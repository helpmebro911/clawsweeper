## Summary

Current `main` still leaves the PDF extraction fallback without `systemPrompt`, while the Codex Responses transport only sends `instructions` when `context.systemPrompt` exists. The PR matches an existing Codex-specific pattern already used for image understanding and remains relevant; it should stay open for maintainer review rather than be auto-closed.
