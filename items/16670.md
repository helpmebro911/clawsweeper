## Summary

Keep #16670 open. Current main still has memory-search config, docs, doctor diagnostics, and `openclaw memory status --deep`, but the onboarding/configure flows still do not include the requested dedicated memory embedding setup step for provider, model, API key/SecretRef, or verification. The April 26 Codex/maintainer comment in the issue already kept this open for the same gap, and the relevant setup/configure/docs files are unchanged since that reviewed SHA.

## Best Possible Solution

Keep #16670 open until OpenClaw adds a dedicated onboarding/configure memory setup path or a maintainer explicitly decides it should remain optional. The likely path is a memory/embeddings configure section or onboarding step that prompts for provider, model, and API key or SecretRef under `agents.defaults.memorySearch`, verifies readiness with an embedding probe or `openclaw memory status --deep`, and updates public docs for API-key, Codex OAuth, and daemon environment behavior.

## Risks / Open Questions

- A fix should account for daemon/LaunchAgent environment visibility and SecretRef-backed credentials, not just the interactive shell environment.
