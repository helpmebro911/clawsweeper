## Summary

Keep this PR open. The hardening request is still valid on current main, but the PR targets stale `src/browser` files while the live noVNC bootstrap route now lives in the bundled browser plugin under `extensions/browser` and still lacks the requested CSP, nonce, nosniff, and frame-deny headers. The maintainer comments in the provided PR context also explicitly support porting rather than closing or merging as-is.

## Best Possible Solution

Port the narrow hardening to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`: generate a per-response script nonce, return CSP plus HTML together so they cannot drift, set `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY`, and keep the existing one-time observer token flow with the password only in the fragment. Do not merge the current branch until it is rebased or rewritten onto the live browser plugin files.
