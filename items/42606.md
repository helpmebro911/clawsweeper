## Summary

Keep this PR open. The security hardening is still relevant on current main, but the PR targets stale `src/browser` files while the active noVNC bootstrap route now lives under the bundled browser plugin in `extensions/browser` and still lacks the requested CSP, nonce, nosniff, and frame-deny coverage.

## Best Possible Solution

Port the narrow hardening to `extensions/browser/src/browser/bridge-server.ts` and `extensions/browser/src/browser/bridge-server.auth.test.ts`: generate a per-response script nonce, return CSP plus HTML together, set `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, and `X-Frame-Options: DENY`, and keep the current one-time observer token flow with the noVNC password only in the URL fragment.
