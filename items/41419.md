## Summary

Keep this PR open. Current main still drops Discord upload MIME types for standard and component sends, and WebP still goes through the shared optimizer to JPEG. The PR addresses a real open bug, but it should remain under maintainer review because its WebP bypass changes shared media behavior and review has raised validation/privacy questions; there are also parallel open PRs for the same bug that should be consolidated rather than auto-closing this useful work.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it with a focused canonical fix for #41256. The best path is to preserve Discord upload MIME types for both standard and component sends, decide the shared WebP/animated-image policy explicitly, add regression coverage for Discord MIME and WebP frame preservation, address metadata and pixel-limit validation before bypassing optimization, and consolidate the parallel open PRs.
