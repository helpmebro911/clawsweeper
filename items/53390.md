## Summary

Keep this issue open. Current main still lacks a viewport-aware text snapshot contract for the browser tool: the public schema and action forwarding have no current-viewport/visible-only option, and all snapshot backends still build AI/ARIA output from page/root accessibility snapshots or Chrome MCP's full structured snapshot rather than filtering by current scroll viewport. The linked #24951 is a separate closed act-timeout report, not a canonical duplicate or shipped fix.

## Best Possible Solution

Keep this issue open. The best fix is to define an explicit viewport-aware text snapshot contract for the browser tool, implement it consistently across Playwright, raw CDP, and Chrome MCP snapshot paths, and add a long-page regression test that scrolls first and verifies the next text snapshot reflects the currently visible viewport.
