## Summary

Keep open. Current main still does not expose or implement a viewport-aware text snapshot for the browser tool; snapshot remains a page/root accessibility-tree style contract across Playwright, raw CDP, and Chrome MCP paths. The prior ClawSweeper comment also kept this open, and the current HEAD is an unrelated plugin test commit, not a browser snapshot fix.

## Best Possible Solution

Keep this issue open. The best fix is to define an explicit viewport-aware text snapshot mode or separate visible-viewport inspection action, implement it consistently across Playwright, raw CDP, and Chrome MCP snapshot paths, and add a long-page regression test that scrolls then verifies the next text snapshot reflects content currently visible after the scroll.
