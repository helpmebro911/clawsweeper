## Summary

Keep open. Current main still does not implement a viewport-scoped browser text snapshot after scrolling. The tool and route pass snapshot options through to full-page/root accessibility snapshot paths, while scroll/evaluate actions can independently change page scroll state. I found documentation for viewport-only screenshots/labels, but not a documented or tested guarantee that text snapshots reflect the current viewport after scroll.

## Best Possible Solution

Keep this issue open. Maintainers should decide the intended contract for `browser snapshot`: if text snapshots should represent the current viewport, implement viewport-aware filtering or a dedicated viewport snapshot mode across the Playwright, CDP, and Chrome MCP snapshot paths, then add a regression test with a long page, `act kind="evaluate"` or End-key scroll, and a follow-up snapshot that must show post-scroll visible content. If full-tree snapshots are intentional, update the browser docs and expose a clear viewport-specific text alternative instead.
