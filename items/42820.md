## Summary

Keep open. Current main still has the reported failure mode for normal `message` sends when a model supplies non-empty or true poll-looking fields: the shared schema still exposes poll creation params, the send path still rejects before dispatch, and Feishu declares media support but no poll support. The shipped mitigation only covers zero/empty defaults. Open PR #46455 remains an unmerged implementation candidate, so the issue should stay open until that work is reviewed, landed, or replaced.

## Best Possible Solution

Keep the issue open and review, land, or replace PR #46455. The durable fix should either sanitize accidental poll creation params before the send-path guard or narrow the LLM-facing schema so poll creation fields are only visible for poll-capable actions/channels, while preserving `action="poll"` behavior and adding regression coverage for Feishu and Telegram send-with-attachment calls that include auto-populated non-empty poll fields.
