## Summary

Keep this issue open. Current main still enforces a 16,000-token hard minimum for resolved model context windows: the guard sets `shouldBlock` below 16k, the embedded runner throws on that block, tests assert the block, custom-provider onboarding clamps sub-16k model metadata up to the hard minimum, and local-model docs still describe the below-16k preflight block.

## Best Possible Solution

Keep the issue open for maintainer product judgment. A complete fix would replace the runtime hard block with warning-only behavior or an explicit supported override, then update `src/agents/context-window-guard.ts`, `src/agents/pi-embedded-runner/run/setup.ts`, custom-provider onboarding normalization, the guard/onboarding tests, and `docs/gateway/local-models.md` so sub-16k experimentation is consistently allowed while still warning users about limitations.
