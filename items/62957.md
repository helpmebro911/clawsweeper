## Summary

Keep open. Current main still enforces the requested 16k context-window minimum as a hard runtime block, and the tests/docs continue to encode that behavior. The prior automated review reached the same result on an earlier main SHA; current main still has the same guard, throw path, onboarding clamp, and local-model documentation.

## Best Possible Solution

Keep this issue open for maintainer product judgment. A complete fix should change the context-window guard from a hard sub-16k block to warning-only behavior or an explicit override, update the embedded runtime path, decide whether custom-provider onboarding should preserve smaller configured windows, and revise the guard/onboarding tests plus local-model docs so the behavior is consistent.
