## Summary

Keep #71848 open. The issue is fresh and specific, and current main still persists TMPDIR into the macOS LaunchAgent environment, including session-shaped `/var/folders/...` values matching one suspected failure mode. Existing launchd hardening and related closed items cover KeepAlive/restart races, but not this SIGABRT 134 plus stale TMPDIR/TCC/memory-pressure incident.

## Best Possible Solution

Keep the issue open and investigate the macOS LaunchAgent install/runtime path. The most concrete repo change to evaluate is replacing persisted session TMPDIR with a stable temp directory or omitting TMPDIR from LaunchAgent plists, then adding doctor/service-audit detection for stale `/var/folders` TMPDIR values and macOS gateway docs for launchd TCC/Full Disk Access and supported runtime expectations.
