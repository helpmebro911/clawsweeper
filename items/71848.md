## Summary

Keep #71848 open. Current main has macOS launchd hardening and GUI-session bootstrap guidance, but it does not resolve the reported stale LaunchAgent TMPDIR/TCC/SIGABRT scenario. The generated service environment still persists TMPDIR into the LaunchAgent, and tests explicitly assert that behavior, while docs only give generic macOS permission guidance.

## Best Possible Solution

Keep this issue open and narrow it into a macOS LaunchAgent hardening task: make gateway service TMPDIR durable across login sessions while preserving SQLite temp/journal reliability, add doctor/service-audit detection and repair for stale /var/folders LaunchAgent TMPDIR values, document Full Disk Access/TCC requirements for the LaunchAgent runtime process, and improve launchd status/diagnostics for SIGABRT 134 and post-bootstrap crashes.

## Evidence

- No live macOS launchd reproduction was run in this read-only review, so the exact SIGABRT root cause still needs maintainer or reporter validation on an affected host.
