## Summary

Keep this PR open. The checked-out main still implements memory indexing with the fixed-size `chunkMarkdown(content, { tokens, overlap })` path only, and the proposed multi-strategy chunking surface is neither shipped nor clearly superseded. This PR is also not a ClawHub cleanup case because it changes bundled memory-core indexing, config/schema, SDK exports, and reindex metadata. Related open memory chunking items overlap, but none is a merged or clearly canonical replacement for this PR.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should decide the product/API direction for bundled memory-core chunking, coordinate it with related open memory chunking work (#44395, #48795, #48808, #53650), require the bot review conversations to be resolved or re-reviewed, and validate the final shape with focused tests for default fixed-size compatibility, strategy-specific chunking, schema/help drift, and full-reindex metadata changes.
