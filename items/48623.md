## Summary

Keep open. Current main still intentionally ignores `lastGood` for automatic auth-profile ordering, and failover rotation walks the fixed candidate list built from that ordering. The slug-generator mitigation discussed in PR #65492 is not present on main as a dedicated slug task model/timeout override; the current helper still uses the agent default model and only the general agent timeout. The shared-timeout portion appears at least partly mitigated now because each embedded attempt receives `params.timeoutMs` and schedules its own abort timer, but unresolved core behavior remains, so this is not conservative cleanup fodder.
