## Summary

Current main still matches the reported Control UI streaming layout bug. Tool-boundary text is preserved as stream segments, `buildChatItems` emits those segments and the live tail as separate `kind: "stream"` items, and the chat view renders every stream item as its own outer assistant `.chat-group`. The linked fix PR #63986 is closed unmerged, and the current tests do not cover segmented streaming producing exactly one assistant group.

## Best Possible Solution

Keep this issue open and fix the Control UI rendering path. The best implementation is to represent one active assistant run as one outer assistant group, interleaving committed stream segments, visible tool cards, and the live tail inside that group; when tool calls are hidden, merge segment text and live text into a single updating bubble. Add focused coverage in `ui/src/ui/chat/build-chat-items.test.ts` or an unmocked chat rendering test asserting segmented live streaming produces exactly one outer `.chat-group.assistant`. PR #63986 and the related duplicate-bubble PR context are useful references, but current main has not shipped the fix.
