## Summary

Keep #71569 open. Current main partially addresses the visual-spam part by using a single Mattermost draft preview post and finalizing eligible replies in place, but the central config bug remains: `channels.mattermost.streaming` is documented while the Mattermost schema/type/metadata still omit it, and the inbound Mattermost monitor starts draft preview streaming unconditionally instead of honoring `off | partial | block | progress`. The provided related issue #71565 is superseded by this broader report, and the prior ClawSweeper context for #70253 also supports that Mattermost streaming config remains unresolved.

## Best Possible Solution

Keep the issue open and fix it in the Mattermost plugin: add `streaming` to Mattermost config schema, generated metadata, and TypeScript types; resolve `resolveChannelPreviewStreamMode(account.config, ...)` in the inbound monitor; honor `off`, `partial`, `block`, `progress`, and `streaming.preview.toolProgress`; preserve the existing in-place finalization tests; and add regression tests proving `channels.mattermost.streaming` is accepted and affects delivery mode. A separate product choice is still needed for Mattermost final notifications or silent previews because edits alone may not notify users.
