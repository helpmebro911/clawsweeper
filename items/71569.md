## Summary

Keep open. Current main still has the central Mattermost preview-streaming contract mismatch: docs advertise `channels.mattermost.streaming` modes, but the Mattermost config schema, public type, generated metadata, and normal inbound runtime do not implement mode-aware `off | partial | block | progress` behavior. Current main does include same-post draft finalization logic, so part of the visual-spam complaint may be improved, but the documented config key and notification-oriented `progress`/`off` behavior remain missing.

## Best Possible Solution

Implement the documented Mattermost preview-streaming contract in the bundled Mattermost plugin: add `streaming` schema/types/generated metadata and doctor/compat handling for root and per-account configs; resolve `off | partial | block | progress` in the Mattermost runtime; skip draft preview creation for `off`; make `progress` deliver the final answer as a normal notifying post; and add focused config/runtime tests. Keep this issue as the focused tracker and reference #70253 only for the overlapping ignored-config symptom.
