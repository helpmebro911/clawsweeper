## Summary

Keep this PR open. Current main still has no EasyRunner deployment guide, and provider-specific hosting docs are an established OpenClaw docs surface, so this is not implemented, not a duplicate, and not clearly ClawHub-only. The PR is docs-only from a repository supply-chain perspective, but the current content still needs contributor revision and maintainer docs/security review because it targets stale Platforms navigation and normalizes unsafe Control UI reverse-proxy guidance.

## Best Possible Solution

Keep this PR open and ask for a revised, rebased docs contribution. The best path is to place or link EasyRunner under the current Install > Hosting docs surface, add the install index/navigation entries consistently, remove normal-use dangerouslyDisableDeviceAuth guidance, configure a safe auth model with explicit gateway.controlUi.allowedOrigins for the EasyRunner HTTPS domain and tight trusted proxy guidance, keep secrets in env/secret storage, rebuild the local image before redeploy/update, and verify the EasyRunner/Caddy/Podman commands against a real deployment before maintainer docs/security review.
