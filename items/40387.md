## Risks / Open Questions

- The proposed guide normalizes gateway.controlUi.dangerouslyDisableDeviceAuth for reverse-proxy access, which conflicts with current OpenClaw security guidance and could weaken user deployments.
- The update flow pulls source and redeploys without rebuilding localhost/openclaw:latest, so users may keep running the old image.
- The compose example omits a non-loopback gateway bind setting for container proxy access, which current docs indicate can make bridge-networked deployments unreachable.
- The troubleshooting command `podman exec ... env | grep TOKEN` may print secrets; the guide should use a redacted or safer verification path.
- The EasyRunner-specific Podman/Caddy/service commands still need factual maintainer or contributor verification.
