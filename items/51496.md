## Summary

Keep open. Current `main` does not implement the requested fix for normal text-mode CLI subcommands like `openclaw message send --dry-run`. The existing stderr-routing logic is gated on `--json`, while `message` still loads plugins directly and prints text results via stdout, so `[plugins]` logs can still contaminate captured stdout.

## Evidence

- **`message` still loads plugins directly in the command action:** `runMessageAction()` calls `ensurePluginRegistryLoaded()` unconditionally before executing the message command, with no stderr-routing flag or non-TTY handling.
  - file: [src/cli/program/message/helpers.ts:53](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/cli/program/message/helpers.ts#L53)
  - command: `sed -n '1,140p' src/cli/program/message/helpers.ts`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **CLI startup only suppresses stdout for JSON mode:** `resolveCliStartupPolicy()` sets `suppressDoctorStdout` from `jsonOutputMode` only. There is no `process.stdout.isTTY` or generic piped-stdout check here, so non-JSON commands do not opt into stderr routing.
  - file: [src/cli/command-startup-policy.ts:47](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/cli/command-startup-policy.ts#L47)
  - command: `sed -n '1,240p' src/cli/command-startup-policy.ts`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Subsystem logger still sends info logs to stdout by default:** `writeConsoleLine()` writes to `sink.log` unless `loggingState.forceConsoleToStderr` is set or the level is error/fatal. Plugin bootstrap logs are info/warn-style diagnostics, so they remain on stdout unless that flag is enabled.
  - file: [src/logging/subsystem.ts:259](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/logging/subsystem.ts#L259)
  - command: `sed -n '220,330p' src/logging/subsystem.ts`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Text-mode `message` output also uses stdout:** `messageCommand()` sends JSON through `writeRuntimeJson()` only when `opts.json === true`; otherwise it emits formatted lines with `runtime.log()`. `defaultRuntime.log()` calls `console.log()`, so command output shares stdout with plugin diagnostics in text/dry-run mode.
  - file: [src/commands/message.ts:82](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/commands/message.ts#L82)
  - command: `sed -n '1,160p' src/commands/message.ts`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Tests codify that non-JSON plugin loading does not route to stderr:** The preaction coverage explicitly expects stderr routing in `--json` mode and explicitly expects no stderr routing without `--json`, which matches the current unfixed behavior for text-mode commands.
  - file: [src/cli/program/preaction.test.ts:476](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/cli/program/preaction.test.ts#L476)
