## Summary

Current `main` does not implement this PR’s behavior. `openclaw cron runs` still requires `--id`, there is no positional `<id>` alias or conflict check in the command handler, the added regression tests are absent, and docs still present `--id` as the supported form. This PR should stay open unless it is superseded by a different implementation.

- **CLI handler still requires `--id` on main:** The `runs` subcommand is defined with `.requiredOption("--id <id>", "Job id")` and an `.action(async (opts) => ...)` handler that reads `opts.id`; there is no positional argument, no alias handling, and no conflicting-id validation.
