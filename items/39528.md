## Summary

Keep open. Current main has relevant hardening for owner-only gateway tools, protected config paths, and prompt guidance, but it does not fully close the reported behavior: the model-facing gateway restart action still schedules a restart when `commands.restart` is enabled, and config writes are guarded by path/dangerous-flag checks rather than a hard proof of explicit user consent. The context-loss portion also lacks enough source-backed reproduction or fix evidence to close as implemented or cannot-reproduce.
