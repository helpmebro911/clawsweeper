## Summary

Current main still does not implement the requested fish-shell convention. `openclaw completion --install`, doctor, update, setup, docs, and helper tests remain profile-based for fish (`~/.config/fish/config.fish`) rather than installing/detecting `~/.config/fish/completions/<binName>.fish`. The related PR #27135 was directly relevant prior art but is closed and unmerged, so #26428 still tracks real remaining work.

## Best Possible Solution

Keep #26428 open. The best fix is a fish-specific path in `src/cli/completion-runtime.ts` that writes/copies the cached fish completion to `~/.config/fish/completions/<binName>.fish`, detects that installed state there, safely removes legacy OpenClaw source blocks from `config.fish`, updates doctor/setup/update reload messaging plus `https://docs.openclaw.ai/cli/completion`, and adds focused regression tests. PR #27135 is the relevant prior-art implementation, but it needs maintainer review/rebase because it closed unmerged.
