## Summary

Keep #20460 open. Current main still uses adjacent numbered config backups (`openclaw.json.bak`, `.bak.1` through `.bak.4`) and has no implemented `config-backup/` directory or datetime-named automatic config backups. The directly related PR #20877 was closed unmerged, and the later datetime PRs are open implementation candidates rather than shipped behavior.

## Best Possible Solution

Keep #20460 open as a core config enhancement. A complete implementation should update `src/config/backup-rotation.ts` and all config write, recovery, audit, doctor, docs, and tests that currently assume adjacent `.bak` files; it should define migration/compatibility behavior for existing numbered backups and coordinate with #20877, #39923, #39982, and #40392 so one canonical change lands.
