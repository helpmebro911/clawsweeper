## Summary

Keep #20460 open. Current main still writes adjacent numbered config backups (`openclaw.json.bak`, `.bak.1` through `.bak.4`) and does not create a `config-backup/` directory or datetime-suffixed config backup names. The linked PR #20877 is relevant prior art, but it was closed unmerged; later open PRs around datetime suffixes are implementation candidates, not shipped behavior.

## Best Possible Solution

Keep #20460 open as a core config enhancement. A complete fix should implement a dedicated config backup directory and datetime naming in `src/config/backup-rotation.ts`, thread the new path through `writeConfigFile`, include mutation writes, recovery/audit paths, permission hardening, pruning, and docs, and define compatibility for existing adjacent numbered `.bak` backups. Use #20877 as prior art, but resolve its retention issue and coordinate with the open datetime implementation PRs before landing one canonical change.
