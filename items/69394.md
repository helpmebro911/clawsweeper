## Summary

Keep open. Current main still strips a trailing `NO_REPLY` from auto-reply text and delivers the preceding content, and the current tests explicitly assert that behavior. The PR targets a real remaining auto-reply leak, but its proposed `stripped.length < text.length` guard needs revision because `stripSilentToken()` trims whitespace even when no trailing token was removed. Related #66755 overlaps but is intentionally narrower, so it does not clearly supersede this broader PR without maintainer policy judgment.

## Best Possible Solution

Keep this PR open but require changes before merge: replace the trim-length heuristic with explicit trailing-token detection, add regression coverage for non-trailing `NO_REPLY` mentions with trailing whitespace, and coordinate with #66755/#66701 so maintainers choose one canonical policy for reasoning-plus-`NO_REPLY` suppression across auto-reply surfaces.
