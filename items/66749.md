## Summary

Keep open. Current `main` still calls `requestHeartbeatNow()` for subagent-scoped exec events in both reported paths, while heartbeat resolution explicitly remaps subagent session keys back to the main session. That matches the PR’s stated bug, and `main` still lacks the proposed subagent-specific regression test, so there is no strong basis to close this PR as already implemented or obsolete.

## Evidence

- **Background exec exit still wakes heartbeat unconditionally:** `maybeNotifyOnExit()` enqueues the system event, then always calls `requestHeartbeatNow(scopedHeartbeatWakeOptions(sessionKey, { reason: "exec-event", coalesceMs: 0 }))` with no `isSubagentSessionKey()` guard on current `main`. That is the exact callsite the PR proposes to change.
