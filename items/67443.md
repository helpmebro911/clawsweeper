## Summary

Keep open. Current main has not absorbed the PR’s central status fixes: `session_status` still hard-codes group activation fallback to `mention`, and the status summary/runtime context-token resolver is still duplicated rather than shared with the agent resolver. The PR also is not safe to close as implemented or land as-is because the provided review discussion flags two P1 regressions in the PR head.

## Best Possible Solution

Keep the PR open, request changes, and narrow it to the real status/session fixes: share or align context-token resolution without regressing `getSessionDefaults`, preserve valid default-agent aliases, add the `requireMention: false` session_status regression test, and either remove or explicitly document/approve the `openclaw/<provider>/<model>` body-model contract change.
