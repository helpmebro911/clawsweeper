## Summary

Current `main` does not implement this opt-in. Cron webhook delivery still goes through `fetchWithSsrFGuard` without any private-network override, the cron config surface does not expose `webhookAllowPrivateNetwork`, and the existing test suite still locks in the blocked-by-default behavior with no opt-in path. The PR discussion also shows this is a coherent, scoped change: the author responded to review feedback with schema/docs/test follow-ups, so this does not meet any conservative close reason.
