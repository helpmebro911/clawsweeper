## Summary

Keep this PR open. Current main still contains the `scripts/ui.js` Windows `.cmd`/`.bat` shell path that passes a separate args array to `spawn()` and `spawnSync()`, which is the Node v24 DEP0190 pattern this PR targets. The PR is plausible, but its current args-string join has quoting risk and no regression coverage, so the best maintainer path is revision and tests rather than cleanup closure.

## Best Possible Solution

Keep the PR open and ask for a current-main revision that routes `scripts/ui.js` through the existing pnpm runner or an equivalent tested Windows spawn resolver. The fix should avoid `shell: true` plus a separate args array and add regression coverage for `.cmd` launchers, safe forwarded args, paths containing spaces, and args containing spaces.
