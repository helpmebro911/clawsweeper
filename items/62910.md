## Summary

Keep this PR open. Current main still has the Node v24 DEP0190 spawn shape in `scripts/ui.js`, so the bug is not obsolete, but the submitted patch needs revision because it folds Windows shell args with a plain `args.join(" ")`, adds no regression coverage, and could regress argument/path boundary handling already covered elsewhere.

## Best Possible Solution

Keep this PR open and request a revision that routes `scripts/ui.js` through the existing pnpm runner or the same `buildCmdExeCommandLine()`-based `cmd.exe` wrapper. The fix should avoid `shell: true` plus a separate args array, preserve command paths and forwarded args containing spaces, keep unsafe metacharacter rejection, and add regression coverage for `.cmd` launchers, `Program Files` paths, args with spaces, and the final spawn/spawnSync invocation shape.

## Risks / Open Questions

- Merging the PR as-is could fix the DEP0190 warning while regressing forwarded UI args that contain spaces on Windows.
- A naive rebase could regress the current `C:\Program Files\...\pnpm.cmd` quoting behavior for UI install/build/test commands.
- The current tests would not catch a future reintroduction of `spawn(file, args, { shell: true })` for Windows `.cmd` launchers.
