## Summary

Keep this PR open. Current main still has the Node v24 DEP0190-triggering shape in scripts/ui.js for Windows .cmd/.bat/.com UI runner launches, so the PR is not obsolete. The submitted patch is not merge-ready because it changes Windows shell command construction with an unquoted command path plus args.join(" "), adds no regression coverage, and bypasses the existing path-quoting helper.

## Best Possible Solution

Keep this PR open and request a revision that reuses scripts/pnpm-runner.mjs or the same buildCmdExeCommandLine()-based cmd.exe wrapper for scripts/ui.js. The final implementation should avoid shell:true with a nonempty args array, preserve Program Files launcher paths and forwarded args containing spaces, keep unsafe metacharacter rejection, and add regression tests asserting the final Windows spawn/spawnSync invocation shape.

## Evidence

Node documents DEP0190 as a runtime deprecation since v24.0.0 when args are passed with shell:true, because values are only space-separated and not escaped. ([nodejs.org](https://nodejs.org/api/deprecations.html))

A read-only spawnSync('echo', ['hello'], { shell: true }) probe emitted DEP0190, while spawnSync('echo hello', [], { shell: true }) did not. Node v24.0.0 source emits DEP0190 only when options.shell is set and args.length > 0, then builds the shell command by joining file and args with spaces. ## Risks / Open Questions

- Current main can still emit DEP0190 on Node v24 for Windows .cmd/.bat/.com UI runner launches until a revised fix lands.
- Merging the PR as written could break pnpm.cmd paths under Program Files because the new command string no longer quotes cmd before shell execution.
- Merging the PR as written would leave forwarded UI args containing spaces dependent on shell splitting, with no regression test for the final spawn/spawnSync shape.
- The change is in a shell command construction path, so preserving the existing unsafe metacharacter rejection is necessary but not sufficient; quoting and argument-boundary behavior also need coverage.
