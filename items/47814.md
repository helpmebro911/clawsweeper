## Summary

Keep this PR open. Current main still does not implement the named-DM session infrastructure proposed here, and the predecessor #47403 was closed unmerged as superseded by this PR rather than shipped. The PR is also not obsolete: it is a focused core gateway/session-store change, but it still needs maintainer review and rework for the parser, persisted-store, and chat-type/send-policy edge cases raised in review.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. If named-DM /resume routing is still desired, rebase against current main and make the implementation fail closed: add strict named-DM key parsing, validate or ignore malformed persisted activeNamedSession values, treat dm-named keys as direct before group/channel token matches in both chat-type and send-policy paths, and keep docs scoped to contributor-facing primitives until routing ships.
