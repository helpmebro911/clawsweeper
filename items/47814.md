## Risks / Open Questions

- The PR's named-DM parser design relies on parseAgentSessionKey, whose current implementation removes empty segments; a strict named-DM parser should reject malformed keys such as agent:main:dm-named:123::work instead of collapsing them into valid-looking input.
- A persisted activeNamedSession containing ':' could make getActiveNamedSessionKey throw unless the implementation validates or ignores malformed stored values. Current normalizeSessionStore only normalizes delivery/model fields.
- Chat-type and send-policy fallback logic currently checks group/channel tokens before direct/dm. A named session called channel or group could be misclassified unless dm-named gets explicit priority in both paths.
- The PR is dirty against current main and includes broad CHANGELOG synchronization unrelated to the focused feature, so it likely needs a narrow rebase before maintainer review.
