## Summary

Keep this PR open. Current main still has the no-cfg early return that skips the default on-disk session store, and the PR is a narrow, recent implementation candidate with targeted regression coverage and no visible security-sensitive scope creep.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing. The best path is to preserve the narrow subagent-depth fix that lets omitted cfg fall through to the default on-disk session store, keep the explicit acct-suffixed requester regression tests, and optionally clean up the temp directories created by the new tests before merge.
