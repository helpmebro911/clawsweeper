## Summary

Keep open. Current main still has the no-cfg early return in subagent depth resolution, so persisted explicit requester sessions still cannot reach the default on-disk session store through this path. The PR’s central bug fix remains relevant, though the branch should be cleaned/rebased before merge because it carries unrelated drift fixes.

## Best Possible Solution

Keep the PR open, but ask for a scoped/rebased patch that preserves the subagent-depth fix and regression tests for default no-cfg store lookup, explicit acct-suffixed requester keys, missing-store fallback, and announce internal/external classification. Drop or split the unrelated drift fixes before landing.
