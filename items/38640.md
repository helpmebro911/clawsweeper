## Summary

Keep PR #38640 open. Current main still calls `activeSession.prompt(...)` directly and does not ship the PR's prompt-level `withLlmRetry`/exponential-backoff wrapper. Main has adjacent SDK retry-wait caps, auth-profile rotation, cooldowns, and model fallback, but those are different from bounded retry of the same prompt call. Related configurable retry work (#49376/#61443) remains open rather than landed or explicitly superseding this PR.

## Best Possible Solution

Keep #38640 open for maintainer judgment. The best path is to either fold it into the configurable LLM/provider retry direction tracked by #49376/#61443, or rework this PR at the chosen runner/provider boundary with abortable backoff, shared failover-status parsing, clear interaction with SDK retries and model fallback, and focused tests for retry parsing plus cancellation behavior.
