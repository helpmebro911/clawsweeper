## Summary

Keep this PR open. Current main still does not implement automatic low default temperatures for tool-enabled or shell-like agent runs, so the PR is not redundant. However, the PR should not be merged as-is because the supplied diff and review history show unresolved agent-runner edge cases plus unrelated secrets/config and lockfile changes mixed into the branch.

## Best Possible Solution

Keep this PR open, but require a narrow rework before maintainer review: rebase on current main, drop unrelated secrets/config, lockfile, and changelog changes, implement the heuristic through the agent extra-params/runtime-plan path for both normal attempts and compaction, preserve explicit user temperatures, avoid provider thinking/cache-retention regressions, and add focused tests for plain chat, generic tools, shell-like tools, explicit temperature, thinking-enabled cases, and compaction.
