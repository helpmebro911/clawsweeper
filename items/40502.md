## Summary

Keep this PR open. Current main still does not implement dynamic low default temperatures for tool-enabled or shell-like embedded agent runs, and the PR remains a plausible implementation candidate. It needs a narrow rebase/split before merge because the provided diff also includes unrelated security-sensitive secrets/config and lockfile/package-surface changes.

## Best Possible Solution

Keep this PR open, but require a focused rebase before maintainer review: drop already-shipped or unrelated secrets/config/lockfile/test-loosening hunks, implement the dynamic temperature heuristic through the embedded runner extra-params/runtime-plan path for normal attempts and compaction, preserve explicit configured temperatures, respect thinking/reasoning provider constraints, and add focused tests for plain chat, generic tools, shell-like tools, explicit temperature, thinking-enabled cases, and compaction.
