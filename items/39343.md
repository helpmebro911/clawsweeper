## Summary

Keep #39343 open. Current main still only satisfies the Telegram album portion: Telegram buffers `media_group_id` updates into one agent turn, but LINE still processes each image/media webhook event independently, and the documented/configured inbound debounce contract remains text-only with media flushing immediately.

## Best Possible Solution

Keep the issue open and implement the missing bundled-channel behavior: preserve Telegram `media_group_id` album buffering, add LINE same-sender/chat/time-window media batching with clear timer and text-flush semantics, expose/document either a media-specific delay or an explicit extension of `messages.inbound`, and add LINE plus Telegram regression coverage for batching, independent keys, replay-cache timing, and reply-token behavior.
