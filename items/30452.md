## Summary

Keep #30452 open. Current main implements the top-level compaction model override, but the requested `memoryFlush.model` and `memoryFlush.resetAfterFlush` controls are still absent from config types, strict schema, generated/help labels, docs, the memory flush plan, and the runtime flush path.

## Best Possible Solution

Keep the issue open or split it into narrower follow-ups. The product path is to add `agents.defaults.compaction.memoryFlush.model` and `memoryFlush.resetAfterFlush` to config types, zod schema, generated schema/help/labels, docs, and relevant UI surfaces; route the silent memory-flush turn through the configured model override; then implement reset-after-flush by reusing existing session reset machinery only after a successful flush, with tests for session store updates, queue refresh, transcript handling, active-run cleanup, subagent/ACP state, and expected channel behavior.
