## Summary

Keep open. Current `main` has generic plugin conversation hooks (`llm_input`/`llm_output`) that a third-party plugin could use, but the shipped OpenTelemetry path still exports only OpenClaw-specific aggregate telemetry (`openclaw.*`) rather than GenAI semantic spans or OpenLLMetry/IITM compatibility. That means the request is not implemented on `main`, and the issue remains a valid enhancement request rather than cleanup fodder.

## Evidence

- **Bundled OTel exporter still emits only OpenClaw-specific metrics and spans:** `createDiagnosticsOtelService()` defines counters/histograms such as `openclaw.tokens`, `openclaw.cost.usd`, `openclaw.run.duration_ms`, and emits a single model trace span named `openclaw.model.usage` with `openclaw.tokens.*` attributes. There is no `gen_ai.*` attribute mapping, no provider span names like `anthropic.chat`, and no OpenLLMetry/IITM integration path in this service.
