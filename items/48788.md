## Summary

Keep issue #48788 open. Current main does not yet contain the requested centralized filename decoding utility: Feishu and generic media fetch still have separate private Content-Disposition parsers, and there is no shared src/media/filename.ts helper on main. PR #71517 is directly related and useful, but it is still open and explicitly describes itself as a partial implementation that does not migrate every channel path yet, so it is not strong enough to supersede the broader issue today.
