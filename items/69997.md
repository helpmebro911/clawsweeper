## Summary

Keep open. Current main still has the unsafe `normalizeStoredOverrideModel` path and session-store loading does not validate corrupted `providerOverride`/`modelOverride` values before returning entries. The PR is not merge-ready as-is because it also bundles unrelated heartbeat behavior changes, but the model-override defensive guard is still useful work rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open, but ask the contributor to split or drop the heartbeat changes, add regression coverage for corrupted session-store override fields, and consider normalizing `providerOverride`/`modelOverride` at the session-store boundary or through a shared `normalizeOptionalString`-style helper so all persisted-session consumers get the same protection.
