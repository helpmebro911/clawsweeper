## Summary

Keep open. Current `main` does not implement `requireMentionInThread` for Feishu: the config schema exposes only `requireMention`, the reply-policy logic has no thread-specific override path, the Feishu bot does not pass thread context into mention gating, and docs/tests likewise only cover the existing group-level mention behavior.

## Evidence

- **Config schema lacks the new option:** `FeishuGroupSchema` and `FeishuSharedConfigShape` include `requireMention` and `replyInThread`, but no `requireMentionInThread` field on current `main`.
  - file: [extensions/feishu/src/config-schema.ts:143](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/extensions/feishu/src/config-schema.ts#L143)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '138,186p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Reply policy has no thread override:** `resolveFeishuReplyPolicy` only resolves from per-group `requireMention`, top-level `requireMention`, and `groupPolicy`; it accepts no `isThreadReply` input and contains no thread-specific branch.
  - file: [extensions/feishu/src/policy.ts:180](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/extensions/feishu/src/policy.ts#L180)
