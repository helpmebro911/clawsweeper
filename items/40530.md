## Summary

Keep this PR open. Current main still lacks a Feishu thread-specific mention override, while the PR is a plausible but not yet landing-ready implementation candidate. The latest PR shape also needs maintainer review because its `groupSession.replyInThread` predicate appears too broad against current main semantics.

## Best Possible Solution

Keep this PR open for Feishu maintainer review. The best path is to reconcile it with #48945, then land one rebased implementation that adds schema, generated config metadata, docs, and tests for topic groups, `replyInThread: "enabled"` groups, bidirectional overrides, ordinary quote replies, top-level mention gating, and ideally live Feishu validation.
