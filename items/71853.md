## Summary

Keep open. The issue is recent, has a corroborating user report, and current main still preserves the reported google-vertex ADC marker path: ADC resolves to the synthetic `gcp-vertex-credentials` string, that value can be stored/forwarded as runtime auth, and the Google transport turns any non-JSON `apiKey` into `x-goog-api-key`. The Node 25 fetch concern is also not safely obsolete because the locked gaxios package still dynamically imports `node-fetch` instead of using `globalThis.fetch`.

## Best Possible Solution

Keep this issue open and fix it in the bundled Google provider/runtime auth path. The likely direction is google-vertex-specific ADC runtime auth preparation that strips `gcp-vertex-credentials` before any transport call, configures Vertex mode with project/location or a real bearer/OAuth credential instead of `apiKey`, and adds regression coverage for ADC marker handling through the run and compaction paths. Separately, update or patch the `@google/genai`/gaxios path, or avoid the affected SDK fetch path, so Node 25+ does not hit the dynamic `node-fetch` crash.
