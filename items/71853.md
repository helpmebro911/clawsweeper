## Summary

Keep open. Current main still has a plausible google-vertex ADC credential leak path: ADC resolution returns the synthetic `gcp-vertex-credentials` marker as the runtime apiKey, the Google transport injects resolved apiKey values into provider-owned requests, and the Google auth helper serializes any non-JSON string as `x-goog-api-key`. The Node 25/gaxios portion may need separate dependency verification, but the ADC bug is not implemented away on main.

## Best Possible Solution

Keep this issue open and fix the google-vertex auth path in the Google provider boundary: ADC should not be forwarded as an API key header, and Vertex requests should use a dedicated Vertex-aware path that passes `vertexai: true` plus project/location or otherwise sends proper OAuth/ADC auth. Add regression coverage for ADC marker stripping and, separately, verify whether the Node 25/gaxios failure is fixed by the current `@google/genai`/`gaxios` versions or needs an upstream/package override.
