## Summary

Keep #38633 open. Current main still shows `pidsLimit: 256` as a sandbox Docker config example while the resolver leaves `pidsLimit` unset unless configured, and Docker args only include `--pids-limit` for a positive configured value. The related PRs that would have fixed this are closed unmerged, and the broader #38604 security issue does not replace this narrower docs mismatch.

## Best Possible Solution

Keep #38633 open until maintainers either clarify the docs that `pidsLimit: 256` is an explicit example and remains unset unless configured, or intentionally add a runtime default in `resolveSandboxDockerConfig` with matching tests, schema/type wording, and docs. Coordinate any runtime default with the broader #38604 security/default discussion; a docs-only fix can remain narrow.
