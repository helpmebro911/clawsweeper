## Summary

Keep open. Current main has a real Control UI i18n framework and generated locale bundles, but the issue's central request is still valid: several user-visible Web UI surfaces continue to render literal English outside the translation map.

## Best Possible Solution

Keep the issue open as Control UI localization debt. The best path is to migrate remaining user-visible Web UI literals into `ui/src/i18n/locales/en.ts`, replace call sites with `t(...)`, regenerate non-English bundles with the existing Control UI i18n workflow, and add or extend checks/tests so new raw English UI copy is harder to introduce. Start with Agents Overview, Quick Settings, command palette/slash commands, config form metadata, and chat action labels.
