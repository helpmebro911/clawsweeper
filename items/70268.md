## Summary

Keep #70268 open. Current main still has the wake-hook routing gap the PR targets: direct `/hooks/wake` and mapped `action: "wake"` paths carry only text/mode into `dispatchWakeHook`, and the gateway dispatcher still resolves the main session unconditionally. The PR is not obsolete or superseded by the provided related work; #35231 is closed unmerged, while #64556 remains the linked open bug this PR is meant to fix.

## Best Possible Solution

Keep #70268 open as the active focused fix for wake-hook session routing. Maintainers should review the latest PR head against the remaining global-scope/session-policy comments, require the focused gateway tests described in the PR, and then land it or ask for targeted changes so #64556 can close from the merged implementation.
