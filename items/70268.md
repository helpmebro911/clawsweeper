## Summary

Keep PR #70268 open. Current main still routes direct and mapped wake hooks through the default main session path and does not carry `sessionKey`/`agentId` through wake actions, while the PR is a focused attempt to fix the open bug #64556. The older related PR #35231 is closed unmerged, so it does not supersede this work.

## Best Possible Solution

Keep #70268 open and continue it as the focused fix for wake-hook session routing. The best implementation path is to add direct `/hooks/wake` `sessionKey` normalization and policy checks, carry `agentId`/`sessionKey` metadata through mapped wake actions, canonicalize and prefix-check the final dispatch target including agentId-only mapped wake, update the wake-hook docs, and retain the new regression coverage before landing.
