## Summary

Keep open. Current main still has the Linux sudo/systemd mismatch this PR is trying to address: unit files are written under the resolved HOME path while systemctl is deliberately aimed at SUDO_USER's user scope under sudo. The PR is a plausible fix candidate, not obsolete, but it should not land as-is because the current head is stale/narrow, has unresolved review feedback, and includes unrelated build/admin-side-effect changes.

## Best Possible Solution

Keep this PR open as a real bug-fix candidate, but require a rebase onto current main and a narrower implementation. The best fix is to reject sudo-with-SUDO_USER at a shared Linux systemd install boundary reached by all gateway install entrypoints, return a clear hint to run openclaw gateway install without sudo and run sudo loginctl enable-linger separately, keep the existing systemctl --machine fallback for management actions, remove the unrelated tsdown sourcemap change, and update focused tests for the guard plus affected daemon-cli install mocks.
