## Summary

Keep open. The PR is stale as written because current main no longer has the old manual Feishu permission JSON block, no longer keeps localized docs in this repo, and renamed the onboarding surface. But the underlying manual-permission docs gap is not fully resolved: the fallback setup help still hardcodes old broad scope names while the current docs tell users to grant required scopes without listing the current Lark tenant-scope set.

## Best Possible Solution

Keep this open for rework against current main: update only the English Feishu docs and `extensions/feishu/src/setup-surface.ts` fallback help with current official Lark tenant-scope guidance or a precise official-doc link, remove the `docs/zh-CN` changes, preserve the verified contact-scope alias note if needed, and explicitly handle the `requireMention: false` group-message scope caveat.
