## Summary

Keep this PR open. Current main still deletes Telegram commands before setting a changed non-empty menu, while the PR is a narrow follow-up that moves deletion to the empty-menu path, adds regression coverage, and does not touch security-sensitive supply-chain surfaces.

## Best Possible Solution

Keep this PR open for maintainer review and validation. The best path is to land the narrow Telegram plugin change if targeted tests and required CI pass: call setMyCommands directly for changed non-empty menus, preserve deleteMyCommands for explicit empty-menu clearing, and keep the new regression coverage. If maintainers want extra hardening, add a focused fallback or retry policy for setMyCommands failures rather than closing this follow-up as already handled by #32059.
