## Summary

Keep this PR open. Current main still uses a shared `deviceId+clientIp` control-plane write budget whenever either identity field is present, so the per-connection isolation proposed here is not implemented on main. The PR is focused and useful, but it needs maintainer/product review plus rebase work because current main now has bucket pruning and a hard cap that the stale branch must preserve.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to ask the contributor to rebase onto current main, preserve stale-bucket pruning and the 10,000-entry cap, rename/update the stale keying test, keep a focused regression test for two concurrent sessions from the same device/IP, and update docs if maintainers choose per-connection control-plane write budgets as the intended product behavior.
