## Summary

Keep open. Current main still does not persist ACPX child-process stderr to disk. The bundled OpenClaw plugin delegates subprocess handling to `acpx` 0.6.1, and the exact npm runtime only keeps the bounded startup stderr buffer plus verbose stderr mirroring. This is bundled ACPX runtime behavior, not a ClawHub-only optional capability.

## Best Possible Solution

Keep the issue open. Implement durable ACP subprocess stderr teeing in `openclaw/acpx` with a small runtime/config contract, then update OpenClaw's `extensions/acpx` dependency pin, config schema, docs, and focused tests. Do not close this as ClawHub: it concerns the bundled ACPX runtime's subprocess diagnostics and the current plugin API/config surface does not already provide this behavior.
