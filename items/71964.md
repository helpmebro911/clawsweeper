## Summary

Keep #71964 open. Current main still emits hard-coded version-manager and package-manager fallback directories into the daemon service PATH without checking whether those directories exist, and the current tests still assert that behavior. The PR is a focused bug-fix attempt for the remaining behavior; related #72109 is closed unmerged, so it does not supersede this PR. The useful next step is maintainer review or author follow-up on the minor test-hermetic note rather than automated closure.

## Best Possible Solution

Keep the PR open and either update #71964 or land an equivalent main-branch fix that gates only hard-coded version-manager/package-manager fallback dirs behind an injectable existence check, while keeping env-configured roots and stable user-bin conventions unconditional. Add deterministic Linux/macOS tests for none-existing, some-existing, and env-configured cases, and address the Nix test-hermetic review note before merge.
