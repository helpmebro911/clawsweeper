## Summary

Keep this PR open. Current main still builds daemon service PATHs with unconditional hard-coded version-manager/package-manager fallback directories, and the doctor audit still treats those same path families as non-minimal when unexpected. The PR is a focused implementation candidate for the remaining behavior behind #71944; the related #72109 was closed unmerged and does not supersede it.

## Best Possible Solution

Review and land this PR, or land an equivalent maintainer patch: add an injectable existence predicate to the service PATH builder, gate only hard-coded version-manager/package-manager fallback directories behind it, keep explicit env roots and stable user-bin conventions unconditional, and add deterministic Linux/macOS regression tests for missing, existing, and env-configured fallback paths.
