## Summary

Keep open. Current main has the session write lock and watchdog, but it does not implement the PR’s core protections: no force-release callback, no runtime lock-ownership check, and no SessionManager write guard. The PR discussion also shows earlier bot-review blockers were addressed on the PR branch, so the work is not obviously obsolete or low-signal.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it on current main, preserve the review fixes already made on the PR branch, and add/refresh regression coverage proving that watchdog force-release aborts the active run/session and that SessionManager writes are rejected after lock ownership is lost. If maintainers dislike private method monkey-patching, replace it with a first-class persistence/lock guard seam around the SessionManager integration rather than closing the bug.
