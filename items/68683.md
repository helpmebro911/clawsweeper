## Summary

Keep #68683 open. Current main has Perplexity web_search credential-resolution code, docs, and regression coverage for the intended config/env paths, but this recent detailed 401 report is not conclusively closed by the existing evidence. The report specifically raises stale scoped config, provider caching, and runtime split concerns, and current code still gives configured plugin credentials precedence over ambient env values, so a stale non-empty configured key can still explain the observed failure mode.

## Best Possible Solution

Keep the issue open and reproduce on current main or latest Homebrew with a controlled Perplexity key across these cases: plugin literal key, plugin SecretRef, PERPLEXITY_API_KEY-only, and stale plugin key plus fresh env. Add a regression or diagnostic proving which credential source is sent for provider=perplexity. If config-over-env precedence is intentional, update doctor/status/docs to surface stale configured keys and tell users to unset or resync plugins.entries.perplexity.config.webSearch.apiKey.

## Evidence
