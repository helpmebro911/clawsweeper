## Summary

Keep this PR open. Current main still lacks sandbox-only Python/Node research-script import validation, and the PR is a plausible core exec hardening change, but it needs changes before merge because review comments identify concrete bypasses in the proposed validator.

## Best Possible Solution

Keep this PR open and request changes. The best path is to rebase onto current main, preserve the existing shell-bleed/fail-closed preflight, harden the import validator for multiline Node import/re-export forms, whitespace in require/import calls, and Python bare-relative denied names, then add regression tests for those bypasses plus sandbox-vs-host behavior before maintainer review.

## Risks / Open Questions

- Merging the PR as-is could create false confidence in sandbox import enforcement because common JavaScript forms such as multiline imports/re-exports and require/import calls with whitespace can bypass the proposed regex scanner.
- The PR is based on an older exec preflight shape; a rebase must preserve current main's expanded shell-bleed and fail-closed interpreter preflight behavior.
- Closing the PR would leave current main without the proposed sandbox-only import hardening for direct Python/Node research scripts.
