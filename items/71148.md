## Summary

Keep open. Current main still ships the Lobster plugin against @clawdbot/lobster@2026.4.6, loads that package in-process, and the published package for that exact version still contains the reported Ajv compile-per-call sites with no validator cache or removeSchema cleanup. I did not find a merged OpenClaw change or newer Lobster dependency that fixes this issue.

## Best Possible Solution

Keep the issue open. Patch @clawdbot/lobster to memoize compiled validators by stable schema content, including the local llm_invoke Ajv path, and add a focused regression such as compile-count/cache-hit coverage for repeated ask/input/resume/output-schema validation. Then bump extensions/lobster to the fixed @clawdbot/lobster release and include the OpenClaw changelog/release evidence.
