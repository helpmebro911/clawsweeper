## Summary

Keep open. Current main still ships the affected in-process Lobster runtime path and pins @clawdbot/lobster 2026.4.6 with Ajv 8.18.0. The pinned upstream tarball still has the reported compile-per-call sites and no visible cache/removeSchema mitigation, while the issue has two independent production reproductions.

## Best Possible Solution

Keep #71148 open. The practical fix should land in the Lobster runtime by caching compiled validators by stable schema identity/content, or by compiling workflow/input schemas once where possible and releasing one-shot validators where safe, then bump @clawdbot/lobster in extensions/lobster and add a regression test/stress check that repeated identical schema validations do not repeatedly compile.
