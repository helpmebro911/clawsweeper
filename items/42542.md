## Summary

Keep PR #42542 open. Current main still has the loader/report asymmetry the PR targets: the Control UI file list hides BOOTSTRAP.md after setup, but `loadWorkspaceBootstrapFiles()` still returns a missing BOOTSTRAP.md entry and the `/context` report can still display it. The PR is recent, focused, has addressed earlier review concerns, and the discussion includes independent reproduction plus live-patch verification.

## Best Possible Solution

Keep PR #42542 open for maintainer review and likely merge an equivalent narrow fix: make `loadWorkspaceBootstrapFiles()` omit BOOTSTRAP.md only when workspace setup is complete and the guarded read failed specifically with ENOENT, while preserving present BOOTSTRAP.md and non-ENOENT diagnostics. The regression tests should cover post-setup missing omission, pre-setup inclusion, present-after-setup inclusion, and directory/non-ENOENT visibility.
