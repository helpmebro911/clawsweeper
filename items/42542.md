## Summary

Keep open. Current `main` still does not make `loadWorkspaceBootstrapFiles` setup-aware, so the asymmetry described in the PR remains: the gateway file list hides `BOOTSTRAP.md` after setup, but the context-report/system-prompt path still carries a missing `BOOTSTRAP.md` entry, and `attempt.ts` still emits the commit reminder whenever a placeholder `BOOTSTRAP.md` exists.
