## Summary

Keep this PR open: current main still uses a 200-message Control UI chat history request and a 200-message render window, so the PR’s central performance change is not implemented, but the PR needs a current-main rebase because the render logic has moved.

## Best Possible Solution

Keep this PR open and ask for a current-main update. The best implementation is to introduce a shared initial chat history window constant in `ui/src/ui/chat/constants.ts`, use it from `ui/src/ui/controllers/chat.ts` for the `chat.history` request and from `ui/src/ui/chat/build-chat-items.ts` for the render slice/notice, then update `ui/src/ui/controllers/chat.test.ts` and add a render-window assertion in `ui/src/ui/chat/build-chat-items.test.ts`. No gateway protocol, server, dependency, workflow, or security-sensitive change is needed.
