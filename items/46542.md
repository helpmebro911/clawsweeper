## Summary

Keep #46542 open for maintainer judgment rather than auto-closing. Current main does not implement the proposed shared memory store, `sharedPaths`, `_shared` reserved agent, or `--shared` memory CLI path. The PR itself is stale against current main because memory CLI/runtime ownership moved into the bundled `memory-core` plugin, but the underlying product problem remains real and is also tracked by open issue #46558.

## Best Possible Solution

Keep the PR open or convert it manually into a maintainer-directed rewrite. The best path is to solve #46558 in the current memory-core architecture: either add an explicit shared-index design to `memory-core` with safe path validation, status/index/search CLI support, docs, and regression tests, or decide that shared document recall should be an external memory plugin using the existing memory plugin runtime APIs. Do not close as implemented because current main still lacks the requested shared store behavior.
