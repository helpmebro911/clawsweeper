## Summary

Keep this PR open. Current main still has the additive legacy `flow_runs` migration without a rebuild, while the write path inserts only `owner_key`; a legacy table that still enforces `owner_session_key TEXT NOT NULL` can plausibly reject new managed flow creation. The PR remains a focused implementation candidate with a matching regression test and no observed supply-chain-sensitive file changes.

## Best Possible Solution

Keep this PR open and review or refresh it against current main. The best path is to land a scoped legacy `flow_runs` rebuild in `src/tasks/task-flow-registry.store.sqlite.ts`, keep the regression coverage in `src/tasks/task-flow-registry.store.test.ts`, and validate with the targeted task-flow store test plus the changed gate before merge.
