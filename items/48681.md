## Summary

Keep #48681 open. Current main still has only a single `workspace` per agent, no `multipleWorkspaces` config/schema surface, and no composite workspace symlink setup. The PR is a focused core-agent/workspace feature rather than ClawHub cleanup, but the PR discussion also shows substantive remaining review work on all workspace setup flows and edge cases before it should land.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is a core implementation that adds a documented `multipleWorkspaces` agent config field, validates it in the runtime schema, resolves absolute workspace roots, creates a composite workspace through the shared `ensureAgentWorkspace` path used by replies/CLI/cron/gateway flows, updates reverse workspace lookup/listing, and adds focused tests for collision, reserved-name, root-path, relative-path, stale-link, and Windows symlink behavior.
