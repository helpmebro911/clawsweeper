## Summary

Keep open. Current main has not adopted this PR's dispatcher-stripping behavior; it still forwards the SSRF dispatcher into the Slack runtime fetch path and tests that contract. The linked Slack attachment bug #63905 remains open, and this PR is a focused implementation candidate that needs maintainer review because it changes the SSRF pinned-dispatcher boundary.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, decide whether Slack media should strip the upstream SSRF dispatcher or preserve the pinned dispatcher path, update the regression test to lock the chosen contract, and then either land a source-backed fix for #63905 or close this PR with a documented maintainer decision.
