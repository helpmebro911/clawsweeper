## Summary

Keep open. Current `main` still does not forward config `env.vars` into sandbox exec environments, so the requested behavior is not implemented. The PR also has an unresolved correctness issue: it proposes forwarding `collectConfigRuntimeEnvVars()` directly, which would bypass the existing `applyConfigEnvVars()` guards that preserve operator env precedence and skip unresolved `${VAR}` placeholders.
