## Summary

Keep open. Current main still does not forward config env.vars into sandbox exec environments, so the PR is not implemented or obsolete. The PR addresses a real gap, but the submitted approach needs adjustment because it appears to bypass existing applyConfigEnvVars guards for existing env precedence and unresolved ${VAR} placeholders.

## Best Possible Solution

Keep the PR open and revise it into a corrected sandbox env propagation patch: thread an effective config env map into sandbox exec, preserve applyConfigEnvVars semantics for existing runtime env and unresolved placeholders, layer it below sandboxEnv and request paramsEnv, and add focused regression tests for sandbox forwarding, precedence, blocked keys, and placeholder behavior. If maintainers choose the stricter documented contract instead, close after linking the canonical #57427 discussion and documenting that sandbox users must use agents.defaults.sandbox.docker.env.
