## Summary

Current `main` does not implement this PR, and the repo docs describe the reported sandbox behavior as intentional rather than a regression. The proposed change is also risky as written because it would bypass the existing `applyConfigEnvVars()` guards that preserve operator-provided env precedence and skip unresolved `${VAR}` placeholders. On conservative cleanup, this should stay open rather than be closed as implemented/obsolete.
