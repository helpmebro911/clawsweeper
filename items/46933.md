## Summary

Keep PR #46933 open. Current main still has the reported `status --deep` last-heartbeat rendering bug: the active overview path appends `ago` to `formatTimeAgo(...)` output and renders missing heartbeat channel metadata as `unknown`. The PR’s proposed fix is still relevant, though it likely needs rebasing/adaptation to the current `status.command-sections.ts` structure.

## Best Possible Solution

Keep the PR open and ask for a rebase that moves the helper or equivalent formatting fix into `buildStatusLastHeartbeatValue` in `src/commands/status.command-sections.ts`, with focused tests asserting that `formatTimeAgo` is not double-suffixed and optional `channel`/`accountId` fields are omitted instead of rendered as `unknown`.
