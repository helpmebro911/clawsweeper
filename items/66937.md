## Summary

Keep open. Current `main` still requires a non-empty LM Studio API key in the interactive onboarding path the report describes, so the bug is not implemented away on `main`. Recent LM Studio work only relaxed non-interactive/config-backed flows via placeholder or synthetic auth.

## Evidence

- **Interactive LM Studio onboarding still makes the API key prompt mandatory:** `promptAndConfigureLmstudioInteractive` always calls `ensureApiKeyFromEnvOrPrompt` with `validate: (value) => (value.trim() ? undefined : "Required")`, and the non-prompter fallback also validates blank input as `Required`. That matches the issue's reported failure when trying to skip the key during onboarding.

There is no interactive test covering a skipped or blank API key, so the reported behavior is still the expected tested behavior on `main`.
