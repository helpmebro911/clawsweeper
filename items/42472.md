## Summary

Keep open. Current main still strips a trailing mixed `NO_REPLY` token in the shared auto-reply normalizer and delivers the preceding text, and the tests explicitly assert that behavior. Main has a narrower cron announce fix, but not a general reply-pipeline fix. The provided GitHub context also shows open, unmerged PR #42873 using closing syntax for this issue, plus follow-up PR #46626, so the issue should remain open until those candidates are reviewed, landed, or closed.

## Best Possible Solution

Keep this issue open. Maintainers should review or rework PR #42873 and PR #46626 against current main, ideally by adding one shared helper for own-line/trailing silent-token intent and applying it consistently in `normalizeReplyPayload`, directive/outbound delivery, gateway live/history projection, cron delivery, and subagent fallback paths, with tests preserving legitimate inline mentions, leading substantive text, exact-token policy, and media payload behavior.
