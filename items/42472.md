## Summary

Keep open. Current `main` does not implement the requested behavior: mixed-content replies containing `NO_REPLY` are normalized by stripping the token and delivering the remaining text, and the tests/docs still encode `NO_REPLY` as an exact-token-only silent contract.

## Evidence

- **Mixed-content NO_REPLY is stripped, not fully suppressed:** `normalizeReplyPayload` removes `NO_REPLY` from mixed-content text and only returns `null` when nothing remains after stripping. That means leading/trailing substantive text is still delivered.
  - file: [src/auto-reply/reply/normalize-reply.ts:64](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/auto-reply/reply/normalize-reply.ts#L64)
  - command: `nl -ba src/auto-reply/reply/normalize-reply.ts | sed -n '1,140p'`
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Unit test expects text before trailing NO_REPLY to remain visible:** The current test suite asserts that `"File's there. Not urgent.\n\nNO_REPLY"` normalizes to a non-null payload that still contains `"File's there"` and no longer contains `NO_REPLY`.
  - file: [src/auto-reply/reply/reply-utils.test.ts:122](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/auto-reply/reply/reply-utils.test.ts#L122)

- **Outbound delivery tests keep mixed NO_REPLY payloads:** `normalizeReplyPayloadsForDelivery` is currently tested to keep `"NO_REPLY thanks for the update"` and `"thanks NO_REPLY"`, while suppressing only the exact sentinel payload.
  - file: [src/infra/outbound/payloads.test.ts:135](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/infra/outbound/payloads.test.ts#L135)
  - command: `nl -ba src/infra/outbound/payloads.test.ts | sed -n '132,160p'`

- **Docs still describe silent behavior as exact-token-only:** The user-facing docs say the exact silent token `NO_REPLY` / `no_reply` means no visible reply, which matches the current code and not the requested broader suppression rule.
