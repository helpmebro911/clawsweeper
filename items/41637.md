## Summary

Keep this PR open. Current main still creates one Feishu webhook HTTP server per account and calls `server.listen(port, host)` for each webhook account, so two accounts sharing the documented default host/port can still collide with `EADDRINUSE`. The PR is a focused implementation candidate for a real remaining Feishu multi-account webhook bug, but it should not merge as-is without a rebase and security/lifecycle fixes for the unresolved review comments.

## Best Possible Solution

Keep the PR open as a real Feishu plugin bug-fix candidate. The best path is a rebased implementation that shares one webhook listener per host:port, routes by token/app/path without cross-account delivery, preserves current signed-request/body/rate-limit guards, handles creator/joiner abort and server error propagation correctly, and adds regression tests for two webhook accounts on the same host:port plus cleanup/error cases.
