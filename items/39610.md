## Summary

Keep #39610 open. Current main at c7a0d9b1889ca09ab5bfd84cda39a8d50d3d5768 still only supports global `agents.defaults.userTimezone`; it does not add `agents.list[].userTimezone`, an agent-aware timezone resolver, schema/docs coverage, or agent-context threading through prompt, heartbeat, envelope, gateway timestamp, cron/current-time, post-compaction, and system-event paths. The provided PR context shows #39610 is the canonical unmerged implementation for closed issue #39579, with useful prior review fixes, but the work is not shipped on main and should not be closed.

## Best Possible Solution

Keep #39610 open as the canonical implementation path for per-agent `userTimezone`. The maintainer-ready path is to add `agents.list[].userTimezone` to typed config, strict runtime schema, generated schema, and docs; centralize agent-aware timezone resolution as per-agent override -> `agents.defaults.userTimezone` -> host timezone; thread resolved `agentId` through prompt, session status/current-time helpers, heartbeat active-hours and body injection, envelopes/system events, gateway/chat timestamp injection, cron, memory flush, post-compaction context, and plugin inbound-envelope surfaces; validate IANA timezone strings consistently; then rebase and run the relevant changed gates.
