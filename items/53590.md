## Summary

Keep this open. Current main still has only a partial configured-primary warmup warning, and it does not catch the reported invalid/retired OpenRouter slug case because OpenRouter dynamic resolution still synthesizes a runtime model for uncatalogued IDs. Doctor validation also remains scoped to `hooks.gmail.model`, not `agents.defaults.model.primary` or configured fallbacks.

## Best Possible Solution

Keep this open and add a non-fatal diagnostic path for configured primary and fallback model refs. Startup warmup and `openclaw doctor` should warn when a provider with a successfully fetched live catalog can prove the configured ref is absent, while preserving OpenRouter's dynamic-model escape hatch when the catalog cannot be reached or the provider intentionally accepts custom IDs.
