## Summary

Keep open. Current `main` still ties indexing concurrency to `this.batch.enabled`, so `agents.defaults.memorySearch.remote.batch.concurrency = 1` is not honored once batch mode is disabled, unsupported, or falls back to normal embedding calls. That matches the report and the linked open PR #66931.

## Evidence

- **Indexing concurrency still ignores configured batch concurrency when batch mode is off:** `getIndexConcurrency()` returns the user-configured batch concurrency only when `this.batch.enabled` is true; otherwise it hardcodes `EMBEDDING_INDEX_CONCURRENCY` (4).

  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:693](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/memory-core/src/memory/manager-sync-ops.ts#L693)
