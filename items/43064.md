## Summary

Keep this issue open. Current main still passes the raw heartbeat responsePrefix through the heartbeat path without the model-selection callback or prefix context needed to interpolate `{model}`, and the supplied GitHub context shows open implementation candidates with closing syntax, including the same-author PR #43065.

## Best Possible Solution

Keep this issue open and review the linked implementation PRs, especially #43065 as the same-author candidate. The fix should stay in `src/infra/heartbeat-runner.ts`: reuse the shared reply-prefix context path, pass `onModelSelected` into `getReplyFromConfig()`, resolve the prefix after model selection, and add focused regression coverage for templated heartbeat alert prefixes plus `HEARTBEAT_OK` stripping.
