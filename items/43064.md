## Summary

Current main still treats heartbeat `responsePrefix` as a raw string on the heartbeat path, so templated values such as `{model}` can still be prepended literally. This issue also has an open same-author implementation candidate, #43065, with closing syntax, so it should remain open until that PR is reviewed, landed, or rejected.

## Best Possible Solution

Keep this issue open and review #43065 as the primary implementation candidate. The fix should stay in `src/infra/heartbeat-runner.ts`: reuse the shared reply-prefix context path, pass `onModelSelected` through `getReplyFromConfig`, resolve the prefix after model selection, and add focused regression coverage for templated heartbeat alert prefixes and prefixed `HEARTBEAT_OK` handling.
