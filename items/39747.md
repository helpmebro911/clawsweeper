## Summary

Keep #39747 open. Current main still normalizes an explicit heartbeat target before any plugin-only channel metadata/bootstrap can make that id deliverable, so the remaining plugin-channel ordering bug is not implemented on main. Telegram is now a bundled channel, making the original example less representative, but the documented/configured contract still permits plugin ids.

## Best Possible Solution

Keep #39747 open, or replace it with a focused successor, until heartbeat target resolution bootstraps or reads plugin channel metadata before deliverable-channel recognition, retries normalization after registry initialization, and adds regression coverage for plugin-only agents.defaults.heartbeat.target. Maintainers should also decide whether heartbeat.target should count as gateway startup channel intent.
