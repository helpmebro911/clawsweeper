## Summary

Keep #36630 open. Current main at e672b614 still does not implement the PR's central bidirectional Signal quote-reply behavior: outbound Signal sends never emit signal-cli quote metadata, Signal outbound adapters do not accept or forward quote author/reply metadata, and inbound Signal quote context still omits ReplyToId plus the richer quote payload model. The PR is still a useful implementation candidate, though it needs maintainer review and hardening before merge.

## Best Possible Solution

Keep #36630 open. The best maintainer path is to land a reviewed, hardened version of this Signal quote-reply work or reimplement it narrowly: strict decimal and safe Signal quote timestamp/id validation, group quote-author resolution from the referenced message, quote-aware Signal outbound delivery through formatted/plugin/monitor/queue paths, inbound ReplyToId plus richer signal-cli quote metadata, extension-boundary-compliant imports, and focused regression coverage for delivery consumption, debounce, queued/recovered delivery, and plugin-boundary cases.
