## Summary

Keep this issue open. The RFC is recent, coherent, and still describes a missing shared OpenClaw contract: current main has tool result `content` plus `details`, ad hoc freshness flags in some surfaces, and a trusted bundled-plugin middleware hook, but no standardized `freshness` envelope or `ToolFreshnessMetadata` type emitted by `session_status`, `memory_search`, or shared tool-result helpers.

## Best Possible Solution

Keep the issue open for maintainer design. The best path is a backwards-compatible core/SDK contract for optional tool-result freshness metadata, documented for plugin authors, with targeted adoption in `session_status`, `memory_search`, cron status/list/runs, and cached web tools. The existing bundled-plugin middleware can support experiments or migrations, but it should not be treated as closing the standardized freshness contract request.
