## Summary

Keep open. Current main has adjacent truthfulness mechanisms, but it does not implement claim-level confidence tiers/source attribution for every factual claim or autonomous completion reporting for unverified claims. Memory citations and memory-wiki claim metadata are scoped to stored memory/wiki content, not the general agent reply and completion contracts.

## Best Possible Solution

Keep the RFC open for maintainer product and protocol design. A credible path would define an additive core/SDK-visible trust signal for agent output, covering claim-level evidence where available, known-vs-inferred wording guidance, and autonomous completion fields for unverified claims. Existing memory citations and memory-wiki claim metadata should be reused as inputs, not treated as the whole solution.
