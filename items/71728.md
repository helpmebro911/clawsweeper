## Summary

Keep #71728 open. Current main has adjacent grounding mechanisms: prompt guidance, memory citations, input provenance, and memory-wiki claim metadata. It does not implement the RFC's requested claim-level confidence tiers, source attribution for every factual assistant claim, known-vs-inferred output structure, or autonomous completion reporting for unverified claims. The remaining work cuts across core reply and gateway/internal-event contracts, so this is not implemented and not a safe ClawHub-only close.

## Best Possible Solution

Keep the RFC open for maintainer design. A credible implementation should define an additive core/SDK-visible trust signal for assistant output, likely touching ReplyPayload or transcript/message metadata plus AgentInternalEvent/Gateway protocol completion fields. It should reuse existing memory citations, input provenance, tool metadata, and memory-wiki claims as evidence sources where available, while documenting channel fallback behavior for clients that can only render plain text.
