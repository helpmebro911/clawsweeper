## Summary

Current main still has the reported behavior: `gateway.trusted_proxies_missing` is emitted only for `bind === "loopback"` and hard-coded as `severity: "warn"`. The existing regression test also still expects the loopback case to be `warn`, and the public audit-checks doc table still lists the check as `warn`. The provided GitHub context shows #70368 as an open, unmerged fix PR, so this issue should remain open until that PR or an equivalent change lands.

## Best Possible Solution

Keep #70357 open and land #70368 or an equivalent scoped fix: change `src/security/audit-gateway-config.ts` so loopback with empty `trustedProxies` emits `gateway.trusted_proxies_missing` at `info`, non-loopback binds with empty `trustedProxies` emit `warn`, add or update regression coverage in `src/security/audit-loopback-logging.test.ts` or the gateway audit tests, and update `docs/gateway/security/audit-checks.md` to document the split.
