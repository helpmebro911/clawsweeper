## Summary

Current main still does not expose OPENCLAW_SESSION_KEY or OPENCLAW_AGENT_ID in exec child-process environments. The session context is available to the exec tool, but local, PTY, sandbox, Docker, and node-host execution paths still pass only the existing env plus OPENCLAW_SHELL=exec, while related PR #68859 remains open and only partially addresses the request.

## Best Possible Solution

Keep this issue open for a focused core exec-runtime fix. Add a shared trusted runtime-env helper that injects both OPENCLAW_SESSION_KEY and OPENCLAW_AGENT_ID after host/sandbox env construction and sanitization, apply it to gateway, PTY, sandbox/Docker/backend, and node-host system.run execution, define precedence over inherited or requested values, update runtime env docs, and add regression tests across those paths. Review #68859 as a starting point, but broaden it before resolving this issue.
