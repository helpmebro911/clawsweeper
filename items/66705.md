## Summary

Keep open. Current main still carries trusted `agentId` and `sessionKey` into the exec tool, but it does not expose `OPENCLAW_SESSION_KEY` or `OPENCLAW_AGENT_ID` in child-process environments. The only documented/runtime-injected exec marker is still `OPENCLAW_SHELL=exec`, and the provided related PR #68859 remains an open partial implementation candidate for `OPENCLAW_SESSION_KEY` only.

## Best Possible Solution

Keep this issue open for a focused core exec-runtime change. Add a shared helper that injects trusted `OPENCLAW_SESSION_KEY` and `OPENCLAW_AGENT_ID` after env sanitization/sandbox env construction, applies it consistently to gateway, PTY, sandbox backend/Docker, and node-host exec paths, documents the new runtime env contract, and adds regression tests for stale inherited/requested values and path coverage. Review open PR #68859 as a starting point, but broaden it before considering this resolved.
