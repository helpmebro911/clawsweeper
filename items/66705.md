## Summary

Keep open: current `main` still only injects `OPENCLAW_SHELL=exec` into exec child environments, and there is no code, docs, or test evidence that `OPENCLAW_SESSION_KEY` or `OPENCLAW_AGENT_ID` are exposed.

## Evidence

- **Exec runtime env injection:** `runExecProcess()` builds the spawned child environment from `opts.env` and adds only `OPENCLAW_SHELL: "exec"`; it does not add `OPENCLAW_SESSION_KEY` or `OPENCLAW_AGENT_ID`.
  - file: [src/agents/bash-tools.exec-runtime.ts:524](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/bash-tools.exec-runtime.ts#L524)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Session and agent context stay internal:** The exec tool resolves `defaults.sessionKey` and derives `agentId`, but later passes the session key only as internal process/session metadata (`sessionKey: notifySessionKey`) rather than merging either value into the child env.
  - file: [src/agents/bash-tools.exec.ts:1360](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/bash-tools.exec.ts#L1360)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Current docs only promise OPENCLAW_SHELL:** The public exec docs say spawned commands receive `OPENCLAW_SHELL=exec` so shell/profile rules can detect exec-tool context; they do not document any session-specific env vars.
  - file: [docs/tools/exec.md:79](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/tools/exec.md#L79)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Background exec docs match the same contract:** The background exec/process docs likewise document only `OPENCLAW_SHELL=exec` for spawned exec commands, with no mention of `OPENCLAW_SESSION_KEY` or `OPENCLAW_AGENT_ID`.
  - file: [docs/gateway/background-process.md:31](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/gateway/background-process.md#L31)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Tests cover only OPENCLAW_SHELL:** Current exec tests assert that PTY and gateway exec expose `OPENCLAW_SHELL`, but there are no corresponding tests for `OPENCLAW_SESSION_KEY` or `OPENCLAW_AGENT_ID`.
  - file: [src/agents/bash-tools.exec.pty.test.ts:39](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/bash-tools.exec.pty.test.ts#L39)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Repo-wide search on current main:** A repo search for `OPENCLAW_SESSION_KEY` and `OPENCLAW_AGENT_ID` returned no matches in `src`, `docs`, `test`, `packages`, or `extensions`, which is strong evidence the requested env vars are not implemented yet.
  - command: `rg -n "OPENCLAW_SESSION_KEY|OPENCLAW_AGENT_ID" src docs test packages extensions`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
