## Summary

Keep open. Current main at f0b327cf686e3a957fef8c94985ee6509953a3b7 still lacks a `silent` field on cron failure-alert config, gateway schemas, UI state/forms, docs, and runtime delivery plumbing. The existing Telegram/outbound delivery stack already supports silent messages, so this remains a narrow cron integration gap rather than a ClawHub-only request or an implemented feature.

## Best Possible Solution

Implement a narrow additive cron contract change on main: add `silent?: boolean` to per-job `CronFailureAlert` and global `cron.failureAlert`; update TypeBox/Zod/generated config docs, CLI docs, and Control UI state/forms; propagate the resolved boolean through `resolveFailureAlert`, `emitFailureAlert`, `sendCronFailureAlert`, `sendGatewayCronFailureAlert`, `sendCronAnnouncePayloadStrict`, and `deliverOutboundPayloads({ silent })`; add focused tests proving Telegram cron failure alerts set `disable_notification` when `silent: true` and preserve current default behavior when omitted.
