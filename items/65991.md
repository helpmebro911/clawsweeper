## Summary

Keep open. Current main still does not expose or propagate a `silent` flag on cron `failureAlert`; the request is a concrete core cron/outbound integration gap, not a duplicate or ClawHub-only plugin idea. The existing Telegram/message path already supports silent delivery, so this is feasible without a new plugin API.

## Best Possible Solution

Implement a narrow cron/outbound contract change on main: add `silent?: boolean` to the per-job `CronFailureAlert` shape and any intended global/default failure-alert config, update TypeBox/Zod schemas, docs, cron CLI/UI/tool schemas, and thread it through `resolveFailureAlert`, `emitFailureAlert`, `sendCronFailureAlert`, and `sendFailureNotificationAnnounce` into `deliverOutboundPayloads({ silent })`. Add regression coverage proving Telegram failure alerts set `disable_notification` when `silent: true` while preserving the default noisy behavior.
