## Summary

Current main still has the reported `openclaw doctor` UX gap: the pre-doctor git update path prints one static update note and calls `runGatewayUpdate` without the existing step-progress controller. The PR is a focused fix for live behavior that remains unimplemented, and the provided review discussion shows the author addressed the initial TTY-gating and cleanup concerns.

## Best Possible Solution

Keep this PR open or consolidate it as the canonical fix for the doctor pre-update UX. The best code path is to land the TTY-gated `createUpdateProgress(Boolean(process.stdout.isTTY))` wiring in `src/commands/doctor-update.ts`, pass `progress` into `runGatewayUpdate`, ensure `stop()` runs in `finally`, and add a narrow regression test if feasible. After that, close or redirect the later duplicate PR #47672 rather than closing this one now.
