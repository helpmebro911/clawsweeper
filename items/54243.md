## Summary

Keep open. Current main still treats `--no-include-workspace` as a planning flag, not an archive filter: the state directory is always an included backup asset, the default workspace lives under `~/.openclaw`, and archive creation tars included source trees whole. CLI help/docs still imply workspace files are excluded, while related exclude-pattern work remains open and unmerged.

## Best Possible Solution

Keep this issue open until native backup either makes `--no-include-workspace` materially exclude nested workspace directories from the state archive, or intentionally clarifies the flag/docs/CLI copy and routes real nested exclusions through shipped `--exclude` / `--exclude-file` support. The likely implementation touches `src/commands/backup-shared.ts`, `src/infra/backup-create.ts`, CLI/docs copy, and regression coverage proving a file under `~/.openclaw/workspace` is absent from the actual archive when workspace exclusion is requested.
