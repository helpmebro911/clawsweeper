## Summary

Current main still does not make `openclaw backup create --no-include-workspace` materially exclude default workspaces nested under the state directory. The flag removes workspace assets from discovery, but the backup plan still includes the state directory and the archive writer tars that source tree without a nested exclusion filter. The related exclude-pattern request #40786 and implementation candidate #44288 remain open, and they do not by themselves prove this flag’s semantics are fixed on main.

## Best Possible Solution

Keep this issue open until `backup create --no-include-workspace` either filters nested workspace directories out of the archived state tree or the CLI/docs contract is explicitly narrowed to say it only skips separate workspace discovery. A robust fix belongs in the backup planner/archive writer surface, with a regression test proving a file under `.openclaw/workspace` is absent from the archive. Maintainers can also use #40786/#44288 as the broader exclude-pattern path, but this issue should stay open until that work lands and covers this flag semantics gap.
