## Summary

Current main still does not make `backup create --no-include-workspace` materially exclude default workspaces under `~/.openclaw`; it only removes workspace assets from the plan while still archiving the state directory as a whole. The CLI/docs still describe the flag as excluding workspace files, and the open exclude-pattern work remains unmerged.

## Best Possible Solution

Keep this issue open until the backup command either filters nested workspace directories out of the archived state tree when `--no-include-workspace` is used, or explicitly changes the CLI/docs contract to say the flag only skips separate workspace discovery and does not filter the state payload. A robust fix should add archive-level regression coverage proving a file under `.openclaw/workspace` is absent, and maintainers can also resolve it by reviewing and landing a focused exclude-pattern implementation such as #44288 if that becomes the canonical path.
