## Summary

Keep open. Current `main` does not contain the proposed probe-specific idle-flush timeout change, and the underlying timeout-budget issue remains plausible in the live probe path. The PR also has an unresolved correctness risk because it keys the shorter cleanup path off any `sessionId` starting with `probe-`, while `openclaw agent --session-id` allows arbitrary user-chosen IDs.
