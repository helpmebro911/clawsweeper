## Summary

Keep open. Current `main` still routes auth probes through the normal post-run idle-flush cleanup with the default 30s wait, so the reported probe-timeout coupling is not already fixed. The submitted PR also uses `sessionId.startsWith("probe-")` as the trigger, and current CLI surfaces still allow arbitrary user-supplied session IDs, so that heuristic can affect non-probe sessions too.
