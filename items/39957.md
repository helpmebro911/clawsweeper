## Summary

Keep PR #39957 open. Current main still lacks the PR's narrow suppression for `message_end` block replies when an assistant turn is `stopReason === "toolUse"` and contains a structured tool-call block, so the reported Telegram-visible intermediate text leak remains plausible on the current code path.

## Best Possible Solution

Keep the PR open and land either this narrow `message_end && stopReason === "toolUse" && has structured tool-call block` suppression with regression coverage, or an equivalent core display-phase fix. Preserve the PR's non-goals: normal `stop` turns, `toolUse` turns without tool-call blocks, and `text_end` block-reply delivery should continue to emit.
