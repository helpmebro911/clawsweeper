## Summary

Keep open. The PR's central behavior change is still not present on current main: empty-output successful background exec completions remain opt-in, the runtime suppresses them unless `notifyOnExitEmptySuccess` is explicitly true, tests still assert the default skips no-op completion events, and docs still state the default is false. The related issue #18237 being closed stale does not supersede this PR because the proposed code change remains unshipped.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. The best implementation path is the PR's narrow change set: flip both `createExecTool` and `runExecProcess` to default `notifyOnExitEmptySuccess` to true unless explicitly false, preserve the opt-out, update the no-op notification tests, and keep config/docs aligned. The PR should also be reviewed against related orchestration work such as #54549 rather than closed as stale or implemented.
