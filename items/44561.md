## Summary

Keep open. Current `main` has a partial mitigation for new heartbeat/cron/exec writes, but it does not implement this PR’s main-session display-name stabilization. Gateway session rows still fall back to persisted `origin.label`, and the session metadata merge path preserves old origin labels when system-event origin updates are skipped. That means previously contaminated main-session entries can still surface transient labels like `heartbeat` in the UI, so this PR still changes real behavior that is not already on `main`.

## Evidence
