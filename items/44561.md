## Summary

Keep open. Current `main` still builds gateway session rows by falling through to `entry?.label ?? origin.label` with no main-session stabilization, and the UI still prefers/render `row.displayName` when present. The PR discussion also surfaced valid edge cases around preserving explicit `displayName` values and honoring configured/global main keys; those concerns remain relevant because `main` has the main-key helpers but does not use them in `buildGatewaySessionRow`.

## Evidence

There is no branch for main-session keys, so a transient origin label can still become the stored/displayed name.
