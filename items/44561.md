## Summary

Keep open. Current main has a broader root-cause mitigation for system-event origin metadata, but it still does not contain this PR's gateway display-name stabilization: `buildGatewaySessionRow` still falls through to `origin.label`, and the Control UI still prefers a row `displayName` over the parsed `Main Session` fallback. The PR discussion also raised real implementation constraints around preserving explicit `displayName` and handling configured/global main keys, so this remains maintainer-reviewable work rather than cleanup-closeable.

## Best Possible Solution

Keep the PR open for maintainer review or replacement by a narrowly scoped gateway fix. The preferred implementation should stabilize only canonical main-session rows, preserve explicit `entry.displayName`, use `resolveAgentMainSessionKey`/`resolveMainSessionKey` for custom and global main keys, and add a regression test covering a main-session row with transient `origin.label: "heartbeat"`.
