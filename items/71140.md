## Summary

Keep #71140 open. Current main still has the split described in the issue and maintainer comment: MEDIA/GIF handling is present, and Telegram has a native reaction primitive, but there is no shared `[[react...]]` directive parser/normalizer path and Telegram reply/outbound delivery does not execute `channelData.telegram.reaction`. The linked PR #71141 is an open draft in the provided context, so it does not supersede or ship the fix yet.

## Best Possible Solution

Keep #71140 open as the canonical bug until a fix lands. The best implementation path is the one already outlined by the maintainer and PR #71141: extend shared reply directive parsing/normalization to emit structured reaction metadata, preserve it through outbound payload planning, allow reaction-only Telegram replies when a reply target exists, call the existing Telegram reaction helper from delivery/sendPayload, and add parser, payload, Telegram delivery, and outbound-adapter regression tests.
