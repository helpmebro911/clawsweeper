## Summary

Keep this issue open. Current main still lacks shared parsing for `[[react...]]` directives and does not execute `channelData.telegram.reaction` during Telegram reply or outbound adapter delivery. The media/GIF half is largely present, but the reaction directive gap remains, and the same author has an open draft PR (#71141) with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and use PR #71141, or a maintainer-owned replacement patch, as the implementation path. The fix should extend shared directive parsing for `[[react:...]]` and `[[react_to_current:...]]`, preserve structured `channelData.telegram.reaction`, allow reaction-only Telegram replies when a valid target exists, execute native Telegram reactions in both reply delivery and outbound adapter paths, and add parser, normalization, delivery, and adapter tests.
