## Summary

Keep this issue open. Current main at 348728c28c1ea9ae7c5824350baba35ee08a275e still has Telegram MEDIA parsing and native Telegram reaction primitives, but the normal outbound reply path does not parse `[[react...]]` / `[[react_to_current:...]]`, does not synthesize `channelData.telegram.reaction`, and Telegram reply delivery still drops empty text/media replies before any reaction handling. Draft PR #71141 is also open, unmerged, authored by the same reporter, and uses closing syntax for this issue, so the right cleanup action is to leave the issue open while that PR is reviewed, landed, or replaced.

## Best Possible Solution

Keep this issue open and use draft PR #71141, or an equivalent maintainer-owned patch, as the active implementation path. The fix should extend shared directive parsing for `[[react:...]]` and `[[react_to_current:...]]`, preserve structured Telegram reaction data through outbound normalization, allow reaction-only Telegram replies when a reply target exists, execute native Telegram reactions in both reply delivery and outbound adapter paths, and add parser, normalization, delivery, and adapter tests.
