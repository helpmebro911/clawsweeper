## Summary

Keep #71140 open. Current main still does not implement the reaction-directive half of the report; media/GIF handling exists, and open draft PR #71141 is the right fix path but is not merged.

## Best Possible Solution

Keep the issue open until PR #71141 or an equivalent fix lands. The right product/code path is to extend the shared directive parser to emit structured reaction metadata, preserve it through outbound normalization, allow reaction-only Telegram replies when a reply target exists, call `reactMessageTelegram`/`setMessageReaction` in Telegram delivery and `sendPayload`, and add parser, delivery, and outbound-adapter regression tests. The existing media/GIF path should be left intact rather than reworked broadly.
