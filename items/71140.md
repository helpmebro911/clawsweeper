## Summary

Keep open. Current main still has the reaction-directive gap: media directives are parsed and Telegram has native media/reaction primitives, but `[[react_to_current:...]]` is not parsed into structured outbound data, reaction-only replies are rejected before Telegram delivery, and `telegramOutbound.sendPayload` does not execute `channelData.telegram.reaction`. The linked PR #71141 is still an open draft and unmerged, so it does not supersede or ship the fix.

## Best Possible Solution

Land a focused fix like #71141 after review: extend the shared reply directive parser to emit structured reaction metadata, carry it through outbound normalization as Telegram channelData, allow reaction-only Telegram replies when a reply target exists, execute the existing Telegram reaction primitive before optional text/media sends, and add parser, delivery, and outbound-adapter regression tests.
