## Summary

Keep #11040 open. Current main has partial lineage support (`spawnedBy`, `parentSessionKey`, `spawnDepth`) but still does not implement the requested Phase 1 chain contract because `rootSessionKey` and a session-chain `traceId` are absent from the persisted session model, gateway schema, spawn write paths, `sessions_list` tool rows, and UI row type. The related implementation PR #11043 is closed unmerged, and the latest provided maintainer comment already kept the issue open for these remaining root/trace gaps.

## Best Possible Solution

Keep the issue open until core either implements the additive Phase 1 contract across `SessionEntry`, gateway schema/patch/list rows, native and ACP spawn write paths, `sessions_list` tool output, UI/shared row types, and regression tests for multi-hop root/trace lineage, or a maintainer explicitly narrows the product direction to the current `spawnedBy`/`parentSessionKey`/`spawnDepth` model. A ClawHub observability plugin can consume current partial lineage, but it cannot create canonical persisted `rootSessionKey` and `traceId` for all spawned sessions without a core contract.
