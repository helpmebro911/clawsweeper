## Summary

Keep #11040 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still has only partial session lineage support: spawnedBy, parentSessionKey, and spawnDepth appear in selected surfaces, but the requested rootSessionKey and stable session-chain traceId contract is not implemented across the persisted session model, gateway protocol/list rows, agent-facing sessions_list output, or native/ACP spawn write paths. The related PR #11043 was closed unmerged, and the latest issue discussion already contains a maintainer Codex review saying to keep this open.

## Best Possible Solution

Keep #11040 open until core implements the additive Phase 1 session-chain contract across SessionEntry, gateway schemas/list rows/events, native and ACP spawn write paths, agent-facing sessions_list output, UI/shared row types where needed, and multi-hop regression tests. If maintainers decide spawnedBy/parentSessionKey/spawnDepth is the intended boundary, close it explicitly with that product rationale rather than via conservative cleanup.
