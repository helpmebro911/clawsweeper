## Summary

Keep #11040 open. Current main still only has partial lineage support (`spawnedBy`, `parentSessionKey`, `spawnDepth`, child links/descendant counts) and does not implement the requested first-class `rootSessionKey` plus stable session-chain `traceId` contract across the persisted session model, gateway protocol/list rows/events, agent-facing `sessions_list`, or native/ACP spawn write paths. The related PR #11043 was closed unmerged, and the existing issue discussion already has a Codex maintainer review keeping this open.

## Best Possible Solution

Keep #11040 open until core implements the additive Phase 1 session-chain contract across SessionEntry, gateway schemas/list rows/events, native and ACP spawn write paths, agent-facing sessions_list output, shared/UI row types where needed, docs, and multi-hop regression tests. If maintainers decide spawnedBy/parentSessionKey/spawnDepth plus current subagent registry traversal is the intended product boundary, close it explicitly with that rationale rather than via conservative cleanup.
