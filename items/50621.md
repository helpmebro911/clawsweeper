## Summary

Current main still applies the 10-minute default cron timeout to `systemEvent` jobs, including `sessionTarget: "main"` / `wakeMode: "now"`, while that path can still await `runHeartbeatOnce()`. The linked fix PRs were closed unmerged, and the requested fire-and-forget or configurable timeout behavior is not present.
