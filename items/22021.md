## Summary

Current main does not implement the requested transport-layer actual-model exposure. The OpenAI-compatible HTTP handlers return the requested agent/model target in response bodies and streams, and there is no X-Actual-Model header implementation or documented response header. The related #25043 was closed as stale and was broader CLI/session observability, so it does not supersede this specific core Gateway HTTP request.

## Evidence

- **No actual-model response header exists:** A targeted search for X-Actual-Model, x-actual-model, Actual-Model, and actual-model across src, docs, packages, test, ui, and extensions returned no matches.
