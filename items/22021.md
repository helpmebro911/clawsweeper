## Summary

Keep #22021 open. Current main still does not expose `X-Actual-Model` or an equivalent per-response actual runtime model on the OpenAI-compatible Gateway HTTP surfaces. The code returns the request-side agent-target model (`openclaw` / `openclaw/<agent>`) and forwards backend overrides into the run, but does not thread the selected provider/model back to HTTP headers or response metadata.

## Best Possible Solution

Keep the issue open. Define a Gateway HTTP contract for actual provider/model metadata, thread selected runtime model data from the fallback/agent-run path into `src/gateway/openai-http.ts` and `src/gateway/openresponses-http.ts`, decide non-streaming versus SSE behavior, document it on the Gateway HTTP API docs, and add focused tests for both compatibility endpoints.
