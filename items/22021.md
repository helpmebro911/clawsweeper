## Summary

Current `main` does not implement the requested transport-layer runtime-model header. The compat HTTP handlers still take the request `model` (or request-side `x-openclaw-model` override) and echo that value back in response bodies/SSE chunks, while newer runtime-model visibility work lives in session/status/trajectory surfaces rather than HTTP response headers.

## Evidence

- **Request-side override only:** Gateway compat model override parsing is request-only: `resolveOpenAiCompatModelOverride` validates `model` and reads `x-openclaw-model` from the incoming request headers. No response header is produced there.
  - file: [src/gateway/http-utils.ts:303](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/gateway/http-utils.ts#L303)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **OpenAI compat uses requested model label:** `handleOpenAiHttpRequest` derives `model` from `payload.model` (defaulting to `openclaw`) before resolving any internal override.
  - file: [src/gateway/openai-http.ts:530](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/gateway/openai-http.ts#L530)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **OpenAI compat responses echo that label:** Both the non-stream JSON response and streaming chunks serialize the local `model` variable into the response payloads; this path does not set any runtime-model response header.
  - file: [src/gateway/openai-http.ts:607](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/gateway/openai-http.ts#L607)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **OpenResponses also echoes requested model:** `createResponseResource` stores `params.model`, and the non-stream/stream response paths pass the request `model` variable into final response objects and SSE events.
  - file: [src/gateway/openresponses-http.ts:373](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/gateway/openresponses-http.ts#L373)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Actual runtime model exists on other surfaces, not HTTP headers:** OpenClaw now tracks the actual selected/fallback model internally (`onModelSelected`) and sessions can recover runtime model data from transcript usage, but that visibility is outside the HTTP response-header surface requested here.
  - file: [src/auto-reply/get-reply-options.types.ts:13](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/auto-reply/get-reply-options.types.ts#L13)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)

## Risks / Open Questions

- HTTP clients still cannot reliably read the actual runtime model from response headers when fallbacks or mid-session model changes occur.
- Compat response bodies may continue to show the requested alias (`openclaw`, `openclaw/<agentId>`, etc.) rather than the concrete provider/model that actually ran the turn.
