## Summary

Keep #22021 open. Current main still does not expose `X-Actual-Model` or an equivalent per-response actual runtime model field on the OpenAI-compatible Gateway HTTP endpoints. The current implementation returns the requested compat model (`openclaw` / `openclaw/<agent>`) and only forwards backend overrides into the runner; actual selected provider/model metadata exists internally but is not threaded back to HTTP responses.

## Best Possible Solution

Keep the issue open. Define a Gateway HTTP contract for actual provider/model metadata, thread the selected model from `onModelSelected`, `agentMeta`, or a new run-result metadata seam into `src/gateway/openai-http.ts` and `src/gateway/openresponses-http.ts`, decide non-streaming versus SSE behavior, document it on https://docs.openclaw.ai/gateway, and add focused tests for both compatibility endpoint files.
