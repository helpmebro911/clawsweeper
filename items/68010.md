## Summary

Keep this PR open. Current main still eagerly resolves the compatibility provider env-var export at module import time, while OpenRouter's auth env var is manifest-owned. The PR is still a relevant implementation candidate, but it needs review fixes rather than cleanup closure.

## Best Possible Solution

Keep this PR open as the implementation candidate for the unresolved main-branch lazy-loading bug. Narrow or approve the production change that makes `PROVIDER_ENV_API_KEY_CANDIDATES` delegate to `PROVIDER_AUTH_ENV_VAR_CANDIDATES`, fix the regression test to assert after module evaluation and before property access, and handle the broader provider env-var security hardening as a reviewed follow-up or clearly scoped part of this PR.

- The PR branch touches provider auth env-var resolution, which can read host env secrets and feed provider authentication. The added hardening in `src/secrets/provider-env-vars.ts` is security-sensitive and should be reviewed or split from the narrow lazy-loading fix.
