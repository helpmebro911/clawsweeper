## Summary

Keep PR #71582 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still uses the old exec-event heartbeat wake routing, does not express the PR's pinned-route clears, and tests still lock in the old behavior. The provided PR context shows the author addressed Greptile's duplicate-helper review by consolidating the helper into `src/routing/session-key.ts` on the PR branch, but that implementation is not present on main.

## Best Possible Solution

Keep PR #71582 open and continue review of the latest head. The best path is to preserve the shared `scopedExecEventWakeOptions` direction from the PR, verify the pinned-route regression coverage for agent, gateway, infra heartbeat, cron, wake, and outbound routing surfaces, and run the appropriate changed gates before merge.
