## Summary

Keep PR #71582 open. Current main still uses the old exec-event heartbeat wake routing shape and does not clear pinned heartbeat delivery fields, so the PR is not implemented or obsolete. The provided PR context also shows the author addressed the Greptile duplicate-helper concern by moving exec-event wake option logic into `src/routing/session-key.ts` on the PR branch, which is useful follow-up but not present on main.

## Best Possible Solution

Keep PR #71582 open and continue review of the latest PR head. The best path is to retain the shared `scopedExecEventWakeOptions` helper, verify the pinned-route regression tests for `target`, `to`, `accountId`, `isolatedSession`, and `global`, and run the changed gates for the touched agent, gateway, infra heartbeat, cron, and outbound routing surfaces before merge.
