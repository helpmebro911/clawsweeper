## Summary

Keep this issue open. Current main still has the concrete auth-context gap described in the discussion: the embedded run path has `agentDir` and `workspaceDir` before provider stream wrapping, but `applyExtraParamsToAgent` does not forward them into the provider `wrapStreamFn` context. That means direct `openai-codex` native web search can still miss persisted auth-profile credentials and resolve as `codex_auth_missing`.

## Best Possible Solution

Keep this issue open. Review and either land or replace #67843 with a focused fix that forwards `agentDir` and `workspaceDir` into the provider stream-wrapper context, then add regression coverage for persisted `openai-codex` auth activating native Codex web search. After that merges, decide whether a narrower follow-up is needed for native-tool attachment diagnostics or planner/tool-event visibility.
