## Summary

Keep open. Current main still lacks `channels.telegram.replyContextLength`, and the existing Telegram reply code/config only covers `replyToMode`, `textChunkLimit`, and fixed quote/reply handling. The discussion workaround (`replyToMode: "all"`) is useful for enabling native replies, but it does not provide configurable quote excerpt length; the related PRs #43010 and #43734 were closed unmerged.

## Best Possible Solution

Keep the issue open until Telegram implements a typed and validated `channels.telegram.replyContextLength` setting, applies it when deriving reply quote text for bot delivery, sends that quote through `reply_parameters.quote` where Telegram supports it, and updates the Telegram docs/config metadata plus regression tests. The existing `replyToMode: "all"` workaround should remain documented as an adjacent native-reply control, not treated as a replacement for configurable excerpt length.
