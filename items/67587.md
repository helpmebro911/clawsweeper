## Summary

Current main still leaves the reported Discord mention reliability gap open. Outbound Discord sends rewrite @name text only when the per-account in-memory Discord directory cache already has a matching handle; unknown handles are deliberately preserved as plaintext, and I did not find a config-backed mention alias source, normal inbound-message cache seeding path, or merged superseding fix for deterministic agent-to-agent mentions.

## Best Possible Solution

Keep this issue open and implement the fix in the Discord plugin. A conservative path is an account-scoped and preferably guild-aware deterministic alias source such as channels.discord.mentionAliases or channels.discord.accounts.<id>.mentionAliases, with optional trusted inbound author/member cache seeding. Update the config type, schema/generated config metadata, docs, and regression tests for configured aliases, account isolation, unknown-name preservation, reserved mentions, code spans/fences, normal sends, webhook sends, and structured outbound sends.
