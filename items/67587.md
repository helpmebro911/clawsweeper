## Summary

Keep this issue open. Current main still rewrites Discord outbound `@Name` references only when the account-scoped in-memory directory cache already contains that handle; unknown handles are intentionally preserved as plaintext. I did not find a deterministic config-backed alias-to-userId map, inbound-message cache seeding, or a superseding related item/PR in the provided context or local history.

## Best Possible Solution

Implement this in the Discord plugin with an account-scoped deterministic source of mention aliases, such as `channels.discord.mentionAliases` or `channels.discord.accounts.<id>.mentionAliases`, and/or seed the existing directory cache from trusted inbound Discord author/member metadata. Add regression tests for configured aliases, account isolation, unknown-name preservation, reserved mentions, code spans, and normal/webhook/structured outbound send paths.

## Risks / Open Questions

- A fix must stay account-scoped because Discord display names and usernames can collide across guilds and bot accounts.
- Outbound rewriting must continue preserving `@everyone`, `@here`, unknown names, and markdown code spans unless the mention is explicitly configured or otherwise safely resolved.
