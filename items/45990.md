## Summary

Keep open. Current main still emits `llm_input` once before prompt submission and `llm_output` once after the embedded attempt completes, so the PR is not implemented on main. The PR also has unresolved review concerns around changing existing hook semantics and losing `llm_input` observability for failures before the first streamed `message_start`.

## Best Possible Solution

Keep the PR open for maintainer or contributor follow-up. A safe implementation should add documented per-round/per-call observability without silently changing the existing aggregate `llm_output` contract, and should preserve input-hook emission for prompt attempts that fail before streaming starts. The related open issue #64268 can inform the output side, but it does not fully supersede this PR's `llm_input` work.

- Per-round `llm_input`/`llm_output` observability for embedded agent loops remains unavailable on current main.

- Emitting `llm_input` only from `message_start` can lose trace data for auth, rate-limit, network, or pre-stream failures.
