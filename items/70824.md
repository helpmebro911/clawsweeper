## Summary

Keep open. Current main still lacks both requested behavior changes: `mergeProviderModels` overwrites explicit `input` with implicit catalog metadata, and `resolveGatewayModelSupportsImages` remains provider-scoped with coverage expecting no cross-provider image fallback. The PR is not obsolete, though its gateway fallback needs maintainer review because a review comment identified a real provider-scoping regression risk.

## Best Possible Solution

Keep the PR open for maintainer rework. The safest path is to split or narrow it: land the explicit `input` preservation in `src/agents/models-config.merge.ts` with focused regression tests, and only change gateway cross-provider image capability after proving the selected provider/routing path can actually handle the image. Otherwise keep provider-scoped capability checks and rely on text-only offload as the fail-closed fallback.
