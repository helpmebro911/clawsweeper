## Summary

Do not auto-close this PR. Current main does not contain the PR's merge-input fix, and current tests still assert the old behavior where explicit `input` is overwritten by the implicit catalog value. The cross-provider fallback portion also has an unresolved correctness concern: current main intentionally fails closed for provider-qualified lookups when only another provider declares image support, and the PR review thread flagged that broad fallback can misclassify provider-specific capabilities.

## Best Possible Solution

Keep the PR open, but ask for it to be narrowed or revised: land the explicit `input` preservation in `src/agents/models-config.merge.ts` with focused tests, and either drop the cross-provider fallback or make image routing/capability resolution prove that attachments are only passed to an actually image-capable provider. If maintainers want cross-provider merge semantics, track that as a separate gateway routing change with provider-scoped regression coverage.
