## Summary

Current main has a small public plugin-SDK gateway helper, but it does not satisfy #49178's requested reusable, platform-agnostic `@openclaw/gateway-client` package. The CLI/Node `GatewayClient` and Control UI `GatewayBrowserClient` remain separate implementations, and no dedicated gateway client workspace package exists on main.

## Evidence

- **No dedicated package on main:** The workspace includes `packages/*`, but `find packages -maxdepth 3 -type f -name package.json -print` lists only `packages/plugin-package-contract`, `packages/plugin-sdk`, and `packages/memory-host-sdk`; there is no `packages/gateway-client` or `@openclaw/gateway-client` package.
  - file: [pnpm-workspace.yaml:1](https://github.com/openclaw/openclaw/blob/d228463120a5057f7ccf9a5c31af62fe4a922088/pnpm-workspace.yaml#L1)
