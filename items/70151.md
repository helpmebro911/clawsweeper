## Summary

Keep open. Current main still has the stale-snapshot empty-turn shape described by PR #70151: the outer runner builds payloads and logs the terminal incomplete-turn error from the attempt snapshot, while the attempt result itself is derived from a single messages snapshot. I did not find an equivalent late-assistant/session-file recovery path or regression coverage on current main or the latest release v2026.4.24.

## Best Possible Solution

Keep PR #70151 open and review a focused fix in the embedded runner: before surfacing an empty-turn error, refresh the persisted session or otherwise re-resolve the latest assistant message, build payloads from that refreshed assistant, and keep the regression test for the stale-snapshot/late-assistant case.
