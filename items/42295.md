## Summary

Keep open. Current `main` still has the reported null-deref path in models-list row rendering: `toModelRow()` calls `model.input.join("+")` without guarding missing `input`, while OpenClaw docs and adjacent runtime code still treat custom-provider `input` as optional and default it to text when absent. This is not implemented on `main` yet.

## Evidence
