## Summary

Keep #12831 open. Current main has partial tool-event plumbing and live tool cards, but it still does not provide a first-class Activity panel / observer mode: Control UI tool cards remain part of the chat item stream, tool output is still coupled to `verbose=full`, verbose is patched as session state rather than per-connection state, TUI tool visibility remains verbose-gated, and related #46604 is a narrower open web split-panel request rather than a clear superseding canonical tracker.

## Best Possible Solution

Keep #12831 open as the broad observer-mode tracker. Build on the existing `tool-events` and `session.tool` plumbing, but add a dedicated Control UI activity panel with independent scroll/state, observer output visibility that is not coupled to chat `/verbose full`, a per-connection setting or documented subscription workflow, TUI split/observer support, and focused tests/docs. Treat #46604 as a narrower web split-panel follow-up unless maintainers explicitly mark it canonical.
