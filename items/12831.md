## Summary

Keep #12831 open. Current main has useful partial plumbing for tool-event delivery and live tool cards, but it still does not provide the requested first-class Activity panel / observer mode with independent UI state, per-connection verbose behavior, output visibility decoupled from `/verbose full`, or TUI observer support.

## Best Possible Solution

Keep #12831 open as the broad observer-mode tracker. The best path is to build on the existing `tool-events` and `session.tool` plumbing, add a dedicated Control UI activity panel with independent scroll/state, consume session-scoped tool events for late-joining observers, decouple observer output visibility from chat `/verbose full`, decide the per-connection/tab semantics, and add focused TUI/docs/tests. Treat #46604 as a narrower web split-panel follow-up unless maintainers explicitly mark it canonical.
