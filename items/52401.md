## Summary

Keep open. Current main does not implement `agent_error` or an equivalent modifying hook before lifecycle error broadcast. Existing `message_sending` hooks cover outbound channel delivery, while `agent_end` is observation-only and fire-and-forget after the run. This PR is a plausible core plugin-API seam, not a ClawHub plugin, but it still needs maintainer/design review because the proposed implementation touches terminal lifecycle timing.

## Best Possible Solution

Keep the PR open for maintainer review. The right path is a documented `agent_error` or equivalent plugin API with contract tests, lifecycle tests, and docs/changelog updates, implemented so plugin latency or failures cannot suppress, falsely delay, or reorder terminal lifecycle error delivery and compaction cleanup.
