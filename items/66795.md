## Summary

Keep open. Current `main` still reconstructs Telegram model-picker state from the global default model instead of the routed agent default, so the PR appears to address a real, still-unfixed bug with a targeted regression test.

## Evidence

- **Current Telegram fallback still uses global default:** In `resolveTelegramSessionState`, the no-override fallback reads `runtimeCfg.agents?.defaults?.model` directly and returns that as `sessionState.model`, instead of resolving the routed agent's default model.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:380](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/telegram/src/bot-handlers.runtime.ts#L380)
  - command: `nl -ba extensions/telegram/src/bot-handlers.runtime.ts | sed -n '340,410p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Picker checkmark comes from that session-state model:** The Telegram model-list handler passes `sessionState.model` to `buildModelsKeyboard`, and `buildModelsKeyboard` adds `✓` only when `currentModel` exactly matches `provider/model`. If `sessionState.model` is the global default, the picker will mark the wrong model for agent-specific defaults.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:1639](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/telegram/src/bot-handlers.runtime.ts#L1639)

  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Agent-aware default helper already exists:** `resolveDefaultModelForAgent` explicitly checks `agentId` and applies `resolveAgentEffectiveModelPrimary`, so the PR's replacement call matches the repo's intended contract for agent-specific defaults.
  - file: [src/agents/model-selection.ts:199](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/model-selection.ts#L199)
  - command: `nl -ba src/agents/model-selection.ts | sed -n '190,245p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)

The only posted review comment is a non-blocking style nit about calling `resolveDefaultModelForAgent` twice; it does not dispute the behavioral fix.
