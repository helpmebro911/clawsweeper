## Summary

Keep this PR open. Current main still has the Telegram model-picker fallback bug: the routed agent is used for the provider list, but the no-override current-model fallback still reads the global default, so an agent-level default can fail to get the checkmark.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this fix or an equivalent one that computes the routed agent default in `resolveTelegramSessionState()`, uses it for the final no-override fallback, keeps the regression test for an agent default that differs from the global default, and verifies the Telegram extension lane before merge.
