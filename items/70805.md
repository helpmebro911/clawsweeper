## Summary

Current main does not implement the PR’s central fix. `gateway:watch` still skips clean runtime-postbuild sync outright, and the runtime-postbuild requirement check still relies on stamps/dirty input heuristics without validating required staged outputs. The provided PR context describes a concrete Telegram failure, targeted regression coverage, and later follow-up commits for review feedback, so this is a plausible focused bug-fix PR rather than a closeable stale/duplicate item.

## Best Possible Solution

Keep the PR open. The best product/code path is to land a reviewed version of this fix, or split it if maintainers want smaller pieces: add required runtime-postbuild output validation before watch-mode launch, preserve/recover bundled plugin package metadata correctly during watch skips, verify staged runtime dependency freshness beyond stamp presence, and keep the targeted regression tests for the Telegram failure path and related runtime artifacts.
