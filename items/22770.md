## Summary

Keep #22770 open. Current main still lacks the requested external-editor compose handoff for the TUI composer: no Ctrl+E/$VISUAL/$EDITOR hook, no temp-file roundtrip, no slash-command substitute, and the public TUI docs/tests still cover only inline composition. The related #55836 remains adjacent in-place vi/vim editing work, not a duplicate or shipped substitute.

## Best Possible Solution

Implement this in the core TUI composer: add a deliberate shortcut or slash command, suspend the TUI, write current composer text to a temporary markdown file, launch $VISUAL then $EDITOR and wait, read the edited content back into CustomEditor without auto-submitting, surface clear missing-editor/non-zero-exit/file-I/O status messages, update https://docs.openclaw.ai/web/tui, and add focused CustomEditor/TUI tests. Keep #55836 separate for in-place vi/vim input mode.
