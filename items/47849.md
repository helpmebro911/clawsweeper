## Summary

Keep #47849 open. Current main still does not provide built-in multi-user Web UI login with per-user session history isolation; the remaining request is a core gateway/auth and session-visibility capability, not a conservative auto-close candidate.

## Evidence

- **Gateway auth modes are still global/auth-delegation modes:** Current main resolves gateway auth mode as one of `none`, `token`, `password`, or `trusted-proxy`; there is no built-in account/user login mode in this core auth contract.
  - file: [src/gateway/auth-resolve.ts:9](https://github.com/openclaw/openclaw/blob/d228463120a5057f7ccf9a5c31af62fe4a922088/src/gateway/auth-resolve.ts#L9)

- **User-facing configuration docs still describe shared token/password or trusted proxy:** The configuration reference says gateway auth means a shared token/password or an identity-aware reverse proxy with `gateway.auth.mode: "trusted-proxy"`; it does not document a built-in multi-user Web UI login system.
  - file: [docs/gateway/configuration-reference.md:393](https://github.com/openclaw/openclaw/blob/d228463120a5057f7ccf9a5c31af62fe4a922088/docs/gateway/configuration-reference.md#L393)
