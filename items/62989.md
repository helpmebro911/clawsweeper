## Summary

Keep this PR open. Current main has related strict-agentic/update_plan handling and a 120s LLM idle timeout, but it still does not implement the PR's central dispatch-level plan-stall fail-close or memory session-delta retry/backoff behavior. Several secondary hunks are also still absent, and the existing review comments identify real regressions that need author or maintainer handling before any landing decision.

## Best Possible Solution

Keep this PR open, request a rebase onto current main, and split it into smaller reviewed fixes. The most useful remaining slices are dispatch-level fail-closed handling for plan/control progress stalls, memory session-delta fetch-failure retry/backoff that preserves pending work, and a carefully tested memory search fallback that preserves current hybrid/vector semantics. Treat shipped strict-agentic behavior as baseline overlap, not a full replacement.
