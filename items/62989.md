## Summary

Keep this PR open. Current main includes later strict-agentic plan-only retry/blocking behavior, but it does not implement this PR’s central dispatch-level stall fail-close guard or memory session-delta retry/backoff that preserves pending work after fetch failures. Several other PR hunks are still absent or intentionally different, and the PR has unresolved review findings that should be handled before any landing decision.

## Best Possible Solution

Keep this PR open, but ask for a rebase and split/narrowing. The best path is to preserve the still-missing fixes as smaller reviewed patches: dispatch-level fail-closed handling for plan/control progress stalls, memory session-delta retry/backoff that preserves pending work, and any memory keyword fallback that maintains current hybrid/vector semantics. Treat the strict-agentic work already on main as the baseline and resolve the existing review findings before landing.
