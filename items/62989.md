## Summary

Keep this PR open. Current main has overlapping strict-agentic GPT-5/update_plan handling and a 120s idle timeout, but it still lacks the PR's central dispatch-level plan-stall fail-close and memory session-delta retry/backoff behavior. The PR also has unresolved review findings on false aborts, listener cleanup, retry backoff, memory fallback semantics, and session-state reconciliation that need author or maintainer handling before it could land.

## Best Possible Solution

Keep this PR open, request a rebase onto current main, and split it into smaller reviewed fixes. The useful slices are dispatch-level fail-closed handling for plan/control progress stalls, memory session-delta fetch-failure retry/backoff that preserves pending work, and memory search fallback that preserves current hybrid/vector semantics. Treat shipped strict-agentic behavior as baseline overlap, not a replacement for the remaining dispatch and memory fixes.
