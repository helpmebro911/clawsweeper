## Summary

Keep open. Current main has newer replay-policy architecture, but it does not implement this PR's central replay-boundary behavior: there is no OpenAI-completions thinking-signature sanitizer on main, OpenAI-specific reasoning cleanup is only applied to Responses APIs, and OpenRouter wrapped Mistral models still do not resolve `strict9` through the current provider replay hooks. The PR discussion also shows the author addressed substantive review feedback, so the remaining work is still useful even if it needs porting to the current file layout.

## Best Possible Solution

Keep the PR open, but port the remaining idea to current main's replay-policy architecture: add boundary-specific `openai-completions` thinking-signature sanitization in `src/agents/pi-embedded-runner/replay-history.ts` or a helper it calls, add regression tests for `thinking` and `redacted_thinking`, and express wrapped Mistral `strict9` handling through the current provider replay hook/model-family policy rather than the old `provider-capabilities.ts` surface.
