## Summary

Keep #71783 open. Current main repairs QMD collection name conflicts only when `collection list` exposes matching path and pattern metadata; it still does not parse the conflicting `Name: life-main (qmd://life-main/)` from the add error, so a metadata-poor `life` versus `life-main` conflict can still leave searches targeting missing `life`.

## Best Possible Solution

Keep the issue open and add a narrow memory-core repair: parse the conflicting collection name from QMD add errors like `A collection already exists for this path and pattern` plus `Name: life-main (qmd://life-main/)`, verify it matches the configured path/pattern when possible, then safely rebind it to the managed name. Add regression coverage for `life` versus `life-main` and for missing-collection search retry after boot reconciliation. If automatic rebinding is too destructive without metadata, add a targeted diagnostic and `openclaw doctor` repair path.
