## Summary

Keep #71783 open. Current main still only rebinds a different-name QMD collection when `collection list` exposes usable path+pattern metadata; the name-only/add-error-only conflict path can still log `qmd collection add skipped` and leave searches targeting the managed name (`life`) while QMD only has the stale name (`life-main`).

## Best Possible Solution

Keep the issue open and add a narrow memory-core repair: parse the conflicting collection name from QMD add errors such as `A collection already exists for this path and pattern` plus `Name: life-main (qmd://life-main/)`, then safely rebind that stale entry to the managed name when the add command path and pattern match the configured collection. Add regression coverage for `life` versus `life-main`, including the missing-collection search retry. If automatic removal/rename is too aggressive, add a targeted diagnostic and `openclaw doctor` repair instead.

## Evidence
