## Summary

Keep #71783 open. Current main still only rebinds a different-name QMD collection when `collection list` exposes usable path and pattern metadata; it does not parse the stale collection name from the add-error text shown in the issue, so the `life` versus `life-main` boot/search failure remains plausible.

## Best Possible Solution

Keep the issue open and add a narrow memory-core repair: parse the conflicting collection name from QMD add errors like `A collection already exists for this path and pattern` plus `Name: life-main (qmd://life-main/)`, verify the failed add command path and pattern match the configured managed collection, then safely remove/re-add or rename/rebind that stale entry to the managed name. Add regression coverage for `life` versus `life-main`, including the missing-collection search retry. If automatic mutation is too risky, add a targeted diagnostic and `openclaw doctor` repair instead.
