## Summary

Current main still does not implement per-agent `agents.list[].compaction` overrides. The config type and strict Zod schema only expose the requested compaction fields under `agents.defaults.compaction`, resolved agent config does not merge compaction, runtime compaction paths still read defaults-only config, and the related implementation PRs #14598 and #19329 are closed unmerged.

## Best Possible Solution

Keep the issue open. A good implementation should add optional `agents.list[].compaction` typing and strict-schema validation, reuse/extract the defaults compaction schema, resolve effective compaction by deep-merging per-agent values over `agents.defaults.compaction` including nested `memoryFlush`, thread the effective agent config through reserve-token, safeguard, timeout, memory-flush, provider, and runtime call sites, and add focused schema/resolver/runtime tests plus config docs. Closed PRs #14598 and #19329 are useful prior art but are not on main.
