## Summary

Keep #13736 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still has compaction configured and consumed through `agents.defaults.compaction` only: per-agent agent entries do not type or validate `compaction`, resolved agent config does not merge it, runtime paths read defaults-only compaction, and docs describe defaults-only configuration. Prior PRs #14598 and #19329 are useful implementation evidence but closed unmerged; #69988 is an open implementation path, not shipped behavior or an explicitly designated canonical replacement for this issue.

## Best Possible Solution

Keep #13736 open. The best implementation path is to add optional `agents.list[].compaction` to the typed config and strict Zod schema, reuse or extract the defaults compaction schema, deep-merge per-agent compaction over `agents.defaults.compaction` including nested `memoryFlush`, thread the effective per-agent config through Pi reserve-token setup, safeguard runtime setup, memory-flush/preflight behavior, and compaction docs/schema labels/help, then land focused resolver/schema/runtime tests. #69988 looks like the active candidate implementation path, but until it merges or maintainers explicitly make it canonical, this issue remains the right user-facing tracker.
