## Summary

Keep #13736 open. Current main still does not support `agents.list[].compaction`: the agent entry type and strict Zod schema omit the field, resolved agent config does not merge it, runtime compaction paths read `agents.defaults.compaction`, and docs/help labels describe defaults-only compaction. Prior PRs #14598 and #19329, plus active PR #69988, are useful implementation evidence, but the feature is not shipped on main.

## Best Possible Solution

Keep #13736 open until the feature lands or maintainers explicitly supersede it. The implementation should add optional `agents.list[].compaction` to the typed config and strict schema, reuse the defaults compaction schema, deep-merge per-agent compaction over `agents.defaults.compaction` including nested `memoryFlush`, thread the effective agent-specific config through Pi reserve-token setup, safeguard runtime setup, memory-flush/preflight behavior, and update schema labels/help/docs with focused resolver/schema/runtime tests. PR #69988 looks like the active candidate path, but it is not merged on current main.
