## Summary

Keep open. Current main still excludes BlueBubbles from the TTS voice-note channel set, so auto-TTS for BlueBubbles is generated as an audio file target rather than a voice-note target. The BlueBubbles outbound media paths also still do not forward `audioAsVoice`/`asVoice` into `sendBlueBubblesMedia`, even though the attachment layer can mark `isAudioMessage`. The reporter's linked PR #19522 correctly identified these missing routes, but the provided GitHub context says it was closed unmerged, and current main still shows the gap.

## Evidence

- **TTS voice-note eligibility excludes BlueBubbles:** `OPUS_CHANNELS` is `telegram`, `feishu`, `whatsapp`, `matrix`, and `discord`; `TRANSCODED_VOICE_NOTE_CHANNELS` is only `feishu`. `shouldDeliverTtsAsVoice` returns false unless the channel is in the native set and the target is `voice-note`, so BlueBubbles cannot get `audioAsVoice` from auto-TTS here.
