## Summary

Keep #12578 open. Current main still lacks a persistent usage-footer default under `agents.defaults`; `responseUsage` is still session-scoped, `/usage` remains the only setter, and the related implementation PRs are open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #12578 open until a focused core config change lands. The maintainer path should choose one public key under `agents.defaults`, add schema/types/generated help/docs, resolve response usage from session override first and the inherited default second, preserve explicit `off` versus clear-to-inherit semantics, and add regression tests for inherited default, explicit off, clear-to-inherit, and session override behavior.
