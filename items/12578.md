## Summary

Keep #12578 open. Current main still does not provide a persistent `agents.defaults` usage-footer default; `responseUsage` remains session-scoped, `/usage` is still the setter, docs still describe per-session persistence, and the two related implementation PRs in the issue context are open/unmerged rather than shipped.

## Best Possible Solution

Keep #12578 open until a focused core config change lands. The maintainer path should choose one public key under `agents.defaults`, add schema/types/generated help/docs, resolve response usage from session override first and inherited default second, preserve explicit `off` versus clear-to-inherit semantics, and add regression tests for inherited default, explicit off, clear-to-inherit, and session override behavior.
