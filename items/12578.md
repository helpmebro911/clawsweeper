## Summary

Current main still does not implement a persistent default for per-response usage footers. The reply runner reads only session `responseUsage`, `/usage` mutates only that session field, session creation preserves an existing session value, and `agents.defaults` schema/types/docs still lack `usageDefault`, `responseUsageDefault`, or `responseUsage`. Related PRs #48635 and #69618 are relevant possible implementations, but they remain open and are not current-main behavior.

## Best Possible Solution

Keep #12578 open until a maintainer chooses and lands the persistent usage-footer default. A focused fix should add the chosen key, likely `agents.defaults.responseUsage` or `agents.defaults.responseUsageDefault`, to schema/types/generated help/docs; apply it in the reply runner only when the session has no explicit `responseUsage`; preserve `/usage off|tokens|full` as the session override; and add tests for inheritance, explicit off, and clear-to-inherit behavior.
