## Summary

Current main still does not implement a persistent default for per-response usage footers. The reply runner resolves footer mode only from session responseUsage state, /usage still mutates the session entry, session construction only preserves an existing responseUsage value, and the agent defaults schema/types/docs do not expose a usageDefault, responseUsageDefault, or agents.defaults.responseUsage config key. The related PRs #48635 and #69618 remain relevant possible implementations, but they are not current-main behavior per the provided GitHub context.

## Best Possible Solution

Keep #12578 open until a focused implementation lands. The likely product path is to add an inherited config key such as agents.defaults.responseUsage or agents.defaults.responseUsageDefault to schema/types/generated help/docs, apply it only when a session has no explicit responseUsage override, preserve /usage off|tokens|full as the per-session override, and add tests for inherited default, explicit off, and clear-to-inherit behavior.
