## Summary

Keep PR #47984 open. Current main still does not implement `appliedTags` for Discord `channelEdit`: thread creation can set `applied_tags`, but channel/thread editing cannot parse or send it. The PR addresses a real missing capability, though it needs to be rebased/adapted to the current Discord action runtime and should preserve explicit empty arrays for tag clearing as the review discussion noted.

## Best Possible Solution

Keep the PR open and ask for a rebase/adaptation onto current main: add `appliedTags?: string[]` to `DiscordChannelEdit`, parse it in `readDiscordChannelEditParams` with explicit empty-array preservation, map it to `applied_tags` in `editChannelDiscord` using an `!== undefined` guard, and add current runtime/REST tests covering both non-empty tag updates and `appliedTags: []` clearing.
