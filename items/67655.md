## Summary

Keep this PR open. Current main still only fails closed for Windows shell wrappers when `cmdInvocation` is true, while PowerShell wrappers are parsed as shell wrappers separately; the PR is a focused fix candidate but needs a small test/docs revision before maintainer review.

## Best Possible Solution

Keep this PR open and request a small revision: block Windows shell wrappers in allowlist mode based on `isWindows && shellWrapperInvocation`, update the stale denial-message assertions, add/keep the PowerShell-wrapper regression, and align the Windows node docs/guidance to mention PowerShell wrappers before maintainer review and landing.
