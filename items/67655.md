## Summary

Keep this PR open. Current main still has the cmd-specific Windows allowlist guard, while PowerShell and pwsh are recognized shell-wrapper transports that can reach policy as `shellWrapperInvocation: true` with `cmdInvocation: false`. The PR is a focused security hardening candidate, but it needs a narrow revision before maintainer review because its denial-message change leaves existing test guidance stale and docs still mention only `cmd.exe /c`.

## Best Possible Solution

Keep this PR open and request a focused revision: block Windows shell wrappers in allowlist mode with `security === "allowlist" && isWindows && shellWrapperInvocation`, keep or cleanly remove the now-less-useful `cmdInvocation` contract, add the PowerShell/pwsh regression for `cmdInvocation: false`, update the stale message assertion and public node docs, then run the targeted node-host policy/invoke tests before maintainer review and landing.
