## Summary

Current `main` does not implement configurable typing-indicator TTL: reply orchestration still hardcodes the controller default path, the typing controller still defaults to a 2-minute TTL, and no `typingTtlSeconds` key exists in the current agent/session config schema or user-facing docs/help. The PR also still has unresolved review concerns around docs/metadata coverage and override semantics, so there is not strong evidence to close it conservatively.

## Evidence

- **Reply path does not pass a configurable TTL:** `getReplyFromConfig` resolves only `typingIntervalSeconds` and constructs `createTypingController(...)` without any `typingTtlMs` override, so current main cannot honor a `typingTtlSeconds` config even if one existed.
  - file: [src/auto-reply/reply/get-reply.ts:231](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/auto-reply/reply/get-reply.ts#L231)
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Typing controller still hardcodes a 2-minute default TTL:** `createTypingController` still defaults `typingTtlMs = 2 * 60_000`, matching the PR's stated problem and showing no shipped configurability on main.
  - file: [src/auto-reply/reply/typing.ts:28](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/auto-reply/reply/typing.ts#L28)

  - file: [src/config/zod-schema.session.ts:54](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/config/zod-schema.session.ts#L54)
