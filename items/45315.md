## Summary

Keep PR #45315 open. Current main still has the early-abort responsePrefix interpolation gap: the prefix context is created with only identity metadata, model/provider/thinking values are populated later by onModelSelected, and the fast-abort path returns before that callback can run. The PR is a focused fix for a real user-facing bug linked to #45314, but the review discussion also identifies accuracy gaps around session overrides, channel overrides, and catalog-backed thinking defaults, so maintainer review or a revised patch is still useful.

## Best Possible Solution

Keep the PR open and ask for maintainer judgment on scope. The best product fix is to ensure abort replies receive enough prefix context before normalization: at minimum pre-seed safe config defaults to avoid raw placeholders, and preferably thread session/channel/catalog context where available so {model} and {thinkingLevel} match the actual active run. Add focused regression coverage for fast abort with responsePrefix templates, plus cases for normal onModelSelected override behavior.
