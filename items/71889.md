## Summary

Keep #71889 open. Current main still contains the reported source path: setup stores the main workspace in `agents.defaults.workspace`, `agents add` uses `resolveAgentWorkspaceDir(cfg, agentId)` as the prompt default, and that resolver nests non-default agents under `agents.defaults.workspace` when it is set. The docs still describe peer-level `workspace-<agentId>` defaults, and the related fix PRs #71899 and #71941 are open/unmerged rather than shipped.

## Best Possible Solution

Keep the issue open until one narrow fix lands. The best path is to update the `agents add` interactive default so the setup-created main workspace `~/.openclaw/workspace` yields a peer-level new-agent suggestion like `~/.openclaw/workspace-<agentId>`, while preserving explicit per-agent workspaces and intentionally custom shared workspace bases. Add regression coverage for new-agent prompt defaults and existing-agent update prompts, then let the merged PR close #71889.
