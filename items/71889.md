## Summary

Keep #71889 open. Current main still seeds the interactive `agents add` workspace prompt from `resolveAgentWorkspaceDir(cfg, agentId)`, and that resolver still appends non-default agent ids under `agents.defaults.workspace`, producing the reported nested `workspace/<id>` path after normal setup writes `agents.defaults.workspace = ~/.openclaw/workspace`. The docs still describe peer-level `workspace-<agentId>` defaults, and the linked fixes (#71899 and #71941) are open rather than shipped.

## Best Possible Solution

Keep the issue open and land one narrow, tested fix, likely the #71941 shape: make `agents add` suggest `~/.openclaw/workspace-<agentId>` only for the setup-default main workspace case, preserve explicit per-agent workspaces and intentional shared-base configs, and add wizard plus resolver coverage before closing #71889.
