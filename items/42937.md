## Summary

Keep this PR open. Current main still supports global `session.dmScope` and route-binding `bindings[].session.dmScope`, but not the PR’s requested `agents.list[].dmScope` per-agent default. The PR remains a plausible core routing/config change, but it needs rework against the newer binding-level override surface and an explicit precedence rule.

## Best Possible Solution

Keep this PR open and ask for a focused rebase/rework. The best implementation would add `AgentConfig.dmScope`, strict zod validation, help/label/generated schema/docs, and route-resolution lookup for the selected agent while preserving binding-level overrides with explicit precedence. Add tests for per-agent override, global fallback, binding interaction, and route-cache behavior.
