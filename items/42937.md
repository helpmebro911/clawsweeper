## Summary

Keep this PR open. Current main still supports global `session.dmScope` and route-binding `bindings[].session.dmScope`, but it does not accept or apply the requested `agents.list[].dmScope` per-agent default. The PR remains a plausible core routing/config change and should be rebased against the newer binding-level override surface with an explicit precedence rule.

## Best Possible Solution

Keep this PR open and ask for a focused rebase/rework. The best implementation would add `AgentConfig.dmScope`, strict zod validation, generated schema/help/label/docs updates, and selected-agent lookup in route resolution while preserving binding-level overrides with explicit precedence. Add tests for per-agent override, global fallback, binding interaction, and route-cache behavior, plus documentation that calls out DM isolation implications.

## Risks / Open Questions

- The PR predates current `bindings[].session.dmScope`; a rebase needs an explicit precedence rule, likely binding override > agent override > global default.
- Allowing per-agent `dmScope: "main"` can intentionally reduce DM isolation for that agent, so docs/help and tests should make that security implication clear.
- The rework should avoid introducing another copy of the `dmScope` literal union or should centralize all schema uses to prevent drift.
