## Summary

Keep open. Current main has input provenance and external-content boundary markers, but the requested durable provenance chain is still not implemented: the plugin hook registry has no `before_persistence_write`/persistent-state hook, `before_tool_call` does not expose input provenance or action classification, and existing transcript hooks only cover tool-result/message persistence. The issue is also security/core-plugin-API design work, not a ClawHub-only plugin cleanup item.

## Evidence
