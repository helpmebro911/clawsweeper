## Summary

Keep open. Current main still builds the webchat session dropdown from persisted `sessions.list` rows plus the active session; `agents.list` is available but is only used for labels, so agents without stored sessions can still disappear. The PR addresses the right user-visible bug, but its latest submitted approach has a review-noted key-construction problem around canonical `mainSessionKey` values and should be refreshed rather than closed as implemented.

## Best Possible Solution

Keep the PR open or ask the contributor to refresh it against `ui/src/ui/chat/session-controls.ts`. The best fix is to merge configured `agents.list` entries into `resolveSessionOptionGroups()`, synthesize valid per-agent main-session keys only when no real row exists, dedupe case-insensitively against persisted rows, preserve existing labels, and add regression tests for session-less configured agents plus custom main-key/global-scope behavior. #41933 remains the canonical user-visible bug, and #41955/#48671 are possible implementation vehicles if corrected.
