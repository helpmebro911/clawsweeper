## Summary

Keep this issue open. Current main has addressed adjacent pieces, including ACP task tracking and hook `deliver:false`, but the central requested capability is still missing: headless persistent ACP sessions are still rejected unless `thread:true`, the public docs preserve that requirement, the spawn result still does not expose the ACP backend/session identity, and hook agent dispatch still runs through the isolated agent path rather than routing ACP session keys to `runTurn`.

## Best Possible Solution

Keep this issue open and narrow it to a conservative headless ACP persistence contract. The likely fix belongs in `src/agents/acp-spawn.ts` and the `sessions_spawn` tool/API docs: add an explicit headless persistent ACP mode or narrowly relax the thread guard when delivery and resume semantics are clear, expose the ACP backend/agent session identity or a documented status lookup path, and add tests for hook-triggered or channel-less `mode:"session"` ACP spawns without thread binding. A follow-up can then decide whether `/hooks/agent` should route known ACP session keys into `acpManager.runTurn`.
