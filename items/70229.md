## Summary

Keep #70229 open. Current main still unconditionally adds Docker `--security-opt no-new-privileges` for sandbox containers, and the config/types/schema/docs do not expose the requested `sandbox.docker.noNewPrivileges` opt-out. The linked bug #43996 includes a concrete Docker repro and logs, and the provided PR context shows the PR is a focused fix with tests/docs plus a later commit addressing the bot's per-agent test-coverage comment.

## Best Possible Solution

Keep the PR open for maintainer review. The right product/code path is a narrowly scoped Docker sandbox config knob with default `true`, agent/global precedence, schema/types/generated config help, dangerous-config reporting for `false`, and regression tests/docs. PR #70229 appears to be a plausible implementation path, with maintainer attention needed for the security tradeoff and final validation gates.
