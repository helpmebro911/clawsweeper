## Summary

Keep #70229 open. Current main still unconditionally applies Docker `--security-opt no-new-privileges`, does not expose `sandbox.docker.noNewPrivileges` in config types/schema/resolution, and does not report `noNewPrivileges=false` as a dangerous config flag. The linked bug #43996 has a concrete reproduction and this PR is a focused, useful fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and continue review toward a mergeable fix for #43996: add a secure-default `sandbox.docker.noNewPrivileges` config knob, wire it through schema/types/generated config docs, omit the Docker flag only when explicitly false, and surface that false value as dangerous configuration. If maintainers prefer automatic fallback instead of a config knob, that product choice should happen on this PR or the linked bug rather than closing the contribution.
