## Summary

Keep open. Current main addresses several sub-symptoms from the report, including stale plugin entries as warnings, scope-limited gateway status as degraded reachability, exec notification suppression, and disabled-plugin safeguards. But this issue is an omnibus report that still contains unresolved Windows/QClaw crash-dump upload and 1.5GB memory-growth claims, plus external skill-interceptor update behavior that is not proven fixed or covered by a canonical open tracker.

## Best Possible Solution

Keep this issue open, but split or retitle it into actionable tracks. Treat current-main gateway status, stale-config, and exec-notification behavior as mostly addressed; route WeChat and skill-interceptor behavior to their owning plugins unless a core API gap is identified; and require a current-release Windows/QClaw reproduction or downstream packaging tracker for crash-dump upload and memory-growth symptoms before closing the upstream report.
