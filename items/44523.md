## Summary

Current `main` does not appear to preserve user-selected `modelOverride`/`providerOverride` when a session rolls over because of an implicit daily freshness reset. The underlying bug described in PR #44523 still looks real, although the March patch would need rebasing onto the newer `resolveResetPreservedSelection(...)` logic added on `main`.

## Evidence

- **User-facing contract says `/model` persists the session model:** Docs state that `/model` persists the new session model immediately, so losing a user selection at the next 4 AM rollover would violate current documented behavior.
  - file: [docs/tools/slash-commands.md:194](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/docs/tools/slash-commands.md#L194)
