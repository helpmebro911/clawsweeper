## Summary

Keep open. Current main still does not implement `tools.exec.shell` or an equivalent per-agent shell override, and this PR remains a plausible implementation attempt for a real exec gap. The related #49931 issue tracks the broader product request, but the provided context and local reports do not show a merged fix or maintainer decision that supersedes this PR.

## Best Possible Solution

Keep the PR open for maintainer review or explicit consolidation into #49931 or a newer canonical PR. A complete solution should define the supported shell override contract, add typed global and per-agent config plus schema/help/docs, route the selected shell through direct exec, PTY supervisor, gateway approval follow-up runs, and node/sandbox policy decisions, then add focused regression tests for precedence, fallback, Windows arguments, validation, approval paths, and docs/schema drift.
