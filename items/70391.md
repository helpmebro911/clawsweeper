## Summary

Keep PR #70391 open. Current main still reloads chat history for every final chat event, including final events that already carry an appendable assistant message, and current tests assert that pre-PR behavior. The PR is a focused Control UI fix with relevant regression coverage, but it has review-comment edge cases that should be addressed rather than auto-closed.

## Best Possible Solution

Keep #70391 open and ask for a small revision: skip the final-event history reload only when the final payload is explicitly displayable as an assistant message, while preserving or replaying deferred `session.message` updates that may represent unrelated transcript changes. Tie the regression tests to the broader #66875 race so maintainers can decide whether to land this as a narrow fix or fold it into the canonical Control UI reload reconciliation work.
