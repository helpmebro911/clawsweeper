## Summary

Keep open. Current `main` contains only a narrow Groq workaround for `qwen/qwen3-32b`; the general reasoning-effort compatibility problem described in the issue is not fully implemented on `main`, and recent issue comments still match the remaining code path.

## Evidence

- **Groq workaround is single-model only:** `detectCompat()` special-cases Groq only when `model.id === "qwen/qwen3-32b"`, mapping all non-off efforts to `default`. There is no broader Groq or provider-generic mapping here.
  - file: [src/agents/openai-transport-stream.ts:1453](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/openai-transport-stream.ts#L1453)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Other reasoning models still send `reasoning_effort`:** The completions request builder still resolves a reasoning effort and assigns `params.reasoning_effort` for reasoning-capable models whenever compat says the provider supports it. Without a matching fallback map, non-OpenAI values still flow through this path.
  - file: [src/agents/openai-transport-stream.ts:1715](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/openai-transport-stream.ts#L1715)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Compat layer only understands OpenAI-style effort vocabulary:** `readCompatReasoningEfforts()` filters model-declared values down to the built-in OpenAI set (`none|minimal|low|medium|high|xhigh`), and `resolveOpenAIReasoningEffortForModel()` only normalizes within that vocabulary. That does not cover provider/model vocabularies like Groq `default` or Qwen/LM Studio `on`/`off` except via ad hoc maps.
  - file: [src/agents/openai-reasoning-effort.ts:15](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/openai-reasoning-effort.ts#L15)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **No targeted regression coverage for the reported Groq/Qwen cases:** A focused search for Groq/Qwen reasoning regression tests returned no matches in current test files.
  - command: `rg -n "qwen/qwen3-32b|qwen3\.6-35b-a3b|reasoning_effort.*default|groq.*reasoning" src/agents/*.test.ts extensions/**/*.test.ts`
- **Recent reporter evidence says the issue family is still live:** Issue comment on 2026-04-23 reports `qwen/qwen3.6-35b-a3b` still warning/falling back because `medium` is unsupported and only `on`/`off` are accepted, which is consistent with the remaining generic-gap in current code.
