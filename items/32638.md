## Summary

Keep #32638 open. Current main still does not preserve provider-native reasoning effort values for Groq or LM Studio/Qwen-style OpenAI-compatible providers; the code still normalizes through the OpenAI-only effort enum and can fall back to generic values such as low/medium/high instead of Groq default or LM Studio on/off.

## Best Possible Solution

Keep the issue open and fix reasoning compatibility at the provider/model contract level. The implementation should allow provider-native request values outside the OpenAI effort enum, preserve discovered LM Studio allowed_options such as on/off, apply Groq none/default handling for all affected Groq reasoning models, and add focused regression tests for Groq and LM Studio OpenAI-compatible completions.
