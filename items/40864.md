## Summary

Keep open. Current main has `sessions_send` acceptance, in-memory idempotency, agent-to-agent follow-up/announce handling, and separate outbound delivery queues, but I did not find the requested durable per-agent inbox or at-least-once retry/ack path for `sessions_send` itself. The related outbound queue work does not cover cross-session injection reliability, and the related receive-ack issue is closed stale rather than a canonical active replacement.

## Best Possible Solution

Keep this issue open and handle it as a core session-tool reliability design item. A good implementation would add a durable `sessions_send`/A2A delivery record before target dispatch, stable idempotency keys, acknowledgement or terminal failure state, startup/heartbeat retry or drain logic, and regression tests for gateway timeout/restart before and after acceptance. If maintainers prefer a plugin surface, first add an explicit core extension point for intercepting and durably acknowledging cross-session sends; the current plugin/ClawHub path alone does not satisfy the requested reliability guarantee.
