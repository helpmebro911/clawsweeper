## Summary

Keep open. Current main still has the fall-through behavior this PR targets: a core `send` result with no `toolResult` is returned as `jsonResult(result.payload)`, and the downstream agent loop treats only explicit `details.status` error/timeout tool results as failures before committing messaging-tool sends. The PR discussion also includes a concrete production Feishu observation, so this is a plausible focused bug fix rather than stale cleanup.

## Best Possible Solution

Keep the PR open and either rebase/land a narrow fix in `src/agents/tools/message-tool.ts` that converts non-dry-run core sends without a concrete delivery `messageId` into an explicit error tool result, or implement the same behavior directly on main with a regression test in `src/agents/tools/message-tool.test.ts`. The fix should preserve dry-run and plugin-handled send behavior.
