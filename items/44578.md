## Summary

Keep open. Current `main` still has the exact fallback this PR is trying to remove: `message-tool` returns `jsonResult(result.payload)` whenever `getToolResult(result)` is empty, with no send-specific check for a confirmed delivery `messageId`. I did not find the proposed guard or its regression test on `main` at `38caa6832d4e1d344736d635b0c5fdad37e4430b`.
