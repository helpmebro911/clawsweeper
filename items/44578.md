## Summary

Keep open. Current `main` still has the fallthrough this PR is trying to fix: `message-tool` returns `jsonResult(result.payload)` whenever `getToolResult(result)` is empty, and the core direct-send path can still produce a send payload with no concrete `result.messageId` under `bestEffort` / partial-failure delivery. I did not find the proposed guard or regression test on `main`.
