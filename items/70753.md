## Summary

Keep #70753 open. Current main still has the same bundled MCP URL-only path that defaults to legacy SSE, builds the SDK `SSEClientTransport` through Undici, and applies the same 30s `client.connect()` timeout reported by the issue. The provided discussion contains a concrete reproduction and a triage comment marking it a valid P1 regression; I found no shipped fix, duplicate canonical issue, or core-scope reason to close it.

## Best Possible Solution

Keep the issue open and fix the core bundled MCP remote transport path. A conservative fix should likely live around `src/agents/mcp-transport-config.ts` and `src/agents/mcp-transport.ts`: either normalize/diagnose `type: "http"` configs, prefer or retry `streamable-http` for modern `/mcp` endpoints, or harden the SSE Undici dispatcher against the reported HTTP/2 stream timeout. Add a focused regression test for URL-only vs explicit `streamable-http` behavior and, if possible, a live/manual AgentMail verification note.
