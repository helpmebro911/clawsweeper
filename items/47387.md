## Summary

Keep #47387 open. The PR discussion correctly identifies an outbound Telegram media prefetch gap: current main has proxy-aware Telegram API and inbound media download transport, but the outbound send/reply media paths still call the generic media loader without Telegram transport/proxy options.

## Best Possible Solution

Keep the PR open and request a rebase/adaptation to current main: thread the current `TelegramTransport` into outbound `sendMessageTelegram` and `deliverReplies` media loading, extend the shared media loader options if needed to carry dispatcher attempts and retry policy, and keep focused regression tests for remote media send and reply prefetch through a configured Telegram proxy.
