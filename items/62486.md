## Summary

Keep open. Current main has clear source-backed improvements for interruption/queued-message handling, including Web `/stop`, `sessions.abort`, queue modes, and QQ Bot urgent `/stop`, but the issue also asks for QQ/Web chat-history synchronization or an official recommendation. Current evidence only shows session-scoped gateway history and default/direct-session routing, not native QQ/Web history mirroring, and the reported NapCat/OneBot surface is not present in the checked-out tree.

## Evidence

- **Web stop and queue behavior exists:** The Web chat handler treats `/stop` and several stop-like inputs as aborts, and when a chat is busy it queues new non-stop messages instead of dropping them.
  - file: [ui/src/ui/app-chat.ts:67](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/ui/src/ui/app-chat.ts#L67)
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Gateway stop command aborts active runs:** `chat.send` detects stop-command text and aborts active runs for the session key, returning the aborted run IDs.
  - file: [src/gateway/server-methods/chat.ts:2112](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/gateway/server-methods/chat.ts#L2112)
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Stop behavior is covered by tests:** Tests cover `/stop` persistence metadata and `sessions.abort` for dashboard runs; multilingual stop-text recognition is also covered.
  - file: [src/gateway/server-methods/chat.abort-persistence.test.ts:197](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/gateway/server-methods/chat.abort-persistence.test.ts#L197)

  - file: [extensions/qqbot/src/engine/gateway/gateway-connection.ts:205](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/extensions/qqbot/src/engine/gateway/gateway-connection.ts#L205)
