## Summary

Keep this PR open. Current main already implements the task timestamp normalization half, but the doctor Gateway SecretRef false-positive remains present, and the related bug #65201 is still open. The PR still contains useful doctor work, but it needs a narrow revision before merge because the current diff resolves SecretRefs before confirming token auth is needed and changes the lazy doctor import shape.

## Best Possible Solution

Keep this PR open and ask for a narrow revision: rebase on current main, drop the already-implemented task timestamp portion, preserve the lazy doctor import boundary by dynamically importing resolveGatewayAuthTokenForService inside runGatewayAuthHealth, gate SecretRef resolution behind the existing token-needed decision, pass the resolved token as authOverride only for token-required paths, and keep regression coverage proving a resolvable Gateway token SecretRef suppresses the doctor false-positive without invoking SecretRef resolution for non-token auth modes.
