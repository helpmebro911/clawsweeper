## Summary

Keep open. Current main still sends Feishu `.opus`/`.ogg` media through the native `audio` message path, but it never computes or passes the `duration` field that the local Feishu upload helper says is required for audio/video uploads. The related closed Feishu media fixes cover upload/routing behavior, not this missing metadata.

## Best Possible Solution

Fix this in the Feishu plugin media path. In `extensions/feishu/src/media.ts`, compute a finite media duration for native Ogg/Opus and voice-intent transcode outputs before `uploadFileFeishu`, convert it to Feishu's expected millisecond integer, and pass it as `duration` for audio, and likely video as well because the helper marks audio/video duration as required. Add focused regression tests that `.opus` media and `audioAsVoice` transcode sends include `duration` in the `im.file.create` payload.
