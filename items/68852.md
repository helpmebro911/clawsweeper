## Summary

Keep this PR open. Current main still extracts wiki links after stripping only the managed Related block, so the reported fenced-code false positive is not implemented on main. The PR is relevant and narrowly scoped, but it still needs maintainer review or revision because the latest diff strips HTML comments before fenced code blocks, matching the live review concern that a literal `<!--` inside code can hide a valid prose link later in the page.

## Best Possible Solution

Keep the PR open as a plausible focused bug fix for `extensions/memory-wiki/src/markdown.ts`. Maintainers should ask for or make a small revision that strips fenced code before broad HTML comments, or otherwise uses a context-aware Markdown scan, then keep the added `extractWikiLinks` coverage and add the missing regression case for `<!--` inside a code fence followed by a real prose link.
