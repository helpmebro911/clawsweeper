## Summary

Keep #14785 open. Current main still builds and forwards broad tool parameter schemas, the message tool is only partially scoped by action/capability rather than by channel-specific parameters, and cron still carries tutorial-style text in its always-sent tool description. The issue discussion also contains a fresh maintainer review explicitly keeping this as the canonical parent tracker.

## Best Possible Solution

Keep #14785 open as the canonical tool-schema overhead tracker. The best path is incremental core work: avoid sending schema payloads when no effective tools remain, remove or make optional duplicated prompt tool-list text, prune message parameters by active channel/capability while preserving cross-channel jobs, move cron tutorial content out of the always-sent description, and design a compatible lazy or slim schema mode for plugin and declarative tool growth. Narrower items such as #14619 can stay linked follow-ups but do not supersede this broader issue.
