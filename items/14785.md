## Summary

Keep #14785 open. Current main still sends broad tool schemas, especially the multi-channel message schema, still includes a human-readable tool list in the system prompt, and still embeds cron tutorial-style guidance in the cron tool description. The issue discussion also explicitly treats #30359 as partial and includes a maintainer Codex review keeping this as the canonical parent tracker.

## Best Possible Solution

Keep #14785 open as the canonical tool-schema overhead tracker. The best maintainer path is incremental core work: remove or make optional duplicated tool-list text, avoid schema payloads when no effective tools remain, prune message parameters by active channel/capability without breaking cross-channel jobs, move cron tutorial content out of the always-sent description, and design a compatible lazy/slim schema mode for plugin and declarative tools. Link narrower items like #14619 as follow-ups rather than treating them as superseding this broader issue.
