## Summary

Keep this open. Current main has a related fix for the Gateway/plugin-runtime-deps side of `tslog` staging, but the reported pnpm-global `openclaw onboard --install-daemon` crash happens during classic CLI startup before command dispatch. That startup path still statically imports the logging barrel, the logger still bare-imports `tslog`, and the package/docs still support the affected Node 24 + pnpm global onboarding path.

## Best Possible Solution

Keep this issue open until maintainers either prove `openclaw onboard --install-daemon` from a packaged pnpm-global install on Node 24 no longer crashes before dispatch, or change the CLI startup/logging packaging path so bare `tslog` resolution cannot prevent onboarding from starting. The #72228 fix should stay, but it is not enough evidence to close this report.
