## Summary

Keep #68996 open. Current main already has shipped baseline Gemma 4 support from #61507, but it does not implement this PR's remaining transport-normalization concern for explicit Gemma configs pinned to `openai-completions`. The PR is recent, includes concrete local verification, and still touches a real Google provider edge case; it should be revised rather than closed as implemented or obsolete.

## Best Possible Solution

Keep the PR open, but ask for a narrower fix that preserves the shipped #61507 dynamic Gemma support, only switches explicit Gemma configs when the model is actually moved to `google-generative-ai`, patches or validates the compatible native baseUrl at the same time, and adds regression tests for both direct Google and `google-gemini-cli` template-source behavior.
