## Summary

Current `main` does not implement the requested sidecar formatted log file. The logger still writes a single JSONL file, the docs still describe file logs as JSON lines, and the main human-readable path is `openclaw logs`, which still depends on `logs.tail` or a narrow pairing-only local fallback rather than a durable formatted on-disk file when the gateway is down.

## Evidence

- **Logger still writes one JSONL file:** `buildLogger()` serializes each event with `JSON.stringify(...)` and appends only to `settings.file`; there is no derived `-formatted` companion path or second append call.
  - file: [src/logging/logger.ts:153](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/logging/logger.ts#L153)
