## Summary

Keep #32438 open. Current main still persists only the configured JSONL gateway log file; human-readable formatting exists in CLI/UI presentation paths, but no sibling `-formatted` file is derived or written. The earlier duplicate candidate #29378 is closed and PR #32442 was closed unmerged, so this issue remains the active tracker for the requested durable formatted log file.

## Best Possible Solution

Keep this issue open until maintainers either implement the logger behavior or explicitly reject the product direction. A complete fix should derive a sibling path from `logging.file`, write redacted human-readable lines whenever the JSONL file is written, preserve current JSONL consumers, define rotation and retention behavior for the sibling file, add focused logger/log-tail tests, and update the public logging docs.
