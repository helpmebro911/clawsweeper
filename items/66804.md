## Summary

Keep open. Current main has useful MiniMax and Active Memory latency mitigations, including default thinking-off behavior and MiniMax Anthropic-stream thinking disabling, but it still does not implement fallback-after-timeout for a selected Active Memory recall model. A timeout still returns `summary: null`, and the prompt hook injects no memory context. The provided discussion also has multiple recent confirmations, including OpenClaw 2026.4.22.

## Best Possible Solution

Keep this issue open unless maintainers intentionally consolidate it into #65309. The likely fix belongs in `extensions/active-memory/index.ts`: add bounded fallback-on-timeout behavior for `config.modelFallback`, switch slow MiniMax recall to a configured highspeed/dedicated recall model, or synthesize a local `memory_search`-based summary when the selected recall model times out, without extending the foreground wait beyond the configured budget. Add regression coverage for selected-model timeout fallback.
