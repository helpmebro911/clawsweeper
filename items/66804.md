## Summary

Keep open. Current main has Active Memory mitigations and MiniMax-specific latency controls, but the reported remaining behavior is still real in code: a selected recall model timeout returns `status: "timeout"` with `summary: null`, and `modelFallback` is only used during initial model selection when no explicit/current/agent model resolves, not as a retry after MiniMax-M2.7 times out. The issue also has multiple recent user confirmations, including OpenClaw 2026.4.22.

## Best Possible Solution

Keep this issue open unless maintainers intentionally consolidate it into #65309. The likely fix belongs first in `extensions/active-memory/index.ts`: add bounded fallback-on-timeout behavior for `config.modelFallback`, or produce a local `memory_search`-based summary when the selected recall model times out, without extending the foreground wait beyond the configured budget. Add regression coverage for selected-model timeout fallback and live MiniMax validation when credentials are available.
