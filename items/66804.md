## Summary

Keep open. Current `main` still uses a 15s default Active Memory timeout, still resolves `modelFallback` only when no model resolves, and still returns `status: "timeout"` without retrying another model after a timeout. The provided issue discussion also includes multiple fresh April 2026 reproductions on newer releases, so this is neither implemented nor obsolete.
