## Summary

Keep this PR open. Current main still forwards gateway `agent` `extraSystemPrompt` unchanged for inter-session `task_completion` wakes, while the subagent announce path sends those wakes with `inter_session` provenance and no explicit prompt context. Normal chat still assembles inbound/group prompt context separately, so the completion-wake prompt-context gap remains. The PR is a relevant implementation candidate, but it needs a refresh for current inbound metadata and group-context behavior before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for the inter-session `task_completion` completion-wake prompt-context gap. Refresh it onto current main, align the synthesized context and tests with `openclaw.inbound_meta.v2` and current group-context behavior, preserve explicit `extraSystemPrompt` pass-through, synthesize persisted inbound/group context only when existing session delivery metadata supports it, and run the targeted gateway tests plus `pnpm check:changed` before merge. Keep #63030 open for the broader heartbeat/cache-digest drift unless the PR grows to cover that path too.
