## Summary

Keep this PR open. Current main still forwards gateway `agent` requests with `extraSystemPrompt: request.extraSystemPrompt`, so inter-session `task_completion` wakes that omit it still do not reconstruct the session-derived Inbound Context / Group Chat Context that normal chat turns build. The PR remains a plausible implementation candidate for open issue #63030 rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate for #63030. The best path is to rebase or refresh it onto current main, keep the regression tests for synthesized persisted session context and explicit prompt pass-through, and have gateway maintainers review whether the helper matches normal chat prompt assembly across session types.
