## Summary

Keep this PR open. Current main still has the reported completion-wake prompt-context gap: gateway `agent` forwards `request.extraSystemPrompt` unchanged, subagent announce paths send `inter_session` `task_completion` wakes without an explicit prompt suffix, and normal chat remains the path that assembles `Inbound Context` and group-chat context. The PR is still a focused implementation candidate for the completion-wake portion of #63030, and the observed diff is limited to gateway TypeScript source/tests with no security-sensitive workflow, dependency, script, secret, or publishing changes.

## Best Possible Solution

Keep this PR open as the implementation candidate for the completion-wake part of #63030. Refresh it onto current main, preserve explicit `extraSystemPrompt` pass-through, synthesize persisted inbound/group prompt context only for `inter_session` `task_completion` wakes with persisted delivery metadata, keep the focused regression tests, and leave any remaining heartbeat prompt-ordering work tracked under #63030.
