## Summary

Keep open. Current `main` still drops Telegram direct-thread context when cron infers delivery from a session key, the direct-thread regression test proposed in this PR is not present on `main`, and the PR discussion itself points to unresolved correctness gaps plus a follow-up PR (`#44412`) rather than a landed fix.

## Evidence

- **Current main still strips `:thread:` and delivers to parent peer only:** `inferDeliveryFromSessionKey()` calls `stripThreadSuffixFromSessionKey()` before parsing, then builds `CronDelivery` from the remaining `peerId`. For a Telegram DM thread key like `agent:main:telegram:direct:<peer>:thread:<chatId>:<threadId>`, that logic resolves `to` as the plain direct peer instead of `<chatId>:topic:<threadId>`.
