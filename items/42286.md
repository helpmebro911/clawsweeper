## Summary

This PR adds an optional tracing plugin, but current OpenClaw direction is to keep optional capabilities out of core and publish them as community plugins. Main already exposes the public plugin seams needed for tracing hooks, CLI commands, and HTTP routes, so this belongs on ClawHub/npm rather than as a bundled `extensions/*` addition.

## Close Comment

Closing this PR after a Codex review because it fits the ClawHub/community-plugin path rather than OpenClaw core. Current project direction says optional capability should usually ship as plugins, be hosted and maintained in its own repository, and clear a high bar before being added to core (`VISION.md:54-61`). The plugin docs also say you do not need to add a plugin to this repo and should publish it to ClawHub or npm instead (`docs/plugins/building-plugins.md:16-19`). Main already exposes the public seams this tracing plugin needs: typed lifecycle hooks including `llm_input`, `after_tool_call`, `session_start`, `subagent_spawning`, `subagent_spawned`, and `subagent_ended` (`src/plugins/hook-types.ts:56-85`, `src/plugins/hook-types.ts:662-746`), plus plugin-owned CLI and HTTP registration (`docs/plugins/sdk-overview.md:92-99`, `src/plugins/types.ts:1953-1965`, `src/plugins/types.ts:2062-2068`). This looks useful, but it should be published and maintained as a community plugin on ClawHub/npm instead of landing as a new bundled `extensions/*` plugin.
