## Summary

Keep open. Current main has not implemented the PR’s three central changes: gateway shutdown still uses defaultRuntime/process.exit, model fallback still has early and same-provider cooldown probe paths, and cron has no public startupStaggerMs config. The PR is stale against current main and should not land as-is, but it remains a plausible implementation candidate that needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open, but require a rebase and narrower maintainer review before landing. The best path is to split or update the work so gateway shutdown handling is proven on the Node gateway path, cooldown changes preserve current timeout/model-scoped recovery semantics unless maintainers intentionally change that contract, cron startup staggering includes config/docs/generated-schema/test coverage, and the incidental secret allowlist change is removed or justified.
