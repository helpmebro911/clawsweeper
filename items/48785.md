## Summary

Keep open. Current main still models `tools.sessions.visibility` as the coarse string enum `self|tree|agent|all`; there is no structured per-capability object form for send/history/list/metadata. The remaining request is core session-tool authorization behavior, not an optional ClawHub/plugin cleanup candidate, and the linked same-gateway peer-send issue #48784 does not supersede this permission-model request.
