## Summary

Current main already hardens the shared chat normalization/rendering paths that would have produced this undefined `.map()` crash, and the TUI path now guards non-array content before any mapping.

## Evidence

- **UI message normalization defaults content to an array:** `normalizeMessage()` starts with `let content: MessageContentItem[] = [];` and only maps user content after that initialization, so assistant/user messages no longer carry undefined `content` into rendering.
  - file: [ui/src/ui/chat/message-normalizer.ts:264](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/ui/src/ui/chat/message-normalizer.ts#L264)

  - file: [ui/src/ui/chat/build-chat-items.ts:150](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/ui/src/ui/chat/build-chat-items.ts#L150)
