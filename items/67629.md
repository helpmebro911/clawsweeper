## Summary

Keep this issue open. Current main exposes cron job identifiers and running state in the model, Gateway schema, and CLI, but the Control UI cron job cards still do not display each job's `id` or turn `state.runningAtMs` into a visible running state. The prior ClawSweeper comment reached the same conclusion, and current source inspection at `aac83e00cfe72e5395b4cdfe77ead1c534fe7ef8` still confirms the gap.

## Best Possible Solution

Keep this open and scope it as a Control UI cron visibility enhancement. The likely fix belongs in `ui/src/ui/views/cron.ts`: render each cron job's `id`, compute an active running state from `job.state?.runningAtMs`, keep last-run status as separate history state, add concise English i18n labels, and add focused `ui/src/ui/views/cron.test.ts` coverage for running, idle, disabled, and identifier display.
