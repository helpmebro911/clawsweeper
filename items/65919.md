## Summary

Current main still does not keep run-side typing presence alive for a long quiet active run until run completion plus dispatch idle. The implementation has partial refreshes on run/tool/text/reasoning signals, but the keepalive tick itself does not refresh the internal two-minute run-side TTL. An open targeted PR (#71828) directly proposes that missing refresh and regression coverage, so this issue should remain open for maintainer review.

## Best Possible Solution

Keep this issue open and review PR #71828 or an equivalent narrow lifecycle-owned active-run presence fix. The implementation should refresh internal run-side liveness on keepalive ticks or an explicit active-run lease, preserve stale-presence cleanup, avoid new default progress/disclosure surfaces, avoid increasing provider send cadence, and stop only after run completion plus reply dispatch idle.
