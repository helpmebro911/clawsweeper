## Summary

Keep open. Current main still has the active-run typing TTL gap: dispatch-idle cleanup is present, but the run-side keepalive tick does not refresh the run-side TTL, so configured typing can still stop during a healthy long run before completion and dispatch idle. The discussion also includes a fresh Discord reproduction and an open targeted PR candidate, so the best maintainer path is to review that implementation rather than close this issue.

## Best Possible Solution

Keep this issue open and review/land PR #71828 or an equivalent active-run presence lease. The implementation should refresh the run-side presence lease while the run is active, preserve stale-presence cleanup, avoid increasing provider-facing keepalive cadence, and stop only after both run completion and dispatch idle. If reactions or channel-local progress surfaces have the same lifecycle problem, track those as follow-up work under the same internal lease shape.
