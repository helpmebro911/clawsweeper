## Summary

Keep this PR open for maintainer review or contributor rework. The core update-restart health gap is not implemented on current main: `openclaw update` still waits for restart health without the Windows unknown-listener stale-PID fallback that the shared helper supports and daemon restart already uses. However, the PR is not merge-ready as-is because it is stale and mixes the update fix with unrelated config, workflow, Matrix contract, Tlon dependency, lockfile, and secrets-baseline changes, including security/supply-chain-sensitive dependency resolution churn.

## Best Possible Solution

Keep the PR open but require a focused refresh before merge: rebase onto current main, keep only the update restart-health change plus targeted regression coverage, adapt it to the current `maybeRestartService` expected-version/package-update flow, preserve the delayed Windows fallback semantics from the review discussion, and split or drop the unrelated config, Matrix contract, workflow, Tlon dependency, lockfile, and baseline changes. Security-sensitive dependency and CI changes should be reviewed in their own PR if still needed.
