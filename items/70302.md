## Summary

Keep open. Current main still lacks the PR's requested fallback: `buildGatewaySessionRow` returns only `entry?.thinkingLevel`, so fresh/unpersisted sessions can still produce `undefined` for `sessions.list` and `sessions.changed`. The documented resolution order says per-agent `agents.list[].thinkingDefault` should apply before global/model fallback, and current UI code relies on the session row/defaults it receives rather than resolving per-agent config itself.

## Best Possible Solution

Keep PR #70302 open or land an equivalent current-main patch in `src/gateway/session-utils.ts`: preserve persisted `entry.thinkingLevel` priority, fall back to `resolveAgentConfig(cfg, sessionAgentId)?.thinkingDefault`, then the shared model/global resolver, and add focused coverage for both `listSessionsFromStore` and `loadGatewaySessionRow`/event-fed rows. The companion chat.history work in #70259 should be evaluated separately because it targets a different RPC path.
