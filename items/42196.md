## Summary

Keep this issue open. Current main still allows `openclaw cron add` without `--agent`, passes an omitted agent through as `undefined`, and cron execution still resolves missing agent ids to the configured default agent. The provided GitHub context also shows open PR #42245 with closing syntax for this issue, so maintainer policy says the issue should remain open until that PR or an equivalent fix is merged or deliberately closed.

## Best Possible Solution

Keep this issue open and review PR #42245 or an equivalent focused patch. The likely maintainer path is to warn when an agent-turn cron job is created without `--agent`, with a cron CLI regression test; if maintainers prefer stricter behavior, require `--agent` for agent-turn cron creation and update the CLI and scheduled-task docs accordingly.
