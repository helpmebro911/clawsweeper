## Summary

Keep #56693 open. Current main still has no OpenAI Codex active-workspace validation, no deactivated-workspace retry/fallback, and no regression coverage for `deactivated_workspace`. The existing fixes only cover nearby token/accountId extraction and stale-profile cases, not selecting or recovering from a deactivated ChatGPT workspace when another workspace is active.

## Best Possible Solution

Keep the issue open. The best path is a provider-owned OpenAI Codex auth hardening change: persist returned `accountId` on login/refresh, add an active-workspace validation/probe where the ChatGPT backend exposes enough metadata, classify `deactivated_workspace` clearly, retry once with a rediscovered active account when safe, persist the recovered account id, and add targeted tests for login, refresh, usage/runtime error handling, and user-facing remediation.
