## Summary

Keep open. Current `main` still has a concrete `group:openclaw` mismatch: the group is built only from tools tagged `includeInOpenClawGroup`, but `exec` is not tagged, while the docs say `group:openclaw` includes all built-in tools. That is relevant to the reported regression and there is not strong evidence on `main` that the issue is fully fixed.

## Evidence

- **`exec` is not tagged for `group:openclaw`:** In the current tool catalog, `exec` is defined without `includeInOpenClawGroup: true`, so it will not be part of the synthesized `group:openclaw` set.
  - file: [src/agents/tool-catalog.ts:83](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/agents/tool-catalog.ts#L83)

- **`group:openclaw` only includes flagged tools:** `group:openclaw` is assembled by filtering `CORE_TOOL_DEFINITIONS` to only entries with `includeInOpenClawGroup`, which excludes unflagged tools like `exec`.
  - file: [src/agents/tool-catalog.ts:337](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/agents/tool-catalog.ts#L337)
