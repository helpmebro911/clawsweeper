## Summary

Keep this PR open. Current main still validates `entry?.config` unconditionally in the non-activating snapshot and CLI metadata loader paths, and `validatePluginConfig` still normalizes `undefined` to `{}` before schema validation. That leaves the false-positive strict-schema invalid-config behavior targeted by this PR present on main. The provided PR discussion shows recent, focused follow-up work and no merged replacement.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent loader fix that skips schema validation only for non-activating loads when config is actually `undefined`, preserves normalized/defaulted `api.pluginConfig` for non-activating registration, keeps activation and explicit-config validation strict, and retains focused regression coverage in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`.
