## Summary

Keep PR #70811 open. Current main still validates `entry?.config` unconditionally in both non-activating snapshot loads and CLI metadata loads, and `validatePluginConfig` still coerces `undefined` to `{}` before schema validation. That matches the false-positive strict-schema failure described in #70658. The PR discussion shows a focused fix with follow-ups addressing the scoped `validatedConfig` regression, normalized `pluginConfig`, and schema defaults rather than work that is obsolete or superseded.

## Best Possible Solution

Keep PR #70811 open for maintainer review or land an equivalent loader fix on main. The fix belongs in `src/plugins/loader.ts`: skip or defer validation only when config is truly `undefined` in non-activating snapshot/CLI metadata paths, continue validating activation and explicit config, and preserve normalized/defaulted `api.pluginConfig`. Keep the regression coverage in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`.
