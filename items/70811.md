## Summary

Keep this PR open. Current main at 646a268d2710bc54e478349ef5690f3c8803df4f still validates omitted plugin config as `{}` in the relevant non-activating loader paths, so the linked false-positive validation bug is not implemented on main. The live PR is open, unmerged, clean/mergeable, has focused regression tests for the omitted-config cases, and #64946 is related validation noise but a different wrapped/unwrapped schema issue.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent loader change that distinguishes omitted config in non-activating metadata/snapshot loads from real activation, preserves normalized/defaulted `api.pluginConfig`, keeps activation and explicit-config validation strict, retains the focused regressions in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`, and explicitly resolves the Aisle validation-bypass concern before merge.
