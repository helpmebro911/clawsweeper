## Summary

Keep PR #70811 open. Current main still validates `entry?.config` even when it is `undefined`, because `validatePluginConfig` turns missing config into `{}` before JSON Schema validation. That validation runs before both manifest-only snapshot returns and CLI metadata registration, so the false-positive path described in #70658 is still present on main. The PR is a focused, recent bug-fix attempt with regression coverage in its branch and is not safely obsolete or superseded by the older related cleanup reports.

## Best Possible Solution

Keep #70811 open and review or land a scoped loader fix that defers validation only for non-activating loads with `entry?.config === undefined`, preserves a normalized/defaulted `api.pluginConfig` for metadata registration, and keeps real activation plus explicit-config validation strict. After that ships, close #70658; older related reports such as #62169 do not cover the current CLI metadata and manifest-only snapshot paths still present on main.
