## Summary

Keep #19289 open. Current main still does not provide an additive browser backend registration API for third-party MCP/custom drivers selectable through browser profiles. Existing support covers OpenClaw-managed CDP, remote CDP under the managed driver path, and Chrome DevTools MCP existing-session attach, but custom driver ids are rejected or normalized away across config, schema, runtime resolution, control routes, CLI/client types, docs, and Plugin SDK surfaces.

## Best Possible Solution

Keep #19289 open for maintainer design. The best implementation path is an additive browser driver extension point in core and the Plugin SDK: define a typed browser driver/adapter contract, let plugins register driver ids, broaden browser profile config/schema/help/client/CLI/control routes to accept registered ids, route profile resolution and runtime dispatch through that registry, document the public contract at https://docs.openclaw.ai/tools/browser and the plugin SDK docs, and add contract tests proving MCP/custom adapters can bind without replacing the bundled browser plugin wholesale. The closed Patchright item #52190 remains relevant as an optional whole-plugin or backend-specific discussion, but it should not supersede this core API request.
