## Summary

Current `main` does not implement a pluggable browser-backend/driver interface for third-party adapters. The bundled browser plugin can be fully replaced by another plugin, but built-in browser profile drivers remain hard-coded and schema-validated to the existing driver set, so the issue’s requested seam is still missing.

## Evidence

- **Browser profile config only allows fixed driver enum:** `BrowserProfileConfig.driver` is limited to `"openclaw" | "clawd" | "existing-session"`; there is no extensible driver registration hook in the public config type.
  - file: [src/config/types.browser.ts:7](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/config/types.browser.ts#L7)

- **Runtime schema rejects external driver ids:** Config validation hard-codes the same driver literals and requires non-`existing-session` profiles to use CDP fields, which blocks arbitrary third-party driver names.
  - file: [src/config/zod-schema.ts:393](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/config/zod-schema.ts#L393)
