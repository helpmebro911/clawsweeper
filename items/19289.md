## Summary

Keep #19289 open. Current main still lacks an additive browser backend/plugin registration interface for profile-selectable third-party MCP/custom drivers. The shipped browser support covers the bundled browser plugin, OpenClaw-managed/remote CDP profiles, and Chrome DevTools MCP existing-session attach, but config, schema, runtime resolution, control routes, client types, docs, and tests still constrain profile drivers to built-in modes.

## Best Possible Solution

Keep the issue open for maintainer design. The best path is an additive browser driver/adapter extension point in core and the Plugin SDK: define a typed browser backend contract, let plugins register driver ids, broaden browser profile config/schema/help/client/CLI/control routes to accept registered ids, route runtime dispatch through that registry, document the contract at https://docs.openclaw.ai/tools/browser and the plugin SDK docs, and add contract tests proving MCP/custom adapters can bind without replacing the bundled browser plugin wholesale. The closed Patchright item #52190 may remain a related backend-specific discussion, but it should not supersede this core API request.
