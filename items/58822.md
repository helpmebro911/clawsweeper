## Summary

Current `main` still encodes the reported regression: both the agent and cron subagent model resolvers prefer the parent agent's own model before `agents.defaults.subagents.model`, and the accompanying tests assert that behavior even though the docs describe the global subagent default as the way to override caller inheritance.

## Evidence

- **Agent resolver still uses buggy precedence:** `resolveSubagentConfiguredModelSelection` currently resolves `agentConfig.subagents.model ?? agentConfig.model ?? cfg.agents.defaults.subagents.model`, so a parent agent model shadows the global subagent default.
