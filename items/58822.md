## Summary

Keep open. Current main still resolves configured subagent models in the reported problematic order, checking the parent agent's own model before the global subagent default. The public subagent docs describe the intended behavior as inheriting the caller only unless agents.defaults.subagents.model or per-agent subagents.model is set. The issue also has a confirming user comment and a linked PR (#58823), so this is a live, actionable regression rather than cleanup-close material.

## Evidence

- **Current main still has reported precedence:** resolveSubagentConfiguredModelSelection returns agentConfig.subagents.model, then agentConfig.model, then cfg.agents.defaults.subagents.model. That means a main agent model shadows the global subagent default exactly as reported.
  - file: [src/agents/model-selection.ts](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/src/agents/model-selection.ts)
  - command: `public GitHub blob inspection of main`
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Public docs support issue expectation:** The Sub-Agents docs say the default model inherits the caller unless agents.defaults.subagents.model or agents.list[].subagents.model is set, and explicit sessions_spawn.model wins. That makes the configured global subagent model higher priority than caller inheritance.
  - file: [docs/tools/subagents.md](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/tools/subagents.md)
