## Summary

Keep open. Current main still routes `openclaw configure` through the same interactive Gateway-location Clack select before the model/auth section, and the command has no non-interactive mode/bypass flag for that step. Existing `models auth setup-token`/`paste-token` paths are useful workarounds, as noted in the issue discussion, but they do not fix the reported configure wizard input hang or provide an Anthropic browser OAuth URL from configure.
