## Summary

Keep #39223 open. Current main still routes `openclaw configure` and `openclaw configure --section model` through the Gateway-location Clack select before model/auth setup, and the configure command still has no auth-only, local, non-interactive, or skip-Gateway bypass. The direct `openclaw models auth ...` commands remain a useful documented workaround, but they do not fix the reported configure wizard path.

## Best Possible Solution

Keep the issue open until the configure path itself is robust. A good fix would detect non-interactive or unsuitable TTY conditions before the Gateway Clack select and print actionable direct-auth guidance, or add a model/auth-only configure flag/path that skips Gateway-location selection when the user only needs auth setup. Continue documenting `openclaw models auth login/setup-token/paste-token` as workarounds, not as the closing fix for this issue.
