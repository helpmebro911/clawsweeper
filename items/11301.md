## Summary

Keep #11301 open. Current main still mounts Docker/Podman runtime container workspaces only at `/home/node/.openclaw/workspace`; `/workspace` remains a sandbox-container path, not a runtime container alias. The related PR #12764 was closed unmerged in the provided GitHub context, so it is useful prior art but not a shipped fix.

## Best Possible Solution

Keep #11301 open until maintainers either implement a safe Docker/Podman runtime compatibility alias or bind mount for `/workspace -> /home/node/.openclaw/workspace` and align docs/smokes, or explicitly decide `/workspace` is sandbox-only and update runtime docs/examples to remove the ambiguity. Treat PR #12764 as prior art, not a superseding fix.
