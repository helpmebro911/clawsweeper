## Summary

Keep open. Current main still does not add the requested runtime-container `/workspace -> /home/node/.openclaw/workspace` compatibility alias. The existing `/workspace` support is for sandbox containers, and the directly related implementation PR #12764 is closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep #11301 open until a maintainer either implements a safe runtime Docker/Podman compatibility alias or bind mount for `/workspace -> /home/node/.openclaw/workspace` and aligns docs/smokes, or explicitly decides that `/workspace` is sandbox-only and updates the Docker/runtime documentation and issue expectations accordingly.
