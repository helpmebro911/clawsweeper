## Summary

Keep #11301 open. Current main still does not add a runtime-container `/workspace` alias; Docker, Docker setup, Podman, Quadlet, and live Docker helpers continue to mount the agent workspace only at `/home/node/.openclaw/workspace`. The existing `/workspace` behavior belongs to sandbox tool containers, not the gateway/CLI runtime container. The related PR #12764 was closed unmerged, so it is prior art rather than a shipped or superseding fix.

## Best Possible Solution

Keep #11301 open until maintainers either implement a safe runtime compatibility alias or second bind mount for `/workspace -> /home/node/.openclaw/workspace` across Dockerfile, Compose, setup helpers, Podman/Quadlet, and relevant smokes/docs, or explicitly decide that `/workspace` is sandbox-only and update docs/examples to remove runtime-container ambiguity. Treat PR #12764 as useful prior art, not a canonical fix.
