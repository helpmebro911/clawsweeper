## Summary

Keep this PR open. Current main still suppresses text-only tool result payloads for ordinary group chats without using the persisted session verbose level, and the PR discussion includes a concrete DingTalk AI Card plugin use case. The branch needs rework before merge because current main now expects native command tool summaries/media delivery, while the PR diff preserves native suppression.

## Best Possible Solution

Keep this PR open as a plausible focused fix, but require a rebase and implementation update against current `src/auto-reply/reply/dispatch-from-config.ts`. The best path is to let explicit session verbose `on` or `full` opt ordinary non-Slack group chats into text tool summaries while preserving existing forum, Slack non-DM, sendPolicy, media-only, exec-approval, preview-suppression, and current native-command behavior unless maintainers intentionally change those contracts. Add focused regression tests and coordinate the broader config-option discussion in #54663.
