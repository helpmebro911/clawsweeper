## Summary

Keep open. Current main still keeps the WebChat pending queue and draft in component memory, so the user-visible problem is not implemented on main. PR #71336 is an active, focused attempt to address the queue part after the author dropped the unrelated session-store changes noted in review; it should get maintainer review rather than cleanup closure. The patch is not merge-ready yet because the current PR diff appears to import `flushChatQueue` from the wrong module and lacks regression coverage.

## Best Possible Solution
