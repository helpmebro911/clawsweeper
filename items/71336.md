## Summary

Keep PR #71336 open. Current main still keeps the WebChat follow-up queue in component memory, clears it on session switch, and has no chat queue localStorage persistence module or restore path. The PR is recent, contributor-authored, and responds to maintainer feedback by dropping the unrelated session-store changes in favor of the still-open WebChat persistence bug #51549, so it is not conservative cleanup-close material.

## Best Possible Solution

Keep #71336 open as the active focused WebChat queue-persistence implementation path. The best product/code direction is to require a clean UI-only patch that persists queued follow-ups with the existing safe storage helper, restores per-session queue state on reconnect/session switch, drains only when idle, removes empty/stale entries, and adds targeted unit/integration coverage; track any remaining #51549 draft/history scope separately if this PR intentionally stays queue-only.
