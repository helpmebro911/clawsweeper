## Summary

Keep #2597 open. Current main still lacks the requested passive agent-visible `context=X%` field in the system-prompt Runtime line. Existing `/status`, `session_status`, `/context`, UI usage indicators, and the community `openclaw-workspace-sync` plugin help with inspection or recovery, but they do not implement the requested per-turn prompt primitive. The latest maintainer-facing comment also explicitly kept the issue open.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded context-usage plumbing from the latest reliable session token snapshot into embedded-runner params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused tests/docs, or a maintainer explicitly decides that `/status`, `session_status`, `/context`, UI warnings, context/memory plugins, and `openclaw-workspace-sync` are the intended product boundary.
