## Summary

Keep #2597 open. Current main has explicit context visibility in `/status`, `session_status`, `/context`, usage/UI surfaces, and a community `openclaw-workspace-sync` recovery plugin, but it still does not implement the requested passive agent-visible `context=X%` field in the system-prompt Runtime line. The latest maintainer-facing discussion also explicitly kept the issue open.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded context-usage plumbing from the latest reliable session token snapshot into embedded-runner params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused tests/docs, or a maintainer explicitly decides that explicit `/status`, `session_status`, `/context`, UI warnings, context/memory plugins, and `openclaw-workspace-sync` are the intended product boundary.
