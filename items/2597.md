## Summary

Keep #2597 open. Current main has meaningful state-loss mitigations, including pre-compaction memory flush, reset hooks, `/status`/`session_status` context reporting, and the discussion points to the useful `openclaw-workspace-sync` community plugin workaround. But the requested passive agent-visible `context=X%` field in the Runtime line is still not implemented on main.

## Best Possible Solution

Keep the issue open until OpenClaw either implements bounded `contextPercent` plumbing from session usage into `RuntimeInfoInput` and `buildRuntimeLine()`, or a maintainer explicitly decides that context pressure should remain available only through `/status`, `session_status`, memory flush, hooks, or plugin workflows.
