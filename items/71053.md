## Summary

Keep #71053 open. Current main has an internal trusted env-proxy fetch mode and helper plumbing, but `web_fetch` still does not expose or read the requested `tools.web.fetch.useEnvProxy` config toggle. The linked PR #71056 is the active implementation attempt, but it is still open and unmerged, so the issue is not implemented or safely obsolete.

## Best Possible Solution

Keep the issue open and use PR #71056, or a maintainer-owned replacement, as the concrete path to add `tools.web.fetch.useEnvProxy` across config types, runtime schema, docs, and `createWebFetchTool` wiring while preserving the existing ALL_PROXY/NO_PROXY SSRF gating tests.
