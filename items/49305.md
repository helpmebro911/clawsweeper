## Summary

Keep open. Current main has adjacent WhatsApp lifecycle improvements, including status visibility and the `channels.start` gateway handoff after CLI login, but the reported logged-out monitor path still stops without clearing WhatsApp auth or invoking the gateway logout lifecycle. Because WhatsApp configuration is still derived from persisted auth state, stale credentials can remain enough for the gateway to keep trying the dead account instead of moving cleanly to an unlinked/relink state. The April 21 issue comment is therefore still materially accurate, and the open #67816 follow-up is adjacent timeout/status handling rather than a proven fix for this monitor cleanup gap.
