## Summary

Keep PR #66415 open. On target `origin/main` (`cf834e2a21eac7fd218989fb5b85e7e8dd4c3246`), the central non-strict OpenAI-compatible tool-schema behavior is still not implemented: non-strict tool parameters still return the generic normalizer output, which preserves object schemas without adding explicit `required: []`. The PR is focused, has regression coverage, and its discussion shows the author addressed the bot-reviewed concern about mutating literal objects under `enum`, `const`, `default`, and `examples`.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or adapt the OpenAI-specific non-strict normalization behind `normalizeOpenAIStrictToolParameters`, limited to schema-bearing keywords, with regression coverage for top-level and nested object schemas plus literal objects under `enum`, `const`, `default`, and `examples`. Keep generic non-OpenAI schema normalization and strict-mode behavior unchanged.
