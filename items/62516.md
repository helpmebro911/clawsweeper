## Summary

Keep #62516 open. Current main still has the same meaningful behavior gap for the reported configuration: `tools.profile: "coding"` is applied as an early allowlist, only `alsoAllow` is merged into that profile stage, and later `agents.list[].tools.allow` cannot re-add plugin/browser tools after profile filtering. The docs and tests document `alsoAllow` as the workaround, but that does not implement the reporter's requested/observed expectation for `tools.allow` under a profile.
