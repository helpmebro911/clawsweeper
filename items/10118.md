## Summary

Keep #10118 open. Current main partially supports multiline composition through the pinned `pi-tui` editor, but the user-facing acceptance criteria are still incomplete: TUI docs and `/help` do not mention Shift+Enter/newline composition, OpenClaw exposes no config or CLI bridge for configurable submit/newline keybindings, and related PR #43327 is still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and narrow it to the remaining shipped work: document newline composition and terminal limitations in `docs/web/tui.md` and TUI `/help`, add focused assertions for shortcut/help visibility, and decide whether OpenClaw should expose configurable submit/newline keybindings through config or CLI wiring to `pi-tui`'s keybinding layer. Treat PR #43327 as related context until it is merged, updated, or explicitly superseded.

## Risks / Open Questions

- Terminal Shift+Enter support is terminal-dependent, so docs/help should qualify the shortcut and mention reliable alternatives or fallbacks rather than promise universal behavior.
- Configurable Enter-vs-Ctrl/Cmd/Shift+Enter behavior needs product/API judgment because it would bridge OpenClaw config/CLI into `pi-tui` keybindings and may affect submit, autocomplete, and overlay behavior.
- Related PR #43327 is useful prior work but remains open/unmerged, so closing this issue now would lose the remaining tracked acceptance criteria.
