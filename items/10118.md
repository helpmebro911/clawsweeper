## Summary

Keep #10118 open. Current main already gets the basic Enter-submit and Shift+Enter-newline behavior from pinned pi-tui, and OpenClaw preserves submitted multiline text, but the issue also asks for docs/help visibility and optional configurable send/newline behavior. Those remain missing on current main, and a maintainer comment already explicitly kept this open for the same remaining scope.

## Best Possible Solution

Keep the issue open and narrow the next change: document Shift+Enter in docs/web/tui.md and slash /help, add focused tests for the visible shortcut text, and make an explicit maintainer product decision on configurable Enter-vs-Ctrl/Cmd+Enter behavior using pi-tui's keybinding surface or an OpenClaw config-backed wrapper. Treat PR #43327 as related but not canonical while it remains open and unmerged.
