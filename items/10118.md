## Summary

Keep #10118 open. Current main partially covers the core shortcut through the pinned pi-tui editor, but the issue's remaining user-facing acceptance criteria are still missing: TUI docs and /help do not mention Shift+Enter, there is no OpenClaw config/CLI bridge for configurable send/newline behavior, and related PR #43327 is still open and unmerged.

## Best Possible Solution

Keep the issue open and narrow it to the remaining shipped work: document Shift+Enter/newline composition and terminal limitations in docs/web/tui.md and TUI /help, add focused assertions for shortcut/help visibility, and decide whether OpenClaw should expose configurable submit/newline keybindings through config or CLI wiring to pi-tui's keybinding layer. Treat PR #43327 as related context until it is either updated and merged or superseded.

## Risks / Open Questions

- Terminal-specific Shift+Enter support depends on keyboard protocol support, so docs should qualify the shortcut rather than promise universal behavior.
- Configurable Enter-vs-Ctrl/Cmd+Enter behavior is a product decision and likely needs explicit OpenClaw config/CLI wiring to pi-tui keybindings.
- PR #43327 is useful related prior work, but it is open/unmerged and its patch does not represent current main behavior.
