## Summary

Keep open. Current `main` still does not implement the stale-history reset this PR proposes: `beforeToolCall` still runs `detectToolCallLoop` before `recordToolCall`, `detectToolCallLoop` reads `state.toolCallHistory` with no age-based purge, and diagnostic session state retains `toolCallHistory` for up to 30 minutes while idle. I also found no landed `staleThresholdMs` config/schema/docs surface on `main`, so this PR is not superseded by an equivalent implementation.

## Evidence

- **Production call order still checks before recording:** `beforeToolCall` calls `detectToolCallLoop(...)` first and only then `recordToolCall(...)`, so any cross-cycle cleanup must happen before or inside detection to affect the first call of a new cycle.
  - file: [src/agents/pi-tools.before-tool-call.ts:146](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/agents/pi-tools.before-tool-call.ts#L146)
  - command: `nl -ba src/agents/pi-tools.before-tool-call.ts | sed -n '130,210p'`
- **No stale-history purge in loop detection:** `detectToolCallLoop` loads `const history = state.toolCallHistory ?? []` and evaluates streaks/counts from it directly. There is no timestamp or staleness gate before the history is consumed.
  - file: [src/agents/tool-loop-detection.ts:438](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/agents/tool-loop-detection.ts#L438)
  - command: `nl -ba src/agents/tool-loop-detection.ts | sed -n '360,700p'`
- **Session state can preserve tool history across heartbeat gaps:** Diagnostic session state stores `toolCallHistory` on the session object and only prunes idle sessions after `SESSION_STATE_TTL_MS = 30 * 60 * 1000`, which is long enough for repeated heartbeat cycles to reuse the same history.
  - file: [src/logging/diagnostic-session-state.ts:9](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/logging/diagnostic-session-state.ts#L9)
  - command: `nl -ba src/logging/diagnostic-session-state.ts | sed -n '1,220p'`
- **No landed config/schema/docs support for a stale threshold:** Current `ToolLoopDetectionConfig`, runtime Zod schema, and loop-detection docs expose `enabled`, `historySize`, thresholds, and detectors only; there is no `staleThresholdMs` or equivalent field on `main`.
  - file: [src/config/types.tools.ts:165](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/config/types.tools.ts#L165)
  - command: `nl -ba src/config/types.tools.ts | sed -n '145,190p'; nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '500,509p'; nl -ba docs/tools/loop-detection.md | sed -n '21,75p'`
