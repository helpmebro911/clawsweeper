## Summary

Keep open. Current main at 8bbb143ab87e0a45f9d3f986768d75168750b672 still drops native Error message/stack/cause from stability bundles: the bundle error type only contains name/code, the writer only reads name/code, and existing tests explicitly assert startup error text is absent. Related PR #71155 is still open, while #71092 and #71352 were closed unmerged, so the issue is not implemented on main.

## Best Possible Solution

Keep #71071 open and land a focused fix in src/logging/diagnostic-stability-bundle.ts that preserves Error.message and Error.stack, and likely bounded/redacted cause metadata, in writer-produced bundles while keeping imported-bundle parsing conservative. Add regression tests for gateway.startup_failed and gateway.restart_startup_failed bundles that verify actionable message/stack preservation plus redaction/truncation behavior.
