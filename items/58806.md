## Summary

Keep open. Current `main` still does not pass `requesterSenderId` or `senderIsOwner` into `ChannelAgentToolFactory`, so the requested channel-plugin seam is not implemented yet.

## Evidence

- **Factory type is still cfg-only:** `ChannelAgentToolFactory` on current `main` still accepts only `{ cfg?: OpenClawConfig }`, with no requester identity or owner flag in the public contract.
  - file: [src/channels/plugins/types.core.ts:32](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/channels/plugins/types.core.ts#L32)

## Risks / Open Questions

- Closing this now would hide a real plugin-SDK gap: channel plugins still cannot build sender-aware `agentTools` closures from trusted requester identity on current `main`.
- Because bundled plugins do not appear to use this seam today, the gap can stay latent until third-party or new bundled channel plugins need per-sender access control.
