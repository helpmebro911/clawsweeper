## Summary

Keep #69060 open. Current main still does not implement `x-openclaw-session`; the OpenAI-compatible HTTP session resolver only honors `x-openclaw-session-key`, the OpenAI `user` field, and a random UUID fallback. The linked broader issue #68920 remains open, but this PR is a focused, plausible core gateway fix for the stable-session portion rather than an obsolete duplicate.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to implement an agent-scoped `x-openclaw-session` branch in `resolveSessionKey`, preserve the current priority order (`x-openclaw-session-key` > `user` > new stable hint > random UUID), add focused OpenAI HTTP regression tests for priority and agent scoping, and update the public OpenAI HTTP API docs if merged. Keep #68920 open for the larger `lightContext`/voice-mode work.
