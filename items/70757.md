## Summary

Keep this PR open. Current main still uses the BlueBubbles API password as the inbound webhook auth secret, while this PR is the active implementation candidate for the linked security issue #70755 and its review comments were followed up in the PR head.

## Best Possible Solution

Keep this PR open, or land an equivalent maintainer-approved fix. The right path is to add a dedicated BlueBubbles webhookSecret surface, authenticate inbound BlueBubbles webhooks only against that secret, enforce that it differs from the API password across inherited account configs, expose it in setup/CLI metadata, and update docs, tests, and SecretRef metadata consistently.
