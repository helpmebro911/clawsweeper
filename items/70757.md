## Summary

Keep PR #70757 open. Current main still uses the BlueBubbles API password as the inbound webhook authentication secret, the config/setup surfaces do not expose a distinct webhookSecret, and the public docs still instruct operators to put the password in the webhook URL. The PR remains a plausible focused fix for linked issue #70755 rather than an obsolete or closeable item.

## Best Possible Solution

Keep #70757 open and review/land a focused version or equivalent fix that adds a distinct BlueBubbles webhookSecret across runtime auth, config schemas, setup/CLI metadata, secret registry, docs, and regression tests. After that ships on main, close linked issue #70755 with the fixing commit and release evidence.
