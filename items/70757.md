## Summary

Keep this PR open. Current main still uses the BlueBubbles API password as the inbound webhook credential, and this PR is the active same-author implementation candidate for the linked security bug #70755. The PR diff is security-sensitive but aligned with the stated fix, and I did not see unexplained workflow, dependency, lockfile, package-resolution, or script-execution expansion.

## Best Possible Solution

Keep this PR open for maintainer review, or land an equivalent maintainer-approved fix. The best product path is to add a dedicated BlueBubbles webhookSecret, authenticate inbound webhooks only against that secret, enforce that it is present and distinct from the API password across top-level and inherited account configs, expose it through setup/CLI metadata and SecretRef surfaces, and update docs plus regression tests. After merge, close #70755 as implemented.
