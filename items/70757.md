## Summary

Keep open. Current main still uses `channels.bluebubbles.password` as the BlueBubbles webhook auth token, still documents publishing that password in the webhook URL, and lacks the `webhookSecret` config/setup surface that PR #70757 proposes. The PR directly addresses linked issue #70755 and the provided review context shows the author followed up on bot review gaps, so this is a plausible security-boundary fix rather than cleanupable stale work.

## Best Possible Solution

Keep PR #70757 open. The best path is a maintainer/security review of the PR's dedicated `webhookSecret` implementation, ensuring runtime auth, config schema, setup CLI/wizard, secret metadata, docs, and regression tests all land together and that the addressed review comments remain resolved.
