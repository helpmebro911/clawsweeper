## Summary

Keep open. Current `main` has a plugin-side `gateway_stop` hook and a gateway-client `shutdown` event, but it still does not provide the requested internal `gateway:shutdown` / `gateway:pre-restart` hook events, and the follow-up `activeTurns` ask is not exposed on `sessions.list` or the session list payload.

## Evidence

- **Internal hooks docs only advertise startup:** The internal hooks event table lists `gateway:startup` but no shutdown or pre-restart lifecycle event.
  - file: [docs/automation/hooks.md:36](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/automation/hooks.md#L36)
  - command: `nl -ba docs/automation/hooks.md | sed -n '36,56p'`
  - sha: [7e16e3d07723](https://github.com/openclaw/openclaw/commit/7e16e3d07723cc3903f40ce94f53a3920c88b1fc)
- **Gateway only dispatches internal startup hook:** Gateway startup explicitly creates and triggers `gateway:startup`; there is no matching internal shutdown dispatch in this lifecycle path.
  - file: [src/gateway/server-startup-post-attach.ts:268](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/gateway/server-startup-post-attach.ts#L268)

  - file: [src/gateway/server.impl.ts:909](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/gateway/server.impl.ts#L909)
