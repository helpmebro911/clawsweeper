## Summary

Keep open. Current main still contains the cross-agent target-agent binding lookup described by #70574, and the linked issue has a concrete Telegram confirmation. The PR is not merge-ready as written because the `!==` to `===` change would re-enable binding lookup for same-agent spawns and can overwrite the active requester account, contradicting current same-agent coverage and the surrounding invariant.

## Best Possible Solution

Keep #70607 and #70574 open, but revise the implementation. The likely direction is to preserve the caller/requester account for cross-agent completion routing while retaining the same-agent invariant that avoids binding re-resolution; add regression coverage for the Telegram-style parent-account route and keep or explicitly adjust the Matrix thread-bound account behavior.
