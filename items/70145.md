## Summary

Keep open. I did not find a high-confidence close reason: current main still contains the same initialization gate that emits the reported 503 before slash-command state is active, the docs still say a healthy initialized endpoint should answer GET with 405, and I found no Mattermost slash-command changes between the latest release snapshot provided and current main that would prove this was already fixed.
