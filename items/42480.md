## Summary

Current main still has the Discord debounced synthetic-message bug this PR is trying to fix. The PR is useful but stale against the current plugin layout and needs a port plus regression coverage, not closure.

## Best Possible Solution

Keep this PR open as useful bug-fix work. Port the mention accumulation into `extensions/discord/src/monitor/message-handler.ts`, preserve `mentionedUsers`, `mentionedRoles`, and `mentionedEveryone` across all debounced entries, add a regression test for earlier-entry mention metadata in the Discord handler queue/preflight coverage, then run the relevant Discord plugin test lane before landing.
