## Summary

Keep this PR open. Current main still has the Discord debounced synthetic-message path that spreads the last Carbon Message into a plain object without materializing mention metadata, while preflight still depends on mentionedUsers, mentionedRoles, and mentionedEveryone for mention gating. The PR addresses the right bug after its follow-up commit, but it targets the old src/discord path and needs to be ported to the current Discord plugin path with regression coverage before landing.

## Best Possible Solution

Keep this PR open as useful bug-fix work. Port the all-entry mention accumulation into extensions/discord/src/monitor/message-handler.ts, preserving mentionedUsers, mentionedRoles, and mentionedEveryone on the synthetic message, then add a focused Discord debounce/preflight regression test where mention metadata appears before the last debounced entry before landing.
