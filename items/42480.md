## Summary

Current `main` still appears to have the reported Discord debounce mention-loss bug. The debounced path in `extensions/discord/src/monitor/message-handler.ts:162-182` builds a synthetic message with object spread from `last.data.message` and does not materialize `mentionedUsers`, `mentionedRoles`, or `mentionedEveryone`. Preflight still relies on those fields for mention gating in `extensions/discord/src/monitor/message-handler.preflight.ts:709-716`, while `resolveDiscordMessageText()` rewrites `<@id>` mentions to `@name` before preflight in `extensions/discord/src/monitor/message-utils.ts:632-658`, and hydration only retries when raw mention syntax remains in text in `extensions/discord/src/monitor/message-handler.preflight.ts:327-342`. I found no equivalent fix on current `main`.
