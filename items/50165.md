## Summary

Keep open. Current main has a strong partial fix for premature upstream completion when descendant subagents are still pending, shipped in v2026.4.24, but the broader issue report and follow-up comment also cover background exec / external work still running after a subagent is marked done. Current subagent status surfaces still resolve completion from endedAt plus pending descendant subagents only, and background exec completion is handled through a separate process/system-event path rather than keeping the subagent itself running.
