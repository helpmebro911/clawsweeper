## Summary

Keep open. Current `main` still has the same unguarded Anthropic `input_json_delta` parsing path described in #59076, and that path still turns thrown parse errors into fatal assistant errors that channel payload building can surface to users. I do not see strong evidence of a fix on `main`, so conservative maintainer cleanup should not close this issue.

## Evidence

- **Anthropic partial tool-call JSON is still parsed eagerly without a guard:** `src/agents/anthropic-transport-stream.ts:910`-`917` appends each `input_json_delta` chunk and immediately calls `parseStreamingJson(block.partialJson)`, and `src/agents/anthropic-transport-stream.ts:959`-`962` parses again on `content_block_stop`. There is no local `try/catch` around either parse.
  - file: [src/agents/anthropic-transport-stream.ts:910](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/anthropic-transport-stream.ts#L910)

  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)

That matches the reporter's later comment that the session moved to failed/error state.
