## Summary

Keep PR #44288 open. Current main still does not implement backup exclude patterns, --smart-exclude, .backupignore loading, protected-path guards, or excludedStats manifest support. The PR is recent and directly targets open backup-size/stall reports plus #40786; it has useful follow-up work after several review fixes, but one remaining semantics question around assets outside stateDir still needs maintainer review rather than auto-close.

## Best Possible Solution

Keep #44288 open and continue review as the candidate implementation for backup exclude patterns. The best path is to land a focused, CI-green core backup change that registers the exclude CLI flags, applies gitignore-style filters safely to all intended backup roots, records pattern-level manifest stats without leaking filenames, and either fixes or explicitly tracks the external-asset exclude semantics raised in review.
