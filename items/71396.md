## Summary

Keep #71396 open. Current main still contains the Feishu automatic mention cascade that the PR targets: incoming `mentionTargets` are extracted, passed into reply dispatch, then prepended to first plain/card replies and streaming-card closes as executable Feishu mention markup. The linked issue #70065 is closed only because this PR is the canonical active fix, not because main already implements it. The PR still has unresolved review/security concerns that need maintainer handling before merge.

## Best Possible Solution

Keep #71396 open and finish the Feishu plugin fix there or with an equivalent patch: stop unconditional propagation of inbound `mentionTargets` into fallback/error/ack/ordinary replies, keep mention context safe for the agent, and preserve explicit mention behavior through structured allowlisted conversion or sanitization with regression coverage for plain text, card, streaming-card, and fallback paths.
