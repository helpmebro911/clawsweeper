## Summary

Keep this PR open. Current main still has the Feishu mention-cascade behavior: inbound `mentionTargets` flow into reply dispatch, and the dispatcher still attaches/prepends them to outbound replies. The PR is an active implementation candidate for the linked Feishu bug, with recent author follow-up and remaining review/security questions rather than an obsolete cleanup candidate.

## Best Possible Solution

Keep the PR open and have the Feishu plugin owner/maintainer review the latest head. The best path is to land or revise this bundled Feishu plugin fix so fallback, error, ack, and ordinary replies stop automatically cascading inbound mentions, while intentional explicit mentions remain supported where the Feishu card/text formats require conversion and the mention context hint is safely bounded/escaped.
