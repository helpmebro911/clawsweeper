## Summary

Keep this PR open. Current main still does not have the proposed main-specific host-level OpenClaw self-inspection boundary, and the PR remains a plausible but security-sensitive implementation candidate with unresolved bypass concerns.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. The best path is to decide whether main should get this narrow self-inspection exception, then revise the implementation to use parser/segment-based detection instead of a command regex, make absolute path handling cross-platform, reject all mutating doctor aliases and flag forms, block mixed shell and pipeline forms, and add focused tests for those bypass cases before merge.
