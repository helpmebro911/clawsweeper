## Summary

Keep open. Current `main` still requires `agents.defaults.cliBackends.<id>.command` in the TypeScript config type, Zod schema, and generated base schema, so a modelAliases-only override remains unsupported. The PR addresses a real gap, but the provided review discussion also identifies an unhandled generated-schema update before it can be considered complete.

## Best Possible Solution

Keep PR #69123 open and ask for a focused update: make `CliBackendSchema.command` and `CliBackendConfig.command` optional for input partials, preserve a resolved-backend type or guard with `command: string` after `resolveCliBackendConfig`, add the modelAliases-only regression test, and regenerate/check the base config schema so schema-driven consumers accept the same partial override contract.
