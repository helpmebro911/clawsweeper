## Summary

Keep #26037 open. Current main has bundled Qwen/Coding Plan support, but it still does not officially enable Bailian/Qwen Coding Plan thinking for the requested qwen3.5-plus, glm-5, and kimi-k2.5 path.

## Best Possible Solution

Keep the issue open and implement this in the bundled Qwen provider plugin: verify Bailian Coding Plan thinking semantics for qwen3.5-plus, glm-5, kimi-k2.5, and current Coding Plan models; add Qwen-owned reasoning metadata plus the correct enable_thinking/thinking-budget payload adapter; document the setup at docs.openclaw.ai/providers/qwen; and add tests for catalog discovery, payload construction, config validation, and the reported WebUI/gateway restart path.
