## Summary

Keep #26037 open. Current main has bundled Qwen/Coding Plan support and generic pass-through config, but it still does not provide official Bailian/Qwen Coding Plan thinking support for the requested qwen3.5-plus, glm-5, and kimi-k2.5 path.

## Best Possible Solution

Keep the issue open and implement this in the bundled `qwen` provider plugin: verify Bailian Coding Plan thinking semantics for qwen3.5-plus, glm-5, kimi-k2.5, and MiniMax-M2.5; add Qwen-owned reasoning metadata and a provider thinking profile; map OpenClaw `/think` levels to the correct `enable_thinking` and budget payload fields; document the setup on `docs.openclaw.ai/providers/qwen`; and add Qwen-owned tests for catalog discovery, payload construction, config validation, and the reported WebUI/gateway restart path.
