## Summary

Keep open. Current main still maps 403 HTML provider responses to the generic auth HTML failure, does not expose a Cloudflare challenge-specific failure kind or browser-like TLS profile, and does not document the cloudscraper/reverse-proxy workaround. PR #67717 is an open implementation candidate with closing syntax, so this issue should remain open until that PR is reviewed and either merged or closed.

## Best Possible Solution

Review PR #67717 as the narrow fix for the misleading 403 HTML diagnostic. If it is correct, land it with the regression tests and then close this issue as implemented. Treat a built-in browser TLS profile or challenge-bypass transport as separate maintainer-scoped design work, and document a safe workaround if maintainers choose not to ship transport emulation.
