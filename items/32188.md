## Summary

Keep #32188 open. Current main still has useful related pieces, including chat session/model/thinking selectors, transient fallback/compaction/context notices, per-message usage metadata, slash-command usage reporting, overview auth/quota cards, and session-table data, but it does not implement the requested persistent in-interface status area showing the active agent/model plus live token/quota/subscription context during operations. No relevant Control UI files changed after the prior ClawSweeper keep-open review.

## Best Possible Solution

Keep the enhancement open. The best product path is a first-class persistent status surface in the Control UI chat/application chrome that shows the active agent/session/run state and selected provider/model, then adds privacy-aware context usage, token/cache/cost, provider quota windows, auth/subscription state, and fallback information using existing sessions.list, sessions.changed, models.authStatus, usage/cost, session event, and model/auth data where possible.

## Risks / Open Questions

- Quota and subscription state is provider-specific and privacy-sensitive; an implementation should avoid exposing account details unnecessarily and degrade cleanly when providers do not report quota windows.
- Several existing surfaces overlap with the request, so the remaining work should be scoped to persistent Control UI/application chrome during active runs rather than another slash command, historical session table, or dashboard-only report.
