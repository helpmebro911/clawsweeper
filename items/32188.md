## Summary

Keep #32188 open. Current main exposes related data and reports, including session/model selectors, context-pressure notices, fallback toasts, overview auth/quota cards, usage dashboards, and `/usage`/`/status` style reports, but it still does not provide the requested persistent in-interface status area showing the active agent/model plus live token/quota/subscription context during operations.

## Best Possible Solution

Keep the enhancement open. The best path is a first-class persistent Control UI status surface in the chat/application chrome that shows active agent/session/run state and selected provider/model, then layers in privacy-aware context usage, token/cache/cost, provider quota windows, auth/subscription state, and fallback state using existing sessions.list, sessions.changed/session.message payloads, models.authStatus, usage/cost, and model/auth data where available.
