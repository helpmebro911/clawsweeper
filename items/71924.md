## Summary

Current main still has the GitHub Copilot catalog returning an empty model list and relies on synthetic dynamic-model metadata for uncataloged Copilot models, so PR #71924 is not obsolete. The linked issue #71936 is closed, but the shipped/current code does not yet implement the PR’s central behavior: fetching Copilot `/models` and mapping GPT-5.5 capability metadata into the provider catalog.

## Best Possible Solution

Keep PR #71924 open and review it as the active implementation path. The best fix is plugin-local: fetch GitHub Copilot `/models` during catalog discovery, map Copilot capability limits into `ModelDefinitionConfig`, add GPT-5.5 to the relevant thinking profile handling, and keep focused tests for mapping, endpoint selection, and no-unmocked-network catalog behavior.
