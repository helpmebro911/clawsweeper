## Summary

Keep PR #71924 open. Current main still has the exact GitHub Copilot provider behavior the PR is trying to change: catalog discovery returns an empty model list, GPT-5.5 is not in the xhigh thinking allowlist, and unknown Copilot models fall back to conservative synthetic metadata rather than Copilot /models capabilities.

## Best Possible Solution

Keep the PR open and review it as the active implementation candidate. The likely right path is a focused GitHub Copilot plugin change that fetches Copilot /models during catalog discovery, maps capability limits into ProviderRuntimeModel metadata, adds GPT-5.5 to xhigh thinking support, and covers the behavior with mocked catalog-discovery tests.
