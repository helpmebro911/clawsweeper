## Summary

Keep this PR open. Current main still bypasses message hook emission for Slack no-mention skips, so the PR is not obsolete, but the submitted implementation needs revision because it reads the generic group policy surface instead of Slack's channel config surface.

## Best Possible Solution

Keep this PR open and ask for a revision: add a Slack-specific ingest policy on channels.slack.channels, wire it through the Slack type/schema/docs/config metadata and channel config resolver with wildcard and channel-specific override behavior, emit internal message:received from the no-mention skip path only when that Slack policy is true, simplify the unreachable slackFrom branch, and keep focused regression tests for wildcard ingest plus channel-specific ingest:false override.

## Risks / Open Questions

- Merging as submitted would likely not enable the intended behavior for valid Slack configs because the PR reads generic groups policy rather than channels.slack.channels.
- The Slack ingest contract is not yet in the Slack type, strict Zod schema, generated metadata, or public docs, so operators would not have a supported way to configure this behavior.
- The PR patch contains a minor dead branch noted in review: the slackFrom ternary is inside an isRoom-guarded block, so the slack:group fallback is unreachable.
- Silent ingest observes skipped channel messages; the final implementation should make that content-exposure behavior explicit and opt-in through Slack's channel policy surface.
