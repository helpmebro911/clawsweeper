## Summary

Current `main` appears to have fixed the silent-failure part of #42515 by surfacing user-visible error replies for empty `stopReason:"error"` turns and generic provider failures, but it does not implement generic session recovery for provider 400 deadlocks. The only automatic resets on this path are for compaction failures, role-ordering conflicts, and a specific Gemini corruption case, so the reported "offending message stays in history until /reset" behavior is still plausible.
