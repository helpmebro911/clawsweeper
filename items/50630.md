## Summary

Current main still allows the reported `gateway.auth.mode="none"` + `gateway.tailscale.mode="serve"` configuration. The runtime guard only rejects `funnel` without password auth and unauthenticated non-loopback binds; Tailscale serve forces loopback, so the non-loopback guard does not apply. A current test also proves this config resolves successfully instead of throwing, and auth mode `none` still accepts connections unconditionally. The linked fix PR #50631 is still open/unmerged in the provided GitHub context, so this is not a conservative cleanup close.
