## Summary

Keep open. The report is recent, detailed, non-maintainer-authored, and has no protected labels, but the available evidence is not strong enough to close as implemented or non-reproducible. The cross-referenced PR #60574 appears related to Gmail/Tailscale hook delivery, but the issue describes healthy Gmail watch delivery with missing OpenClaw hook session creation and Signal delivery, so it may still cover a distinct mapping/session dispatch failure. Current docs also show Gmail per-message session keys require explicit `hooks.allowRequestSessionKey` and `hooks.allowedSessionKeyPrefixes`, which is relevant but does not by itself prove the reported regression is fixed or obsolete.

## Evidence

- **Issue is actionable and recent:** Issue #59002 was opened 2026-04-01 with concrete versions, environment, logs, repro outline, and observed/expected behavior. It is not older than 60 days, so stale_insufficient_info does not apply.
- **Related PR is only circumstantial:** The timeline cross-references PR #60574, titled `fix(hooks): remove hooks token from Tailscale Funnel push endpoint URL`, but the provided issue body says `gog gmail watch status` reports healthy delivery and the missing behavior is OpenClaw `hook:gmail:<id>` session creation plus Signal delivery after watcher receipt.
- **Current docs describe Gmail session-key requirements:** The current configuration reference says the built-in Gmail preset uses `sessionKey: "hook:gmail:{{messages[0].id}}"` and instructs users to set `hooks.allowRequestSessionKey: true` plus matching `hooks.allowedSessionKeyPrefixes` when keeping per-message routing. This is relevant to the symptom but is not enough to prove the issue is fixed.
