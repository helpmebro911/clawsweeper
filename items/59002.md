## Summary

Keep open. Current `main` still makes the Gmail preset depend on templated `sessionKey` routing, and current gateway code/tests intentionally reject that route unless `hooks.allowRequestSessionKey` plus `hooks.allowedSessionKeyPrefixes` are configured. Docs now describe that requirement, but there is no strong evidence on `main` of an automatic migration, a restored pre-2026.3.31 behavior, or another clear fix for the reported regression. The linked follow-up in the issue timeline is an open PR about Funnel hook-token URLs, not proof that Gmail hook-session dispatch is resolved.
