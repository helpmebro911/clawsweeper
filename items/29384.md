## Summary

Keep #29384 open. Current main still does not expose Docker sandbox GPU, IPC, raw Docker create/run args, `capAdd`, `devices`, or `privileged` support for the requested `--gpus all` / `--ipc=host` workflow. Related OpenShell GPU support is a different backend, and the related capAdd PR is adjacent only.

## Best Possible Solution

Keep the issue open until a maintainer-approved Docker sandbox config extension lands. The best path is a narrow explicit contract such as `agents.defaults.sandbox.docker.gpus` plus a reviewed `ipc` policy or break-glass override, not broad raw-args passthrough by default. Update the config type, strict Zod schema, generated schema/help/docs, resolver and config-hash/recreate behavior, runtime validation/audit messaging, Docker create-arg builder, and focused regression tests.

## Risks / Open Questions

- The requested Docker flags weaken sandbox isolation: GPU/device exposure and host IPC are materially different from current default `capDrop: ["ALL"]`, no-new-privileges, and blocked namespace-join posture.
- A broad raw Docker args passthrough would bypass existing validation and auditing unless constrained carefully.
- OpenShell GPU support may help some users, but treating it as resolution would leave Docker-backend AI/ML workflows without the requested support.
