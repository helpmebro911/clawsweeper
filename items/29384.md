## Summary

Keep open. Current `main` does not implement arbitrary sandbox Docker flag passthrough or dedicated `gpus`/`ipc` settings. The sandbox Docker config is a fixed schema, and container creation only emits a curated set of arguments with security validation, so this feature request is still unmet and is not a conservative cleanup close.

## Evidence

- **Sandbox Docker config has no `gpus`, `ipc`, or generic extra-args field:** `SandboxDockerSettings` enumerates supported Docker knobs (`image`, `network`, `cpus`, `ulimits`, `binds`, etc.) and stops there; there is no field for GPU flags, IPC mode, or arbitrary docker-run parameters.
  - file: [src/config/types.sandbox.ts:3](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/config/types.sandbox.ts#L3)
  - command: `nl -ba src/config/types.sandbox.ts | sed -n '1,80p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Runtime schema rejects unknown Docker keys:** `SandboxDockerSchema` is `.strict()`, so even if a user tried to add ad hoc Docker parameters in config, unknown keys would be rejected rather than passed through.
  - file: [src/config/zod-schema.agent-runtime.ts:103](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/config/zod-schema.agent-runtime.ts#L103)

- **Container creation uses a curated allowlist of Docker CLI flags:** `buildSandboxCreateArgs()` appends specific flags like `--network`, `--user`, `--memory`, `--cpus`, `--ulimit`, and `-v`; there is no hook for arbitrary passthrough flags such as `--gpus all` or `--ipc=host`.
  - file: [src/agents/sandbox/docker.ts:362](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/sandbox/docker.ts#L362)

- **Docs expose only the same fixed Docker settings:** The public configuration reference lists the supported `sandbox.docker` options and examples, and it does not document GPU/IPC settings or a generic extra-args escape hatch.
  - file: [docs/gateway/configuration-reference.md:1542](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/gateway/configuration-reference.md#L1542)

## Risks / Open Questions

- A generic Docker-flag passthrough could bypass existing sandbox hardening and validation, so maintainers may want a narrowly-scoped design instead of raw flag injection.
- `--ipc=host` in particular expands host sharing/isolation surface; the existing Docker sandbox intentionally uses curated, validated options rather than unrestricted runtime flags.
