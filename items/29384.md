## Summary

Keep #29384 open. Current `main` still does not expose Docker sandbox GPU, host IPC, raw Docker args, devices, privileged mode, or `capAdd` support for the requested `--gpus all` / `--ipc=host` workflow. The earlier ClawSweeper comment remains consistent with current code: OpenShell has separate GPU support, and related Docker capability work is adjacent rather than a shipped replacement for Docker backend users.

## Best Possible Solution

Keep the issue open until a maintainer-approved Docker sandbox config extension lands. The best path is a narrow explicit contract such as `agents.defaults.sandbox.docker.gpus` plus a reviewed `ipc` policy or guarded break-glass override, with aligned updates to public config types, Zod schema, generated schema/help/docs, resolver/config-hash behavior, runtime validation/audit messaging, Docker create args, and focused tests. Avoid broad raw Docker arg passthrough unless maintainers explicitly accept the security and validation tradeoffs.

- OpenShell GPU support may help some users, but closing this as implemented would leave Docker-backend AI/ML workflows without the requested support.
