## Summary

Keep #29384 open. Current main still does not expose Docker sandbox `--gpus`, `--ipc`, raw Docker create args, or equivalent GPU/IPC fields. The OpenShell `gpu` option is a separate sandbox backend, and the related #53108 capAdd PR is open/unmerged in the provided context and would not satisfy the requested Docker GPU/IPC workflow.

## Best Possible Solution

Keep the issue open until a maintainer-approved Docker sandbox config extension lands. Best path is a constrained explicit contract such as `agents.defaults.sandbox.docker.gpus` plus a carefully reviewed `ipc` policy or dangerous override, not a broad raw-args passthrough by default. Update the TypeScript config type, strict Zod schema, resolver/hash behavior, runtime validation/audit messaging, Docker create-arg builder, generated schema/help/docs, and focused regression tests. Treat #53108 as related capAdd prior work only, not closure for GPU/IPC support.
