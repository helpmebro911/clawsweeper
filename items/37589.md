## Summary

Keep #37589 open. Current main still recreates array-backed message objects and every text part in stripInlineDirectiveTagsFromMessageForDisplay even when no inline directive tags are stripped, and the related fix attempts referenced in the issue are closed unmerged.

## Best Possible Solution

Implement the narrow identity-preservation fix in src/utils/directive-tags.ts: reuse unchanged text-part objects, track whether any text block changed, and return the original message object when no directives were stripped. Add focused Vitest coverage with toBe assertions for unchanged array-backed messages and mixed changed/unchanged text parts; if maintainers decide this helper should not remain plugin-facing, remove or de-export it deliberately with SDK docs/API updates.
