## Summary

Keep #11517 open. Current main still has the import-heavy Pi attempt runner and Discord monitor files named by the issue, and the merged harness/runtimePlan work is an adjacent plugin/harness seam rather than the requested RunContext/AgentContext dependency aggregation. The latest issue discussion also already reached a keep-open conclusion.

## Best Possible Solution

Keep #11517 open as the canonical architecture tracker. The best path is an incremental refactor that introduces a prepared RunContext/AgentContext or smaller service bundles for the Pi attempt runner and Discord monitor, preserves plugin SDK boundaries, separates pure file moves from behavior changes, and adds focused tests around each extracted seam.
