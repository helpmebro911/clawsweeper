## Summary

Keep open. Current main still has the import-heavy Pi embedded attempt runner and Discord monitor surfaces named in #11517, and the current harness/plugin seams are adjacent rather than a RunContext/AgentContext-style dependency aggregation for these files.

## Best Possible Solution

Keep #11517 open as the canonical tracker. The best path is an incremental maintainer-approved refactor that introduces prepared RunContext/AgentContext or smaller service bundles for the Pi attempt runner and Discord monitor, preserves plugin SDK boundaries, and adds focused tests around extracted seams before any large pure-file split.
