## Summary

Keep #12602 open. Current main already implements authored Slack Block Kit payloads, shared presentation blocks, and interactive buttons/selects, but the remaining native Slack markdown table/rich Markdown delivery path is still not implemented: `markdown.tables: "block"` is schema-valid in the shared Markdown IR but is coerced back to text/code rendering before Slack delivery. The issue discussion also records recent maintainer/Codex review keeping this umbrella open until that remaining Slack table/rich output work is resolved or explicitly superseded.

## Best Possible Solution

Keep #12602 open unless maintainers explicitly make #57708 or another narrower item canonical. The concrete remaining implementation belongs in the bundled Slack plugin: carry `markdownToIRWithMeta` table metadata through outbound Slack delivery when `markdown.tables: "block"` is configured, render supported Markdown tables as Slack Block Kit table/rich blocks, add focused Slack outbound tests and public docs, and avoid duplicating the already-shipped raw `channelData.slack.blocks`, shared `presentation`, and interactive directive support.
