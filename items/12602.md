## Summary

Keep #12602 open. Current main already implements major Slack Block Kit slices, including authored Slack blocks, semantic presentation rendering, and interactive reply controls, but the remaining richer/table-rendering gap is still real: `markdown.tables: "block"` is schema-valid but coerced to `code`, tests assert that behavior, and the Slack renderer has no outbound table block path. The latest provided maintainer discussion also explicitly kept this umbrella open until that work is implemented or superseded.

## Best Possible Solution

Keep #12602 open as the umbrella for the remaining Slack rich-output work unless a maintainer explicitly supersedes it with a narrower canonical issue. The likely implementation belongs in the Slack plugin: wire markdown table/rich markdown metadata into native Slack Block Kit table or rich rendering end to end, add focused outbound tests, and update the public message-presentation or Slack docs. Treat raw Block Kit payloads, shared presentation, and interactive reply directives as already shipped.
