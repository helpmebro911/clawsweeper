## Summary

Keep #39428 open. Current main still routes Discord inbound messages through the generic route resolver, which only checks explicit binding tiers before falling through to the configured default agent. I found no shipped implicit Discord accountId-to-agentId matching tier on main, and the direct fix PR #39485 is still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #39428 open until PR #39485 or an equivalent maintainer-approved fix lands on main. The likely implementation belongs after explicit binding tiers and before default-agent fallback, should preserve the literal default account behavior, and should add regression coverage for Discord accountId matching plus the default-account guard.
