## Summary

Keep this PR open. Current main still lacks the proposed passive recovery prompt discipline, Recovery Bundle / Task Ledger state files, and checkpoint writes for update_plan, sessions_yield, and sessions_spawn. The existing restart/orphan recovery code is active recovery from interrupted sessions, not the passive informational checkpoint layer proposed here.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase against current main, address the redaction and synchronous-I/O review comments, keep recovery strictly passive and informational, prove approvals/auth/permissions are never inherited, add focused tests for update_plan, sessions_yield, sessions_spawn, prompt composition, and persisted recovery state, and decide whether the new recovery-state surface needs user-facing docs.

- The design should be reconciled with existing restart-aborted main-session recovery, subagent orphan recovery, reset behavior, compaction, transcript snapshots, and session-store maintenance before landing.
