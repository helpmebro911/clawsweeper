## Summary

Keep open. Current main does not implement the PR's central session-archiving feature: there is no persisted session `archivedAt` field, no `includeArchived`/`archivedAt` gateway protocol support, no default archive filtering in `sessions.list`, and no Sessions UI archive/unarchive control. The related ClawSweeper items are adjacent chat-history/session-management work, but none supersede this specific PR.

## Best Possible Solution

Keep the PR open for maintainer/product review. The best path is a rebased, focused session-archiving implementation with additive gateway protocol support, persisted `archivedAt`, default list hiding plus explicit `includeArchived`, UI archive/unarchive controls, inbound auto-unarchive behavior, async-safe archive-summary writing, Swift/client type updates, docs if user-facing, and regression tests for gateway list/patch, UI controller/view behavior, and internal lookup paths such as resolve, announce target, and spawned-session visibility. Split or remove the unrelated Telegram topic-name command unless maintainers want it bundled.
