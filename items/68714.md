## Summary

Keep open. Current main still lacks the PR's post-rename chmod on the async config hot-save path, and the PR is a focused, plausible security hardening change rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and finish it as a small security hardening change: add `chmod(configPath, 0o600)` after the successful async rename in `src/config/io.ts`, add a regression test that starts with a permissive existing `openclaw.json` and verifies `writeConfigFile` leaves it at `0600` on POSIX, and consider aligning the sync migration helper if maintainers want the same invariant for every config rewrite path.
