## Summary

Keep open. Current main has partial startup-scoping and setup-entry machinery, but the WhatsApp channel still lacks the opt-in and narrow runtime setter path needed to avoid full configured-channel module loading at gateway startup. The related #70533 is useful and open, but it tracks manifest discovery/allowlist overhead rather than this issue's configured WhatsApp runtime evaluation path, so it is not a clean superseding item.
