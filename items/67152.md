## Summary

Keep open. Current main still contains the reported request-scoped fallback plus cleanup path: `memory-core` writes the local fallback diary entry when `subagent.run` throws `RequestScopedSubagentRuntimeError`, returns without a run id, and then the `finally` block still calls `subagent.deleteSession` on the same runtime. Existing tests still expect cleanup after request-scoped fallback, so the warning remains possible. The related PR #70554 is an implementation candidate, but the provided context says it is still open, not merged.

## Best Possible Solution

Keep this issue open until PR #70554 or an equivalent memory-core fix lands. The fix should either use a gateway-valid background subagent runtime for dreaming narrative work or skip request-scoped cleanup after fallback, and should add regression coverage proving the fallback writes the diary entry without emitting the cleanup warning.
