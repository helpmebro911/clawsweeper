## Summary

Keep PR #69647 open. Current main still has the stale Control UI gateway URL path: `loadSettings()` falls back from the current page-derived scoped key to the shared legacy settings key, then preserves a sibling-path `gatewayUrl` when it differs from the current page URL. The PR is recent, focused, and the provided review timeline shows substantive follow-up commits addressing earlier heuristic and page-scoped cache concerns, so this is not a conservative cleanup-close candidate.

## Best Possible Solution

Keep the PR open for maintainer review or author revision. The likely product path is a small Control UI storage fix that distinguishes page-derived gateway defaults from explicit user overrides, writes/updates a page-derived scoped settings record, and adds regression tests for `/a` to `/b` reverse-proxy navigation plus same-host custom WebSocket paths and token scoping.
