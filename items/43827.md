## Summary

Current main still has hard-coded Chat UI strings, so the central i18n work is not implemented. The PR remains useful in direction, but it is stale against the current Chat component split, the Control UI locale-generation pipeline, and the current `t()` fallback contract. Security review did not find a concrete supply-chain or secrets-handling concern in the PR diff.

## Best Possible Solution

Keep this PR open for author or maintainer revision: rebase onto current main, localize the current Chat modules (`ui/src/ui/views/chat.ts` and relevant `ui/src/ui/chat/*` helpers), add English source keys, regenerate all supported non-English Control UI locale bundles through the i18n pipeline, and either remove the no-op `fallback` params or add a tested fallback API.

## Risks / Open Questions

- Applying the PR as-is would leave current hard-coded Chat strings behind because Chat controls, queue rendering, and status indicators have moved into `ui/src/ui/chat/*` modules.
- The PR's `{ fallback: "..." }` parameters are misleading with the current `t()` API; missing English keys would render raw key paths rather than the intended fallback text.
- The PR hand-edits generated non-English locale bundles and covers only six locales, while current main supports fourteen locales through the Control UI i18n pipeline.
- Current tests still assert literal button titles in Chat run controls, so a revised implementation should update or preserve those expectations intentionally.
