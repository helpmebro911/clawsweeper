## Summary

Keep open. The PR targets a real remaining Web UI streaming defect on current main: `handleChatEvent` stores chat deltas as the full latest text snapshot, `handleAgentEvent` still commits that full cumulative `host.chatStream` into each stream segment when a tool card starts, and `buildChatItems` renders every segment plus the current live stream as separate assistant streaming groups. The PR branch’s follow-up commit `c7517091065c11d7d9f7c3b5a927f980b622db46` positively addresses the Greptile off-by-one review by slicing from total committed segment length, but that code is not present on current main and the current tests do not cover the multi text/tool/text cycle.

## Best Possible Solution

Keep the PR open or replace it with an equivalent narrow main-branch fix. The best implementation is to commit only the newly streamed suffix when a tool starts, using the total length of already committed stream segments rather than the last segment text, and add a targeted `app-tool-stream.node.test.ts` regression for multiple cumulative text snapshots interrupted by multiple tool calls. If this PR is preferred, land the corrected `c7517091065c11d7d9f7c3b5a927f980b622db46` behavior with tests rather than closing it as obsolete.
