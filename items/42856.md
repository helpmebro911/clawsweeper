## Summary

Keep open. Current main still appears to reproduce the reported source-path bug: `gateway` action `config.schema.lookup` is compared as the full compound action name against simple read-only verbs, so lookup misses can still be marked mutating and then rendered as user-facing warning payloads. Two open, unmerged PRs (#42898 and #42962) explicitly reference this issue with closing intent, so the issue should remain open until a fix is reviewed and merged or the PRs are replaced.

## Best Possible Solution

Keep this issue open and review the linked implementation candidates. The best fix likely belongs in `src/agents/tool-mutation.ts`: classify compound `gateway`/`cron`/`canvas` actions by their leaf read-only verb, or explicitly treat `config.schema.lookup` as non-mutating, with regression coverage for mutation classification and payload warning behavior for `config schema path not found`.
