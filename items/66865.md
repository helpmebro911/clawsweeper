## Summary

Keep this PR open. Current main still lacks the central update invariant: skipped update results are reported as successful to the UI, and update-triggered restarts do not have an update-specific, health-checked process replacement path. The PR is an active implementation candidate with recent author follow-up and directly targets the still-open behavior in #62492 plus related stale-chunk reports.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to review and land a focused fix that makes `update.run` non-ok for skipped/error outcomes, blocks unsafe package self-updates before replacing global files, performs update-specific process replacement with health/version verification, and preserves the regression coverage shown in the PR. After merge, close #62492 as implemented and reassess the remaining scope in #64892 and #65668 separately.
