## Summary

Keep #37843 open. Current main still trims `payload.model` and resolves any non-empty value as a model override, so `model: "default"` is not treated as an unset/runtime-default sentinel. The related fix PRs in the supplied GitHub context are closed unmerged, and current docs/tests do not show the requested behavior as shipped.

## Best Possible Solution

Keep the issue open. Implement a shared cron helper that trims `payload.model` and treats exact `default` as unset before model resolution; use the same helper in `resolveCronFallbacksOverride` so fallback behavior matches omitted model. Add focused tests for undefined, empty, whitespace-only, `default`, whitespace-trimmed `default`, explicit provider/model, invalid/disallowed models, and fallback-chain behavior, then update CLI/UI/docs if `--model default` is intended as a supported user-facing value.
