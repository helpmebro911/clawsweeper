## Summary

Keep #71049 open. Current main still always enables the Microsoft Teams DM streaminfo/informative-status path for personal chats, and the existing `typingIndicator` flag only controls the native typing keepalive, not the Copilot-style stream status. The issue is recent, has no protected-label or maintainer-author blocker, and the requested behavior belongs in the bundled `msteams` plugin config/runtime rather than an external ClawHub item.

## Best Possible Solution

Keep the issue open and implement a bundled `msteams` plugin config option for DM stream-status behavior. The runtime change should let personal chats skip `TeamsHttpStream`/`replyOptions.onPartialReply`, use native typing keepalive, and send a single final message when disabled; the schema, generated metadata, config types, docs, and dispatcher/stream-controller tests should be updated together.
