## Summary

Current main still lacks the requested MS Teams DM opt-out. Personal conversations always get a TeamsHttpStream-backed stream controller and an informative stream update on reply start; `typingIndicator: false` only disables the native typing keepalive, not the Copilot-style stream status. The accepted config schema exposes `typingIndicator`, `blockStreaming`, and `blockStreamingCoalesce`, but no `streaming`, `dmStreamStatus`, or equivalent DM stream-status control, and the schema is strict.

## Best Possible Solution

Keep this issue open and implement it in the bundled MS Teams plugin. Add a plugin-owned config option under `channels.msteams` that gates personal-chat streaminfo/informative status creation, falls back to native typing plus final-message delivery, and update the TS config type, strict zod schema, generated config metadata, MS Teams docs, and dispatcher/controller tests.
