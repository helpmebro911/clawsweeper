## Summary

Keep #58823 open. Current main still resolves an agent's own model before `agents.defaults.subagents.model` in both the shared subagent helper and isolated cron path, and checked-in tests assert that behavior. The PR is a focused, still-relevant fix for open issue #58822, but it conflicts with the behavior merged in #58003 and now has a bot-raised security concern, so it needs maintainer judgment rather than cleanup closure.

## Best Possible Solution

Keep the PR open. Maintainers should decide whether `agents.defaults.subagents.model` is intended to outrank an agent's own `model.primary` for all agents or only for the main-agent case, then land a narrow update across `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression tests for explicit spawn override, per-agent `subagents.model`, global subagent default, and fallback to the agent primary model. Review the bot-raised allowed-model validation concern before merge.
