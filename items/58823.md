## Summary

Keep open. Current main still gives the agent primary model priority over `agents.defaults.subagents.model` in both the shared subagent helper and isolated cron path, and checked-in tests assert that behavior. PR #58823 is not implemented on main and remains a plausible focused fix for #58822, but it conflicts with the behavior intentionally merged in #58003 and needs maintainer product judgment before landing.

## Best Possible Solution

Keep the PR open for maintainer review. Decide the intended model-precedence contract, then either land this narrow two-path fix with regression tests for explicit override, per-agent `subagents.model`, global `agents.defaults.subagents.model`, and fallback to agent primary model, or preserve #58003's named-agent behavior with an explicit main-agent distinction and updated docs/tests.
