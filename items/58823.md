## Summary

Keep this PR open. Current main still implements and tests the opposite subagent model precedence in both the shared helper and isolated cron path, while the docs support the global subagent-default expectation. The PR is paired with open issue #58822 by the same author and has an unresolved Aisle allowlist-validation concern in the review discussion.

## Best Possible Solution

Keep this PR open for maintainer review. Maintainers should decide the intended subagent model precedence contract, then either land a narrow fix in `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts` with regression coverage plus subagent-spawn allowlist validation, or keep #58003's current precedence and update `docs/tools/subagents.md` plus related config docs before resolving #58822.
