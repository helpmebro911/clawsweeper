## Summary

This PR should stay open. Current main still implements and tests the opposite precedence from the PR, while the user-facing docs support the reporter's expectation that `agents.defaults.subagents.model` sets the default sub-agent model. The PR is also paired with open same-author issue #58822 via closing syntax, and the security review raised a concrete model-allowlist validation point that needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. If `agents.defaults.subagents.model` is intended to outrank an agent's own `model.primary`, land this PR or a narrow equivalent in `src/agents/model-selection.ts` and `src/cron/isolated-agent/model-selection.ts`, update regression tests, and address or explicitly document allowlist validation for normal `sessions_spawn` model persistence. If #58003's named-agent precedence is the intended contract, update `docs/tools/subagents.md` and `docs/gateway/config-tools.md`, then close #58822 and this PR with that product decision.
