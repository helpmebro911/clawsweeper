## Summary

Keep open. Current `main` still does not implement the requested lifecycle distinction: embedded runs emit only `start`/`end`/`error`, `agent.wait` still maps aborted terminal lifecycle events to `timeout`, and subagent outcome plumbing preserves that generic timeout classification. There is a partial ACP-side mitigation that emits parent-facing start/no-output notices, but it does not add a `first-progress`/`startup-failed` lifecycle signal or change `agent.wait` semantics.
