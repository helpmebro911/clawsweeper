## Summary

Keep open. Current main still does not implement Discord cross-instance channel ownership: slash commands and message preflight continue to apply only local Discord allowlist state, and the repo has no Discord instance-claim registry or claim lookup. The open PR is a relevant implementation candidate for the closed #55651 report, but it still needs maintainer review and likely revision before landing because review comments identify real routing and concurrency risks.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is a Discord-plugin-owned fix under `extensions/discord/src/monitor`: robust per-instance/per-bot ownership claims, lookup before local allowlist denial in message preflight and native slash-command paths, and focused tests covering same-token multi-instance routing, guild-wide and channel-scoped allowlists, bot scoping, denied/disabled channels, named/wildcard entries, and lifecycle cleanup. Close only after that implementation lands, or after an explicit maintainer decision that same-token multi-instance Discord routing is unsupported.
