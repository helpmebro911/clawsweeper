## Summary

Current `main` still lacks a plugin-facing way to observe the resolved tool set with JSON schemas. There is no `after_tools_resolved` hook or `getResolvedTools()`-style SDK API, and the adjacent gateway inventory APIs only expose names/descriptions/provenance, not tool parameter schemas.

## Evidence

- **No resolved-tools plugin hook or API on current main:** `src/plugins/hook-types.ts:71`-`85` lists plugin hook names and includes `before_tool_call` / `after_tool_call`, but nothing like `after_tools_resolved`. `src/plugins/types.ts:1922`-`1947` shows `OpenClawPluginApi` methods such as `registerTool`, `registerHook`, and `registerGatewayMethod`, but no `getResolvedTools()` or equivalent read API.
  - file: [src/plugins/hook-types.ts:71](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/plugins/hook-types.ts#L71)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Targeted search found no proposed names:** Repository search for `after_tools_resolved` and `getResolvedTools(` returned no matches in `src/plugins`, `src/plugin-sdk`, or plugin docs, which is strong negative evidence that the requested surface has not landed under the proposed API names.
  - command: `rg -n "after_tools_resolved|getResolvedTools\(" src/plugins src/plugin-sdk docs/plugins docs/concepts docs/tools`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Adjacent gateway APIs do not expose schemas:** `src/gateway/protocol/schema/agents-models-skills.ts:409`-`437` defines `ToolsEffectiveEntry`/`ToolsEffectiveResult` with only `id`, `label`, `description`, `rawDescription`, `source`, `pluginId`, and `channelId`. There is no `parameters` or schema field.
  - file: [src/gateway/protocol/schema/agents-models-skills.ts:409](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/gateway/protocol/schema/agents-models-skills.ts#L409)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Runtime inventory builder drops tool schemas:** `src/agents/tools-effective-inventory.ts:156`-`168` constructs each effective-tool entry from `tool.name`, label, and descriptions plus provenance, and does not include `tool.parameters`. This confirms the runtime-effective inventory cannot satisfy the issue as filed.
  - file: [src/agents/tools-effective-inventory.ts:156](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/agents/tools-effective-inventory.ts#L156)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Catalog API is operator-scoped and metadata-only:** `src/gateway/server-methods/tools-catalog.ts:26`-`33` defines catalog entries without any schema field, and `docs/gateway/protocol.md:428`-`439` documents `tools.catalog` / `tools.effective` as operator gateway methods, not plugin SDK hooks. That is adjacent functionality, but it does not provide plugins the full resolved tools plus JSON schemas requested here.
  - file: [src/gateway/server-methods/tools-catalog.ts:26](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/gateway/server-methods/tools-catalog.ts#L26)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)

## Risks / Open Questions

- The new gateway `tools.catalog` and `tools.effective` surfaces can make this look implemented at a glance, but they are not plugin-facing SDK hooks and they omit tool parameter schemas.
- PR #18860 is still referenced from the issue context as open, so closing this as implemented would be premature without a merged commit on `main`.
