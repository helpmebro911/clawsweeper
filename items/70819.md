## Summary

Keep open. Checked-out main still rejects any `node_modules` symlink whose real target escapes the install root, and the trusted OpenClaw peer-link exception proposed by this PR is not present on current GitHub `main` either. The PR is still open, its head exists, and its patch adds the exact allowlist plus regression coverage for the missing behavior.

## Best Possible Solution

Keep the PR open for maintainer review or land a narrowed equivalent: allow only exact `node_modules/openclaw` and `node_modules/.bin/openclaw` peer links when their real target resolves inside the trusted host OpenClaw package root, while preserving negative coverage for spoofed, nested, arbitrary, or non-exact symlink names. Consider splitting unrelated CI stabilization if reviewers want the security-scanner fix isolated.
