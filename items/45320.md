## Summary

Keep #45320 open. Current main does not implement the requested Discord partial-mode assistant-boundary preview rotation under the proposed name or an equivalent current config shape. The PR is stale against the pluginized Discord layout and newer nested streaming config, and it has a review comment identifying a correctness gap in the draft-stream hardening, but the central capability is still plausibly useful and is not safely closed as implemented, duplicate, ClawHub, or unactionable.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If this behavior is accepted, update it against current main by using the current nested streaming config shape, likely under channels.discord.streaming.preview, add schema/types/UI hints/docs, wire the Discord message handler off a resolver instead of discordStreamMode === "block", move tests to extensions/discord, and fix the stale in-flight send failure path plus throttle-window reset in createDiscordDraftStream. If maintainers reject another Discord-specific knob in favor of the broader progress/status-lane design, close it explicitly with that product rationale.
