## Summary

Keep open. Current `main` does not implement this PR’s reset-refresh behavior: `sessions.reset` only emits `sessions.changed`, while the TUI only handles `chat`, `chat.side_result`, and `agent` events. The TUI’s in-memory run-state reset is keyed to session-key changes, so an external reset of the same session still has no matching event path to clear stale run/tool state or reload history.

## Evidence
