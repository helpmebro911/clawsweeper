## Summary

Keep open. Current main still contains the reported Overview behavior, so this PR is not obsolete, and the same-author bug #41545 remains open. The exact patch also needs maintainer review or rework because it preserves a gateway token across arbitrary WebSocket endpoint edits, while current storage, docs, and tests treat gateway tokens as selected-gateway-scoped credentials.

## Best Possible Solution

Keep this PR open for maintainer review or author rework. The best fix is to separate in-progress URL editing from confirmed endpoint changes: preserve the typed token for same-target or whitespace-only edits, but clear or require explicit confirmation/token entry when the normalized gateway target changes, with regression tests for both cases and no changes to backend auth or storage schema.
