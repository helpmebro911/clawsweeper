## Summary

Keep #10354 open. Current main still exposes the shared message tool `channel` parameter as a bare optional string with no description or enum, while the requested one-line schema metadata change remains unmerged. Prompt guidance and runtime fallback behavior do not satisfy the tool-schema request.

## Best Possible Solution

Land a narrow core change in `src/agents/tools/message-tool.ts` that adds a `channel` schema description explaining it expects a messaging provider or channel plugin id, such as `slack` or `discord`, not a conversation/channel target id. Add a focused `src/agents/tools/message-tool.test.ts` assertion that `createMessageTool().parameters.properties.channel.description` is present. Defer an enum unless maintainers derive it dynamically from configured or registered channel/plugin ids.
