## Summary

Keep #10354 open. Current main still exposes the shared `message` tool `channel` parameter as a bare optional string with no description or enum, while `target` has parameter-level guidance. Prompt text and runtime fallback reduce some failures, but they do not implement the requested schema metadata, and the linked fix PRs were closed unmerged.

## Best Possible Solution

Keep the issue open and land a narrow core change in `src/agents/tools/message-tool.ts` that adds a `channel` schema description explaining it expects a messaging provider/channel plugin ID such as `slack` or `discord`, not the conversation target. Add a focused schema test proving the description is exposed; defer an enum unless maintainers choose a dynamic-safe source for plugin channel IDs.
