## Summary

Keep #10354 open. Current main still exposes the shared `message` tool `channel` parameter as a bare optional string with no description or enum, and the linked attempts #10364/#13584 were closed unmerged. Prompt guidance and current-channel fallback do not provide the requested tool-schema metadata.

## Best Possible Solution

Land a narrow core change in `src/agents/tools/message-tool.ts` adding a `channel` schema description that says it expects a messaging provider or channel plugin id such as `slack` or `discord`, not the conversation/channel target id. Add a focused `src/agents/tools/message-tool.test.ts` assertion that `createMessageTool().parameters` exposes that description. Defer an enum unless maintainers derive it dynamically from the channel/plugin registry.
