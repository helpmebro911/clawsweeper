## Summary

Keep open. Current main still does not contain this PR's durable pending-inbound store, startup replay, or file-backed active-turn recovery. The PR remains a live implementation candidate, with overlapping but not yet accepted follow-up design in #65650 and unresolved correctness/security-sensitive review findings in the provided discussion.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should either review it and require fixes for the remaining correctness/security concerns, or explicitly supersede it with a canonical durable restart-resilience design such as #65650 after that design is accepted.

## Risks / Open Questions

- The PR touches gateway startup, command draining, embedded-run lifecycle, system-event trust semantics, and JSON atomic file helpers, so landing it without maintainer review could introduce restart, data-loss, or local file-I/O regressions.
- The provided security review specifically calls out untrusted inbound content being replayed as trusted system events and symlink/chmod/read-size concerns in the PR's file-backed store path.
- #65650 overlaps the restart-resilience problem with a SQLite-based direction, but it is an open issue rather than an accepted/merged superseding implementation.
