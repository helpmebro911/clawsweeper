## Summary

Current main still appears to bypass the hook-aware outbound seams for the WebChat/Gateway TUI `chat.send` path. The related umbrella #50126 is still the right broader tracker, but this issue remains a concrete default-surface regression rather than cleanup-closeable duplicate work, especially because #12584 was closed unmerged in the provided context.

## Best Possible Solution

Keep this issue open and fix the WebChat/Gateway TUI `chat.send` path. The best implementation is to route `chat.send` reply delivery through a shared hook-aware dispatcher/observer seam, or add equivalent `message_sending`, `message_sent`, and `before_message_write` calls around the existing WebChat reply delivery and transcript append paths. Add regression coverage for WebChat `chat.send` and Gateway-backed `openclaw-tui` so plugin hooks observe assistant replies on these default local surfaces before closing.
