## Summary

Current main still matches the reported WebChat/Gateway TUI outbound hook gap. The `chat.send` handler used by WebChat creates a raw reply dispatcher and calls `dispatchInboundMessage` directly, bypassing the hook-aware wrapper that wires `message_sending`. The reply dispatcher itself only runs `beforeDeliver` and `deliver`, with no `message_sent` emission, and the WebChat transcript append path writes through `SessionManager.appendMessage` without the existing `before_message_write` hook wrapper. The related #12584 PR is closed unmerged, and the broader #50126 umbrella does not make this specific default-surface regression obsolete.

## Best Possible Solution

Keep this issue open. Fix the Gateway `chat.send` path used by WebChat and Gateway-backed TUI by routing replies through a shared hook-aware outbound observer/dispatcher, or by adding equivalent `message_sending` and `message_sent` handling around the current delivery path. Also apply `before_message_write` to Gateway-injected assistant transcript appends, then add focused regression tests for WebChat `chat.send` and TUI/Gateway `sendChat` coverage. Coordinate any broader unification with #50126, but do not close this specific default-surface regression until the code and tests cover it.
