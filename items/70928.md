## Summary

Keep open. Current main still has a real webchat/openclaw-tui outbound hook gap: webchat `chat.send` builds a plain reply dispatcher and later broadcasts/persists final assistant messages without `message_sent` emission, while TUI either delegates to `chat.send` or runs embedded internal agent output that only reaches `deliverOutboundPayloads` for non-internal external delivery. The related #12584 PR is closed unmerged, and #50126 remains an umbrella rather than a shipped fix for this specific webchat/TUI path.

## Best Possible Solution

Keep #70928 open and fix it in core. Add a shared outbound lifecycle observer or a minimal webchat/TUI bridge that fires `message_sending` before final internal delivery and `message_sent` after broadcast/persist success or failure, with session/run context. Cover `chat.send`, Gateway TUI, and embedded TUI in regression tests, and add an interim docs matrix for hook/channel guarantees if the architectural unification stays in #50126.
