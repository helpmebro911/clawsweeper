## Summary

Current main partially covers Telegram sender context by exposing SenderName, SenderId, SenderUsername, ChatType, WasMentioned, and OriginatingChannel, but it does not expose Telegram msg.from.is_bot or an equivalent SenderIsBot field to the agent context. The issue's central bot-vs-user classification request remains missing, so it should stay open.

## Risks / Open Questions

- The remaining work needs a small context-contract decision, likely a typed field such as SenderIsBot rather than dumping raw Telegram msg.from into prompts.
- Bot status is channel-specific and user-controlled metadata should stay clearly marked as untrusted where it reaches the agent prompt.
