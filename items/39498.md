## Summary

Keep #39498 open. Current main still has a plausible ByteString crash path for pasted non-Latin1 API keys, especially Comfy Cloud's `X-API-Key`, but the PR's global `normalizeSecretInputString` change is too broad and can mutate valid non-header secrets such as Matrix passwords.

## Best Possible Solution

Keep the PR open, but request a narrower patch. Preserve trim-only semantics for `normalizeSecretInputString`, `hasConfiguredSecretInput`, and non-header credentials. Apply `normalizeSecretInput` or `normalizeOptionalSecretInput` only where secrets become HTTP header values, starting with Comfy Cloud `X-API-Key` and any shared provider request auth/header sanitizer. Add regression coverage for the Comfy pasted-U+2502 path and a guard proving Matrix-style password/body credentials keep non-Latin1 characters.
