## Summary

Keep #70443 open. Current main still has no `--offline` argument path for `scripts/docker/setup.sh`: the script unconditionally builds `openclaw:local` or pulls `OPENCLAW_IMAGE` before continuing, and the sandbox branch still builds the sandbox image when `Dockerfile.sandbox` exists. The Docker install docs and e2e tests also lack offline setup coverage. The related #69143/#69314 context is about cross-platform build targeting, not this airgapped pre-loaded-image rerun flow.

## Best Possible Solution

Implement the requested explicit `--offline` flag in `scripts/docker/setup.sh`: parse it before setup work, skip the main `docker build`/`docker pull`, require `docker image inspect "$OPENCLAW_IMAGE"` for the gateway image with a clear `docker load` message, warn rather than fail for the optional sandbox image, then leave env synchronization, permission repair, onboarding, gateway config sync, compose startup, and sandbox overlay/config behavior unchanged. Add focused coverage in `src/docker-setup.e2e.test.ts` and document the airgapped rerun flow in the Docker install docs.
