## Summary

Keep open. Current main still has no `--offline` path for Docker setup: `scripts/docker/setup.sh` unconditionally builds `openclaw:local` or pulls `OPENCLAW_IMAGE` before running the permission fix, onboarding, gateway config sync, and Compose startup steps the issue wants to preserve. The related #69143/#69314 context is cross-platform build support, not a canonical offline rerun implementation.

## Best Possible Solution

Keep this issue open and implement the focused Docker setup feature in `scripts/docker/setup.sh`: add Bash 3.2-compatible `--offline` argument parsing, skip the main build/pull only when set, require `docker image inspect "$OPENCLAW_IMAGE"` with a clear `docker load` message if missing, inspect the sandbox image and warn rather than fail when absent, then leave env upsert, permission repair, onboarding, gateway config sync, compose startup, and sandbox overlay handling intact. Add targeted `src/docker-setup.e2e.test.ts` coverage and update `docs/install/docker.md` for the public offline rerun workflow.
