## Summary

Keep this PR open. Current main still derives memory-wiki local-file source identity solely from the slugified title and writes directly to the same `sources/<slug>.md` path, so same-slug inputs can still overwrite each other. The open PR is a focused memory-wiki implementation candidate with regression coverage and no unrelated security-sensitive changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or refine a memory-wiki-owned fix that preserves readable source page names, adds a collision-resistant source-path-derived suffix for new source pages, reuses legacy source pages where possible, and keeps the regression tests focused on source-page collisions.
