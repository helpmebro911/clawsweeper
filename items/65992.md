## Summary

Keep open. Current main still derives memory-wiki ingest source `pageId` and `pagePath` only from the normalized title slug, so different inputs that share a title/slug can still overwrite the same `sources/<slug>.md` page. This PR remains the active implementation candidate for the linked collision problem rather than something already shipped or superseded.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing if the source-page scan trade-off is acceptable. The best code path is still a memory-wiki-owned fix that preserves readable source page names while adding a stable, collision-resistant source identity suffix and regression tests for legacy reuse, symlinks, same-slug titles, and same-title different paths.
