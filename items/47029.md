## Summary

Keep #47029 open for maintainer judgment. Current main covers the Feishu unresolved-SecretRef inspection path and has a newer gateway auth SecretRef resolution path for status probes, but it does not implement the PR's loopback paired-device identity behavior; current tests explicitly keep unauthenticated loopback probe identity disabled. The bot review comments about avoiding identity-file writes were material, and current main appears to have moved toward a non-mutating probe policy rather than the PR's proposed unauthenticated loopback identity reuse.

## Best Possible Solution

Keep the PR open for maintainer review, but require a rebase/split against current main. The likely path is to keep the shipped Feishu/status auth improvements, drop obsolete command-target-set work if the newer probe auth resolver is sufficient, and make an explicit maintainer decision on whether loopback probes should ever reuse an existing paired operator identity without shared auth.
