## Summary

Current main still canonically uses `HEARTBEAT.md`; the requested rename to `Proactive.md` has not been implemented or superseded. Runtime prompts, workspace bootstrap, lightweight context filtering, docs, templates, config types, and tests all still depend on `HEARTBEAT.md`, while repository search found no `Proactive.md`/`PROACTIVE.md`/`proactive.md` implementation. The prior issue discussion also explicitly kept this open, and related PR #48610 is only an open docs guide about proactive heartbeat practices, not a rename or canonical tracker.

## Best Possible Solution

Keep #14968 open. If maintainers accept the rename, implement it as a compatibility-aware migration: add `Proactive.md` support across prompts, workspace templates/bootstrap, heartbeat runner file reads, lightweight context filtering, docs, config help, and tests while preserving `HEARTBEAT.md` as a legacy alias or fallback.
