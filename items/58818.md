## Summary

Keep open. Current `main` has partial adjacent features, but it does not implement the requested guarantee that the last N raw messages survive both compaction and session reset. Resets still mint a fresh `sessionId` and archive the previous transcript, startup context only reloads recent daily memory files, and there is no `session.minTailMessages` / `carryForwardOnReset` config surface.
