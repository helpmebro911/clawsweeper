## Summary

Keep open. Current main has global `agents.defaults.compaction` controls for compaction mode, provider, token budgets, model, notices, and memory flush behavior, but it does not implement the requested `agents.defaults.sessionCompactionLimit` or an equivalent global per-session/message compaction limit. The current `session_status` tool also only accepts `sessionKey` and `model`, so there is no current persisted per-session `compactionLimit` surface to default globally.

## Best Possible Solution

Keep the issue open as a concrete feature request. The likely implementation path is to decide the intended unit and semantics first, then add a typed/schema-validated `agents.defaults` compaction-limit setting, wire it into session creation/reset or the compaction trigger path, document it in the public config and compaction docs, and add regression coverage for default inheritance plus per-session override behavior. Existing `agents.defaults.compaction` and `agents.defaults.contextTokens` should be documented as adjacent token/context-window controls, not treated as fulfilling this requested global session limit.
