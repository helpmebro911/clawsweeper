## Summary

Current main still does not implement the daemon heap-size config or service NODE_OPTIONS injection, and the PR remains a plausible implementation candidate. It should stay open, but needs a rebase and a security-conscious adjustment before maintainer review can merge it.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to refresh it against current main, wire a documented gateway.nodeMaxOldSpaceMb through both gateway and node service install plans, update schema/help/generated docs and focused tests, and construct only a safe heap-specific NODE_OPTIONS value rather than carrying arbitrary user startup flags.
