## Summary

Current main still does not implement a Feishu/Lark option to hide or customize the Agent/Model/Provider runtime footer. The footer is still constructed unconditionally for card replies, threaded into streaming and non-streaming cards, rendered as a card note, and there is no matching Feishu config/schema/manifest/docs/test coverage for a footer-specific setting.

## Best Possible Solution

Keep this issue open. Implement the setting in the Feishu plugin first, likely as `channels.feishu.showRuntimeFooter` or a small `replyFooter` mode/template available at top level and per account; thread it through resolveCardNote, streaming start/close, and non-streaming card delivery; update the Feishu zod schema, plugin manifest metadata, public Feishu docs, and reply-dispatcher tests. Consider a cross-channel presentation contract only if maintainers want the same behavior beyond Feishu/Lark.
