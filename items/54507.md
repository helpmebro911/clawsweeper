## Summary

Current main still unconditionally adds the Feishu/Lark runtime footer to interactive card replies and does not expose a footer-specific hide/customize setting. The issue is recent, concrete, unprotected, and remains actionable in the bundled Feishu plugin rather than being implemented, duplicated, or out of scope.

## Best Possible Solution

Keep this issue open and implement the setting in the bundled Feishu plugin. A conservative path is to add a typed per-channel/per-account option such as `channels.feishu.showRuntimeFooter` or a small `replyFooter` mode/template, expose it in the schema, generated metadata, plugin manifest, and docs, make `resolveCardNote` return `undefined` or templated text when configured, and add Feishu dispatcher tests covering hidden footer behavior for both streaming and non-streaming card replies while preserving raw-text render behavior.
