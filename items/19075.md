## Summary

Keep #19075 open. Current main still hardcodes Brave Search API endpoints, the Brave plugin config schema does not accept a webSearch.baseUrl field, and the related #45843 work is an open implementation attempt rather than shipped behavior. The latest issue discussion also already identified #19075 as the canonical tracker.

## Best Possible Solution

Keep #19075 open as the canonical tracker until the bundled Brave plugin supports a provider-owned baseUrl. The likely fix belongs in extensions/brave: add plugins.entries.brave.config.webSearch.baseUrl to the manifest/UI hints and Brave config type, normalize and validate it, build both /res/v1/web/search and /res/v1/llm/context URLs from the normalized base while preserving the default api.search.brave.com behavior, include the base URL in cache identity, update https://docs.openclaw.ai/tools/brave-search, and add focused tests. PR #45843 or a refreshed successor is the natural implementation path.
