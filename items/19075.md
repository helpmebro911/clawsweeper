## Summary

Keep open. Current `main` does not implement a configurable Brave Search `baseUrl`: the Brave bundled plugin schema exposes `webSearch.apiKey` and `webSearch.mode` only, the runtime still hardcodes Brave API endpoints, and the user-facing docs show no `baseUrl` option. This remains a valid feature request rather than something already fixed or obsolete.

## Evidence

- **Brave plugin config schema lacks baseUrl:** The bundled Brave plugin manifest defines UI hints and schema entries for `webSearch.apiKey` and `webSearch.mode`, but no `webSearch.baseUrl` property.
  - file: [extensions/brave/openclaw.plugin.json:7](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/extensions/brave/openclaw.plugin.json#L7)
