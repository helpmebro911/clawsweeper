## Summary

Keep open. Current main partially addresses the parsing side by decoding Feishu text JSON and stripping bot mentions before command probing, but the reported group-command failure still has live code paths: unmentioned group commands return before command probing, and group commands with no configured command allowlist resolve CommandAuthorized=false, after which whole-message control commands are suppressed instead of delivered as normal text. Related PR #49958 tracks the no-allowFrom authorization part, but it is open and not present on current main; related PR #44118 tracks the unmentioned group-command path.
