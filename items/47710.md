## Summary

Keep #47710 open. Current main still treats WhatsApp login status 408 as a terminal login failure, while only status 515 has socket restart handling. The PR addresses a real remaining bug, but its patch should not be merged as-is because review found missing retry bounds and missing QR callback preservation; the current architecture now points to a bounded retry inside the shared WhatsApp login waiter instead.

## Best Possible Solution

Keep the PR open or ask for a refreshed patch. The best implementation is a bounded 408 retry in `waitForWhatsAppLoginResult`, alongside the existing 515 restart path, preserving `onQr` and `onSocketReplaced`, closing the stale socket, and adding focused tests for 408 retry success, retry exhaustion, and fresh QR propagation.
