## Summary

Keep open. Current main still resolves macOS gateway auth only when `gateway.auth.token` / `gateway.auth.password` are plain strings, while the config contract allows SecretInput/SecretRef values. PR #69494 is a focused active fix for a real desktop browser-proxy auth gap, and the latest maintainer feedback asks for one more regression test on the actual `localConfig()` / browser-proxy endpoint path before merge.

## Best Possible Solution

Keep PR #69494 open. The best maintainer path is to finish the requested regression coverage by threading an internal/defaulted SecretRef resolver through `GatewayEndpointStore.localConfig()` / `_testLocalConfig(...)`, assert token and password SecretRefs resolve on the browser-proxy endpoint path, keep remote credentials string-only unless explicitly changed, then merge once the macOS lane remains green.
