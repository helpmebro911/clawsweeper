## Summary

Keep #48711 open. Current main has memory-search, pre-compaction memory flush, and opt-in Active Memory improvements, but I did not find a high-confidence implementation of the reported default project/task memory reliability contract: durable seed capture, current-state-over-stale recall, open-task semantics, and clearer prompt-cache/session-key validation. The issue is recent, non-protected, and salvageable, so conservative cleanup should not close it.

## Evidence

- **Issue discussion reports a broad trust bug with related state-loss failures:** The provided issue body describes live conversation-gate failures for seed durability, fresh-session drift, completed-task leakage, and misleading long-session-id prompt-cache errors. The only comment links related memory/session failure reports (#47884, #47888, #48400, #48238, #48709), supporting that this is a broader unresolved reliability theme rather than a one-off support request.
- **Current memory-core recall still targets generic memory files, not project/task state semantics:** The memory_search tool description requires semantic search over MEMORY.md, memory/*.md, and optional session transcripts before answering about prior work, decisions, preferences, or todos. That is useful recall plumbing, but it is not a canonical PROJECTS.md/TASKS.md mutation model or an open-task-only retrieval contract.
