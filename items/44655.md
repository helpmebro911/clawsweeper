## Summary

Keep open. Current `main` does not implement Z.AI model discovery from the provider `/models` endpoint. The bundled Z.AI plugin still seeds a fixed catalog, and its only dynamic behavior is a narrow forward-compat path for unknown `glm-5*` ids synthesized from the `glm-4.7` template.

## Evidence

- **Static Z.AI catalog is still hardcoded during onboarding/config seeding:** `ZAI_DEFAULT_MODELS` is a fixed array of named GLM ids and `applyZaiPreset(...)` passes that static list as `catalogModels` for the provider preset.
  - file: [extensions/zai/onboard.ts:14](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/zai/onboard.ts#L14)
  - command: `nl -ba extensions/zai/onboard.ts | sed -n '1,120p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Dynamic resolution is limited to unknown `glm-5*` ids, not provider-wide discovery:** `resolveGlm5ForwardCompatModel(...)` returns `undefined` unless the model id starts with `glm-5`, then synthesizes metadata from the `glm-4.7` template. The provider registers only this `resolveDynamicModel` hook; there is no Z.AI `prepareDynamicModel` hook or `/models` fetch path here.
  - file: [extensions/zai/index.ts:36](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/zai/index.ts#L36)
  - command: `nl -ba extensions/zai/index.ts | sed -n '20,330p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Docs still describe a built-in seeded catalog, not live discovery:** The provider docs say OpenClaw 'currently seeds the bundled `zai` provider with' a fixed table of model refs, then separately note only forward-resolution for unknown `glm-5*` ids.
  - file: [docs/providers/zai.md:85](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/docs/providers/zai.md#L85)
  - command: `nl -ba docs/providers/zai.md | sed -n '80,130p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Generic `/models add` support is not wired for Z.AI bootstrap/discovery:** The custom-model add adapters only cover `openai-codex`, `ollama`, and `lmstudio` here, so there is no existing Z.AI-specific path that fetches upstream model metadata into the picker/config flow.
  - file: [src/auto-reply/reply/models-add.ts:244](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/auto-reply/reply/models-add.ts#L244)
