## Summary

Current main still does not implement the requested replay-context pruning of consecutive assistant NO_REPLY turns. The existing code handles adjacent mitigations only: replay sanitization, heartbeat-pair filtering, history turn limiting, tool-pair repair, display-only chat.history suppression, and preservation of legitimate empty/silent assistant turns. The linked implementation PR #13388 is closed and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #13387 open. Implement a focused replay-context pruning pass near `sanitizeSessionHistory`, `validateReplayTurns`, or immediately after `limitHistoryTurns` that collapses consecutive assistant silent replies while preserving isolated legitimate NO_REPLY turns. Reuse shared silent-token detection such as `isSilentReplyPayloadText`, cover text blocks, legacy string content, JSON silent envelopes, and empty-content silent stop turns, and only add config/docs if maintainers decide this should be user-configurable.
