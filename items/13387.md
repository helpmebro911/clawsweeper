## Summary

Current main still does not implement auto-pruning of consecutive assistant NO_REPLY/silent-reply turns from replay context. The replay pipeline sanitizes, validates, heartbeat-filters, limits history, and repairs tool pairs, but has no run-collapse pass; related mitigations are narrower UI/output/heartbeat behaviors. The linked implementation PR #13388 is closed and unmerged, so the issue should remain open.

## Best Possible Solution

Keep #13387 open. Implement a focused replay-context pruning pass near sanitizeSessionHistory, validateReplayTurns, or immediately after limitHistoryTurns that collapses consecutive assistant silent replies while preserving isolated legitimate NO_REPLY turns. Reuse shared silent-token detection such as isSilentReplyPayloadText, cover legacy string content, text blocks, JSON silent envelopes, and legitimate empty-content silent stop turns, and add tests around replay context rather than only UI history suppression.
