## Summary

Current main still does not implement automatic pruning/collapsing of consecutive assistant `NO_REPLY` transcript turns. The related PR #13388 was closed unmerged, and current code only has adjacent mitigations: heartbeat pair filtering, UI/history display suppression, tool-result context pruning, image-history replay pruning, and empty assistant-turn retry/repair. None collapse repeated silent assistant replies before model replay.

## Best Possible Solution

Keep #13387 open. Implement a focused replay-context pruning pass near `sanitizeSessionHistory`, `validateReplayTurns`, or the post-`limitHistoryTurns` assignment that collapses consecutive assistant silent replies while preserving isolated legitimate `NO_REPLY` turns. Add tests for text-block `NO_REPLY`, legacy string content, and empty-content silent stop turns; update docs/config only if maintainers decide this should be configurable.
