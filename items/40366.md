## Summary

Keep this PR open. Current main still lacks the two central behaviors this PR is trying to add: command delivery does not synthesize a fallback when non-empty raw agent payloads normalize/project to zero deliverable outbound payloads, and Slack draft/preview progress that ends without a delivered final still clears state and returns silently. The related open #45290 is broader delivery recovery work, but the provided context does not establish it as a canonical replacement for this PR's normalized-empty command fallback or Slack preview-only no-final fallback.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rebase. The best path is to port a narrow normalized-empty fallback after outbound projection in `src/agents/command/delivery.ts` or an equivalent shared outbound seam, and a narrow Slack preview-only no-final fallback in `extensions/slack/src/monitor/message-handler/dispatch.ts`. Preserve intentional `NO_REPLY`, tool-only success, block-stream success, same-thread status delivery, draft cleanup, and JSON output behavior, and drop unrelated UI formatting churn before merge review.
