## Summary

Keep this PR open. The checked-out main at aac83e00 still lacks both central behaviors: agent command delivery can normalize a non-empty upstream payload list to zero deliverable outbound payloads without synthesizing a fallback, and Slack preview/progress turns that end without a final reply still clear state and return without an in-thread no-final fallback. The PR discussion is meaningful: automated review found real false-positive risks, and the contributor followed up with targeted guardrails for block-stream success, tool-only/NO_REPLY success, directive-normalized silence, thread-safe delayed status, draft cleanup, and JSON output consistency.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rebase. The best path is to port the normalized-empty command fallback into src/agents/command/delivery.ts after outbound projection, and the Slack preview-only no-final fallback into extensions/slack/src/monitor/message-handler/dispatch.ts, while preserving the guardrails already worked through in review. Ask the contributor to remove unrelated UI formatting churn before merge review. Related #45290 is broader delivery recovery work, but it is not established as the canonical replacement for this PR's normalized-empty command fallback and Slack preview-only no-final behavior.
