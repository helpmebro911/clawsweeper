## Summary

Keep open. Current main still does not implement `--profile-id` for `openclaw models auth login`; the flag exists only on `paste-token`, and the login/persistence path has no field or remap policy for an operator-chosen OAuth profile alias. The related open PR #49315 remains the appropriate implementation candidate in the provided context.

## Best Possible Solution

Keep this issue open as the tracker for explicit OAuth profile aliases. The best path is to review and land #49315 or equivalent work that adds `--profile-id <id>` to `models auth login`, validates the profile ID for the selected provider, plumbs it through `LoginOptions` and auth persistence, rejects ambiguous multi-profile results, and updates CLI docs plus parser/persistence tests.
