## Summary

Keep open. Current main still lacks a `--profile-id` flag for `openclaw models auth login`. The shipped Codex identity work improves automatic profile naming from JWT email or stable token identity, but it does not provide the requested operator-chosen OAuth profile alias. The related direct PR #40608 was closed unmerged, and #49315 remains an open implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep the issue open until a safe implementation lands. The likely path is to add `--profile-id <id>` to `models auth login`, validate provider prefix and collisions, plumb it through `LoginOptions` and provider auth persistence without breaking provider-owned auth state, reject ambiguous multi-profile remaps, then update CLI docs and add parser plus auth persistence tests.
