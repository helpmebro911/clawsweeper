## Summary

Keep open. Current `main` no longer matches the original Codex-overwrites-`default` bug, because OpenAI Codex login now derives identity-based profile names from JWT email or a stable subject. But the specific requested enhancement, a `--profile-id` override on `openclaw models auth login`, is still not implemented in the CLI, auth command path, provider auth context, or docs.
