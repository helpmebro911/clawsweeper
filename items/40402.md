## Summary

Keep open. Current main still does not implement `--profile-id` for `openclaw models auth login`; only `paste-token` accepts that flag. The OpenAI Codex identity work fixes automatic profile naming for many multi-account cases, but it does not provide operator-chosen OAuth aliases. The related open PR #49315 remains the implementation candidate to review or supersede; closed PRs #40608 and #68477 are not present on current main.

## Best Possible Solution

Keep this issue open as the tracker for explicit OAuth profile aliases. Review and land #49315 or equivalent work that adds `--profile-id <id>` to `models auth login`, carries it through login options and provider auth persistence, validates that the requested id belongs to the selected provider, handles collisions and ambiguous multi-profile results safely, and updates CLI docs plus parser/persistence tests.
