## Summary

Keep open. Current main still does not generate `ProcessType=Interactive` in macOS LaunchAgent plists, so the central behavior remains unimplemented. Main partially covers the `WorkingDirectory` concern differently by defaulting macOS gateway installs to the OpenClaw state directory, so this PR likely needs rebase/alignment rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it on current main, preserve or explicitly reject the `ProcessType=Interactive` plist behavior with focused `src/daemon/launchd.test.ts` coverage, and reconcile `WorkingDirectory` with the shipped state-dir default instead of defaulting to `/`. Keep #58061 open until this PR or another canonical ProcessType fix is merged or explicitly declined.
