## Summary

Keep open. Current main does not implement the PR's graceful child-mode supervisor cancellation: agent CLI runs use supervisor child mode, and the supervisor still requests SIGKILL on cancel/timeout. The PR is also not merge-ready as written because the discussion flags its SIGTERM-only approach as missing a bounded SIGKILL fallback, while current main already has stale lock reclamation that should be preserved.

## Best Possible Solution

Keep the item open for maintainer/author follow-up. The best product fix is a narrow supervisor/adapter change for child-mode agent runs that sends SIGTERM first, waits a bounded grace period, then escalates to SIGKILL if the process does not exit, with tests updating the current SIGKILL expectations and preserving existing stale-lock reclamation. If maintainers do not want to salvage this broad PR, close it manually in favor of the canonical #70026 issue or a narrower replacement PR rather than marking it implemented.
