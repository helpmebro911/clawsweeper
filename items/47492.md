## Summary

Keep open. Current main has exec approvals, allowlists, turn-source routing metadata, and limited input provenance for inter-session/system-origin turns, but it does not implement a gateway-enforced verified-human instruction-provenance policy for sensitive targets. The related CaMeL/RFC work is still open and taint-focused, while this issue’s clean-session/autonomous-action case remains a distinct security hardening request.
