## Summary

Keep open. Current main still enforces the 2 MiB local avatar file limit and the current CLI/gateway docs still omit that limit. PR #72172 is a focused docs patch against the right current docs pages and addresses open issue #65312; it should not be closed as implemented or as a stale duplicate of older PR #65675.

## Best Possible Solution

Keep #72172 open as a viable current-main docs fix. Ask for the small CLI note-placement cleanup and wording that clearly says local avatar image files must be under the 2 MiB limit, then merge one canonical PR and close #65312 plus supersede the older duplicate PR #65675 as appropriate.
