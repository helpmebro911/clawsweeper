## Summary

Keep this issue open. Current main still has the reported LiteLLM/OpenAI schema compatibility gap for bundle MCP tools: MCP input schemas are materialized raw, provider schema normalization runs before bundle MCP tools are appended, LiteLLM has no provider schema hook, and the non-strict OpenAI completions path still falls back to the generic normalizer that does not recursively sanitize array-valued JSON Schema `type` fields. The linked #66415 is an open implementation candidate, not a shipped fix.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review #66415 as an implementation candidate, require regression coverage that feeds bundle MCP-style schemas through materialization and the LiteLLM/OpenAI completions request path, and add a post-materialization OpenAI-compatible schema sanitizer or provider-family hook that handles nested array-valued `type`, missing object metadata, and optional fields without changing non-OpenAI provider behavior. Diagnostics should also expose the rejected tool name and schema path.
