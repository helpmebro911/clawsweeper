## Summary

Keep open. Current main still only supports a global model catalog/allowlist at `agents.defaults.models`; per-agent config can set primary/fallback model selection and params, but there is no per-agent model allowlist field and the `/model` enforcement path still resolves allowed models from global defaults. Related #9010 was closed as a duplicate of #7424, so this remains the canonical tracker.

## Best Possible Solution

Keep #7424 open until maintainers implement or explicitly reject an agent-scoped model allowlist. The likely implementation path is to choose a config shape such as `agents.list[].models` or `agents.list[].model.allowlist`, add it to TS types/zod/generated schema/docs, and thread it through `buildAllowedModelSet`, `/model` list/set, session override validation, and UI pickers with tests for per-agent visibility and rejection behavior.
