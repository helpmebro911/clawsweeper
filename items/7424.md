## Summary

Keep #7424 open. Current main still lacks an ordinary per-agent model allowlist: agents can override primary/fallback models, and `agents.defaults.models` acts as a global catalog/allowlist, but there is no `agents.list[]` allowlist schema or agent-scoped `/models`/`sessions.patch` enforcement. Related #9010 is closed as a duplicate of #7424, so #7424 remains the canonical tracker.

## Best Possible Solution

Keep #7424 open as the canonical feature request. Add an agent-scoped model allowlist, likely under `agents.list[].model.allowlist` or `agents.list[].models`, then enforce it anywhere models are listed or selected while preserving documented global `agents.defaults.models` catalog semantics.

## Evidence

- **agent-config-has-no-allowlist:** `AgentConfig` exposes `model?: AgentModelConfig` plus unrelated per-agent settings such as skills, params, runtime, tools, and subagents, but no `models`, `allowedModels`, or `model.allowlist` field for ordinary agents.

## Risks / Open Questions

- The future product decision is how per-agent allowlists should combine with `agents.defaults.models`: inherit, intersect, or replace.
- The implementation will need to cover config types/schema/generated reference docs, `models.list`, `/model`, `sessions.patch`, OpenAI-compatible model overrides, Control UI model pickers, and regression tests.
