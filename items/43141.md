## Summary

Keep this issue open. Current main still does not make command-lane enqueue abort-aware, and the provided GitHub context shows open, unmerged PR #43243 with closing syntax for this issue. That PR should be reviewed, landed, or rejected before cleanup closes the report.

## Best Possible Solution

Review PR #43243 or an equivalent focused patch. The implementation should add AbortSignal support to enqueueCommandInLane, reject immediately if already aborted, remove and reject still-queued entries when the signal fires, clean up abort listeners on all settle paths, and thread params.abortSignal through both session-lane and global-lane embedded-runner enqueue calls. Add command-queue and embedded-runner regression coverage, then close this issue after the fix lands on main.

## Evidence
