## Summary

Keep PR #48377 open. Current main still does not implement the PR’s central behavior: `feishu_doc` create only creates the document, while write/append/insert are the only branches that consume markdown `content`. The PR is a focused, plausible fix for linked issue #48373, though it should add targeted regression coverage and ideally avoid reusing the full replace/write path for a brand-new document.

## Best Possible Solution

Keep the PR open and ask for a small maintainer-quality update: add regression coverage for `feishu_doc` create with `content`, handle explicit empty content deliberately (`p.content !== undefined` if intended), and consider a create-populate helper or `appendDoc`-style path that avoids clearing a brand-new document. The linked bug issue #48373 can remain tied to this PR until the behavior lands or maintainers decide that create-with-content is not part of the Feishu doc contract.
