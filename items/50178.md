## Summary

Keep open. Current main has a provider-specific `reasoning_content` replay fix for DeepSeek V4 and Venice DeepSeek V4, but I did not find equivalent handling wired into the bundled Qianfan provider or generic OpenAI-compatible completions path for the reported Baidu Qianfan tool-call replay failure. The related #71150 discussion is useful evidence for the same bug class, but its implemented fix is scoped to DeepSeek V4 models, not the Qianfan model/provider reported here.
