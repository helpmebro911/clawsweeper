## Summary

Keep this PR open. Current main still suppresses non-forum group tool summaries and tool-start statuses through the shared verbose-progress gate, without any `CommandAuthorized` plus real-control-command exception. The PR remains a focused implementation candidate, but it needs rebase/adaptation to the current dispatch shape and final review of group-visible progress behavior.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. Rebase it onto current main, implement a provider-agnostic authorized group command predicate at the current verbose-progress gate, apply it consistently to tool summaries and tool-start statuses, preserve Slack non-direct and normal group-chatter suppression, and retain regression coverage for authorized, unauthorized, mention-prefixed, forum-topic, and path-like slash inputs.
