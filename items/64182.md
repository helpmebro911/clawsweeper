## Summary

Keep this PR open. Current main still suppresses non-forum group tool summaries and tool-start statuses through a shared verbose-progress gate, with no `CommandAuthorized` plus real-control-command exception. The latest PR context is a focused implementation candidate and the security review found no supply-chain-sensitive file changes.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. Rebase/adapt it to current main, preserve a provider-agnostic authorized group command predicate, apply it consistently to tool summaries and tool-start statuses, keep Slack non-direct and normal group-chatter suppression intact, and retain regression coverage for authorized, unauthorized, mention-prefixed, forum-topic, and path-like slash inputs.
