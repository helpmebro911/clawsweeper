## Summary

Keep #45626 open. Current main still exposes Slack `download-file` but does not include `fileId` in the shared `message` tool schema, and Slack does not add a plugin-owned schema contribution to compensate. The linked #45574 problem remains real on current main, so this focused PR or an equivalent fix is still useful.

## Best Possible Solution

Keep the PR open or land an equivalent small fix on main: add `fileId` to the shared message tool schema, and include a regression test that proves Slack `download-file` can be called through the agent-facing `message` tool schema with `fileId`. The linked #45574 bug should remain tied to this fix until that schema path is covered.
