## Summary

Keep open. Current main still does not implement client-provided Chat Completions tools on `/v1/chat/completions`, while open PR #66278 explicitly remains the implementation candidate for this missing protocol support. Under the review rules, an issue with an open PR using closing syntax should stay open until that PR merges or is closed.

## Best Possible Solution

Keep this issue open and treat PR #66278 as the implementation candidate. The maintainer path is to review, revise, and either land that PR or replace it with an equivalent gateway fix that parses Chat Completions `tools`/`tool_choice`, forwards client tools into the agent run, emits OpenAI-compatible `tool_calls` for non-stream and stream responses, and handles `role: "tool"` follow-up messages with focused tests. After that merges, close this issue as implemented.
