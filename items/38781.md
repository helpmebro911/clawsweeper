## Summary

Keep #38781 open. Current main still has the reported macOS node contract gap: the agent node-exec path unconditionally invokes `system.run.prepare`, while the Swift macOS companion node does not define, advertise, or dispatch that command. Related PR #69725 is relevant alternate work, but it is open/unmerged in the provided context and no equivalent fallback or Swift prepare implementation is present on current main.

## Best Possible Solution

Keep #38781 open until maintainers either merge a direct Swift macOS `system.run.prepare` implementation or intentionally supersede it with a merged fallback such as #69725. The best direct fix is to align the Swift macOS node with the TypeScript node-host contract: define the prepare command and payload types, advertise `system.run.prepare`, dispatch it in `MacNodeRuntime`, return the canonical approval-plan shape expected by `executeNodeHostCommand`, and add focused macOS/agent tests for declared commands, prepare payload shape, wrapper/rawCommand consistency, and path hardening.
