## Summary

Keep this PR open. Current main still has no `neverReply` config or equivalent no-LLM group suppression behavior, and the plugin-first alternative is not yet a fully shipped replacement because the related global `inbound_claim` dispatch work is still open in #71988. The PR also remains an active, concrete implementation with tests and user discussion, so it is not safely closeable as implemented, duplicate/superseded, or ClawHub-only yet.

## Best Possible Solution

Keep this PR open while maintainers choose the architecture. The cleanest path is either to land a corrected generic plugin seam (#71988 or equivalent) and move listen-only/passive observation to the already-posted `https://github.com/flowolforg/openclaw-listen-only` plugin/ClawHub path, or to refactor this PR into a smaller core-owned `neverReply` seam with focused channel contract tests and docs. Do not close this PR until one of those paths is accepted or shipped.
