## Summary

Keep PR #39322 open. Current main still does not implement legacy `imsg`/`channels.imessage` tapback reaction forwarding, and the related feature issue #39031 remains open rather than being shipped or superseded. The PR is a plausible implementation attempt, but it still needs maintainer review around the imsg reaction RPC contract and group mention-gating behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to refresh it against current main, verify the current `imsg` reaction RPC contract, add the `includeReactions` config/schema/docs surface, add parser and monitor tests, and explicitly define group reaction mention-gating semantics. If maintainers decide legacy `imsg` should not gain this feature, supersede the PR with a clear product decision on #39031 rather than closing it as already implemented.
