## Summary

Current main still does not implement the requested user-facing framework control to suppress sub-agent announcements. The public sessions_spawn schema/docs do not expose announce/announcePolicy/silent/suppressAnnounce, the config schema has no subagent announce policy, and the announce flow still suppresses only on model output tokens or internal lifecycle flags.

## Best Possible Solution

Keep #8299 open until OpenClaw either adds a documented config-backed sub-agent announce policy and/or sessions_spawn parameter, or maintainers explicitly replace it with a broader canonical announce routing/dispatch design. The implementation should update the public tool schema, config types/zod/generated schema/help, docs, and regression tests around cleanup/retry semantics.
