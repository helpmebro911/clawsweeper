## Summary

Keep #8299 open. Current main still lacks a documented config option or public `sessions_spawn` parameter that deterministically suppresses sub-agent completion announcements. The implementation still supports suppression only through exact model/token output (`ANNOUNCE_SKIP` or `NO_REPLY`) or internal lifecycle state, while top-level completion delivery still makes a follow-up agent call with external delivery. The linked implementation PRs #8306 and #8393 were closed unmerged, and adjacent routing issues do not supersede this specific suppression request.

## Best Possible Solution

Keep #8299 open until OpenClaw adds a documented framework-level sub-agent announce suppression control, either under `agents.defaults.subagents` and/or as a public `sessions_spawn` parameter. A complete fix should update TypeScript config types, zod/generated schema/help, the public tool schema/parser, registry and delivery contracts, docs at `docs/tools/subagents.md`, and regression tests proving deterministic skip behavior without relying on `ANNOUNCE_SKIP` or `NO_REPLY`. Coordinate final naming with adjacent announce routing/orchestration work, but do not treat those issues as superseding this narrower suppression request.
