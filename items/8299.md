## Summary

Keep #8299 open. Current main still does not provide a documented config option or public `sessions_spawn` parameter that deterministically suppresses sub-agent completion announcements. Suppression remains model/token-driven (`ANNOUNCE_SKIP` / `NO_REPLY`) or tied to internal routing state, and the prior implementation PRs in the provided discussion were closed unmerged.

## Best Possible Solution

Keep #8299 open until OpenClaw adds a documented framework-level sub-agent announce suppression control, either under `agents.defaults.subagents` and/or as a public `sessions_spawn` parameter. A complete fix should update TypeScript config types, zod/generated schema/help, the public tool schema/parser, registry and delivery contracts, docs at `docs/tools/subagents.md`, and regression tests proving deterministic skip behavior without relying on `ANNOUNCE_SKIP` or `NO_REPLY`.
