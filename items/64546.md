## Summary

Keep this PR open. Current main still derives the Mattermost interaction signing secret from the bot token using the public HMAC key, and this PR is the active implementation candidate for open bug #64545. The PR diff is security-relevant and scoped, but its random in-memory fallback still needs revision because Mattermost button signing and gateway verification can run in different processes or across gateway restarts.

## Best Possible Solution

Keep this PR and #64545 open. Revise the Mattermost fix so interaction signing uses a persistent per-account secret shared by send and gateway monitor paths, supports config/SecretRef or credential-store persistence, handles existing deployments intentionally, updates schema/types/docs/setup/secret metadata, and adds regression coverage for both bot-token forgery resistance and CLI/gateway cross-process verification before merge.
