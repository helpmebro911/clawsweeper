## Summary

Current main still does not implement the requested `agents.setDefault` Gateway RPC. The handler surface, advertised method list, ADMIN scope map, protocol schemas/exports, and protocol docs all stop at the existing agent CRUD/file helpers, and the Control UI still stages `agents.defaultId`, matching related open bug #59070.

## Best Possible Solution

Keep this issue open. Implement an additive core Gateway method `agents.setDefault` with params/result schemas, protocol exports, advertised method entry, ADMIN scope entry, docs, and focused handler tests. The handler should reject unknown configured agent ids, update `agents.list[]` so only the selected entry has `default: true`, persist config, and return `{ ok: true, defaultId }`. Then update the Control UI Set Default action to call this RPC, which should also address related #59070.
