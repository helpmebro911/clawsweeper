## Summary

Current main still does not implement the requested `agents.setDefault` Gateway RPC. The server handlers, advertised method list, ADMIN scope map, protocol schemas, and protocol docs only expose `agents.list/create/update/delete` plus agent file helpers. The existing Control UI path still stages invalid `agents.defaultId`, which matches the related open bug #59070 rather than resolving this request.

## Best Possible Solution

Keep this issue open. Implement an additive core Gateway method `agents.setDefault` with params/result schemas, protocol exports, method registry entry, ADMIN scope entry, docs, and handler tests. The handler should reject unknown configured agent ids, set only the selected `agents.list[]` entry to `default: true`, clear or omit `default` on the others, write config, and return `{ ok: true, defaultId }`. The Control UI Set Default action should then call this RPC instead of staging `agents.defaultId`, which should also address the related #59070 invalid-config path.
