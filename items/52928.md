## Summary

Current main still does not implement the requested `agents.setDefault` Gateway RPC. The core Gateway surface exposes `agents.list`, `agents.create`, `agents.update`, and `agents.delete`; `agents.update` cannot set the per-agent `default` flag, the method registry and ADMIN scope do not include `agents.setDefault`, and the Control UI Set Default path still writes the invalid `agents.defaultId` config key described by related #59070.

## Best Possible Solution

Keep this issue open and implement an additive core Gateway method. Add typed `agents.setDefault` params/result schemas, register it in the method list and ADMIN scope, implement the server handler to mark only the requested configured agent as `default: true`, reject unknown agent ids, return `{ ok: true, defaultId }`, update protocol docs, and switch the Control UI Set Default action away from writing `agents.defaultId`. Add focused server-method tests and a UI/controller test for the Set Default path; this should also resolve the related #59070 invalid-config symptom.
