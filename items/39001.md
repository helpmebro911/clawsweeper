## Summary

Keep PR #39001 open. Current main still expands Feishu `merge_forward` API items without preserving each forwarded sub-message sender and without resolving item-level `@_user_N` mention placeholders, while the PR remains a focused bundled Feishu plugin fix and its latest head addresses prior review edge cases.

## Best Possible Solution

Keep PR #39001 open for maintainer review and land this patch or an equivalent bundled Feishu plugin change in `extensions/feishu/src/bot-content.ts`, with focused regression tests in `extensions/feishu/src/bot.test.ts` or a colocated helper test covering sender IDs, forwarded mention normalization, key-collision ordering, object/string/union_id IDs, and literal replacement text.
