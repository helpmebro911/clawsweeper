## Summary

Keep this PR open. Current main still lacks Unicode smart-quote delimiter repair for tool-call argument JSON, and the related merged #70294 only enabled the existing malformed-argument wrapper for openai-completions; it did not implement this parser behavior. The PR is outdated and needs rebase/tests, but it targets a real remaining gap tied to the still-open #67488.

## Best Possible Solution

Keep this PR or an equivalent follow-up open. Rebase it onto current main, move the delimiter-aware scanning to the current shared/helper structure as appropriate, preserve #70294's openai-completions enablement, and add focused tests for U+201C/U+201D structural delimiters plus content-level smart quotes inside ASCII strings with trailing junk before maintainer review/merge.

## Risks / Open Questions

- The PR is based on an older shape of the repair code; current main now imports the balanced JSON helper from `src/shared/balanced-json.ts`, so the branch needs a careful rebase rather than direct merge.
- The PR currently changes parser behavior without adding regression tests in the diff; maintainers should require tests for Unicode structural delimiters and for ASCII-delimited content containing smart quotes plus trailing junk.
- Because this path repairs tool-call arguments before tool execution, the implementation should remain fail-closed with strict prefix/suffix limits and should not broaden arbitrary malformed JSON acceptance.
