## Summary

Keep #9394 open. Current main still does not support iMessage rich-text attributes, text sizing, or the requested Shake/Nod/Explode/Ripple/Bloom/Jitter text-animation effects. Legacy imsg delivery is plain text/media/reply oriented, and BlueBubbles has only narrow classic effectId support while stripping Markdown before send. The existing issue discussion already kept this open for the same gaps, and current main still matches that assessment.

## Best Possible Solution

Keep #9394 open for maintainer product/API direction. The likely path is a typed BlueBubbles/iMessage rich-text and text-effect send surface covering supported attribute spans, text size, and animation/effect enums, wired through outbound delivery and message actions with tests and docs. Legacy imsg should either gain upstream-supported capability handling or document these attributes as unsupported.
