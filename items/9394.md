## Summary

Keep open. Current main still only partially addresses the request: BlueBubbles has a `sendWithEffect` action for classic iMessage effects, but legacy `imsg` iMessage still sends plain text/media/reply metadata, BlueBubbles strips Markdown formatting before delivery, and there is no implementation for rich text attributes, text size, or the requested Shake/Nod/Explode/Ripple/Bloom/Jitter text animations.

## Best Possible Solution

Keep #9394 open until a maintainer decides the product direction and the missing behavior is implemented or explicitly documented as unsupported. The most concrete path is to add a typed BlueBubbles/iMessage rich-text and Apple text-effect send surface, wire it through message actions/outbound delivery with tests and docs, and clarify the legacy imsg limitations if upstream imsg cannot carry those attributes.
