## Summary

Keep #9394 open. Current main still lacks iMessage rich-text attributes, text sizing, and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter text-animation effects. Legacy imsg outbound remains plain-text/media/reply oriented, while BlueBubbles only has partial classic effectId support and strips Markdown before send. The existing issue comment by steipete already kept this open for the same gaps, and current main still matches that assessment.

## Best Possible Solution

Keep #9394 open for maintainer product/API direction. The likely implementation is a typed BlueBubbles/iMessage rich-text and text-effect send surface covering supported attributes, text size, and animation/effect enums, wired through outbound delivery and message actions with tests and docs. Legacy imsg should either gain upstream-supported capability handling or be documented as unsupported for those attributes.
