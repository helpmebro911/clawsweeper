## Summary

Keep this issue open. Current main still routes `openclaw models set` through the generic config replacement path, while the config writer still rejects payloads that shrink below half the previous file size as `CONFIG_WRITE_REJECTED`; no current source, test, doc, or recent history evidence shows a narrow models-set fix has landed.

## Best Possible Solution

Keep this issue open. The likely fix belongs in the models command/config mutation path or in a narrowly scoped config-writer exception that recognizes a valid semantic update to `agents.defaults.model.primary` while preserving the size-drop guard for real whole-file clobbers. Add a regression test that exercises a real `openclaw models set` write against a valid shrink-prone config instead of mocking `updateConfig`.
