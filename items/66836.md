## Summary

Keep this PR open. Current main still does not implement the requested sandbox initialization timeout, public `agents.defaults.sandbox.initTimeoutMs` config, cooperative backend startup cancellation, or targeted timeout regression coverage. The PR is the active successor to closed, unmerged PR #33556, so it remains a plausible implementation candidate, but its current diff still needs maintainer review for abort propagation and a security-sensitive sandbox browser hash rollback before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The maintainer path is to request a revised patch that adds and documents `agents.defaults.sandbox.initTimeoutMs`, wires it through config types/schema/generated docs as needed, wraps sandbox initialization in a bounded timeout, makes cancellation a real backend startup contract, threads the signal through Docker/SSH/OpenShell/prune/browser operations, preserves the current sandbox browser security hash epoch, keeps focused timeout/abort regression tests, and splits unrelated OpenShell e2e guard changes unless they are required for the fix.
