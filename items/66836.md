Keep this PR open. Current main still does not implement a configurable sandbox initialization timeout or cooperative startup cancellation, and this PR remains the active implementation candidate superseding closed PR #33556. It should get maintainer review and revision rather than conservative cleanup because the PR diff still has correctness and security-review concerns.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best maintainer path is to request a revised patch that adds and documents `agents.defaults.sandbox.initTimeoutMs`, wires it through public config types, strict schema, generated schema/help/docs as needed, wraps `resolveSandboxContext` in a bounded init timeout, makes startup cancellation effective through backend factories, Docker/SSH/OpenShell startup, prune, browser image/network/container setup, CDP polling, and port reads, preserves the current sandbox browser security hash epoch, and keeps focused timeout/abort regression tests.
