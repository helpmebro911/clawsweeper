## Summary

Keep open. Current `main` still does not implement a configurable delay between consecutive model/provider API requests. The existing knobs cover provider request transport overrides, auth-profile cooldowns, and fallback/rotation behavior, but not a user-configurable inter-request delay. Related PR #49221 tracks an attempted implementation and remains open, so the feature request is still substantive rather than conservative cleanup.
