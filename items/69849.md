## Summary

Keep #69849 open. Current main has moved to @agentclientprotocol/sdk 0.20.0, but the latest SDK package still emits generated Zod schemas with optional array fields that lack `.default([])`, and OpenClaw has no ACP/Claude schema-normalization layer or regression test proving this report is obsolete.

## Best Possible Solution

Keep the issue open and ask maintainers to choose the product boundary: either add a provider-gated Claude/Anthropic schema normalizer that fills missing defaults for optional array schemas before tool registration, or track an upstream @agentclientprotocol/sdk fix and bump once released. A focused regression should cover the affected ACP lifecycle/tool schemas against Claude-style strict validation.
