## Summary

Keep this issue open. Current main still supports ordinary skill metadata, skill visibility/status, heartbeat tasks, and generic cron jobs, but it does not implement declarative SKILL.md schedule/event trigger metadata, automatic skill-to-cron reconciliation, lifecycle-event skill triggers, last-triggered skill status, or a skills audit command. The related context covers nearby frontmatter robustness and model-routing work, not a canonical shipped tracker for this trigger contract.

## Best Possible Solution

Keep the issue open for maintainer product/API design. The strongest path is either a versioned `metadata.openclaw.triggers` contract with validation, cron reconciliation, lifecycle event mapping, persisted last-triggered state, `openclaw skills list/info/audit` output, docs, and tests, or a deliberate plugin-first API that lets a ClawHub plugin inspect skill metadata/status and reconcile cron jobs without importing private `src/agents/**` internals.
