## Summary

Current main still does not implement declarative schedule/event trigger metadata for SKILL.md, automatic skill-to-cron registration, last-triggered skill status, or an `openclaw skills audit --never-triggered` surface. The existing cron and plugin hooks are adjacent but not enough to satisfy the issue because the public skill metadata/status contract does not expose normalized trigger data or trigger history.

## Best Possible Solution

Keep this open for maintainer product/API design. A strong path is to add a versioned `metadata.openclaw.triggers` contract to the SKILL.md metadata parser and docs, validate schedule/event forms, reconcile schedule triggers into gateway cron jobs, map approved lifecycle events to skill execution, persist last-triggered history, and expose the result through `openclaw skills list/info/audit`. If maintainers want this outside core, first add a public plugin API for normalized skill metadata/status discovery and cron reconciliation so a ClawHub plugin can implement it without private imports.
