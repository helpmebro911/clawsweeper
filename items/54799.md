## Summary

Keep this issue open. Current main still lacks declarative SKILL.md trigger metadata, scheduler reconciliation from skill metadata, event-triggered skill registration, last-triggered skill status, and an `openclaw skills audit --never-triggered` surface. Cron, heartbeat tasks, hooks, and task auditing are adjacent systems, but none provide the requested SKILL.md metadata/status contract.

## Best Possible Solution

Keep this open for maintainer product/API design. A strong path would add a versioned `metadata.openclaw.triggers` contract for SKILL.md, validate schedule and event trigger forms, reconcile schedule triggers into managed cron jobs, define approved lifecycle events for event-triggered skills, persist last-triggered history, and expose it through `openclaw skills list`, `openclaw skills info`, and a new skill audit surface. If maintainers prefer ClawHub/plugin ownership, first add the missing public skill metadata/status and cron reconciliation APIs so an external plugin can implement it without private imports.
