## Summary

Current main already makes bundled runtime dependency installs hermetic for the reported non-isolated staging root path. `installBundledRuntimeDeps` now creates an OpenClaw-owned `package.json` in the install execution root before invoking npm, always runs npm with that root as `cwd`, strips global/prefix npm config, and has a regression test covering a parent directory with unrelated package context.

## Best Possible Solution

Close this as implemented on main. Keep the current unconditional staging `package.json` creation plus project-local npm env hardening, and let the fix ship with the next OpenClaw release; reopen only if the issue reproduces on a build containing `a87edd732d369a79308038c620e810469de58532` or newer.
