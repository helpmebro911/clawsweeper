## Summary

Current main still does not provide one canonical outbound message.send path that is simultaneously hook-bearing, deliverable, and auditable. The same author also has open draft PR #63007 targeting the remaining gateway identity and guarded-denial gaps, so this issue should stay open until that PR is reviewed/landed or replaced by a maintainer-approved canonical send contract.

## Best Possible Solution

Keep this issue open. Review and either land #63007 or replace it with a maintainer-approved canonical outbound send contract that threads agent/session/request identity through message_sending, returns a real provider delivery result on success, returns an explicit blocked/denied result on hook cancellation, and aligns direct CLI, gateway send, and plugin send action behavior or exposes the missing plugin API needed to do so.

## Evidence

Its body says it makes gateway call send sufficient for one truthful guarded outbound path by passing agentId/sessionKey into message_sending and surfacing guarded cancellation explicitly. That is an implementation candidate, not a reason to close this issue before merge.
