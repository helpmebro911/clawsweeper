## Summary

Keep this issue open. Current main still lacks one canonical outbound send path that is simultaneously hook-bearing, deliverable, and auditable: gateway send still collapses empty delivery into `No delivery result`, `message_sending` cancellation is represented as a skipped payload rather than a structured blocked outcome, the hook call still lacks outbound agent/session/request identity, and Telegram direct CLI sends can be handled by the plugin action runtime before the core delivery path. The same author also has open draft PR #63007 targeting these gaps, so this issue should remain open until that PR is reviewed, landed, or explicitly superseded.

## Best Possible Solution

Keep this issue open. Review and either land #63007 or replace it with a maintainer-approved canonical outbound send contract that threads agent/session/request identity into `message_sending`, returns real provider delivery results on success, returns explicit guarded-denial results on hook cancellation, and aligns direct CLI, gateway send, and plugin action behavior or documents the intentional canonical path.
