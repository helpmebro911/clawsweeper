## Summary

Keep this issue open. Current main still has no skill-level model routing contract in SKILL.md frontmatter or `metadata.openclaw`, and adjacent routing paths are session/run, cron, subagent, or plugin-hook level rather than enforced per skill invocation. The related open PR #58142 remains a candidate implementation, not proof that the issue is fixed.

## Best Possible Solution

Keep this issue open as the canonical product request. The best path is to review and land, revise, or replace PR #58142 with a maintainer-approved implementation that adds a documented skill metadata contract, parses and persists preferred model/fallback fields, applies them at skill invocation/read time, defines precedence and failure behavior, and adds focused tests for frontmatter parsing, slash skill invocation, prompt/read routing, and interaction with existing model-routing hooks.
