## Summary

Keep #23585 open. Current main still requires the configured trusted-proxy identity header and rejects requests with `trusted_proxy_user_missing` when it is absent; there is no accepted `defaultUser` or `fallbackUser` config field, and the related discussion distinguishes this from #25293 and #17761 rather than superseding it.

## Best Possible Solution

Keep #23585 open. If maintainers accept the feature, implement an explicit trusted-proxy fallback/default user option across `GatewayTrustedProxyConfig`, zod and generated schemas, config help/docs/security audit guidance, and auth tests; only apply it after trusted proxy source and required-header checks pass, and define how `allowUsers` applies to the fallback identity. If maintainers reject fallback identity on security grounds, document Cloudflare Zero Trust IP-whitelist/no-identity WebSocket setups as unsupported.
