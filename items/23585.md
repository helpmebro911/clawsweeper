## Summary

Keep open. Current main still requires the trusted-proxy identity header and returns trusted_proxy_user_missing when it is absent; the config contract has no defaultUser/fallbackUser option, and related discussion says #25293 did not fix this and #17761 is a separate trusted-proxy fallback problem.
