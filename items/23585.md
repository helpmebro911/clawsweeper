## Summary

Keep open. Current main still fail-closes trusted-proxy auth when the identity header is missing, and there is no `defaultUser`/`fallbackUser` support in the gateway auth schema, implementation, docs, or tests.

## Evidence

- **Trusted-proxy auth still rejects missing identity header:** `authorizeTrustedProxy()` returns `trusted_proxy_user_missing` when the configured user header is absent or blank, and `authorizeGatewayConnect()` forwards that failure without any fallback user path.
  - file: [src/gateway/auth.ts:263](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/gateway/auth.ts#L263)
  - command: `nl -ba src/gateway/auth.ts | sed -n '230,335p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Public config type has no fallback/default user field:** `GatewayTrustedProxyConfig` only exposes `userHeader`, `requiredHeaders`, and `allowUsers`. There is no `defaultUser` or `fallbackUser` field on current main.
  - file: [src/config/types.gateway.ts:124](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/config/types.gateway.ts#L124)
  - command: `nl -ba src/config/types.gateway.ts | sed -n '120,170p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Runtime schema also lacks fallback/default user support:** The Zod schema for `gateway.auth.trustedProxy` only allows `userHeader`, `requiredHeaders`, and `allowUsers`, so current config validation would reject the requested feature shape.
  - file: [src/config/zod-schema.ts:720](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/config/zod-schema.ts#L720)
  - command: `nl -ba src/config/zod-schema.ts | sed -n '716,732p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Docs still describe missing user header as an error to fix at the proxy:** Trusted-proxy docs still list `userHeader` as required, explain `trusted_proxy_user_missing` as a missing/empty header problem, and do not document any fallback user behavior.
  - file: [docs/gateway/trusted-proxy-auth.md:90](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/docs/gateway/trusted-proxy-auth.md#L90)
