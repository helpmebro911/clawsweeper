## Summary

Keep open. Current `main` still treats zombie detection as Linux-only, so the reported macOS stale-lock path is not implemented on `main`. The submitted PR also needs maintainer cleanup before it would be safe to land: it includes committed runtime log files and its added macOS test does not verify zombie detection behavior.

## Evidence

- **Current main is Linux-only:** `isZombieProcess()` immediately returns `false` on every non-Linux platform, so current `main` still has no macOS zombie detection in the shared PID liveness helper.
  - file: [src/shared/pid-alive.ts:11](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/shared/pid-alive.ts#L11)
  - command: `nl -ba src/shared/pid-alive.ts | sed -n '7,20p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Session stale-lock cleanup depends on PID liveness:** Session lock inspection marks a lock stale only when `isPidAlive(pid)` is false; a zombie PID reported as alive will block stale-lock reclamation.
  - file: [src/agents/session-write-lock.ts:331](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/agents/session-write-lock.ts#L331)
  - command: `nl -ba src/agents/session-write-lock.ts | sed -n '330,353p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Gateway lock resolution also depends on PID liveness:** Gateway owner resolution returns `dead` only when `isPidAlive(pid)` is false, so the missing macOS zombie handling still matters on current `main`.
  - file: [src/infra/gateway-lock.ts:200](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/infra/gateway-lock.ts#L200)
  - command: `nl -ba src/infra/gateway-lock.ts | sed -n '193,205p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **PR contains unrelated runtime logs:** The PR adds `openclaw-2026-03-20.log` and `openclaw-2026-03-21.log`, which are runtime artifacts unrelated to the fix and should not be merged as part of a maintainer cleanup.
  - file: [openclaw-2026-03-20.log](https://github.com/openclaw/openclaw/blob/786d90dfc89f8d0dd868eaa467ccfd81389660aa/openclaw-2026-03-20.log)

- **PR test does not prove the fix:** The added macOS test only asserts that `isPidAlive(process.pid)` does not throw on Darwin; it does not mock or create a zombie state, so it does not verify the new macOS zombie-detection branch.
  - file: [src/shared/pid-alive.test.ts](https://github.com/openclaw/openclaw/blob/786d90dfc89f8d0dd868eaa467ccfd81389660aa/src/shared/pid-alive.test.ts)
