## Summary

Keep this issue open. Current main has restored 24-hour duplicate heartbeat suppression, but the plugin-channel heartbeat fallback remains incomplete: `resolveHeartbeatDeliveryTarget()` can recover the stored session route, then still routes it through outbound plugin resolution and converts an unresolved/unbootstrapped plugin channel into `channel: "none", reason: "no-target"`. The related #46203 is closed and unmerged, so it does not supersede the remaining work.

## Best Possible Solution

Keep this issue open and narrow it to the remaining plugin-channel heartbeat fallback. The best fix is in `src/infra/outbound/targets.ts`: make `resolveHeartbeatDeliveryTarget()` preserve a trusted stored plugin route for `target: "last"` when session metadata has a plugin channel and destination but outbound plugin normalization fails because the registry is empty or unbootstrapped. Keep the restored duplicate suppression unchanged, and add regression coverage using the real registry-backed normalization path with an empty plugin registry so the route cannot degrade to `no-target`.
