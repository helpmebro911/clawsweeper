## Summary

Keep open. Current main still leaves the Docker label Go-template hardening and thread-ownership URL path-segment hardening unimplemented. The TLS/private-WS and Control UI dangerous flags are documented, tested break-glass behavior, so changing them needs maintainer security/product judgment rather than automated closure.

## Best Possible Solution

Keep this issue open for maintainer security triage. Land narrow hardening for readDockerContainerLabel label validation and thread-ownership path-segment encoding or strict validation, with regression tests. Separately decide whether the private-WS break-glass and Control UI dangerous flag policy or error wording should change, since those are currently documented and tested operator escape hatches.
