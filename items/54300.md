## Summary

Keep open. Current main still contains the two concrete hardening gaps called out in the existing ClawSweeper comment: Docker label names are interpolated into a Docker Go template without validation, and the thread-ownership plugin still concatenates raw channel/thread values into a URL path. The TLS/private-WS and Control UI dangerous flags are documented and tested as intentional break-glass behavior, so that part needs maintainer security/product judgment rather than automated closure.

## Best Possible Solution

Keep this issue open for maintainer security triage. Land a narrow hardening change that validates Docker inspect label names in readDockerContainerLabel, encodes or strictly validates thread-ownership channel/thread path segments with regression tests, and separately decide whether to split or revise the private-WS and Control UI break-glass policy and user-facing error wording.
