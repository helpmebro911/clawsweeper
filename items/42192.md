## Summary

Keep open. Current `main` still intentionally warns on duplicate plugin IDs across precedence buckets, the docs/tests codify that behavior, and this PR still has an unaddressed review finding about missed same-origin duplicate warnings after same-root origin promotion.

## Evidence

- **Current main still emits duplicate warnings after precedence resolution:** `loadPluginManifestRegistry` keeps a single winner by precedence but still pushes a `duplicate plugin id detected` warning for the overridden candidate. Nothing on `main` implements the PR's proposed cross-origin warning suppression.
