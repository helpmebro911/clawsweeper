## Summary

Keep open. Current main still uses the Unix shell/xargs docs-format pipeline that this PR is trying to replace, and no equivalent Node helper has landed. The PR discussion also shows the initial bot review issues were addressed in the branch, so this remains potentially useful implementation work rather than an obsolete or safely closeable PR.

## Best Possible Solution

Keep this PR open for maintainer review or ask the author to rebase/amend it into the canonical fix. The best implementation is a single docs-format wrapper that removes the `xargs` dependency, preserves the current tracked-file set, handles Windows command shims using an existing repo pattern such as `pnpm exec` or Windows `shell: true` where needed, reports child-process launch errors clearly, and ideally covers both `format:docs:check` and `format:docs`. After one corrected PR lands, close the remaining duplicate PRs against that shipped fix and the canonical issue #44293.
