## Summary

Keep open. Current main supports the maintainer-suggested Matrix DM isolation path, but that feature is opt-in, predates the reporter's version, and does not prove the reported WebChat-visible transcript/session failure is fixed. The maintainer request for a fresh isolated-DM repro and logs is still the right next discriminator.

## Best Possible Solution

Keep this issue open. Maintainers should wait for, or request again, the fresh reproduction using the suggested isolated DM config plus gateway logs and session JSONL. If it reproduces, the likely fix belongs around the Matrix monitor to reply-dispatch/transcript visibility path, with a regression proving that a Matrix DM inbound event both routes to the intended session and produces the WebChat/session-list-visible transcript update or a clear logged drop/queue reason.
