## Summary

Keep open. Current main does not implement the requested WhatsApp GIF metadata extraction: inbound video messages still normalize to plain `<media:video>` without checking `gifPlayback` or `contextInfo.externalAdReply`, and no current code references `externalAdReply` or `<media:gif>`. The linked PR #49103 directly attempted the requested fix but is closed unmerged, so it is useful prior work rather than a superseding canonical fix.
