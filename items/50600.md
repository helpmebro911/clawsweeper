## Summary

Keep open. Current main partially addresses the empty Telegram text-send failure by filtering whitespace-only reply chunks before sendMessage, but the requested configurable busy acknowledgment is still not implemented. The public queue config/type/schema only expose mode, per-channel mode, debounce, cap, and drop policy; the follow-up queue state/settings/enqueue path has no busyAckMs, busyAckText, or queue:wait-exceeded hook/timer that can send an interim acknowledgment to the originating channel. Related ClawSweeper reports cover adjacent lane configurability/diagnostics, but none is a canonical replacement for this specific channel busy-ack request.
