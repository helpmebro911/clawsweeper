## Summary

Keep open. Current main has clearer ACP/acpx and Discord binding documentation than the original report, but it still does not implement the requested Pi Gateway -> Mac mini ACP harness execution path. ACP session creation can choose backend, harness agent, mode, model/thinking, and cwd; it cannot choose a remote OpenClaw node as the process host.

## Best Possible Solution

Keep this open for maintainer product/docs judgment. The short-term path is to document that bundled ACP/acpx harnesses run on the Gateway host today, recommend running the Gateway/acpx on the Mac or using an operator-managed wrapper for multi-machine setups, and clarify Discord binding guidance. If OpenClaw should support this topology, add an explicit ACP runtime node selector and node-host process execution contract across config schema, ACP runtime SDK, acpx integration, and Discord `/acp spawn` docs.
