## Summary

Keep open. Current main has clearer ACP docs and better Discord binding behavior, but it still does not implement the reported remote Mac node harness topology. ACP/acpx remains a Gateway-host runtime path with backend, agent, mode, and cwd controls, but no node or remote execution selector.

## Best Possible Solution

Keep this open for maintainer product judgment. The near-term docs path is to explicitly state on the ACP agents and Discord docs that ACP/acpx harnesses execute on the Gateway host today, and recommend running the Gateway/acpx on the Mac or using an operator-managed wrapper if that is the intended deployment. If OpenClaw should support the reported topology, design an explicit ACP runtime node selector plus node-host process execution support rather than overloading cwd, thread bindings, or existing backend ids.
