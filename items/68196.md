## Summary

Keep this PR open. Current main still returns a hard `timeout` from `sessions_send` immediately after `agent.wait` times out, and the proposed post-timeout compensation helper is not present. The PR is paired with open bug #68065 by the same author, uses closing syntax, and has supportive production discussion, so the maintainer path is review/revise/land rather than cleanup-close.

## Best Possible Solution

Keep the PR open and review it as the implementation candidate for #68065. The best code direction is to make post-timeout handling explicit in `sessions_send`: after a successful `agent` RPC acceptance, re-read the latest assistant snapshot to return `ok` for a new post-baseline reply, otherwise return `accepted` for the still-running accepted run, while preserving hard `timeout` for paths without acceptance proof. Keep the focused regression tests, including the `timeoutSeconds: 0` guard, and address the accepted-state proof review comment before landing.
