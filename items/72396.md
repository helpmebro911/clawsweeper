## Summary

Keep open. Current main still treats `OPENCLAW_PLUGIN_STAGE_DIR` as one writable external base directory, then checks and installs against a single resolved install root. The requested layered read-only baseline plus writable persistent stage is not implemented and belongs in core plugin runtime-dependency resolution, not ClawHub.

## Best Possible Solution

Keep #72396 open and implement this in core plugin runtime-dependency handling. The likely path is to extend `src/plugins/bundled-runtime-deps.ts` with an explicit read-roots plus final write-root model, either by parsing `OPENCLAW_PLUGIN_STAGE_DIR` as a delimiter-separated list or by adding a separate read-only env var. Missing-dependency checks and alias/loader registration should search roots left to right, while locks, retained manifests, npm installs, and cleanup writes stay scoped to the final writable root. Add regression coverage for read-only baseline hits, writable-root installs for new deps, doctor scan/repair, loader aliases, and docs for Docker/systemd operators.
