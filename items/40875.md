## Summary

Keep this PR open. The heartbeat docs still lack fallback-specific guidance, but the PR is stale against current docs structure and its wording is inaccurate against current fallback behavior: different-provider overrides skip unrelated configured fallbacks, but override runs can still fall through to the configured primary. Security review found no code-execution or supply-chain surface in the provided docs/changelog-only diff.

## Best Possible Solution

Keep this PR open and request a narrow revision against current main: add the fallback note inside the current `heartbeat.model` `ParamField`, align it with `docs/concepts/model-failover.md`, say no override and same-provider overrides follow the standard fallback chain, say different-provider overrides skip unrelated configured fallbacks unless the override is already in the configured fallback chain, and mention that override runs can still fall through to `agents.defaults.model.primary`. Fix the changelog attribution or remove the changelog entry.
