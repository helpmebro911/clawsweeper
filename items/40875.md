## Summary

Keep this PR open. Current main still lacks the heartbeat-specific fallback clarification, but the proposed wording is not accurate for current fallback behavior because cross-provider heartbeat overrides can still try the configured primary as the final candidate. The related #23718 is a separate open feature request for richer heartbeat.model fallback configuration, not a duplicate of this docs correction.

## Best Possible Solution

Keep this PR open and request a narrow revision against current main: update the `heartbeat.model` field note to say no override and same-provider overrides use the standard fallback chain; different-provider overrides skip unrelated configured fallbacks unless the override is itself in `agents.defaults.model.fallbacks`; and override runs can still end at `agents.defaults.model.primary` as the final candidate. Link or align with the public model failover docs, and fix or remove the changelog entry. Treat #23718 separately as the product request for explicit per-heartbeat `{ primary, fallbacks }` support.
