## Summary

Current main does not yet implement the PR’s core request. `chat.history` still reads transcript rows, projects them through the shared display projection, and that projection does not drop synthetic compaction markers or pre/post-compaction memory artifacts. The related #32068 was closed unmerged, so it is useful discussion but not a shipped or canonical open fix. PR #70348 also has a valid review concern: its raw text-prefix filtering can hide legitimate user-authored content, so the work should stay open for a safer implementation rather than be conservatively closed.

## Best Possible Solution

Keep the item open, but revise the implementation around the shared `src/gateway/chat-display-projection.ts` path. A safer fix would filter synthetic compaction messages by `__openclaw.kind === "compaction"` only if maintainers decide those dividers should not be visible in `chat.history`, and handle memory-flush/post-compaction artifacts via explicit internal provenance or other metadata rather than raw text prefixes. Add gateway-level `chat.history` regression coverage plus false-positive tests proving normal user-authored text with those prefixes remains visible.
