## Summary

Keep open. Current main has not implemented this PR's requested filtering: `chat.history` still projects `readSessionMessages()` output through the shared display projection, while that projection only drops assistant control replies, empty text-only user messages, and heartbeat noise. The PR should likely be reworked because its text-prefix filter has a real false-positive risk, but the underlying gateway history cleanup is still unresolved.

## Best Possible Solution

Keep the PR open but request a safer implementation in the shared gateway chat display projection: hide synthetic `__openclaw.kind="compaction"` artifacts if they should not be user-visible, and handle memory-flush/post-compaction artifacts using provenance or synthetic metadata instead of broad raw text prefixes. Add gateway-level tests proving internal artifacts are hidden while user-authored messages with similar text remain visible.
