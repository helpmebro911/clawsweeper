## Summary

Keep open. Current main has provider lists, media capability tables, plugin compatibility/deprecation tracking, model-catalog row statuses, and runtime auth/probe statuses, but it still does not publish the requested provider x model-family x capability compatibility matrix with stable/experimental/deprecated/broken lifecycle states, deprecation deadlines, tracked issue links, or doctor output.

## Best Possible Solution

Keep the issue open until maintainers approve and ship a provider compatibility policy plus a machine-readable matrix. A good implementation would extend provider/plugin model metadata or add a dedicated provider-compat registry with provider, model family, capability, lifecycle status, deprecation deadline, replacement, and canonical issue links, then generate public docs and surface a concise view in doctor or models status while keeping live auth probes separate from lifecycle support state.
