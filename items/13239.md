## Summary

Keep #13239 open. Current main still has provider docs, model catalog metadata, live auth/probe status, and plugin compatibility/deprecation policy, but not the requested public provider x model-family x capability compatibility matrix with stable/experimental/deprecated/broken lifecycle states, deadlines, tracked issue links, generated docs, and doctor output. The linked Matrix plugin issue (#52899) is useful motivation, but it was a specific closed plugin regression and does not supersede this broader provider policy request.

## Best Possible Solution

Keep the issue open until maintainers approve and ship a provider compatibility policy plus a machine-readable registry keyed by provider, model family, capability, lifecycle status, deadline, replacement, and canonical issue link. Generate the public docs page from that registry and surface a concise doctor or models-status view while keeping live auth/probe health separate from support lifecycle state.
