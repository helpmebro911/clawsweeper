## Summary

Keep #13239 open. Current main still does not have the requested public provider x model-family x capability compatibility matrix with stable/experimental/deprecated/broken lifecycle states, deadlines, tracked issue links, generated docs, and doctor output. The related Matrix plugin issue #52899 is useful motivation, but it was a closed plugin-specific regression and does not supersede this broader provider support policy request. The latest maintainer-side discussion also explicitly kept this item open, and there were no relevant provider/docs/model-status changes between that review point and current HEAD e672b61417af5c45b0431df6d9109a1f4b618ef5.

## Best Possible Solution

Keep the issue open until maintainers approve and ship a provider compatibility policy plus a machine-readable registry keyed by provider, model family, capability, lifecycle status, deadline, replacement, and canonical issue link. Generate a public docs page from that registry and surface concise lifecycle state in CLI doctor or models status output, while keeping live auth/probe health separate from support lifecycle state.
