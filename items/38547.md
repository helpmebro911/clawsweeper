## Summary

Keep #38547 open. Current main still does not provide native remote gateway delegation for `sessions_spawn` or `sessions_send`: the tool schemas do not expose `gatewayUrl`/`gatewayToken`, the subagent and ACP spawn paths dispatch through the current gateway without URL/token overrides, and the docs describe same-gateway subagent orchestration. Existing gateway override plumbing is present for other tools, so this remains an unimplemented core session-tool capability rather than an already-shipped feature.

## Best Possible Solution

Keep #38547 open until maintainers either implement or explicitly reject native remote session delegation. A concrete implementation should add documented `gatewayUrl`/`gatewayToken` support to `sessions_spawn` and `sessions_send`, reuse the existing safe gateway override validation/credential resolution, thread resolved remote call options through native subagent and ACP dispatch, and add tests proving model calls, tool execution, transcripts, and credential/account use occur on the remote gateway.
