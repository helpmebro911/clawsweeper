## Summary

Keep open. Current `main` still resolves implicit providers before any `models.mode` handling, while docs and schema help still say `models.mode: "replace"` uses only configured providers. The reported semantic mismatch is therefore still present on `cc0f3e0`, and the latest release `v2026.4.23` also still contains the same logic.
