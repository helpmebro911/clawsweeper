## Summary

Keep open. Current main still invokes implicit provider discovery before checking `models.mode`, so `replace` mode can still pay discovery cost and merge implicit providers despite the documented configured-providers-only contract. The earlier ClawSweeper comment reached the same conclusion, #68965 is already closed as a duplicate of this issue, and open PR #63552 is related performance work rather than a merged replace-mode fix.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `src/agents/models-config.plan.ts`: branch on `cfg.models?.mode === "replace"` before invoking implicit provider discovery, use only explicitly configured providers on that path, preserve normalization and source-managed secret enforcement, and add regression coverage proving the injected implicit resolver is not called in replace mode while merge/default mode still discovers providers. Review #63552 separately unless it is updated to explicitly satisfy this replace-mode contract.
