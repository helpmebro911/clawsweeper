## Summary

Current main still has the reported configured ACP session-key collision path for Discord threads that inherit a parent-channel binding. The Discord match returns the parent channel id, the configured-binding registry materializes from that match, and the ACP persistent key hashes only the materialized conversation id. A same-author implementation PR, #64322, is still open and should be reviewed or replaced rather than closing this issue.

## Best Possible Solution

Keep this issue open and review, rebase, land, or replace #64322. The implementation should make inherited Discord parent-channel ACP bindings materialize distinct persistent configured ACP session keys per runtime thread, preserve parentConversationId and direct thread bindings, avoid breaking non-Discord canonical binding behavior, and add regression coverage for two Discord thread ids under one configured parent resolving to different target session keys.
