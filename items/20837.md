## Summary

Keep #20837 open. Current main carries channel metadata through ordinary agent runs and has origin-based reply routing, but the slash-command system-prompt rebuild used by `/context` and `/export-session` still drops the available channel/capability/action/shell runtime metadata. The discussion is also useful: evansantos noted the partial channel-awareness gap while working on SlimClaw and opened #21271 for this exact command path, but that PR is closed unmerged.

## Best Possible Solution

Keep the issue open. Implement the missing slash-command runtime metadata in `src/auto-reply/reply/commands-system-prompt.ts`, preferably by reusing or extracting the embedded-runner pattern for machine/shell details, channel normalization, account-aware capabilities, message-tool capabilities, reaction guidance, and channel actions. Add focused regression coverage showing `/context` or `/export-session` from Telegram rebuilds a prompt containing `channel=telegram` and expected capabilities. Track selective prompt-driven reply-routing rules as a separate follow-up once metadata is consistently exposed.
