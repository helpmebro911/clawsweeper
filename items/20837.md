## Summary

Keep open. Current `main` already passes the active message channel and channel capabilities into the normal agent runtime/system prompt, so the core chat flow is at least partially implemented. But the shared `/context` and `/export-session` commands path still builds runtime info with `host/os/arch = "unknown"` and does not pass `channel`, `capabilities`, or `shell`, so the repo does not fully implement the requested channel-awareness surface across all agent-facing prompt/report paths. Current routing code also suggests the old "broadcast to dashboard + telegram" symptom has been addressed separately, but that does not fully resolve this feature request.
