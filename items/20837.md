## Summary

Keep #20837 open. Current main already exposes channel metadata to ordinary agent runs through inbound context and runtime prompt wiring, but the slash-command prompt rebuild used by `/context` and `/export-session` still drops the available command channel and channel capabilities/actions. The linked PR #21271 was closed unmerged, and the latest maintainer-side Codex comment correctly identified the remaining core gap.

## Best Possible Solution

Keep the issue open. Implement the missing slash-command runtime metadata in `src/auto-reply/reply/commands-system-prompt.ts` by mirroring the embedded runner pattern for machine/shell details, channel normalization, account-aware capabilities, message-tool capabilities, reaction guidance, and channel actions. Add focused regression coverage proving `/context` or `/export-session` from Telegram includes channel metadata in the rebuilt system prompt. Track selective reply routing as a separate follow-up once the metadata gap is closed.
