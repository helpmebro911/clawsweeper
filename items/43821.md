## Summary

Keep this issue open. Current main has meaningful partial HTTP(S) env-proxy support, but the broad global proxy request is not fully implemented, and the provided GitHub context shows open PR #43919 still links this issue as an implementation target.

## Best Possible Solution

Keep this issue open while maintainers review #43919 against current main. If the PR is superseded by newer shipped work, close or rebase that PR first, then either close this issue only after verifying the broad behavior is implemented or split the remaining gaps into narrower issues for ALL_PROXY/SOCKS, browser, web tools, model-provider, and channel/plugin proxy paths.
