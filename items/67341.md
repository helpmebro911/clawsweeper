Keep this PR open. Current main still lacks the requested `amazon-bedrock-mantle` `config.discovery.enabled` opt-out, and the linked bug remains a valid implementation target. ## Risks / Open Questions

- The PR patch captures `api.pluginConfig` at registration time; current provider patterns resolve plugin config from the live runtime snapshot, so the implementation likely needs a small adaptation before merge.
- The PR gates only the catalog handler. Passing config into `resolveImplicitMantleProvider` and gating before IAM token generation would protect direct callers and make the behavior easier to unit test.
- The PR currently lacks regression tests for `discovery.enabled: false`, `true`, and unset/default behavior.
