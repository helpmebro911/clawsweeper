## Summary

Current main still keeps the Usage header and Daily Cost chart on different backend summaries. The header is based on `sessions.usage` totals, while the daily chart is still populated from `usage.cost`; `sessions.usage` discovers all configured agents, but `usage.cost` still scans a single agent transcript directory. The linked fix PR #42063 was closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open and fix the gateway/UI contract so the Usage header and Daily Cost chart use the same aggregation scope and cost unit. The strongest path is to revive the core idea from #42063: aggregate `usage.cost` across the configured gateway agents or derive both header and daily chart data from one shared backend summary, then add regression coverage proving multi-agent header totals and daily bars match for the selected range. If legacy MiniMax/DashScope transcripts can store micro-dollar values, add explicit normalization or doctor/repair handling with tests instead of formatting around it in the UI.
