## Summary

Current main still leaves the Usage dashboard backed by two different summaries: the header uses `sessions.usage` totals while the daily chart uses `usage.cost` daily rows. `sessions.usage` discovers configured agents, but `usage.cost` still loads a single default-agent transcript directory, so the reported header/chart mismatch remains actionable. The linked candidate fix #42063 was closed unmerged, and #68066 is a separate OpenRouter billing-accuracy issue rather than a canonical replacement.

## Best Possible Solution

Keep this issue open. The best fix is to make the Usage header and Daily Cost chart derive from one shared backend aggregation scope and one documented cost unit, likely by reviving the useful part of #42063: aggregate `usage.cost` across all configured gateway agents or return both header totals and daily bars from the same gateway summary. Add regression coverage proving multi-agent totals and daily bars match for the selected date range, and add explicit data-layer normalization/repair coverage if legacy provider transcripts can store non-dollar cost units.
