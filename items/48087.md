## Summary

Keep open. Current main has strong adjacent MiniMax auth/baseUrl fixes that likely address the related #37142 MiniMax 401 path, but #48087’s specific report is about an intermittent Control UI-visible provider 401 that clears after page refresh. The issue discussion points to possible credential-source drift or wrong MiniMax key class, and current UI code still persists terminal chat errors in `lastError` until a later send/history reload rather than proving automatic stale-error recovery.
