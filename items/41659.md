## Summary

Keep this PR open. Current main still serves Control UI index HTML without rewriting favicon or apple-touch-icon hrefs for a configured basePath, while the source HTML contains root-relative icon links. The PR is still a focused implementation candidate for the linked #41205 regression, but it needs a rebase onto current main because the index-serving path now also computes CSP hashes.

## Best Possible Solution

Keep this PR open as a valid focused bug-fix candidate. Rebase it onto current main, integrate the favicon basePath rewrite into `serveResolvedIndexHtml` while preserving `computeInlineScriptHashes`/CSP behavior, keep the function replacer, and run the targeted gateway tests plus the changed gate before merge.
