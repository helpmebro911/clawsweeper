## Summary

Keep open. Current `main` still has the underlying launchd handoff behavior, and the submitted PR is not safe to close as implemented: the PR discussion identifies two concrete correctness problems, while the current code still only schedules a detached helper and waits for the current PID to exit. This looks like a real bug that needs a revised fix rather than cleanup closure.

## Evidence

- **Current main only schedules detached handoff for in-process launchd restarts:** `restartLaunchAgent()` still takes the `isCurrentProcessLaunchdServiceLabel(label)` branch by scheduling `scheduleDetachedLaunchdRestartHandoff({ mode: "kickstart", waitForPid: process.pid })` and returning `{ outcome: "scheduled" }`; it does not trigger an in-process restart on current `main`.
  - file: [src/daemon/launchd.ts:672](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/daemon/launchd.ts#L672)
  - command: `nl -ba src/daemon/launchd.ts | sed -n '669,683p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **Detached handoff explicitly waits for caller PID to exit:** The launchd handoff shell script loops on `kill -0 "$wait_pid"` before it runs `launchctl kickstart -k`, so the scheduled helper does nothing until the current process terminates.
  - file: [src/daemon/launchd-restart-handoff.ts:85](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/daemon/launchd-restart-handoff.ts#L85)
  - command: `nl -ba src/daemon/launchd-restart-handoff.ts | sed -n '85,105p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **Authorized SIGUSR1 restarts must go through the restart helper:** `emitGatewayRestart()` increments authorization with `authorizeGatewaySigusr1Restart()` before emitting SIGUSR1. That is the internal contract for a local in-process restart.
  - file: [src/infra/restart.ts:121](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/infra/restart.ts#L121)
  - command: `nl -ba src/infra/restart.ts | sed -n '121,145p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **Gateway run loop ignores unauthorized SIGUSR1 by default:** The SIGUSR1 handler consumes restart authorization first; if not authorized and external restarts are not allowed, it logs `SIGUSR1 restart ignored` and returns without restarting.
  - file: [src/cli/gateway-cli/run-loop.ts:230](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/cli/gateway-cli/run-loop.ts#L230)

  - file: [src/daemon/launchd.ts:465](https://github.com/openclaw/openclaw/blob/2ddc1913fa1323e3aadc8735d8f869596d18951c/src/daemon/launchd.ts#L465)

The current launchd test for this branch only asserts that a detached handoff is scheduled with `waitForPid: process.pid`; it does not cover emitting an authorized restart signal first.
  - file: [src/crestodian/operations.ts:339](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/crestodian/operations.ts#L339)
