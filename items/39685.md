## Summary

Keep open. Current main has SSRF hostname allowlist support and a Codex-native search allowDomains setting, but it does not implement the requested global/agent-level web tool network policy with allowedDomains/denyDomains for web_fetch and managed web_search. The companion filesystem issue #39672 is adjacent but does not supersede this network egress request.

## Evidence

- **ToolsConfig lacks requested web tool domain policy:** The current tools.web.search config only exposes provider/openaiCodex.allowedDomains, and tools.web.fetch.ssrfPolicy only exposes allowRfc2544BenchmarkRange. There is no tools-level net policy, no denyDomains, and no generic allowedDomains for web_fetch/managed web_search.
