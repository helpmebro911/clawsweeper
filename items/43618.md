## Summary

Keep this PR open. Current main still lacks the central operator-facing queue diagnostics surface: no live queue snapshot helper, no `diagnostics.queue` Gateway RPC, and no `openclaw gateway diagnostics queue` CLI command. The related `queue_status` request remains adjacent, not a replacement.

## Best Possible Solution

Keep this PR open for maintainer review or an updated successor. The best path is a small core-owned, read-only queue snapshot seam in `src/process/command-queue.ts`, exposed through `diagnostics.queue` and `openclaw gateway diagnostics queue`, scoped to `operator.read`, integrated with the current diagnostics command layout, documented, and covered by focused queue, Gateway, CLI, and authorization tests. Keep #9797 separate for the agent-facing `queue_status` tool discussion.
