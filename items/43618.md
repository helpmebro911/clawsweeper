## Summary

Keep open. Current main still lacks the PR's central feature: a live queue diagnostics snapshot exposed as `diagnostics.queue` and `openclaw gateway diagnostics queue`. The existing stability diagnostics are bounded event records and docs still direct operators to logs for queue depth, so the PR is not implemented or superseded. Related #9797 is a distinct agent-facing `queue_status` tool request, not a replacement for this operator Gateway/CLI diagnostics surface.

## Best Possible Solution

Keep this PR open for maintainer review or an updated successor. The best path is a minimal core-owned, read-only queue snapshot seam in `src/process/command-queue.ts`, exposed through `diagnostics.queue` and `openclaw gateway diagnostics queue`, scoped to `operator.read`, documented, and covered with focused queue, Gateway, CLI, and scope tests. Keep #9797 separate for the in-agent `queue_status` tool discussion.
