## Summary

Keep open. Current main still matches the report: the message tool scopes the `action` enum and plugin-provided schema contributions, but the shared core property groups are still added unconditionally. The earlier Codex review comment is still accurate against current main SHA 348728c28c1ea9ae7c5824350baba35ee08a275e.

## Best Possible Solution

Keep this issue open and implement the action-aware core schema trim in `src/agents/tools/message-tool.ts`: thread the resolved action set into `buildMessageToolSchemaProps()`/`buildMessageToolSchemaFromActions()`, include each shared core schema group only when active actions can use it, preserve always-needed routing/send fields and cross-channel behavior, and add focused schema tests for minimal send/react-only or disabled-action configurations that assert unsupported core fields are absent while supported fields remain.
