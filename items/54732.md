## Summary

Keep this issue open. Current main still trims the `message` action enum and plugin-provided schema contributions, but the shared core parameter groups are still added unconditionally, so the reported token/schema bloat remains present at c2d82b87eee7a86bbd0e851169e0ca5559c44fe6.

## Best Possible Solution

Implement action-aware shared schema trimming in `src/agents/tools/message-tool.ts`: pass the resolved action set into the shared property builder, include each shared schema group only when one active action can use it, keep stable routing/send/reaction/read fields available, preserve valid cross-channel actions, and add focused tests proving basic send/react configurations omit unsupported poll/sticker/thread/event/moderation/gateway/channel-management/presence fields while supported actions still expose their required parameters.
