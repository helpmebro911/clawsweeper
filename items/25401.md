## Summary

Keep #25401 open. Current main still lacks a provider-level `models.providers.<id>.enabled` contract, and the requested behavior is not implemented in the type/schema, models.json planning, fallback candidate resolution, configured model catalog, or `openclaw models status`. The related implementation PR #25415 was closed unmerged, so the issue remains the active tracker.

## Best Possible Solution

Keep the issue open until OpenClaw adds a documented `models.providers.<provider>.enabled?: boolean` contract that defaults to enabled, preserves disabled config blocks, excludes disabled providers from generated `models.json`, configured catalogs, fallback candidate selection, and probe/status paths, and displays disabled state in `openclaw models status` JSON and rich output. Add focused tests for schema acceptance, models-config planning, fallback behavior, configured catalog/listing, and status output.
