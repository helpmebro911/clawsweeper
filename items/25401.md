## Summary

Keep #25401 open. The related PR #25415 was closed unmerged, and current main still does not implement a provider-level `models.providers.*.enabled` contract across config typing, models.json generation, fallback selection, or `models status`.

## Evidence

- **Provider config type lacks enabled:** `ModelProviderConfig` exposes baseUrl, apiKey/auth/api, headers, authHeader, request, and models, but no provider-level `enabled` field. The nearby `BedrockDiscoveryConfig.enabled` is a separate discovery toggle, matching the issue's observation that this pattern exists only elsewhere.
