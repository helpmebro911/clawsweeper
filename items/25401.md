## Summary

Keep #25401 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still does not define or honor a provider-level models.providers.<id>.enabled flag. The related implementation PR #25415 was closed unmerged, and the current code still merges explicit providers into models.json, fallback/model catalogs, and models status without a disabled-provider filter.

## Best Possible Solution

Keep the issue open until core adds a documented models.providers.<provider>.enabled?: boolean contract that defaults to enabled, preserves disabled config blocks, excludes disabled providers from models.json generation, configured model catalogs, and fallback candidate selection, and surfaces disabled state in openclaw models status JSON and rich output. PR #25415 remains useful prior art, but it was closed unmerged and should be rebased or reimplemented with focused tests around config schema, models-config planning, fallback resolution, and status output.
