## Summary

Keep open. Current main still lacks the requested `openclaw agent` SIGTERM handling and the fallback documentation warning for cron/coreutils `timeout` use. The related signal/supervisor issues are adjacent, but none supersedes this specific top-level agent CLI leak under external cron.

## Best Possible Solution

Keep this issue open. Implement a first-class `openclaw agent` shutdown path: create a top-level AbortController, bind SIGTERM/SIGINT to abort, pass the signal through gateway and embedded/local paths, request gateway run abort when a run id is known, cancel managed CLI-provider child runs, flush logs/transcript state, and exit within a bounded grace window. Add docs for external cron wrappers such as `timeout -k` as a defensive fallback until and after the runtime fix.
