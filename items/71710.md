## Summary

Keep open. Current main still lacks a bounded SIGTERM shutdown path for `openclaw agent`, the CLI docs do not document the `timeout -k` mitigation requested by the reporter, and the closest related issue (#66399) is adjacent process-supervisor work rather than a canonical fix for OS-cron killing the agent CLI process.

## Best Possible Solution

Keep the issue open and fix the agent CLI surface directly: add centralized SIGTERM/SIGINT handling for `openclaw agent`, cancel pending gateway RPCs and embedded runs via AbortSignal, flush/deliver bounded cleanup, then exit within a documented grace window. Also fix the multi-listener re-raise pattern in signal cleanup helpers, add regression coverage for combined internal listeners, and document `timeout -k <delay>` or equivalent supervisor escalation as the interim cron-safe wrapper.
