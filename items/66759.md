## Summary

Current `main` still hardcodes the Feishu card footer to `Agent`/`Model`/`Provider` and exposes no config or documented surface for custom footer fields, so this feature request is not implemented yet.

## Evidence

- **Static footer is hardcoded in the Feishu reply dispatcher:** `resolveCardNote()` always builds the footer from `Agent`, optional `Model`, and optional `Provider`, with no user-configurable fields or directive parsing.
  - file: [extensions/feishu/src/reply-dispatcher.ts:58](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/feishu/src/reply-dispatcher.ts#L58)

  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **That static note is applied to both streaming and normal card replies:** The dispatcher passes the resolved note into streaming card startup/close and into `sendStructuredCardFeishu(...)` for regular card sends, so current card replies inherit the same fixed footer.
  - file: [extensions/feishu/src/reply-dispatcher.ts:286](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/feishu/src/reply-dispatcher.ts#L286)
