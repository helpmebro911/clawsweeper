## Summary

Current main has a new interrupt-and-steer path (`sessions.steer`, plus Control UI `/redirect`), but the normal chat-input path still queues follow-up user messages whenever a run is active instead of starting a new turn immediately. That means the issue’s requested default behavior is not implemented well enough to close conservatively.

## Evidence
