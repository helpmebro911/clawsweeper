## Summary

Keep open. Current main has already shipped Google Chat threaded-send plumbing, but it does not implement this PR's remaining `channels.googlechat.sessionThread` config, per-thread session key partitioning, or supporting envelope timestamp plumbing. The PR is partially superseded, not obsolete.

## Best Possible Solution

Keep #68967 open. The best path is to rebase or rescope it around the still-missing Google Chat per-thread session mode: add the config/schema/docs surface, preserve existing space-based route matching, derive a thread-safe session key without exposing raw case-sensitive thread names, pass that key through envelope/session metadata paths, and keep the already-shipped threaded-reply behavior as background rather than the main reason for the PR.
