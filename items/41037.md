## Summary

Keep this PR open. Current main still validates raw `auth.profiles` directly against a schema that requires `provider`, so the linked config shape with `auth.profiles["qwen-portal:default"] = { mode: "oauth" }` still appears to fail before CLI commands can continue. The PR is a focused implementation candidate with regression coverage and positive review context, not obsolete cleanup.

## Best Possible Solution

Keep this PR open and have a maintainer rebase it or apply the same narrow normalization on current main: infer a missing `auth.profiles[*].provider` from the profile id before legacy checks and schema validation, preserve explicit providers, and keep focused regression coverage in `src/config/config.schema-regressions.test.ts`. After rebase, run the changed config validation/test lane.
