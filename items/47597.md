## Summary

Keep open. Current main still rejects `sessions_spawn({ runtime: "subagent", streamTo: "parent" })` before reaching the subagent spawn path, and the native subagent implementation has no parent progress relay equivalent to the ACP-only relay. The issue discussion remains relevant: one commenter reproduced the exact hard error, and another noted a model/provider behavior that auto-attaches `streamTo`, with OpenAI Responses as a workaround rather than an OpenClaw fix.
