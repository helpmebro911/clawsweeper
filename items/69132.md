## Summary

Keep #69132 open. Current main removed the exact `/api/experimental/web_search` call, but it still performs a single request to the configured/local Ollama host at `/api/web_search` and throws on any non-OK response. The reporter explicitly observed that local `/api/web_search` also returns 404 on Ollama 0.16.0, and the linked PRs #69178/#69222 remain candidate fixes rather than merged canonical resolution.

## Best Possible Solution

Keep #69132 open and resolve it by choosing or consolidating the active fix path from #69178/#69222: add tested compatibility handling for local 404s, decide when `OLLAMA_API_KEY` may trigger `https://ollama.com/api/web_search`, keep bearer-token scoping explicit, and update `docs.openclaw.ai/tools/ollama-search` plus Ollama provider docs to match the final local/cloud behavior.
