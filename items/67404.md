## Summary

Keep this PR open. Current main still lacks QMD session-export refresh from transcript-update events, and this PR is a focused implementation candidate, but it is not ready to merge as-is because the transcript-triggered update can be dropped while a QMD update is already pending.

## Best Possible Solution

Keep the PR open and ask for revision or maintainer review. The best path is to adapt it to current main, wire QMD session exports to transcript-update events only when QMD session export is enabled, debounce and filter by agent, clean up timers/subscriptions on close, and use the forced/queued QMD update path so transcript updates during pendingUpdate always trigger a later export/index pass. Add focused qmd-manager tests for debounce, agent filtering, cleanup, and the in-flight update race.

## Risks / Open Questions

- Merging the PR as-is can still leave QMD session exports stale when a transcript update arrives during an in-flight QMD update, because the new session-delta path is non-forced and does not queue a follow-up update.
- The branch is behind current main and touches a public SDK subpath that has since been reshaped into an agent-harness barrel over agent-harness-runtime; it likely needs rebase/adaptation before maintainer review.
- The public SDK export should be deliberately accepted or moved to the already-documented plugin runtime event surface; no supply-chain or CI/dependency regression was observed in the PR diff.
