## Summary

Keep this issue open. Current main still does not show a systemd-specific command-level handoff or regression test for the reported direct `openclaw gateway restart` child process exiting with `code === null`. The linked fix PR is closed unmerged, and the current systemd path still calls `systemctl restart` synchronously and returns structured completion.

## Best Possible Solution

Keep open and fix narrowly in the gateway restart command path. First reproduce whether the CLI is being killed because it runs inside the managed systemd service cgroup; if so, add a systemd command-level detached handoff similar in spirit to the launchd/update paths, while preserving health checks and non-zero failure behavior. Add a focused regression that observes the child process exit result rather than only stdout.
