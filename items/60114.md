## Summary

Keep open. Current `main` still does not implement TUI prompt aliases, so this PR is not obsolete or already shipped, but the PR also has unresolved review concerns: the proposed alias parsing path appears to lose user-authored spacing/escaping, and the patch bundles a large unrelated Matrix test rewrite/regression alongside the TUI feature.

## Risks / Open Questions

- Merging the PR as-is could persist a different prompt than the user typed whenever the alias text depends on repeated spaces or shell-style escaping.
- Because the branch also rewrites unrelated Matrix tests, it is harder to review and increases regression risk outside the TUI alias feature.
- Closing this PR would discard a still-unshipped feature request; the safer maintainer action is to keep it open until the parsing issue and unrelated test churn are resolved or split out.
