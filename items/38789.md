## Summary

Keep PR #38789 open. Current main still reuses persisted skill snapshot prompts in non-rw sandbox runs, so sandboxed agents can still receive host-side skill paths instead of prompt paths rebuilt from the sandbox workspace. The PR targets a real remaining agents bug, but it should not land unchanged because review feedback identifies correctness gaps around empty sandbox skill sets, preserved skill filters, and env override alignment.

## Best Possible Solution

Keep this PR open or replace it with a focused agents fix that rebuilds sandbox prompts from the same sandbox-aware skill set used for runtime setup in both attempt and compaction paths. The fix should preserve the saved skill filter, remote eligibility note, and env overrides, return no host snapshot prompt when sandbox entries are empty, and add regression coverage for snapshot-present non-rw sandbox runs plus attempt/compaction parity.
