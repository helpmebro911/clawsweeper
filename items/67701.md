## Summary

Keep this PR open. Current main still does not persist `compactResult.result.tokensAfter` from overflow auto-compaction into session `totalTokens` when provider usage and prompt snapshots are unavailable, so the core bug is not implemented on main. The PR is useful but should not be merged as-is because its branch also contains unrelated ACP, cron delivery, and Telegram changes.

## Best Possible Solution

Keep this PR open, but ask for a focused rebase or maintainer cherry-pick: thread overflow and timeout auto-compaction `tokensAfter` into current session accounting, use it when usage and prompt snapshots are unavailable, preserve existing `promptTokens`/`lastCallUsage` behavior, add regression coverage for a session with no prior `totalTokens`, and split the unrelated ACP, cron delivery, and Telegram changes into separate PRs.
