## Summary

Keep open. Current `main` still matches the PR's root-cause description: reminder-like cron jobs created as `systemEvent` default to `sessionTarget: "main"`, and main-session cron runs still record `deliveryStatus = "not-requested"` instead of outbound chat delivery. The existing PR discussion comment about scheduler drift is adjacent but separate; it does not show that the reminder-delivery problem is obsolete or already fixed.
