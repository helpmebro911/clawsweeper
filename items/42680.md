## Summary

Keep open. Current `main` still starts the cron timeout before isolated-agent execution reaches the agent runner queue, so the reported premature-timeout behavior is not implemented away. The PR discussion also surfaced an unresolved CLI-provider timeout-contract concern in the proposed patch, so there is no strong basis to auto-close this PR.
