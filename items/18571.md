## Summary

Keep #18571 open. Current main still has outbound `message_sending` cancellation/rewrite support, but the shared pre-send delivery hook is not wired with `sessionKey`, `runId`, or a bounded transcript/tool-call history view needed for deterministic verification before delivery.

## Best Possible Solution

Keep the issue open and implement an additive plugin hook contract: propagate `sessionKey` and a real `runId` into `message_sending` and `message_sent` on shared outbound delivery and direct reply paths, expose a bounded documented transcript or tool-call/result-history view for deterministic verification, and add regression tests for correlation plus cancel/rewrite behavior.
