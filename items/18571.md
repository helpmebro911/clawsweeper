## Summary

Keep #18571 open. Current main has `message_sending` cancellation/rewrite support and docs that describe message hook correlation fields, but the outbound pre-send hook still is not wired with `sessionKey` or `runId` on the shared delivery path, `OutboundSessionContext` still cannot carry `runId`, and the hook contract still lacks bounded tool-call/result history or a transcript handle for deterministic verification.

## Best Possible Solution

Keep the issue open and implement an additive plugin hook contract: propagate `sessionKey` and a real `runId` into `message_sending` and `message_sent` across shared outbound delivery and direct reply paths, expose a bounded documented transcript or tool-call/result-history view suitable for deterministic pre-send verification, and add regression tests for correlation plus cancel/rewrite behavior.
