## Summary

Keep open. Current `main` still treats Nextcloud Talk `object.content` as the final inbound text and applies mention regexes directly to that raw body. Official Nextcloud Talk bot docs describe chat-message `object.content` as a JSON-encoded `{ message, parameters }` payload whose rendered text can contain rich-object placeholders such as `{mention-call1}`, so the reported mention-gating failure is still plausible on current `main`. ([nextcloud-talk.readthedocs.io](https://nextcloud-talk.readthedocs.io/en/stable/bots/))

## Risks / Open Questions

- The bug is likely broader than mention detection: current parsing appears to pass rich-object JSON payloads through as message text instead of rendered chat text.
- A real fix will need parser, type, and test updates for `text/markdown` webhook payloads, not just a regex tweak.
