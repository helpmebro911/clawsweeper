## Summary

Keep open. Current main has the generic SecretRef env/file/exec transport from #26155, but it still does not ship Bitwarden/Vaultwarden docs, a bw resolver, a Bitwarden plugin, or a public secret-provider plugin contract. Maintainer discussion also points toward porting this after #72548 or an equivalent manifest-first secret-provider API, so this PR is not safely closeable as implemented, duplicate, or ClawHub-only.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best path is to port it after #72548 or an equivalent manifest-first secret-provider plugin API lands, likely as an extensions/secrets-bitwarden plugin declaring secret-provider ownership, exposing a Bitwarden provider factory, and carrying docs, tests, package metadata, and security review. If maintainers choose the generic exec-wrapper route instead, rebase this PR and fix docs/script/test/package alignment rather than closing it as implemented or ClawHub-only.
