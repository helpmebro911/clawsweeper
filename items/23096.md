## Summary

Keep open. Current main still has only the built-in env/file/exec SecretRef transports and does not ship Bitwarden/Vaultwarden docs, a resolver, a bundled plugin, or a secret-provider plugin contract. The PR is stale and needs a rebase or port, but the discussion and current code make it a plausible implementation candidate rather than something safely closeable as implemented, duplicate, superseded, or ClawHub-only.

## Best Possible Solution

Keep this PR open as the focused Bitwarden/Vaultwarden implementation candidate. The best maintainer path is to first land or settle #72548 or an equivalent manifest-first secret-provider plugin API, then port this work into an extensions/secrets-bitwarden plugin with contracts.secretProviders ownership, a Bitwarden provider factory, package metadata/runtime dependency handling, docs, tests, and a dedicated security review. If maintainers choose the generic exec-wrapper route instead, rebase this PR and fix the docs/script/package/security issues rather than closing it as implemented or ClawHub-only.
