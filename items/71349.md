## Summary

Keep open. Current main has a related memory-lancedb live-config wiring fix in the latest release notes, but the reported low autoCapture behavior is not conclusively fixed: capture is still narrow, heuristic-only, fire-and-forget, and the adjacent doctor warning path still exists when no active memory runtime backend resolves.

## Best Possible Solution

Keep the issue open. The best maintainer path is to decide the intended product contract for memory-lancedb autoCapture, then either make it durable/observable across agent_end paths or document it as a narrow heuristic. A code fix likely belongs in extensions/memory-lancedb/index.ts plus focused tests for multi-session capture, long user turns, duplicate skips, failed async writes, and cron/CLI completion. The adjacent doctor UX should be handled in src/commands/doctor-memory-search.ts by recognizing an active memory slot/plugin separately from legacy memory backend config, or by rewording the diagnostic so a valid memory-lancedb slot is not reported as absent.
