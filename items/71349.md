## Summary

Keep open. Current main has a memory-lancedb auto-capture hook, but it is narrow, heuristic-only, and fire-and-forget; the report’s “rarely fires” behavior is still plausible and not covered by a shipped fix or canonical duplicate. The adjacent doctor warning also still maps through the memory-core runtime backend path, while memory-lancedb does not register that runtime capability.

## Best Possible Solution

Keep #71349 open. Add a focused reproduction/regression around memory-lancedb `agent_end` payloads from gateway/Telegram/cron/CLI paths, then decide whether `autoCapture` should remain trigger-only or capture per-turn/session summaries. Also fix or clarify doctor’s memory-plugin health path so slot-selected memory-lancedb does not produce a misleading generic “No active memory plugin” warning.
