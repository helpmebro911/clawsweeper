## Summary

Keep open. Current main has a memory-lancedb auto-capture hook, but it still only captures user messages that match a narrow heuristic trigger set and are under the configured length cap; it does not implement the issue's expected per-session/per-turn persistence. The adjacent doctor warning also still has a plausible current-code path because doctor relies on active memory runtime registration, while memory-lancedb does not register a memory runtime/capability.

## Best Possible Solution

Keep the issue open for a maintainer-owned fix or product clarification. The likely path is to either broaden/replace memory-lancedb autoCapture so it captures the intended session events, add observable skip/failure diagnostics, and cover real gateway/channel agent_end paths, or document the current heuristic-only behavior and add a separate fix for doctor to recognize a selected memory-lancedb slot even without memory-core runtime registration.
