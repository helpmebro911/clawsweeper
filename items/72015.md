## Summary

Keep open. Current main partially fixes the hard-timeout sub-claim, but the broader reliability/defaults report remains valid: Active Memory still runs as a blocking `before_prompt_build` sub-agent, still inherits the current/session model before fallback, and QMD gateway startup still arms QMD managers for every configured memory-enabled agent while `onBoot` defaults true.

## Best Possible Solution

Keep #72015 open as the broader reliability/defaults tracker. Preserve the shipped hard-timeout behavior, then add explicit nonblocking/deadline or opt-in blocking semantics for Active Memory, safer model/default warnings, per-agent/session recall concurrency limits, QMD boot concurrency/lazy/active-only controls, and status/doctor visibility for high-cost configurations.
