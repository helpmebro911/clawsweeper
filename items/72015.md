## Summary

Keep open. Current main fixes one related failure mode, the hard active-memory timeout overrun, but the central reliability request is still present: active-memory remains a blocking `before_prompt_build` embedded-agent pass, it still inherits the current/session or agent primary model before fallback, and QMD gateway startup still arms QMD managers for every configured memory-enabled agent with `onBoot` defaulting on.

## Best Possible Solution

Keep the issue open as a real reliability/defaults item. A maintainer-quality fix likely belongs in the bundled active-memory plugin and QMD startup path: add an explicit nonblocking or short-deadline default for pre-prompt recall, avoid inheriting slow primary models without an explicit choice or warning, add per-agent/session recall concurrency/drop behavior, and add global QMD boot-update concurrency, jitter, or lazy/active-agent startup controls. The existing hard-timeout work should be treated as a partial mitigation, not a full resolution.
