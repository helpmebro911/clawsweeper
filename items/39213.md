## Summary

Keep PR #39213 open. Current main still rotates Telegram answer previews across assistant-message boundaries in `streamMode: "partial"`, the PR's stream-revival/refinalization mechanism is absent, and current tests still assert split preview messages in partial mode. There is also an explicit maintainer hold requiring rebase/alignment with #38906 and boundary-regression fixes before merge.

## Best Possible Solution

Keep #39213 open as the active Telegram partial-preview coalescing work, or replace it with a rebased maintainer-owned PR. The implementation should live under `extensions/telegram`, explicitly account for current #38906 DM preview transport behavior, preserve block-mode rotation, add a safe partial-mode stream revival/refinalization path, and include focused regression tests for early partials, sequential tool-call rounds, and media-only assistant boundaries.
