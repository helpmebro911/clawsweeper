## Summary

Keep open. Current main still ignores per-agent `agents.list[].reasoningDefault` in the Telegram and Feishu channel preview paths, while the config contract, docs, and core reply directive resolution already expose/use the per-agent default. The PR is a focused active fix with regression tests and relevant review discussion, not obsolete cleanup.

## Best Possible Solution

Keep PR #71817 open for Telegram and Feishu maintainer review. The best path is to merge or update a focused implementation that makes channel preview resolution fall back to `agents.list[].reasoningDefault`, preserves explicit session overrides including `off`, and keeps Feishu stream-only preview semantics. Track global/per-model defaults and any agent ID normalization policy in separate canonical issues.
