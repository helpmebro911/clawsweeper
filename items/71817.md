## Summary

Keep open. Current main still ignores per-agent `reasoningDefault` in the Telegram and Feishu channel preview paths, while the core config/docs already expose the per-agent setting. The PR is a focused implementation with tests and active review discussion, not obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer/channel-owner review. The best path is a focused merge or update that makes Telegram and Feishu reasoning preview resolution fall back to `agents.list[].reasoningDefault` while preserving explicit session overrides and Feishu's stream-only preview behavior. Broader `agents.defaults` or per-model default behavior should remain tracked separately.
