## Summary

Keep open. Current main still feeds recent light-phase entries into a fresh dream-diary narrative each sweep, appends a new timestamped DREAMS.md entry, and has no per-phase last-emitted cutoff or duplicate-body suppression. The existing diary dedupe intentionally preserves identical bodies when timestamps differ, which matches the reported hourly duplicate shape rather than fixing it. Related #68882 is promotion-candidacy filtering, and #70737 is cron/heartbeat decoupling, so neither supersedes this diary write-path bug.

## Best Possible Solution

Keep #72096 open. Implement a per-workspace/per-phase emission cursor or recent narrative fingerprint under `memory/.dreams/`, then suppress repeated light/REM diary narratives when the candidate/snippet set or generated body has already been emitted in the current window. Add a regression test with two light-phase cycles at different timestamps producing the same snippets/body, plus an optional doctor cleanup for existing duplicate-body diary entries.
