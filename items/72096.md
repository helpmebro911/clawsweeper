## Summary

Keep open. Current main has ingestion checkpoints and a manual exact-diary dedupe action, but the light/dream diary emission path still selects recent recall entries from the whole lookback window and appends phase narratives without a per-phase last-emitted cutoff or automatic duplicate suppression. The reported repeated-summary symptom is plausible against current code.

## Best Possible Solution

Implement this in `memory-core` as phase emission state, not just ingestion state: persist a per-workspace/per-phase emitted cutoff or content fingerprint set, have light/dream diary generation skip entries already emitted in the current sweep/window, and add a regression test that runs repeated managed dreaming cycles with unchanged top candidates plus growing REM/session corpus and verifies `DREAMS.md` does not accumulate duplicate light/work-summary entries. Keep #70737 as orthogonal context, not the canonical fix.
