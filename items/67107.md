## Summary

Current main has not resolved the reported macOS Big Sur installation UX gap. OpenClaw still intentionally requires Node >=22.14, but the installer and install docs still do not preflight or clearly document the macOS 11/Big Sur incompatibility before attempting Homebrew Node installation.

## Best Possible Solution

Keep this issue open and narrow the fix to installer/docs UX. The conservative path is to keep runtime support at Node >=22.14, add an early macOS version preflight in `scripts/install.sh` before Homebrew or Node installation, and document the tested minimum macOS version in `docs/install/index.md` and `docs/install/node.md`. Add or adjust installer tests around the macOS preflight so Big Sur gets a clear unsupported-OS message.
