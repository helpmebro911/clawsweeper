## Summary

Keep #70241 open. Current main still documents and scripts `clawhub package publish --dry-run` for plugin publishing, while the latest published `clawhub` package is still 0.9.0 and its command registration lacks `--dry-run`, `--json`, and the documented GitHub-source publish forms. The provided maintainer live-verification comment also explicitly says the docs/client mismatch is current and should stay open.

## Best Possible Solution

Keep the issue open. The best path is to either publish/update the `clawhub` client so `package publish` implements the documented `--dry-run`, `--json`, and GitHub-source behavior, or correct OpenClaw docs and release helpers to match the currently shipped client. The ClawHub service/client timeout should be investigated in the ClawHub publish path, with clearer diagnostics around whether the backend is still processing after the client upload deadline.
