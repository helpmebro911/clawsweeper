## Summary

Keep #45276 open. Current main still has the reported IPv6 canonical-form bug in `isLocalGatewayAddress`: it normalizes the candidate IP but compares the stored tailnet IPv6 address as a lowercased raw string. Current tests do not cover this helper, and the PR patch is focused on the implicated gateway code path with the bot review's mocking concern already addressed in a follow-up commit.

## Best Possible Solution

Keep the PR open. The best path is to land or adapt the narrow gateway fix so `isLocalGatewayAddress` normalizes both candidate and stored tailnet IPv4/IPv6 values with the existing IP normalizer, and add reliable hoisted-mock regression coverage in `src/gateway/net.test.ts`. If maintainers decide the helper is dead code, that should be handled as an explicit cleanup instead of closing this PR as already implemented.
