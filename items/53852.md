## Summary

Current main still matches the reported failure class. Isolated `agentTurn` cron runs still reach the embedded agent stream path, OpenAI-compatible tool-call argument chunks are still appended into `partialArgs` without detecting repeated complete/cumulative JSON payloads, completed tool-call blocks still retain that scratch field, and the repair wrapper still rejects a duplicated complete JSON object as unsafe trailing content.

## Best Possible Solution

Keep this issue open. The best fix belongs at the OpenAI-compatible streaming/tool-call boundary: detect cumulative or repeated complete argument payloads before appending, or safely repair identical duplicated balanced JSON objects, strip scratch `partialArgs` from completed tool-call blocks, and add a focused regression test for isolated `agentTurn` cron using an OpenRouter/Kimi-shaped stream that emits the same complete arguments twice.
