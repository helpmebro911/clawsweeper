## Summary

Keep this issue open. Current main still has the reported OpenAI-compatible streaming shape: isolated cron agent turns run through the embedded agent path, streamed `toolCall.function.arguments` chunks are appended directly into `partialArgs`, completed tool-call blocks retain that scratch field, and the existing malformed-argument repair does not accept a second full JSON object as a suffix.

## Best Possible Solution

Keep this open and fix the OpenAI-compatible streaming/tool-call boundary. The fix should either detect cumulative or repeated full argument payloads before appending, or deterministically repair duplicate balanced JSON objects. Add a focused regression test for an isolated `agentTurn` cron path using an OpenRouter/Kimi-shaped stream that emits the same complete arguments twice, and ensure completed tool-call blocks do not expose malformed `partialArgs` scratch state.
