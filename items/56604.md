## Summary

Keep open. Current main still captures the dialed phone number but does not use it to select a per-number voice, persona, agent, response model, prompt, greeting, realtime instructions, or TTS voice. The recent maintainer discussion also explicitly says the feature remains missing, and the related #59245 tracks broader outbound task-call behavior rather than this inbound per-number routing request.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled voice-call plugin: add a typed/documented per-number route table to the config schema and manifest; normalize event.to at call creation for Twilio, Telnyx, and Plivo; persist a stable per-call route id or effective config; and make inbound greeting, agent/model/system prompt, realtime consult/instructions, and TTS voice selection use that effective config with global fallback. Add focused tests for per-number matches, fallback behavior, and provider To-number normalization.
