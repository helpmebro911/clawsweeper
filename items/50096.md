## Summary

Keep open. Current main has added substantial memory infrastructure since the issue was filed, including dreaming/promotion, opt-in session transcript indexing, memory-wiki, and plugin-backed memory options. But the two clearest core asks in the discussion are still not resolved: `MEMORY.md` is still part of normal workspace bootstrap injection unless using broad bootstrap controls, and `memoryFlush` remains a pre-compaction/per-turn safeguard rather than a gateway shutdown/restart flush.
