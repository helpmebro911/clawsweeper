## Summary

Keep PR #48130 open. Current main moved this browser code under the browser plugin, but the same Windows executable extraction bug is still present there, so the PR is not obsolete. It needs to be rebased/ported and should add a focused regression test.

## Best Possible Solution

Keep the PR open and port the two regex fixes into `extensions/browser/src/browser/chrome.executables.ts`. Add focused regression coverage, preferably in `extensions/browser/src/browser/chrome.default-browser.test.ts`, for quoted and unquoted Windows registry command strings containing backslashes and spaces, then run the changed browser plugin checks before landing.
