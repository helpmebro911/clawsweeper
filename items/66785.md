## Summary

Keep this issue open. Current main still appears to force the installer onboarding path after detecting an existing config, even though large model-list filtering and OpenRouter onboarding support are present.

## Best Possible Solution

Fix the installer or Umbrel setup entrypoint so an existing `OPENCLAW_CONFIG_PATH` or legacy config exits the onboarding path after doctor/dashboard handling instead of continuing into `openclaw onboard`. Add a focused regression test for the existing-config install path, then verify the packaged Umbrel setup can filter large model catalogs and select OpenRouter.
