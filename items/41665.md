## Summary

Keep this PR open. Current main still has the reported Windows-onboarding problem for brew-backed skill installers: install-spec OS filtering exists and is documented/tested, but several current skill manifests still omit `os` on brew install entries, so onboarding can still surface brew-only install actions on Windows. The PR is directionally useful and the provided discussion includes a Windows Git Bash repro plus a review note about `xurl`; however, it is stale/incomplete against current main and should be rebased/updated rather than closed.

## Best Possible Solution

Keep the PR open as a plausible focused bug-fix candidate. The best maintainer path is to ask for a rebase/update against current main, add `os` filters to all current brew-only install specs, preserve or add valid non-brew Windows installers where they exist, and then land the metadata-only change with the existing OS-filter tests as contract coverage.
