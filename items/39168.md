## Summary

Keep PR #39168 open. The submitted patch is obsolete against current main because OpenClaw now depends on @mariozechner/pi-agent-core 0.70.2 and has no pi-agent-core patchedDependency, but the user-visible mcp_exec/mcp_read/mcp_write dispatch problem is still not fixed in the current OpenClaw-side tool-call normalization path.

## Best Possible Solution

Rework the PR against current main by implementing guarded mcp_ prefix normalization in src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts. The normalizer should first try the raw name, then strip mcp_ only when the stripped name is non-empty and resolves to exactly one allowed registered tool. Preserve mcp__bundle__* names, avoid a pnpm patch, and add focused tests for mcp_exec, mcp_read, mcp_write, nonexistent mcp_foo, bare mcp_, and registered mcp__bundle__* names.
