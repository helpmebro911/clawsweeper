## Summary

Current main still does not implement native Telegram thread-bound subagent session spawning. The core `sessions_spawn({mode:"session", thread:true})` path requires a channel `subagent_spawning` hook to return `threadBindingReady`, while the Telegram bundled entry still lacks `registerFull` subagent lifecycle hook wiring. Telegram has adjacent ACP/topic binding infrastructure, but docs still describe native thread-bound subagent sessions as Discord-only. Open PR #42707 remains the relevant implementation candidate, so this should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this open and review/land #42707 or an equivalent focused implementation. The fix should add Telegram `registerFull` lifecycle wiring for `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`, use the existing Telegram thread-binding manager and SessionBindingAdapter, honor `channels.telegram.threadBindings.enabled` and `spawnSubagentSessions`, clean up bindings on end/failure, resolve completion delivery targets, and update native subagent docs/tests to distinguish this from Telegram ACP topic binding.
