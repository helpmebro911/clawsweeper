## Summary

Keep this issue open. Current main still gates native `sessions_spawn({ mode: "session", thread: true })` on a registered `subagent_spawning` lifecycle hook, and the Telegram bundled plugin still does not register the native subagent lifecycle hooks. Telegram has adjacent ACP/current-conversation topic binding support, but that is not the native subagent session binding requested here. Open PR #42707 remains the focused implementation candidate rather than shipped behavior on main.

## Best Possible Solution

Keep this issue open and review/land #42707 or an equivalent focused implementation. The fix belongs in the Telegram plugin entry/runtime: register `subagent_spawning`, `subagent_delivery_target`, and `subagent_ended`; bind native session-mode subagent spawns through Telegram topic/current-conversation state; honor Telegram thread-binding feature flags; clean up bindings on failure/end; resolve completion delivery targets; and update native subagent docs/tests so ACP topic binding and native subagent thread binding remain distinct.
