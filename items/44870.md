## Summary

Keep open. Current `main` still treats non-Azure custom “OpenAI-compatible” endpoints as `/chat/completions` during verification and stores them as `api: "openai-completions"`, so a Responses-only OpenAI-compatible endpoint would still fail in the same way the issue describes.
