## Summary

Keep open. Current `main` has only lightweight compaction visibility: a `notifyUser` boolean, fixed start/end notices, and transient UI markers. It does not implement the core request for previewable, confirmable, user-correctable, or rollbackable context compaction, and Codex-harness docs explicitly note that human-readable compaction summaries/auditable kept-entry lists are still missing.

## Evidence

- **Compaction config exposes notices, not confirmation flow:** `AgentCompactionConfig` includes `truncateAfterCompaction` and `notifyUser`, but no preview/approval/rollback fields or multi-mode user control surface.
  - file: [src/config/types.agent-defaults.ts:469](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/config/types.agent-defaults.ts#L469)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **User-facing docs describe brief notices only:** The compaction docs say compaction is silent by default and `notifyUser: true` only shows short start/complete notices.
  - file: [docs/concepts/compaction.md:133](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/concepts/compaction.md#L133)

- Because lightweight notices already exist, maintainers could overestimate current coverage unless the remaining approval/review/rollback gaps are called out explicitly.
