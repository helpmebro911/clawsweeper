## Summary

Keep this PR open. The linked bug is still present on current main, so the work is not obsolete; however, the current PR head described in the review context still sets `hasText` unconditionally, so media-only sends would still suppress the following text reply. Reviewer comments already identified the right minimal correction.

## Best Possible Solution

Keep the PR open and ask for the small targeted update already suggested in review: compute `hasText` from `content` or `message`, keep `pendingMessagingTexts` tracking outside `sendTarget` extraction for `sessions_send`, store `sendTarget.hasText = hasText`, and add regression coverage for media-only same-target message-tool sends preserving the final text reply.
