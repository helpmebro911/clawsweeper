## Summary

Keep open. Current main still only prepends model fallback and fallback-cleared reply notices when verbose mode is enabled, the existing e2e coverage explicitly asserts verbose-off suppression, and the requested independent opt-in config is absent from config types/schema/docs. There is also an open same-author implementation PR, #69399, with closing syntax for this issue, so the issue should remain open until that PR lands or maintainers choose another implementation path.

## Best Possible Solution

Keep this issue open and review #69399. The likely implementation path is a default-false agent-defaults option that lets `runReplyAgent` prepend fallback and fallback-cleared notices when either verbose mode is enabled or the opt-in is true, with config type/schema/help/docs updates and e2e coverage for once-per-state behavior.
