## Summary

Keep open. Current main has a generic transport-activity stale-socket policy and a Mattermost update_at watchdog, but Mattermost still does not publish transport activity or implement WebSocket ping/pong keepalive. The linked PR #57621 remains open as a proposed fix, so the reported silent WebSocket stall is not fixed on main.
