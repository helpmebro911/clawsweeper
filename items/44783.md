## Summary

Current `main` still uses the nested workspace-state path the issue reports. The request is not implemented on `38caa6832d4e1d344736d635b0c5fdad37e4430b`, so there is not strong evidence to close this issue conservatively.

## Risks / Open Questions

- The report likely mixes two behaviors: current main intentionally stores workspace metadata at `<workspace>/.openclaw/workspace-state.json`, while the extra outer `.openclaw` segment in `C:\Users\Administrator\.openclaw\.openclaw\workspace` could also come from setting `OPENCLAW_HOME` to an already-suffixed `.openclaw` directory. `docs/help/environment.md:118` documents `OPENCLAW_HOME` as a home-directory override, not a state-directory override.
