## Summary

Keep this PR open. Current main still contains the exact `asRecord(details?.debug) ?? {}` fallback that makes payload-less `memory_search` transcript entries truthy, and current tests do not cover the interleaved empty-then-populated transcript case added by the PR. The provided related item #68837 covers a different Active Memory config-validation claim and is not a duplicate or superseding fix.

## Best Possible Solution

Keep this PR open and land it, or an equivalent patch, after normal maintainer validation. The best code path is to remove the `?? {}` fallback in `readActiveMemorySearchDebug` and keep the regression test proving a newer payload-less `memory_search` result does not shadow older real debug data.

## Evidence

- **Current main still has the implicated fallback:** `readActiveMemorySearchDebug` scans transcript JSONL entries newest-first, then sets `debug` to `asRecord(details?.debug) ?? {}` before checking `if (!debug && !warning && !action && !error)`.
