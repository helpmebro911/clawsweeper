## Summary

Keep #68773 open. Current main still has the exact `asRecord(details?.debug) ?? {}` fallback that makes payload-less `memory_search` transcript entries look truthy, and current tests do not cover the interleaved empty-then-populated transcript case the PR adds. The PR is a focused bug fix, not obsolete or duplicated by the provided related item.

## Best Possible Solution

Keep #68773 open and land this PR, or an equivalent patch, after normal maintainer validation. The best code path is to remove the `?? {}` fallback in `readActiveMemorySearchDebug` and keep the regression test that proves a newer payload-less `memory_search` result does not shadow older real debug data.
