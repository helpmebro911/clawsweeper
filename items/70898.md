## Summary

Keep open. Current main still does not seed a fresh transcript during silent auto-reply session rotations, so the user-visible #70853 problem is not implemented on main. PR #70898 is a focused active fix with author follow-up, but the latest head still has review concerns around replaying raw same-role runs, reading whole transcripts for a small tail, and a changelog attribution gap.

## Best Possible Solution

Continue this PR or a replacement patch in the auto-reply session-reset surface. The fix should seed silent rotated transcripts from prior context, but normalize or collapse same-role `user`/`assistant` runs into provider-safe turns before appending, use a bounded tail reader rather than full transcript reads, add regression coverage for #70853 including consecutive same-role tails, and fix the changelog attribution.
