## Summary

Keep PR #70898 open. Current main still rotates silent auto-reply sessions onto a fresh sessionId/sessionFile without seeding prior DM context, so the linked #70853 problem is not implemented on main. The PR is focused and relevant, but its latest revision still has unresolved review concerns around replaying invalid same-role tails and reading whole transcripts on the reset path.

## Best Possible Solution

Keep the PR open and ask for a revision rather than closing it. The best path is to preserve DM continuity on silent compaction/role-ordering resets, but implement the replay with a bounded tail reader and normalize the retained user/assistant tail into provider-valid ordering before appending it. Rebase onto current main and cover the reset helper plus the transcript replay edge cases in focused tests.
