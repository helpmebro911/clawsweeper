## Summary

Current main still does not implement the requested Slack outbound minimum-delay/throttle setting. Slack sends are serialized per account/token/target/thread, but there is no last-send timestamp, sleep, configurable `sendThrottleMs`, or `outbound.interMessageMs` equivalent in the Slack send path, config types, schema, metadata, docs, or tests. The related PR #26732 was closed unmerged, so it is not shipped.

## Best Possible Solution

Keep #13627 open. Implement a focused Slack outbound minimum-delay feature in the bundled Slack plugin send path, with typed config, strict schema, config metadata/UI help, docs, and fake-timer coverage for normal sends, chunks, media follow-ups, DMs, channel messages, thread replies, Slack action sends, and slash/respond follow-ups if those are in scope. If maintainers want a generic channel throttle instead, track that as a separate design item; the closed unmerged #26732 does not supersede this issue.
