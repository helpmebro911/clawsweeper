## Summary

Keep PR #45921 open. Current main does not implement Prompt Inspector or an equivalent AI-based prompt-injection detection path, and the remaining work is not safely auto-closeable as ClawHub-only because the PR targets core security ingress points where the current plugin surface does not fully provide an async pre-model/blocking content-safety seam. The PR may need redesign into a generic content-safety capability plus a vendor plugin, but the underlying security/product request is still live.

## Best Possible Solution

Keep the item open, but steer it away from a one-off Prompt Inspector core client. The best maintainer path is to define a generic prompt/content-safety capability or plugin hook that can asynchronously inspect and optionally block untrusted inbound content, wrapped external content, and model-visible tool results before they reach the model, then implement Prompt Inspector as an external or official plugin using that contract if maintainers want that vendor integration.
