## Summary

Keep open. Current main still does not prove a fix for the reported Control UI latency mismatch: Sessions navigation still refreshes through `sessions.list`, Overview still overlaps several secondary refreshes, and the Usage path still calls the slow `sessions.usage`/`usage.cost` endpoints named by the reporter. Related #71631 is open and broader, but it is not a merged fix or explicit maintainer supersession for this concrete Windows timing report.

## Best Possible Solution

Keep this issue open as a concrete Control UI performance bug. The best next step is to instrument the full path from Sessions tab click to request enqueue/send, gateway handler start/end, state assignment, and visible render, while correlating concurrent `usage.cost`, `sessions.usage`, `node.list`, `cron.runs`, `config.schema`, and `channels.status` calls on a Windows/global-npm setup. Then make slow secondary panels lazy, cancellable, stale-guarded, or independently rendered. If maintainers want #71631 to become canonical, they should say so explicitly and close this only after a landed change covers this reproduction.
