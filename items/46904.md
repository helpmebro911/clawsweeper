## Summary

Keep open. Current main still rejects `message` tool `action="send"` calls when non-empty poll creation params are present, and tests explicitly lock in that rejection. The PR addresses a real remaining part of #41199, with useful discussion and focused tests, though it needs maintainer policy review and a current-main rebase/fix before merge.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with #41199/#46455. The best code direction is a narrowly scoped send-path normalization decision: strip only poll-creation params before `handleSendAction()` when `action="send"`, preserve all `action="poll"` validation, cover camelCase and snake_case variants in helper and runner tests, and update the branch for current-main exports/types before merge.
