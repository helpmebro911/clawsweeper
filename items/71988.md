## Summary

Keep #71988 open. Current main still does not broadcast `inbound_claim` after plugin-bound fallback, and the PR is a focused, recent attempt at a real core/plugin hook gap rather than something implemented, superseded, stale, or suitable for ClawHub alone. The PR likely needs changes before merge because review comments identify missing handling for dedupe commit and synthetic replies on the new early-return path.

## Best Possible Solution

Keep the PR open but request a small follow-up patch: add the fallback broadcast in `src/auto-reply/reply/dispatch-from-config.ts`, mirror the targeted plugin-bound handled path by delivering `broadcastResult.reply`, commit or release inbound dedupe before any new terminal return, and add positive regression tests for missing-plugin/no-handler broadcast handling including handled-with-reply and dedupe behavior. #48434 can remain consolidated under this PR unless maintainers want a fresh canonical issue.
