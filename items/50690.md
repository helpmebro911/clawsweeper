## Summary

Keep open. Current main has useful adjacent work: a generic outbound `message_sending` hook, phase-aware visible-text extraction that hides commentary in some surfaces, and a narrow isolated-cron interim-ack retry/suppression path. It does not implement the requested runtime-enforced, execution-state-aware pre-send policy across user-facing delivery surfaces, repeated-ack suppression, progress-debt faults, or the requested structured policy metrics/tests.

## Evidence

- **Outbound pre-send hook is generic and limited:** `applyMessageSendingHook` runs before delivery and can cancel or rewrite content, but the event is built from the payload, target, channel/account, reply/thread ids, and media metadata. It does not pass recent session history, recent tool calls, task metadata, execution state, or progress-debt state required by #50690.
