## Summary

Current main does not implement first-class direct script cron jobs. The CLI, protocol, type model, runtime dispatcher, docs, and Control UI still expose cron execution as either a main-session system event or an isolated/current agent turn. The linked ollama-exec-shim is a useful workaround, but it is not the requested core cron/dashboard integration, and the current plugin cron hook does not provide a custom cron payload/executor seam that would make this a clean ClawHub-only close.
