## Risks / Open Questions

- Do not merge this PR as submitted: provided review comments identify a syntax error and unused import that would break validation.
- The proposed HMAC enforcement is security-sensitive and appears to risk first-upgrade lockout for existing configs that have a gateway token but no `.sig` sidecar yet.
- The proposed best-effort sidecar write combined with read-side missing-signature rejection can create a self-inflicted startup failure after transient signature-write I/O errors.
- The proposed token lookup bypasses injected filesystem/home dependencies, making tests likely to miss the integrity path.
- The PR mixes unrelated CodexBar onboarding changes into a config security PR, broadening review scope beyond the stated fix.
