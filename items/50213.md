## Summary

Keep open. Current main still does not implement a Gemini web_search baseUrl override: the bundled Google plugin schema exposes only apiKey/model for Gemini web search, and the runtime builds requests from the fixed Google API base. The issue is recent, concrete, and the discussion describes a valid proxy/third-party Gemini use case; no provided related item supersedes this specific config gap.

## Evidence

- **Issue discussion validates the requested behavior:** #50213 asks for a Gemini web search baseUrl so third-party/proxy Gemini-compatible APIs can be used, and the comments explicitly call out the same gap: OpenAI-compatible providers have a baseUrl pattern, but Gemini web search does not.
- **Google plugin schema does not accept webSearch.baseUrl:** The bundled google plugin configSchema has webSearch.additionalProperties=false and only declares apiKey and model, so plugins.entries.google.config.webSearch.baseUrl is not an accepted config field on current main.
