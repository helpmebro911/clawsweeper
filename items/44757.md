## Summary

Current `main` does not implement this request. The WebChat/Gateway attachment parsing path still hardcodes a 5 MB limit in all three active call sites, and the Gateway config surface only exposes `gateway.webchat.chatHistoryMaxChars`, not any attachment size override.
