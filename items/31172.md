## Summary

Keep PR #31172 open. Current main still does not implement a configurable Feishu `streamingThrottleMs`: the runtime hardcodes a 160 ms streaming card throttle, the dispatcher does not pass a throttle option, and the validated schema, manifest, docs, and tests do not expose or cover the setting. The linked issue #31136 was stale-closed rather than shipped, and the PR author recently reiterated the Feishu stability need.

## Best Possible Solution

Keep the PR open for maintainer review or a rebased Feishu plugin patch. The best implementation is to add `streamingThrottleMs` to the Feishu config schema and plugin manifest, resolve it through the account/top-level config merge, pass it into `FeishuStreamingSession`, document it on the public Feishu channel docs page, and add focused tests for both default behavior and configured throttle plumbing. Maintainers should decide whether the default should preserve current main's 160 ms behavior or adopt the PR's 100 ms proposal.
