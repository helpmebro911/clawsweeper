## Summary

Keep PR #31172 open. Current main still has a fixed Feishu streaming card update throttle and does not expose or plumb `streamingThrottleMs` through schema, manifest, docs, or tests. The linked issue #31136 was stale-closed rather than shipped, and the PR author recently reiterated the Feishu stability need.

## Best Possible Solution

Keep the PR open for maintainer review or a rebased Feishu plugin patch. The best implementation is to add `streamingThrottleMs` to the Feishu config schema and plugin manifest, resolve it through the account/top-level config merge, pass it into `FeishuStreamingSession`, document it on the Feishu channel docs page, and add focused tests for both default behavior and configured throttle plumbing. Maintainers should explicitly decide whether the default should preserve current main's 160ms behavior or adopt the PR's 100ms proposal.
