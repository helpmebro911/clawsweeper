## Summary

Keep PR #49914 open. Current main does not implement chat-channel `/sessions` or `/session <number|sessionId|back>` switching, and the PR is a concrete, recent implementation attempt for the same user problem tracked by #9959. It is not safe to close as implemented, duplicate/superseded, clawhub, stale, or not-actionable; the best maintainer path is to review/repair the PR or use it as the basis for a core implementation.

## Best Possible Solution

Keep the PR open and have a maintainer decide whether to repair and land it or replace it with a smaller core implementation. The likely core path is to add a chat-only `/sessions` command and `/session` switch targets, persist bounded per-SessionKey history in `SessionEntry`, retain transcript files until LRU eviction, restore the full set of session-scoped state that affects later turns, wire docs/config/schema/tests, and explicitly handle in-flight run/lifecycle-hook semantics before merge.
