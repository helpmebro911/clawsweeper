## Summary

Keep this issue open. Current main still does not expose a `skills` selector for `sessions_spawn({ runtime: "acp" })`, and the ACP execution path sends the raw task text through the ACP runtime before the normal OpenClaw skill snapshot/prompt path is reached. The related context-injection PR was closed unmerged and did not supersede ACP skill-context injection.

## Best Possible Solution

Implement this as an additive ACP/session-spawn feature in the ACP-owned spawn path: add a typed skill selection surface or documented inherited default, reuse the existing skill snapshot/filter machinery for rendered instruction text, add a narrow ACP runtime prompt-context seam or safe task-prefixing strategy, and cover explicit lists, default allowlists, no-tool-grant isolation, truncation, and host-side security behavior in ACP-specific tests and docs.

## Risks / Open Questions

- Adding this feature without explicit allowlists and prompt-size limits could leak local skill instructions into external ACP harnesses unexpectedly.
- ACP harnesses run on the host and outside the OpenClaw sandbox, so the implementation should inject rendered instructions only and must not grant skill env vars, OpenClaw-only tools, or sandbox privileges implicitly.
- Automatic relevance detection would need conservative deterministic ordering and auditability; an explicit skill list or documented default allowlist is safer as the first additive contract.
