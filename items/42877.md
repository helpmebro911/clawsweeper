## Summary

Keep this issue open. Current main still provides memory search/read, active-memory recall, startup/read caps, append-only flush, and promotion/dreaming paths, but it does not implement the requested bounded model-facing memory CRUD/write tool with hard per-store character budgets, rejected over-limit writes, substring replace/remove, duplicate handling, capacity reporting, or credential screening.

## Best Possible Solution

Keep this open for a focused memory-core or active-memory design. Add a structured `memory` or `memory_write` tool with config-backed per-store character budgets, atomic add/replace/remove semantics, duplicate detection, credential-pattern screening, and over-limit responses that include usage/current entries. If enforcement must be hard rather than advisory, add a generic memory-store write guard seam so generic write/edit, memory flush, and promotion/dreaming cannot bypass the same budget.
