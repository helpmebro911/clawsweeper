## Summary

Keep this PR open. Current main has shipped the generic empty/NO_REPLY no-rerun behavior and a narrower exact-NO_REPLY tool-result preservation path, but it still does not implement the PR’s broader raw-stdout contract: recovering a successful non-NO_REPLY tool result as the final cron output when the final assistant payload is empty. The submitted patch is also not ready as-is because review comments identify prompt-keyword gating, a timestamp-fragile test, and a retry-suppression regression when no successful tool result was recovered.

## Best Possible Solution

Keep this PR open, but require a rebase onto the current run-executor and embedded-runner structure. The best fix is a narrower raw-output recovery path that only uses a successful in-run tool result when one is actually present, avoids prompt-keyword-only intent detection if possible, preserves the existing interim-ack retry when no tool result was recovered, and adds regression coverage for arbitrary raw stdout recovery, exact NO_REPLY, acknowledgement-without-tool-result retry, and stale/older transcript entries.
