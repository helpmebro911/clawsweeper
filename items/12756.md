## Summary

Keep #12756 open. Current main still exposes the shared `read` message action, and Slack/Discord implement it, but the bundled Telegram plugin does not advertise, map, dispatch, document, or test Telegram message reads. The related PR #14367 is useful prior art for an inbound-store design, but it was closed unmerged in the provided GitHub context and its behavior is not present on main.

## Best Possible Solution

Keep #12756 open and implement Telegram `message read` end-to-end in the bundled Telegram plugin: advertise the action in discovery, map shared `read` to a Telegram runtime `readMessages` handler, maintain a bounded Telegram-safe inbound message store or equivalent source, document platform limitations, align CLI/tool parameters, and add focused adapter/runtime/contract tests. Treat closed PR #14367 as prior art, not shipped proof.
