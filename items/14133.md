## Summary

Keep #14133 open. Current main still does not implement a dedicated Reminders tab in Control UI/Mission Control, and the cron model/protocol/listing path still lacks the metadata/tag/type contract needed to distinguish one-shot reminder jobs from other cron jobs. The provided discussion also includes a 2026-04-26 maintainer-side Codex review from steipete keeping this same issue open, which matches the current checkout.

## Best Possible Solution

Keep this issue open as a core Control UI plus cron protocol enhancement. A good fix would add an additive cron classification/display metadata contract, update reminder-producing paths to set it, expose typed list filtering, keep Cron jobs filterable by non-reminder jobs, and add a Reminders tab with human-readable reminder text, fire time, channel/recipient, and delete actions.
