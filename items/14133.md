## Summary

Keep #14133 open. Current main still exposes reminders only through the generic cron surface: there is no Reminders navigation tab or render branch in Control UI, no reminder classification/display metadata in the cron wire contract, and no backend cron.list/listPage filter that can separate reminder one-shot jobs from ordinary cron jobs. The provided issue discussion also includes a same-day maintainer-authored Codex review from steipete explicitly keeping this open for the same missing core UI/protocol work.

## Best Possible Solution

Keep this open as a core Control UI plus Gateway cron protocol enhancement. The best path is an additive cron classification/display metadata contract, reminder-producing cron paths that set that metadata, typed list filtering for reminder versus non-reminder jobs, Cron tab filtering or hiding for reminder jobs, and a Reminders tab showing cleaned reminder text, fire time, channel/recipient, and delete actions with protocol/service/UI tests and docs updates.
