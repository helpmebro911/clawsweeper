## Summary

Keep #14133 open. Current main still has only the generic Control UI Cron tab, no Reminders route/tab, no reminder-specific cron metadata/display contract, and no cron.list/listPage filter that can separate reminder one-shot jobs from ordinary cron jobs. The existing issue discussion also contains a same-day Codex review from steipete keeping this open for the same missing core UI/protocol work.

## Best Possible Solution

Keep this open as a core Control UI plus Gateway cron protocol enhancement. The best path is an additive cron classification/display metadata contract, reminder-producing cron paths that set that metadata, typed list filtering for reminder/non-reminder jobs, Cron tab filtering or hiding for reminder jobs, and a Reminders tab showing cleaned reminder text, fire time, channel/recipient, and delete actions with protocol/service/UI tests and docs updates.
