## Summary

Keep this PR open. Current main still has the 60s Gateway model-pricing fetch timeout that this PR changes to 10s, and the timeout-warning regression test still expects 60s for both OpenRouter and LiteLLM. The PR is narrow and not implemented or superseded on the reviewed checkout; however, the change reverses a changelog-documented 60s slow-upstream tolerance, so it needs Gateway maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for Gateway maintainer review. If maintainers prefer fail-fast optional pricing enrichment, merge or adapt the two-file timeout/test change and separately evaluate whether startup should decouple pricing refresh even more. If the 60s slow-upstream tolerance remains intentional, close the PR with that product rationale.
