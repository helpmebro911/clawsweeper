## Summary

Keep this issue open. Current main still only provides session selection, label editing, filtering/sorting, and runtime override controls; it does not implement the requested left-side conversation management features: manual drag ordering, folder/tag/custom grouping, or moving conversations between groups.

## Best Possible Solution

Keep this open as a Control UI conversation-management enhancement. The best path is to design a first-class persistent session organization contract, likely extending SessionEntry/session row data plus gateway protocol APIs for reorder and move/group assignment, then add a left conversation list with drag ordering, folder/tag/custom category assignment, move actions, and double-click/context-menu rename. Reuse the existing label support for display-name editing, and add focused gateway/UI tests plus docs once the product shape is settled.
