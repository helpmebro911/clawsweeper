## Summary

Keep this issue open. Current main still exposes the Control UI agent file surface as a fixed bootstrap/core-file editor: `agents.files.list` only returns the bootstrap file set plus `MEMORY.md`, `agents.files.get/set` still reject names outside `ALLOWED_FILE_NAMES`, the protocol has no scope/path parameter, and the UI renders the surface as “Core Files” tabs rather than a recursive workspace or system browser. The stale bot comment is not enough to close this March 12, 2026 feature request, and the requested system-level browsing is security-sensitive core/gateway product work rather than a completed or clearly external ClawHub item.

## Best Possible Solution

Keep this issue open for maintainer product/security design. A good implementation would add an additive gateway protocol for file-browser scopes, start with a safe recursive agent-workspace mode using existing within-root safe-open primitives, update the Control UI with a directory tree/search and scope selector, and treat any system-filesystem mode as a separate admin-scoped capability with read/write policy, auditability, and docs. Existing bootstrap-file behavior should remain the default compatibility path.
