## Summary

Keep #64953 open. Current main still has the reported replace-mode catalog gap in both CLI picker paths and the gateway catalog path: the call sites still load the general model catalog, and the catalog loader still returns the registry plus plugin-augmented entries without filtering on `models.mode`. The PR remains a focused fix candidate for open issue #64950, although the review discussion correctly leaves a maintainer design question around whether replace mode should include implicit/plugin-discovered models.

## Best Possible Solution

Keep #64953 or a canonical successor open. Maintainers should define the replace-mode catalog contract, then implement one shared mode-aware catalog path for `promptDefaultModel`, `promptModelAllowlist`, and gateway `models.list`, with regression tests covering strict configured-provider replacement and any intended implicit/plugin-discovery behavior.
