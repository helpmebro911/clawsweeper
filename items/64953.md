## Summary

Keep this PR open. Current main still routes the CLI picker and gateway model catalog through the full model catalog without a `models.mode === "replace"` branch, while the same-author linked bug #64950 remains open. The PR is still a plausible fix candidate, but maintainers should settle the review-thread question about implicit/plugin-discovered models and add focused regression coverage before landing.

## Best Possible Solution

Keep this PR, #64950, or a maintainer-chosen successor open. The best fix is a shared mode-aware catalog path for CLI picker and gateway `models.list` that honors `models.mode: "replace"`, explicitly defines implicit/plugin-discovery semantics, preserves configured metadata and aliases, and adds regression tests for the default picker, allowlist picker, and gateway catalog output.
