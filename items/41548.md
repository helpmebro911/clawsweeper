## Summary

Keep open. Current main has shared session routing and identity-link primitives, but it still does not implement the requested unified inbox or automatic TUI/Telegram/iMessage cross-surface sync. TUI live events and history are scoped to the active session, channel delivery remains origin-channel based, and the closest related issues (#40678 and #51691) remain open rather than proving this broader request is shipped or obsolete.

## Best Possible Solution

Keep this open as a core TUI/Gateway product issue. Implement an explicit unified inbox or opt-in mirror/observe mode that uses existing gateway session events, labels source channel/session clearly, preserves origin-channel delivery safeguards, and defines whether replies are read-only, active-session scoped, or explicitly routed. Coordinate scope with #40678 for TUI observe mode and #51691 for real-time Telegram/TUI sync rather than closing this broader issue now.
