## Summary

Keep PR #47479 open. Current main still has the Docker setup .env handling gaps the PR is trying to address, although the patch would need rebasing onto the moved implementation in `scripts/docker/setup.sh`.

## Best Possible Solution

Keep the PR open and request a rebase/update: implement a safe project `.env` loader in `scripts/docker/setup.sh` that preserves caller-env precedence, make `upsert_env()` quote/escape whitespace-containing values without corrupting comments or quoted values, forward `OPENCLAW_INSTALL_BROWSER` when building, and add/update focused e2e coverage for `.env` startup defaults plus quoted multi-package values.
