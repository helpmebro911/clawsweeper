## Summary

Keep open. Current main still treats Feishu `requireMention` as a reply gate, not a context-recording boundary: unmentioned group messages are buffered into shared group history and injected into the next handled turn, and there is still no Feishu `contextIsolation`, `contextPolicy`, `excludeOtherBots`, or mention-only history policy.

## Best Possible Solution

Keep this issue open. Implement a backwards-compatible Feishu plugin context policy, likely global and per-group, defaulting to current behavior. When enabled, `extensions/feishu/src/bot.ts` should avoid recording unmentioned group messages into pending history and should filter messages addressed to other bots once reliable multi-bot mention identity is available. Update `extensions/feishu/src/config-schema.ts`, generated config metadata if required, `docs/channels/feishu.md` / public Feishu docs, and Feishu tests covering unmentioned pending-history replay plus multi-bot mention handling.
