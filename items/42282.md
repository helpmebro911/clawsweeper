## Summary

Keep open. Current main undercuts the issue's stated root cause and its silent-failure claim, but it still lacks the requested progress UX for long-running backups, and this read-only review did not run a live multi-GB reproduction. That is not strong enough evidence to close conservatively.

## Evidence

- **Backup creation still uses node-tar file mode:** `createBackupArchive()` still calls `tar.c({ file: tempArchivePath, gzip: true, ... }, ...)`. The `tar@7.5.13` package README says async commands with a `file` option return a Promise, and stream mode is only used when `file` is omitted. That weakens the report's claim that current code buffers the whole archive in memory before writing.
  - file: [src/infra/backup-create.ts:354](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/infra/backup-create.ts#L354)
  - command: `nl -ba src/infra/backup-create.ts | sed -n '320,370p'; npm view tar@7.5.13 readme --json`
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Current code cleans temp artifacts and propagates tar failures:** The archive write is wrapped in `try/finally`, and both the temp archive and manifest temp dir are removed in `finally`. Publication is also atomic via hard-link/copy fallback before the temp file is removed.
  - file: [src/infra/backup-create.ts:338](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/infra/backup-create.ts#L338)
  - command: `nl -ba src/infra/backup-create.ts | sed -n '142,176p'; nl -ba src/infra/backup-create.ts | sed -n '334,374p'`
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Tests cover tar failure cleanup:** `backup.atomic.test.ts` explicitly mocks `tar.c()` rejecting with `disk full`, expects `backupCreateCommand()` to reject, and verifies no final archive or leftover temp files remain. That contradicts the issue's current 'silent dangling .tmp file' claim.
  - file: [src/commands/backup.atomic.test.ts:58](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/commands/backup.atomic.test.ts#L58)
  - command: `nl -ba src/commands/backup.atomic.test.ts | sed -n '58,76p'`

- **No progress feedback on current main:** `backupCreateCommand()` only logs the final formatted summary after `createBackupArchive()` completes. There is no progress callback, byte counter, or interactive status update in the command path.
  - file: [src/commands/backup.ts:23](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/commands/backup.ts#L23)
