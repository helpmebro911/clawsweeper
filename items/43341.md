## Summary

Keep this issue open. Current main still does not implement live transcript subscription or session-message handling in the bundled TUI, even though the gateway and Control UI have the needed real-time primitives. The linked standalone `oclaw` TUI is useful prior art and workaround, but it does not fix `openclaw tui`.

## Best Possible Solution

Keep this issue open. Implement bundled TUI live transcript sync by adding a TUI subscription surface, subscribing to the active canonical session on connect and session changes, handling `session.message` with active-run deferral/history reload semantics, applying `sessions.changed` metadata/session-list refreshes, and adding focused TUI regression tests. Treat the linked standalone `oclaw` TUI as useful prior art or a workaround, not as a reason to close the OpenClaw core issue.
