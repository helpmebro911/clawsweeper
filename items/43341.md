## Summary

Current main still does not wire the bundled TUI into the gateway transcript event stream. The gateway exposes and tests `session.message` / `sessions.changed`, and Control UI consumes that path, but the TUI only handles `chat`, `chat.side_result`, and `agent` events and has no session subscription method.

## Best Possible Solution

Keep this issue open. Implement bundled TUI live transcript sync by adding TUI backend subscription methods, subscribing to the active canonical session on connect and session changes, handling `session.message` with active-run deferral/history reload semantics, updating session metadata from `sessions.changed`, and adding focused TUI regression tests. Coordinate with #51691 only if maintainers choose a canonical tracker explicitly.
