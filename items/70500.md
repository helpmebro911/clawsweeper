## Summary

Keep open. Current main does not contain PR #70500’s docs warning about granting macOS Accessibility to a shared `node` runtime, and the closed predecessor #69654 is not merged. The existing docs cover stable TCC/app identity and PeekabooBridge security, but they do not give the specific user-facing warning this PR adds. Greptile’s comment also supports the PR as accurate docs-only hardening, and PRtags correctly identifies #70500 as the active reopening of #69654.

## Best Possible Solution

Keep #70500 open for maintainer review or apply the same docs change directly: add the Node/CLI Accessibility warning to `docs/platforms/mac/permissions.md`, link it from `docs/platforms/mac/peekaboo.md`, then run the repository docs check when the docs toolchain is available.
