## Summary

Keep open. Current main still has the reported Lobster behavior: unresolved workflow-file candidates are swallowed by `detectWorkflowFile` and then sent to the embedded runtime as inline pipelines. This PR is a focused implementation candidate for open bug #68101, but it should not merge as written because its extension-only heuristic can misclassify valid inline pipelines ending in `.lobster`, `.yaml`, `.yml`, or `.json`.

## Best Possible Solution

Keep this PR open as an implementation candidate for #68101, but require a revision before merge: preserve path errors only for inputs that are clearly workflow-file references, keep ordinary inline pipelines on the runtime path, and add paired regression tests for both a missing relative workflow file and a valid inline pipeline containing a workflow-like file argument. Close this PR only after #68101 is fixed on main or explicitly superseded by a merged PR.
