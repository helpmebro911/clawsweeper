## Summary

Current main still does not implement the reported OpenRouter/Hunter Alpha failure mode. The observed reply arrives only in a thinking block; current parsing still treats bare `reasoning`, `reasoning_content`, and `reasoning_text` as thinking, visible extraction still ignores thinking blocks, and the reasoning-only retry guard does not cover OpenRouter/Hunter Alpha.

## Best Possible Solution

Keep this issue open. Add a targeted regression for an OpenRouter/Hunter Alpha completions stream where the answer starts with `[[reply_to_current]]` but arrives only in a bare reasoning field, then fix the OpenRouter-owned path conservatively: either disable proxy reasoning injection for Hunter Alpha when it cannot separate hidden reasoning from visible content, or add a narrowly scoped compatibility path that surfaces only proven answer text through normal reply directive parsing for webchat and Telegram.

## Risks / Open Questions

- A broad fallback that promotes all thinking-only content into visible text could leak genuine hidden reasoning. The fix should be scoped to the affected OpenRouter route or to provider-marked visible output.
- OpenRouter route behavior may change upstream, so regression coverage should mock the observed bare `reasoning`, `reasoning_content`, or `reasoning_text` stream shape rather than depending on live availability.
