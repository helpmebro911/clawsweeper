## Summary

Current `main` does not implement this PR's requester-channel hydration in `sessions_send` A2A. The flow still uses `params.requesterChannel` directly and falls back to internal `webchat` behavior when that field is omitted; newer helper-level guards only protect already-established external routes, not the broader omission case this PR targets.

## Evidence

In the ping-pong and announce prompts it still passes `params.requesterChannel` through directly, and `sourceChannel` becomes `undefined` when that param is omitted.
