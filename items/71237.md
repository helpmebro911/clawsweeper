## Summary

Keep PR #71237 open. Current main has partial orphan-transcript handling through doctor archival and disk-budget pressure cleanup, but it does not implement the requested `pruneOrphanedTranscripts` utility or an equivalent explicit sessions-cleanup path that unlinks old unreferenced JSONL transcripts. The PR is recent, focused, and the provided discussion shows author follow-up on review feedback, so this needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best product path is to either merge the utility with its tests and a clear caller contract, then wire it into `openclaw sessions cleanup`, or fold the same safe orphan-pruning logic directly into the cleanup command with union-of-references handling across stores and checkpoint preservation. Current main should not be treated as already implementing this PR.
