## Summary

Keep this PR open. Current main has related session maintenance, missing-transcript cleanup, doctor orphan archival, and pressure-gated disk-budget cleanup, but it still does not provide the proposed reusable orphan-transcript pruning utility or an ordinary on-demand cleanup path that scans already-unindexed JSONL transcripts and unlinks them.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to merge the tested primitive as a staged building block and follow with explicit openclaw sessions cleanup wiring, or ask the author to include the cleanup command wiring now, with preservedPaths built from all relevant stores, resolveSessionTranscriptCandidates, topic transcript derivations, and compaction checkpoint metadata.
