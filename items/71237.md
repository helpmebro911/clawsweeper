## Summary

Keep PR #71237 open. Current main has overlapping session maintenance, doctor orphan detection, and disk-budget cleanup, but it does not include the proposed reusable `pruneOrphanedTranscripts` utility or an unconditional on-demand orphan transcript sweep. The PR is recent, scoped to a real sessions performance/storage problem, and review comments have already driven safety improvements around concurrency and checkpoint preservation contracts.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either to land a hardened reusable orphan-transcript primitive and then wire it into `openclaw sessions cleanup` with multi-store preserved-path collection, or supersede the PR with an equivalent main implementation that gives operators a dry-run/enforce orphan sweep while preserving current doctor/disk-budget safety rules.
