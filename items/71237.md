## Summary

Current main has related session maintenance, but it still lacks the proposed `pruneOrphanedTranscripts` primitive or an equivalent on-demand sweep that scans a sessions directory and unlinks already-unindexed `.jsonl` transcripts. This PR is recent, active, and technically scoped, with review follow-up addressing concurrency and checkpoint-preservation concerns, so it remains useful for maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is either to merge the well-tested primitive as a staged building block and follow with explicit `openclaw sessions cleanup` wiring, or supersede it with a narrower PR that implements the full cleanup command directly while collecting preserved paths from all relevant stores, transcript candidates, topic transcripts, and compaction checkpoint snapshots.
