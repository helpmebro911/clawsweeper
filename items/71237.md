## Summary

Keep this PR open. Current main has related session cleanup, doctor orphan archival, and pressure-gated disk-budget cleanup, but it still does not provide the proposed reusable orphan-transcript pruning primitive or an ordinary on-demand path that scans already-unindexed JSONL transcripts and unlinks them. The PR is recent, contributor-authored, scoped to session maintenance plus tests, and active review feedback is still being addressed.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to address the current checkpoint-pattern review, then decide whether to merge the tested primitive as a staged building block or require the same PR to wire `openclaw sessions cleanup` with a union-of-live-paths strategy across all relevant stores, `resolveSessionTranscriptCandidates`, topic transcript derivations, and compaction checkpoint metadata.

## Risks / Open Questions

- The PR introduces an opt-in file-unlink primitive; even though it is not wired into runtime paths, the preserved-path contract and file classification need maintainer review before merge.
- The latest review comment indicates the checkpoint skip may overmatch valid orphan transcript filenames containing `.checkpoint.` and leave real orphans behind.
- Merging only the utility will not yet give operators the full cleanup workflow; follow-up CLI wiring must collect preserved paths across custom stores, neighboring stores, topic transcripts, symlink-normalized paths, and compaction checkpoints.
