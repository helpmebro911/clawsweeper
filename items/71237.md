## Summary

Keep PR #71237 open. Current main has partial orphan-transcript mitigation through store-maintenance archival, doctor repair, and budget-gated cleanup, but it does not contain the proposed `pruneOrphanedTranscripts` utility and `sessions cleanup` still has no direct orphan transcript sweep. The PR is recent, active, and its discussion addressed review feedback rather than being obsolete.

## Best Possible Solution

Keep this open for maintainer review. The best path is either to merge/refine a shared orphan-prune primitive and then wire it into `openclaw sessions cleanup` with dry-run/enforce reporting, or supersede it with an equivalent implementation that safely gathers all live transcript paths across stores and checkpoints before deleting old orphan `.jsonl` files.

## Risks / Open Questions

- The PR utility is intentionally not wired into runtime or `sessions cleanup`, so merging it alone would not reclaim historical orphans until follow-up CLI integration lands.
- The caller contract must preserve all live transcript paths, including compaction checkpoint snapshots; the eventual command should build that union explicitly rather than relying only on primary transcript candidates.
