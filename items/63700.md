## Summary

Keep this issue open. Current main still does not implement the requested repeatable `--file` surface for `openclaw infer model run`: the command is registered as prompt-only, execution forwards only the prompt string to local and gateway paths, and the documented file-based image/audio commands are separate workflows rather than custom-prompt multimodal model runs. The existing discussion's note that #69030 is adjacent raw-mode work remains accurate, not a duplicate or shipped fix.

## Best Possible Solution

Keep this issue open and implement the core CLI capability. Add repeatable `--file` handling to `src/cli/capability-cli.ts` metadata and command registration, read and MIME-validate supported files, forward image inputs to local `agentCommand` through `images`/`imageOrder`, forward gateway runs through `attachments`, decide the audio representation before claiming audio support, and update `docs/cli/infer.md` plus focused CLI tests. Coordinate with #69030 because raw mode and file attachments both affect `infer model run`, but do not treat it as a duplicate.
