## Summary

Keep PR #48682 open. The linked bug is still real on current main: GitHub Copilot implicit discovery still resolves the first stored profile instead of `auth.order`, and the current tests still assert first-profile behavior. The PR contains useful targeted regression coverage and addresses #46031 conceptually, but the patch is stale against the current plugin-owned discovery layout and needs rework to avoid bypassing plugin disablement.

## Best Possible Solution

Keep the PR open and ask for a rebase/rework: implement the fix in the GitHub Copilot plugin token path, likely by using the existing order-aware provider-auth SDK helper or `resolveAuthProfileOrder`, and add focused tests for `auth.order`, unresolved ordered token fallback, stale stored order fallback, and plugin discovery disablement. Keep #46031 as the linked canonical bug until that rebased fix lands.
