## Summary

Keep open. Current main still does not implement a Wear OS companion, phone-side Wear proxy, Wear Gradle module, Data Layer dependency, tile/complication surface, or Wear docs. The PR is large and contributor-abandoned, but it is concrete, unsuperseded feature work with manual Pixel Watch/emulator verification and user interest, so no conservative auto-close reason applies.

## Best Possible Solution

Keep #47604 open or convert it into an explicit maintainer-owned Wear OS plan. The practical path is to split and rebase it into reviewable pieces: shared Android gateway protocol helpers, phone-side Wear proxy, standalone Wear app module, then tile/complication/notification polish with focused tests and manual Pixel Watch proof. If maintainers decide not to ship Wear OS in core, that should be an explicit product decision rather than automated stale cleanup.
