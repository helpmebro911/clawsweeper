## Summary

Keep open. Current `main` has partial reply-metadata display support, but the WebChat UI still does not implement the requested native reply/quote workflow: no hover Reply action, no composer state for selecting a parent message, and no structured reply field on `chat.send` for the WebChat client path.

## Evidence

- **WebChat uses the shared Control UI chat client:** The browser client for this UI connects in `webchat` mode, so the chat rendering/sending code under `ui/src/ui/**` is the relevant WebChat implementation.
  - file: [ui/src/ui/app-gateway.ts:285](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/app-gateway.ts#L285)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '275,292p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Message hover actions only expose existing non-reply actions:** `grouped-render.ts` computes `hasActions` from `canCopyMarkdown || canExpand` and renders only expand/copy buttons in `.chat-bubble-actions`; there is no Reply/Quote action wired into the hover toolbar.
  - file: [ui/src/ui/chat/grouped-render.ts:1414](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/chat/grouped-render.ts#L1414)

  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Current reply UI is display-only:** `renderReplyPill()` only renders a pill when a message already contains parsed reply metadata (`replyTarget`); it does not provide any interaction for creating a reply from WebChat.
  - file: [ui/src/ui/chat/grouped-render.ts:793](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/chat/grouped-render.ts#L793)
  - command: `nl -ba ui/src/ui/chat/grouped-render.ts | sed -n '790,810p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **WebChat send path has no reply parameter:** The WebChat controller sends only `sessionKey`, `message`, `deliver`, `idempotencyKey`, and `attachments` via `chat.send`, and the `ChatSendParamsSchema` likewise has no `replyToId`/quote field. That means the current UI has no native way to attach a selected parent message when sending.
  - file: [ui/src/ui/controllers/chat.ts:232](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/controllers/chat.ts#L232)

- **Existing support is manual-tag parsing, not native reply UX:** The message normalizer parses inline directive tags like `[[reply_to:<id>]]` / `[[reply_to_current]]` into `replyTarget` for rendering. That is compatible with reply metadata arriving from elsewhere, but it is not the requested WebChat-side hover-to-reply feature.
  - file: [ui/src/ui/chat/message-normalizer.ts:193](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/chat/message-normalizer.ts#L193)

## Risks / Open Questions

- This review is source-inspection only. I did not run the UI interactively in this environment.
- There is partial reply metadata support via inline directive tags, so someone skimming the UI code could mistake that for a full WebChat reply feature; it is not equivalent to the requested native hover/composer flow.
