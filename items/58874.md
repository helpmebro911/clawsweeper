## Summary

Keep open. Current `main` still hardcodes `sourceChannel` to `INTERNAL_MESSAGE_CHANNEL` in both the descendant-wake path and the completion announce path, so the channel-propagation bug described in #55931 is not implemented on `main`. The PR discussion also shows the proposed change was narrowed after review and never landed a complete fix.
