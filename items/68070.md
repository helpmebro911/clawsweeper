## Summary

Keep this PR open. Current main still returns plain parameter-validation errors and wraps tool failures into ordinary JSON tool results without a `retryable: false` marker, so the PR is not obsolete or implemented on main. The related UX issue #65235 is adjacent user-facing cleanup, not a superseding fix for validation-error retryability.

## Best Possible Solution

Review this PR as an active focused fix candidate. The best path is to add regression coverage showing read/write/edit parameter validation failures produce non-retryable tool-result metadata and stop repeated model retries, then prefer a typed local validation error or documented adapter seam if the Pi result contract supports it.
