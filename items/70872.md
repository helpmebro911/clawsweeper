## Summary

Keep #70872 open. Current main still forwards bundle MCP tool inputs without schema-aware coercion for nested object/array properties, so a model-provided `{ "params": "{}" }` remains a string inside the MCP `arguments` object. The linked #70882 PR directly tracks a plausible fix, but it is open and unmerged, so the issue is not implemented or superseded on main yet.

## Best Possible Solution

Keep the issue open and review/land PR #70882 or an equivalent schema-aware fix. The fix should live at the bundle MCP materialization/runtime boundary, parse stringified JSON only for properties whose MCP `inputSchema` expects `object` or `array`, preserve already-correct values, and add regression tests for `params: "{}"`, arrays, invalid JSON strings, and unchanged object inputs.
