## Summary

Keep #70872 open. Current main and latest release v2026.4.24 still pass bundled MCP tool inputs through without schema-aware coercion of stringified object/array properties, so `{ "params": "{}" }` can still reach strict MCP servers as a string. The linked PR #70882 proposes the right targeted fix and tests, but it is open and unmerged.

## Best Possible Solution

Keep the issue open until #70882 or an equivalent maintainer-reviewed fix lands. The likely fix belongs in `src/agents/pi-bundle-mcp-materialize.ts`: before `SessionMcpRuntime.callTool`, inspect the MCP tool `inputSchema.properties` and parse only string values whose property schema declares `type: "object"` or `type: "array"`, with regression tests covering `params: "{}"`, stringified arrays, already-correct objects, and invalid JSON pass-through.
