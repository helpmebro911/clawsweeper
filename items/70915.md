## Summary

Keep #70915 open. Current `main` still routes ZAI media-understanding requests through the generic model-backed image path with the configured provider `baseUrl`, and the bundled ZAI plugin does not currently rewrite coding endpoints for vision models. The linked bug has concrete reproduction data, and the PR targets the implicated bundled ZAI media-understanding surface, though the request-scoped config rewrite should be reviewed against the noted shared `models.json` race before merge.

## Best Possible Solution

Keep the PR or linked issue active. The best fix is a bundled ZAI-owned image routing change, ideally in the media-understanding provider or a suitable provider runtime transport seam, that maps ZAI vision-model image requests from `/api/coding/paas/vN` to `/api/paas/vN` without mutating shared provider config and with focused regression tests for `glm-4.6v`, `glm-4.5v`, and `glm-5v-turbo`.
