## Summary

Current `main` does not implement PR #51292’s configurable exec approval timeout. The old 120s pressure appears reduced because `main` now hardcodes a longer shared default of 1,800,000 ms, but there is still no `approvals.timeoutMs` config/schema/docs surface or env-var-based timeout resolver on current `main`.

## Evidence

- **Approvals config type has no timeout field:** `ApprovalsConfig` currently exposes only `exec` and `plugin`; there is no `timeoutMs` property matching the PR.
  - file: [src/config/types.approvals.ts:29](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/config/types.approvals.ts#L29)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Approvals schema still rejects extra keys:** `ApprovalsSchema` is a strict object with only `exec` and `plugin`, so `approvals.timeoutMs` is not part of the validated config surface on `main`.
  - file: [src/config/zod-schema.approvals.ts:23](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/config/zod-schema.approvals.ts#L23)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Docs/help surface lacks the requested knob:** The approvals help section documents forwarding settings only; there is no `approvals.timeoutMs` help entry on `main`.
  - file: [src/config/schema.help.ts:661](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/config/schema.help.ts#L661)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Main uses a hardcoded shared exec approval timeout:** `DEFAULT_EXEC_APPROVAL_TIMEOUT_MS` is hardcoded to `1_800_000` ms on current `main`, with no resolver layered on top here.
  - file: [src/infra/exec-approvals.ts:167](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/infra/exec-approvals.ts#L167)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Gateway still falls back to the constant:** `exec.approval.request` uses the request’s explicit `timeoutMs` when present, otherwise falls back directly to `DEFAULT_EXEC_APPROVAL_TIMEOUT_MS`; it does not consult config.
  - file: [src/gateway/server-methods/exec-approval.ts:145](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/gateway/server-methods/exec-approval.ts#L145)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Agent-side registration still sends an explicit default timeout:** Agent exec approval registration builds the gateway payload with `timeoutMs: DEFAULT_APPROVAL_TIMEOUT_MS`, so the current flow remains hardcoded rather than config-driven.
  - file: [src/agents/bash-tools.exec-approval-request.ts:55](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/agents/bash-tools.exec-approval-request.ts#L55)

## Risks / Open Questions

- The original symptom behind `#51287` may be partly obsolete because `main` now defaults approvals to 30 minutes instead of 120 seconds, but that is not the configurable feature this PR proposes.
- Current exec-approval code on `main` has moved since March 2026, so reviving this work would likely require a fresh rebase/reimplementation against the newer approval request helpers rather than merging the PR as-is.
