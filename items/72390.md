## Summary

Keep #72390 open. Current main does not support SecretRef objects in top-level `openclaw.json` `env.vars`: the config schema and public config type are string-only, and the runtime env collection path only accepts literal string values. Related MCP server env SecretRef support exists, but it is scoped to `plugins.entries.acpx.config.mcpServers.*.env.*` and does not implement this issue’s requested global `env.vars` behavior.

## Best Possible Solution

Keep the issue open as a core config/secrets enhancement. The likely implementation path is to promote `env.vars.*` values from `string` to `SecretInput`, add a core SecretRef collector/resolver for `env.vars.*`, update schema/docs/generated config metadata, and test runtime env creation, process-env application, and service-env persistence semantics.
