## Summary

Current main still does not implement optional marquee or overflow-scrolling behavior for long status information. The prior Codex review remains valid: the relevant Control UI and TUI surfaces use static controls, ellipsis, wrapping, or ordinary chat/log scrolling rather than an accessible overflow-status component.

## Best Possible Solution

Keep this issue open as a product/UI enhancement. The best path is a reusable accessible overflow-status component/helper, with explicit opt-in adoption in selected Control UI and TUI status/footer surfaces. It should render statically when content fits, scroll only on real overflow, respect reduced-motion preferences, pause or allow user control on hover/focus, and treat #13670 as related narrower model/token visibility work rather than a replacement.
