## Summary

Keep PR #39389 open. Current main still lacks Feishu-specific documentation for `match.accountId`, and the PR addresses a real docs gap, but its proposed wording is not safe to auto-close or merge as-is because Feishu `channels.feishu.defaultAccount` is documented and implemented as default account selection, while route bindings with omitted `match.accountId` are indexed under the literal `default` account bucket unless the inbound route itself uses that id.

## Best Possible Solution

Keep the PR open and revise the Feishu docs patch in `docs/channels/feishu.md`. The best version should document `match.accountId` directly in the Feishu routing fields: omitted means the literal default account routing scope, explicit ids such as `main` match that named Feishu account, and `accountId: "*"` matches any configured Feishu account. It should also say `channels.feishu.defaultAccount` controls outbound/default account selection and should not be read as an inbound binding wildcard. Coordinate with #49769 for generic wording, but keep the Feishu-specific example on the Feishu channel page.
