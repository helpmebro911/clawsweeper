## Summary

Keep this PR open. Current main still does not contain an OpenShell platform deployment page or navigation entry, and the existing OpenShell docs cover OpenClaw using OpenShell as a sandbox backend from the host, not operating OpenClaw inside an OpenShell sandbox. The PR is docs-only and active, but its operational guidance is security-sensitive and needs maintainer docs/security review before merge.

## Best Possible Solution

Keep this PR open for maintainer docs/security review. The best path is to reconcile the new platform guide with docs/gateway/openshell.md and current Slack/proxy/fetch-guard behavior, remove duplicate glossary entries, then either merge it as a dedicated Platforms deployment guide or ask the author to fold the durable operator guidance into the canonical OpenShell docs.
