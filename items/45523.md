## Summary

Keep #45523 open. Current main still does not add `gateway.auth.mode=none` or trusted-proxy `allowUsers` coverage to the dangerous-config-flags collector used by the gateway startup warning. Main has related audit/runtime safeguards, including an audit warning for missing trusted-proxy allowlists and runtime rejection of whitespace-only `trustedProxy.userHeader`, but those do not implement the PR's stated startup dangerous-flag behavior or schema hardening.

## Best Possible Solution

Keep the PR or a successor open. The best path is to add the two gateway auth entries to `src/security/dangerous-config-flags.ts`, cover both `undefined` and `[]` for `trustedProxy.allowUsers`, add focused tests in `src/security/dangerous-config-flags.test.ts` and startup/audit coverage as needed, and decide whether to keep the schema-level `userHeader` refine in addition to the existing runtime validation.
