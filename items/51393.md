## Summary

Current `main` does not implement statement support for `browser act kind=evaluate`. The Playwright-backed evaluate path still wraps user input as `eval("(" + fnBody + ")")`, which only parses expressions/functions, and the request normalization/CLI/docs still present `fn` as function source rather than auto-wrapping statements.

## Evidence

- **Playwright evaluate still expression-only for element refs:** The element evaluate path parses the provided `fnBody` with `eval("(" + fnBody + ")")` and throws `Invalid evaluate function` on parse errors, so statement bodies like `var x = 1` still fail.
