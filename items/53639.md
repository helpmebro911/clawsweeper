## Summary

Keep open. Current main only adds a partial local/self-hosted endpoint mitigation; it still schedules gateway model-pricing refresh on normal startup, still fetches OpenRouter and LiteLLM pricing catalogs with 60s timeouts for external model refs, and still has no `models.pricing.enabled` disable surface. The linked open PR #68651 is the direct implementation candidate with closing syntax, so this issue should remain open until that PR is reviewed and merged, replaced, or closed.

## Best Possible Solution

Keep this issue open and review #68651 as the canonical fix candidate. A complete fix should add a validated `models.pricing.enabled` config key with default behavior preserved, skip all pricing bootstrap/catalog network work when disabled, update schema/help/labels/docs/generated baselines and changelog as appropriate, and add focused tests proving disabled pricing does not fetch OpenRouter or LiteLLM while default/enabled behavior still works.
