## Summary

Keep open. Current main still starts model-pricing refresh during normal gateway startup, still performs OpenRouter and LiteLLM catalog fetches with a 60s timeout, and still has no validated `models.pricing.enabled` disable surface. The provided GitHub context also shows open PR #68651 as the implementation candidate that explicitly proposes the missing disable switch, so this issue should remain open until that PR is reviewed/merged or closed.

## Best Possible Solution

Keep this issue open and review #68651 as the canonical implementation candidate. A solid fix should add a validated `models.pricing.enabled` config key with default behavior preserved, skip all pricing bootstrap/refresh network work when disabled, update config docs/schema/help, and include focused gateway/config tests. PR #72033 may improve diagnostics, but it does not replace the disable or non-blocking startup fix requested here.
