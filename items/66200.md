## Summary

Keep this PR open. The hardening goal is still relevant, but the submitted patch is not merge-ready because it wraps the legacy BlueBubbles envelope `Body` while current `main` still routes the model-facing text through raw `BodyForAgent`/`BodyStripped`. The review discussion already identified both the missing empty-body guard and the more important model-visible path gap.

## Best Possible Solution

Keep this PR open and request a focused revision or replacement. The fix should harden the actual BlueBubbles model-facing path, likely by setting an appropriate wrapped `BodyForAgent` or `UntrustedContext` while preserving raw `RawBody`, `CommandBody`, and `BodyForCommands` for command parsing, reset handling, dedupe, and metadata. It should include the Discord-style non-empty guard and focused BlueBubbles tests for the prompt-bound context.

## Risks / Open Questions

- Merging the PR as-is would create a false sense of prompt-injection hardening because the normal BlueBubbles prompt path can still use raw `BodyForAgent`.
- A correct fix must preserve raw command/dedupe fields while hardening only the agent-facing text or structured untrusted context path.
- The patch currently lacks targeted BlueBubbles tests proving inbound text reaches the agent with the intended untrusted-content boundary.
