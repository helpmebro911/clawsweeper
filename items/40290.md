## Summary

Keep open. Current `main` still strips admin-configured sandbox env vars that match the credential blocklist, so the main bug described by PR #40290 is not implemented yet. The PR also still appears to need follow-up because its skill-filter bypass would override the documented `skills.entries.<key>.enabled: false` disable behavior.
