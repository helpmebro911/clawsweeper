## Summary

Keep open. Current main still lacks a configured-key escape hatch for Docker sandbox env sanitization, so credential-shaped keys explicitly set in sandbox Docker config are still stripped. The PR targets a real bug, but its current diff still includes unrelated skill eligibility bypass behavior and lacks focused regression tests.

## Best Possible Solution

Keep this PR open for a focused revision. Narrow it to the Docker sandbox env bug: add a configured-key escape hatch to sanitizeEnvVars, apply it only to keys explicitly present in resolved sandbox.docker.env when building Docker create args, preserve value validation and host/process-env blocking, add sanitizer plus Docker create-args regression tests, and drop the unrelated skill eligibility bypass unless maintainers separately approve that policy change.
