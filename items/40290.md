## Summary

Keep open. Current main still lacks an admin force-allow path for credential-shaped keys in configured Docker sandbox env, so the core sandbox bug is not implemented or obsolete. The PR contains a plausible env fix, but it should not land as-is because the skill-filter change can bypass existing skill eligibility policy.

## Best Possible Solution

Keep the PR open for author or maintainer follow-up. Narrow it to the sandbox Docker env behavior: add a `forceAllowKeys`-style sanitizer option for keys explicitly present in configured `sandbox.docker.env`, pass those keys from `buildSandboxCreateArgs`, add sanitizer and create-args regression tests, remove the unused `validateEnvVars` helper unless there is a real caller, and drop or redesign the skill-filter bypass so `shouldIncludeSkill` remains authoritative for disabled, bundled-disallowed, and runtime-ineligible skills.
