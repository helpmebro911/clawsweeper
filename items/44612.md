## Summary

Keep open. Current `main` still rejects whitespace in `OPENCLAW_CONFIG_DIR` and mount specs, so the reported Docker bug is not implemented yet. This PR also became unsafe for conservative cleanup because it mixes the Docker fix with unrelated browser-route changes that conflict with the current browser plugin contract and were already called out in review as regressions.

That means the underlying Docker path-with-spaces problem remains on `main`, including the `OPENCLAW_HOME_VOLUME` compose overlay path.
