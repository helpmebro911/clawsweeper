## Summary

Keep open. Current main still has the reported mismatch in repo-owned guidance: the CLI and main devices docs correctly make `approve --latest` preview-only, but bundled node-connect skill docs and Android README still present `openclaw devices approve --latest` as the approval step. Related PR #70667 appears to be the right fix, but it is open and not merged into current main.

## Best Possible Solution

Keep the secure preview-only CLI behavior from #64160. Land or otherwise apply the docs-only fix from PR #70667: update `skills/node-connect/SKILL.md` and `apps/android/README.md` to show the two-step flow (`devices list` / preview, then `devices approve <requestId>`), and audit nearby recovery text such as status fallback wording if maintainers want to avoid implying `--latest` approves.
