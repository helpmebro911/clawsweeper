## Summary

Keep this issue open. Current main still allows `sessions_history` to forward an oversized caller-supplied limit to `chat.history`, while the Gateway contract still rejects values above 1000. The provided GitHub context also shows an open, unmerged PR with closing syntax for this issue, so the issue should remain open until that PR is reviewed and landed or rejected.

## Best Possible Solution

Review #66650 against current main, then land it or an equivalent focused fix that adds the Gateway maximum to the `sessions_history` schema, clamps outgoing `chat.history` requests to 1000, and adds regression coverage for oversized requested limits. If maintainers reject that PR, close or supersede it first and keep this issue tracking the remaining bug.
