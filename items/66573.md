## Summary

Current main still has the reported bug: `sessions_history` accepts any numeric `limit >= 1` and forwards it to `chat.history` without clamping, while the Gateway protocol validator rejects `chat.history.limit > 1000` before the handler's internal hard cap can apply. The provided GitHub context also shows open PR #66650 with closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #66650 or an equivalent narrow fix that adds the Gateway maximum to the `sessions_history` schema, clamps outgoing `chat.history` requests to 1000 before the Gateway call, and adds regression coverage for oversized caller-supplied limits.
