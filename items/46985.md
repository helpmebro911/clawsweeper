## Summary

Keep open. Current main still commits full cumulative WebChat stream text into tool-interrupted stream segments and then renders the unmodified live stream separately, so the duplicated-prefix bug remains plausible. The related PR #47377 is narrower and only changes segment commit logic; it does not cover #46985's live-preview dedupe path, so it is not a complete superseder.

## Best Possible Solution

Keep the item open, but treat the best path as a focused rebase/port of the WebChat streaming dedupe into current `ui/src/ui/app-tool-stream.ts` and `ui/src/ui/chat/build-chat-items.ts`, with regression tests for segment slicing, live preview dedupe, unchanged cumulative streams, and repeated identical incremental segments. Reconcile with #47377 for the narrower segment-slicing work, and split the unrelated audit/inbound/plugin-registry changes into separate PRs if they are still wanted.
