## Summary

Keep this PR open. Current main still has the over-broad Feishu external-key guard that rejects legitimate opaque keys containing `..`, and the PR is a focused fix for that bundled Feishu plugin path. It should not land as-is until the validator also rejects exact dot-segment keys such as `.` and `..`.

## Best Possible Solution

Refresh this PR with a narrow Feishu plugin validator fix: keep empty/length/control-character and `/`/`\` rejection, allow embedded `..` inside opaque Feishu keys, explicitly reject exact `.` and `..`, and add regression tests for valid embedded-dot keys plus invalid dot-segment/path-separator inputs. No OpenClaw core change is needed.
