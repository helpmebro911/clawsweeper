## Summary

Keep this PR open. Current main still rejects legitimate Feishu media keys containing embedded `..`, so the linked Feishu image-analysis regression is not implemented on main. The PR is focused on the right bundled Feishu plugin surface, but it should be refreshed before merge to preserve fail-closed handling for exact dot-segment keys.

## Best Possible Solution

Refresh this PR with the narrow Feishu plugin fix: keep empty, length, control-character, `/`, and `\` rejection; allow embedded `..` inside opaque Feishu keys; explicitly reject exact dot-segment tokens such as `.` and `..`; and keep regression tests for valid embedded-dot keys plus invalid path-separator and dot-token inputs. No OpenClaw core or ClawHub move is needed.
