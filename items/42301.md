## Summary

Keep open. Current `main` still rejects any Feishu external key containing `".."`, so the bug this PR addresses is still present and the proposed safe fix is not already implemented on `main`.

## Evidence

- **Current validator still blocks legitimate `..` substrings:** `normalizeFeishuExternalKey()` returns `undefined` for any string containing `/`, `\\`, or `..`. That still rejects keys like `img_v2_test..key` described in the PR.
  - file: [extensions/feishu/src/external-keys.ts:15](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/extensions/feishu/src/external-keys.ts#L15)

  - sha: [d29eaeafc1fc](https://github.com/openclaw/openclaw/commit/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132)
- **Rejected keys are dropped before Feishu attachments are processed:** Post parsing only appends image/file keys when `normalizeFeishuExternalKey()` succeeds, so a legitimate key containing `..` is filtered out before downstream media handling.
  - file: [extensions/feishu/src/post.ts:153](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/extensions/feishu/src/post.ts#L153)
