## Summary

Keep open. Current `main` still passes the configured workspace path straight through to Docker bind mounts, and the docs now describe Docker-out-of-Docker as requiring host-path parity rather than automatically remapping gateway-container paths back to host paths. I found no current code or tests showing the reported Docker-install `workspaceAccess` bind problem has been fixed on `main`.

## Evidence

- **Sandbox context uses configured workspace path directly:** `ensureSandboxWorkspaceLayout()` resolves `agentWorkspaceDir` from the configured workspace path and, for `workspaceAccess: "rw"`, uses that same path as `workspaceDir` with no host/container path translation layer.
  - file: [src/agents/sandbox/context.ts:35](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/sandbox/context.ts#L35)
  - command: `nl -ba src/agents/sandbox/context.ts | sed -n '20,80p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Workspace mount strings are literal path passthroughs:** `appendWorkspaceMountArgs()` formats Docker `-v` arguments directly from `workspaceDir` and `agentWorkspaceDir` (`hostPath: workspaceDir` / `hostPath: agentWorkspaceDir`). There is no remap from gateway-container paths to host-visible paths before mount creation.
  - file: [src/agents/sandbox/workspace-mounts.ts:14](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/sandbox/workspace-mounts.ts#L14)
  - command: `nl -ba src/agents/sandbox/workspace-mounts.ts | sed -n '1,80p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Docker container creation uses those paths unchanged:** `createSandboxContainer()` calls `appendWorkspaceMountArgs()` after building create args, so the sandbox container is created from the same resolved workspace strings without any inspection of the gateway container's bind mounts.
  - file: [src/agents/sandbox/docker.ts:482](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/sandbox/docker.ts#L482)
  - command: `nl -ba src/agents/sandbox/docker.ts | sed -n '470,520p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Current docs describe a constraint, not an implemented remap:** The sandbox docs explicitly say that in Docker-out-of-Docker the `workspace` config must use the host absolute path and the gateway container must mount that exact host path identically. That is consistent with the current direct-path code and indicates the issue's requested automatic remap is not implemented on `main`.
  - file: [docs/gateway/sandboxing.md:85](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/gateway/sandboxing.md#L85)
