## Summary

Keep #65309 open. Current main contains useful mitigations for Active Memory, including toolsAllow forwarding and a hard timeout deadline, but it still runs Active Memory synchronously in the before_prompt_build path and can block an eligible direct-chat reply until config.timeoutMs expires. The issue also has fresh 2026-04-26 confirmation, so it is not obsolete or superseded by the narrower closed mitigation issues.

## Best Possible Solution

Keep #65309 open as the canonical Active Memory direct-chat latency/reliability tracker unless a maintainer intentionally splits it. A real fix should make Active Memory safe for primary chat by adding a non-blocking/background recall mode, a much smaller foreground fail-open budget with optional late/background memory update, or an explicit prewarm/readiness gate so cold memory/indexing cost is not paid inside the first user turn. The closed #67878 and #69892 fixes should remain treated as partial mitigations, not full supersession of this broader product bug.
