## Summary

Current `main` still triggers heartbeat wakes for subagent exec completions and intentionally reroutes those wakes onto the main session instead of suppressing them. Existing tests codify that fallback-to-main heartbeat behavior, so the reported spurious wake/token issue is still actionable rather than already fixed.

## Evidence

- **Subagent exec paths still request heartbeat wakes unconditionally:** Both background-exec completion helpers enqueue a system event and immediately call `requestHeartbeatNow(scopedHeartbeatWakeOptions(sessionKey, { reason: "exec-event", coalesceMs: 0 }))` without filtering subagent session keys.
  - file: [src/agents/bash-tools.exec-runtime.ts:279](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/bash-tools.exec-runtime.ts#L279)

- **Scoped wake options preserve subagent session keys:** `scopedHeartbeatWakeOptions` adds `sessionKey` to heartbeat wake requests for any parsed agent session key, so subagent exec completions are explicitly routed into the heartbeat runner rather than dropped before scheduling.
  - file: [src/routing/session-key.ts:35](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/routing/session-key.ts#L35)
