## Summary

Keep this issue open. Current main still has the reported unguarded exec-event heartbeat wake path for subagent session keys, and #66749 remains the open same-author implementation PR with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review #66749 as the narrow fix path, or explicitly retarget to #69492 if maintainers want the broader event-audience design first. The implementation should skip requestHeartbeatNow for isSubagentSessionKey(sessionKey) in both exec completion paths while preserving enqueueSystemEvent, with regression coverage for both maybeNotifyOnExit and emitExecSystemEvent.
