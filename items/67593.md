## Summary

Keep this issue open. Current main still does not expose Kimi or Moonshot provider usage: the provider usage ID/list is closed over other providers, default status/usage surfaces fan out through that list, the bundled Kimi and Moonshot plugins do not implement usage auth or snapshot hooks, and docs list the supported usage providers without Kimi/Moonshot. The related PR #20343 was closed unmerged, so it is useful prior art but not a shipped fix.

## Best Possible Solution

Keep this issue open and implement the feature in the bundled provider boundary: add Kimi/Moonshot usage auth and snapshot fetching through `resolveUsageAuth` and `fetchUsageSnapshot`, extend the usage provider list or generalize usage-provider discovery so `kimi`/`kimi-coding`/`moonshot` are queryable by `/status`, Control UI, `openclaw status --usage`, and channels status, then add regression tests and update https://docs.openclaw.ai/concepts/usage-tracking. Treat `/quota` as a separate product decision because current main documents provider quota under `/status` and `openclaw status --usage`, not as a standalone chat command.
