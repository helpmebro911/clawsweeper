## Summary

Keep open. Current main has improvements for the mixed text/tool delivery path and reasoning tag stripping, but the central Gemini textSignature bloat is still present: the Google transport stores incoming Gemini thought signatures on text blocks as `textSignature`, and later replays same-provider `textSignature` back to Gemini as `thoughtSignature`. The generic replay sanitizer strips `thoughtSignature`/`thought_signature`, but not `textSignature`, so the reported session-history bloat path remains plausible on current main.

## Evidence
