## Summary

Keep open. Current main still does not include this PR's proposed Trivy/KICS/TruffleHog workflow template or staged-content TruffleHog pre-commit guide, and the existing security docs/CI use different tools rather than an equivalent user-copyable scanner template. The predecessor PR #8951 is closed unmerged because this PR replaced it, so it is not a canonical superseding item. The PR also has unresolved template correctness/security review concerns, leaving this as a maintainer docs/security scope decision rather than a conservative cleanup close.

## Best Possible Solution

Keep this PR open for maintainer docs/security judgment. If the template belongs in OpenClaw docs, fix the staged-path handling by streaming NUL-delimited paths directly, pin every GitHub Action reference to immutable SHAs, add docs navigation, and align the page with the existing detect-secrets, zizmor, dependency-audit, and openclaw security audit guidance. If maintainers decide generic scanner templates are out of scope for core docs, close manually with that product rationale.
