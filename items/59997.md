## Summary

Keep PR #59997 open. Current main still appears to classify only numeric `sessions.json.bak.<digits>` store backups, while the PR broadens this to all non-empty `sessions.json.bak.*` suffixes and adds targeted regression coverage for the reported real backup names.

## Evidence

- **Current main still uses numeric-only backup classification:** `isSessionArchiveArtifactName()` checks `LEGACY_STORE_BACKUP_RE = /^sessions\.json\.bak\.\d+$/`, so names like `sessions.json.bak.session-maint-*` are not covered by the dedicated store-backup path on current main.
  - file: [src/config/sessions/artifacts.ts:4](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/src/config/sessions/artifacts.ts#L4)
