## Summary

Keep open. Current `main` still uses a numeric-only matcher for `sessions.json.bak.*`, so the PR’s reported gap is not implemented on `main`, and the cleanup/quota path still depends on that matcher.

## Evidence

- **Current matcher is still numeric-only:** `isSessionArchiveArtifactName` still special-cases only `sessions.json.bak.<digits>` via `LEGACY_STORE_BACKUP_RE = /^sessions\.json\.bak\.\d+$/`, so named backups like `sessions.json.bak.safe-cleanup-*` would not be classified as archive artifacts on current `main`.

If named `sessions.json.bak.*` files are not classified, they still contribute to total bytes but are skipped as removable archive artifacts.
