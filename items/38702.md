## Summary

Current main still only has partial WebChat input history. ArrowUp/ArrowDown recall is limited to an empty draft and a local send-time stack, without session-message-derived entries, draft restoration, caret/selection/modifier/IME guards, or focused tests for the requested behavior. The linked PR #38709 remains the right implementation path, but it is open and unmerged in the provided GitHub context.

## Best Possible Solution

Keep #38702 open until PR #38709 or an equivalent focused change lands. The implementation should derive in-session recall entries from current user-role text messages, preserve and restore the active draft, enforce modifier/IME/selection/caret guards, reset safely around loading/reload boundaries, and add focused UI/controller tests for ArrowUp/ArrowDown behavior.
