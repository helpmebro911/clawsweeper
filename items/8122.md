## Summary

Keep #8122 open. Current main still scopes automatic session reset policy to global, chat type, and channel-level overrides; Telegram group/topic configuration does not accept `session.reset`, and the reset runtime does not consult Telegram group/topic config. The linked implementation PR #8134 was closed unmerged, and #20888 is a broader closed session lifecycle request rather than a canonical tracker for this per-Telegram-group feature.

## Best Possible Solution

Keep #8122 open. Implement an additive, account-aware reset override for Telegram group/topic config, or add a generic channel plugin reset resolver if maintainers want this across plugins. Update Telegram types, strict Zod schema, generated config metadata/help, public docs, and focused precedence tests for topic/group over channel over type/global, including multi-account cases.
