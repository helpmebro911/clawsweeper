## Summary

Keep #8122 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still supports automatic session reset only at global, chat-type, and channel override levels. Telegram group/topic config does not accept a `session.reset` block, the strict Telegram schema would reject the proposed config shape, and the runtime reset resolver only receives a channel-level override. Related PR #8134 was closed unmerged, and #20888 is a broader closed lifecycle request rather than a canonical tracker for this per-Telegram-group feature.

## Best Possible Solution

Keep #8122 open. Implement an additive, account-aware reset override for Telegram group/topic config, or introduce a generic channel plugin reset resolver if maintainers want conversation-scoped policies across plugins. Update Telegram config types, strict Zod schema, generated config labels/help, public docs, and focused precedence tests covering topic/group over channel over type/global, including multi-account Telegram cases.
