## Summary

Keep #8122 open. Current main still does not accept or apply per-Telegram-group session reset overrides, the related implementation PR #8134 was closed unmerged, and the broader #20888 lifecycle issue does not supersede this specific Telegram group configuration request.

## Best Possible Solution

Keep this issue open. The best path is an additive, account-aware implementation that lets Telegram group or topic config override reset policy, or a generic channel group reset resolver if maintainers want this beyond Telegram. Update Telegram config types, Zod schema, generated metadata/help, public docs, and focused tests covering precedence such as group/topic over channel over type/global, including multi-account behavior.

## Risks / Open Questions

- A naive implementation could reproduce the #8134 multi-account caveat by selecting a group reset policy without tying it to the inbound Telegram account context.
- The eventual fix should avoid hard-coding Telegram-specific behavior in core when a generic channel/group reset seam would better preserve the plugin boundary.
