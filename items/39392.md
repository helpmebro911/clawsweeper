## Summary

Current main at 6b6dcaf still does not implement the Control UI agent-switch model-select fix. The Agents Overview subtree is still reused across selected agents, the Primary model select still relies on direct `.value` binding without option-level selected state, and the related fix PRs remain open/unmerged rather than shipped.

## Best Possible Solution

Keep #39392 open until a focused Control UI fix lands on main. The likely repair is to isolate the Agents Overview model controls by selectedAgent.id, or explicitly bind select and option selected state to a per-agent selected value, and add a regression test that switches main -> another agent -> main while verifying the displayed Primary model matches the selected agent's resolved config. #54724 appears to be the broader current candidate, with #39401 and #52948 covering narrower parts of the same surface.
