## Summary

Keep #70803 open. Current main still has the exact WhatsApp target-normalization gap the PR fixes: runtime LID conversion accepts `@hosted.lid`, but the shared target normalizer only accepts `@lid`, and the current regression tests cover only `@lid`. The provided PR discussion and Greptile review describe a narrow, plausible fix with focused test coverage.

## Best Possible Solution

Keep the PR open for maintainer review/merge or a maintainer-equivalent patch. The likely correct implementation is the PR’s narrow change: extend `WHATSAPP_LID_RE` in `extensions/whatsapp/src/normalize-target.ts` to accept `hosted.lid` and keep the added lower/upper-case regression coverage in `extensions/whatsapp/src/resolve-target.test.ts`.
