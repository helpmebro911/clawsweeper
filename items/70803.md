## Summary

Keep open. Current main still rejects WhatsApp `@hosted.lid` targets in `normalizeWhatsAppTarget` / `isWhatsAppUserTarget`, while the runtime JID-to-E.164 path already accepts hosted LIDs. PR #70803 is a focused fix for a real drift between two WhatsApp target paths, not obsolete cleanup.

## Best Possible Solution

Keep PR #70803 open for maintainer review and likely landing as a narrow WhatsApp plugin bug fix: align `extensions/whatsapp/src/normalize-target.ts` with the existing hosted-LID runtime support and keep the added regression coverage in `extensions/whatsapp/src/resolve-target.test.ts`. Before merge, run the relevant changed lane or targeted WhatsApp extension tests in a dependency-complete worktree.
