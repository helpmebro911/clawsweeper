## Summary

Keep open. Current main already preserves Feishu streaming defaults and supports auto-mode card streaming, but it still only accepts boolean `channels.feishu.streaming`. The PR's remaining object-config work is not implemented on main and the stale PR head is not safe to merge as-is.

## Best Possible Solution

Keep this PR open for focused rework or maintainer review. The best path is a current-main rebase that implements Feishu object-form streaming config across `extensions/feishu/src/config-schema.ts`, `extensions/feishu/openclaw.plugin.json`, public Feishu docs, generated config metadata if applicable, and runtime wiring, while preserving `streaming !== false` default behavior. Add regression tests for omitted/default streaming, `streaming: false`, `{ enabled: false }`, custom `throttleMs`, and custom title behavior.

## Risks / Open Questions

- Directly merging the stale PR head could regress current default-enabled Feishu streaming for configs that omit `streaming`.

- Configurable card title and throttle touch Feishu Card Kit runtime behavior and should be bounded, documented, and tested before merge.
