## Summary

Keep this PR open. Current main implements Feishu auto-mode streaming and preserves boolean default/disabled behavior, but it still does not accept object-form `streaming` config or wire config-driven `throttleMs`/title. The PR is directionally useful but stale and unsafe to merge as-is because its head still regresses omitted/default streaming and does not update the public config metadata or docs.

## Best Possible Solution

Keep this PR open for focused rework or maintainer review. The best path is to rebase onto current main, add a typed Feishu streaming config resolver that accepts boolean and object forms while preserving `undefined` as default-enabled and `false`/`{ enabled: false }` as disabled, wire bounded `throttleMs` and title/header behavior without regressing identity-derived headers/notes, update plugin manifest/docs/generated metadata, and add regression tests for omitted/default streaming, disabled forms, object config, custom throttle/title, and auto-mode card/plain-text routing.
