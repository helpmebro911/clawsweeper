## Summary

Keep open. Current main still has only the binary `tools.fs.workspaceOnly` filesystem policy and does not implement `tools.fs.allowedRoots` or an equivalent multi-root read/write/edit/apply_patch boundary. This PR is the active implementation candidate after #33708 was closed, and the diff touches security-sensitive filesystem sandbox and apply_patch paths, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate for `tools.fs.allowedRoots`. Maintainers should review the final rebased diff, verify the filesystem sandbox and apply_patch security properties, run the relevant changed lanes, and either land a cleaned-up version or replace it with an equivalent core implementation.
