## Summary

Current main still preserves the reported ACP/media-understanding gap. ACP dispatch invokes media understanding for inbound media, but passes only ctx and cfg; provider-backed image understanding still requires agentDir, and activeModel is still needed for native-vision skip and text-only fallback routing. The prior ClawSweeper comment is still consistent with the current checkout.

## Best Possible Solution

Keep this issue open. Fix ACP dispatch by resolving the effective ACP agent directory from the canonical ACP session or resolved ACP agent, pass that agentDir into applyMediaUnderstanding, and pass activeModel when ACP runtime/session configuration can provide it. Add a focused dispatch-acp.test.ts regression for an inbound image ACP turn with configured tools.media.image, asserting media understanding receives the agentDir and enriches the prompt before runTurn. Treat the Windows OPENCLAW_AGENT_DIR doctor expectation as a separate follow-up if it still reproduces after the ACP fix.
