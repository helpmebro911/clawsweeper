## Summary

Keep open. Current main still has the ACP-specific gap reported here: ACP dispatch calls media understanding for inbound media but still omits `agentDir`, while provider-based image understanding requires `agentDir` and the normal reply path passes it. This makes the reported Windows workaround and MiniMax/text-only model failure still actionable in OpenClaw core.

## Best Possible Solution

Keep this issue open. Fix ACP dispatch by resolving the effective ACP agent directory from the canonical session/resolved ACP agent and passing it to `applyMediaUnderstanding`; also pass active model context when it can be derived from ACP runtime options or the configured agent model so native-vision skip behavior matches the normal reply path. Add a focused `dispatch-acp.test.ts` regression for an inbound image ACP turn with configured `tools.media.image`, asserting media understanding receives `agentDir` and enriches the prompt for a text-only ACP model before `runTurn`. Treat the Windows environment-variable/doctor note as a separate follow-up if it still reproduces after the ACP fix.
