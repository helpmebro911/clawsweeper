## Summary

Keep #71555 open. Current main still uses the pinned `@mariozechner/pi-coding-agent@0.70.2` session manager, the upstream `SessionManager.open()` path still fully parses the session file before constructing a manager that loads it again, and OpenClaw's embedded attempt path still calls that API for agent runs. Related large-session issues are adjacent, but none clearly supersedes this specific embedded context rebuild/double-load crash report.

## Best Possible Solution

Keep this issue open as the specific embedded-runner large-session crash report. The best fix is to avoid the double full-file parse in the embedded Pi session path, ideally by changing `pi-coding-agent` so `SessionManager.open()` can read only the header or accept preloaded entries, or by adding an OpenClaw-side adapter/streaming parser with regression coverage for a large JSONL containing long tool-result lines. Link it to the broader large-session tracking issues, but do not close it into them because this report identifies a distinct current code path.
