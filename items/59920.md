## Summary

Keep open. Current main still has the parser gap this PR is meant to fix: `parseCliJsonl()` only handles Claude `type:"result"` specially and otherwise collects generic `item.text`, with no `task_complete.last_agent_message`, `phase:"final_answer"`, or commentary-phase handling. The related merged PRs address adjacent visible-text surfaces, not this CLI JSONL parser path.

## Evidence

- **Current parser still uses generic item text fallback:** On current main, `parseCliJsonl()` initializes `texts`, checks Claude JSONL result handling, then pushes `parsed.item.text` for message-like items and returns the joined text. There is no terminal-field priority for `task_complete.last_agent_message` or phase-aware final-answer selection in this function.
