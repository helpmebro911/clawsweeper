## Summary

Keep open. Current `main` still has the Codex/OpenAI JSONL parser gap this PR targets, so it is not cleanup-closeable as already implemented or obsolete. The PR discussion also contains a credible regression concern: as written, the patch may make commentary-only successful runs fall back to raw JSONL, so this needs revision/review rather than closure.

## Evidence

- **Current main parser still ignores terminal reply fields:** `parseCliJsonl()` on `main` only special-cases Claude `result` events, then appends generic `parsed.item.text`. There is no handling for `task_complete.last_agent_message`, `phase:"final_answer"`, or commentary suppression in this code path.
