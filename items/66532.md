## Summary

Keep open. Current main still has the abort/wait/isActive/queue helpers internally, but the plugin runtime surface injected as api.runtime.agent only exposes runEmbeddedAgent and runEmbeddedPiAgent plus agent/session utilities. The adjacent Gateway chat.abort work is real and shipped on main, but it is explicitly a different surface from the CoClaw plugin-facing request in this issue.

## Best Possible Solution

Keep this issue open and implement the additive plugin runtime contract: re-export abortEmbeddedPiRun, waitForEmbeddedPiRunEnd, isEmbeddedPiRunActive, and queueEmbeddedPiMessage from src/plugins/runtime/runtime-embedded-pi.runtime.ts; lazy-bind them in createRuntimeAgent(); declare them on PluginRuntimeCore["agent"]; and update the runtime shape test plus plugin SDK runtime docs/API inventory. Treat the Gateway chat.abort fix as related shipped work, not the canonical resolution for this plugin-facing request.
