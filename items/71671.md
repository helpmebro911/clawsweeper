## Summary

Keep PR #71671 open. Current main still has the reported stale-history behavior: `getNoProgressStreak` skips over non-matching or missing-result records with `continue`, so older identical no-progress results can still count after another tool call. The PR is a focused bug fix with relevant regression coverage, and I did not find an implemented main fix or a canonical duplicate that supersedes this narrower tail-semantics bug.

## Best Possible Solution

Keep PR #71671 open and review or land this focused fix, or apply an equivalent patch on main: make `getNoProgressStreak` count only the contiguous current tail by breaking on the first non-matching or missing-result record, and keep the added regression tests for both known polling and the global no-progress circuit breaker. Treat broader idle-history cleanup such as #40217 as related but separate work.
