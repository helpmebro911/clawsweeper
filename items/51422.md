## Summary

Keep open. Current `main` does not implement Feishu message recall/delete support: the Feishu plugin does not advertise `delete`/`unsend`, its dispatcher has no recall/delete branch, and its runtime surface exports no recall/delete helper.

## Evidence

- **Feishu action discovery omits delete/unsend:** The Feishu message-tool discovery list on current `main` includes `send`, `read`, `edit`, `thread-reply`, `pin`, `list-pins`, `unpin`, `member-info`, `channel-info`, and `channel-list`, but not `delete`, `unsend`, or any recall action.
  - file: [extensions/feishu/src/channel.ts:204](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/feishu/src/channel.ts#L204)
  - command: `nl -ba extensions/feishu/src/channel.ts | sed -n '190,235p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **User-facing Feishu hints still exclude recall/delete:** The plugin prompt text says Feishu supports `send`, `read`, `edit`, `thread-reply`, pins, channel/member lookup, and reactions when enabled. It does not mention message recall or delete support.
  - file: [extensions/feishu/src/channel.ts:586](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/feishu/src/channel.ts#L586)
  - command: `nl -ba extensions/feishu/src/channel.ts | sed -n '575,595p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Dispatcher has no Feishu recall/delete path:** The Feishu action handler implements `read`, `edit`, `pin`, `unpin`, `list-pins`, `channel-info`, `member-info`, `channel-list`, `react`, and `reactions`, then falls through to `Unsupported Feishu action`, with no branch for `delete`, `unsend`, or recall.
  - file: [extensions/feishu/src/channel.ts:1089](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/feishu/src/channel.ts#L1089)
  - command: `nl -ba extensions/feishu/src/channel.ts | sed -n '756,866p'; nl -ba extensions/feishu/src/channel.ts | sed -n '1070,1110p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Runtime exports no message recall/delete helper:** The Feishu runtime surface exports send/get/edit, pin, reaction, chat, and directory helpers only; there is no exported message recall/delete operation to back the requested feature.
  - file: [extensions/feishu/src/channel.runtime.ts:29](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/feishu/src/channel.runtime.ts#L29)
  - command: `nl -ba extensions/feishu/src/channel.runtime.ts | sed -n '1,80p'`

- **Core contract can support this, but Feishu does not implement it:** The shared channel action vocabulary already includes generic `unsend` and `delete` action names, so this is not blocked by a missing core action contract; it remains a Feishu-plugin implementation gap on `main`.
