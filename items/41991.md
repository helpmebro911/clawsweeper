## Summary

Keep open. The PR is a focused diagnostic improvement and the underlying behavior is still present on current main: Gemini CLI credential extraction failures still collapse to null and `resolveOAuthClientConfig()` still throws the generic `Gemini CLI not found` message. The old `extensions/google-gemini-cli-auth` path has since been folded into the bundled `extensions/google` plugin, so the useful work needs to be rebased/retargeted rather than closed as implemented.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up, but require it to be rebased onto the current bundled Google plugin. The best patch would add diagnostic state or a small result type in `extensions/google/oauth.credentials.ts`, reset it in `clearCredentialsCache()`, append details from `resolveOAuthClientConfig()`, and add focused tests for missing binary, missing `oauth2.js` including recursive-search wording, parse failure, and unexpected extraction exceptions while preserving the current install-layout coverage.
