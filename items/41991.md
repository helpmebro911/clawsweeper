## Summary

Keep open. Current main still collapses Gemini CLI OAuth credential extraction failures into a generic `Gemini CLI not found` error, while this PR proposes useful diagnostic detail. The PR is stale because the old `extensions/google-gemini-cli-auth` plugin path has been consolidated into the bundled `extensions/google` plugin, and the related security discussion in #54289 should inform whether this diagnostic is ported or the extraction path is removed/gated instead.

## Best Possible Solution

Keep this PR open as a focused implementation candidate, but ask the contributor to rebase/port the change to `extensions/google/oauth.credentials.ts` and add regression tests around `resolveOAuthClientConfig()` for missing binary, missing `oauth2.js`, parse failure, and unexpected extraction exceptions. Coordinate the final decision with #54289: if automatic Gemini CLI credential extraction is removed or gated there, fold the diagnostic behavior into that policy change or close this PR as superseded after the canonical security work lands.
