## Summary

Current `main` does not implement the requested Feishu fallback identity resolution. The inbound mention payload type already exposes `open_id`, `user_id`, and `union_id`, but the actual mention gate still matches only the bot's own `open_id`, and group messages are dropped when that exact match fails. That aligns with the reported app-scoped `open_id` mismatch, so there is not strong evidence to close this issue as fixed or obsolete.
