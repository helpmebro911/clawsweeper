## Summary

Keep open. Current `main` still awaits the pre-turn memory flush, still lacks the linked suppression fix for memory-flush-triggered compaction side effects, and the dedicated fix PR (#58975) remains open and unmerged.

## Evidence

- **Active turn still waits behind memory flush on main:** `runReplyAgent()` awaits `runMemoryFlushIfNeeded()` before it builds the follow-up turn runner, and `runMemoryFlushIfNeeded()` performs a real embedded agent run for the flush. That means the user turn is still sequenced behind the pre-compaction flush path.
