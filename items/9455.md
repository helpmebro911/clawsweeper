## Summary

Keep #9455 open. Merged #9510 added the Cron Jobs filter bar and some filtering, but current main still misses two explicit acceptance paths: Jobs search does not match job IDs, and Jobs last-run-status filtering cannot select absent/unknown status.

## Best Possible Solution

Keep #9455 open and narrow the remaining work on top of #9510: add job.id to the cron.list/Jobs search haystack, add an "unknown" Jobs last-status filter value and option, normalize absent or lastRunStatus-only status into that bucket, and add focused tests for job-id search plus unknown-status filtering.
