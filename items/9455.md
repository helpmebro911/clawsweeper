## Summary

Keep #9455 open. Current main already has a Cron Jobs filter bar with search, enabled, schedule-kind, last-run-status, sorting, reset controls, and tests, but it still does not satisfy two explicit requested behaviors: search does not match job id, and the Jobs last-run status filter has no explicit unknown option. The related #16799/#16831 context is about CLI/status readability for cron-heavy setups, not a canonical duplicate for this dashboard filter request.

## Best Possible Solution

Keep the issue open but narrow the remaining work: include job.id in cron.list search matching, add an explicit unknown last-run-status filter that matches jobs with no last status, and update the UI/controller/protocol tests around those two behaviors. The related status UX issue/PR should remain separate because it targets CLI session/status readability, not this dashboard filter bar.
