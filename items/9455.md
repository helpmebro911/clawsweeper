## Summary

Keep #9455 open. Current main includes the Cron Jobs filter bar and partial filtering from merged #9510, but two explicit requested behaviors are still missing: Jobs search does not match job IDs, and the Jobs last-run-status filter still cannot select jobs with absent/unknown status.

## Best Possible Solution

Keep #9455 open, but narrow it to the remaining delta on top of merged #9510: include job.id in the cron.list search haystack or equivalent client-side Jobs search path, add an unknown Jobs last-status filter value/option, normalize absent lastStatus/lastRunStatus into that unknown bucket, and add focused tests for job-id search plus unknown-status filtering.
