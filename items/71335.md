## Summary

Keep #71335 open. Current main has not implemented the requested gateway-specific default: `memorySearch.sync.watch` still falls back to `true`, and both builtin and QMD memory managers create chokidar watchers whenever that resolved setting is true. A recent watcher repair narrowed/changed watched paths, but it did not add a gateway-mode default or preserve the requested CLI-vs-gateway distinction.

## Best Possible Solution

Keep the issue open and implement a runtime-aware default: pass an explicit execution context into memory sync resolution or gateway memory startup/tool paths, default `sync.watch` to false only for headless gateway execution, keep explicit per-agent/default `sync.watch: true` winning, and add tests proving CLI/default behavior stays true while gateway mode without an override resolves false for builtin and QMD watchers.

## Risks / Open Questions

- A naive `cfg.gateway?.mode` check could change CLI/desktop memory behavior for users whose config already contains `gateway.mode: "local"`.
- Both builtin and QMD memory backends consume the resolved sync setting, so a fix needs coverage for both watcher paths and explicit `sync.watch: true` overrides.
