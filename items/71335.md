## Summary

Keep this issue open. Current main at 646a268d2710bc54e478349ef5690f3c8803df4f still resolves unset memory search `sync.watch` to `true` without any gateway/headless runtime input, and the gateway startup path passes the raw config through to memory managers. Both builtin and QMD memory managers still create chokidar watchers when the resolved value is true. The related #64549 was a closed auto-indexing bug in the same subsystem, but it does not supersede this gateway default-policy request.

## Best Possible Solution

Keep this issue open and implement a narrow runtime-aware memory sync default. Thread an explicit gateway/headless execution context into memory sync resolution or manager creation, default `sync.watch` to false only for that context, keep explicit user overrides authoritative, and add resolver plus builtin/QMD manager coverage for gateway and non-gateway behavior. Update schema help and the public memory docs if the default changes.
