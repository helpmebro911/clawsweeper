## Summary

Keep this issue open. Current main still defaults unset `memorySearch.sync.watch` to `true` with no gateway/headless runtime context, and both builtin and QMD memory managers still create chokidar watchers when that resolved setting is true. Recent gateway/memory work reduces some startup/runtime fanout, but it does not implement the requested gateway-mode default policy.

## Best Possible Solution

Implement a narrow runtime-aware memory sync default: pass gateway/headless execution context into memory sync resolution or manager creation, default `sync.watch` to false only for that context, keep explicit `sync.watch: true` and `sync.watch: false` authoritative, and add resolver plus builtin/QMD manager tests for gateway and non-gateway behavior. Update schema/help docs if the default changes.
