## Summary

Keep #71335 open. Current main still defaults `memorySearch.sync.watch` to `true` when neither agent defaults nor per-agent overrides set it, and that resolver is used by the memory-core managers that create chokidar watchers. The request is a concrete core/bundled-plugin default change, not stale, duplicate, or ClawHub-only.

## Best Possible Solution

Keep the issue open and implement a gateway-runtime-aware default for memory search sync watching. The fix should preserve explicit per-agent/default overrides, avoid changing standalone CLI/desktop resolution unintentionally, update schema help/docs if behavior changes, and add tests around gateway runtime defaulting plus watcher creation in memory-core.

## Risks / Open Questions

- A naive `cfg.gateway.mode` check inside the shared resolver could also affect CLI memory commands on machines whose config contains `gateway.mode`; preserving CLI/desktop behavior likely needs explicit runtime context or a gateway-only resolution path.
- Changing the default needs focused regression coverage for unset defaults, explicit `sync.watch: true`, explicit `sync.watch: false`, and gateway versus non-gateway runtime behavior.
