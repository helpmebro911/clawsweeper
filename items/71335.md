## Summary

Keep #71335 open. Current main still defaults `agents.defaults.memorySearch.sync.watch` to `true` with no gateway-mode branch, and both builtin and QMD memory managers still create chokidar watchers when that resolved flag is true. The related #64549 report is same subsystem but tracks auto-index correctness, not this gateway default policy.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the memory config resolution/startup path: add a context-aware default so gateway server mode resolves `memorySearch.sync.watch` to false only when neither agent nor defaults explicitly set it, while CLI/desktop behavior remains true by default. Add regression coverage for gateway-mode default false, explicit `sync.watch: true` winning, and non-gateway/default CLI behavior staying unchanged; cover both builtin and QMD manager paths if the default is applied below the shared resolver.
