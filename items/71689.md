## Summary

Keep open. Current main still logs task-registry SQLite restore failures and continues without a corruption recovery path, bad-database snapshot/rebuild, or task-registry audit/operator finding. The related items in the provided context are title-overlap only and do not supersede malformed `tasks/runs.sqlite` recovery.

## Best Possible Solution

Keep this issue open. Implement task-registry-owned SQLite corruption recovery in `src/tasks/task-registry.store.sqlite.ts` and the restore/audit surface in `src/tasks/task-registry.ts` / `src/tasks/task-registry.audit.ts`: detect SQLite malformed/corrupt errors during open, PRAGMA/schema setup, and load; close the handle; snapshot `runs.sqlite` plus sidecars; attempt safe recovery/rebuild or fall back to an explicit empty registry only with an operator-facing audit/event; and add regression tests for malformed database restore.
