## Summary

Keep open. Current main still does not expose a typed Gemini safety-settings config surface, the strict provider schema would reject a provider-level `safetySettings` key, and the direct Google Generative AI request builder does not add `safetySettings` to Gemini payloads. The related PR #49529 is relevant, but the provided GitHub context shows it remains open and unmerged, so the user-facing capability is still missing.

## Best Possible Solution

Keep #12008 open until PR #49529 or an equivalent implementation lands. The likely maintainer path is to add a typed, schema-validated `models.providers.*.safetySettings` config surface, document it, map it only on the direct `google-generative-ai` request path, preserve explicit request-level safety settings if such a path is supported, and add config-validation plus payload tests for both the bundled `google` provider and custom provider keys using the Google adapter.
