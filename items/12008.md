## Summary

Keep open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still lacks a schema-validated Gemini safety-settings config surface and does not map `safetySettings` into direct `google-generative-ai` payloads. The provided GitHub context identifies PR #49529 as the relevant implementation path, but it is still open/unmerged there, and the latest maintainer review comment already kept #12008 open.

## Best Possible Solution

Keep #12008 open until PR #49529 or an equivalent implementation lands. The maintainer-ready path is a typed and schema-validated `models.providers.*.safetySettings` config field, generated schema/docs updates, runtime mapping into direct `google-generative-ai` payloads while preserving explicit request-level settings, and regression tests for both bundled Google and custom provider keys using the Google adapter.
