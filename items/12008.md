## Summary

Keep open. Current main still does not expose a schema-validated Gemini safety-settings config field and the direct Google Generative AI payload builder still does not emit `safetySettings`. The related implementation PR #49529 is still open and unmerged, so the feature remains real work rather than a closeable duplicate or implemented request.

## Best Possible Solution

Keep #12008 open until PR #49529 or an equivalent implementation lands. The maintainable path is a typed and schema-validated `models.providers.*.safetySettings` config field, generated schema/help/docs updates, runtime mapping into direct `google-generative-ai` payloads for both bundled Google and custom Google-compatible provider keys, and regression tests that preserve explicit request-level `safetySettings` precedence.
