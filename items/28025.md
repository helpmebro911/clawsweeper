## Summary

Keep #28025 open. Current main still has no `channels.slack.thread.replyOnToolUse` or equivalent Slack policy; Slack reply routing is configured by `replyToMode`, existing thread state, and delivery/streaming plans that are chosen before tool callbacks can affect the thread target.

## Best Possible Solution

Keep #28025 open. If accepted, implement a default-false typed Slack config option in the bundled Slack plugin/schema/config metadata/docs, carry a reliable per-turn tool-use signal into Slack reply planning before final delivery or streaming commits to a target, preserve explicit thread replies plus `thread.inheritParent` and `replyToMode` semantics, and add Slack tests for DMs, channels, existing threads, reply-mode interactions, streaming, and tool/progress callbacks.
