## Summary

Current main still does not implement the requested Slack tool-use-gated reply routing. Slack threading is configured and planned through `replyToMode`, chat type, and existing thread context; tool callbacks only drive status/progress, and docs/tests still describe reply-mode threading rather than `channels.slack.thread.replyOnToolUse` or an equivalent policy.

## Best Possible Solution

Keep #28025 open for maintainer product judgment. If accepted, implement a default-false Slack config such as `channels.slack.thread.replyOnToolUse`, wire it through Slack types/schema/generated metadata/docs, carry a per-turn tool-use flag into reply planning without breaking existing explicit threads, `thread.inheritParent`, `replyToMode`, message-tool sends, typing indicators, or streaming, and add focused Slack tests for DMs, channels, existing threads, default-off behavior, and tool/progress callbacks.
