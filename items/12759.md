## Summary

Keep #12759 open. Current main still has only per-channel labeler rules, no `channel: ALL`/`channel: NEW` catch-all label, and no repo-controlled workflow logic that collapses many channel labels into one. The related implementation PR #71548 was closed unmerged, so it does not supersede or ship the request.

## Best Possible Solution

Keep the issue open until repo-controlled labeling implements the catch-all behavior. The likely path is to add the maintainer-approved label and metadata, update `.github/labeler.yml` and/or `.github/workflows/labeler.yml` so broad channel changes receive one catch-all label instead of many individual `channel:*` labels, sync the GitHub label, and verify with a representative multi-channel PR.
