## Summary

Keep #12759 open. Current main still lacks a repo-controlled `channel: ALL` or equivalent catch-all label and collapse behavior; labeler and sync code still handle individual channel labels plus unrelated size/maintainer/beta/PR-limit labels. Related PR #71548 attempted this work but was closed unmerged, so it does not supersede or ship the request.

## Best Possible Solution

Keep the issue open until repo-controlled labeling implements the maintainer-approved catch-all behavior: add the label/rule and metadata, update `.github/labeler.yml`, `scripts/sync-labels.ts`, or workflow/agent logic as needed so broad channel changes get one catch-all label instead of many individual channel labels, then verify with a PR touching multiple channel surfaces.
