## Summary

Keep this PR open, but it needs a rework before merge. Current main moved thinking support to provider-owned profiles, and OpenAI/OpenAI Codex/GitHub Copilot already expose xhigh there. The Vercel AI Gateway proxy case described by this PR still appears missing: the plugin catalogs upstream OpenAI refs like `openai/gpt-5.4`, but it does not declare a thinking profile hook, while `sessions.patch` validates xhigh against the effective provider/model. The PR is also stale against main and contains unrelated/broken web-search registry churn, so the useful behavior should be salvaged in the current provider-profile architecture rather than merged as-is.

## Best Possible Solution

Keep this PR open for maintainer/contributor rework. The best path is to rebase onto current main, drop unrelated web-search and test-seam churn, and implement the proxy xhigh behavior through provider-owned thinking profiles, likely in the Vercel AI Gateway provider or a narrow shared provider helper that validates trusted upstream refs. Add focused `thinking` and `sessions.patch` regression tests for accepted `vercel-ai-gateway` OpenAI refs and rejected unsupported refs.
