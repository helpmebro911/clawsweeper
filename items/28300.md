## Summary

Keep open. Current main has overlapping Control UI theming through Claw/Knot/Dash, light/dark/system mode, roundness controls, and one browser-local tweakcn-backed Custom import slot, but it still does not implement the requested six curated preset themes or an in-app Custom Theme Studio with hue generation, gradient controls, grouped color editing, and per-variable override/reset UI.

## Best Possible Solution

Keep this open for Control UI product/design handling. A good implementation would deliberately extend the existing Appearance theme model: decide whether to add accepted built-in preset families, a local generator/editor, or a narrower extension of tweakcn imports; then align `ThemeName`/`resolveTheme`, CSS token generation, browser-local persistence, quick settings, i18n, docs, and focused tests.
