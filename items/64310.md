## Summary

Keep this PR open. Current main still has the mutable module-global workspacePackagePaths array, collectWorkspacePackagePaths still appends to it on every main() call, and buildMissingPackages still consumes the accumulated module state. The provided PR is a focused script-only fix and has not been implemented or superseded on main.

## Best Possible Solution

Review and land a focused version of this PR rather than closing it. The maintainer path is to make collectWorkspacePackagePaths side-effect-free, pass the fresh deterministic package-path list into buildMissingPackages, address the redundant-sort review note, and add a small regression test in test/scripts/audit-seams.test.ts that calls main twice in-process and verifies missingPackages does not duplicate entries.
