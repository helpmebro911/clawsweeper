## Summary

Keep this PR open. Current main still uses a mutable module-global workspacePackagePaths array, collectWorkspacePackagePaths still appends to it, and buildMissingPackages still consumes the accumulated array. The PR is a focused, plausible fix that has not landed or been superseded; it needs maintainer review, the small redundant-sort cleanup, and regression coverage rather than closure.

## Best Possible Solution

Review and land a focused version of this PR. The maintainer path is to make collectWorkspacePackagePaths side-effect-free, pass the fresh deterministic package-path list into buildMissingPackages, address the redundant sort review note, and add a small regression test in test/scripts/audit-seams.test.ts that calls main twice in-process and verifies missingPackages does not duplicate entries.
