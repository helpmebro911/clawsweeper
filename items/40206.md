## Summary

Keep open. Current checked-out `main` still builds isolated cron run session keys by appending `:run:<uuid>`, and the Moonshot provider explicitly disables the OpenAI-compatible tool-call ID sanitization path. Main does contain a generic 40-character sanitizer, but I did not find evidence that this fix is applied to Moonshot cron-isolated runs, so there is not strong closure evidence.

## Evidence
