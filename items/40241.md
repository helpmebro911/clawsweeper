## Summary

Keep open. Current `main` does not implement Microsoft Clarity, and the PR still has unresolved instrumentation bugs: it would record `gateway_connected` before the real gateway handshake and `gateway_disconnected` on component teardown instead of actual gateway close events.

## Evidence

- **Not implemented on main:** `ui/index.html` on current `main` has only the local theme bootstrap in `<head>` and no Clarity loader snippet. `ui/src/ui/analytics.ts` does not exist in the checkout.
  - file: [ui/index.html:11](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/ui/index.html#L11)
  - command: `nl -ba ui/index.html | sed -n '1,30p'; test -f ui/src/ui/analytics.ts; echo $?`
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **Real success point is gateway hello:** The control UI marks the gateway as connected inside `onHello`, after the handshake succeeds. Emitting `gateway_connected` from `handleConnected()` would fire before this success path and over-count failed/offline/auth-error attempts.
  - file: [ui/src/ui/app-gateway.ts:287](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/ui/src/ui/app-gateway.ts#L287)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '275,337p'`
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **Real disconnect point is gateway close:** The control UI marks the gateway disconnected inside `onClose`. Emitting `gateway_disconnected` from `handleDisconnected()` would instead track Lit component teardown, missing live connection-loss events and counting page unmounts as disconnects.
  - file: [ui/src/ui/app-gateway.ts:338](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/ui/src/ui/app-gateway.ts#L338)
  - command: `nl -ba ui/src/ui/app-gateway.ts | sed -n '338,350p'`
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **PR wires analytics at the wrong lifecycle layer:** Current `handleConnected()` calls `connectGateway()` from a `bootstrapReady.finally(...)` block, and `handleDisconnected()` is the component unmount path. The PR proposal adds analytics at exactly these two lifecycle hooks, which do not correspond to successful handshake and actual socket close.
  - file: [ui/src/ui/app-lifecycle.ts:52](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/ui/src/ui/app-lifecycle.ts#L52)
  - command: `nl -ba ui/src/ui/app-lifecycle.ts | sed -n '52,80p'`
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **Existing review feedback is still unresolved:** The supplied PR context already flags the same two functional problems and also notes unused `setTag` imports in `ui/src/ui/app-lifecycle.ts` and `ui/src/ui/app-settings.ts` on PR commit `c543c70705fa616521ec938fb42296f94866b7fe`.
  - sha: [c543c70705fa](https://github.com/openclaw/openclaw/commit/c543c70705fa616521ec938fb42296f94866b7fe)
