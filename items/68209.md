## Summary

Keep open. Current main has adjacent Codex route, harness-pinning, auth-profile, and cumulative-usage hardening, but it does not prove the exact route-switch regression is fixed. Native compaction still lacks a post-compaction token count, and status rendering can still use stored totalTokens without checking totalTokensFresh.

## Best Possible Solution

Keep this issue open as a focused child of #66251. The best maintainer path is to reproduce the current-main route switch with a clean session, capture /status, agentHarnessId, totalTokensFresh, app-server auth/runtime events, Codex login status before and after the switch, and then either close with proof or split fixes for stale context reporting, native compaction freshness, thread/runtime isolation, and auth-mode isolation.
