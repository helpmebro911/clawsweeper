## Summary

Keep open. Current main includes the narrower #70678 WhatsApp quiet-session watchdog fix, but the broader reported regression still matches current code paths: reconnect gaps can still leave no active listener, health-monitor restarts can still stack on a reconnecting/disconnected WhatsApp account, and group hydration failure is still logged and continued without retry or recovery coverage.

## Best Possible Solution

Keep this issue open as a real WhatsApp plugin regression. Preserve the #70678 transport-activity watchdog fix, then add focused fixes and tests for retryable 408/428 reconnect churn, health-monitor coordination while WhatsApp is already reconnecting, outbound delivery behavior during short reconnect gaps, and recovery from `groupFetchAllParticipating` 1006 plus Baileys/libsignal Bad MAC failures so group events and sends do not silently disappear.
