## Summary

Keep open. Current main has the narrow quiet-session watchdog mitigation for #70678, but the broader reported WhatsApp regression still has unfixed code paths: outbound sends still fail immediately without an active listener, retryable reconnects still clear the active listener during backoff, the health monitor can still restart disconnected/reconnecting channels, group hydration failure is still one-shot, and the recent maintainer PR context explicitly says related reports including this issue should remain open unless their exact symptoms are proven fixed.

## Best Possible Solution

Keep this issue open as a real WhatsApp plugin regression. Preserve the shipped #70678 transport-activity watchdog mitigation, review #72621 as a narrow related auth-conflict/listener-state candidate, and add focused WhatsApp fixes/tests for retryable 408/428 reconnect churn, health-monitor coordination while reconnecting, outbound delivery behavior during short listener gaps, retry/recovery after `groupFetchAllParticipating` 1006, and Baileys/libsignal Bad MAC session recovery before considering this report fixed.
