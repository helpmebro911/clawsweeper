## Summary

Keep #36617 open. Current main has Japanese localization for the browser Control UI and the docs publish pipeline, but the requested core UX scope still remains English-first across CLI help, onboarding prompts, and workspace templates; there is no shared core/key-based i18n layer outside the UI bundle.

## Best Possible Solution

Keep #36617 open until maintainers either narrow it to the already-shipped Control UI/docs language support or implement a shared core i18n path for CLI help, onboarding/wizard prompts, common labels, and workspace templates. The concrete implementation path is locale resolution outside the browser UI, English fallback, ja-JP resource files for the agreed core surfaces, and focused tests for fallback, help rendering, wizard prompts, and template generation.
