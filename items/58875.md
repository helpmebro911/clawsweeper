## Summary

Do not close. Current main still formats the footer input label from `usage.input` directly, while cached tokens are shown only in a separate cache suffix. That does not implement the linked PR comment's stated fix for cached-token-heavy runs where `7 + 6993 cache` should show as about `7.0k in`.

## Evidence

- **Current footer uses raw input for main input label:** `formatResponseUsageLine` assigns `const input = usage.input` and then formats `inputLabel` from that value, without adding `cacheRead` or `cacheWrite` into the main `in` count.
  - file: [src/auto-reply/reply/agent-runner-usage-line.ts:52](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/src/auto-reply/reply/agent-runner-usage-line.ts#L52)
  - command: `web/open GitHub blob for current main; local exec unavailable due sandbox bwrap failure`
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Cache is appended separately:** The same formatter appends cache details as `cache <cached> cached / <new> new`, so a cached-heavy response can still show a small `in` value rather than the prompt total requested by the issue.
