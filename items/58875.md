## Summary

Keep open. Current `main` still builds the per-response usage footer from uncached `usage.input`, so cached-prompt runs can still render misleading input counts like `7 in` while only surfacing the cached portion in the separate cache suffix. The linked issue discussion describes summing `input + cacheRead + cacheWrite`, but that behavior is not present in the checked-out code.

## Evidence

- **Footer still formats raw input only:** `formatResponseUsageLine` reads `usage.input` directly for the `... in / ... out` footer label and returns that value unchanged apart from `formatTokenCount`; it does not fold in cached prompt tokens.
