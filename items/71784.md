## Summary

Keep #71784 open. Current main still lacks a bounded retry around live builtin memory-search query embeddings. Remote OpenAI-compatible and Gemini query paths still perform a single guarded HTTP POST, while the existing retry loop is applied to batch/probe paths and its classifier still excludes the reported socket/TLS transport failures. Related PR #44167 is open and batch/reindex-scoped, so it does not supersede this live-query bug.

## Best Possible Solution

Keep #71784 open. A conservative fix should add a narrow transient transport-error classifier and bounded retry/backoff for live memory query embeddings, ideally near `embedQueryWithTimeout()` or the shared single-embed remote HTTP helper, while preserving immediate failures for auth/config/status errors. Add focused regression tests for live query retries in `extensions/memory-core` and the shared remote fetch helper; then live-validate whether dispatcher keep-alive tuning is also needed.
