## Summary

Keep #71784 open. Current main still has no bounded retry for live builtin memory-search query embeddings, and the remote OpenAI-compatible/Gemini query paths still perform a single guarded HTTP request. The related batch/reindex reports do not supersede this per-query live-search bug.

## Best Possible Solution

Keep #71784 open. A conservative fix should add a narrow transient transport-error classifier for memory embedding HTTP calls and apply bounded retry/backoff to the live query path (`embedQueryWithTimeout()` or the shared single-embed remote HTTP helper), covering `fetch failed`, `other side closed`, `ECONNRESET`, `EPIPE`, `UND_ERR_SOCKET`, and TLS socket disconnects while preserving immediate failures for auth/config errors. Add regression tests in `extensions/memory-core` and `src/memory-host-sdk`, then live-validate whether dispatcher keep-alive tuning is also needed.
