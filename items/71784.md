## Summary

Keep #71784 open. Current main still lacks a bounded retry for live builtin memory-search query embeddings, and the related open PR/report in the provided context targets batch reindex resilience rather than the per-query `openclaw memory search` failure described here.

## Best Possible Solution

Keep #71784 open. A conservative fix should add a narrow transient transport-error classifier and bounded retry/backoff around live query embedding, either in `embedQueryWithTimeout()` or the shared single-embed remote HTTP helper, while preserving immediate failures for auth/config/provider errors. Add regression tests for the live query path separately from batch reindex, then live-validate whether dispatcher tuning is also required.
