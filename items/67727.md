## Summary

Keep this PR open. Current main still supports only uniform/per-channel inbound debounce, and the PR remains a plausible but incomplete implementation path because production channel call sites do not yet pass bot-origin state into debounce resolution. The PR discussion explicitly acknowledges that call-site wiring was deferred, so this is not obsolete or safely closeable as implemented.

## Best Possible Solution

Keep this PR open and either scope it explicitly as a preparatory config/API change or complete the product path before merge: add the config type/schema/generated metadata/tests, then thread reliable bot-origin state through the shared debounce resolver/helper and the relevant channel/plugin handlers. The existing Greptile baseline/schema artifact comments appear addressed by later PR commits, but the production senderIsBot wiring remains the material blocker.
