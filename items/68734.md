## Summary

Keep #68734 open. Current main does not implement dynamic prompt-hook narrowing for tools or skills: the public before_prompt_build event/result types still only cover prompt/messages and prompt text mutations, the hook merger and prompt-build helper do not propagate toolsAllow or skillsAllow, and the runner only applies the existing static params.toolsAllow path before hook execution. The PR discussion also contains useful real-world MCP-heavy validation and notes that #51258 is complementary rather than a superseding fix.

## Best Possible Solution

Keep #68734 open for maintainer review. The best path is an additive plugin contract that exposes the complete post-policy tool and skill catalogs to before_prompt_build, lets plugins narrow but never expand the model-visible surface, updates the active session/tool prompt state before the model call, preserves existing prompt mutations/provider transforms, includes client-hosted tools, and adds focused helper/seam tests plus plugin docs.
