## Summary

Keep #48325 open for maintainer judgment. The PR is not implemented on current main, and it is not a clean ClawHub close because the full proposal depends on selected structured app-log/diagnostic access that current public plugin APIs do not expose to non-bundled plugins. It also overlaps with the existing privileged diagnostics-otel exporter, so the right path is a product/security/API decision rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open. Maintainers should decide whether compact gateway event review should be folded into the existing bundled `diagnostics-otel` path, exposed through a narrow public diagnostics/log subscription API so an external ClawHub plugin can implement it, or explicitly accepted as a new bundled privileged telemetry plugin after security/API review and rebase.

- The PR likely needs a current-main rebase/API update before technical review because the provided diff references log transport behavior that is not exposed as a public root SDK API on current main.
