## Summary

Current main still implements the chat header and mobile chat controls as one session select grouped by agent, followed by Model and Thinking controls. The session-bound Model/Thinking part of the request is already true, but the requested separate Agent and Agent Session controls are still missing. The related open agent-switcher report #68959 is complementary rather than a complete superseding tracker for this split-picker behavior.

## Best Possible Solution

Keep this issue open. Implement the split in the Control UI chat surfaces: add an Agent selector backed by agentsList, add an Agent Session selector filtered from sessionsResult with parseAgentSessionKey(...).agentId, switch agents to that agent's most recent session or buildAgentMainSessionKey({ agentId }) as fallback, and keep Model/Thinking sessions.patch behavior targeting state.sessionKey. Mirror the desktop header change in the mobile chat controls and add focused tests for agent filtering, agent-switch fallback, and preserving session-bound Model/Thinking overrides. Coordinate with #68959, but do not close this issue unless a canonical item explicitly tracks the complete split-picker behavior.
