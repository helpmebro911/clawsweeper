## Summary

Keep PR #71573 open. Current `origin/main` at `cf834e2a` still lacks the proposed `update.run` restart continuation: the server method writes an update sentinel without `continuation`, the update tests still assert `continuation` is undefined, and the agent gateway tool advertises `continuationMessage` but drops it for the `update.run` action. The linked issue #71178 remains an open, detailed reproduction, and the PR is a focused, recent attempt to fix that surface rather than obsolete cleanup.

## Best Possible Solution

Continue review of PR #71573 or supersede it with an equivalent maintainer patch. The best fix is to add `continuationMessage` to the `update.run` protocol/schema and generated clients, parse and forward it from the agent gateway tool, build a restart success continuation only when `runGatewayUpdate` succeeds and a session context exists, and cover the default, explicit-message, no-session, and failed-update cases in tests. Before landing, resolve the security review concerns around session authorization and message size bounds.
