## Summary

Keep PR #71573 open. Current main still lacks the `update.run` restart-continuation producer path, while the PR is a focused, current fix for the linked #71178 subset. It is not implemented on main, not obsolete, and not superseded by another shipped change.

## Best Possible Solution

Keep #71573 open as the active code fix for the `update.run` continuation gap. The best path is to finish maintainer/security review, address or explicitly justify the `sessionKey` authorization and `continuationMessage` size-bound concerns, keep the protocol schema/generated Swift/test updates together, and land the focused producer-side fix. After landing, maintainers should decide whether #71178 closes as narrowed to this subset or remains open for graceful-drain and post-reset bridge follow-ups.
