## Summary

Keep open. Current main still ignores `mode` in WhatsApp outbound target resolution and rejects explicit direct-message targets that are outside a non-empty `allowFrom`; PR #48387 remains a focused live behavior change, with related #49549 tracking a broader `allowTo` design rather than replacing this PR outright.

## Best Possible Solution

Keep #48387 open for maintainer review. If maintainers want the narrow behavior, port the one-condition change to `extensions/whatsapp/src/resolve-outbound-target.ts`, add the explicit-target regression in `extensions/whatsapp/src/resolve-outbound-target.test.ts`, and preserve implicit/heartbeat/action authorization gating. If maintainers prefer outbound-specific policy, use #49549 as the broader design path rather than closing this as implemented.
