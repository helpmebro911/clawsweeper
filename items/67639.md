## Summary

Keep this PR open. Current main still drops the status scan source config before the memory status probe boots the memory runtime, so the PR remains a plausible fix for the false-positive `plugins.allow is empty` warning reported in #62049. The branch likely needs a scoped rebase or split, not automatic closure.

## Best Possible Solution

Keep this PR open and review a scoped rebase: thread `overview.sourceConfig` as `activationSourceConfig` through `executeStatusScanFromOverview`, `resolveStatusMemoryStatusSnapshot`, `resolveSharedMemoryStatusSnapshot`, `getActiveMemorySearchManager`, and the memory runtime bootstrap. Keep `activationSourceConfig` internal to runtime loading, do not pass it to plugin `getMemorySearchManager` hooks, preserve omission of optional `purpose`, and split unrelated extension/type cleanup out of the branch before validation.
