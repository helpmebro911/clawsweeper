## Summary

Keep open. Current main at 348728c28c1ea9ae7c5824350baba35ee08a275e still lacks the requested CDP/IME text insertion action: the browser tool schema, route kind gate, typed request union, normalization, managed Playwright execution path, existing-session Chrome MCP path, and docs only expose the existing `type`/`fill` style actions. The related reports found in the provided context cover CDP process reuse, stale ports, downloads, or attach timeouts, not an `Input.insertText` action, and the prior ClawSweeper comment correctly identified the same remaining gap against an older main SHA.

## Best Possible Solution

Keep this issue open and implement the capability in the bundled browser plugin as an additive `browser act` kind. The likely maintainer path is to extend `BrowserActRequest`, schema, normalization, CLI/docs, and `/act` handling; focus the requested ref/selector/tab; issue CDP `Input.insertText` for managed/raw CDP profiles with the same timeout and navigation-safety expectations as other interactions; and either support existing-session profiles through a real Chrome MCP/CDP route or return a documented unsupported error. Add focused regression coverage for schema acceptance, route normalization, execution dispatch, and docs describing when to prefer the new action over `type`/`fill`.
