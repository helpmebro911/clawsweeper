## Summary

Keep this issue open. Current main still has no additive browser act kind for CDP/IME text insertion, and the existing text paths continue to normalize and dispatch only `type`/`fill` semantics through Playwright fill/type or Chrome MCP fill.

## Best Possible Solution

Implement this in the bundled browser plugin as an additive browser act kind, likely `insertTextIme` or `cdpInsertText`: extend the schema, `BrowserActRequest`, `ACT_KINDS`, normalization, CLI/tool docs, timeout handling, and `/act` dispatch; focus the target tab/element; send CDP `Input.insertText` for managed/raw CDP profiles; define existing-session behavior explicitly; and add schema, normalization, dispatch, and docs tests.
