## Summary

Current main still lacks the ordering guard this issue asks for. The Control UI model picker performs an optimistic async `sessions.patch`, but the send path has no shared pending model-switch state to await, the send button is not disabled for such a pending patch, and `chat.send` cannot carry the selected model directly.

## Best Possible Solution

Keep this issue open and implement the UI-side ordering guard: store a pending model-switch operation in Control UI state, disable the model picker and send controls while it is pending, have `handleSendChat` await that operation before `chat.send`, roll back and do not send if `sessions.patch` fails, and add a regression test that holds the patch promise unresolved while an immediate send is attempted.
