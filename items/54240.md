## Summary

Keep this issue open. Current main still has the Control UI model-switch/send ordering race: changing the dropdown performs an optimistic local update and an async `sessions.patch`, while the send path neither waits for that patch nor includes a model override in `chat.send`. Adjacent model-picker fixes on main address qualified/stale display behavior, not this in-flight ordering barrier.

## Best Possible Solution

Implement the proposed Control UI ordering guard: track the pending model-switch operation in UI state, disable the model picker and send while it is pending, make `handleSendChat` await the pending switch before calling `chat.send`, roll back and avoid sending if `sessions.patch` fails, and add a regression test that changes the model with unresolved `sessions.patch` and immediately attempts to send.
