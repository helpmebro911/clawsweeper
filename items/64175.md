## Summary

Keep this issue open. Current main still preserves OpenRouter `supported_parameters` for scanning only, but the runtime OpenRouter capability cache and dynamic model resolver do not map the upstream `tools` signal into `compat.supportsTools`. As a result, the normal agent path still defaults dynamic OpenRouter models to tool-capable and can attach OpenClaw tools to `openrouter/perplexity/sonar-deep-research`.

## Best Possible Solution

Fix this at the OpenRouter provider/runtime boundary: preserve whether OpenRouter `supported_parameters` includes `tools`, carry that through `OpenRouterModelCapabilities`, set `compat.supportsTools` for dynamic OpenRouter models, handle old cache entries conservatively, and add regression coverage for `openrouter/perplexity/sonar-deep-research` on the normal agent runtime path.
