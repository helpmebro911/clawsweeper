## Summary

Keep this issue open. Current main still only applies path-date temporal decay to top-level `memory/YYYY-MM-DD.md` files; nested and date-suffixed memory files remain unmatched, and #68155 is open and unmerged as the implementation candidate.

## Best Possible Solution

Review and either land or close #68155. If it does not land, update `extensions/memory-core/src/memory/temporal-decay.ts` so temporal decay recognizes dated markdown files under `memory/` subdirectories while keeping non-dated memory topic files evergreen, and add focused regression tests in `extensions/memory-core/src/memory/temporal-decay.test.ts` for nested dated paths, date-prefixed filenames if intended, and evergreen non-dated paths.
