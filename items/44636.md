## Summary

The reported root cause does not match OpenClaw’s code path. In the reporter’s cited version `2026.3.8` (`3caab9260cb0a0064e6a37b2de3bedc8a547e599`), Google audio transcription already routed through a shared Gemini inline-data helper that sets `contents[0].role = "user"`, and current `main` still does the same.
