## Summary

Keep open. Current main has targeted lifecycle/cleanup guards for some retry and transcript cases, but it does not implement the requested standardized async resource lifecycle manager or equivalent shared shutdown protocol. At least one motivating cleanup path still has the same immediate-close/delete shape, and related remaining work is not fully superseded by a narrower canonical item.

## Evidence

- **No centralized ResourceManager implementation found:** A targeted repo search found no `ResourceManager` or `PluginResourceManager` implementation in source, plugins, packages, docs, or tests.

  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
