## Summary

Keep open. Current main still has hard-coded config-recovery notices and restart-sentinel wake/delivery behavior, and the gateway config schema/types do not expose `gateway.systemEvents.recoveryNotice` or an equivalent suppression control. The related items in the provided context do not supersede this: #71289 narrowed when whole-file recovery happens, #49579 is a narrow WhatsApp banner filter PR, and #39307 asks for more user-visible reset signaling rather than this operator-controlled suppression knob.

## Best Possible Solution

Keep the issue open and implement this as a core Gateway config feature, not a plugin. The narrow path is to add a typed `gateway.systemEvents.recoveryNotice` policy to schema/help/generated docs, gate both restart-sentinel notice surfaces and `enqueueConfigRecoveryNotice`, preserve restart continuations and actual recovery behavior, and add focused tests for the default user-visible mode plus log-only/off modes.
