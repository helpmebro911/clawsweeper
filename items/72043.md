## Summary

Keep open. Current main still has hard-coded config-recovery/restart-sentinel notices and no `gateway.systemEvents.recoveryNotice` config surface. The related #49579 WhatsApp banner PR and #39307 context-reset visibility issue do not supersede this narrower recovery-notice control request.

## Best Possible Solution

Keep this issue open and implement it in core gateway config, not as a plugin. Add a typed `gateway.systemEvents.recoveryNotice` mode to the config schema/help/docs, default it to current behavior, and thread it through config recovery notice and restart-sentinel startup handling so `user`, `log`, and `off` gate prompt injection and out-of-band delivery without changing recovery or continuation execution.
