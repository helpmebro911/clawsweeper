## Summary

Keep #72148 open. Current main still prepends the user-facing "Automatic session resume failed, so sending the status directly." text when exec approval follow-up session resume fails, and tests still assert that behavior. The PR is a recent, focused fix for #72143 and is not superseded by the related #72268 parser PR.

## Best Possible Solution

Keep #72148 open for maintainer review. The best path is to land a focused fix that removes the internal resume-failure prefix from user-facing exec approval fallback messages, logs resume failures internally with robust redaction/sanitization, and tracks retry/backoff separately unless maintainers decide it belongs in the same issue scope. Treat #72268 as related parser work, not a superseding fix.
