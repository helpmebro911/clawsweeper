## Summary

Keep this issue open. Current main still has the reported framework-level boundary-loss behavior: the chunker consumes paragraph/newline separators, the coalescer can only reinsert a joiner while chunks remain in the same buffered flush, and downstream block reply payloads carry plain text without boundary metadata for later streamed deliveries.

## Best Possible Solution

Keep this issue open and fix it in the shared block streaming pipeline. The likely path is to either keep consumed separators attached to emitted chunks while normalizing only dedupe comparisons, or carry explicit boundary metadata from EmbeddedBlockChunker through block reply coalescing/delivery. Add a regression test that streams Markdown like `# Title\n\nFirst paragraph.` across at least two flushed text_end deliveries and asserts the reconstructed visible text remains structurally equivalent.

## Risks / Open Questions

- A fix must avoid double-inserting separators in the already coalesced case while preserving paragraph/newline semantics across separate flushes.
- Current dedupe normalizes whitespace when comparing streamed fragments to a final payload, so any boundary-preservation change needs focused tests around final-payload suppression as well as visible streamed text reconstruction.
