## Summary

Keep #40205 open. Current main still uses Ollama's deprecated `/api/embeddings` contract in the active bundled Ollama memory embedding provider, and the current tests still assert that old endpoint/request/response shape. The PR's exact files are stale because this code now lives under `extensions/ollama`, but the requested behavior remains useful and should be rebased rather than closed.

## Best Possible Solution

Keep this PR open and ask for a rebase/update against `extensions/ollama/src/embedding-provider.ts` and `extensions/ollama/src/embedding-provider.test.ts`: use `POST /api/embed`, send `{ model, input }`, parse the first vector from `embeddings`, update assertions away from `prompt`, and preserve the current serialized batch behavior for local Ollama stability. Then run the relevant extension changed checks before merge.
