## Summary

Keep #13487 open. Current main still only detects Discord mentions/replies relative to the currently running bot account and does not implement a cross-account target router that suppresses the default owner when another OpenClaw bot is mentioned or replied to. Default-agent fallback and optional `ignoreOtherMentions` behavior remain, and the existing tests/docs cover current-bot mention gating rather than the requested mention > reply-target > default-owner suppression matrix.

## Best Possible Solution

Keep #13487 open until the Discord plugin makes one deterministic target decision before normal/default routing: map configured Discord account bot user IDs to OpenClaw accounts/agents, choose eligible responders from Discord mention metadata first, referenced-message author second, and only then fall back to the per-channel/default owner. Suppress non-target/default responders whenever a targeted mention or reply applies, and add regression coverage for explicit mention, native reply, reply-plus-mention, and untagged default-owner cases. #39428/#39485 may help account routing, but they should not close this issue unless they implement this exact suppression matrix.
