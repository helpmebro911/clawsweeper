## Summary

Keep #13487 open. Current main still has per-account Discord handling, not a Discord-wide target-selection step that chooses explicit bot mention, then reply target, then default owner while suppressing all non-target/default responders. The existing issue discussion is also concrete: GTZhou reproduced reply-shaped guild-channel failures, and the latest maintainer-context comment already kept this open after comparing related #39428/#39485.

## Best Possible Solution

Keep #13487 open until the Discord plugin performs one deterministic cross-account target decision before default routing: map configured Discord bot user IDs to owning OpenClaw accounts/agents, select explicit bot mentions first, then referenced-message author, and use the per-channel default owner only for untagged non-reply messages. Suppress all non-target/default responders whenever mention or reply targeting applies, and add regression tests for explicit mention, native reply, reply-plus-mention, and untagged default-owner cases.
