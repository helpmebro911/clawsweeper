## Summary

Keep #13487 open. Current main still performs Discord handling per account/bot and only uses mention/reply metadata to gate the current handler, not to make one Discord-wide target-selection decision. The requested mention > reply-target > default-owner precedence, including suppression of the default owner when a targeted bot applies, remains missing. The existing issue discussion includes a concrete multi-bot reproduction and an April 26 maintainer-context Codex review that also kept the item open after considering related #39428/#39485.

## Best Possible Solution

Keep #13487 open until the Discord plugin performs one deterministic cross-account target decision before default routing: map configured Discord bot user IDs to their OpenClaw accounts/agents, choose explicit bot mentions first, then referenced-message author, and use the per-channel default owner only for untagged non-reply messages. Suppress all non-target/default responders whenever mention or reply targeting applies, and add regression tests for explicit mention, native reply, reply-plus-mention, multiple mentioned bots, and untagged default-owner cases.
