## Summary

Keep PR #71898 open. Current main still has the reported memory_search result-shaping bug: session-filtered memory-manager hits can reach the output path, but the surfaced result object hard-codes every memory-manager hit as `corpus: "memory"`. The PR is a focused, recent contributor bug fix with a regression test and an addressed bot-review readability comment, so conservative cleanup should not close it.

## Best Possible Solution

Keep PR #71898 open for normal maintainer review and CI. The best product/code path is to preserve `MemorySearchResult.source` when shaping `memory_search` results and keep the focused regression test that asserts session transcript hits surface as `corpus: "sessions"`.
