## Summary

Keep this PR open. Current main still uses ASCII-only Discord outbound mention parsing, so Unicode/CJK handle rewriting is not implemented. The PR remains useful, but it is now stale against the moved Discord plugin path and still has an unresolved correctness review around wrapped URL tokens; related #59818 overlaps but is also open and broader rather than a shipped superseding fix.

## Best Possible Solution

Keep this PR open as a useful but needs-revision candidate. Ask the author to rebase onto the current `extensions/discord` plugin path, fix the wrapped URL-token guard by recognizing or stripping leading URL wrappers, and add focused regressions for Unicode handles, Unicode local-part emails, plain URL paths, wrapped URL tokens, reserved mentions, and existing ASCII behavior. Maintainers should then compare it with #59818 and either land the narrower safe fix or consolidate the broader bot-to-bot work into one canonical Discord plugin PR.
