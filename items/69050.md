## Summary

Keep PR #69050 open. Current main still does not implement the proposed build-all change: `pnpm build` and `pnpm build:ci-artifacts` route through `scripts/build-all.mjs`, the full and `ciArtifacts` step lists still omit `ui:build`, and the build script still cleans `dist/` before rebuilding. Current release docs/scripts work around this by running `pnpm ui:build` separately, and gateway startup can try to repair missing assets, but that does not make the build-all pipeline produce `dist/control-ui/`.

## Best Possible Solution

Keep the PR open for maintainer review. The likely product/code path is either to merge the focused build-all addition with matching `test/scripts/build-all.test.ts` updates and validation that `pnpm build`/`build:ci-artifacts` produce `dist/control-ui`, or to explicitly document and enforce that `pnpm build && pnpm ui:build` remains the supported release/dev artifact contract and close manually as a product decision.
