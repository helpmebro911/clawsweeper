## Summary

Keep PR #71863 open. Current main still has the Signal daemon shutdown race: `SignalDaemonHandle.stop` is synchronous, only sends SIGTERM, and `monitorSignalProvider` calls it without awaiting during abort/finally cleanup. The PR directly addresses the linked #22676 behavior and its bot review follow-ups were addressed in the latest PR commit.

## Best Possible Solution

Keep #71863 open and review it as the active fix for #22676. The right product/code direction is to make Signal daemon shutdown await process exit, add a bounded SIGKILL fallback, await cleanup in the monitor abort/finally path, and keep the regression tests proposed by the PR.
