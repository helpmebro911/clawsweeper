## Summary

Keep PR #71594 open. Current main still lacks the PR's IPv4-only BYOH documentation and onboarding hint, while current gateway bind/link code does use IPv4-only paths for the relevant `lan`, `tailnet`, and `custom` BYOH surfaces. The PR remains focused docs/test work rather than something already implemented or superseded.

## Best Possible Solution

Keep PR #71594 open for maintainer review or merge after normal docs/test validation. The best product path is to document the current IPv4-only BYOH behavior in gateway CLI/config/onboarding surfaces now, while leaving native IPv6 BYOH support to a separate behavior/API change if maintainers want it.
