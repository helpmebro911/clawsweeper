## Summary

Keep open. Current main still does not propagate restart reasons through the restart emitter or deferred restart path that this PR targets, and the user-visible `reason=none` gateway-tool log remains possible when callers omit a reason. The linked bug may have later root-cause fixes, but this observability improvement is not implemented or clearly superseded on main.

## Best Possible Solution

Keep the PR or replace it with an updated maintainer patch that threads a sanitized restart reason through `scheduleGatewaySigusr1Restart`, `deferGatewayRestartUntilIdle`, `emitPreparedGatewayRestart`, and `emitGatewayRestart`, logs only after successful signal emission, passes config-reload reasons from `server-reload-handlers.ts`, and adds focused regression tests for emitted/deferred restart reason logging.
