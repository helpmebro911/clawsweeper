## Summary

Current main still does not appear to implement the reported WhatsApp trusted-contact natural-language relay path. Explicit WhatsApp outbound sends and same-provider cross-context sends are supported, but a direct-chat request like “tell Jack …” still has no WhatsApp-owned way to resolve the name “Jack” to an approved E.164/JID target.

## Best Possible Solution

Keep this issue open. Add a focused WhatsApp regression where an authorized direct sender asks to relay to another approved direct contact by name, then implement a WhatsApp-owned target-resolution path. Maintainers should decide whether names come from explicit config aliases, WhatsApp live contact metadata, prior session metadata, or only configured E.164/JID IDs. Use the existing directory entry name/handle fields or messaging.targetResolver.resolveTarget before adding any new generic core/plugin API seam.
