## Summary

Keep this issue open. Current main supports explicit WhatsApp outbound sends and same-provider cross-context sends, but I still do not find a WhatsApp-owned alias or regression path that turns a trusted DM sender's natural-language “tell Jack ...” request into an approved target for Jack unless the model already supplies Jack's E.164/JID target.

## Best Possible Solution

Keep this issue open. Add a focused WhatsApp regression where an authorized direct sender asks to relay to another approved direct target, then implement the smallest WhatsApp-owned fix that gives the agent a reliable target resolution path for that relay. That likely belongs in WhatsApp directory/alias/session-route guidance or message-tool prompt context, with a generic core/API seam only if WhatsApp cannot expose the needed contact metadata through the existing plugin directory contract.
