## Summary

Keep open. Current main still returns memory corpus supplements in registration/re-registration order, while the adjacent prompt supplement path is explicitly sorted for deterministic prompt-cache behavior. The PR is a narrow implementation candidate with focused regression tests and no observable security or supply-chain expansion in its provided diff.

## Best Possible Solution

Review or land the focused fix after any needed rebase: sort `listMemoryCorpusSupplements()` by `pluginId` in `src/plugins/memory-state.ts`, and keep regression tests covering reverse registration and same-plugin re-registration stability.
