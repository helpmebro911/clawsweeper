## Summary

Keep open. Current main still does not default the memory-core SQLite index database to WAL or `synchronous=NORMAL`, and this PR remains a plausible focused implementation candidate rather than something already shipped or safely superseded.

## Best Possible Solution

Keep this PR open as a current implementation candidate. The best path is to amend it with either a `journal_mode` result check or an explicit rebuildable-index tradeoff comment, add focused memory-core tests for fresh opens and safe reindex with WAL sidecars, then merge if maintainers accept WAL as the default. Older overlapping WAL PRs can be reconciled after this cleaner current-path review is resolved.
