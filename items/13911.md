## Summary

Keep #13911 open. Current main still has no first-class per-channel suppression or reroute policy for subagent completion announcements, and the source/docs/tests still preserve requester-channel completion delivery for top-level channel sessions. #69492 may eventually subsume the design, but it remains broader/open and is not shipped behavior.

## Best Possible Solution

Keep #13911 open. The best path is a core-supported subagent completion audience/delivery policy, wired through config schema/types/docs/tests and applied before external requester-channel delivery and fallback sends. It should cover Telegram/iMessage-style channels without relying on AGENTS.md behavior, string-based sanitizer proxies, or the current origin-only plugin hook; if maintainers want this plugin-owned, first extend the plugin API to express suppression explicitly.
