## Summary

Current main still lacks a first-class per-channel sub-agent completion announce suppression or reroute policy. The requester-channel announce path remains documented, implemented, and covered by tests, and the latest maintainer comment explicitly kept this issue open while noting #69492 is only a broader open design proposal.

## Best Possible Solution

Keep #13911 open. The best path is a core-supported sub-agent completion delivery/audience policy in config, schema, docs, and tests, applied before external requester-channel delivery is resolved. It should cover Telegram/iMessage-style channels without relying on AGENTS.md instructions, sentinel replies, outbound sanitizer proxies, or a plugin hook that currently cannot express suppression.
