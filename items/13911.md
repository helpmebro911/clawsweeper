## Summary

#13911 should remain open. Current main still has no first-class per-channel or per-subagent completion announce suppression/reroute policy, while source, docs, and tests continue to preserve requester-channel completion delivery for top-level channel sessions. The provided issue discussion also includes Telegram and iMessage confirmations plus a recent maintainer-authored Codex review keeping the item open; #69492 may eventually subsume the design but is still broader/open and not shipped behavior.

## Best Possible Solution

Keep #13911 open. The best path is a core-supported subagent completion delivery/audience policy, wired through config schema, generated config docs, user docs, and regression tests, and applied before external requester-channel delivery and fallback sends. It should cover Telegram, iMessage, Discord, and similar channel sessions without relying on AGENTS.md behavior or sanitizer proxies; if maintainers want this plugin-owned, first extend the plugin API so a hook can explicitly suppress subagent completion announcements by provenance and channel.
