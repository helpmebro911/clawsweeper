## Summary

Keep open. Current main still has unsafe non-array assistant content handling in the OpenAI Responses conversion path, while the related fix PRs are closed unmerged and only part of the replay-string case is covered today.

## Best Possible Solution

Keep this issue open and land a focused core fix around the OpenAI-compatible ingress/replay boundaries: guard or normalize non-array assistant `content` in `buildAssistantMessageFromResponse`, preserve the existing string replay repair, decide whether object/null content should coerce or fail soft, and add regression tests for string, object, and null provider payloads. #43864 is useful prior analysis but should be reviewed or reimplemented against current main.
