## Summary

Keep #71463 open. Current main still emits per-call voice-call restore startup logs, so the cleanup is not implemented or superseded. The PR is a plausible narrow logging improvement, but maintainer and bot review both identify remaining gaps and missing regression tests before it should land.

## Best Possible Solution

No docs or changelog are necessary unless maintainers want to mention operational log noise reduction.
