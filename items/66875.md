## Summary

Keep this issue open. Current main still has the redundant same-run final chat.history reload path that the report identifies, while the shipped fixes only mitigate active session.message reloads, empty/stale history results, and loading-skeleton flicker. The open maintainer PR #72325 remains the right implementation candidate for the remaining final reconciliation race.

## Best Possible Solution

Keep this issue open and resolve it through #72325 or an equivalent narrow Control UI/WebChat change. The fix should suppress redundant history reloads for same-run final assistant payloads already incorporated locally, preserve legitimate hydration for tool/no-message finals and deferred session.message reloads, and update the app-gateway, chat-event-reload, and controller tests before closing.
