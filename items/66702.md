## Summary

Keep open. Current main contains the narrower #71868 mitigation for old pruned replay markers, but it still does not scope image hydration to current-turn media or exclude injected memory/untrusted-context blocks from image-reference scanning.

## Best Possible Solution

Implement a focused embedded-runner/media fix: make image hydration current-turn aware, prefer only explicit current-turn attachments when the current turn has media unless the user asks to compare prior images, exclude injected `<relevant-memories>` and `Untrusted context` blocks from prompt image-ref detection, and add regression tests for memory/untrusted-context stale refs plus recent image-heavy history.
