## Summary

Keep open. Current `main` does not implement this fix, and the PR discussion identifies unresolved correctness gaps in the submitted patch: the console filter would be restored before later `confirm()` prompts, and there is no error-path restore test.

## Evidence

- **Current main still runs `secrets configure` without any console filter guard:** In `main`, the `configure` action calls `runSecretsConfigureInteractive(...)` directly, then continues into two later `confirm(...)` prompts. There is no save/restore of console subsystem filters around this interactive flow.
