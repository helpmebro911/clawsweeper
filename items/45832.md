## Summary

Keep PR #45832 open. Current main has moved the WhatsApp Web code from the old `src/web` files into `extensions/whatsapp`, and the exact PR patch is stale, but the central hardening request is not implemented on main: several WhatsApp credential reads still follow symlinks, and the auth directory is still created through the default `ensureDir` path without explicit `0o700`/chmod hardening. The PR discussion is still useful: Greptile’s directory-mode gap was addressed on the PR branch in `2775428`, but that commit is not present on current main.

## Best Possible Solution

Keep the PR open but ask for a rebase/port against `extensions/whatsapp/src/{auth-store.ts,creds-files.ts,session.ts}`. The best fix is a small shared symlink-rejection helper for WhatsApp credential files, regression tests for `creds.json` read/status/self-id/age and backup restore behavior, and explicit `0o700` plus best-effort chmod for the auth directory in the current session creation path.
