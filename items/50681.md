## Summary

Keep open. Current main already blocks unknown Telegram groups before assistant execution when group allowlisting is active, so part of the security posture exists. The requested first-class unknown-group action surface remains missing: there are no accepted `unknownGroupAction`/message/cooldown config fields, no Telegram runtime path to warn and call `leaveChat`, and the tests cover rejection only, not warning or auto-leave.

## Evidence

- **Unknown groups are currently fail-closed by allowlist:** `evaluateTelegramGroupPolicyAccess` checks the chat allowlist first and returns `group-chat-not-allowed` when the chat is not allowed.
  - file: [extensions/telegram/src/group-access.ts:161](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/extensions/telegram/src/group-access.ts#L161)
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **Group preflight happens before assistant processing:** Inbound Telegram handling calls `shouldSkipGroupMessage` and returns before `processInboundMessage`, so blocked groups do not reach assistant execution.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:1891](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/extensions/telegram/src/bot-handlers.runtime.ts#L1891)
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **Requested config fields are not in the strict Telegram schema:** `TelegramAccountSchemaBase` is a strict object and includes existing `groups`, `allowFrom`, `groupAllowFrom`, and `groupPolicy` fields, but no `unknownGroupAction`, `unknownGroupMessage`, or `unknownGroupCooldownMs`.
  - file: [src/config/zod-schema.providers-core.ts:207](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/config/zod-schema.providers-core.ts#L207)
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **Requested config fields are not in Telegram types:** `TelegramAccountConfig` documents the current Telegram access-control fields and has no unknown-group action/message/cooldown fields.
  - file: [src/config/types.telegram.ts:94](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/config/types.telegram.ts#L94)
