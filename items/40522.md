## Summary

Keep open. Current `main` still does not redirect gateway `/` to the configured Control UI `basePath`; with `basePath` set, `/` falls through the Control UI router and the server returns `404 Not Found`. The PR’s requested behavior is not implemented on `main`.

## Evidence

- **Base-path router still rejects `/`:** In the basePath branch, requests outside `${basePath}/` and `basePath` immediately return `not-control-ui`. There is no special case for `pathname === "/"`.
  - file: [src/gateway/control-ui-routing.ts:41](https://github.com/openclaw/openclaw/blob/171322644dece55402569c52b26370fd81769ca6/src/gateway/control-ui-routing.ts#L41)
  - command: `nl -ba src/gateway/control-ui-routing.ts | sed -n '1,220p'`
  - sha: [171322644dec](https://github.com/openclaw/openclaw/commit/171322644dece55402569c52b26370fd81769ca6)
- **Unhandled control-ui requests fall through:** `handleControlUiHttpRequest()` delegates to `classifyControlUiRequest()`. When the route is `not-control-ui`, it returns `false` instead of issuing a redirect.
  - file: [src/gateway/control-ui.ts:696](https://github.com/openclaw/openclaw/blob/171322644dece55402569c52b26370fd81769ca6/src/gateway/control-ui.ts#L696)

  - sha: [171322644dec](https://github.com/openclaw/openclaw/commit/171322644dece55402569c52b26370fd81769ca6)

  - file: [src/gateway/server-http.ts:1147](https://github.com/openclaw/openclaw/blob/171322644dece55402569c52b26370fd81769ca6/src/gateway/server-http.ts#L1147)
