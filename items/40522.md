## Summary

Keep this PR open. Current main still lacks the requested root-to-Control-UI-basePath redirect, while the PR remains a focused implementation candidate for the linked basePath Not Found workflow and includes the review-requested query-string regression test in the provided context.

## Best Possible Solution

Keep this PR open for maintainer review and rebase/reapply. The best implementation is the narrow gateway routing change in `classifyControlUiRequest`: when `basePath` is non-empty and `pathname === "/"`, redirect `GET` and `HEAD` to `${basePath}/${search}`, preserve non-read fallthrough for plugin/webhook routes, keep regression tests for root redirect, query preservation, and write-method fallthrough, and either harden or explicitly document the accepted `basePath` shape before merge.
