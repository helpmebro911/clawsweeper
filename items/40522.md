## Summary

Keep open. Current main still lacks the PR's requested root-to-Control-UI-basePath redirect: when `gateway.controlUi.basePath` is set, GET `/` is classified as `not-control-ui`, falls through the Control UI handler, and the gateway returns plain `Not Found`. The PR remains a focused fix for the linked #34298 behavior, and the provided PR discussion shows the only review-requested test gap, query-string preservation, was addressed.

## Best Possible Solution

Keep the PR open. A maintainer should merge it or reapply the same narrow change on main: make `classifyControlUiRequest` redirect read-method requests for `/` to `${basePath}/` while preserving the query string, keep non-read root requests falling through for plugin/webhook safety, and retain regression tests for GET root redirect, query-string preservation, and POST fall-through.
