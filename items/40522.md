## Summary

Keep this PR open. Current main still lacks the requested gateway-root redirect when `gateway.controlUi.basePath` is configured, while the PR remains a narrow implementation candidate with tests for root redirect, query-string preservation, and non-read fallthrough.

## Best Possible Solution

Keep this PR open. A maintainer should merge, rebase, or reapply the same narrow gateway change on current main: in `classifyControlUiRequest`, when `basePath` is non-empty and `pathname === "/"`, redirect read methods to `${basePath}/${search}`, keep non-read root requests as `not-control-ui`, and retain regression tests for root redirect, query-string preservation, and POST fallthrough.
