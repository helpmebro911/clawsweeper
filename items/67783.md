## Summary

Keep this PR open. Current main still uses the Feishu reply target as the typing-reaction target, while topic-mode and replyInThread group flows intentionally make that reply target the thread/topic root. The PR is a narrow implementation candidate for a still-present Feishu/Lark UX bug, not obsolete or superseded by the related closed bug report.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or reimplement the narrow fix: thread ctx.messageId into createFeishuReplyDispatcher as an optional inbound-message target, use it for addTypingIndicator with replyToMessageId fallback, and keep/add focused bot and dispatcher tests covering topic/replyInThread behavior plus fallback behavior for existing flows.
