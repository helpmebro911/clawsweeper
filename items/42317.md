## Summary

Keep this PR open. Current main still does not honor session.mainKey in the core routing path: runtime config materialization still rewrites any configured value to "main", and resolveAgentRoute still builds both peer and main session keys with DEFAULT_MAIN_KEY. The PR is still open, unmerged, and remains the canonical implementation candidate for the closed linked feature issue, but it needs a rebase/path adaptation and the custom-main-key session-delivery guard concern addressed before landing.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. Maintainers should rebase it onto current main, port stale Telegram and WhatsApp hunks to the current plugin paths, preserve configured session.mainKey through runtime defaults and all route builders, add guard coverage in session-delivery.ts for custom main keys, align docs/config help, then run the changed config/routing/plugin tests before merging or requesting revisions.
