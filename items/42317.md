## Summary

Keep open. Current `main` does not implement this PR’s behavior: runtime config normalization still overwrites `session.mainKey` to `"main"`, and route/session builders still hardcode `DEFAULT_MAIN_KEY` in the inbound routing paths this PR targets. There is adjacent support for custom main keys elsewhere, so the request is still relevant rather than obsolete, but it is not already implemented on `main`.
