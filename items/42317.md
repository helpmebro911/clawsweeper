## Summary

Keep this PR open. Current main still does not implement the central behavior: runtime config still forces a configured session.mainKey back to "main", and several routing/build paths still hard-code or omit the configured main key. The PR remains a focused implementation candidate for linked #41790, but it needs a rebase/port plus the unresolved main-session delivery guard fix before it is safe to merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, port the channel/outbound hunks to current plugin paths, preserve configured session.mainKey through runtime defaults and every route/session-key builder, update delivery guards to recognize configured custom main keys, align docs/schema help, and run the changed config/routing/plugin verification before merge or maintainer-requested revisions.
