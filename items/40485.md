## Summary

Keep this PR open: current main still appears to have the Telegram native-command reply-threading behavior the PR is trying to fix, but the PR needs to be ported from removed `src/telegram` paths to the current Telegram plugin layout.

## Best Possible Solution

Keep this PR open and port the narrow fix to `extensions/telegram/src/bot-native-commands.ts`: allow native command delivery to override `replyToMode`, set it to `"off"` only for direct-chat native `new` and `reset`, preserve group/topic and other-command threading behavior, and add regression tests for both `/new` and `/reset` with global Telegram `replyToMode: "first"`.
