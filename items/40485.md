## Summary

Keep this PR open. Current main still sends Telegram native command replies through the configured account-level replyToMode and still falls back to replying to the triggering slash-command message. The PR addresses a plausible current bug, but it now targets removed legacy src/telegram paths and needs to be ported to the active extensions/telegram plugin with /reset coverage added.

## Best Possible Solution

Keep this PR open and port the narrow fix to extensions/telegram/src/bot-native-commands.ts. Add a per-command override or equivalent delivery option that sets replyToMode to "off" only for private-chat native /new and /reset replies, preserve configured threading for groups, topics, plugin commands, and other native commands, and add regression coverage for both /new and /reset with configured replyToMode "first".
