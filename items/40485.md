## Summary

Keep open. Current `main` still lacks the PR's narrow Telegram DM override: native `/new` and `/reset` replies still inherit the account-level `replyToMode`, and the native-command delivery path still injects `replyToId` from the triggering message. That means the reported `Deleted message` reply target remains plausible on current `main`, so there is not strong evidence to close this PR as already implemented or obsolete.
