## Summary

Keep this PR open. The branch is a plausible implementation attempt for the linked hook regression, but the author’s own follow-up says it remained incomplete after live WhatsApp testing, and current main still does not provide a complete generic received-hook emission at the active-session enqueue/drain boundary. The PR also changes when inbound content becomes plugin-visible, so it needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review, but request a revision or replacement that defines the real generic inbound hook boundary, covers active-session enqueue/drain behavior with regression tests, and explicitly reconciles plugin message_received/internal message:received semantics with WhatsApp’s opt-in privacy policy. If maintainers decide the WhatsApp-specific main implementation fully supersedes the linked bug, close this PR only with that canonical evidence.
