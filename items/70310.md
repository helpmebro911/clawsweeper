## Summary

Keep #70310 open. Current main does not implement the requested QQBot token/API SSRF fake-IP policy: token and REST API requests still use raw `fetch()`, while the RFC2544/fake-IP allowance is only present on QQBot media downloads. The provided related title matches do not supersede this QQBot-specific bug.

## Best Possible Solution

Implement the QQBot fix in the bundled plugin: route token and REST API requests through `fetchWithSsrFGuard` or a plugin SDK guarded-fetch helper, apply a narrow QQBot hostname policy with `allowRfc2544BenchmarkRange: true`, remove the QQBot token/API entries from the temporary raw-fetch allowlist, and add regression tests where `bots.qq.com`/`api.sgroup.qq.com` resolve to `198.18.x.x`.
