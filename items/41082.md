## Summary

Keep open. Current main has substantial compaction hardening, including default safeguard mode, recent-turn preservation, and a quality audit that retries when the compacted summary omits the latest user ask. However, I did not find a source or regression test proving the specific reported failure is fixed: a new user question appended after compaction being answered as if the old compacted topic were still current. The issue is not protected/maintainer-authored and is not old enough for stale-insufficient-info closure.
