## Summary

Keep open. Current main has meaningful compaction mitigations, but they do not conclusively close the reported post-compaction topic-switch bug. The code preserves recent turns, re-distills prior summaries, and injects an AGENTS.md refresh after compaction, yet I did not find a regression test or implementation that specifically detects a new topic after compaction and demotes the old summary. The public docs still describe stale post-compaction context as a troubleshooting case.

## Best Possible Solution

Keep this issue open and add a targeted compaction-context regression around the embedded run path: create a transcript with an old-topic compaction summary, append a fresh unrelated user question after the compaction boundary, then assert the assembled context/current prompt clearly makes the fresh user question the actionable request. If the test exposes ambiguity, implement the fix near prompt assembly or compaction-summary guidance rather than closing this as already handled.
