## Summary

Keep PR #70598 open. Current main already implements the central exec heartbeat prompt fix, but this PR still appears to contain useful follow-up work for mixed event queues: main filters exec text into the exec prompt, then consumes the full inspected event queue, so non-exec reminders in the same inspected batch can be dropped without being rendered or deferred.

## Best Possible Solution

Keep the PR open or supersede it with a focused follow-up that preserves current main's shipped exec prompt behavior while adding the missing deferred-event restoration and regression tests for mixed exec plus cron/reminder queues, including duplicate reminder preservation.
