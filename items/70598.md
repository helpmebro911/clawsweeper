## Summary

Keep PR #70598 open. Current main already implements the central exec heartbeat prompt fix, but this PR still contains useful follow-up work that is not on main: preserving deferred non-exec system events, including duplicate reminders, when an exec-event heartbeat consumes the inspected queue. The provided PR discussion shows reviewers called out that remaining data-loss path, and current main still has the source shape that causes it.

## Best Possible Solution

Keep #70598 open or ask the author to rebase/narrow it to the remaining behavior: after an exec-event heartbeat handles only exec-classified entries, preserve deferred non-exec pending events without routing them through consecutive-duplicate suppression. The best final patch should add/keep focused tests for mixed exec+cron queues and duplicate deferred reminders, then close this PR only after that work lands or a canonical replacement tracks it.
