## Summary

Keep #39031 open. Current main still does not forward legacy iMessage `imsg rpc` tapback reactions to the agent; the BlueBubbles path has reaction system events, but that does not satisfy this legacy `channels.imessage` request. The provided related-item context also treats #39031 as canonical, with #39322 still open as the implementation PR and #60677 open as a filtering-only PR.

## Best Possible Solution

Keep #39031 open as the canonical legacy iMessage `imsg rpc` tapback-to-agent feature request. Maintainers should review, rebase, or supersede #39322 with a typed implementation that subscribes to reaction notifications, parses target/reaction metadata, routes them as reaction/system events rather than ordinary text, defines access-control and dedupe behavior, and updates config/docs/tests. If legacy `imsg` reaction forwarding is intentionally out of scope, document that explicitly and point users to BlueBubbles for reaction-event support.
