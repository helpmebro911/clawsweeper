## Summary

Keep open. Current main still enforces same-root realpath containment for skills and does not implement the PR's cross-configured-root symlink allowlist. The PR is addressing a plausible gap, but the submitted approach has an unresolved review concern: loading a symlinked bundled skill through a workspace/managed/extra root would keep the non-bundled source and could bypass `skills.allowBundled`. Related PR #59688 overlaps but is narrower, still open, and not a merged/superseding fix.

## Best Possible Solution

Keep the PR open, but require a revised implementation. The best path is to allow symlinks only among safe non-bundled configured roots or to preserve the target root's source policy when a symlink resolves into bundled skills, then add regression tests for both the intended cross-root symlink case and the `skills.allowBundled` bypass case. Coordinate with the narrower open #59688 so maintainers can land one consolidated skills symlink fix rather than competing partial patches.
