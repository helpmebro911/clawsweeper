## Summary

Keep PR #38981 open. Current main still does not implement `workspaceConfig.allowedExternalPaths`, and the requested behavior is a core agent/gateway workspace trust-boundary change rather than ClawHub/plugin cleanup. The PR discussion shows real review issues were found and follow-up commits addressed write parity, bootstrap context loading, per-agent allowlist wiring, and absolute-prefix enforcement, so the work is not obsolete.

## Best Possible Solution

Keep PR #38981 open for maintainer review. The best path is to either land a carefully reviewed core implementation covering config types, zod schemas/help/docs, bootstrap context reads, extra bootstrap files, gateway `agents.files.get/set`, and symlink/write regression tests, or make an explicit maintainer product/security decision that external symlinked workspace context remains unsupported and document that boundary. This belongs in OpenClaw core because the missing behavior is in the agent/gateway workspace safety layer, not in a plugin or ClawHub surface.
