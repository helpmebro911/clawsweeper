## Summary

Keep PR #70821 open. Current main still lacks the Bedrock plugin profile change: `amazon-bedrock` only advertises `off/minimal/low/medium/high` plus `adaptive` for Claude 4.6, so `xhigh` remains unavailable for `amazon-bedrock/...claude-opus-4-7` even though the shared Anthropic transport can map Opus 4.7 `xhigh` correctly.

## Best Possible Solution

Keep the PR open and land a narrow Bedrock-owned fix: add an Opus 4.7 model-id matcher in `extensions/amazon-bedrock/register.sync.runtime.ts`, advertise `{ id: "xhigh" }` only for matching Claude Opus 4.7 Bedrock ids, preserve the existing Claude 4.6 `adaptive` default behavior, and keep the focused regression test in `extensions/amazon-bedrock/index.test.ts` asserting xhigh is present for Opus 4.7 and absent for Opus 4.6/Sonnet 4.6.
