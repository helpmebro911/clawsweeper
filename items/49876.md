## Summary

Keep open. Current main has a narrow fail-closed guard for explicit tool allowlists that resolve to no callable tools, but it does not implement the issue's requested unattended cron/session delivery policy: suppressing fabricated placeholder/mock output, blocking unsupported success claims against a tool-call ledger, or failing closed when required tool calls fail or produce missing data. The linked discussion also broadened the problem beyond Gemini cron runs to interactive unsupported success claims; #45049 remains open for simulated tool-call behavior, while #68051 was a companion proposal rather than evidence that this issue's cron delivery gap is implemented here.
