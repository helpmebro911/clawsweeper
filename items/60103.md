## Summary

Keep open. Current `main` still has the Feishu mention-format mismatch the issue describes: agents are told to expect/write text-style `<at user_id="...">name</at>` tags, but outbound card sends pass reply text through unchanged into card markdown, and the non-card path sends a `post` markdown payload rather than normalizing mention tags into Feishu post/card-native structures.

## Evidence

- **Inbound prompt still teaches text-style mention tags:** Feishu message context tells the agent that content may include mention tags in the form `<at user_id="...">name</at>`, which matches the issue’s report about the syntax agents naturally reuse.
  - file: [extensions/feishu/src/bot.ts:168](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/feishu/src/bot.ts#L168)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Separate helper functions confirm card and text/post mention syntaxes differ:** The Feishu extension still has distinct helpers: text uses `<at user_id="...">name</at>` while cards use `<at id=...></at>`. There is no shared normalization helper here.
  - file: [extensions/feishu/src/mention.ts:78](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/feishu/src/mention.ts#L78)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Card send path does not normalize agent-authored mention tags:** `sendStructuredCardFeishu`/`sendMarkdownCardFeishu` only rewrite mentions when explicit `mentions` targets are separately supplied; otherwise they pass `text` straight into the card markdown payload. A reply body that already contains `<at user_id="...">...` remains unchanged in card output.
  - file: [extensions/feishu/src/send.ts:661](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/feishu/src/send.ts#L661)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Non-card send path also lacks post-format mention normalization:** `sendMessageFeishu` wraps the outbound body into a `post` payload with a single `tag: "md"` text node. It only prepends `buildMentionedMessage(...)` for explicit mention targets and does not convert inline `<at ...>` tags into Feishu post `tag:"at"` objects.
  - file: [extensions/feishu/src/send.ts:454](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/feishu/src/send.ts#L454)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Docs still describe streaming card output as the default:** The public Feishu docs state that streaming replies use interactive cards and document `streaming: true` / `blockStreaming: true` as defaults, which matches the issue’s reported default card-rendering path.
  - file: [docs/channels/feishu.md:238](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/docs/channels/feishu.md#L238)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
