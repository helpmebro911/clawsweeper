## Summary

Keep this PR open. Current main still uses a direct `meta.token !== token` comparison for diffs artifact viewer tokens, while the PR is a narrow one-file hardening change that reuses the already exported `safeEqualSecret` helper without adding unrelated security-sensitive surface.

## Best Possible Solution

Keep this PR open as the focused implementation candidate. The best path is to land the narrow `DiffArtifactStore.getArtifact` change using `safeEqualSecret` from `openclaw/plugin-sdk/browser-security-runtime`, then run the repository's changed-check or targeted diffs/plugin validation before merge.
