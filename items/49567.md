## Summary

Keep open. Current main still does not support per-binding tool allow/deny policy in route or ACP bindings: the binding Zod schemas and exported binding types have no `tools` field, the binding schemas remain strict, and docs only describe global/per-agent tool restrictions. The related PR #49593 is still open and explicitly only accepts the schema field while leaving runtime filtering for follow-up, so it is not a complete canonical replacement for this issue.

## Evidence

- **Route binding schema rejects extra fields:** `RouteBindingSchema` contains `type`, `agentId`, `comment`, `match`, and `session`, then calls `.strict()`. There is no `tools` field, so `tools.deny` would still be rejected by config validation on current main.
