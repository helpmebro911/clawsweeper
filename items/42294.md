## Summary

Keep open. Current main still preserves `audioAsVoice` in outbound routing, but the Telegram outbound adapter does not translate it into Telegram's `asVoice` send option, so the PR's bugfix intent is still relevant. The original patch shape is stale, though: current code moved to `extensions/telegram/*`, and the prior review's text-only regression concern still applies to the broad `sendPayload` routing condition.
