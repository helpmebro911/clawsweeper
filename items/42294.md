## Summary

Keep this PR open. Current main has moved Telegram outbound code into the Telegram plugin and includes adjacent routing/transport pieces, but the current payload adapter still drops `audioAsVoice` before calling `sendMessageTelegram`. The PR is stale against the new layout, yet the core fix idea remains useful.

## Best Possible Solution

Keep this PR open and ask for or perform a rebase onto the current Telegram plugin layout. The likely fix belongs in `extensions/telegram/src/outbound-adapter.ts`/`extensions/telegram/src/channel.ts` via `sendTelegramPayloadMessages`, mapping `payload.audioAsVoice === true` to Telegram `asVoice: true`, plus a focused regression in `extensions/telegram/src/outbound-adapter.test.ts`. Recheck the text-only routing review comment while rebasing and run the targeted Telegram outbound/delivery tests before merge.
