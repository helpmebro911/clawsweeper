## Summary

Keep this PR open. Current main has part of the plumbing, but the Telegram outbound `sendPayload` path still does not translate `audioAsVoice` into the `asVoice` option required for native Telegram voice-note sends. The PR is still functionally relevant, but it needs a rebase/adaptation to the current `extensions/telegram` layout and should address the existing text-only chunking review concern.

## Best Possible Solution

Keep this PR open for contributor or maintainer follow-up. The best fix is to rebase onto current main, implement the Telegram-specific mapping in `extensions/telegram/src/outbound-adapter.ts` or the shared `sendTelegramPayloadMessages` path so media sends receive `asVoice: true` when `payload.audioAsVoice` or context `audioAsVoice` is true, avoid changing text-only chunking behavior, and add focused regression coverage in `extensions/telegram/src/outbound-adapter.test.ts` plus the relevant outbound delivery test surface.
