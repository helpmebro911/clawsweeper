## Summary

Keep this PR open. Current main still prepends `acp` unconditionally in the ACP debug client, so the duplicated-argument bug described by the PR is still plausible and not implemented on main. The production change is narrow, but the added regression test in the PR does not exercise the changed helper path, matching the existing bot review concern.

## Best Possible Solution

Keep this PR open and ask for a focused test fix before merge: either expose a small internal `buildServerArgs` helper for direct unit coverage or test `createAcpClient` with a mocked spawn/connection path so `serverArgs: ['acp', '--verbose']` is proven to spawn `openclaw acp --verbose` exactly once. The production change itself is plausibly the right owner-module fix in `src/acp/client.ts`; it should not be closed as stale or obsolete while current main still has the duplicated-arg path.
