## Summary

Keep open. Current main still does not support outbound WhatsApp sticker sends: WebP media is routed through the generic image path, shared web media still optimizes non-GIF images by default, and WhatsApp message actions only advertise react/poll. The related implementation PR #7502 was closed unmerged, so it is not a canonical shipped fix.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin rather than closing the issue: add an explicit outbound sticker path, preferably a typed asSticker/sticker action contract, preserve valid WebP sticker media before JPEG optimization, route the send through Baileys { sticker: buffer }, and cover the send-api/media-loader behavior with focused tests and WhatsApp docs updates.
