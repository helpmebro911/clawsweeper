## Summary

Keep #7476 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still recognizes inbound WhatsApp stickers but has no outbound WhatsApp sticker send path: WebP/image media goes through the image payload, there is no `asSticker` option, and the WhatsApp action adapter does not support `action=sticker`. The prior implementation PR #7502 is closed unmerged, and the latest maintainer/Codex issue comment already kept this open.

## Best Possible Solution

Keep the issue open and implement this in the bundled WhatsApp plugin. Add a deliberate outbound sticker contract, preferably explicit `action=sticker` or `asSticker` rather than captionless WebP autodetection alone; preserve valid WebP sticker media before image optimization; route outbound sends through Baileys `{ sticker: buffer }` or equivalent `WAMediaUpload`; advertise and dispatch the WhatsApp sticker action or option consistently; and add focused send-api, media-loading, message-action, docs, and regression test coverage.
