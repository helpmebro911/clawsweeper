## Summary

Keep open. Current main supports multiple configured models, allowlists, and fallbacks after setup, but the onboarding wizard still only prompts for a single default model. The request is therefore not implemented on current `main`; it is only partially covered by later `configure` flows and existing model-switching commands.

## Evidence

- **Onboarding still prompts only for a default model:** `runSetupWizard()` calls `promptDefaultModel()` in both the `skip` path and the auth-selected path, then applies only the chosen primary model. There is no `promptModelAllowlist()` call in the onboarding flow.
