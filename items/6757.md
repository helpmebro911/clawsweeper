## Summary

Keep #6757 open. Current main still supports manual `/compact`, admin/control-plane `sessions.compact`, UI-triggered compaction, auto-compaction, and context-engine/plugin compaction hooks, but it does not expose the requested agent-triggered self-compaction action through `session_status`, the agent-facing `gateway` tool, or a dedicated active-session CLI command. The latest issue discussion explicitly kept this item open, and the directly related PR #43401 attempted `session_status(action="compact")` but was closed unmerged.

## Best Possible Solution

Keep #6757 open until OpenClaw implements a first-class in-session self-compaction API, such as `session_status(action="compact")` or an equivalent active-session agent tool. A correct implementation should use direct compaction when already inside a run to avoid lane deadlocks, preserve visibility/permission semantics, mark the action as mutating, update compaction counters/token snapshots, and add focused tests for `session_status`, mutation tracking, and active-run behavior. PR #43401 remains useful prior art but is not shipped.
