## Summary

Keep #6757 open. Current main still has manual `/compact`, admin-scoped `sessions.compact`, automatic/preemptive compaction, and context-engine/plugin compaction hooks, but it does not expose the requested agent-triggered self-compaction action through `session_status`, the agent-facing gateway tool, or a dedicated active-session CLI command. The latest issue discussion also explicitly kept this item open, and PR #43401 attempted the exact `session_status(action="compact")` shape but was closed unmerged.

## Best Possible Solution

Keep #6757 open until OpenClaw implements a first-class in-session self-compaction API, such as `session_status(action="compact")` or an equivalent agent/session tool using direct compaction for the active run, or maintainers explicitly decide agents must not compact their own active session. A correct implementation should preserve session visibility and permission semantics, mark the action as mutating, avoid active-run deadlocks, update compaction counters/token snapshots, and add focused tests for `session_status`, mutation tracking, and active-run behavior. PR #43401 remains useful prior art but is not shipped.
