## Summary

Keeping open. The requested agent-triggered self-compaction path is still not present on current main: `session_status` can report usage and set a model override, but it has no compact action; tool mutation detection only treats model overrides as `session_status` mutations; the agent-facing gateway tool still exposes a fixed set of gateway actions and not arbitrary `sessions.compact`; and the existing `sessions.compact` RPC remains an admin-scoped gateway method rather than an ordinary in-session agent tool. The related PR #43401 proposed the exact missing `session_status(action="compact")` behavior but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until maintainers either implement a first-class self-compaction path, most plausibly `session_status(action: "compact")` with focused mutation/test coverage, or make an explicit product decision that agents should not be allowed to compact their own active session. The existing admin `sessions.compact` RPC should not be treated as solving the in-session agent tool request.
