## Summary

Keep this issue open. Current main still has only a boolean fallback retry image strip, no provider-level image trust policy, and prompt-detected local images are still loaded without a fallback trust guard. There is also an open same-author PR (#57959) whose body says it closes this issue, so the issue should remain open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Keep this issue open while maintainers review, update, land, or close PR #57959. The best implementation path is a focused fallback-boundary change that threads primary/target provider and failure context through `runWithModelFallback` and `runAgentAttempt`, centralizes a tested image trust decision, suppresses prompt-detected image loading when the fallback target is not trusted, and documents any user-facing trust configuration if maintainers choose configurable behavior over conservative cross-provider stripping.
