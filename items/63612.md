## Summary

Current main still has the reported pre-provider crash path: OpenClaw delegates active-history token estimation to the pinned upstream estimator, and that estimator still has unchecked `.length` reads for malformed assistant/toolResult/custom/bash/summary shapes. PR #63636 remains open and unmerged with closing syntax for this work, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and have maintainers review, land, or supersede #63636. The durable fix should add a guarded OpenClaw-side token estimator or normalize malformed replay history before every compaction token-estimation call site, with regression tests for malformed assistant content, missing text/thinking/name fields, non-array toolResult/custom content, and missing bash/summary fields.
