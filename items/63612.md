## Summary

Keep this issue open. Current main still delegates the reported pre-prompt token-estimation path to the pinned upstream estimator that has unsafe `.length` reads, and the linked implementation PR remains open and unmerged with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and have maintainers review, land, or supersede PR #63636. The durable fix should harden OpenClaw’s local token-estimation path, or normalize malformed replay history before every pre-prompt/compaction estimation call, with regression tests for malformed assistant content, missing text/thinking/name fields, non-array or missing toolResult/custom content, and missing bash/summary fields.
