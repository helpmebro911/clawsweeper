## Summary

Keep #71142 open. Current main has Control UI image upload support, but the upload limit remains a hardcoded 5,000,000-byte Gateway chat attachment limit, and the `gateway.controlUi` config schema/docs expose no upload-size setting. Related prior cleanup confirms image upload itself is implemented, but this newer request is specifically about configurability and remains actionable in core.

## Best Possible Solution

Keep the issue open and implement this in core, likely as a documented `gateway.controlUi` upload limit that is validated in the config schema, surfaced in generated docs/UI metadata, and used by the `chat.send` attachment path instead of the current hardcoded `5_000_000`. Add focused tests for the default, configured larger image uploads, oversize rejection, and interaction with the WebSocket payload cap.

## Risks / Open Questions

- Raising Control UI upload limits needs to account for base64 expansion and the Gateway WebSocket `MAX_PAYLOAD_BYTES` envelope, currently documented as 25 MiB.
- The issue proposes audio/video/document limits too, but current Control UI composer only accepts `image/*`; maintainers should decide whether to scope the first fix to images or broaden the upload surface.
