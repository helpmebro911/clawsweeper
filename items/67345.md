## Summary

Keep this PR open. Current main still does not contain the broad claw-code parity bridge work, this PR is paired with open same-author PR #67333 for part of the stack, and the branch includes security-sensitive tool surfaces that need maintainer review before merge or cleanup.

## Best Possible Solution

Keep this PR open for maintainer product and security review, but ask for a split or restack. A practical path is separate PRs for runner/tool-call normalization, MCP resources/auth support, LSP dispatch extensions, sandbox/parity-harness work with #67333, and any Hermes/task/team surfaces maintainers actually want in core. Compatibility utility tools should either move to plugin/ClawHub surfaces or land only with SSRF-guarded fetch, redacted or allowlisted config reads, owner-only persisted mutations, bounded waits, and schema patterns that match repo guardrails.
