## Summary

Keep open. Current main does not implement the PR's claw-code parity alias suite, compatibility tools, unified MCP/LSP wrapper tools, Hermes bridge, task/team tools, parity harness, or sandbox/fs hardening changes. The PR is also broad, paired with another open same-author PR, and has unresolved security and correctness review findings, so it is not a conservative cleanup close.

## Best Possible Solution

Keep the PR open for maintainer review, but request a split or restack. A practical path is separate focused PRs for runner/tool-call normalization gaps, MCP/LSP bridge extensions, task/team or Hermes surfaces if maintainers want them in core, and optional compatibility utilities as plugin or ClawHub work where current APIs are sufficient. Config, network, and control-plane tools should require explicit owner authorization, safe defaults, and a dedicated security review before any core merge path.
