## Summary

Keep open. Current main still runs `before_tool_call` before the browser tool executes and passes only the tool-call params supplied by the model/user; the browser `act` schema has no `_currentUrl`/`currentUrl` field, and the current URL is only added to `/act` responses after the browser route resolves the tab. The issue comment’s openclaw-credential-vault v1.0.0-beta.5 cache workaround is valid and useful, but it does not obsolete the requested native hook/API improvement for domain-aware security checks.
