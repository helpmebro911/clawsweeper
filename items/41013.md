## Summary

Keep open. Current main still does not strip leading `<session-recap>` blocks from the shared user-message cleanup path, so this PR is not obsolete or implemented. The PR is a focused implementation candidate for the linked UI/TUI leak, but it needs revision before landing because the submitted helper can erase malformed user text and can mutate whitespace when no recap block is stripped.

## Best Possible Solution

Keep the PR open as a focused candidate, but require a small revision before landing: add session-recap support in `src/auto-reply/reply/strip-inbound-meta.ts`, wire it into both full and leading-only stripping paths, only remove well-formed leading blocks with a matching close tag, preserve the original string when no recap block is stripped, and add regression tests for well-formed, malformed/no-close, mid-message, leading blank line, and whitespace-preservation cases. Once revised and reviewed, it can close the linked UI/TUI leak report.
