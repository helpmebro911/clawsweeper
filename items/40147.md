## Summary

Keep this PR open. Current main still forces Feishu thread replies from inbound `thread_id` or `root_id` even when `replyInThread` is disabled, so the submitted behavior is not implemented. The related #39507 is overlapping but proposes different semantics, so this needs maintainer product judgment rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or consolidate it with #39507 only after deciding Feishu root/thread semantics. A viable current-main fix should update `extensions/feishu/src/bot-content.ts` and `extensions/feishu/src/reply-dispatcher.ts` together, scope `root_id` hard-forcing to intended group surfaces, preserve or intentionally change p2p quote behavior, update the thread-id-only disabled test expectation, and add regression coverage for normal groups, topic groups/topic-sender routing, and p2p quote replies.
