## Summary

Keep this PR open. Checked-out main still forces Feishu thread replies from inbound thread_id/root_id even when replyInThread is disabled, and the current tests still assert that behavior. The overlapping #39507 is useful related work, but it is not an exact duplicate because it treats disabled replyInThread as authoritative even for root_id, while this PR intentionally makes root_id force thread replies.

## Best Possible Solution

Keep this PR open for maintainer review or explicitly consolidate it with #39507 after deciding Feishu root_id semantics. A current-main fix should update extensions/feishu/src/bot-content.ts and extensions/feishu/src/reply-dispatcher.ts together, add/replace regression tests for thread_id-only disabled groups, root_id normal groups, topic-group/topic-sender routing, and p2p quote replies, and avoid broadening thread delivery outside the intended group routing surface.
