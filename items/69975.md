## Summary

Keep open. Current main still has the misleading cron-only `--tz` help in both `cron add` and `cron edit`, so the PR is not implemented or obsolete. The PR is useful as the active clean respin for #59456, but it should not be closed or merged as-is because its proposed `default: UTC` wording is wrong for cron schedules, which default to the host/gateway timezone when `--tz` is omitted.

## Best Possible Solution

Keep #69975 open and request a narrow wording update. The final help should mention that `--tz` applies to `--cron` and offset-less ISO `--at` datetime strings, while separating the defaults: cron schedules without `--tz` use the host/gateway timezone, and offset-less one-shot `--at` values are UTC unless `--tz` is supplied. After that, the PR can close #59456 and the older closed duplicate PRs can remain superseded.
