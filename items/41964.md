## Summary

Keep open. Current main still has the opposite behavior from this PR: non-Control-UI-visible runs are gated out of chat/node-session fanout, and tests assert that suppression. The reported TUI live-display gap remains plausible, but the PR's one-line guard removal is security-sensitive and needs maintainer design review rather than stale closure.

## Best Possible Solution

Keep the PR open for maintainer review, but request a revised design before merge. A safer implementation should preserve `isControlUiVisible` for general Control UI and node-session fanout while adding an authorized TUI/operator live-view path, likely by teaching the TUI to subscribe to and render the existing `session.message` transcript surface. Coordinate with #38829 and #45417 so inbound user transcript updates and assistant streaming are handled consistently.
