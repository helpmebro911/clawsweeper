## Summary

Keep open. Current main still has useful partial hardening for selected external content and channel metadata, but it does not implement the central request: generic trust-boundary wrapping for non-owner current user messages and open-world tool results at the shared agent/provider boundary, plus a stable system-prompt trust anchor for those wrappers. The linked implementation PR #62973 was closed unmerged.

## Best Possible Solution

Keep this issue open and implement a core-owned, tested boundary strategy: add stable system-prompt guidance; wrap or otherwise label non-owner current messages at prompt assembly time; classify/wrap open-world tool results in the shared transport or tool-result persistence path; and add regression coverage for owner vs non-owner messages, web_fetch/web_search/x_search, external MCP results, and existing external-content marker sanitization.
