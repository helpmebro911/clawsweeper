## Summary

Keep open. Current main still has partial prompt-injection hardening for web/file/media external content, inbound metadata, special-token sanitization, and owner-only tool policy, but it does not implement the central request: a shared trust-boundary contract for non-owner current message bodies and generic/open-world tool results, with stable system-prompt guidance. The linked implementation PR #62973 is closed unmerged, and the broader CaMeL PR #39231 is related but not a shipped replacement for this narrower boundary request.

## Best Possible Solution

Keep this issue open and implement a core-owned, tested trust-boundary strategy: add stable pre-cache system-prompt guidance, wrap or otherwise label non-owner current message bodies during prompt assembly, classify/wrap open-world tool results at a shared transport or tool-result boundary, preserve existing external-content sanitizer behavior, and add provider-shape regression coverage. Treat #62973 as a closed unmerged attempt and #39231 as related broader CaMeL work, not as a shipped replacement for this issue.
