## Summary

Keep this PR open. Current main still lets Discord startup continue when fetchUser("@me") rejects or yields no usable bot id, and this PR remains a focused candidate for the linked security bug, even though the submitted patch is stale and incomplete.

## Best Possible Solution

Keep this PR open, ask for a rebase onto current main, and revise the fix in extensions/discord/src/monitor/provider.startup.ts so Discord startup fails when fetchUser("@me") rejects or returns no normalized bot id. Add targeted regression tests for rejection and invalid identity responses, then consider tightening extensions/discord/src/monitor/message-handler.preflight.ts to trust mentionDecision.shouldSkip without the extra botId guard for #42219.
