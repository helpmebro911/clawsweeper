## Summary

Current main still lacks an abandoned-stream cleanup path in the shared guarded provider response wrapper. Anthropic Messages still streams through that wrapper, the guarded fetch release hook is still the dispatcher/socket cleanup path, and the relevant tests still do not cover a response body abandoned before EOF without explicit cancel.

## Best Possible Solution

Keep this issue open. Patch `src/agents/provider-transport-fetch.ts` so a managed streamed response reliably invokes `release()` when abandoned before EOF without explicit `cancel()`, then add focused regression coverage in `src/agents/provider-transport-fetch.test.ts`. During the fix, audit Anthropic Messages and Anthropic Vertex/pi-ai streaming consumers so they either drain/cancel explicitly or are protected by the shared transport safety net.

## Evidence
