## Summary

Current main still contains the reported provider transport lifecycle gap: streamed guarded fetch responses release the undici dispatcher only on no body, EOF/read error, or explicit cancel, with no abandoned-stream safety net. The supplied macOS report, Linux confirmation, and linked external FinalizationRegistry prior art are credible, and the affected Anthropic/Vertex streaming paths still route through this surface.

## Best Possible Solution

Keep this issue open. Patch the shared provider transport lifecycle in `src/agents/provider-transport-fetch.ts` so the guarded fetch `release()` runs even when a streamed `Response` is abandoned before EOF and without explicit `cancel()`. Add focused regression coverage in `src/agents/provider-transport-fetch.test.ts` for abandoned streamed responses, and audit the Anthropic Messages and Anthropic Vertex streaming consumers so they either drain/cancel explicitly or are protected by a robust wrapper-level safety net.
