## Summary

Keep this PR open. It is paired with the still-open same-author bug #63069, current main still hardcodes shell completion profile paths, and the fetched PR diff does not yet wire its new resolver into the runtime install/check path that actually writes and detects completion.

## Best Possible Solution

Keep this PR open and request changes rather than closing it. The maintainer path is to move or expose a shared env-aware profile resolver in src/cli/completion-runtime.ts, use it consistently from installCompletion, isCompletionInstalled, usesSlowDynamicCompletion, doctor/setup/update-facing hints, preserve the intended bash fallback behavior, and add focused runtime tests for zsh ZDOTDIR, fish XDG_CONFIG_HOME, bash .bashrc/.bash_profile selection, and non-Windows PowerShell XDG_CONFIG_HOME if kept in scope. Then review and land it as the fix for #63069, or close both only after an equivalent implementation ships on main.
