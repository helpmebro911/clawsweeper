## Summary

Keep #24118 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still does not expose `channels.signal.linkPreview` in Signal config types, strict schema, generated metadata, or Signal docs, and the bundled Signal send path still only passes message/text-style/account/attachments/target fields to signal-cli JSON-RPC. Upstream signal-cli has link-preview controls, and the provided maintainer/Codex issue comment already distinguished closed #29466 as unrelated Webchat preview-card work, so this remains a concrete bundled Signal plugin enhancement rather than a closeable duplicate or ClawHub item.

## Best Possible Solution

Keep #24118 open until the bundled Signal plugin adds a backward-compatible `channels.signal.linkPreview` surface across top-level and per-account config types, strict schema, generated metadata, and docs; resolves account inheritance; threads the setting through formatted/direct text and media sends; passes supported preview metadata to signal-cli JSON-RPC; and adds focused schema and send-path tests. If maintainers decide Signal link previews are out of scope, record that explicit product decision instead of auto-closing this actionable bundled plugin enhancement.
