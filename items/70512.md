## Summary

Keep #70512 open. Current main still routes Feishu/Lark DMs by sender only and has group-only topic session isolation. The related #10242 report covers restoring Feishu DM thread/new-thread sending behavior, but it does not add inbound DM topic/thread-based session keys.

## Best Possible Solution

Keep this open and implement it as a bundled Feishu plugin feature: add an opt-in DM topic/thread session scope with a default that preserves current per-user routing, derive a stable direct conversation/session id from the Feishu p2p topic/thread fields, and update Feishu config schema, docs, routing/ACP binding behavior, and regression tests for both ordinary DM topics and group topic parity.

## Evidence

- **DM routing ignores topic/thread fields:** Feishu inbound handling resolves group sessions only when `isGroup`; for DMs, `peerId` is `ctx.senderOpenId`, and that direct peer is passed to `resolveAgentRoute`. This means any DM `root_id`/`thread_id` on the event does not affect the routed session.
