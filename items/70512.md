## Summary

Keep #70512 open. Current main still routes Feishu/Lark direct messages by sender open_id only, even when the inbound event has root_id/thread_id metadata, and the Feishu config/docs only expose topic-scoped session isolation for groups. The related #10242 context covers Feishu DM/new-thread send behavior, but not inbound DM topic-based session isolation.

## Best Possible Solution

Implement this inside the bundled Feishu plugin rather than core routing: add an explicit Feishu DM topic/session-scope config, derive direct-message session peers from sender plus root_id/thread_id when enabled, preserve existing per-user default behavior, update config metadata/docs, and add focused Feishu tests for p2p/private events with and without topic metadata.
