## Summary

Keep #70512 open. Current main still routes Feishu direct messages by sender id only; Feishu `root_id`/`thread_id` fields are parsed, but the topic-aware session-key logic is only implemented for group/topic-group routing. The related #10242 review is useful context for Feishu DM thread creation/reply support, but it does not implement per-topic DM session isolation.

## Best Possible Solution

Keep the issue open and implement this in the bundled Feishu plugin as an opt-in, backward-compatible direct-message routing mode. A conservative path is to add a Feishu DM session-scope config with `per-user` as the default and `per-topic` as opt-in, build direct peer/conversation ids from sender plus stable topic/root id when present, preserve existing per-sender sessions when no topic id exists, and add focused tests for two Feishu DM topics from the same sender routing to distinct `SessionKey` values plus docs/config metadata for the new option.
