## Summary

Keep this issue open. Current main still parses Feishu/Lark DM root/thread metadata but routes direct messages by sender Open ID only, while topic session IDs, parent peers, thread bootstrap, and MessageThreadId remain gated to group/topic-group sessions. The config schema and public Feishu docs still expose group topic routing but no opt-in DM per-topic session-scope option.

## Best Possible Solution

Keep this issue open. Implement the feature inside the bundled Feishu plugin with an explicit opt-in DM topic/session-scope config, preserve the current sender-scoped default, derive direct-message conversation IDs from sender plus canonical topic/root/thread metadata, propagate that conversation ID through routing, configured/runtime bindings, replies, and thread context, then add focused DM topic tests plus Feishu config schema and docs updates.

## Risks / Open Questions

- Feishu/Lark direct-message topic event semantics still need verification before choosing canonical precedence among root_id, thread_id, parent_id, and message_id fallbacks.
- Changing DM conversation IDs by default would break existing sender-scoped sessions, bindings, dynamic-agent creation, and ACP workflows, so the feature should remain opt-in.
- Existing ACP conversation binding support for Feishu DMs is only a partial workaround for bound conversations; it does not provide general per-topic DM session isolation for normal inbound routing.
