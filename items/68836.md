## Summary

Keep this PR open. Current main still has the observability gap the PR targets: text Telegram sends emit success logs, but media sends in the auto-reply delivery path do not. The PR is a focused implementation candidate for the linked Telegram media logging issue, though it should probably gain regression assertions and be checked against the direct send path before landing.

## Best Possible Solution

Keep this PR open as a plausible focused fix for the Telegram media success-log gap. Best maintainer path: review the patch, add focused delivery tests that assert the six media success log lines, and decide whether the direct sendMessageTelegram media path should receive the same observability before landing or in a linked follow-up for #68770.
