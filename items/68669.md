## Summary

Keep open. Current main still has the duplicated cleanup-wrapper path this PR targets: embedded subagent completion can reach `completeSubagentRun(..., triggerCleanup: true)` through both the gateway wait path and the in-process lifecycle listener, and `completeSubagentRun` still calls `cleanupBrowserSessionsForLifecycleEnd` without a per-run dispatch guard. The PR is a focused bug fix with a regression test and an explicit maintainer comment saying it is correct and worth landing.

## Best Possible Solution

Keep PR #68669 open for maintainer landing. The best path is to merge the narrow dispatch-flag fix, including the `SubagentRunRecord.browserCleanupDispatchedAt` field, synchronous guard in `completeSubagentRun`, rearm reset in `subagent-registry-run-manager.ts`, and the parallel-completion regression test, after normal maintainer CI/rebase checks.
