## Summary

Current main still returns and tests the raw `Usage: /btw <side question>` producer text when the `/btw` side question is missing. The issue remains a concrete core auto-reply cleanup item, not a close candidate.

## Best Possible Solution

Keep this issue open. Update the producer-owned usage text in `src/auto-reply/reply/commands-btw.ts` to render the full command usage as inline code or escape the placeholder, update `src/auto-reply/reply/commands-btw.test.ts`, and audit nearby auto-reply usage strings with raw angle-bracket placeholders for the same markdown/HTML rendering issue.
