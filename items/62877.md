## Summary

Current main still returns the raw `Usage: /btw <side question>` text for a missing `/btw` question, and the unit test still asserts that exact raw output. The issue is recent, concrete, and remains actionable in core producer code rather than being a close candidate.

## Best Possible Solution

Keep this issue open. Fix the producer-owned usage text in `src/auto-reply/reply/commands-btw.ts` by rendering the command usage as inline code or escaping the placeholder, update `src/auto-reply/reply/commands-btw.test.ts`, and audit nearby auto-reply usage strings with raw angle-bracket placeholders so markdown/HTML-rendering channels receive unambiguous text.
