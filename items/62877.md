## Best Possible Solution

Keep this issue open. Fix the producer-owned usage text in `src/auto-reply/reply/commands-btw.ts`, preferably as `Usage: `/btw <side question>`` or an escaped placeholder, update `src/auto-reply/reply/commands-btw.test.ts`, and audit nearby auto-reply usage strings with angle-bracket placeholders for the same markdown-channel rendering problem.
