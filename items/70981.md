## Summary

Keep open. Current main still has the reported gateway/UI mismatch: `buildGatewaySessionRow` receives `cfg` but populates `GatewaySessionRow.thinkingDefault` with the model-only `resolveThinkingDefaultForModel`, while the UI renders `Default (...)` from that row field and the runtime/config resolver already reads configured thinking defaults. The related PR #71033 was closed unmerged as a duplicate of this landing path in the provided GitHub context, so #70981 is not superseded by a merged fix.

## Best Possible Solution

Keep PR #70981 open as the current landing path for #70967. The best code direction is to switch `buildGatewaySessionRow` to the config-aware thinking default resolver and keep the regression test; during maintainer review, decide whether to also cover per-agent `agents.list[].thinkingDefault` and `getSessionDefaults` parity so the row/default contract fully matches the docs.
