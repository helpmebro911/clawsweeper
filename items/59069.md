## Summary

Keep open. Current `main` already normalizes Telegram `:topic:` targets when the Telegram channel plugin is loaded, but this PR's fallback is not implemented on `main`, and the remaining delta appears to target an unloaded-plugin/test-double path rather than a clearly obsolete bug. The existing review comment also identifies a plausible precedence bug in the proposed patch, so there is not strong evidence for a conservative close.

## Risks / Open Questions

- If cron delivery can run before the Telegram plugin is loaded in some real runtime path, the current main code may still leave `to` unnormalized and lose the explicit topic `threadId` in that narrow case.
- Closing this PR now would discard a potentially valid regression fix without proving the unloaded-plugin path is impossible on current main.
- Merging the PR as-is would likely need a small follow-up for the review concern that explicit `:topic:` should probably win over session-derived `threadId` when they disagree.
