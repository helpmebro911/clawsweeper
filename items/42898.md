## Summary

Current main still appears to misclassify dotted read-only gateway actions such as `config.schema.lookup` as mutating, so this PR remains a focused implementation candidate rather than cleanup to close. The PR also already addressed the main review concern by scoping the dotted leaf fallback to `gateway` only and keeping `cron`/`canvas` fail-closed.

## Best Possible Solution

Keep this PR open and review or land the gateway-only leaf-verb mutation-classification fix, with regression coverage for `config.schema.lookup`, other read-only dotted gateway actions, mutating writes like `config.set`/`config.patch`, missing actions, and unsupported cron/canvas dotted actions. After that merges, close #42856 as implemented and treat payload-only suppression work as secondary unless it covers a separate severity policy.
