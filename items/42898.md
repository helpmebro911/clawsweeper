## Summary

Current main still has the reported classifier bug: the gateway/cron/canvas branch checks the full dotted action string against a flat read-only verb set, so `config.schema.lookup` is still treated as mutating even though the gateway declares it read-scope and exposes it as a runtime RPC. This PR remains a focused implementation candidate; the related payload-level PR #42962 does not supersede the classifier fix.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best path is a gateway-only classifier fix, ideally anchored to known read-scope gateway methods or otherwise tightly restricted, with regression coverage for `config.schema.lookup`, `config.get`, `agents.files.list`, mutating writes like `config.patch`/`config.apply`, missing actions, and unsupported cron/canvas dotted actions. After the classifier fix lands, #42856 can close as implemented; #42962 should only proceed if maintainers still want an additional payload-specific severity policy.
