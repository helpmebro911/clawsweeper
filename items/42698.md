## Summary

Keep this PR open. Current main moved Bonjour/Ciao handling into the bundled Bonjour plugin and has useful suppression for several Ciao cancellation/assertion paths, but it still does not match the exact upstream `IPv4 address changed from undefined to defined` assertion reported by this PR. The submitted branch is stale against current main because it targets removed `src/infra/bonjour-ciao.*` files, so the useful fix should be ported rather than merged as-is.

## Best Possible Solution

Keep this PR open as useful bug-fix evidence, but ask for a rebase or maintainer port: update `extensions/bonjour/src/ciao.ts` so the interface assertion classifier matches both upstream Ciao wordings, add focused coverage in `extensions/bonjour/src/ciao.test.ts` and advertiser handler coverage if needed, then run the Bonjour/plugin test lane. Treat the old `src/infra` patch as the reference implementation, not a directly mergeable diff.
