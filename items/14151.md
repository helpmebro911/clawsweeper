## Summary

Current main at cf834e2a21eac7fd218989fb5b85e7e8dd4c3246 still does not implement a configurable force-prepend <think> option. OpenClaw has reasoning visibility, provider thinking defaults, native/tagged reasoning parsing, final-tag enforcement, and provider-specific request params, but no typed config key or runner/subscription path that synthesizes a missing opening <think> tag before streaming/non-streaming parsing. Related PR #14323 attempted this feature but was closed unmerged in the provided GitHub context, and the prior Codex review comment correctly identified the remaining gaps.

## Best Possible Solution

Keep #14151 open. A complete fix should add a typed global and/or per-model config option, validate and document it, resolve it through model/agent config, thread it through RunEmbeddedPiAgentParams and subscribeEmbeddedPiSession, inject before both streaming and final-message reasoning parsing, guard against double-prepending/native reasoning events, and add focused tests for streaming, non-streaming, existing <think> tags, native thinking events, and enforceFinalTag interactions.
