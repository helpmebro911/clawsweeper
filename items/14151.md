## Summary

Keep #14151 open. Current main still has no configurable force-prepend think-tag option, and the current reasoning pipeline only parses/strips existing tagged or native reasoning. The directly related PR #14323 was closed unmerged, and the only relevant change since the prior Codex keep-open review is unrelated WhatsApp documentation/runtime work.

## Best Possible Solution

Keep #14151 open. A complete fix should add a typed global and/or per-model config option, validate and document it, resolve it through agent/model config, thread it through RunEmbeddedPiAgentParams and SubscribeEmbeddedPiSessionParams, inject exactly once before streaming and final-message reasoning parsing, guard existing tags and native thinking events, and add focused tests for streaming, non-streaming, existing <think> tags, native thinking, and enforceFinalTag interactions.
