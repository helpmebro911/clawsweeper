## Summary

Keep open. Current main still reports the root CLI version commit through resolveCommitHash, and that resolver still prefers live git metadata before dist/build-info.json with no injected build-time commit source. The provided PR directly targets that remaining behavior with a narrow resolver/build-config change and regression tests, and I did not find a superseding implementation on main.

## Best Possible Solution

Keep this PR open for maintainer review, rebase if needed, and land it or an equivalent implementation. The best code path is to bake a sanitized build-time commit into node build outputs, consult it after explicit GIT_COMMIT/GIT_SHA env overrides and before live git, and keep regression coverage for injected priority, fallback behavior, env override, and cache stability.
