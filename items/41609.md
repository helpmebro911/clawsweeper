## Summary

Keep open. Current main has a narrow Feishu interactive-card parser for fetched/read messages and quoted-card context, but the inbound `im.message.receive_v1` event path still does not parse `message_type: "interactive"` card content into the agent-visible message body. The merged maintainer repair explicitly says it did not close the broader inbound card parsing work, and open PR #56795 is still the live implementation candidate for that remaining path.

## Best Possible Solution

Keep this issue open and review, land, or replace #56795 with a focused Feishu plugin change that parses inbound `message_type: "interactive"` events, fetches or preserves raw card JSON when needed, exposes readable card text to the agent, and adds tests for the direct inbound event path plus fetched/read card fallback behavior. After that lands on main, close this issue as implemented.
