## Summary

Keep open. Current main still has the reported Discord DM path: `resolveDiscordAutoThreadReplyPlan` creates a channel-shaped reply target, DMs do not get an auto-thread context, and `processDiscordMessage` assigns `ctx.To` from that channel-shaped reply target before computing the `user:<id>` DM conversation target. The related fix PR was closed unmerged, so this issue still tracks real remaining work.

## Best Possible Solution

Land a narrow Discord message-handler fix that computes the DM conversation target before `effectiveTo` and uses it for direct-message `ctx.To` while leaving `deliverTarget` channel-based for Discord API delivery. Add a regression test that exposes and asserts `ctx.To`, `ctx.From`, and `ctx.ChatType` for inbound Discord DMs, and then decide separately whether the component-interaction DM surface should keep channel-shaped `To` or follow the same user-shaped semantic.
