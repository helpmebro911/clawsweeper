## Summary

Keep open. Current main has meaningful partial UX work, including Quick Settings, schema-backed config editing, a Control UI ClawHub skills panel, and CLI/native ClawHub plugin installs. It still does not satisfy the central request for a broader WordPress-like guided dashboard experience with dashboard-native ClawHub plugin discovery, installation, and configuration.

## Best Possible Solution

Keep this open as a product/UI umbrella or split it into narrower issues for guided Control UI configuration flows, contextual setup help, dashboard-native ClawHub plugin discovery/install/configuration, and any additional gateway/plugin API needed to expose those flows safely.

## Evidence
