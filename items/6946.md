## Summary

Keep open. Current main still lacks the requested configurable Telegram standalone processing-indicator message lifecycle. The code has adjacent mechanisms, including ack/status reactions, live preview tool progress, and native-command progress placeholders, but not a normal inbound Telegram placeholder that is sent immediately, tracked, deleted on final reply, and edited on failure. The latest maintainer/Codex issue comment also explicitly kept #6946 open, and the related implementation PRs #43021 and #43720 were closed unmerged.

## Best Possible Solution

Keep #6946 open as the canonical Telegram feature request. The best implementation path is an opt-in bundled Telegram plugin change: add typed config/schema/generated metadata/docs, send a standalone placeholder after an eligible inbound Telegram message is accepted, carry its message id through dispatch, delete it after successful final delivery, and edit or replace it on dispatch failure, with focused Telegram dispatch tests for success, error, preview streaming, group/topic, and cleanup behavior.
