## Summary

Keep #6946 open. Current main still lacks the requested configurable Telegram standalone processing-indicator message lifecycle: no accepted `processingIndicator`/`deleteOnReply`/failure-edit config, no inbound dispatch placeholder message that is sent immediately, tracked, deleted after final reply, or edited on failure. Current Telegram feedback remains adjacent mechanisms: ack/status reactions, preview-stream tool progress, and heartbeat typing support. The related implementation PRs #43021 and #43720 were closed unmerged, and the latest provided issue discussion already kept this open.

## Best Possible Solution

Keep #6946 open as the canonical Telegram feature request. The likely best implementation is an opt-in bundled Telegram plugin change: add typed config/schema/generated metadata/docs, send a standalone placeholder after an eligible inbound Telegram message is accepted, carry its message id through the Telegram message context/dispatch path, delete it after successful final delivery, and edit or replace it on dispatch failure. Add focused Telegram dispatch/config tests for success, error, preview-streaming interaction, group/topic handling, superseded turns, and cleanup failures.
