## Summary

Current main still advertises Nextcloud Talk reactions but does not wire the existing reaction sender into the shared message action dispatch path. PR #72348 directly tracks the right fix and its head ref still exists, but it is not on this checkout's main, so the issue should remain open until that PR or an equivalent implementation lands.

## Best Possible Solution

Keep #70110 open and review or land PR #72348, or an equivalent fix that adds a Nextcloud Talk `ChannelMessageActionAdapter`, registers it on the channel plugin, dispatches `react` to `sendReactionNextcloudTalk`, and keeps focused regression coverage for both adapter wiring and reaction request shape.
