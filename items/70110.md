## Summary

Current main still appears to have the reported Nextcloud Talk reaction gap. The low-level `sendReactionNextcloudTalk` helper exists, and docs/capabilities say reactions are supported, but the Nextcloud Talk plugin does not expose a shared `message` tool action adapter or handler for `react`, so normal agent/message-action dispatch has no production path to call the helper.

## Best Possible Solution

Keep #70110 open. The likely fix belongs in the bundled Nextcloud Talk plugin: add a `ChannelMessageActionAdapter` or equivalent message-tool API that advertises `react`, validates room/message/emoji params, calls `sendReactionNextcloudTalk` with the resolved cfg/account, and adds regression coverage for shared `message` tool discovery and dispatch.
