## Summary

Keep open. The issue is a clear, recent feature request to restore direct CLI shell execution on paired nodes, and it is not eligible for stale closure. Current user-facing docs still describe `exec host=node` as the replacement for `openclaw nodes run`, so this needs maintainer/product judgment rather than conservative cleanup.

## Evidence

- **issue_is_recent_actionable_feature_request:** Issue #59337 was opened on 2026-04-02, has only the `enhancement` label, and asks specifically to restore or add a CLI path for executing shell commands on paired nodes without starting an agent/session.
- **current_docs_describe_exec_host_node_as_replacement:** The current Nodes CLI docs say `system.run` and `system.run.prepare` are blocked through `nodes invoke` and instruct users to use the `exec` tool with `host=node` instead of `openclaw nodes run`. That is the behavior the reporter finds inconvenient, not evidence that the request is already resolved.
  - file: [docs/cli/nodes.md](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/cli/nodes.md)
- **current_cli_tree_does_not_show_nodes_run:** The current CLI reference command tree lists `nodes status`, `describe`, `list`, `pending`, `approve`, `reject`, `rename`, `invoke`, `notify`, `push`, and capability subcommands, but does not list `nodes run`.
  - file: [docs/cli/index.md](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/cli/index.md)
