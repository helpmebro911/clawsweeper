## Summary

Keep open. Current main has the clearer startup guard and docs for `hooks.enabled` requiring `hooks.token`, but the requested schema/config-write dependency validation is still missing from `config.patch`, `config.set`, `writeConfigFile`, and `config.schema.lookup`.

## Best Possible Solution

Keep this issue open. Add a shared cross-field config validator that rejects `hooks.enabled=true` unless `hooks.token` is a non-empty string, and run it through the validation path used by `config.patch`, `config.set`, and `writeConfigFile`. If `config.schema.lookup` is meant to guide UI/custom clients, add explicit dependency or warning metadata there too. Keep `resolveHooksConfig` as the last-resort startup/reload guard and add focused regression tests around the write/lookup path.
