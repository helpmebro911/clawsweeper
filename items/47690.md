## Summary

Keep open. Current main does not implement the requested Telegram group cross-account forwarding/broadcast behavior. Routing still resolves a single agent route for one channel/account input, Telegram account config has no `forwardToAccounts`/`broadcastToAccounts`-style field, and the Telegram code explicitly treats bot-to-bot communication as supported via channel posts, not group messages. The related context also points to still-open follow-up work for Telegram allow-bot and multi-account routing behavior, so this is not cleanly implemented or safe to auto-close.

## Evidence

- **Single-destination route resolver:** `resolveAgentRoute` normalizes one `accountId`, loads bindings for that channel/account, returns the first matched binding, or falls back to one default agent. There is no multi-destination forwarding/broadcast result.
