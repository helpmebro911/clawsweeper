## Summary

Keep open. Current main has related hardening, but #8719 is a broad native security-profile proposal and remains materially unimplemented: filesystem policy is still only `workspaceOnly`, unrestricted host FS behavior is still supported when that flag is false/unset, network/data-classification gaps are still documented as residual risk, and audit/plugin/sandbox/session controls are partial rather than the requested data-centric enforcement model. The issue also has an existing maintainer/Codex review comment saying to keep it open, plus related open PathGuard work that covers only a subset of the request.

## Best Possible Solution

Keep #8719 open as a product/security design umbrella until maintainers either implement a native security profile or split it into canonical narrower issues. The likely path is to track filesystem PathGuard work separately (#39672/#52621/#60981), acknowledge ClawMoat as complementary external prior art, and define native follow-ups for data classification, network egress policy, sensitive-action audit logging, TTL elevation, session isolation, and plugin/skill verification.
