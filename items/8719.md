## Summary

Keep #8719 open. Current main still has narrower hardening primitives, especially optional `tools.fs.workspaceOnly`, sandbox bind validation, exec approvals, and plugin trust audit warnings, but it does not implement the requested native data-centric, secure-by-default Security Profile v1.1. The latest discussion also includes an explicit steipete keep-open review and useful external prior art via ClawMoat; the open PathGuard work is filesystem-only and does not supersede this broader product/security umbrella.

## Best Possible Solution

Keep #8719 open as the native product/security umbrella until maintainers either define OpenClaw Security Profile v1.1 or split it into canonical narrower issues. Track filesystem sandboxing through #60981/#7722, use ClawMoat as useful external prior art, and create or keep follow-ups for data classification/security levels, protected OS presets, cross-tool filesystem/exec/export/network policy, sensitive-action audit logs, approval/TTL elevation, session isolation, self/config integrity, and plugin/skill verification.
