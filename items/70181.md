## Summary

Keep open. Current `main` still appears to have the reported gap: bundled CLI metadata loads prefer `extensions/memory-core/cli-metadata.ts`, but that entry only registers the `memory` CLI and does not register the memory capability's `publicArtifacts`. `wiki bridge import` still depends on `listActiveMemoryPublicArtifacts(...)`, and existing bridge tests confirm that a zero-artifact result prunes previously imported bridge pages. That matches the issue report's destructive behavior. The linked fix PR is still only referenced from the issue timeline, not present in the checked-out `main`.
