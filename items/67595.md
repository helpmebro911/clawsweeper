## Summary

Keep this open. Current main appears to have the LaunchAgent managed-service-env/token embedding fix, and a maintainer explicitly confirmed that partial fix, but the separate `openclaw channels list` failure remains present: usage fetching is still enabled by default, human output still prints raw provider fetch errors, and JSON output still awaits provider usage without the same non-fatal catch.

## Best Possible Solution

Keep this issue open, or split it only after creating a focused follow-up for the remaining `openclaw channels list` behavior. The remaining fix should make provider usage enrichment optional and non-fatal for both human and JSON output, avoid raw `TypeError: fetch failed` output for a simple channel/auth listing, and add regression coverage for no configured chat channels plus auth profiles with a failing usage provider.
