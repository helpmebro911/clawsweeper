## Summary

Keep open. Current main includes the narrower #70752 stale-loaded-service refresh, but the broader macOS plaintext LaunchAgent token projection and `channels list` usage-fetch failure path are still not clearly resolved.

## Best Possible Solution

Keep this open, or split it into two focused issues. For the gateway side, make macOS service install/doctor converge on a LaunchAgent definition that never inlines `OPENCLAW_GATEWAY_TOKEN` while preserving the systemd EnvironmentFile behavior and updating audit hints. For `channels list`, make empty chat-channel configurations succeed cleanly by skipping usage when appropriate or converting provider usage fetch failures into consistent non-fatal per-provider warnings in both human and JSON output.
