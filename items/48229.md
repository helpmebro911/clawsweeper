## Summary

Keep open. Current main still clears self-declared operator scopes for device-less WebSocket operator connections authenticated only by shared token/password, except for a narrow direct-local backend gateway-client path. That matches the reported custom plugin/WebSocket failure more than an implemented fix, and the linked PR is still only proposed context rather than proof that main fixes this token-auth case.

## Evidence

- **Current WebSocket handshake clears unbound scopes:** For device-less clients, the WebSocket handshake calls shouldClearUnboundScopesForMissingDeviceIdentity() and then clearUnboundScopes(), which empties the requested scopes. The nearby comment says shared token/password auth can bypass pairing but device-less clients still clear self-declared scopes by default, except for the local backend gateway-client handoff.
