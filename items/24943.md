## Summary

Keep open. Current main still does not strip general markdown before sending through the legacy iMessage plugin. The related PR #25349 attempted this exact fix but was closed unmerged, and the current send path only converts markdown tables plus strips internal directive tags before passing text to `imsg`. The linked upstream `imsg` discussion is relevant, but it does not prove OpenClaw has implemented the requested send-time plain-text fallback.

## Evidence

- **iMessage send path lacks markdown stripping:** `sendMessageIMessage` converts markdown tables for the `imessage` channel, then strips inline directive tags, and later sends `params.text` to the RPC client. There is no `stripMarkdown` or equivalent markdown-to-plain conversion in this path.
  - file: [extensions/imessage/src/send.ts:139](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/extensions/imessage/src/send.ts#L139)
  - command: `nl -ba extensions/imessage/src/send.ts | sed -n '1,260p'`
  - sha: [bd796d1c85c7](https://github.com/openclaw/openclaw/commit/bd796d1c85c7dec0dcb89ed11502e4271ded1314)
- **Outbound plugin sanitizer is not the requested markdown stripping:** The iMessage outbound plugin uses `chunkerMode: "text"` and `sanitizeForPlainText(text)`, but that helper targets HTML/plain-text cleanup; its tests preserve lightweight markup such as `*bold*` rather than stripping markdown syntax.
  - file: [extensions/imessage/src/channel.ts:241](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/extensions/imessage/src/channel.ts#L241)
  - command: `nl -ba extensions/imessage/src/channel.ts | sed -n '1,330p' && nl -ba src/infra/outbound/sanitize-text.ts | sed -n '1,180p'`
  - sha: [bd796d1c85c7](https://github.com/openclaw/openclaw/commit/bd796d1c85c7dec0dcb89ed11502e4271ded1314)
- **Shared stripMarkdown utility is not wired into iMessage:** A targeted search found `stripMarkdown` in the shared utility and BlueBubbles send path, but not under `extensions/imessage`, so the legacy iMessage plugin has not adopted the existing stripping behavior.
  - command: `rg -n "stripMarkdown" extensions/imessage src/shared/text/strip-markdown.ts extensions/bluebubbles/src/send.ts`
  - sha: [bd796d1c85c7](https://github.com/openclaw/openclaw/commit/bd796d1c85c7dec0dcb89ed11502e4271ded1314)
- **Related GitHub discussion does not supersede the open issue:** Provided context shows PR #25349 directly tried to close #24943 but was closed unmerged. PR #26465 was also closed unmerged and only addressed a `stripMarkdown()` edge case, not wiring markdown stripping into legacy iMessage on current main.
