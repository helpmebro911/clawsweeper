## Summary

Keep #24943 open. Current main still does not strip or convert general Markdown before iMessage delivery; it only converts Markdown tables, strips inline directive tags, and performs HTML-to-lightweight plain-text sanitization in some outbound paths. The provided GitHub discussion also includes a maintainer comment explicitly keeping this open until PR #56283 or an equivalent fix lands.

## Best Possible Solution

Keep #24943 open until PR #56283 or an equivalent main-branch implementation lands. The best path is shared outbound Markdown stripping for non-Markdown-capable/plain-text channels, applied after existing HTML/table/internal-directive handling, with focused regression tests for iMessage direct sends, monitor replies, media captions, config overrides if added, and empty-after-strip behavior.
