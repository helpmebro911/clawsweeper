## Summary

Keep #24943 open. Current main still does not strip general Markdown before iMessage delivery; it only converts Markdown tables, strips internal directive/scaffolding text, and applies HTML-to-lightweight plain-text sanitization. The related broad fix PR #56283 is still open in the provided context, and the latest owner comment explicitly kept this issue open pending that or an equivalent fix.

## Best Possible Solution

Keep #24943 open until PR #56283 or an equivalent main-branch implementation lands. The likely maintainer path is shared outbound Markdown stripping for non-Markdown/plain-text channels, applied after existing HTML/table/internal-scaffolding handling and covered by focused iMessage direct-send, monitor reply, media-caption, and empty-after-strip regression tests.
