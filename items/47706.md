## Summary

Keep open. Current main has Raspberry Pi docs and manual low-memory advice, but it does not implement the PR's adaptive resource profile or core/resource-limit behavior. The same surfaces still use fixed FFmpeg buffers, static image resize/quality ladders, fixed embedding index concurrency, fixed plugin registry cache limits, fixed LanceDB recall defaults, and systemd units without MemoryHigh/MemoryMax support. The PR is still a plausible core feature, but it needs a current-main rebase and review fixes before it could land.

## Best Possible Solution

Keep the PR open for maintainer review, but do not merge it as-is. The best path is to rebase onto current main, implement the adaptive profile in the current owners of each surface (media helpers, extensions/memory-core, extensions/memory-lancedb, src/plugins, and src/daemon), fix the remaining test-global leak, add focused tests for each adaptive default and override, and update the public Raspberry Pi/Linux docs if the defaults are accepted.
