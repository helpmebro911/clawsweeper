## Summary

Keep #27065 open. Current main still renders the Control UI New session toolbar button as a direct click to `onNewSession`, and the parent still wires it straight to `handleSendChat("/new", { restoreDraft: true })`. There is no confirmation dialog, relocation, danger treatment, or undo guard on this path. The direct fix PR #27110 is closed and unmerged in the provided GitHub context, and the existing maintainer/Codex comment also explicitly kept this issue open.

## Best Possible Solution

Keep #27065 open and implement a focused Control UI safety step before the toolbar New session action dispatches `/new`. The likely change belongs in `ui/src/ui/chat/run-controls.ts` or the `onNewSession` wiring in `ui/src/ui/app-render.ts`, with regression coverage in `ui/src/ui/chat/run-controls.test.ts` for confirm and cancel behavior. Broader `/new` and `/reset` policy can remain tracked separately; this issue still describes a real current-main toolbar misclick risk.
