## Summary

Keep #27065 open. Current main still exposes the Control UI New session toolbar button as a direct click action, and the parent wires it straight to `handleSendChat("/new", { restoreDraft: true })`. I found no confirmation dialog, danger treatment, relocation, or undo guard on this path. The provided GitHub context also says the direct fix PR #27110 was closed unmerged, and the existing maintainer/Codex comment already kept this issue open against the same behavior.

## Best Possible Solution

Keep #27065 open and implement a focused Control UI guard before the toolbar New session action dispatches `/new`. The likely fix belongs in `ui/src/ui/chat/run-controls.ts` or the `onNewSession` wiring in `ui/src/ui/app-render.ts`, with regression coverage in `ui/src/ui/chat/run-controls.test.ts` for both confirm and cancel behavior. Broader `/new` and `/reset` policy can remain tracked separately; this issue still describes a real current-main toolbar misclick risk.
