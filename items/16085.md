## Summary

Keep #16085 open. Current main still implements the bundled Signal plugin against native signal-cli JSON-RPC plus SSE only, while this PR remains the active, unmerged implementation candidate for bbernhard/signal-cli-rest-api REST/WebSocket container support after linked issue #10240 was closed as tracked by this PR.

## Best Possible Solution

Keep #16085 open as the canonical implementation candidate for bundled Signal container REST/WebSocket support. Review and finish the adapter/container client design, align config schema/help/docs/tests, verify native signal-cli JSON-RPC/SSE behavior remains unchanged, and continue using it as the follow-up for closed issue #10240 unless maintainers choose a different implementation path.
