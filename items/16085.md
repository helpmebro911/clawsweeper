## Summary

Keep #16085 open. Current main still only implements the bundled Signal plugin against native signal-cli JSON-RPC plus SSE, while this PR remains the active implementation candidate for bbernhard/signal-cli-rest-api REST/WebSocket container support after linked issue #10240 was closed as tracked by this PR.

## Best Possible Solution

Keep #16085 open as the canonical implementation candidate for bundled Signal container REST/WebSocket support. Finish review of the adapter/container client design, align config schema/help/docs/tests, preserve native signal-cli JSON-RPC/SSE behavior, and continue using this PR as the follow-up for closed issue #10240 unless maintainers choose a different implementation path.
