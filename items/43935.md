## Summary

Current main still models channel hot reloads as channel-wide restarts, while the same-author implementation PR #43938 remains open and contains closing syntax for this issue. This should stay open until that PR or an equivalent account-scoped reload fix lands.

## Best Possible Solution

Keep this issue open while #43938 remains open. The maintainer path is to review and land that PR or an equivalent gateway change that detects strictly account-scoped paths under channels.<channel>.accounts.<accountId>.*, plans account-specific restarts, routes them through stopChannel(channelId, accountId) and startChannel(channelId, accountId), and preserves full-channel restarts for channel-global or mixed changes with focused planner and handler tests.
