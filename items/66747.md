## Summary

Current main still does not fully resolve the reported macOS package/probe and task registry chmod symptoms. The related wrapper work is useful and shipped for #69400, but it is an opt-in service wrapper path; the default published CLI remains shebang-based, still loads dist entrypoints, and the task/task-flow SQLite stores still call chmod synchronously without EPERM-tolerant restore handling.

## Best Possible Solution

Keep this issue open. The maintainer path is to fix or clearly diagnose macOS 26 npm provenance failures in the packaged CLI and LaunchAgent gateway path, provide a supported shebang-free or equivalent default-safe entrypoint if needed, make task and task-flow registry permission hardening tolerate benign chmod EPERM when mode/ownership are already acceptable, and add macOS/package acceptance coverage for npm-installed `openclaw gateway probe`, `openclaw status`, and startup registry restore.
