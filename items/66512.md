## Summary

Keep open. Current main still lacks the requested systemd sd_notify readiness contract: generated Linux units do not use Type=notify or NotifyAccess, the gateway ready boundary only updates internal state/logs, and direct systemctl restart still completes after systemctl returns rather than after READY=1.

## Best Possible Solution

Keep this open for a focused Linux daemon/gateway change: add a small systemd notify helper that detects NOTIFY_SOCKET and sends READY=1 only after the existing gateway ready boundary, update generated and documented Linux units to use Type=notify with NotifyAccess=main, add unit/startup-hook tests, and coordinate optional WatchdogSec/WATCHDOG=1 support with #11973.
