## Summary

Keep open. Current main still renders the Linux gateway systemd unit without Type=notify/NotifyAccess and the gateway readiness path only logs readiness; there is no sd_notify/NOTIFY_SOCKET integration. Existing CLI-side restart health polling mitigates some OpenClaw commands, but it does not satisfy the requested systemd readiness contract for systemctl restart consumers.

## Best Possible Solution

Implement this in OpenClaw core, not ClawHub: add a small systemd notify helper for Linux that detects NOTIFY_SOCKET and sends READY=1 after the same point that currently logs gateway ready, render generated systemd units with Type=notify and NotifyAccess=main when supported, document the behavior, and add focused unit tests for unit rendering plus a startup-path test for the notify hook. Treat WATCHDOG=1/WatchdogSec as either a follow-up here or coordinate with the existing watchdog issue #11973.
