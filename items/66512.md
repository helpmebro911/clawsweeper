## Summary

Keep open. Current main still does not implement the requested systemd readiness notification contract: the generated Linux unit has no Type=notify/NotifyAccess directive, gateway readiness only updates OpenClaw logging/probes, and the systemd restart path still completes after systemctl restart plus optional OpenClaw-side polling rather than making systemd block on sd_notify READY=1.

## Best Possible Solution

Keep this issue open for a focused core daemon/gateway change: add a small systemd notification helper that detects NOTIFY_SOCKET and sends READY=1 only after the WebSocket listener and intended readiness boundary are ready, update generated Linux units and public docs to Type=notify with NotifyAccess=main, add renderer and notification-hook tests, and coordinate WATCHDOG=1/WatchdogSec with #11973 rather than folding hang recovery into this readiness-only change.
