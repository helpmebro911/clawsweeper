## Summary

Keep open. Current `main` still preserves dirty config state globally, so the explicit-reload bug described in the PR is not implemented on `main`; but this PR's unconditional `configFormDirty = false` inside shared `loadConfig()` is too broad because the same loader is used by automatic tab refresh and reconnect paths, which would silently discard unsaved edits outside an explicit Reload action.
