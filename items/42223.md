## Summary

Keep this PR open. Current main still keeps a collapsed sidebar group expanded when the active tab belongs to that group, and the latest release has the same render logic. The submitted fix targets the right root cause, but it is stale against current `nav-section` markup and should add a focused regression test before merge.

## Best Possible Solution

Keep this PR open as the canonical fix path. Update it against current main by changing the current `showItems` logic to remove the active-tab exception while preserving icon-only sidebar behavior when `navCollapsed` is true, then add a focused Control UI browser regression test for collapsing the active group.
