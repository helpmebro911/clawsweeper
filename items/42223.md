## Summary

Keep this PR open. Current main still has the active-tab exception that this PR is trying to remove, so a collapsed sidebar group remains visible when it contains the active tab. The linked bug #42167 was closed as superseded by this PR, not because main already shipped an equivalent fix.

## Best Possible Solution

Keep this PR open as the canonical fix path for #42167. Update it against current main by changing the current `showItems` logic so `isGroupCollapsed` hides child items even when the active tab is inside that group, then add a focused Control UI/browser regression test for collapsing the active group.
