## Summary

Current main still renders `openclaw memory status` through the deep/short-term dreaming resolver, so a light-enabled and deep-disabled configuration can still appear as `Dreaming: off`. The issue should remain open because the provided GitHub context also shows open PR #67910 as an unmerged implementation candidate with auto-closing syntax for this issue.

## Best Possible Solution

Review #67910 against current `memory-core` status behavior. If the patch is acceptable, land it with focused coverage for global dreaming enabled, light enabled, and deep disabled so the PR closes this issue. If #67910 is not suitable, implement the same phase-aware summary in `extensions/memory-core/src/cli.runtime.ts` using the full dreaming config or phase resolvers, with a regression test in `extensions/memory-core/src/cli.test.ts`.
