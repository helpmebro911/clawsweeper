## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before the Mattermost inbound mention gate and agent wake path. Mattermost upstream emits `post_edited` with a `post` payload, but OpenClaw only parses/dispatches `posted`. A complete fix also needs edit-aware replay handling, because current Mattermost replay dedupe is keyed only by message id and would suppress an edited version of a previously seen post.

## Best Possible Solution

Keep the issue open and fix it in the bundled Mattermost plugin: accept `post_edited` events carrying a post payload, route them through the same mention gate/enqueue path when the edited body newly qualifies, make replay dedupe edit-aware so a previously seen unmentioned post can wake after an edit while non-qualifying edits do not re-wake, and add focused `monitor-websocket` plus replay/mention regression tests.
