## Summary

Current main still does not implement the requested force-origin-channel or immutable persisted-origin behavior; it preserves some external route metadata, but routing policy and tests still block dashboard/WebChat turns from delivering to the external origin unless explicit delivery is set.

## Best Possible Solution

Keep this open for maintainer product judgment. The best path is a narrow reply-routing design at the session/reply boundary: either an explicit force-origin setting or an immutable persisted-origin route captured from the first external user turn, with regression tests for Telegram/Discord/WhatsApp-style inbound sessions while WebChat/dashboard is active and safeguards against stale-route or wrong-channel delivery.

- PR #57802 is useful prior art but would need current-main review and regression tests against newer route-preservation behavior before revival.
