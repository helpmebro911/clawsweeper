## Summary

Keep open. Current main has related route-preservation work, but it still does not implement the requested force-origin reply behavior or immutable persisted-origin fields, and WebChat/dashboard-originated turns remain intentionally blocked from external origin routing unless an explicit delivery route is set. The closed, unmerged #57802 is useful prior art rather than shipped behavior.

## Best Possible Solution

Keep this issue open for a maintainer-reviewed reply-routing design. The likely path is a narrow session/reply-boundary change: either a documented explicit force-origin setting or an immutable first-external-origin route with safeguards for stale shared sessions, plus regression tests around Telegram/Discord/WhatsApp-origin sessions while WebChat/dashboard is active. #57802 can be revived or replaced as prior art, but current main should not be treated as already shipping the requested behavior.
