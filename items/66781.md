## Summary

Keep open. Current main narrows the Active Memory example, and fresh local onboarding now defaults to the `coding` tool profile, but the broader least-privilege default requested here is still not implemented: unset/`full` profiles remain unrestricted, no-policy and empty-allowlist matching allow tools, empty runtime `toolsAllow` preserves the full tool list, and docs still document broad full/unset and sub-agent behavior.

## Best Possible Solution

Keep this open for an explicit maintainer product decision on least-privilege tool defaults. A complete fix should define per-agent/per-run default policy semantics, preserve an intentional escape hatch such as `tools.profile: "full"`, warn or migrate existing broad configs, update schema/help/public docs, and add regression coverage for embedded runs, Codex harness runs, plugin/MCP tools, sub-agent policy resolution, and runtime `toolsAllow` behavior.
