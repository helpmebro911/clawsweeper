## Summary

Keep open. Current main fixes the Active Memory example and improves fresh local onboarding, but the broader least-privilege default requested by this issue is still not implemented. Unset/`full` tool profile remains unrestricted, missing policy still allows tools, and sub-agents remain broad unless a restrictive profile or allowlist is configured.

## Best Possible Solution

Keep this issue open for an explicit maintainer decision on least-privilege tool defaults. A complete solution should define the desired default per-agent/per-run policy, preserve a deliberate escape hatch such as `tools.profile: "full"`, migrate or warn existing broad configs, update config schema/help and public docs, and add regression coverage across core runs, embedded runs, plugin/MCP tools, and sub-agent policy resolution.
