## Summary

Keep open. Current main still only refreshes the gateway service environment when the service is loaded, while the not-loaded restart path only re-bootstraps/kickstarts the existing LaunchAgent plist. That leaves the PR's central #48992 fix relevant, but the PR should be revised before merge because it also includes unrelated git-commit build injection changes with a review concern.

## Best Possible Solution

Keep the PR open, but ask for a focused revision: keep the `refreshGatewayServiceEnv = true` move in `src/cli/update-cli/update-command.ts`, add a regression test proving `openclaw update` refreshes/reinstalls the gateway service env when `isLoaded()` returns false, and drop or split the unrelated git-commit/tsdown changes into a separate PR that addresses the existing live-git precedence tests.
