## Summary

Keep open. Current main still does not implement `sessions_broadcast`, and the closed linked issue #47470 was consolidated into this PR rather than solved on main. The PR is a focused implementation, but it still needs maintainer/code review because the reviewed head uses a non-current provenance kind and does not address the subagent tool-deny boundary raised in review comments.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, use an accepted provenance model or extend the provenance contract deliberately, add explicit subagent deny/allow policy handling with tests, update the session tool docs/profile tables, and then decide whether a first-class all-session broadcast primitive belongs in core. If maintainers decide the feature should not be core, close it after that product decision rather than through automated cleanup.
