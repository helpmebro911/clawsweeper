## Summary

Keep #12508 open. Current main still exposes in-process plugin hook chains where handlers can rewrite tool params, outbound message content, transcript-adjacent messages, and prompt/context fields, but there is no host-enforced checksum/signature handoff, per-stage tamper detection, modification audit trail, or manifest-level per-hook allow-list. The existing maintainer-side Codex review comment also explicitly kept this issue open, and current source/docs/tests still support that conclusion.

## Best Possible Solution

Keep #12508 open for core/plugin-runtime security design. The likely solution belongs in the core hook runner/registry/config/docs contract, with tests around modifying hook chains, per-hook permissions, deterministic modification auditing, and host-enforced integrity checks. This should not be moved to ClawHub because an external plugin cannot reliably enforce integrity between other plugins' handlers inside the core runner.
