## Summary

Keep #12508 open. Current main still has trusted in-process plugin hook chains with broad mutation surfaces and coarse policy gating, but no host-owned checksum/signature handoff, tamper detection, per-handler modification audit trail, or per-hook manifest/config allow-list. The existing April 26, 2026 maintainer-side Codex review comment in the issue also kept this open, and current source/docs/tests still support that conclusion.

## Best Possible Solution

Keep this open for core plugin-runtime security design. The likely path is a host-owned change across the hook runner, registry/config policy, manifest/docs contract, and focused tests for sequential mutation, per-hook permissions, deterministic modification auditing, and integrity verification. Do not move this to ClawHub unless core first exposes an API that lets an external plugin enforce these guarantees across the full hook chain.
