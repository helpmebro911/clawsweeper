## Summary

Keep #12508 open. Current main still has trusted in-process plugin hook chains where modifying hooks run sequentially and can rewrite prompt context, tool params, outbound messages, and transcript-adjacent messages, but there is still no host-owned checksum/signature handoff, tamper detection, per-handler modification audit trail, or per-hook manifest allow-list. The existing maintainer-side Codex comment keeping the issue open still matches current main.

## Best Possible Solution

Keep this open for core plugin-runtime security design. The likely path is a host-owned change across the hook runner, registry/config policy, manifest/docs contract, and focused tests for sequential mutation, per-hook permissions, deterministic modification auditing, and integrity verification. This should not move to ClawHub unless core first exposes an API that lets an external plugin enforce these guarantees across the full hook chain.
