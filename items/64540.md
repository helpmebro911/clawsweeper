## Summary

Keep this PR open. Current main still has the reported empty-heartbeat fallback behavior, and this PR is a focused implementation candidate, but the one-file patch is incomplete because other heartbeat policy paths still use the documented `every: "0m"` disable contract and would diverge from the changed summary helper.

## Best Possible Solution

Keep this PR open for maintainer review or contributor rework. If OpenClaw wants `heartbeat: {}` to mean disabled, add one shared empty-heartbeat predicate and apply it consistently across `src/infra/heartbeat-summary.ts`, `src/infra/heartbeat-runner.ts`, `src/agents/heartbeat-system-prompt.ts`, and `src/agents/acp-spawn.ts`; then add regression tests for defaults and per-agent empty heartbeat configs, including inherited defaults, and update the public heartbeat/config docs. If maintainers want `every: "0m"` to remain the only disable contract, document that on #64293 and close or revise this PR instead of landing a partial behavior change.
