## Summary

Keep open. Current `main` still intentionally defaults thread-bound ACP spawns to persistent/session mode when `thread: true` or `/acp spawn --thread ...` is used, so the issue title behavior remains. Main has added mitigations like visible thread-binding intro messages, idle/max-age expiry, and stale-binding cleanup, but those reduce fallout rather than changing the default.
