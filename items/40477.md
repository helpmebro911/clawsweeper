## Summary

Keep open. Current `main` still uses the generic forwarded exec-approval hint instead of embedding ready-to-send `/approve <id> ...` commands, and the tests explicitly lock that behavior in.

## Evidence

- **Forwarded exec approval text still uses placeholder:** `buildRequestMessage()` appends `Reply with: /approve <id> ${decisionText}` rather than concrete commands per decision.
  - file: [src/infra/exec-approval-forwarder.ts:268](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/src/infra/exec-approval-forwarder.ts#L268)
  - command: `nl -ba src/infra/exec-approval-forwarder.ts | sed -n '227,272p'`
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **Current tests expect the generic placeholder:** The forwarder test asserts `Reply with: /approve <id> allow-once|allow-always|deny`, and the ask=always case asserts `Reply with: /approve <id> allow-once|deny`. That means current main intentionally preserves the unfixed text.
  - file: [src/infra/exec-approval-forwarder.test.ts:539](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/src/infra/exec-approval-forwarder.test.ts#L539)
  - command: `nl -ba src/infra/exec-approval-forwarder.test.ts | sed -n '532,560p'`
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **Related approval surfaces already support prefilled commands:** The shared pending-reply renderer prints concrete `/approve ...` command fences (`Run:` plus `Other options:`), and Telegram native approvals build from that renderer. So the requested UX exists in native/shared approval flows, but not in the forwarded fallback text used by `exec-approval-forwarder.ts`.
  - file: [src/infra/exec-approval-reply.ts:302](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/src/infra/exec-approval-reply.ts#L302)
  - command: `nl -ba src/infra/exec-approval-reply.ts | sed -n '302,313p'`
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **Telegram native path uses the richer renderer:** Telegram native approval payloads call `buildExecApprovalPendingReplyPayload(...)`, which is the path that already emits concrete commands and inline buttons.
  - file: [extensions/telegram/src/approval-handler.runtime.ts:72](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/extensions/telegram/src/approval-handler.runtime.ts#L72)
  - command: `nl -ba extensions/telegram/src/approval-handler.runtime.ts | sed -n '66,90p'`
