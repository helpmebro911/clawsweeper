## Summary

Keep this PR open. Current main still normalizes Signal shared allowlist entries and DM-policy entries through phone-only E.164 formatting, while the documented Signal contract supports `uuid:<id>` sender IDs. The PR is a focused Signal source/test change, and the provided diff does not touch CI, dependencies, lockfiles, install/build/release scripts, publishing metadata, generated/vendor code, or secrets.

## Best Possible Solution

Rebase and review this PR, then land it or an equivalent Signal-owned normalizer used by both `signalConfigAdapter.formatAllowFrom` and `signalSecurityAdapter.normalizeDmEntry`. Keep the regression coverage for `uuid:`, `signal:uuid:`, wildcard preservation, and E.164 phone normalization, and either canonicalize compact UUIDs or document why that edge is deferred.
