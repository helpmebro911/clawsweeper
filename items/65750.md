## Summary

Keep open. Current main still has the exact clear-path reload that this PR removes: `/clear` calls `sessions.reset`, zeroes local chat state, then immediately calls `loadChatHistory`, whose implementation can write returned history back into `chatMessages`. The linked bug report is closed, but the PR is still a focused implementation candidate rather than cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is to remove the immediate post-reset history reload from the active `/clear` implementation, add a regression test that would fail if stale `chat.history` repopulates `chatMessages`, and confirm whether the unused render-level clear callback should be updated or deleted as part of the same cleanup.
