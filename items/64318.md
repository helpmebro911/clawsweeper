## Summary

Keep open. `main` does not already ship this Amazon plugin, and the latest PR head still conflicts with existing OpenClaw guardrails and runtime contracts. The prior discussion is useful context, especially the earlier Polly PR links, but the current branch still has actionable correctness issues rather than something suitable for conservative cleanup-closing.

## Evidence

## Risks / Open Questions

- Landing the PR as-is likely trips an existing bundled-plugin naming guardrail before runtime review even starts.
- The realtime voice path appears unsafe for voice-call usage because transient stream failures can still force hangup and the audio-rate conversions do not match the telephony contract.
- Amazon Polly talk-mode configuration and explicit voice/model overrides do not line up with current speech-core conventions, so user configuration would not behave as expected.
