## Summary

Keep open. Current main still has the legacy memory-core dreaming path: a request-scoped subagent runtime falls back to a local snippet write instead of retrying through a gateway-owned background subagent. The PR is recent, focused on a real current bug also reflected by related issue #67152, and no current-main implementation or superseding canonical PR closes out the central behavior.

## Best Possible Solution

Keep #70554 open as the active implementation candidate for the memory-core dreaming background-runtime bug. Maintainers should trim unrelated changes if needed, resolve the remaining review comments, preserve the narrow memory-core host-runtime seam, add or keep regression coverage for background retry and cleanup, and run the changed gateway/memory-core lanes before merge.
