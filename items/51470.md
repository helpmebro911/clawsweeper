## Summary

Keep open. Current `main` does not implement this PR's change: managed service proxy env forwarding still omits `NODE_USE_ENV_PROXY`, so the reported gap remains plausible and the PR is not obsolete.

## Evidence

- **Current main omits the requested env key:** `SERVICE_PROXY_ENV_KEYS` on `main` includes only `HTTP_PROXY`, `HTTPS_PROXY`, `NO_PROXY`, `ALL_PROXY`, and lowercase variants. `NODE_USE_ENV_PROXY` is not present, so `readServiceProxyEnvironment()` cannot preserve it.
  - file: [src/daemon/service-env.ts:47](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/daemon/service-env.ts#L47)

  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Managed service environment only forwards keys from `proxyEnv`:** `buildCommonServiceEnvironment()` spreads `sharedEnv.proxyEnv` into the service environment. Because `proxyEnv` is built from `SERVICE_PROXY_ENV_KEYS`, the missing key is not propagated to installed gateway/node services on current `main`.
