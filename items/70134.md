## Summary

Keep PR #70134 open. Current main still uses the `auth-profiles.json` filesystem mtime in the `models.json` ready-cache fingerprint, and there is no equivalent semantic auth-profile fingerprint module or regression coverage on main. The PR targets a real remaining cache-invalidation problem and is not superseded; it still needs normal maintainer review because the provided discussion includes a remaining correctness tradeoff around OAuth token content fingerprinting.

## Best Possible Solution

Keep the PR open for maintainer review. The best implementation direction is to replace the mtime-only `auth-profiles.json` component with a semantic auth-profile fingerprint that preserves meaningful profile order, normalized alias and SecretRef behavior, parse/read-error invalidation, OAuth identity/token changes that affect discovery, and existing `models.json` mtime invalidation, with focused regression tests for no-op rewrites and meaningful auth changes.
