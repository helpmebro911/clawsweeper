## Summary

Keep #69707 open. Current main still forwards configured `embedding.dimensions` into the `memory-lancedb` plugin's OpenAI embeddings request, and the current colocated test asserts that behavior. The PR addresses a real remaining compatibility gap for OpenAI-compatible `baseUrl` providers that reject `dimensions`, and it is not clearly duplicate or superseded by the adjacent `encoding_format` or preserve-dimensions reports.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best path is to fix the plugin so providers that reject `dimensions` can still use configured local vector sizes, while preserving clear validation for invalid or too-short embeddings and adding focused regression coverage for the outbound request shape and local truncation behavior.
