## Summary

Keep open. Current `main` still bypasses image sanitization for OpenAI-compatible `image_url` inputs when the prompt text has no detected image refs, so the core 5MB/resize bug remains. The streaming-error half of the report appears fixed on `main`, which makes this issue partially addressed but not closable.

## Evidence

- **Gateway extracts `image_url` parts and forwards them as prompt images without resize/sanitization at ingress:** `resolveImagesForRequest()` parses each `image_url`, enforces count/total-byte limits, and returns `ImageContent[]`; `handleOpenAiChatCompletions()` then passes those images into `buildAgentCommandInput({ prompt: { ..., images }})`. There is no call to the image sanitization/resize pipeline in this gateway step.
  - file: [src/gateway/openai-http.ts:332](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/gateway/openai-http.ts#L332)
