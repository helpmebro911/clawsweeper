## Summary

Keep PR #68848 open. Current main still has the exact no-registration cleanup gap the PR targets: `maybeWakeNodeWithApns` creates `nodeWakeById` state before loading APNs registration, then returns `path: "no-registration"` without deleting that state. The only current deletion path is `clearNodeWakeState`, called from node WebSocket disconnect cleanup, which does not cover unregistered or typo node IDs. The linked issue #68847 and Greptile comment both describe the same valid bug, and prior merged PR #63709 covered a different registered-node disconnect cleanup path.

## Best Possible Solution

Keep the PR open and land #68848 or an equivalent minimal patch that deletes `nodeWakeById` on the `!registration` early-return path, with a focused regression test proving repeated unregistered node wake attempts do not grow the module-scope map. Prior #63709 should be treated as complementary, not superseding, because it only covers registered-node WS disconnect cleanup.
