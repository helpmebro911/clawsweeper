## Summary

Current `main` does not implement this PR's memory embedding proxy fix. The shared memory HTTP helper still calls `fetchWithSsrFGuard` without `withTrustedEnvProxyGuardedFetchMode`, so embedding requests remain on the default strict path that ignores env proxy settings. The related fetch-guard hardening from the PR is also absent on `main`.

## Evidence
