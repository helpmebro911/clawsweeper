## Summary

Keep #70334 open. The report is recent, concrete, corroborated by another user, and #70591 has already been closed as a duplicate of this same compaction/session-freeze failure. Current main has related safeguards, but I did not find direct source, test, changelog, or history evidence proving the successful overflow-compaction retry path can no longer leave the session in `processing` with `queueDepth=0`.

## Best Possible Solution

Keep #70334 open as the canonical tracker for the compaction retry/session-lock freeze. Add focused regression coverage for explicit overflow compaction followed by retry success, retry error, and lost retry/lifecycle resolution; assert diagnostic state returns to idle or the run is boundedly aborted, active-run state clears, and queued follow-up messages can drain. If the root cause is stale active-run state rather than compaction wait resolution, wire recovery through existing abort/cleanup primitives instead of leaving `session.stuck` as warn-only.
