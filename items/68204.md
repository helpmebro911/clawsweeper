## Summary

Keep this issue open. Current main still has separate agent events, trajectory bundle events, ACP relay logs, and task/Task Flow ledger records rather than one canonical run trace schema adopted across agent, ACP, subagent, and task-flow paths. The related run-trace PR remains open/unmerged in the provided context, so it is useful prior art but not shipped coverage.

## Best Possible Solution

Keep this open as an architecture/observability tracking issue. The best path is to define a small versioned run-trace contract, decide how it relates to existing trajectory and agent-event surfaces, then adapt agent events, ACP parent-child relay, subagent lifecycle, task records, and Task Flow state changes into that schema with focused tests. Review #43145 as related prior art or an implementation candidate, but do not close this issue until the cross-flow schema is merged on main.
