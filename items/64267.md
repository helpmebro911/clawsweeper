## Summary

Keep open. Current `main` still has the stale streaming thinking-tag regex in `src/agents/pi-embedded-subscribe.ts`, so the issue's core leak path is not fully fixed even though a newer MiniMax-specific mitigation landed.

## Evidence

- **Streaming path still uses narrow regex on current main:** `src/agents/pi-embedded-subscribe.ts` still scans only plain `<think>`/`<thinking>`/`<thought>`/`<antthinking>` tags and requires a bare `>` terminator, so it will miss namespaced or attributed tags in streamed block stripping.
  - file: [src/agents/pi-embedded-subscribe.ts:38](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/pi-embedded-subscribe.ts#L38)
  - command: `nl -ba src/agents/pi-embedded-subscribe.ts | sed -n '34,44p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Canonical sanitizer is broader than the streaming path:** `src/shared/text/reasoning-tags.ts` accepts `antml:`-prefixed tags and tag attributes via `\b[^<>]*>`, which confirms the mismatch the issue discussion called out is still present on current `main`. The shared tests also cover namespaced and attributed tags.
  - file: [src/shared/text/reasoning-tags.ts:7](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/shared/text/reasoning-tags.ts#L7)
