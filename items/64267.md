## Summary

Keep open. Current main has adjacent reasoning-tag fixes, including canonical sanitizer support for attributed and namespaced thinking tags, but the streamed/block reply scanner still uses a narrower tag grammar. That leaves a plausible user-visible leak on the same streaming path identified in the issue discussion.

## Best Possible Solution

Keep this issue open and make a narrow core fix in the embedded streaming utilities. Share the canonical reasoning-tag grammar with `stripBlockTags`, `splitThinkingTaggedText`, and reasoning extraction, including `antml:` prefixes and attributes, while preserving code-span awareness and chunk-boundary state. Add focused streaming/block regression tests for attributed `<think>`, `<thinking>`, and `<antml:thinking>` across chunk boundaries.
