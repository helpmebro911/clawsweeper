## Summary

Keep this issue open. Current main still does not expose `stopReason`, `hasToolCalls`, or an equivalent agent completion context to `message_sending` hook handlers. The requested signal exists in agent run metadata, but the shared outbound delivery params and hook event builder only pass delivery/routing metadata. Related PR #39761 remains an open implementation candidate, not shipped behavior.

## Best Possible Solution

Keep the issue open and implement a backward-compatible hook contract change. The likely path is to thread agent completion metadata from `EmbeddedPiRunMeta` through agent command delivery and `DeliverOutboundPayloadsCoreParams`, then expose optional `stopReason` and `hasToolCalls` or a broader typed `agentContext` on `PluginHookMessageSendingEvent`, with focused tests for shared outbound delivery and agent reply delivery. If #39761 becomes the chosen implementation, close this issue only after that PR merges and current main demonstrably exposes the requested signal.
