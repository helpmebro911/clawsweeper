## Summary

Keep PR #46858 open. Current main still has the bug the PR targets: heartbeat runs read `messages.responsePrefix` as a raw string and pass it through heartbeat OK/alert handling without the `onModelSelected` callback or `resolveResponsePrefixTemplate()` context that normal replies use. The linked issue #43064 remains a valid current-code problem rather than an obsolete or already-implemented request.

## Best Possible Solution

Keep the PR or an equivalent core fix open: update `runHeartbeatOnce` to use the same response-prefix context pattern as normal replies, pass `onModelSelected` through heartbeat `getReplyFromConfig` options, resolve the prefix before heartbeat OK/alert normalization and delivery, and add regression coverage for templated alert replies plus prefixed `HEARTBEAT_OK` suppression.
