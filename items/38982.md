## Summary

Keep #38982 open. Current main has startup light/dark TUI palette support, but it still does not ship this PR's central behavior: no runtime `/theme` command, no `/theme list`, and no searchable picker with live preview, cancel/revert, and commit behavior.

## Best Possible Solution

Keep #38982 open unless a maintainer explicitly chooses #44382 as the canonical TUI theming path. The best product path is to coordinate the overlapping PRs and land one focused implementation that adds a runtime theme API, registers `/theme`, supports `/theme list`, opens a searchable picker with live preview, commits on select, reverts on cancel, repaints all TUI surfaces consistently, updates public TUI docs, and adds targeted command/theme tests.
