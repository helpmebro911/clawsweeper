## Summary

Keep PR #39975 open. Current main still emits `message_reference` in Discord Execute Webhook payloads when `replyTo` is present, and Discord's current webhook docs still do not list that field for Execute Webhook while Create Message does. The PR's branch is stale because it edits the old `src/discord/send.outbound.ts` path, but the focused behavior change is still needed in `extensions/discord/src/send.outbound.ts` with a regression test.

## Best Possible Solution

Keep the PR open or replace it with an equivalent refreshed patch against `extensions/discord/src/send.outbound.ts`: remove `message_reference` only from `sendWebhookMessageDiscord`, preserve bot-token/channel reply behavior in `sendMessageDiscord` and media/component paths, and add a focused test proving `replyTo` does not serialize into Execute Webhook payloads.
