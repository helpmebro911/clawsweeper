## Summary

Keep open. Current main still requires `models.providers.*.models`, so the reporter's config with `baseUrl` and `apiKey` but no `models` remains invalid. The memory embedding client would use `models.providers.openai.baseUrl` after config validation, but the schema still blocks the exact configuration described in #39589. The linked #46977 was closed unmerged and #46980 is still open, so the fix is not implemented on main; closing as implemented or superseded would be premature.
