## Summary

Keep #38246 open. The prior ClawSweeper keep-open assessment still matches current main at e672b614: WhatsApp visible auto-replies are delivered through the inbound message's reply/sendMedia closures, which target the inbound chat JID. Main-DM pinning and doctor/audit warnings address shared-session metadata risk, but they do not block, reroute, or force explicit targets for plain replies in allowlisted third-party DMs.

## Best Possible Solution

Keep the issue open and implement a WhatsApp plugin routing guard in the visible auto-reply delivery path before deliverWebReply sends payloads. Add typed config/schema/docs for the chosen policy, and regression coverage for owner DMs, allowlisted non-owner DMs where plain replies are blocked or rerouted, and explicit message-tool sends that still work intentionally.
