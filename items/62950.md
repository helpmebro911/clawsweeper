## Summary

Current main still has the reported directive-preservation bug. Mixed-content inline directives are parsed, then `clearInlineDirectives(parsedDirectives.cleaned)` replaces the directive object before run-level resolution, which drops `thinkLevel` and `verboseLevel` and causes defaults to be used. This PR remains a focused bug-fix candidate; it is not implemented on main or superseded by the current code.

## Best Possible Solution

Keep this PR open as a valid bug-fix candidate. The best maintainer path is to review the small preservation change in `src/auto-reply/reply/get-reply-directives.ts`, prefer the repository's spread-reassignment style if requested, add regression coverage for mixed-content `/think` and `/verbose` plus unauthorized senders, then land it or replace it with an equivalent fix on main.
