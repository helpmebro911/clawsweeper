## Summary

Current main still clears mixed-content `/think` and `/verbose` directive levels before run resolution, so this PR is not obsolete. The proposed direction is plausible and narrowly scoped, but it still needs regression coverage and a maintainer check that preserving `hasThinkDirective`/`hasVerboseDirective` does not accidentally persist one-shot inline levels as session defaults.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. The best maintainer path is to land this preservation change or an equivalent spread-style version, add regression tests around `resolveReplyDirectives` and Gateway `chat.send` mixed-content thinking/verbose input, and explicitly verify that preserved inline levels affect only the intended run unless maintainers want session persistence. #38729 remains related Anthropic xhigh provider work, not a replacement for this directive-preservation fix.

## Risks / Open Questions

- The PR as written may preserve `hasThinkDirective`/`hasVerboseDirective` strongly enough to persist mixed-content levels as session overrides; that needs an explicit regression test or a small implementation adjustment if inline levels are meant to be current-run only.
- Regression tests should cover authorized mixed-content `/think` and `/verbose`, unauthorized mixed-content messages, and the `chat.send`/`sessions.send` `thinking` injection path.
- The direct property mutation noted by Greptile is functionally small, but a spread-style reassignment would match the surrounding code style.
