## Summary

Keep this PR open. Current main still does not implement explicit message-tool agent routing: the message tool has no `agentId` option, derives routing from `agentSessionKey`, and the embedded tool path only passes the resolved agent as `requesterAgentIdOverride` for broader tool/session context rather than message send attribution. The provided PR discussion also shows a real fallback hazard was identified and addressed in the PR head, so this remains useful maintainer work rather than cleanup-close material.

## Best Possible Solution

Keep this PR open and review or rebase a guarded version onto current main: add an explicit `agentId` path through `createOpenClawCodingTools` -> `createOpenClawTools` -> `createMessageTool`, prefer it only when it is a trusted runtime-provided explicit value, preserve config-aware fallback routing for opaque/non-agent session keys, add focused tests for explicit override and default-agent fallback behavior, and fix changelog attribution if the entry remains.
