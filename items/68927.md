## Summary

Keep PR #68927 open. Current main still renders friendly/catalog-derived chat model labels while preserving qualified values, and the PR intentionally proposes a product-visible change to show exact qualified refs instead. The related closed PRs cover value preservation/nested refs, not this label policy change.

## Best Possible Solution

Keep the draft PR open for maintainer product review. If maintainers want exact qualified refs in the chat picker, update `ui/src/ui/chat-model-ref.ts`, `ui/src/ui/chat-model-select-state.ts`, and the current friendly-label tests together; otherwise close it explicitly as a product decision, not as already implemented.
