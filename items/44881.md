## Summary

Keep open. Current `main` does not show a fix for PM2-managed gateway upgrades: service lifecycle support is limited to launchd/systemd/schtasks, unmanaged `gateway stop` only sends SIGTERM to the current listener PID, PM2 is not detected as a supervisor, and the repo’s own npm-update smoke still force-stops old gateway processes before and after update because same-host npm upgrades can leave the old gateway process/version around.

## Evidence

- **Gateway service commands only target built-in service managers:** `gateway install|start|stop|restart|uninstall` are described as managing only `launchd/systemd/schtasks`, not PM2.
  - file: [src/cli/daemon-cli/register-service-commands.ts:76](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/cli/daemon-cli/register-service-commands.ts#L76)

- **Service registry has no PM2 backend:** `resolveGatewayService()` only registers darwin launchd, linux systemd, and win32 Scheduled Task implementations.
  - file: [src/daemon/service.ts:172](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/daemon/service.ts#L172)

  - file: [src/infra/supervisor-markers.ts:1](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/infra/supervisor-markers.ts#L1)
