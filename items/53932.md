## Summary

Current main still does not expose or wire a Slack option to preserve original image bytes during outbound uploads. Slack uploads continue through the optimized outbound-media helper, the Slack config/docs only expose `mediaMaxMb` for this area, and the related implementation attempt #53988 is closed unmerged in the provided context.

## Best Possible Solution

Keep this open and implement the opt-out in the bundled Slack plugin. Add a documented Slack config key that defaults to current optimized behavior, plumb it through Slack send/upload via an additive outbound-media SDK option or a deliberate raw media loader path, update schema/types/generated metadata/docs, and add regression coverage proving PNG uploads can preserve original bytes and MIME when disabled while the existing optimized default remains intact.
