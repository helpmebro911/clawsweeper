## Summary

Keep this issue open. Current main still uploads Slack media through the optimized outbound media path, and Slack config/docs expose only media size limits, not an option to preserve original image bytes or disable optimization. The related PR #53988 was closed unmerged, so it does not supersede the request.

## Best Possible Solution

Implement this in the bundled Slack plugin as an explicit documented config control, likely on channels.slack and per-account Slack config. Keep the current optimized behavior as the default, route Slack uploads through raw outbound media loading or an SDK optimizeImages=false option when disabled, and preserve maxBytes, SSRF, proxy, and local-root safeguards. Add schema/types/metadata/docs plus regression coverage proving PNG uploads can preserve original MIME/bytes when the opt-out is enabled.
