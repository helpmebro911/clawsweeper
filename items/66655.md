## Summary

Keep open. Current `main` still does not implement the PR's requested behavior: the Control UI dropdown label reads `thinkingDefault` from gateway session data, but the gateway currently fills that field from provider/model fallback logic rather than `agents.defaults.thinkingDefault` from config.

## Evidence

- **Control UI label still depends on gateway session defaults:** `resolveChatThinkingSelectState` builds the dropdown label from `activeRow?.thinkingDefault ?? state.sessionsResult?.defaults?.thinkingDefault ?? ...`, so the UI only shows the configured default if gateway session data already carries it.
  - file: [ui/src/ui/chat/session-controls.ts:194](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/ui/src/ui/chat/session-controls.ts#L194)

That means the Web UI receives the provider/model fallback default instead of `agents.defaults.thinkingDefault`.
