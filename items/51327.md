## Summary

`main` does not implement this PR's fix. Deleted agents with `deleteFiles=false` still lose `sessions.usage` history because agent deletion removes config/store visibility while keeping transcript files, and usage discovery still scans only the visible gateway agent list.

## Evidence

- **Deletion keeps transcript files but removes session-store visibility:** `agents.delete` defaults `deleteFiles`, purges agent session-store entries, and only trashes `agentDir`/`sessionsDir` when `deleteFiles` is true. The existing test also asserts no file deletion happens when `deleteFiles: false`.
  - file: [src/gateway/server-methods/agents.ts:634](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/gateway/server-methods/agents.ts#L634)
  - command: `nl -ba src/gateway/server-methods/agents.ts | sed -n '610,670p'; nl -ba src/gateway/server-methods/agents-mutate.test.ts | sed -n '972,1025p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Visible gateway agents intentionally exclude disk-only agents:** `listAgentsForGateway()` builds agent IDs from configured/default/main keys only; it does not scan `agents/*/sessions`. The scope-boundary test explicitly verifies that an explicit `agents.list` hides disk-only agents on disk.
  - file: [src/gateway/session-utils.ts:714](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/gateway/session-utils.ts#L714)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '680,760p'; nl -ba src/gateway/session-utils.test.ts | sed -n '620,700p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **`sessions.usage` still discovers sessions only through visible agents:** `discoverAllSessionsForUsage()` calls `listAgentsForGateway(config).agents` and then `discoverAllSessions({ agentId })` for each returned agent. `discoverAllSessions()` resolves only `resolveSessionTranscriptsDirForAgent(params?.agentId)`, so preserved transcripts for a deleted, disk-only agent are never scanned.
  - file: [src/gateway/server-methods/usage.ts:305](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/gateway/server-methods/usage.ts#L305)
