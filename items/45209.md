## Summary

Keep open. Current main still does not implement the Feishu Bitable tool toggle requested by PR #45209/#45195: `FeishuToolsConfig`, the Zod schema, defaults, and multi-account merge helper all omit `bitable`, while `registerFeishuBitableTools` still registers all `feishu_bitable_*` tools whenever Feishu accounts exist. The PR remains a focused bundled Feishu plugin improvement rather than an obsolete, duplicate, or ClawHub-only change.

## Best Possible Solution

Keep PR #45209 open for maintainer review or landing. The best path is to add `bitable?: boolean` through the Feishu config type/schema/default/merge path, guard `registerFeishuBitableTools` the same way drive/wiki/perm are guarded, and add focused tests proving `tools.bitable: false` suppresses `feishu_bitable_*` registration while the default remains enabled.
