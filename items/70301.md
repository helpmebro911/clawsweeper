## Summary

Keep open. Current main already shipped the central `/bot-approve` `requireAuth: true` auth-gate, but PR #70301 still contains useful unmerged review follow-up: a QQ-only framework-command guard and `/bot-approve ?` help preservation. Those gaps remain visible in current main.

## Best Possible Solution

Keep #70301 open or recreate its remaining delta on top of current main. The best path is to preserve the shipped #70706 auth-gate, add the QQ-only channel guard in `extensions/qqbot/src/bridge/commands/framework-registration.ts`, restore `/bot-approve ?` help in the handler, and add focused regression coverage for non-QQ framework contexts plus unauthorized pre-dispatch non-execution/config non-mutation.
