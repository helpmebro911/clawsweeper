## Summary

Keep open. Current main already has the core auth-gating flag for `/bot-approve`, but this PR still contains meaningful security/behavior work that is not on main: a QQ-only framework-command guard and stronger regression coverage for the pre-dispatch bypass. Closing as implemented would discard a plausible remaining control-surface fix identified in the PR review discussion.

## Best Possible Solution

Keep the PR open for maintainer review or rework. The best path is to land an equivalent QQ-only framework guard plus focused regression tests for `/bot-approve` help behavior and pre-dispatch non-execution/config immutability on top of current main, then close this PR only after that remaining delta is merged or superseded by a canonical follow-up.
