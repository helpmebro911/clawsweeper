## Summary

Keep open. The report is recent, corroborated by two follow-up comments, and current `main` does not show a landed fix for the affected memory-provider selection paths. The intended behavior is that explicit `memorySearch.provider` values like `ollama` are honored and `fallback: "none"` disables provider fallback, but there is no evidence that this issue was fixed on `main` after the latest release.

## Evidence
