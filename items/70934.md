## Summary

Keep open. Current main has QuickStart, `--skip-bootstrap`, and the Crestodian modern preview, but the shipped first-run path still runs setup before first chat and the bootstrap flow explicitly collects identity/persona-style preferences before normal replies. I did not find a shipped value-first sequence with example tasks or an initial useful task before customization.

## Best Possible Solution

Keep this open for a focused onboarding UX change. The likely fix is to make first-run bootstrap value-first: present a few ready-to-run example tasks or open a working Control UI/TUI chat first, then defer BOOTSTRAP identity/persona/preferences until after a successful initial result. The implementation probably needs coordinated changes in `docs/reference/templates/BOOTSTRAP.md`, bootstrap prompt handling in `src/agents/system-prompt.ts`/`src/agents/bootstrap-prompt.ts`, and the setup finalize hatch flow, while preserving `--skip-bootstrap`, QuickStart, and Crestodian as compatible paths.
