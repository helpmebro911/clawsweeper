## Summary

Keep #70934 open. Current main already reduces some onboarding friction with QuickStart defaults, skip paths, and chat welcome suggestions, but it does not implement the requested value-first onboarding sequence: show bootstrap/example tasks, let the user complete a useful task first, then prompt for optional personalization/customization. This is a recent product/UX enhancement with no duplicate or maintainer resolution in the provided discussion.

## Best Possible Solution

Keep the issue open for maintainer product triage. The best path is a focused onboarding design change that preserves QuickStart defaults, opens a concrete first-use task or chat template before optional identity/avatar/context-profile customization, and adds a regression test or scripted onboarding proof for the first-run sequence.
