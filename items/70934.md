## Summary

Current main still does not implement the requested value-first first-run onboarding. QuickStart, Crestodian, and macOS onboarding reduce setup friction, but the primary flows still configure model/auth, workspace, gateway, channels/search/skills, and bootstrap/persona handoff before a new user can pick and run a useful example task/template.

## Best Possible Solution

Keep this open for maintainer UX/product triage. A good implementation would add a first-run value-first path with example task templates and one quick useful runnable task before optional identity, persona, reply-style, channel, search, skill, and other customization steps, likely touching src/wizard/setup.ts, src/wizard/setup.finalize.ts, the macOS onboarding page/chat flow, docs/start/wizard.md, and focused wizard/onboarding tests.
