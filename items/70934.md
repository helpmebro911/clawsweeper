## Summary

Current main still does not implement the requested value-first first-run onboarding. QuickStart, the Control UI welcome suggestions, and macOS onboarding chat reduce some friction, but the primary first-run paths still complete setup/bootstrap/persona work before presenting a runnable example task/template experience.

## Best Possible Solution

Keep this open for maintainer UX/product triage. The best implementation would add a first-run path that performs only the minimum required model/session setup, presents a small set of bootstrap templates or example tasks, lets the user run one useful task immediately, and then offers optional agent name, reply style, symbols/avatar, persona, channel, search, skill, and other customization steps. Likely touch points are the CLI wizard and finalization handoff, macOS onboarding order and kickoff prompt, Control UI first-chat affordances, public onboarding docs, and targeted wizard/onboarding tests.
