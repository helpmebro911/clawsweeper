## Summary

Keep open. Current main has useful partial mitigations, including QuickStart defaults, a skippable channel step, Control UI welcome suggestions, and `--skip-bootstrap`, but it does not implement the requested value-first first-run flow. The documented and tested flow still configures model/auth, gateway, channels, daemon/health, skills, and then bootstraps identity/preferences before the user has run a chosen useful task.

## Best Possible Solution

Keep this open for product/design work. The best path is to reuse the existing QuickStart, Control UI suggestions, and `--skip-bootstrap` pieces, but add a true value-first first-run path: present a small template/task picker, let the user run one low-friction useful task in Control UI/TUI, then offer identity/persona/bootstrap customization as an optional follow-up. Update CLI/macOS/mobile docs and tests around the chosen first-run sequence.
