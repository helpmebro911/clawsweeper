## Summary

Keep this PR open. Current main still lacks the `/v1/chat/completions` tool protocol support the PR targets, while the adjacent `/v1/responses` path already has client-tool parsing, tool-choice handling, forwarding, and pending-tool-call response mapping. The PR is a focused implementation candidate for the remaining #54405 work and the related #54174 streaming gap, but the stream-finalization review concern should be resolved before landing.

## Best Possible Solution

Keep this PR open and review or rework it on current main. The best path is to fix the stream-finalization race so tool-call streams wait for the command result or another proven-safe condition before finalizing, keep the focused Chat Completions parsing/forwarding/response mapping, preserve the regression coverage in `src/gateway/openai-http.test.ts`, and then land it as the implementation for the open #54405 client-tool passthrough gap and the related #54174 streaming tool-call behavior.
