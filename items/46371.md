## Summary

Keep PR #46371 open. Current main has moved Gemini CLI OAuth into the bundled Google plugin and already covers several npm-shim/global-install cases, but it does not include the PR’s no-PATH `%APPDATA%\npm\node_modules` fallback. Current main also does not implement `WORKING.md` as a workspace bootstrap file. The related PRs in the supplied context are closed/unmerged, so they do not supersede this open PR, although this PR would need to be rebased and split or updated for the current code layout.

## Best Possible Solution

Keep the PR open for maintainer review, but do not land it as-is. The best path is to port the Windows Gemini CLI fallback into `extensions/google/oauth.credentials.ts` with a focused regression test that restores all mutated env state, and handle `WORKING.md` as a separate agent-workspace change with explicit docs/tests for main, subagent, cron, and compaction behavior.
