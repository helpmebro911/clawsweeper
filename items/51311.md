## Summary

Keep open. Current `main` still resumes the `sendPing()` checked continuation directly from `URLSessionWebSocketTask.sendPing`'s callback with no at-most-once guard, so the PR's crash fix is not implemented on `main` or in the latest release `v2026.4.22`.

## Risks / Open Questions

- If `URLSessionWebSocketTask.sendPing(pongReceiveHandler:)` invokes its callback more than once during connection abort, current `main` can still hit a `CheckedContinuation` double-resume fatal error in the keepalive path.
- Because there is no test coverage for this bridge, leaving the PR open means the crash path remains unverified and unprotected on `main`.
