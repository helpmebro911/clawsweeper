## Summary

Keep open. Current main contains partial Feishu streaming-card fixes, but it still does not close the specific late distinct-final case reported in #71977. Distinct final payloads coalesce only while the streaming session is still active; after `onIdle` closes the streaming card, only exact duplicate text is suppressed. A later final segment with different text still starts a new `FeishuStreamingSession`, matching the reported extra-card behavior.

## Best Possible Solution

Keep #71977 open and fix Feishu’s per-turn streaming-card lifecycle. Once a streaming card has been closed for a turn, the dispatcher should not call `startStreaming()` for later distinct final segments from the same turn; it should either suppress finals already represented by the card, append/update the existing card if Feishu supports it, or deliberately fall back to non-card text. Add a regression test for `onReplyStart`/partial streaming, `onIdle`, then two different final payloads, asserting only one streaming session/card is created.
