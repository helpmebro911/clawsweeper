## Summary

Keep #6731 open. Current main still documents and implements permissive host exec defaults unless operators opt into stricter exec policy/sandboxing, and it explicitly says there is no separate heuristic dangerous-command gate in YOLO mode. The Apr 26 maintainer-side Codex review comment in the provided issue context also explicitly kept this item open.

## Best Possible Solution

Keep #6731 open until maintainers either implement a native hard/default dangerous-operation gate in core exec/tool policy or explicitly decide that the current opt-in exec approvals, sandboxing, and tool-policy model is final. A concrete implementation path should cover default safe mode, Windows PowerShell destructive-command classification such as Remove-Item, approval UX, sandbox interaction, and regression tests/docs.
