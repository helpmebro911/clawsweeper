## Summary

Keep open. Current main has soft safety prompts and configurable hard controls, but it still documents host exec as no-approval by default and explicitly says prompt injection is not solved. The issue discussion asks for a native hard/default dangerous-operation gate, so this is still core safety/product scope rather than cleanup-closeable.

## Best Possible Solution

Keep #6731 open until maintainers either implement a native hard/default dangerous-operation gate or make an explicit product decision that documented opt-in exec approvals, tool policy, and sandboxing are the intended final model. A good implementation path would likely live in core exec/tool policy, with docs and tests covering defaults, dangerous command classification, approval behavior, and sandbox interaction.
