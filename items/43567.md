## Summary

Keep this issue open. Current main still does not provide the requested distinct system/runtime presentation for internal prompts such as the pre-compaction memory flush. The prior Codex review comment already identified the same gap, and current main at c2d82b87 still matches it: the memory flush prompt is submitted as a normal embedded prompt, chat.history lacks runtime/system-origin classification for visible user-role entries, Control UI system roles still fall through to the normal "other" bubble path, and the TUI still renders user-role history as user messages.

## Best Possible Solution

Keep this issue open. Implement a generic runtime/system-message presentation contract: tag runtime-originated transcript entries with stable provenance/kind or make internal-only turns use transcriptPrompt: "", project that metadata through chat.history, render it as a muted/badged system block in the Control UI, and route it through TUI addSystem rather than addUser. Add focused coverage for memory-flush projection, Control UI rendering, and TUI history loading.

## Risks / Open Questions

- Some narrower paths already suppress runtime-only context blocks, heartbeat prompts, and NO_REPLY outputs, so the visible leak can be intermittent; that does not satisfy the requested general distinct rendering treatment.
- A text-prefix filter for phrases like "Pre-compaction memory flush" would risk hiding legitimate user-authored content. The safer fix is structural provenance or the existing hidden transcriptPrompt/runtime-only path.
