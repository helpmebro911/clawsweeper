## Summary

Keep open. Current main has not implemented this PR's central plugin-config graceful-degradation change, and the PR is not safely closeable as already shipped or superseded. It also is not merge-ready as-is because the branch bundles unrelated fixes and has unresolved review concerns around media delivery, failed-registry caching, and fail-open runtime behavior.

## Best Possible Solution

Keep the PR open for maintainer judgment, but do not merge it as-is. The best path is to split out unrelated fixes, keep #70371/#61683 as the canonical product problem, and implement a focused plugin-local config failure policy: mark only the bad plugin failed/disabled, preserve clear doctor/status diagnostics, avoid caching transient failed registries incorrectly, and define which runtime paths must still fail closed for required or security-sensitive plugins.
