## Summary

Keep open. Current main still has the two strict plugin-loader settings this PR targets, so the plugin crash/degradation request is not implemented. But the PR is also not a safe cleanup close or obvious merge: `ensurePluginRegistryLoaded` is used for full runtime activation, not only validate/report paths, and the provided PR review context flags unresolved fail-open/cache risks plus unrelated changes.

## Best Possible Solution

Keep the PR open for maintainer-directed rework rather than closing it. The best path is to split unrelated commits, then solve #70371/#61683 with an explicit plugin-local degradation policy: fail soft for metadata/status or optional-plugin quarantine where intended, preserve fail-closed behavior for required/runtime-critical plugins, avoid caching failed full-runtime registries, and add focused regression tests for startup, CLI bootstrap, doctor/status output, and plugin diagnostics.
