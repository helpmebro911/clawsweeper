Keep this PR open. Current main still does not expose Slack Socket Mode ping/pong timeout config, and the PR is a plausible implementation candidate for the still-open timeout request, but it should be rebased and adjusted for the current Slack dependency/API surface before landing.

## Best Possible Solution
