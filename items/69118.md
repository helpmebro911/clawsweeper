## Summary

Keep #69118 open. Current main has a partial static-prompt hashing fix for volatile inbound metadata, but the specific group-channel failure remains: `extraSystemPromptStatic` still includes `groupIntro`, and `groupIntro` is intentionally present on the first group turn and absent on later turns, so `resolveCliSessionReuse` can still invalidate with `reason=system-prompt` on turn 2.

## Best Possible Solution

Keep the issue open and patch the group-channel path so `extraSystemPromptStatic` excludes turn-variable content such as `groupIntro` before hashing, or otherwise make CLI session reuse tolerant of expected prompt drift. Add a focused regression test that simulates a group-channel first turn followed by a second turn and asserts the CLI session is reused instead of invalidated with `reason=system-prompt`.
