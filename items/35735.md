## Summary

Keep #35735 open. Current main still registers `openclaw sessions` for listing plus `sessions cleanup` only, and there is no `src/commands/sessions-delete.ts` or equivalent `rm`/`clear` command module. Gateway/UI per-key deletion exists, but it does not provide this PR's offline store-targeted `openclaw sessions rm <key>` and bulk `openclaw sessions clear --older-than ...` workflow with `--store`, `--agent`, `--all-agents`, `--dry-run`, and `--json`.

## Best Possible Solution

Keep #35735 open for maintainer judgment. The best path is a rebased, focused implementation of `openclaw sessions rm <key>` and `openclaw sessions clear` that uses safe store mutation semantics, avoids unrelated maintenance pruning, addresses the review comments, documents the public CLI at https://docs.openclaw.ai/cli/sessions, and adds targeted registration/command tests. If maintainers reject this command shape, replace it with a narrower canonical design issue rather than auto-closing this PR as stale.
