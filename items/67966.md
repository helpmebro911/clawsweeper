## Summary

Current main still does not implement the requested local-managed Playwright navigation interception. Local-managed profiles keep `usesPersistentPlaywright: false`, so `openTab` only reaches `createPageViaPlaywright` for remote CDP profiles; explicit strict SSRF mode still requires inspectable redirect-hop handling for HTTP(S) URLs and throws before the raw CDP fallback can open them.

## Best Possible Solution

Keep this issue open and implement a focused browser plugin change in `extensions/browser`: route local-managed OpenClaw profiles through the existing Playwright-backed `createPageViaPlaywright` and `gotoPageWithNavigationGuard` path when strict SSRF policy needs redirect-hop inspection, while preserving existing-session Chrome MCP behavior and safe raw-CDP fallbacks where Playwright is unavailable. Add local-managed strict-mode regression tests for hostname and IP-literal navigation and update browser docs if the capability matrix changes.
