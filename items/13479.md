## Summary

Keep #13479 open. Current main still only partially satisfies the request: cron job cards show the prompt/system payload inline, but clicking a job still selects run history rather than revealing a job-detail view, and the job `description` summary is not rendered in the cron dashboard.

## Best Possible Solution

Keep #13479 open. The best implementation path is to add selected-job detail or expansion state in `ui/src/ui/views/cron.ts`, render `job.description` as the human-readable summary alongside the existing prompt/system payload display, preserve the existing run-history controls, add English i18n strings, and cover the behavior in focused `ui/src/ui/views/cron.test.ts` tests.
