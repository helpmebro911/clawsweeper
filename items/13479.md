## Summary

Keep #13479 open. Current main still renders the cron prompt/system payload inline in the jobs list, but clicking a job is wired to load run history rather than reveal a job detail/summary surface, and the persisted `description` summary is not rendered on the main cron dashboard job card.

## Best Possible Solution

Keep the issue open. The likely maintainer path is to add explicit selected-job detail or expansion state in `ui/src/ui/views/cron.ts`, render `job.description` as the human-readable summary alongside the existing prompt/system payload, preserve run-history controls, add English i18n strings, and cover the behavior with focused `ui/src/ui/views/cron.test.ts` tests.
