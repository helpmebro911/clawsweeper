## Summary

Keep #13479 open. Current main partially shows cron payload text inline, and the cron model already persists an optional description, but the Web UI still has no click-to-reveal job detail/summary surface and does not render `job.description` on the dashboard job card.

## Best Possible Solution

Keep the issue open. The likely implementation belongs in `ui/src/ui/views/cron.ts` and related Control UI state: add explicit selected-job detail or expansion behavior, render `job.description` as the human-readable summary alongside the existing prompt/system payload, preserve run-history controls, add English i18n strings, and cover it with focused `ui/src/ui/views/cron.test.ts` assertions.
