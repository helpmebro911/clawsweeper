## Summary

Keep #44683 open. Current main does not contain a Helm chart, and the PR is a coherent additive deployment feature rather than an implemented, duplicate, ClawHub, or non-repo item. The current Kubernetes docs explicitly choose Kustomize as the supported in-repo path and say Helm can be layered on top of those manifests, so accepting this PR still needs maintainer product judgment plus fixes for the review issues already raised on the PR.

## Best Possible Solution

Keep the PR open for maintainer judgment. If OpenClaw wants an in-repo Helm path, update the chart to resolve the review findings, add Helm lint/template validation in the repo’s accepted check lane, and document it from docs/install/kubernetes.md. If maintainers want to stay Kustomize-only, close manually with that product decision and preferably point contributors to maintain an external chart layered on the published manifests.
