## Summary

Keep #7433 open. Current main still lacks the requested WhatsApp/Baileys reliability hooks in the bundled WhatsApp plugin: no socket-level `getMessage` or `cachedGroupMetadata`, no proactive group metadata event maintenance, and no `lid-mapping.update` persistence path. The related closed #66917 is broader WhatsApp stability work, not a canonical replacement for this narrower implementation request.

## Best Possible Solution

Keep the issue open and implement the reliability work inside the bundled `@openclaw/whatsapp` plugin: add bounded retry message storage and wire it through Baileys `getMessage`, expose a shared group metadata cache through `cachedGroupMetadata`, seed and maintain it from connect hydration plus `groups.upsert`/`groups.update`/`group-participants.update`, and decide whether Baileys `signalRepository.lidMapping` is enough or `lid-mapping.update` should persist reverse mappings. Add focused WhatsApp plugin tests for socket options, retry lookup, group cache updates, participant updates, and LID mapping behavior.
