## Summary

Current `main` has Telegram pin/unpin transport helpers and `delivery.pin` support for sent messages, but the Telegram message tool still does not advertise or dispatch manual `pin`/`unpin` actions. The feature request is therefore not implemented on `main` and should stay open.

## Evidence

- **Telegram discovery omits pin/unpin:** `describeMessageTool()` only returns `send`, `poll`, `react`, `delete`, `edit`, `sticker`, `topic-create`, and `topic-edit`; no `pin` or `unpin` actions are exposed for Telegram.
  - file: [extensions/telegram/src/channel-actions.ts:124](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/telegram/src/channel-actions.ts#L124)
  - command: `nl -ba extensions/telegram/src/channel-actions.ts | sed -n '35,170p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Telegram runtime aliases omit pin/unpin:** The Telegram action alias map accepts `delete`, `edit`, `poll`, `react`, `send`, `sticker`, and topic actions, but not `pin` or `unpin`, so the shared runner cannot dispatch those actions through the Telegram plugin.
  - file: [extensions/telegram/src/action-runtime.ts:59](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/telegram/src/action-runtime.ts#L59)
  - command: `nl -ba extensions/telegram/src/action-runtime.ts | sed -n '35,120p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Contract test expects no pin/unpin support:** The Telegram channel-actions contract test still expects only `send`, `poll`, `react`, `delete`, `edit`, `topic-create`, and `topic-edit`. If pin/unpin were implemented for the message tool, this test would need to include them.
  - file: [extensions/telegram/src/channel-actions.contract.test.ts:19](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/telegram/src/channel-actions.contract.test.ts#L19)
  - command: `nl -ba extensions/telegram/src/channel-actions.contract.test.ts | sed -n '1,80p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Only sent-message delivery pin is wired:** Telegram outbound declares `deliveryCapabilities.pin` and `pinDeliveredMessage`, and the action-runtime tests cover `delivery.pin` on `sendMessage`. That supports pinning a newly sent message, but not manual `pin`/`unpin` actions on existing messages.
  - file: [extensions/telegram/src/outbound-adapter.ts:134](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/telegram/src/outbound-adapter.ts#L134)

## Risks / Open Questions

- `delivery.pin` partially overlaps the reporter's pinning use case for newly sent messages, but unpinning and manual pin/unpin actions for existing messages are still absent; closing as implemented would be premature.
