## Summary

Keep #30518 open. Current main still has Telegram delivery pinning for newly sent messages and low-level `pinMessageTelegram`/`unpinMessageTelegram` helpers, but the bundled Telegram message action surface does not advertise, map, or dispatch manual `pin`/`unpin` actions for existing messages.

## Best Possible Solution

Keep the issue open. Implement this in the bundled Telegram plugin by adding `pin` and `unpin` to Telegram message action discovery and `TELEGRAM_MESSAGE_ACTION_MAP`, adding runtime aliases and branches that call `pinMessageTelegram` and `unpinMessageTelegram`, deciding whether a `channels.telegram.actions.pins` gate/config-doc entry is needed, adding adapter/runtime contract tests, and updating Telegram docs to distinguish manual pin/unpin from send-time `delivery.pin`.
