## Summary

Keep #30518 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f has Telegram send-time delivery pinning and low-level Telegram pin/unpin helpers, but it still does not expose manual Telegram `pin` or `unpin` message actions. The Telegram action adapter does not advertise or map those actions, and the runtime still rejects unsupported Telegram actions before any helper can run.

## Best Possible Solution

Keep the issue open. Implement this in the bundled Telegram plugin by adding `pin` and `unpin` to Telegram message action discovery and `TELEGRAM_MESSAGE_ACTION_MAP`, adding runtime aliases and branches that call `pinMessageTelegram` and `unpinMessageTelegram`, deciding whether explicit Telegram action gates are needed in config/schema/docs, adding focused adapter/runtime contract tests, and updating the message docs to distinguish manual pin/unpin from send-time `delivery.pin`.
