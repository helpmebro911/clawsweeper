## Summary

Keep this PR open. Current main already reshaped the exec PATH fallback and cron deadlock fixes, and docs cover WSL2 remote CDP troubleshooting, but the PR still contains useful browser status/error-diagnostics work that is not implemented in the current `extensions/browser` plugin layout. It needs a rebase or split rather than closure.

## Best Possible Solution

Keep the PR open, but require a rebase or split. Drop or reconcile the exec and cron portions already solved on main, then port the still-missing browser layered status diagnostics and bounded HTML upstream-error summarization into `extensions/browser`. Address the two existing review comments and verify with focused browser CLI/client/server tests plus a security review of status/config and error-reflection paths.

## Risks / Open Questions

- Merging the PR as-is would be stale against the current `extensions/browser` plugin layout and current exec `host=auto` plus cron `cron-nested` semantics.
- The remaining browser diagnostics work reads gateway config state and reports status to operators; it must keep secrets redacted and use a short timeout for best-effort config reads.
- HTML upstream error summarization must be bounded and sanitized so untrusted proxy markup or large response bodies are not reflected into logs, CLI output, or agent-visible errors.
- The PR touches command execution and scheduling surfaces; obsolete exec and cron hunks should be dropped during rebase instead of reintroduced.
