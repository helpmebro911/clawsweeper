## Summary

This issue should stay open. Current main still does not register an agent-facing `feishu_bitable_delete_record` tool. The bundled Feishu Bitable tool family exposes metadata, list/get/create/update record, create app, and create field operations, while record deletion is only used internally for cleanup after new Bitable creation. The related open PRs are implementation candidates, not merged fixes.

## Best Possible Solution

Keep this issue open as the canonical tracker. The likely fix belongs in `extensions/feishu/src/bitable.ts`: add a typed `feishu_bitable_delete_record` helper and schema around the existing Lark/Feishu `client.bitable.appTableRecord.delete` API, register it through the existing Bitable tool helper, preserve `accountId` and contextual `agentAccountId` routing, return a structured success result, and add focused Feishu plugin tests. Review or consolidate #40697 and #41368 rather than closing this before a fix merges.
