## Summary

Keep open. Current main has metadata for avatar resolution and local/data support, but the reported external URL path is still intentionally blocked in the Control UI rendering policy, and direct non-local `/avatar/<agent>` requests still return not found. The related PR #41239 proposed serving remote/data avatars but was closed unmerged, so it does not supersede or implement the issue.
