## Summary

Keep open. Current main has adjacent Telegram retry and polling hardening, including strict safe retry classification for send helpers, but the bot reply delivery path implicated by the issue still sends final reply chunks through direct `bot.api.sendMessage` calls. A current test explicitly expects a network send failure in that path to reject, which matches the reported silent/truncated reply failure mode rather than proving it fixed.
