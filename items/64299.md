## Summary

Keep this issue open. Current main still routes Feishu document write/append through `document.convert`, but public setup guidance and the manual credential prompt still omit the reporter-verified `docx:document.block:convert` scope. A related descendant payload normalization concern also remains on the small descendant insert path.

## Best Possible Solution

Keep this issue open and make a focused Feishu plugin cleanup: update public Feishu setup docs and manual credential help with the document-writing scopes, explicitly note that the Feishu app must be republished after scope changes, add a clearer diagnostic for `document.convert` permission failures if feasible, and fix the small descendant insertion mapper so normalized `children` cannot be overwritten, with a narrow regression test.

## Evidence

- **Reporter-verified root cause remains relevant:** The issue discussion says adding `docx:document.block:convert` in Feishu Open Platform and publishing a new app version made `write` and `append` succeed.
