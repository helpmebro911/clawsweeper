## Summary

Keep this PR open. Current main partially implements CJK-aware token estimation through the shared helper, but it has not fully absorbed this PR's stated scope: soft-trim eligibility still uses raw UTF-16 length, and assistant tool-call argument JSON still uses raw `.length`. The PR itself is also stale against the active `pi-hooks` path, so it needs a rebase or maintainer replacement patch rather than closure as implemented.

## Best Possible Solution

Keep this PR open, or replace it with a rebased maintainer patch against `src/agents/pi-hooks/context-pruning/pruner.ts`. The best fix should keep the shared `src/utils/cjk-chars.ts` helper, use weighted estimation for assistant tool-call argument JSON, and make soft-trim units consistent with regression tests for CJK-heavy tool results.
