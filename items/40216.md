## Summary

Keep open. Current `main` has partial CJK-aware token estimation via `src/utils/cjk-chars.ts`, but the context pruner still has unresolved non-Latin accounting gaps, so this PR is not obsolete and should not be conservatively closed as already implemented.

  const NON_LATIN_RE = /[\u2E80-\u9FFF\uA000-\uA4FF\uAC00-\uD7AF\uF900-\uFAFF\u{20000}-\u{2FA1F}]/gu;
  const nonLatinCount = (text.match(NON_LATIN_RE) ?? []).length;
  const CJK_SURROGATE_HIGH_RE = /[\uD840-\uD87E][\uDC00-\uDFFF]/g;
  const cjkSurrogates = nonLatinCount === 0 ? 0 : (text.match(CJK_SURROGATE_HIGH_RE) ?? []).length;
  const codePointLength = text.length - cjkSurrogates;
  return codePointLength + nonLatinCount * 3;
};
function estimateJoinedTextLength(parts){let len=0; for(const p of parts) len += p.length; len += Math.max(0, parts.length-1); return len;}
