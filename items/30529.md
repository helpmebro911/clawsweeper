## Summary

Keep #30529 open. Current main has internal Discord typing support for inbound replies and heartbeat runs, but it still does not expose a Discord typing indicator as an agent-callable message-tool action. The requested `message(action="typing"...)` / `typing-indicator` capability is absent from the shared action list, agent tool schema, CLI action normalization, Discord action discovery, Discord action handler, docs, and Discord skill guidance.

## Best Possible Solution

Keep the issue open until maintainers either add a shared `typing` / `typing-indicator` message-tool action or explicitly decide automatic reply/heartbeat typing is sufficient. A concrete implementation should add the shared action name and target mode, expose any required schema fields, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to the existing Discord typing sender with account and target resolution, and update `docs/cli/message.md`, `docs/channels/discord.md`, and `skills/discord/SKILL.md`. If cross-provider parity is desired, design the action so Telegram, Slack, Discord, and similar plugins can opt in cleanly.
