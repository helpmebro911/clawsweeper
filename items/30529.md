## Summary

Current main still does not implement the requested Discord typing indicator as an agent-callable message-tool action. Discord has internal typing support for inbound replies and heartbeat runs, but `typing` / `typing-indicator` is absent from the shared message action list, agent tool schema, CLI normalization, Discord action discovery, Discord dispatcher/runtime, docs, and the Discord skill guidance.

## Best Possible Solution

Keep #30529 open until maintainers either add a shared `typing` / `typing-indicator` message-tool action or explicitly decide automatic typing is sufficient. A concrete implementation should add the shared action name and target semantics, define target mode/schema behavior, advertise it from Discord `describeMessageTool`, route it through `handleDiscordMessageAction` to the existing Discord typing sender with account and target resolution, and update `docs/cli/message.md`, `docs/channels/discord.md`, and `skills/discord/SKILL.md`. If cross-provider parity is desired, design the action so Telegram, Slack, Discord, and similar plugins can opt in cleanly.
