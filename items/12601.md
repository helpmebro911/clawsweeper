## Summary

Keep #12601 open. Current main still lacks Slack Canvas content reading: the bundled Slack plugin exposes no `read-canvas` action, dispatches no Canvas API call, has no `canvases:read` setup/docs/config gate, and current tests lock the existing Slack action surface. The related PR #18752 was a useful prior implementation attempt, but the provided GitHub context shows it was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep this open for the bundled Slack plugin. A clean fix should live mostly in `extensions/slack`: add a gated Canvas read action or dedicated Slack Canvas tool, add `canvases:read` to setup/docs/config/schema, call the appropriate Slack Canvas read API through the existing account/token model, return bounded markdown/section data safely, update the shared action contract if the message action surface is used, and add action discovery, runtime dispatch, unit/contract tests, docs, and live Slack validation.
