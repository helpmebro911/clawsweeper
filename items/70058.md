## Summary

Keep #70058 open. Current main still has no opt-in polling fallback for the skills watcher: `ensureSkillsWatcher()` builds a chokidar watcher with native defaults only, the watcher state/reuse key has no polling fields, the colocated tests do not cover polling env vars, and the skills docs only document `watch` and `watchDebounceMs`. The PR addresses a plausible chokidar-supported path with tests/docs and follow-up fixes for review comments, but it should be cleaned before merge because the current diff also includes an unrelated QA scenario change.

## Best Possible Solution

Keep the PR open and ask for a clean, skills-only update: retain the opt-in `OPENCLAW_SKILLS_WATCH_POLLING` / interval wiring, interval validation, watcher reuse-key coverage, and docs; remove or split the unrelated QA scenario change; then run the targeted skills watcher tests and the changed gate before maintainer merge.
