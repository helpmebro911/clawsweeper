## Summary

Keep this PR open. Current main still archives the state directory without an exclusion filter for extension-local node_modules, and the open broader exclude-pattern PR does not clearly supersede this default behavior. The PR remains a focused implementation candidate with review feedback already addressed in the provided discussion.

## Best Possible Solution

Keep this PR open for maintainer review. Either land or recreate the narrow default exclusion with canonical path handling, Windows path normalization, and regression coverage, plus backup/restore docs if maintainers want this default; or explicitly fold the same extensions/*/node_modules behavior into the broader backup exclude-pattern work before closing this PR.
