## Summary

Keep PR #45808 open. Current main still does not implement explicit `agentId` binding on Discord accounts or routing-layer synthesis from `channels.discord.accounts[*].agentId`; users must still write explicit `bindings[].match.accountId` entries. The PR contains useful unmerged work, but the latest review context still raises implementation concerns that should be resolved before merge rather than closing it as obsolete.

## Best Possible Solution

Keep the PR open and ask for an update/rebase against current main. The best product path is to implement the feature as an explicit routing-layer synthesis step, with Discord schema/types/generated metadata/docs updated, explicit bindings preserving precedence, scoped and wildcard semantics aligned with `resolve-route`, tokenless or disabled accounts excluded from outbound-selectable synthesized bindings, status snapshots actually surfacing the effective account `agentId`, and focused tests for `listBindings`, cache invalidation, status snapshots, and tokenless-account behavior.
