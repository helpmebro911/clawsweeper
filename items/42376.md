## Summary

Keep open. The report targets a custom `QQchat`/NapCat plugin that is not present in the `openclaw/openclaw` checkout, so I could not verify the claimed abort path against repo code. Current main does expose the two core controls that would address the described overlap class in supported channels: DM session isolation per account (`per-account-channel-peer`) and non-interrupt queueing by default (`collect`). That is not enough to prove the external plugin issue is implemented on main or obsolete.
