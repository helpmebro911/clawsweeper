## Summary

Keep open. Current main still has the reported slug-generator path: session-memory enables LLM slug generation by default, `generateSlugViaLLM` calls `runEmbeddedPiAgent` on `temp:slug-generator` without `authProfileFailurePolicy: "local"`, and the embedded runner persists classified non-timeout failures such as billing into shared auth profile state. The existing classifier intentionally treats Anthropic extra-usage messages as billing, and billing failures still default to a roughly 5-hour `disabledUntil` window.

## Best Possible Solution

Keep the issue open. Fix the session-memory slug generator by passing `authProfileFailurePolicy: "local"` or an equivalent helper-local failure scope to its `runEmbeddedPiAgent` call, then add regression coverage proving slug-generator billing/auth failures fall back to timestamp/null without writing shared auth-profile failure state. Preserve the existing billing classifier for user-facing/provider runs unless maintainers make a broader product decision.
