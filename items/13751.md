## Summary

Keep #13751 open. Current main still makes Feishu sender-name lookup contact-backed by default, still documents/tests `resolveSenderNames` as default-on, and still routes live peer listing through `client.contact.user.list` with static fallback disabled for channel-list actions. The latest main commit after the prior April 26 review is WhatsApp-only, so the Feishu surface has not changed since that keep-open analysis.

## Best Possible Solution

Keep #13751 open. Make basic Feishu inbound bot operation contact-free by default: use webhook sender fields when available, otherwise attribute with open_id/user_id without calling `contact.user.get`; make profile lookup an explicit opt-in. For directory peers, prefer static configured peers unless the operator explicitly enables live contact-directory lookup. Update Feishu docs, config metadata, and regression tests around the new default.
