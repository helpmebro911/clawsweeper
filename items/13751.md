## Summary

Keep open. Current main still resolves inbound Feishu sender names through `client.contact.user.get()` by default, with `resolveSenderNames` defaulting to true in schema, tests, and docs. Live peer listing also still uses `client.contact.user.list()`. Main has mitigations for the obsolete `contact:contact.base:readonly` scope string, but the central request to avoid contact/profile-directory APIs for basic bot operation remains unresolved.

## Best Possible Solution

Keep #13751 open. The best product/code path is to make basic Feishu inbound operation avoid contact/profile-directory APIs by default: use webhook sender data when present, otherwise attribute messages with `open_id`/`user_id` without calling `contact.user.get`; make display-name lookup an explicit opt-in. Live directory peer listing should also be static-first or explicitly opt-in for contact-directory access, with setup/docs/tests updated to reflect the new default.
