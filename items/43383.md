## Summary

Current main still appears to have the reported sandbox skill prompt mismatch. The non-main sandbox workspace sync exists, but attempt and compaction runs still pass a host-built skills snapshot through unchanged; when that snapshot contains resolved skills, live sandbox skill loading is skipped and the snapshot prompt wins. Related PR #38789 remains open as the focused implementation candidate, while #43526 and #51471 were closed unmerged.

## Best Possible Solution

Keep this issue open and review or land PR #38789, or implement the same fix directly on main: for sandboxed non-rw attempt and compaction runs, rebuild or remap the available-skills prompt from sandbox-readable skill entries instead of returning the host-built snapshot prompt. Preserve skill filters, agent allowlists, environment override behavior, remote eligibility notes, and empty-sandbox behavior, then add regression coverage for snapshot-present attempt and compaction runs.
