## Summary

Keep open. Current `main` still shows the Nostr gateway lifecycle bug this PR is trying to fix, but the patch as proposed is not high-confidence: its `subscribeMany()` change conflicts with current in-repo usage and the review discussion flagged it against the `nostr-tools` API, and its new `sendMedia` fallback would change Nostr from dropping media URLs to inlining them into DM text.

## Evidence

- **Gateway lifecycle bug still exists on main:** The channel manager treats `gateway.startAccount()` as the long-lived runtime task and stores the returned promise for restart/stop handling. Tests model correct plugins as staying pending until `abortSignal` fires. Nostr still returns immediately with a `{ stop() }` object instead, so the lifecycle problem described by the PR still appears real on current `main`.
