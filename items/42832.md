## Summary

Keep this PR open. Current main still contains the two Nostr DM delivery failures the PR targets: `startNostrGatewayAccount` resolves immediately, which the gateway manager treats as a stopped/crashed channel and auto-restarts, and `startNostrBus` still passes an array where `nostr-tools@2.23.3` expects a single `Filter`. The PR is useful but not merge-ready because its `sendMedia` fallback is now partly stale and can expose raw media paths in DMs.

## Best Possible Solution

Keep this PR open as a useful but not merge-ready implementation candidate. The maintainer path should be a refreshed Nostr-owned patch on current main: make `startNostrGatewayAccount` stay pending until `ctx.abortSignal` and close idempotently, pass a single `Filter` object to `subscribeMany`, add focused lifecycle and real subscription-shape regression coverage, and drop the proposed `sendMedia` fallback unless it is restricted to safe public HTTP(S) URLs. The open Nostr restart-loop report #53858 should remain the canonical current user-facing bug until a corrected fix lands.
