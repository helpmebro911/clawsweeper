## Summary

Keep open. Current main still has the Nostr gateway lifecycle and DM subscription-shape failures this PR targets, but the PR is stale against the current Nostr file layout and its sendMedia fallback is a security-sensitive change that should not merge as-is.

## Best Possible Solution

Keep this PR open as a stale but useful implementation candidate. The maintainer path is a refreshed Nostr-owned patch on current main: make startNostrGatewayAccount stay pending until ctx.abortSignal with idempotent cleanup, pass a single Filter object to subscribeMany, add focused lifecycle plus real subscription-shape regression coverage, and leave media unsupported unless a safe public-URL-only media design is explicitly added. #53858 should remain the canonical user-facing bug until that corrected fix lands.
