## Summary

Keep open. Current `main` does not implement `gateway.handshakeTimeoutMs`, and the PR as described is still incomplete against current `main` because the strict gateway config schema and label/help parity do not define that field.

## Evidence

- **Current gateway timeout logic is env-only:** `getPreauthHandshakeTimeoutMsFromEnv()` reads `OPENCLAW_HANDSHAKE_TIMEOUT_MS` (and a test-only fallback) and otherwise returns the default; there is no config-file lookup on current `main`. `ws-connection.ts` uses that env-only helper for each pre-auth socket.
  - file: [src/gateway/handshake-timeouts.ts:37](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/gateway/handshake-timeouts.ts#L37)

  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Gateway config type lacks the requested field:** `GatewayConfig` currently ends at `channelMaxRestartsPerHour?: number;` and does not expose `handshakeTimeoutMs`.
  - file: [src/config/types.gateway.ts:447](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/config/types.gateway.ts#L447)
