## Summary

Keep this PR open. Current main still lacks the standardized fs permission_denied helper and write/edit mutation payload guards, and this PR is paired with the same author's open dependent #67332. The PR is a plausible security-hardening implementation candidate, but the current diff has unresolved review blockers around sandbox mount-boundary classification and missing-file read semantics.

## Best Possible Solution

Keep this PR open and request changes. The maintainer path is to fix the classifier for "escapes allowed mounts", preserve ENOENT/path-miss semantics for sandbox reads, add focused regression coverage for both, confirm the write/edit payload limits are acceptable, and then review this PR together with dependent #67332.
