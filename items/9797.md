## Summary

Keep #9797 open. Current main still lacks the requested agent-facing `queue_status` tool and lacks a stable queue snapshot RPC/CLI surface that exposes per-lane queue depth, active runs, max concurrency, and per-agent workload. The related #43618 proposal is relevant but, per the provided PR context, remains open and explicitly does not add the in-agent tool requested here.

## Best Possible Solution

Keep this issue open and implement a shared read-only queue snapshot provider in core. Expose it through the requested agent-facing `queue_status` tool and, ideally, the operator CLI/RPC surfaces discussed in the comments, with tool catalog/docs/tests updated together. If #43618 becomes the canonical queue snapshot backend, this issue can then track the remaining agent-tool consumer or close once that consumer ships.
