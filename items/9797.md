## Summary

Keep #9797 open. Current main still lacks an agent-facing `queue_status` tool, the existing session/agent tools do not expose per-lane queue depth or per-agent workload, and no merged gateway/CLI diagnostics queue snapshot exists. The issue discussion also already includes an April 26, 2026 Codex/maintainer comment keeping this open and noting that related PR #43618 is only an open operator-diagnostics proposal, not the requested in-agent tool.

## Best Possible Solution

Implement a shared read-only queue snapshot provider in core, then expose it through an agent-facing `queue_status` tool with catalog/profile/policy registration, permissions, docs, and focused tests. If PR #43618 or equivalent operator diagnostics lands first, reuse its snapshot provider and let #9797 track the remaining in-agent tool surface.
