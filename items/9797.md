## Summary

Keep #9797 open. Current main still does not provide an agent-facing `queue_status` tool or an equivalent stable queue snapshot surface for agents, and the related open PR #43618 is explicitly scoped to operator diagnostics rather than the requested in-agent dispatch tool.

## Best Possible Solution

Implement a shared read-only queue snapshot provider in core, then expose it through an agent-facing `queue_status` tool with tool catalog/profile/policy registration, permissions, docs, and focused tests. If the operator diagnostics PR lands first, reuse that backend and keep this issue focused on the remaining in-agent dispatch surface.
