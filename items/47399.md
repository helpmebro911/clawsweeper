## Summary

Keep open. Current main still has the segment-specific duplicate-bubble path this PR is trying to fix: tool events commit the full cumulative `chatStream` into `chatStreamSegments`, and the renderer later displays those stored segments plus the live cumulative stream separately. PR #47399 contains useful coverage for both segment storage and live-stream rendering, but it is not merge-ready because it includes an unrelated binary workspace artifact and has an unresolved whitespace-only delta review issue.

## Best Possible Solution

Keep this open and consolidate the #47188 fix across the duplicate PRs. The best maintainer path is to implement a current-main version of the committed-length/offset model in `ui/src/ui/app-tool-stream.ts` and `ui/src/ui/chat/build-chat-items.ts`, preserve whitespace-only deltas or avoid advancing past skipped text, add focused regression tests for text->tool->text and whitespace separators, and remove the unrelated workspace image before merge.
