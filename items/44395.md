## Summary

Current main still does not implement the central request. Active memory indexing still chunks Markdown by size using only token and overlap settings, so adjacent `##`/`###` sections can still be blended into one searchable chunk. MMR is present as a partial stretch item, and memory-wiki can import/compile source pages, but neither replaces heading-aware active-memory chunking nor automatically extracts imported sources into entities, concepts, or syntheses.

## Best Possible Solution

Keep this issue open. The best maintainer path is to implement heading-aware or section-aware chunking in the active memory-search indexing path, including config/schema/help support, reindex metadata so strategy changes rebuild the index, and focused tests proving separate `##`/`###` sections do not blend into one embedding/FTS chunk. Treat entity extraction and graph memory as a separate memory-core or memory-wiki design track unless maintainers want this umbrella to remain the canonical tracker.
