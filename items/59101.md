## Summary

Current main still leaves `openclaw memory index --force` and `memory status --index` on watcher-backed manager paths. The builtin backend creates a non-status memory manager that starts Chokidar, and the QMD backend creates a full manager that also starts Chokidar. The provided related context shows PR #49851 is still open and unmerged with the direct one-shot CLI manager proposal, so this issue should remain open for that fix or an equivalent implementation.

## Best Possible Solution

Keep this issue open and review, rebase, or replace PR #49851 with an equivalent watcher-free one-shot memory manager mode. The fix should route `memory index` and `memory status --index` through that mode for both builtin and QMD backends, preserve normal Gateway/runtime watching, and add regression tests proving the one-shot CLI paths do not start Chokidar while long-lived managers still watch normally.
