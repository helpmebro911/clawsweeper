## Summary

Keep open. Current `main` still routes `openclaw memory index` through the full memory-manager path, and both builtin and QMD managers enable filesystem watchers by default on that path. That matches the report's cross-backend `EMFILE ... watch` theory, and I did not find a corresponding fix on `main`. The incidental QMD same-name collection warning mentioned in the issue does appear fixed separately, but that does not resolve the watcher crash itself.
