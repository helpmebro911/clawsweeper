## Summary

Current main partially addresses the report for message lifecycle hooks, but the requested managed-hook coverage is not implemented for before_dispatch or agent_end. message_received and message_sent now bridge into internal hooks, while before_dispatch and agent_end still run only through the plugin hook runner. This leaves a meaningful requested capability missing, so this should stay open for maintainer judgment rather than be auto-closed.

## Evidence
