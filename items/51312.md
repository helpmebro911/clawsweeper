## Summary

Keep open. Current `main` still treats `userTimezone` as a global default only; there is no per-agent `agents.list[].userTimezone` field, and runtime consumers continue to read `cfg.agents?.defaults?.userTimezone` directly.

## Evidence

- **Per-agent agent config has no `userTimezone` field:** `AgentConfig` includes per-agent overrides like `model`, `heartbeat`, `sandbox`, and `tools`, but no `userTimezone`. The global default type still defines `userTimezone` under `AgentDefaultsConfig`.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/config/types.agents.ts#L70)
  - command: `nl -ba src/config/types.agents.ts | sed -n '1,220p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Schema would reject `agents.list[].userTimezone`:** `AgentEntrySchema` is `.strict()` and lists no `userTimezone` property, so a per-agent config key of that name is not supported by the current config schema.
  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/config/zod-schema.agent-runtime.ts#L812)

  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Only global default is documented/generated:** Docs and generated schema expose `agents.defaults.userTimezone`; no corresponding `agents.list[].userTimezone` path exists.
  - file: [docs/gateway/config-agents.md:267](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/gateway/config-agents.md#L267)
