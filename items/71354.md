## Summary

Keep PR #71354 open. Current main still short-circuits the Tailscale hostname lookup whenever `gateway.tailscale.mode` is `off`, so the linked status bug is not implemented on main even though the display layer already knows how to render an off-mode DNS suffix. The PR is a focused fix to the implicated code path, and the provided discussion shows the author addressed the cold-start regression review by preserving `skipColdStartNetworkChecks`.

## Best Possible Solution

Keep PR #71354 open and either land it after maintainer review or apply the same focused change on main: in `src/commands/status.scan.bootstrap-shared.ts`, gate `tailscaleDnsPromise` on `skipColdStartNetworkChecks` rather than `tailscaleMode === "off"`, and add a colocated unit test proving normal off-mode scans call `getTailnetHostname` while cold-start fast-path scans skip it. Maintainers should also decide whether the expanded local Tailscale CLI execution needs an absolute-binary or explicit opt-in safeguard.
