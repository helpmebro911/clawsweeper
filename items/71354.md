## Summary

Keep PR #71354 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still short-circuits Tailscale hostname detection when `gateway.tailscale.mode` is `off`, so the PR's requested runtime-state detection is not implemented on main. The related discussion is meaningful: review feedback about the cold-start regression was addressed in the PR's follow-up commit, but remaining security/product concerns and test coverage still need maintainer review rather than auto-closure.

## Best Possible Solution

Keep #71354 open. The best path is to update the PR for current main, preserve `skipColdStartNetworkChecks`, add regression coverage for off-mode runtime lookup, and decide explicitly whether normal `openclaw status` and `status --all` should show DNS/daemon state when exposure mode is off before landing.
