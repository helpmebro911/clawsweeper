## Summary

Keep PR #71354 open. Current main still has the exact config-mode guard the PR removes, so `openclaw status` cannot populate `tailscaleDns` when `gateway.tailscale.mode` is off. The downstream status formatting already supports displaying `off · <tailnet host>` when a DNS name is supplied, and the PR head directly targets that missing runtime detection while preserving the cold-start skip after review feedback.

## Best Possible Solution

Keep the PR open. The best path is to merge or adapt this focused bootstrap change after adding targeted regression tests for mode-off runtime detection and cold-start skipping, and after a maintainer explicitly accepts or adjusts the privacy/PATH behavior raised in review.
