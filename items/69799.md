## Summary

Keep this issue open. Current main still documents and implements an external Node-backed macOS Gateway path, with launchd program arguments ultimately running a Node executable plus the OpenClaw CLI entrypoint. The newer macOS app-owned TCC and PeekabooBridge work mitigates some app/UI automation surfaces, but it does not ship a dedicated or bundled Node runtime for the Gateway or legacy iMessage/FDA process-context paths called out here.

## Best Possible Solution

Keep this issue open as core macOS security architecture work. The best path is to implement and document an OpenClaw-scoped TCC runtime boundary, such as a signed app-owned Gateway/runtime or a dedicated installed Node binary with stable identity, then coordinate it with the open #7227 security thread and the narrower docs/stable-path work from #69561 and #22179.
