## Summary

Keep #69799 open. Current main has useful partial macOS app/TCC work, but it does not ship the requested dedicated or bundled Node runtime for the Gateway. The documented and implemented local Gateway path still depends on an external global CLI plus a user/system Node binary, and iMessage docs still require Full Disk Access for the process context running OpenClaw/imsg.

## Best Possible Solution

Keep the issue open or explicitly consolidate it into a maintainer-owned macOS TCC hardening tracker. The likely product path is a signed OpenClaw-owned runtime boundary: either a bundled app-owned Gateway/runtime or a dedicated signed Node executable/path, with launchd install changes, migration/doctor checks, release packaging updates, and docs for iMessage/FDA and Accessibility.
