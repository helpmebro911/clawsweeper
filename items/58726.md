## Summary

Keep open. Current `main` still implements generic same-tool/same-params loop detection as warning-only, and only hard-blocks on separate critical detectors such as known poll/no-progress, ping-pong, unknown-tool repeats, or the 30-call global no-progress breaker. That does not satisfy this issue's request to hard-abort repeated identical tool calls like `session_status` after a lower threshold.

## Evidence

- **Generic repeat detector is still warn-only:** `detectToolCallLoop()` explicitly labels the generic same-tool/same-params path as warn-only and returns `level: "warning"` once `recentCount >= warningThreshold`; there is no generic critical branch using `criticalThreshold`.
  - file: [src/agents/tool-loop-detection.ts:541](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/tool-loop-detection.ts#L541)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Hard blocking only happens on other critical detectors:** The same file blocks only for `unknown_tool_repeat`, `global_circuit_breaker`, `known_poll_no_progress`, and critical `ping_pong`. Generic repeats are not blocked unless they also hit the separate global no-progress breaker.
  - file: [src/agents/tool-loop-detection.ts:446](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/tool-loop-detection.ts#L446)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Before-tool-call hook executes warnings instead of aborting:** `runBeforeToolCallHook()` returns `{ blocked: true }` only for `loopResult.level === "critical"`; warning-level loop results are merely logged/emitted, then the tool call is still recorded and allowed to proceed.
  - file: [src/agents/pi-tools.before-tool-call.ts:194](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/pi-tools.before-tool-call.ts#L194)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Tests lock in the current behavior:** E2E coverage states `keeps generic repeated calls warn-only below global breaker` and only blocks generic repeats at `GLOBAL_CIRCUIT_BREAKER_THRESHOLD` rather than at the lower critical threshold requested here.
  - file: [src/agents/pi-tools.before-tool-call.e2e.test.ts:227](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/pi-tools.before-tool-call.e2e.test.ts#L227)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Unit tests also assert generic loops stay warning-only at the critical threshold:** Unit coverage expects generic no-progress loops to remain warning-only at `CRITICAL_THRESHOLD`, confirming current `main` does not hard-abort this class of loop at the threshold the issue asks for.
  - file: [src/agents/tool-loop-detection.test.ts:320](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/tool-loop-detection.test.ts#L320)

That means repeated `session_status` calls can legitimately change output and may never accumulate the identical-outcome streak required for the 30-call global breaker, which matches the report's failure mode.
  - file: [docs/concepts/session-tool.md:81](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/concepts/session-tool.md#L81)
