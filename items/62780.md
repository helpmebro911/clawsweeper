## Summary

Keep open. Current main partially addresses nearby surfaces with `message_sending` rewrite/cancel hooks and a built-in empty-response retry path for supported Gemini/strict-agentic runs, but it does not implement the requested after-LLM pre-send hook that can reject content and trigger `agents.defaults.model.fallbacks`. The provided related-item context shows broader pre-delivery hook issues were closed, but none is an open canonical tracker for this narrower fallback-gating behavior.

## Evidence

- **Existing pre-delivery message hook result shape is limited to rewrite/cancel:** `PluginHookMessageSendingEvent` exposes target/content/routing metadata, and `PluginHookMessageSendingResult` only supports `content?: string` and `cancel?: boolean`; there is no `pass`/`modify`/`reject` action, reject reason, finish reason, or provider/model completion metadata in this hook contract.
  - file: [src/plugins/hook-message.types.ts:69](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/plugins/hook-message.types.ts#L69)
  - command: `nl -ba src/plugins/hook-message.types.ts | sed -n '69,80p'`
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Message hook cancellation does not feed model failover:** `runMessageSending` merges only `content` and sticky `cancel`; `cancel=true` is terminal for lower-priority hooks. The code path does not translate cancellation into `FailoverError`, `FailoverReason`, or fallback-model selection.
  - file: [src/plugins/hooks.ts:774](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/plugins/hooks.ts#L774)
  - command: `nl -ba src/plugins/hooks.ts | sed -n '774,795p'`
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Built-in empty-response recovery is a retry instruction, not a plugin-gated fallback:** `resolveEmptyResponseRetryInstruction` returns a continuation instruction when the attempt has no payload and the provider/model is in the supported recovery set; it is not a plugin hook and does not invoke the configured model fallback chain.
  - file: [src/agents/pi-embedded-runner/run/incomplete-turn.ts:383](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/agents/pi-embedded-runner/run/incomplete-turn.ts#L383)

  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Gemini is specifically included in the built-in empty-turn recovery set:** The recovery guard allows strict-agentic models and Gemini provider IDs whose model id matches `gemini*`, so the reported Gemini empty-output symptom is partly mitigated on main, but only by retrying with a continuation instruction.
