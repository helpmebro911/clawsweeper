## Summary

Keep this PR open. Current main still lacks an Android talk-mode language override, and the PR is a concrete implementation candidate. It is not obsolete, duplicated, or ClawHub-only, but it still needs maintainer review and at least one functional fix around offline gateway refresh behavior before landing.

## Best Possible Solution

Keep this PR open. The best path is a rebase onto current main, an endpoint-aware language refresh path that updates future gateway options without setting a false offline failure status, and targeted Android tests or fresh manual evidence for Settings persistence, STT/TTS locale behavior, gateway locale metadata, and `auto` fallback. After that, an Android maintainer should decide whether the hardcoded locale list and UI placement are acceptable.
