## Summary

Keep open. Current main has useful mitigations for this class of WebChat history issues, including a larger 200-message history window and optimistic-tail preservation, so the 92-line diagnostic case may no longer fail the same way. But the core request is for complete session history rendering/sync, and current `chat.history` is still explicitly bounded, tail-sliced, byte-capped, and has no cursor/load-older API. The first issue comment's pagination/load-older diagnosis remains a real remaining gap rather than something conclusively implemented on main.
