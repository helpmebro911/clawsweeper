## Summary

Keep #9132 open. Current main still exposes session lifecycle data from the gateway, but the Control UI Sessions table does not render an active/completed/timed-out status indicator, nor does it support status sorting or filtering. The related #46792 is closed and is not an active canonical replacement.

## Best Possible Solution

Implement the requested lifecycle indicator in the Control UI Sessions view. Add a compact Status column or badge in ui/src/ui/views/sessions.ts that maps status values running, done, failed, killed, and timeout plus subagentRunState/hasActiveSubagentRun into clear active/completed/timed-out labels. Consider adding status sorting/filtering in the same patch or a narrower follow-up, and add focused rendering coverage in ui/src/ui/views/sessions.test.ts.
