## Summary

Keep #9132 open. Current main still carries session lifecycle data through `sessions.list`, including `status`, subagent run state, start/end time, and runtime, but the Control UI Sessions table does not render a lifecycle status badge/column or status-based sorting/filtering. The prior issue discussion also already kept this open, and related #46792 is closed rather than an active canonical replacement.

## Best Possible Solution

Implement the status indicator in the Control UI Sessions view. Add a compact Status column or badge in `ui/src/ui/views/sessions.ts` that maps `running`, `done`, `failed`, `killed`, `timeout`, `subagentRunState`, and `hasActiveSubagentRun` to clear labels, surface last activity/runtime from `updatedAt`, `startedAt`, `endedAt`, and `runtimeMs`, and add focused rendering coverage for the Sessions view. Status sorting/filtering can be included in that patch or split into a narrower follow-up if maintainers want a smaller first change.
