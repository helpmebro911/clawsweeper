## Summary

Keep #9132 open. Current main has session lifecycle status in the gateway/session-row data model, but the Control UI still does not render an active/completed/timed-out indicator in the Sessions table or chat session selector. The existing maintainer-thread Codex review reached the same conclusion, and current main has not changed that user-visible surface.

## Best Possible Solution

Keep this issue open until Control UI renders the existing lifecycle fields. The likely fix belongs in `ui/src/ui/views/sessions.ts` as a Status column/badge with last activity/runtime detail, plus corresponding chat selector treatment if maintainers want the status visible there too, backed by focused UI tests in `ui/src/ui/views/sessions.test.ts` and/or chat selector tests.
