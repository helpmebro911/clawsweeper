## Summary

Keep #14207 open. Current main still hard-codes non-batch memory indexing concurrency at 4 and only exposes `remote.batch.concurrency` for provider Batch API jobs, which does not cover the custom OpenAI-compatible inline embedding proxy path described in the issue. The related #68812/#68822/#66931 work is open follow-up context, not a shipped implementation or a clear canonical replacement for this issue.

## Best Possible Solution

Keep #14207 open until OpenClaw adds an agent-scoped non-batch memory indexing concurrency setting, wires it through `MemorySearchConfig`, `ResolvedMemorySearchConfig`, Zod schema/help/generated metadata, labels/docs, and focused tests, and uses it in `getIndexConcurrency()` for inline/non-batch indexing without changing the existing provider Batch API meaning of `remote.batch.concurrency`. Coordinate with the related open concurrency/retry work (#68812/#68822/#66931) rather than closing this performance-focused report now.
