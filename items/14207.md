## Summary

Keep #14207 open. Current main still hard-codes non-Batch memory indexing fanout at 4 and does not expose an agent-scoped inline indexing concurrency setting. The only public concurrency knob remains `agents.defaults.memorySearch.remote.batch.concurrency`, which is gated on provider Batch API support and does not solve the reported custom OpenAI-compatible/Ollama inline embedding path.

## Best Possible Solution

Keep #14207 open until OpenClaw adds an agent-scoped inline/non-Batch memory indexing concurrency setting, wires it through `MemorySearchConfig`, `ResolvedMemorySearchConfig`, strict Zod schema, generated config help/labels, public memory config docs, and focused tests, then uses it in `getIndexConcurrency()` for non-Batch indexing while preserving `remote.batch.concurrency` for provider Batch API jobs.
