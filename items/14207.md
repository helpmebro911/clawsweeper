## Summary

Keep #14207 open. Current main still hard-codes non-batch memory indexing fanout at 4 and only exposes `remote.batch.concurrency` for provider Batch API jobs. The config contract, strict schema, docs, and focused tests still do not provide an agent-scoped inline indexing concurrency setting. Related #68812/#68822 remain adjacent work, not a clearly shipped or canonical superseder.

## Best Possible Solution

Keep #14207 open until OpenClaw adds an agent-scoped inline/non-Batch memory indexing concurrency setting, wires it through `MemorySearchConfig`, `ResolvedMemorySearchConfig`, strict Zod schema, generated config labels/help, public memory config docs, and focused tests, then uses it in `getIndexConcurrency()` for non-Batch-API indexing while preserving `remote.batch.concurrency` for provider Batch API jobs.
