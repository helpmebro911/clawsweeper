## Summary

Keep this issue open. Current main still has no Dreaming-tab agent selector, and the Dreaming UI/controller/backend path remains scoped to the default agent for status, Dream Diary, and Dream Diary actions.

## Best Possible Solution

Keep this issue open. Add Dreaming selected-agent state and a selector backed by agents.list, pass an optional agentId through loadDreamingStatus, loadDreamDiary, Dream Diary actions, and related wiki refreshes where applicable, update doctor.memory.* gateway handlers to accept and validate that optional agentId with default-agent fallback, and add focused UI/controller/gateway tests. Keep #63829 for memory-wiki vault isolation and #68959 for any broader web UI agent-switcher product design.
