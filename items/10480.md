## Summary

Keep #10480 open. Current main still does not support Workers AI model selection through `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`; the bundled Cloudflare AI Gateway provider remains Anthropic/Claude-oriented. The generic custom-provider path can configure a one-off Workers AI endpoint, but it is not documented as the supported Workers AI answer and does not satisfy the requested Cloudflare AI Gateway onboarding path.

## Best Possible Solution

Keep #10480 open. The best implementation belongs in the bundled `extensions/cloudflare-ai-gateway` provider plugin: add an explicit Workers AI mode or non-interactive model selector, generate an OpenAI-compatible provider config using `https://gateway.ai.cloudflare.com/v1/<account>/<gateway>/workers-ai/v1` and `@cf/...` model IDs, update Cloudflare AI Gateway and onboarding CLI docs, and add focused tests for generated provider/model config. If maintainers decide the generic `custom-api-key` path is the intended answer, document the exact Workers AI command/config before closing.
