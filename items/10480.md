## Summary

Current main still does not implement Workers AI model selection for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`. The bundled Cloudflare AI Gateway path remains Anthropic-only, while Workers AI can only be approximated through custom provider configuration or manual config edits, which does not satisfy the issue’s requested onboarding flow.

## Best Possible Solution

Keep #10480 open until maintainers either add an explicit Workers AI path to the bundled Cloudflare AI Gateway provider plugin or make a product-scope decision not to support it. A concrete fix would likely add a Cloudflare endpoint/provider-type or explicit `@cf/...` model refs to non-interactive onboarding, generate an OpenAI-compatible `/workers-ai/v1` provider config, update `docs/providers/cloudflare-ai-gateway.md` and non-interactive onboarding docs, and add focused tests for generated provider/model config.
