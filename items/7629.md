## Summary

Keep open. Current main still leaves the bundled `memory-lancedb` plugin on vector-only LanceDB retrieval, with no FTS index setup, no hybrid/BM25 config surface, and no query-text path into LanceDB search. The existing BM25+vector implementation is for the separate builtin `memory-core` engine, and PR #7636 was closed unmerged.

## Best Possible Solution

Keep #7629 open as a focused bundled `memory-lancedb` parity enhancement. The likely implementation belongs in `extensions/memory-lancedb`: add typed `hybrid` config plus manifest/docs entries, create or maintain an FTS index on `text`, pass original query text through recall/forget/CLI/hooks and duplicate checks, use verified LanceDB hybrid search or a tested fallback/reranker when enabled, preserve vector-only fallback, and add targeted tests for config validation, FTS setup, exact-keyword recall, fallback behavior, and query-text propagation.
