## Summary

Keep open. Current main at b565e6e96310 still leaves `memory-lancedb` on pure vector retrieval: the plugin creates no FTS index, has no hybrid/BM25 config surface, and never passes query text through to LanceDB search. The related implementation PR #7636 was closed unmerged, and #19028 was an unrelated unmerged MongoDB backend proposal. The prior supporting comment and the recent Codex keep-open review remain consistent with the current code.

## Best Possible Solution

Keep the issue open and implement the requested behavior in the existing `memory-lancedb` plugin if maintainers still want parity: add a `hybrid` config surface to the runtime schema and manifest, create or maintain a LanceDB FTS index on `text`, thread the original query text through recall/forget/CLI/hooks, use LanceDB hybrid search or a verified fallback/reranker, and add focused tests for config validation, FTS setup, vector-only fallback, and exact-keyword recall.
