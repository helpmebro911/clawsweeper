## Summary

Keep #7057 open. Current main still has the reported WSL-sensitive `pi-tools.workspace-paths` and `pi-tools.safe-bins` test paths without WSL-specific guards or temp-dir cleanup hardening, and the two related mitigation PRs in the provided context were closed unmerged.

## Best Possible Solution

Keep #7057 open until a focused test-only fix lands for `src/agents/pi-tools.workspace-paths.test.ts` and `src/agents/pi-tools.safe-bins.test.ts`. The maintainer path should either harden temp-dir creation/cleanup around cwd-sensitive tests or add a narrowly justified WSL guard using the existing WSL detector, with WSL proof or targeted diagnostics and no unrelated changes.
