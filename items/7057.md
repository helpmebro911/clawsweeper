## Summary

Keep open. Current main still does not contain the WSL-specific skip or temp-directory hardening proposed for the reported `pi-tools.workspace-paths` and `pi-tools.safe-bins` failures, and the related mitigation PRs were closed unmerged. The issue is concrete enough to remain actionable.

## Best Possible Solution

Keep #7057 open until a focused test-only fix lands. The likely path is to update `src/agents/pi-tools.workspace-paths.test.ts` and `src/agents/pi-tools.safe-bins.test.ts` with targeted WSL handling using the existing WSL detector where justified, robust temp-dir creation/cleanup around cwd-sensitive tests, and enough diagnostics or platform proof to show the WSL timeout/ENOENT path is addressed. Avoid broad unrelated changes like the closed mitigation PRs bundled.
