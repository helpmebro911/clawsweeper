## Summary

Keep #7057 open. Current main still has the reported WSL-sensitive `pi-tools.workspace-paths` and `pi-tools.safe-bins` test paths without WSL-specific guards or temp-dir cleanup hardening, and the related mitigation PRs were closed unmerged. The existing maintainer comment also explicitly kept this issue open for the same unresolved surface.

## Best Possible Solution

Keep #7057 open until a focused test-only fix lands for `src/agents/pi-tools.workspace-paths.test.ts` and `src/agents/pi-tools.safe-bins.test.ts`. The best path is either to harden temp-dir creation/cleanup around cwd-sensitive tests or add narrowly justified WSL guards using the existing WSL detector, with targeted diagnostics or WSL verification and no unrelated changes.
