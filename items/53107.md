## Summary

Current main still does not implement cron job tags/categories. The prior ClawSweeper comment correctly kept this open, and current main at 348728c28c1ea9ae7c5824350baba35ee08a275e still lacks a supported `tags` field across the persisted cron model, Gateway schema/list filtering, CLI, Control UI, macOS cron editor/model, docs, and agent cron tool.

## Best Possible Solution

Keep this issue open. The best implementation is an additive core cron contract change: add normalized optional `tags?: string[]` to the cron job model, Gateway add/update/list schemas, persistence/normalization, agent cron tool schema, CLI add/edit/list/show and tag filtering, Control UI create/edit/display/filter controls, macOS cron model/editor if still supported, plus docs and regression tests. Tag-based bulk enable/disable can either ship with that list/update contract or be split into a narrower follow-up after tags are persisted and filterable.
