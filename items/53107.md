## Summary

Keep this issue open. Current main still does not implement persisted cron job tags/categories or tag-based filtering. The earlier ClawSweeper keep-open comment remains accurate against c2d82b87eee7a86bbd0e851169e0ca5559c44fe6, and the related items in the provided context cover adjacent cron history/UI/hooks/catch-up work rather than this schema/API/UI feature.

## Best Possible Solution

Keep this issue open. The best path is an additive core cron change: add normalized optional `tags?: string[]` to the cron job model and persistence, accept and validate it in Gateway add/update/list schemas, implement `cron.list` tag filtering, expose create/edit/list/show tag flags in the CLI, update the agent cron tool schema and behavior, add Control UI and macOS display/filter/edit controls, document the feature, and add focused regression tests. Tag-based bulk enable/disable can ship after the foundational tag contract or be tracked separately.
