## Summary

Keep this issue open. Current main still gates estimated dollar-cost display on provider auth mode in both reported display paths, so an Amazon Bedrock provider resolved as `aws-sdk` remains token-only even when explicit model pricing is configured. The same-author PR #53303 was closed unmerged, and current tests/docs still encode API-key-only cost display.

## Best Possible Solution

Keep this issue open. Implement the display-layer fix by resolving explicit model pricing independently of auth mode, then show estimated cost in `/status` and `/usage full` for `aws-sdk`, OAuth, token, mixed, and API-key providers when usable pricing is configured. Add regression coverage for a Bedrock-style `aws-sdk` provider with nonzero model cost in both status and response-usage footer paths, and update the public usage/cost docs.
