## Summary

Current main still gates both `/status` and `/usage full` cost display on auth mode, so Bedrock-style `aws-sdk` providers with explicit model `cost` metadata still show tokens without dollar estimates. The same-author PR #53303 was closed unmerged, and current docs/tests still encode API-key-only cost display.

## Best Possible Solution

Keep this issue open. Implement the decoupling in core usage display by resolving model cost metadata independently of auth mode, then show estimated cost in both `/status` and `/usage full` when the active model has explicit nonzero cost metadata under `aws-sdk`, OAuth, token, or API-key auth. Add regression tests for a Bedrock-style `aws-sdk` provider with explicit costs and update the public usage-cost docs. PR #53303 can be used as prior context, but it was closed unmerged and should not be treated as shipped.
