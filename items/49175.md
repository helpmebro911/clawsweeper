## Summary

Keep open. Current main still has the reported `cron.runs` all-scope name leak path: it builds `jobNameById` from unfiltered `context.cron.list({ includeDisabled: true })`, and the cron service/list-page contracts do not expose caller/session filters. The referenced PR #49197 was closed unmerged, and the suggested helper/filtering is not present on main.

## Evidence

- **All-scope run history still uses unfiltered cron.list for names:** `cron.runs` with `scope === "all"` calls `context.cron.list({ includeDisabled: true })`, builds `jobNameById` from every returned job, then passes that map into `readCronRunLogEntriesPageAll`.
  - file: [src/gateway/server-methods/cron.ts:427](https://github.com/openclaw/openclaw/blob/d228463120a5057f7ccf9a5c31af62fe4a922088/src/gateway/server-methods/cron.ts#L427)

  - sha: [d228463120a5](https://github.com/openclaw/openclaw/commit/d228463120a5057f7ccf9a5c31af62fe4a922088)
- **cron.list has no caller/session filter:** The service `list` method only accepts `includeDisabled` and filters by enabled state before sorting; it has no `agentId`, `sessionKey`, caller, or owner override option.
