## Summary

Keep open. Current `main` still exposes `applyMergePatch` on config mutation paths without the proposed recursion-depth or array-length hardening, so this PR is not implemented on `main`. The PR discussion also already identifies unresolved correctness problems in the proposed fix, which makes conservative auto-close inappropriate.
