## Summary

Keep #38076 open. Current main still parses `init_skill --resources` case-sensitively: `ALLOWED_RESOURCES` contains only lowercase tokens, `parse_resources()` validates stripped raw tokens before any lowercase normalization, and `main()` passes `args.resources` directly to that parser. The current skill docs also document only lowercase values, and no focused `init_skill` regression test exists. The linked fix attempts in the provided GitHub context are closed and unmerged, so there is no canonical landed fix to close against.

## Best Possible Solution

Land the narrow skill-creator fix: normalize each nonempty `--resources` token to lowercase before validation and order-preserving deduplication, keep lowercase output directory names and the existing unknown-resource error behavior, and add focused `skills/skill-creator/scripts/test_init_skill.py` coverage for mixed-case acceptance, lowercase deduplication, and invalid-resource rejection.
