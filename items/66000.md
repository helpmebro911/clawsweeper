## Summary

Keep this PR open. Current main still has the linked profile/launchd precedence bug, and this PR is a focused implementation candidate for #65643. It should not be closed as obsolete, but the implementation needs maintainer review because the proposed rule clears every nonmatching `OPENCLAW_LAUNCHD_LABEL`, including custom labels that current launchd and restart-helper code intentionally preserve.

## Best Possible Solution

Keep this PR open and request a narrower implementation: when `--profile` is explicitly supplied, ignore or clear stale OpenClaw-managed labels from another profile while preserving explicit custom `OPENCLAW_LAUNCHD_LABEL` overrides. Add regression coverage for stale managed labels, matching managed labels, custom labels, absent labels, and the `gateway status` plist-resolution path from #65643; then review and land the revised PR or close it only after a canonical fix lands elsewhere.
