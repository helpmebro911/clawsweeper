## Summary

Keep this PR open. Current main still has the linked profile/launchd precedence bug, and this PR remains a focused implementation candidate, but the current patch is too broad because it clears custom OPENCLAW_LAUNCHD_LABEL overrides that main currently treats as supported behavior.

## Best Possible Solution

Keep this PR open and request a narrower fix: when --profile is explicitly supplied, ignore or clear stale OpenClaw-managed launchd labels from another profile while preserving explicit custom OPENCLAW_LAUNCHD_LABEL overrides. Add regression coverage for stale managed labels, matching managed labels, custom labels, absent labels, and the gateway status plist-resolution path from #65643; then land the revised PR or close it only after a canonical fix lands elsewhere.
