## Summary

Keep open. Current main still has the reported self-chat/allowFrom path: when WhatsApp metadata mentions are present and allowFrom makes the linked self number count as self-chat, the code intentionally skips identity-overlap matching and falls back to text patterns/number text. Existing tests also assert that self-chat JID mentions are ignored, so the issue is not safely obsolete or implemented. The related agent-level mentionPatterns comment appears addressed on current main, but the central LID/native mention bug remains a live product/behavior question.
