## Summary

Current main still defaults TUI startup without an explicit --session to the resolved main session key rather than selecting the most recently updated active/user session. The storage-bloat side has maintenance caps/rotation, but sessions still persist per-session skillsSnapshot data, so the requested behavior is not fully implemented on main.

## Evidence

- **TUI blank session defaults to main session key:** resolveTuiSessionKey returns the agent main session when raw session input is blank and session scope is not global; runTui passes the trimmed --session value and initializes currentSessionKey from that resolver, so a restart with no --session still opens main rather than ranking recent sessions.
