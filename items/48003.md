## Summary

Keep open. Current main still matches the reported failure mode: Discord inbound jobs are serialized by queue key before the steer path can run, and the remaining steer injection path is still gated on streaming state rather than active-run state. Related PRs discuss plausible fixes, but the requested Discord/main-session behavior is not implemented on current main.
