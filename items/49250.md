## Summary

Keep open. Current main has generic Control UI queued-message/steer behavior, but the requested prompt/heartbeat race handling is not implemented: heartbeat runs are skipped or dropped when work is already active, and UI session-message reloads are explicitly deferred while a chat run is active, so the heartbeat is not guaranteed to show immediately while the prompt is visibly queued behind it.

## Evidence

- **Heartbeat dropped during active run:** The active-run queue policy returns `drop` whenever the incoming run is a heartbeat and another run is active, rather than queueing or surfacing a heartbeat/prompt conflict state.
  - file: [src/auto-reply/reply/queue-policy.ts:14](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/auto-reply/reply/queue-policy.ts#L14)
  - command: `nl -ba src/auto-reply/reply/queue-policy.ts | sed -n '1,25p'`
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **Test locks heartbeat drop behavior:** The current test suite explicitly expects heartbeat runs to be dropped while another run is active.
  - file: [src/auto-reply/reply/queue-policy.test.ts:16](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/auto-reply/reply/queue-policy.test.ts#L16)

- **Heartbeat runner skips busy lanes:** `runHeartbeatOnce` skips with `requests-in-flight` when either the main command lane or the resolved session lane has work, so it does not implement the issue's requested 'show heartbeat right away, visually queue prompt' ordering when a prompt is already in flight.

- **UI defers session reload while chat run active:** The Control UI intentionally defers `session.message` history reloads while `host.chatRunId` exists, which can delay heartbeat transcript visibility during an active prompt instead of showing the heartbeat immediately.
  - file: [ui/src/ui/app-gateway.ts:501](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/ui/src/ui/app-gateway.ts#L501)
