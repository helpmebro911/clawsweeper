## Summary

Keep open. Current `main` still has the reported ASCII-only slug normalization bug, and this PR's proposed regex change is not conservative to land as-is because it still strips combining marks called out in review.

## Evidence

- **Current main still strips non-ASCII group/channel names:** `normalizeHyphenSlug` and `normalizeAtHashSlug` still use ASCII-only allowlists: `[^a-z0-9#@._+-]+` and `[^a-z0-9-]+`. That means non-Latin letters are removed on current `main`.
  - file: [src/shared/string-normalization.ts:54](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/shared/string-normalization.ts#L54)
  - command: `nl -ba src/shared/string-normalization.ts`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Reasonable reproduction still fails on current main:** A direct reproduction of the current logic returns empty slugs for `技术讨论组`, `Тестовая группа`, `#技术讨论`, and `#किरण`, confirming the bug is still present rather than obsolete.
  - command: `node -e 'const normalizeOptionalLowercaseString=(v)=>typeof v==="string"?v.trim().toLowerCase():(v==null?undefined:String(v).trim().toLowerCase()); const normalizeHyphenSlug=(raw)=>{const trimmed=normalizeOptionalLowercaseString(raw)??""; if(!trimmed) return ""; const dashed=trimmed.replace(/\s+/g,"-"); const cleaned=dashed.replace(/[^a-z0-9#@._+-]+/g,"-"); return cleaned.replace(/-{2,}/g,"-").replace(/^[-.]+|[-.]+$/g,"");}; const normalizeAtHashSlug=(raw)=>{const trimmed=normalizeOptionalLowercaseString(raw)??""; if(!trimmed) return ""; const withoutPrefix=trimmed.replace(/^[@#]+/,""); const dashed=withoutPrefix.replace(/[\s_]+/g,"-"); const cleaned=dashed.replace(/[^a-z0-9-]+/g,"-"); return cleaned.replace(/-{2,}/g,"-").replace(/^-+|-+$/g,"");}; console.log(JSON.stringify({hyphen_cjk:normalizeHyphenSlug("技术讨论组"), hyphen_cyrillic:normalizeHyphenSlug("Тестовая группа"), at_cjk:normalizeAtHashSlug("#技术讨论"), at_devanagari:normalizeAtHashSlug("#किरण")}))'`
