## Summary

Keep #9993 open. Current main has built-in config validation, base-hash protection, SecretRef preflight, config write guards, and an agent-facing `gateway` tool guard, but it still does not provide the requested synchronous plugin/API-level veto before config writes. Direct RPC and runtime/plugin write paths still persist through `writeConfigFile` without a config pre-apply hook.

## Best Possible Solution

Keep the issue open and coordinate it with the linked config validation/mutation RFC. The best implementation is a documented synchronous config-write interception point invoked by `config.set`, `config.patch`, `config.apply`, and shared write helpers after pending config is computed and built-in validation/preflight succeeds, but before persistence. It should expose current config, pending config, action, raw patch/full payload, baseHash, actor/client metadata, changed paths, and a typed reject result, with tests proving a plugin can reject direct RPC writes without changing disk or active runtime config.
