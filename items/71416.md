## Summary

Keep #71416 open. Current main still catches any Gateway CLI failure, emits only a stderr diagnostic, and runs the embedded agent automatically. The tests and docs describe that behavior, but there is still no opt-in fallback flag, nonzero exit, stdout banner, or stable JSON transport/fallback marker for scripts.

## Best Possible Solution

Keep the issue open for a focused CLI behavior change. The best maintainer path is to make fallback explicit and machine-detectable: either fail closed unless an `--allow-embedded-fallback` flag is passed, or add a stable JSON marker such as `meta.transport`/`meta.fallback` plus clear non-JSON warning behavior, then update `src/commands/agent-via-gateway.ts`, option registration, tests, and `docs/cli/agent.md` together.
