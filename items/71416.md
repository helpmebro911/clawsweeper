## Summary

Keep #71416 open. Current main still catches any Gateway agent failure, logs `Gateway agent failed; falling back to embedded: ...` through stderr, and immediately runs the embedded agent. Tests and docs still codify stderr-only diagnostics for JSON mode, with no `--allow-embedded-fallback` option, nonzero exit path, stdout banner, or stable JSON transport/fallback marker.

## Best Possible Solution

Keep the issue open for a focused CLI behavior change. The best fix is to make Gateway-to-embedded fallback explicit and machine-detectable, likely in `src/commands/agent-via-gateway.ts` plus `src/cli/program/register.agent.ts`, with tests and `docs/cli/agent.md` updated together. A conservative path would add an opt-in `--allow-embedded-fallback` or, at minimum, emit a stable JSON marker such as `meta.transport`/`meta.fallback` while preserving parseable stdout semantics.
