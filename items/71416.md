## Summary

Current main still has the reported behavior: `openclaw agent` catches Gateway-path failures, emits only a stderr diagnostic, then runs the embedded agent. The registered CLI has `--local` but no opt-in fallback flag, JSON embedded output still contains only payload/meta/delivery shape without a transport/fallback marker, and docs/tests currently codify the fallback rather than resolving the detectability problem. The related heartbeat PR #71478 explicitly scoped this CLI fallback issue out, and the existing maintainer-posted Codex review also kept this open.

## Best Possible Solution

Keep this issue open for a targeted CLI behavior change in `src/commands/agent-via-gateway.ts` and `src/cli/program/register.agent.ts`, with regression coverage in `src/commands/agent-via-gateway.test.ts` and docs in `docs/cli/agent.md`. The maintainer choice is whether Gateway failure should fail unless an explicit fallback flag is passed, or whether JSON/non-JSON outputs should expose a clear, stable embedded-fallback signal.

## Risks / Open Questions

- Changing fallback to fail closed by default could break scripts that currently rely on automatic embedded fallback.
