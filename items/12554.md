## Summary

Keep #12554 open. Current main has per-run chat abort and per-agent/session-scoped `process` tool controls, but it still lacks a first-class operator surface that lists and terminates all background exec/process sessions for a target OpenClaw session. The existing `/sessions/:key/kill` HTTP path is subagent-run control, not exec-process cleanup.

## Best Possible Solution

Keep #12554 open. The best path is to add a first-class Gateway/CLI operator surface such as `sessions.exec-list` and `sessions.kill --kill-exec` or `sessions.abort --kill-exec`, backed by the existing process registry/supervisor and session scope metadata. It should list active background exec/process records for a target session key, cancel only that session's records as a unit, enforce operator/requester authorization, and add regression tests proving it does not affect unrelated sessions.
