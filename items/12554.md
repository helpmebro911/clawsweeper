## Summary

Keep #12554 open. Current main has scoped exec/process machinery and active-run abort support, but it still lacks the requested session-wide operator surface to enumerate all exec processes for an OpenClaw session and terminate them as a unit. The existing `/sessions/:sessionKey/kill` route is subagent-run control, not arbitrary exec/process cleanup for a session.

## Best Possible Solution

Keep #12554 open and implement an additive Gateway/CLI/RPC operator surface: resolve the canonical session key, expose an exec/process list for that scope, terminate all matching managed exec runs via process supervisor cancellation plus process-tree fallback, return per-process status, document the contract, and add tests for same-session kill, unrelated-session isolation, background exec, HTTP/RPC/CLI validation, and subagent cancellation interactions.

## Risks / Open Questions

- Do not close this as implemented based on related #39960; that covered active stop/abort for current exec work, while #12554 asks for a session-wide operator list/kill-all surface for already-spawned exec processes.
- A correct implementation needs explicit authorization and session isolation so one operator/session cannot inspect or terminate unrelated sessions' exec processes.
- The fix should preserve intended background exec behavior unless the caller explicitly asks for stronger kill-exec semantics.
