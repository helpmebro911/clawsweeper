## Summary

Keep open. Current `main` does not implement the requested `~/.openclaw` sibling-symlink allowance, and the submitted PR should not be auto-closed because its own diff broadens generic boundary checks beyond symlink hops, affecting shared path-guard callers without regression coverage.

- Closing this PR would be wrong because the requested behavior is not present on current `main`.
