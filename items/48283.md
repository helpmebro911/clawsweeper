## Summary

Keep #48283 open. Current main does not contain the PR's general prompt-abort cleanup, and the linked bug remains plausibly unsolved for the embedded runner path. Main only strips synthetic `sessions_yield` aborted assistant artifacts; ordinary prompt aborts still fall through as prompt errors and then use the current session messages for snapshots/finalization. The PR should be rebased and covered by a focused regression test before maintainer action.

## Best Possible Solution

Keep the PR open, but require a current rebase and a focused regression test. The best fix is to add or extract a shared tail-strip helper for runner-owned aborted assistant artifacts, call it on the ordinary prompt abort path before snapshots/finalization/prompt-error persistence, and prove that stale assistant content from #48241 is not persisted or delivered while intentional `chat.abort` partials with `openclawAbort` metadata still work.
