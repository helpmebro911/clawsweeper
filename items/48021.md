## Summary

Keep open. Current main still returns raw bootstrap file names from `buildBootstrapInjectionStats` and `/context` renders that `name` directly, so the reported `undefined` label remains possible for hook-injected bootstrap files with a valid `path` but missing or empty runtime `name`. PR #48021 is still relevant rather than obsolete, and its review discussion indicates the author already adjusted the patch to use normalized `path.posix.basename`.

## Best Possible Solution

Keep the PR open and either land it after review or apply an equivalent fix on main: derive a stable display label in `buildBootstrapInjectionStats` from non-empty `file.name`, normalized basename of `file.path`, then a placeholder, and add regression coverage for undefined and empty names while preserving injection/truncation tracking.
