## Summary

Keep open. Current main has adjacent surfaces, but not the requested lifecycle-owned SDK seams. Plugins can use raw `api.runtime.events.onAgentEvent`, `agent_end`/subagent hooks, generic prompt hooks, and `gateway_start` cron access, but main does not provide typed sanitized agent-event subscriptions, namespaced run-scoped plugin storage, plugin/session-owned scheduler cleanup, or heartbeat-specific prompt contributions. The linked #71731 is an open docs/RFC PR, and #71676 was closed unmerged, so neither supersedes or ships this request.

## Best Possible Solution

Keep #71737 open as the focused SDK/RFC tracker unless maintainers intentionally consolidate it under the open RFC PR #71731. The best implementation path is additive core SDK work: typed sanitized agent lifecycle subscriptions, plugin-namespaced run storage cleaned on terminal events, plugin/session-owned scheduler helpers with reset/disable cleanup, and deterministic heartbeat prompt contribution handling with fixture plugin tests.
