## Summary

Current main has multi-session primitives, but not the requested tab system. The Control UI exposes a single chat route and a session dropdown, plus a Sessions table and a New session action. Switching sessions replaces the active chat state rather than preserving multiple open chat tabs with independent drafts/state, close/reorder controls, or persisted tab layout.

## Best Possible Solution

Keep the issue open for explicit Control UI product work. A complete fix likely belongs in the chat UI layer: introduce a per-gateway persisted chat-tab collection, render a top tab strip above the chat thread, let users open/switch/close sessions without clearing unrelated tab-local draft/UI state, and optionally add reorder persistence. It can reuse existing `sessions.list`, `chat.history`, `sessions.patch`, and `/new`/reset flows, but should not be treated as already solved by the current session dropdown.
