## Summary

Keep open. Current main still does not serialize OpenClaw-owned writes to openclaw.json with an advisory lock or a locked read-modify-write path. The writer uses temp-file plus rename, and that helps avoid torn writes, but the read/prepare/write window remains unprotected. The linked PR #70655 proposed the right class of fix, but it was closed unmerged after the author said it was opened prematurely.

## Best Possible Solution

Implement a config-specific advisory lock around the whole config read/prepare/validate/write critical section, using the existing file-lock helper or an equivalent core-safe wrapper. Update replaceConfigFile/mutateConfigFile so read-modify-write operations either run under the same lock or retry cleanly on base-hash conflicts. Add focused regression coverage for concurrent config writers, including preservation of disjoint edits and no invalid concatenated JSON on the fallback path.
