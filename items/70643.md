## Summary

Keep #70643 open. Current main has config validation, temp-file rename writes, clobber rejection, and last-known-good recovery, but it does not implement the requested shared advisory lock or a read-merge-write critical section for openclaw.json writers. The linked fix PR #70655 was closed unmerged, and the exact concatenated-two-JSON-object shape is still treated as invalid rather than specifically repaired.

## Best Possible Solution

Implement a shared advisory lock for OpenClaw-owned openclaw.json writes that covers read, merge/projection, validation, temp-file write, backup rotation, and rename. Re-read inside the lock for mutation helpers, gateway config RPCs, CLI config/model/auth writers, and plugin runtime config writes; keep the current validation/rejected-payload and last-known-good recovery behavior. Add a regression test that drives two concurrent config-touching paths and asserts the final file is valid JSON with both intended changes preserved, plus coverage for lock cleanup/re-entrancy.
