## Summary

Keep this issue open. Current main still hard-gates verbose/tool-result delivery for ordinary group chats and has no typed config option under channel defaults or group/topic settings to opt back in. The related #44027 is only an open implementation candidate and covers session `/verbose` behavior, not the requested schema-backed config policy.

## Best Possible Solution

Keep this issue open. The best path is a narrow typed config policy such as `channels.defaults.allowVerboseToolResults` plus provider/group/topic overrides, defaulting to false. Wire it into `dispatch-from-config` and ACP projection, add regression tests proving ordinary groups remain silent by default but deliver text summaries when explicitly enabled, preserve forum/media/exec-approval behavior, update public docs with a trusted-group warning, and coordinate with #44027.

## Risks / Open Questions

- Verbose tool-result output in shared chats can expose commands, paths, URLs, plugin diagnostics, and tool-returned data; any implementation should default to false and document trusted-group use only.
- The config policy should be reconciled with #44027 so OpenClaw does not grow separate, inconsistent mechanisms for group tool-result delivery.
