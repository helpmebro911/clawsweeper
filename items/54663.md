## Summary

Keep this issue open. Current main still suppresses text-only tool results in ordinary group chats and has no schema-backed config option under channel defaults, Telegram groups/topics, or WhatsApp groups to opt back in. The related #44027 remains only an open implementation candidate and addresses session `/verbose` behavior rather than the requested config policy.

## Best Possible Solution

Keep this issue open. The best path is a narrow, typed, schema-backed opt-in such as `channels.defaults.allowVerboseToolResults` plus provider/group/topic overrides, defaulting to false. Wire that policy into `dispatch-from-config` and ACP projection, add regression tests proving ordinary groups stay silent by default but deliver text summaries when explicitly enabled, preserve forum/media/exec-approval behavior, update public docs with a trusted-group warning, and coordinate the behavior with #44027.

## Risks / Open Questions

- Verbose tool-result output in shared chats can expose commands, paths, URLs, diagnostics, and tool-returned data; the implementation should default to false and document trusted-group use only.
- The requested config policy should be reconciled with #44027 so OpenClaw does not grow separate, inconsistent mechanisms for group tool-result delivery.
