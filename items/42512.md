## Summary

Keep open. Current `main` still lacks the WhatsApp/TTS behavior this PR is trying to add: bare `[[tts]]` directives are not parsed, WhatsApp auto-reply still suppresses tool payloads entirely, and WhatsApp audio delivery still forces push-to-talk for all audio instead of only voice-note-compatible payloads.
