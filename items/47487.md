## Summary

Keep open. Current main still contains the reported behavior: global `tools.exec` is implicitly added back to the profile allowlist, even when a messaging-profile agent uses an explicit `alsoAllow` list that omits `exec`. The related PR #47991 targeted this exact bug, but the provided context shows it was closed unmerged, and current main still has the old logic and test expectation.
