## Summary

Current `main` still does not implement skill-driven plugin context visibility. The skill metadata type and frontmatter parser have no `metadata.openclaw.activates.plugins` contract, skill snapshots do not carry plugin activation dependencies, and request-time plugin tools are still assembled from the enabled plugin registry plus static tool policy/allowlists. The linked PR #72533 remains open and unmerged with closing syntax, so this issue should stay open while that PR is reviewed or closed separately.

## Best Possible Solution

Keep this issue open for maintainer design and implementation. The likely path is to add a typed `metadata.openclaw.activates.plugins` skill metadata contract, parse and preserve it through skill entries/snapshots, define a concrete pre-model activation trigger, feed activated plugin ids into request-time plugin tool planning while preserving deny/allow and optional-tool precedence, document the public contract, and review or close #72533 separately because it is open, unmerged, and README-only.
