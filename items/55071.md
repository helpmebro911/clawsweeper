## Summary

Current main still does not implement skill-driven plugin context visibility. Skill metadata has no `activates.plugins` contract, the parser/snapshot path does not preserve activation dependencies, plugin tools are built from enabled plugin registry plus static allowlist policy, and linked PR #72533 remains open, unmerged, and README-only despite closing syntax.

## Best Possible Solution

Keep this issue open for maintainer design and PR handling. The likely implementation path is to add a typed `metadata.openclaw.activates.plugins` skill metadata contract, parse and preserve it through skill entries and snapshots, define a concrete pre-model activation trigger, feed activated plugin ids into request-time plugin tool planning, document the contract, and add tests that prove deny/allow policy and optional-tool precedence are preserved. Maintainers should separately review or close PR #72533 because it is README-only and does not implement the feature.
