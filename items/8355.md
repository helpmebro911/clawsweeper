## Summary

Keep #8355 open. Current main still batches the classic voice-call STT/agent/TTS path: it awaits one completed embedded-agent response string, then awaits one completed telephony TTS buffer before Twilio sends 20ms frames. Realtime voice support exists, but docs and config describe it as a separate full-duplex mode, not the requested sentence-level LLM-to-TTS pipeline. Maintainer comments in the provided issue context already treat #8355 as canonical, with #42187 closed unmerged and #42159 closed as the narrower duplicate.

## Best Possible Solution

Keep #8355 open as the canonical provider-generic voice-call feature. The best path is to add a sentence-level pipeline in the voice-call plugin, expose or extend a streaming telephony TTS capability with buffered fallback, feed embedded-agent partial replies or sentence chunks into that capability, stream chunks through Twilio and Telnyx media handlers where supported, and preserve queue clearing, abort, timeout, and barge-in behavior for in-flight synthesis and playback.
