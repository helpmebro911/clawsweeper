## Summary

Keep PR #45216 open. Current main still does not recognize FastAPI/Starlette-style top-level `{"detail":"..."}` error payloads, so the raw JSON leak described by the PR is still plausible on the shared assistant error-formatting path.

## Best Possible Solution

Keep this PR or land an equivalent core fix: update `src/shared/assistant-error-format.ts` to classify string top-level `detail` payloads and use `detail` as the message fallback, with precise regression tests for both `formatAssistantErrorText` and `formatRawAssistantErrorForUi`. Track provider-credential gating or silent suppression separately from this narrow formatter fix.
