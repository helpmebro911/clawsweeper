## Summary

Keep #11492 open. Current main still does not expose Slack `channel-info` or `channel-list` as message-tool actions: Slack discovery, dispatch, runtime exports, docs, and contract tests all omit them, while generic/Discord surfaces already include the requested action names. The provided discussion also has a recent maintainer/Codex keep-open comment; PR #11491 is closed unmerged, and PR #53199 is still open rather than shipped on main.

## Best Possible Solution

Keep the issue open until Slack exposes the requested message-tool actions on main. The likely fix belongs in `extensions/slack`: add `channel-info` and `channel-list` to Slack action discovery behind the existing `channelInfo` gate, dispatch them through `message-action-dispatch`, add runtime helpers for Slack channel metadata/listing, and update Slack docs plus focused discovery/dispatch/runtime tests. If PR #53199 becomes canonical, it should explicitly cover both requested actions or document a maintainer-approved split.
