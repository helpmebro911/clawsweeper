## Summary

Keep #11492 open. Current main still does not expose Slack `channel-info` or `channel-list` as agent-facing message actions, even though the generic action names and Slack `channelInfo` config flag exist. Related PR #11491 is closed unmerged, and open PR #53199 only partially overlaps unless it also adds `channel-info`.

## Best Possible Solution

Keep #11492 open until the bundled Slack plugin exposes both `channel-info` and `channel-list` through the message tool. The likely implementation belongs in `extensions/slack`: register discovery behind the existing `channelInfo` gate, add dispatcher branches, add runtime/action helpers around Slack `conversations.info` and paginated `conversations.list`, then update Slack docs and focused contract/runtime tests. PR #53199 can help with the `channel-list` portion but should not supersede this issue unless `channel-info` is also implemented or a narrower canonical follow-up remains open.
