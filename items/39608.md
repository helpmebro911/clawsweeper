## Summary

Keep #39608 open. Current main still constructs the WhatsApp Baileys socket with `syncFullHistory: false` and no `shouldSyncHistoryMessage` override, while Baileys 7.0.0-rc.9 replaces the default history-sync predicate with `() => !!syncFullHistory` when that override is absent. The recent issue comments report the same group-inbound loss on 2026.3.28 and 2026.4.20, so this is not stale and not implemented on main.
