## Summary

Keep #39608 open. Current main still creates the WhatsApp Baileys socket with `syncFullHistory: false` and no explicit `shouldSyncHistoryMessage`, while the pinned Baileys version defaults an omitted predicate to `() => !!syncFullHistory`. The recent issue comments report the same DM-only/group-silent failure on 2026.3.28 and 2026.4.20, and the related #14069 was closed stale rather than replaced by a live canonical fix.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin either passes an explicit Baileys history-sync predicate, exposes a deliberate config/product choice for linked-device history sync, or documents with maintainer approval that WhatsApp linked-device group inbound after relink is unsupported. The likely code change belongs in `extensions/whatsapp/src/session.ts`, with a focused regression test in `extensions/whatsapp/src/session.test.ts` asserting the options passed to `makeWASocket`; a live linked-device group smoke would be useful before closing as fixed.
