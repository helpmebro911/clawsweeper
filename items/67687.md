## Summary

Keep open. Current main still does not clearly satisfy the reported Feishu config shape: `groups.<chat_id>.requireMention: false` is honored only after the group is admitted, while the runtime admission check still uses `groupAllowFrom` under the schema-defaulted `groupPolicy: "allowlist"`. The issue is better framed as Feishu group admission/config contract, not just the mention gate.

## Best Possible Solution

Keep this issue open and fix or clarify Feishu group admission. The conservative implementation path is to either treat explicit `channels.feishu.groups.<chat_id>` entries as admitted groups under `groupPolicy: "allowlist"`, or add a targeted doctor/config warning that `groups.<chat_id>.requireMention=false` does not admit the group without `groupAllowFrom` or `groupPolicy: "open"`. Add a regression test that runs the reporter's config shape through `validateConfigObjectWithPlugins` and then verifies a Feishu group text event reaches session/route dispatch. Update `docs/channels/feishu.md` if the intended contract remains `groupAllowFrom`-only.
