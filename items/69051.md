## Summary

Keep this PR open. Current main has the provider-agnostic TTS persona foundation, but it still lacks this PR's source/plain TTS metadata contract, speech-provider capability declaration, ElevenLabs v3 expressive-tag allowlist, and per-attempt fallback text routing. The PR is active and split into a dependent stack, so cleanup should leave it for maintainer review or a clearly linked superseding implementation.

## Best Possible Solution

Keep this PR open as the active PR-A implementation candidate. Maintainers should resolve the remaining fallback/summarization and SDK-surface review threads, verify focused speech-core and changed-lane tests, then either land this provider-aware source/plain TTS router or close it only after linking a superseding PR that implements the same contract.
