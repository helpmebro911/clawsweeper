## Summary

Keep #69102 open. Current main has configurable model params, but the Gemini transport only forwards a small allowlist into the Google request and does not map `serviceTier`/`service_tier` to the REST `service_tier` field that Google documents for Flex inference. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/flex-inference))

## Best Possible Solution

Keep the issue open and implement it in the bundled Google provider: accept `agents.defaults.models["google/<model>"].params.serviceTier` and/or `service_tier`, map valid Flex values to top-level `service_tier` in `extensions/google/transport-stream.ts`, add focused Google transport and extra-param tests, and document the Google-specific config in `docs/providers/google.md` and `docs/concepts/model-providers.md`.
