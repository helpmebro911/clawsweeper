## Summary

Keep this PR open. Current main still has the narrow final-tag regexes in both affected paths, so the `<final/>` leak described by the PR and linked #65867 is not implemented on main. The PR is a plausible implementation candidate, but its latest parser shape still needs maintainer review for the malformed-attribute strict-mode concern raised in review comments.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this fix or an equivalent shared final-tag parser with regression coverage for `sanitizeUserFacingText`, streaming with enforcement off, streaming with enforcement on, self-closing `<final/>` and `<final />`, slash-containing attributes, and malformed-attribute fail-closed behavior; after that lands, #65867 can be closed as implemented. #69685 was a companion persistence-side PR and is closed unmerged, so it does not supersede this streaming/sanitizer fix.
