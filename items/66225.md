## Summary

Keep this PR open. Current main still has the narrow final-tag regexes in both the streaming subscriber and exported user-facing sanitizer, so the self-closing `<final/>` leak targeted by this PR is not implemented on main. The linked bug #65867 remains open, and #69685 was a separate persistence-side PR that closed unmerged rather than superseding this streaming/sanitizer fix.

## Best Possible Solution

Keep this PR open and have a maintainer review the branch. The best path is to land this fix or an equivalent shared final-tag parser with regression coverage for `sanitizeUserFacingText`, streaming with enforcement off, streaming with enforcement on, and malformed-attribute parser hardening; after that lands, #65867 can be closed as implemented.
