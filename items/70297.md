## Summary

Keep open. Current main still allows channel sessions in the active `src/memory-host-sdk` QMD default scope, and the regression test still asserts that default channel access is allowed. The provided PR diff remains a focused change to remove that channel allow-rule and flip the test expectation to denied, so it is not implemented on main or superseded by the related reports provided.

## Best Possible Solution

Keep #70297 open for maintainer review and either merge this focused source/test change or apply an equivalent fix in `src/memory-host-sdk/host/backend-config.ts` plus the regression test. If direct plus channel was an intentional product decision from the shipped changelog entry, maintainers should close it manually with that rationale and align the conflicting config-help wording.
