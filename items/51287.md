## Summary

Current main no longer matches the issue’s cited 2-minute default, but the requested configurability is still not implemented. Exec approval expiry is hardcoded to 30 minutes and the request/transport wait uses derived hardcoded constants; there is no `openclaw.json` approvals timeout field on main.

## Evidence

- **Hardcoded exec approval default:** `DEFAULT_EXEC_APPROVAL_TIMEOUT_MS` is still a constant set to `1_800_000` ms (30 minutes), not a config-backed value.
  - file: [src/infra/exec-approvals.ts:167](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/infra/exec-approvals.ts#L167)

  - file: [src/agents/bash-tools.exec-runtime.ts:122](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/agents/bash-tools.exec-runtime.ts#L122)
