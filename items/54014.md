## Summary

Keep open. Current main partially honors OPENCLAW_HOME for default state/config, agent workspace, and agentDir resolution, but the broader issue remains: git checkout defaults and installer/onboarding paths still derive from os.homedir() or $HOME, while docs still promise OPENCLAW_HOME replaces the system home for internal path resolution.

## Best Possible Solution

Keep this issue open and narrow it to the OPENCLAW_HOME path contract. If git checkout and installer/onboarding defaults are considered internal OpenClaw paths, update src/cli/update-cli/shared.ts, scripts/install.sh, docs/help/environment.md, docs/cli/update.md, docs/install/installer.md, and focused tests so OPENCLAW_HOME seeds those defaults unless OPENCLAW_GIT_DIR, OPENCLAW_STATE_DIR, or OPENCLAW_CONFIG_PATH is explicitly set. If maintainers want git checkout, personal ~/.agents skills, trash, or host-file tilde expansion to remain OS-home based, document those as explicit exceptions.
