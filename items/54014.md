## Summary

Keep open. Current main partially respects OPENCLAW_HOME for the shared effective-home resolver, state dir, config path, and profile-derived state/config paths, but the issue’s explicit OPENCLAW_GIT_DIR default request is still not implemented. The update CLI and shell installer still default git checkout paths to the OS/HOME-based ~/openclaw unless OPENCLAW_GIT_DIR is set, and installer onboarding config checks still use $HOME/.openclaw/openclaw.json rather than an OPENCLAW_HOME-derived default.

## Best Possible Solution

Keep the issue open and narrow it to the remaining OPENCLAW_HOME contract. A maintainer should decide whether OPENCLAW_HOME should seed default OPENCLAW_GIT_DIR and installer onboarding paths; if yes, update src/cli/update-cli/shared.ts, scripts/install.sh, docs, and tests. Review #65536 separately for sandbox tilde expansion, because it is related but does not supersede this broader report.
