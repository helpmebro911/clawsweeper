## Summary

Keep open. Current main still has the remaining send-path false positive for non-empty or true poll-looking fields, and the provided GitHub context shows open PR #46455 still references this issue with closing syntax. Main partially mitigates the report by hiding Discord native `components` from the generic message tool schema and ignoring zero-valued poll defaults, but the broader action-aware schema/runtime split requested here is not implemented.

## Best Possible Solution

Keep this issue open. Maintainers should review, rebase, land, or explicitly supersede PR #46455 or an equivalent patch that makes poll creation fields action-aware, strips model-populated poll/component noise from `action="send"`, preserves strict validation on explicit poll/component paths, and adds focused regression coverage for Telegram/Feishu/Discord send-with-file plus Discord empty-modal skeleton behavior.
