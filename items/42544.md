## Summary

Keep open. Current `main` still unconditionally runs stale-gateway cleanup whenever `OPENCLAW_SERVICE_MARKER` is set, and that cleanup kills any verified OpenClaw gateway listener on the port without a health check. The PR also has unresolved correctness issues called out in review, so there is not strong evidence to close it as fixed, obsolete, or non-reproducible.

## Evidence

- **Current main still does unconditional service-mode cleanup:** `runGatewayCommand` still calls `cleanStaleGatewayProcessesSync(port)` solely because `OPENCLAW_SERVICE_MARKER` is present; there is no health probe before the kill path.
  - file: [src/cli/gateway-cli/run.ts:440](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/cli/gateway-cli/run.ts#L440)
