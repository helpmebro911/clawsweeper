## Summary

Keep open. Current `main` does not implement this PR's behavior: service-mode startup still unconditionally runs stale-gateway cleanup when `OPENCLAW_SERVICE_MARKER` is set, and that cleanup still terminates verified OpenClaw gateway listeners on the port without any health probe. The PR also has unresolved correctness concerns in review, so there is not strong evidence to close it as already implemented or obsolete.

## Evidence

- **Service-mode path still kills before any health check:** On current `main`, `runGatewayCommand` still does `cleanStaleGatewayProcessesSync(port)` immediately when `OPENCLAW_SERVICE_MARKER` is set. There is no `/health` probe or skip path in `src/cli/gateway-cli/run.ts:440`.
