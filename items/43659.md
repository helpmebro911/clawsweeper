## Summary

Keep open. Current main still reuses a single unowned GitHub Copilot API-token cache entry across profiles, and this PR remains a plausible fix candidate for the same-author paired issue #43658.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Port the cache-ownership check to `src/agents/github-copilot-token.ts`, preserve the SDK/plugin re-export, add regression tests for different GitHub tokens and legacy unbound cache entries, and review it together with #43658.

## Risks / Open Questions

- The PR cannot be merged as-is without porting because it targets the old `src/providers/github-copilot-token.ts` path while current main keeps the helper in `src/agents/github-copilot-token.ts` and exports it through `openclaw/plugin-sdk/github-copilot-token`.
- The proposed fingerprint approach writes a stable credential-derived identifier to disk; maintainers should decide whether that is acceptable or prefer a profile/account-owned cache key that does not derive from the raw PAT.
- Even with the PR's fingerprint check, a single shared cache file would still evict valid tokens when alternating profiles, causing extra token exchanges; that is an efficiency follow-up, not a reason to close this correctness fix.
