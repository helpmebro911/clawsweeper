## Summary

Keep open. Current main still has the unowned shared GitHub Copilot API token cache that this PR is trying to fix, and the paired bug #43658 remains open. The PR is a plausible focused fix candidate, but it targets the old `src/providers` path and needs porting to the current `src/agents` / plugin SDK path before it can land.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Port the cache-ownership check to `src/agents/github-copilot-token.ts`, preserve `openclaw/plugin-sdk/github-copilot-token` behavior, add regression tests for different GitHub tokens and legacy unowned cache entries, and review it together with #43658. Consider whether the cache should be keyed per profile/fingerprint instead of storing a stable credential-derived field in a single shared file.

## Risks / Open Questions

- The PR cannot land as-is because it edits the old `src/providers` location; the fix needs to be ported to `src/agents/github-copilot-token.ts` and covered through the current SDK/plugin re-export path.
- Persisting a truncated SHA-256 of the GitHub PAT is not the raw secret, but it is a stable credential-derived identifier. Maintainers may prefer a profile-owned cache key, per-profile cache file, or keyed derivation.
- Even with a fingerprint mismatch check, a single cache file would still evict valid tokens when alternating profiles; that is an efficiency follow-up, not a reason to close the correctness fix.
