## Summary

Current main still uses schtasks for Windows gateway Scheduled Task install, run, stop/restart, and runtime status, and still documents/tests the Startup-folder fallback. The PR is a focused core daemon change rather than obsolete cleanup, but it remains unmerged and has Windows-specific functional/security-sensitive command-execution review risks that need maintainer review before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a focused Windows daemon change in src/daemon/schtasks.ts that makes PowerShell ScheduledTasks the primary path, preserves a valid schtasks fallback, keeps uninstall cleanup of Scheduled Task plus Startup-folder/script artifacts, updates docs/tests for the chosen admin/fallback policy, and adds targeted tests for PowerShell success, PowerShell-unavailable fallback, denied/non-admin behavior, restart, runtime parsing, and uninstall cleanup.
