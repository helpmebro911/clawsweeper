## Summary

Keep open. Current `main` still does not implement the PR's static built-in-channel fallback for `sandbox explain`, and the sandbox CLI defaults to running without plugin loading, so the reported `(unknown)` channel path remains plausible on `main`.

## Evidence

- **`sandbox explain` still uses runtime-only channel normalization:** `inferProviderFromSessionKey()` returns `normalizeAnyChannelId(candidate) ?? undefined`, and `resolveActiveChannel()` also only tries `normalizeAnyChannelId()` before falling back to the same helper. There is no `normalizeChatChannelId()` fallback in current `main`.
  - file: [src/commands/sandbox-explain.ts:61](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/commands/sandbox-explain.ts#L61)
  - command: `nl -ba src/commands/sandbox-explain.ts | sed -n '1,130p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **`normalizeAnyChannelId()` depends on an already-loaded runtime plugin registry:** `listRegisteredChannelPluginEntries()` returns an empty array when no active channel registry is present, and `normalizeAnyChannelId()` only resolves through that registry. It does not fall back to bundled built-in channel metadata.
  - file: [src/channels/registry.ts:27](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/channels/registry.ts#L27)
  - command: `nl -ba src/channels/registry.ts | sed -n '1,120p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Static built-in channel normalization exists separately:** `normalizeChatChannelId()` resolves against the bundled channel catalog and would cover built-in channel ids without requiring the runtime plugin registry.
  - file: [src/channels/ids.ts:46](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/channels/ids.ts#L46)
  - command: `nl -ba src/channels/ids.ts | sed -n '1,120p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **CLI startup defaults to not loading plugins:** The default CLI command policy sets `loadPlugins: "never"`, and there is no sandbox-specific override in the command catalog, so `openclaw sandbox explain` does not reliably have a populated runtime channel registry.
  - file: [src/cli/command-path-policy.ts:5](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/cli/command-path-policy.ts#L5)
