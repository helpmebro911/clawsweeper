## Summary

Keep #8813 open. Current main still builds the Skills system-prompt section with an unconditional `read` instruction, even though prompt construction already normalizes runtime tool names and preserves caller casing. I found no `Skill`-tool branch in source, tests, or docs; existing tests and docs still encode the current read-based behavior.

## Best Possible Solution

Keep #8813 open. The best fix is in `src/agents/system-prompt.ts`: pass normalized available-tool information and casing resolution into `buildSkillsSection`, prefer invoking a registered Skill tool when present, preserve the existing `read` fallback otherwise, then update `src/agents/system-prompt.test.ts` and the public Skills/system-prompt docs for both behaviors.
