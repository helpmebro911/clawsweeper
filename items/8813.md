## Summary

Keep #8813 open. Current main still hardcodes the Skills section to tell the model to read SKILL.md with the read tool, even though buildAgentSystemPrompt already computes the normalized available tool set. Tests and docs still assert/describe read-based skill loading, and the provided issue discussion already includes a recent Codex review reaching the same conclusion.

## Best Possible Solution

Keep the issue open until core prompt composition supports this explicitly. The likely fix is to pass the normalized availableTools set and casing resolver into buildSkillsSection, detect a registered `skill`/`Skill` tool, emit Skill-tool invocation guidance instead of read-based guidance when present, and update system prompt tests plus public docs for both paths.
