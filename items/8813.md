## Summary

Keep #8813 open. Current main still emits read-based Skills guidance unconditionally; `buildSkillsSection` cannot inspect the registered tool set, even though prompt construction already receives runtime tool names. Existing tests and docs still assert/document loading `SKILL.md` via `read`, and the related items do not supersede this narrower Skill-tool preference request.

## Best Possible Solution

Keep #8813 open. The best fix is in `src/agents/system-prompt.ts`: pass normalized available-tool information plus casing resolution into `buildSkillsSection`, prefer invoking a registered `Skill` tool when present, preserve the current `read` fallback otherwise, then update `src/agents/system-prompt.test.ts` and the public Skills/system-prompt docs for both behaviors.
