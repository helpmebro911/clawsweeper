## Summary

Keep open. Current main has explicit per-call ACP runTimeoutSeconds plumbing, but it still does not implement the PR's central acp.defaultRunTimeoutSeconds config fallback for omitted ACP sessions_spawn calls. The linked feature request #38419 was closed stale rather than shipped, so this PR remains the focused implementation candidate after a rebase.

## Best Possible Solution

Keep this PR open and ask for a narrow rebase onto current main. Preserve explicit per-call runTimeoutSeconds precedence, including 0; add acp.defaultRunTimeoutSeconds to the public config contract; resolve ACP timeout as explicit per-call value -> ACP config default -> 0; apply the resolved value consistently to ACP runtime initialization and Gateway dispatch; and keep focused tests for omitted/default, explicit nonzero override, and explicit 0 overriding a nonzero default.
