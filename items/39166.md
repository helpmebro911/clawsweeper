## Summary

Keep this PR open. Current main already forwards explicit ACP `runTimeoutSeconds` from `sessions_spawn(runtime="acp")`, but it still has no `acp.defaultRunTimeoutSeconds` config key, schema support, docs, or ACP timeout resolver for omitted values. The linked feature request #38419 was closed stale, not shipped, and this PR remains a focused implementation candidate rather than something to close before merge.

## Best Possible Solution

Keep this PR open and ask for a focused rebase onto current main. The maintainer path is to preserve current explicit ACP timeout forwarding, add `acp.defaultRunTimeoutSeconds` to the public config type, strict Zod schema, generated schema metadata, help/labels, and user docs, then resolve ACP timeout as explicit per-call value -> ACP config default -> `0`. Tests should cover omitted config default, explicit nonzero override, and explicit `0` overriding a nonzero config default across ACP runtime initialization and Gateway dispatch. Because this PR is the open implementation candidate for #38419, the linked issue should only be considered implemented after the PR or equivalent work lands.
