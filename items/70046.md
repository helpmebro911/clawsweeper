## Summary

Keep #70046 open. Current main still does not support bare `HH:MM` / `HH:MM:SS` values in `cron add --at`, and the PR is the active clean respin for the open linked bug #59441 after #59444 was closed unmerged.

## Best Possible Solution

Keep #70046 open for maintainer review rather than auto-closing it. The likely best path is to land a focused fix in `src/cli/cron-cli/shared.ts` that detects valid `HH:MM` / `HH:MM:SS`, resolves today's date in the supplied IANA timezone using the existing offset-less datetime helper, returns `null` for invalid timezones or out-of-range times, and adds targeted parser tests. Help-text-only wording can remain tracked separately by #59456 / #69975 if maintainers want that split.

## Evidence
