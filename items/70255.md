## Summary

Keep PR #70255 open. Current main does not implement the one-shot exit behavior for `openclaw hooks` inspection/mutation commands, and the PR remains a focused, relevant fix for a real gap: hooks diagnostics load plugin modules and execute plugin `register(api)`, so unmanaged plugin handles can keep the CLI alive after output is printed. The PR discussion also shows review feedback was addressed in follow-up commits.

## Best Possible Solution

Keep PR #70255 open and land it, or an equivalent patch, after normal maintainer CI. The right product/code direction is a scoped one-shot hooks action wrapper for `hooks list`, `info`, `check`, `enable`, `disable`, and bare `hooks`, guarded by the shared Vitest runtime-env detection and leaving deprecated `hooks install`/`update` plus relay behavior unchanged.
