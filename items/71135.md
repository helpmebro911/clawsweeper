## Summary

Keep PR #71135 open. Current main does not already implement the proposed per-session working-context capsule, the canonical issue #67511 remains open, and the PR is a recent draft with a focused storage primitive plus tests rather than an obsolete or low-signal change. It needs maintainer product/design review on the storage shape and integration points, not conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to review the proposed capsule schema and storage location, then either land the utility module with an explicit follow-up plan for #67511 or ask the author to fold in the first runtime integration point so the persisted state is populated and visible in `/status`. Closing as implemented, duplicate, or stale would lose a recent focused contribution for an open, unsolved agents/session-continuity request.
