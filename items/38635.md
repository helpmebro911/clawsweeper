## Summary

Current main still has the UX bug this PR targets: chat focus mode suppresses the nav/drawer, while the visible topbar hamburger only toggles drawer state and never exits user-initiated focus mode. The PR remains useful, though its patch likely needs rebasing to the current `.topbar-nav-toggle` path.

## Best Possible Solution

Keep this PR open or replace it with an equivalent current-main patch: when user-initiated `chatFocusMode` is active, clicking `.topbar-nav-toggle` should clear `chatFocusMode` and open the nav/drawer; onboarding-controlled focus should remain controlled by onboarding. Add a browser regression test that activates `.shell--chat-focus`, clicks `.topbar-nav-toggle`, and verifies focus mode exits and the nav/drawer becomes visible.
