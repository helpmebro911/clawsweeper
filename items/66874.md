## Summary

Keep this open. Current main still supports the bundled Anthropic Claude CLI backend, but the official Docker image does not pre-install `@anthropic-ai/claude-code`, the Docker release workflow does not add it, and the user-facing Docker docs still do not provide a Docker-specific persistent install plus `~/.claude` credential recipe.

## Best Possible Solution

Keep this open until maintainers choose the Docker product path. A conservative first step is a Docker-specific Claude CLI section in `docs/install/docker.md`, cross-linked from `docs/providers/anthropic.md` and `docs/gateway/cli-backends.md`, with a persistent npm prefix or `/home/node` volume, `~/.claude` persistence, an explicit `agents.defaults.cliBackends.claude-cli.command` plus `agentRuntime.id: "claude-cli"` example, and Anthropic API-key fallback guidance. If maintainers want the lowest-friction happy path, add a pinned `@anthropic-ai/claude-code` install to the official Docker image and cover it in Docker release/test/docs.

## Risks / Open Questions

- Pre-installing Claude Code in the official image would add third-party package pinning, image-size, update-policy, and supply-chain maintenance work; maintainers should choose that deliberately rather than treating it as an incidental Dockerfile change.
- A docs-only fix needs to be precise about both binary persistence and OAuth credential persistence, because persisting `/home/node/.openclaw` alone does not preserve Claude Code's `~/.claude/.credentials.json` state.
