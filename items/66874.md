## Summary

Keep open: current `main` does not preinstall the `claude` CLI in the official Docker image, and the existing Docker docs only provide generic persistence/custom-image guidance, not the Claude-specific official-image path or persistent install instructions requested in #66874.

## Evidence

- **Official Docker image does not include Claude CLI:** The repository Dockerfile has no `claude` or `@anthropic-ai/claude-code` reference. Its runtime stage installs base apt utilities, Corepack/pnpm, optional apt packages, and optional Playwright browsers, but no Claude CLI package.
  - file: [Dockerfile:158](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/Dockerfile#L158)
  - command: `rg -n "@anthropic-ai/claude-code|\\bclaude\\b" Dockerfile || true; nl -ba Dockerfile | sed -n '158,220p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Anthropic provider docs assume Claude CLI is already installed:** The Anthropic docs tell users to 'Ensure Claude CLI is installed and logged in' and verify with `claude --version` before onboarding. That confirms current docs do not claim the Docker image ships the binary.
  - file: [docs/providers/anthropic.md:70](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/providers/anthropic.md#L70)
  - command: `nl -ba docs/providers/anthropic.md | sed -n '70,98p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **CLI backend docs do not add Docker-specific setup:** The CLI backend docs say Claude Code must already be logged in on the same host and only mention `agents.defaults.cliBackends.claude-cli.command` when the binary is not on `PATH`. There is no Docker-specific persistent install recipe there.
  - file: [docs/gateway/cli-backends.md:179](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/gateway/cli-backends.md#L179)
  - command: `nl -ba docs/gateway/cli-backends.md | sed -n '179,190p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Docker docs persist OpenClaw state, but not Claude CLI setup:** The Docker install docs document persistence for `/home/node/.openclaw` and mention optional `OPENCLAW_HOME_VOLUME`, but they do not mention `claude`, `claude-cli`, or a persistent Claude CLI install path.
  - file: [docs/install/docker.md:168](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/install/docker.md#L168)
  - command: `rg -n "claude|Claude CLI|claude-cli" docs/install/docker.md docs/install/docker-vm-runtime.md || true; nl -ba docs/install/docker.md | sed -n '168,181p'; nl -ba docs/install/docker.md | sed -n '273,286p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Current Docker VM guidance is generic custom-image advice:** `docs/install/docker-vm-runtime.md` already says runtime-installed binaries are lost on restart and external binaries must be baked into the image at build time, but the examples are generic and not Claude-specific. That is adjacent guidance, not a direct resolution of this issue's request for the official image or explicit Claude docs.
