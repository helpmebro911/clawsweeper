## Summary

Keep open. Current `main` still does not implement the requested tree-scoped creator-owned ACP exception. Tree visibility rejects cross-agent targets before considering creator ownership, `sessions_send` only narrows label lookup by `spawnedBy` for sandbox-restricted flows, and there is no `tools.sessions.ownedAcp` config/schema/docs support on `main`.

## Evidence

- **Tree visibility still blocks cross-agent targets outright:** `createSessionVisibilityGuard()` computes `isCrossAgent` from the target/requester agent ids and returns forbidden unless visibility is `all`; the tree-spawned check runs only after that, so creator-owned ACP sessions under another agent id are still excluded.
  - file: [src/agents/tools/sessions-access.ts:206](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/agents/tools/sessions-access.ts#L206)
  - command: `nl -ba src/agents/tools/sessions-access.ts | sed -n '185,245p'`
  - sha: [d29eaeafc1fc](https://github.com/openclaw/openclaw/commit/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132)
- **Label+agent lookup is not narrowed by spawnedBy in normal tree mode:** `sessions_send` adds `spawnedBy: effectiveRequesterKey` only when `restrictToSpawned` is true. In ordinary unsandboxed tree mode, label resolution is still broad and can hit unrelated rows or fail before the later visibility guard.
  - file: [src/agents/tools/sessions-send-tool.ts:149](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/agents/tools/sessions-send-tool.ts#L149)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '116,170p'`
  - sha: [d29eaeafc1fc](https://github.com/openclaw/openclaw/commit/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132)
- **Spawned-session plumbing exists, but only as generic spawnedBy filtering:** The helper paths do call `sessions.list` / `sessions.resolve` with `spawnedBy`, and gateway store filtering accepts `entry.spawnedBy` or `entry.parentSessionKey`, so the missing piece is the higher-level tree visibility exception rather than raw spawnedBy storage.
  - file: [src/agents/tools/sessions-resolution.ts:50](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/agents/tools/sessions-resolution.ts#L50)
  - command: `nl -ba src/agents/tools/sessions-resolution.ts | sed -n '50,100p'`

- **Gateway spawnedBy filtering can see owned ACP rows:** `listSessionsFromStore()` filters by `entry.spawnedBy === spawnedBy || entry.parentSessionKey === spawnedBy`, which supports owned ACP rows once queried narrowly.
  - file: [src/gateway/session-utils.ts:1506](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/gateway/session-utils.ts#L1506)

No `ownedAcp` option exists in docs, schema, or types, and `rg -n "ownedAcp" src docs test packages extensions` returned no matches.
