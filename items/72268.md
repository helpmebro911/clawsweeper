## Summary

Keep open. Current main still has the exact parser weakness #72268 fixes: exec approval result parsing uses first-close-paren regexes, while the gateway can emit nested metadata such as `approval-timeout (allowlist-miss)`. That nested denial is therefore not recognized as denied and can flow into the generic completion/fallback path. The related #72148 addresses the resume-failed prefix, not this nested-metadata classification bug.

## Best Possible Solution

Keep #72268 open and land this PR or an equivalent balanced-metadata parser with the nested denial/finished regression tests. #72148 can proceed separately for the user-facing resume-failed prefix, but it does not supersede this parser fix.
