## Summary

Keep this PR open. Current main still parses `Exec denied (...)` and `Exec finished (...)` metadata with `[^)]*`, while current exec gateway code can emit nested metadata such as `approval-timeout (allowlist-miss)`. That still misclassifies the denied result and sends it down the generic completion path. The related #72148 is adjacent wording work, not a parser replacement.

## Best Possible Solution

Keep this PR open and review or land the focused parser fix: replace the regex metadata extraction with balanced parsing for `Exec denied` and `Exec finished`, keep the nested-parentheses parser and followup-routing regression tests, and treat #72148 as a separate wording/logging fix.
