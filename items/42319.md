## Summary

Keep open. Current `main` does not contain the Google Chat auth rewrite or the gateway health runtime-snapshot wiring from PR #42319, and the PR discussion identifies a real regression risk in the proposed health refactor: `buildChannelAccountSnapshot()`'s default path still drops `probe` while multiple channel summaries read `snapshot.probe`. This is useful work, but it is not implemented on `main` and is not safe to auto-close as obsolete.

## Evidence

- **Google Chat auth fix is not on current main:** `extensions/googlechat/src/auth.ts` on `dfa52aaab0f4ca5a9416c1eea21b20b63631e85f` still builds a `GoogleAuth` client and calls `auth.getClient().getAccessToken()`. It also still returns the shared cache key `"none"` for accounts without inline/file credentials, so the PR's manual JWT exchange and `none:<accountId>` cache-key fix are not present on main.
