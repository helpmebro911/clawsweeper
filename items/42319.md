## Summary

Keep this PR open, but it should not merge as written. Current main still does not feed live channel runtime state into health snapshots, so the health-side goal remains useful. The PR is stale against current main, though: its Google Chat auth rewrite would bypass the current SSRF-guarded google-auth transport and strict Google endpoint validation, and its health refactor still has the probe-preservation regression called out in review comments.

## Best Possible Solution

Keep this PR open as a stale but useful implementation candidate. The best path is a narrow health/runtime patch: pass the gateway `ChannelRuntimeSnapshot` into `getHealthSnapshot`, reuse the existing `buildChannelAccountSnapshot`/status-summary contract, preserve `probe`, `lastProbeAt`, configured/enabled fields, and read-only channel plugin discovery, and leave Google Chat auth on the current SSRF-guarded google-auth runtime unless a separately evidenced auth bug is fixed with equivalent endpoint and credential safeguards.
