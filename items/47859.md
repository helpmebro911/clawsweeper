## Summary

Current main still does not implement the PR’s TUI `/image` command or `@path` attachment flow. The gateway and WebChat already have a usable `chat.send` attachment contract, so this PR is plausible TUI surface work rather than obsolete core work. However, the provided review comments identify real correctness issues in the PR diff, so it should not be closed as implemented or merged as-is.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. The best path is a focused TUI implementation on current main that extends the TUI backend/client to pass existing `chat.send` attachments, adds `/image` and `@path` parsing with tests, preserves user text when no image reference is consumed, and handles attachment-only messages without leaking local paths into the prompt.
