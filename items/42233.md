## Summary

Keep open. Current `main` still reports `sessions_send` wait timeouts as plain `status: "timeout"` without a separate unknown-delivery state, and it does not attach delivery confirmation metadata on that timeout path. The codebase also explicitly acknowledges that `sessions_send` timeouts/errors may still have delivered the message, which matches the report rather than disproving it.

## Evidence

- **`sessions_send` still returns plain timeout with no delivery status:** The tool sets `delivery` only for accepted/ok flows, but if the nested wait times out it returns `{ status: "timeout", error, sessionKey }` directly, with no `delivery` field and no distinction like `timeout_waiting_ack` vs known failure.
