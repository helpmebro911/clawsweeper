## Summary

Current `main` does not implement configurable cost metadata for auto-discovered Amazon Bedrock models. The discovery config still has no pricing field, discovered models are still hard-coded to zero cost, and the explicit-provider merge path still replaces implicit discovered models rather than overlaying per-model cost metadata.

## Evidence

- **Bedrock discovery config has no cost overlay field:** `BedrockDiscoveryConfig` only exposes `enabled`, `region`, `providerFilter`, `refreshInterval`, `defaultContextWindow`, and `defaultMaxTokens`.
  - file: [src/config/types.models.ts:126](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/config/types.models.ts#L126)
  - command: `nl -ba src/config/types.models.ts | sed -n '120,150p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Discovered foundation models still get zeroed pricing:** Bedrock discovery defines `DEFAULT_COST` as all zeros and assigns `cost: DEFAULT_COST` when building discovered model definitions.
  - file: [extensions/amazon-bedrock/discovery.ts:139](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/amazon-bedrock/discovery.ts#L139)
