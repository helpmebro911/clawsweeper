## Summary

Current main has some of the underlying proxy/TLS behavior and partial docs, but it does not already contain this enterprise systemd proxy/SSL-inspection guide. The PR still addresses a real docs gap; it just needs updating to current docs conventions and current service naming rather than being auto-closed.

## Evidence

- **Current VPS docs use newer service naming and edit flow:** The current Linux server doc tells operators to edit the user unit with `systemctl --user edit openclaw-gateway.service`, and to use `sudo systemctl edit openclaw-gateway.service` only for a deliberate system unit install. That means the PR's older `sudo systemctl edit openclaw` examples are stale against current main, but the docs topic itself is not already implemented there.
