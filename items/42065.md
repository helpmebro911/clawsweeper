## Summary

Keep this PR open. Current main partially supersedes it with a generated Control UI i18n pipeline and an Overview-card language picker, but the requested global topbar locale switch is still absent and some topbar/dashboard copy remains hardcoded in English. The PR should be reworked before merge because it adds a parallel i18n checker/hand-edited generated locale output and its switcher saves settings before async locale loading completes.

## Best Possible Solution

Rebase or rewrite the PR against current main: keep scripts/control-ui-i18n.ts as the single sync/check path, decide explicitly whether the locale picker should move to the topbar or remain Overview-only, localize remaining topbar/search/sidebar and relevant Overview strings through ui/src/i18n/locales/en.ts plus generated locale output, and make any revived switcher await successful i18n.setLocale before saving settings.
