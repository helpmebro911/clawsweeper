## Summary

Keep this PR open. Current main has a newer generated Control UI i18n system and an Overview-page language picker, so much of the PR is stale, but main still does not implement the PR's explicit topbar/global locale switcher placement and still has hardcoded topbar theme-mode copy. There is no canonical follow-up or maintainer decision saying the current Overview placement is the intended replacement.

## Best Possible Solution

Ask for a rebase/rework or make a maintainer follow-up: use the existing scripts/control-ui-i18n.ts sync/check pipeline, move or duplicate the current Overview language picker into the topbar only if maintainers want that placement, localize remaining topbar theme-mode copy, await successful i18n.setLocale before saving settings, and update docs/tests for the chosen placement. If maintainers intentionally prefer the current Overview-only picker, close manually with that product rationale.
