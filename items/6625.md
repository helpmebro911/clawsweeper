## Summary

Keep #6625 open. Current main still implements hard sub-agent run timeouts plus post-timeout partial-progress recovery, but I found no configurable grace period, no sessions_spawn pre-timeout warning parameter, and no native or ACP scheduler that injects a checkpoint warning before runTimeoutSeconds expires.

## Best Possible Solution

Keep #6625 open for core sub-agent lifecycle work. Add a documented sub-agent grace or warning setting, optionally a sessions_spawn override, schedule a warning before runTimeoutSeconds, deliver it through the child-session control/message path for native and ACP runs where feasible, and cover the behavior with focused tests and docs while preserving existing post-timeout partial-progress recovery.
