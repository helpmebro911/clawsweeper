## Summary

Keep #6625 open. Current main still supports hard sub-agent run timeouts and post-timeout announce/partial-progress recovery, but I found no configurable grace period, no pre-timeout warning field in the sessions_spawn schema, and no native or ACP scheduler that injects a checkpoint warning before the run timeout expires.

## Best Possible Solution

Keep #6625 open for core sub-agent lifecycle work. Add a documented sub-agent grace/warning setting, optionally a deliberate sessions_spawn override, schedule a warning before runTimeoutSeconds, deliver it through the child-session control/message path for native and ACP runs where feasible, and cover the behavior with focused tests and docs while preserving existing post-timeout partial-progress recovery.
