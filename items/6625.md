## Summary

Current `main` already preserves timed-out subagent progress well enough to make the reported data-loss problem obsolete: timeout announce now recovers recent assistant output or a partial-progress summary from `chat.history`, and the timed-out subagent session/transcript remains inspectable instead of being immediately deleted.

## Close Comment

Closing from a ClawSweeper/Codex review because current `main` already covers the underlying timeout-loss problem well enough. Timed-out subagents now recover progress from `chat.history` during announce, including recent assistant text or a `[Partial progress: N tool call(s) executed before timeout]` summary (`src/agents/subagent-announce-output.ts:233`, covered by `src/agents/subagent-announce.timeout.test.ts:426` and `src/agents/subagent-announce.timeout.test.ts:509`). The timed-out child is also not immediately discarded: `/subagents info` exposes the transcript path, `sessions_history` can recall it, and `runTimeoutSeconds` only stops the run rather than deleting the session (`docs/tools/subagents.md:34` and `docs/tools/subagents.md:152`). This landed on `main` in `467fcb17912e14797bc8caa6689c30d290e39d01`. There is not enough evidence here to name a shipped release yet, so I am only marking it implemented on `main`.
