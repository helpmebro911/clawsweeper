## Summary

Keep open. Current main still has the Docker VM runtime URL problems and an AGENTS Node-baseline mismatch, and external checks confirm the reported binary URLs are real failures. However, the PR should not be merged as-is: its gateway AGENTS hunk conflicts with the current CLI because `gateway restart` only accepts `--json`, while `--deep` belongs to `gateway status`. The related #72154 overlaps the gog/goplaces Docker URL work but does not fully supersede this PR because it leaves wacli and AGENTS cleanup out of scope.

## Best Possible Solution

Keep the PR open but request a focused revision: fix or drop the AGENTS gateway wording so `--deep` is tied to `gateway status`, update the root AGENTS Node baseline only if maintainers want that contributor-facing rule aligned to `>=22.14.0`, and coordinate with #72154 so the Docker URL fix lands once, including a deliberate decision on whether the Linux Docker example should remove wacli or document a different install path.
