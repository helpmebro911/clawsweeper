## Summary

Keep open. Current main has some model-display fixes for stored session rows, but the exact TUI defaults path reported here is still global-default based: `sessions.list` filters by active agent but returns `getSessionDefaults(cfg)` without an agent id, and the TUI uses those defaults when the active session has no persisted row. The related PR #48110 identified the same missing `agentId`-aware defaults fix, but that PR is closed unmerged and the current signature still lacks the proposed parameter.
