## Summary

Keep open. Current main still hard-blocks `qwen3.6-plus` on Qwen Coding Plan endpoints in the catalog, config normalization, runtime suppression, tests, and docs. PR #70957 is relevant but narrower than the remaining guards because it only changes `extensions/qwen/models.ts` and `extensions/qwen/provider-catalog.test.ts`.

## Best Possible Solution

Keep #70957 open unless maintainers explicitly choose PR #66367 or another broader patch as canonical. The best implementation should remove or revise all hard-coded `qwen3.6-plus` Coding Plan guards in `extensions/qwen/models.ts` and `extensions/qwen/index.ts`, update `extensions/qwen/provider-catalog.test.ts`, and align `docs/providers/qwen.md` with the verified Alibaba/Qwen Coding Plan contract.
