## Summary

Keep #8295 open. Current main still does not ship Telegram `allowBots` for groups/topics in the public config types, strict schema, generated config metadata/help, docs, or inbound runtime policy. The related Plaiground ClawHub skill is a useful Discord workaround, and PR #68325 is relevant prior/current work, but neither closes the requested bundled Telegram plugin behavior on main.

## Best Possible Solution

Keep #8295 open until the bundled Telegram plugin either implements `allowBots` across account/group/topic config types, strict schema, generated metadata/UI help, docs, inbound filtering, and regression tests, or maintainers explicitly document that Telegram group bot-to-bot delivery is not feasible and point users to supported alternatives such as Telegram channels or the Plaiground Discord/ClawHub workaround.
