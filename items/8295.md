## Summary

Keep #8295 open. Current main still does not implement Telegram `allowBots` for groups/topics in config types, strict schema, generated metadata/help, docs, or inbound runtime filtering. The related Plaiground ClawHub skill is a useful Discord workaround, and PR #68325 is relevant active work, but neither closes the requested bundled Telegram plugin behavior on main.

## Best Possible Solution

Keep #8295 open until the bundled Telegram plugin implements `allowBots` across account/group/topic config types, strict schema, generated metadata/help, docs, inbound group/topic filtering, and regression tests. PR #68325 is the active candidate to review; if Telegram bot-to-bot group delivery is not feasible, maintainers should document that limitation and point users to supported alternatives such as Telegram channels or the Plaiground Discord/ClawHub workaround.
