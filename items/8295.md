## Summary

Keep open. Current main still does not implement Telegram allowBots parity: the Telegram config schema/types do not accept the field, and the Telegram inbound group path has no allowBots policy comparable to Discord/Slack. The existing Telegram bot-to-bot support is channel_post/channel-oriented, not the requested group/topic allowBots control. PR #8310 attempted this but was closed unmerged; the Plaiground ClawHub note is a useful Discord workaround, not Telegram group support.

## Best Possible Solution

Keep #8295 open until Telegram implements a real allowBots policy. The likely fix belongs in the bundled Telegram plugin/config contract: add account/group/topic allowBots types and strict Zod schema support, update config metadata/docs/UI surfaces, filter delivered bot-authored group/topic messages consistently with Discord/Slack while preserving self-message and channel_post behavior, and add regression tests plus live QA coverage around BotFather Bot-to-Bot Communication Mode.
