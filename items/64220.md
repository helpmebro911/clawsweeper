## Summary

Current main still matches the reported Control UI stale-state bug. Session switching now clears compaction/fallback status in one helper, but active/retrying compaction still has no stale timeout, the renderer still shows it indefinitely, and terminal/reconnect/clear-history/send-failure cleanup paths still clear chat state without clearing compactionStatus/fallbackStatus. The related #39908 is a closed broader symptom report, not a shipped fix or canonical replacement for this narrower state-machine issue.

## Best Possible Solution

Keep this issue open and fix it in the Control UI state layer. Add a run/session-scoped stale timeout or watchdog for active/retrying compaction, centralize cleanup of compactionStatus/fallbackStatus and their timers, call that cleanup on terminal chat events, reconnect/disconnect, session/agent switch, overview/chat clear-history, reset, and send-failure paths, then add regression tests for missed compaction end after final reply plus reconnect and clear-history cleanup.
