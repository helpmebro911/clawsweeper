## Summary

Current `main` still only fires `message_sending` inside direct outbound delivery, not along the Message Tool paths this PR targets, so the underlying bug remains real. The PR should stay open rather than be closed, but the review thread also surfaces unresolved correctness risks around cancelled direct `sendMessage()` callers, CLI reporting, and attachment metadata parity that merit maintainer review before merge.

## Risks / Open Questions

- Closing this PR would leave a still-open, user-visible bug on `main`: Message Tool sends do not receive `message_sending` hooks today.
- Merging the PR without resolving the reviewed cancellation semantics could create false-positive delivery reporting for direct `sendMessage()` callers and CLI output.
- If the upstream hook path omits `metadata.mediaUrls`, policy plugins that inspect attachments could regress on Message Tool sends even while text-only cases are fixed.
