## Summary

Current main does not implement the requested Dashboard Sessions prune workflow. The UI supports filtering, selecting rows, and deleting explicitly selected sessions, while generic CLI/session-store maintenance prunes by configured age or missing transcript files. There is no Dashboard Prune button, no 48-hour age input/default, no eligibility preview for orphaned Telegram/Discord channels/topics, and no source path that checks whether those remote channels still exist before cleanup. Related #50310 is narrower Discord thread cleanup work and remains open; it does not supersede this broader Dashboard request.
