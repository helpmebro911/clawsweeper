## Summary

Current main still has a SQLite-backed memory search index over Markdown files, not the requested durable typed SQLite memory record model with canonical rows, revision history, lifecycle statuses, validity windows, confidence, and normalized entities. The existing keep-open ClawSweeper comment remains consistent with the current checkout.

## Best Possible Solution

Keep this issue open. The best path is a versioned durable-memory storage contract in the memory owner surface, likely memory-core plus the memory-host SDK: add migrations and contract tests for canonical memories rows, memory_revisions, explicit type/status/confidence/timestamp/validity/entity fields, and docs that explain how the current Markdown/index backends interact with the new source-of-truth model. Coordinate with #42647, #42648, and #42650 before landing the schema.
