## Summary

Current main still preserves the reported memory-core behavior: default dreaming promotion requires repeated signal and diversity gates, and tests explicitly assert that one-off high-scoring recalls do not reach MEMORY.md under default thresholds. Manual CLI/config threshold overrides exist, but they are not a durable priority lane for important single-event feedback.

## Best Possible Solution

Keep this open for memory-core maintainer/product design. The best implementation is a tested durable lane for high-priority single-event candidates, such as explicit remember-directive metadata, a manual priority flag, or a critical-feedback classifier, with narrowly scoped bypass semantics for signal and diversity gates. Update memory-core tests and https://docs.openclaw.ai/concepts/dreaming once the behavior is defined.
