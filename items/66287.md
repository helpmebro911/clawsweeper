## Summary

Current main still has the reported memory-core gap: default deep dreaming promotes only candidates that pass score, signal-count, and diversity gates, and the regression tests explicitly keep a one-off high-scoring recall out of MEMORY.md. Manual CLI threshold overrides and pre-compaction flushes exist, but they are not the requested durable single-event priority path.

## Best Possible Solution

Keep this open for memory-core maintainer/product design. A good implementation would add a tested durable lane for high-priority single-event candidates, such as explicit remember-directive metadata, a manual priority flag, or a critical-feedback classifier, and let that lane bypass or separately satisfy the signal/diversity gates without weakening default promotion for ordinary singleton notes. Update memory-core tests and the public memory/dreaming docs once the chosen behavior is defined.
