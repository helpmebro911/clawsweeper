## Summary

Keep this PR open. Current main still blocks `qwen3.6-plus` on Qwen Coding Plan endpoints in the catalog builder, config normalization, runtime suppression, tests, and user docs. The open maintainer PR #72664 is related but narrower: it allows explicit configured use while intentionally keeping built-in catalog suppression unless maintainers choose the broader provider-truth policy separately.

## Best Possible Solution

Keep this PR open for maintainer product and security review. If maintainers want `qwen3.6-plus` advertised on Coding Plan endpoints, rebase this PR or land an equivalent maintained patch covering `extensions/qwen/models.ts`, `extensions/qwen/index.ts`, `extensions/qwen/provider-catalog.test.ts`, and `docs/providers/qwen.md`. If maintainers choose #72664's narrower explicit-config policy as canonical, close this PR only after that policy lands or is explicitly accepted as the replacement.
