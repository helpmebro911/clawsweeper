## Summary

Keep this issue open. Current main still maps the first `agent.wait` timeout in `sessions_send` directly to a hard `status: "timeout"`, and the same author has an open, unmerged implementation candidate (#68196) that is explicitly paired with this issue. Per cleanup rules, the issue should stay open until that PR is reviewed and merged, revised, or closed.

## Best Possible Solution

Keep this issue open and use #68196 as the active implementation path. The maintainer path is to review or revise that PR so `sessions_send` distinguishes soft wait-window expiry from real delivery uncertainty, returns a late reply when one appears after the baseline snapshot, reports accepted delivery when acceptance can be proven, and only returns a hard timeout when acceptance cannot be proven. Close this issue after the accepted fix lands on main.
