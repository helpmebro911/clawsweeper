## Summary

Current main still has the reported bug: exec approvals paths are resolved from the home-based `~/.openclaw` defaults, while the shared mutable-state resolver honors `OPENCLAW_STATE_DIR`. The linked PR #62938 is still an open implementation candidate for the same issue, so this issue should remain open until that fix or an equivalent replacement lands.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #62938. The fix should live in `src/infra/exec-approvals.ts`, make the exec approvals file and socket path resolve under `OPENCLAW_STATE_DIR` / `resolveStateDir()` before the existing `OPENCLAW_HOME` and home defaults, add focused regression coverage in `src/infra/exec-approvals-store.test.ts`, and preserve the symlink-safety behavior added around exec approvals paths.
