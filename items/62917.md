## Summary

Current main still has the reported behavior: exec approvals file and socket resolution remains hard-coded through the home-based `~/.openclaw` defaults and does not consult `OPENCLAW_STATE_DIR`, while the shared state resolver and doctor warning path do. Also, PR #62938 is still an open implementation candidate that uses closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or close PR #62938. The best fix belongs in `src/infra/exec-approvals.ts`: resolve the exec approvals file and socket under `OPENCLAW_STATE_DIR` / `resolveStateDir()` before the existing home default, add focused regression coverage in `src/infra/exec-approvals-store.test.ts`, preserve trusted-root symlink protections, and update user-facing approvals docs if the path wording changes.
