## Summary

Keep this issue open. Current main still resolves exec approvals file and socket paths through the home-based `~/.openclaw` defaults rather than `OPENCLAW_STATE_DIR`, and the provided GitHub context shows open PR #62938 remains an unmerged implementation candidate for this exact problem.

## Best Possible Solution

Keep this issue open and review, update, land, or replace PR #62938. The likely fix belongs in `src/infra/exec-approvals.ts`: resolve the approvals file and socket under the explicit `OPENCLAW_STATE_DIR`/`resolveStateDir()` contract first, then fall back to the existing `OPENCLAW_HOME` and default-home behavior, with regression tests in `src/infra/exec-approvals-store.test.ts` for state-dir precedence and the existing symlink-safety behavior.
