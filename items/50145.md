## Summary

Keep open. Current main still matches the reported root cause: interrupt mode can abort the active run, the aborted attempt can retain the last assistant from full session history, `run.ts` passes that assistant into payload construction unguarded, the payload builder falls back to it when no streamed assistant text exists, and `runReplyAgent` has no aborted-run early exit before final payload handling. The linked PR #50147 is an exact open/unmerged fix candidate, but it has not landed on main; #71190 is related queue-abort fallout, not a canonical replacement for this specific interrupt replay bug.
