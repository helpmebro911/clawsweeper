## Summary

Keep PR #70971 open. Current main partially mitigates the reported exec-only MCP leak by avoiding bundle MCP materialization for allowlists that cannot reference bundle MCP tools, but it still does not apply runtime `toolsAllow` to the final combined tool list. Bundled LSP tools are still materialized whenever tools are enabled and appended after the runtime allowlist pass, so the PR is not obsolete.

## Best Possible Solution

Keep the PR open, but ask for a rebase/rework against current main. The best fix is to make runtime `toolsAllow` authoritative over the final model tool list after bundled MCP/LSP materialization, while preserving intended plugin-group semantics such as `bundle-mcp`, and to add focused regression coverage for the final assembled tools.
