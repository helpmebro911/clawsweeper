## Summary

Keep this PR open. Current main still appends bundled MCP/LSP tools after the runtime toolsAllow pass, so the reported path remains real. The submitted one-line fix is plausible for exec-only cron jobs, but likely needs maintainer adjustment to preserve plugin-key semantics such as bundle-mcp plus focused regression coverage.

## Best Possible Solution

Keep this PR open or replace it with a maintainer patch in src/agents/pi-embedded-runner/run/attempt.ts that applies runtime toolsAllow after bundled MCP/LSP materialization using the same plugin-aware policy semantics as the final tool policy pipeline. Add regression coverage for toolsAllow: ["exec"], an exact bundled MCP tool name, toolsAllow: ["bundle-mcp"], and configured bundled LSP tools before landing.
