## Summary

Keep this PR open. Current main has a partial bundle-MCP creation gate, but it still does not make runtime `toolsAllow` authoritative over the final bundled MCP/LSP tool list, and the PR is a plausible implementation candidate for the remaining behavior.

## Best Possible Solution

Keep this PR open or supersede it with a maintainer patch that applies runtime `toolsAllow` after bundled MCP/LSP materialization, preserves intended `bundle-mcp` plugin/group semantics, supports exact MCP and LSP tool names, and adds focused regression coverage for `toolsAllow: ["exec"]`, one exact MCP tool, `toolsAllow: ["bundle-mcp"]`, and configured bundle LSP.
