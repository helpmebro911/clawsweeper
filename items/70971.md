## Summary

Keep #70971 open. Current main partially avoids the reported exec-only MCP leak by not creating bundled MCP runtimes unless toolsAllow contains a bundled MCP-style name, but it still appends bundled LSP tools to effectiveTools without reapplying toolsAllow. The PR's central final-allowlist hunk is not present on main, and existing tests cover only the helper/MCP-gating behavior, not the final bundled MCP/LSP effectiveTools contract.

## Best Possible Solution

Keep the PR open for maintainer review or a rebased equivalent. The best fix is to make toolsAllow authoritative over the final bundled-tool set on the scheduled embedded attempt path, including LSP, while preserving the current MCP optimization and adding a regression test for an exec-only allowlist with bundled MCP/LSP configured.
