## Summary

Keep PR #70971 open. Current main has a partial mitigation for the reported exec-only MCP leak, but it still does not apply runtime `toolsAllow` to the final combined tool list, so the PR is not obsolete. The branch likely needs rebasing and regression coverage rather than auto-close.

## Best Possible Solution

Keep the item open and rework it against current main: apply runtime `toolsAllow` after combining base and bundled tools, or equivalently filter `filteredBundledTools` by the same runtime allowlist, then add focused regression coverage for scheduled/embedded runs with bundled MCP and LSP tools. The existing MCP materialization skip can remain as an optimization, but it should not be the only enforcement mechanism.
