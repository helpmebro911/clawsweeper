## Summary

Keep open. Current main mitigates generic text-only tool-result delivery when Discord preview streaming owns progress, but Discord partial preview still enables tool-progress updates by default and routes tool/progress labels into the edited preview message. That does not satisfy the issue's requested boundary that tool activity/output should not be user-visible through Discord streaming by default.

## Best Possible Solution

Keep #70561 open for explicit maintainer/security product judgment. A likely fix is to make Discord partial preview distinguish assistant-visible tool events from user-visible assistant text by default, either disabling `streaming.preview.toolProgress` for Discord, replacing it with a strictly sanitized generic status, or requiring an explicit verbose/full opt-in; add Discord regression tests covering `onToolResult`, `onItemEvent.progressText`, `onApprovalEvent.command`, and `onCommandOutput.output` so raw or sensitive tool data cannot reach `draftStream.update` or normal delivery unexpectedly.
