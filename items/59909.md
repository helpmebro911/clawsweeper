## Summary

Current `main` does not implement configurable TUI footer display. The footer content is still hard-coded in the TUI renderer, the CLI exposes no footer option, and the docs still state that agent and session are always shown in the footer.

## Evidence

- **TUI footer is hard-coded:** `updateFooter()` always builds the footer from agent, session, model, thinking/fast/verbose/reasoning, and token data, then renders that joined string. There is no branching for hidden/compact/footer-off modes.
  - file: [src/tui/tui.ts:816](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/tui/tui.ts#L816)

  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **No CLI footer option exists:** The `openclaw tui` command defines `--local`, `--url`, `--token`, `--password`, `--session`, `--deliver`, `--thinking`, `--message`, `--timeout-ms`, and `--history-limit`, but nothing for footer display.
