## Summary

Keep open. Current main has implemented the Discord file/path upload route, but the direct `buffer`/`filename`/`contentType`/`mimeType` path advertised by ordinary `message.send` is still neither delivered nor rejected. Related open work (#41779, #44555, #57335, #63578) points to the same shared send-schema/runtime mismatch rather than proving this is fixed.

## Best Possible Solution

Keep this issue open and scope the fix to direct buffer attachments on ordinary Discord `message.send`. The best path is to review or refresh the shared buffer-send work in #57335, coordinate with #41779 and #44555, and either deliver `buffer` plus `filename` plus `contentType`/`mimeType` end to end into Discord multipart `files` with regression coverage, or remove/fail-fast those fields for ordinary `send` if buffers are intentionally unsupported. #63578 is related for plugin send-media hydration but does not by itself prove this Discord buffer case is fixed on main.
