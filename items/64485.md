## Summary

Keep open. Current `main` still requires `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` to be set in `docker-compose.yml`, and a direct `docker compose config` reproduction fails when they are unset. The repo's manual Docker docs still tell users to run `docker compose` directly without first exporting those variables, so the report is still live rather than implemented or obsolete.

## Evidence

- **Current compose file still has no defaults for the bind mounts:** Both services in `docker-compose.yml` still mount `${OPENCLAW_CONFIG_DIR}` and `${OPENCLAW_WORKSPACE_DIR}` directly, with no fallback/default expression.
  - file: [docker-compose.yml:14](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/docker-compose.yml#L14)
  - command: `nl -ba docker-compose.yml | sed -n '1,140p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Reasonable reproduction still fails on current main:** From the repo root, `env -u OPENCLAW_CONFIG_DIR -u OPENCLAW_WORKSPACE_DIR docker compose config` warned that both variables were unset and failed with `invalid spec: :/home/node/.openclaw: empty section between colons`.
