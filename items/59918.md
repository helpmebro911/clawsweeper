## Summary

Keep open. Current `main` still has the JSONL parser gap described in #59918: non-Claude JSONL output is reduced to generic `item.text` collection, without preferring `task_complete.last_agent_message` or `phase: "final_answer"`, and that parsed text is still used as the returned fallback reply.

## Evidence

- **Generic JSONL parser still ignores terminal fields and assistant phase:** `parseCliJsonl()` special-cases Claude `type: "result"`, but the generic path only collects `parsed.item.text`, joins it, and returns it. It does not inspect `task_complete.last_agent_message`, `payload.phase`, or `final_answer` markers.
  - file: [src/agents/cli-output.ts:440](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/cli-output.ts#L440)
  - command: `nl -ba src/agents/cli-output.ts | sed -n '420,560p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Returned parser text is still used for CLI fallback output:** `executePreparedCliRun()` calls `parseCliOutput()` on CLI stdout and then returns `parsed.text` as the final CLI output after text transforms, so this parser behavior still affects user-visible fallback replies.
  - file: [src/agents/cli-runner/execute.ts:522](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/cli-runner/execute.ts#L522)
  - command: `nl -ba src/agents/cli-runner/execute.ts | sed -n '500,560p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Docs still promise final-message extraction for JSONL backends:** The CLI backend docs say `output: "jsonl"` extracts the final agent message for backends like Codex CLI, which does not match the current generic parser implementation above.
  - file: [docs/gateway/cli-backends.md:246](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/docs/gateway/cli-backends.md#L246)
  - command: `nl -ba docs/gateway/cli-backends.md | sed -n '240,270p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Current parser tests only cover Claude stream-json cases:** `parseCliJsonl` tests exercise Claude `result`/stream-json behavior and related error handling, but there is no regression coverage here for OpenAI/Codex-style phased JSONL with `task_complete.last_agent_message` or `phase: "final_answer"`.
  - file: [src/agents/cli-output.test.ts:162](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/cli-output.test.ts#L162)

- **Issue discussion already included a narrowly scoped proposed fix:** The reporter linked PR #59920 in the issue comments with the same intended behavior: prefer `task_complete.last_agent_message`, then `phase: "final_answer"`, and skip commentary in the generic fallback path. That aligns with the remaining gap in current `main`.
