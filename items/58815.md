## Summary

Keep open. Current `main` still has the reported Android reconnect window in `GatewaySession`: `connect()` only relaunches the loop when `job == null`, while `disconnect()` clears `job` asynchronously later. The PR's change is not on `main`, and the review discussion identified a separate race in the proposed patch, so there is not strong evidence to close this as implemented or obsolete.

## Evidence

- **Current main still gates reconnect only on null job:** `GatewaySession.connect()` still starts `runLoop()` only when `job == null`; it does not relaunch when the cached job exists but is already inactive, which is the exact condition described in the PR.
