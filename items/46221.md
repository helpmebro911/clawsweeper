## Summary

Keep #46221 open. Current main still enqueues manual wake/system events onto the canonical main session while `drainFormattedSystemEvents()` drains only the active session key, so the per-channel-peer gap described by #42919 is not implemented on main. The PR is relevant but should be rebased/reworked before merge because its proposed fallback drains `resolveMainSessionKey(cfg)`, and the review discussion raised a real multi-agent/default-agent boundary concern.

## Best Possible Solution

Keep the PR open and ask for a rebased patch that implements an agent-scoped main-key fallback in `src/auto-reply/reply/session-system-events.ts`, with regression coverage for per-channel-peer delivery and a non-default-agent case. Maintainers should also explicitly confirm whether all non-main sessions, not only isolated DMs, should be allowed to consume main-key system events.
