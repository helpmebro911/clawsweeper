## Summary

Keep #19859 open. Current main still hard-codes browser WebChat Enter-to-send behavior, exposes no persisted UI or gateway config setting for send shortcut behavior, and the native macOS WebChat composer is also fixed to Return sends. The related #8147 overlaps, but it remains open and the latest maintainer-side comment explicitly kept #19859 open unless maintainers consolidate it.

## Best Possible Solution

Keep #19859 open unless maintainers explicitly consolidate it into #8147. A complete fix should add a typed chat send-shortcut preference, wire it through the browser Control UI/WebChat composer and native Swift WebChat composer, support Ctrl/Cmd+Enter to send when plain Enter inserts a newline, and update settings UI, config schema/types/help, public WebChat docs, and focused composer tests.
