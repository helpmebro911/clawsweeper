## Summary

Keep PR #39118 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still lacks the PR's central doctor safety behavior: dependency-only companion units can still be reported as gateway-like, extra gateway-like services are not active-state filtered, and ExecStart repair has no running-gateway guard. The PR discussion also contains concrete review issues, so the best outcome is revision rather than cleanup-close.

## Best Possible Solution

Keep #39118 open and request a revised implementation. The fix should distinguish dependency-only companion references from actual gateway service declarations without hiding custom-named gateways, make active-service filtering scope-aware for user versus system units, apply the running-gateway ExecStart guard after all ExecStart-level audit issues are known, and add regression tests for companion dependencies, custom launchers, inactive extras, active extras, legacy cleanup, and system-scope deep scans.
