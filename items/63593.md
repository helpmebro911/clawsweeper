## Summary

Keep this PR open. Current main still treats an unresolved explicit `--tag latest` lookup as downgrade risk because `targetVersion == null` remains enough to trigger the non-interactive downgrade stop for registry-resolvable tags. The PR is a narrow implementation candidate, includes regression coverage for unresolved `latest`, and preserves the unresolved non-latest guard after the review comment was addressed.

## Best Possible Solution

Keep this PR open, refresh it against current main, and review or land the narrow CLI change in `src/cli/update-cli/update-command.ts` with the regression tests in `src/cli/update-cli.test.ts`. The intended shape is to exempt unresolved explicit `latest` lookups from downgrade confirmation while preserving the conservative stop for unresolved non-latest registry tags and real resolved downgrades.
