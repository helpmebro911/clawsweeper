## Summary

Current main implements the reported MCP stdio restart leak path: gateway restart closes the old server, gateway shutdown now disposes all cached/in-flight session MCP runtimes, and stdio MCP transports reap the process tree when graceful close does not exit. The report came from OpenClaw 2026.4.22; I did not find a tagged release containing the current gateway-shutdown disposal, so this is implemented on main rather than a known shipped release.

## Best Possible Solution

Close this issue as implemented on main. Keep the current shutdown/runtime/stdio process-tree cleanup; if maintainers still want startup scavenging for pre-upgrade orphan processes or a per-server idleTimeoutMs shape instead of the current global mcp.sessionIdleTtlMs, track that as a narrower enhancement rather than keeping this restart-leak bug open.
