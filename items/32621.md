## Summary

Keep open. Current `main` still does not implement `message_sending` for all explicit `message(action=send)` sends. The hook runs in `deliverOutboundPayloads`, but `executeSendAction` first delegates to channel plugin `handleAction("send")` and returns early when that path handles the send. Bundled channels such as Slack and Discord still implement explicit send as plugin-owned actions that go straight to their runtime send handlers.

## Evidence

- **Explicit message-tool send returns before core delivery hook path when a channel plugin handles send:** `executeSendAction` calls `tryHandleWithPluginAction({ action: "send" })` and immediately returns `pluginHandled` if present, so those sends never reach the core `sendMessage`/`deliverOutboundPayloads` path.
  - file: [src/infra/outbound/outbound-send-service.ts:143](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/infra/outbound/outbound-send-service.ts#L143)
  - command: `nl -ba src/infra/outbound/outbound-send-service.ts | sed -n '120,180p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **`message_sending` is wired only inside outbound payload delivery:** `deliverOutboundPayloads` computes `hasMessageSendingHooks` and runs `applyMessageSendingHook(...)` before sending. This is the hook-bearing path, not the plugin action fast path.
  - file: [src/infra/outbound/deliver.ts:834](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/infra/outbound/deliver.ts#L834)

- **Core send path would use the hook-bearing delivery pipeline, but only after plugin handling falls through:** `sendMessage` calls `deliverOutboundPayloads(...)` for direct/core sends, confirming the hook exists there; however that path is reached only after the earlier plugin-action shortcut does not handle the send.
  - file: [src/infra/outbound/message.ts:287](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/infra/outbound/message.ts#L287)

  - file: [extensions/slack/src/channel-actions.ts:25](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/slack/src/channel-actions.ts#L25)

## Risks / Open Questions

- Some channels may already fall through to the core `deliverOutboundPayloads` path, so the bug is partial by channel/runtime. The request is still not implemented end-to-end for all explicit sends, which is what the issue asks for.
- `gh` inspection was unavailable in this environment because `GH_TOKEN` was not set, and the local clone appears shallow for some referenced commits. That limits PR-history confirmation, but the current `main` source is sufficient to show the issue is not fully fixed.
