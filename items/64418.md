## Summary

Current `main` still reprocesses historical image blocks on replay: `replay-history` sanitizes the full transcript every turn, and `tool-images` does not memoize or persist resized results back to stored session history. The latest release tag shows the same code path, and the issue timeline already points at an open follow-up PR, so there is not strong evidence to close this item conservatively.
