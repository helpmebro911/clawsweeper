## Summary

Keep #47677 open. Current main receives Telegram reaction updates and stores them as system-context events, but it does not implement the central requested behavior: reaction-only inputs do not wake the agent, custom emoji reactions are not normalized as inbound signals, and there is no per-emoji reaction semantics config. The related #64752 bug remains open and #47687 is an unmerged implementation proposal, so the work is still pending.

## Best Possible Solution

Keep the issue open and either review/refresh PR #47687 or implement the same direction in the Telegram plugin: documented reaction semantics config, normalized standard/custom emoji keys, configured wake behavior for reaction-only updates, observable accept/reject/executed outcomes, and regression tests proving reaction-only inputs can start an agent turn. Keep #64752 linked as the narrower wake-trigger repro.
