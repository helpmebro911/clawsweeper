## Summary

Keep this PR open. Current main still does not implement the proposed local `tsgo` host-pressure refusal, force override, or redirected temp environment, and the open PR is a focused change to the active `tsgo` wrapper path with relevant tests. The review comments identify issues that should be fixed before landing, especially checking the filesystem backing the effective temp directory instead of hard-coding `/tmp`.

## Best Possible Solution

Keep this PR open and revise it. The best landing path is to add the host-pressure refusal and explicit force override in `scripts/lib/local-heavy-check-runtime.mjs`, route real local tsgo temp files through an artifact temp directory only when a heavy tsgo run will proceed, check the filesystem backing the effective temp directory, preserve the sparse guard and heavy-check lock behavior, and keep focused tests for pressure refusal, force bypass, and temp-dir routing.
