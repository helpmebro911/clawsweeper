## Summary

Keep open. Current main still has the reported cross-agent requester-origin behavior: `resolveRequesterOriginForChild()` resolves `accountId` from the target/child agent when `targetAgentId !== requesterAgentId`, and that origin is stored as the subagent run requester origin used by completion delivery. The open PR #70607 is a relevant fix attempt, but it is not merged, and current main plus the latest release tag still show the problematic branch.

## Best Possible Solution

Keep #70574 open until a core agents fix lands. The best path is to preserve the parent/requester `accountId` in the completion `requesterOrigin`, while still allowing a separate target-agent-bound account/origin for child session startup and thread-binding hooks where that behavior is intentional. Add regression tests around `resolveRequesterOriginForChild()` or the spawn registration path, plus an integration-style completion delivery test proving cross-agent Telegram DM completion returns through the parent requester session and nested `streamTo="parent"` still routes to the immediate parent.
