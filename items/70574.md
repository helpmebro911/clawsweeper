## Summary

Keep open. Current main still contains the exact cross-agent requester-origin override described in #70574, and the linked fix PR #70607 is open rather than shipped. This is an active product bug, not a stale/duplicate cleanup candidate.

## Best Possible Solution

Keep #70574 open and review/land PR #70607 or an equivalent patch that preserves the parent accountId in the completion requesterOrigin while retaining explicit target-agent bound account behavior only for child thread/bootstrap delivery. Add regression coverage for cross-agent Telegram/no-thread completion routing plus same-agent and nested `streamTo="parent"` cases.

## Risks / Open Questions

- Fixing this by simply removing target-agent account resolution could regress existing thread-bound interactive child delivery paths that currently rely on the target agent's bound account.
- The likely implementation needs separate delivery contexts: parent requesterOrigin for completion/task ownership, and target-agent bound origin for child bootstrap/thread binding where that behavior is intentional.
