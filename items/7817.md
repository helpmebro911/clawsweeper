## Summary

Keep open. Current main still has flat/agent-grouped session selection, a Sessions table, parent/child session metadata, and checkpoint branching, but it does not implement the requested Control UI project tree, threaded-entry routing dropdown/fanout, arbitrary message split/triage, or seed-to-branch workflow. The existing related #55349 is narrower quote-reply/threaded-reply work, not a canonical replacement for this broader feature.

## Best Possible Solution

Keep #7817 open as the broad Control UI product feature. A complete solution likely needs explicit project metadata in the session contract, a project/tree session navigator, composer-level target/thread routing semantics, and transcript-level split/move/seed operations with matching Gateway protocol, docs, and UI tests. Keep #55349 as a narrower quote-reply/threaded-reply follow-up rather than treating it as a replacement.
