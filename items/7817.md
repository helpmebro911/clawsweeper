## Summary

Keep #7817 open. Current main has useful session primitives, including parent/child session metadata, child-session listing, checkpoint branching, grouped session selection, and display-only reply pills, but it still does not implement the requested Control UI project tree navigator, composer-level routing/fanout, arbitrary selected-message split/move triage, or seed-elsewhere workflow. The related #55349 item is narrower quote-reply UX and does not supersede this broader project/thread organization request.

## Best Possible Solution

Keep #7817 open as the broad Control UI product feature. A complete implementation should add additive Gateway/session support for project and session-tree metadata, composer route targets or fanout, selected-message split/move workflows, and seed/copy branch behavior, then wire those into a Control UI project tree navigator with focused protocol/UI tests and docs. Reuse parentSessionKey, childSessions, checkpoint branching, and reply-pill primitives where they fit; keep #55349 separate for quote-reply/threaded-reply UX.
