## Summary

Keep open. Current main still rejects `auth-profiles.json` entries whose credential `type` is `aws-sdk` as `invalid_type`; the related PR #69753 proposed the targeted skip/marker behavior but is closed unmerged and its implementation is not present on main. There is separate current coverage for provider-level `auth: "aws-sdk"` fallback/IMDS behavior, so the final fix should be narrow: stop treating SDK-managed marker profiles as invalid or stop writing them, with regression coverage for the store-load warning path.

## Best Possible Solution

Keep the issue open and land a narrow auth-profile store fix: either silently ignore SDK-managed `aws-sdk` marker entries during load, as PR #69753 proposed, or stop generating/persisting those markers. Add regression tests that an `aws-sdk` marker does not count as an invalid dropped entry while real invalid types still warn, and keep the existing Bedrock provider-level AWS SDK/IMDS fallback tests intact.
