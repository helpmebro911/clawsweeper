## Summary

Keep open. Current main has several cancellation primitives, but it does not yet implement the issue's central request: unified per-session run generation invalidation with stale-output fencing, pre-tool gates, and immediate new-message takeover. The related PR #70363 is useful and directly relevant, but it is open/draft and explicitly partial, with pre-tool gating, new-message takeover, and several emission sites deferred.

## Best Possible Solution

Keep this issue open as the canonical umbrella until main has a cohesive run invalidation contract. The best implementation path is to land a focused core change that adds a per-session run-generation primitive, captures it on ReplyOperation, bumps it on /stop and new-message takeover, cancels owned embedded/process-supervisor work, gates tool dispatch before each tool, fences visible emission sites including typing/block/non-block/followup delivery, and adds regression tests for skipped tools, stale output suppression, and new-message supersession.
