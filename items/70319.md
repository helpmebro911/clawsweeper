## Summary

Keep open. Current main still has useful abort primitives, but it does not implement the issue's central unified run-generation invalidation or stale-output fencing guarantee. The linked PR #70363 is open, draft, unmerged, and explicitly partial, so it is not a shipped fix or a full superseding tracker.

## Best Possible Solution

Keep #70319 open as the umbrella for stronger run interruption semantics. Maintainers should review or reshape #70363 into focused PRs that add a per-session generation/current-run contract, wire it into abort and new-message takeover, gate pre-tool execution, and fence all visible emission paths with regression tests.
