## Summary

Keep open. Current main has several abort primitives and an interrupt queue mode, but it does not yet provide the requested unified per-session generation invalidation or broad stale-output fencing. The linked draft PR #70363 is explicitly partial and open, so it does not supersede the issue.

## Best Possible Solution

Keep #70319 open as the umbrella until maintainers decide the product semantics and land a complete or intentionally scoped implementation. The likely path is to review/split #70363, add a session run-generation/current-run primitive, bump it from /stop and new-message takeover paths, propagate current-run checks through reply delivery, typing/progress/tool-result/followup emissions, add a pre-tool gate in the embedded runner, wire session-owned subprocess cancellation, and cover those paths with regression tests.
