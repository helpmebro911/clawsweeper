## Summary

Keep open. Current main still has abort, queue, and process-cancel primitives, but it does not implement the unified per-session run-generation invalidation, broad stale-output fencing, pre-tool skip gate, or immediate new-message takeover requested here. The same-author draft PR #70363 remains open and explicitly partial, so it is an implementation candidate rather than a reason to close this issue.

## Best Possible Solution

Keep this issue open as the canonical core interruptibility tracker. Review or split #70363, then land focused core changes for a session/run generation primitive, stale-output fences across reply/typing/followup/tool-progress/event emission paths, a pre-tool abort/currentness gate in the Pi tool path, new-message takeover that invalidates and cancels the active run, and targeted regression tests proving stale runs cannot emit visible effects or launch further tools.
