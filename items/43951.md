## Summary

Keep this PR open. Current main still drops `accountId` before `/hooks/agent` dispatch, while the cron delivery path already has the lower-level `accountId` support this PR is trying to thread through. The PR is paired with open #43866 from the same author and remains a plausible implementation candidate, but it needs maintainer review and rebase/adaptation against the current cron message-tool policy.

## Best Possible Solution

Keep the PR open. Best path is to rebase it onto current main, preserve the focused hook-side fix (`accountId` normalization, propagation into `CronDelivery`, observable normalization failure, replay-key scoping, and regression tests), adapt or remove stale cron tool-policy changes against the current `deliveryMode`/`forceMessageTool` implementation, update the public hook docs, and fix changelog attribution before maintainer review.
