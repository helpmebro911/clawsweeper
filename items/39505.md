## Summary

Current main still reproduces the reported formatter bug for markdown/backtick-wrapped Home Assistant-style entity IDs. The TUI sanitizer strips edge punctuation into a candidate but still checks the raw token for the file-like underscore preservation branch, so wrapped dotted IDs are classified as non-copy-sensitive and split with literal spaces. PR #39565 remains open and unmerged as the focused fix; PR #39553 was closed unmerged.

## Best Possible Solution

Keep #39505 open until PR #39565 or an equivalent focused fix lands. The fix should classify the punctuation-stripped candidate for copy-sensitive URL/path/file-like checks, preserve backtick/quote/markdown-wrapped dotted entity IDs exactly, add regression tests for bare and wrapped Home Assistant-style IDs, and retain guardrails that still chunk generic long prose or punctuation-only tokens for narrow-terminal safety.
