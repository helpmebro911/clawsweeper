## Summary

Current main still matches the reported Windows Gmail setup crash path. Gmail setup accepts `gcloud` through PATH/PATHEXT-style dependency detection, but the actual setup commands still spawn a bare `gcloud` argv, while the shared Windows command shim only handles npm/npx/corepack/pnpm/yarn-style commands. There is no targeted Gmail Windows `.cmd` regression coverage in current tests.

## Best Possible Solution

Keep this issue open and fix Gmail webhook setup's Windows command resolution. The maintainer-safe path is to route `gcloud`, `gog`, and `tailscale` through a Windows-aware executable resolver, likely the existing `src/infra/executable-path.ts` helper or a narrow Gmail setup wrapper, then add win32 regression tests proving `gcloud.cmd` in PATH is the command executed without enabling shell parsing.
