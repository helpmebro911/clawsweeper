## Summary

Keep open. Current main has related OpenAI Codex browser OAuth and device-code login support, but it does not implement the requested non-interactive Codex OAuth setup via environment-supplied callback/redirect URL. The direct `models auth login` path still requires an interactive TTY before provider auth runs, and the OpenAI plugin only advertises `OPENAI_API_KEY` as an env-var auth path for the direct OpenAI provider, not for `openai-codex`.
