## Summary

Keep open. Current main still lacks first-class Telegram custom emoji support for normal bot markdown/model output: `<tg-emoji>` is escaped in markdown mode, there is no markdown-level custom emoji syntax, and captions/replies flow through the same markdown-to-HTML renderer. Trusted `textMode: "html"` can pass raw markup, but that does not satisfy the issue's requested default messages, captions, and replies behavior.
