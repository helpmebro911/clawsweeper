## Summary

Keep #48034 open. Current main still does not make managed/bundled skill files accessible for the common sandbox `workspaceAccess: "rw"` path, but this PR is not mergeable as-is: the provided review comments identify a broken import and the proposed sync block remains unreachable for typical rw agent-scope workspaces. The unrelated locale hunk is already present on main.

## Best Possible Solution

Keep the item open or replace it with a focused maintainer PR that makes managed/bundled skill prompt locations and sandbox fs visibility agree in `workspaceAccess: "rw"` mode, adds a regression test for rw agent-scope skill reads, and drops the already-shipped locale hunk. The fix should avoid importing `resolveBundledSkillsDir` from `src/utils.ts` and should account for the active rw workspace/mount path rather than only removing the inner guard.
