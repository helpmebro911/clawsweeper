## Summary

Keep open. Current main still gates Anthropic beta-header wrapping on non-empty resolved extra-param betas, so the OAuth beta injection path is not guaranteed to run when Anthropic beta headers come only from model/provider header configuration. The proposed change is still directionally relevant, but it needs to be rebased onto the current Anthropic plugin implementation rather than the older core runner path.

## Best Possible Solution

Keep this PR open, rebase or rewrite it against `extensions/anthropic/stream-wrappers.ts`, change `wrapAnthropicProviderStream` to apply `createAnthropicBetaHeadersWrapper` for Anthropic even when `resolveAnthropicBetas` returns no explicit betas, and add focused coverage for OAuth tokens with model/provider-level `anthropic-beta` headers but no extra-param beta.
