## Summary

Keep #70990 open. Current main does not implement the proposed `model_failover` or `model_failure_terminal` plugin hooks, and this is a plausible core plugin API extension rather than ClawHub-only work because external plugins cannot observe these failover seams without the missing hook surface. The linked #70976 is closed, but the observable capability is still absent from main; #65824 still contains the broader user-facing gap.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to add a narrow, documented core plugin hook API for failover decisions and terminal all-models-failed outcomes, with tests at the hook runner and failover emission seams, then leave broader user-visible fallback notices, prompt re-injection, and circuit-breaker behavior to separate issues such as #65824.
