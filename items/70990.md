## Summary

Keep #70990 open. Current main does not implement the requested `model_failover` or `model_failure_terminal` plugin hooks, and the PR is a plausible core plugin-contract change rather than ClawHub work because external plugins cannot observe these failover decisions without a core hook surface. The linked issue #70976 is closed, but the observable implementation is not present on current main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to either land a tightened version of #70990 with public hook docs/API coverage and changed-lane validation, or have a maintainer explicitly redirect the plugin failover observability design to a different canonical core issue/PR. This should not be closed as ClawHub work because the missing piece is the core hook emission contract that external plugins would consume.
