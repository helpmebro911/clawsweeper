## Summary

Keep open. Current main still lacks the requested `model_failover` and `model_failure_terminal` plugin hooks, while the existing code has internal failover and terminal-failure seams that are only exposed through logs/user-facing fallback handling. This PR remains a plausible narrow core plugin API change rather than ClawHub-only work, because external plugins cannot observe these events until core exposes the missing hook surface.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide the exact `model_failover` and `model_failure_terminal` event contract, address the remaining review comments, add or keep focused hook-runner and failover-emission tests, document the new hooks, and then land it if maintainers accept this as the narrow core plugin API surface for failover observability. Broader fallback notices, workspace-context re-injection, and circuit-breaker policy should stay under #65824 or separate follow-ups.
