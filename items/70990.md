## Summary

Keep this PR open. Current main still lacks the proposed `model_failover` and `model_failure_terminal` plugin hooks, and the work is a plausible narrow core plugin API extension because external plugins cannot observe these failover seams without core hook names, handler types, runner methods, and emission points.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to finalize the `model_failover` and `model_failure_terminal` contracts, decide whether terminal failure payloads need conversation-access gating or narrower fields, add public hook documentation, retain the focused hook-runner and failover-emission tests, and then land if maintainers accept this narrow core plugin API surface. Broader fallback notices, workspace-context reinjection, and circuit-breaker policy should remain under #65824 or separate follow-ups.
