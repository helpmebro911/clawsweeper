## Summary

Keep #70990 open. Current main does not contain the proposed `model_failover` or `model_failure_terminal` hook names, event types, runner methods, docs, or tests. Existing `model_call_started` / `model_call_ended` hooks provide provider-call telemetry, but they do not expose failover decisions or terminal all-models-failed summaries, which is the central API gap described by #70976 and this PR. This is a recent, focused core plugin API PR with relevant tests in the provided diff, not a ClawHub-only plugin implementation or an obsolete duplicate.

## Best Possible Solution

Keep the PR open for maintainer review or revision. The best path is a core plugin hook/API change, not ClawHub-only work, because external plugins cannot observe structured failover decisions or terminal all-models-failed summaries without a core hook seam. Before landing, align the hook payloads with existing `model_call_*` telemetry conventions, update public plugin docs/SDK surfaces, and ensure the targeted plugin/failover tests pass.
