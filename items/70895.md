## Summary

Keep #70895 open. Current main has not adopted the PR's conditional session-reuse behavior: webhook agent hooks still dispatch isolated cron-style jobs, `prepareCronRunContext` still passes `forceNew: true` for every isolated job, and `resolveCronSession` therefore bypasses stable `sessionKey` reuse. The PR is focused, includes a regression test in the provided diff, and directly addresses linked issue #70894; it should be reviewed/landed or replaced by an equivalent fix rather than closed.

## Best Possible Solution

Keep #70895 open and review/land this narrow fix or an equivalent implementation: compute `forceNewSession` so ordinary isolated cron runs still force a fresh session, but hook/webhook-sourced isolated runs with `externalContentSource` or `hook:*` session keys pass `forceNew: false`; keep/add the regression test in `src/cron/isolated-agent/run.skill-filter.test.ts`, and consider a docs note clarifying `/hooks/agent` sessionKey reuse.
