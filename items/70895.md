## Summary

Keep #70895 open. Current main still has the reported bug: hook-dispatched agent runs are converted into `sessionTarget: "isolated"` cron jobs, and the isolated runner still passes `forceNew: input.job.sessionTarget === "isolated"` to `resolveCronSession`, which bypasses the resolver's existing session reuse path. The PR is narrow, linked to open bug #70894, and its proposed regression test targets the missing behavior without removing the existing fresh-session expectation for normal isolated cron runs.

## Best Possible Solution

Keep the PR open and land a narrow version of its approach: compute a hook-specific `forceNewSession` in `src/cron/isolated-agent/run.ts`, pass `forceNew: false` only for hook/webhook-sourced isolated runs, retain the existing normal isolated cron `forceNew: true` test, add the webhook regression test, and consider a concise docs update for `/hooks/agent` `sessionKey` reuse.
