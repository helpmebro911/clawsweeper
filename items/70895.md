## Summary

Keep #70895 open. Current `main` still has the exact webhook-isolated-session rollover path described by the PR: `/hooks/agent` dispatches an isolated cron agent turn with a stable hook session key and `externalContentSource: "webhook"`, but `prepareCronRunContext` still passes `forceNew: input.job.sessionTarget === "isolated"` to `resolveCronSession`, which bypasses the existing fresh-session reuse branch. The PR is focused, recently opened, linked to the concrete open bug #70894, and adds the missing regression coverage, so it is still useful maintainer work rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and either merge the focused fix after maintainer review or apply the same product behavior on `main`: preserve session reuse for webhook/hook-sourced isolated agent runs by passing `forceNew: false` when the run carries hook provenance, while keeping ordinary scheduled isolated cron jobs on fresh sessions. The regression test should assert both sides of that boundary.
