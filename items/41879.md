## Summary

Keep open. Current main still contains the reported default-agent-only fallback, and the regression test still asserts that `agents.defaults.heartbeat` with multiple agents enables only the default agent. Several related fix PRs were opened but closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open as the canonical tracker. The likely fix belongs in the shared heartbeat scope logic: update `isHeartbeatEnabledForAgent()` and `resolveHeartbeatAgents()` together so `agents.defaults.heartbeat` semantics match the documented global/default behavior, add regression coverage for defaults-only multi-agent configs and explicit per-agent scoping, and explicitly decide whether zero-config default heartbeat remains intended or should be disabled.

## Risks / Open Questions

- The no-configuration part of the issue conflicts with current docs that describe heartbeats as enabled by default at a default cadence, so maintainers should clarify intended product behavior before changing that part mechanically.
