## Summary

Current main still implements tool interception as typed plugin hooks, not as managed/internal HOOK.md events. The internal hook type/dispatcher/loader remain unable to carry a pre-tool blocking or parameter-rewrite result, and the tool execution paths call the global plugin hook runner directly. The issue should remain open as the canonical product/API request unless maintainers decide that tool-call interception should stay plugin-only.

## Best Possible Solution

Keep #13364 open as the canonical request for maintainer product/API judgment. A complete implementation would add typed internal tool_call before/after events, define how managed handler return values block or rewrite params, bridge tool execution without double-firing plugin hooks, and update docs plus focused internal HOOK.md tests. If maintainers choose plugin-only policy instead, close later with an explicit product decision and point users to plugin hooks/ClawHub.
