## Summary

Current `main` still appears to have the reported Google Chat threading bug. The Google Chat outbound adapter resolves `thread` as `threadId ?? replyToId` for both `sendText` and `sendMedia`, while the only `replyToMode` handling in this plugin is a config resolver; I did not find a suppression path or regression test proving `replyToMode: "off"` is honored for these sends.

## Evidence
