## Summary

This is a real, recent regression report with an open linked fix PR, and current main does not yet contain the proposed protections for unstable channel retry idempotency keys or user transcript idempotency.

## Best Possible Solution

Keep #72176 open as the canonical regression until PR #72290 or an equivalent narrow fix lands. The fix should add stable duplicate suppression for retry paths with changing idempotency keys, persist user transcript idempotency, and add regression coverage for duplicate user transcript delivery and duplicated assistant replies.
