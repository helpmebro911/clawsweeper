## Summary

Keep this issue open. Current main still lacks the requested `plugins.entries.amazon-bedrock-mantle.config.discovery.enabled` contract: the Mantle manifest rejects that key, registration does not read plugin config, and the resolver still attempts IAM bearer-token generation when no explicit token is present. The provided context also shows open PR #67341 as an implementation candidate with closing syntax, so the maintainer path is to review and land or close that PR before resolving this issue.

## Best Possible Solution

Review PR #67341 against current main or replace it with an equivalent focused fix in `extensions/amazon-bedrock-mantle`: add `discovery.enabled` to the manifest schema and UI hints, read startup/runtime plugin config in `register.sync.runtime.ts`, pass it into `resolveImplicitMantleProvider`, short-circuit before IAM token generation and `/v1/models` when disabled, update `docs/providers/bedrock-mantle.md`, and add regression coverage proving disabled discovery makes no IAM or fetch call while enabled/default discovery remains unchanged.
