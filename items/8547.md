## Summary

Current main still does not implement user-configurable Web UI or WebChat font size/font scale. The prior issue discussion explicitly kept #8547 open after comparing it with closed #3027, and current source/docs/tests still show only theme, custom theme, roundness, assistant identity, and seam color controls.

## Best Possible Solution

Keep #8547 open for a core Control UI accessibility/customization change. A practical path is to add a bounded text-size or font-scale field to browser-local `UiSettings`, expose it in Appearance and Quick Settings, apply it via CSS custom properties across chat messages, sidebars, tool cards, and compose inputs, and extend gateway `ui` schema/types/docs only if maintainers want server-side font sizing as a public config contract.
