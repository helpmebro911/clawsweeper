## Summary

Keep #8547 open. Current main still has no user-configurable Control UI/WebChat font size or font scale in browser-local UI settings, Appearance/Quick Settings controls, gateway `ui` config schema, docs, or CSS variables. The closed related issue #3027 does not supersede the remaining work, and the latest issue discussion explicitly kept #8547 open for a core UI accessibility/customization change.

## Best Possible Solution

Implement a bounded Control UI text-size or font-scale setting in `UiSettings`, expose it in Appearance and Quick Settings, apply it through CSS custom properties across chat text, compose input, sidebars, tool cards, and compact panels, and add focused UI/storage tests plus docs. Extend the gateway `ui` schema/types/docs only if maintainers want server-side font sizing as a public config contract.
