## Summary

Keep open. Current main still does not implement user-configurable Web UI font size or font scale: browser-local UI settings, the Appearance panel, gateway `ui` config schema/types, docs, and chat CSS all remain limited to theme/identity/roundness plus fixed font sizes. The related #3027 is closed and does not provide an open canonical tracker, while the existing maintainer/Codex comment on #8547 already reached the same keep-open conclusion.

## Best Possible Solution

Keep #8547 open for maintainer product/design decision and implementation. A concrete fix likely belongs in the core Control UI settings model and Appearance panel, backed by CSS custom properties for chat/body/input text, schema/type/docs updates if a gateway-level config knob is desired, and focused UI/storage/config tests. #3027 can remain historical context but should not be treated as the canonical open tracker because it is closed.
