## Summary

Keep open. Current main improves the situation with compact skills formatting, documented budget knobs, and per-agent allowlists, but #50677’s central complaint is still not fully resolved: skills can still be truncated out of the model-facing catalog, the runtime warning is still embedded in the prompt rather than surfaced as an operator-visible warning, omitted skill names are not listed, and there is no skill priority/pinning config. Ryce’s comment frames this as a state-visibility problem; current main still leaves that visibility gap. Related PR #42637 remains open and only covers omitted-name reporting, so it is not a complete canonical resolution for this broader issue.
