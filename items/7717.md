## Summary

Keep #7717 open. Current main still lacks a Discord guild/channel role-mention trigger surface: role IDs are only modeled as sender allowlists or generic mention metadata, and the mention preflight path never compares mentioned role IDs against configured trigger roles. The existing maintainer/Codex discussion also explicitly kept this issue open, and the related #20617 is a closed mention-format request rather than a canonical replacement for role-trigger behavior.

## Best Possible Solution

Keep #7717 open and implement this in the bundled Discord plugin as an additive guild/channel config surface such as triggerOnRoles?: string[]. Wire it through config types, strict Zod schema, generated metadata/help, docs, and preflight. Preflight should compare message.mentionedRoles / mention_roles against configured role IDs and treat a match as a wake mention, with focused schema/metadata tests plus preflight tests for matched and unmatched role mentions.
