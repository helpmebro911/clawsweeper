## Summary

Keep #7717 open. Current main still does not provide a Discord guild/channel role-mention trigger surface; existing `roles` fields are sender allowlists/routing inputs, and preflight never compares mentioned role IDs against configured trigger role IDs.

## Best Possible Solution

Keep #7717 open and implement it in the bundled Discord plugin as an additive guild/channel config surface such as `triggerOnRoles?: string[]`. Wire it through config types, strict Zod schema, generated metadata/help, docs, and preflight. Preflight should compare `message.mentionedRoles` / Discord `mention_roles` against configured role IDs and treat a match as a wake mention, with focused config/schema tests and preflight tests for matched and unmatched role mentions.

## Risks / Open Questions

- A user can approximate some cases with configured mention patterns matching raw Discord role mention text such as `<@&roleId>`, but that is not the requested typed role-ID trigger contract.
- The implementation needs to preserve existing `requireMention`, `ignoreOtherMentions`, @everyone/@here, reply-to-bot, sender `users`/`roles` allowlist, and routing behavior.
