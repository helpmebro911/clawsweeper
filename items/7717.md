## Summary

Keep #7717 open. Current main still does not implement first-class Discord role-mention wake triggers: role mentions are recognized as mention metadata, but the configured Discord guild/channel schema has no `triggerOnRoles`-style role-ID trigger list, and preflight still only wakes on @everyone, explicit bot mention, configured regex mention patterns, or reply-to-bot behavior. The related #20617 is about resolving mentions to readable names and was closed stale; it does not supersede this trigger request.

## Best Possible Solution

Implement this in the bundled Discord plugin as an additive Discord guild/channel config surface, likely `triggerOnRoles?: string[]`, with schema/types/config metadata/docs updates. In preflight, compare Discord `mentionedRoles`/`mention_roles` against the effective configured role IDs and treat a match as a wake mention while preserving existing `requireMention`, `ignoreOtherMentions`, @everyone, explicit bot mention, and reply-to-bot behavior. Add focused preflight and config-schema tests, plus a short Discord docs note at https://docs.openclaw.ai/channels/discord.
