## Summary

Keep #68975 open. Current main still defaults the built-in local memory embedding provider to embeddinggemma, so the bge-m3 default requested by the PR is not implemented or otherwise superseded. The PR is useful but needs rebasing onto the current memory-host-sdk layout plus docs/package mirror updates before maintainer review.

## Best Possible Solution

If maintainers want bge-m3 as the default, rebase the change onto src/memory-host-sdk/host/embedding-defaults.ts, update the package mirror/tests and user-facing memory docs, and run the changed memory/plugin SDK validation lanes.
