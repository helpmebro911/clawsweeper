## Summary

Keep open. Current main still rejects user-supplied chat image attachments above 5,000,000 bytes before any resize/compression step. The Control UI reads selected images as data URLs and forwards the original base64 payload; `chat.send` passes that payload to `parseMessageWithAttachments` with `maxBytes: 5_000_000`, where the exact reported error is still thrown. Existing image optimization code applies to `loadWebMedia` sources, not this base64 chat attachment path, and the related items in the provided context do not supersede this remaining UX request.
