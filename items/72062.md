## Summary

Keep #72062 open. Current main still has tool-name allow/deny plus positive exec allowlist/approval controls, but no per-agent deny-wins policy that matches resolved shell wrapper script paths before approval. The related approval-cascade issues and the closed #57568 PR do not supersede this focused core exec-policy gap.

## Best Possible Solution

Keep the issue open as a focused core exec-policy feature. The best path is to add a documented deny-wins wrapper/script path policy, probably `tools.exec.denyWrappers` plus `agents.list[].tools.exec.denyWrappers` or an equivalent maintainer-approved name, validate it in strict config schema/types, resolve shell/interpreter script operands with the existing exec wrapper helpers, apply it before approval prompts and positive allowlist satisfaction, and cover gateway plus node-host behavior with tests and docs. Keep #69570/#69386 for the separate approval/stop cascade work.
