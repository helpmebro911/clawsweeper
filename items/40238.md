## Summary

Keep this PR open. Current main still dispatches inline skill tools through raw OpenClaw tools plus owner-only filtering, while normal command runs use the broader createOpenClawCodingTools policy pipeline. The PR remains a plausible security-policy fix, but it should be rebased/adapted to preserve the current lazy runtime boundary.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase or adapt it on current main so inline skill command-dispatch: tool resolves through createOpenClawCodingTools, removes the standalone owner-only fallback, preserves the lazy runtime boundary, forwards route/session/group/sender/model/config context including message target and thread context, and keeps focused regression tests for profile-disallowed tools plus argument forwarding.
