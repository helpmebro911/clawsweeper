## Summary

Keep this PR open. Current main still routes inline skill `command-dispatch: tool` through the raw OpenClaw tool factory plus owner-only filtering, while normal command runs use `createOpenClawCodingTools` and the broader profile/provider/group/subagent policy pipeline. The PR remains a relevant security/policy fix, but it should be rebased/adapted to current main and preserve the current lazy runtime boundary plus full command-route context.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase or adapt it so inline skill `command-dispatch: tool` resolves tools through `createOpenClawCodingTools` on current main, preserves the current lazy runtime pattern, forwards the same route/session/group/sender/model/config inputs as normal command runs, removes the standalone owner-only fallback, and keeps focused regression tests for non-owner/profile-disallowed tools plus route-context preservation before running the targeted inline dispatch test and `pnpm check:changed`.
