## Summary

Keep PR #48433 open. Current main has since shipped related host routing and pluggable sandbox backend work, but it has not implemented the PR's core contract: `tools.exec.allowedHosts` plus per-host `exec-approvals.json` allowlist buckets. This is core exec/security policy work, not a ClawHub/plugin item, and the PR's remaining value should be reviewed by a maintainer after rebase and resolution of the open safety review concerns.

## Best Possible Solution

Keep the PR open for maintainer judgment. The best path is to rebase it onto current main, reconcile it with the newer `host=auto` and pluggable sandbox backend model, and either implement `tools.exec.allowedHosts` plus per-host approval buckets with focused tests/docs or split the design into a smaller core exec-policy proposal. If maintainers decide `host=auto` is the intended replacement, close manually with that product rationale rather than treating this as already implemented.
