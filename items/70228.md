## Summary

Current main still appears to have the reported cosmetic false positive. The Darwin minimal PATH resolver unconditionally includes common user-bin/version-manager fallback directories, and the service audit compares the LaunchAgent PATH against that full expected set, emitting the exact "missing required dirs" warning for any omitted entries. The status command surfaces these service config audit issues, so the user-visible report is still actionable and not superseded by the PATH-adjacent related items.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the gateway service audit: for Darwin PATH audits, warn only on meaningful missing entries such as existing directories, explicit env-configured dirs, and installed runtime bin dirs, or downgrade/rename the fallback superset wording from `required dirs` to recommended/preferred. Add a focused service-audit test for a healthy macOS LaunchAgent PATH that omits nonexistent user-bin fallbacks.
