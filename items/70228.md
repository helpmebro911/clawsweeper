## Summary

Keep open. Current `main` still implements the behavior the issue reports: the macOS gateway audit compares the service PATH against a hard-coded Darwin superset, warns on every missing entry, and labels those entries as "required" even when they are only heuristic defaults.

## Evidence

- **PATH audit still warns on every missing expected entry:** `auditGatewayServicePath` builds `expected` from `getMinimalServicePathPartsFromEnv`, diffs it against the service PATH, and emits `Gateway service PATH missing required dirs: ...` for any missing entries. There is no existence or relevance filter before warning.
  - file: [src/daemon/service-audit.ts:282](https://github.com/openclaw/openclaw/blob/5376a4a5d6dd6332904481cf39be9537132b5c8a/src/daemon/service-audit.ts#L282)
  - command: `nl -ba src/daemon/service-audit.ts | sed -n '270,330p'`
  - sha: [5376a4a5d6dd](https://github.com/openclaw/openclaw/commit/5376a4a5d6dd6332904481cf39be9537132b5c8a)
- **Darwin path builder unconditionally adds many user-bin defaults:** `resolveDarwinUserBinDirs` appends common user-bin paths plus fnm/pnpm fallback locations such as `~/Library/Application Support/fnm/aliases/default/bin`, `~/.fnm/aliases/default/bin`, `~/Library/pnpm`, and `~/.local/share/pnpm` whenever `HOME` is set. The function does not check whether those directories exist.
