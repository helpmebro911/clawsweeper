## Summary

Keep #8892 open. Current main still has no native `openclaw tui --agent <id>` startup selector; it only supports explicit agent-scoped `--session agent:<id>:<session>`, cwd workspace inference, default-agent fallback, and post-launch `/agent <id>`. The workaround noted in the issue discussion helps dedicated workspace launches but does not satisfy the requested generic-directory startup selector.

## Best Possible Solution

Add a native `--agent <id>` selector for `openclaw tui`: register the flag in `src/cli/tui-cli.ts`, extend `TuiOptions`/`runTui` with a typed startup agent id, validate it against configured agents, define precedence versus `--session agent:<id>:...` and cwd workspace inference, and update focused TUI tests plus `docs/cli/tui.md` and `docs/web/tui.md`. Keep #8892 open as the canonical user-facing request; #19247 is the same request but is already closed, and #40176 only provides a workspace-inference workaround.
