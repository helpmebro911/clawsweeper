## Summary

Keep #8892 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still has no native `openclaw tui --agent <id>` startup selector; it only supports session-derived routing, workspace inference, and post-launch `/agent`.

## Best Possible Solution

Add a native `--agent <id>` selector for `openclaw tui`: register the flag in `src/cli/tui-cli.ts`, extend `TuiOptions`/`runTui` with a typed startup agent id, validate it against configured agents, define precedence versus `--session agent:<id>:...` and cwd workspace inference, and update focused TUI tests plus `docs/cli/tui.md` and `docs/web/tui.md`.
