## Summary

Keep open. Current main still does not implement a native `openclaw tui --agent <id>` flag. The TUI can infer an agent from CWD, switch via `/agent`, or use a fully qualified `--session agent:<id>:<key>`, but the requested CLI flag and the reported `--session agent:ollama` shorthand remain unsupported.

## Best Possible Solution

Keep the issue open until OpenClaw implements a native TUI agent selector: add `--agent <id>` in `src/cli/tui-cli.ts`, add an `agent`/`agentId` field to `TuiOptions`, validate the configured agent similarly to `openclaw agent --agent`, define precedence versus `--session agent:<id>:...` and CWD auto-discovery, update TUI docs, and add focused tests for the new startup routing behavior.
