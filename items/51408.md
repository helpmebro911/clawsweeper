## Summary

Keep open. Current `main` still has the core ingredients from the report: heartbeat runs default to the main session, heartbeat-specific model overrides are routed into the active run model, compaction safeguard runtime is still sessionManager-scoped and populated from the current run model/context window, and replay sanitization still writes `model-snapshot` entries for whichever model just ran. I did not find a fix on `main` that isolates heartbeat/auxiliary model state from the shared main-session transcript or compaction runtime.
