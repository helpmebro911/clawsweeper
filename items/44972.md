## Summary

Keep #44972 open. Current main still emits `response.completed` for the streamed tool-call path even though it builds a response with `status: "incomplete"`; the schema/test coverage also does not contain the PR's `response.incomplete` event. The PR remains a focused gateway contract fix, not obsolete.

## Best Possible Solution

Keep the PR open or land an equivalent maintainer-reviewed patch: add `ResponseIncompleteEventSchema` to `src/gateway/open-responses.schema.ts`, emit `response.incomplete` from `src/gateway/openresponses-http.ts` when streamed responses stop with pending tool calls, and update the regression test so streamed tool-call responses assert `response.incomplete` plus `[DONE]`.
