## Summary

Keep open. The PR is still a small, relevant fix: current main still passes `job.name` directly into `truncate(...)` in `printCronList`, while the linked issue shows a real `openclaw cron list` crash and this PR adds the missing `?? "-"` guard. The related issue is closed, but I did not find source-backed evidence that this exact guard is present on current main, so this is not safe to close as implemented.

## Evidence
