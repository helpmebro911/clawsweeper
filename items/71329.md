## Summary

Keep this PR open. Current main still couples Slack lifecycle status reactions to the ack-reaction path, so `messages.statusReactions.enabled: true` alone does not activate the lifecycle controller. The PR addresses a real remaining Slack behavior gap, with one small test assertion cleanup noted in review.

## Best Possible Solution

Keep this PR open and review or adapt its narrow Slack prepare change: let `messages.statusReactions.enabled === true` activate lifecycle reactions independently of `ackReaction`, preserve `enabled: false` as an explicit disable, keep the existing ack-reaction scope-gated fallback for backwards compatibility, and update the explicit-disable test to assert the plain ack reaction promise behavior noted in review.
