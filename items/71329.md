## Summary

Keep this PR open. Current main still requires Slack lifecycle status reactions to pass the ack-reaction gate, so `messages.statusReactions.enabled: true` does not independently activate the existing controller. The PR is a narrow implementation candidate, with one test assertion gap already identified in review and no obvious security-sensitive expansion in its two-file diff.

## Best Possible Solution

Keep this PR open and review or adapt its narrow Slack change: let `messages.statusReactions.enabled === true` activate lifecycle status reactions independently of the ack-reaction gate, preserve `enabled: false` as an explicit status-controller disable, keep the plain ack-reaction fallback, and tighten the explicit-disable regression test before merging.
