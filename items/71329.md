## Summary

Keep #71329 open. Current main still gates Slack lifecycle status reactions through the ack-reaction gate, so `messages.statusReactions.enabled: true` does not independently activate the controller in DMs/default `group-mentions` cases or when ack reactions are disabled. The PR is a focused Slack plugin fix, but should address the review-noted test assertion before merge.

## Best Possible Solution

Keep the PR open and either merge or adapt its focused Slack prepare change: treat `messages.statusReactions.enabled === true` as an independent opt-in, preserve `enabled: false` and existing ack-scope behavior for the compatibility path, and add regression tests for explicit enable without ackReaction plus explicit disable with the plain ack reaction still firing.
