## Summary

Keep PR #71329 open. Current main still gates Slack lifecycle status reactions through the ack-reaction path, so `messages.statusReactions.enabled: true` does not independently activate the controller in the scenario this PR targets. The PR is focused, recent, and not superseded by the related Slack progress/thread-status items.

## Best Possible Solution

Keep the PR open. The best path is to retain the small prepare-gate change, add the missing assertion requested in review for the `statusReactions.enabled: false` branch, then run the Slack extension targeted tests and maintainer review it as a bundled Slack plugin behavior fix.
