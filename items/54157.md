## Summary

Keep this issue open. Current main still has zero `cacheRead` and `cacheWrite` values for both Volcengine/Doubao and BytePlus model catalogs, and usage-cost estimation still multiplies cached-token usage by those catalog fields. PR #55913 is an open implementation candidate that uses GitHub closing syntax for this issue, so the issue should remain open until that PR is reviewed and landed or closed.

## Best Possible Solution

Keep this issue open and review PR #55913 as the implementation candidate. The fix should update `extensions/volcengine/models.ts` and `extensions/byteplus/models.ts` with source-backed non-zero `cacheRead` and `cacheWrite` pricing, correct any related per-million-token pricing if the vendor docs support it, and add focused regression tests in the current plugin test locations.
