## Summary

Current main has passwordless Microsoft Teams support via the newer `authType: "federated"` certificate/managed-identity path, but it does not implement this PR's `defaultAzureCredential` auth type or the automatic Azure credential chain/local `az login` behavior described in the PR. The PR itself is stale against the current Teams SDK implementation, so it needs rework, but the requested capability is not already shipped or clearly superseded.

## Best Possible Solution

Keep the item open for maintainer/product judgment. The best path is a fresh, current-main implementation or follow-up issue that decides whether Microsoft Teams should add a `defaultAzureCredential` mode on top of the existing `authType: "federated"` managed-identity support, with separate identity-client-id handling and docs for AKS/Azure Container Apps/local development.
