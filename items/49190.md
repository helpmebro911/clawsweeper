## Summary

Keep open. Current main still lacks a first-class cron warning status and still records non-best-effort delivery-target/delivery failures as error. Delivery diagnostics are tracked separately and best-effort delivery can avoid failing some runs, but that is opt-in and renders as ok/not-delivered rather than the requested warning state in Control UI. The related #67377 context is adjacent classifier work, not a superseding implementation of this execution-vs-delivery split.
