## Summary

Keep this PR open as a focused but stale implementation candidate. Current main still rejects the requested WhatsApp `group:` and `whatsapp:group:` target forms in the active bundled WhatsApp plugin, while the submitted patch targets obsolete `src/whatsapp/*` paths rather than `extensions/whatsapp/src/*`.

## Best Possible Solution

Keep this PR open, but ask the author to refresh it against `extensions/whatsapp/src/normalize-target.ts` and `extensions/whatsapp/src/resolve-target.test.ts`. The best implementation is a narrow normalizer update that strips repeated `whatsapp:` and `group:` prefixes for WhatsApp group targets, with regression coverage for valid group prefixes, mixed case, empty-group invalid cases, and repeated prefixes, followed by the WhatsApp plugin test lane.
