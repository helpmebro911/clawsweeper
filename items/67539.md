## Summary

Keep this issue open. Current main has Gemini TTS runtime support and human-facing docs, but the requested provider-owned TTS prompt-hint hook and active-provider injection path are still missing.

## Best Possible Solution

Add an additive optional prompt-hint hook to SpeechProviderPlugin and the public SDK; have speech-core resolve only the active provider, resolved provider config/model/voice, target, and modelOverrides before appending that provider's short hint; then let the Google provider own the Gemini square-bracket audio-tag guidance and cover active-provider-only and policy-gated cases in tests.

## Risks / Open Questions

- Adding this hook changes the public speech-provider/plugin SDK contract, so SDK exports, provider docs, and contract tests should change together.
- Prompt hints must be gated by the resolved active provider/model/target and modelOverrides policy so inactive provider syntax or disabled controls are not advertised.
