## Summary

Current main still does not implement the requested OpenRouter behavior. The OpenRouter Anthropic wrapper applies `cache_control` through a helper that marks system/developer messages and strips assistant thinking-block markers, while the direct Anthropic path separately tags the trailing user message. Current tests and docs still encode system/developer-only OpenRouter cache markers. The same-author implementation candidate, PR #63062, is open and unmerged with closing syntax for this issue, so the issue should stay open pending maintainer review of that PR.

## Best Possible Solution

Review PR #63062 against current main. If maintainers accept the behavior, land a focused change that reuses the existing trailing-user-message cache-control helper on the OpenRouter Anthropic path, preserves verified-route gating and assistant thinking-block cleanup, adds regression coverage for user string and array content, and updates the prompt-caching/OpenRouter docs. If maintainers intentionally want OpenRouter to remain system/developer-only, document that product decision in the issue and docs, then close with explicit maintainer judgment.
