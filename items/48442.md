## Summary

Keep PR #48442 open. Current main does not implement the proposed `agents.defaults.systemPrompt.identityLine` or `disabledSections` contract, and the PR remains potentially useful prior art, but it is not mergeable as written because the provided review discussion correctly identifies missing schema support and missing runtime wiring.

## Best Possible Solution

Keep the PR open for maintainer/product judgment, but request a rework before merge: add the strict Zod/generated config/docs surface, decide whether the option is defaults-only or per-agent too, thread it through embedded, CLI, auto-reply, compaction, and prompt-report paths, and add tests that exercise real config validation plus runtime prompt assembly. If maintainers do not want public built-in section names as core config, the better path is a plugin/API design that can transform or replace the assembled prompt deliberately rather than this one-off config shape.
