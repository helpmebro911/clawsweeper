## Summary

Keep this PR open. Current main still hardcodes bundled session-memory snapshots under `<workspace>/memory/`, and the discussion describes a real memory indexing/noise problem that this PR is trying to solve. The work is not implemented or superseded, but the PR should be revised before merge for symlink-safe directory creation and docs i18n policy.

## Best Possible Solution

Keep this PR open and request a small revision: preserve the default `memory` output, add optional workspace-relative `hooks.internal.entries.session-memory.outputDir`, reject empty/absolute/traversal values, write via `writeFileWithinRoot` rooted at the workspace with relative `outputDir/filename`, avoid any pre-write mkdir that can follow symlinks outside the workspace, add regression tests for custom output dirs, invalid fallback, and symlink alias behavior, and remove localized docs edits. Coordinate with #54524 for broader duplicate-indexing guidance or dedupe policy.
