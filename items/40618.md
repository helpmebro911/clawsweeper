## Summary

Keep this PR open. Current main still hardcodes session-memory snapshots under `<workspace>/memory/` and has no `outputDir` implementation, docs, or tests. The PR addresses a real workflow problem raised in discussion, but the proposed head needs revision before merge because its pre-write `fs.mkdir` can create directories through an in-workspace symlink outside the workspace, and it edits localized docs that repo policy says not to edit.

## Best Possible Solution

Keep this PR open and request a focused revision: preserve the default `memory` output, add optional workspace-relative `hooks.internal.entries.session-memory.outputDir`, reject empty/absolute/traversal values with fallback to `memory`, route writes through `writeFileWithinRoot` rooted at the workspace with `relativePath` set to `outputDir/filename`, remove or replace the separate pre-write mkdir with an alias-safe helper, add regression coverage for custom output, invalid fallback, and symlink alias behavior, and drop localized docs edits. Coordinate the docs/design angle with #54524, which tracks the broader duplicate-indexing guidance.

## Risks / Open Questions

- The PR head should not merge as-is because a separate pre-write `fs.mkdir(outputRoot, { recursive: true })` on a configured path can create directories through an in-workspace symlink outside the workspace before the later alias-safe write rejects the final file.
- The PR edits localized `docs/zh-CN/**` files, which conflicts with the repo docs i18n policy; English docs should be updated here and localization should flow through the publish/i18n pipeline.
- Coverage should include the custom output directory, invalid absolute/traversal fallback, and a symlink alias case so the workspace-relative guarantee is executable.
