## Summary

Keep this PR open. Current main still drops `ReplyToId` and `ReplyToBody` before the plugin `inbound_claim` event, while the same finalized context already exists upstream and is used by the agent inbound metadata path. The PR remains a narrow additive implementation candidate, not obsolete, duplicated, or superseded.

## Best Possible Solution

Keep this PR open and review it as a small plugin hook API fix. The best path is to add optional `replyToId` and `replyToBody` to the canonical inbound hook context, map them from `FinalizedMsgContext`, expose them in `inbound_claim` metadata, and add a focused regression test in `src/hooks/message-hook-mappers.test.ts`.
