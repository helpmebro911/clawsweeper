## Summary

Keep open. Current main still excludes markdown from the built-in skill/plugin code-safety scanner, while this PR is a focused security-hardening follow-up to #10705. The PR should not be closed as stale, but it also should not merge as-is because the proposed critical markdown download-and-exec rule still misses common execution forms.

## Best Possible Solution

Keep this PR open as the active core security-hardening candidate for markdown skill scanning. Ask for a revision that broadens `markdown-download-exec` detection for wrapped interpreters, absolute paths, versioned interpreters, and line-continuation pipelines; add regression tests for those cases; then run the relevant changed gate before merge.
