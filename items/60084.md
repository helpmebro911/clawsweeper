## Summary

Keep open. Current `main` still hardcodes `node:sqlite` for the memory index and other file-backed SQLite stores, exposes no alternate driver/fallback in config, and the issue is recent and specific enough that it is not a candidate for stale closure. I could not do a direct ARM64/Kubernetes reproduction in this checkout-only environment, but the code/docs show the reported failure mode remains unaddressed on `main`.
