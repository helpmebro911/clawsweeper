## Summary

Keep this PR open. Current main still lacks abort-aware queued-entry removal in `enqueueCommandInLane`, while the cron timeout signal already reaches `runEmbeddedPiAgent`; this PR remains a focused implementation candidate for the linked open bug #43141, not conservative cleanup material.

## Best Possible Solution

Keep this PR open and review or land it after addressing the minor review comments and running targeted command-queue and embedded cron-runner validation. An equivalent fix should add queued-entry abort removal to `enqueueCommandInLane`, preserve active-task behavior, clean up abort listeners, thread the existing cron abort signal through the default embedded-runner enqueue wrappers, and keep regression coverage for pre-aborted, queued-aborted, active-after-dequeue, and never-run aborted entries.
