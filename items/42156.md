## Summary

Current main still matches the reported Telegram callback timeout path: callback acknowledgements are performed inside the downstream `callback_query` handler, while grammY `sequentialize` is installed earlier and ordinary callback updates share the normal chat/topic lane. The linked fix PRs are closed unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Implement the Telegram plugin fix on current main: add a pre-sequentialize middleware in `extensions/telegram/src/bot-core.ts` that immediately answers callback queries with caught/logged failures, remove or avoid redundant handler-level acknowledgement in `extensions/telegram/src/bot-handlers.runtime.ts`, and add a regression test proving acknowledgement happens before the queued callback action is released when a same-topic turn is busy.
