## Summary

Keep this open. Current main still generates macOS LaunchAgent StandardOutPath/StandardErrorPath from the OpenClaw state directory under `.openclaw/logs`, writes those paths verbatim into the plist, and only checks whether the service is loaded after install rather than proving the gateway RPC is healthy. The issue discussion contains concrete macOS exit-78 confirmations and workarounds, so this remains an actionable core LaunchAgent bug.

## Best Possible Solution

Keep this issue open and fix the CLI-managed macOS LaunchAgent path in `src/daemon`: write `StandardOutPath` and `StandardErrorPath` to a launchd-safe non-symlinked user log location such as `~/Library/Logs/OpenClaw`, or resolve the generated log directory with `realpath` before writing the plist. Add coverage for symlinked state directories and plist regeneration, update runtime hints/docs, and consider an install-time health check that clearly reports launchd EX_CONFIG failures.
