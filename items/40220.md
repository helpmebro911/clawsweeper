## Summary

Current main still leaves the raw file-log JSON timestamp mismatch in place: top-level `time` is formatted with a local offset, while `_meta.date` remains tslog's `Date` metadata and will serialize as UTC. The related PR #48717 identified the same root cause, but it is closed unmerged and now has no diff/files, so it does not supersede the issue.

## Best Possible Solution

Keep this issue open. Patch `src/logging/logger.ts` so the file transport computes one host-local timestamp string and serializes both top-level `time` and `_meta.date` with that same value, preserving the rest of `_meta` without mutating the original tslog object. Add focused coverage in `src/logging/logger-timestamp.test.ts` asserting `_meta.date === time` and the explicit offset format. Treat any broader request to honor `agents.defaults.userTimezone` in logs as a separate product decision because current docs scope that setting to prompt context.
