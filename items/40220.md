## Summary

Current main still appears to leave the reported raw JSON log timestamp mismatch unresolved. The file transport writes a local-offset top-level `time`, but serializes tslog's original `_meta.date` Date object unchanged, so JSON readers can still see `_meta.date` as UTC `Z` while `time` uses the OpenClaw formatter. The related PR #48717 was closed unmerged in the provided GitHub context, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open. Patch `src/logging/logger.ts` so the file-log JSON payload copies `_meta` and serializes `_meta.date` with the same timestamp string used for top-level `time`, then add focused coverage around file-log JSON output asserting `_meta.date === time` and the explicit offset format. Treat switching file logs from host/TZ local time to `agents.defaults.userTimezone` as a separate product/docs decision unless maintainers explicitly want that broader behavior here.
