## Summary

Keep open. Current main still documents and tests inline/bare TTS directive behavior, while this PR changes that public contract to standalone-line-only parsing and also carries unrelated Slack/config-validation changes. A maintainer has already said the work needs to be split and reworked before merge.

## Best Possible Solution

Keep this PR open for a narrower revision. Split or drop the unrelated Slack/config-validation hunks, rebase on current main, then make the TTS change only after maintainers decide whether inline/bare directives remain supported or are intentionally migrated to standalone-only syntax. A focused follow-up should update `src/tts/directives.ts`, `src/tts/directives.test.ts`, `test/helpers/plugins/tts-contract-suites.ts`, `docs/tools/tts.md` / https://docs.openclaw.ai/tools/tts, and changelog coverage together.

## Risks / Open Questions

- Merging as-is would silently change documented and tested inline/bare TTS directive behavior used by tagged-mode auto-TTS.
- The PR mixes a public parser contract change with unrelated Slack and config-validation changes, increasing review and regression risk.
- The Slack `ignoreSelf` change is security-sensitive because self-event filtering affects bot loop prevention; current main has a safer dedicated middleware path that this PR needs to rebase around.
- Standalone-only syntax, if desired, still needs explicit product/API approval plus docs, tests, runtime prompt hints if any, and changelog coverage.
