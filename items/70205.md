## Summary

Keep open. Current main still has the reported bug: `sessions.usage` scans configured agents, but unscoped `usage.cost` still calls the transcript-backed cost summary without an agent scope, so it resolves to the default agent session directory only. The PR appears to address a real remaining gap, though its diff should be cleaned up because it includes unrelated changes outside the gateway/UI usage surface.

## Best Possible Solution

Keep the PR/open work alive, but require a maintainer-quality cleanup before merge: isolate the gateway `usage.cost` aggregation and UI date-only label fix from unrelated branch drift, keep explicit `agentId` scoping and cache-key scoping, retain the multi-agent gateway tests plus the timezone label test, and leave #13219 open for the broader dedicated usage log proposal.
