## Summary

Keep open. Current main still routes unscoped `usage.cost` through a single transcript directory, while `sessions.usage` already scans configured agents. The PR is a focused bug fix with useful reproduction evidence and bot-review follow-up; it is not implemented on main and is not safely closable as duplicate because the related PR #42063 is also open and this PR appears to carry broader tests/UI coverage.

## Best Possible Solution

Keep a focused fix open: aggregate unscoped gateway `usage.cost` across the configured gateway agents, include the resolved agent scope in the cache key, preserve explicit `agentId` scoping/validation, add regression coverage for multi-agent cost aggregation and UI date-only labels, then reconcile or close the older related PR #42063 as superseded once a canonical patch lands.
