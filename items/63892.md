## Summary

Current main still has the reported behavior: `runPreflightCompactionIfNeeded` returns early for fresh persisted token totals when the transcript-byte trigger is inactive, before token-threshold evaluation runs. The provided context also shows an open same-author implementation PR with closing syntax, so this issue should stay open until maintainers land or explicitly supersede that PR.

## Best Possible Solution

Keep this issue open. Review and land or explicitly supersede #64384, or make #66716 the canonical implementation if maintainers prefer that broader approach. The fix should let fresh persisted token totals at or above the preflight threshold reach `shouldRunPreflightCompaction`, preserve the below-threshold fast path, and add a focused regression test for `totalTokensFresh: true`, cached tokens above threshold, and no transcript-byte trigger.

## Evidence

- **Fresh-token path still returns before threshold check:** `runPreflightCompactionIfNeeded` computes `freshPersistedTokens`, then returns when persisted totals are fresh and `shouldCompactByTranscriptBytes` is false. That return occurs before `shouldRunPreflightCompaction` is called, so cached totals above the token threshold are skipped.
