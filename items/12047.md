## Summary

Keep #12047 open. Current main partially supports Telegram document attachments by downloading them and exposing paths/MIME types to shared media extraction, and shared extraction can handle text-like files plus PDFs. The central requested Office/.docx-style capability remains missing: Office MIME types are explicitly treated as binary before extraction, default file MIME support omits Office formats, the bundled document extractor only declares PDF support, and tests assert that application/vnd Office attachments are skipped.

## Best Possible Solution

Keep this as an OpenClaw issue. Extend the shared media/document extraction path so supported Office MIME types such as .docx can reach a safe document-extractor contract with size limits, untrusted-content wrapping, tests, and docs. Telegram should continue to supply MediaPath/MediaType into the shared path rather than owning document parsing itself.
