## Summary

Keep #12047 open. Current main still only partially satisfies the request: Telegram downloads document attachments and forwards their MediaPath/MediaType into the shared media pipeline, and shared extraction handles text-like files plus PDFs. The requested Office-style document support remains missing because default file MIME allowlists omit DOCX/XLSX/PPTX, application/vnd Office MIME types are skipped as binary before extraction, and the bundled document-extract plugin is PDF-only. A same-day Codex review comment on the issue already kept it open for the same remaining gap, and the relevant files are unchanged since that review.

## Best Possible Solution

Keep this open as an OpenClaw product/contract issue. Telegram should continue forwarding attachments through the shared MediaPath/MediaType pipeline; the fix should extend shared file/media extraction so safe Office MIME types can pass configured/default allowlists, route non-PDF document MIME types through the generic document-extractor contract, add or bundle an Office document extractor implementation, and cover Telegram attachments plus generic shared extraction with tests and docs.
