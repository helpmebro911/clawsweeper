## Summary

Current main only partially covers the request. Telegram document attachments are downloaded, type-tagged as `<media:document>`, and forwarded into the shared media context, and the shared path can extract text/Markdown/JSON/PDF. The remaining requested Office-style document support is still missing: default file MIME allowlists omit Office formats, `application/vnd.*` Office MIME types are treated as binary before extraction, dispatch only sends PDFs to document extractors, and the bundled document-extract plugin only declares a PDF extractor. The existing issue discussion already has a same-day maintainer/Codex review keeping this open for the same DOCX/XLSX/PPTX gap.

## Best Possible Solution

Keep this open as an OpenClaw product/contract issue. Extend the shared media/input-file extraction path so safe Office MIME types can reach document extractors, add or bundle an Office document extractor implementation, and cover Telegram attachments plus generic shared extraction with tests and docs. Telegram should continue forwarding MediaPath/MediaType into the shared path rather than parsing documents itself.
