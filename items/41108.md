## Summary

Keep open. Current main still has the MS Teams bug this PR targets: implicit mention detection only checks `activity.replyToId` against the sent-message cache. Main parses `conversation.id;messageid=...` for Teams thread routing, but does not use that thread-root ID in mention gating or cache it when the bot replies.

## Best Possible Solution

Keep the PR open and have the Teams maintainer adapt the fix to current main: derive implicit mention from either `replyToId` or `extractMSTeamsConversationMessageId(rawConversationId)`, seed the lookup with the thread root when replying, and add tests that exercise the production mention helper/path with RSC requirements left documented.
