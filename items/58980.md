## Summary

Keep open. Current `main` still has the exact gap this PR fixes: `normalizeProxyReasoningPayload` deletes `reasoning_effort` but returns early for `thinkingLevel === "off"` without deleting an injected `reasoning` object. OpenRouter and Kilocode still use that helper on `main`. There is a newer retry-to-`minimal` fallback for reasoning-mandatory endpoint errors, but that is a mitigation after a failed request, not the direct payload cleanup proposed here.
