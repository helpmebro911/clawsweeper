## Close Comment

Closing as implemented on `main` after ClawSweeper/Codex review. The requested compatibility/support path already exists: unlisted providers can be added through Custom Provider onboarding (`docs/start/onboarding-overview.md:58`, `docs/start/wizard-cli-reference.md:191`), the onboarding flow detects OpenAI/Anthropic-compatible endpoints and writes a custom provider entry with `baseUrl`, `api`, `apiKey`, and model metadata (`src/commands/onboard-custom.ts:244`, `src/commands/onboard-custom-config.ts:531`), and `main` now includes a community-proxy guide showing OpenClaw configured against an OpenAI-compatible subscription proxy (`docs/providers/claude-max-api-proxy.md:78`, added in `467fcb17912e14797bc8caa6689c30d290e39d01`). That is enough to cover this class of low-cost proxy integration without adding a new dedicated core integration. This is on `main` after `v2026.4.22`, so there is no shipped release tag to point to yet.
