## Summary

Keep open. Current `main` at `7d9172d5e093435d37ed3e00e476a4eaf84a252b` does not include the PR's Control UI stale-run recovery. The UI still marks chat busy solely from `chatRunId`, only clears that state on terminal chat events or reconnect, and has no Control UI `agent.wait`/watchdog path to recover a missed terminal event while the connection stays up.
