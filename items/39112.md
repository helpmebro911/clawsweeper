## Summary

Keep PR #39112 open. Current main still persists native Ollama responses with omitted `prompt_eval_count` / `eval_count` as zero usage, and the Control UI usage path consumes persisted transcript usage rather than estimating later. The PR remains useful, but its patch targets the removed `src/agents/ollama-stream.*` layout and needs porting to the current bundled Ollama plugin.

## Best Possible Solution

Keep this PR open, or land an equivalent current-main fix in `extensions/ollama/src/stream.ts`: estimate native Ollama input/output usage only when counters are absent, preserve exact nonnegative counters including explicit zero, count relevant message/image/tool payloads in the prompt estimate, and add focused regression coverage in the current Ollama plugin tests for missing counters, explicit zero counters, streamed responses, and multimodal/tool payload estimates.
