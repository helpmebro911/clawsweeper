## Summary

Keep open. Current `main` still matches the reported mismatch: Feishu config code supports and writes top-level single-account credentials under `channels.feishu`, while the Feishu docs only document credential fields under `channels.feishu.accounts.<id>`. The same mismatch is present in the latest shipped release `v2026.4.22`.

## Evidence

- **Docs only document nested account credentials:** The Feishu channel docs list `channels.feishu.accounts.<id>.appId` and `channels.feishu.accounts.<id>.appSecret` in the configuration reference, with no matching top-level `channels.feishu.appId` / `appSecret` entry nearby.
  - file: [docs/channels/feishu.md:386](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/docs/channels/feishu.md#L386)
  - command: `nl -ba docs/channels/feishu.md | sed -n '371,390p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Schema still supports top-level single-account credentials:** The Feishu config schema explicitly keeps top-level `appId` and `appSecret` as backward-compatible single-account fields.
  - file: [extensions/feishu/src/config-schema.ts:212](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/extensions/feishu/src/config-schema.ts#L212)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '208,236p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Setup flow still writes default account to top-level config:** The setup surface comment and implementation state that `DEFAULT_ACCOUNT_ID` writes to top-level `channels.feishu`, not to `channels.feishu.accounts[accountId]`. That means user-generated config can still have the shape reported in the issue.
  - file: [extensions/feishu/src/setup-surface.ts:76](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/extensions/feishu/src/setup-surface.ts#L76)
  - command: `nl -ba extensions/feishu/src/setup-surface.ts | sed -n '74,110p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Latest release still ships the same docs mismatch:** The released `v2026.4.22` docs still show only `channels.feishu.accounts.<id>.appId` / `appSecret` in the Feishu configuration reference.
  - file: [docs/channels/feishu.md:386](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/docs/channels/feishu.md#L386)
  - command: `git show v2026.4.22:docs/channels/feishu.md | nl -ba | sed -n '371,390p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

## Risks / Open Questions

- This may be partly a docs bug and partly a product-shape inconsistency: users can still be led into the undocumented top-level layout by setup flows.
- If maintainers intend to standardize on `accounts`, a real fix likely needs either docs for both shapes or a setup/config migration away from top-level single-account serialization.
