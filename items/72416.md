## Summary

Keep this PR open. Current main still only accepts stable `openclaw.install.minHostVersion` floors in `>=x.y.z` form, so the prerelease floor form this PR adds is not implemented. The PR is recent, focused, has matching docs/tests in the provided diff, and the security review found no unexplained workflow, dependency-source, lockfile, publishing, secret, or script changes.

## Best Possible Solution

Keep the PR open and review or land it as the implementation candidate. The likely best path is to update `src/plugins/min-host-version.ts` to accept prerelease semver floors while preserving the `>=`-floor-only contract, keep enforcement centralized for install and manifest-registry loading, and merge the accompanying regression tests and public plugin-author docs updates.
