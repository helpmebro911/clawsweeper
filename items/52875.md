## Summary

Current main still does not implement the requested recovery path. `sessions.send` / `sessions_send` still require an existing session, there is no registered `sessions.start` / `sessions_start` surface, and timeout responses still lack the delivery metadata discussed in the issue. The discussion positively identifies PR #52921 as the implementation candidate, and because it is still open and uses closing syntax for this issue, this issue should remain open until maintainers review, land, or close that PR.

## Best Possible Solution

Keep this issue open while maintainers review PR #52921. If the PR is correct and safe, land it and let this issue close through the linked PR. If not, keep this issue as the canonical report and implement the missing behavior in the gateway/session owner surfaces: an idempotent `sessions.start`/`sessions_start`, `sessions_send` auto-bootstrap for known agent configs, structured not-found errors, delivery metadata on timeout/accepted paths, plus focused regression tests and docs updates.
