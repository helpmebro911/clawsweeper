## Summary

Current main does not implement the requested recovery path. `sessions_send` still requires an existing resolved session, missing label/key resolution still returns `No session found` as `INVALID_REQUEST`, there is no registered `sessions.start`/`sessions_start` API or tool, and the related fix PR #52921 is still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #52875 open until a focused fix lands on main. The best path is to finish or replace PR #52921 with an additive implementation: idempotent `sessions.start`/`sessions_start` for known agent IDs, optional `sessions_send` auto-bootstrap via the existing `sessions.create` path, structured missing-session errors, accepted/message tracking for timeout/fire-and-forget sends, and matching protocol docs plus regression tests.
