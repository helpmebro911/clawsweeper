## Summary

Keep #24107 open. Current main still has generic stepped auth-profile cooldowns plus OpenAI Codex WHAM-specific cooldown probing, but it does not provide the requested generic provider-level `rpm`/`rpd` config contract, quota-window counters, or pre-request quota skip in `runWithModelFallback`.

## Best Possible Solution

Keep #24107 open as a core model fallback/auth-profile enhancement. A complete fix should add a validated `rpm`/`rpd` provider quota contract, generated schema/docs updates, persisted minute/day quota state with reset timestamps, pre-request skip and `cooldownUntil` marking in `runWithModelFallback`, and tests for RPM/RPD reset behavior alongside the existing WHAM and model-scoped cooldown coverage.
