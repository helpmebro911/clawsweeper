## Summary

Keep #24107 open. Current main has narrower cooldown improvements, including OpenAI Codex WHAM-aware cooldown probing and a shorter stepped generic cooldown, but it still does not implement the requested generic provider-level `rpm`/`rpd` config contract, persisted quota-window counters, or pre-request quota skip in model fallback.

## Best Possible Solution

Keep #24107 open as a core model fallback/auth-profile enhancement. A complete fix should add a validated provider quota config/API contract for `rpm`/`rpd`, generated schema/docs updates, persisted per-provider or per-profile minute/day counters and reset timestamps, pre-request skip/mark-cooldown logic in `runWithModelFallback`, integration with `cooldownUntil` and probe decisions, and focused tests for RPM and RPD reset behavior while preserving the existing OpenAI Codex WHAM-specific cooldown behavior.
