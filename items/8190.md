## Summary

Keep #8190 open. Current main still supports single-session hard reset and current-session soft reset, but not a global auth-refresh/reset action that updates all active sessions while preserving conversation context. The related #7533 account-switching work is not a superseding fix for this remaining cross-session lifecycle capability.

## Best Possible Solution

Keep #8190 open as the canonical tracker. The best implementation is an additive gateway/session capability, such as `sessions.refreshAuth` or an explicit all-sessions auth-refresh mode, that enumerates active sessions, invalidates auth-backed runtime and CLI caches, preserves transcripts and channel/session context where possible, and adds protocol, gateway, session-store/channel tests plus user-facing docs. Treat #7533 as shipped account switching, not as a superseding fix for this remaining global session refresh work.

## Risks / Open Questions

- A naive implementation that loops over `sessions.reset` would rotate session ids and archive transcripts, conflicting with #8190's context-preservation requirement.
