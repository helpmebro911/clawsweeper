## Summary

Keep #8190 open. Current main still has only per-session reset and per-session soft reset/auth-reuse invalidation; it does not provide the requested global auth-switch/session-refresh action across all active channel sessions with preserved context.

## Best Possible Solution

Keep this issue as the canonical tracker for a core gateway capability: add an explicit global session auth-refresh/reset flow, likely as an additive gateway protocol method or scoped reset parameter, that refreshes auth-backed runtime state for all active sessions while preserving transcripts/context. Reuse the existing per-session soft reset and CLI auth-epoch invalidation pieces, but add multi-session protocol, server behavior, channel/session-store tests, and docs before closing.
