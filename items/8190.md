## Summary

Keep #8190 open. Current main still exposes only single-session reset and current-session soft reset paths; it does not provide a global gateway/API/chat action that refreshes auth for all active sessions while preserving conversation context. Related #7533 is closed and covers gateway-level auth/account switching, not the remaining all-session auth refresh capability.

## Best Possible Solution

Keep #8190 open as the canonical tracker. The best implementation is an additive gateway/session capability, such as `sessions.refreshAuth` or an explicit all-sessions auth-refresh mode, that enumerates active sessions, invalidates auth-backed runtime and CLI caches, preserves transcripts and channel/session context where possible, and adds protocol, gateway, session-store/channel tests plus user-facing docs. Related #7533 should remain treated as shipped auth/account switching, not as a superseding fix for this remaining global session refresh work.
