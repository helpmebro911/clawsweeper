## Summary

Keep open. Current `main` still routes completion announces through long direct gateway calls with transient retry/backoff before queue fallback, so the primary gateway-destabilization report is not proven fixed. The comment-thread's adjacent `kind=cli` zombie-task concerns are partly improved on `main`, but those changes do not resolve the announce-timeout path described here.
