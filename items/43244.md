## Summary

Keep open. Current main still has unqueued gateway config write handlers, so this PR is still a relevant implementation candidate for the gateway RPC lost-update race. It needs rebase/review before landing because current main has additional config semantics and the submitted PR still has unresolved review feedback.

## Best Possible Solution

Keep this PR open as a still-relevant implementation candidate. The best maintainer path is to rebase or reimplement the config-write serialization on current main, preserve current config handler semantics, add race coverage for all three gateway write methods, address the queue-rejection review feedback, and then land it as the gateway-side fix for the lost-update race. The broader #64973 report should remain separately tracked until verified.
