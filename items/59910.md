## Summary

Current `main` still appears to allow the reported Slack -> Anthropic size mismatch. Slack inbound media is accepted/stored up to 20MB by default, then later image loading for model input still uses the shared 6MB `MAX_IMAGE_BYTES` cap rather than Anthropic's 5MB limit, with no provider-aware clamp or Slack/Anthropic regression coverage visible in the inspected tree.

## Evidence

- **Shared image cap is still 6MB:** `MAX_IMAGE_BYTES` remains `6 * 1024 * 1024`, so the general image-loading path still targets 6MB, not Anthropic's 5MB limit.
  - file: [src/media/constants.ts:1](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/media/constants.ts#L1)
  - command: `nl -ba src/media/constants.ts | sed -n '1,40p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Media loader trusts explicit caps and otherwise uses the shared image cap:** `loadWebMedia` uses the caller-provided `maxBytes` when present, otherwise falls back to `maxBytesForKind(...)`; image optimization/clamping therefore stays tied to the 6MB shared cap unless a caller overrides it.
  - file: [src/media/web-media.ts:447](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/media/web-media.ts#L447)

  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Slack inbound attachments still default to 20MB:** Slack monitor runtime sets `mediaMaxBytes` to `(opts.mediaMaxMb ?? slackCfg.mediaMaxMb ?? 20) * 1024 * 1024`. The public Slack docs also still say inbound attachment cap defaults to 20MB.
  - file: [extensions/slack/src/monitor/provider.ts:184](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/slack/src/monitor/provider.ts#L184)

- **Slack stores inbound files using that larger cap:** Slack media download passes `params.maxBytes` into both `fetchRemoteMedia(...)` and `saveMediaBuffer(...)`, so a 5-6MB image still survives Slack ingest when the channel cap is 20MB.
  - file: [extensions/slack/src/monitor/media.ts:247](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/slack/src/monitor/media.ts#L247)
