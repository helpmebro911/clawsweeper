## Summary

Keep this PR open. Current main already covers the chat-control and markdown-sidebar aria-label pieces with regression coverage, but the PR still contains unresolved accessibility behavior: its grouped tool disclosure label would replace contextual accessible text, and the streaming `aria-busy` change is not present on main and needs explicit screen-reader validation. The same author also has paired open accessibility work in #67432, so closing only this PR would be premature.

## Best Possible Solution

Keep this PR open for scoped contributor follow-up: rebase it on current main, drop the already-shipped chat-control and markdown-sidebar label changes, preserve contextual accessible names for grouped tool disclosures or append state without overriding context, and split or explicitly validate the streaming `aria-busy` behavior with a targeted screen-reader test plan. Coordinate the remaining accessibility work with #67432 rather than closing only one half of the same-author sweep.
