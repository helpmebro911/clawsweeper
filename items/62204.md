## Summary

Keep open. Current main does not contain the PR's env-filtering change, and the reported failure path is still plausible: exec host commands sanitize inherited process.env through the host env security policy, that policy still does not block OPENCLAW_SERVICE_* or OPENCLAW_SYSTEMD_UNIT, and gateway credential resolution still treats OPENCLAW_SERVICE_KIND=gateway as service runtime state.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. The best code path is still a narrow host-env policy change that strips inherited OPENCLAW_SERVICE_* and OPENCLAW_SYSTEMD_UNIT from exec-spawned host child processes, keeps OPENCLAW_SHELL=exec, updates the generated Swift policy, and lands with the focused regression test from the PR or an equivalent test on current main.
