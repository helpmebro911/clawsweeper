## Summary

Current main still does not provide an upgrade-safe Control UI/WebChat chat message max-width setting. The chat group width remains hard-coded in CSS, message rendering has no config-derived style hook, runtime bootstrap config does not carry a layout value, and the public typed/validated config/docs expose only existing Control UI security/hosting settings plus WebChat history-size tuning.

## Best Possible Solution

Keep this issue open. Implement a narrow Control UI chat layout setting, likely either gateway.controlUi.chatMessageMaxWidth or a browser-local Appearance field, validate/sanitize the allowed CSS length expression, pass it through the Control UI bootstrap or UI settings state, set a root or shell CSS custom property, update ui/src/styles/chat/grouped.css to use a safe var(..., min(900px, 68%)) fallback, and cover the schema/bootstrap/CSS/docs behavior with focused tests.
