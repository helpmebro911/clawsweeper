## Summary

Keep open. Current `main` still clears followup/command queues during `/new` reset without draining the per-session system-events map, while the reply path still drains queued system events into the next prompt for the same session key. The latest release `v2026.4.23` shows the same code path, and the issue discussion already has two linked open fix PRs (#66868, #67049), so this is neither stale nor already implemented.
