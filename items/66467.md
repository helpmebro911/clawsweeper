## Summary

Keep this issue open. Current main still pins the bundled ACPX/OpenClaw ACP path to acpx 0.6.1 and @agentclientprotocol/sdk 0.20.0, whose published session/update schema requires usage_update.used and size to be numbers. A used: null notification can still be rejected by SDK validation before OpenClaw/acpx runtime projection can degrade it into a best-effort status event.

## Best Possible Solution

Fix the ACPX/OpenClaw ACP runtime boundary, preferably by updating or pinning ACPX/@agentclientprotocol/sdk to a schema that accepts nullable or absent usage fields, or by normalizing/dropping nullable usage_update fields before SDK validation rejects the notification. Add focused regression coverage for a session/update payload with sessionUpdate: "usage_update", used: null, and numeric size so a Discord-bound ACP turn remains stable and projects a best-effort status event.
