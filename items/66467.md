## Summary

Keep this issue open. Current main still uses the ACPX/runtime dependency combination that rejects `usage_update.used = null` during SDK `session/update` validation before OpenClaw's ACPX projection layer can treat usage fields as optional.

## Best Possible Solution

Fix the ACPX/OpenClaw ACP runtime boundary: update or pin acpx and @agentclientprotocol/sdk to compatible behavior, or normalize/drop nullable `usage_update` fields before SDK validation rejects the notification. Add regression coverage for a `session/update` payload with `sessionUpdate: "usage_update"`, `used: null`, and numeric `size`, ideally through the ACPX runtime path used by Discord-bound sessions.
