## Summary

Keep open. Current `main` still has the old frontmatter descriptions for `gemini`, `notion`, and `spotify-player`, and the runtime explicitly tells the model to choose skills from those `description` entries before loading any skill body. The ClawHub policy in `VISION.md` is scoped to new skills, while this PR edits already bundled skills, so there is not strong evidence for a conservative close.
