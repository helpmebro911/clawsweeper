## Summary

Keep open. Current main still has the short bundled `gemini`, `notion`, and `spotify-player` frontmatter descriptions that #44329 proposes to improve, and those descriptions are still used as the model-facing skill-selection surface. The ClawHub/new-skill policy does not cleanly apply because this PR edits existing bundled skills, and the related broader issue #43410 remains open while the similar PR #40453 covers different skills.

## Best Possible Solution

Keep #44329 open for maintainer review or fold it into the broader #43410 skill-description cleanup. If accepted, update the three bundled `SKILL.md` descriptions with concise trigger-oriented wording, and consider a small frontmatter/style test or review checklist so future bundled skill descriptions stay useful without growing too long.
