## Summary

Close as `clawhub`. Current `main` already exposes sandbox backends as a plugin seam, the existing OpenShell backend is implemented as a plugin using that seam, and the repo vision/docs explicitly steer optional integrations like this to external plugins published via ClawHub or npm instead of adding them to core/bundled scope by default.

## Evidence

- **Vision prefers optional capability as plugins, not core:** `VISION.md` says OpenClaw is slimming core while expanding plugins, that plugin authors should host and maintain plugins in their own repositories, and that the bar for adding optional plugins to core is intentionally high.
