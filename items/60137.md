## Summary

Current `main` still lacks the PR's two behavior changes: bare Slack `slack_webapi_request_error` wrappers with empty/no useful message are not classified as transient, and `undefined`/`null` unhandled rejection reasons still fall through to the fatal exit path. Given the code on `main`, the unchanged tests/release surface, and the prior discussion citing real reproductions, this PR still addresses a live gap rather than something already implemented or obsolete.

## Evidence

- **Transient classifier still misses empty Slack wrapper cases:** `isTransientNetworkError()` immediately returns `false` for falsy inputs and skips candidates with an empty normalized message; there is no special-case handling for Slack's `slack_webapi_request_error` wrapper code or wrapper-prefix-only messages.
  - file: [src/infra/unhandled-rejections.ts:231](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/infra/unhandled-rejections.ts#L231)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Unhandled rejection handler still exits on unmatched reasons:** `installUnhandledRejectionHandler()` checks handled/abort/fatal/config/transient cases, then logs `Unhandled promise rejection` and exits. There is no `reason == null` guard before the fatal fallback, so `undefined`/`null` reasons still reach the exit path.
  - file: [src/infra/unhandled-rejections.ts:349](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/infra/unhandled-rejections.ts#L349)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Current tests still encode the old Slack-wrapper behavior:** The unit test suite on `main` still asserts that a Slack error with message `A request error occurred` and code `slack_webapi_request_error` is non-transient.
  - file: [src/infra/unhandled-rejections.test.ts:171](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/infra/unhandled-rejections.test.ts#L171)

- **Current tests still treat null/undefined-style inputs as non-transient:** The unit tests explicitly include `null` and `undefined` in the `returns false for non-network input` table, which matches the current implementation and shows the PR's added coverage is not already present on `main`.
  - file: [src/infra/unhandled-rejections.test.ts:183](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/infra/unhandled-rejections.test.ts#L183)
