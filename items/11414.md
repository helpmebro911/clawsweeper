## Summary

Keep #11414 open. Current main still supports Telegram forum topic create/edit only; the requested close, reopen, and delete lifecycle actions are absent from the generic action list, Telegram adapter/runtime, send helper surface, config schema, docs, and tests. The related PR #41410 is open and deletion-only, so it does not supersede this dedupe-parent issue.

## Best Possible Solution

Keep #11414 as the canonical tracker until bundled Telegram topic lifecycle support is complete. The likely implementation path is to add generic topic-close, topic-reopen, and topic-delete action names; map and advertise them in the Telegram message action adapter; implement runtime/send helpers for Telegram closeForumTopic, reopenForumTopic, and deleteForumTopic; add config/schema/docs coverage; and extend Telegram action contract/runtime/send tests. PR #41410 can be considered for deletion, but close and reopen still need first-class support.
