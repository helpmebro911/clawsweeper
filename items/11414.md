## Summary

Keep #11414 open. Current main supports Telegram forum topic creation and editing, but the requested close, reopen, and delete topic lifecycle actions are still absent from the shared message action surface, Telegram action discovery/map, runtime dispatch, transport exports, config schema, docs, and tests. The related PR #41410 is useful prior art for deletion only and is still open/unmerged in the provided context, so it does not supersede this dedupe-parent tracker.

## Best Possible Solution

Keep #11414 open as the canonical Telegram forum topic lifecycle tracker. A complete fix should add close, reopen, and delete forum topic support through shared message action names, Telegram action map/discovery, runtime dispatch, transport/runtime helper exports, config schema/metadata, public Telegram docs, and focused Telegram contract/runtime/transport tests. PR #41410 can be used as prior art for deletion, but it should either be completed with the remaining lifecycle actions or linked from this parent as a partial follow-up.
