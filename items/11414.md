## Summary

Keep #11414 open. Current main has Telegram forum topic creation and editing, but not the requested close, reopen, or delete lifecycle actions. The related PR #41410 remains open in the provided context and only addresses deletion, so it does not supersede this dedupe-parent tracker.

## Best Possible Solution

Keep #11414 open as the canonical Telegram forum topic lifecycle tracker. A complete fix should add close, reopen, and delete topic support through shared message action names, Telegram action map/discovery, runtime dispatch, transport/runtime helper exports, config schema/metadata, docs, and focused Telegram contract/runtime tests. PR #41410 can remain useful prior art for deletion, but it is not a complete replacement for this issue.
