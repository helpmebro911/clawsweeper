## Summary

Keep open. The request is a fresh, plausible Feishu bundled-plugin configuration request, not stale, not maintainer-authored, and not protected-label gated. Current public Feishu docs list quota knobs for typing indicators and sender-name resolution, but no probe TTL setting. ClawHub has community workaround skills that patch Feishu probe TTLs, which supports user demand, but those workarounds mutate installed/source files and do not replace an upstream configurable option.

## Evidence

- **Issue requests configurable Feishu probe TTL:** Issue #58755 asks to increase the success-probe TTL and add a `channels.feishu.probeTTL` setting because the current Feishu probe cache is too short for multi-account deployments.
- **Current Feishu docs expose other quota knobs, not probe TTL:** The Feishu docs' quota optimization section documents `typingIndicator` and `resolveSenderNames`; the configuration reference lists Feishu settings through `resolveSenderNames` without a `probeTTL` or equivalent probe-cache option.
