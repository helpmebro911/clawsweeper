## Summary

Current main still has the reported failure mode. The Gemini CLI local OAuth path only prints the authorization URL if ctx.openUrl throws, while the host openUrl adapter discards the boolean failure result from the browser launcher. A directly linked PR, #71469, tracks the exact fix and adds a regression test, but it is still open and unmerged, so this issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #70964 open until PR #71469 or an equivalent fix lands. The best code path is to print the Gemini OAuth URL before best-effort browser launch and add or keep a regression test covering a resolved openUrl call where the local callback flow still needs the URL visible to the user.
