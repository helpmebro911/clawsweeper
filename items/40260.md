## Summary

Current `main` already handles Discord forum/media thread creation for `message action=thread-create`: it detects forum-like parents, sends the required `message.content` starter payload, forwards optional `appliedTags`, and covers the behavior with focused tests. The implementation landed after the latest tagged release, so this looks fixed on `main` but not yet shipped in `v2026.4.22`.
