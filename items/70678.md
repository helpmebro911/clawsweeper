## Summary

Keep open. Current main still contains the WhatsApp extension-local watchdog that treats missing inbound messages as stale liveness and force-closes the socket with status 499 after the configured/default 30-minute timeout. The newer gateway health policy no longer relies on app-level events when no transport activity is reported, but that does not remove the WhatsApp-local watchdog. The issue is also marked as the dedupe parent, has an open fixing PR (#71466), and a maintainer comment explicitly confirmed the report is still valid and should remain open.
