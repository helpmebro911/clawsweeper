## Summary

Keep open. Current main at ffa84cdc025b3396d5044e322fbe6ff27cbeaf22 still does not implement caller-provided cron job ids or duplicate-id validation: the create contract excludes id, createJob always assigns a UUID, cron.add schema/CLI do not accept an id, and add() pushes without a duplicate guard. The PR addresses a real core cron/API gap, but should be rebased, narrowed, and hardened before maintainer landing.

## Best Possible Solution

Keep the PR open and converge it into a focused core cron contract change: add optional stable id support to CronJobCreate/createJob, preserve it through normalization, Gateway validation, CLI --id, cron-tool flat-param recovery, and Swift protocol models; reject duplicate ids case-insensitively under the add lock with a string guard for malformed persisted rows; document the CLI/API behavior and keep targeted regression tests. Drop unrelated secret-scanning and fixture changes from this PR.
