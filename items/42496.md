## Summary

Keep open. Current `main` does not implement this PR's runtime-port discovery for TUI connections: the TUI still resolves its Gateway URL from env/config/default only, and the gateway lock payload on `main` still does not persist the active runtime port.

## Evidence

- **TUI still resolves connection without a lock-file port override:** `resolveGatewayConnection()` loads config/env and immediately calls `buildGatewayConnectionDetails(...)`; there is no `applyGatewayRuntimePortEnvOverride()` or equivalent pre-step in the current file.
  - file: [src/tui/gateway-chat.ts:249](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/tui/gateway-chat.ts#L249)
  - command: `nl -ba src/tui/gateway-chat.ts | sed -n '249,271p'`
  - sha: [d29eaeafc1fc](https://github.com/openclaw/openclaw/commit/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132)
- **Gateway URL still derives local port from `resolveGatewayPort`:** `buildGatewayConnectionDetailsWithResolvers()` computes `localPort` from `resolveGatewayPort(...)` and then builds `ws://127.0.0.1:${localPort}`. There is no lock-file lookup in this path on `main`.
  - file: [src/gateway/connection-details.ts:37](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/gateway/connection-details.ts#L37)
  - command: `nl -ba src/gateway/connection-details.ts | sed -n '34,43p'`
  - sha: [d29eaeafc1fc](https://github.com/openclaw/openclaw/commit/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132)
- **Port resolution still only uses env/config/default:** `resolveGatewayPort()` checks `OPENCLAW_GATEWAY_PORT`, then `cfg.gateway.port`, then falls back to `DEFAULT_GATEWAY_PORT` (`18789`). It does not read runtime lock state.
  - file: [src/config/paths.ts:285](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/config/paths.ts#L285)
  - command: `nl -ba src/config/paths.ts | sed -n '285,302p'`
  - sha: [d29eaeafc1fc](https://github.com/openclaw/openclaw/commit/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132)
- **Gateway lock payload on `main` has no `port` field:** `LockPayload` and `LockPayloadSchema` contain `pid`, `createdAt`, `configPath`, and optional `startTime` only; `acquireGatewayLock()` writes that payload without persisting the active port.
  - file: [src/infra/gateway-lock.ts:18](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/infra/gateway-lock.ts#L18)
