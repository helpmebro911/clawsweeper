## Summary

Keep open. Current main implements `agents.defaults.compaction.model` for compaction summarization, including local-model use, but pre-compaction memory flush still routes through the active run's provider/model fallback path and startup workspace files are loaded into prompt context rather than processed by a dedicated housekeeping model.

## Best Possible Solution

Keep this issue open and narrow the implementation target. Add a dedicated maintenance or memory-flush model override in core config, route `runMemoryFlushIfNeeded` through it, and cover schema/docs/tests for provider/model parsing, auth-profile behavior, fallback, local Ollama-style models, and memory tool permissions. Treat startup-file preprocessing as a separate maintainer decision unless the same maintenance-model contract is explicitly extended to it.
