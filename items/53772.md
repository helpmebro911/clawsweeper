## Summary

Keep open. Current main implements `agents.defaults.compaction.model` for compaction summarization, including local-model use, but it does not route the pre-compaction memory-flush turn, startup-context reads, or a broader housekeeping class through a separate model override.

## Best Possible Solution

Keep this issue open and narrow the implementation target. The likely core path is to extend the existing compaction model-routing design or add a dedicated `maintenance`/`memoryFlush` model override, route `runMemoryFlushIfNeeded` through that override, and add schema/docs/tests. For startup reads, maintainers should explicitly decide whether they remain active-prompt injection with existing `contextInjection`, `bootstrapMaxChars`, and `startupContext` controls, or become a separate summarized-startup-context feature.
