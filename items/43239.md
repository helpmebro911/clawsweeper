## Summary

Keep this PR open for maintainer review/requested changes. Current main still does not thread a caller-provided filename through the generic message send pipeline into Feishu media sends, so the core problem is real; however, the PR remains too broad and unsafe to merge as-is because it mixes the Feishu fix with unrelated repository/CI/local-script/skill changes and review-flagged Feishu media regressions.

## Best Possible Solution

Keep this PR open only as an implementation candidate and request a narrow update or replacement: thread `filename` from message tool params through `MessageSendParams`, outbound payload planning, `ChannelOutboundContext`, and Feishu `sendMediaFeishu`; preserve `loadWebMedia`/`mediaLocalRoots` and `sanitizeFileNameForUpload`; add focused tests for explicit filename forwarding, local basename fallback, remote non-image URLs, local-root rejection, and upload sanitization; drop all unrelated workflow, CODEOWNERS, BuildTools/BuildDocument, skills, `uv.lock`, and OpenAI websocket changes.
