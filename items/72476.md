## Summary

Keep this PR open. Current main still merges configured `agents.defaults.models[model].params.thinkingBudgets` into effective extra params but does not forward that field into pi-ai `SimpleStreamOptions`, while the pinned pi-ai contract and OpenClaw transport code already consume `thinkingBudgets`. The PR is a focused implementation candidate, not obsolete or superseded, but it should tighten validation so non-positive or fractional token budgets are not forwarded.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to update the helper to accept only positive integer token budgets, keep the regression proving configured budgets reach pi-ai stream options, then run the targeted extra-params test and the changed gate before merge.
