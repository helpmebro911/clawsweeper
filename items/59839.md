## Summary

Current `main` still makes the Control UI/WebChat New Session action send `/new`, which rotates the active session under the same session key and archives the prior transcript instead of exposing it as a separate switchable session. The session picker only renders `sessions.list` rows, so archived reset history is still not recoverable from the UI the way the issue describes. The newer #29683 canonicalization fix is present, so the commenter’s earlier workspace theory does not resolve this report.
