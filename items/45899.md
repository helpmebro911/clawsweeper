## Summary

Keep open. Current main partially covers the docs side with broader outbound MEDIA local-path policy, but it still does not implement the PR's Feishu runtime behavior: blocked local-media send failures only reach the generic Feishu reply error handler, with no allowed-root hint and no one-time plain-text fallback.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main: retain or adapt the newer outbound-media docs, add Feishu-specific LocalMediaAccessError/path-not-allowed detection in the reply dispatcher, send the one-time final-reply plain-text fallback, and cover final/non-final/guard-reset cases in extensions/feishu/src/reply-dispatcher.test.ts. Coordinate with #51375 so valid local paths are passed through mediaLocalRoots rather than falling into the fallback unnecessarily.
