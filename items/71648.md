## Summary

Keep PR #71648 open. Current main still has the unbounded MCP channel-bridge pending maps described by the PR and linked issue #71646: `pendingClaudePermissions` and `pendingApprovals` are stored without TTL/sweeper, are not cleared by `close()`, and are only removed on narrow success paths. The PR is recent, active, linked to a concrete open bug report, includes focused tests in the provided diff, and the Greptile review thread shows a real follow-up issue was addressed in head commit `9f16dd4823547354c6d7f85b0dab7fe672008010`. This is not obsolete, implemented on main, or superseded.

## Best Possible Solution

Keep PR #71648 open for maintainer review. The best path is to land this TTL sweeper, close-clear, post-close guard, and regression-test coverage or an equivalent fix on main, then handle the separate FIFO/cap concern and any session-binding interaction with #56420 as follow-up work.
