## Summary

Keep this PR open. Current main still has the MCP channel bridge leak pattern the PR targets: `pendingClaudePermissions` and `pendingApprovals` are raw Maps with no TTL/sweeper, no close-clear, and only conditional deletion paths. The PR is paired with still-open same-author issue #71646 and remains a focused implementation candidate rather than something to close as stale, duplicate, or already implemented.

## Best Possible Solution

Keep this PR open for maintainer review and normal validation. The best path is to land a focused MCP bridge fix that wraps pending map entries with stable timestamps, adds a lazy self-terminating TTL sweeper, clears both pending maps on close, guards post-close writes, and leaves cap/FIFO hardening as a separate narrower follow-up if maintainers want it.
