## Summary

Current main still duplicates the subagent task in both the subagent system prompt and first child user message, so the linked bug is not implemented on main. PR #72028 is an active focused fix attempt, but its user-message-only task binding has a security/product tradeoff already flagged in discussion, and open PR #72053 tracks the same #72019 fix with the safer system-bound-task direction. This should remain a maintainer choice until a canonical PR lands.

## Best Possible Solution

Keep this open for maintainer review unless #72053 or an equivalent canonical fix is selected and merged. The best product direction is to store the concrete subagent task in a privileged/system-bound prompt or immutable runtime metadata, make the first user turn a short kickoff that does not repeat the task, preserve multiline task text verbatim, and keep regression coverage for both the system prompt and spawned child payload. After that lands, close duplicate PRs and the linked #72019 bug with the shipped commit.
