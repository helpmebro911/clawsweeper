## Summary

Keep open. Current main still duplicates the native `sessions_spawn` task text in both the subagent system prompt and the initial child message, so the linked bug is not already fixed. This PR is an active focused implementation candidate, but the PR discussion still has a real design question around where the canonical task should live for persistent `mode="session"` subagents; overlapping #72053 remains open and unmerged, so maintainers need to choose or consolidate the final approach.

## Best Possible Solution

Keep this PR open for maintainer review and consolidate the competing #72019 fixes. The final implementation should keep the long delegated task in exactly one authoritative model-visible location, preserve multiline and structured task text, avoid duplicating it in the initial provider payload, and include regression coverage for the actual `agent` call payload. If #72053 or another equivalent fix lands first, close this PR as superseded by that shipped commit; otherwise review whether this PR should add persistent-session task durability before landing.
