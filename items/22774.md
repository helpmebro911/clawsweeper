## Summary

Keep #22774 open. Current main has partial `senderLabel` rendering and limited inter-session provenance, but it still lacks the durable sender attribution contract and Chat UI identity rendering needed to distinguish human, main-agent, and sub-agent messages across send, steer, transcript/history, and announce flows.

## Best Possible Solution

Keep the issue open. The best path is an additive core sender attribution contract carrying durable sender id, sender kind, display name, source session/agent, and renderable identity through `chat.send`, `sessions.send`, `sessions.steer`, agent input provenance, and sub-agent announce delivery. Persist it in transcript/history events, expose it through `chat.history` and live chat updates, then update the Chat UI to render distinct human/main-agent/sub-agent identities with tests for main-to-sub-agent steer and sub-agent-to-main announce flows.
