## Summary

Current main has partial user sender-label support, but it does not implement the requested multi-role Chat UI attribution for main-agent versus sub-agent messages. The linked PRs are closed and unmerged, and the later related PR explicitly described itself as an unblocker rather than a direct fix for #22774.

## Evidence

- **Sub-agent workflow remains a first-class core surface:** The docs describe sub-agents as isolated background runs, expose /subagents send and /subagents steer, and state that completion handoffs are runtime-generated internal context sent back to the requester session. That is the exact workflow the issue asks the Chat UI to attribute clearly.
  - file: [docs/tools/subagents.md:10](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/docs/tools/subagents.md#L10)

There is no origin role, source agent label, senderLabel, or avatar metadata in that handoff.
