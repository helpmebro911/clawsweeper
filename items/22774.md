## Summary

Keep #22774 open. Current main still only has partial sender-label display plus trace-oriented provenance; it does not provide durable sender attribution for human/main-agent/sub-agent identities across send, steer, announce, transcript/history, and Chat UI rendering. The related PRs were closed unmerged, and #58110 is a broader multi-agent chat UX request rather than a canonical replacement for this narrower attribution contract.

## Best Possible Solution

Keep this issue open as the specific tracker for durable multi-agent sender attribution. The best implementation is an additive sender attribution contract carrying sender id, sender kind, display name, source session/agent, and renderable identity through chat.send, sessions.send, sessions.steer, agent input/announce delivery, transcript persistence, chat.history, and live chat updates. Then update the Control UI to render distinct human, main-agent, and sub-agent identities, with regression coverage for main-to-sub-agent steer, sub-agent-to-main announce, nested announce, and history reload. Coordinate with #58110 only as broader multi-agent chat UX work; it does not supersede this narrower attribution contract.
