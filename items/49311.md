## Summary

Keep open. Current main partially addresses the gateway install side by parsing existing LaunchAgent EnvironmentVariables and preserving safe custom values when that existing environment is passed into the install plan. However, the doctor repair path still rebuilds the service plan without passing the existing service environment, so `doctor --repair` can still rewrite the LaunchAgent without custom plist-only variables such as `ANTHROPIC_API_KEY`.
