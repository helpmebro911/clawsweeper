## Summary

Keep this PR open. Current main still does not implement gateway-side available-provider narrowing for `models.list`, and the open adjacent PRs are not merged canonical replacements. The PR is plausible but needs auth-parity fixes before maintainer review can land it.

## Best Possible Solution

Keep this PR open for maintainer review or fold it into an explicit canonical successor. The best implementation is a shared gateway/model-picker provider availability path that preserves the full-catalog fallback for unconfigured installs, reuses the current auth availability semantics, and adds focused regression coverage for Bedrock AWS SDK credentials, synthetic/local providers, gateway `models.list`, and UI/TUI consumers.

## Risks / Open Questions

- The submitted patch can hide usable Bedrock models for users authenticated through AWS SDK credentials when another provider makes the filtered path active.
- The submitted patch can hide usable local or synthetic-auth providers because its helper does not match current `resolveApiKeyForProvider` and `hasAvailableAuthForProvider` semantics.
- There are overlapping open model selector/catalog PRs (#59693 and #64953), so maintainers should choose a canonical behavior before landing overlapping gateway filtering changes.
