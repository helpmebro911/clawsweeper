## Summary

Keep PR #39244 open. Current main still lacks the requested `tools.exec.nsSandbox` / bubblewrap implementation for safeBins-approved gateway exec commands. Existing safeBins code provides trusted-path and argv-policy validation, and existing sandbox backends cover full sandbox runtimes, but neither implements the PR’s safeBins-only Linux namespace filesystem isolation.

## Best Possible Solution

Keep PR #39244 open for maintainer/security judgment. The best implementation path is a narrowed, rebased optional Linux bwrap layer for safeBins-approved gateway exec, wired through config types, strict config schema/help/docs, per-agent defaults, gateway allowlist and approval follow-up execution, runtime spawning, and regression tests. If maintainers reject core bwrap support, close only with an explicit product/security rationale and a documented alternative using current sandbox backends.
