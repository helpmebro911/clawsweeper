## Summary

Keep open. Current `main` carries subagent session `label` and terminal `status` in the gateway/session model, but the inspected Control UI surfaces still do not use both fields to render a friendlier finished-state label such as `<task> — completed`.

## Evidence

- **Gateway session rows already expose the needed fields:** `GatewaySessionRow` includes both `label` and terminal `status` values (`running|done|failed|killed|timeout`), and `buildGatewaySessionRow` forwards them into `sessions.list` rows.
  - file: [src/gateway/session-utils.types.ts:16](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/gateway/session-utils.types.ts#L16)
  - command: `nl -ba src/gateway/session-utils.types.ts | sed -n '10,60p'; nl -ba src/gateway/session-utils.ts | sed -n '1384,1428p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Web Control UI uses label/displayName only:** `resolveSessionDisplayName` and `resolveSessionScopedOptionLabel` only choose between `row.label`, `row.displayName`, and key-derived fallbacks; they never append or map `row.status` into a completed/failed/cancelled label.
  - file: [ui/src/ui/chat/session-controls.ts:390](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/ui/src/ui/chat/session-controls.ts#L390)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '390,412p'; nl -ba ui/src/ui/chat/session-controls.ts | sed -n '598,614p'; rg -n "row\.status|status" ui/src/ui/chat/session-controls.ts`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Native macOS Control UI still drops label/status on decode:** The macOS Control UI `GatewaySessionEntryRecord` decodes `displayName` and token fields, but not `label` or `status`, so that surface cannot show the spawned task label or a `completed`/`failed` state even though the gateway provides them.
  - file: [apps/macos/Sources/OpenClaw/SessionData.swift:9](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/apps/macos/Sources/OpenClaw/SessionData.swift#L9)
