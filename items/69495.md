## Summary

Current main still lacks the central behavior from PR #69495: `agents.*.heartbeat.model` remains string-only and heartbeat runs only pass a single `heartbeatModelOverride`, with no heartbeat-specific fallback override. The PR addresses a real gap tracked by #69434/#66232, but its own review comments identify issues that still need author or maintainer follow-up, so it should stay open rather than be auto-closed.

## Best Possible Solution

Keep PR #69495 open as a plausible implementation for the real #69434/#66232 heartbeat fallback gap. The best path is to update this PR or a successor so `heartbeat.model` uses the shared model config union, heartbeat ticks pass primary plus fallback overrides through the reply/run pipeline, explicit `fallbacks: []` is preserved when intended to disable fallback inheritance, docs/schema help/tests cover the semantics, and the existing runWithModelFallback retriable-error behavior is reused.
