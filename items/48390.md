## Summary

Keep open. Current main fixed the stale-event part of the report and has better WhatsApp listener registry/status visibility, but the 440 conflict path still exits as a non-manual channel stop and the gateway restart machinery can start it again. That leaves the core requested behavior, a conflict-stopped state that is not auto-restarted until operator action, not fully implemented. The issue discussion also includes a reporter-confirmed community patch from DarkReitor/openclaw-wa-fix, which supports that this is a real remaining lifecycle gap rather than just a stale report.
