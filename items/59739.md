## Summary

Keep open. This is a plausible feature request for Telegram group chats to share the main DM session, and the current documented behavior still says group chats use isolated non-main session keys while DMs use the main session. The issue is recent, externally authored, has no protected labels, and the discussion includes a positive supporting comment plus a related open cross-reference rather than evidence that the request was resolved.

## Evidence

- **Issue request:** #59739 asks for a per-group Telegram option such as `session: main`/`groupsAsMain` so selected group chats route into the main DM session instead of isolated group sessions.
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Current documented behavior:** Public OpenClaw group/session docs describe DMs as using the main session and group chats as using `agent:<agentId>:<channel>:group:<id>` non-main group session keys, with Telegram topics further isolated by topic.
