## Summary

Keep open. Current main still does not implement the PR's live-tail inbound user-message behavior for TUI --session mode: TUI does not consume transcript session.message events, its chat event contract has no user-message state, and current code only reloads history after terminal chat events. The PR discussion also acknowledged that the patch's chat.send-only approach misses real external channel ingress paths.

## Best Possible Solution

Keep the item open, but revise the approach before merge. The best fix is to make TUI --session subscribe to and render session.message transcript updates, or add a lower-level dispatch/transcript hook that covers real external channel ingress as well as chat.send, with local echo suppression and regression tests for external-channel sessions and internal chat.send.
