## Summary

Keep #71656 open. Current main has meaningful partial fixes for session-corpus noise and scan fairness, and deep promotion ranking is weighted/recency-aware, but the reported flat daily/session ingestion scoring is still present and the broader pre-filtering/deduplication request is not fully implemented. The related #57307 and #68876 context does not supersede the full issue scope.

## Best Possible Solution

Keep the issue open and narrow it into implementation work: add content-type-aware source scoring before daily/session calls to recordShortTermRecalls, extend session-corpus filtering/deduplication for repeated blocks and low-signal test/status content, and add regression coverage that proves session-corpus scans keep advancing across files/days under budget pressure. Cross-link #57307 for the broader importance/time-decay policy, but keep #71656 tracking the corpus filtering and scan-rotation specifics unless maintainers split those into narrower issues.
