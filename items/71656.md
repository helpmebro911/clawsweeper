## Summary

Keep #71656 open. Current main has partial fixes for session-corpus noise and some scan fairness, and deep promotion ranking is weighted/recency-aware, but daily and session ingestion still enter the short-term store with fixed source scores. The remaining request is concrete, recent, and not fully covered by the related scoring issue or corpus-filtering PR.

## Best Possible Solution

Keep the issue open and narrow it into implementation work: add content-type-aware source scoring before `recordShortTermRecalls` for daily and session dreaming ingestion, extend corpus filtering/deduplication for repeated blocks and low-signal status/test chatter, and add regression coverage proving session-corpus scans rotate across files and days under budget pressure. Link the scoring portion to #57307 if maintainers want a canonical scoring-policy thread, but keep #71656 for the corpus filtering and scan-stall specifics.
