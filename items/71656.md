## Summary

Keep open. Current main has meaningful progress on session corpus filtering and scan checkpointing, but the central ingestion-quality problem remains: daily and session corpus signals still enter short-term recall with fixed source scores, and session dedupe is still tied to transcript position rather than repeated content identity.

## Best Possible Solution

Keep this open and narrow the remaining work to `memory-core`: add content-type-aware scoring for daily and session ingestion before `recordShortTermRecalls`, add content-level dedupe for repeated session blocks, and extend budget-rotation regression coverage across multiple large corpus sources. Treat #68876 as a related corpus-noise improvement, not a complete fix for this issue.
