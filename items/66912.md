## Summary

Keep this PR open. Current main still downloads reply-target media without checking whether the replied-to Telegram message was authored by the bot, so the reported bug remains real. The current PR head appears to wire the self-authored guard after the earlier bot review, but it still needs a focused regression test and removal or explanation of unrelated Plugin SDK generated baseline hash churn.

## Best Possible Solution

Keep this PR open and ask for a clean Telegram plugin patch: add or keep the self-authored reply-target guard in resolveReplyMediaForMessage, add regression coverage in extensions/telegram/src/bot.test.ts for reply_to_message.from.id === ctx.me.id with no getFile/media fetch, preserve ordinary reply-media ingestion, and remove or justify the docs/.generated/plugin-sdk-api-baseline.sha256 change.
