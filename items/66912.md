## Summary

Keep open. Current main still lacks the self-authored reply-target guard in Telegram reply-media resolution, while this PR is an active, focused candidate to add it. It should not be closed as obsolete; it needs regression coverage and removal or justification of unrelated generated Plugin SDK baseline churn before maintainer review.

## Best Possible Solution

Keep this PR open and ask for a clean Telegram plugin patch: add the self-authored reply-target guard in resolveReplyMediaForMessage, add regression coverage in extensions/telegram/src/bot.test.ts for reply_to_message.from.id === ctx.me.id with no media fetch, preserve the existing normal reply-media ingestion behavior, and remove or justify the unrelated docs/.generated/plugin-sdk-api-baseline.sha256 change.
