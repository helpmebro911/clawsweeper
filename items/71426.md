## Summary

Keep open. Current main still does not implement the requested `sessions.patch` plugin extension hook: the wire schema is closed to known core fields, the gateway handler applies only core session patch logic, and the plugin SDK exposes adjacent hooks/RPC registration but no session patch action registration or dispatch. This is an in-scope OpenClaw plugin API gap, not a ClawHub-only plugin request.

## Best Possible Solution

Keep this open until OpenClaw either implements an additive, documented plugin session patch seam or explicitly folds the work into a canonical broader RFC. A likely path is adding a namespaced `sessions.patch` extension envelope plus SDK/manifest registration for plugin-owned patch actions, dispatching through the plugin registry before persisting the entry, and covering unknown plugin/action, handler failure, and returned entry patch behavior in gateway/protocol and plugin SDK tests.
