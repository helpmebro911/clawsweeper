## Summary

Keep open. Current main still matches the maintainer-validated bug: ACP `streamTo:"parent"` completion events are mirrored/enqueued through the heartbeat wake path, but ACP spawn wake reasons are not action-priority and the heartbeat prompt builder still only creates event-carrying prompts for exec and cron events. The direct task-delivery mirror path also still sends/mirrors completion without waking the parent session. Related PR #56442 is a plausible fix direction, but it is open and not present on current main, so this issue should remain as the validated regression tracker.

## Best Possible Solution

Keep this issue open until main implements an explicit ACP parent-completion wake/announce path. The likely maintainer path is to adapt #56442's `parentUpdates:"notify"`/subagent-announce approach or otherwise make ACP terminal events action-priority with an event-carrying heartbeat prompt, then add regression coverage for heartbeat-enabled and heartbeat-disabled parent completion and clarify `streamTo:"parent"` docs.
