## Summary

Keep open. Current `main` still matches the reported ACP parent-wake gap: ACP parent-stream wakes are classified as generic `wake`, not action-priority; heartbeat prompt routing still only special-cases exec/cron events; and terminal task mirror delivery still does not request a parent wake. The issue is also explicitly maintainer-triaged as valid.

## Evidence

- **Maintainer triage already marked the report valid:** Issue comment from `steipete` on 2026-04-22 says the report is valid and calls out the same two gaps: `acp:spawn:*` not treated as action-priority, and heartbeat prompt routing only handling exec/cron while ACP terminal events fall through.
- **ACP parent-stream reasons are still not action-priority:** `resolveHeartbeatReasonKind` maps `acp:spawn:*` to `wake`, but `isHeartbeatActionWakeReason` still returns true only for `manual`, `exec-event`, and `hook`. That leaves ACP parent-stream wakes at default priority.
