## Summary

Current main still leaves the reported ACP parent-wake gap open. The stream relay now mirrors ACP progress/completion into the parent session and requests a heartbeat wake, but `acp:spawn:*` is still not action-priority and the heartbeat prompt resolver still only builds event-carrying prompts for exec completions and cron events. Direct task terminal delivery also mirrors the completion into the parent transcript without waking the owner session on the successful direct-send path. Maintainer triage in the issue explicitly marked the report valid and pointed to the same source gaps; the open #56442 design is a plausible implementation direction, but it is not merged on current main.

## Best Possible Solution

Keep #70264 open as the focused bug report. The best fix is to keep `streamTo: "parent"` as lightweight progress relay and add or land an explicit ACP parent completion mode, likely based on #56442's `parentUpdates: "notify"`/subagent-announce direction. If the heartbeat path remains involved, ACP terminal events should become action wakes and `resolveHeartbeatRunPrompt` should build an event-carrying ACP completion prompt. Add regressions for heartbeat-enabled `runtime:"acp", streamTo:"parent"` after `sessions_yield`, heartbeat-disabled completion, and direct delivery/mirror behavior.
