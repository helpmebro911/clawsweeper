## Summary

Keep PR #45344 open. Current main still does not reject explicit unknown agent selectors on the OpenAI-compatible `/v1/chat/completions` and `/v1/responses` paths: the header/model helpers normalize explicit IDs, the HTTP handlers build an agent-scoped session key from that value, and `agentCommandFromIngress` only validates `opts.agentId`, not an agent ID inferred from `sessionKey`. The PR is still a plausible focused gateway security/behavior fix, though it should preserve documented explicit `main` behavior while adding the unknown-ID rejection.

## Best Possible Solution

Keep the PR or an equivalent focused fix open. The best path is to validate explicit agent selections for `/v1/chat/completions` and `/v1/responses` against the configured agent IDs before invoking `agentCommandFromIngress`, return a 400 `invalid_request_error` for unknown IDs, preserve implicit default routing and documented explicit `main`/default-agent behavior, and add regression tests for unknown header and model selectors on both HTTP paths.
