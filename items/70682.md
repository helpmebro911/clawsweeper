## Summary

Keep open. Current main still wires the host read tool through upstream `createReadTool` and only applies OpenClaw's workspace guard when `tools.fs.workspaceOnly` is true; the default remains false. The linked security report #70573 is still open, and this PR is a plausible implementation candidate, but it is XL and bundles unrelated security-sensitive changes that need maintainer/security review or splitting.

## Best Possible Solution

Keep this PR open, but ask for a narrower branch. The best path is to split unrelated fixes, decide the intended privacy default for read-tool isolation, then land a focused change in `src/agents/pi-tools.ts` and `src/agents/pi-tools.read.ts` with regression coverage for cross-agent memory reads, absolute paths, parent traversal, symlink/hardlink escapes, and explicit `workspaceOnly=false` behavior.
