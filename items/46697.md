## Summary

Keep #46697 open. Current main still delivers the A2A announce-step output to the visible target/thread, not the full target reply requested by the PR, and existing tests/docs still describe and assert announce delivery. The PR discussion shows useful iteration after review feedback, but the requested behavior is not implemented on main and there is no merged canonical superseding item in the provided context.

## Best Possible Solution

Keep the PR open, or convert it into a focused follow-up if maintainers prefer a fresh patch. The best implementation would post the final post-ping-pong `latestReply` to the resolved announce target/thread, preserve `threadId`, make the interaction with `ANNOUNCE_SKIP` explicit, avoid an unused announce model run when direct delivery succeeds, keep requester/target provenance coherent, and add targeted tests for announce targets with and without ping-pong.
