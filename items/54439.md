## Summary

Keep open. Current main still has the unguarded Control UI cron payload dereference, the UI loader still trusts raw `cron.list` rows, and the linked focused fixes #54550 and #54552 are closed unmerged with no equivalent regression test found on main.

## Best Possible Solution

Keep this issue open. The best maintainer path is to revive or reimplement the focused #54552 approach on current main: guard Control UI cron payload dereferences, validate or filter malformed `cron.list` rows at the UI boundary, and add a jsdom regression test proving malformed cron entries do not prevent switching back to Chat.
