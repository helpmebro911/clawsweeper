## Summary

Current main still leaves the reported Control UI crash path plausible. `renderApp` still reads `job.payload.kind` from `state.cronJobs` while building cron model suggestions, and the Cron list loader still accepts `cron.list` rows into UI state without runtime shape filtering. The focused fixes linked in the discussion, #54550 and #54552, are closed unmerged.

## Best Possible Solution

Keep this issue open. The best maintainer path is to revive or reimplement the focused #54552 approach on current main: validate or filter malformed `cron.list` rows at the UI and/or server list boundary, guard shared Cron UI helpers before reading `payload.kind`, and add a jsdom/browser regression test proving malformed Cron entries do not prevent switching back to Chat.
