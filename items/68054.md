## Summary

Keep open. Current main has adjacent cron, heartbeat, dreaming, and legacy nano-banana cleanup, but there is still no source/test/history evidence that conclusively fixes the reported user-visible behavior: unexpected image sends on OpenClaw 2026.3.2 while `cron list` and the persisted cron store were empty.

## Best Possible Solution

Keep this open for maintainer diagnosis. Ask the reporter to retry on a current release/current main and provide a redacted bundle: `openclaw version`, `openclaw cron status`, `openclaw cron list --all --json`, `openclaw cron runs` around the suspected times, `openclaw system heartbeat last`, heartbeat config/status, `/dreaming status` or `openclaw memory status --json` if memory-core is enabled, affected channel/plugin, active session/task state, and gateway logs around the unexpected sends. If it reproduces, investigate `src/infra/heartbeat-*`, `src/cron/service/*`, `extensions/memory-core/src/dreaming.ts`, active session/task state, and outbound media/message delivery paths; also confirm whether `openclaw doctor --fix` migrates any legacy nano-banana config.
