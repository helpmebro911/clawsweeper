## Summary

Current main still does not discover the reported grouped managed-skill layout by default. The loader only accepts a skill root or immediate child directories with `SKILL.md`, so `~/.openclaw/skills/coze/koze-retrieval/SKILL.md` remains skipped by the same path used by `openclaw skills list`. Two open PRs explicitly target this issue with closing syntax, so the issue should stay open until one canonical fix lands or the PRs are closed.

## Best Possible Solution

Keep this issue open. Review and land one canonical fix that adds bounded grouped managed-skill discovery for `~/.openclaw/skills/<group>/<skill>/SKILL.md` in `src/agents/skills/workspace.ts`, preserves containment and limits, and adds regression coverage in `src/agents/skills.loadworkspaceskillentries.test.ts`; then close or deduplicate the overlapping PRs.
