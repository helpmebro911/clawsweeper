## Summary

Keep this PR open. Current main still resolves the effective OpenClaw home through OPENCLAW_HOME, HOME, USERPROFILE, and os.homedir() only, so the Android/Termux `/home` crash from the linked report remains plausible. The PR is a focused implementation candidate and its second commit addresses the earlier review gap by sharing the Termux fallback with OPENCLAW_HOME tilde expansion.

## Best Possible Solution

Keep this PR open, refresh it against current main if needed, and review or land the focused Termux home-resolution fix with its regression tests. Preserve existing OPENCLAW_HOME, HOME, USERPROFILE, OPENCLAW_STATE_DIR, and OPENCLAW_CONFIG_PATH precedence while tightening the Termux PREFIX guard if maintainers want a stricter path check.
