## Summary

Keep this issue open. Current main still has the reported helper paths using embedded execution directly, and cron still checks CLI provider status before resolving canonical providers to configured CLI runtimes. Same-author PR #72499 is open, unmerged, and contains closing syntax for this issue, so the maintainer path is to review/land or close that PR rather than close this issue now.

## Best Possible Solution

Keep this issue open and review PR #72499, or an equivalent scoped patch, that applies the existing resolveCliRuntimeExecutionProvider(...) ?? provider plus isCliProvider(...) dispatch pattern to queued followups, models status --probe, LLM slug generation, and cron. The accepted fix should preserve runtime config, auth profile, session binding, routing and cancellation context, and include focused regression coverage for those paths before closing the issue.
