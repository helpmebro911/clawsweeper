## Summary

Keep open. Current `main` already flushes buffered assistant text before `state:"final"`, and `v2026.4.23` shipped the same `src/gateway/server-chat.ts` behavior, so I do not have evidence that OpenClaw core currently drops buffered chat deltas. But the reporter’s later logs show the OpenClaw side itself emitted no assistant events before lifecycle end, and the discussion/logs point at a Java/Netty relay outside this repo. That is not strong enough to close as fixed or obsolete; the report still describes a plausible live bug somewhere in the end-to-end stack.
