## Summary

Keep open. Current main still lacks the requested `openclaw config set agents.defaults.model.primary ...` warning when per-agent model overrides are present. The underlying behavior is real and documented: `agents.list[].model` takes precedence over `agents.defaults.model`. The PR is narrow and security-low-risk, but needs rebase/adaptation to current `runConfigSet` and batch-mode plumbing before it can land.

## Best Possible Solution

Keep the PR open and rebase/adapt it onto current main. The implementation should add a small helper near `runConfigSet`, detect any successful non-dry-run operation whose parsed `setPath` is `agents.defaults.model` or `agents.defaults.model.primary`, scan the resulting config for explicit `agents.list[].model` values, and add focused tests for object-style, string-style, bracket notation, no-overrides, unrelated paths, and batch mode. No core model-resolution or schema change is needed.
