## Summary

Keep this PR open. Current main still lacks configurable `sessions_spawn` label templates: the tool reads only the explicit `label`, config schemas do not accept `labelTemplate`, and docs only describe `label` as an optional human-readable parameter. The linked request #72074 was closed as superseded by this active implementation PR, not as shipped. The patch is relevant, but review discussion still identifies requester/target-agent semantics plus missing tests/docs/generated schema follow-through.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. If accepted, implement label templating in the core `sessions_spawn` path with explicit-label precedence, compute `{agentId}` from the resolved spawn target, resolve per-agent template policy from the requester agent with defaults fallback, and update tests, docs, config types, strict schemas, and generated schema artifacts.
