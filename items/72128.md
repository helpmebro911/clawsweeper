## Summary

Keep PR #72128 open. Current main does not implement configurable subagent label templates, and the open PR is the active candidate for the linked request rather than something already shipped or superseded. The PR also has unresolved behavior-review concerns around resolved target-agent labels and requester-scoped subagent config, so it needs maintainer/contributor follow-up instead of conservative cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review or contributor revision. The best path is to implement label templating in the core sessions_spawn path using the resolved target agent for `{agentId}`, read per-agent `subagents.labelTemplate` from the requester-scoped config with defaults fallback, preserve explicit-label precedence, and add focused tests plus docs/config-schema updates. Close only if a separate merged core implementation supersedes it or a maintainer explicitly rejects the product direction.
