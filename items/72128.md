## Summary

Keep #72128 open. Current main does not implement configurable label templates for `sessions_spawn`: labels still come only from the explicit `label` argument, and the proposed `labelTemplate` key is absent from config types, zod schemas, and user docs. The PR is a focused, recent implementation attempt for open issue #72074, but it still needs maintainer review because the provided review comments identify unresolved target/requester semantics for template resolution.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to resolve the target/requester semantics explicitly, add focused tests for explicit-label precedence, default template fallback, per-agent override behavior, omitted-agentId fallback, and repeated token replacement, then update the config docs/generated schema surface before landing against #72074.
