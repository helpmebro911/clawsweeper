## Summary

Keep #17213 open. Current main has useful narrower safeguards, including strict-agentic plan-only recovery, lifecycle end/error fallback, and stale run-context cleanup, but it still does not implement the requested atomic run.begin -> run.exec -> mandatory run.commit/run.abort contract with structured abort blockers across the shared event and gateway protocol surfaces. The provided maintainer discussion also already keeps this issue open, and related PR #40710 was closed unmerged rather than shipped.

## Best Possible Solution

Keep #17213 open as a core reliability/product-contract item. The best path is a maintainer-approved, backward-compatible lifecycle design that defines explicit begin intent, execution evidence, mandatory commit or abort terminal state, and structured abort/blocker details across AgentEvent types, Gateway protocol/schema, agent emitters, plugin or harness-facing surfaces, clients, docs, and regression tests. Treat #40710 and #55721 as useful prior art and narrower fixes, not as superseding this issue.
