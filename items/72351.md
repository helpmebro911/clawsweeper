## Summary

Keep open. Current main still lets `--history-limit 0` reach `chat.history` as `limit: 0`, while the Gateway protocol rejects zero. The PR is a focused TUI-side fix with regression coverage and is not superseded by the adjacent group/channel history-limit items.

## Best Possible Solution

Keep the PR open and land this narrow TUI guard plus regression test after normal maintainer validation. The fix belongs in `src/tui/tui-session-actions.ts`: interpret `historyLimit === 0` locally as skip history loading while preserving the normal local side effects, rather than weakening the Gateway `chat.history` schema where zero remains invalid.
