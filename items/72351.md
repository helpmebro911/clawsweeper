## Summary

Keep this PR open. Current main still accepts `--history-limit 0`, preserves that explicit zero through the TUI session history path, and forwards it unchanged to `chat.history`, while the Gateway protocol rejects `limit: 0`. The PR remains a focused implementation candidate with regression coverage, and the provided review comments support that it is narrow rather than superseded.

## Best Possible Solution

Keep this PR open and either land the narrow TUI-side guard with its regression test or recreate the same behavior on main. The Gateway protocol should continue rejecting `chat.history.limit: 0`; the TUI should interpret explicit `--history-limit 0` locally as no-history mode and avoid sending `chat.history` at all.
