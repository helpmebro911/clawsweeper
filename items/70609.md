## Summary

Keep open. Current main still lacks the generic outbound metadata path this PR is trying to add across gateway send, direct delivery, channel plugin context, and queue recovery. The PR should not be merged as-is, but the remaining work is real core/API plumbing rather than something that can be closed as already implemented, duplicate, or ClawHub-only.

## Best Possible Solution

Keep the PR open, but ask for a narrower patch: add an opaque outbound metadata contract through `sendMessage`, gateway `send`, `deliverOutboundPayloads`, `ChannelOutboundContext`, delivery queue storage, and recovery replay, with focused tests. Split out the unrelated auth commit and move Hermes-specific builders/hardcoded task/media/exec-approval injection into a plugin/config-driven follow-up or ClawHub work unless a maintainer explicitly wants it in core.
