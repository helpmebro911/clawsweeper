## Summary

Current main already ships direct Android/iOS gateway clients, a documented Gateway WebSocket protocol for mobile clients, and device pairing/token management. The latest release v2026.4.22 also contains the mobile-platform docs, so this request is implemented well enough on shipped OpenClaw.

## Evidence

- **Android direct gateway flow is documented:** The Android platform doc explicitly says Android connects directly to the Gateway WebSocket, supports Setup Code and Manual modes, and uses `openclaw devices` approval for pairing.
  - file: [docs/platforms/android.md:24](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/docs/platforms/android.md#L24)

## Risks / Open Questions

- The repo does not appear to expose a separate standalone Kotlin SDK package or REST-first mobile API; this close relies on the shipped mobile apps, shared Swift protocol/client package, documented Gateway WS protocol, and device-management surface satisfying the underlying request.

## Close Comment

Closing as implemented on current OpenClaw. A Codex review of `main` found that OpenClaw now ships direct mobile-to-gateway support rather than only third-party chat-channel access: `docs/platforms/android.md:24` documents direct Android WebSocket connection with Setup Code/Manual pairing, `docs/platforms/ios.md:12` documents the same for iOS, `docs/gateway/protocol.md:10` documents the Gateway WebSocket protocol used by iOS/Android nodes, and `docs/cli/devices.md:9` documents device approval, rotation, and revocation flows. The repo also contains the client implementations (`apps/android/app/src/main/java/ai/openclaw/app/gateway/GatewaySession.kt:281`, `apps/shared/OpenClawKit/Package.swift:5`, `apps/ios/Sources/Chat/IOSGatewayChatTransport.swift:15`), and these docs are already present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still want a separate public Kotlin/Swift SDK package beyond the shipped app/protocol surface, please open a new narrowly scoped issue for that specific gap.
