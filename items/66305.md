## Summary

Keep this PR open. Current main still immediately clears the shared model catalog cache after a successful empty catalog load, and no current test covers brief negative caching. The linked user issue appears narrowed by later model-selection work, but shared catalog surfaces still use `loadModelCatalog()`, so the PR remains a plausible focused fix that needs rebase/porting over current `readOnly` semantics.

## Best Possible Solution

Keep this PR open and review/rebase it, or port the same fix to current main: briefly cache successful empty catalog loads, retry after a bounded window, keep import/filesystem failures retryable, make `useCache: false` and cache resets clear negative-cache state, preserve current `readOnly` non-mutating semantics, guard stale concurrent loads, and keep focused regression tests in `src/agents/model-catalog.test.ts`.
