## Summary

Keep this PR open. Current main still clears the shared model catalog cache after a successful empty catalog load, so the PR's core behavior is not implemented on main. The linked user issue was closed because ordinary MiniMax allowlist-backed turns now skip runtime catalog loading, but that narrower shipped fix does not obsolete this PR's broader shared-cache fix for model catalog surfaces that still call loadModelCatalog().

## Best Possible Solution

Keep this PR open and review/rebase it, or port the equivalent fix to current main: briefly cache successful empty catalog loads, retry after a bounded window, keep import/filesystem failures retryable, make useCache:false and cache resets clear negative-cache state, preserve current readOnly semantics, guard stale concurrent loads, and retain focused regression tests in src/agents/model-catalog.test.ts.
