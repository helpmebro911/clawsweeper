## Summary

Current main still lacks the behavior this PR proposes: the direct Anthropic path marks the trailing user message for cache control, while the OpenRouter helper only handles system/developer prompt blocks and assistant thinking cleanup. The paired bug report remains open, and this PR is still the active implementation candidate rather than stale or superseded cleanup.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or revise the proposed helper change with the included regression coverage, confirm SDK/GitHub Copilot caller impact, and update OpenRouter prompt-caching documentation if conversation-message markers become supported behavior.
