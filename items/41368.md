## Summary

Keep open. Current main still lacks user-facing Feishu Bitable delete tools, while the PR is a focused bundled Feishu plugin change and is broader than the related single-record delete PR/issue. The Lark SDK and Feishu docs support the proposed record, batch-record, and field delete APIs; this should get maintainer review/rebase/polish rather than cleanup closure.

## Best Possible Solution

Rebase and review a Feishu plugin change in extensions/feishu/src/bitable.ts that exposes feishu_bitable_delete_record, feishu_bitable_batch_delete_records, and feishu_bitable_delete_field through the existing registerBitableTool/account-routing pattern. Before landing, preserve current main behavior, fix the schema/response review comments, add targeted tests, and make the destructive nature of these tools explicit enough for maintainer comfort.
