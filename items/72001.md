## Summary

Keep PR #72001 open. Current main still has the Gmail wizard/config-validator gap the PR targets, and the PR is a plausible focused fix with useful discussion, but it likely needs maintainer/code updates before landing because the templated Gmail session key also requires the runtime request-session-key opt-in and there is a valid edge case in the review comment.

## Best Possible Solution

Keep the PR open for maintainer review. The best fix is to make the Gmail setup wizard emit a complete validator- and runtime-valid hooks configuration for the Gmail preset while preserving existing user policy: add/merge the required prefixes only when compatible, handle the allowRequestSessionKey requirement or choose a static-session mapping path, and add regression coverage for startup validation, actual mapped /hooks/gmail dispatch, and the defaultSessionKey/static-shadow edge case raised in review.
