## Summary

Keep open. Current `main` has generic `preferOver` logic for competing auto-enable candidates, but the reported `channels.feishu` + `plugins.entries.openclaw-lark.enabled: true` path is not strongly covered by the in-tree code or tests. `channels.feishu` still resolves to the built-in `feishu` plugin first, and I did not find evidence that merely enabling `openclaw-lark` creates the competing candidate needed to suppress the built-in doctor auto-enable.
