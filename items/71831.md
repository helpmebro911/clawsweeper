## Summary

Keep PR #71831 open. Current main still documents the bare WebSocket runtime fallback in `docs/tools/browser.md`, but it does not mention that `openclaw browser doctor` readiness diagnostics now use the same discovery-first/direct-WebSocket fallback. The current implementation and tests support the PR's docs-only sentence, and the provided maintainer discussion already confirms the patch remains valid.

## Best Possible Solution
