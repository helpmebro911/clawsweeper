## Summary

Keep open. Current main still lacks a Telegram `/settings` inline-keyboard control panel, and this PR is a plausible implementation candidate rather than something already shipped or superseded. It needs a rebase/port to the current bundled plugin layout and another security-focused review because it adds a privileged config-write callback surface.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to port it onto current main under `extensions/telegram`, reuse the current native command and interactive callback structure, preserve `configWrites`, fresh-config loading, DM/topic authorization parity, owner/commands allowlist checks, and add focused tests in the Telegram extension before maintainer review.
