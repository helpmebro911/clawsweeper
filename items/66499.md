## Summary

Keep #66499 open. Current main still has the gateway chat-delta path stripping inline directive tags without normalizing the leading whitespace before merge, while session/history display paths trim the stripped text. The PR is not merged or superseded, the linked bug #66497 remains open, and the review discussion shows this is a plausible focused gateway bug fix that needs a narrower implementation and stronger regression coverage rather than cleanup closure.

## Best Possible Solution

Keep the PR or linked issue #66497 open as the active gateway bug-fix thread. The best implementation is a narrowly scoped normalization in the WebSocket chat-delta merge path that removes only whitespace introduced by leading inline directive tags, preserves intentional user-visible leading/trailing whitespace and delta separators, and adds a regression test through `createAgentEventHandler` or an intentionally exported merge helper for directive-prefixed cumulative text plus incremental deltas.
