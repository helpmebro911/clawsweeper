## Summary

Keep this PR open. Current main still routes Gateway/TUI live assistant output through directive stripping that preserves leading whitespace, then merges snapshots/deltas with overlap dedupe, so the reported corruption path is still plausible. The PR is a useful implementation candidate, and the discussion shows the author incorporated review feedback, but it needs rebasing/adaptation to the current shared live-chat projector and a narrower whitespace fix.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. Rebase it onto current main, move the production change into src/gateway/live-chat-projector.ts, normalize only whitespace introduced by stripped leading directive tags before merging, preserve intentional leading/trailing delta separators, and add a regression test through the shared projector helper or real Gateway event path.

- An unconditional trimStart-style fix can remove intentional leading newlines or indentation when no directive introduced that whitespace; the implementation should normalize only whitespace caused by stripped leading directive tags.
