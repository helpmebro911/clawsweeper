## Summary

Keep this PR open. Current main still has the same risky live-chat merge shape for directive-prefixed assistant streams, but the PR is stale against the current shared projector and should be adapted rather than closed.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, move the production fix into src/gateway/live-chat-projector.ts, normalize only whitespace introduced by stripped leading inline directive tags before merging, preserve intentional leading/trailing delta separators, and add a regression test through the shared projector or Gateway event path proving the xumum/GOOGLE corruption no longer occurs.
