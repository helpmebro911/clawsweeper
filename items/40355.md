## Summary

Keep #40355 open. Current main has partial model-misconfiguration visibility at gateway startup and onboarding, but it still does not implement the PR's requested `openclaw doctor` warning for invalid `agents.defaults.model.primary` or `agents.list[].model`. The linked issue #39811 remains a valid product/DX problem, and the PR discussion shows the author addressed useful review feedback rather than abandoning the change.

## Best Possible Solution

Keep the PR open and ask for a rebase/port to the current doctor health contribution structure. The best implementation is a dedicated doctor health contribution or shared validation helper that checks `agents.defaults.model.primary` and each `agents.list[].model` against the current model catalog, resolves configured aliases the same way runtime selection does, sanitizes warning output, and adds focused regression tests. #39811 should remain the canonical issue for the broader validation/fallback visibility problem.
