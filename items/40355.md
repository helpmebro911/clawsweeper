## Summary

Keep this PR open. Current main still lacks the requested `openclaw doctor` warning for invalid `agents.defaults.model.primary` or `agents.list[].model` primaries, and the linked bug #39811 is still open. The PR is a plausible implementation candidate, but it needs a port to the current doctor health contribution structure and a fresh pass on terminal-safety/catalog-loading behavior before merge.

## Best Possible Solution

Keep this PR open and ask for a current-main port into `src/flows/doctor-health-contributions.ts` or a shared validation helper used by that contribution. The implementation should check `agents.defaults.model.primary` and each `agents.list[].model` primary against the catalog, resolve aliases the same way runtime selection does, sanitize terminal-facing warning text, choose an explicit readOnly/offline catalog-loading behavior, and keep #39811 as the canonical broader tracking issue until the warning ships.

## Risks / Open Questions

- The PR is stale against the current doctor health contribution architecture and likely needs a rebase/port before it can be evaluated for merge.
- The fetched PR diff still prints config-derived warning text without terminal sanitization, despite discussion mentioning a follow-up fix; maintainers should require the final branch diff to include safe output handling.
- Doctor catalog validation needs an explicit side-effect policy. Non-readOnly catalog loading may refresh or write `models.json` and run provider catalog augmentation, which is heavier than a purely advisory config check.
- The final implementation should resolve model aliases consistently with runtime/model-picker selection to avoid false warnings for valid alias-based configs.
