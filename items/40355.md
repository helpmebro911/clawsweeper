## Summary

Keep this PR open. Current main still lacks the requested `openclaw doctor` warning for invalid `agents.defaults.model.primary` and `agents.list[].model` primaries, while the linked bug remains open and this PR is still a plausible implementation candidate. The PR should be refreshed against current main and reviewed for alias-consistent resolution, terminal sanitization, and catalog-loading side effects before merge.

## Best Possible Solution

Keep this PR open and ask for a current-main-ready implementation in the doctor health contribution layer or a shared validation helper used by that layer. The final patch should warn for invalid default and per-agent primary model refs, resolve aliases and shorthand the same way runtime/model-picker code does, sanitize all terminal-facing config-derived text, use an explicit read-only/offline catalog-loading policy for diagnostics, and add focused tests for defaults, per-agent models, aliases, empty catalogs, and terminal sanitization. Keep #39811 as the canonical issue until this behavior ships.
