## Summary

Current main already routes spawned subagents through the standard agent prompt-build pipeline, and OpenClaw documents prompt mutation as a plugin hook concern. This PR adds a new core config knob for behavior that existing hook-only plugins can already implement and scope to subagent sessions, which cuts against the repo's plugin-first direction for optional capability.

## Risks / Open Questions

- There is no bundled example plugin for a file-backed subagent prompt wrapper, so closing this PR may still leave discoverability/documentation work for the plugin path.
- A plugin implementation would need its own tests for subagent-session filtering and file-loading behavior; current main only provides the generic hook seam.

## Close Comment

Closing from a Codex review because this is a better fit for the existing plugin/ClawHub path than a new OpenClaw core config surface. On current `main`, `sessions_spawn` already sends the child task through the normal agent pipeline (`src/agents/subagent-spawn.ts:889`, `src/agents/subagent-spawn.ts:969`), and that pipeline already supports prompt mutation through `before_prompt_build` / legacy `before_agent_start` (`src/agents/pi-embedded-runner/run/attempt.ts:1931`, `src/plugins/hook-before-agent-start.types.ts:21`). The plugin architecture explicitly supports hook-only plugins and says to prefer `before_prompt_build` for prompt mutation work (`docs/plugins/architecture.md:83`, `docs/plugins/architecture.md:101`), and hook context includes `sessionKey` so a plugin can target only subagent sessions (`src/plugins/hook-types.ts:154`). Given `VISION.md:54` says optional capability should usually ship as plugins, this subagent prompt-wrapper behavior belongs as a plugin/ClawHub package rather than another core config knob.
