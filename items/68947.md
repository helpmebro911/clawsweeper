## Summary

PR #68947 should stay open. Current main still has the export-session placeholder regression, so the work is not obsolete, but the PR is not safely closeable or merge-obsolete because the current replacement path can mutate inlined JavaScript containing `$` replacement sequences.

## Best Possible Solution

Keep PR #68947 open and ask for a small follow-up: preserve the restored `{{MARKED_JS}}`, `{{HIGHLIGHT_JS}}`, and `{{JS}}` placeholders with prettier-ignore guards, change `generateHtml()` to insert script assets with function replacers or another raw-text helper, and extend the regression test to read the real template/assets and assert no raw placeholders remain and `$` sequences are preserved. That would directly fix the open #49957 failure without needing a broader redesign.
