## Summary

Keep PR #68774 open. Current main still has the exact gap the PR targets: the dreaming contamination detector only accepts `Candidate:`/`Reflections:` at the start after prefix stripping, and promotion rehydration has no managed dreaming-fence overlap check before appending to `MEMORY.md`. The linked issue #67580 and the PR discussion describe a plausible production leak, and the bot review/comment thread supports the PR as a focused correctness fix with only a deferred P2 performance follow-up.

## Best Possible Solution

Keep #68774 open and have a maintainer merge it or an equivalent patch that broadens the dreaming narrative-lead check, rejects rehydrated ranges inside `openclaw:dreaming:*` fences, and keeps the new regression tests. Keep #67580 as the canonical user-facing bug until that fix lands.
