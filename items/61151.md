## Summary

Keep this PR open. Current main still does not strip `partialJson` from otherwise valid tool-call blocks during session transcript repair, and the PR remains a focused implementation candidate that preserves existing `sessions_spawn` attachment redaction while adding targeted regression coverage.

## Best Possible Solution

Review and either land this focused transcript-repair change or request final adjustments. The best implementation path is to strip `partialJson` from otherwise complete retained tool-call blocks inside `repairToolCallInputs`, continue dropping genuinely incomplete artifacts, and keep `sessions_spawn` attachment redaction in the same fall-through path with regression coverage for both behaviors.
