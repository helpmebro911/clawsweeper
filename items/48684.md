## Summary

Keep open. Current main still has the exact shared debounce behavior reported: `shouldDebounceTextInbound()` calls `hasControlCommand()`, while bare abort phrases are only recognized by the broader abort/control-command path. The cross-referenced PR #48916 describes the right narrow fix, but the provided GitHub context says it was closed unmerged, and the current checkout does not contain that change.
