## Summary

Keep this issue open. Current main still consumes Discord interactive components by the clicked component id, not by panel/message group, so a non-reusable panel with sibling buttons can still deliver a second callback through an unclicked sibling. The broader related #54233 is closed and should not supersede this focused remaining bug.

## Best Possible Solution

Keep this issue open as the focused tracker. Fix it in the Discord plugin component registry/interaction path by assigning a message or panel consumption group when registering component entries, then invalidating all sibling entries in that group when any non-reusable entry is consumed before callback dispatch. Add a regression test with two buttons in one non-reusable component message proving the second sibling click returns the expired response and does not dispatch to the agent.
