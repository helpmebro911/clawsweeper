## Summary

Current main still has completion-focused subagent announce delivery, internal spawn hooks/events, an in-memory announce queue, and opt-in task state-change notifications, but it does not implement the central request: framework-owned immediate requester-channel spawn acknowledgements, native per-run progress lifecycle notifications, or durable announce persistence across gateway restarts/channel outages.

## Best Possible Solution

Keep this open as the umbrella runtime architecture request. The best implementation path is to extend the existing subagent run registry, completion announce flow, and task ledger with runtime-owned lifecycle events for spawned/running/progress/completed/failed/cancelled, an immediate requester-channel spawn acknowledgement, durable persisted announce retry with idempotency keys, and delivery that is not starved by the parent session lane. Coordinate with #22358 and #54488, but do not close this until the broader lifecycle delivery contract ships.
