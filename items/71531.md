## Summary

Current main does not implement `channels.slack.logLevel`. The Slack config schema has no `logLevel` field, the Slack runtime constructs a custom `SocketModeReceiver` and Bolt `App` without any log-level plumbing, and the exact linked PR #71550 is still open/unmerged rather than shipped. The issue remains a concrete, actionable Slack channel RFE.

## Best Possible Solution

Keep #71531 open as the canonical user problem until main ships a Slack log-level config. The implementation should add a validated `channels.slack.logLevel` field, document it, and forward it to the custom `SocketModeReceiver` as well as any Bolt/App or WebClient logging paths needed to suppress socket-mode WARN noise. PR #71550 is the active starting point, but it should be adjusted before merge so the receiver that owns the socket-mode logger actually receives the level.
