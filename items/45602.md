## Summary

Keep PR #45602 open. Current main still shows the original one-Enter `/status` problem path: `/status` is not marked local, Enter in the slash menu only fills the draft for non-local commands, and the existing regression test explicitly expects `/status` to remain on the agent path. The PR/discussion is useful, but the best fix likely needs to decouple "run on menu Enter" from "execute locally" or fully preserve the agent status default-resolution behavior.

## Best Possible Solution

Keep this open, but rework the patch before merge. The safest product direction is to preserve `/status` on the agent/canonical status path and fix the slash-menu UX by allowing exact no-argument commands like `/status` to submit on Enter without requiring `executeLocal`. If maintainers prefer local execution, add a gateway/API surface that returns the same effective status data the agent path uses, including per-session model thinking and fast-mode defaults, then cover it with targeted UI and gateway tests.
