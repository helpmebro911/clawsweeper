## Summary

Keep open. Current `main` does not implement the proposed compaction-cleanup wait, and the PR discussion identifies unresolved correctness issues in the patch as submitted, so there is not strong evidence for a conservative close.

## Evidence

- **Current main still unsubscribes immediately during compaction teardown:** `unsubscribe()` is still synchronous and calls `abortCompaction()` followed immediately by `sessionUnsubscribe()`, so there is no wait for a final compaction-end event to be handled before the listener is removed.

  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Production teardown still treats unsubscribe as synchronous:** Runner cleanup calls `unsubscribe()` inside a synchronous `try/catch` in `finally` and does not await it. That matches the Codex review comment in the PR discussion: making `unsubscribe` async would change teardown semantics and could bypass this catch path.
