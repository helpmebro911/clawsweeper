## Summary

Keep open. Current `main` does implement and document no-prompt exec policy (`tools.exec.ask: "off"` with permissive host policy), but the specific stale-popup failure reported in #59010 is not clearly fixed: the Control UI still preserves pending approvals across reconnects and leaves a queue item in place when resolve fails, which matches the reported `unknown or expired approval id` / popup-flood behavior closely enough that a conservative cleanup should not close it.
