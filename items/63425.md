## Summary

Keep this issue open. Current main still has a source-reachable `/status` path where `channels.modelByChannel` selects the runtime provider/model, but a stale positive `sessionEntry.contextTokens` can remain authoritative in status formatting. The existing ClawSweeper comment matches the current source, and I did not find a targeted regression test or merged source change that closes this channel-override context-token case.

## Best Possible Solution

Keep this issue open and fix the `/status` context-token reconciliation for channel-selected models. A narrow maintainer fix would detect when `channels.modelByChannel` selected the active model without an explicit session `/model` override, then ignore, clear, or recompute stale `sessionEntry.contextTokens` before status formatting. Add a regression test with a stale 1M persisted session context, a channel override to a smaller MiniMax-style model window, and `/status` showing the active model's configured context window.
