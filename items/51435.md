## Close Comment

Thanks for the idea. After Codex review against current `main`, this looks better scoped as a ClawHub plugin/skill rather than an OpenClaw core feature. Current project direction explicitly says new skills should go to ClawHub first and that new core skills are out of scope when they can live there (`VISION.md:68`, `VISION.md:101`). The extension model also already points optional tools/skills to plugins (`docs/tools/plugin.md:11`), and ClawHub is documented as the canonical surface for community capabilities (`docs/plugins/community.md:9`). I also checked current `main` for a built-in `report_bug` / GitHub issue creation surface and did not find one. Closing this as `clawhub`; if someone wants to build this, a ClawHub plugin/skill is the right path.
