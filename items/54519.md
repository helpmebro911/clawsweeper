## Summary

Current main still does not implement the requested per-account read-only channel mode or a hard gateway/transport no-outbound guarantee. Existing `sendPolicy` is session-scoped and only suppresses selected reply paths; direct/gateway sends, message actions, polls, and WhatsApp transport side effects still have no account-level `readOnly` gate. The related #71382 PR is useful but remains open/unmerged and WhatsApp-only, so it should not close this broader generic issue yet.

## Best Possible Solution

Keep this issue open. The best product path is a generic channel/account outbound delivery policy, exposed in shared config/schema/docs/UI metadata and enforced before gateway sends, direct delivery, message actions, polls, typing/presence, reactions, read receipts, and plugin-owned transport side effects where feasible. Coordinate with #71382 for WhatsApp transport-layer enforcement, but do not close this broader issue until the generic contract or a maintainer-approved scoped replacement exists.
