## Summary

Current main still does not implement the requested generic per-account read-only channel mode. Shared channel config and WhatsApp config/schema lack a `readOnly` or no-outbound account policy, and gateway/core outbound paths still deliver sends and polls without a hard account-level gate. The related #71382 PR is open, unmerged, and WhatsApp-scoped, so it is an implementation candidate rather than a reason to close this broader issue.

## Best Possible Solution

Keep this issue open. The best product path is a generic channel/account outbound policy exposed in shared config schema, docs, and relevant UI metadata, then enforced before every outbound surface: automatic replies, direct gateway sends, message actions, polls, reactions, typing/presence, read receipts, and plugin-owned transport side effects where feasible. Coordinate with #71382 for the WhatsApp transport subset, but close this broader issue only after a maintainer-approved generic policy or explicitly scoped replacement ships with tests and docs.
