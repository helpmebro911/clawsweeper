## Summary

Keep #18985 open. Current main still does not support the reporter's Windows 11 MSYS2/MINGW `install.sh` path, and installer PATH persistence/warnings remain bash/zsh-oriented rather than fish-aware. Existing fish completion support and MSYSTEM-aware TUI handling are adjacent surfaces, not installer support.

## Best Possible Solution

Keep #18985 open until maintainers either implement explicit MSYS2/MINGW `install.sh` support with correct OS detection, Windows/MSYS path and package-manager handling, fish `config.fish` PATH persistence or guidance, docs, and installer smoke coverage; or make a documented product decision that Windows Unix-like environments should use WSL2 or `install.ps1` instead.
