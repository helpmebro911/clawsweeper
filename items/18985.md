## Summary

Keep open. Current `main` does not implement the requested Windows 11 MSYS2 + fish-shell install path: `scripts/install.sh` still hard-fails on non-macOS/Linux/WSL environments, the installer docs still scope `install.sh` to macOS/Linux/WSL and Windows to `install.ps1`, and the installer's PATH/profile guidance remains bash/zsh-only rather than fish-aware.

## Evidence

- **`install.sh` still rejects unknown OS values:** `detect_os_or_die()` sets `OS="unknown"` unless `OSTYPE` matches macOS or Linux/WSL, then exits with `Unsupported operating system` and tells Windows users to use `install.ps1`. That matches the issue report rather than fixing it.
  - file: [scripts/install.sh:254](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/scripts/install.sh#L254)
  - command: `nl -ba scripts/install.sh | sed -n '254,290p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Installer docs still do not claim `install.sh` works on Windows/MSYS:** The current installer docs list `install.sh` and `install-cli.sh` as `macOS / Linux / WSL`, and `install.ps1` as the Windows installer. The `install.sh` section explicitly says it is recommended for macOS/Linux/WSL and that OS detection supports macOS and Linux (including WSL).
  - file: [docs/install/installer.md:12](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/docs/install/installer.md#L12)
  - command: `nl -ba docs/install/installer.md | sed -n '1,90p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Windows docs still direct native Windows users to PowerShell/WSL paths:** The Windows platform doc says OpenClaw supports native Windows and WSL2, but the native Windows path called out as working is the website installer via `install.ps1`, with WSL2 still recommended. There is no MSYS2/install.sh support documented.
  - file: [docs/platforms/windows.md:10](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/docs/platforms/windows.md#L10)
  - command: `nl -ba docs/platforms/windows.md | sed -n '1,50p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Installer shell-profile handling is still bash/zsh-only:** `warn_shell_path_missing_dir()` tells users to fix PATH in `~/.zshrc` or `~/.bashrc` only, and the installer's PATH-writing logic appends only to `.bashrc`/`.zshrc`. That means the fish-shell part of the request is also not implemented in the installer flow.
  - file: [scripts/install.sh:1788](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/scripts/install.sh#L1788)
  - command: `nl -ba scripts/install.sh | sed -n '1788,1804p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Fish is known elsewhere in the repo, so this is a real installer gap rather than a repo-wide lack of fish awareness:** The setup/completion flow already knows the fish config path (`~/.config/fish/config.fish`), but the installer does not use that knowledge.
  - file: [src/wizard/setup.completion.ts:21](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/wizard/setup.completion.ts#L21)
