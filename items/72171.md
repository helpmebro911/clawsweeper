## Summary

Keep #72171 open. Current main still hardcodes new non-Azure custom-provider models to text-only input, does not refresh existing non-Azure model input on re-onboard, and runtime transports drop image payloads when the selected model metadata lacks `image`. The PR is a focused fix for the still-open linked bug #51869; review comments raise scope/coverage refinements, not evidence that the work is obsolete or already shipped.

## Best Possible Solution

Keep #72171 or a maintainer-chosen canonical follow-up open. The best fix belongs in `applyCustomApiConfig()`: detect or explicitly ask for image-capable custom models, write `input: ["text", "image"]` for new matching non-Azure custom models, upgrade existing text-only entries on re-onboard without clobbering richer user metadata, and cover new plus existing-config paths in `src/commands/onboard-custom-config.test.ts`. Maintainers should decide whether Azure custom deployment vision detection is in this PR or a separate follow-up.
