## Summary

Keep open. The reported lifecycle:end payload gap still exists on current main: the pi-embedded lifecycle emitter still omits `aborted` and `stopReason`, while the fallback emitter that includes them remains guarded behind `!lifecycleEnded`. The related PR #66574 was closed unmerged, so the issue's concrete bug is not implemented on main.

## Best Possible Solution

Keep the issue open and implement the fix in `src/agents/pi-embedded-subscribe.handlers.lifecycle.ts`: derive `stopReason` from the terminal assistant/result state, include `aborted` and `stopReason` in both `emitAgentEvent` and `onAgentEvent` lifecycle:end payloads, and add regression tests for natural completion and aborted runs so the fallback in `agent-command.ts` is no longer the only enriched path.
