## Summary

Current main still does not include `aborted` or `stopReason` on the pi-embedded `lifecycle:end` event emitted by `handleAgentEnd`. The richer fallback in `agent-command.ts` remains gated off once the subscribe path emits a terminal lifecycle event, and the related fix PR in the provided context is closed unmerged.

## Best Possible Solution

Keep this issue open. The fix should update `src/agents/pi-embedded-subscribe.handlers.lifecycle.ts` so both the global `emitAgentEvent` and direct `onAgentEvent` lifecycle end payloads include `aborted` and `stopReason`, deriving them consistently from the terminal assistant or attempt result. Add regression tests for natural completion and aborted runs, and keep the gateway lifecycle/status mapping covered.
