## Summary

Keep this issue open. Current main still emits the normal pi-embedded `lifecycle:end` event without `aborted` or `stopReason`, while the richer fallback in `agent-command.ts` is skipped once the subscribe path emits the terminal lifecycle event. The related fix PR #66574 is closed unmerged, and existing tests still assert field-less lifecycle end payloads.

## Best Possible Solution

Update the pi-embedded subscribe lifecycle bridge so both global `emitAgentEvent` and direct `onAgentEvent` lifecycle end payloads include `aborted` and `stopReason`, sourced from explicit terminal attempt metadata or an extended typed terminal lifecycle metadata seam. Add focused regression coverage for natural completion, explicit abort, and timeout/fallback behavior, while preserving current gateway session lifecycle status mapping.

## Risks / Open Questions

- A minimal fix that infers `aborted` only from `stopReason` could misclassify timeouts or other nonstandard terminal reasons; the safer path is to pass explicit runner result metadata through a typed terminal lifecycle metadata seam.
- Changing terminal lifecycle payloads should preserve existing error handling, retry-grace behavior, and gateway session status mapping for `done`, `timeout`, `killed`, and `failed`.
