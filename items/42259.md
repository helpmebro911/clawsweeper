## Summary

Keep this PR open. Current main still does not implement configurable image-tool JPEG quality or resize-side compression controls, and #41771/#42058 were closed with this PR as the remaining implementation thread. The PR should not merge as-is because the proposed fallback behavior can violate its own constraints and the branch bundles unrelated docs/gateway/provider-link changes.

## Best Possible Solution

Keep this PR open as the canonical implementation thread, but require a focused rebase before merge. The best patch should add a typed and documented `agents.defaults` compression contract, thread quality/max-side options from `createImageTool` into `loadWebMedia` and `optimizeImageToJpeg`, honor out-of-ladder values such as quality 95/85 and max side 600, add regression tests for those edge cases, and split unrelated docs/gateway/provider-link changes into separate PRs if they are still wanted.
