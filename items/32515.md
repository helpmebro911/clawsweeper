## Summary

Keep #32515 open. Current main still does not implement the requested one-click Anthropic browser OAuth onboarding flow. Anthropic setup is still Claude CLI reuse, manual setup-token paste, or API-key auth; current token-prefix handling is compatibility after a credential already exists, not browser sign-in/account detection/configuration.

## Best Possible Solution

Keep the issue open for maintainer/product handling. If accepted, implement it in the bundled Anthropic provider as a provider-owned OAuth/browser auth method using the existing `ProviderAuthContext.openUrl` and OAuth helper seams, persist the resulting auth profile, choose safe API-key-versus-subscription defaults, update onboarding/configure/docs, and add focused regression coverage. If Anthropic cannot support a sanctioned browser OAuth flow, maintainers should explicitly retitle or close around the supported Claude CLI/setup-token/API-key UX.
