## Summary

Current `main` does not implement the requested one-click Anthropic OAuth/browser login flow. The Anthropic provider currently supports three setup paths: API key, manual `setup-token` paste, or reuse of an already-authenticated local Claude CLI session. Because the requested UX is still absent, this issue should remain open.

## Evidence

- **Anthropic docs list only API key and Claude CLI reuse:** Provider docs describe exactly two primary auth routes: direct API key auth and reusing an existing Claude CLI login on the same host. The onboarding instructions tell the user to choose one of those routes; they do not describe an OpenClaw-started browser/OAuth login.
  - file: [docs/providers/anthropic.md:10](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/providers/anthropic.md#L10)

  - file: [extensions/anthropic/register.runtime.ts:414](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/anthropic/register.runtime.ts#L414)
