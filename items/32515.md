## Summary

Keep #32515 open. Current main still does not implement the requested one-click Anthropic browser OAuth onboarding flow. Anthropic auth setup remains Claude CLI reuse, manual setup-token paste, or API-key auth; existing OAuth/token compatibility handles already-obtained Anthropic tokens but does not open a browser, sign in with Anthropic, detect account type, or configure everything automatically.

## Best Possible Solution

Keep the issue open for maintainer/product handling. If accepted, implement this in the bundled Anthropic provider as a provider-owned OAuth auth method using the existing ProviderAuthContext `openUrl` and OAuth callback helper seams, persist the resulting auth profile, choose safe API-key-versus-subscription defaults, update onboarding/configure/docs, and add focused regression coverage. If Anthropic cannot support a sanctioned browser OAuth flow for this use, maintainers should explicitly retitle or close around the supported Claude CLI/setup-token/API-key UX.
