## Summary

Keep #70664 open. Current main still has the exact Unix/macOS lsof command-name filter the issue reports: `parsePidsFromLsofOutput` only returns PIDs whose `c` token contains `openclaw`, so an lsof `cnode` listener for a Node-hosted `openclaw-gateway` is still dropped. The related PR #70681 is an open, direct fix, but it has not landed on current main.

## Best Possible Solution

Keep the issue open until PR #70681 or an equivalent fix lands. The likely product-correct path is to make Unix lsof parsing collect listener PIDs without classifying by `c`, verify candidate PIDs via full argv (`ps -ww -p <pid> -o command=` or an equivalent robust process-argv reader), update `pollPortOnce` to use the same verifier, and add regression coverage for `cnode` plus rewritten `openclaw-gateway` argv on the macOS/Unix path.
