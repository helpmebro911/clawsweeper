## Summary

Keep open. Current `main` does not include a bundled Yandex/YandexGPT provider or docs, only the generic custom-provider path for OpenAI-compatible endpoints. The earlier PR was not superseded by code on `main`; it was self-closed, and the issue is now linked to an active replacement PR (#66515), so there is no strong basis for any close reason.

## Evidence

- **No bundled Yandex provider on current main:** A repo-wide search across provider/plugin surfaces found no Yandex provider artifacts on `main`; the only `yandex` matches are browser executable-name detections, not model-provider code.
  - command: `rg -n "yandex" package.json docs/providers src/plugin-sdk scripts/lib src/plugins extensions`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Provider directory omits Yandex:** The current provider index lists many bundled providers, ending with `xAI`, `Xiaomi`, and `Z.AI`, but there is no Yandex entry.
  - file: [docs/providers/index.md:27](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/docs/providers/index.md#L27)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Main supports a workaround, not the requested native provider:** OpenClaw documents `models.providers` custom providers and explicitly supports non-native `api: "openai-completions"` endpoints, which could cover Yandex as a manual OpenAI-compatible backend. That is useful context, but it does not satisfy the request for a first-class bundled provider.
  - file: [docs/concepts/model-providers.md:618](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/docs/concepts/model-providers.md#L618)
