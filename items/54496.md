## Summary

Current main still does not implement silent reasoning persistence. OpenClaw separates thinking production (`thinkingLevel`) from user/channel visibility (`reasoningLevel`), but there is no documented or configured mode that generates thinking while stripping `type: "thinking"` or `redacted_thinking` blocks before they are stored in the session transcript. Existing stripping is provider replay hygiene and explicitly preserves the latest assistant thinking block, while `sessions.get` and ACP loadSession still expose/replay stored thinking blocks.

## Best Possible Solution

Keep this issue open and implement a core transcript-persistence policy independent of thinking production and reasoning visibility. A maintainer-quality fix should keep `thinkingLevel` for model reasoning effort, keep `reasoningLevel` for user/channel visibility, add a documented persistence/discard mode, strip `type: "thinking"` and `redacted_thinking` before durable transcript writes or expose only replay-safe opaque state outside user/history/memory transcript APIs, and cover `sessions.get`, ACP loadSession, compaction/replay, built-in session memory, and provider replay compatibility with tests.
