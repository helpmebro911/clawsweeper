## Summary

Current main still does not implement silent reasoning persistence. OpenClaw has thinking production controls, reasoning visibility controls, display sanitizers, plugin hooks, and provider replay sanitizers, but no schema-backed mode that lets reasoning run while discarding thinking/redacted_thinking blocks from the durable session transcript and transcript APIs.

## Best Possible Solution

Keep this issue open. The best product direction is a core transcript-persistence policy separate from `thinkingLevel` model reasoning production and `reasoningLevel` user/channel visibility. Add a documented persist/discard setting for thinking blocks, define where provider-required replay signatures or opaque reasoning state live, and cover transcript writes/reads, `sessions.get`, ACP `loadSession`, compaction/replay sanitization, memory/session-history indexing, UI/config docs, and plugin hook expectations.

## Risks / Open Questions

- A naive strip-before-write implementation could break providers that require signed or opaque thinking blocks for follow-up replay, so the design needs a clear split between provider replay state and user/history/memory transcript surfaces.
- The existing `before_message_write` hook can support narrow plugin workarounds, but it is not a documented core setting and does not settle `sessions.get`, ACP loadSession, compaction/replay, UI/config, or memory indexing semantics.
