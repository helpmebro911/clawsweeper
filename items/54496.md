## Summary

Current main still does not implement silent reasoning persistence. OpenClaw has separate model-thinking and reasoning-visibility knobs, but no documented or schema-backed mode that lets reasoning run while discarding `thinking`/`redacted_thinking` blocks from durable transcripts. Current transcript APIs and ACP replay still preserve or return thinking blocks.

## Best Possible Solution

Keep this issue open. Implement a core transcript-persistence policy separate from model thinking effort and reasoning visibility: keep `thinkingLevel` for production, keep `reasoningLevel` for user/channel visibility, add a documented persist/discard setting for thinking blocks, decide how provider-required opaque replay state is stored outside user/history/memory transcript APIs, and cover `sessions.get`, ACP `loadSession`, compaction/replay, session-history and memory indexing, UI/config docs, and plugin hook expectations.
