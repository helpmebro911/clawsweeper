## Summary

Keep open. Current main still renders the shared status footer with only the session key plus last-updated/no-activity text, and neither `/status` nor `session_status` adds elapsed session duration or the requester-approved absolute start time. The session start timestamp plumbing exists, so this is a concrete small feature rather than stale or unactionable cleanup.

## Best Possible Solution

Implement this in the shared status renderer so both `/status` and `session_status` benefit. Resolve a start timestamp from `sessionEntry.sessionStartedAt` first, fall back through `resolveSessionLifecycleTimestamps()` when needed, then render either `started <absolute time>` or `duration <elapsed>` before the existing `updated ...` text. Add focused coverage around `buildStatusMessage()` or `buildStatusText()` for the session footer and keep broader response-time/status timing work tracked separately in #57442.
