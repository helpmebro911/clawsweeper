## Summary

Keep open. Current main has a partial mitigation for orphaned trailing user turns during retries, but it does not prove the reported raw JSONL duplicate persistence and compaction/bootstrap re-append cascade is fixed. Maintainer discussion also explicitly keeps #66443 as the concrete repro under umbrella #69208 rather than treating it as a strict duplicate.

## Best Possible Solution

Keep #66443 open as the concrete Track A repro under #69208. Add a focused regression that forces overflow recovery/tool-result truncation and inspects both the active branch and raw JSONL for duplicate role=user entries, cloned compaction entries, and repeated openclaw:bootstrap-context:full records. Then implement transcript-write idempotency or recovery-specific dedupe using stable message identity/hash windows without breaking SessionManager branch semantics.
