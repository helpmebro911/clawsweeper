## Summary

Keep #7006 open. Current main still keeps `openrouter/auto` as the runtime model, does not read OpenRouter's resolved response `model`, and still formats response footers from normalized token usage plus local pricing rather than OpenRouter's provider-reported `usage.cost` / `cost_details`. The ClawRouter comment is useful prior art for alternate routing, but it is a separate service and does not satisfy transparency for OpenRouter's own auto route. Related #9016 is cost-focused and still open; PR #43026 was closed unmerged, so neither supersedes the resolved-model request.

## Best Possible Solution

Keep #7006 open. The best implementation path is to capture OpenRouter's response `model` and provider-reported `usage.cost` / `cost_details` in the OpenAI-compatible transport, preserve selected-versus-resolved model metadata in assistant/session records, and thread that through response footers, usage/session/status surfaces, transcripts, and regression tests for `openrouter/auto`. Treat #9016 as related cost-only work and #43026 as useful prior art, not as a replacement for this issue.
