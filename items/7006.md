## Summary

Keep open. Current `main` still computes and displays the per-reply usage footer as tokens/cache/cost only, and the OpenRouter path still does not capture or persist a routed downstream model for `openrouter/auto` replies.

## Evidence

- **Usage footer formatter has no model field:** `formatResponseUsageLine(...)` only renders input/output tokens, cache counters, and optional estimated cost. The returned string is `Usage: ...` with no provider/model placeholder.
  - file: [src/auto-reply/reply/agent-runner-usage-line.ts:9](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/auto-reply/reply/agent-runner-usage-line.ts#L9)

  - sha: [7e16e3d07723](https://github.com/openclaw/openclaw/commit/7e16e3d07723cc3903f40ce94f53a3920c88b1fc)
- **Reply runner only appends usage/cost/session key:** `runReplyAgent` resolves `modelUsed`/`providerUsed`, but when `/usage` is enabled it only calls `formatResponseUsageLine(...)` and optionally appends the session key. No model label is added to the footer.
  - file: [src/auto-reply/reply/agent-runner.ts:1452](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/auto-reply/reply/agent-runner.ts#L1452)
