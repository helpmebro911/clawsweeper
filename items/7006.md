## Summary

Keep #7006 open. Current main still does not expose OpenRouter's resolved `openrouter/auto` model in the response footer or agent/session metadata, and it still estimates cost from the selected model's local pricing instead of preserving OpenRouter's numeric `usage.cost` / `cost_details`. The ClawRouter comment is useful prior discussion, but the reporter correctly distinguished it as a separate routing service rather than transparency for OpenRouter's own auto mode. Related #9016 remains cost-focused, and PR #43026 was closed unmerged, so neither supersedes the resolved-model request.

## Best Possible Solution

Keep #7006 open. The best product/code path is to capture OpenRouter's response `model` and provider-reported `usage.cost` / `usage.cost_details` in the OpenAI-compatible transport, preserve selected-versus-resolved model metadata in assistant/session records, and thread that through response footers, usage/session/status surfaces, transcripts, and regression tests for `openrouter/auto`. Treat #9016 as related cost-only work, not a replacement for this resolved-model transparency request; #43026 remains useful prior art but was closed unmerged.
