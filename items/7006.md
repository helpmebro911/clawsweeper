## Summary

Keep open. Current main still does not expose the concrete model chosen by OpenRouter for `openrouter/auto` in the response usage footer or agent-visible metadata. Cost reporting exists as estimated/local usage infrastructure, but the OpenAI-compatible stream path still keeps the selected model id and does not consume a routed OpenRouter model/cost for auto routing. The ClawRouter comment is useful prior discussion, but it is a separate routing service; #9016 covers cost-only work and #43026 was closed unmerged/not present here.

## Best Possible Solution

Keep the issue open. The best path is to teach the OpenRouter/OpenAI-compatible transport to capture provider-reported routed model and cost metadata for `openrouter/auto`, persist it through assistant usage/session metadata, expose it to `session_status` or equivalent agent-visible surfaces, and append it in the response usage footer with focused regression coverage for auto routing.
