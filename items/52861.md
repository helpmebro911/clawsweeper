## Summary

Keep this issue open. Current main has useful partial mitigations for skills, compaction, explicit tool allowlists, and optional bootstrap skipping, but it still does not implement the central request: automatic conditional tool/schema injection plus an on-demand or summary/index default for recurring workspace bootstrap files.

## Best Possible Solution

Keep this issue open as the broad per-turn overhead tracker. Use #14785 for the narrower tool-schema work, but solve this issue with a policy-preserving automatic tool relevance/schema-routing design plus an opt-in or better default summary/index mode for recurring workspace bootstrap files. Keep the existing compact skills catalog and compaction behavior as partial shipped mitigations.

## Risks / Open Questions

- Automatic tool relevance routing must preserve explicit allow/deny policy semantics, plugin compatibility, safety/recovery tools, and provider schema constraints.
- Workspace bootstrap summary/index mode needs deterministic ordering and cache-stable behavior so token savings do not make prompts harder to reproduce or hide standing instructions.
- Closing this now would lose the broader tracker while #14785 only covers the narrower tool-schema footprint and does not own workspace bootstrap streaming/default summarization.
