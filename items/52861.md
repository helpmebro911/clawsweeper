## Summary

Keep open. Current main has meaningful prompt-cost mitigations, including compact/on-demand skill catalogs, static tool profiles and allow/deny policy, bootstrap caps plus continuation-skip/lightweight modes, and automatic compaction. It still does not implement the central request for automatic per-turn conditional tool/schema relevance routing, and the workspace bootstrap path still defaults to injecting the configured bootstrap files rather than a summary/index/on-demand model. The related #14785 remains the narrower open tracker for tool-schema footprint, but this issue covers broader per-turn overhead across tools, skills, workspace files, and compaction.

## Best Possible Solution

Keep this issue open as the broad per-turn prompt overhead tracker. Use #14785 for narrower tool-schema footprint work, but solve this issue with a compatible automatic relevance/schema-routing design plus a default or opt-in workspace bootstrap summary/index/on-demand mode. Preserve the shipped mitigations: compact skill catalogs, explicit tool profiles and allowlists, bootstrap caps/contextInjection controls, lightweight cron/heartbeat behavior, and compaction.

## Risks / Open Questions

- Automatic conditional tool/schema routing needs a policy-preserving design for explicit allow/deny lists, owner-only tools, provider-specific schema constraints, client tools, plugin tools, bundled MCP/LSP tools, and recovery paths.
- A default workspace bootstrap summary/index mode could hide standing instructions unless it preserves deterministic ordering, clear cache behavior, and a reliable route for the agent to read full files on demand.
- Closing this issue now would drop the broad workspace/skill/bootstrap scope; #14785 only tracks the narrower tool-schema footprint.
