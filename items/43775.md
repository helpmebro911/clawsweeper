## Summary

Keep this issue open. Current main has a scoped future-config guard for service/process mutations and `gateway --force`, but ordinary config loading still only warns and non-service gateway startup can continue with a future `meta.lastTouchedVersion`. The linked auto-downstamp PR was closed unmerged, and the merged GPT-5.4 runtime hardening work addresses adjacent delivery/runtime paths without proving this future-stamped-config Discord/OpenAI-Codex failure is fixed.

## Best Possible Solution

Keep the issue open until maintainers either extend the future-config guard/remediation to ordinary gateway runtime paths that can perform channel delivery, or add focused regression/manual proof showing the current service/process guard fully eliminates the Discord/OpenAI-Codex silent-delivery failure. A good fix would make the startup warning actionable and point users to the split-brain install repair flow without blocking safe read-only inspection.
