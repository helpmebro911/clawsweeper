## Summary

Keep open. Current `main` still serves WebChat history only from the live session-store entry for the requested session key, and current session-maintenance enforcement can still evict non-active session entries plus their transcript files when disk budget cleanup runs. I found no strong evidence in the checked-out `main` that this issue’s underlying session-loss path is already fixed well enough to close conservatively.

## Evidence

- **WebChat history depends on current session-store pointer:** The Gateway history endpoint resolves the requested session key through `sessions.json`, returns `404` when no current entry with `sessionId` exists, and only then reads transcript messages from that entry’s session file. If the pointer is evicted, history disappears even if tokens were previously consumed.
  - file: [src/gateway/sessions-history-http.ts:133](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/gateway/sessions-history-http.ts#L133)
  - command: `nl -ba src/gateway/sessions-history-http.ts | sed -n '100,180p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Disk-budget enforcement still deletes non-active session entries and transcript files:** `enforceSessionDiskBudget()` removes orphan/archive files first, then, if still over the high-water mark, deletes oldest store entries except the single `activeSessionKey`; when an entry’s last reference is removed it also deletes that transcript file.
  - file: [src/config/sessions/disk-budget.ts:254](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/config/sessions/disk-budget.ts#L254)
  - command: `nl -ba src/config/sessions/disk-budget.ts | sed -n '245,340p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Normal enforced maintenance wires the same eviction path:** `saveSessionStoreUnlocked()` preserves only the current `activeSessionKey` during stale/cap pruning, then invokes `enforceSessionDiskBudget()` in enforce mode with that one active key. Other session pointers remain eligible for eviction.
  - file: [src/config/sessions/store.ts:284](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/config/sessions/store.ts#L284)

- **Current docs still describe entry-and-transcript eviction as the enforced behavior:** The session-management docs explicitly say enforced disk-budget cleanup may evict oldest session entries and their transcript files until usage drops below `highWaterBytes`, which matches the current code path rather than a shipped fix for lost history.
