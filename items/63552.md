## Summary

Keep this PR open. Current main still runs broad non-live provider discovery during models.json generation even when explicit providers are already available, and this PR is a narrow implementation candidate with regression coverage. The overlapping #72352 and related #66957 are both still open, so neither is a high-confidence superseding close reason yet.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt the focused optimization: pass explicit providers into the discovery-filter path, resolve their owning plugin IDs through the existing helper, fall back to broad discovery when any explicit provider has no owner, preserve live all and mixed live-filter behavior, and keep regression coverage. If maintainers prefer #72352 as the canonical broader fix, close this PR only after that implementation is explicitly selected or merged.
