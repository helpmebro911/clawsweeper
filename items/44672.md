## Summary

Keep open. Current main has partial support for distinguishing pairing upgrade reasons in shared gateway error mapping, but the macOS control/chat path still collapses `PAIRING_REQUIRED` into a generic pairing message and does not consume the newer role-upgrade-specific mapper. Auto-recovery also appears indirect rather than an immediate post-approval refresh.

## Evidence
