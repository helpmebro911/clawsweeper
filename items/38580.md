## Summary

Current `main` still does not satisfy #38580. The Telegram plugin exposes send/poll/react/delete/edit/sticker/topic actions and delivery pinning, but no on-demand action or native command that reads `getChat.pinned_message`. The related implementation attempts #38613 and #38718 are useful prior art from the discussion, but both are closed and unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep #38580 open until the bundled Telegram plugin implements an on-demand pinned-message read path, likely as a `pins`/`list-pins` message action or explicit `/getpinned` native command backed by Telegram Bot API `getChat.pinned_message`. The implementation should add Telegram discovery/runtime plumbing, an account/action gate, docs, and regression tests for no pin, API errors, target resolution, formatting, and multi-account/thread behavior. If maintainers want this outside the bundled Telegram plugin, record that product decision explicitly because current `main` cannot satisfy the request.
