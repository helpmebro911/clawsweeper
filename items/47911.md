## Summary

Keep open. Current main has overlapping diagnostics (`openclaw status`, `openclaw sessions --all-agents`, `openclaw agents list`, and `openclaw models status --agent`), but it does not implement the requested unified `openclaw fleet status` command or an equivalent fleet-wide view combining per-agent health, model auth state, channel health, session stuck/active state, and recent errors. The linked PR #50066 is still open and explicitly only adds `sessions_send` JSONL logging that fleet status could consume; the closed #50072 signal-layer proposal/prototype does not supersede this core observability request.
