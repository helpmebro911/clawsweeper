## Summary

Keep this open. Current main has partial private-network support for SearXNG web_search and model-provider request transport, but the central request is still not implemented for generic web_fetch, image/media URL references, or Gateway OpenResponses input_file/input_image URL fetches. The related implementation PR was closed unmerged, and the provider-request work covers a different surface.

## Best Possible Solution

Keep this issue open and scope the remaining product/security decision around a narrow private-network opt-in for generic web_fetch, image/media URL references that share web_fetch SSRF policy, and Gateway OpenResponses input_file/input_image URL fetches. The implementation should preserve default private-network blocking, add explicit config schema/help/docs, wire the policy through runtime guards, and add SSRF regression tests. Existing SearXNG and model-provider allowPrivateNetwork support should be treated as precedent, not full resolution.
