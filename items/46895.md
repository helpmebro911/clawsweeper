## Summary

Keep open. Current main still routes Control UI `/model status` and `/model list` through the local `/model` executor, and that executor only treats empty args as model info; non-empty `status` or `list` still goes to `sessions.patch`. The PR is stale and should be refreshed, but it targets a real remaining Web UI parity gap that is also documented and covered in the shared directive path.

## Best Possible Solution

Keep the PR or linked work open, but refresh it against current main and implement Control UI parity with the shared `/model` directive contract: `/model list` must not call `sessions.patch` and should show the compact browser/list behavior; `/model status` must not call `sessions.patch` and should show status/detail behavior or deliberately forward to the shared agent directive path. Add focused tests for both keywords plus the normal `/model <provider/model>` patch path.
