## Summary

Current main still only partially addresses the reported Control UI ANSI leak. The Overview Gateway Logs tail strips ANSI for display, but the dedicated Logs tab still fetches `logs.tail`, parses JSON/raw lines without sanitizing ANSI, and renders `entry.message ?? entry.raw` directly. The related fix PR #64431 was closed unmerged, so this issue should remain open.

## Best Possible Solution

Keep this issue open and land a focused Control UI logging fix. The best implementation is a shared ANSI display strategy for every `logs.tail` UI consumer: strip or safely render ANSI in both Overview and the dedicated Logs tab, keep export behavior deliberate, and add regression tests for ANSI-bearing JSON messages, subsystem/context parsing, and raw fallback lines. PR #64431 is useful prior art but cannot close the issue because it was not merged.
