## Summary

Keep #13597 open. Current main still lacks an AWS deployment guide for EC2/Lightsail, ECS/Fargate, or Lambda applicability; existing docs only provide generic Docker/Kubernetes/VPS material and Bedrock provider setup. The issue discussion is concrete and actionable, including Docker Compose as the likely primary AWS path, persistence/EFS, gateway exposure, Secrets Manager, multi-agent topology, and an AWS sample repo to consider.

## Best Possible Solution

Keep the issue open until OpenClaw ships an AWS install page, likely `docs/install/aws.md`, linked from `docs/install/index.md`, `docs/vps.md`, and `docs/docs.json`. The guide should make Docker Compose the primary EC2/Lightsail path, cover durable state/EFS or equivalent storage, gateway bind/reverse proxy/ALB/TLS/auth/origin settings, security groups, IAM roles, Secrets Manager, logging/monitoring, backups, troubleshooting, performance and hardening, ECS/Fargate task/service/networking, Lambda applicability or non-support, multi-agent topology, and should evaluate/link the AWS sample repo from the issue discussion.
