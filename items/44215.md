## Summary

Keep PR #44215 open. Current `main` still does not add Windows-specific bootstrap directories in `ensureOpenClawCliOnPath`, and the PR is a plausible targeted fix, but it needs maintainer/author changes before it could land because the proposed diff has unresolved PATH ordering/test concerns and an unrelated config labels change.

## Best Possible Solution

Keep the PR open, but require a rebase and a narrower patch: drop the unrelated config-labels change, decide the intended security ordering for Windows dirs, avoid prepending user-writable WindowsApps/npm/pnpm ahead of trusted/existing entries unless maintainers explicitly accept that behavior, use a platform-aware delimiter for simulated Windows PATHs, and add focused tests covering SystemRoot/System32, Program Files and ProgramFiles(x86), and any user-profile dirs in their final ordering.

## Risks / Open Questions

- Landing the PR verbatim could put user-writable Windows locations such as WindowsApps/pnpm/npm ahead of existing PATH or trusted system entries, changing executable resolution for aliases like Python.
- The current PR diff includes an unrelated `src/config/schema.labels.ts` gateway push label addition that should be dropped from a path bootstrap fix.
- The Windows regression test needs platform delimiter handling and stronger ordering assertions before it can reliably prove the intended behavior on non-Windows CI.
