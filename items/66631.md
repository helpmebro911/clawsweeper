## Summary

Keep open. Current `main` still routes Feishu topic replies with `ctx.rootId ?? ctx.messageId`, and the bundled tests explicitly lock in `message_id` as the topic root when `root_id` is absent. There is no alternate code path that resolves a topic root from `thread_id`, so the reported behavior remains plausible on `main` and in the latest release.

## Evidence

(ctx.rootId ?? ctx.messageId) : ctx.messageId`, so topic-mode or `replyInThread` flows still reply to the triggering message whenever `root_id` is missing.
