## Summary

Keep open. Current main still requires a Feishu group message to mention the bot before dispatch, with no thread-follow-up exception or `threadFollowUp` config in the Feishu schema. The PR addresses a real remaining Feishu topic-thread usability gap, has useful linked discussion and tests in its branch context, and is not safely superseded just because related issues were later closed.

## Best Possible Solution

Keep the PR open and have a Feishu/channel maintainer either fix the remaining scoped review issue on this branch or consolidate it with the canonical Feishu thread-follow-up PR. The shipped path should add an explicit config/schema/docs surface, preserve top-level mention gating, support `group_topic` and `group_topic_sender` thread follow-ups, cover broadcast and multi-account cases, and include regression tests for active/unrelated/off behavior.
