## Summary

Keep open. Current main still deliberately bypasses Discord `requireMention` for bot-owned `autoThread` threads and bound thread sessions, matching the follow-up comment that `autoThread: true` creates a bot-started thread where later messages are handled without a fresh mention. The related closed PR was explicitly scoped to ACP thread binding and did not supersede this mention-gating behavior.

## Best Possible Solution

Keep this issue open and fix the Discord plugin behavior explicitly. A conservative path is to add a Discord thread-reply mention policy under the guild/channel entry or `channels.discord.thread`, wire it through config types, Zod schema, generated config/help docs, and Discord preflight, then add regression tests for bot-owned autoThreads and bound thread follow-ups with `requireMention: true`. Coordinate with #52568 as the narrower product-shape discussion if maintainers want that issue to become canonical.
