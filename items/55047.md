## Summary

Keep this issue open. Current main still has an unfixed Control UI crash path consistent with the report and prior discussion: malformed persisted cron jobs can reach the browser, and the global render/edit paths still dereference `job.payload` without guards.

## Best Possible Solution

Keep this issue open. The maintainer path is to reproduce against current main with a synthetic or reporter-provided stale cron row missing a valid payload, then add a defensive normalization/filter layer before `cron.list` returns jobs plus null-safe guards in Control UI render/edit paths. Add a focused jsdom/controller regression test that proves malformed cron rows do not blank unrelated Control UI sections, and wire doctor repair only as a cleanup path rather than relying on users to repair before the UI can render.
