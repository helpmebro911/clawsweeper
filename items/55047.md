## Summary

Keep this issue open. Current main still has a credible Control UI blank-screen path matching the report and prior discussion: persisted cron jobs can be returned without payload validation, while global Control UI render and Cron view/edit paths still dereference `job.payload`. The related #54552 fix was closed unmerged, so it did not resolve this on main.

## Best Possible Solution

Keep this issue open. Reproduce against current main with a synthetic or reporter-provided stale cron row missing a valid payload, then add defensive normalization/filtering before `cron.list` exposes jobs plus null-safe guards in the Control UI global render, Cron list, and edit/clone paths. Add a focused jsdom/controller regression test proving malformed cron rows do not blank unrelated Control UI sections; doctor repair can remain cleanup but should not be required before the UI can render.
