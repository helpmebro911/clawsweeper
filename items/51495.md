## Summary

Current `main` still appears to reproduce the reported `status --all` misclassification: scope-limited gateway probes are treated as unreachable outside the dedicated `gateway status` command, so the issue is not strong-closeable yet.

## Evidence

- **`status --all` still marks reachability false unless `probe.ok === true`:** `resolveGatewayProbeSnapshot()` sets `gatewayReachable` from `gatewayProbe?.ok === true` only. A probe that connected successfully but returned `missing scope: operator.read` still has `ok: false`, so it remains classified as unreachable for the shared status pipeline.
  - file: [src/commands/status.scan.shared.ts:131](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/commands/status.scan.shared.ts#L131)
  - command: `nl -ba src/commands/status.scan.shared.ts | sed -n '83,154p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Top-level status formatting still renders that path as unreachable:** `buildGatewayStatusSummaryParts()` renders `unreachable (<error>)` whenever `gatewayReachable` is false, which matches the issue's `openclaw status --all` symptom.
  - file: [src/commands/status-all/format.ts:362](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/commands/status-all/format.ts#L362)
  - command: `nl -ba src/commands/status-all/format.ts | sed -n '342,379p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Shared status helpers short-circuit follow-up RPC when reachability is false:** `resolveStatusGatewayHealthSafe()` and `resolveStatusLastHeartbeat()` both bail out when `gatewayReachable` is false, so the misclassification propagates through the rest of `status` output.
  - file: [src/commands/status-runtime-shared.ts:79](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/commands/status-runtime-shared.ts#L79)
  - command: `nl -ba src/commands/status-runtime-shared.ts | sed -n '68,107p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Dedicated `gateway status` command was fixed separately:** Gateway-status helpers now explicitly treat `missing scope: operator.read` as a degraded-but-reachable probe via `isScopeLimitedProbeFailure()` and `isProbeReachable()`. This shows the repo already recognizes the condition, but only in that command path.
  - file: [src/commands/gateway-status/helpers.ts:272](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/commands/gateway-status/helpers.ts#L272)
