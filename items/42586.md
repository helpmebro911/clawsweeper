## Summary

Keep this PR open. Current main still has the reported gateway health/runtime mismatch: health snapshots are built without channel runtime state, while `channels.status` does read the channel manager runtime snapshot. The PR is a focused contributor fix with no obvious security or supply-chain red flags, but it is overlapped by open maintainer PR #72417, which explicitly credits this PR and asks not to close duplicates or superseded PRs until that repair is validated.

## Best Possible Solution

Keep this PR open while #72417 remains open and explicitly asks maintainers not to close duplicates. The maintainer path is to review and land #72417 if it validates; if it does not land, salvage this PR by rebasing it onto current main, threading runtime through the current `server-startup-early`/gateway context path, preserving runtime and probe fields after plugin summaries, and adding regression coverage for `getHealthSnapshot` plus gateway health refresh.
