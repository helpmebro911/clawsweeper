## Summary

Keep open: current `main` still clears requested operator scopes for device-less shared-token WebSocket operator clients, with only a narrow direct-local backend exception, and the compatibility test suite explicitly codifies that behavior.

## Evidence

- **Current scope-clearing policy still targets token-auth clients:** `shouldClearUnboundScopesForMissingDeviceIdentity()` returns true not only when `decision.kind !== "allow"`, but also for allowed device-less clients using `token`, `password`, or `trusted-proxy` auth. That means successful shared-token auth does not preserve requested scopes by itself.

- **Handshake path still clears scopes by default:** The handshake comment says device-less clients still clear self-declared scopes by default, with one narrow exception for the direct-local backend gateway-client handoff. The code calls `clearUnboundScopes()` when there is no device and `skipLocalBackendSelfPairing` is false.
