Keep this issue open. Current main supports generic custom-provider image routing and configurable image-tool timeouts, but I found no source, test, docs, changelog, live proof, or issue discussion showing the NVIDIA NIM Kimi K2.5 image abort is fixed.

## Best Possible Solution

Keep this issue open. Add focused regression or live/manual proof for NVIDIA NIM Kimi K2.5 through the same configured-provider image path used by Feishu and Telegram. First verify whether raising tools.media.image.timeoutSeconds above the observed provider latency fixes it; if yes, document the required config or revisit the default. If it still fails, fix the NVIDIA/Kimi OpenAI-compatible image payload or streaming behavior, or surface a clear unsupported-provider error.
