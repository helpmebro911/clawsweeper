## Summary

Keep #9835 open. Current main has partial Telegram reaction support: inbound reaction system events and outbound `message` tool reactions exist, and the prompt can receive a generic reaction-guidance adapter. The requested product behavior is still missing: no public `reactionGuidance: "off" | "minimal" | "full"` config, no strict schema/docs/generated metadata for it, no prompt gating by `reactionNotifications`, and no incoming-reaction soft-CTA guidance.

## Best Possible Solution

Keep #9835 open. If maintainers want this behavior, implement it in the bundled Telegram plugin and agent prompt composition path: add and document `reactionGuidance` with `off|minimal|full` semantics, define how it composes with `reactionNotifications` and existing `reactionLevel`, gate incoming soft-CTA prompt guidance on enabled reaction notifications, update strict schema/generated config metadata/docs, and add focused tests for off/minimal/full plus notification gating.
