## Summary

Keep #9835 open. Current main still has Telegram inbound reaction system events and outbound `message` tool reactions, but it does not implement the requested `reactionGuidance: "off" | "minimal" | "full"` config, does not gate prompt guidance on `reactionNotifications`, and does not teach the prompt to treat incoming user reactions as soft CTAs.

## Best Possible Solution

Keep #9835 open. If maintainers want this behavior, implement it in the bundled Telegram plugin and prompt composition path: add and document `reactionGuidance` with off/minimal/full semantics, define how it composes with `reactionNotifications` and existing `reactionLevel`, inject incoming soft-CTA guidance only when enabled, update generated config metadata/docs, and add focused tests for off/minimal/full plus notification gating.
