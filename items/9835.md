## Summary

Keep #9835 open. Current main still has the Telegram reaction plumbing and a prompt section for outbound reaction etiquette, but the requested configurable two-way guidance is not implemented: there is no user-facing `reactionGuidance` config, prompt injection is driven by `reactionLevel` rather than enabled inbound `reactionNotifications`, and the prompt still lacks the requested incoming-reaction soft CTA guidance.

## Best Possible Solution

Keep this issue open and implement the remaining product behavior in the Telegram/plugin prompt path: add a documented user-facing `reactionGuidance` setting if maintainers still want that shape, gate prompt injection on enabled inbound `reactionNotifications` plus guidance mode, add minimal/full prompt text that covers both sparse sending and incoming reaction soft CTA interpretation, update config schema/generated metadata/docs, and add focused tests for off/minimal/full and notification gating.
