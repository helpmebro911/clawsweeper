## Summary

Keep #37591 open. Current main still requires `system.run.prepare` for `exec host=node`, while the Swift macOS companion node still advertises and dispatches `system.run` without defining or handling `system.run.prepare`. The issue discussion includes fresh 2026.4.x reports and the related fixes (#38781/#69725) are still unmerged implementation directions, not shipped replacements.

## Best Possible Solution

Keep #37591 open as the canonical tracker until maintainers ship a reviewed fix. The likely path is to add `system.run.prepare` to the shared Swift command model, advertise it from `MacNodeModeCoordinator`, dispatch it in `MacNodeRuntime`, return the gateway-compatible canonical plan payload, and add focused Swift plus agent/gateway regression coverage. A separately reviewed compatibility fallback for already-shipped macOS nodes that only advertise `system.run` may also be useful, but should preserve the approval-plan security contract.
