## Summary

Current main still exposes Slack `dmHistoryLimit` as config/docs surface but does not wire it into Slack 1:1 DM history hydration. The Slack monitor derives only `historyLimit`, the monitor context has no DM history limit, and inbound history injection remains gated on `isRoomish`, which excludes `channel_type: "im"` direct messages.

## Best Possible Solution

Keep this issue open and implement the missing Slack plugin behavior in `extensions/slack`: resolve per-DM `channels.slack.dms.<user>.historyLimit` before top-level `channels.slack.dmHistoryLimit`, pass the effective DM limit through `SlackMonitorContext`, fetch recent `conversations.history` for fresh top-level `channel_type: "im"` sessions, inject that context consistently with existing inbound history, and add focused tests for DM history hydration, override precedence, current-message exclusion, and fresh-session/reset behavior.
