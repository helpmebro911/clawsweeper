## Summary

Keep PR #39941 open. Current main still recurses into the first object-valued anyOf/oneOf variant during config form coercion, so the PR addresses a real remaining UI config bug rather than obsolete cleanup.

## Best Possible Solution

Keep PR #39941 open. The best path is to land or rework a small discriminator-aware fix in ui/src/ui/controllers/config/form-coerce.ts that prefers object variants with matching top-level const constraints, falls back to the first object variant only when no discriminator match exists, preserves primitive union behavior, and keeps regression coverage for anyOf, oneOf, and the unconstrained-variant edge case discussed in review.
