## Summary

Keep #7524 open. Current main still has no `groupScope` or equivalent configurable path for selectively routing group/channel peers into the agent main session. Core routing applies `dmScope` only to direct peers, route binding session overrides only accept `dmScope`, docs still describe groups/channels as isolated/non-main, and the related open #19929 has not been maintainer-designated as the canonical replacement in the provided discussion.

## Best Possible Solution

Keep #7524 open as the focused tracker unless a maintainer explicitly marks #19929 canonical. The best implementation is an additive core routing/config contract for selective group/channel/thread-to-main or named shared sessions, wired through `buildAgentPeerSessionKey()`, `buildAgentSessionKey()`, `resolveAgentRoute()`, route binding schemas/types/help/generated metadata, relevant channel group configs, public docs, and regression coverage for BlueBubbles, Signal, Telegram, Matrix, Discord/Slack channel routing, and generic group/channel session keys.
