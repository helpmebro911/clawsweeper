## Summary

Keep #7524 open. Current main still lacks a global or per-group `groupScope`/shared-session config that selectively routes group/channel peers into the agent main session. The newer conversation-binding path is useful for supported channels, but it is runtime/configured ACP or focused-session binding, not the requested routing/config contract, and it does not cover the Signal use case raised in the comments.

## Best Possible Solution

Keep #7524 open as the focused tracker unless a maintainer explicitly marks #19929 canonical. The best implementation is an additive routing/config contract for selective group/channel/thread-to-main or named shared sessions, wired through `buildAgentPeerSessionKey()`, `buildAgentSessionKey()`, `resolveAgentRoute()`, strict config schemas/types/help/generated metadata, relevant channel group configs, public docs, and regression coverage for BlueBubbles, Signal, Telegram, Matrix, Discord/Slack channel/thread routing, and generic non-direct session keys.
