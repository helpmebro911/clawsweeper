## Summary

Keep #7524 open. Current main still does not provide a declarative `groupScope`/group-to-main session option: non-direct peers are still keyed as isolated channel/group buckets, route and outbound session construction only propagate `dmScope`, and the public config schemas/docs only describe DM scoping plus isolated group/channel sessions. Current conversation binding/focus machinery can override routing for some bindable conversations, but it is not the requested per-group config contract and does not cover all cited channels such as Signal.

## Best Possible Solution

Keep #7524 open as the focused tracker unless a maintainer deliberately folds it into a broader shared-session design such as #19929. The best implementation is a core session-routing/config contract for selective group/channel-to-main behavior, propagated through `buildAgentPeerSessionKey()`, `buildAgentSessionKey()`, `resolveAgentRoute()`, route-binding session overrides, outbound session route construction, docs, generated config metadata, and regression tests for inbound and outbound group routing across at least one built-in channel such as BlueBubbles or Signal.
