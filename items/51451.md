## Summary

Keep open. Current `main` does not implement a per-request HTTP workspace override for `/v1/chat/completions`. The compat HTTP surface documents and parses `x-openclaw-model`, `x-openclaw-agent-id`, `x-openclaw-session-key`, and `x-openclaw-message-channel`, but there is no `x-openclaw-workspace` handling and the OpenAI HTTP path never forwards a workspace override into `agentCommandFromIngress`. There is an existing internal-only `workspaceDir` seam for spawned subagent runs, which shows the runtime can accept a workspace override, but it is not exposed on the external HTTP API.

## Evidence
