## Summary

Keep #72143 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still returns the user-facing "Automatic session resume failed" prefix on session-resume fallback, sends it in direct delivery, and has tests explicitly expecting that prefix. The implementation also makes a single `callGatewayTool("agent", ...)` attempt before fallback, so the requested retry/backoff behavior is not present. Related PR #72148 is relevant and useful, but it is still open in the provided context and explicitly defers retry/backoff, so it does not fully supersede the issue yet.

## Best Possible Solution

Keep #72143 open until main changes the exec approval follow-up path to retry the session resume/follow-up call with bounded backoff, log resume failure metadata internally, and send direct fallback content without the internal prefix. PR #72148 can be the prefix-removal portion, but the retry/backoff requirement should either land there or be split into a clear canonical follow-up before this issue closes.
