## Summary

Keep #72143 open. Current main still contains and tests the user-facing "Automatic session resume failed" prefix, and the exec approval follow-up path still makes a single session follow-up attempt before direct fallback. The linked PR #72148 is open and appears to address the prefix only; its own notes explicitly defer retry/backoff, so the issue is not yet implemented or fully superseded on main.

## Best Possible Solution

Keep the issue open until main removes the direct-delivery prefix, logs session-resume failures internally with useful metadata, and either implements the requested short retry/backoff around the `callGatewayTool` session follow-up path or links a narrower accepted follow-up that tracks that remaining resilience work. PR #72148 can be a useful partial fix, but it should not close the issue by itself unless maintainers explicitly split the retry/backoff requirement.
