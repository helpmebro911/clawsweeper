## Summary

Keep open. Current main still has only scheme/empty-string validation for gateway-status WebSocket URLs, and the explicit/configured target is passed through to the gateway probe. The PR addresses a real remaining surface, but its submitted patch is incomplete because review comments identify an IPv4-mapped IPv6 bypass for link-local metadata addresses.

## Best Possible Solution

Keep the PR open and request a rework against current main: implement gateway-status URL host validation using the shared SSRF/IP classification helpers rather than ad hoc string prefixes, preserve intentional localhost/private-LAN gateway behavior if maintainers want that exception, explicitly reject metadata/link-local/wildcard/internal/local targets, and add focused `resolveTargets()` tests for plain IPv4, hostnames, wildcard, localhost/LAN allowed cases, and IPv4-mapped IPv6 bypass forms.
