## Summary

Keep open. Current `main` still routes workspace bootstrap files through a workspace-root boundary check that rejects symlink hops outside the workspace, then reports those files as `missing`, which matches the issue report rather than fixing it.

## Evidence

- **Bootstrap loader marks failed reads as missing:** `loadWorkspaceBootstrapFiles()` reads each default bootstrap file through `readWorkspaceFileWithGuards()`. When that guarded read is not `ok`, it pushes the file with `missing: true`, which is what later renders as `[MISSING] Expected at: ...`.
  - file: [src/agents/workspace.ts:526](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/agents/workspace.ts#L526)
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Boundary path resolver rejects symlink escapes:** The boundary resolver throws when a symlink hop resolves outside the workspace root canonical path. That means a bootstrap file symlink pointing outside the workspace is rejected before read, even if the target file exists and is readable.
  - file: [src/infra/boundary-path.ts:311](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/infra/boundary-path.ts#L311)
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Out-of-workspace bootstrap aliases are intentionally excluded in tests:** Current tests explicitly assert that extra bootstrap files outside the workspace are excluded, while only a symlinked workspace root itself is supported. This aligns with the reported behavior still being present on `main`.
  - file: [src/agents/workspace.load-extra-bootstrap-files.test.ts:41](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/agents/workspace.load-extra-bootstrap-files.test.ts#L41)
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Sandbox bootstrap seeding also skips symlinked seed files:** Related sandbox workspace tests assert that symlinked bootstrap seed files are skipped, reinforcing that current code treats bootstrap symlink aliases conservatively rather than following them.
  - file: [src/agents/sandbox/workspace.test.ts:37](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/agents/sandbox/workspace.test.ts#L37)
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)

## Risks / Open Questions

- This may be an intentional security boundary, so resolving the issue likely needs an explicit product/security decision about trusting out-of-workspace bootstrap symlinks.
