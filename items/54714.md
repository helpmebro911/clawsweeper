## Summary

Keep open. Current main still allows the reported Control UI Agent Context regression: the selected subagent identity is loaded through `agent.identity.get`, but the resolver prefers global `ui.assistant` name/avatar before `agents.list[].identity`, and the Agent Context panel then prefers that runtime identity over the per-agent list payload.

## Best Possible Solution

Keep this issue open. Land a focused fix that makes non-default agents prefer `agents.list[].identity` for name/avatar/emoji over global `ui.assistant`, while preserving default-agent behavior. Update Agent Context so emoji/avatar state is shown clearly instead of only `custom` or `—`, and add regression tests for default global priority, subagent per-agent name, subagent emoji/avatar, missing identity fallback, and partial fallback.
