## Summary

Keep open. Current main has auth-profile ordering, last-used round-robin, cooldown/disabled failover, and session stickiness, but it does not implement the requested opt-in provider-profile pool routing layer: no routing policy config, no task/none stickiness mode, no per-profile in-flight/concurrency caps, no pool strategy selection, and no `models routing` CLI surface. This also should not be closed as ClawHub-only: the remaining work is in core/runtime auth selection and provider control-plane behavior, with only a narrow provider hook currently exposed.
