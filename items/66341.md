## Summary

Keep this PR open. Current main still requires admin scope for cron mutation methods and has no persisted creator-scope ceiling, so the requested core authorization change is not implemented or obsolete. The branch remains plausible, but it needs focused maintainer security review and cleanup before it can land.

## Best Possible Solution

Keep this PR open for maintainer review. Ask the author to split unrelated browser and QA-lab changes, then review a focused cron-only branch that lowers the intended cron method scope, persists creator scopes at creation time, enforces the ceiling in both embedded and CLI/MCP execution paths, and adds regression coverage proving write-scoped callers cannot update, remove, or run admin-created jobs to gain admin-level tools.
