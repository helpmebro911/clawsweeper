## Summary

Keep #72165 open. Current main still has the reported eager session-list materialization and synchronous transcript-read paths, and the older #57715 tracker does not fully cover this issue's direct sessionId lookup and MCP getConversation lookup scope.

## Best Possible Solution

Keep #72165 open unless maintainers explicitly consolidate its direct lookup and MCP scope into #57715 or another canonical item. The best implementation is a focused session summary/index path: persist or cache summary metadata at write time, filter/sort/limit before transcript hydration where semantics allow, provide direct lookup by sessionId, change MCP getConversation away from a 500-row sessions.list scan, and add regression coverage proving non-returned rows do not perform transcript reads.
