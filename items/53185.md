## Summary

Keep this issue open. Current main has shipped substantial `web_fetch` hardening and `web_search` outputs are generally wrapped as untrusted, but the issue discussion evolved into a broader pre-model tool-result preprocessing hook with custom script interception and structured threat metadata. Current main does not expose that remaining capability for external plugins or config, and #54508 is related broader security work rather than a full replacement for this narrower preprocessing-hook request.

## Best Possible Solution

Keep this issue open, but narrow it. Treat the shipped `web_fetch` hidden-content sanitizer as done, then decide whether OpenClaw should add a trusted core/plugin API for pre-model `web_fetch`/`web_search` result preprocessing, add built-in `web_search` threat metadata, or explicitly fold the taint/risk-class pieces into #54508 while keeping the script/custom-hook question tracked here until resolved.
