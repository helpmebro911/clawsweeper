## Summary

Keep open. Current `main` still does not preserve Slack custom agent identity on edited messages, so the PR targets a real gap, but the submitted branch is not a conservative cleanup close because review already identified unresolved correctness issues in the PR itself.

## Evidence

- **Current main edit path drops custom identity:** `extensions/slack/src/actions.ts:206-220` updates Slack messages with only `channel`, `ts`, `text`, and optional `blocks`. It does not forward `username`, `icon_url`, or `icon_emoji`, so edited messages on `main` still revert to the bot default identity.
  - file: [extensions/slack/src/actions.ts:206](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/slack/src/actions.ts#L206)

  - file: [extensions/slack/src/draft-stream.ts:22](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/slack/src/draft-stream.ts#L22)
