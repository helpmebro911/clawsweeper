## Summary

Current `main` already persists ordinary session continuity across reconnects via the gateway-owned session store and transcript files, but it does not implement this PR’s separate cross-session memory/preferences restore flow. There is also an existing hook/plugin-style memory path, so this looks partly redundant and partly a product/scope change rather than a clear auto-close case.
