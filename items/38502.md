## Summary

Keep PR #38502 open. Current main still lacks the active-run Ctrl+C status hint, while Ctrl+C remains a local clear/warn/exit flow and `/abort` remains the explicit abort path. The provided PR diff is still a narrow one-file TUI UX fix against the reported mismatch in #38501, and it already incorporated the prior review feedback by advertising `/abort` only rather than unconditional Esc behavior.

## Best Possible Solution

Keep PR #38502 open or land an equivalent narrow change on main: compute an active-run hint from `activeChatRunId` inside `handleCtrlC`, append `; run still active (/abort to stop)` to only the clear and warn Ctrl+C statuses, leave exit behavior and abort/keybinding semantics unchanged, and add focused status-text coverage if the TUI status surface is practical to test. The linked issue #38501 remains the right user-problem anchor.
