## Summary

Keep open. Current `main` still has the old TUI Ctrl+C status messages with no `activeChatRunId`-based hint, while `/abort` remains the explicit active-run abort path. The PR is a narrow text-only fix for the linked UX bug #38501 and is not obsolete or superseded by shipped behavior.

## Best Possible Solution

Keep this PR open or land an equivalent focused TUI change on `main`: append an `activeChatRunId`-gated hint to the Ctrl+C clear/warn status messages, word it around `/abort` as the reliable stop command, and add focused status-message coverage if the TUI handler surface can be tested cleanly.
