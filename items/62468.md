## Summary

Keep open. Current main still does not resolve a bare Feishu display name/username like `jaydenli` as a user target, and the remaining behavior matches the reported `Unknown target ... Hint: <chatId|user:openId|chat:chatId>` failure path.

## Evidence

- **Feishu target resolver only recognizes explicit IDs/prefixes:** Feishu registers `looksLikeFeishuId` plus the existing hint, but no plugin `resolveTarget` fallback. `looksLikeFeishuId` accepts provider/user/chat prefixes and `oc_`/`ou_` IDs, not a bare username like `jaydenli`.
  - file: [extensions/feishu/src/channel.ts:1166](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/extensions/feishu/src/channel.ts#L1166)
  - command: `nl -ba extensions/feishu/src/channel.ts | sed -n '1138,1178p'`
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Bare targets default to group lookup:** The shared resolver treats an unprefixed target as `group` unless the plugin infers otherwise. Feishu does not provide `inferTargetChatType`, so `jaydenli` searches group directories rather than user peers.
  - file: [src/infra/outbound/target-resolver.ts:144](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/infra/outbound/target-resolver.ts#L144)

- **Unknown-target error remains generic:** On a directory miss with no plugin fallback, current main returns `Unknown target "<raw>" for <provider>. Hint: <hint>`, which is the same class of error reported in the issue.
  - file: [src/infra/outbound/target-resolver.ts:428](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/infra/outbound/target-resolver.ts#L428)
