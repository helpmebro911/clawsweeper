## Summary

Keep this PR open. Current main still does not expose configurable memory-lancedb embedding timeout or retry settings, and the PR remains the active implementation candidate rather than obsolete cleanup. It needs maintainer review before merge for current-main drift, the unresolved baseUrl/API-key security concern, and an unexplained generated checksum change.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is a refreshed, scoped memory-lancedb change that adds validated embedding.timeoutMs and embedding.maxRetries to config.ts, openclaw.plugin.json, UI hints, runtime OpenAI client/request wiring, and focused tests, while preserving current encoding_format: "float" behavior and resolving the configurable baseUrl/API-key security concern plus generated checksum drift before merge.

## Risks / Open Questions

- The PR should not merge until maintainers resolve the configurable embedding.baseUrl security concern: the configured OpenAI API key can be sent to a non-default endpoint, and retries may repeat that request.
- The PR patch is stale against current main’s encoding_format: "float" behavior; it must preserve the production request shape and update tests accordingly after rebase.
- The PR changes docs/.generated/plugin-sdk-api-baseline.sha256 without an obvious plugin SDK API surface change in the patch, so that generated drift should be removed or regenerated coherently.
- The PR body still mentions a LaunchAgent ThrottleInterval change, but the current changed-file list no longer includes the daemon launchd file; the scope/body should be cleaned up before merge.
