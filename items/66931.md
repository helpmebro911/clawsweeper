## Summary

Keep open. Current `main` still hardcodes indexing concurrency to `4` whenever batch mode is disabled, so the reported bug is not implemented on `main`. The PR remains materially useful, and its follow-up commit incorporated the review suggestion to reuse the already-clamped `this.batch.concurrency`; conservative cleanup should not close it.

## Evidence

- **Current main still ignores configured concurrency outside batch mode:** `getIndexConcurrency()` on `main` returns `this.batch.concurrency` only when `this.batch.enabled` is true; otherwise it falls back to `EMBEDDING_INDEX_CONCURRENCY` (`4`). That means a user-set `agents.defaults.memorySearch.remote.batch.concurrency` is ignored whenever batch mode is unavailable or disabled.
  - file: [extensions/memory-core/src/memory/manager-embedding-ops.ts:466](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/memory-core/src/memory/manager-embedding-ops.ts#L466)
  - command: `nl -ba extensions/memory-core/src/memory/manager-embedding-ops.ts | sed -n '430,500p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Resolved batch config already carries the user concurrency value:** `resolveBatchConfig()` always computes `concurrency: Math.max(1, batch?.concurrency ?? 2)`, while `enabled` depends on both config and provider batch support. So current `main` can have a resolved user concurrency value that `getIndexConcurrency()` still discards when `enabled` is false.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:1062](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/memory-core/src/memory/manager-sync-ops.ts#L1062)

- **The ignored value affects real indexing work:** Memory indexing uses `runWithConcurrency(..., this.getIndexConcurrency())` for both file-entry construction and file indexing tasks, so the fallback value directly controls request fan-out during indexing.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:688](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/memory-core/src/memory/manager-sync-ops.ts#L688)

Current `main` does not honor that when batch mode is unavailable, matching the PR's reported mismatch.
