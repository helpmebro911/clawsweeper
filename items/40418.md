## Summary

Keep #40418 open. Current main has partial `/new`/`/reset` preservation and startup hydration, but it still saves bounded raw user/assistant transcript text to slugged memory files and reloads existing files as untrusted notes. It does not implement the requested synthesis, session-state JSON, full-transcript anchor links, or generated daily continuation package. Related PR #50584 remains open/unmerged, and related bug #69943 shows the raw persistence path still has unresolved safety implications.

## Best Possible Solution

Keep #40418 open until maintainers either land a scoped core implementation or explicitly redirect the feature to a ClawHub/context-engine plugin design. A strong implementation should coordinate with #50584 and #69943: opt-in synthesis, sanitizer/filtering before persistence, compact continuation packages, canonical daily summary append, receipt links to full transcripts, timezone-aware filenames, and regression tests for `/new`/`/reset` preservation and startup hydration.
