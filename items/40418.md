## Summary

Keep open. Current `main` has partial support for reset-time preservation via the bundled `session-memory` hook and first-turn startup-context loading, but it does not implement the full request for automatic cross-session synthesis with links/anchors back to full transcripts.

## Evidence

- **Bundled feature is narrower than the issue request:** The docs describe `session-memory` as a bundled hook that saves session context on `/new` or `/reset`, specifically by extracting the last 15 user/assistant messages and writing `<workspace>/memory/YYYY-MM-DD-slug.md`. That is preservation, but not the requested synthesized continuation package with transcript links.
  - file: [docs/automation/hooks.md:156](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/docs/automation/hooks.md#L156)
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **Implementation only writes recent messages into one markdown artifact:** The handler resolves the prior session, reads a configurable recent-message window, generates only a filename slug via LLM, and writes a markdown file containing session metadata plus `## Conversation Summary`. There is no code here that creates anchor links, stores the full transcript separately for drill-down, or synthesizes durable facts/TODOs/preferences into a structured package.
  - file: [src/hooks/bundled/session-memory/handler.ts:127](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/src/hooks/bundled/session-memory/handler.ts#L127)
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **Transcript extraction is limited, not full-fidelity export:** The transcript helper reads JSONL, filters to user/assistant text messages, drops command messages and some inter-session provenance, and returns only the last `messageCount` entries. That does not match the issue's proposed full conversation export plus summary/linking flow.
  - file: [src/hooks/bundled/session-memory/transcript.ts:24](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/src/hooks/bundled/session-memory/transcript.ts#L24)
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **New-session startup re-injects raw daily notes rather than synthesizing them:** The startup-context path enumerates recent `memory/*.md` files and prepends their contents as quoted 'Untrusted daily memory' blocks on the first `/new` or `/reset` turn. It loads existing notes verbatim; it does not synthesize prior sessions into a canonical daily summary or generate transcript anchors.
  - file: [src/auto-reply/reply/startup-context.ts:309](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/src/auto-reply/reply/startup-context.ts#L309)
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **Product docs confirm first-turn raw memory loading only:** The system-prompt docs say `memory/*.md` daily files are not part of normal bootstrap, except that bare `/new` and `/reset` can prepend recent daily memory as a one-shot startup-context block for the first turn. That matches reinjection of notes, not the requested synthesis/linking workflow.
  - file: [docs/concepts/system-prompt.md:119](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/docs/concepts/system-prompt.md#L119)
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)

## Risks / Open Questions

- The issue is partially addressed already, so maintainers may want to retitle or narrow it to the remaining gap: structured synthesis and transcript-linking, not basic preservation.
- There is active surface area around this feature; separate bugs in the current hook/startup-context path could complicate whether the desired behavior should be expanded in core or adjusted through the hook mechanism first.
