## Summary

Keep open. The current main implementation of `startGatewayConfigReloader` still constructs a chokidar watcher only for `opts.watchPath` and does not import or call `collectIncludePathsRecursive`, nor dynamically add/unwatch include paths. The related PR #59632 was closed, but the requested behavior is not present on current main.

## Evidence

- **Current reloader watches only the main config path:** `startGatewayConfigReloader` creates `chokidar.watch(opts.watchPath, ...)` and registers add/change/unlink handlers on that watcher; there is no include-path sync, `watcher.add()`, or `watcher.unwatch()` logic for `$include` files.
  - file: [src/gateway/config-reload.ts:7](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/src/gateway/config-reload.ts#L7)
  - command: `web open raw.githubusercontent.com/openclaw/openclaw/main/src/gateway/config-reload.ts`
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Include scanner exists but is not wired into config reload:** `collectIncludePathsRecursive()` exists and recursively collects `$include` paths, matching the infrastructure mentioned in the issue, but current `config-reload.ts` does not use it.
  - file: [src/config/includes-scan.ts](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/src/config/includes-scan.ts)
  - command: `web open raw.githubusercontent.com/openclaw/openclaw/main/src/config/includes-scan.ts`
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Current reloader tests do not cover include-file watching:** `config-reload.test.ts` covers reload planning, missing snapshots, restart callback failures, and diffing, but there are no tests asserting included files are added to or removed from the watcher.
  - file: [src/gateway/config-reload.test.ts](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/src/gateway/config-reload.test.ts)
  - command: `web open raw.githubusercontent.com/openclaw/openclaw/main/src/gateway/config-reload.test.ts`

- **Issue discussion points to an attempted fix but not a landed implementation:** The issue timeline cross-references PR #59632, `feat: watch $include-referenced files for config hot-reload`, which is closed. Current main still lacks the described change, so the issue should not be closed as implemented.
