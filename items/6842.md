## Summary

Keep open. Current `main` only documents and implements same-instance agent/session A2A flows, not external A2A protocol support with Agent Cards or HTTP JSON-RPC. A plugin/ClawHub path is plausible, but the repo’s scope docs frame that as a guardrail rather than a hard exclusion, and the provided GitHub context shows recent activity plus linked open PR `#60502`, so there is not strong enough evidence for a conservative close.

## Evidence

- **Current A2A config is internal session access:** `tools.agentToAgent` is documented as cross-agent targeting for session tools, not as an external protocol surface.
  - file: [docs/gateway/configuration-reference.md:2428](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/gateway/configuration-reference.md#L2428)

  - file: [docs/tools/acp-agents.md:336](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/tools/acp-agents.md#L336)
