## Summary

Keep #39138 open. Current main still replays pending outbound delivery queue entries on gateway startup using retry/backoff and a wall-clock recovery budget, but I found no age-based expiry or max replay count that would prevent old queued messages, including 15-hour-old entries, from being resent once eligible.
