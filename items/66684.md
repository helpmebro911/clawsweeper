## Summary

Keep open. Current main still contains the actionable remote-installer pipe patterns in Docker build surfaces and user-facing Docker rebuild docs. The cited credential-pattern examples look stale or false-positive, so the issue should be narrowed rather than closed.

## Best Possible Solution

Keep the issue open and narrow it to Docker/supply-chain hardening. Replace or justify the Bun and Homebrew remote installer pipes in `Dockerfile`, `Dockerfile.sandbox-common`, and `docs/install/docker.md` with pinned or verified installation steps where practical, then separately acknowledge the stale credential-pattern examples as false positives under the security policy.
