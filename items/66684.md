## Summary

Keep open. Current main still contains the actionable remote installer execution patterns in Docker build surfaces and public Docker guidance. The credential-pattern examples look stale or false-positive, so the issue should be narrowed to Docker/supply-chain hardening rather than closed.

## Best Possible Solution

Keep this issue open and narrow it to Docker/supply-chain hardening. Replace or explicitly justify the Bun and Homebrew remote installer execution in `Dockerfile`, `Dockerfile.sandbox-common`, and Docker docs with pinned or verified installation steps where practical, then separately acknowledge the stale credential-pattern examples as false positives under the existing security reporting policy.
