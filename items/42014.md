## Summary

Keep this PR open. Current main still lacks the requested session sync observability in memory manager status and `openclaw memory status`, while the PR is a focused feature with useful review follow-up. It needs a rebase from the old `src/memory` / `src/cli/memory-cli.ts` layout into the current `extensions/memory-core` plugin-owned implementation before normal maintainer review can decide whether to land it.

## Best Possible Solution

Keep the PR open and ask for a rebase into `extensions/memory-core`. The best implementation path is to add a typed session-sync status helper/state on the memory-core manager, expose it through `MemoryIndexManager.status()` under `custom.sessionSync`, render it in `openclaw memory status` and JSON output, preserve dirty/error state on failed session syncs, avoid transcript-content leakage, and add focused manager plus CLI deep-output tests. During review, carry forward the timestamp and Map aggregation fixes from `2df928fbc4df1b5e6f1137c04cb6cbb2bf547706`.
