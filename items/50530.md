## Summary

Keep open. Current main still preserves single mid-paragraph newlines in the Telegram Markdown-to-HTML send path, and there is no Telegram collapse-newlines option in the exposed formatting config. Existing tests also codify single-newline preservation for chunking, so the requested cosmetic behavior remains real work rather than already implemented cleanup.

## Evidence

- **Telegram formatter preserves source soft breaks:** Telegram outbound Markdown is parsed through markdownToIR and then rendered to Telegram HTML; markdownToIR currently handles both softbreak and hardbreak tokens by appending a literal newline, so a mid-paragraph source newline survives rendering.
  - file: [extensions/telegram/src/format.ts:75](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/extensions/telegram/src/format.ts#L75)
  - command: `nl -ba extensions/telegram/src/format.ts | sed -n '1,260p'; nl -ba src/markdown/ir.ts | sed -n '620,980p'`
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **Single-newline preservation is expected by tests:** The chunking tests explicitly expect newline mode to keep a single paragraph as "Line one\nLine two" rather than collapsing it to a space.
  - file: [src/auto-reply/chunk.test.ts:491](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/auto-reply/chunk.test.ts#L491)

  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **Telegram config exposes formatting fields but no collapse option:** Telegram docs list the high-signal formatting/delivery fields as textChunkLimit, chunkMode, linkPreview, and responsePrefix; targeted searches found no collapseNewlines or equivalent Telegram setting on current main.
