## Summary

Keep open. Current `main` still does not expose `session.readRecentMessages` (or an equivalent typed `SessionHistoryMessage` contract) on the channel-plugin runtime, and the nearest existing history readers are either internal helpers or gateway-only subagent APIs.

## Evidence

- **Channel runtime type lacks the requested method:** `PluginRuntimeChannel.session` currently exposes only `resolveStorePath`, `readSessionUpdatedAt`, `recordSessionMetaFromInbound`, `recordInboundSession`, and `updateLastRoute`; there is no `readRecentMessages` slot on `main`.

- **Channel runtime wiring also lacks it:** `createRuntimeChannel()` wires the same five session helpers and does not attach any recent-history reader.
  - file: [src/plugins/runtime/runtime-channel.ts:220](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/plugins/runtime/runtime-channel.ts#L220)
