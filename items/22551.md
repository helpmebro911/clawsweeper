## Summary

Current `main` does not expose a configurable Discord-compatible server base URL. The Discord integration still hardcodes Discord REST and gateway endpoints, and the only related escape hatch is a loopback-only HTTP(S) proxy setting. That looks like a partial workaround, not implementation of the requested feature.

## Evidence

## Risks / Open Questions

- `channels.discord.proxy` may let some advanced users bridge a Discord-compatible server through a local reverse proxy, so the issue is partially addressable today.
- Because that path is indirect, loopback-only, and not documented as Discord-compatible server support, closing as implemented would be too aggressive.
