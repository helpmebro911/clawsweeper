## Summary

Current `main` still does not appear to fix the reported LiteLLM onboarding regression. The bundled LiteLLM provider only prompts for an API key, its onboarding config falls back to `http://localhost:4000` unless `models.providers.litellm.baseUrl` already exists, and the docs still treat custom LiteLLM URLs as manual config rather than an onboarding step. I found no evidence on `main` of a shipped fix for the skipped LiteLLM URL prompt or the resulting one-model fallback behavior.
