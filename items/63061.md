## Summary

Keep open. Current main still implements the behavior this PR targets: `--profile` with `OPENCLAW_HOME` derives a profile-suffixed state dir, while the generic config/state resolver treats `OPENCLAW_HOME` as the home root and defaults to an unsuffixed `.openclaw`. The PR remains a focused implementation candidate rather than obsolete or superseded.

## Best Possible Solution

Keep this PR open for maintainer review. Decide the intended precedence between `OPENCLAW_HOME` physical isolation and named profile suffixing, then either land a normalized version of this PR with matching help/docs and daemon/service-path alignment, or explicitly document that profile suffixing should win under `OPENCLAW_HOME` and close the PR after that product decision.
