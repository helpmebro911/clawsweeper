## Summary

Keep this PR open. Current main still has the behavior the PR targets: CLI profile handling derives a suffixed state dir under OPENCLAW_HOME, while the generic config path resolver defaults OPENCLAW_HOME to an unsuffixed .openclaw state dir. The provided PR context remains a focused implementation candidate and shows the later head commit addressed the review feedback around sentinel OPENCLAW_HOME values and dev-profile coverage.

## Best Possible Solution

Keep this PR open for maintainer review. The likely path is to land a normalized version of the PR if OPENCLAW_HOME is intended to be the physical isolation boundary, with matching docs/help updates and a quick review of service install/start paths so CLI and daemon state resolution stay aligned.
