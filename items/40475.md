## Summary

Keep open. Current `main` has Feishu topic/thread routing and thread-aware replies, but it still gates group handling on an explicit bot mention and does not expose a Feishu config option for implicit thread follow-ups.

## Evidence

- **Mention detection still depends on explicit @mentions:** `parseFeishuMessageEvent()` sets `mentionedBot` only from `checkBotMentioned(event, botOpenId)`. Thread metadata (`root_id`, `thread_id`) is captured separately and is not used to mark a thread reply as an implicit mention.
  - file: [extensions/feishu/src/bot.ts:106](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/extensions/feishu/src/bot.ts#L106)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '90,140p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Group messages are still dropped when mention gating is on:** In the main Feishu dispatch path, after resolving group policy, the handler returns early on `if (requireMention && !ctx.mentionedBot)`. There is no exception for replies inside a bot-started topic/thread.
  - file: [extensions/feishu/src/bot.ts:474](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/extensions/feishu/src/bot.ts#L474)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '420,500p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **No config surface for thread-follow-up mention bypass:** The Feishu config schema exposes `requireMention`, `groupSessionScope`, `topicSessionMode`, and `replyInThread`, but nothing like `threadFollowUp` or `requireMentionInThread`.
  - file: [extensions/feishu/src/config-schema.ts:141](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/extensions/feishu/src/config-schema.ts#L141)

  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Docs still describe only global/per-group mention gating:** The Feishu docs document `channels.feishu.requireMention` and `channels.feishu.groups.<chat_id>.requireMention` as the mention controls. They do not document any thread-specific follow-up behavior or override.
  - file: [docs/channels/feishu.md:68](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/docs/channels/feishu.md#L68)
