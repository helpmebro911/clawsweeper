## Summary

Keep PR #48293 open. Current main does not contain the proposed pure transcript-tail detector or the `agents.defaults.compaction.guard` config scaffold it is stacked on, and the linked umbrella issue #48238 remains open with a concrete production-session failure mode. Main has adjacent compaction safeguard and quality-audit behavior, but it does not classify the four requested tail-signal families: repeated tool failures, duplicate assistant drift, stale system/reminder recurrence, and trailing ungrounded user turns.

## Best Possible Solution

Keep the PR or its canonical umbrella issue active. Maintainers should decide whether to land the detector after resolving the open config-scaffold dependency (#48278), or supersede the whole stack with a smaller maintainer-owned implementation that still provides a pure tail-signal detector before runtime guard wiring.
