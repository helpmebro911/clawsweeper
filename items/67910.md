## Summary

Keep open. Current main still renders `openclaw memory status` dreaming state through the deep-promotion config, so the linked light-only status problem is not implemented on main. This PR is still a plausible implementation thread, but its latest patch needs maintainer review because it now expects light-only and REM-only configs to show `Dreaming: off`, and it bundles adjacent `--force` and timezone cleanup changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrower memory-core status change that reports the effective scheduled managed sweep accurately while also exposing configured light/deep/REM phase state clearly enough to solve #67868. Address the existing deep-disabled and `limit=0` review concerns, and split the unrelated `--force` and timezone cleanup unless maintainers want them included.
