## Summary

Keep #71839 open. Current main still has the Telegram draft-preview behavior the PR removes: when reply threading is enabled, draft streams receive an implicit `replyToMessageId`, and the agent system prompt still tells assistants to prefer `[[reply_to_current]]` for native replies. The PR is recent, unmerged, non-maintainer-authored, and has no protected labels, so it should receive normal maintainer review rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to preserve the Telegram behavior change and regression coverage, keep explicit native reply tags working for useful quote context, update the system prompt guidance as proposed, and either split or justify the QA-lab mock-server changes before landing.
