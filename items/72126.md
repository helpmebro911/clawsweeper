## Summary

Keep open. Current main still reads the OpenAI Realtime API key from provider config or process.env.OPENAI_API_KEY and uses that value directly for browser-session and voice-bridge auth, so the keychain-ref bug described in #72120 is not implemented on main. PR #72126 is a fresh, focused bug-fix candidate rather than an obsolete cleanup item, though it should get regression coverage and resolve the review concerns before merge.

## Best Possible Solution

Keep #72126 open as an active bug-fix PR. Before merge, add focused regression tests proving keychain refs are resolved before browser-session and voice-bridge auth, avoid permanently caching transient lookup failures, and consider a small shared OpenAI realtime API-key resolver if realtime transcription should receive the same behavior.
