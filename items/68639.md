## Summary

Keep #68639 open. Current main still resolves the active memory manager with `agentSessionKey` but drops that session key when `searchMemoryWiki()` calls shared memory search. Shared memory/QMD already accept `sessionKey` and use it for scope checks, and the current memory-core search tool forwards the same context correctly. The PR is a focused, still-useful fix with a matching regression test.

## Best Possible Solution

Keep this PR open for maintainer review. The best code path is to merge or recreate the focused change in `extensions/memory-wiki/src/query.ts`: call shared memory search with `{ maxResults, sessionKey: params.agentSessionKey }`, and keep the regression test that asserts `wiki_search` forwards the active session key.
