## Summary

Keep #25883 open. Current main still does not implement the requested Redis/BullMQ-style horizontal job orchestration: cron and task tracking remain Gateway-local/file-backed or in-process, and the existing maintainer/Codex issue comment explicitly kept the ScalyClaw prototype attached as useful prior art rather than treating the request as implemented or out of scope.

## Best Possible Solution

Keep #25883 open for maintainer design review. The best path is to decide whether OpenClaw should support a distributed scheduling/job abstraction or plugin-capable queue backend with worker claiming, retry/backoff, dead-letter/failure state, cron support, observability, and shared-state contracts, while preserving the linked ScalyClaw prototype as prior art.
