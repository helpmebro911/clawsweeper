## Summary

Keep this issue open. Current main still lacks a gateway startup/runtime-auth diagnostic for legacy CLAWDBOT_*/MOLTBOT_* env vars, and PR #53667 is an open, unmerged implementation candidate that explicitly targets this issue with closing syntax.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #53667. The best product fix belongs in the gateway startup/runtime-auth path: detect CLAWDBOT_*/MOLTBOT_* keys early, emit clear OPENCLAW_* replacement guidance plus doctor/migration pointers, improve misleading auth/channel errors, and add focused regression tests while preserving the intentional removal of legacy fallback support.
