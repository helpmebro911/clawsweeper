## Summary

Do not close this PR. Current `main` already fixes the narrow `delivery.mode="none"` symptom, but it does not contain this PR's mode-aware implementation; `main` still keys delivery status off `plan.requested`, while webhook plans are marked `requested: false`, so the PR is not redundant and should not be closed as implemented on `main`.

## Evidence

- **Current main still uses the broad `!requested` guard:** `resolveDeliveryState` returns `not-requested` before checking `delivered`, based solely on `resolveCronDeliveryPlan(...).requested`. That is different from the PR's `plan.mode === "none"` guard.
  - file: [src/cron/service/timer.ts:261](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/cron/service/timer.ts#L261)
