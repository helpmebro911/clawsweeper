## Summary

Keep open. Current `main` does not implement this PR’s fix: `resolveDeliveryState` still returns `"not-requested"` for any delivery plan with `requested === false`, and `resolveCronDeliveryPlan` marks `requested` false for `webhook` as well as `none`. The PR discussion and patch show the contributor already addressed the earlier webhook-regression review by switching the guard to `plan.mode === "none"` and adding webhook coverage, so this is still active, useful work rather than conservative cleanup.
