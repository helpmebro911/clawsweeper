## Summary

Keep PR #68924 open. Current main still has the ambiguous `Cost` wording and does not thread `usageCostSummary` into the overview cards, so the PR is not implemented or superseded. The PR is recent and focused, but the existing review discussion identifies real follow-up work around i18n, model-auth preservation, date-window semantics, and incomplete cost data that should be resolved before merge.

## Best Possible Solution

Keep the PR open and ask for a focused revision: preserve the model-auth overview card, keep all changed labels behind existing i18n keys, pass `usageCostSummary` into overview cards, compute any 7d/30d hints from true calendar windows in the same data scope as the displayed value, hide or qualify cost-window hints when `missingCostEntries` indicates incomplete data, and add targeted overview-card tests for those cases.
