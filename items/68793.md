## Summary

Keep open. Current main has already moved the raw WebSocket dispatch JSON to debug, but the QQ Bot gateway still logs inbound user-visible message text at info during normal processing, so the central privacy/noise request is not fully implemented.

## Best Possible Solution

Keep this issue open and fix it in the QQ Bot plugin. Demote routine message-processing logs to debug or replace them with metadata-only info logs such as message id, sender/peer id, type, and content length; keep full payloads behind debug/explicit verbose paths. Add focused QQ Bot gateway tests that assert info logs do not include inbound message content while debug logs may include opt-in diagnostics.
