## Summary

Keep this PR open. Current main still emits `gateway.trusted_proxies_missing` for `gateway.bind: "loopback"`, so the reported noisy audit finding is not implemented on main. The PR is still a plausible fix candidate, but it is stale against the current split gateway audit implementation and its raw `bind !== "loopback"` condition would misclassify effective loopback cases such as custom loopback binds or non-container auto binds.

## Best Possible Solution

Keep this PR open and request a revision against `src/security/audit-gateway-config.ts`. The fix should base `gateway.trusted_proxies_missing` on effective gateway exposure, stop warning for definitely local-only loopback-resolving binds, still warn for genuinely exposed reverse-proxy cases, and add regression coverage for plain loopback, LAN, custom loopback, and exposed custom/auto behavior.
