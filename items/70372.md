## Summary

Keep #70372 open. Current main has phase-aware suppression for explicit `commentary` / `final_answer` metadata, but it does not track that an unphased Anthropic/Bedrock assistant message has emitted native thinking events. The Anthropic stream still emits thinking and text blocks without phase metadata, so the reported leak path remains plausible on main. The PR should not be auto-closed; it needs a scoped implementation and regression coverage, or an explicit maintainer decision to fold it into the broader related #70344 work.

## Best Possible Solution

Keep the PR open for maintainer review. The best fix is a provider-scoped regression-tested change: derive phase-aware delivery only for unphased native Anthropic/Bedrock thinking messages, preserve that state across stream item resets, and add a test with `thinking_start`/`thinking_delta`/`thinking_end` followed by unphased `text_delta`/`text_end` proving narration does not reach partial replies, block replies, or `assistantTexts` while OpenAI phase-pending behavior remains unchanged. If #70344 becomes the canonical broader fix, explicitly close this PR as superseded then.
