## Summary

Keep open. Current main does not implement the PR’s core cron-session heartbeat targeting guard: exec/background completion wakes still preserve cron run session keys, and heartbeat forced-session resolution still accepts cron run sessions. The PR is not obsolete, though the provided review comments show it likely needs revision to preserve agent scoping when dropping the cron run session key.

## Best Possible Solution

Keep the PR open or replace it with a revised core fix that maps cron-run-originated heartbeat wakes to the originating agent’s main heartbeat target instead of the isolated cron run session. The revision should preserve agent scoping for non-default agents, avoid unscoped all-agent wakes, use `isCronSessionKey`/canonical parsing consistently, and add regression coverage for `scopedHeartbeatWakeOptions`, forced `runHeartbeatOnce` cron session keys, and non-default-agent cron exec completions.
