## Summary

Keep this issue open. Current main still has the reported policy behavior: mutating tool errors return `showWarning: true` before the recoverable-error suppression path, and regression tests explicitly expect a recoverable mutating error to produce a warning. The existing edit recovery wrapper is useful partial mitigation, and the discussion around #59868 and PR #60664 is relevant, but no merged current-main fix covers this warning-policy request.

## Best Possible Solution

Keep this issue open. The likely maintainer path is a focused change around `src/agents/pi-embedded-runner/run/payloads.ts` and possibly upstream `lastToolError` clearing, with regression tests showing that recoverable or post-write-applied mutating failures do not emit false `failed` notifications while real mutating failures remain visible. Coordinate with #59868 and PR #60664, but do not treat either as shipped replacement work.
