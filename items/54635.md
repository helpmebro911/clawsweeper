## Summary

Keep this issue open. Current main still emits user-facing `⚠️ ... failed` payloads for recoverable mutating tool errors; the current suppression is limited to cases where the assistant explicitly acknowledges the failed action, and nearby edit recovery work does not cover the broader warning-policy/state problem reported here.

## Best Possible Solution

Keep this issue open. The best path is a focused fix around `src/agents/pi-embedded-runner/run/payloads.ts` and possibly `src/agents/pi-embedded-subscribe.handlers.tools.ts`: suppress false mutating failure notifications only for recoverable or proven-applied failures, clear `lastToolError` when a mutation is proven applied, and add regression tests proving recoverable/proven-applied mutating failures do not emit `failed` payloads while real mutating failures remain visible. Coordinate with #59868 and PR #60664, but do not close this issue until the warning-policy behavior is actually fixed on main.
