## Summary

Keep #29387 open. Current main still loads bootstrap files only from the resolved workspace directory, while `agentDir` remains a separate per-agent state/auth directory. The bootstrap resolver and major callsites do not pass or inspect `agentDir`, so the requested warning or merge/override behavior is still missing. The prior fix PRs mentioned in the issue were closed unmerged and their head commits are not present in this checkout.

## Best Possible Solution

Keep #29387 open as the canonical bug. The best maintainer path is either a narrow diagnostic that threads explicit agentDir context into bootstrap callsites and warns on recognized bootstrap filenames in agentDir, or a deliberate per-agent bootstrap merge/override contract for explicitly configured agentDir files with allowlist filtering, hook precedence, caller coverage, regression tests, and docs that clearly distinguish workspace bootstrap files from agentDir state.
