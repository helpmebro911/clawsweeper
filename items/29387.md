## Summary

Keep #29387 open. Current main still resolves bootstrap files from the configured workspace only; `agentDir` is resolved separately for state/auth and is available in some runtime params, but it is not passed into `resolveBootstrapContextForRun`, so neither agentDir merge/override nor the requested misplaced-file warning exists. The prior fix PRs discussed in the issue (#29450 and #29545) are closed and their head commits are not present in this checkout.

## Best Possible Solution

Keep #29387 open as the canonical bug. The best maintainer path is either a narrow runtime diagnostic that threads explicit agentDir/agentId context into bootstrap callsites and warns on recognized bootstrap filenames in agentDir, or a deliberate merge/override contract for explicitly configured agentDir files with allowlist filtering, hook precedence, caller coverage, regression tests, and docs that distinguish workspace bootstrap files from agentDir state.
