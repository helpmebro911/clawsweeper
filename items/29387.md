## Summary

Keep open. Current `main` still resolves injected bootstrap files exclusively from `workspaceDir`; `agentDir` is a separate runtime-state path and I found no current warning/diagnostic path for bootstrap-named files placed there. The issue’s core report still matches current code.

## Evidence

- **Bootstrap resolution still only reads workspaceDir:** `resolveBootstrapFilesForRun()` accepts `workspaceDir` and loads either cached workspace files or `loadWorkspaceBootstrapFiles(params.workspaceDir)`; there is no `agentDir` input or merge/warning path here.
  - file: [src/agents/bootstrap-files.ts:217](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/bootstrap-files.ts#L217)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Workspace loader enumerates only workspace bootstrap filenames under the workspace directory:** `loadWorkspaceBootstrapFiles()` builds file paths by joining the resolved workspace directory with `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, and `MEMORY.md` only.
  - file: [src/agents/workspace.ts:477](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/workspace.ts#L477)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **agentDir remains a distinct state path, separate from workspace:** `resolveAgentWorkspaceDir()` and `resolveAgentDir()` are separate resolvers; `agentDir` resolves to `.../agents/<id>/agent`, while workspace context comes from the workspace resolver.
  - file: [src/agents/agent-scope-config.ts:149](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/agent-scope-config.ts#L149)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Docs still describe workspace as the context location and agentDir as auth/state storage:** The workspace doc says the workspace is the only working directory used for workspace context, and the configuration reference documents per-agent profiles under `<agentDir>/auth-profiles.json`, not bootstrap prompt files.
  - file: [docs/concepts/agent-workspace.md:9](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/concepts/agent-workspace.md#L9)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **No current agentDir-bootstrap warning text found:** Repository search for warning/diagnostic strings around ignored `agentDir` bootstrap files returned no matches, which supports the report that the current behavior remains silent.
