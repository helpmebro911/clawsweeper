## Summary

Keep #12855 open. Current main has a real but narrower default-off package auto-updater under `update.auto`, but it does not implement the requested first-class workflow: configurable wall-clock/cron schedule, timezone, confirm and notify-only modes, configured channel notifications, or post-restart version/changelog/health messaging.

## Best Possible Solution

Keep #12855 open as the canonical implementation tracker for the first-class auto-update workflow. Extend the existing `update.auto` package updater with schema/types/help/docs for `mode`, `schedule`, `timezone`, and notification settings; add HH:MM or cron timezone scheduling; support `notify-only`, `confirm`, and `silent` flows through configured channels; persist post-restart update state; and report previous/new version, changelog link or summary, and health after restart. Treat #65011 as the separate UI/discoverability follow-up and #12896 as historical closed/unmerged work.
