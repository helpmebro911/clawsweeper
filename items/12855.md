## Summary

Keep #12855 open. Current main has a useful default-off package auto-updater under `update.auto`, but it still does not implement the requested first-class workflow: configurable confirmation/notify-only/silent modes, HH:MM or cron timezone scheduling, configured post-update channel notification, and update-health summary. The related PR #12896 closed unmerged, and #65011 is an open UI/discoverability follow-up rather than a replacement for this issue.

## Best Possible Solution

Keep #12855 open as the product and implementation tracker for the remaining first-class update workflow. The likely path is to extend `update.auto` through config schema/types/help/docs, runtime scheduling, confirmation or notify-only channel delivery, post-update restart notification, changelog or health summary support, and tests. Keep #65011 as the UI/discoverability follow-up for the existing updater, not as a replacement for this workflow request.
