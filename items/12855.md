## Summary

Keep #12855 open. Current main has a default-off package auto-updater under `update.auto`, but it is still narrower than the requested first-class workflow: no wall-clock/cron schedule, timezone, confirm/notify-only modes, configured channel notifications, or post-restart version/changelog/health message.

## Best Possible Solution

Keep #12855 open as the canonical engine/workflow tracker. Extend the existing core `update.auto` package updater with schema/types/help/docs for `mode`, `schedule`, `timezone`, `notifyAfterUpdate`, and `notifyChannel`; add HH:MM or cron timezone scheduling; support `notify-only`, `confirm`, and `silent` flows through configured channel targets; persist update state across restart; and report previous/new version, changelog link or summary, and health after restart. Leave #65011 as the separate Control UI/onboarding/discoverability follow-up.
