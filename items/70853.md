## Summary

Keep #70853 open. Current main still rotates an auto-reply session to a fresh sessionId/sessionFile for the same stable session key without replaying or injecting prior user/assistant turns, and the directly related fix PR #70898 is still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #70853 open as the canonical bug until PR #70898 or an equivalent main-branch fix lands. The best code direction is to preserve user-visible continuity in resetReplyRunSession for silent compaction/role-ordering rotations, either by seeding a bounded prior user/assistant transcript tail into the new live session file or by injecting equivalent preserved context into the next model turn, while keeping explicit /new and /reset as fresh sessions and adding regression coverage for the Telegram/direct-message style session-key rebind case.
