## Summary

Keep #70853 open. Current main still has the silent auto-reply reset path that rotates an existing session key to a fresh session id without replaying or hydrating prior user/assistant transcript history, and the directly linked fix PR #70898 remains open rather than shipped on main.

## Best Possible Solution

Keep #70853 open and use PR #70898, or an equivalent maintainer-approved patch, as the implementation path: preserve recent prior user/assistant turns for silent auto-reply session rotations in src/auto-reply/reply/agent-runner-session-reset.ts while keeping explicit /new and /reset fresh, and add regression coverage proving a Telegram/direct-style stable session key keeps usable context across the rebind.
