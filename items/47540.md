## Summary

Current main still does not implement PR #47540's core request: the Control UI context notice remains a single model-context usage notice, there is no `gateway.controlUi.chat.showPricingThreshold` config surface, and the bootstrap contract does not carry a chat context-notice setting. The related merged maintainer PR #71297 improved freshness/compaction CTA behavior but explicitly left #47540 related-not-closed because threshold behavior was outside that PR's scope.

## Best Possible Solution

Keep #47540 open and ask for a focused rebase onto current main: preserve the useful config/bootstrap/UI split for pricing-threshold hints, align with the merged #71297 context freshness and compact CTA code, base threshold warnings on fresh context/request usage rather than cumulative run input, and update the Control UI tests plus config docs/schema baselines.
