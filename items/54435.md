## Summary

Current main still matches the reported literal `session.store` failure mode: `sessions.list` loads the combined gateway session store, but the combined-store helper returns after reading one non-templated store as the default agent and never reaches all-agent discovery. The linked #54716 is an open, unmerged implementation candidate for the same bug, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while #54716 is reviewed and either landed or replaced with an equivalent fix. The fix should update `src/config/sessions/combined-store-gateway.ts` so literal per-agent paths under `agents/<id>/sessions/sessions.json` use all-agent discovery while preserving true shared literal-store behavior, and it should add regression coverage for that literal path.
