## Summary

Keep open. Current main still routes `sessions.list` through `loadCombinedSessionStoreForGateway`, and that helper still returns immediately for a literal non-templated `session.store`, reading only that one file and assigning entries to the default agent. The all-agent discovery helper can scan disk-backed agent stores, but this path is only reached after the literal-store branch is skipped. PR #54716 remains open with closing syntax and is the implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while PR #54716 is reviewed. The best fix is to adapt the PR's literal per-agent store discovery behavior to the current `src/config/sessions/combined-store-gateway.ts` implementation, preserve the shared literal-store fallback for non-agent-layout paths, add a literal-path regression test, and close this issue only after that or an equivalent fix lands.
