## Summary

Keep #70807 open. Current main does not contain the requested compat timeout propagation, media timeout widening, or Ollama ops scripts, so it is not implemented or superseded. The PR also has concrete unresolved review concerns, so the best maintainer path is a revised/split implementation rather than conservative auto-close.

## Best Possible Solution

Keep the PR open but ask for a focused revision. The core fix should pass the active agent timeout to the OpenAI-compatible handler through a runtime-only option that works even when chatCompletions has no config object, preserve the public config schema/type boundary, and add regression coverage for both configured and runtime-enabled endpoints. The media timeout widening should remain a separate reviewed change. The Ollama scripts should either be removed from this PR or hardened with configurable hosts/ports, standard 11434 defaults, and complete socket cleanup.
