## Summary

Keep #70807 open. Current main does not include the compat timeout propagation, the media timeout change, or the Ollama helper scripts, and the PR still contains unresolved review feedback that affects correctness and mergeability.

## Best Possible Solution

Keep the PR open, but do not merge it as-is. The best path is to split the core timeout fix from optional Ollama ops tooling, thread the active agent timeout through a runtime-only OpenAI compat option without widening the schema-backed user config type, add a regression test for the endpoint-enabled-without-config case, keep invalid negative timeout handling aligned with `agent-command.ts`, and either fix/parameterize the Ollama scripts or move them to a separate reviewed tooling/plugin discussion.
