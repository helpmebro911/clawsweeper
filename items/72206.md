## Summary

Keep #72206 open. The report is current, has a concrete reproduction and logs, and current main still contains the same Claude live-session behavior as the reported beta for the relevant files. I did not find a merged fix or a canonical open issue that fully covers the fresh-live-session failure while same-model reuse succeeds; the linked #71793 buffer issue is already addressed separately, and #71803 is a different watchdog/idle-timeout failure mode.

## Best Possible Solution

Keep the issue open and investigate the fresh Claude live-session path in `src/agents/cli-runner/claude-live-session.ts` and `src/agents/cli-runner/execute.ts`. A good fix should preserve warm-session reuse, surface the real stderr/result detail instead of only `Claude CLI failed.`, and add a regression test or live-smoke coverage for a cold `claude-cli` live turn started from a system-event wake. Also verify whether the CLI `system event --expect-final` handling should actually wait for the agent-backed final response on this path.
