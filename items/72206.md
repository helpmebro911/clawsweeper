## Summary

Keep #72206 open. The report is fresh, concrete, and still maps to current claude-cli live-session behavior: OpenClaw can create a fresh Claude stdio live session, later reuse it through a separate path, and still turns an empty-stderr Claude process exit into `FailoverError` with `reason=unknown` and message `Claude CLI failed.` The newer main changes only adjust live model-switch fallback routing; they do not fix or test the reported fresh live-session startup failure.

## Best Possible Solution

Keep the issue open. Maintainers should reproduce the `system event --mode now` fresh claude-cli live-session path, capture redacted process argv/stdout/stderr/exit timing, then fix the fresh live-session startup path or diagnostics in `src/agents/cli-runner/claude-live-session.ts`. Add a focused regression test that distinguishes first live-session creation from reuse and verifies either success or a classified, actionable error instead of `reason=unknown` / `Claude CLI failed.`
