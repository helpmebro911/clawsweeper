## Summary

Keep open. The issue's direct `curl` 401 is expected on non-loopback Gateway binds, but current `main` still has a plausible macOS-specific gap: canvas/A2UI access depends on short-lived node-scoped capability URLs, and the macOS A2UI path still does a single load/poll before throwing the exact `A2UI_HOST_UNAVAILABLE` error without refreshing that capability. iOS already has explicit capability-refresh recovery on the same shared node session API, so there is not strong evidence this macOS report is obsolete or already fixed on `main`.

## Evidence

- **Non-loopback canvas/A2UI requires auth and expiring node capability URLs:** Gateway docs say `gateway.bind` beyond loopback requires auth for canvas/A2UI routes, node WebViews usually do not send auth headers, and the advertised node-scoped capability URLs expire quickly.
  - file: [docs/gateway/configuration-reference.md:537](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/gateway/configuration-reference.md#L537)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Server enforces bearer-or-capability auth for A2UI routes:** `authorizeCanvasRequest` accepts canvas/A2UI HTTP only via bearer auth or a live `canvasCapability`; expired/missing capabilities fall back to unauthorized.
  - file: [src/gateway/server/http-auth.ts:24](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/gateway/server/http-auth.ts#L24)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **macOS still throws exact error after one load attempt, with no capability refresh:** The macOS node runtime resolves `canvasHostUrl`, loads `__openclaw__/a2ui/`, polls for readiness, and then throws `A2UI_HOST_UNAVAILABLE: A2UI host not reachable`; there is no `node.canvas.capability.refresh` retry in this path.
  - file: [apps/macos/Sources/OpenClaw/NodeMode/MacNodeRuntime.swift:442](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/apps/macos/Sources/OpenClaw/NodeMode/MacNodeRuntime.swift#L442)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Shared refresh API exists and iOS already uses it:** The shared node session exposes `refreshNodeCanvasCapability()`, and iOS calls it before returning host-unavailable, which highlights the missing equivalent recovery on macOS.
  - file: [apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayNodeSession.swift:276](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayNodeSession.swift#L276)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **iOS A2UI path retries with capability refresh:** `ensureA2UIReadyWithCapabilityRefresh` navigates, waits, then refreshes the node canvas capability and retries before giving up.
  - file: [apps/ios/Sources/Model/NodeAppModel+Canvas.swift:58](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/apps/ios/Sources/Model/NodeAppModel%2BCanvas.swift#L58)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
