## Summary

Keep open. Current main still does not implement Ralph Loop or a per-agent max Ralph iteration/default loop setting, and the issue already has a same-day maintainer-visible Codex review comment keeping it open. The current agent config type, strict schema, resolver, docs, and embedded runner expose strict-agentic retry/tool-loop guardrails, not a STOP-driven repeated same-prompt loop with per-agent precedence.

## Best Possible Solution

Keep #6890 open for maintainer product judgment. If accepted, implement an OpenClaw-native per-agent loop/default config surface in AgentConfig, strict zod schemas, generated/help docs, and resolveAgentConfig, then wire the embedded runner with explicit precedence against higher-priority runtime or CLI overrides. Add tests for global default, per-agent override, override precedence, disabled, finite, and unlimited semantics, and keep the feature distinct from strict-agentic retry handling and tools.loopDetection. If maintainers decide this should live outside core, first define the missing plugin/runtime API needed to express it rather than closing the request as already possible through ClawHub.
