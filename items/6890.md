## Summary

Keep open. Current main still lacks a Ralph Loop feature and lacks any per-agent `loop.max_ralph_iterations` or equivalent agent config surface; existing loop-related behavior is tool-loop detection and strict-agentic retry handling, which do not implement a STOP-driven repeated same-prompt loop or the requested precedence model.

## Best Possible Solution

Keep #6890 open for maintainer product judgment. If accepted, implement an OpenClaw-native loop/default config surface in `AgentConfig`, strict zod schemas, generated/help docs, and `resolveAgentConfig`, then wire the embedded runner with explicit precedence against higher-priority runtime or CLI overrides. Add focused tests for global default, per-agent override, override precedence, disabled, finite, and unlimited semantics. If maintainers decide this belongs outside core, first define the missing plugin/runtime API needed to express it safely rather than closing it as already possible through ClawHub.
