## Summary

Keep this PR open. Current main does not implement an `agents.defaults.systemPromptFile` config field or any runner-side file read that prepends operator-managed prompt text. The PR is a focused implementation candidate for the closed companion feature request, and the existing review discussion was materially addressed, but it still needs maintainer review for public config docs/generated schema and the security semantics of per-session file reads.

## Best Possible Solution

Keep the PR open as the active implementation candidate. Best path is maintainer review of the product/API choice, then either land a tightened version that updates config docs/generated schema and clearly defines path/read semantics, or intentionally redirect the feature to a plugin/hook design if maintainers decide this should not become core `agents.defaults` behavior.

## Risks / Open Questions

- The PR adds a new public config key but the provided file list does not include generated schema/help or user docs, so it likely needs config documentation/generated-artifact follow-through before merge.
- The proposed per-session read is security-relevant: if an operator points the file into an agent-writable workspace, future sessions could be influenced by agent edits. Maintainers should decide whether documentation is enough or whether the implementation should require an absolute/outside-workspace path or snapshot at startup/config load.
- This read-only review did not run tests or CI by design; before merge, the appropriate changed gate should cover config schema drift plus CLI and embedded prompt composition behavior.
