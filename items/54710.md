## Summary

Keep this issue open. Current main still does not have a targeted fix, regression test, or live Discord-routed exec proof for the reported env-value mutation. The shared exec path appears to pass environment values through, and local shell checks preserve `pw_pw##`, but that is not enough to close a recent Discord-specific behavior report.

## Best Possible Solution

Keep this issue open until there is focused proof. Add a regression or live verification that invokes the exec tool from a Discord-routed session with `SOME_PASSWORD=pw_pw##` and asserts `echo "${SOME_PASSWORD}"` or `printf` matches `printenv SOME_PASSWORD`. If current main passes that end-to-end Discord route, close as cannot-reproduce with the proof; if it fails, fix the observed mutation layer in Discord streaming/final delivery, transcript/session replay, shell startup, sandbox/Docker exec, or the shared exec environment path.
