Keep open. Current main still lacks a targeted fix or regression test for the Discord-routed exec/env edge case. ## Best Possible Solution

Keep this issue open and add a focused regression or live verification that invokes exec from a Discord-routed session with `SOME_PASSWORD=pw_pw##`, asserting `echo "${SOME_PASSWORD}"` matches `printenv SOME_PASSWORD`.
