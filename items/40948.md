## Summary

Current main does not implement the requested session-recap stripping. The shared stripper still fast-path returns unchanged when no existing inbound metadata sentinel is present, there is no session-recap recognizer or regression coverage, and the Control UI/TUI still rely on that shared stripper for user-visible rendering. Related PR #41013 remains open and appears to track the same remaining fix, but the issue should stay open until a fix lands or maintainers explicitly choose the PR as canonical.
