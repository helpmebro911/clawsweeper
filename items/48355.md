## Summary

Keep open. Current main has partial overlap through the global/runtime agent event stream, including `thinking`, but it does not implement this PR’s plugin hook/API contract: no typed `thinking_start`/`thinking_end` plugin hooks, no top-level `api.onAgentEvent`, and no `tokenUsage` or `toolCallCount` on `agent_end`. The PR is stale and likely needs a rebase/redesign, but it is still useful unfinished work rather than an auto-close candidate.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a rebase or successor that chooses the intended public surface: either extend the existing `api.runtime.events.onAgentEvent` contract with safe scoping and docs, or add typed plugin hooks such as `thinking_start`/`thinking_end` plus enriched `agent_end` payloads. Any path should address the existing review comments, add focused tests for native and tagged thinking flows, and coordinate with the canonical lifecycle-hook work rather than silently closing this PR.
