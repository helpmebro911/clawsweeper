## Summary

Keep #69527 open. Current main still has OpenRouter `auto` identity inconsistencies around `openrouter/auto`, and `openclaw infer model run --json` still hardcodes `attempts: []`, so the reporter's fallback-visibility expectation is definitely unresolved even if some runtime paths may now reach the OpenRouter provider. The exact related PR #69580 was closed unmerged, not shipped.

## Best Possible Solution

Keep the issue open. A proper fix should make OpenRouter auto canonical across onboarding/config, status, catalog, dynamic model resolution, and the actual OpenRouter request payload, then add regression coverage for `agents.defaults.model.primary = "openrouter/auto"` with configured fallbacks. The `infer model run` command should also return real fallback attempt details instead of hardcoded empty arrays so users can distinguish primary success from fallback success.
