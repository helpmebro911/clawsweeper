## Summary

Keep open. Current main still preserves `openrouter/auto` as the OpenClaw runtime model id, while OpenRouter's bundled catalog uses bare `auto` as the API-side id and the OpenAI-compatible transport sends `model.id` directly. `openclaw infer model run --json` also still hardcodes `attempts: []`, so fallback visibility remains missing. Related PR #69580 targeted this bug but was closed unmerged.

## Best Possible Solution

Keep this issue open. The best fix is provider-owned: preserve `openrouter/auto` as the public OpenClaw model ref, translate the OpenRouter runtime/API payload model id to bare `auto`, keep `openrouter/@preset/...` refs working, add regression coverage around OpenRouter dynamic resolution/model run, and surface embedded fallback `executionTrace.attempts` in the `openclaw infer model run --json` envelope.
