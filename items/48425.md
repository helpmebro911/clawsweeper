## Summary

Keep #48425 open. Current main still writes the complete session store with plain JSON.stringify, still budgets against that same full serialization, and active session paths continue to persist skillsSnapshot into session entries. The PR addresses a real remaining bug, but the best path is to revise it so serialization drops only the top-level SessionEntry.skillsSnapshot while preserving systemPromptReport and unrelated nested keys.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: add a shared session-store serialization helper that omits only top-level SessionEntry.skillsSnapshot during store writes and disk-budget measurements, preserves systemPromptReport for warning dedupe, and covers nested-key preservation plus save/load behavior in tests. That would solve the current main bug without introducing a broad JSON key filter.
