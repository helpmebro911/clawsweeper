## Summary

Keep open. Current main still has no `memorySearch.ignorePaths` or equivalent exclusion contract, and `memory/dreaming/**` remains within both builtin and QMD default memory-search scope. The related merged dreaming-promotion fix reduces durable-memory contamination, but it does not address search indexing, QMD collection scope, or watcher scope.

## Best Possible Solution

Keep this issue open. The best fix is a real memory-search exclusion contract: add schema/type/help/docs support for `memorySearch.ignorePaths` or a similarly named field, merge defaults and per-agent overrides, apply it consistently to builtin `listMemoryFiles`, builtin chokidar watching, QMD collection generation/update/watch scope, and reindex scope hashing, then add focused tests for `memory/dreaming/**`, dependency dirs, `extraPaths`, and builtin/QMD watcher-index parity.
