## Summary

Keep #71285 open. Current main still does not provide a `memorySearch.ignorePaths`/exclude config surface, and the active indexing and watcher paths still include `memory/dreaming/**` unless it is handled elsewhere. The related merged dreaming fixes address promotion/self-ingestion, not the search index or watcher behavior requested here; the earlier ignorePaths PR was closed unmerged.

## Best Possible Solution

Keep the issue open and implement a config-backed memory search exclusion surface if maintainers want this capability: add the field to config schema/types/help/docs, resolve it alongside default/per-agent memorySearch config, apply the same matcher to `listMemoryFiles()` and both builtin/QMD watchers, and add focused tests for `memory/dreaming/**`, default dependency-directory ignores, and watch/index parity. The merged dreaming promotion work should remain separate because it does not solve search-result pollution.
