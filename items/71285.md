## Summary

Keep #71285 open. Current main does not implement `memorySearch.ignorePaths`, does not hardcode `memory/dreaming/` out of memory search indexing, and does not exclude `dreaming` from memory watchers. The adjacent merged dreaming self-ingestion work prevents promotion pollution, but it does not solve the reported `memory_search` index/result pollution. The earlier related ignore-path work in #6362 was closed unmerged, and #6109 was closed not planned, so neither is a live canonical item that supersedes this concrete current request.

## Best Possible Solution

Keep the issue open and implement a first-class exclusion path for memory search. The likely complete fix should add schema/types/help/docs, resolved config, scope-hash/reindex behavior, builtin `listMemoryFiles()` filtering, builtin chokidar filtering, QMD collection/watch parity, and tests covering `memory/dreaming/**` plus at least one generic ignored directory.
