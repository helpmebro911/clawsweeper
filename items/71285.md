## Summary

Current main still does not implement the requested configurable memory-search exclusion. `memorySearch` exposes `extraPaths` but no `ignorePaths`/exclude contract, builtin indexing still recursively walks `memory/**`, builtin and QMD watchers only use hardcoded dependency/cache directory ignores, and QMD default collections still include `memory/**/*.md`. The related dreaming fixes address promotion/self-ingestion, not search index or watcher scope.

## Best Possible Solution

Keep this issue open and implement a unified memory-search exclusion contract, likely `memorySearch.ignorePaths` or a clearly named equivalent. It should be added to config schema/types/help/docs, config resolution, builtin `listMemoryFiles`, builtin chokidar watching, QMD collection generation/watch/update scope, and reindex/scope hashing, with regression tests for `memory/dreaming/**`, dependency/cache defaults, `extraPaths`, and builtin/QMD parity.
