## Summary

Keep this PR open. Current main still builds gateway outbound session identity, but the public `message_sending` hook context does not receive `agentId`, cancellation is not reported back from outbound delivery, and gateway send still collapses a hook-cancelled delivery to `No delivery result`. The PR is also paired with the same-author open umbrella #63011, so it should be reviewed or replaced rather than closed.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent focused maintainer patch: add optional `agentId` to the public message hook context, pass outbound session `agentId` and `sessionKey` into `message_sending`, preserve cancellation reason through a narrow delivery callback or blocked-result shape, surface an explicit guarded gateway-send denial, and cover the gateway/outbound path plus docs. Review it alongside #63011 rather than closing one half prematurely.
