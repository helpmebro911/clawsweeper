## Summary

Keep this PR open. Current main still derives an outbound session for gateway send, but it does not pass that identity into the `message_sending` hook context, the public message hook context still lacks `agentId`, and a cancelled delivery still falls through to the generic `No delivery result` gateway failure.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent focused maintainer patch: add optional `agentId` to the public message hook context, pass `session.agentId` and `session.key` into `message_sending`, expose cancellation details through a narrow delivery callback/result, surface an explicit guarded gateway-send denial, and cover it with gateway/outbound regression tests plus a small plugin hook docs update.
