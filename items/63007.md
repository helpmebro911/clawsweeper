## Summary

Keep this PR open. Current main still has the gap the PR targets: gateway send passes an outbound session into delivery, but `message_sending` receives only channel/account/conversation context, hook cancellation still produces an empty result with no cancellation callback, and gateway send still reports that as the generic `No delivery result`. The same-author umbrella #63011 is also open, so this focused PR should be reviewed, rebased, or replaced rather than closed.

## Best Possible Solution

Keep this PR open, or replace it with an equivalent focused maintainer patch reviewed alongside #63011: add optional `agentId` to the public message hook context, pass outbound `agentId` and `sessionKey` into `message_sending`, preserve cancellation reason through a narrow delivery callback or blocked-result shape, surface an explicit guarded gateway-send denial, update hook docs/types, and cover the gateway/outbound behavior with tests.
