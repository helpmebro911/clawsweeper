## Summary

Keep #9764 open. Current main still has only Google Chat service-account auth, no user OAuth config/schema/secret surface, no refresh-token credential resolution, and no per-operation token routing for reactions, attachment uploads, reaction typing, or proactive DM lookup. The linked external Google Chat work is useful prior art, but it is not shipped in openclaw/openclaw.

## Best Possible Solution

Keep the issue open. The bundled Google Chat plugin should add a strict user OAuth credential model with appropriate secret/redaction handling, resolve refresh-token credentials safely, construct scoped user OAuth clients, route reactions, attachment uploads, reaction typing, and proactive DM/email-alias lookup through user auth, preserve service-account auth for webhook verification and bot-level sends, and document clear fallback warnings when user OAuth is absent.
