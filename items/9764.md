## Summary

Keep #9764 open. Current main still lacks Google Chat user OAuth configuration, secret handling, refresh-token resolution, scoped OAuth client construction, and per-operation token selection for reactions, attachment uploads, reaction typing, or proactive DM lookup. The prior external Google Chat work may be useful implementation reference, but it is not shipped in this repository.

## Best Possible Solution

Keep the issue open. The bundled Google Chat plugin should add a strict user OAuth credential model with SecretRef/redaction support, resolve refresh-token credentials safely, construct a scoped user OAuth client, route reactions, attachment uploads, reaction typing, and proactive DM/email-alias lookup through user auth, preserve service-account auth for webhook verification and bot-level sends, and warn or fall back clearly when user OAuth is absent.
