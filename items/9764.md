## Summary

Keep open. Current main still has Google Chat wired only for service-account authentication, while reactions, attachment upload, and proactive DM resolution all use the same account access-token path. The prior discussion and the external closed Google Chat file-upload PR are useful context, but they do not supersede the missing bundled Google Chat user OAuth credential surface.

## Best Possible Solution

Keep the issue open until the bundled Google Chat plugin has an explicit user OAuth credential model, config/schema/docs, token selection for reactions/uploads/proactive DMs, and tests covering graceful fallback when user OAuth is not configured. If maintainers prefer this outside the bundled plugin, they should first make that product decision explicit and identify any plugin API gap needed for an external implementation.
