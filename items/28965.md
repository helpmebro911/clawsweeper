## Summary

Keep #28965 open. Current main still has useful targeted model normalization, model status output, and per-session reset paths, but it does not implement the requested first-class `openclaw models reconcile` command or scoped bulk `openclaw sessions model-reset` workflow.

## Best Possible Solution

Keep this issue open until OpenClaw implements deliberate core CLI support for model configuration reconciliation plus scoped bulk session model override reset, or maintainers explicitly designate a canonical issue for the same remaining work. The likely implementation belongs in `openclaw models`, `openclaw sessions`, config update helpers, session-store maintenance code, regression tests for dry-run/scoped reset behavior, and user-facing model migration/troubleshooting docs.

## Risks / Open Questions

- A bulk session override reset would be operationally destructive without dry-run, JSON reporting, explicit scope filters, and clear docs.
- The proposal overlaps broader config simplification work, so maintainers may choose to supersede it with a canonical design issue rather than implement the exact command names.
