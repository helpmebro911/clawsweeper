## Summary

Keep #28965 open. Current main still has targeted model status/list/set/fallback/auth commands and per-session reset/patch helpers, but not the proposed first-class model configuration reconciliation workflow or scoped bulk session model override reset.

## Best Possible Solution

Keep #28965 open until OpenClaw either implements a documented `openclaw models reconcile` flow and a scoped `openclaw sessions model-reset` command, or maintainers explicitly consolidate it into a canonical model/config single-source design issue. The likely implementation belongs in the existing models and sessions CLI surfaces with `--dry-run`, `--json`, `--agent`, `--all-agents`, `--store`, and match/filter support, plus regression coverage for clearing `providerOverride`, `modelOverride`, `modelOverrideSource`, and optional auth-profile override fields without disturbing unrelated session data.
