## Summary

Keep #28965 open. Current main still lacks a first-class `openclaw models reconcile` command and lacks the requested scoped bulk `openclaw sessions model-reset` workflow. Existing code has targeted `models` commands, session listing/cleanup, per-session model patch/reset behavior, and some status visibility, but those do not satisfy the proposal’s central single-source reconcile plus operator reset request. The prior ClawSweeper keep-open comment remains accurate on current HEAD; there were no relevant changes in the inspected model/session CLI, session override, or docs paths since that review.

## Best Possible Solution

Keep #28965 open until OpenClaw either implements a documented model configuration reconciliation flow plus scoped bulk session model override reset, or maintainers explicitly consolidate it into a canonical model/config single-source design issue. The likely implementation belongs in the existing `models` and `sessions` CLI surfaces with `--dry-run`, `--json`, `--agent`, `--all-agents`, `--store`, and match/filter support, plus regression tests for clearing `providerOverride`, `modelOverride`, `modelOverrideSource`, and optional auth-profile override fields without disturbing unrelated session data.
