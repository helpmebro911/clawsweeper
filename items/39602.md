## Summary

Keep open. Current main still lacks Feishu/Lark create/update-time announce target validation, and #39599 remains the canonical open bug. PR #39602 is unsafe as submitted because its head replaces the cron CLI registration module with `404: Not Found`, but the underlying bug fix is focused and salvageable rather than a conservative cleanup close.

## Best Possible Solution

Keep this PR open for contributor or maintainer repair, or supersede it only with a clean canonical fix. The best implementation should restore `src/cli/cron-cli/register.cron-add.ts`, add gateway/service validation that rejects Feishu/Lark announce jobs only when no explicit or deterministic target can be resolved, add clear CLI feedback if desired, and keep #39599 as the canonical bug until regression coverage lands.
