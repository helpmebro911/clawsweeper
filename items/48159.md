## Summary

Keep open. Current main does not implement per-session ephemeral/secure chat mode: the session store schema has no durable `ephemeral` flag, normal turns still resolve and persist a JSONL transcript, plugin/context-engine hooks receive `sessionFile`/`sessionKey` but no hard ephemeral guard, and docs still describe transcript, compaction, and memory flush persistence as normal behavior. The related #62436 is narrower HTTP passthrough work and does not supersede this all-channels secure-chat request.
