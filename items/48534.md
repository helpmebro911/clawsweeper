## Summary

Keep #48534 open. Current main still awaits plugin hook handlers directly in the shared modifying-hook runner, so a never-resolving `before_agent_start` handler can still block that run path. The linked PR #48741 attempted this fix but closed unmerged, and the current config/types/docs do not expose a lifecycle hook timeout such as `plugins.hookTimeoutMs`.
