## Summary

Keep this PR open for maintainer handling. Current main still lacks the pending-tool-call hint, adapter, and re-wait loop proposed for the embedded-runner retry gap, so the narrow bug-fix idea is not implemented or clearly superseded. The current PR branch should not be merged as-is because the diff is heavily contaminated with unrelated generated artifacts, UI/runtime/hook changes, and security-sensitive execution paths.

## Best Possible Solution

Keep this PR open for maintainer handling, but request or salvage a clean replacement. The replacement should stay scoped to the embedded runner: verify the pi-agent-core `state.pendingToolCalls` contract, add a best-effort pending-tool-call hint, bridge it into attempt and compaction cleanup, preserve total timeout budgeting, keep cleanup probes non-throwing, and add focused retry-gap regression tests. Do not merge the current contaminated branch as-is.
