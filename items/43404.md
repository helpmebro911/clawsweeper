## Summary

Keep this PR open. Current main still has both reminder-guard gaps the PR targets: existing-cron suppression is session-key-only, and compaction retry still clears `successfulCronAdds`. The PR remains a focused implementation candidate for the linked open bug, with one functional call-site concern still worth maintainer review before landing.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the linked open bug. The best maintainer path is to preserve `successfulCronAdds` across compaction retry, add existing isolated-cron suppression by `agentId` only for enabled `sessionTarget: "isolated"` jobs, avoid deriving a default `main` agent id from an absent session key, and keep regression coverage for session-bound, isolated, disabled, empty-store, store-error, missing-agentId, missing-sessionKey, and main-without-sessionKey cases.
