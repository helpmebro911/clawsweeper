## Summary

Keep this PR open. Current main still rejects the reported Windows drive-letter Docker bind at both config validation and runtime sandbox validation, while the PR targets a real Windows/Docker Desktop bug. The patch is not safe to auto-close as obsolete, and it is also not ready to merge without maintainer review because it broadens sandbox host bind acceptance and needs Windows-specific source-root and canonicalization coverage.

## Best Possible Solution

Keep this PR open for maintainer review or replace it with a maintainer-owned patch. The best fix should reuse `splitSandboxBindSpec` in config validation, update runtime validation and host-path canonicalization for Windows drive paths without bypassing sandbox policy, compare Windows roots safely, add regression coverage for uppercase/lowercase drives, binds with and without mode, blocked paths, outside-root rejection, and Windows symlink/junction escapes, then update the public sandbox docs if native Windows drive-letter binds are officially supported.

## Risks / Open Questions

- The current PR adds no regression tests or docs for native Windows Docker binds, even though it changes security-sensitive validation behavior.
