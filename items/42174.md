## Summary

Current main still does not implement this PR's requested native Windows drive-letter Docker bind support. The config schema still splits a bind at the drive-letter colon and rejects source `D`, while the runtime sandbox security path still rejects any bind source that does not start with `/`. The PR targets a real remaining Windows bug, but it changes sandbox bind validation and should be reviewed or revised with Windows-specific regression coverage rather than closed as obsolete.

## Best Possible Solution

Keep this PR open or replace it with a maintainer-owned fix. The fix should reuse `splitSandboxBindSpec` in config validation, update runtime source validation and allowed-root canonicalization for Windows drive paths without bypassing sandbox policy, compare Windows drive roots safely, and add targeted Windows-path regression tests for valid binds, lowercase drives, no-mode binds, blocked paths, outside-root rejection, and symlink/junction escapes. Update sandbox docs if native Windows drive-letter binds become supported.
