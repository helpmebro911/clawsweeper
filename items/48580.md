## Summary

Keep open. Current main has meaningful ACPX recovery for OpenClaw-managed ACP turns, and the bundled plugin now delegates to upstream acpx@0.6.1, but I did not find strong evidence that the specific direct `acpx codex sessions new` stale session JSON / dead pid behavior is fixed. The remaining behavior appears distinct from the already-implemented early `acpx exited` retry path.
