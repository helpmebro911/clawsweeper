## Summary

Keep this PR open. Current main still does not pass `tools.exec.host`, `tools.exec.security`, or `tools.exec.ask` into the chat-bash `createExecTool` path, while the normal agent exec path does inherit those settings. The PR addresses a real policy-consistency gap, but its second commit also changes `!` alias routing in a way that an existing review correctly flags as a regression for valid punctuation-prefixed shell commands.

## Best Possible Solution

Keep the PR open and ask for a focused update: preserve the `tools.exec.host/security/ask` inheritance in `handleBashChatCommand`, add regression coverage for the inherited policy, and either drop or rework the markdown-image/bang-routing change so valid `!` aliases such as absolute paths, relative paths, and shell-variable-prefixed commands still route to chat-bash.
