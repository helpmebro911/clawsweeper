## Summary

Do not auto-close this PR. Current main has superseded part of the Overview stat-card work by rendering Overview cards as native buttons wired to tab navigation, but the broader affordance cleanup is not implemented: generic `.card`, `.stat`, and `.account-card` hover styles still apply to non-clickable panels. The PR is also dirty as submitted because it includes large unrelated Spryng/routing artifacts, so it needs maintainer/author changes rather than cleanup closure under the allowed reasons.

## Best Possible Solution

Keep this open as needing changes, or ask for a clean replacement PR against current main. The focused solution should preserve the shipped `overview-cards.ts` button-based Overview cards, remove or scope hover affordances from non-interactive `.card`, `.stat`, and `.account-card` surfaces, add focused UI coverage for Overview card navigation if desired, and drop all unrelated Spryng/routing and hook-loader changes.
