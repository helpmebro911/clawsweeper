## Summary

Keep this PR open. Current main and v2026.4.24 still persist `systemSent: true` inside `ensureSkillSnapshot()` before the LLM run, so the PR is not obsolete. The patch is a focused core-session fix with regression tests and no supply-chain/CI/dependency churn, but it still needs maintainer review for successful queued/final followup paths before landing.

## Best Possible Solution

Keep this PR open. The best path is to finish the focused fix by removing `systemSent` persistence from `ensureSkillSnapshot()`, then marking it delivered through a shared post-success helper that runs for normal replies, silent/NO_REPLY completions, messaging-tool-only completions, and queued followup completions, while skipping provider/meta-error outcomes. Add targeted regression coverage for failed first call, successful retry with `isNewSession=false`, payloadless success, messaging-tool success, and queued/final followup success before maintainer review and landing.
