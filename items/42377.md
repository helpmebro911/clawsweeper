## Summary

Keep open. Current `main` still has a real mixed-type crash path for `env.vars`, so this PR is not obsolete, but the evidence shows the report's root cause is not `containsEnvVarReference` being called from arbitrary config traversal. The live path is `src/config/config-env-vars.ts`, and this PR also bundles unrelated Feishu changes and three root-level personal docs, so there is not a supported auto-close reason.

## Evidence

  - file: [src/config/config-env-vars.ts:21](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/config/config-env-vars.ts#L21)
