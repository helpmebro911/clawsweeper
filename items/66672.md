## Summary

Keep open. Current main has safety primitives, tool profiles, sandbox read-only workspace access, exec approvals, and an experimental update_plan tracker, but it still does not provide first-class Ask, Plan, and Modify workflow modes with enforced read-only operation, preview/validation, and explicit execution transitions.

## Best Possible Solution

Keep this issue open for core product/runtime design. A strong implementation would add an enforced Ask mode that blocks mutating tools at the policy boundary, a Plan mode that captures intended file/action changes and blocks execution until explicit approval, and a Modify mode that enables existing write/exec-capable paths under current tool policy, sandbox, and exec approval rules. It should include UI/session controls and tests proving enforcement at runtime/tool-policy boundaries, not only prompt wording.
