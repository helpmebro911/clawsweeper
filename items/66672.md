## Summary

Keep open. Current `main` has partial safety controls for read-only execution, but it does not implement the requested first-class Ask/Plan/Modify workflow. In particular, planning is only progress tracking, and supported GPT-5 runs now explicitly reject plan-only turns instead of exposing them as a distinct user mode.

## Evidence

- **Read-only safety exists, but as sandbox/tool-policy config rather than a named Ask mode:** `docs/gateway/security/index.md:1030` documents a read-only profile built from `sandbox.workspaceAccess: "ro"` or `"none"` plus tool deny-lists, and `docs/gateway/security/index.md:1171` shows the example as per-agent access control, not a first-class Ask/Plan/Modify workflow surface.
