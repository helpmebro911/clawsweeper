## Summary

Keep open. I re-verified current main at 646a268d2710bc54e478349ef5690f3c8803df4f and the prior ClawSweeper comment is still accurate: OpenClaw has safety primitives such as tool profiles, exec approvals, sandboxing, ACP permission checks, and the experimental update_plan tool, but it does not implement the requested first-class Ask, Plan, and Modify workflow modes with enforced read-only, preview/validation, and execution states.

## Best Possible Solution

Keep open for maintainer product/design work in core runtime, config, and UI. The likely path is an enforced read-only Ask mode at the tool-policy boundary, a Plan mode that captures structured intended actions and blocks mutating tools until explicit confirmation, and a Modify mode that enables existing write/exec-capable paths after validation. Regression tests should prove enforcement at runtime/tool-policy boundaries, not only prompt wording.
