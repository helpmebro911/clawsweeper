## Summary

Keep this PR open. Current main has partially moved since the patch: subagent announce delivery now defaults to 120s and is configurable, but the sessions_send A2A path still has the 60s wait cap and adjacent hardcoded budgets. The PR is useful implementation direction, but it needs a rebase/rework and a bounded timeout policy before maintainer cleanup closure would be safe.

## Best Possible Solution

Keep this PR open for maintainer review, but rework it against current main: preserve or explicitly change the shipped 120s subagent announce default, add a bounded/configurable timeout policy for sessions_send A2A, remove inappropriate 60s caps consistently, and add focused regression coverage for waits above 60s and the fire-and-forget A2A path.
