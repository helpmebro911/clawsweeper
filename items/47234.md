## Summary

Keep PR #47234 open. Current main still does not implement per-binding group/channel routing into the main session: `groupScope` is absent from source/docs, route bindings only expose `session.dmScope`, and non-direct peers still always resolve to group/channel session keys. The PR also remains a useful concrete implementation candidate, though it needs rebase/splitting and must address the outbound and Telegram propagation review comments before merge.

## Best Possible Solution

Keep the PR open for maintainer/product review. The best path is to split the IRC active-client fix into #45662 or rebase after it, then implement the group/session feature against current main's binding-session model, with schema/types/generated config metadata, docs/security wording, routing/session-key tests, outbound session derivation, and Telegram topic override propagation covered explicitly.
