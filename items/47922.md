## Summary

Keep open. Current main does not implement `messages.outbound.text.emDash`, the PR has not been merged or superseded locally, and the existing plugin hook surface makes a ClawHub/plugin direction plausible but not enough to auto-close this PR because the PR also targets shared core config, gateway mirroring, and post-hook normalization semantics.

## Best Possible Solution

Keep the PR open for maintainer product-boundary review. If OpenClaw wants this as a core global policy, fix the reviewed rewrite edge cases, add tests for leading em dashes and pre-existing trailing commas, and document exact ordering around `message_sending` hooks and transcript mirroring. If maintainers prefer the lean-core direction, close it manually as a ClawHub/plugin item and point contributors to a configurable hook plugin, with a separate narrow follow-up only if delivery mirror rewriting needs a public plugin seam.
