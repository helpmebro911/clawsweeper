## Summary

Current `main` appears to have addressed the `agents list` SecretRef failure, but I did not find strong evidence that the separate CLI hang is fixed. `agents` commands still bootstrap the full plugin registry, while CLI teardown only closes memory managers, so the non-memory plugin-handle part of the report remains plausible.

## Evidence

- **`agents list` now degrades unresolved SecretRefs instead of crashing:** `buildProviderStatusIndex()` uses read-only account inspection when available and catches `unresolved SecretRef` errors, recording the account as `not configured` rather than rethrowing.
