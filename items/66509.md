## Summary

Keep this issue open. Current main still has the QQBot outbound path buffering `kind: "tool"` text behind a 60s fallback, while `streaming.mode: "partial"` only affects block streaming and does not enable immediate tool/progress delivery. Telegram has a separate tool-progress preview path, but QQBot does not.

## Best Possible Solution

Keep the issue open and implement a QQBot-specific streaming fix in `extensions/qqbot/src/engine/gateway/outbound-dispatch.ts`: when `streaming.mode` is partial, deliver visible tool/progress text immediately or through a QQBot preview/edit path, while preserving media fallback and suppression rules. Add a focused regression test in `extensions/qqbot/src/engine/gateway/outbound-dispatch.test.ts` for tool-kind text followed by a block/final reply.
