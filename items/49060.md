## Summary

Keep open. Current main still does not route ACP bridge diagnostics away from stdout before setting up the ACP stdio JSON-RPC stream, so the reported core bug remains live. The linked PR #49072 and related PR #49085 are useful active fixes/discussion, but neither is merged into main.

## Evidence

- **ACP server still uses stdout for protocol without stderr routing:** serveAcpGateway() loads config and bootstraps the gateway, then binds ndJsonStream() to process.stdout/process.stdin. The file has no routeLogsToStderr import or call before that protocol setup.
  - file: [src/acp/server.ts:15](https://github.com/openclaw/openclaw/blob/d228463120a5057f7ccf9a5c31af62fe4a922088/src/acp/server.ts#L15)
