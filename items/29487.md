## Summary

Keep open. Current main still implements Feishu quoted/replied context only through `message.parent_id`; it does not model, parse, or fetch the UI quote identifier `quote.message_id` identified in the issue discussion, so a quote-only Feishu event can still reach the agent without the quoted message body.

## Best Possible Solution

Keep #29487 open. Implement the missing behavior inside the bundled Feishu plugin by verifying the Feishu UI quote payload shape, extending the Feishu event/context types with a separate quoted-message id, parsing `quote.message_id` or the verified equivalent, preferring that id when fetching quoted content while retaining `parent_id` fallback, and adding regression tests for quote-only UI replies plus existing parent-id replies under current group context visibility rules.
