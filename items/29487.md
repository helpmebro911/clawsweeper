## Summary

Keep open. Current main still only fetches Feishu quoted/replied context from `message.parent_id`; it does not model, parse, or fetch from the UI quote identifier `quote.message_id` called out in the issue discussion, and no current test covers a quote-only Feishu event.

## Best Possible Solution

Keep #29487 open. Implement this in the bundled Feishu plugin by verifying the Feishu UI quote payload shape, extending `FeishuMessageEvent` and `FeishuMessageContext` with a quoted-message id, parsing `quote.message_id` or the verified equivalent, preferring that id when fetching quoted content while retaining `parent_id` fallback, and adding regression tests for quote-only UI replies plus existing parent-id replies under group context visibility rules.
