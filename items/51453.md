## Summary

Current `main` no longer matches the reported Z.AI hourly JWT-refresh path. The bundled Z.AI provider is API-key/auth-profile based, its config patch only sets provider/base-url/catalog state, and `missing-meta-before-write` is not a write-blocking condition by itself in current config I/O.

## Evidence

- **Z.AI auth is API-key/auth-profile based:** The bundled Z.AI plugin prompts for `ZAI_API_KEY`, builds an API-key credential, and stores it via `upsertAuthProfile(...)`; it does not define a rotating JWT-in-config flow.
  - file: [extensions/zai/index.ts:108](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/zai/index.ts#L108)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Z.AI config patch only sets provider catalog/base URL:** The Z.AI onboarding/config helper writes provider preset metadata (`providerId`, `api`, `baseUrl`, `catalogModels`, aliases, primary model ref). It does not write per-model auth headers such as `models.providers.zai.models[].headers.X-Authorization`.
  - file: [extensions/zai/onboard.ts:43](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/zai/onboard.ts#L43)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Current docs describe Z.AI as API-key auth:** Current provider docs say Z.AI uses `ZAI_API_KEY` and Bearer auth, with onboarding through `openclaw onboard --auth-choice zai-api-key`.
  - file: [docs/providers/zai.md:9](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/providers/zai.md#L9)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **`missing-meta-before-write` is non-blocking on current main:** Current config I/O only treats `size-drop:*` and `gateway-mode-removed` as blocking write reasons. `missing-meta-before-write` is logged as suspicious but is not itself a blocker.
  - file: [src/config/io.ts:344](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/config/io.ts#L344)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Repo-wide search found no current AutoGLM/JWT-header refresh path:** Searches on current main found no references to `X-Authorization`, `autoclaw-proxy`, or `AutoGLM` under the current Z.AI provider/config code paths, which is inconsistent with the issue’s reported reproduction mechanism.
  - command: `rg -n "X-Authorization|autoglm-api|autoclaw-proxy|AutoGLM" extensions/zai docs/providers/zai.md src -g '!**/dist/**'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
