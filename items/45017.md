## Summary

Keep open. Current main has media/attachment hardening that justified closing the linked #44791, but it still does not tolerate the malformed `null`/`undefined` content-array entries this PR is specifically trying to handle. The dashboard rendering path still calls `normalizeMessage` and `extractThinkingCached`, and those helpers still dereference `item.type`/`p.type` after unchecked casts.

## Best Possible Solution

Keep the PR open or ask the author/maintainer to rebase it onto current main. The best fix is still a small non-object guard in `message-normalizer.ts` and `message-extract.ts`, updated for the current attachment/canvas normalization logic, plus regression tests for `null`, `undefined`, and array-valued content entries. Since #44791 is now closed for media-specific hardening, this PR should be treated as a narrower malformed-content hardening follow-up rather than closed as implemented.
