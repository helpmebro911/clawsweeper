## Summary

Keep #72130 open. Current main has session maintenance and optional disk-budget cleanup, but it does not ship a default hard disk budget for VPS/server deployments, and the OpenClaw compaction checkpoint path still creates full transcript-copy snapshots. The reported Hermes/git-backed checkpoint store is not clearly covered by the existing session-store cleanup, so this needs maintainer investigation rather than auto-close.

## Best Possible Solution

Keep the issue open and add a bounded retention path for server deployments: identify the exact Hermes checkpoint directory, include it in OpenClaw-owned maintenance or expose a plugin/dependency cleanup hook, choose a safe VPS default or first-run profile for `session.maintenance.maxDiskBytes`, and add regression coverage for multi-GB binary/blob checkpoint artifacts.
