## Summary

Keep open. Current `main` still leaves Control UI run recovery and queued-message draining dependent on terminal `chat` events. The gateway/backend already supports `agent.wait` for `chat.send` runs that finish without lifecycle events, but the Control UI does not call that recovery path, so the reported stale-busy/queued-message stall remains plausible on `7d9172d5e093435d37ed3e00e476a4eaf84a252b`.

## Evidence

- **Busy state only clears on terminal chat events:** `handleChatEvent` clears `chatRunId` only for `final`, `aborted`, and `error`. There is no alternate reconciliation path in this controller when those events are missed.
  - file: [ui/src/ui/controllers/chat.ts:451](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/ui/src/ui/controllers/chat.ts#L451)
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Queue flushing is wired to terminal-event handling:** `handleTerminalChatEvent` drains queued messages only after `handleChatEvent` returns a terminal state. If no terminal `chat` event arrives, this flush path does not run.
  - file: [ui/src/ui/app-gateway.ts:401](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/ui/src/ui/app-gateway.ts#L401)
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Session-message recovery is deferred while run is active:** When a `session.message` event arrives during an active run, the UI stores a deferred reload token and returns early. That deferred reload is replayed only after a later terminal chat state clears `chatRunId`.
  - file: [ui/src/ui/app-gateway.ts:461](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/ui/src/ui/app-gateway.ts#L461)
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Backend already covers the missing-lifecycle case:** Gateway test explicitly proves `agent.wait` resolves `chat.send` runs that finish without lifecycle events.
  - file: [src/gateway/server.chat.gateway-server-chat.test.ts:998](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/gateway/server.chat.gateway-server-chat.test.ts#L998)
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Control UI does not use agent.wait:** `rg -n "agent\\.wait" ui/src/ui` returned no matches, so current Control UI code has no visible `agent.wait` reconciliation path.
  - command: `rg -n "agent\.wait" ui/src/ui`
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
