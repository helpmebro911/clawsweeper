## Summary

Keep open. Current main still hard-codes the TUI footer and has no footer template, footer field config, balance/cooldown display, or plugin status-provider seam wired into TUI rendering. Existing usage and cost surfaces are adjacent, but they are `/status`, per-response `/usage`, and CLI usage-window summaries, not the persistent customizable TUI footer requested here. The provided GitHub context also shows an open PR with closing syntax for this issue, so this issue should remain open until that PR is reviewed/closed/reworked or a real implementation lands.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and close or rewrite the README-only closing PR, then either implement a real TUI footer configuration surface across src/tui/tui.ts, TuiOptions, CLI/config/docs, and focused tests, or split the work into a canonical footer-template issue plus a reviewed plugin status-provider API. Existing usage/cost and provider quota plumbing can supply data, but the persistent customizable TUI footer still needs core product/API work.
