Keep open. Current main still has a hard-coded TUI footer and no configurable footer template, footer fields, balance/cooldown field source, or plugin status-provider API. Keep this issue open. The best path is to close or request a complete rewrite of PR #72483, then implement a real TUI footer configuration surface in src/tui/tui.ts, TuiOptions, CLI/docs, and, if plugins should supply live cost/balance/cooldown data, a documented public plugin status-provider API aligned with #64086. Maintainers may also manually consolidate the issue with #59909/#64086 if they choose one canonical footer tracker.
