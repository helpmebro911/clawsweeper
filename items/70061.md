## Summary

Keep open. Current `main` still produces Telegram DM topic session keys in the `agent:main:main:thread:...` form, while Active Memory only recognizes `:group:`, `:channel:`, `:direct:`, `:dm:`, a 3-part main session, or `webchat`. For unrecognized keys it returns `undefined`, and the `before_prompt_build` hook immediately skips the plugin for that session.

## Evidence

- **Active Memory does not recognize thread-form session keys:** `resolveChatType()` checks `:group:`, `:channel:`, `:direct:`/`:dm:`, then a 3-part `agent:<id>:<main>` session, then `webchat`. There is no `:thread:` branch or Telegram-specific fallback here.
