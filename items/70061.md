## Summary

Keep #70061 open. Current main still matches the reported failure mode: Active Memory only classifies session keys containing `:group:`, `:channel:`, `:direct:`, `:dm:`, a 3-part main-agent key, or webchat. Telegram DM topics on the default DM route are still produced as `agent:main:main:thread:<chatId>:<topicId>`, which does not satisfy any Active Memory chat-type branch, so `isAllowedChatType()` returns false and the hook exits before running memory recall.

## Best Possible Solution

Keep the issue open and fix it in `extensions/active-memory`: classify thread-suffixed DM session keys whose base session is the configured main key as `direct`, ideally with a regression test using `agent:main:main:thread:<chatId>:<topicId>` plus existing `:direct:` thread coverage. Consider surfacing a debug/status line for chat-type filter skips so future session-key format gaps are observable.
