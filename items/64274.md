## Summary

Current `main` still has the auth-profile runtime snapshot path-key collision described in this issue and in the closed, unmerged PR #64320. The MiniMax portal provider still reads OAuth availability through the affected per-agent auth store, while the multi-agent docs still promise per-agent auth isolation.

## Best Possible Solution

Keep this issue open. Revive PR #64320 or apply an equivalent fix that keys runtime auth snapshots by a stable original identity, such as raw `agentDir` plus a distinct default/main sentinel, updates get/has/set/replace/save/load callers consistently, and adds a regression test where `OPENCLAW_AGENT_DIR` points at the requested agent while main and that agent have distinct `minimax-portal` profiles.
