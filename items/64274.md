## Summary

Keep open. Current `main` still lets an agent-scoped environment override collapse the implicit main-agent auth-store lookup onto the explicit agent store, so the reported MiniMax auth-resolution regression remains plausible and is not clearly implemented on `main`.

## Evidence

- **Implicit "main" lookup follows env override:** `resolveOpenClawAgentDir()` returns `OPENCLAW_AGENT_DIR` / `PI_CODING_AGENT_DIR` when either is set, so callers that pass no `agentDir` do not necessarily target the real main-agent directory.
  - file: [src/agents/agent-paths.ts:6](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/agent-paths.ts#L6)
  - command: `nl -ba src/agents/agent-paths.ts | sed -n '1,12p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Auth-store path derivation collides for undefined vs explicit agentDir:** `resolveAuthStorePath(agentDir)` joins `auth-profiles.json` onto `agentDir ?? resolveOpenClawAgentDir()`. A read-only reproduction with `OPENCLAW_AGENT_DIR=/tmp/kate/agent` prints the same path for `resolveAuthStorePath(undefined)` and `resolveAuthStorePath('/tmp/kate/agent')`.
  - file: [src/agents/auth-profiles/path-resolve.ts:12](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/auth-profiles/path-resolve.ts#L12)
  - command: `node -e 'const path=require("node:path"); const env={OPENCLAW_AGENT_DIR:"/tmp/kate/agent"}; const resolveOpenClawAgentDir=(env)=>env.OPENCLAW_AGENT_DIR?.trim()||env.PI_CODING_AGENT_DIR?.trim()||"/tmp/state/agents/main/agent"; const resolveAuthStorePath=(agentDir)=>path.join(agentDir ?? resolveOpenClawAgentDir(env), "auth-profiles.json"); console.log(resolveAuthStorePath(undefined)); console.log(resolveAuthStorePath("/tmp/kate/agent"));'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Runtime snapshots are still keyed by resolved auth-store path:** `runtimeAuthStoreSnapshots` is keyed via `resolveAuthStorePath(agentDir)`, not raw `agentDir`. That means the implicit main snapshot and the explicit sub-agent snapshot can alias to the same key under an agent-scoped env override.
  - file: [src/agents/auth-profiles/runtime-snapshots.ts:6](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/auth-profiles/runtime-snapshots.ts#L6)
  - command: `nl -ba src/agents/auth-profiles/runtime-snapshots.ts | sed -n '1,58p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Store merge logic still compares against the collided main key:** `resolveRuntimeAuthProfileStore()` computes both `mainKey` and `requestedKey` with `resolveAuthStorePath(...)`; if `OPENCLAW_AGENT_DIR` already points at the requested sub-agent, `mainKey === requestedKey`, so the code treats the sub-agent snapshot as the main snapshot path.
  - file: [src/agents/auth-profiles/store.ts:60](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/auth-profiles/store.ts#L60)
