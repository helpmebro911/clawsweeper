## Summary

Keep this issue open. Current main has raw Control UI editing for workspace bootstrap files such as USER.md and MEMORY.md, and those files are injected into agent context, but it does not provide the requested pinned key-value facts panel, structured pinned-facts storage, inline add/edit/delete UX, or a plugin API for persistent Control UI sidebar panels. The comment about `openclaw-lifework` is useful evidence of the broader missing surface: plugins can register tools/hooks/routes/methods today, but not a Control UI panel mount that auto-renders in the dashboard.

## Best Possible Solution

Keep this open and route it through a Control UI/product design pass. A good implementation would either add a first-class pinned facts editor backed by USER.md or a dedicated pinned-context file, with structured gateway methods and tests proving those facts are injected into agent context, or first add a generic persistent Control UI panel registration API that can also cover the related sidebar/panel requests. The existing raw Agents Files editor should be treated as a foundation, not a complete solution.
