## Summary

Current main still does not implement a Control UI pinned facts panel or the broader persistent Control UI panel extension seam described in the discussion. The only related UI today is raw workspace-file editing plus per-chat pinned message summaries, and the plugin SDK can add tools, commands, routes, gateway methods, and memory prompt supplements but cannot register a sidebar/dashboard panel for the Control UI.

## Best Possible Solution

Keep this issue open unless maintainers split out a narrower canonical issue for the underlying Control UI panel extension API. The best path is to design an additive gateway/plugin-SDK panel registration and mount contract for persistent Control UI surfaces, then implement the pinned facts editor as either a first-party panel or a plugin-backed panel with explicit storage and context-injection semantics.
