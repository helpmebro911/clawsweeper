## Summary

Keep open. Current main has a permanent delivery-error classifier and immediate failed/ path, but it still does not classify the reported local-media allowlist rejection or the proposed Telegram "message is too long" error. The later #69183 context is related and broader, but it does not fully supersede this narrow delivery-queue classifier request.

## Best Possible Solution

Keep #66489 open and implement the narrow classifier fix in `src/infra/outbound/delivery-queue-recovery.ts`, with policy/recovery tests for the local-media allowlist error and any still-observable Telegram too-long API error. If maintainers prefer the enqueue-time validation path, it should be added as a separate improvement while retaining the classifier as a safety net for already-queued items.
