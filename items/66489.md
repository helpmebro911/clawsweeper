## Summary

Current main still does not implement the requested permanent delivery classification. The exact local-media allowlist error is still thrown, but `PERMANENT_ERROR_PATTERNS` does not include it, live send failures still call `failDelivery`, and queued recovery only moves directly to `failed/` for classifier matches.

## Best Possible Solution

Keep this issue open. The narrow fix belongs in `src/infra/outbound/delivery-queue-recovery.ts` with delivery-queue policy/recovery tests: classify the exact local-media allowlist rejection as permanent so poisoned queued entries move directly to `failed/` instead of burning retry budget. A follow-up can consider enqueue-time validation and a `lastErrorClass` field; the Telegram `message is too long` case should be verified separately first.
