## Summary

Keep this PR open as candidate work, but not as a wholesale merge. Current main only partially covers the heartbeat replay problem and still lacks the Feishu websocket health snapshot, proposed exec failure semantics, and first-visible diagnostics surface; the PR branch also has unresolved module-load/regression and supply-chain-sensitive changes that need a split/rebase review.

## Best Possible Solution

Keep this PR open only as a source of candidate fixes. Ask for a current-main rebase or replacement split: a focused Feishu websocket health PR, a separate exec/process terminal-status semantics PR with product approval, a separate first-visible diagnostics design/implementation PR, and an ACPX dependency/command-launch PR with explicit security and maintainer review. The heartbeat slice should be compared against main's consume-on-success implementation before carrying anything over.
