## Summary

Current `main` does not show a strong maintainer-close condition for #40269. The reported user-visible leak remains plausible: the active run path only strips thinking blocks when `dropThinkingBlocks` is enabled, but Claude Sonnet 4.6 is explicitly configured to preserve them, and the dedicated Anthropic-thinking recovery helper exists only as unused code plus tests. If the provider still returns this invalid-request error, `formatAssistantErrorText()` will still surface it to chat as `LLM request rejected: ...` rather than suppressing or internally recovering.
