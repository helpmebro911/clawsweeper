## Summary

Keep open. Current main still lacks the requested `MEDIA:/path#doc` document-delivery hint, and WhatsApp still treats image MIME media as images unless the loaded media kind is already `document`. The related implementation PR #42044 was closed unmerged, and current docs/tool schema only expose document forcing as Telegram-only behavior.

## Best Possible Solution

Keep this issue open and implement a deliberate outbound media document hint. The most direct path is to support the proposed `#doc` suffix or a structured `asDocument`/delivery hint by stripping it before media resolution, preserving content type and filename, forcing downstream delivery as `document`, wiring that through WhatsApp send and auto-reply paths, and adding focused parser/loader plus WhatsApp delivery tests. Update user-facing docs if the behavior becomes supported outside Telegram.

## Risks / Open Questions

- The existing Telegram-only `forceDocument` option can be mistaken for a full fix, but it is not wired into WhatsApp delivery and does not let agent-emitted `MEDIA:` directives carry a document hint.
- A `#doc` suffix should be implemented carefully so real local filenames, remote URL fragments, filename derivation, MIME preservation, and host-read safety checks remain well-defined.
