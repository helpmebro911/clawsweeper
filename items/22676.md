## Summary

Keep #22676 open. Current main still matches the reported Signal daemon restart race: the daemon stop handle is synchronous, monitor teardown does not await signal-cli exit, and Signal config hot reload still stops then immediately starts the channel. The related candidate fix PR #71863 is open, while earlier PR #46357 was closed unmerged, so no close reason applies yet.

## Best Possible Solution

Keep #22676 open until PR #71863 or an equivalent focused fix lands on main. The fix should make the Signal daemon stop contract asynchronous, send SIGTERM, wait for signal-cli exit, escalate to SIGKILL on timeout, and await that shutdown through monitor/channel reload teardown before starting a replacement Signal channel. Add regression coverage for daemon stop wait/escalation and abort or reload-driven monitor shutdown.
