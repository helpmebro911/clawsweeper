## Summary

Current main still contains the Signal daemon restart race described in #22676. The daemon stop handle remains synchronous and fire-and-forget, Signal monitor teardown still does not await the signal-cli child process exit, and channel hot reload still stops then immediately starts the Signal channel. PR #71863 is an open candidate fix, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #22676 open until PR #71863 or an equivalent focused fix lands on main. The fix should make the Signal daemon stop contract asynchronous, send SIGTERM, wait for signal-cli exit, escalate to SIGKILL on timeout, and await that shutdown through monitor/channel reload teardown before starting a replacement Signal channel. Add regression coverage for daemon stop wait/escalation and aborted or reload-driven monitor shutdown.
