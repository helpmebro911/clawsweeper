## Summary

Keep open. Current main still clears the Overview Gateway Token when the WebSocket URL input changes, and the linked bug #41545 remains an open product bug. The PR is a focused attempted fix with regression coverage, but the exact preservation behavior needs maintainer judgment because current storage/connect code scopes tokens by gateway URL and sends `settings.token` to the configured gateway endpoint.

## Best Possible Solution

Keep the PR open for maintainer product/security judgment. The likely path is a reworked UI fix that avoids clearing the token on every keystroke while still respecting per-gateway token scoping, with regression coverage in `ui/src/ui/navigation.browser.test.ts` and/or a small helper test for the chosen state transition.
