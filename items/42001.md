## Summary

Keep open. Current main still has the UI behavior this PR targets, and the linked bug #41545 remains unresolved. The PR is a plausible focused fix with regression coverage, but the exact preservation behavior needs maintainer judgment because current storage/docs/connect paths treat gateway tokens as scoped to the selected gateway URL.

## Best Possible Solution

Keep the PR open for maintainer review or rework. The best fix should stop clearing a manually entered Gateway Token on ordinary WebSocket URL edits while preserving the selected-gateway credential boundary, with regression coverage in the Overview/navigation UI tests and any helper introduced for that state transition.
