## Summary

Current main still has only first-party `openclaw backup create` and `openclaw backup verify`. There is no shipped `openclaw backup restore <archive>` command, implementation, docs, or tests, so this issue should remain open. The keepmyclaw ClawHub plugin mentioned in the discussion is a useful workaround, and related #41274/#59280/#13616 show active demand, but none of that supersedes the missing core CLI restore workflow.

## Best Possible Solution

Keep this issue open until OpenClaw ships a safe first-party `openclaw backup restore <archive>` workflow or maintainers explicitly consolidate it into a canonical native backup/restore tracker. The likely implementation belongs beside the existing backup CLI and infra: reuse archive verification, add dry-run/force behavior, stage restore payloads, handle conflicts and rollback, update `docs/cli/backup.md`, and add focused command/restore-path tests. keepmyclaw can remain a documented community workaround, not a close reason for the core CLI gap.
