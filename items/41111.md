## Summary

Keep #41111 open. Current main still has only `openclaw backup create` and `openclaw backup verify`; there is no registered `backup restore` command, implementation, tests, or docs. The linked keepmyclaw ClawHub plugin is a useful workaround, and PR #41274 was relevant prior work, but the requested first-party restore command remains missing from core.

## Risks / Open Questions

- Closing as `implemented_on_main` would be incorrect because the requested command is absent from current main.
- Closing as `clawhub` would likely be too aggressive: the workaround plugin is useful, but OpenClaw core already owns archive creation/verification and the issue asks for the missing first-party restore half of that CLI lifecycle.
