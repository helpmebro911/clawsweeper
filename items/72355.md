## Summary

Keep this open. Current main appears to have mitigated the fatal Bonjour/CIAO crash-loop path, but the remaining hostname behavior is still real: the bundled Bonjour plugin still defaults the advertised host and TXT lanHost to openclaw/openclaw.local unless OPENCLAW_MDNS_HOSTNAME is set, and the supported config contract still has no discovery.mdns.hostname key.

## Best Possible Solution

Keep this issue open and narrow it to Bonjour hostname selection/configuration. Maintainers should decide whether the bundled Bonjour plugin should default to a sanitized system hostname or add a supported hostname override, then update MdnsDiscoveryConfig, the strict zod schema, generated schema/help/docs, docs.openclaw.ai gateway docs, and Bonjour tests while preserving the current CIAO cancellation suppression and bounded advertiser recovery.
