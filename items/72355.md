## Summary

Keep this issue open. Current main mitigates the fatal Bonjour/CIAO crash-loop path, but the reported hostname behavior remains: the bundled Bonjour plugin still defaults the advertised hostname and TXT lanHost to `openclaw` / `openclaw.local` unless `OPENCLAW_MDNS_HOSTNAME` is set, and there is no supported `discovery.mdns.hostname` or Bonjour plugin config field.

## Best Possible Solution

Keep this issue open and narrow it to Bonjour hostname selection/configuration. Maintainers should decide whether the bundled Bonjour plugin should default to a sanitized system hostname or add a supported hostname override, then update `extensions/bonjour`, `MdnsDiscoveryConfig`, the strict zod schema, generated config docs/help, the Bonjour plugin manifest if plugin-owned config is chosen, public gateway docs, and Bonjour tests while preserving the existing CIAO cancellation suppression and bounded advertiser recovery.
