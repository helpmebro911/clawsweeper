## Summary

Keep this PR open. Current main still lacks per-agent scoping for the bundled memory-lancedb plugin, and no merged/superseding implementation was found. The PR is a plausible implementation candidate, but it remains draft and has unresolved correctness and data-scope concerns from review comments, so the right outcome is review/fix rather than cleanup closure.

## Best Possible Solution

Keep this PR open as the remaining implementation candidate for bundled memory-lancedb per-agent isolation. The maintainer path is to rebase it onto current main, keep the fix inside extensions/memory-lancedb, use trusted ctx.agentId consistently from tool factories and hook contexts, resolve LanceDB filter/query-ordering and agentId validation semantics, log or surface migration failures, and add regression coverage for agent A/B isolation, legacy shared memories, duplicate detection, forget/delete scoping, and default unscoped behavior. After that, review and land it or intentionally supersede it with a canonical replacement PR.
