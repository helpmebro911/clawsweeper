## Summary

Keep this PR open. Current main has partial MCP loopback hardening for owner/non-owner visibility, but it still does not implement this PR's central per-backend token-bound scope model. The linked security issue #64993 is still open in the provided context and is paired with this PR by the same author, so this remains an implementation candidate rather than an obsolete close.

## Best Possible Solution

Keep this PR open for maintainer security review, or supersede it with a narrower patch for #64993. The best path is to preserve the shipped owner/non-owner bearer mitigation, add a per-backend token registry that binds session key, account id, message channel/provider, and owner status to the bearer token, stop emitting scope headers/env vars, invalidate scoped cache entries on revocation, add negative spoofing tests for all scope fields, fix token lifecycle/race issues, and split or justify the unrelated plugin loader change.
