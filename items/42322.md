## Summary

Keep open. Current `main` does not implement Feishu `replyToMode`, so this PR is not obsolete or already shipped. The requested behavior is still missing from the Feishu schema/runtime/docs on `main`, and the PR discussion also surfaced a remaining streaming-path concern that still needs maintainer review rather than cleanup-closing.

## Evidence

- **Feishu config schema on main has `replyInThread` but no `replyToMode` field.:** `extensions/feishu/src/config-schema.ts` defines `ReplyInThreadSchema` and wires it into `FeishuGroupSchema` and `FeishuSharedConfigShape`, but there is no `ReplyToModeSchema` or `replyToMode` property for Feishu at all on current `main`. That means the requested config is not implemented yet.
