## Summary

Keep this PR open. Current main still lacks a Feishu `replyToMode` config surface, and Feishu assistant replies still pass a fixed reply target through the bot and dispatcher. The PR addresses a real Feishu-owned gap, but it is not ready to merge as-is because it needs rebasing onto current reply-target/threading behavior plus tests/docs/metadata and a complete first-mode policy across streaming and media sends.

## Best Possible Solution

Keep this PR open and request a Feishu-owned implementation pass: rebase onto current main, add `replyToMode` to the strict Feishu schemas with clear group/account/top-level precedence, use the shared reply-policy helpers or an equivalent single state machine across streaming card start, text/cards, and media, update plugin manifest/generated config metadata/public docs, and add focused tests for `off`, `first`, and `all`.
