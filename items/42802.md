## Summary

Current main still lacks the requested read-only `routing.list` Gateway method and the selected-agent Channel Bindings display in the Agents tab. The existing config/docs support route and ACP bindings, and the same-author external implementation candidate is useful, but nothing equivalent has landed in `openclaw/openclaw`.

## Best Possible Solution

Keep this issue open. The maintainer path is to review or port the linked same-author implementation candidate, add a config-derived `routing.list` or equivalent read-only Gateway API under the correct read scope, define protocol/UI types, load route and ACP bindings into Control UI state, render only the selected agent's bindings in Agents -> Channels, and add focused Gateway and UI coverage before closing as implemented.
