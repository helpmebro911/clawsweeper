## Summary

Keep #9820 open. Current main still does not implement native JSONL support for `memorySearch.extraPaths`: builtin discovery admits Markdown plus enabled multimodal files, `memory_get` still rejects non-`.md` paths, watcher globs are Markdown/multimodal-only, and docs/config help still describe extra paths as Markdown-oriented. Session JSONL indexing and plugin/QMD supplement paths are separate workarounds, not native JSONL support for builtin extra paths.

## Best Possible Solution

Keep #9820 open until maintainers either add native JSONL support to the builtin memory indexer or explicitly document the supported alternative. A complete fix should update file admission, JSONL line/chunk handling, watcher globs, `memory_get` rendering, config help/docs, and tests; if arbitrary JSONL logs are out of scope, document the intended QMD custom collection, memory corpus supplement, or converter-plugin path.
