## Summary

Keep #9820 open. Current main still does not implement native JSONL support for builtin `memorySearch.extraPaths`: discovery admits `.md` plus enabled multimodal files, indexing treats admitted text files as Markdown, `memory_get` rejects non-`.md` targets, watcher globs remain Markdown/multimodal-only, and public docs/config help still describe extra paths as Markdown-oriented. Existing session JSONL indexing and QMD session export paths are separate mechanisms, not arbitrary JSONL support for builtin extra paths.

## Best Possible Solution

Keep #9820 open until maintainers either add native JSONL support to the builtin memory indexer or explicitly document the supported alternative. A complete core fix should update file admission, JSONL parsing/chunking and line mapping, watcher globs, `memory_get` behavior, config help/docs, and focused tests; if arbitrary JSONL logs are out of scope, document the intended session-memory, QMD custom collection, or converter-plugin workflow.
