## Summary

Keep open. Current main still does not implement native JSONL support for builtin `memorySearch.extraPaths`: the file lister admits Markdown plus enabled multimodal media only, `memory_get` rejects non-Markdown paths, and the builtin sync path indexes whatever that lister returns. OpenClaw does have related session transcript JSONL indexing and message hook support, but that is scoped to OpenClaw session transcripts rather than arbitrary hook-created `.jsonl` logs in `extraPaths`.

## Best Possible Solution

Keep tracking this issue until maintainers either implement native builtin JSONL support for `memorySearch.extraPaths` or explicitly narrow the product direction. A complete fix should extend the allowed file set, add JSONL-aware chunking and `memory_get` read behavior, update watcher/index tests, and update public memory docs. If maintainers prefer not to support arbitrary JSONL in the builtin engine, document the intended path via session transcript memory or QMD/custom collections instead.
