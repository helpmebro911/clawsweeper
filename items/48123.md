## Summary

Keep open. Current main still silently drops the reported non-image attachment cases in `parseMessageWithAttachments()`: sniffed non-image payloads and unknown non-image MIME payloads log warnings and `continue`, so `chat.send` never reaches its error response path for those inputs. The current tests also assert the drop-and-log behavior. The linked PR #67572 appears to be the active implementation path for non-image offload support, but it is still open and not present on main.
