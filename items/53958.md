## Summary

Keep open. Current main still does not visibly show the active config file path in the Control UI Overview or prominently in Config/Settings. The backend already has admin-scoped config path data, and the Config view receives a path, but the UI only uses it as the Open button tooltip. The related PR #54025 was closed unmerged, so it did not resolve this issue.

## Best Possible Solution

Add a visible, copyable active config path field in the Control UI Overview snapshot card or Config/Settings header, sourced from the existing admin-scoped hello snapshot or config.get snapshot path. Add focused UI coverage proving the path is rendered for authorized Control UI sessions and not exposed through low-privilege snapshots.
