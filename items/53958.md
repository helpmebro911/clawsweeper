## Summary

Keep open. Current main still does not visibly show the active config file path in Control UI Overview or prominently in Settings/Config. The backend already exposes the path only to admin-scoped sessions, and Settings/Config receives it, but the UI uses it only as an Open-button tooltip. Related PR #54025 was closed unmerged, so it does not resolve the request.

## Best Possible Solution

Keep this issue open and add a visible, copyable active config path in Control UI, preferably in the Overview snapshot card or Settings/Config header, sourced from existing admin-scoped hello snapshot/config.get data. Add focused UI coverage proving the path renders for authorized Control UI sessions and that low-privilege snapshots still do not expose sensitive paths.
