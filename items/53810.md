## Summary

Keep open. Current main still supports the reported Qwen/legacy ModelStudio models on the generic OpenAI-compatible stream path, and that path still does not parse Kimi-style tagged tool calls from reasoning/thinking content. The prior Codex review comment is still accurate against the current main SHA.

## Best Possible Solution

Keep this issue open and fix it in provider-owned stream normalization. The likely path is a Qwen/ModelStudio wrapper or shared OpenAI-compatible seam that detects validated Kimi-style tagged tool-call sections in reasoning/thinking content, converts them to structured `toolCall` blocks, and normalizes `stopReason` only after extraction succeeds. Add focused regression coverage for `qwen/kimi-k2.5`, `qwen/MiniMax-M2.5`, and the existing Kimi wrapper so text-block and thinking-block behavior are explicit.
