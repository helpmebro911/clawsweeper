## Summary

Current main still does not implement the reported provider/model-specific fix. The affected Qwen/legacy ModelStudio models remain bundled and documented, the Qwen provider still uses the generic OpenAI-compatible completions transport without a provider-local stream wrapper, and that generic stream path still only promotes structured `delta.tool_calls` into tool calls. Kimi-style tagged tool-call sections are only parsed by the Kimi provider wrapper, and only when they appear as standalone text blocks, so marker sections embedded in Qwen/ModelStudio reasoning/thinking text remain unhandled.

## Best Possible Solution

Keep this issue open. The best fix is provider-owned stream normalization, either in a Qwen/ModelStudio wrapper or a narrowly gated shared OpenAI-compatible compat seam, that detects validated Kimi-style tagged tool-call sections in reasoning/thinking text, converts them into structured `toolCall` blocks, and normalizes `stopReason` to `toolUse` after extraction succeeds. Add focused regression coverage for `qwen/kimi-k2.5`, `qwen/MiniMax-M2.5`, and the existing Kimi wrapper so text-block, thinking-block, and structured-tool-call behavior are explicit.
