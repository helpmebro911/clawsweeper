## Summary

Keep open. Current `main` still enforces a 10_000 ms preauth/connect-challenge timeout, applies that timeout in the gateway WebSocket handshake path, and does not expose the requested `gateway.handshakeTimeoutMs` config surface. The report therefore does not appear implemented on `main`.

## Evidence

  - file: [src/gateway/handshake-timeouts.ts:1](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/gateway/handshake-timeouts.ts#L1)
