## Summary

Keep open. Current main has some recovery and task-reconciliation improvements, but it still does not satisfy the core request: default one-shot subagent runs keep their child session entries, those entries remain visible through `sessions.list`/Control UI unless separately cleaned up, and session cleanup is warn-mode by default. The April 20 issue comment also reports the broader stale-running/task-panel symptom still reproducing on 2026.4.15, so this is not safe to close as implemented or superseded.
