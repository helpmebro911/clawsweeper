## Summary

Keep #70052 open. Current main still uses the upstream `estimateTokens` chars/4 estimator in the compaction and preemptive-compaction paths, while OpenClaw's CJK-aware estimator remains limited to other surfaces such as context pruning. The linked #70112 appears to be the right fix path, but it is an open PR and not shipped on main.

## Best Possible Solution

Keep #70052 open and use #70112, or an equivalent maintainer-owned patch, as the implementation path. The fix should introduce one shared CJK-aware message-token estimator for compaction, route `estimateMessagesTokens`, embedded compaction metrics/hooks, full-session before/after accounting, and pre-prompt synthetic messages through it, and add focused Japanese/Chinese/Korean regression tests for both compaction and preemptive compaction.
