## Summary

Keep #13613 open. Current main still runs cron inside each gateway with local JSON/state files plus in-process/shared-file guards; it has no distributed coordination config, Redis/etcd/DynamoDB backend, leader visibility, or public scheduler lock-provider plugin seam. The existing maintainer/Codex comment also kept this open, and #13582 is only a separate closed local scheduler reliability issue.

## Best Possible Solution

Keep #13613 open until maintainers either implement explicit HA cron coordination in core, or first add a documented scheduler/lock-provider plugin API that lets ClawHub backends provide Redis/etcd/DynamoDB-style coordination safely. The implementation should include config/schema/docs, health or CLI visibility for leadership/lock state, and multi-process or multi-node regression coverage. Treat #13582 as a separate closed local scheduler reliability bug, not the canonical tracker for this HA request.
