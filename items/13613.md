## Summary

Keep #13613 open. Current main still implements cron as gateway-local scheduling with process-local locking and no leader election, external lock backend, coordination config, or status/CLI health surface. The existing discussion correctly treats related #13582 as local scheduler reliability work, not HA cron coordination.

## Best Possible Solution

Keep #13613 open until OpenClaw either implements distributed cron coordination in the core scheduler or adds a documented scheduler/lock-provider extension API. The implementation should cover config/schema/docs, due-job ownership semantics across gateways, leader or lock health in status/CLI, and multi-process or multi-node regression coverage. Treat #13582 as shipped/local scheduler reliability work, not the canonical tracker for this HA request.
