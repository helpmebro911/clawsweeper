## Summary

Keep #13613 open. Current main still implements cron as a Gateway-local scheduler with process-local locking only; there is no cron coordination config/schema, distributed lock or leader-election backend, or leader/lock health surface. The related #13582 remains a closed local scheduler reliability bug, not a canonical tracker for multi-gateway HA cron coordination.

## Best Possible Solution

Keep #13613 open until OpenClaw either implements distributed cron ownership in the core scheduler or adds a documented scheduler/lock-provider extension API that can support it. The complete path should include config/schema/docs, explicit due-job ownership semantics across Gateways, leader or lock health in status/CLI, and multi-process or multi-node regression coverage. Treat #13582 as shipped local scheduler reliability work, not the canonical tracker for HA coordination.
