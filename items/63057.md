## Summary

Keep this PR open. Current main still contains the targeted Anthropic `options.thinkingBudgetTokens || 1024` fallback, and the resolver can still produce an explicit zero budget before payload construction. The PR is a plausible narrow fix, but it should add regression tests and may need to guard against all Anthropic-invalid budgets, not only `0`.

## Best Possible Solution

Keep this PR open as an implementation candidate. Update it to add focused regression tests in `src/agents/anthropic-transport-stream.test.ts` for the explicit-zero path and the undefined-budget default, and consider tightening the guard or resolver so the legacy Anthropic payload never sends a positive budget below the provider minimum. Then maintainers can decide whether to merge this narrow fix or fold it into broader Anthropic reasoning cleanup.
