## Summary

Keep this PR open. Current main still has the exact `options.thinkingBudgetTokens || 1024` fallback in the non-adaptive Anthropic thinking branch, and the zero-budget value this PR guards against is still reachable from the current token adjustment logic. The PR is a plausible focused fix, but the existing review comments are right that it should add regression coverage before merge.

## Best Possible Solution

Keep this PR open as an implementation candidate. Before merge, add targeted tests in `src/agents/anthropic-transport-stream.test.ts` that exercise a non-adaptive reasoning model with a resolved zero budget and with the default positive budget path, then let maintainers decide whether #63016's broader undefined-config default should be tracked separately.
