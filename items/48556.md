## Summary

Keep open. Current main still lacks FormData normalization in the shared proxy fetch wrappers that this PR targets. Runtime-fetch and Discord-specific multipart fixes exist, but they do not cover `makeProxyFetch()` / `resolveProxyFetchFromEnv()` or the public plugin SDK proxy helper surface. The newer #66243 overlaps, but it is also still open and unmerged, so this PR is not implemented or clearly superseded on main.

## Best Possible Solution

Keep the item open and either update #48556 or fold it into #66243 as a single maintained core proxy-fetch fix. The implementation should normalize global FormData to Undici FormData inside both `makeProxyFetch()` and `resolveProxyFetchFromEnv()`, keep the public SDK behavior covered by tests, and account for the overlapping Request-normalization work from #66243.
