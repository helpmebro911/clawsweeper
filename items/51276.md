## Summary

Keep open. Current `main` does not implement `cron.payload.kind="exec"`, and the public cron surfaces still only support `systemEvent` and `agentTurn`. This is neither implemented nor obsolete.

## Evidence

- **Core cron types still exclude exec payloads:** `CronPayload` and `CronPayloadPatch` on `main` only include `systemEvent` and `agentTurn`; there is no `exec` variant.
  - file: [src/cron/types.ts:112](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/cron/types.ts#L112)
  - command: `nl -ba src/cron/types.ts | sed -n '90,150p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Job validation still rejects isolated exec-style jobs:** `assertSupportedJobSpec` requires isolated/current/session cron jobs to use `payload.kind="agentTurn"`, so an `exec` payload would be rejected by the service layer.
  - file: [src/cron/service/jobs.ts:156](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/cron/service/jobs.ts#L156)

- **Gateway protocol schema has no exec payload branch:** The wire schema for `cron.add`/`cron.update` only unions `systemEvent` and `agentTurn` payloads, so the requested feature is not available through the public Gateway API.
  - file: [src/gateway/protocol/schema/cron.ts:130](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/gateway/protocol/schema/cron.ts#L130)

- **Agent cron tool surface still advertises only two payload kinds:** `CRON_PAYLOAD_KINDS` is `['systemEvent','agentTurn']`, and the tool payload schema exposes message/text/model/thinking/tools fields only; there are no `command`, `shell`, or `env` fields.
  - file: [src/agents/tools/cron-tool.ts:28](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/agents/tools/cron-tool.ts#L28)
