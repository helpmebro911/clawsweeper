## Summary

Keep #70588 open. Current main has Discord webhook support for internal thread-bound session delivery, but it does not expose an agent-callable `message` action named `webhook-create`, does not advertise that action in Discord discovery, and does not document it in the Discord message-tool surfaces. The request is recent, unprotected by maintainer labels, and still describes a real missing Discord plugin capability rather than stale or duplicate cleanup.

## Best Possible Solution

Implement this as a Discord-owned bundled plugin message action rather than a core migration: add `webhook-create` to the shared message-action name/target metadata, advertise it from Discord discovery behind an appropriate action gate, implement runtime creation via Discord's channel webhook route with target-channel resolution and optional name/avatar handling, return the generated URL in a secret-aware result, and cover it with focused Discord action tests plus docs/skill updates.
