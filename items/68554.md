## Summary

Keep this PR open. Current main still has both locked `loadSessionStore(..., { skipCache: true })` reads that the PR removes, and `skipCache` still bypasses the in-process cache before doing synchronous `readFileSync` plus `JSON.parse`. The PR is a narrow, relevant performance fix, strengthened by the Greptile review and the related live symptom report in #70857. It needs maintainer review rather than cleanup closure because the exact change intersects with an existing test that preserves external same-mtime writer behavior for `updateSessionStoreEntry`.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to remove the redundant locked-path synchronous disk parse if locked session-store updates are the supported write contract, or introduce a lock-aware fast path that avoids repeated large `sessions.json` parses while preserving whatever external-writer protection maintainers still require. Add or update focused coverage for queued locked writes and the accepted stale-cache edge case before landing.
