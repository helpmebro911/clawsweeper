## Summary

Keep this PR open. Current main still contains the two locked session-store reads this PR removes, and `skipCache` still bypasses the object cache before synchronous `readFileSync` and `JSON.parse`. The related discussion is also live: a commenter connected the same symptom to open issue #70857 on OpenClaw 2026.4.24. The PR remains a plausible focused fix, but it needs maintainer review because current tests also encode a stale-cache/external-writer protection contract.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to remove the redundant locked-path disk parse if the cache contract can be narrowed, or replace it with a lock-aware fast path that avoids repeated large synchronous parses while preserving any external-writer protection maintainers still require. Add focused coverage for queued locked writes and the accepted stale-cache edge case before landing.
