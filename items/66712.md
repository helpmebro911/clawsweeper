## Summary

Current `main` does not implement the PR’s slot-level cron delivery dedup guard or the `deliveryAttempted` propagation fix in startup catch-up. Existing code only dedups by `executionId = createCronExecutionId(jobId, runStartedAt)`, so a same-slot replay with a different `runStartedAt` remains plausible and the PR still addresses a real gap.

## Evidence

- **Current main only has execution-level dedup cache:** `COMPLETED_DIRECT_CRON_DELIVERIES` is the only delivery cache in `delivery-dispatch.ts`, and the direct-delivery idempotency key is built from `createCronExecutionId(jobId, runStartedAt)`, not from the planned slot timestamp.
