## Best Possible Solution

Keep the issue open and add a targeted regression harness that uses the production `sessions_spawn`/registry/control-runtime composition, then runs `/subagents list` from the same controller session and expects the active or recent run to appear. If the harness confirms duplicate module instances, anchor `subagentRuns` and related pending state in a process-global singleton or consolidate the runtime import boundary; if it instead confirms controller-key or persistence timing drift, patch that narrower path and keep parent completion delivery in the same validation lane.
