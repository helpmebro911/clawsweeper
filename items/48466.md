## Summary

Keep open. Current main still has the Telegram streaming-preview gap this PR targets: `draft-stream.ts` sends/edits rendered HTML with `parse_mode: "HTML"` and stops the preview on any thrown error, with no HTML parse-error plain-text fallback or per-generation disable path. Other Telegram delivery paths already have HTML parse fallback, so this remains a plausible focused bug fix rather than cleanup-close material.

## Best Possible Solution

Keep the PR open. The right path is to rebase or refresh it onto current `main`, review the latest head for the review-thread race fixes, and land a tested `draft-stream.ts` implementation that retries Telegram HTML parse failures as plain text for message and draft preview transports without corrupting generation state. If the PR is not continued, the same behavior should be implemented directly in the Telegram plugin with colocated regression tests.
