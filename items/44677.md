## Summary

Keep open. Current main still routes Feishu tools through the same Lark SDK auth path, still depends on `@larksuiteoapi/node-sdk` `^1.61.1`, and I did not find a landed OpenClaw-side fix or regression test proving the reported `tenant_access_token` failure is gone.
