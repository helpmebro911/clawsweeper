## Summary

Keep this PR open. Current main still installs Docker Playwright browsers under `/home/node/.cache/ms-playwright`, the public Docker/browser docs still describe that cache path, and the paired same-author bug report #42819 remains open. The PR is a plausible in-repo Docker/docs fix, but it needs maintainer review and updates before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to update it against current main, keep #42819 open until an implementation lands, remove or justify the apt-cache deletion, preserve non-root runtime writeability for both preinstalled and runtime-installed browsers, add focused Dockerfile/docs assertions for the new path and symlink, and then land if Docker build verification is clean.

## Risks / Open Questions

- Merging the PR as supplied may preserve the reviewed BuildKit cache regression if `rm -rf /var/lib/apt/lists/*` remains inside the cached `/var/lib/apt` mount.
- The Docker image should prove `/opt/playwright` is writable by the non-root `node` runtime user both with and without `OPENCLAW_INSTALL_BROWSER=1`.
- The symlink creation should preserve build failure propagation so a failed browser install cannot leave a dangling `/usr/local/bin/chromium` while the build succeeds.
