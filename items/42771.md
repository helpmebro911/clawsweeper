## Summary

Current main still uses the hidden Playwright cache path in the Docker image and public docs, and the same-author paired bug #42819 remains open. This PR is a plausible in-repo implementation candidate, but its current patch still needs Dockerfile ownership/cache fixes and docs reconciliation before it should merge.

## Best Possible Solution

Keep this PR open for maintainer review and contributor fixes. The best path is to rebase it onto current main, keep #42819 open until merge, set `/opt/playwright` as the Docker browser registry path with node ownership in both default and preinstall builds, recursively chown after preinstall, remove the BuildKit apt-cache deletion, create the chromium symlink only after a successful install, update `docs/install/docker.md` plus `docs/tools/browser-control.md`, and add focused Dockerfile assertions for path, ownership, symlink, and cache behavior.

## Risks / Open Questions

- Merging the current patch as-is can leave `/opt/playwright` unwritable by `USER node` in the default build path where `OPENCLAW_INSTALL_BROWSER` is unset.
- The patch still deletes `/var/lib/apt/lists/*` inside the named BuildKit apt lists cache mount, which can degrade rebuild performance without reducing the committed image layer.
- The docs patch is stale against current main because Docker Playwright guidance now lives in `docs/tools/browser-control.md` as well as `docs/install/docker.md`.
- The PR is currently not implemented on main and should stay paired with open bug #42819 until a corrected implementation lands.
