## Summary

Keep open. Current main still exposes `sessions_spawn.streamTo` in the ACP-capable schema without a field description explaining that it requires `runtime="acp"`, while execution still rejects `streamTo` when the selected runtime is `subagent`. The linked PR #69170 appears to be the right narrow fix, but it is open and not merged, so the issue is not implemented on main or the latest checked release.

## Best Possible Solution

Merge or recreate the narrow schema/documentation fix from PR #69170: add a `streamTo` description that says it requires `runtime="acp"` and should be omitted for `runtime="subagent"`, optionally mirror that in `describeSessionsSpawnTool`, and add a regression assertion in `src/agents/tools/sessions-spawn-tool.test.ts` so the hint cannot disappear again.
