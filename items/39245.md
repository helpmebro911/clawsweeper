## Summary

Keep this PR open. Current main still does not implement the PR's central behavior: pairing and sanitizing mixed `functions.exec:0` / `functions exec:0` IDs and stripping `functions.` / `functions ` prefixes from persisted transcript tool names. The PR is active, recently rebased, and its predecessor #39091 is closed unmerged rather than a canonical shipped fix.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a narrow agent-side fix that normalizes only the intended leading `functions ` versus `functions.` provider mangling where pairing requires it, preserves replay-safe raw IDs, keeps `sessions_spawn` attachment redaction covered for prefixed names, and adds regression tests for mixed dot/space IDs and prefixed tool names. Any unrelated node workdir changes should either be justified as required by current main or split away.
