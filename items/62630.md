## Summary

Keep #62630 open. Current main still preserves some cross-root absolute sessionFile paths and can pass them into transcript/lock mkdir paths, while legacy state migration and migration docs do not provide a Docker-to-host /home/node rewrite, diagnostic, or doctor repair path.

## Evidence

- **Cross-root absolute session paths are still preserved:** resolveSessionFilePath keeps canonical cross-root .../agents/<agentId>/sessions/<file> absolute paths as a structural fallback instead of rewriting them to the current state dir.

- **Regression test confirms old absolute roots are intentionally returned:** The test for OPENCLAW_STATE_DIR differing from stored paths expects a stored /original/state absolute transcript path to be returned unchanged, which is the same class of portability issue as Docker /home/node paths on a host install.
  - file: [src/config/sessions.test.ts:618](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/config/sessions.test.ts#L618)
