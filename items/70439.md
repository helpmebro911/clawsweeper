## Summary

Current main only partially addresses the report: Signal now preserves attachment paths/MIME metadata so the generic media-understanding pass can classify voice notes after a message is accepted, but Signal still has no channel preflight transcription path or configurable preflight hook like Telegram, Discord, or WhatsApp. The maintainer discussion already made the same keep-open call on April 25, 2026, and current main still matches that assessment.

## Best Possible Solution

Keep the issue open. The best fix is a shared channel audio-preflight helper wired into Signal before mention gating/agent dispatch, reusing the existing media-understanding transcribeFirstAudio path, setting Transcript/BodyForAgent/RawBody/CommandBody and MediaTranscribedIndexes, and adding Signal tests/docs for DMs plus requireMention group voice notes. A shell preflight_cmd should be a separate product/API decision if maintainers still want that configurable hook.
