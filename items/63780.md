## Summary

Keep this PR open. Current main still has the generic uncaught-exception handler registry, but it does not implement the PR's two remaining behaviors: suppressing transient uncaught network failures in the CLI/global handlers and wiring Telegram polling recovery through uncaught exceptions. The linked ENETUNREACH issue remains open, and a production user comment corroborates the Telegram polling stall symptoms.

## Best Possible Solution

Keep this PR open for maintainer review or rebase. The best path is to port the PR's transient uncaught suppression and Telegram uncaught polling restart onto current main, preserve fatal hooks for non-transient uncaught exceptions, keep suppression limited to AbortError/transient-network and Telegram-tagged polling failures, add the CLI ENETUNREACH and Telegram uncaught HttpError/getUpdates regression tests, and resolve or consciously accept the duplicate legacy-handler logging issue.
