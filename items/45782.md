## Summary

Keep #45782 open. Current main still has the silent-failure patterns this PR targets, but the submitted patch is not safe to merge as-is because review comments identify real regressions in the proposed logging calls and the TLS chmod warning severity remains too low.

## Best Possible Solution

Keep the PR open but require a revised implementation: use the existing subsystem logger or a correctly typed warning path instead of logWarn(message, { error }), log TLS permission restriction failures at warn severity, update the current allow-from-store-file.ts path, and add focused tests for malformed allow-from JSON and any cleanup logging that must stay non-throwing.
