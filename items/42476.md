## Summary

Keep open. Current `main` does not implement the PR’s cache-busting change for temporary `/media/:id` responses, so there is no basis to close this as already fixed or obsolete.

## Evidence

- **Current media route still only sets `X-Content-Type-Options`:** On `main` (`d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132`), the `/media/:id` handler sets `X-Content-Type-Options: nosniff` and then proceeds directly into request handling. There is no `Cache-Control`, `Pragma`, or `Expires` header set in this route.
  - file: [src/media/server.ts:49](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/media/server.ts#L49)
  - command: `nl -ba src/media/server.ts | sed -n '1,220p'`
  - sha: [d29eaeafc1fc](https://github.com/openclaw/openclaw/commit/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132)
- **Current tests do not cover the proposed anti-cache headers:** The shared response assertion helper only checks status and optional `x-content-type-options`, and the missing-media case asserts `expectedNoSniff: true` only. Current `main` therefore has no regression coverage for the headers proposed in the PR.
  - file: [src/media/server.test.ts:60](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/media/server.test.ts#L60)
  - command: `nl -ba src/media/server.test.ts | sed -n '1,240p'`
  - sha: [d29eaeafc1fc](https://github.com/openclaw/openclaw/commit/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132)
- **Related sensitive-serving code already uses cache suppression elsewhere:** A comparable file-serving path in the canvas host explicitly sets `Cache-Control: no-store`, which reinforces that the media route’s lack of cache suppression is a real gap rather than an intentional repo-wide policy.
  - file: [src/canvas-host/server.ts:403](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/canvas-host/server.ts#L403)
  - command: `nl -ba src/canvas-host/server.ts | sed -n '392,420p'`
  - sha: [d29eaeafc1fc](https://github.com/openclaw/openclaw/commit/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132)

## Risks / Open Questions

- Temporary capability-style media URLs on current `main` can still be cached by browsers or intermediaries because the requested anti-cache headers are not present on `/media/:id` responses.
