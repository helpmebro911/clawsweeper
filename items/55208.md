## Summary

Keep open. Current main still does not implement a macOS LaunchDaemon/system-scope mode for the Gateway service. The managed macOS path remains LaunchAgent-only in the CLI, daemon registry, launchd implementation, macOS app integration, tests, and docs; the existing status scanner can detect system LaunchDaemons, but OpenClaw does not install or manage one.

## Best Possible Solution

Keep this issue open for maintainer product and security judgment. The best implementation would add an explicit macOS system-scope service mode, likely on the `openclaw gateway install`/lifecycle surface rather than only `start`, with a LaunchDaemon service implementation for `/Library/LaunchDaemons`, `UserName` and environment/config-path semantics, sudo install/uninstall flow, `launchctl system/...` lifecycle support, status/doctor/docs coverage, and regression tests that preserve the current per-user LaunchAgent as the default.
