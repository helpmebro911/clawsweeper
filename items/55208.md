## Summary

Current main still does not implement the requested macOS LaunchDaemon/system-scope Gateway mode. The managed macOS Gateway lifecycle remains LaunchAgent-only: the CLI has no daemon/scope option, the typed install contract cannot carry that intent, Darwin service resolution maps only to the LaunchAgent adapter, launchd writes under the user's LaunchAgents directory and uses the gui domain, the macOS app invokes the same LaunchAgent install path, and docs still describe custom LaunchDaemons as not shipped.

## Best Possible Solution

Keep this issue open for explicit macOS Gateway lifecycle design. The likely path is an opt-in system-scope LaunchDaemon mode on the managed service surface, with matching install/start/stop/restart/status/doctor/docs/tests and security review, while keeping the current per-user LaunchAgent default.
