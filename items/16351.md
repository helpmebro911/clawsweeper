## Summary

Keep #16351 open. Current `main` already implements `openclaw config validate` and config write protections for invalid or secret-resolved payloads, but the explicit request for bare `openclaw config` to show help instead of entering the wizard on an already configured system is still unresolved. The existing maintainer-posted Codex review on the issue remains accurate against current HEAD.

## Best Possible Solution

Keep #16351 open until a maintainer either implements the remaining root `openclaw config` behavior or explicitly decides that the wizard default is intentional. The likely implementation belongs in `src/cli/config-cli.ts` and `src/commands/configure.commands.ts`: when no subcommand and no `--section` are supplied on an already configured system, print config help instead of entering guided setup, while preserving the shipped `config validate` and SecretRef/env-preserving write behavior and updating docs/tests accordingly.
