## Summary

Keep #16351 open. Current main already covers `openclaw config validate` and protects config writes from invalid/destructive or secret-resolved payloads, but the remaining root-command request is still unresolved: bare `openclaw config` still dispatches into the guided configure wizard, and the docs still document that behavior.

## Best Possible Solution

Keep #16351 open until a maintainer either implements the remaining bare `openclaw config` behavior or explicitly decides the wizard default is intentional. The likely change belongs in `src/cli/config-cli.ts` and `src/commands/configure.commands.ts`: when no subcommand and no `--section` are supplied on an already configured system, print config help instead of entering guided setup, while preserving the shipped `config validate` and SecretRef/env-preserving write behavior and updating docs/tests.
