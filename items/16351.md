## Summary

Keep #16351 open. Current main addresses important parts of the request, including `openclaw config validate` and write-back protection for existing env/SecretRef-style secrets, but the issue’s first requested behavior remains missing: bare `openclaw config` still intentionally starts the configure wizard instead of showing help on an already configured system. That is a meaningful remaining CLI behavior request, so this should not be closed as implemented.
