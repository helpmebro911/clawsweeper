## Summary

Keep open. Current `main` does not implement this PR's node-unavailable fallback behavior. The pre-reply path now avoids treating mixed-cause `FallbackSummaryError`s as pure rate limits, but the mid-turn no-content path still rewrites mixed `rate_limit` + `node invoke timed out` errors into generic rate-limit copy, and the node-timeout regression tests from the PR are not present on `main`.

## Evidence

- **Pre-reply path has no node-unavailable fallback branch on main:** `fallbackText` only special-cases billing, rate-limit/overloaded, context overflow, and role-ordering errors. There is no node-specific user message or classifier in the current pre-reply error ladder.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:1575](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/auto-reply/reply/agent-runner-execution.ts#L1575)

A mixed message like `429 ... | errorCode=UNAVAILABLE ... node invoke timed out` would still be rewritten as rate-limit copy here.
