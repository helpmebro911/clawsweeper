## Summary

Keep this issue open. Current main still treats model pricing as USD-only, has no `costCurrency` or equivalent schema/type/config surface, and multiple user-visible cost surfaces still format with a hardcoded dollar sign. The prior ClawSweeper comment correctly identified the same gap, and no later main commit appears to have implemented it.

## Best Possible Solution

Keep this issue open for a real core config/display enhancement. Maintainers should decide whether currency metadata belongs globally, per provider, or per model; then add a validated schema/type field with USD as the default, update generated config docs/help and public docs, and thread currency-aware formatting through `/status`, `/usage full`, `/usage cost`, CLI usage summaries, Control UI usage cards, and chat metadata. Mixed-currency totals should be labeled, split, or suppressed rather than silently summed under one symbol.

## Risks / Open Questions

- A display-only currency symbol could mislead if one session aggregates usage from providers priced in different currencies; implementation needs a clear global, provider, or model-level currency policy and mixed-currency behavior.
- Changing the cost contract away from USD affects config schema, generated schema/help, docs, plugin manifest model catalog semantics, status/usage/chat UI display, and transcript/session aggregation expectations.
