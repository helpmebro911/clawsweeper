## Summary

Current main still lacks `costCurrency` or an equivalent currency metadata/display contract. Pricing config, runtime cost structs, docs, diagnostics, chat/CLI/status output, and Control UI formatters remain USD/dollar-oriented, so the requested user problem is still actionable core work.

## Best Possible Solution

Keep this issue open for a real core pricing-currency contract. Maintainers should choose the boundary first, likely global, per-provider, per-model, or explicit USD storage plus display conversion, then update config schema/types, generated/public docs, plugin provider catalog semantics, runtime cost lookup, transcript/session/diagnostic fields, CLI/chat/status output, Control UI usage views, and mixed-currency aggregation behavior.
