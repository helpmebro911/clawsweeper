## Summary

Keep open. Current main partially addresses the old hang class by emitting ACP lifecycle end/error events and registering ACP child runs as background tasks, but it does not implement this PR's central fallback: recovering missing parent relay assistant output from gateway `chat.history` or terminal state from `agent.wait` when local relay events do not arrive. The PR also has unresolved review comments about duplicate-output and stale-history races, so it needs a current-main rework rather than cleanup closure.

## Best Possible Solution

Keep this PR open or replace it with a current-main version of the same fix: add a bounded gateway-backed fallback inside `startAcpSpawnParentStreamRelay` that reads child assistant output from `chat.history` and terminal state from `agent.wait` only when local events are missing, with explicit dedupe/race guards for the review-comment cases and regression tests for missing assistant, missing terminal, stale history at lifecycle end, and mixed history-first/local-event-second ordering.
