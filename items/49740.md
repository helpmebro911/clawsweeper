## Summary

Keep open. Current main has one-shot cron retry policy and a recurring-job error backoff floor, but it still does not implement per-job `--retry-count`, `--retry-delay`, or `--retry-backoff` for recurring cron jobs, and daily recurring failures can still wait for the next natural schedule. The linked PR #51001 tracks the request but is still open/unmerged, so the issue remains actionable.
