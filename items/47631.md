## Summary

Keep #47631 open. Current main still forces Telegram DM answer and reasoning preview lanes onto message transport, and the current tests/docs assert sendMessage/editMessageText behavior rather than native draft transport for the DM answer lane. The PR is not implemented on main, and the linked closed PR #42479 is not a shipped superseding change.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrowed Telegram patch that restores `previewTransport: "auto"` only for the DM answer lane, keeps reasoning on message transport, and includes the draft cleanup plus ambiguous-materialize regression coverage before replacing the current message-transport tests/docs.
