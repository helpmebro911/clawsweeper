## Summary

Keep open. Current main does not implement the PR's central behavior: cron failure alerts still show only the raw last error, cron run-log entries and gateway schema do not expose or backfill errorReason, and CLI JSON output is still passed through unchanged. The PR discussion also contains unresolved protocol/client compatibility review comments and useful field evidence about broader cron timeout/exec-policy behavior, so this needs maintainer review rather than conservative cleanup.
