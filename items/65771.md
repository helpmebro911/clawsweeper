## Summary

Keep open. Current main does not implement the PR's central behavior: cron failure alerts still print only raw error text, cron JSON output is passed through unchanged, and run-log entries/protocol schema still lack `errorReason`. The PR is a plausible focused fix with regression coverage in its provided diff, though the protocol enum and generated Swift surfaces still need maintainer review before merge.

## Best Possible Solution

Keep the PR open and ask for a small maintainer-quality finish: reuse or centralize the complete cron failover reason schema, validate/backfill persisted run-log `errorReason` values before exposing them through gateway protocol, keep generated Swift surfaces source-compatible where needed, and land the focused regression tests for alert text, CLI JSON, and older run-log entries.
