## Summary

Keep open. Current main still has the core visibility gap described in #49380: `/status` reads `SessionEntry.compactionCount`, but plugin context engines running proactive/background compaction from `afterTurn()` still have no callback, return value, or stats path to report those compactions into the session store. The existing increment paths cover built-in auto-compaction and explicit `compact()` calls, not leaf/proactive `afterTurn()` compactions owned by third-party engines.

## Evidence
