## Summary

Keep open. Current main has adjacent improvements for sub-agent timeouts, Ollama context sizing/stream assembly, and empty-error retries, but I did not find source-backed evidence that the reported large-context Ollama sub-agent behavior is fixed. The issue is recent, non-maintainer-authored, and still concrete enough to investigate; the KeepMyClaw comment is a workaround for recovery after long runs, not evidence that OpenClaw core no longer fails.

## Evidence

- **Sub-agent timeout default is no timeout, but this only covers one likely cause:** Current main resolves omitted sub-agent runTimeoutSeconds to the configured default or 0, and docs define 0 as no timeout. That may address the timeout hypothesis but does not prove the short code-regurgitation/no-analysis symptom is gone.
