## Summary

Keep this issue open. Current main has meaningful partial session/topic management, but it does not yet ship the full requested topic-management UI and metadata surface: no first-class Control UI create/import/export/archive-restore flow, no TUI rename/delete/archive management, no session row message count, and no description/tags/createdAt topic metadata contract.

## Best Possible Solution

Keep this issue open, but narrow it into concrete remaining work: add a first-class Control UI create/manage workflow around sessions.create, sessions.patch, and sessions.delete; decide whether description, tags, createdAt, and messageCount belong in the Gateway session row contract; add TUI rename/delete/archive actions only if maintainers want management beyond the existing picker; and define deleted-archive restore plus import/export semantics with docs and focused tests.
