## Summary

Keep #8724 open. Current main still does not provide a typed global or per-model text-generation timeout for `agents.defaults.models`; the existing controls are a whole-run timeout, an idle/no-chunk LLM watchdog, and media/tool-specific request timeouts. Related first-token/request-timeout items are adjacent but do not solve the reported active-streaming case where a model keeps emitting non-useful output past a model-specific budget.

## Best Possible Solution

Keep #8724 open until OpenClaw implements and documents a typed global/per-model text-generation timeout contract, or a maintainer explicitly rejects that product direction. A complete implementation should add schema/type/generated-doc support, resolve the effective timeout for each fallback candidate, abort and log timeout events, integrate timeout classification with fallback handling, and add regression coverage for a stream that continues emitting chunks past the configured budget.
