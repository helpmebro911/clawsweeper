## Summary

Keep #8724 open. Current main still does not implement a typed global or per-model text-generation timeout for `agents.defaults.models`; the existing controls are whole-run timeout, idle/no-chunk watchdog, and media/tool-specific request timeouts. The maintainer/Codex discussion on 2026-04-26 also explicitly distinguishes this active-streaming generation-budget request from adjacent first-token/request-timeout items.

## Best Possible Solution

Keep #8724 open until OpenClaw implements and documents a typed global/per-model text-generation timeout contract, or a maintainer explicitly rejects the product direction. A complete fix should add schema/type/generated-doc support, resolve the effective timeout per fallback candidate, abort and log total generation-budget expirations, classify them for fallback handling, and add regression coverage for a stream that continues emitting chunks past the configured budget.
