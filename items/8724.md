## Summary

Keep #8724 open. Current main still has only a global agent runtime timeout and a global LLM idle watchdog; it does not provide the requested global/per-model generation timeout config under `agents.defaults.models`, and the idle watchdog will not stop a model that keeps streaming looping text.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a documented generation-timeout contract or a maintainer explicitly rejects the product direction. A complete fix should add typed and schema-validated global/per-model config, plumb it into embedded model attempts and fallback accounting per candidate, keep it distinct from idle/first-token timeout behavior, document it, and add regression tests for a model that keeps streaming non-useful planning chunks past the configured generation budget.
