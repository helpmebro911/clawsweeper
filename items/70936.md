## Summary

Keep open. The PR targets the old core `src/media/pdf-extract.ts` implementation, but current `main` delegates PDF extraction to the bundled `document-extract` plugin. The underlying `standardFontDataUrl` behavior is still missing in the active plugin PDF.js call, so this is not implemented on main and should not be closed as obsolete.

## Best Possible Solution

Keep the item open or consolidate it with the canonical PDF standard-font tracking item. The concrete fix should move into `extensions/document-extract/document-extractor.ts`: resolve `pdfjs-dist/package.json` from the plugin/runtime package root, pass a trailing-separator filesystem `standardFontDataUrl` into PDF.js `getDocument`, update the local PDF.js type surface, and add focused coverage in `extensions/document-extract/document-extractor.test.ts`.
