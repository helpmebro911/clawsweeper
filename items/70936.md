## Summary

Keep open. Current main has moved PDF fallback extraction behind the bundled `document-extract` plugin, and the active PDF.js call in that plugin still omits `standardFontDataUrl`. PR #70936 targets the older core `src/media/pdf-extract.ts` implementation, so it needs retargeting, but the linked issue #51455 and PR discussion describe a real remaining bug rather than obsolete cleanup.

## Best Possible Solution

Keep the item open, but ask for the package-root `standardFontDataUrl` fix and focused regression test to be retargeted to the bundled `document-extract` plugin, which is the current owner of PDF.js fallback rendering. The linked issue #51455 remains the canonical bug until that active path passes standard font data into PDF.js.
