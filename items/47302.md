## Summary

Keep open. Current main still does not implement `openclaw doctor --check-config` or an equivalent doctor deep config validation mode. Existing config validation is schema-level, and current doctor/model/TTS surfaces do not provide the PR’s combined schema + model resolution/catalog/fallback + TTS + channel CI gate.

## Best Possible Solution

Keep #47302 open for maintainer review or rework. The best product path is a narrow core `doctor --check-config` implementation that reuses current config, model-selection, TTS, and channel/plugin contracts, exits non-zero for real failures, and includes targeted regression tests for the failure modes documented in #45552 and #20909. Close only after a separate merged PR ships the same deep validation surface.
