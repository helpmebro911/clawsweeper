## Summary

Keep PR #48608 open. Current main still does not contain `docs/concepts/multi-agent-patterns.md` or an equivalent practical orchestration-patterns guide; existing docs cover multi-agent routing, session tools, subagents, and task-flow primitives, but not the proposed hub-and-spoke/cost-tiered/channel-bound/heartbeat/pipeline/anti-patterns guide. The linked issue #43034 was closed as duplicate/superseded in favor of this open PR, so closing the PR would drop the canonical remaining docs work. The PR needs revision for the existing review comments, but that is an actionable docs review path rather than a cleanup close reason.

## Best Possible Solution

Keep PR #48608 open as the canonical docs work for #43034. The best path is to revise the new guide against the current session-access contract: add `engineer` to the Pattern 1 allowlist or change the diagram, include `tools.sessions.visibility: "all"` where cross-agent `sessions_*` calls are expected, then have docs review decide whether the guide belongs as a standalone `docs/concepts/multi-agent-patterns.md` page or should be folded into the existing multi-agent/session-tool docs.
