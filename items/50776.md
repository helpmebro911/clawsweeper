## Summary

Keep open. Current main still does not make the reported Slack directory CLI commands use Slack's live directory API, and Slack `directory self` still falls back to the generic null default. The issue is recent, non-maintainer-authored, has no protected labels, and is a concrete bundled Slack plugin behavior gap rather than ClawHub work or a duplicate of the provided token-selection report.

## Evidence

- **CLI list path is config-backed only:** `openclaw directory peers list` and `groups list` select `plugin.directory.listPeers` / `listGroups`, not the live variants, before invoking the adapter.
  - file: [src/cli/directory-cli.ts:169](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/cli/directory-cli.ts#L169)

- **Slack registers live methods separately:** Slack registers config-backed `listPeers`/`listGroups` and separately wires `listPeersLive`/`listGroupsLive`; the directory CLI path above does not call those live methods.
  - file: [extensions/slack/src/channel.ts:427](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/extensions/slack/src/channel.ts#L427)
