## Summary

Keep #72094 open. Current main has a bundled Codex app-server harness, /codex chat binding, ACP Codex fallback, and PeekabooBridge, but it does not yet expose the requested Codex Desktop Computer Use route, computer_use availability probe, desktop-control provider selection, or channel-safe policy for Discord/Slack-originated desktop-control tasks. The issue is a recent, specific follow-up to #67776 and is not superseded by the broader closed computer-use/VNC discussions in the supplied context.

## Best Possible Solution

Keep this open as a focused Codex plugin/harness feature request. The best path is a narrow, version-gated design in the bundled Codex runtime: probe Codex Desktop/app-server for computer_use availability, expose an explicit codex-desktop or computer-use routing option, route channel-originated requests through that path only behind explicit policy, and keep Peekaboo as a fallback. If Codex app-server lacks a stable computer-use task API, track that upstream contract/API gap rather than closing the OpenClaw issue.
