## Summary

Keep #59514 open. Current main has added a separate Amazon Bedrock Mantle provider that covers Bearer-token auth for OpenAI-compatible Bedrock/Mantle endpoints, and the issue discussion correctly recognizes that as a useful adjacent path. However, the specific requested behavior remains unsupported: `api: "bedrock-converse-stream"` with a plain `Authorization: Bearer <api-key>` path that bypasses AWS SDK SigV4 signing for Bedrock Converse-format proxies.

## Evidence

- **Issue scope is specific to Bedrock Converse-format proxy auth:** The reporter asks for `auth: "api-key"` on `api: "bedrock-converse-stream"`, sending `Authorization: Bearer <api-key>` and skipping SigV4. A follow-up comment says `authHeader: true` still fails because the AWS SDK signing step overwrites the Bearer header.
  - command: `provided GitHub context for issue #59514`
- **Standard Bedrock provider remains the AWS SDK / Converse provider path:** The bundled `amazon-bedrock` provider registers the standard Bedrock provider, with `auth: []` and Bedrock-specific implicit provider/auth resolution rather than an `auth: "api-key"` transport branch for Converse-format requests.
