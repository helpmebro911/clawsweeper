## Summary

Keep this issue open. Current main still exposes `running | done | failed | killed | timeout` as the persisted, gateway-facing, UI-facing, and `sessions_list` status vocabulary, without an additive `resumable`, `lastTurnStatus`, `lifecycleStatus`, or `taskStatus` field or clear agent-facing documentation that `failed`, `timeout`, and `done` can still be followed up via `sessions_send`. The closed related issues #69202 and #71756 are useful adjacent lifecycle/recovery context, but they do not supersede this status/resumability contract problem.

## Best Possible Solution

Keep this issue open and treat it as an agent-facing session API contract bug. Add a backward-compatible `resumable` field plus a distinct `lastTurnStatus`, `lifecycleStatus`, or `taskStatus` field to `sessions.list` rows, document the semantics on https://docs.openclaw.ai/concepts/session-tool, and update tool descriptions or prompt guidance so orchestrators prefer `sessions_send` for known visible sessions unless the session is deleted or inaccessible. Add focused tests for the returned row shape and guidance; defer any status-string rename to a versioned migration.
