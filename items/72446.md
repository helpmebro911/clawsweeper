## Summary

Keep open. Current main still exposes all succeeded Microsoft Foundry deployments during Entra onboarding and still routes the built-in provider through OpenAI-compatible Foundry paths, so the linked bug in #60546 is not already implemented or obsolete. This PR is a focused, recent implementation candidate with provider-local changes and no evident supply-chain/security-sensitive file changes, but it should get normal maintainer review before landing because the proposed diff appears to filter the prompt choices without also filtering the persisted discovered deployment catalog.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to finish the provider-owned guardrail in `extensions/microsoft-foundry`: filter Anthropic deployments consistently from both onboarding choices and persisted discovered model catalog, keep the custom-provider workaround note, and add regression coverage for mixed and Anthropic-only deployment lists before landing it as the implementation candidate for #60546.
