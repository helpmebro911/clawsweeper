## Summary

Keep open. Current main still uses the old `!result.delivered` fallback in `dispatchAgentHook`, so a `deliver:false` shared hook can still enqueue a main-session system event. The PR is not obsolete or superseded by a shipped fix; it remains a focused candidate for the open hook-fallback bug lineage.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or refine a hooks-local announcement policy that suppresses successful silent/shared hook results, avoids duplicate fallback after delivered or handled delivery paths, still surfaces real hook errors, and updates the gateway tests that currently encode the old `deliver:false` fallback behavior.
