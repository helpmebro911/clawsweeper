## Summary

Keep open. Current main partially addresses the reported CI race by waiting for marker content, shipped in v2026.4.24 via 63653f51d82d0bf2a5ce17eb678c6702b37f0054, but the issue also reports a Windows stopped-marker failure and current main still uses the unguarded SIGTERM marker oracle. The related fuller fix in #70669 was closed unmerged, so there is no canonical merged resolution for the remaining test determinism gap.

## Best Possible Solution

Keep the issue open until the remaining test oracle is resolved or explicitly descoped. The narrow code path is `extensions/qa-lab/src/lab-server.test.ts`: preserve the content wait, record the fixture process PID, verify process exit after `lab.stop()`, and only use the graceful SIGTERM stopped-marker assertion on platforms where that signal handler is reliable. Then validate the focused QA-lab extension test on the supported CI lane, and ideally on Windows if the Windows reproduction remains in scope.
