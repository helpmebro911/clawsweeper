## Summary

Keep open. Current main supports remote MCP over SSE/streamable HTTP with static headers, but it does not expose or wire an MCP OAuth authProvider, browser callback flow, finishAuth handling, or persisted OAuth credentials for remote MCP servers such as Heptabase. This is a generic MCP client capability gap, not a ClawHub-only plugin close, because the missing piece is the core/runtime transport authentication flow or an extension API for it.

## Evidence
