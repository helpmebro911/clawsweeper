## Summary

Keep this PR open. Current main still drops Feishu card-action fields outside `action.value`, and the same-author issue it closes remains open. The patch is a plausible implementation candidate, but it needs a current-main rebase because Feishu card actions now pass through parser, decoder, and fallback layers that the posted diff does not fully update.

## Best Possible Solution

Keep this PR open as an active implementation candidate for #43202, or have maintainers choose one canonical Feishu card-action PR if they want to consolidate #42908 and this PR. The best fix should extend the Feishu callback parser/event type to preserve `form_value`, `input_value`, `name`, `option`, and `options`, adapt the current decoder/fallback flow, and add parser-to-handler tests while preserving existing plain command extraction.
