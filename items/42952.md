## Summary

Keep open. Current main still does not implement a Feishu `ignoreFileContent` or equivalent attachment-content opt-out. Feishu inbound file/media messages are still downloaded into `MediaPaths`, and the shared media-understanding path can still extract eligible text/PDF attachment contents into `<file>` prompt blocks before the agent turn.

## Best Possible Solution

Keep this issue open and implement a Feishu-owned inbound attachment content opt-out. A conservative fix would add a documented per-account/top-level Feishu setting, wire it before `resolveFeishuMediaList` downloads file/document resources or before those entries enter `mediaPayload`, preserve placeholder text such as `<media:document> (name)`, and add focused Feishu plus shared media-understanding regression tests. Update Feishu schema, generated config metadata/help, and https://docs.openclaw.ai/channels/feishu.
