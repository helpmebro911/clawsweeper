## Summary

Keep open. Current main still lacks a Feishu `ignoreFileContent` config surface and still routes downloaded Feishu file/media attachments into the shared media-understanding path, where text-like and PDF file contents can be extracted and appended to the agent prompt.

## Best Possible Solution

Keep this issue open and implement a real Feishu-owned or shared inbound-attachment opt-out. The likely path is `channels.feishu.ignoreFileContent` with per-account inheritance, wired before `resolveFeishuMediaList` or before shared `extractFileBlocks`, plus regression tests proving Feishu file messages produce only a placeholder and do not add downloaded `MediaPaths` or extracted `<file>` prompt blocks when enabled. Docs should explain how it interacts with `mediaMaxMb` and media understanding.
