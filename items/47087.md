## Summary

Keep #47087 open. The current main implementation still starts `sessions_spawn` child runs without `inputProvenance`, while the existing runner and transcript plumbing already support consuming that field once supplied. The related #57849 is adjacent hook-message preservation work, not a superseding fix for spawning child task provenance.

## Best Possible Solution

Keep the PR or its linked issue #47070 open. The best implementation is still the small spawn-path change: add `inputProvenance: { kind: "inter_session", sourceSessionKey: requesterInternalKey, sourceChannel: requesterOrigin?.channel, sourceTool: "sessions_spawn" }` to the child `agent` gateway params in `src/agents/subagent-spawn.ts`, plus a focused regression assertion in `src/agents/subagent-spawn.workspace.test.ts`. #57849 should remain separate because it addresses preserving already-written provenance in plugin hook messages, not adding provenance to spawned child tasks.
