## Summary

Keep open. Current main still has the narrow parser weakness the PR targets: `parseAbsoluteTimeMs` calls `input.trim()` without a nullish guard, and malformed cron schedule rows can still reach that parser through CLI schedule formatting. The cron fix is plausible and not implemented on main, but the branch also bundles an unrelated package-update early exit that needs review or splitting before merge.

## Best Possible Solution

Keep this PR open for maintainer review, but ask the author to split or remove the update-command change and focus the fix on the cron path: harden `parseAbsoluteTimeMs` or its call sites for nullish input, then add a regression test through cron list/show or schedule formatting.
