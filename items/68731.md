## Summary

Keep open. Current main still only enriches BlueBubbles contact names for group participants, not 1:1 inbound DM sender labels, and the email-addressed sender extension requested in the issue is also explicitly absent. The issue has useful downstream implementation evidence and maintainer follow-up asking for a PR, so it is not stale, duplicate, or a ClawHub cleanup candidate.

## Best Possible Solution

Implement this in the bundled BlueBubbles plugin: add a reusable sender-name resolver with usable-name detection, extend the existing Contacts lookup/cache to handle 1:1 sender ids and email handles, preserve the legacy group config flag through a compatibility shim if renaming, and cover the new DM and email paths with focused tests plus docs updates.
