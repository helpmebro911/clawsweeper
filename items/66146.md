## Summary

Keep this PR open. Current main still has the OpenAI-compatible transport gap the PR targets: Responses and Completions only forward max-token fields when stream options explicitly include `maxTokens`, while OpenRouter model discovery already populates `model.maxTokens`. The PR is still useful, but it should not land as-is because it bundles an unrelated TUI refactor with review-reported regressions and lacks focused fallback regression tests.

## Best Possible Solution

Keep this PR open, but ask the contributor to rebase and narrow it to the transport fix. The best patch should resolve an effective positive max-token value from explicit stream options first and `model.maxTokens` second, send it through the existing `max_output_tokens` / `max_tokens` / `max_completion_tokens` compatibility logic, and add focused tests for both Responses and Completions when `options.maxTokens` is omitted. The unrelated TUI refactor should be dropped from this PR or handled separately after its review comments are fixed.
