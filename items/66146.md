## Summary

Keep this PR open. Current main still omits the model-derived max-token fallback in both OpenAI-compatible stream request builders, so the agents transport bug remains. The PR is a relevant implementation candidate, but it should be narrowed before landing because it bundles unrelated TUI changes and currently lacks focused omitted-options regression tests.

## Best Possible Solution

Keep this PR open as a relevant implementation candidate, but ask for a rebase and a narrower patch. The best fix belongs in src/agents/openai-transport-stream.ts: resolve an effective positive max-token value from options.maxTokens first and model.maxTokens second, preserve the existing max_output_tokens / max_tokens / max_completion_tokens compatibility behavior, and add focused regression tests for omitted options.maxTokens in both request builders. Drop the unrelated TUI refactor or handle it separately after its review comments are resolved.
