## Summary

Current main still does not implement per-agent memory-wiki vault configuration or ctx-aware vault path resolution. The bundled memory-wiki plugin resolves one plugin config at registration, the loader supplies config from the single global `plugins.entries.memory-wiki.config` entry, and strict agent config still has no `plugins` field. The related implementation candidate #67584 remains the right path to review rather than closing this issue first.

## Best Possible Solution

Keep this issue open as the narrow memory-wiki vault isolation request. Review and land #67584 or an equivalent memory-wiki-owned implementation that resolves the vault root per invocation context, covers `wiki_search`, `wiki_get`, `wiki_apply`, status/lint/gateway/CLI behavior or clearly documents their scope, preserves the current single-vault default, and adds tests proving two agents on one gateway cannot read or write each other's wiki vaults.
