## Summary

Keep open. Current main has adjacent multi-agent, per-agent tool policy, group toolsBySender, cron toolsAllow, and Feishu-specific dynamic-agent primitives, but it still does not provide the requested first-class org/team deployment package: no team workspace scaffold CLI, no declarative agents.yaml-style deployment manifest, and no complete per-session/operator-scope RBAC surface. The narrower RBAC work is still represented by open RFC #49183 and open PRs #53496/#53530 in the provided discussion.

## Best Possible Solution

Keep this open as the umbrella product/design tracker unless maintainers split it. The practical path is to review or close the focused RBAC RFC/PRs (#49183, #53496, #53530), then decide whether workspace scaffolding and deployment manifests belong in core CLI/config, a documented plugin API, or a ClawHub/plugin pattern.
