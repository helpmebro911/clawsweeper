## Summary

Keep #45200 open. Current main still has a real gap after subagent announce retry exhaustion: the live and resumed give-up paths mark task delivery failed, log, complete cleanup, and may emit lifecycle hooks, but they do not send a last-resort user-visible notification with the recovered result. The broader linked bug #44925 remains open, and this PR is still a plausible focused fix for Pattern 1, though it needs to be adapted to current main and covered by tests.

## Best Possible Solution

Keep the PR open, but ask for a current-main revision that adds a tested last-resort notification in both live and resumed announce give-up paths. Prefer a small helper in the subagent lifecycle/delivery boundary that skips intentionally killed runs, requires a deliverable `requesterOrigin.channel` and `to`, uses a stable per-run idempotency key, sends the warning/result through the outbound message path or otherwise documents the LLM-roundtrip tradeoff, and keeps failures best-effort with an audit log.
