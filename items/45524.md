## Summary

Keep open. Current main still does not add `agents.defaults.sandbox.mode=off` to the startup dangerous-config flag collector. The startup warning path exists, but it only reports flags returned by `collectEnabledInsecureOrDangerousFlags`, and that collector currently covers core flags, hooks/browser/fs flags, sandbox Docker dangerous booleans, and plugin-declared flags, not sandbox mode itself. The PR is plausible but not merge-ready as-is because its patch also adds auth warnings outside the stated scope, and review already identified that its trusted-proxy allowlist check would miss the common `allowUsers: undefined` allow-all case.

## Best Possible Solution

Keep the PR open for maintainer review or contributor revision. The best path is a narrow patch that adds `agents.defaults.sandbox.mode === "off"` to `collectEnabledInsecureOrDangerousFlags`, adds regression coverage for explicit off versus absent config, and either splits the gateway-auth checks into a separate PR or fixes them to match current trusted-proxy runtime semantics, including `allowUsers: undefined`.
