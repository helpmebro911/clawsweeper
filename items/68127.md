## Summary

Keep this PR open. Current main still calls `.trim().toLowerCase()` on `params.model.id` and `params.model.name` in the legacy OpenAI Codex alias canonicalizer, and the PR is a narrow candidate fix for the reported agent bootstrap TypeError. It is not superseded by #69997, which was closed unmerged and touched a different stored-override path. The main remaining ask is a focused regression test for undefined or non-string model metadata.

## Best Possible Solution

Keep the PR open as a valid bug-fix candidate. Add a focused regression test in `src/agents/pi-embedded-runner/model.test.ts` that feeds an OpenAI Codex resolved model with undefined or non-string `id`/`name` through the legacy `gpt-5.4-codex` canonicalization path, then land this guard or an equivalent maintainer patch after the targeted checks pass.
