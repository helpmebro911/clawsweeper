## Summary

Keep open. Current main does not implement Telegram or shared `ackReactionTiming`: the config types/schemas/docs have no timing field, and Telegram still creates the ack/status reaction during message-context construction rather than deferring it to `onAgentRunStart`. The PR is focused and still useful; the review discussion also shows the author incorporated the Greptile/Codex feedback for the `run-start` + status-reactions `setThinking()` gap. The companion Slack #45421 and Discord #45449 PRs remain open, so this is not superseded by a merged canonical item.

## Best Possible Solution

Keep #46258 open. The best path is to rebase it onto current main, preserve the default `received` behavior for compatibility, land the shared config/schema/help/docs additions once the cross-channel contract is accepted, and keep the Telegram regression coverage for deferred simple reactions plus deferred status reactions that transition from queued to thinking after `onAgentRunStart`.
