## Summary

Current main still has the cache-stability problem this PR targets: runtime-sensitive Messaging and Reactions text are assembled before the stable Project Context and before the internal cache boundary. The PR is useful but stale against current main's newer boundary design and test layout, so it should be refreshed rather than closed.

## Best Possible Solution

Keep this PR or its linked issue open, then refresh the implementation against current main. Preserve SYSTEM_PROMPT_CACHE_BOUNDARY, move or split channel/session-sensitive text such as inline-button runtime guidance, Reactions, Group Chat Context, and comparable per-session additions so stable Project Context remains in the reusable prefix, and add a regression test in the current system-prompt test layout asserting Project Context/cache-boundary ordering for those dynamic sections. Use the public prompt-caching docs at https://docs.openclaw.ai/reference/prompt-caching as the product contract.
