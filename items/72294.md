## Summary

Keep this PR open. Current main still only suppresses exact duplicate Feishu finals after a streaming card closes, so the distinct-late-final duplicate-card problem from #71977 is not implemented on main. The PR is a plausible implementation candidate, but review discussion identifies real behavior/race questions that should be resolved before merge rather than closing the PR as obsolete.

## Best Possible Solution

Keep this PR open as the active implementation candidate for the Feishu late-final duplicate-card bug. Before merge, maintainers should decide whether distinct late finals should be suppressed, appended, or delivered as a follow-up, and the implementation should address the async onError/onIdle closeout race with per-reply scoping or serialized/awaited cleanup plus regression coverage for late distinct finals, media passthrough, and error recovery.
