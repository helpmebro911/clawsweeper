## Summary

Current main still matches the reported BlueBubbles allowlist failure mode: the shared channel config schema still accepts numeric allowFrom entries without converting them back to +E.164 strings, and BlueBubbles handle normalization still returns all-digit handles without a leading '+'. The linked fix PR #62101 is useful and directly related, but it is open/unmerged in the provided GitHub context, so the issue should stay open until the fix lands or a maintainer chooses a canonical tracker.
