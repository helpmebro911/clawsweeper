## Summary

Do not auto-close #68654. Current main has related CLI startup mitigations that skip plugin CLI registration for builtin primaries, but it has not absorbed the PR's narrower changes: validated plugin CLI config still does a second config load, and plugin CLI registration entries still use the full plugin loader path rather than a metadata-entry registration path. The PR also contains review discussion that corrected an important regression risk, so it may still be useful after rebasing against current main.

## Best Possible Solution

Keep the PR open for maintainer review or author rework. The best path is to rebase it on current main, preserve the useful validated-config snapshot reuse if still correct, and either drop or redesign the metadata-entry registration path around the existing builtin skip policy and the plugin-root fallback behavior covered by tests.
