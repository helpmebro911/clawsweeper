## Summary

Keep this PR open. Current main has overlapping CLI startup work, including manifest activation narrowing and built-in command skip logic, but it has not implemented this PR's concrete remaining changes: validated plugin CLI config still performs a second config load, and lazy plugin CLI registration still always builds entries through the full plugin loader rather than a metadata-entry path. The PR is recent, contributor-authored, not protected, and its diff is narrow enough to merit maintainer review/rebase rather than cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a rebased, current-main version that separately validates the low-risk `snapshot.config` reuse and the metadata-entry lazy-loading optimization, while preserving full-loader fallback for plugin commands that lack activation metadata and confirming the optimization still matters after current `run-main` routing.
