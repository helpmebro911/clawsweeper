## Summary

Keep #40878 open. Current main still lacks the PR's adaptive thinking picker entry and still mutates the streaming chat scroll position synchronously inside the `streamingAssistantText` `onChange` handler. The latest PR head is recent, open, non-draft, and now includes the reviewed inner guard in the deferred scroll task, so this remains plausible iOS chat UI bug-fix work rather than an implemented, duplicate, ClawHub, or unactionable item.

## Best Possible Solution

Keep #40878 open for maintainer review. The best path is to align the PR title/body with the current two-file iOS chat UI diff, then have a maintainer review and validate the adaptive picker plus rapid streaming scroll behavior. Longer term, the shared iOS/macOS chat composer should likely consume provider/profile-advertised thinking levels instead of extending a hard-coded picker list one value at a time.
