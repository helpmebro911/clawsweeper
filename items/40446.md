## Summary

Keep open. Current main still does not implement this PR's passive after-turn execution-health monitor, `agents.defaults.executionHealth` config, or the proposed file-burst/no-effect/error-cascade signals. Existing loop detection and strict-agentic behavior are adjacent protections, not a full replacement, and the PR still needs maintainer review rather than automated closure.

## Best Possible Solution

Keep this PR open for maintainer review. If passive execution-health monitoring belongs in core, rebase the branch, split unrelated payload/parser changes where possible, preserve payload callback chaining, make any `tool_use` parsing consumer-specific or prove it safe, and refresh tests for cross-turn file bursts, no-effect loops, error cascades, bounded arg hashing, and config/docs. If maintainers want this outside core, close it explicitly as a ClawHub/plugin direction and point contributors at the current hook surfaces.
