## Summary

Keep open. Current main still does not implement the PR's fail-fast behavior for missing local probe auth: it resolves auth warnings, then continues into gateway/status/daemon probe calls with empty credentials. The open PR remains a plausible implementation candidate for the #61179 loopback log-spam path, but it still needs maintainer/security review around failure-reason redaction and output sanitization before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate. Before landing, revise it so only local loopback probes with explicitly required but unresolved auth fail fast; preserve explicit URL/RPC override behavior and unset/none/trusted-proxy auth modes; make normal user-facing failure reasons generic or redacted; sanitize every terminal/status surface that renders probe errors; then run the targeted gateway, daemon status, and shared status regression tests.
