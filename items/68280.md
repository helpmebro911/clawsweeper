## Summary

Keep this PR open. Current main still lacks the requested fail-fast behavior for missing local loopback probe auth, and the PR remains a focused implementation candidate, but it still needs maintainer review around overridden target credential policy and failure-reason output handling before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate. Review and revise it so fail-fast applies only to local loopback probes where auth is explicitly required but unresolved; preserve unset/none/trusted-proxy and explicit override behavior; avoid sending auto-resolved credentials to user-supplied targets without explicit intent; redact or sanitize failure reasons across all human and structured output paths; then land with focused regression coverage.
