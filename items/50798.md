## Summary

Keep open. Current main has useful partial coverage for the reporter's Option C: Discord inbound messages in a bound thread can resolve to the bound ACP/session route, including allowed bot relay posts. But the requested capability is not fully implemented: `sessions_send` still has no `visible` option and injects through `deliver:false` on the internal channel, while the outbound `message`/gateway send path still derives and records a Discord outbound session entry before delivery with no `proxyOnly` or bound-thread exclusion. The route-polluting stub described by the issue can therefore still be created by the proxy-send workaround.
