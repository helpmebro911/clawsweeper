## Summary

Current main still leaves the generic exec-approval forwarding fallback with the old placeholder `/approve <id>` instruction, so the PR is not obsolete. It remains useful, but should be rebased onto the current allowed-decision-aware renderer path before merge.

## Best Possible Solution

Keep this PR open and rework it against current main. The implementation should update `src/infra/exec-approval-forwarder.ts` so the generic plain-text fallback renders concrete per-decision `/approve <actual-id> <decision>` lines, preferably via `buildExecApprovalActionDescriptors` or equivalent shared helpers, while preserving allowed-decision filtering, native Telegram/shared interactive rendering, and non-markdown channel behavior. Update `src/infra/exec-approval-forwarder.test.ts` for markdown-capable fallback, non-markdown fallback, and `ask=always` without `allow-always`.
