## Summary

Current `main` does not contain this PR's actual fallback-text change. Exec approval notifications still render the generic `Reply with: /approve <id> ...` hint, so it is not safe to close as implemented. That said, `main` now also attaches shared interactive `/approve` buttons and Telegram renders them as inline buttons, so the PR is partially superseded but not fully replaced for plain/non-button channels.
