## Summary

Current main still has the generic exec-approval fallback text with `/approve <id>` placeholder, so the PR is not obsolete. The proposed change remains useful, but the submitted diff needs rework because it hard-codes all three decisions and can advertise `allow-always` when current main intentionally filters that decision out for `ask=always`.

## Best Possible Solution

Keep this PR open, but rework it against current main. The implementation should update the generic fallback in `src/infra/exec-approval-forwarder.ts` to render concrete per-decision `/approve <actual-id> <decision>` lines while preserving markdown/plain-channel behavior and the existing allowed-decision filtering. Prefer using or mirroring `buildExecApprovalActionDescriptors`, and add focused coverage in `src/infra/exec-approval-forwarder.test.ts` for markdown-capable fallback, non-markdown fallback, and `ask=always` without `allow-always`.
