## Summary

Keep open. Current main still generates the Windows Startup-folder fallback as a `.cmd` file containing `start "" /min cmd.exe /d /c ...`, and the existing startup-fallback test asserts that exact behavior. The issue is recent, concrete, non-maintainer-authored, unprotected, and targets a distinct login-item surface rather than the related update-restart issue #57682.

## Best Possible Solution

Keep the issue open and fix the Windows Startup-folder fallback in `src/daemon/schtasks.ts`: replace the generated `start /min cmd.exe` launcher with a truly hidden launcher or equivalent shim, preserve the existing `.cmd` path used for detection, write and remove any companion launcher file during install/uninstall, and update `src/daemon/schtasks.startup-fallback.test.ts` to assert the silent-launch behavior.
