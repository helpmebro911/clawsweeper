## Summary

Keep open. Current main appears to have addressed the specific `gateway.mode` source mismatch by routing doctor and config get through the same config snapshot path, but the central false-positive problem remains: Darwin LaunchAgent scanning still treats broad `openclaw`/`clawdbot` matches in plist contents as gateway-like, and both doctor and status still print cleanup hints for the canonical active gateway service whenever any extra service is detected.

## Best Possible Solution

Keep this issue open. Fix `src/daemon/inspect.ts` so Darwin LaunchAgent detection requires an explicit gateway service marker or parsed `ProgramArguments` that actually invoke the gateway, then change doctor/status cleanup hints to target only detected stale or inactive extra services and never the loaded/running active gateway. Review #39118 and #52515 as implementation candidates, then add regression coverage for non-gateway OpenClaw ecosystem plists and active-gateway cleanup suppression.
