## Summary

Keep open. Current main still has the macOS doctor false-positive and cleanup-hint behavior described by this issue. The config-source complaint appears partially obsolete because doctor preflight and `config get` both read through `readConfigFileSnapshot`, but that does not resolve the remaining LaunchAgent scanner and cleanup-hint problems. The related open PRs #39118 and #52515 are implementation candidates, not shipped fixes.

## Best Possible Solution

Keep this issue open and fix the doctor gateway-service scanner. The best path is to make macOS LaunchAgent detection require an explicit `OPENCLAW_SERVICE_KIND=gateway` marker or parsed ProgramArguments that actually invoke the gateway, stop reporting arbitrary OpenClaw ecosystem plists as competing gateways, and make cleanup hints target only detected stale/inactive extra gateway services while never suggesting removal of the active profile gateway. Add fixture-based macOS plist tests and coordinate with #39118 and #52515 before closing.
