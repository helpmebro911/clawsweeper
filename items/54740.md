## Summary

Keep open. Current main has likely fixed the `gateway.mode` source-mismatch subclaim, but the central macOS LaunchAgent false-positive and unsafe cleanup-hint behavior is still present. The related prior discussion is useful: #15849 described the same launchd cleanup hazard before being closed stale, and open PRs #39118 and #52515 are still implementation candidates rather than shipped fixes.

## Best Possible Solution

Keep this issue open. Fix `src/daemon/inspect.ts` so Darwin LaunchAgent detection requires an explicit gateway service marker or parsed `ProgramArguments` that actually invoke the gateway, then change doctor/status cleanup hints to target only stale or inactive extra services and never the loaded/running canonical gateway. Review #39118 and #52515 as implementation candidates, and add regression coverage for non-gateway OpenClaw ecosystem plists plus active-gateway cleanup suppression.
