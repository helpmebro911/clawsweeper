## Summary

Keep #71205 open. Current main still unconditionally tightens the default state/config directory to 0700 when group/other bits are present, and there is no `OPENCLAW_SKIP_STATE_DIR_HARDEN` implementation, test, changelog entry, or docs entry on main. The PR is a focused core config I/O bug fix with concrete multi-uid sidecar reproduction details, and the provided discussion shows the bot observability feedback was addressed in the PR branch rather than superseded elsewhere.

## Best Possible Solution

Keep the PR open and review it as a focused core config I/O fix. The best path is a small implementation in `src/config/io.ts` that checks a truthy `OPENCLAW_SKIP_STATE_DIR_HARDEN` before stat/chmod, logs the accepted env option, adds a Linux regression test preserving a 02770 setgid group-writable state dir, and documents/changelogs the opt-in operator responsibility if maintainers accept the escape hatch.
