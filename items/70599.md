## Summary

Keep PR #70599 open. Current main still lacks the PR's central tool-result `NO_REPLY` handling, so it is not obsolete; however the PR head still has a real unresolved review concern because its silent-tool fast path can bypass provider-error handling.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: inspect only current-attempt successful tool results, require the latest tool result text to be exact silent `NO_REPLY`, and gate the fast path so provider-error, timeout, abort, and tool-error cases still surface through the existing retry/error handling. Add regression tests for the intended empty-final success, stale prior-turn `NO_REPLY`, provider-error after `NO_REPLY`, and non-silent/latest-error tool results.
