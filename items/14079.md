## Summary

Keep #14079 open. Current main still lacks Linux desktop notification delivery for the existing `system.notify` / `openclaw nodes notify` surface. The headless node host does not advertise or handle `system.notify`, docs still split `system.notify` as macOS app behavior versus headless node-host commands, and plugins cannot currently fill the same command because `system.notify` is reserved by core. The related closed #24616 covers error wording, not Linux delivery.

## Best Possible Solution

Keep #14079 open. The best path is a focused core/node-host implementation for Linux `system.notify`: detect desktop-session availability, advertise the command only when delivery is available, use `notify-send` first with a Freedesktop D-Bus/libnotify fallback, degrade cleanly on headless hosts, add gateway/node-host/CLI coverage, and update the public nodes docs. If maintainers prefer external implementation, core first needs a documented way for plugins to implement or augment the reserved `system.notify` surface.
