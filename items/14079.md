## Summary

Current main still does not implement Linux desktop notifications for the existing `system.notify` / `openclaw nodes notify` surface. The headless node host does not advertise or dispatch `system.notify`, docs still describe notification delivery as macOS-app behavior, and there is no `notify-send`, libnotify, dunstify, or Freedesktop D-Bus notification backend in the source tree. The related #24616 was a narrower error-UX issue and does not supersede this feature request.

## Best Possible Solution

Keep #14079 open. The best path is a focused core/node-host change: detect Linux desktop-session availability, advertise `system.notify` only when delivery is available, implement `notify-send` with a D-Bus/libnotify fallback, degrade cleanly on headless hosts, cover Gateway/node-host/CLI behavior with tests, and update the public nodes/CLI docs. If maintainers prefer an external plugin, core first needs a documented extension point for the reserved `system.notify` surface.
