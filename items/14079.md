## Summary

Keep #14079 open. Current `origin/main` still lacks Linux desktop notification delivery for the existing `system.notify` / `openclaw nodes notify` surface: the headless node host does not advertise `system.notify`, its dispatcher has no notify handler, docs and CLI still describe notify as macOS-node-only, and plugins cannot replace this exact command because core reserves it. The related closed #24616 only addressed the misleading Linux error experience, not Linux notification delivery.

## Best Possible Solution

Keep the issue open. The best product fix is to add Linux `system.notify` support in the node-host notification path: advertise the command when appropriate, implement `notify-send` with libnotify/D-Bus fallback and graceful headless no-op behavior, add focused tests for command advertisement, dispatch, and error behavior, and update the nodes CLI/docs. If maintainers prefer this outside core, first add an explicit plugin API that can safely own or extend the core notification surface.
