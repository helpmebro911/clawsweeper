## Summary

Keep open. Current main partially covers the `<available_skills>` portion through existing agent skill allowlists, but the requested user-facing prompt gates for the CLI quick reference, reply-tag/output-directive guidance, and messaging component guidance are not implemented. The `systemPromptOverride` docs still describe only a full replacement, not the requested required-vs-optional cookbook. The provided related ClawSweeper report #66612 is about memory-flush sandbox writes and does not supersede this prompt-gating request.

## Best Possible Solution

Keep the issue open and narrow the implementation around the missing pieces: add default-true config/schema/docs/test coverage for the CLI quick reference, reply-tag/output-directive guidance, and messaging component guidance if maintainers agree with those gates; document the existing agent skill allowlist as the current `<available_skills>` filter path; and add a short `systemPromptOverride` cookbook that identifies structural blocks and cache-boundary requirements.
