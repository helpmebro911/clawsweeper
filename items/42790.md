## Summary

Keep open. Current `main` already skips the Tailscale hostname probe when `gateway.tailscale.mode` is `off`, so the original ENOENT noise looks addressed, but the `status --all` diagnosis path still marks off-mode as unhealthy/unknown rather than healthy. That means the PR's follow-up behavior from the review discussion is not implemented on `main`, so this is not safe to auto-close as already fixed.
