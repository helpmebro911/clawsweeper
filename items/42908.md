## Summary

Keep this PR open. Current main still does not preserve Feishu card-action `option`, `options`, or `form_value` from live callbacks through the parser/fallback path, and this PR is paired with open same-author issue #42754. The patch is a useful implementation candidate, but it is stale against current main because the production parser currently reconstructs `action` with only `value` and `tag`, while fallback serialization is now in `card-interaction.ts`.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a current-main update: preserve Feishu native action fields in `parseFeishuCardActionEventPayload`, extend the typed event and `buildFeishuCardActionTextFallback` serialization for `option`, `options`, and `form_value` while preserving existing `value.text` and `value.command` behavior, and add regression coverage from a raw `card.action.trigger` payload. Coordinate with #42754, #43202, and #43953 so one canonical Feishu card-action pass-through fix lands.
