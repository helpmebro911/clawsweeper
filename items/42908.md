## Summary

Keep this PR open. Current main still drops Feishu card-action sibling fields before fallback serialization can reach the agent, and this PR is paired with the same-author open issue #42754. The patch remains useful, but it needs a current-main rebase because the live parser now strips fields before the handler sees them.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best path is to rebase on current main, preserve Feishu-native sibling action fields in `parseFeishuCardActionEventPayload`, extend the event/fallback serialization path for `option`, `options`, `form_value`, and likely the broader `input_value`/`name` fields tracked in #43202 and #43953, while preserving `value.text` and `value.command` behavior. Add regression tests from raw `card.action.trigger` payloads, including empty arrays and empty objects.
