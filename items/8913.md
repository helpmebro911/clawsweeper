## Summary

Current `main` still lacks configurable reasoning/thinking display formatting per channel or per agent. The shared formatter still emits fixed italic Markdown, the embedded runner/subscriber and Telegram reasoning lane have no display-format parameter, config/docs expose reasoning visibility and provider transport `thinkingFormat` rather than user-facing display style, and the linked implementation PR #25336 was closed unmerged after review found incomplete threading.

## Best Possible Solution

Keep #8913 open. A complete fix should define and validate a reasoning display-format contract, preserve italic as the compatibility default, choose the config scope and override order, thread the selected value through embedded runner payloads, subscribe/message-end/streaming callbacks, and Telegram reasoning-lane dispatch, then add regression coverage for default italic plus configured code/spoiler/blockquote behavior and document the public config.
