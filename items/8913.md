## Summary

Keep #8913 open. Current main still hardcodes reasoning/thinking display as italic Markdown and does not expose a per-channel or per-agent display-format setting. The related implementation PR #25336 was closed unmerged after review found incomplete threading, and current config/docs cover reasoning visibility or provider transport `thinkingFormat`, not user-facing display style.

## Best Possible Solution

Keep #8913 open. A complete fix should define a documented reasoning display-format contract, preserve italic as the compatibility default, validate the config shape, choose the channel/agent/global override order, thread the selected value through embedded run payloads, subscribe/message-end/streaming callbacks, and the Telegram reasoning lane, then add regression tests for default italic plus configured code/spoiler/blockquote behavior.
