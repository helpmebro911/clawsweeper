## Summary

Keep open. Current main still hardcodes visible reasoning formatting as italic Markdown and has no per-channel or per-agent `thinkingFormat`/`reasoningFormat` config that is threaded through the agent runner, subscribe handlers, or Telegram reasoning lane. The closed related PR #25336 did not merge, and the latest provided maintainer/Codex comment also explicitly kept this issue open.

## Best Possible Solution

Keep the issue open until OpenClaw implements the feature or maintainers choose a different product direction. A complete fix should add a documented `ReasoningFormat`/`thinkingFormat` contract, validate it in agent and/or Telegram channel config, thread it through runner payloads, subscribe handlers, and Telegram split/dispatch paths, and add integration coverage proving code/spoiler/blockquote or other supported formats work on the visible reasoning paths.
