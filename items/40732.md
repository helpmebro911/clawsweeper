## Summary

Keep open. Current main still uses a native textarea for the raw config editor, with no CodeMirror component, status bar, path breadcrumbs, or current-main tests for those behaviors. This PR is focused, has author follow-up on review feedback, and overlaps with #40009 and #56401, but neither has shipped or been selected as the canonical replacement.

## Best Possible Solution

Keep this PR open for maintainer review or explicitly consolidate it with the canonical raw config editor effort. The likely product path is one selected implementation in ui/src/ui/views/config.ts plus a dedicated editor component, with tests for raw save/change behavior, redaction, status text, breadcrumbs, JSON5 edge cases, formatting/validation expectations, and dependency impact.

## Risks / Open Questions

- The PR adds CodeMirror packages and lockfile entries, so maintainers should review dependency footprint, licensing, integrity, and bundle impact before merge.
- The raw editor path resolver is hand-written for JSON5-like syntax; review comments already found edge cases, and escaped identifier keys should still be verified against the current PR head before merge.
- ui/AGENTS.md says non-English locale bundles are generated output; this PR edits multiple locale files, so maintainers should ensure the i18n generator metadata/workflow is satisfied rather than accepting generated drift.
- There is overlapping open work in #40009 and #56401; maintainers should pick one canonical raw-editor direction to avoid landing conflicting implementations.
