## Summary

Keep #72120 open. Current `main` still has the reported OpenAI Realtime SecretRef gap: the OpenAI realtime voice provider resolves configured `apiKey` values, but both the browser-session client-secret path and the server-side bridge path still fall back to `process.env.OPENAI_API_KEY` and pass that value directly to OpenAI. The linked PR #72126 is an open, unmerged fix path, not evidence that `main` is fixed.

## Best Possible Solution

Keep the issue open and land PR #72126 or an equivalent OpenAI plugin fix that resolves and caches supported keychain/SecretRef API-key values before both realtime client-secret creation and realtime voice bridge creation. Add focused extension tests for the env fallback so unresolved placeholders cannot be sent to OpenAI again.
