## Summary

Keep #18967 open. Current main still has only broad opt-in paths for giving subagents `sessions_send` and widening session visibility; it does not implement the requested native parent-scoped send path with direct-parent-only, same-tree, no-chaining enforcement.

## Best Possible Solution

Keep #18967 open. The best path is a narrow native `sessions_send_parent` tool, or a literal `sessionKey: "parent"` mode on `sessions_send`, resolved from stored direct-parent metadata. It should enforce direct-parent-only access within the same OpenClaw agent tree, avoid automatic forwarding further up the tree, preserve the existing broad opt-in as a separate escape hatch, and add resolver/policy/visibility/docs coverage.
