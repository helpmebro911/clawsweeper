## Summary

Keep #18967 open. Current main still does not provide a native parent-scoped `sessions_send` path for sub-agents; it only has default denial plus broad opt-in tool and session-visibility settings. The recent prior review comment also explicitly kept this open, and the only code change since then is unrelated WhatsApp docs/code.

## Best Possible Solution

Keep #18967 open. The best implementation is a narrow native parent-send path, either a dedicated `sessions_send_parent` tool or a literal `sessionKey: "parent"` mode, resolved from stored direct-parent session metadata. It should enforce direct-parent-only access within the same OpenClaw agent tree, avoid automatic forwarding further up the tree, preserve broad `sessions_send` opt-ins as separate high-trust configuration, and add resolver, policy, visibility, docs, and regression-test coverage.
