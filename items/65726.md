## Summary

Current main still does not implement the requested ACP binding notify policy or a bound-thread duplicate guard. The typed ACP config and strict schemas omit `notifyPolicy`, binding materialization has no field to carry it, ACP background task callers still omit `notifyPolicy` so pending task delivery defaults to visible `done_only`, and the newer Slack thread-routing work does not suppress outbound replies from other sessions targeting an active ACP-bound thread.

## Best Possible Solution

Keep this issue open. Add typed `notifyPolicy` support to `AgentRuntimeAcpConfig` and/or `AgentAcpBinding.acp`, update strict Zod schemas, generated help/docs, and binding materialization, then pass the resolved policy into ACP background task creation so `silent` suppresses bound-session completion notices. Separately design a channel/account/conversation/thread ownership guard, likely around session binding/outbound delivery, with tests covering Slack ACP-bound threads and ordinary manual replies.
