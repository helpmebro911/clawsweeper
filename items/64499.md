## Summary

Keep open. Current `main` still appears to have the reported mismatch: `exec host=node` always invokes `system.run.prepare`, but the macOS companion node only advertises and handles `system.run`/`system.which` plus approvals/notify. That matches the reporter's `INVALID_REQUEST ... does not support "system.run.prepare"` log, so this is still a real bug rather than something already fixed or obsolete.
