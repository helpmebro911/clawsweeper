## Summary

Keep #70905 open. Current main still does not expose a stable agent id on `ChannelOutboundContext` for channel outbound adapters. The agent id is already carried through adjacent outbound/session/action paths, but the adapter context type and the context assembly path omit it, so the requested additive SDK/core seam remains real.

## Best Possible Solution

Keep the issue open and implement a narrow additive core/SDK change: add optional `agentId` to `ChannelOutboundContext`, pass `params.session?.agentId ?? params.mirror?.agentId` through `ChannelHandlerParams` into `createChannelOutboundContextBase()`, and add focused tests proving `sendText`, `sendMedia`, `sendPayload`, and formatted outbound callbacks receive it for session/mirror-backed sends without changing human/system sends.
