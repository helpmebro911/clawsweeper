## Summary

Keep #70905 open. Current main still does not expose `agentId` on `ChannelOutboundContext`; the outbound pipeline carries agent context in `OutboundSessionContext` and several caller paths, but `deliverOutboundPayloads` only uses it internally for media/access and hooks before building an adapter context that omits it. This is a concrete additive plugin SDK/core seam request, not a ClawHub-only plugin task or a duplicate of the provided related items.

## Best Possible Solution

Keep the issue open. The best implementation is an additive `agentId?: string` on `ChannelOutboundContext`, populated in `deliverOutboundPayloads` from the outbound session or mirror agent context and covered by focused tests for text, media, payload/formatted delivery plus docs/API baseline updates.

## Risks / Open Questions

- Adding this field touches a public plugin contract, so the eventual fix should update the SDK/API baseline, docs, and focused outbound contract tests.
- Callers should define precedence carefully when both `session.agentId` and `mirror.agentId` are present, likely matching the existing media-access precedence unless maintainers choose otherwise.
