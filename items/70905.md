## Summary

Keep this issue open. Current main still does not expose a stable agent id on ChannelOutboundContext, even though upstream outbound paths already carry agentId and deliver.ts uses it for adjacent media-root behavior. This is a core plugin SDK plumbing gap, not a ClawHub-only plugin request.

## Best Possible Solution

Add optional agentId?: string to ChannelOutboundContext, thread a documented source such as params.session?.agentId ?? params.mirror?.agentId through ChannelHandlerParams and createChannelOutboundContextBase, and verify it reaches sendText, sendMedia, sendPayload, sendFormattedText, sendFormattedMedia, and renderPresentation contexts. Update channel plugin docs to distinguish stable agent attribution from display identity.
