## Summary

Keep #70905 open. Current main carries agentId through upstream outbound/session/tool-action paths, but the plugin-facing ChannelOutboundContext still does not expose it or receive it when outbound adapter contexts are assembled. This is an actionable additive core SDK gap, not a ClawHub-only plugin request, and the issue author offered to contribute once the shape is approved.

## Best Possible Solution

Keep the issue open and implement the additive core SDK change: add optional agentId?: string to ChannelOutboundContext, thread params.session?.agentId ?? params.mirror?.agentId through ChannelHandlerParams/createChannelOutboundContextBase, cover the adapter contexts in outbound delivery tests, and update the plugin SDK docs/API baseline so third-party channel authors can rely on the field.
