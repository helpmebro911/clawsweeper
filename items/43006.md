## Summary

Keep open. Current main has a pre-compaction memory flush, but it remains a single-plan, single-threshold mechanism suppressed after one flush per compaction cycle. The requested progressive staged reminders/flushes and forced-summary handoff are not implemented, and the related docs-only PR #43102 was closed unmerged rather than superseding this feature request.

## Best Possible Solution

Keep this issue open for maintainer design. A complete implementation likely belongs in the gateway/core memory-flush pipeline plus a versioned memory-plugin contract update: staged threshold configuration, stage-specific prompts, persisted per-stage completion state, deduplication semantics, and a defined forced-summary/compaction handoff, with schema, docs, and tests updated together.
