## Summary

Current main already has pre-compaction memory flush, but it is still a single-threshold, once-per-compaction-cycle mechanism. The requested progressive 80/90/95-style stages, stage-specific prompts, and forced-summary behavior remain unimplemented. The related PR #43102 was a docs-only RFC and is closed unmerged, so it does not supersede the remaining work.

## Best Possible Solution

Keep this issue open for maintainer product/design review. A complete implementation likely belongs in the Gateway/core memory-flush pipeline plus the bundled memory-core plugin contract: add staged threshold configuration or a staged flush-plan API, persist per-stage completion metadata instead of only `memoryFlushCompactionCount`, support stage-specific prompts, define the final forced-summary/compaction handoff, and update schema, docs, and tests.
