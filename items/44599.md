## Summary

Keep open. Current `main` still hard-fails when `OPENCLAW_CONFIG_DIR` or `OPENCLAW_WORKSPACE_DIR` contains whitespace, which matches issue #44599 rather than proving it fixed or obsolete.

- **The Docker setup path is always validated:** `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` are populated from the environment/defaults and immediately passed into `validate_mount_path_value`, so a whitespace-bearing path still fails on current `main`.
  - file: [scripts/docker/setup.sh:235](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/scripts/docker/setup.sh#L235)
