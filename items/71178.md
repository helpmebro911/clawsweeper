## Summary

Keep #71178 open. Current main has restart-drain/recovery mitigation, but the specific `update.run` continuation gap called out by the maintainer is still present: `update.run` writes a restart sentinel without `continuation`, the protocol schema has no `continuationMessage`, the agent gateway tool does not forward it for `update.run`, and tests still assert continuation is absent. The provided context also shows PR #71573 as the open/unmerged targeted fix.

## Best Possible Solution

Keep #71178 open. Land or otherwise implement the #71573-style fix: add `continuationMessage` to `UpdateRunParamsSchema` and restart request parsing, forward it from the agent gateway tool's `update.run` branch, build a `buildRestartSuccessContinuation(...)` payload in `src/gateway/server-methods/update.ts` only for successful updates with a session key, and update the current tests that assert `continuation` is undefined. After that, decide whether external CLI `openclaw update`, graceful-drain/fallback notices, and system-prompt reset bridge behavior stay here or move to narrower follow-ups.
