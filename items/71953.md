## Summary

PR #71953 should stay open. Current `main` still has the hook-manifest gap the PR describes: bundled hook handlers are emitted under `dist/bundled/<hook>/handler.js`, hook discovery requires `HOOK.md`, full builds run `copy-hook-metadata`, but the `gatewayWatch` profile only runs `tsdown`, `runtime-postbuild`, and `build-stamp`, while `runRuntimePostBuild()` does not copy hook manifests. The PR body and Greptile comment both describe a concrete reproduction and a narrow tested fix, and the related items in the provided context cover adjacent bundled-runtime issues rather than superseding this specific dev-gateway hook metadata problem.

## Best Possible Solution

Keep PR #71953 open for maintainer review. The best fix is a narrow runtime-postbuild change like the PR proposes: copy bundled hook `HOOK.md` manifests during `runRuntimePostBuild()` and add focused tests for the happy path plus missing-source/missing-manifest cases, so both `gateway:watch` and runtime restaging paths ship the manifests hook discovery requires.
