## Summary

Keep PR #71953 open. Current main still has the gap the PR targets: `gateway:watch` routes through `runRuntimePostBuild`, but that function does not copy bundled hook `HOOK.md` metadata, while the standalone `copy-hook-metadata` step is outside the gateway-watch build profile. The PR is a focused, plausible bug fix with tests in its diff and is not superseded by the related bundled-plugin/runtime PRs in the provided context.

## Best Possible Solution

Keep the PR open and have a maintainer either merge a vetted version of this patch or implement the same behavior on main: fold bundled hook `HOOK.md` copying into `runRuntimePostBuild`, keep the `gatewayWatch` path covered, and retain focused tests for copied metadata plus missing-manifest/source-dir cases.
