## Summary

Keep open. Current main has absorbed the internal cron `delivery.threadId` delivery-plan and target-forwarding pieces, but the PR is not obsolete: the public gateway cron schema still rejects/omits `delivery.threadId`, and failure-destination deduplication still ignores the primary delivery thread, matching the PR review feedback that the author later addressed on the PR branch.

## Best Possible Solution

Keep this PR open for maintainer review or author rebase. The best path is to preserve current main's internal `delivery.threadId` plumbing, then port only the remaining pieces: add `threadId` to the cron gateway delivery/patch schema with round-trip tests, update failure-destination deduplication so explicit topic-routed primary delivery does not suppress a same-chat failure route, and sync any public protocol/client model surfaces required by the gateway schema change.
