## Summary

Current main has absorbed the internal cron `delivery.threadId` plumbing, but this PR still contains useful unlanded work for the public gateway cron schema and failure-destination deduplication. It should not be closed as obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or a focused rebase. Preserve the internal `delivery.threadId` plumbing already on main, then port the remaining gateway schema round-trip and failure-destination deduplication fixes with targeted tests. Because the gateway schema is a public protocol surface, sync any generated/client artifacts required by that boundary before landing.
