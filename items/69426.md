## Summary

Keep PR #69426 open. Current main still gates `params.context1m` on provider id `anthropic` or `claude-cli`, so the reported `github-copilot` and other proxied-Anthropic path remains unresolved. The PR is a focused bug fix with relevant regression coverage proposed, and no protected-label or maintainer-authorship rule requires closure.

## Best Possible Solution

Keep the PR open and either merge/adapt its focused fix or replace it with a capability-based helper that treats Claude Opus/Sonnet model ids consistently across Anthropic-family proxy providers. Add regression coverage for `github-copilot` and, if feasible, one Bedrock/LiteLLM-style proxied path, while preserving the prefix gate so non-1M Claude models remain capped.

## Evidence
