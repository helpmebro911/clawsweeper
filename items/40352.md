## Summary

Current main still preserves dirty Control UI config form state on the ordinary reload path, so the reported stale Agents model dropdown behavior is not proven fixed. The issue also has an open unmerged closing PR, and a maintainer-labeled repair PR is still open, so it should remain open until maintainers land or close the implementation candidate.

## Best Possible Solution

Keep this issue open and let maintainers review, land, or close the active implementation candidates. The best fix is to make explicit Reload Config discard stale local config state while passive refresh preserves unsaved edits, then land the selected-option binding fix from PR #52948 or add browser regression coverage proving it is unnecessary. Regression coverage should exercise the Agents primary model dropdown after reload and initial render.
