## Summary

Keep this issue open. Current main at aac83e00 still routes Control UI Reload Config through the ordinary `loadConfig` path, which applies snapshots without clearing dirty form state, and the Agents model select still renders plain options without option-level selected binding. The GitHub context also shows open implementation PRs, including #52948 with closing syntax and maintainer-labeled #72624, so the issue should remain open until a canonical fix lands or those PRs are closed.

## Best Possible Solution

Keep this issue open and review/land a canonical Control UI fix. The best path is to split passive config refresh from explicit Reload Config, make explicit reload discard stale dirty config state across Agents/Config/Channels call sites, add selected-option bindings where Lit select rendering can show stale values, and include regression coverage for the Agents primary model dropdown after reload.
