## Summary

Keep open. Current main still contains the stale-reload path the issue describes: the Agents Overview reload button calls `loadConfig()` directly, `loadConfig()` does not clear `configFormDirty`, `applyConfigSnapshot()` refuses to refresh `configForm` while dirty, and the model dropdown renders from `configForm`. On code inspection, an edited/stale model selection can therefore survive an explicit reload.

## Evidence

- **Reload handler does not discard dirty form state:** Agents Overview wires `onConfigReload` straight to `loadConfig(state)` with no reset step.
  - file: [ui/src/ui/app-render.ts:1943](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/ui/src/ui/app-render.ts#L1943)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '1920,1975p'`
  - sha: [799a42bd13de](https://github.com/openclaw/openclaw/commit/799a42bd13de586a1a7163cf0bd51673c31de253)
- **`loadConfig()` never resets `configFormDirty`:** `loadConfig()` fetches `config.get` and immediately calls `applyConfigSnapshot(state, res)`; there is no `configFormDirty = false` before applying the fresh snapshot.
  - file: [ui/src/ui/controllers/config.ts:39](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/ui/src/ui/controllers/config.ts#L39)
  - command: `nl -ba ui/src/ui/controllers/config.ts | sed -n '39,52p'`
  - sha: [799a42bd13de](https://github.com/openclaw/openclaw/commit/799a42bd13de586a1a7163cf0bd51673c31de253)
- **Fresh snapshots are ignored while dirty:** `applyConfigSnapshot()` only replaces `configForm` and `configFormOriginal` inside `if (!state.configFormDirty)`, so a reload cannot refresh the form-backed values after the form becomes dirty.
  - file: [ui/src/ui/controllers/config.ts:101](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/ui/src/ui/controllers/config.ts#L101)

  - file: [ui/src/ui/views/agents-panels-overview.ts:52](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/ui/src/ui/views/agents-panels-overview.ts#L52)
