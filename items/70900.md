## Summary

PR #70900 should remain open. Current main still has the exact `surface_error` throw guard that the PR targets, and the surrounding runner code can still produce `failoverFailure=false` with a classified `failoverReason`, so this is plausible useful bug-fix work rather than obsolete cleanup.

## Best Possible Solution

Keep PR #70900 open for maintainer review and likely land a narrow version of it: add `params.failoverFailure` to the non-abort, non-timeout `surface_error` throw guard in `src/agents/pi-embedded-runner/run/assistant-failover.ts`, keep #70124's real provider-error surfacing intact, and add the focused regression test for `failoverFailure=false` with a classified `failoverReason`.
