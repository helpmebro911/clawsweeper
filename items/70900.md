## Summary

Keep PR #70900 open. Current main still has the ungated assistant `surface_error` throw path, while the PR targets a real gap left after merged PR #70848: a classified stale `errorMessage` can drive failover even when the assistant did not stop with `stopReason === "error"`.

## Best Possible Solution

Keep the PR open and land this fix or an equivalent change: gate the `surface_error` throw on `params.failoverFailure` while preserving #70848's real provider-error surfacing, and add the focused regression test for `failoverFailure=false` with a classified `failoverReason`.
