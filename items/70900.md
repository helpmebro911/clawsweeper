## Summary

Keep this PR open. Current main still has the unguarded assistant `surface_error` throw path, and the PR is a focused source/test fix for a plausible success-to-error regression where `failoverReason` is classified from stale text while `failoverFailure` is false.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent minimal fix in `src/agents/pi-embedded-runner/run/assistant-failover.ts` that requires `params.failoverFailure` before throwing from the assistant `surface_error` branch. Preserve the #70848/#70124 behavior for genuine provider failures and include the proposed regression test for `failoverFailure=false` with a classified `failoverReason`.
