## Summary

Keep open. Current main still has the reported behavior: the lean `status --json` path skips memory inspection by returning `memory: null`, but the scan result and JSON payload have no collection-state marker, and the shared memory formatter still renders every enabled/null memory snapshot as `unavailable`. The PR remains a plausible focused implementation candidate because its latest supplied diff adds explicit checked/skipped state and preserves `unavailable` for attempted probe failures.

## Best Possible Solution

Keep this PR open for maintainer review, rebase, and CI. The best implementation path is to land this focused checked/skipped memory-collection plumbing, or reimplement the same idea on main: lean `status --json` without `--all` should report skipped collection as `not checked`, while attempted probe failures and unavailable memory runtimes should continue to render as `unavailable`.
