## Summary

Keep open. Current main still cannot distinguish a deliberately skipped fast-path memory probe from an attempted-but-unavailable memory probe, and the renderer still labels enabled null memory as unavailable. The PR remains a focused implementation candidate and its latest patch appears to address the prior review concern by preserving unavailable for checked probe failures.

## Best Possible Solution

Keep this PR open for maintainer review and CI. The best fix is to land this focused checked/skipped memory-collection state, or reimplement the same behavior on main: lean `status --json` without `--all` should report skipped memory collection as `not checked`, while attempted probe failures and unavailable memory runtimes should continue to render as `unavailable`.
