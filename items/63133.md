## Summary

Current main still has the reported behavior: lean `openclaw status --json` skips memory inspection and returns `memory: null`, but neither the scan result nor the JSON/human renderers carry a checked-vs-skipped state. This PR remains a focused implementation candidate, and the latest provided patch addresses the earlier review concern by preserving `unavailable` for attempted probe failures while using `not checked` only for the explicit skipped fast path.

## Best Possible Solution

Keep this PR open for maintainer review, rebase if needed, and CI. The best fix is to land this focused checked/skipped memory-collection state, or reimplement the same behavior on main: lean `status --json` without `--all` should report skipped memory collection as `not checked`, while attempted probe failures and unavailable memory runtimes should continue to render as `unavailable`.
