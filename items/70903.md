## Summary

Keep open. Current main has partial recovery work, but it still does not fully solve the reported lockout. Billing-disabled profiles are probed only for no-fallback setups or near expiry; a same-provider fallback chain sharing one auth profile can still skip every Anthropic candidate for the long billing window. The added comment scenario also remains plausible because `openclaw configure --section model` goes through the auth-choice path, while stale cooldown clearing is currently wired only into `openclaw models auth login`.

## Best Possible Solution

Keep this issue open and finish the recovery semantics in core: treat a fallback chain that only has the same provider/profile like a single-provider setup for periodic billing probes, and clear the affected provider's usage/cooldown state from the shared provider auth-choice/configure flow after successful re-auth. Add focused regression tests for same-provider billing fallback chains and `openclaw configure --section model` re-auth clearing stale `cooldownUntil`/`disabledUntil` state.
