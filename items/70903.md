## Summary

Keep open. Current main has partial mitigations: billing disables are bounded and no longer extend on mid-window retries, explicit model auth login clears stale provider lockouts, and a no-fallback billing-disabled primary can be probed. The reported recovery gap remains for shared-profile same-provider fallback chains: billing disables are still persisted for 5-24 hours and far-from-expiry billing-disabled providers with configured fallbacks are skipped rather than re-probed after the user fixes billing.

## Best Possible Solution

Keep the issue open and narrow the fix around persisted billing cooldown recovery. The likely product/code path is to re-probe billing-disabled primary providers on a short throttle for same-provider-only/shared-profile fallback chains, or add a provider health-check/reset path that clears disabledUntil after successful provider validation. Add regression coverage for primary and fallback models sharing one auth profile, plus coverage that provider re-auth/configure paths clear stale cooldown state for token providers such as GitHub Copilot.
