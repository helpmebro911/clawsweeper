## Summary

Keep open. Current main has partial recovery work, including a single-candidate billing probe and direct `models auth login` cleanup, but it does not fully solve the reported recovery bug: same-provider fallback chains still count as fallback candidates and suppress billing probes until near expiry, and `openclaw configure --section model` still reauthenticates through the plugin auth-choice path without clearing stale usage state.

## Best Possible Solution

Keep this issue open and finish the shared provider auth recovery work. Treat same-provider-only fallback chains that share disabled profiles as having no effective fallback for billing recovery probes, and make successful provider re-auth through `openclaw configure --section model` clear stale cooldown/disabled usage state for the affected provider/profile. Add regression coverage for Anthropic same-profile Opus fallback billing recovery and GitHub Copilot configure re-auth clearing a `format` cooldown.
