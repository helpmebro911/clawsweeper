## Summary

Current main contains partial cooldown recovery work, but the issue's central same-provider fallback billing lockout and the later `openclaw configure --section model` re-auth stale-cooldown path are still not fully handled.

## Best Possible Solution

Keep this issue open and implement the remaining recovery gaps in the shared provider auth/fallback path: probe same-provider-only fallback chains that share disabled billing profiles on a throttled schedule, and make successful provider re-auth through `openclaw configure --section model` clear stale usage state for the affected provider just like `models auth login` does. Add regression coverage for Anthropic same-profile primary plus same-provider fallback billing recovery and GitHub Copilot configure re-auth clearing a `format` cooldown.

## Risks / Open Questions

- Re-auth cleanup should clear only the affected provider/profile usage state and preserve unrelated auth profile history, ordering, and last-good metadata.
