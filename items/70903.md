## Summary

Keep open. Current main has partially improved this area, but the exact reported Opus-only/same-provider fallback shape can still stay blocked by a persisted billing `disabledUntil` until the cooldown is near expiry. The no-fallback billing probe does not cover a primary plus same-provider fallback that shares the same disabled auth profile.

## Best Possible Solution

Keep the issue open and adjust billing recovery around `src/agents/model-fallback.ts`: treat a fallback chain with no actually usable alternate provider/profile as recoverable on the same throttled probe path as the no-fallback case, then clear persisted `disabledUntil` through the existing successful-use path. Add regression coverage for an Anthropic primary plus Anthropic fallback sharing one disabled auth profile, far from cooldown expiry, recovering after a successful probe.
