## Summary

Keep #9465 open. Current main still does not implement cron job hooks or the requested per-job shell pre-hook gate. Main has a newer cron-aware `before_agent_reply` plugin seam that can short-circuit some cron agent runs before the model/subprocess starts, but it is not a CronJob `hooks.pre`/`post`/`onError` contract, not exposed through the cron API/CLI, and does not satisfy the issue’s job-level hook design. The linked PR #45129 remains open and unmerged, so it is useful related work rather than shipped behavior.

## Best Possible Solution

Keep the issue open until maintainers choose the product scope and either land a per-job cron hook contract through the cron type, gateway schema, timer execution path, CLI/tool/docs, and tests, or explicitly supersede this broad request with a narrower canonical cron pre-hook/lifecycle-hooks tracker such as the shell gate work in #45129.
