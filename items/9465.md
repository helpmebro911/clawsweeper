## Summary

Keep #9465 open. Current main still lacks first-class persisted per-job cron hooks: the cron types, Gateway schemas, scheduler/manual execution paths, CLI, and Control UI do not expose `hooks.pre`, `hooks.post`, or `hooks.onError`. Existing plugin hooks can short-circuit some cron-triggered agent runs with `ctx.jobId`, but that is a global plugin hook workaround, not the job-defined scheduler lifecycle requested here. Related PR #45129 remains open/unmerged in the provided GitHub context, PR #9466 was closed unmerged/docs-only, and the latest maintainer comment on #9465 explicitly kept this issue open.

## Best Possible Solution

Keep #9465 open until maintainers either land an additive first-class cron hook lifecycle across persisted cron types, Gateway schemas/RPC, scheduler due/startup/manual paths, CLI, Control UI, docs, and tests, or explicitly supersede it with a canonical narrower tracker such as #45129 for shell pre-hook gating. Existing plugin hooks can remain documented as a workaround for some deployments, but they should not be treated as closing this per-job cron hooks request.
