## Summary

Keep #9465 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still does not provide the requested persisted per-job cron hook lifecycle (`hooks.pre`, `hooks.post`, `hooks.onError`) across cron types, Gateway schemas, scheduler/manual execution, CLI, or Control UI. The existing plugin `before_agent_reply` seam can short-circuit some cron-driven agent turns with `ctx.jobId`, but that is a global plugin workaround rather than a job-defined scheduler contract. The provided issue discussion adds concrete demand for deterministic pre-LLM gating, related PR #45129 remains open/unmerged, PR #9466 was closed unmerged as docs/proposal-only, and the latest maintainer comment explicitly kept #9465 open.

## Best Possible Solution

Keep #9465 open until maintainers either land an additive first-class cron hook lifecycle across persisted cron types, Gateway protocol/schema/RPC, scheduler due/startup/manual paths, CLI, Control UI, docs, and tests, or explicitly supersede it with a canonical narrower tracker such as #45129 for shell pre-hook gating. Existing plugin hooks can remain documented as a workaround for some deployments, but they should not close this per-job cron hooks request.
