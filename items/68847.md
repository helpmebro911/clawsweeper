## Summary

Keep open. Current main still matches the reported leak path: `maybeWakeNodeWithApns` stores a `nodeWakeById` entry before the APNs registration lookup, returns `path: "no-registration"` without deleting it, and the only delete path remains the node WebSocket close cleanup. The linked fix PR #68848 is still open/unmerged in the provided GitHub context, so the issue remains an actionable gateway bug rather than something already implemented on main.

## Best Possible Solution

Keep the issue open and review or refresh PR #68848, or apply the same narrow fix on main: remove the speculative `nodeWakeById` entry on the `no-registration` branch while preserving concurrent `state.inFlight` sharing. Add a focused regression test for repeated unregistered node wake attempts, preferably with a narrow read-only testing seam or another existing gateway test pattern that proves the map does not retain entries after the no-registration return.
