## Summary

Keep this open. Current main still has the reported enqueue-followup behavior: it enqueues and handles stale-active drain, then unconditionally calls `typing.cleanup()` even when the active run is still live.

## Best Possible Solution

Update `src/auto-reply/reply/agent-runner.ts` to compute post-enqueue liveness once. If the active snapshot is stale, finalize and drain immediately. If the followup is truly queued behind a live run, keep or refresh the typing signal, for example through `typingSignals.signalToolStart()`, and do not call `typing.cleanup()` until the queued turn actually runs or is otherwise resolved. Add focused tests for stale drain and still-active queued liveness.
