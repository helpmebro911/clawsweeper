## Summary

Keep this issue open. Current main still enqueues a real followup behind an active run and then unconditionally cleans up the queued followup typing controller, so the reported Discord liveness bug remains present.

## Best Possible Solution

Keep this issue open. Fix `src/auto-reply/reply/agent-runner.ts` by computing whether the followup is still queued behind an active run after enqueue: drain immediately for stale active snapshots, but keep or refresh typing for true queued-behind-active cases instead of calling `typing.cleanup()`. Add focused tests for stale drain and still-active queued liveness, including an assertion that cleanup is not called immediately when `isRunActive()` remains true.
