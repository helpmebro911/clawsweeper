## Summary

Keep #66385 open. Current main still rejects skill roots and SKILL.md files whose realpaths escape the configured skills root, and the strict skills config schema/types/docs do not expose `skills.allowExternalSkillsIn` or an equivalent allowlist. The PR is a focused core skills-loader/config change, not ClawHub-only work, because the behavior is enforced inside OpenClaw's core skill discovery path. The provided PR context also shows the initial bot-review concerns were addressed by later PR commits rather than making the PR obsolete.

## Best Possible Solution

Keep the PR open for maintainer review as a core skills-loader/config feature. The best path is to review the current PR head against the security comments already addressed, ensure the config schema/help/generated baseline and skills containment tests are green, and merge only if maintainers want this explicit trusted-external-skills escape hatch.
