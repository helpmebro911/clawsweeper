Keep this PR open. Current main still does not implement `skills.allowExternalSkillsIn`, and the PR remains a plausible focused implementation candidate for trusted external skill symlink roots. It should not be auto-closed as implemented, duplicate, or ClawHub-scoped because this changes the core skills/config trust boundary and needs maintainer product/security review.

## Best Possible Solution

Keep this PR open for maintainer review. If OpenClaw wants trusted external symlinked skill roots, land a focused core skills/config change with safe realpath containment, generated config/docs artifacts, and regression tests for default rejection, allowed external containment, prefix-boundary bypasses, canonical prefix resolution, empty entries, and target-workspace sync behavior.
