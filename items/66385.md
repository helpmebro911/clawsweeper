## Summary

Keep this PR open. Current main still does not implement `skills.allowExternalSkillsIn`: skill loading still rejects any skill directory or `SKILL.md` whose realpath escapes its configured root, the strict skills config type/schema omit the requested key, docs still describe strict containment, and tests lock in rejection of escaped symlinked skills. The PR is a focused contributor implementation candidate, not obsolete; its security-sensitive trust expansion still needs maintainer review before merge or explicit product/security rejection.

## Best Possible Solution

Keep this PR open for maintainer review. If OpenClaw wants trusted external symlinked skill roots, land a focused core skills/config change with safe realpath containment, generated config/docs artifacts, and tests covering default rejection, allowed external containment, prefix-boundary bypasses, canonical prefix resolution, empty entries, and target-workspace sync behavior. If maintainers do not want this trust expansion, close it only after explicit product/security judgment.
