## Summary

Current main still has the reported Synology Chat webhook behavior: HEAD is rejected with 405, successful POST acknowledgements return 204 with no body, and tests still assert that old behavior. This issue also has open implementation candidates, including same-author PR #53441, so it should remain open until a fix is reviewed and landed or intentionally superseded.

## Best Possible Solution

Keep this issue open. Review and either land the focused same-author PR #53441 or intentionally supersede it with #53522. The code change belongs in extensions/synology-chat/src/webhook-handler.ts: accept HEAD probes with 200 and change successful POST ACKs from 204/no body to 200 with {"success":true}, with updated webhook and integration tests covering both behaviors.
