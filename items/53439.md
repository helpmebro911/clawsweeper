## Summary

Current main still has the reported Synology Chat webhook behavior: HEAD is rejected with 405 and successful POST ACKs return 204/no body. The issue is paired with open implementation candidates, including the reporter's focused PR #53441, so it should remain open until a fix lands or is explicitly superseded.

## Best Possible Solution

Keep this issue open. Review and either land the focused same-author PR #53441 or explicitly supersede it with #53522. The code change belongs in extensions/synology-chat/src/webhook-handler.ts: accept HEAD probes with 200 and change successful POST ACKs from 204/no body to 200 with {"success":true}, with updated webhook and integration tests covering both behaviors.
