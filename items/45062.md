## Summary

Keep #45062 open. Current main has since added a real system-prompt cache boundary and Anthropic payload splitting, so the PR is stale and cannot land as-is. But the PR’s broader optimization is not fully implemented on main: several change sources named in the PR, including skills, model aliases, workspace notes, docs/config text, tool listings, and MEMORY.md, still sit above the cache boundary and can still invalidate the stable prefix.

## Best Possible Solution

Keep the PR open as useful performance work, but require a rebase or replacement patch that targets the current `SYSTEM_PROMPT_CACHE_BOUNDARY` architecture. The best path is a smaller, current-main PR that moves demonstrably volatile prompt inputs below the boundary where safe, preserves current transport shaping, adds focused stability tests for the named change events, and resolves the existing review concerns.
