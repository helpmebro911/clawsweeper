## Summary

Keep this PR open. Current main still cannot clear cron payload model, fallbacks, or thinking overrides with null, and this PR is a focused implementation candidate. It should not be closed as obsolete, but it still needs the patch-only null semantics review concern resolved before landing.

## Best Possible Solution

Keep this PR open and request a small revision: make null-clear a patch-only contract for CronPayloadPatch types/schema, preserve null through normalizeCronJobPatch without changing normalizeCronJobCreate behavior, delete model/fallbacks/thinking in mergeCronPayload when patch values are null, coerce null to undefined only when building a replacement payload, and add regression coverage for cron.update null-clear plus cron.add optional-null compatibility.
