## Summary

Keep PR #38731 open. Current main has cron run-session retention/reaping, but it still does not implement the PR’s specific high-frequency `--every <30m` advisory in CLI or gateway add/update paths. The PR remains plausible focused work, but it needs rework before merge because the proposed patch is incomplete and review identified a logger argument-order bug.

## Best Possible Solution

Keep the PR open or replace it with a revised implementation. The best path is to centralize a small high-frequency every-schedule helper and apply it consistently in CLI `cron add`, CLI `cron edit --every`, gateway `cron.add`, and gateway `cron.update`. Keep sub-30-minute schedules valid, emit/log the advisory non-fatally through the correct CLI warning/log channel and `SubsystemLogger.warn(message, meta)` order, and add focused regression tests for add and update paths.
