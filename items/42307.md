## Summary

Current `main` does not contain the PR's web Adaptive Card renderer, and the repo's documented direction for rich message UI is the shared `MessagePresentation` contract rather than plugin-specific marker parsing. That is not enough basis for any auto-close reason, so this should stay open for maintainer/product review.
