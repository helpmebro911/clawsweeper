## Summary

Keep open. Current main still implements `/v1/chat/completions` as a normal agent run, creates or accepts a session key for every valid request, and the agent path persists session metadata/skills snapshots. The chat completions config/schema also has no passthrough or no-persistence mode. Related #48159 covers a broader ephemeral-session idea, but it does not supersede this narrower LLM relay/passthrough request.
