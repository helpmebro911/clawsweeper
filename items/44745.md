## Summary

PR #44745 is out of scope for OpenClaw core on current `main`. The proposed smart-router/escalation-gate stack is a new orchestration layer plus custom memory/trace system, while current architecture explicitly pushes optional capability into plugins/context engines and says not to merge heavy orchestration layers that duplicate existing agent/tool infrastructure.

## Close Comment

Closing after a Codex review. The prototype is understandable, but this is not a good fit for OpenClaw core as it exists on `main`. Current project direction is plugin-first: `VISION.md:55-70` says optional capability should usually ship as plugins and new skills should go to ClawHub first, and `VISION.md:101-107` explicitly calls out heavy orchestration layers that duplicate existing agent/tool infrastructure as out of scope. Current main already exposes `plugins.slots.memory` and `plugins.slots.contextEngine` for this class of extension (`docs/tools/plugin.md:109-149`, `src/config/types.plugins.ts:24-29`, `src/context-engine/registry.ts:462-475`). This PR instead adds a separate top-level `lib/*` router/memory/trace stack, so the right home is an external plugin or ClawHub package rather than OpenClaw core.
