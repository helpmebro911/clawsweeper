## Summary

Keep open. The issue is recent, author is not protected, and it is a concrete feature request with production logs and follow-up discussion. Current documented behavior appears to cover global auth-profile rotation/retry knobs (`auth.cooldowns.overloadedProfileRotations`, `overloadedBackoffMs`, `rateLimitedProfileRotations`), but not the requested per-fallback-candidate or provider/model retry granularity. The linked PR #55118 was discussed positively, but the reporter and commenter explicitly identified this remaining config-scope gap.

## Evidence

- **Issue discussion narrows remaining request:** After PR #55118 was linked, the reporter acknowledged its reason-aware retry work but clarified the remaining need is per-fallback-candidate retry granularity: proxy candidate retry 3x, direct candidate retry 1x. A follow-up comment agreed provider/model-level retry/cooldown config would be the natural first layer.
  - command: `Provided GitHub context comments 4186126064, 4186866844, 4186894531`
- **Public docs show only global auth cooldown retry knobs:** The configuration reference documents `auth.cooldowns.overloadedProfileRotations`, `overloadedBackoffMs`, and `rateLimitedProfileRotations`; it does not show a per-candidate, per-fallback-entry, or provider/model override for retry-before-fallback behavior.
  - file: [docs/gateway/configuration-reference.md](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/gateway/configuration-reference.md)
  - command: `Web search/open of current public docs because local command sandbox failed`
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Model failover docs describe same-provider auth-profile retry before model fallback:** The model failover docs say OpenClaw usually allows one same-provider auth-profile retry before switching to the next configured model fallback, tunable by the global auth cooldown fields. That is related but not the requested per-candidate/provider-model fallback-chain budget.
