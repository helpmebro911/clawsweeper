## Summary

Keep PR #71862 open. Current main at 5c4c33c7ded3afe04ed9a710ac13bb9a486ea008 still exposes `before_model_resolve` as a provider/model override hook only, and the embedded runner still chooses auth profiles from request/session state rather than hook output. The provided related context shows #68621 is an older overlapping open PR, not a shipped or canonical superseding item.

## Best Possible Solution

Keep #71862 open as the active candidate implementation. The best path is to add `authProfileOverride` to `before_model_resolve` and legacy `before_agent_start` result types, merge it with the same first-defined priority rules as provider/model overrides, return it from runner setup, consume it as an automatic auth-profile preference only when no explicit user pin is present, preserve legacy compaction-marker semantics, update SDK/docs/tests, and then close #68621 as superseded if #71862 lands.
