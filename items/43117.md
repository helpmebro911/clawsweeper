## Summary

Current main still only partially satisfies this feature request. Gemini Embedding 2, outputDimensionality, and opt-in image/audio multimodal indexing from memorySearch.extraPaths are present, matching the helpful prior discussion. The central requested conversation-history attachment path remains missing: session transcript indexing still extracts text only and does not embed persisted MediaPath/MediaPaths attachment files from chat or channel history.

## Best Possible Solution

Keep this issue open and narrow it to the missing product behavior: extend the memory-core/session transcript indexing path to discover eligible persisted conversation attachments from MediaPath/MediaPaths, resolve them through existing media safety boundaries, and index them as structured Gemini embedding inputs. Keep the shipped outputDimensionality and extraPaths support, add regression coverage for chat or Telegram attachment recall, and update docs/reference/memory-config.md to distinguish extraPaths media indexing from conversation-history media indexing.
