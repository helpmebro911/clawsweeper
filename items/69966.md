## Summary

Keep open. Current `main` still treats bootstrap profile settings as global `agents.defaults.*` only; per-agent config supports other overrides such as `skillsLimits` and `contextLimits`, but not `contextInjection`, `bootstrapMaxChars`, or `bootstrapTotalMaxChars`, so the requested feature is not implemented on `main` or documented as supported.

## Evidence

- **Per-agent agent config has no bootstrap fields:** `AgentConfig` includes per-agent overrides like `skillsLimits` and `contextLimits`, but there are no `contextInjection`, `bootstrapMaxChars`, or `bootstrapTotalMaxChars` properties under `agents.list[]`.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/a4fc6c2409f9c28e55e04e22c2e243c6900d66a3/src/config/types.agents.ts#L70)
  - command: `nl -ba src/config/types.agents.ts | sed -n '60,130p'`
  - sha: [a4fc6c2409f9](https://github.com/openclaw/openclaw/commit/a4fc6c2409f9c28e55e04e22c2e243c6900d66a3)
- **Schema for `agents.list[]` rejects/omits bootstrap override keys:** `AgentEntrySchema` enumerates allowed per-agent keys and includes `skillsLimits` and `contextLimits`, but not bootstrap/profile settings, confirming the config surface does not currently support the requested per-agent bootstrap overrides.
  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/a4fc6c2409f9c28e55e04e22c2e243c6900d66a3/src/config/zod-schema.agent-runtime.ts#L812)

- **Runtime resolution reads only global defaults:** Bootstrap/context injection resolution functions read from `config.agents.defaults` only: `resolveContextInjectionMode()` returns `config?.agents?.defaults?.contextInjection`, and bootstrap size helpers read only `cfg?.agents?.defaults?.bootstrapMaxChars` / `bootstrapTotalMaxChars`. There is no agent-specific lookup path.

That matches the current code and does not cover the requested feature.
