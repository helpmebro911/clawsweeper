## Summary

Current main does not implement per-agent bootstrap profile support. The requested knobs remain global under `agents.defaults`; `agents.list[]` does not expose `contextInjection`, `bootstrapMaxChars`, or `bootstrapTotalMaxChars`, the runtime resolvers read only defaults, and the Control UI profile selector stages presets under `agents.defaults`. Related reports in the provided context are adjacent per-agent features, but none supersedes this specific bootstrap-profile request.

## Best Possible Solution

Keep the issue open. Add first-class per-agent bootstrap profile overrides by extending `AgentConfig`/`AgentEntrySchema` with `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars`, merging them over `agents.defaults` in agent-scope resolution, passing `agentId` through bootstrap/runtime resolver paths, and updating docs/schema help/tests. Control UI can then either expose per-agent profile editing or continue as a global defaults selector while documenting direct config overrides.

## Risks / Open Questions

- Implementing this touches the public config contract, strict Zod schema, runtime bootstrap injection, docs, schema help/labels, and likely Control UI behavior; `agents.defaults` inheritance must remain backward-compatible.
- Named reusable profiles would be a larger product surface than direct per-agent overrides; direct `agents.list[]` overrides appear to be the smallest path that solves the reported multi-agent problem.
