## Summary

Keep open. Current main does respect `contextPruning.mode: "off"` for pruning and now documents that pruning and compaction are separate, but the remaining user problem is not fully resolved: compaction still has no config-level `off`/hard-limit-only mode, compaction defaults to `safeguard` when unset, and `agents.defaults.contextTokens` is only a cap rather than a way to raise a model to Anthropic 1M context. The issue discussion’s later diagnosis that pruning and compaction are separate is supported, but the requested hard-disable/control behavior remains missing.
