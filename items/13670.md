## Summary

Keep #13670 open. Current main still does not implement an always-visible Control UI topbar display for the active model plus token/context usage. The relevant data exists in Debug, Usage, Overview, and chat-scoped surfaces, but not in the topbar/header requested by the issue.

## Best Possible Solution

Keep #13670 open until maintainers either implement a focused Control UI topbar status component or explicitly decline that placement. A good implementation should live in the topbar render path, derive the active model from the current session/default model state, derive token/context usage from fresh session usage rows rather than promoting Debug-tab polling into global UI state, and add focused responsive UI tests for fresh, stale, and unavailable usage states.
