## Summary

Current main still does not implement the requested Control UI topbar display for active model plus token/context usage. The repo has adjacent model and usage surfaces in chat controls, context notices, per-message metadata, debug snapshots, slash commands, TUI/Web TUI, CLI, and docs, but the always-visible Control UI topbar status component requested in #13670 is absent.

## Best Possible Solution

Keep #13670 open until maintainers either implement a focused Control UI topbar status component or explicitly decline that placement. A good implementation should live in the topbar render path, derive the active model from current session/default model state, derive context/token usage from fresh session usage data where possible, avoid using debug-tab-only polling as normal UI state, and add focused UI tests plus docs if the surface ships.
