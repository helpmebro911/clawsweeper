## Summary

Keep #13670 open. Current main still does not implement an always-visible Control UI topbar display for the active model plus token/context usage. Model and usage data exist in adjacent Debug, Usage, Overview, and chat surfaces, but the topbar/header path still renders only navigation/search plus mobile-chat and theme controls.

## Best Possible Solution

Keep #13670 open until maintainers either implement a focused Control UI topbar status component or explicitly decline that placement. A good implementation should live in the topbar render path, derive the active model from current session/default model state, derive token/context usage from fresh session usage rows, handle stale/unavailable states, and add focused responsive UI coverage.
