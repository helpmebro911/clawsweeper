## Summary

Current main still lacks the requested pre-tool-call model/session routing contract. The existing `before_tool_call` path supports params rewriting, blocking, and approval, while model selection is handled earlier by `before_model_resolve`; neither provides a result contract to redirect an already emitted tool call to another model/session. The linked PRs #58079 and #62701 are useful context-field work, but they are open/unmerged and do not add routing semantics.

## Best Possible Solution

Keep #13225 open as a plugin/runtime API design request. The best path is an additive typed contract, either on `before_tool_call` or as a separate pre-tool routing hook, that preserves existing params/block/approval behavior while explicitly defining whether plugins may inspect preceding assistant context and redirect a pending tool call to a selected provider/model/session with clear permission, audit, and failure semantics. Treat #58079 and #62701 as related context-field work unless they merge and are extended to cover routing.
