## Summary

Keep #13225 open. Current main still exposes `before_tool_call` as a params/block/approval hook, and `before_model_resolve` only supports pre-run provider/model overrides before session messages load. The requested per-tool-call routing contract with preceding assistant output and model/session redirect remains missing. The existing discussion is also aligned with keeping this open: the latest maintainer/Codex review comment explicitly noted that the TTS argument-injection use case is likely covered by `ctx.agentId` plus params rewrite, but the model/session delegation request is not.

## Best Possible Solution

Keep #13225 open as a plugin/runtime API design request. Preserve the shipped `before_tool_call` params/block/approval behavior and the run-level `before_model_resolve` model override hook, but only close this issue after maintainers either ship or deliberately reject a typed contract that lets plugins inspect relevant preceding assistant output and redirect a pending tool call to another model or session. Treat #62701 and #58079 as partial context-plumbing work, not full superseding coverage.

## Risks / Open Questions

- A routing hook would expose more model/session context to plugins, so the design should explicitly handle plugin permissions and data exposure.
- The already-supported TTS-style argument injection should stay scoped to params rewrite and `ctx.agentId`; the remaining issue should focus on typed pre-tool-call model/session delegation.
