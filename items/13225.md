## Summary

Keep #13225 open. Current main still lacks the requested pre-tool-call model/session routing contract: `before_tool_call` can rewrite params, block, or require approval, and `before_model_resolve` can choose a model only before session messages load. The per-agent TTS argument-injection comment is partially covered by current `ctx.agentId` plus params rewriting, but the issue’s central request for preceding assistant output plus redirect/model/session routing remains unimplemented. Related PRs #58079 and #62701 are useful context work, but the provided GitHub context shows both are open/unmerged and neither adds redirect semantics.

## Best Possible Solution

Keep #13225 open as a plugin/runtime API design request. The best path is an additive typed contract, either on `before_tool_call` or as a separate pre-tool routing hook, that preserves existing params/block/approval behavior while explicitly deciding whether plugins may inspect preceding assistant context and redirect a pending tool call to a selected provider/model/session with clear permission, audit, and failure semantics. Treat #58079 and #62701 as related context-field work unless they merge and are extended to cover routing.
