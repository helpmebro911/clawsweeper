## Summary

Keep this PR open. Current main still enqueues 160-character inbound body previews in Slack, MS Teams, and Mattermost system-event labels, while the full message body is separately delivered through each channel's inbound envelope path. The PR is a focused implementation candidate for the paired open bug #67503, not obsolete or already implemented.

## Best Possible Solution

Keep this PR open and review a rebased version, or land an equivalent patch, that removes inbound body previews from Slack, MS Teams, and Mattermost system-event labels while preserving full message delivery through the inbound envelope path and keeping preview variables only for debug/type-compatible uses. Add targeted regression coverage for the queued system-event text where practical.
