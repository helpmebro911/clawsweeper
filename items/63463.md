## Summary

Keep this issue open. Current main has the generic pieces that should preserve slash-containing model IDs and send the bare model ID once an OpenAI-compatible custom provider is configured correctly, but the report is recent, concrete, unresolved, and current main still has no exact MLX regression or maintainer-confirmed retry with the documented custom-provider shape.

## Best Possible Solution

Keep this issue open for maintainer triage. Ask the reporter to retry with models.providers.mlx.baseUrl set to http://127.0.0.1:8080/v1, api set to "openai-completions", and models[].id set to the bare MLX model id. If it still fails, add a focused regression for mlx/mlx-community/Qwen3-30B-A3B-6bit that proves custom-provider resolution and verifies the Chat Completions payload uses the bare model id; consider adding a clearer validation warning for OpenAI-compatible local providers missing api or /v1.
