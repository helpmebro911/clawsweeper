## Summary

Keep this issue open. Current main still supports WhatsApp inbound debounce only at global/channel/account scope, while WhatsApp group config remains limited to mention/tool/prompt behavior and the runtime passes a single scalar debounce value into the listener. The requested per-group `debounceMs` behavior is not implemented, and the original image/image/text workflow has an additional media batching question because WhatsApp media currently bypasses debounce.

## Best Possible Solution

Implement the WhatsApp case first in the WhatsApp plugin: add `debounceMs?: number` to the group config type, strict Zod schema, generated metadata/help/UI/docs, and regression tests. At runtime, resolve exact group then wildcard group debounce per inbound group message and pass it through the existing `createInboundDebouncer.resolveDebounceMs` seam, falling back to account/channel/global debounce for other chats. Add tests proving one group override does not affect other groups, and explicitly decide the media batching behavior for the reported image/image/text sequence. Treat Telegram/Discord/other channels as follow-up plugin-specific design unless maintainers want a shared public contract.
