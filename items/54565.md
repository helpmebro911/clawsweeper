## Summary

Keep this issue open. Current main still supports WhatsApp inbound debounce only at global/channel/account scope, while per-group WhatsApp config remains strict and does not accept `debounceMs`. The runtime still computes one scalar debounce value before listener attach, so it cannot vary the debounce window by exact group or wildcard group entry.

## Best Possible Solution

Implement the WhatsApp case in the WhatsApp plugin first: add `debounceMs?: number` to `WhatsAppGroupConfig`, the strict Zod schema, generated metadata/help/UI/docs, and focused regression tests. At runtime, resolve exact group then wildcard group debounce per inbound group message and feed it through the existing `createInboundDebouncer.resolveDebounceMs` seam, falling back to account/channel/global debounce for other chats. Explicitly decide and test whether media messages are included in the batching window for the reported image/image/text sequence. Treat Telegram, Discord, and other channels as plugin-specific follow-ups unless maintainers choose a shared public contract.
