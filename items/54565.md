## Summary

Keep open. Current main still supports inbound debounce globally, per channel, and for WhatsApp at channel/account scope, but not inside `channels.whatsapp.groups.<jid>`. The prior Codex review comment remains consistent with current source: the WhatsApp config type/schema still reject group-level `debounceMs`, and the live monitor still resolves one account/channel debounce value before creating the listener.

## Best Possible Solution

Keep this issue open. The best path is a WhatsApp-owned implementation first: add `debounceMs?: number` to the WhatsApp group config type/schema/generated docs surface, resolve exact group and wildcard group debounce from the effective account/channel group map for each inbound group message, pass that through `createInboundDebouncer.resolveDebounceMs`, and add regression tests proving group-specific values override account/channel/global debounce only for the targeted group. Treat Telegram/Discord/other channel support as explicit follow-up design or per-plugin additions.
