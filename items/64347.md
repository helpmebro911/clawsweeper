## Summary

Keep open. Current main still has no native warm-reuse path for `sessions_spawn`: the native subagent path generates a fresh UUID-backed child session key for every spawn, the public tool/config schemas expose no native reuse key or pool selector, and the current docs/tests explicitly keep `resumeSessionId` scoped to ACP rather than native subagents.

## Best Possible Solution

Keep this issue open. The maintainer path is to design an explicit native subagent reuse contract around `src/agents/subagent-spawn.ts`, `src/agents/tools/sessions-spawn-tool.ts`, `src/config/zod-schema.agent-defaults.ts`, and `docs/tools/subagents.md`, choosing opt-in reuse versus automatic pooling, documenting isolation/history tradeoffs, handling concurrency and archive cleanup, and adding focused native `runtime="subagent"` tests. Treat #64468 and #72331 as related cache/runtime cleanup, not as canonical resolution.

## Risks / Open Questions

- A native warm-reuse implementation could leak or mix unrelated child history unless reuse is explicit, scoped, and documented.
- Automatic pooling would need locking/allocation semantics so concurrent spawns do not reuse an active child session unsafely.
- Archive cleanup, follow-up routing, and thread/session-mode behavior need a defined contract for reused native child sessions.
