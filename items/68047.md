## Summary

Keep this PR open. Current main still only applies the Claude-name image-support fallback to `claude-cli`, not the direct `anthropic` provider, and the current gateway tests still lack the direct Anthropic stale-catalog and catalog-miss cases this PR adds. Main has a text-only offload mitigation, but that does not implement the PR's gateway capability fix.

## Best Possible Solution

Review or update this focused PR against current main. The likely best path is to land the direct `anthropic` Claude-name fallback with the two regression tests, or replace it with an explicit provider-owned image-capability seam if maintainers do not want more provider ids in the gateway resolver. Keep this PR open until main includes that behavior or a maintainer explicitly decides the offload path is the intended replacement.
