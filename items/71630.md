## Summary

Keep #71630 open. Current main has local chat lifecycle signals and send/stop controls, but it does not render the requested compact composer runtime status chip, and the broader gateway-owned session activity API referenced by the issue is still absent/open rather than shipped.

## Best Possible Solution

Keep #71630 open as a focused Control UI feature. The best path is to add a compact composer status chip in `ui/src/ui/views/chat.ts`/chat styles with tests, derive an initial `In progress`/`Done`/`Interrupted` state from existing chat lifecycle and reconnect handling, and wire it to the gateway-owned `sessions.activity` surface when #39127 lands or as part of the same product slice.

## Risks / Open Questions

- A fully authoritative session-scoped indicator likely depends on the open #39127 activity API; using only current Web UI local lifecycle signals would be approximate around reconnects, child agents, and background finalization.
- The UI work should cover responsive/mobile layout, accessibility labels/tooltips, and locale generation so the chip does not create layout shift or untranslated strings.
