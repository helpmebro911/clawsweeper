## Summary

Current main still does not implement the requested compact Web UI composer runtime status indicator. The issue remains actionable as a focused Control UI feature and is distinct from the broader per-session activity API work in #39127 and channel status-surface work in #52640.

## Best Possible Solution

Keep this issue open. Implement a compact, accessible Control UI composer status chip near the attachment/voice/Talk controls in ui/src/ui/views/chat.ts, with stable desktop/mobile layout styles and focused render tests. Use #39127's session-scoped activity API when available; an interim version may use existing chatRunId/chat event lifecycle signals, but it should fail closed after disconnect/reconnect uncertainty and must not add transcript messages.
