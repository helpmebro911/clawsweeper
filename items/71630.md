## Summary

Keep #71630 open. Current main has local Web UI chat lifecycle state and existing send/stop controls, but it does not render a compact In progress/Interrupted/Done status chip in the composer and does not expose the gateway-owned session activity API that the issue wants to consume. Related #39127 remains open and is the right backend dependency, but it does not supersede this UI-specific composer request.

## Best Possible Solution

Keep #71630 open as a focused Control UI chat feature. Implement a compact composer status chip in the existing chat toolbar, backed by #39127's per-session activity API when available; a minimal interim version could map current chatRunId/chat terminal events/reconnect handling to In progress, Done, and Interrupted, but it should fail closed after disconnects or missed terminal events.
