## Summary

Keep #36754 open. Current main still skips symlinked plugin directories during auto-discovery of global and workspace extension roots. The explicit `plugins.load.paths` workaround follows symlinks via `fs.statSync`, but that is a separate code path and does not fix scanning `~/.openclaw/extensions` or `<workspace>/.openclaw/extensions`. The related PR attempts in the supplied context are closed and unmerged, so they do not supersede the issue.

## Best Possible Solution

Keep #36754 open. The best fix is a narrow change in `src/plugins/discovery.ts`: resolve symlink `Dirent`s in `discoverInDirectory` before the file/directory branch, skip or diagnose broken symlinks, then reuse the existing package, bundle, index, candidate-validation, and security checks. Add regression coverage for symlinked plugin directories under both global and workspace extension roots while keeping the existing symlink escape tests intact.
