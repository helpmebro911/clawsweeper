## Summary

Keep #38573 open. Current main still forces resolved topic/thread cron deliveries through the direct outbound path, and the supported cron delivery model, gateway schema, agent tool surface, UI type, and docs do not expose an opt-in route for text-only topic/forum completions to run through main-session announce/agent-turn judgment before posting.

## Best Possible Solution

Keep this open for maintainer product/API work. Add a backward-compatible cron completion-route option that preserves direct topic/thread delivery by default, lets text-only topic/forum completions opt into main-session announce/agent-turn delivery, keeps structured payloads on direct delivery, defines fallback behavior, and updates cron storage/types, normalization, gateway protocol, agent tool schema, CLI/UI, docs, and tests.

## Risks / Open Questions

- Changing the default topic/forum cron route would risk regressing the current reliable direct-send behavior, so the safe product shape is an explicit opt-in with clear fail-closed versus direct-fallback semantics.
- A narrow dispatch-only patch would leave clients unable to request the behavior consistently; the supported option needs to cross cron types/storage/normalization, gateway protocol, agent tool schema, CLI/UI, docs, and regression tests.
