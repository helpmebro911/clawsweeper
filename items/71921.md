## Summary

Keep open. Current main supports Bedrock vision metadata when fresh discovery succeeds, but the reported stale explicit/static `amazon-bedrock` catalog path still preserves non-image `input` metadata and runtime image support still depends on `input` containing `image`.

## Best Possible Solution

Keep the issue open. The best fix is provider-owned: refresh or merge discovered Bedrock model metadata into stale explicit/static `amazon-bedrock` rows, including rows missing `input` or stuck at `["text"]`, while preserving intentional user overrides. Add regression coverage for Claude Opus/Sonnet 4.x foundation and regional/global inference-profile IDs, and consider a narrow Bedrock Claude repair shim only as defense-in-depth.
