## Summary

Current main does not fully implement the requested Bedrock Claude vision-capability fix. The Bedrock plugin has context-window knowledge for the affected Claude 4.x IDs, but image capability still comes from AWS discovery metadata or from an already discovered backing foundation model. If those rows arrive as text-only, OpenClaw still emits `input: ["text"]`, matching the reported `Model does not support images` path.

## Best Possible Solution

Keep #71921 open. The best fix is provider-owned Bedrock capability metadata in `extensions/amazon-bedrock/discovery.ts`: recognize the affected Claude Opus 4.7, Opus 4.6, and Sonnet 4.6 base IDs plus regional/global inference-profile forms as text+image capable, apply that override when mapping foundation models and inference profiles, and add focused discovery tests for stale text-only or missing modality data.
