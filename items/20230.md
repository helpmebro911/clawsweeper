## Summary

Keep #20230 open. Current main has shipped the related active assistant name/avatar and distinct user bubble styling, but the central remaining request is still missing: no per-agent chat bubble color contract, and assistant chat rendering still uses one active assistant identity rather than per-message/per-agent attribution.

## Best Possible Solution

Keep the issue open until maintainers either implement or explicitly reject per-agent chat styling and per-message assistant attribution. A complete fix likely belongs across the Control UI chat model, session/transcript metadata, identity/config validation, and CSS rendering: persist assistant identity per message/group, add a validated optional color/styling field or render hook, render assistant-specific CSS variables/classes, and add focused tests for mixed-agent and historical-session attribution.
