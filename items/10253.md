## Summary

Keep #10253 open. Current main still lacks a hook/webhook-specific lifecycle policy for cleanup, archival, delayed deletion, or transcript retention. Hook agent runs are dispatched through the isolated cron runner, but hook session keys such as `hook:gmail:*` are persisted as ordinary hook sessions, while the existing cron reaper only prunes canonical cron run-session aliases. The prior issue discussion also kept this item open, and the relevant hook config, dispatch, runner, reaper, session-list, and docs surfaces remain unchanged for this request.

## Best Possible Solution

Keep #10253 open. The best maintainer path is a core hook-session lifecycle policy under `hooks`, with config types/schema/help/docs, dispatch and finalization wiring for `/hooks/agent` plus mapped/Gmail hooks, explicit archive/delete/transcript-retention semantics, and focused tests for Gmail per-message keys, delayed cleanup, `sessions_list` visibility, and interaction with existing cron/session retention settings.
