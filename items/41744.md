## Summary

Keep open. The current main branch has partial media preservation, but it still lacks the start-args fallback described by the later reproductions: image tool results without `MEDIA:` text or `details.path` still extract no media. There are also open implementation PRs tracking this exact issue, including #46268 with closing syntax and #50359 from the issue author, so the issue should remain open until a canonical fix lands or the PRs are closed.

## Best Possible Solution

Keep this issue open and review the linked implementation candidates. The likely fix belongs in the embedded tool-result media extraction path: pass the stored `tool_execution_start` args into media extraction, fall back to `path`/`file_path`/`filePath` only when trusted image content exists and no explicit media path is present, then preserve that media through the existing pending-tool-media merge and Feishu `sendMediaFeishu` delivery path. Once one PR lands with focused regression coverage, close this issue as implemented and close or supersede the duplicate PR.
