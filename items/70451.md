## Summary

Keep #70451 open. Current main fixes the broader #64306-style bootstrap preload for status/health/gateway-status paths, but the hooks CLI still builds a hook report through plugin diagnostics, which enters broad plugin module loading during `hooks list/info/check/enable/disable`. That leaves the Windows `openclaw hooks enable ...` timeout/SIGKILL report unproven as fixed.

## Best Possible Solution

Keep #70451 open and narrow the hooks CLI implementation. `openclaw hooks list/info/check/enable/disable` should use cold hook/plugin metadata by default, or load only the selected hook's owning package, instead of `buildPluginDiagnosticsReport()` with `loadModules: true`. Add regression coverage that `hooks enable <standalone-hook>` and `hooks list --json` do not invoke broad plugin module loading, and document the `hooks.internal.entries.<id>.enabled` config fallback for users blocked by the CLI.
