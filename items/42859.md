## Summary

Keep this PR open. The linked prompt-noise problem still exists on current main, but the PR's blanket removal of gateway sender fields is still too broad because those fields feed command identity and sender-bound ownership checks.

## Best Possible Solution

Keep this PR open, but request a narrower fix: suppress synthetic Control UI/internal gateway sender metadata from prompt-visible untrusted metadata while preserving trusted gateway caller identity for command authorization, run attribution, and sender-bound lifecycle/focus/ACP ownership checks. Add paired regression tests for both behaviors.
