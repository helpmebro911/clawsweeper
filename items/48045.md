## Summary

Keep open. Current main has partial unmanaged Playwright download handling, but the reported navigate-to-download failure is still not handled: navigation still calls `page.goto()` and rethrows non-retryable errors, the `/navigate` response still only returns the URL/target, and there is no CDP protocol-level download handling in the browser plugin code.

## Evidence

- **Navigate API has no download result surface:** `navigateViaPlaywright` is typed to return only `{ url: string }`; the route spreads that result into `{ ok: true, targetId, ...result }`, so current `/navigate` cannot return the requested `download.triggered: true` payload.
  - file: [extensions/browser/src/browser/pw-tools-core.snapshot.ts:319](https://github.com/openclaw/openclaw/blob/d228463120a5057f7ccf9a5c31af62fe4a922088/extensions/browser/src/browser/pw-tools-core.snapshot.ts#L319)
  - command: `nl -ba extensions/browser/src/browser/pw-tools-core.snapshot.ts | sed -n '300,410p'`
  - sha: [d228463120a5](https://github.com/openclaw/openclaw/commit/d228463120a5057f7ccf9a5c31af62fe4a922088)
- **page.goto errors still propagate:** `gotoPageWithNavigationGuard` calls `opts.page.goto(...)` and its catch block rethrows errors unless a navigation-policy block was recorded, so a Playwright download-navigation error is not converted into success.
