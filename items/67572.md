## Summary

Keep open. This contributor PR is paired with open #48123 by the same author, and current main only partially overlaps it: non-image chat.send attachments are saved as media:// refs and transcript media, but ordinary dispatch still does not populate the ctx.MediaPaths/MediaTypes workspace-staging contract this PR is trying to add.

## Best Possible Solution

Keep this PR open as the implementation candidate paired with #48123. The best maintainer path is a rebase or smaller split that preserves current WebChat/file-upload behavior, then either implements the missing ordinary chat.send ctx.MediaPaths/MediaTypes workspace-staging contract with size/error/security tests, or explicitly decides that the shipped media://inbound plus tool/media-ref path supersedes workspace staging and closes both paired items together.

## Risks / Open Questions

- The PR broadens data access by persisting arbitrary non-image user payloads and staging them into the agent workspace; even with existing media-store and sandbox guards, that is security-sensitive enough for maintainer review.

- Current main still treats non-image offloads as imageOrder: offloaded; mixed image/non-image ordering and staging behavior need focused tests if this PR is revived or split.
