## Summary

Keep this issue open. Current main still converts provider billing/rate-limit failures into normal final reply text, WhatsApp has no suppress/custom/owner-only error policy surface, and Telegram’s existing error policy does not cover the provider-failure final-reply path described by the reporter. The prior ClawSweeper comment reached the same conclusion, and the current checkout at 646a268d2710bc54e478349ef5690f3c8803df4f still matches that behavior.

## Best Possible Solution

Implement a shared outbound provider/runtime error-reply policy before channel delivery. The likely maintainer path is a generic final-error classification on ReplyPayload or the reply pipeline, plus per-channel/account/group/topic overrides for normal reply, silent suppression, custom text, and owner-only routing. Cover WhatsApp groups, WhatsApp DMs, Telegram groups/topics/DMs, billing/rate-limit provider failures, and unchanged tool-error behavior; preserve full diagnostics in logs/status/control surfaces.
