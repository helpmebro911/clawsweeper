## Summary

Keep open. Current main still lacks Signal inbound status reaction lifecycle wiring, while the PR is a focused attempt to fix a real gap reported in #72020. The review comments point to implementation details that need resolution, not to the PR being obsolete, implemented, duplicated, or out of scope.

## Best Possible Solution

Keep the PR open and finish it as the canonical fix for #72020: wire Signal inbound handling through createStatusReactionController using sendReactionSignal, pass lifecycle callbacks into dispatchInboundMessage, handle direct and group targets, gate by the actual ack/mention decision, avoid done on queuedFinal=false, add regression tests for direct, group, error, and non-final paths, and update the status reaction docs once behavior ships. #65919 should remain the broader presence-lifetime follow-up.
