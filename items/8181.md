## Summary

Keep #8181 open. Current main has useful partial progress for explicit local embeddings: `provider: "local"` falls back to a default Hugging Face GGUF model and calls `node-llama-cpp`'s model resolver. The central zero-config request is still unmet because `node-llama-cpp` is not installed as a runtime dependency, local setup errors and docs still instruct operators to install/approve/rebuild the optional native package, and `provider: "auto"` still skips local embeddings on a clean config unless `local.modelPath` points to an existing local file. The latest issue discussion already reached the same conclusion, and no relevant implementation files have changed since that keep-open review.

## Best Possible Solution

Keep #8181 open until maintainers either implement true turnkey local embeddings or explicitly decide that local embeddings remain a manual optional runtime and update the issue/docs accordingly. A complete implementation should handle native runtime install/build or prebuilt binaries, first-use GGUF download with clear progress and failure messaging, clean `auto` versus explicit `local` semantics, and matching docs plus tests.
