## Summary

Keep #8181 open. Current main partially supports the helpful issue comment: explicit `provider: "local"` uses `node-llama-cpp` and a downloadable default GGUF model. The requested zero-config setup is still not implemented because the native runtime is still optional/not installed by OpenClaw, user-facing docs and setup errors still require manual install/build steps, and auto mode intentionally skips the default downloadable local model unless a real local file path already exists.

## Best Possible Solution

Keep #8181 open until maintainers either implement a true turnkey local embedding path or explicitly decide the product should remain optional/manual. A complete fix would need package/install/update handling for the `node-llama-cpp` native runtime or prebuilt binaries, first-use model download with clear progress/failure messaging, and docs/tests aligned around `memorySearch.provider: "local"` as the only required user config.
