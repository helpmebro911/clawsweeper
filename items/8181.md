## Summary

Keep #8181 open. Current main has partial support for explicit local embeddings: `provider: "local"` defaults to the built-in EmbeddingGemma GGUF model and resolves it through `node-llama-cpp`. The central zero-config request is still unmet because the native runtime is not installed as a normal dependency, setup/error paths still tell users to install/approve/rebuild `node-llama-cpp`, and `provider: "auto"` still refuses to select the default downloadable local model on a clean config.

## Best Possible Solution

Keep #8181 open until maintainers either implement true turnkey local embeddings or explicitly decide that local embeddings remain a manual optional runtime. A complete fix should handle native runtime install/build or prebuilt binaries, first-use GGUF download with clear progress and failure messaging, clean `auto` versus explicit `local` semantics, and matching docs plus tests.
