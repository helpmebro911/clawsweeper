## Summary

Keep open. Current main does not show the linked fix as shipped: `openclaw mcp serve` still lacks the stdout isolation used by the other MCP stdio server, and the only directly related fix is still an open PR (#68800), not merged into the checked-out main.

## Best Possible Solution

Keep #68587 open and either land a tested fix equivalent to #68800 or replace it with a maintainer-verified fix that keeps `openclaw mcp serve` stdout protocol-only and adds a regression test proving `tools/list` returns a normal MCP response for strict stdio clients such as Hermes/mcp-inspector.
