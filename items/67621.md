## Summary

Keep open. Current main still applies the `$[A-Z_]...` scanner to raw Python/Node source text without string-literal awareness whenever script preflight runs. The exact `security=full` plus `ask=off` path is now skipped, but that is only a partial mitigation. There is also an open implementation candidate, #67924, linked with closing syntax, so the maintainer-safe path is to review/land or close that PR before resolving this issue.

## Best Possible Solution

Keep this issue open while #67924 is reviewed, updated if necessary, and either landed or deliberately closed. The likely fix belongs in `src/agents/bash-tools.exec.ts`: make script preflight ignore `$VAR` matches inside Python/Node string and comment contexts while preserving rejection for actual shell-variable syntax in executable source, with focused regression coverage in `src/agents/bash-tools.exec.script-preflight.test.ts`.
