## Summary

Keep open. Current `main` still lacks WhatsApp per-group `allowFrom`: the public config type, Zod schema, runtime access-control path, docs, and tests all operate on channel/account-level `groupAllowFrom` only. The request remains actionable and in-scope for the bundled WhatsApp plugin.

## Evidence

- **WhatsApp group config has no per-group sender allowlist field:** `WhatsAppGroupConfig` only exposes `requireMention`, `tools`, `toolsBySender`, and `systemPrompt`; there is no `allowFrom` key on `channels.whatsapp.groups.<jid>` in the current public config type.
  - file: [src/config/types.whatsapp.ts:25](https://github.com/openclaw/openclaw/blob/bc2d53dacd4eda5488af5f884791ecfb80eb06ee/src/config/types.whatsapp.ts#L25)

  - file: [src/config/zod-schema.providers-whatsapp.ts:21](https://github.com/openclaw/openclaw/blob/bc2d53dacd4eda5488af5f884791ecfb80eb06ee/src/config/zod-schema.providers-whatsapp.ts#L21)
