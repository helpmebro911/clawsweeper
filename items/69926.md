## Summary

Keep #69926 open. Current main still lacks WhatsApp per-group `allowFrom`: the WhatsApp group type and strict Zod schema only accept `requireMention`, tool policy, and `systemPrompt`, while inbound sender authorization still resolves one account-level `groupAllowFrom` with `allowFrom` fallback and never looks up `groups[<jid>].allowFrom`. The requested parity is real: Telegram and other channel docs/code already have per-group sender allowlist behavior, and related #69874 is a separate maintainer-owned inheritance issue rather than a canonical tracker for this feature.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin and config contract: add optional `allowFrom` to `WhatsAppGroupConfig` and `WhatsAppGroupEntrySchema`, update generated config metadata/docs, resolve the effective group entry for the inbound group JID at both root and account scopes, and use `groups[<jid>].allowFrom` as an override over account/channel `groupAllowFrom` with regression tests for allow, block, wildcard, empty override, fallback, and multi-account group-map behavior. Keep #69926 open until that lands or a narrower PR tracks it.
