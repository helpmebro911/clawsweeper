## Summary

Keep this PR open. Current main still appends the workspace HEARTBEAT.md path hint to any resolved heartbeat prompt mentioning `heartbeat.md`, including user-configured prompts, while the user-facing heartbeat docs say custom heartbeat prompt bodies are sent verbatim. The linked bug remains open, and the related PRs in the provided context were closed duplicate attempts rather than merged replacements.

## Best Possible Solution

Keep this PR or the linked open bug #40255 active. The best path is a small heartbeat-runner fix that distinguishes non-empty user-configured heartbeat prompts from built-in/generated heartbeat prompts, preserves the workspace-path hint and idempotency for default/internal prompts, adds a regression test for custom prompts mentioning HEARTBEAT.md, and explicitly resolves the prompt-safety tradeoff before merge.

## Risks / Open Questions

- A bot review raised a real prompt-safety/product tradeoff: skipping the appended workspace path hint for custom prompts also removes the current path-disambiguation guardrail. Maintainers should decide whether the documented verbatim custom-prompt contract is enough or whether heartbeat file selection needs a separate tool-layer/non-overridable guard.
- The PR exports `appendHeartbeatWorkspacePathHint` for direct unit testing. That is likely acceptable inside a runner module, but maintainers may prefer testing through `runHeartbeatOnce()` or another existing seam if they want to avoid widening the module surface.
