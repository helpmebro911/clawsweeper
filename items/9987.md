## Summary

Keep #9987 open. Current main has backend support to enumerate pending exec approvals via `exec.approval.list`, but the requested user-facing CLI command `openclaw approvals pending` is still absent from command registration, CLI docs, and CLI tests. The related implementation PRs #10013 and #10341 were closed unmerged, and the latest issue comment also recorded a keep-open Codex review for the same gap.

## Best Possible Solution

Keep #9987 open until OpenClaw adds a real `openclaw approvals pending` CLI surface that calls the existing `exec.approval.list` gateway method, renders a useful table with command details, supports `--json`, and documents/tests the behavior. `--watch` can either ship in the same change or be split into a narrower follow-up if maintainers want an incremental implementation.
