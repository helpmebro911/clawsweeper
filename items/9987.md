## Summary

Keep #9987 open. Current main has backend support for listing pending exec approvals, and channel runtimes can replay those pending approvals, but the requested user-facing `openclaw approvals pending` CLI command is still missing from the approvals CLI, CLI tests, and CLI docs. The provided GitHub discussion also records a maintainer Codex review keeping this item open, and the related implementation attempts #10013 and #10341 are closed unmerged.

## Best Possible Solution

Keep #9987 open until OpenClaw adds `openclaw approvals pending` in `src/cli/exec-approvals-cli.ts`, backed by the existing `exec.approval.list` gateway method. The first implementation should provide human-readable table output with command details, `--json` output for monitoring, focused CLI tests in `src/cli/exec-approvals-cli.test.ts`, and docs updates at `docs/cli/approvals.md` and `docs/cli/index.md`. Treat `--watch` as part of that work only if maintainers want live updates in the initial CLI surface; otherwise split it to a narrower follow-up.
