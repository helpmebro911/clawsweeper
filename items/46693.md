## Summary

Keep #46693 open. Current `main` still does not pass explicit `cacheRetention` through for OpenAI or OpenAI-Codex from the configured extra-params path. The OpenAI transport can emit `prompt_cache_retention: "24h"` when it receives `options.cacheRetention === "long"` on a direct OpenAI host, and docs describe that behavior, but the shared resolver used by `applyExtraParamsToAgent` still returns `undefined` unless the model is Anthropic-family or eligible Google. The PR remains a plausible focused fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR or land an equivalent focused fix on `main`: update `src/agents/pi-embedded-runner/prompt-cache-retention.ts` so explicit `cacheRetention` values pass through for `openai` and `openai-codex` without adding an implicit default, then add regression coverage proving configured OpenAI/OpenAI-Codex model params reach the stream options and that direct OpenAI Responses emits `prompt_cache_retention: "24h"` only when the downstream host policy allows it.
