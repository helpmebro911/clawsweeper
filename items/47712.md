## Summary

Keep open. Current main does not implement the requested generic `messages.tts.filterEmoji` option: the TTS config schema/types/docs do not expose it, the main speech-core path only strips Markdown before synthesis, and the agent `tts` tool passes the requested text through to the TTS runtime. The only emoji stripping found is provider-local behavior in `tts-local-cli`, which does not satisfy a global TTS config option/default.

## Risks / Open Questions

- Default emoji filtering is a product behavior choice because it changes spoken text globally across providers; implementation should decide whether filtering is default-on, opt-in, and whether it applies before or after TTS directive parsing and Markdown cleanup.
