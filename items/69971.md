## Summary

Keep open. Current main still lacks both requested behaviors: plugin install does not pass a profile-aware extensionsDir into the local/npm installer paths, and discovery still skips symlinked plugin package directories. The PR also has an unresolved substantive review concern: using resolveStateDir for install while discovery roots still use resolveConfigDir can split install and discovery under OPENCLAW_CONFIG_PATH-only or legacy-dir setups, so the PR likely needs revision rather than closure.

## Best Possible Solution

Keep the PR open and ask for a revision that computes the managed plugin install root from the same source as plugin discovery, or otherwise updates discovery/install together so they remain aligned for --profile, OPENCLAW_STATE_DIR, OPENCLAW_CONFIG_PATH-only, and legacy-dir cases. Keep the symlink discovery change, but add regression tests for profile-aware install forwarding, directory symlink discovery, broken symlink diagnostics, and the root-alignment edge case raised in the review comment.
