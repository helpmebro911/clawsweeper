## Summary

Current main still does not implement the reported Volcengine Plan imageModel path. The explicit imageModel configuration still selects the configured provider/model, but the bundled Volcengine coding catalog and docs remain text-only for `volcengine-plan`, the reported `doubao-seed-2.0-pro` model is absent from the Volcengine catalog, and there is no Volcengine media-understanding provider or regression coverage for this Ark coding image endpoint. A related open PR, #55688, is only a partial candidate and is not merged on main.

## Best Possible Solution

Keep this issue open. Maintainers should verify the Volcengine Ark coding image endpoint with `doubao-seed-2.0-pro`, then either add a Volcengine-owned image-capable catalog/media-understanding path with regression coverage, or explicitly document that `volcengine-plan` imageModel is unsupported and point users to a supported `volcengine/*` image-capable model or a custom provider config with `input: ["text", "image"]`. Review #55688 as a partial catalog-capability candidate, but do not treat it as a complete fix for this issue until the reported model and endpoint are covered or ruled unsupported.
