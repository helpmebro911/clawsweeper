## Summary

Keep PR #45342 open. Current main still calls provider usage endpoints directly from `loadProviderUsageSummary` with no TTL/cache layer and has no `usage.cacheTtlMs` config surface. The PR is a focused bug fix for #45332 and the provided review timeline shows the author addressed the concrete cache-key, clock, and transient-error review concerns in later commits, so it remains useful rather than obsolete.

## Best Possible Solution

Keep #45342 open and either land it after maintainer review/CI or implement the same fix on main: add a bounded provider usage summary cache keyed by agent directory and requested providers, expose `usage.cacheTtlMs` through config schema/types/help, bypass cache for explicit auth/test scenarios, avoid caching transient failures, and keep regression coverage for cache hit, cache disabled, provider-key separation, clock consistency, and retry-after-error behavior.
