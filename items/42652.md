## Summary

Current main still has the reported Discord A2A announce account routing gap. Discord announce targets are parsed from the session key and returned before session hydration, so stored `deliveryContext.accountId` / `lastAccountId` is not merged into the final gateway `send`. Related #51626 was closed in favor of this issue, so this remains the canonical tracker rather than a duplicate to close.

## Best Possible Solution

Keep this issue open and fix the Discord `sessions_send` A2A announce path. The narrow maintainable path is to opt Discord into `preferSessionLookupForAnnounceTarget` or adjust `resolveAnnounceTarget` to merge missing account/thread fields from stored session metadata without regressing parse-only routing. Add a regression test where a parsed Discord target session has `deliveryContext.accountId` or `lastAccountId` and the final gateway `send` receives that account id.
