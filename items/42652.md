## Summary

Keep open. Current main still leaves Discord A2A announce delivery without the session-specific accountId when the target can be parsed from the session key, because Discord does not opt into session-backed announce target hydration. The related duplicate #51626 was closed in favor of this issue, so this remains the canonical tracker.

## Best Possible Solution

Fix the Discord `sessions_send` A2A announce path so parsed Discord targets hydrate missing accountId from stored session metadata. The narrowest current path is to set Discord `preferSessionLookupForAnnounceTarget: true`, or adjust `resolveAnnounceTarget()` to merge missing account/thread fields from `sessions.list` without regressing parse-only channel behavior. Add a regression test where a parsed Discord target session has `deliveryContext.accountId: "work"` and the final gateway `send` receives `accountId: "work"`.
