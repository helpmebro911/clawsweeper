## Summary

Keep this issue open. Current main partially addresses the enum-docs and cleanup pieces, but the main requests remain real: no nearest-value hint for `tools.exec.host`, no unified operator view of process/MCP/subagent runtime fanout, and no documented model-free local maintenance-runner pattern.

## Best Possible Solution

Keep this issue open for maintainer triage, then split it into focused follow-ups: add a small operator hint for invalid `tools.exec.host` values if desired; design a unified runtime fanout/status surface spanning process supervisor sessions, bundled/user MCP runtimes, subagent/task state, and Control UI/CLI status; document the deterministic local maintenance-runner pattern with systemd user timers or equivalent scripts for work that does not need model turns; separately consider inline dashboard log review UX if maintainers want that product surface.
