## Summary

Keep this PR open. Current main still has the reported Discord `thread-reply` attachment gap: the handler reads only `media`, while the sibling `send` path supports `media`, `path`, and `filePath`; the linked same-author bug remains open, so this PR is still an implementation candidate rather than cleanup.

## Best Possible Solution

Keep this PR open and update it against current main. The best patch is narrow: in `extensions/discord/src/actions/handle-action.guild-admin.ts`, make `thread-reply` read `media ?? path ?? filePath`, pass the same scoped media options expected by the Discord send/runtime path, and add current-location tests for `filePath`, `path`, precedence, and media option passthrough before landing it with the linked bug.
