## Summary

Keep open. Current main does not implement the exact registrar-level Slack message-event tracking requested for subtype and skipped app_mention paths, and this is a focused recent PR with useful regression-test intent. The patch should not be merged as-is, though, because accepted message/app_mention flows already call trackEvent downstream in createSlackMessageHandler, so the proposed top-level calls would double-update common inbound events.

## Best Possible Solution

Keep the PR open for maintainer/author revision. The best fix is to avoid double-counting accepted messages while covering only message-event branches that return before createSlackMessageHandler and are intentionally considered inbound activity, with tests for subtype system events, DM-skipped app_mentions, mismatched drops, and no double update on normal messages.
