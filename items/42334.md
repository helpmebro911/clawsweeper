## Summary

Keep open. Current `main` does not implement this behavior: `createOpenAIDefaultTransportWrapper` still forces OpenAI models to `transport: "auto"` with `openaiWsWarmup: true` regardless of `baseUrl`, and the only built-in mitigation is reactive HTTP fallback after websocket failure. That means the PRâ€™s requested endpoint-based SSE default is not already on `main`, so there is no strong basis to close it as implemented or obsolete.
