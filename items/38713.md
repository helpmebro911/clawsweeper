## Summary

Keep PR #38713 open. Current `main` still resolves `openclaw skills info <name>` with exact-only `name`/`skillKey` matching, while the PR addresses a still-present CLI mismatch where list-visible skills can fail lookup because of case or separator differences. The predecessor PR #38701 was closed unmerged, and the linked bug #38546 was closed stale rather than fixed on main.

## Best Possible Solution

Keep PR #38713 open and finish or land the focused CLI resolver in `src/cli/skills-cli.format.ts`, with regression tests in `src/cli/skills-cli.test.ts`. The best implementation trims the requested name, preserves exact matches first, allows only unique case-insensitive and separator-normalized fallback matches, sanitizes not-found output, and tracks any separate agent-facing skill activation issue independently if it still reproduces.
