## Summary

Keep #18860 open. Current main still does not implement the requested `after_tools_resolved` typed plugin hook or an equivalent resolved-tools/schema API, and the linked issue #21330 points back to this PR rather than superseding it. A maintainer has also already left a keep-open Codex review comment on the PR discussion.

## Best Possible Solution

Keep #18860 open as useful implementation and design context. The best path is a narrow, maintainer-approved core plugin/runtime seam: add a typed resolved-tools hook or equivalent resolved-tools API in `src/plugins/hook-types.ts`, `src/plugins/hooks.ts`, and the embedded runner after `effectiveTools` is finalized, then add focused contract tests and public plugin docs for schema visibility and lifecycle semantics.
