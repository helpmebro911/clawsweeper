## Summary

Keep this PR open. Current main still lacks `after_tools_resolved` or an equivalent plugin-facing API for resolved per-attempt tool/schema discovery, and the linked feature issue #21330 is closed as superseded by this still-open PR. The latest maintainer discussion in the provided context also explicitly says to keep this PR open.

## Best Possible Solution

Keep this PR open as the active implementation/design path for plugin-facing resolved tool/schema discovery. A maintainer-quality solution should add a narrow additive contract across `src/plugins/hook-types.ts`, `src/plugins/hooks.ts`, `src/plugins/types.ts`, and `src/agents/pi-embedded-runner/run/attempt.ts`, plus focused contract tests and public docs at `https://docs.openclaw.ai/plugins/hooks`. If maintainers prefer the pull-based `getResolvedTools` design discussed in #21330, they should explicitly designate that replacement before superseding this PR.
