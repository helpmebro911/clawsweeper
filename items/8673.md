## Summary

Keep #8673 open. Current main still lacks a general retry/backoff path for transient OAuth token refresh failures; the only retry-like recovery is targeted to `refresh_token_reused` credential rotation. The related fix PR #45975 is still open/unmerged in the provided context, and #8677 was closed unmerged, so this issue remains a valid tracker.

## Best Possible Solution

Implement a bounded, tested retry policy around the actual OAuth refresh operation on the current auth path, preferably reusing the existing retry helpers. It should retry clearly transient network/timeout/5xx failures, preserve immediate handling for permanent auth failures such as invalid grants, and avoid swallowing the existing `refresh_token_reused` recovery path. Add focused tests under `src/agents/auth-profiles/` for success-after-transient-failure, exhaustion preserving the final error, and non-retryable auth failure behavior.
