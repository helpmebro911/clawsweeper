## Summary

Keep open. Current main does not expose `runtime.gateway.dispatchMethod` or any equivalent arbitrary gateway RPC dispatch surface to plugins. The motivating gap still exists: plugin runtime helpers can call selected gateway-backed helpers (`subagent`, `nodes`), but `subagent.getSessionMessages()` still routes through raw `sessions.get`, while `chat.history` remains a separate sanitized/budgeted gateway RPC with no in-process plugin runtime API. This PR is a focused plugin runtime API proposal and needs maintainer product/API judgment, not automated cleanup.

## Best Possible Solution

Keep the PR open for maintainer API review. The likely product path is either to land a carefully scoped `api.runtime.gateway.dispatchMethod` contract with authorization/docs/tests, or to provide a narrower plugin runtime helper for sanitized session history if broad gateway RPC dispatch is not acceptable.
