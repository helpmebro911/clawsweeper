## Summary

Keep open. Current main still sends Feishu message-resource downloads as type=image or type=file only; there is no retry/fallback from type=file to type=media for 502/5xx responses. The linked fix PRs #49967 and #50164 directly address the reported bug, but both are still open/unmerged in the provided GitHub context, so the issue should remain as the canonical bug until a fix lands or maintainers choose a canonical PR.

## Evidence

- **Downloader only forwards the provided type:** downloadMessageResourceFeishu accepts only type "image" | "file" and calls client.im.messageResource.get once with params: { type }; there is no catch/retry path for 502/503/504 and no alternate type=media attempt.
