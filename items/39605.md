## Summary

Keep #39605 open. Current main still is not a safe close: Discord and Slack native/slash command routing continue to resolve routes from the provider startup config snapshot, while `resolveAgentRoute` still derives DM session keys from the supplied `cfg.session.dmScope`. Telegram now has a fresh-runtime-config path and a regression test, but that only partially addresses the report. The related fix PR #39617 is still open and unmerged in the provided GitHub context, so the issue is neither implemented on main nor safely superseded.

## Best Possible Solution

Keep the issue open and land a focused maintainer-reviewed fix, likely via #39617 or an equivalent patch, that reloads or otherwise supplies current runtime config before Discord native command route resolution and Slack slash command route resolution. Add Discord and Slack regression tests matching the existing Telegram coverage, proving non-main DM routing or route-binding changes are honored between slash/native command invocations without recreating the provider.
