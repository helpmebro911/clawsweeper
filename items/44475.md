## Summary

Keep PR #44475 open. Current main still does not implement passthrough provider-model fields in `agents.defaults.models`: the schema is strict, the public type only exposes `alias`/`params`/`streaming`, and `applyModelDefaults()` never reads per-model default entries when normalizing provider model definitions. The PR remains a focused implementation for the closed linked request #44468, but it should be rebased and the normalized model-ref lookup review comment should be resolved before merge judgment.

## Best Possible Solution

Keep the PR open as the active implementation candidate for #44468. The best path is to rebase it onto current main, preserve the passthrough schema/type/defaults/test coverage, add normalized provider/model-ref lookup parity with existing model selection behavior, then run the targeted config test lane and changed gate before maintainer merge review.
