## Summary

Keep open. Current `main` still defines `llm_output` without any per-call timing field, emits that event without `latencyMs`/`durationMs`, and has no `llm_call` event surface to carry the data instead. The report is recent and specific, so none of the conservative close reasons apply.

## Evidence
