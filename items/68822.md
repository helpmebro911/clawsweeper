## Summary

Keep open. Current main still hard-codes the memory embedding retry and non-batch indexing concurrency values, so this PR is not obsolete or implemented elsewhere. The PR is also not merge-ready as described in the review discussion: the proposed config must be added to the validated/resolved memorySearch contract and read from the agent-scoped settings path, and the unrelated log artifact should be removed.

## Best Possible Solution

Keep the PR open and ask for a revised implementation that adds typed, validated, documented agent-scoped memorySearch knobs, resolves defaults/per-agent overrides into `ResolvedMemorySearchConfig`, wires `MemoryIndexManager` to those resolved settings instead of raw config casts, adds schema/resolution/manager tests, and removes the unrelated log file. Once that lands, use it to close or cross-link #68812 and the overlapping #14207 concurrency request as appropriate.
