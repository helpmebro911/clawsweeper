## Summary

Keep open. Current main only has partial overlap: non-batch index concurrency can be influenced through resolved `memorySearch.remote.batch.concurrency`, but embedding retry attempts/backoff remain hardcoded. This PR is still a plausible implementation candidate for #68812, but it is not merge-ready because it reads an invalid top-level config path, bypasses the typed/validated config contract, omits schema/docs/tests for the new knobs, and includes an unrelated root log file.

## Best Possible Solution

Keep this PR open for revision or maintainer closure after review. A mergeable version should remove the log file, add typed and validated agent-scoped config under `agents.*.memorySearch`, document the public keys, add config/runtime regression tests, and reconcile concurrency naming with the existing `remote.batch.concurrency` behavior and related #14207/#57733 work. Retry attempts/base/max delay are the still-missing part on main and are the best reason to continue the implementation rather than close it as obsolete.
