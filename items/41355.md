## Summary

Keep #41355 open. Current main still does not implement the requested Discord-side guard: the Discord adapter still forwards ThreadStarterBody unconditionally, and the shared reply path can still prepend `[Thread starter - for context]` on non-new sessions when that field is present. The discussion usefully points to #50522 as the direct proposed fix and #44447 as a manual-patch confirmation, but that work is not reflected in the checked-out main code.
