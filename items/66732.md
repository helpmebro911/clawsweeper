## Summary

Current main still does not resolve the reported Telegram-only reminder-note leak. The core guard can still append the exact note when successfulCronAdds is zero, reminder telemetry is still direct cron.add-only in the inspected paths, and Telegram direct text/media/payload delivery still forwards resolved text without an exact reminder-note suppressor. The prior ClawSweeper keep-open assessment remains consistent with current main.

## Best Possible Solution

Keep this issue open. The best fix is a narrow, regression-tested Telegram plugin change that suppresses only the exact false unscheduled-reminder note variants across direct text, media, and payload fallback sends, or a broader telemetry fix that reliably marks every successful reminder creation path before the reminder guard runs. Tests should cover Telegram direct text, media, interactive/payload fallback, direct cron.add, and any supported helper/exec-created reminder path.
