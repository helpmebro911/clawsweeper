## Summary

Current main still does not resolve the reported Telegram reminder-note leak. The exact guard note still exists, the reply runner can still append it when successful cron telemetry is zero, telemetry is still direct `cron.add`-only, and Telegram outbound paths still forward direct text, media captions, and payload fallback text without a specific suppressor.

## Best Possible Solution

Keep this issue open. The best fix is a narrow Telegram plugin change that suppresses only the exact false unscheduled-reminder note variants across direct text, media captions, and payload/interactive fallback sends, or a broader telemetry fix that reliably marks every successful reminder creation path before the guard runs. Add focused regression tests for Telegram direct text, media, payload fallback, direct `cron.add`, and any supported helper-created reminder path.
