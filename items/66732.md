## Summary

Keep open. Current `main` still appends the unscheduled-reminder note whenever `successfulCronAdds` stays `0`, the Codex app-server bridge still increments that counter only for direct `cron.add` tool calls, and there is no Telegram-side sanitizer for the bogus note. That is not strong evidence that #66732 is fixed or obsolete.
