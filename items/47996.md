## Summary

Keep PR #47996 open. Current main still does not implement stale queued-delivery expiry or the proposed lower-level direct reply/outbound dedupe, and the PR is not safely obsolete: the provided review context shows unresolved correctness concerns in the dedupe design plus broad unrelated UI formatting churn that would need cleanup before merge.

## Best Possible Solution

Keep the PR open, but ask for a narrow outbound-only revision. The stale recovery expiry can be landed with tests if it rechecks age at the actual replay point and preserves recovery/drain semantics. The direct reply dedupe should either be dropped in favor of existing gateway idempotency or redesigned with a complete behavior key covering queue identity/skipQueue, session, mirror transcript content, delivery flags, payload reply targets, scopes, abort/callback semantics, and best-effort partial failures. Remove unrelated UI formatting changes before merge consideration.
