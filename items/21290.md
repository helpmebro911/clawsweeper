## Summary

Keep open. Current `main` does not contain this PR's structured OpenTelemetry/GenAI diagnostics work: the diagnostics event schema still exposes only `model.usage`, the bundled `diagnostics-otel` plugin still records only `model.usage` into a single `openclaw.model.usage` span, docs still describe only that older event model/protocol support, and the core paths this PR instrumented do not emit the proposed run/inference/tool diagnostic events on `main`. This is not implemented-on-main, not obsolete, and still appears to describe a live gap.
