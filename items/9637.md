## Summary

Current main still does not implement a TUI plain-text/accessibility mode. The related PR #9685 is useful prior art, but it was closed unmerged; the core config schema, generated schema/docs, and TUI/tool rendering paths still expose emoji and Unicode glyphs without an opt-out.

## Best Possible Solution

Keep #9637 open and implement it as core TUI/config work. Add a documented ui.accessibility.plainText or equivalent config key to OpenClawConfig, zod schema, generated schema/help, and public docs; load it during TUI startup; thread it through ChatLog, ToolExecutionComponent, selectors, settings/theme rendering, and tool summary formatting; provide ASCII alternatives for emoji and Unicode glyphs; and add focused config validation plus TUI rendering tests. Treat closed PR #9685 as prior art, not as a completed or superseding fix.
