## Summary

Current main still does not implement the requested TUI plain-text accessibility option. The config schema has no `ui.accessibility.plainText` key, tool display and TUI tool headers still render emojis unconditionally, and list components still use Unicode separator/arrow characters. The related PR #9685 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep this issue open. The best path is to implement a real `ui.accessibility.plainText` or equivalent config contract across `src/config/types.openclaw.ts`, `src/config/zod-schema.ts`, generated schema/help/docs, then thread the resolved option through the TUI so tool display, tool execution headers, selectors, separators, cursors, and related tests can switch to ASCII/plain text. PR #9685 can serve as prior-art context, but it would need to be refreshed against current main and reviewed for the current config/TUI architecture.
