## Summary

Current main still does not implement a TUI plain-text/accessibility mode. The requested capability remains concrete core TUI/config work: the UI config schema accepts only appearance settings, tool/TUI render paths still emit emoji and Unicode glyphs unconditionally, and the related PR #9685 was closed unmerged rather than shipped.

## Best Possible Solution

Keep #9637 open and implement it as core TUI/config work. Add a documented `ui.accessibility.plainText` or equivalent key to `OpenClawConfig`, zod schema, generated schema/help, and public docs; load it once during TUI startup; thread it through `ChatLog`, `ToolExecutionComponent`, selector/theme rendering, settings UI, and tool summary formatting; provide ASCII alternatives for emoji and Unicode glyphs; and add focused config validation plus TUI rendering tests. Treat closed PR #9685 as useful prior art, not as a completed or superseding fix.
