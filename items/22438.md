## Summary

Keep #22438 open. Current main has partial bootstrap/context controls, but it still does not implement the requested schema-backed `agents.defaults.bootstrapTier` option or a `minimal | standard | full` resolver. The linked implementation PR #22439 is still open and explicitly tracks this issue.

## Best Possible Solution

Keep #22438 open until #22439 or a successor lands, or until maintainers explicitly decide that fixed allowlists plus per-run `lightContext` are the intended product design. A complete solution should wire `agents.defaults.bootstrapTier` through config schema/types/help/docs, bootstrap resolution, hook behavior, subagent/cron defaults, and focused tests for `minimal | standard | full`.
