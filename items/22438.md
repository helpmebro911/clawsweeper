## Summary

Keep #22438 open. Current main partially addresses the old motivation by giving subagent/cron bootstrap a broader fixed allowlist and by supporting lightweight per-run context, but it still does not implement the requested schema-backed `agents.defaults.bootstrapTier` (`minimal`/`standard`/`full`) control. The linked implementation PR #22439 is still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #22438 open until maintainers either merge a schema-backed `agents.defaults.bootstrapTier` design, likely through #22439 or a successor, or explicitly decide that the existing fixed subagent/cron allowlist plus per-run `lightContext` is the intended product direction. If implemented, the change should cover `src/agents/workspace.ts`, `src/agents/bootstrap-files.ts`, agent-defaults schema/types/help/labels/generated schema, public config docs, the bootstrap-extra-files hook behavior, and focused workspace/bootstrap tests.
