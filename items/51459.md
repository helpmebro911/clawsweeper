## Summary

The request is useful, but it is Lossless Claw/plugin-scope rather than an OpenClaw core cleanup item. Current main already provides plugin/context-engine hooks to implement automatic post-compaction recall without adding LCM-specific behavior to core.

## Close Comment

Closing as `clawhub` after a Codex review. This request is aimed at Lossless Claw/LCM behavior, and current `main` already treats that as plugin-owned rather than OpenClaw-core behavior: `VISION.md:52` keeps optional capability in plugins, `docs/plugins/community.md:64` documents Lossless Claw as the external `@martian-engineering/lossless-claw` plugin, and `docs/concepts/context-engine.md:28` shows context engines are installed and configured as plugins. The needed core extension points already exist on `main`: `src/plugins/hook-before-agent-start.types.ts:21` exposes `before_prompt_build` prompt injection, `src/plugins/hook-types.ts:661` exposes `after_compaction`, and `src/agents/harness/prompt-compaction-hook-helpers.ts:110` wires the post-compaction hook. There is also an in-tree example of this pattern in `extensions/active-memory/index.ts:2034`, where a plugin performs automatic recall before the main reply and injects the result with `prependContext`. Given that, “auto-LCM recall after compaction” belongs in the LCM plugin / ClawHub side, not as OpenClaw core cleanup.
