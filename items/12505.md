## Summary

Keep #12505 open. Current main has useful opt-in sandbox backends and hardening for tool execution, but it still does not implement the central request: secure-by-default tiered sandbox presets, native plugin process isolation, IPC-proxied plugin SDK calls, or per-plugin manifest policies for env/fs/net/resources. The existing issue discussion also already treats #29974 as partial Docker sandbox progress, not a complete resolution.

## Best Possible Solution

Keep #12505 open as the canonical sandbox/isolation architecture tracker unless maintainers split it into focused follow-up issues. The best product path is a core design for secure-by-default tiered sandbox presets, native plugin/skill isolation or an equivalent stronger execution boundary, IPC/proxy SDK enforcement, manifest-declared env/fs/net/resource policies, credential scoping, and migration guidance that treats current Docker/SSH/OpenShell sandbox backends as partial groundwork.
