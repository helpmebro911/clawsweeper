## Summary

Keep #12505 open. Current main has real opt-in tool sandboxing and sandbox backend extensibility, but it does not implement the issue’s central request: secure-by-default tiered presets plus plugin/skill process isolation, IPC-proxied SDK calls, manifest env/fs/net/resource capability enforcement, and credential scoping.

## Best Possible Solution

Keep #12505 open as the canonical sandbox/isolation architecture tracker unless maintainers split it into narrower issues. The best path is core design for secure-by-default tiered sandbox presets, plugin/skill process isolation or a safer bundle-first execution path, manifest-declared env/fs/net/resource policy, credential scoping, and an IPC/proxy SDK boundary. Treat PR #29974 and current Docker/SSH/OpenShell backend work as partial opt-in tool-sandbox progress, not closure.
