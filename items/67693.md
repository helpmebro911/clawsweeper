## Summary

Keep this PR open. Current main still forwards wizard select and multiselect labels directly into Clack without a blank-label fallback, and the current test file only covers tokenized filtering. The related onboarding crash work in the discussion addresses channel metadata, higher-level trim guards, or text prompt results rather than this select/multiselect adapter path.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to land a narrow label-normalization helper in src/wizard/clack-prompter.ts with unit coverage for select, multiselect, and preferably searchable paths, while clarifying the exact crash claim or reframing the PR as defensive fallback/display hardening if no crash repro is available.
