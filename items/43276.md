## Summary

Keep this PR open. Current main still injects the blanket group-chat instruction that forbids same-group message-tool sends, while the outbound send path already supports media payloads and channel auto-thread resolution. The PR is a plausible narrow fix, but it needs a rebase because current main now has existing prompt tests and fixtures asserting the old wording, plus prompt-injection hardening that the PR's older added tests should not undo.

## Best Possible Solution

Keep this PR open and ask for a rebase onto current main. The maintainer path is to update buildGroupChatContext so normal text replies remain direct while files/images/attachments may use the message tool targeting the same group, update the existing groups.test.ts assertion and group-intro prompt fixture, preserve the current exclusion of untrusted GroupSubject/GroupMembers from the persistent group context, then run the targeted auto-reply tests and the changed gate before landing.
