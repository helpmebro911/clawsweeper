## Summary

Keep this issue open. Current main still renders the Control UI Logs view inside `.log-stream`, and the active CSS still caps that scroll area at fixed pixel heights. #53916 is an open, unmerged implementation candidate with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, refresh, and land #53916 if its CSS approach is acceptable. An equivalent fix should make the Control UI Logs `.log-stream` height respond to available viewport space while preserving usable behavior on shorter screens; after the fix merges, this issue can close as implemented.
