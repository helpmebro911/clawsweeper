## Summary

Current main does not implement the requested Chrome OptGuide/WasmTts model-download suppression. The managed launcher still disables only Translate and MediaRouter via `--disable-features`, the sandbox browser entrypoint has no `--disable-features` argument at all, and there is no dedicated `browser.disableModelDownloads`-style config surface. The related PR #41213 explicitly targeted this issue but was closed unmerged, so it does not make the issue obsolete.

## Best Possible Solution

Keep this issue open. The likely maintainer path is a focused browser-plugin/sandbox fix that centralizes the Chrome disabled-feature list, adds the OptGuide/WasmTts model-download features for managed headless/server and sandbox launches, updates the sandbox/browser docs, and adds regression coverage around `buildOpenClawChromeLaunchArgs` plus a lightweight check for the sandbox entrypoint defaults.
