## Summary

Current main still has the reported gap. Ping-pong detection can warn on alternating tools, but critical blocking still requires stable no-progress result hashes, the global circuit breaker is scoped to the current tool/signature, and enforcement blocks only the current invocation rather than both tools in the pair.

## Best Possible Solution

Keep this issue open. A maintainer fix should make tool-loop protection pair-aware: either allow sustained A/B alternation to escalate without requiring identical per-side result hashes, add a pair-scoped threshold before the per-tool global breaker, or persist short-lived session-level blocks for both tools/signatures when a critical ping-pong or pair breaker trips. Add regression coverage for variable-result memory_search <-> memory_get alternation and for blocking both sides of the offending pair.
