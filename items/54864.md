## Summary

Keep this issue open. Current main has the lower-level `--session-id` isolation behavior, but it still does not implement the requested `openclaw agent --new-session` CLI flag or generated `sessionId` JSON behavior.

## Best Possible Solution

Implement this as a narrow core CLI feature: add `--new-session` to `openclaw agent`, plumb `newSession` through `AgentCliOpts`, reject it with `--session-id`, generate one `crypto.randomUUID()` in `agentCliCommand`, pass that id as the effective `sessionId` through gateway/local/fallback paths, include the generated id as a top-level `sessionId` in `--json` output, and update parser, gateway/embedded JSON, session-resolution, and docs coverage.
