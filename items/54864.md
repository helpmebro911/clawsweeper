## Summary

Keep this issue open. Current main still does not register or document `openclaw agent --new-session`, does not plumb a `newSession` option through `AgentCliOpts`, does not generate a CLI-side UUID before choosing gateway versus embedded execution, and does not add a generated top-level `sessionId` to `--json` output. Main does include the lower-level `--session-id` isolation fix, so the prior related PR remains useful prior art but the requested CLI feature is still missing.

## Best Possible Solution

Implement a narrow core CLI feature in the existing `openclaw agent` path: add a `--new-session` boolean option, reject it with `--session-id`, generate one `crypto.randomUUID()` in `agentCliCommand`, pass that id as the effective `sessionId` through both gateway and embedded paths, include the generated id as a top-level `sessionId` in `--json` output, and update parser/session-resolution/JSON-output tests plus the public agent CLI and agent-send docs.
