## Summary

Keep open. Current main has fixed part of the lower-level `--session-id` isolation behavior, including `--agent` plus `--session-id`, but it still does not expose the requested `openclaw agent --new-session` flag, generate the fresh id in the CLI path, or return that generated id as a top-level JSON field. The related PR #69108 remains closed and unmerged, so it is an implementation reference rather than a superseding item.

## Best Possible Solution

Implement a narrow core CLI feature in the existing `openclaw agent` path: add a `--new-session` boolean option, reject it together with `--session-id`, generate one `crypto.randomUUID()` in `agentCliCommand` before choosing gateway versus `--local`, pass that id as the effective `sessionId` to both paths, surface it as a top-level `sessionId` in `--json` output, and update CLI docs plus parser/session-resolution tests. Preserve the current default behavior where no `--session-id` or `--new-session` reuses the agent main session.
