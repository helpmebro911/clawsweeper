## Summary

Keep open. Current `main` still installs the native Windows gateway as a per-user, interactive Scheduled Task (`/SC ONLOGON`, `/RL LIMITED`, and usually `/RU <user> /NP /IT`) with a per-user Startup-folder fallback. Docs and tests confirm that user-session-oriented model. There is evidence of Windows gateway robustness fixes, but not strong proof that the reported disconnect-after-closing-PowerShell behavior is fully resolved on current `main`.

## Evidence
