## Summary

Keep open. Current main has useful mitigations for controlled restarts and a narrow main-session recovery path, but it does not fully implement the requested persistent streaming state, status tracking, or guaranteed post-restart delivery/resume for interrupted assistant generation. The related #40512 context is a different thinking-block replay problem, not a superseding tracker for this broader recovery feature.
