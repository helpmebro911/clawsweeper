## Summary

Keep #39349 open. Current main still disables an explicit `pty:true` request whenever exec routes through the sandbox, and I found no current warning branch, regression test, docs update, merged duplicate, or shipped release evidence that makes this PR obsolete. The linked #38601 remains the same unresolved core exec UX bug.

## Best Possible Solution

Keep #39349 open or land an equivalent narrow core exec fix: push a user-facing warning onto the existing exec `warnings` path when `params.pty === true && sandbox`, verify it appears in both running and foreground result text, add regression coverage for the sandbox and non-sandbox cases, and optionally clarify the public exec docs at `https://docs.openclaw.ai/tools/exec`.
