## Summary

Keep open. Current main still sends `sessions_send` target turns as generic `inter_session` messages, still prompts the A2A reply/announce steps with `REPLY_SKIP` and `ANNOUNCE_SKIP`, and still suppresses final visible delivery when the announce step returns `ANNOUNCE_SKIP`. I did not find a typed delivery-intent mode, a prompt carve-out for explicit outbound delivery requests, or a regression test for the WhatsApp case. Related #43848 overlaps the sessions_send skip behavior, but it is not a clear canonical replacement for this WhatsApp explicit-delivery issue.
