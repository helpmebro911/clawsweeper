## Summary

Keep PR #72348 open. Current main still advertises Nextcloud Talk reactions and contains the HTTP reaction sender, but the channel does not register a message action adapter, so the shared message action dispatch path cannot reach `sendReactionNextcloudTalk`. The linked bug #70110 and this PR’s proposed adapter/tests are directly relevant, not obsolete or superseded.

## Best Possible Solution

Keep PR #72348 open for maintainer review and CI. The best path is to land a focused Nextcloud Talk `ChannelMessageActionAdapter` that exposes `send`/`react`, forwards `react` to `sendReactionNextcloudTalk`, and keeps the new adapter and HTTP sender tests. A later generic contract test for channels advertising `capabilities.reactions` would be useful, but it is not required to keep this PR actionable.
