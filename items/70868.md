## Summary

Keep open. Current main has a new downstream mitigation for zero-token empty Claude `stop` turns, but #70868 reports a `before_prompt_build` + `prependSystemContext` interaction on Anthropic models and does not include usage data. The implemented guard does not prove that hook path is deterministic or that nonzero-usage empty `stop` turns are fixed.

## Best Possible Solution

Keep #70868 open. Add a focused regression harness for a plugin that returns `prependSystemContext` from `before_prompt_build`, then verify the Anthropic/mock runtime behavior for empty `stop` responses with realistic usage. If the model can still return an empty stop, broaden the guard or halt/retry before persistence, and keep replay repair as a downstream safety net.
