## Summary

Keep #71575 open. Current main still deliberately renders Telegram tool-progress preview updates by default when preview streaming is active: it builds a visible `Working…` draft preview, appends tool names/progress text/command summaries, and only offers `streaming.preview.toolProgress: false` or `streaming.mode: off` as opt-outs. The related #71663 is closed, but current source, tests, and docs show the broader requested behavior, “only final assistant reply,” is not implemented on main.

## Best Possible Solution

Keep the issue open for maintainer product/behavior work. The likely fix is in the Telegram preview streaming path: either make Telegram hide tool-progress previews by default, add a stricter final-only delivery mode for Telegram sessions, or sanitize/summarize progress so raw tool output never appears. Update `extensions/telegram/src/bot-message-dispatch.ts`, the Telegram dispatch tests, and the Telegram/streaming docs together.
