## Summary

Current main still intentionally shows Telegram tool-progress preview edits by default during active preview streaming. The issue is recent, unprotected but still reproducible from source/tests/docs, and the related closed #71663 context supports this as the canonical remaining Telegram final-only behavior request rather than a close candidate.

## Best Possible Solution

Keep this issue open for a Telegram-owned behavior change. The strongest path is to add or switch Telegram to a final-only/default-hidden tool-progress mode in extensions/telegram/src/bot-message-dispatch.ts, preserve an explicit opt-in for users who want live progress previews, and update the Telegram dispatch tests plus docs/channels/telegram.md and docs/concepts/streaming.md together.
