## Summary

Keep #71575 open. Current main still intentionally shows Telegram tool-progress preview lines by default, and the current event path can feed command partial output into those visible preview edits. The related closed #71663 is narrower around partial streaming/tool names and does not supersede this broader report while #71575 remains the open canonical item.

## Best Possible Solution

Keep the issue open as the canonical Telegram leak/default-visibility bug. The best path is to decide whether Telegram direct chats should default to final-only delivery or keep progress previews; then either default `streaming.preview.toolProgress` to false for Telegram or scrub the progress pipeline so previews contain only safe high-level status, never raw command/tool output. Add regression coverage in `extensions/telegram/src/bot-message-dispatch.test.ts` and the Pi tool event tests for exec partial output and duplicate progress events.
