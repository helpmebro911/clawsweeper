## Summary

Keep #17931 open. Current main still does not implement automatic read-only per-skill bind mounts. The #22669-style overlapping custom bind behavior exists as a manual workaround, but sandbox setup still uses writable workspace exposure in `rw` and copy-based skill staging in non-`rw` modes.

## Best Possible Solution

Keep the issue open until OpenClaw either implements automatic read-only handling for eligible skill directories in sandboxed sessions or explicitly rejects that product/security direction. A robust fix should derive deterministic eligible skill source directories, add read-only overlays after broader workspace mounts for `rw`, avoid writable copied skill state where source mounts are available for `ro`/`none`, update fs-bridge/config-hash/docs, and add tests for all three `workspaceAccess` modes while treating #22669 as only a manual workaround.
