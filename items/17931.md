## Summary

Keep open. Current main still mirrors eligible skills into writable sandbox workspace copies for `workspaceAccess: "ro"`/`"none"`, and `workspaceAccess: "rw"` still exposes workspace skills through the writable workspace mount. Docker setup now appends custom binds after workspace mounts, which addresses the related #22669 ordering bug, but there is still no automatic per-skill read-only bind mount behavior requested by #17931.
