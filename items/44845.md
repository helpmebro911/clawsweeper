## Summary

Keep open. The issue body's proposed root cause is outdated because current main already normalizes `prompt_tokens` / `completion_tokens`, but the underlying Volcengine bug does not appear to be implemented away on `main`. Volcengine coding models are still plain `openai-completions` models with the Volcengine coding base URL and no Volcengine-specific compat override, the Volcengine host still falls through to the generic `custom` endpoint class, and the OpenAI completions transport still unconditionally sends `stream_options.include_usage`. If Volcengine omits usage in stream chunks/final chunks, `/status` can still stay at `0/<window>`.
