## Summary

Keep #17761 open. Current main still handles `trusted-proxy` as an exclusive auth path: proxy-auth failure returns before shared-secret or Tailscale auth can run, token + trusted-proxy is rejected by startup validation, local direct token tests expect rejection, and the docs describe the fail-closed contract. The original fallback PR #17746 is closed unmerged, while #64122 is still open related work rather than a shipped fix.

## Best Possible Solution

Keep #17761 open until maintainers either land an intentional trusted-proxy shared-secret fallback or explicitly decide that the current fail-closed contract is final. A complete fix should coordinate `src/gateway/auth.ts`, credential resolution/planning, startup validation, regression/e2e tests, and `docs/gateway/trusted-proxy-auth.md`; #64122 is the current related implementation thread to continue or supersede.
