## Summary

Keep #17761 open. Current main still treats `trusted-proxy` as an exclusive fail-closed auth path: proxy-auth failure returns before Tailscale/token/password auth, mixed trusted-proxy plus token startup config is rejected, local-direct token tests expect rejection, and docs describe the fail-closed contract. The linked fallback PR #17746 is closed unmerged, while #64122 is still open related work rather than a shipped fix.

## Best Possible Solution

Keep #17761 open until maintainers either land an intentional trusted-proxy shared-secret fallback or explicitly decide the current fail-closed contract is final. A complete fix should coordinate `src/gateway/auth.ts`, `src/gateway/auth-resolve.ts`, `src/gateway/credential-planner.ts`, client credential resolution, regression/e2e tests, and `docs/gateway/trusted-proxy-auth.md`; #64122 is the current related implementation thread to continue or supersede.
