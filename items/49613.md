## Summary

Current main still has the reported behavior: heartbeat enablement is based on per-agent heartbeat presence or the default agent fallback, so an `agents.defaults.heartbeat` block does not enable heartbeat for a non-default agent that lacks `agents.list[].heartbeat`. The existing regression test also asserts `ops` is disabled for the exact defaults-only multi-agent shape from the issue. The related fix PRs in the provided context were closed/unmerged, so this remains an actionable core bug rather than something to close.

## Evidence
