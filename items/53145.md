## Summary

Keep this issue open. Current main still routes peer `sessions_send` fire-and-forget runs into the A2A announce path, still uses the literal announce placeholder as an inter-session user input, and still only treats `REPLY_SKIP`/`ANNOUNCE_SKIP` as A2A control tokens rather than swallowing exact `NO_REPLY` before peer delivery. The existing discussion and prior ClawSweeper analysis remain consistent with current source; related #44213 is cron-specific and closed, while #61433 is a broader raw-metadata hardening issue rather than a canonical replacement for this peer `sessions_send` bug.

## Best Possible Solution

Keep this issue open. The best fix belongs in `src/agents/tools/sessions-send-tool.a2a.ts` and adjacent helpers: apply the shared silent/control-reply policy before ping-pong and announce delivery, treat exact `NO_REPLY` as silent in this path, avoid submitting the announce placeholder as a normal inter-session user turn, prevent acknowledgement/follow-up text from being reinjected as peer input, and add focused regression coverage for `sessions_send({ timeoutSeconds: 0 })` covering placeholder suppression, `NO_REPLY` suppression, and no echo loop.
