## Summary

Keep PR #70739 open. Current main still lacks the proposed SSE heartbeat for the core `/v1/chat/completions` and `/v1/responses` streaming gateway endpoints, while the repo already has a session-history SSE keepalive precedent. The PR is a focused core gateway fix, not a ClawHub/plugin item or duplicate, though its review comments should be addressed before merge.

## Best Possible Solution

Keep the PR open and either merge this approach after addressing the remaining review comments, or land an equivalent core gateway implementation: a shared, backpressure-aware, error-safe SSE comment heartbeat wired into the OpenAI-compatible streaming endpoints, with focused unit tests and no new public schema/config surface unless maintainers explicitly want one.
