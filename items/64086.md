## Summary

Keep open. Current main still lacks a public `PluginStatusProvider` / `registerStatusProvider` API and the TUI footer is built from fixed core fields. The issue also has open PR #72557 by the same author with resolving syntax, so the right maintainer path is to review that implementation candidate rather than close this issue early. The ContextClaw discussion is a concrete plugin-author use case, and #53774 remains the broader footer-customization tracker.

## Best Possible Solution

Keep this issue open. Review #72557 as the implementation candidate for the public `PluginStatusProvider` / `registerStatusProvider` seam, decide whether it should include TUI footer rendering in the same PR, and coordinate the UX details with #53774. Close only after merged main gives ContextClaw and other plugins a supported way to contribute TUI footer status without monkeypatching.
