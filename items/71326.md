## Summary

Keep open. The issue is recent, concrete, and the current main code still matches the reported gateway-host execution shape: each gateway exec run builds a fresh shell argv from `getShellConfig()` and spawns it through the process supervisor. I found no current-main mitigation for APFS stale cross-exec reads, no stale-read regression coverage, and no canonical duplicate/superseding item in the provided related context.

## Best Possible Solution

Keep #71326 open. The best path is to add a macOS/APFS reproduction or diagnostic harness for gateway-host `security=full` + `ask=off`, instrument the exec/preflight path around shell spawning and file reads after atomic rename, then fix the narrow execution/read-coherence issue if confirmed. Candidate fixes should preserve exec approval/security behavior and include regression coverage where feasible.
