## Summary

Keep open. The issue is recent, detailed, and targets OpenClaw core gateway-host exec behavior. Current main still skips the Python/Node script preflight in gateway YOLO mode, then runs each host exec through a shell child process; I found no current source, docs, tests, release note, or post-v2026.4.24 diff that adds a stale-read/cache-coherence mitigation or supersedes the report.

## Best Possible Solution

Keep the issue open for maintainer investigation. The next useful path is a focused macOS/APFS repro against v2026.4.19, v2026.4.20, v2026.4.24, and current main using gateway-host exec with security=full and ask=off. If confirmed, the fix likely belongs in the core host-exec/runtime path or in a documented gateway-host filesystem consistency diagnostic, with regression coverage around atomic rename followed by reads from separate exec calls.
