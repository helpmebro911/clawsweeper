## Summary

Keep open. #71326 is recent, detailed, and reports a macOS/APFS gateway-host exec regression that is not resolved by current main. Current code still runs gateway-host exec commands through a freshly spawned shell child, and the 2026.4.20 YOLO/script-preflight change is present and covered by tests/docs, but I found no implementation, test, or docs path that addresses cross-process stale file reads, F_NOCACHE-style mitigation, persistent exec reuse, or read-after-rename coherence. The provided related items are adjacent stale/preflight reports, not a canonical duplicate for this filesystem-coherence symptom.

## Best Possible Solution

Keep #71326 open for maintainer investigation. The next useful path is a macOS reproduction harness that writes via atomic rename and reads through repeated OpenClaw gateway-host YOLO exec calls on current main and v2026.4.20, with instrumentation around shell argv, process ids, inode/mtime/size, and cron/write provenance. If reproduced, the fix should land in the host exec/runtime or a documented macOS diagnostic/mitigation path, with targeted regression coverage; do not close it as the already-shipped YOLO preflight unblock because that solves a different problem.
