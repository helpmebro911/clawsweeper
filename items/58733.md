## Summary

Keep open. Current `main` still sends bundled Grok `web_search` through the shared 30-second default timeout, and there is no xAI-specific default or plugin-local `webSearch.timeoutSeconds` schema entry that would indicate this was fixed on `main`. The generic `tools.web.search.timeoutSeconds` setting exists as a workaround, but that does not resolve the reported provider-default bug.

## Evidence

- **Grok web_search still uses shared default timeout:** `executeXaiWebSearchProviderTool(...)` passes `resolveTimeoutSeconds(searchConfig?.timeoutSeconds, DEFAULT_TIMEOUT_SECONDS)` into `runXaiWebSearch(...)`, so the xAI web_search path still inherits the shared fallback instead of a longer provider-specific default.
