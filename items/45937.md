## Summary

Keep #45937 open. Current main does not yet classify the supplied MiniMax Chinese rate-limit message as `rate_limit`, and MiniMax's bundled provider registration does not provide a provider-owned `classifyFailoverReason` hook. The PR should not land as-is because review comments identify real false-positive risks in the generic `请稍后重试` pattern and the non-error assistant text path, but the underlying user problem remains actionable.

## Best Possible Solution

Keep the PR open, but request a narrower implementation: add MiniMax-specific failover classification in `extensions/minimax/provider-registration.ts` via the existing provider `classifyFailoverReason` hook, include tests for the exact 2062 Chinese message, and avoid standalone matching for generic `请稍后重试` or arbitrary non-error assistant text unless it is scoped to an explicit provider error payload.
