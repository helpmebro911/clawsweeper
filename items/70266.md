## Summary

Keep open. Current `main` still renders the macOS Talk Mode overlay as the generic orb, and the macOS talk-config/state path only carries seam color, not assistant avatar identity. The reporter's follow-up is correct that existing Control UI identity work does not cover this macOS Talk Mode surface.

## Evidence

- **Talk overlay still hardcodes orb rendering:** `TalkOverlayView` always instantiates `TalkOrbView`, and `TalkOrbView` renders only `Circle()`-based gradients/rings/arcs for paused, listening, speaking, and thinking states. There is no avatar/image branch in the overlay view.
  - file: [apps/macos/Sources/OpenClaw/TalkOverlayView.swift:13](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/apps/macos/Sources/OpenClaw/TalkOverlayView.swift#L13)

  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)

  - file: [apps/macos/Sources/OpenClaw/TalkModeGatewayConfig.swift:15](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/apps/macos/Sources/OpenClaw/TalkModeGatewayConfig.swift#L15)
