## Summary

Current `main` does not implement the ACP token-only auth flag or any equivalent ACP opt-out for device identity, so this PR is not stale/obsolete and should remain open.

## Evidence

- **ACP server still always uses default GatewayClient device-identity behavior:** `serveAcpGateway()` constructs `GatewayClient` without any `deviceIdentity` override, so ACP cannot request the explicit null path that disables auto-loading.
  - file: [src/acp/server.ts:55](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/acp/server.ts#L55)

- **ACP CLI exposes no device-identity opt-out flag:** The `openclaw acp` command registers `--url`, `--token`, `--password`, session flags, `--no-prefix-cwd`, and `--provenance`, but nothing like `--skip-device-identity` / `--no-device-identity`.
  - file: [src/cli/acp-cli.ts:15](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/cli/acp-cli.ts#L15)
