## Summary

Current `main` only overlaps with part of PR #51362. The 404 failover behavior is already implemented on `main`, but the session-store clone reduction, the startup hook-config fallback, and the 15s/clamped handshake-timeout change are not. The hook fallback also has a real correctness risk against current startup wiring, so this is not a safe cleanup close.
