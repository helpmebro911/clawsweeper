## Summary

Keep #70308 open for maintainer review rather than auto-closing. The PR is not implemented on current main, and the proposed `src/infra/process-respawn.ts` gateway hunk still does not affect Windows because current main returns before the detached spawn on `win32`. Current main already has `windowsHide: true` on several actual Windows scheduled-task/restart paths, but the linked regression #70238 remains open in the provided context and this PR also touches adjacent CLI/TUI spawn paths that are not present on main. That leaves a real, salvageable bug-fix discussion rather than a high-confidence cleanup close.

## Best Possible Solution

Keep the PR open and request changes or maintainer triage. The best path is to either patch a reachable Windows path that actually reproduces #70238, with a narrow mocked regression test asserting `windowsHide` on that path where feasible, or explicitly retarget/close the PR in favor of the canonical #70238/#54120 discussion if maintainers determine current scheduled-task and restart-helper coverage already solves the gateway flicker. The defensive `process-respawn.ts` addition alone should not be treated as a gateway bug fix because it is unreachable on Windows today.
