## Summary

Keep #21207 open. Current main has adjacent groundwork (`parallel_tool_calls`, context-engine lifecycle/compaction, `update_plan`, and strict-agentic planning-only retry handling), but it still does not implement the requested native batched file/search tools, AI-callable named context block save/load/unload/list tools, or stateful plan submit/done/status enforcement API.

## Best Possible Solution

Keep #21207 open and split it into concrete maintainer-owned tracks: native or gateway-level batching for file/search tools; an explicit AI-callable context block API if named save/load/unload semantics are desired; and runner-owned plan state with opt-in strict gating if enforceable sequential execution is a product goal. Treat current `parallel_tool_calls`, context-engine compaction, strict-agentic retries, and `update_plan` as useful groundwork, not closure.
