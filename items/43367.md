## Summary

Keep open. Current main has partial mitigations for OAuth refresh races, failover classification, task visibility, and cancellation surfaces, but this broad multi-agent reliability report is not fully implemented. The config-write race is still handled by optimistic conflict detection/atomic rename rather than serializing the full read-modify-write sequence, session write lock timeouts still exist, and a same-author open PR remains an implementation candidate for the config-write symptom.

## Best Possible Solution

Keep this issue open. First review and either land or close #44997 for serialized config mutation. Then split or link the remaining work into narrower maintainer-owned fixes for per-agent/session run serialization, cancellation and operator visibility for detached CLI work, subagent completion recovery/notification, and any remaining model-failover behavior after local lock failures.
