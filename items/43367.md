## Summary

Keep open. Current main and the supplied discussion show this is still an actionable multi-agent reliability report: #44997 is an open same-author implementation candidate with closing syntax for the config-write symptom, and the remaining session-lock/subagent delivery behavior is only partially mitigated. OAuth refresh serialization appears mostly implemented, but that resolves one symptom rather than the full orchestration problem.

## Best Possible Solution

Keep this issue open until #44997 is reviewed and either merged or closed. Then maintainers should split or continue the remaining work: serialized config mutation for `agents add`, robust per-agent session-run queue/lock behavior, CLI/gateway cancellation and visibility for detached child work, and subagent completion delivery/recovery. Treat the OAuth refresh serialization on main as one resolved symptom, not a reason to close the broader orchestration report.
