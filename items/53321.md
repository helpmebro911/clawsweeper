## Summary

Keep open. Current main has long-session mitigations, including preflight compaction, memory flush, optional successor transcript rotation, queued-followup retargeting, and context-overflow reset recovery. It still does not implement the requested first-class configurable auto session split policy with token/window trigger config, invisible rollover semantics, active-tool-call boundaries, and continuity payload handling for active tasks or pending confirmations.

## Best Possible Solution

Keep this issue open as the canonical core product/design tracker. Build on current preflight compaction, memory flush, successor transcript rotation, and queued-followup retargeting, then add a supported session.autoSplit-style policy with configurable token/window thresholds, active run/tool-call boundaries, session-key routing updates, continuity payload handling for summaries/tasks/pending confirmations, docs, and focused regression tests.
