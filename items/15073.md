## Summary

Keep #15073 open. Current main still supports per-agent model fallback lists, but fallback execution remains a provider/model retry inside the same active agent session, workspace, bootstrap context, skills snapshot, and transcript. The related closed issues cover narrower context-budget/profile ideas and do not replace the broader fallback-agent/workspace request; the provided discussion also includes a same-day maintainer/Codex comment keeping this issue open.

## Best Possible Solution

Continue tracking #15073 until OpenClaw either adds explicit fallback-agent/workspace/context-profile semantics or a maintainer chooses a narrower model-specific bootstrap/context-budget design as the canonical product direction. A real fix likely needs agent config schema changes, fallback candidate metadata beyond provider/model, retry plumbing that can resolve a fallback agent/workspace without corrupting the original session, bootstrap/context assembly updates, docs, and regression tests proving the fallback uses the intended agent context.
