## Summary

Keep #15073 open. Current main still has model fallback as provider/model retry within the active run, while the requested fallback-agent/workspace/context-profile behavior is absent from config types, strict schemas, runtime fallback candidates, docs, and tests. The issue discussion also already includes a 2026-04-26 maintainer/Codex comment explicitly keeping it open; related #7351, #13529, and #13557 are narrower closed context-budget/profile ideas, not an implementation of this request.

## Best Possible Solution

Keep #15073 open until maintainers choose and implement a first-class design: either fallback-agent/workspace/context-profile semantics or a narrower model-specific context-budget/profile mechanism. A complete fix likely belongs in core config schema/types/docs, fallback candidate metadata, retry plumbing, bootstrap/context assembly, and regression tests proving fallback retries use the intended alternate context without corrupting the original session.
