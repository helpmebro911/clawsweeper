## Summary

Keep #72202 open. The report is recent, specific, and not safely closeable from current main: the OpenAI image provider has a no-Codex-profile path that should use the direct Images API, but there is no exact Windows/config-env regression proof showing the reported `openclaw.json` env timing issue is fixed. The related #70703 work is shipped, but it covers adding Codex OAuth image generation and intentionally prefers Codex when an `openai-codex` profile is active, which is distinct from this reporter's no-Codex-profile failure.

## Best Possible Solution

Keep the issue open and add a focused regression around `image_generate` on a config with `env.vars.OPENAI_API_KEY`, `agents.defaults.imageGenerationModel.primary = "openai/gpt-image-2"`, and an auth store containing no `openai-codex` profiles, ideally with Windows-path/startup coverage. If the repro still routes Codex, fix the auth/env ordering or profile detection in the OpenAI image provider path; if provider-level config persistence is involved, link or split the config-overwrite piece to #44786 rather than closing this image-route report.

## Evidence

- **Issue report is concrete and recent:** #72202 reports Windows 11 on OpenClaw 2026.4.24, `OPENAI_API_KEY` configured via the `openclaw.json` env section, no `openai-codex` OAuth profile, and an observed Codex Responses HTTP 400 while using `openai/gpt-image-2`. The follow-up explicitly links #70703 and raises a Windows env-load ordering hypothesis rather than a vague stale failure.
- **Current route selection still has distinct Codex and direct paths:** Current main selects the Codex Responses route only when the base URL is the public OpenAI API, no explicit direct OpenAI provider config is present, and `hasCodexOAuthProfileConfigured(req)` is true; otherwise it resolves `provider: "openai"` and sends direct requests to `/images/generations` or `/images/edits`. That source shape does not explain the reporter's 'no Codex OAuth profile' Codex route and needs targeted reproduction before closure.
