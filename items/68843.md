## Summary

Keep PR #68843 open. Current main still has the reported ACP stale-binding gap: dispatch cleanup only recognizes metadata/resource-not-found init failures, and reset-in-place failures return the error without unbinding the stale conversation. The PR is a focused replacement for closed, unmerged #60720 and the bot review's only style finding was addressed in the PR's second commit.

## Best Possible Solution

Keep #68843 open for maintainer review and eventual landing if CI/changed gates pass. The best fix is the PR's narrow shape: share the ACP stale-session predicate and unbind reason, add the missing-working-directory phrase, apply it to both dispatch and reset-in-place cleanup, and keep regression tests for both paths while preserving the generic init-failure no-unbind behavior.
