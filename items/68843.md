## Summary

Current main still does not implement the PR's missing-cwd stale ACP binding cleanup. The existing dispatch stale-session classifier only matches missing metadata / resource-not-found style init failures, and reset-in-place returns the close/init failure without unbinding stale conversation bindings. The open PR remains a plausible focused bug fix and replacement for the earlier closed unmerged #60720.

## Best Possible Solution

Keep this PR open, rebase if needed, and review/land the focused ACP stale-binding fix after changed-lane validation. The implementation should keep the classifier narrow, share the stale-unbind reason, and preserve both dispatch-time and reset-in-place regression coverage.
