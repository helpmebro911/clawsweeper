## Summary

Keep PR #72085 open. The docs gap is real on current main: `commands.bashForegroundMs` is bounded/clamped at 0..30000 ms in code and schema, but the human-facing docs/help text still omit the upper bound. The PR is not merge-ready as submitted because the patch adds a large duplicated block to `docs/gateway/configuration-reference.md` for a one-bullet docs fix.

## Best Possible Solution

Keep the PR open and request a narrow docs cleanup: update `docs/tools/slash-commands.md` and likely `src/config/schema.help.ts` plus generated schema docs to state `commands.bashForegroundMs` is an integer from 0 to 30000 ms, default 2000, with 0 backgrounding immediately. Optionally add the same range to the command details in `docs/gateway/config-channels.md`. Avoid duplicating channel/config sections inside `docs/gateway/configuration-reference.md`.
