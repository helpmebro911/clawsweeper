## Summary

Keep PR #72085 open. Current main still has a real documentation gap: `commands.bashForegroundMs` is bounded to 0..30000 ms in source/schema, but the user-facing Slash Commands and channel config docs only show the default and `0` backgrounding behavior. The submitted PR is not merge-ready because the provided PR metadata shows a much larger one-file docs diff than the stated narrow bullet, but the requested docs fix is still useful and belongs in this repo.

## Best Possible Solution

Keep the PR open and request a narrow docs cleanup. The best fix is to update `docs/tools/slash-commands.md` and the schema help source/generated config docs so `commands.bashForegroundMs` is documented as an integer range `0` to `30000` ms, default `2000`, with `0` backgrounding immediately. Optionally add the same concise range note to `docs/gateway/config-channels.md`. Avoid duplicating channel/config reference content inside `docs/gateway/configuration-reference.md`.
