## Summary

Keep PR #72085 open. The underlying docs gap still exists on current main: runtime and schema enforce/clamp `commands.bashForegroundMs` to 0..30000 ms, but user-facing docs and generated help text still only mention the default and `0` behavior. The submitted PR is not merge-ready as-is because the provided GitHub context shows a one-file docs patch with 2693 additions for a one-bullet change, but that is a cleanup request rather than a close reason under this conservative review.

## Best Possible Solution

Keep the PR open and ask for a narrow docs cleanup: add the 0..30000 ms range/default wording to `docs/tools/slash-commands.md` and the source help in `src/config/schema.help.ts` with regenerated schema metadata if required, and consider adding the same concise note near the Commands example in `docs/gateway/config-channels.md`. Avoid duplicating large sections in `docs/gateway/configuration-reference.md`.
