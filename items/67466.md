## Summary

Keep this PR open for maintainer rework. Current main still lacks the focused Feishu setup/status probe timeout cap, and the PR is not safe to merge as-is because it detaches gateway-managed Feishu monitors from the long-lived lifecycle promise that the gateway tracks.

## Best Possible Solution

Keep this PR open but request a narrower rework: preserve the long-lived `monitorFeishuProvider({ accountId })` return contract, coordinate the multi-account bot-info probe behavior with #63481/#63475 or an equivalent lifecycle-safe queue, and extract the Feishu setup/status 3000 ms probe cap with focused regression coverage.
