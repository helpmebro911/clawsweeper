## Summary

Keep this PR open for maintainer rework, but do not merge it as-is. Current main still lacks the PR's Feishu setup/status probe timeout cap, and the startup-latency area remains active. The PR diff also changes the gateway-used single-account Feishu monitor path to fire-and-forget, which conflicts with the gateway lifecycle contract and can cause restart loops or detached background monitors.

## Best Possible Solution

Keep this PR open for maintainer judgment, but require rework before merge. Preserve the long-lived single-account `monitorFeishuProvider({ accountId })` promise for gateway lifecycle tracking, let #63481 or an equivalent queue-based change handle the multi-account startup probe serialization tracked by #63475, and split the Feishu setup/status probe timeout into a small lifecycle-safe patch with focused tests.
