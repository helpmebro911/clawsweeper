## Summary

Current `main` does not implement this PR's multi-workspace hook loading. Gateway startup still loads internal hooks exactly once for `defaultWorkspaceDir`, so workspace-local hooks for non-default agent workspaces are not covered by the current startup path.

## Evidence

- **Gateway startup still uses single-workspace loader:** `startGatewaySidecars` imports `loadInternalHooks` and calls it once with `params.defaultWorkspaceDir`. There is no startup path here that enumerates all configured agent workspaces.
