## Summary

Current main still has the reported runtime behavior: missing `agents.defaults.contextInjection` resolves to `always`, tests and docs lock in that default, `MEMORY.md` remains part of normal bootstrap injection, and there is no `bootstrapDynamicFiles`/`bootstrapFileOrder` configuration surface. The maintainer-linked umbrella #69208 explicitly says concrete reports with unique evidence should stay open, and the current open closing PRs are unmerged implementation candidates, not grounds to close this issue.

## Best Possible Solution

Keep this issue open. The best product path is to decide the default policy for `agents.defaults.contextInjection`, then implement that decision in `src/agents/bootstrap-files.ts`, config schema/help/docs, and regression tests. Treat the `MEMORY.md` stub/lazy-load behavior and configurable dynamic bootstrap-file ordering as either scoped follow-up work or canonical links to #65438. Review or close the open README-only closing PRs rather than letting them resolve this runtime bug by closing syntax alone.
