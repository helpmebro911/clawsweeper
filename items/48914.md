## Summary

Keep open. Current main still classifies unavailable built-in tool groups such as `group:memory` through the tool-ID-only warning path, so the reported misleading plugin-only warning is not implemented away. The PR targets the right code path and has a real user log in the discussion, but it still needs revision before merge because the provided diff regresses the existing warning-dedupe test and has a changelog attribution style issue.

## Best Possible Solution

Keep the PR open and revise it: add a core group-aware classifier such as `isKnownCoreToolOrGroupId` using an own-property check against `CORE_TOOL_GROUPS`, switch the warning classifier to use it, add the focused `group:memory` regression test, preserve the existing warning-dedupe eviction replay, and fix the changelog attribution style before running the targeted tool-policy test lane.
