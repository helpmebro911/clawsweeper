## Summary

Keep this issue open. Current main has useful mitigations, including deterministic heartbeat scheduling, wake coalescing, generic main/subagent lane caps, and per-session serialization, but it still does not implement a heartbeat-origin resource policy for the reported macOS local-gateway load path. Heartbeat admission checks main/session queue pressure, not shared subagent backlog, and subagent work still runs in the same gateway process.

## Best Possible Solution

Keep this issue open as a core gateway and agent-scheduling performance bug. The best maintainer path is to add or validate heartbeat-origin resource policy: cap or serialize subagent spawns initiated by heartbeat turns, account for shared subagent-lane backlog before starting heartbeat runs, expose lane/backlog/drain/event-loop signals, and add a focused regression or stress test with multiple heartbeat-enabled agents spawning background subagents while channel responsiveness remains bounded.
