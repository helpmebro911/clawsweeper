## Summary

Keep open. Current main does not implement the central Feishu hardening in #70471: recipient gating, duplicate final-reply suppression after visible tool delivery, angle-bracket escaping, account-scoped typing state, or fresh credential-aware bot identity probing. The related #66427 is closed/unmerged in the provided context, so it is not a shipped canonical fix.

## Best Possible Solution

Keep #70471 open as the active implementation path, or ask the author/maintainer to split it into narrower Feishu patches if review size is the blocker. The shipped code should add recipient gating before comment-context fetch, record/suppress successful visible comment deliveries without dropping unrelated finals, escape Feishu Drive comment text, scope typing cleanup by account, and refresh bot identity with a credential-aware cache plus safe fallback behavior.
