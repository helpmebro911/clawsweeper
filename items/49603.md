## Summary

Keep open. Current main has a partial acquisition-time mitigation for legacy same-PID lock files that lack a valid `starttime`, but it does not implement the requested gateway-boot sweep for self-owned session locks. New locks on Linux record `starttime`, and the current orphan-self check explicitly refuses to reclaim locks with a valid `starttime`, so the reported same-process/lost-HELD_LOCKS case is not closed with high confidence. PR #49660 is the direct open implementation track for this issue.
