## Summary

Keep this PR open. Current main still lacks the central Grok web_search fixes, but the PR should be rebased and narrowed because it targets an old xAI file layout and mixes unrelated status-command changes into the provider bug fix.

## Best Possible Solution

Keep this PR open as useful implementation work, but ask for a rebase and a narrower patch. The best path is to drop or split the status-command changes, port the Grok work to current `extensions/xai/src/*` files, add null/object guards in the xAI Responses parser, wrap xAI web-search timeout AbortErrors into plain `Error`s with `{ cause }` using `isAbortError()`, make Grok default to 60 seconds while preserving explicit user timeout overrides, and add focused xAI regression tests.
