## Summary

Keep this PR open. Current main still lacks the central xAI/Grok web_search fixes: malformed/null response entries can still crash parsing, Grok still inherits the shared 30s timeout, and internal timeout AbortErrors are not wrapped into structured tool errors. The PR has useful implementation direction, but it needs to be ported to the current xAI plugin layout and separated from unrelated status-command changes.

## Best Possible Solution

Keep this PR open as useful contributor work, but ask for a narrow rebase/port. The maintainer path is to move only the xAI/Grok fixes into `extensions/xai/src/responses-tool-shared.ts`, `extensions/xai/src/web-search-shared.ts`, and `extensions/xai/src/web-search-provider.runtime.ts`; use the shared `isAbortError()` behavior with `{ cause }` when wrapping internal timeout aborts; add a Grok-specific 60s default while preserving explicit user overrides; and add focused xAI regression tests. Drop or split the unrelated `src/commands/status*` changes.
