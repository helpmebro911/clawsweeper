## Summary

Do not close #69086. Current main still has the reported any-assistant session-history guard, and the fallback retry prompt remains hard-coded without a visible config or hook override. The related PR #69109 was closed unmerged and explicitly left the retry-prompt hook request out of scope.

## Best Possible Solution

Keep #69086 open. Land the session-history fix in `src/agents/command/attempt-execution.helpers.ts` with regression tests for user -> assistant -> fresh user returning false and assistant after the latest user returning true, reusing #69109 as prior patch evidence. Then decide a supported operator surface for retry prompt tuning, likely a documented `agents.defaults` config option or a trusted plugin hook, and document it in the model failover/config docs.
