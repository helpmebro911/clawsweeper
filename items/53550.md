## Summary

Keep open. Current main still leaves the reporter's central path unresolved: `agents.defaults.memorySearch.experimental.sessionMemory` enables the builtin session source, but QMD transcript export is still controlled by the separate `memory.qmd.sessions.enabled` switch, and session-sourced `memory_search` hits are still filtered by session-tool visibility. Under the default `tree` visibility, a same-agent gateway session is not visible from an unrelated DM session unless visibility is widened.

## Best Possible Solution

Keep this issue open. The maintainer path is to either bridge `agents.defaults.memorySearch.experimental.sessionMemory` plus `sources: ["sessions"]` into QMD transcript export when `memory.backend` is `"qmd"`, or explicitly diagnose/document that QMD transcript recall requires `memory.qmd.sessions.enabled: true` plus a broader session visibility setting such as `tools.sessions.visibility: "agent"` or `"all"`. Add a regression test with a gateway-style same-agent session key queried from an unrelated DM key so the intended gateway-to-DM recall contract is explicit.
