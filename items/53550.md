## Summary

Keep open. Current main has real session-memory infrastructure, including builtin session indexing, QMD session export when separately enabled, `memory_search corpus=sessions`, and visibility filtering. The reported QMD + gateway-dispatched cross-channel recall path is still not solved by the reporter's config: `agents.defaults.memorySearch.experimental.sessionMemory` does not enable QMD session export, and session hits remain hidden from unrelated same-agent gateway/DM sessions under the default `tools.sessions.visibility=tree`. Related #19913 fixed the inverse ranking problem, while the local related reports for #51386 and #59933 support keeping this broader multi-session continuity/visibility gap open.

## Best Possible Solution

Keep this issue open. A conservative maintainer fix is either to bridge `agents.defaults.memorySearch.experimental.sessionMemory` into QMD session export when `memory.backend="qmd"`, or explicitly document/diagnose that QMD transcript recall requires `memory.qmd.sessions.enabled` plus `tools.sessions.visibility="agent"` or `"all"` for same-agent cross-channel recall. Add a regression test with a gateway-style `agent:<id>:paperclip:issue:<uuid>` transcript and a DM requester such as `agent:<id>:whatsapp:direct:*`.
