## Summary

Keep open. The reported bug is still present on current main: the browser plugin owns the browser tool now, but its registration and createBrowserTool path do not thread config into screenshot image results, so agents.defaults.imageMaxDimensionPx cannot affect browser:screenshot. The PR's original patch targets the old src/agents/tools browser tool path and needs to be ported to extensions/browser.

## Best Possible Solution

Keep the PR open and ask for a port to the current browser plugin architecture: thread ctx.runtimeConfig ?? ctx.config through extensions/browser/plugin-registration.ts into createBrowserTool, resolve image sanitization through an SDK-safe helper or local equivalent, pass it into browser:screenshot and likely labeled browser:snapshot imageResultFromFile calls, and add focused tests in extensions/browser/src/browser-tool.test.ts plus a registration test proving config is forwarded.
