## Summary

Keep #70309 open. Current main supports storing `mcp.servers`, hot-disposes cached MCP runtimes after config changes, and can materialize MCP tools during agent runs, but it does not expose a Control UI MCP connectivity panel with per-server connected/error state, server-scoped tool lists, one-click reconnect/refresh, or structured error details. The related context does not show a canonical duplicate or shipped implementation; the ClawHub-related Swiggy MCP skill item is a different service-specific integration request, not this core operator UI/API gap.

## Best Possible Solution

Keep the issue open. The best path is an additive Gateway operator API for MCP diagnostics, backed by the session MCP runtime/config loader, that reports configured server name, transport summary, connected/error state, listed tools, last redacted error, and a scoped refresh/reconnect action. Then render it under Control UI Infrastructure > MCP and document the public behavior without exposing secrets.
