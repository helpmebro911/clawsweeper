## Summary

Keep open. Current main supports saved MCP server definitions and per-run MCP tool materialization, but it does not provide a generic Control UI MCP connectivity panel, per-server live status, tool-name listing, reconnect/refresh action, or surfaced connection errors for configured `mcp.servers`. The existing Codex `/codex mcp` status is specific to the Codex app-server plugin and does not solve generic OpenClaw MCP server health in Control UI.

## Best Possible Solution

Add a core Gateway read/control surface for generic `mcp.servers` health, likely a scoped `mcp.status`/`mcp.refresh` RPC that loads configured servers, probes or reuses the bundled MCP runtime safely, reports per-server status, tool count/names, and redacted errors, then render it in the Control UI Infrastructure/MCP section with focused tests and docs. Keep Codex app-server `/codex mcp` as plugin-specific status, not the generic solution.
