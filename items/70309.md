## Summary

Keep this issue open. Current main still has MCP config editing and session-time MCP tool materialization, but not the requested Control UI connectivity panel with per-server connected/error state, tool names, manual refresh/reconnect, or surfaced redacted error details.

## Best Possible Solution

Keep this issue open. The best path is an additive Gateway operator diagnostics API, likely under the Gateway protocol/server-methods surface and backed by the MCP config/runtime loader, returning each configured server name, transport summary, connected/error state, tool names/count, and redacted last error, plus a scoped refresh/reconnect action. Then render it in Control UI under Infrastructure > MCP and document the operator behavior.
