## Summary

Keep this issue open. Current main still matches the reported behavior: `embedInterval` only suppresses the dedicated periodic embed timer, while boot, first-run, forced sync, and queued forced update paths can still reach `qmd embed`. The comments and related cleanup history support this issue as the canonical tracker after narrower QMD embed-disable reports were closed or superseded.

## Best Possible Solution

Keep this issue open and implement a documented QMD embed-disable control in `extensions/memory-core/src/memory/qmd-manager.ts` plus the public memory config schema/help/docs. The fix should prevent `qmd embed` from spawning for boot, first-run, manual/forced sync, queued forced sync, session-start/post-compaction, and periodic maintenance when disabled, while preserving `qmd update` and BM25 refresh behavior. Add focused QMD manager tests for disabled embeds across forced and non-forced paths.
