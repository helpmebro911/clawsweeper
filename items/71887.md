## Summary

Keep #71887 open. Current main still has the stale `status --deep` guidance in the `channels status` CLI description and both channel-status footer paths, while `channels status` only registers `--probe`, `--timeout`, and `--json`. The PR is a focused fix for a real current-main docs/CLI mismatch, though it likely needs a rebase/update because config-only status formatting now lives in `src/commands/channels/status-config-format.ts`.

## Best Possible Solution

Keep the PR open and update or land the same narrow fix on current main: change the `channels status` description and both channel-status footer formatters to say `openclaw status --deep` is the top-level broader gateway probe, then add regression coverage that asserts the config-only footer contains the clarified wording and does not suggest `channels status --deep`.
