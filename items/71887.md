## Summary

Keep PR #71887 open. Current main still has the stale `status --deep` guidance in `openclaw channels status` help text and both status footer paths, while `channels status` only registers `--probe`, `--timeout`, and `--json`. The PR is still a useful focused fix for the user-visible bug described in #69341, though it likely needs a small rebase/update because current main now has the config-only footer in `status-config-format.ts`.

## Best Possible Solution

Keep the PR open as the canonical remaining fix for #69341. Update it against current main so it changes `src/cli/channels-cli.ts`, `src/commands/channels/status.ts`, `src/commands/channels/status-config-format.ts`, and `docs/cli/channels.md`, then keep the regression assertion that confirms `openclaw status --deep` is present and `channels status --deep` is absent.
