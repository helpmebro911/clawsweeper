## Summary

Keep open. Current main still lacks cursor-based chat-history pagination, the related freeze issue #48514 remains open, and PR #56383 is not superseded by a merged implementation. The PR also has unresolved post-fix review concerns around byte-budget truncation and untimestamped messages, so conservative cleanup should not close it.

## Best Possible Solution

Keep the PR open for maintainer/product review or a revised successor. The best implementation path is to add a documented `chat.history` pagination contract across the TypeBox schema, gateway handler, generated Swift models, and Web UI, with regression coverage for strict cursor progression, byte-budget truncation, duplicate timestamps, and untimestamped/imported transcript entries. It should also continue to track the broader performance work in #48514.
