## Summary

Current main does not implement the requested built-in `POST /hooks/message` ingress. The gateway hook handler still has explicit branches for `wake` and `agent`, then optional configured mappings, then `404`; docs list `/hooks/wake`, `/hooks/agent`, and mapped hooks only. PR #62528 is the related implementation attempt and is still open/unmerged, so the issue remains a valid canonical tracker until that work lands or maintainers choose a different design.
