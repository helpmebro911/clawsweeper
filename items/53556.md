## Summary

Keep this issue open. Current main partially covers the Feishu examples, but the central backward-compatibility request is not fully implemented: `groupPolicy: "allowall"` and legacy `topicSessionMode` are still accepted, while old Feishu `botId` credentials are not accepted or migrated to `appId`, and Feishu has no doctor legacy-config rule for that field migration.

## Best Possible Solution

Keep this issue open and scope it to a Feishu-owned compatibility path. Add explicit handling for `botId` -> `appId`, preferably through Feishu doctor `legacyConfigRules`/`normalizeCompatibilityConfig` and focused schema or doctor tests; keep or document the existing `topicSessionMode` and `allowall` compatibility; update the Feishu docs with the supported upgrade path. Treat #52885 as related WeChat plugin SDK history, not the canonical tracker for this Feishu migration work.
