## Summary

Current main still only partially satisfies the requested Feishu backward-compatibility work. Legacy `groupPolicy: "allowall"` and `topicSessionMode` are handled, but the reported `botId` to `appId` compatibility path is still missing from the Feishu schema, runtime credential resolution, setup detection, doctor migration contract, metadata, docs, and tests. The linked #52885 is a closed Weixin plugin-SDK compatibility issue, not a canonical Feishu config migration fix.

## Best Possible Solution

Keep this open and scope the fix to Feishu-owned compatibility: add explicit `botId` to `appId` handling for top-level and account configs, preserve `appId` precedence when both fields exist, wire doctor compatibility via Feishu `normalizeCompatibilityConfig`/`legacyConfigRules` or an equivalent plugin-owned seam, update setup/status and credential resolution as needed, add focused schema/account/setup/doctor tests, and document the upgrade path on https://docs.openclaw.ai/channels/feishu.
