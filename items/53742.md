## Summary

Keep this issue open. Current main has fixed part of the token-drift family, including SecretRef-aware gateway token drift checks and a stale embedded-token auto-refresh path, and Linux/systemd now has an EnvironmentFile-style path for state-dir `.env` values. The macOS LaunchAgent install path still remains in scope: the install plan merges durable `.env`, config service env, auth-profile env refs, and preserved existing env values into `plan.environment`, and the LaunchAgent renderer serializes every non-empty environment entry inline under `EnvironmentVariables` in the plist. That means the broader plaintext LaunchAgent projection problem is not fully implemented on main.

## Best Possible Solution

Keep this issue open and finish the macOS install-path fix: avoid serializing secret-bearing durable `.env`, config service env, auth-profile, provider, plugin, and channel values into LaunchAgent plist `EnvironmentVariables`; load them through a launchd-safe runtime env/SecretRef entrypoint or equivalent external source; scrub stale inline plist entries during repair/reinstall; and add regression coverage for forced install, update/reinstall, ENV/FILE/structured SecretRefs, auth profiles, provider/channel/plugin secrets, and gateway auth token behavior.
