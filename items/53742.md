## Summary

Keep this issue open. Current main has a real partial fix for durable `.env` / config-service env projection and gateway-token drift, but the macOS LaunchAgent install path still serializes non-durable auth-profile env refs inline and there is no source/test proof covering the broader FILE, structured SecretRef, provider/channel/plugin secret, update, and npm reinstall vectors raised in the discussion.

## Best Possible Solution

Keep this issue open as the umbrella for macOS LaunchAgent install-path hardening. Finish the implementation by preventing secret-bearing auth-profile, provider, plugin, channel, durable `.env`, and config service values from being serialized into LaunchAgent `EnvironmentVariables`; load them through a launchd-safe runtime source or wrapper; scrub legacy inline copies during repair/reinstall/update; and add regression coverage for `gateway install --force`, update/npm reinstall, ENV/FILE/structured SecretRefs, and gateway-token behavior. Coordinate narrower follow-ups like #67595 and the service-identity design in #69066 rather than closing this report prematurely.
