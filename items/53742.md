## Summary

Keep open. Current main has fixed some gateway-token drift and reinstall sub-symptoms, but the central macOS LaunchAgent install-path concern remains: `gateway install` still builds a service environment from durable `.env`/config/auth-profile values, and the LaunchAgent renderer still serializes those values inline under plist `EnvironmentVariables`.

## Best Possible Solution

Keep this issue open and finish the macOS install-path fix. The likely maintainer path is to stop putting secret-bearing durable environment values into the LaunchAgent plist, load `$OPENCLAW_STATE_DIR/.env` or supported SecretRefs at gateway startup through a launchd-safe runtime entrypoint, clean up stale plist entries, and replace the existing dotenv-projection tests with coverage proving `gateway install --force`, update/reinstall, env refs, file refs, structured SecretRefs, provider/channel secrets, auth-profile refs, and `gateway.auth.token` do not persist plaintext secrets in the plist.
