## Summary

Keep #16711 open. Current main still has only the capped inline Control UI composer and does not implement the requested Expand button, large composer modal, Apply/Cancel/Send workflow, or modal keyboard behavior where Enter inserts a newline and Ctrl/Cmd+Enter sends. The existing issue discussion and related ClawSweeper reports also do not identify a canonical replacement that tracks or ships this long-prompt composer request.

## Best Possible Solution

Keep this open for first-class Control UI work. A good implementation should add an Expand control near the composer actions, modal draft state, Close/Cancel, Apply, and Send semantics, modal keyboard behavior where Enter inserts a newline and Ctrl/Cmd+Enter sends, and focused UI tests around draft ownership, send/queue behavior, attachments, and existing modal patterns.
