## Summary

Keep this PR open. Current main still implements the exact-action clearing policy for mutating tool failures, while this PR proposes a behavioral change for same-tool/different-target recovery and adds tests around that change. The implementation is not shipped on main, not superseded by a merged PR, and current tests still encode the opposite behavior.

## Best Possible Solution

Keep this PR open for maintainer review. The maintainer path should decide whether same-tool/different-target mutating retries should clear the warning, then either land a narrowed implementation with the PR's regression coverage and fail-closed safeguards, or explicitly close it after deciding the current exact-fingerprint behavior is intentional.
