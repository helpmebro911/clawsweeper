## Summary

Keep open. Current main does not implement Feishu text-based keyword triggers: the Feishu schema has `requireMention` but no `mentionTriggers`, and the group gate still returns whenever `requireMention` is true and `ctx.mentionedBot` is false. The PR addresses a real missing Feishu behavior, but the existing review discussion identifies changes needed before merge, especially broadcast handling, `WasMentioned`, empty trigger values, and likely per-group support.

## Best Possible Solution

Keep the PR open and ask for a focused revision. The best path is to implement Feishu text-pattern mention gating with the shared channel inbound helpers where possible, or fully support the proposed `mentionTriggers` field with non-empty validation, account/group inheritance, broadcast active-agent semantics, correct `WasMentioned`, Feishu unit tests for single-agent and broadcast groups, and docs updates.

## Risks / Open Questions

- Merging the PR exactly as described could silently fail for keyword-triggered messages in Feishu broadcast groups unless `activeAgentId` and `WasMentioned` handling are updated.
- A raw `z.array(z.string())` trigger list would let empty strings match every message unless the schema or runtime filters them.
- The product choice should be clarified: use Feishu-specific `mentionTriggers` or the existing shared `groupChat.mentionPatterns` helpers so text-trigger behavior stays consistent across channels.
