## Summary

Keep open. Current main still lacks Feishu text keyword triggers under requireMention, so this PR is not obsolete. The patch targets the bundled Feishu plugin, but it needs revision before merge because it does not carry keyword matches through broadcast active-agent/WasMentioned handling, accepts empty trigger strings, and omits tests/docs/generated config metadata.

## Best Possible Solution

Keep the PR open and ask for a revision in the bundled Feishu plugin. The implementation should validate non-empty trigger strings, resolve triggers consistently across top-level, account, wildcard group, and exact group config, treat a keyword match as an effective mention for single-agent and broadcast dispatch, update activeAgentId and WasMentioned semantics, and add focused tests plus Feishu docs/generated config metadata updates.

## Risks / Open Questions

- Merging the PR as-is can silently fail in Feishu broadcast groups: a keyword-only message can pass the new gate while activeAgentId remains null, so no active Feishu reply dispatcher is selected.
- The proposed z.array(z.string()) plus ctx.content.includes(t) accepts empty strings; includes("") is always true and would effectively bypass requireMention for every group message if configured.
- The public config field would drift from tests, docs, and generated config metadata unless those are updated with the implementation.
- Security review: the live patch touches only Feishu source/config schema files and does not change CI, dependencies, lockfiles, scripts, package publishing, downloaded artifacts, or secret handling; the security-sensitive concern is the empty-trigger mention-gate bypass above.
