## Summary

Keep open. Current main still requires `models.providers.<id>.models` in both the runtime Zod schema and generated JSON schema, while the Ollama discovery code can already use a configured `baseUrl` when no explicit models are present. PR #71636 tracks the same bug, but it is still open and prior review comments show it is incomplete because the generated schema and exported TypeScript type still need alignment.

## Best Possible Solution

Keep #70982 open until a canonical fix lands. The best implementation is to make provider `models` optional consistently across `ModelProviderSchema`, `src/config/schema.base.generated.ts`, and `ModelProviderConfig` or the normalization boundary, then add regression coverage for `models.providers.ollama` with a remote `baseUrl` and no `models` array so `/api/tags` discovery runs against that configured host. PR #71636 can be the starting point if it is expanded to cover those contract surfaces.
