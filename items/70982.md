## Summary

Keep open. Current main partially addresses the runtime path because `models: []` now lets Ollama discovery call the configured remote `baseUrl`, but the issue's schema/docs contradiction remains: an Ollama provider block with `baseUrl` and omitted `models` is still rejected by the Zod schema, generated config schema, and exported config type. Related PRs #71515/#71636 closed unmerged, and #39766 is the broader open optional-provider-models work.

## Best Possible Solution

Keep #70982 open until maintainers either make `models.providers.<id>.models` optional/defaulted across Zod, generated schema, exported config types, validation, and docs, or explicitly document `models: []` as the required remote-Ollama discovery sentinel. Add focused coverage proving `models.providers.ollama` with a remote `baseUrl` and omitted `models` validates and discovers `/api/tags` from that host, or update the public Ollama docs to show the supported `models: []` form.
