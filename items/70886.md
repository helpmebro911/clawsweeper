## Summary

Keep #70886 open. Current main still appears to route ZAI vision requests through the configured provider-level baseUrl, and ZAI coding endpoint onboarding still applies the coding baseUrl to the provider catalog without a vision-model override. The linked fix attempt #70915 remains open, while #70908 was closed unmerged, so the issue should stay open until a fix lands.

## Best Possible Solution

Keep the issue open and use #70915 or a maintainer-authored equivalent as the fix vehicle. The fix should live in the ZAI plugin/provider routing path, add regression coverage for `zai-coding-global` and `zai-coding-cn` with vision models, and verify that image requests for `glm-4.6v`, `glm-4.5v`, and `glm-5v-turbo` use the matching non-coding endpoint while text/coding models keep the configured coding endpoint.
