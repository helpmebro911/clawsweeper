## Summary

Keep #70886 open. Current main still routes ZAI onboarding through a single provider-wide coding baseUrl for coding auth choices, while the ZAI vision model definitions do not override to the non-coding endpoint. The related PR #70915 directly tracks the likely fix but is still open, so the original bug should remain open until a fix lands.

## Best Possible Solution

Land a provider-owned fix, likely by routing ZAI vision models (`glm-4.6v`, `glm-4.5v`, `glm-5v-turbo`) to the matching non-coding ZAI endpoint when the provider was onboarded with a coding endpoint, and add regression coverage for onboarding/model resolution plus the image-tool media-understanding path. PR #70915 is the active related fix to review or supersede with a maintainer patch.
