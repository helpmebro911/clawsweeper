## Summary

Keep open. Current main still lacks a ZAI-specific route correction for vision models on Coding Plan endpoints, and open PR #70915 is the active implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open while maintainers review and either land or close #70915. The best fix belongs in the ZAI plugin boundary: when the configured ZAI provider endpoint is a Coding Plan endpoint, route vision models such as `glm-4.6v`, `glm-4.5v`, and `glm-5v-turbo` to the same-region non-coding multimodal endpoint, with regression tests proving text/coding models stay on the Coding Plan URL and vision models use the standard URL.
