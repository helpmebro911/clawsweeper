## Summary

Keep #70886 open. The current `main` checkout still appears to route ZAI image models through the provider-level Coding Plan base URL, and the linked fix attempts are not reflected in the shipped ZAI source. The MiniMax-related class of failures has separate evidence in the changelog, but that does not resolve the ZAI `glm-4.6v` 404 reported here.

## Best Possible Solution

Keep #70886 open and use the linked PR discussion as useful root-cause evidence. The best fix likely belongs in the bundled ZAI provider/onboarding and the model-resolution contract: route ZAI vision models such as `glm-4.6v`, `glm-4.5v`, and `glm-5v-turbo` to the matching non-coding ZAI base URL when the provider is configured for a Coding Plan endpoint, ensure model-level `baseUrl` overrides are actually honored by the image/runtime resolver, and add regression tests for `zai-coding-global` and `zai-coding-cn` image-model execution. Update the ZAI docs/changelog once shipped.
