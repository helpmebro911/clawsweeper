## Summary

Keep open. Current main still does not implement the PR's configurable session-level model-error circuit breaker: there is no circuitBreaker config/schema/type, no src/agents/circuit-breaker module, and no persisted cb* session state. Main has adjacent provider/profile cooldowns, cron error backoff/alerts, tool-loop circuit breaking, and context-overflow reset behavior, but those are not the requested per-session pause/reset/alert state machine. The PR remains potentially useful, but the review thread and diff show correctness and outbound-alert validation work still needed before merge.

## Best Possible Solution

Keep this PR open and request a focused revision before merge. The best path is to add a structured model-failure signal from reply/cron execution, persist breaker state on the actual run session including isolated heartbeat sessions, make reset semantics intentional for heartbeat dedupe state, validate outbound alert routing and error text, and cover heartbeat, cron, and interactive reply paths with targeted tests. If maintainers want a smaller feature, split the generic session-state/config API from optional alert/reset actions.
