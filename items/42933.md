## Summary

Keep open. Current main still does not implement the requested configurable session-level model-error circuit breaker: there is no `circuitBreaker` config/schema/type surface, no `src/agents/circuit-breaker` module, no persisted `cb*` session state, and the heartbeat/cron/reply paths do not maintain per-session pause/reset/alert breaker state. Existing auth-profile cooldowns, cron error backoff, context-overflow compaction, and tool-loop circuit breaking are adjacent mitigations, not this PR's session-level state machine. The PR remains a plausible implementation candidate, but the review thread shows correctness work still needed before merge.

## Best Possible Solution

Keep this PR open and request a focused revision before merge. The best path is a core-owned session-level breaker with typed config/schema, persisted per-session breaker state, a structured model-failure signal from reply/heartbeat/cron execution, correct isolated-heartbeat session ownership, clear reset semantics for heartbeat dedupe state, validated/sanitized alert routing, and targeted tests across heartbeat, cron, and interactive reply paths. If maintainers want to reduce scope, split the generic config/state and model-failure signal from optional alert/reset actions.
