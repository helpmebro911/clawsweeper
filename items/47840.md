## Summary

Keep open. Current main has provider-id normalization for current aliases such as `z-ai` -> `zai` and `aws-bedrock` -> `amazon-bedrock`, but the Control UI Usage display/filter paths still render `modelProvider`/`providerOverride` directly. The original `bailian` example does not appear to be a current core alias, and the Sessions table no longer shows a provider column, but the broader normalized-vs-displayed-provider inconsistency remains for supported aliases. The related #47620/#47621 discussion is useful context but covered chat picker value/dedupe behavior, not these remaining Usage provider display paths.
