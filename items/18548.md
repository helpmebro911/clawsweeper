## Summary

Keep #18548 open. Current main still trims oversized bootstrap files by fixed head/tail character slices with no relevance-aware or structure-aware middle selection, and active run paths still feed workspace bootstrap files through that trimmer. The compaction and observation-extraction parts may be better split into narrower context-engine/plugin trackers, but the core bootstrap relevance request remains unresolved.

## Best Possible Solution

Keep #18548 open as the core tracker for bootstrap relevance, or split it so bootstrap relevance remains the OpenClaw core issue while incremental compaction follows #58398 or a dedicated compaction tracker. The likely code change belongs in src/agents/pi-embedded-helpers/bootstrap.ts: replace fixed head/tail slicing for oversized bootstrap files with a relevance-aware or structure-aware selector, then update positional tests and docs/config wording. Treat observation extraction as plugin/hook/context-engine work unless a concrete missing SDK API is identified.
