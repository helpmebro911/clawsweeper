## Summary

Keep #18548 open. Current main still truncates oversized bootstrap files by fixed head/tail character slices, and the active run paths still feed workspace bootstrap files through that trimmer. Context-engine, compaction-provider, hook, and Lossless Claw plugin surfaces help with the compaction/observation parts, but they do not resolve the core bootstrap relevance request already called out in the prior maintainer/Codex review.

## Best Possible Solution

Keep #18548 open, preferably as the core tracker for bootstrap relevance. The likely code change belongs in src/agents/pi-embedded-helpers/bootstrap.ts: replace fixed head/tail slicing for oversized bootstrap files with a relevance-aware or structure-aware selector, then update positional tests and docs/config wording. Track incremental compaction through #58398, context-engine, compaction-provider, and Lossless Claw work; treat observation extraction as plugin/hook/context-engine work unless a concrete missing SDK API is identified.
