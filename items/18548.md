## Summary

Keep #18548 open. Current main still uses fixed positional head/tail slicing for oversized bootstrap files, and the active run path, tests, and docs continue to protect and describe truncation rather than relevance-aware or structure-aware selection. Incremental compaction and observation extraction have plausible plugin/context-engine paths, but they do not resolve the core bootstrap relevance request.

## Best Possible Solution

Keep #18548 open as the core tracker for bootstrap relevance, or split it so bootstrap relevance remains in OpenClaw core while incremental compaction follows the context-engine/Lossless Claw/#58398 path and observation extraction remains hook/plugin-driven unless a missing SDK API is identified. The likely core change belongs around src/agents/pi-embedded-helpers/bootstrap.ts and buildBootstrapContextFiles: replace fixed head/tail slicing with a relevance-aware or structure-aware selector, then update positional tests and docs/config wording.
