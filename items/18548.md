## Summary

Keep #18548 open. Current main still uses fixed head/tail positional slicing for oversized bootstrap files in the active run path, and tests/docs still describe and protect truncation rather than relevance-aware or structure-aware bootstrap selection. The compaction and observation-extraction parts have plausible context-engine/plugin paths, including Lossless Claw and #58398, but that does not resolve the core bootstrap relevance request.

## Best Possible Solution

Keep #18548 open as the core tracker for bootstrap relevance, or split it so bootstrap relevance stays in OpenClaw core while incremental compaction follows #58398/context-engine/Lossless Claw work and observation extraction remains hook/plugin-driven unless a missing SDK API is identified. The likely core change belongs around src/agents/pi-embedded-helpers/bootstrap.ts and buildBootstrapContextFiles: replace fixed head/tail slicing with a relevance-aware or structure-aware selector, then update positional tests and docs/config wording.
