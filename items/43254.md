## Summary

Keep this PR open. Current main still scopes Control UI session tokens by the exact parsed gateway URL, so the reported localhost / 127.0.0.1 / [::1] token fragmentation is not implemented on main. The PR targets the correct UI storage surface and has useful follow-up test work, but the proposed `host.startsWith("127.")` loopback check can also match ordinary DNS hostnames such as `127.example.com` and `127.0.0.1.nip.io`, so it needs a small security fix before merge.

## Best Possible Solution

Keep this PR open and request a small revision: normalize only `localhost` and actual numeric loopback IP literals, using the shared canonical IP helper if suitable for the UI bundle or an equivalent UI-local parser; preserve protocol, port, and pathname isolation; add negative tests for `127.example.com` and `127.0.0.1.nip.io`; keep the IPv4/IPv6 alias reuse tests; then run the appropriate UI changed gate before landing.

## Risks / Open Questions

- The PR as written can broaden token-scope sharing beyond true loopback aliases because DNS hostnames such as `127.example.com` and `127.0.0.1.nip.io` satisfy `host.startsWith("127.")`.
- The added tests cover positive alias reuse and port isolation, but they do not include negative cases for DNS hostnames that begin with `127.`.
- Current main still has the underlying UX footgun until a corrected implementation lands.
