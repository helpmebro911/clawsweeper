## Summary

Keep this PR open. Current main still conflates a non-matching session-kill path with a matched path whose session key cannot be decoded, so a route-shaped malformed request can return `false` from the session-kill handler instead of a deterministic `400 invalid_request_error`. The PR is a narrow gateway bug fix with a focused regression test and no unrelated security-sensitive file changes.

## Best Possible Solution

Keep this PR open for normal maintainer review and CI. The best code path is to land the narrow handler change plus regression test, preserving existing valid session auth, ownership, kill, and RPC semantics while making malformed matched session-kill paths return a deterministic API error.
