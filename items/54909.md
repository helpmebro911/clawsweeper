## Summary

Current main still does not satisfy the documented Telegram inline-button callback text contract. The callback_query route exists, but generic callback data is still forwarded to the agent as the raw payload rather than `callback_data: <value>`, and the linked open PR remains the implementation candidate.

## Best Possible Solution

Keep this issue open. Review and land PR #54962, or apply an equivalent change in `extensions/telegram/src/bot-handlers.runtime.ts` that forwards generic Telegram callback fallback text as `callback_data: ${data}` while preserving native `tgcmd:` callbacks and the existing approval/plugin/model callback paths. Add a regression test that asserts the documented prefix for generic callbacks, then close after the fix merges.
