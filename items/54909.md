## Summary

Current main still does not implement the documented Telegram inline-button callback text contract for generic callbacks. The handler receives callback_query updates and forwards them into the agent flow, but the generic fallback still uses the raw payload instead of `callback_data: <value>`. An open, unmerged PR with closing syntax already targets this exact issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #54962 or an equivalent patch in `extensions/telegram/src/bot-handlers.runtime.ts` that forwards generic Telegram callback fallback text as `callback_data: ${data}` while preserving native command callbacks. Add or update a regression test that asserts the documented prefix, then close this issue after the fix merges.
