## Summary

Keep PR #70109 open. Current main still encodes the old behavior: reply media normalization failures are caught, logged, and converted into media-less payloads, and media dedupe still strips final replies when sent-media target metadata is missing. The related #69310 is overlapping but not a clear superseder for #70109's same-target/missing-target dedupe changes, and #70109 has unresolved review concerns that need maintainer/author handling rather than automated closure.

## Best Possible Solution

Keep #70109 open. Maintainers should decide whether the fail-loud error-payload approach in #70109 or the broader dropped-media pipeline in #69310 is the preferred product behavior, then require the chosen patch to cover both normalization/staging failures and the missing-target-metadata media dedupe case with regression tests. If #69310 becomes canonical, it should explicitly absorb #70109's same-target proof behavior before #70109 is closed as superseded.
