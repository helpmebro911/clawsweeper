## Summary

Keep open. Current `main` intentionally treats `qwen/qwen3.6-plus` as unavailable on Qwen Coding Plan endpoints and documents that users should switch to a Standard endpoint or `qwen/qwen3.5-plus`, but the reporter's remaining complaint about still getting the generic `Unknown model` error appears valid on `main`: the suppression-specific error text is defined in the Qwen plugin yet dropped before the final error is built.

## Evidence

- **Qwen plugin suppresses `qwen3.6-plus` on Coding Plan and defines a descriptive error:** `suppressBuiltInModel` returns a Qwen-specific message when the provider is `qwen`/`modelstudio`, the model is `qwen3.6-plus`, and the configured base URL is a Coding Plan endpoint.
  - file: [extensions/qwen/index.ts:177](https://github.com/openclaw/openclaw/blob/88c91675/extensions/qwen/index.ts#L177)
  - command: `nl -ba extensions/qwen/index.ts | sed -n '1,240p'`
  - sha: [88c91675](https://github.com/openclaw/openclaw/commit/88c91675)
- **Current catalog logic omits `qwen3.6-plus` from Coding Plan endpoints:** `isQwenCodingPlanBaseUrl` matches `coding.dashscope.aliyuncs.com` and `coding-intl.dashscope.aliyuncs.com`, and `buildQwenModelCatalogForBaseUrl` filters out `qwen3.6-plus` for those endpoints.
  - file: [extensions/qwen/models.ts:111](https://github.com/openclaw/openclaw/blob/88c91675/extensions/qwen/models.ts#L111)
  - command: `nl -ba extensions/qwen/models.ts | sed -n '1,220p'`
  - sha: [88c91675](https://github.com/openclaw/openclaw/commit/88c91675)
- **Model resolution does suppress the built-in model when config/baseUrl is available:** `resolveExplicitModelWithRegistry` passes `providerConfig?.baseUrl` and `config` into `shouldSuppressBuiltInModel`, so a Coding Plan config will mark the Qwen model as suppressed before lookup.
  - file: [src/agents/pi-embedded-runner/model.ts:413](https://github.com/openclaw/openclaw/blob/88c91675/src/agents/pi-embedded-runner/model.ts#L413)
  - command: `nl -ba src/agents/pi-embedded-runner/model.ts | sed -n '400,460p'`
  - sha: [88c91675](https://github.com/openclaw/openclaw/commit/88c91675)
- **But final error construction drops the config/baseUrl needed for the descriptive message:** After suppression, `buildUnknownModelError` calls `buildSuppressedBuiltInModelError` with only `{ provider, id }`. `src/agents/model-suppression.ts` needs `config` or `baseUrl` to re-evaluate the Qwen suppression rule, so this path falls back to plain `Unknown model: provider/model` instead of the plugin's descriptive error.
  - file: [src/agents/pi-embedded-runner/model.ts:886](https://github.com/openclaw/openclaw/blob/88c91675/src/agents/pi-embedded-runner/model.ts#L886)
  - command: `nl -ba src/agents/pi-embedded-runner/model.ts | sed -n '880,920p'; nl -ba src/agents/model-suppression.ts | sed -n '1,120p'`
  - sha: [88c91675](https://github.com/openclaw/openclaw/commit/88c91675)
- **Docs now describe `qwen3.6-plus` as a Standard-endpoint model, not a Coding Plan model:** The public Qwen docs say to prefer Standard for `qwen3.6-plus` and state that Coding Plan configs omit models only known to work on Standard.
  - file: [docs/providers/qwen.md:29](https://github.com/openclaw/openclaw/blob/88c91675/docs/providers/qwen.md#L29)
  - command: `sed -n '1,240p' docs/providers/qwen.md`

That matches the current code path above.
