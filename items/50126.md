## Summary

Keep open. Current main has fixed part of the Telegram direct reply gap, but the issue is an umbrella about inconsistent outbound hook coverage and meaningful gaps remain in current code: plugin-handled outbound actions bypass `message_sent`, webchat/TUI-style gateway replies still append/broadcast without message hook emission, and Telegram preview-finalized delivery currently emits only the internal hook.

## Evidence
