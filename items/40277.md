## Summary

Keep open. Current main still lacks the central web-chat UI behavior: live chat error events only update `lastError`, assistant `stopReason: "error"` history entries with no text are not converted into visible text, and the renderer has no highlighted model-error bubble. Open #55923 overlaps the live-error path, but its diff only appends a generic warning-text message and does not cover this PR's persisted error-only history fallback or styled model-error rendering, so it is not a complete superseding item.

## Best Possible Solution

Keep this PR open for maintainer review, or fold its narrower UI behavior into a fresh branch or #55923. The desired implementation should append structured assistant error messages for live chat errors, render persisted assistant `stopReason: "error"` entries including empty-detail fallbacks, style model failures distinctly in web chat, and cover controller, extraction, and rendering behavior with focused tests on current main.
