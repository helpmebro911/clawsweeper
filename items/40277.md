Keep open. Current main still lacks the PR's central behavior: live web chat error events only set hidden `lastError`, persisted assistant error metadata is not extracted into display text, and grouped chat rendering has no highlighted model-error bubble. The PR remains a focused UI fix, with a small rebase/security review need around current connect-error formatting and sanitized provider error display.

Keep this PR open for maintainer review or fold its focused UI behavior into the canonical inline-error work.
