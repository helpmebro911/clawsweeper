## Summary

Keep open. Current `main` has generic session-store maintenance for stale `sessions.json` entries, but Feishu still does not expose the requested Feishu-specific idle/max-age session lifecycle controls or command/config surface that would match the issue.

## Evidence

- **Feishu config schema has no lifecycle override keys:** `FeishuSharedConfigShape`, `FeishuAccountConfigSchema`, and `FeishuConfigSchema` enumerate Feishu channel settings and end in `.strict()`. They include routing/rendering/group settings, but no `threadBindings`, `idleHours`, `maxAgeHours`, `idleTimeout`, or `sessionMaxAge` fields for Feishu.
  - file: [extensions/feishu/src/config-schema.ts:155](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/feishu/src/config-schema.ts#L155)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '130,240p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Feishu plugin does not provide lifecycle update hooks:** The Feishu channel plugin advertises only `conversationBindings.defaultTopLevelPlacement` and `buildModelOverrideParentCandidates`; it does not define `setIdleTimeoutBySessionKey` or `setMaxAgeBySessionKey`, which are the hooks the session lifecycle commands use.
  - file: [extensions/feishu/src/channel.ts:595](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/feishu/src/channel.ts#L595)
  - command: `nl -ba extensions/feishu/src/channel.ts | sed -n '588,610p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Session lifecycle commands remain unavailable without those hooks:** `/session idle` and `/session max-age` explicitly require a channel with current-conversation binding support plus `setIdleTimeoutBySessionKey` / `setMaxAgeBySessionKey`; otherwise OpenClaw replies that the feature is only available on supported channels.
  - file: [src/auto-reply/reply/commands-session.ts:463](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/auto-reply/reply/commands-session.ts#L463)
  - command: `nl -ba src/auto-reply/reply/commands-session.ts | sed -n '460,515p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Feishu docs still expose no session lifecycle settings:** The Feishu channel docs list the available `channels.feishu.*` settings, and the table does not include any Feishu session idle/max-age or `threadBindings` options.
  - file: [docs/channels/feishu.md:375](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/channels/feishu.md#L375)
  - command: `nl -ba docs/channels/feishu.md | sed -n '372,408p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **There is an existing generic cleanup mechanism, but it is different:** OpenClaw already documents repo-wide `session.maintenance` cleanup (`pruneAfter`, `maxEntries`, `rotateBytes`) for `sessions.json`, which helps bound store growth, but that is not the Feishu-specific idle/max-age lifecycle requested here.
  - file: [docs/reference/session-management-compaction.md:72](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/reference/session-management-compaction.md#L72)
