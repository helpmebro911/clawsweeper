## Summary

Keep #23451 open. Current main still has exec approvals and plugin-level `before_tool_call.requireApproval`, but not the requested built-in `tools.confirmation` risk-tier policy or assistant-text-aware pause before the first same-response tool executes.

## Best Possible Solution

Keep #23451 open for maintainer/product work at the agent tool-execution boundary. The likely implementation should add a documented `tools.confirmation` config/schema/help surface, deterministic tool risk classification, access to same-response assistant text before first tool execution, user-visible text flush before holding tools, approval/timeout execute-or-skip semantics, and transcript handling that preserves the existing plugin `before_tool_call.requireApproval` primitive as a lower-level custom policy hook.
