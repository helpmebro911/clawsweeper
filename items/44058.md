## Summary

Keep open. Current main still exposes TTS as global `messages.tts`, Feishu account config has no per-account `tts` field, and the shared TTS runtime/delivery path does not carry channel account context. The issue is still valid, and there is already an open follow-up PR (#63338) that a maintainer explicitly left open pending a better design.

## Evidence

- **Speech core still resolves only global TTS config:** `resolveTtsConfig` and the effective auto-state both read from `cfg.messages?.tts`, so the shared runtime still treats TTS as a global config surface.
  - file: [extensions/speech-core/src/tts.ts:313](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/extensions/speech-core/src/tts.ts#L313)

  - file: [src/plugin-sdk/tts-runtime.types.ts:89](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/plugin-sdk/tts-runtime.types.ts#L89)
