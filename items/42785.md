## Summary

Keep open. Current `main` does not include this note-body contrast change, and the PR as proposed still has unresolved regression risk for light-background terminals from the hard-coded near-white body color called out in review.

## Evidence

- **Current main still uses terminal-default note body text:** `note()` wraps the message and passes it straight to `clackNote(...)` with no `isRich()`/theme body-color override, so the reported web-terminal contrast problem is not implemented on `main`.
  - file: [src/terminal/note.ts:182](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/terminal/note.ts#L182)

  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **No body color token exists on main:** The current Lobster palette exports `accent`, `accentBright`, `accentDim`, `info`, `success`, `warn`, `error`, and `muted`, but no `body` token; `theme.ts` likewise has no `theme.body` helper.
  - file: [src/terminal/palette.ts:3](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/terminal/palette.ts#L3)
