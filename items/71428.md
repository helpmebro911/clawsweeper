## Summary

Keep this issue open. Current main still gates conversation-content hooks from per-plugin host config only, and the native plugin manifest contract has no `permissions.conversationAccess` field or loader/registry merge path. The prior review comment remains consistent with current main at `348728c28c1ea9ae7c5824350baba35ee08a275e`.

## Best Possible Solution

Keep the issue open. Add an explicit additive native manifest contract, likely `permissions.conversationAccess`, in `src/plugins/manifest.ts`; normalize it in `loadPluginManifest`; carry it through `PluginManifestRecord`; merge it into loader hook policy with documented config precedence; then update manifest/hooks/config docs and add tests for manifest-only allow, config-only backward compatibility, both-present precedence, and default rejection.
