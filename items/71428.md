## Summary

Keep #71428 open. Current main partially solved the older schema-patch problem by accepting the host config flag, but it still does not implement the requested manifest-owned `permissions.conversationAccess` contract. The loader and registry enforce conversation-hook access only from `plugins.entries.<id>.hooks.allowConversationAccess`, and the public manifest docs/type do not expose a permissions field.

## Best Possible Solution

Keep this open. Add an additive manifest-level `permissions.conversationAccess` contract in `src/plugins/manifest.ts` and `src/plugins/manifest-registry.ts`, merge it with the existing `plugins.entries.<id>.hooks.allowConversationAccess` override in the loader/registry policy path, preserve the current config flag for backward compatibility, and document/test precedence for blocked, manifest-allowed, config-allowed, and explicit-disable cases.
