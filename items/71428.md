## Summary

Keep open. Current main has the config-level `plugins.entries.<id>.hooks.allowConversationAccess` policy and schema support, but it still does not implement the requested manifest-owned `permissions.conversationAccess` contract. The prior issue discussion already called out this distinction, and the current checkout still matches that assessment.

## Best Possible Solution

Add an explicit additive manifest permission contract, likely `permissions.conversationAccess`, normalize it through `src/plugins/manifest.ts` and `src/plugins/manifest-registry.ts`, merge it into the loader/registry hook policy with the existing config override preserved, then document and test precedence for non-bundled conversation hooks.
