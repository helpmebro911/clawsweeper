## Summary

Current main still matches the reported Windows encoding failure mode: `runExec` hardcodes UTF-8 decoding, and `runCommandWithTimeout` appends child-process output with default UTF-8 `Buffer.toString()`. The repo docs still describe Windows Chinese mojibake as an active workaround case, and related issue #56462 confirms the same bug on 2026.3.24. There is no merged main fix to close this as implemented, and #50519 remains a concrete, actionable original bug report with linked attempted fixes.
