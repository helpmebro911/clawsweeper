## Summary

Keep open. Current `main` still has the reported gap: `extensions/google/provider-models.ts` routes `gemma-*` forward-compat models through the Gemini flash template and only patches `reasoning` for `gemma-4`, so Gemma models can still inherit the template `contextWindow`. The resolver remains active in the Google provider registration paths, and current tests cover Gemma reasoning behavior but not the context-window regression this PR addresses.

## Evidence

- **Gemma forward-compat path has no context-window override:** In current `main`, the `gemma-*` branch sets `googleTemplateIds`/`cliTemplateIds` to `GEMMA_TEMPLATE_IDS` and only applies `patch = { reasoning: true }` for `gemma-4`. There is no `contextWindow` patch for Gemma models.
