## Summary

Keep #9656 open. Current main still only falls back to `envelope.editMessage?.dataMessage` and then processes the edited body as a normal inbound Signal message. The requested edit-specific behavior remains missing: no modeled `targetSentTimestamp` on edit envelopes, no edited marker/metadata, no dedup against the original message timestamp, and no edit-specific reply suppression. PR #9701 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until the bundled Signal plugin implements explicit edit handling: model `editMessage.targetSentTimestamp`, mark edited content in agent/session context, use the target timestamp for deduplication or transcript replacement, and suppress edit-triggered replies unless the edit explicitly mentions the agent. Add focused Signal plugin tests for edit envelopes and mention-gated group behavior.
