## Summary

Keep #9656 open. Current main still flattens Signal `editMessage` envelopes into normal inbound messages, does not model or preserve edit `targetSentTimestamp`, uses the new envelope timestamp as `MessageSid`, and has no edit-aware reply suppression or focused edit-envelope tests. The existing maintainer-facing comment and closed-unmerged PR #9701 both point to the same remaining bundled Signal plugin gap.

## Best Possible Solution

Implement this in the bundled Signal plugin. Add a typed edit-message envelope with validated `targetSentTimestamp`, preserve edit metadata or an `[edited]` marker in agent/session context, use the target timestamp for dedupe or transcript replacement, suppress edit-triggered replies unless the edited text explicitly mentions the agent, and add focused Signal tests for edit envelopes, `MessageSid`/dedupe behavior, and mention-gated group edits. PR #9701 is useful historical context, but it was closed unmerged and should not be treated as shipped.
