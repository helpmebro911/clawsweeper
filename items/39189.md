## Summary

Keep PR #39189 open. Current main still does not cover the exact Anthropic billing wording, and the duplicate-reply path is still gated on same-target suppression, while the PR's proposed unconditional text dedupe and broad billing matcher have real false-positive risks noted in review discussion.

## Best Possible Solution

Keep the PR open for rework or split it into two focused fixes: add a narrow Anthropic billing matcher for the exact "specified API usage limits" / "regain access" wording without shadowing periodic usage limits, and address multi-handle duplicate suppression through identity-aware target matching or a channel-owned alias/suppression hook instead of unconditional text-only dedupe. Add positive tests for the Anthropic sample and same-person multi-handle case, plus negative tests for cross-target same-text delivery and periodic usage-window limits.

## Risks / Open Questions

- Anthropic "specified API usage limits" errors can still be classified as rate_limit, causing cooldown/retry behavior instead of billing failover behavior.
- Merging the PR as-is could suppress a legitimate final reply when the message tool sends the same text to a genuinely different provider, account, channel, or target.
- The PR's broad billing matcher could classify ordinary periodic usage-window limits as billing, conflicting with existing 402 temporary-limit tests.
