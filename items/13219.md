## Summary

Keep #13219 open. Current main has useful adjacent usage surfaces, but still lacks the requested native append-only per-call usage ledger with durable event IDs, source/job/run attribution, idempotency/backfill semantics, and ledger-first readers.

## Best Possible Solution

Keep #13219 open until OpenClaw adds a typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam. The ledger should store provider/model, agentId, sessionKey/sessionId, run or call attribution, source/job metadata, token/cache/cost fields, status, cost status/breakdown, latency, and durable idempotency metadata without prompt/completion content. Then usage.cost, CLI/UI summaries, and an explicit backfill command should prefer the ledger while preserving transcript scanning as a compatibility fallback; provider quota snapshots and optional real-time/plugin tools should remain adjacent surfaces.
