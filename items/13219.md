## Summary

Keep open. Current main still provides transcript/session-log based cost summaries, a Gateway usage-cost command, provider quota snapshots, diagnostics/OTel usage events, and UI aggregation, but not the requested native append-only per-call usage ledger with event ids, source/job/run attribution, durable `logs/usage.jsonl`-style storage, and ledger-first readers. The recent issue discussion also explicitly distinguishes #49064 and #70205 from this ledger request.

## Best Possible Solution

Keep #13219 open until OpenClaw adds a typed append-only usage ledger written from the normalized model-call usage/cost seam. The best product path is to store provider/model, agentId, sessionKey, sessionId, runId/callId, source/job attribution, token/cache/cost fields, status, and idempotency metadata without prompt/completion content; then make `usage.cost`, session usage readers, CLI/UI summaries, and any backfill command prefer the ledger while preserving transcript scanning as compatibility fallback. Provider quota snapshots and optional external real-time tools should remain separate adjacent surfaces.
