## Summary

Keep #13744 open. Current main still lacks a public `openclaw.json` or `OPENCLAW_*` configuration surface for session write-lock `timeoutMs`/`staleMs`, and the embedded run still holds the session write lock across setup, prompt/model execution, and teardown. Main has partially addressed the error-classification request by adding `SessionWriteLockTimeoutError` and excluding it from model-timeout failover, but the core configurability and lock-scope requests remain unresolved.

## Best Possible Solution

Keep #13744 open until OpenClaw adds a supported `openclaw.json` and/or `OPENCLAW_*` surface for session write-lock `timeoutMs` and `staleMs`, wires it into embedded run and transcript mutation paths, and either narrows the lock to actual transcript read/repair/prewarm/append/rewrite critical sections or explicitly documents the broad per-session lifecycle lock as intentional. Preserve the current `SessionWriteLockTimeoutError` failover suppression and add focused contention/config coverage for concurrent same-session callers.
