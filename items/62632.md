## Summary

Keep open. The issue is recent, has no protected maintainer-auto-close exception, and the provided triage comments already confirm a real Docker-to-native migration bug. Current main has session-path hardening, but it still accepts structurally valid cross-root absolute transcript paths and persists/uses them; the legacy migration code canonicalizes session keys and layouts, not Docker-era `/home/node` absolute paths embedded in session entries or workspace files.

## Evidence

- **Maintainer triage context confirms the bug:** The issue comments from daviddaadams explicitly mark Docker to native host migration as a real bug, identify stale absolute `/home/node/...` paths in persisted session/workspace metadata as the root cause, and recommend startup/doctor detection plus path normalization rather than closing as unsupported.
- **Current session path resolver can still reuse cross-root absolute paths:** `resolvePathWithinSessionsDir` treats older absolute `sessionFile` paths specially.
