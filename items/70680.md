## Summary

Keep open. Current main still has the reported doctor bug: `*.trajectory.jsonl` matches the shared primary-transcript helper, and doctor uses that helper when building orphan transcript candidates. Runtime/docs confirm trajectory sidecars are written beside session files by default. The related fix PR #70812 is a good canonical implementation path, but it is still open and unmerged in the provided GitHub context, so the issue should remain open until a fix lands.

## Best Possible Solution

Keep #70680 open and land a narrow bug fix, likely via PR #70812 or an equivalent maintainer patch: exclude `*.trajectory.jsonl` from primary and usage-counted transcript classification, add helper coverage, and add a doctor regression test proving a real orphan `*.jsonl` is still offered for archival while a neighboring trajectory sidecar is ignored.
