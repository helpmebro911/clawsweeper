## Summary

Keep open. Current main still has the WebChat history shape that this issue reports: `chat.history` has no cursor parameter, the gateway reads and slices one bounded tail page, the Control UI always requests `limit: 200`, and the chat renderer still renders a capped non-virtualized list. Main does include payload truncation/byte-budget mitigations, but those do not implement the requested pagination, lazy loading, virtual scrolling, or UI history lifecycle boundary. The related pagination PRs in the provided context were closed unmerged, so there is no canonical merged fix to close against.
