## Summary

Keep #9546 open. Current main still renders the WebChat `New messages` pill directly from scroll state, and neither persisted Control UI settings nor `gateway.webchat` config expose a way to disable, customize, dismiss, or replace it with forced autoscroll. The prior keep-open review still matches the current touched files.

## Best Possible Solution

Keep #9546 open. A narrow implementation should add a persisted WebChat/Control UI preference or documented `gateway.webchat` option for the new-message indicator mode, feed that setting into both `scheduleChatScroll` and `renderChat`, and update focused UI tests plus the public WebChat/config docs.
