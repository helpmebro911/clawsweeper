## Summary

Keep #9546 open. Current main still derives the WebChat "New messages" pill directly from scroll state, renders it when that state is true, and exposes no persisted Control UI or gateway.webchat option to disable, customize, dismiss, or replace it with forced autoscroll.

## Best Possible Solution

Keep #9546 open. A narrow implementation should add a persisted WebChat/Control UI preference or documented gateway.webchat option for the new-message indicator mode, feed that setting into both scheduleChatScroll and renderChat, and update focused UI tests plus the public WebChat/config docs.
