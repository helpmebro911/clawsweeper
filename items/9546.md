## Summary

Keep #9546 open. Current main still implements the WebChat/Control UI “New messages” indicator with no user or gateway setting to disable/customize it, and the existing docs/config surfaces only expose `gateway.webchat.chatHistoryMaxChars` for WebChat. The prior automated review comment also kept this open, and current source evidence still matches that conclusion.

## Best Possible Solution

Keep the issue open until maintainers choose and implement the product surface. A concrete fix should add either a browser-local Control UI preference or a documented `gateway.webchat` config setting, gate `showNewMessages`/scroll behavior through it, and update UI tests plus WebChat/config docs.
