## Summary

Keep open. Current main still has the headline web_fetch problem: after a direct network failure and a missed provider fallback, it rethrows the original error without adding the requested URL or the underlying cause. The PR is not obsolete, but it should be narrowed because its live diff bundles unrelated Feishu, Telegram, model/config/table/tool-result, image-helper, and package override changes, including security/install-sensitive side changes.

## Best Possible Solution

Keep this PR open for maintainer rework or replacement. The best path is a narrow web_fetch-only patch that wraps direct network-level failures only after provider fallback misses, extracts error.cause.message or error.cause.code, preserves the original error as cause, includes a safe requested-URL signal, and adds a regression test for the no-fallback network-error case. The unrelated Feishu, Telegram, config schema, model/table, image-helper, tool-result, and package override changes should be split into separate PRs or dropped.
