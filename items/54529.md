## Summary

Keep this issue open. Current main still implements `channels.telegram.apiRoot`, but the reported migration failure is not fully fixed: Telegram update-offset state is still scoped only to account/bot identity rather than the Bot API endpoint, and `deleteWebhook` setup still treats Telegram 400 `Logged out` as a fatal non-recoverable error. The prior related apiRoot work and ClawSweeper closures are relevant, but they cover basic custom endpoint support, not this endpoint-switch offset and migration-cooldown bug.

## Best Possible Solution

Keep this issue open and fix it in the Telegram plugin. Persist a normalized `apiRoot` or endpoint fingerprint with update-offset state, ignore/reset stored offsets when the endpoint changes, handle Telegram 400 `Logged out` during local Bot API migration without crash-looping, and add focused regression tests plus docs for local Bot API migration.
