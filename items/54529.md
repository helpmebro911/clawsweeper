## Summary

Keep open. Current main already has broad Telegram `apiRoot` passthrough, but the narrower local Bot API migration bug remains: persisted offsets are still account/bot scoped rather than endpoint scoped, lower local-server update IDs can still be skipped before dispatch, and `deleteWebhook` 400 `Logged out` is still treated as a fatal non-recoverable setup error.

## Best Possible Solution

Keep this open and fix it in the Telegram plugin: persist a normalized `apiRoot` or endpoint fingerprint with update-offset state, ignore/reset the stored offset when the endpoint changes, handle Telegram `400: Logged out` during local Bot API migration without crash-looping, and add focused regression tests plus local Bot API migration docs.
