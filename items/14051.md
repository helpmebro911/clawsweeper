## Summary

Keep #14051 open. Current main still does not implement activity-based heartbeat idle timeout: heartbeat config/schema/docs expose interval, active-hours, delivery, timeout, light-context, isolation, and reasoning controls, while the runner remains interval/wake driven and does not gate heartbeats on recent allowed inbound user activity. The supplied issue context also shows #59567 was closed as a duplicate in favor of #14051, so this remains the canonical tracker.

## Best Possible Solution

Keep #14051 open as the canonical heartbeat idle-timeout tracker. A proper fix should add a maintainer-approved heartbeat activity/idle config key to exported types, strict zod schema, schema help/generated docs, config-honor inventory, and https://docs.openclaw.ai/gateway/heartbeat. Runtime should gate heartbeat scheduling or execution on recent allowed inbound user/channel activity at the correct agent/session/channel boundary, likely using lastInteractionAt or a dedicated activity ledger, while preserving existing interval, active-hours, wake, requests-in-flight retry, lightContext, isolatedSession, and per-agent merge behavior.
