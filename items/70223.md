## Summary

Keep open. Current main still does not provide a WorkingDirectory for the normal packaged gateway install path, and configured relative agentDir values still resolve through process cwd. The fresh default-config detail appears narrower on current main because local setup now writes agents.defaults.workspace rather than a relative agentDir, but the reported launchd/service install gap remains real for existing or explicit relative agentDir configs.

## Best Possible Solution

Keep the issue open and fix the service boundary: have packaged gateway install plans provide a deterministic WorkingDirectory, preferably the user home or OpenClaw state/config root, and add a regression test that the macOS LaunchAgent plist includes it for non-dev installs. A defense-in-depth follow-up should either reject or anchor relative agents.list[].agentDir values to a stable config/state directory rather than process.cwd().
