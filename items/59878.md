## Summary

Keep open. Current `main` still has the stale-session-state gap described in #59878: `sessions.abort` only aborts live runs and returns `"no-active-run"` without reconciling persisted session rows, `sessions.list` surfaces persisted `status`/`endedAt` directly, and gateway startup only cleans stale lock files rather than clearing stale `running` session state. There is a partial unrelated/adjacent fix in `CHANGELOG.md` for chat-run registry linkage (#69747), but it does not implement the broader persisted-state reconciliation requested here.

## Evidence

- **`sessions.abort` does not clear stale persisted state:** `src/gateway/server-methods/sessions.ts:1242` delegates to `chat.abort`; if no live run is found it responds with `status: "no-active-run"`, and it only emits `sessions.changed` when `abortedRunId` exists. There is no patch to the session store on the `no-active-run` path.
  - file: [src/gateway/server-methods/sessions.ts:1242](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/gateway/server-methods/sessions.ts#L1242)
  - command: `nl -ba src/gateway/server-methods/sessions.ts | sed -n '1225,1282p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Session rows still read persisted `status` directly:** `src/gateway/session-utils.ts:1421` builds the gateway session row from `entry?.status`, `entry?.startedAt`, and `entry?.endedAt` for non-subagent sessions. A stale persisted `running` row will therefore still be reported as running.
  - file: [src/gateway/session-utils.ts:1421](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/gateway/session-utils.ts#L1421)

I found no separate reconciliation path for runs that died without a terminal lifecycle event.
  - file: [src/gateway/session-lifecycle-state.ts:102](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/gateway/session-lifecycle-state.ts#L102)
