## Summary

Keep #45704 open. Current main still lacks the PR's central scrollbar fix: global UI CSS keeps 6px WebKit scrollbars, has no global Firefox `scrollbar-width`/`scrollbar-color`, and the config layout still contains `overflow: clip` on the same containers the PR changes. The related bug #45695 being closed is not enough to close this PR because the observable source changes are not present on main.

## Best Possible Solution

Keep the PR open or land an equivalent maintainer-owned CSS fix. The likely path is to add theme-aware scrollbar tokens, apply Firefox `scrollbar-width`/`scrollbar-color` at `:root`, decide the desired global scrollbar size, and remove or justify the config `overflow: clip` rules with manual browser QA for Chrome and Firefox.
