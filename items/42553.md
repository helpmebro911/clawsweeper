## Summary

Keep open. I found no evidence that current `main` fixes this report: the Signal inbound path still opens `/api/v1/events` with an optional `account` query, external-daemon mode still skips readiness checks and immediately enters the SSE loop, and there have been no Signal code changes since release `v2026.4.22`. The existing tests only mock `streamSignalEvents`; they do not cover a real `signal-cli` SSE stream or the reported external-daemon/account-scoped receive path.
