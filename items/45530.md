## Summary

Keep open. Current main still does not implement persistent TTS voice switching via `[[tts:... persist=true]]`: the directive result has no persistence field, speech-core prefs have no per-provider voice map, auto-reply and Discord voice paths pass directive overrides as one-shot synthesis overrides, and the public docs describe directives as single-reply overrides. The PR is stale against the current plugin-based TTS layout, but the requested behavior remains a coherent core TTS prefs/policy feature rather than a cleanup-close candidate.

## Best Possible Solution

Keep the PR open or ask the contributor/maintainer to rework it onto current main. The best implementation path is a small core TTS prefs feature in `extensions/speech-core/src/tts.ts` plus `src/tts/directives.ts`/provider override integration, with Discord voice and telephony behavior explicitly covered, docs for `persist=true`, and focused tests for parse policy, per-provider voice prefs, best-effort write failures, and current runtime call sites.
