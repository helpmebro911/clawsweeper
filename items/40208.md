## Summary

Keep open. Current `main` still exposes plugin load control only at gateway scope (`plugins.allow` / `plugins.deny` / `plugins.entries.*`) while per-agent scoping exists only for tool policy. I found no `agents.list[].plugins` or agent-scoped plugin lifecycle support in schema/docs, and plugin background services remain registered in the global plugin registry, so the requested feature is not implemented on `main`.

## Evidence

- **No per-agent plugin config surface:** Search for `agents.list[].plugins` and `agents.defaults.plugins` returned no matches under `src/`, `docs/`, `packages/`, or `extensions/`, which is consistent with the config schema lacking any per-agent plugin allow/deny section.
  - command: `rg -n "agents\.list\[\]\.plugins|agents\.defaults\.plugins" src docs packages extensions -g '!**/dist/**'`
  - sha: [799a42bd13de](https://github.com/openclaw/openclaw/commit/799a42bd13de586a1a7163cf0bd51673c31de253)
- **Plugin policy is documented as gateway-wide:** The config help only defines `plugins.allow` and `plugins.deny` as plugin-level load controls, with no agent-scoped variant.
  - file: [src/config/schema.help.ts:1087](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/src/config/schema.help.ts#L1087)
  - sha: [799a42bd13de](https://github.com/openclaw/openclaw/commit/799a42bd13de586a1a7163cf0bd51673c31de253)
- **Per-agent scoping exists for tools, not plugins:** Tool policy docs and implementation support per-agent `agents.list[].tools.allow` / `tools.byProvider.allow`, which matches the reporter's distinction between tool scoping and plugin lifecycle scoping.
  - file: [docs/gateway/config-tools.md:46](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/docs/gateway/config-tools.md#L46)
  - sha: [799a42bd13de](https://github.com/openclaw/openclaw/commit/799a42bd13de586a1a7163cf0bd51673c31de253)
- **Plugin services remain global registrations:** The plugin registry registers services into a shared `registry.services` collection keyed by plugin record, which is gateway-global rather than agent-specific.
  - file: [src/plugins/registry.ts:1091](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/src/plugins/registry.ts#L1091)
  - sha: [799a42bd13de](https://github.com/openclaw/openclaw/commit/799a42bd13de586a1a7163cf0bd51673c31de253)
- **Docs still describe plugin activation as global plugin registration:** Plugin docs state activation calls `register(api)` and can register routes, context engines, and background services, with enablement rules only at plugin/global slot scope.
  - file: [docs/tools/plugin.md:184](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/docs/tools/plugin.md#L184)
  - sha: [799a42bd13de](https://github.com/openclaw/openclaw/commit/799a42bd13de586a1a7163cf0bd51673c31de253)
