## Summary

Keep this PR open. Current main at 646a268d2710bc54e478349ef5690f3c8803df4f still lacks the embedded-runner egress guard for `tools: []` and orphaned `tool_choice`, while the PR is a focused candidate for the remaining strict-provider payload bug. The already-landed `toolsAllow` forwarding work is present on main, so maintainers should review this PR or a successor for the sanitizer and focused regression coverage rather than closing it as obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or a successor patch. The best fix is a narrow embedded-runner egress sanitizer that strips empty `tools` arrays and removes orphaned `tool_choice`, composes with existing payload hooks, covers session HTTP fallback plus boundary-aware/custom embedded stream paths, confirms provider-owned stream coverage, and adds focused regression tests. Do not redo the already-present `toolsAllow` forwarding work.
