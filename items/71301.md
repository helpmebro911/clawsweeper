## Summary

Keep this issue open. Current main has partial support: authored docs are bundled in the npm package, agents are pointed at local docs when available, and config field docs are exposed through `config.schema.lookup`. The core request remains unimplemented: there is no native local-first docs retrieval surface such as `docs.search`, `docs.get`, docs API/schema lookup beyond config fields, or changed-since tooling, and the CLI docs command still uses the live docs MCP endpoint.

## Best Possible Solution

Keep this issue open and narrow acceptance criteria to the missing core capability: ship a release/package docs manifest plus local search/get index over bundled docs, expose a local-first retrieval surface to agents, keep live Mintlify search as an explicit fallback, and add tests for package-installed docs discovery plus agent/tool behavior. A bundled skill or ClawHub component can be used only if it preserves version-coupled packaged docs and a reliable default retrieval path.
