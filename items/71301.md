## Summary

Keep open. Current main partially overlaps the request by packaging `docs/`, resolving a local docs path, and telling agents to consult local docs, but it does not yet provide the requested version-matched docs retrieval layer or native agent tools such as `docs.search`, `docs.get`, `docs.api`, or `docs.changed_since`. The existing `openclaw docs` command searches the live docs index, not a bundled local index.

## Best Possible Solution

Keep #71301 open as a core onboarding/docs capability request. The likely path is to build on the existing packaged `docs/` and `resolveOpenClawDocsPath` flow, add a release-time manifest/search index, expose read-only agent docs actions such as docs search/get/API or an equivalent gateway/tool surface, keep `config.schema.lookup` as the exact config-schema source, and make live Mintlify search an explicit fallback rather than the default for installed-version answers.
