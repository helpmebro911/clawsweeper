## Summary

Keep open. Current main has useful partial support: docs are included in the npm package, agents can resolve a local docs path, the system prompt tells agents to consult local docs first, and config schema lookup is available through the gateway tool. The broader request is still not fully implemented because there is no first-class local docs retrieval/search/get API for agents, and `openclaw docs` still searches the live docs index via docs.openclaw.ai.

## Best Possible Solution

Keep this open as a core agent/onboarding capability issue. Build on the existing bundled `docs/` packaging and local-docs prompt guidance by adding a local-first docs manifest/search/get retrieval surface for agents and `openclaw docs`, with live docs as explicit fallback and `config.schema.lookup` remaining the authoritative field-level config source.
