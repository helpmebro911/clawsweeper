## Summary

Keep #71301 open. Current main has useful partial implementation: the npm package now includes the docs tree, agents discover a local docs path, and prompts tell agents to consult local docs plus `config.schema.lookup`. However the requested native local docs retrieval layer is still missing: `openclaw docs` is explicitly a live-docs search wrapper around `https://docs.openclaw.ai/mcp.SearchOpenClaw`, and the agent/gateway tool surfaces do not expose local `docs.search`, `docs.get`, or equivalent bundled-doc retrieval actions.

## Best Possible Solution

Keep the issue open and narrow the implementation target: retain the current bundled docs and prompt guidance, then add a first-class local docs retriever over the bundled `docs/` tree, exposed to agents through a gateway/MCP/tool surface such as `docs.search` and `docs.get`, with local-first behavior and optional live docs fallback. Cover package-install docs discovery, generated-doc exclusions, and agent prompt/tool behavior in focused tests.
