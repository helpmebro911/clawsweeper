## Summary

Keep this issue open. Current main has partial bundled-docs support and prompt guidance, but the central request remains unimplemented: agents still lack a first-class, local-first, version-matched docs retrieval surface such as docs.search/docs.get, and the existing docs CLI searches the live docs index.

## Best Possible Solution

Keep the issue open and narrow the remaining work: add a release/package docs manifest and local search/get index over bundled versioned docs, expose local-first docs retrieval through the agent gateway/MCP/tool surface, keep live docs as an explicit fallback, and add focused tests for package-installed docs discovery plus agent/tool behavior.
