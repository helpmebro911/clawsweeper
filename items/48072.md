## Summary

Keep open. Current main still does not pre-create `/home/node/.openclaw` in the Docker image before switching to `USER node`, so the named-volume EACCES fix is not implemented. The setup-script chown path only covers the managed setup flow, and the PR’s latest patch addressed the review concern by setting mode `0700`, so this remains useful maintainer work rather than an auto-close candidate.

## Best Possible Solution

Keep #48072 open and consolidate the Docker named-volume fix with #61279 and #63959. The best implementation is a single Dockerfile change before `USER node`, preferably `install -d -o node -g node -m 0700 /home/node/.openclaw` or equivalent, plus changelog/docs guidance for existing root-owned volumes and a focused Docker named-volume smoke or source-level regression check.
