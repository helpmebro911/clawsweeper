## Summary

Keep open. Current main still hardcodes MiniMax VLM routing to `MiniMax-VL-01`, while the linked bug report has a concrete MiniMax-M2.7 image-understanding failure and direct endpoint workaround. A newer broader PR, #70422, overlaps this work but is still open and has unresolved concerns, so it is not a clear canonical supersession for this smaller focused PR.

## Best Possible Solution

Keep this PR open as a focused candidate, but require maintainer verification of the MiniMax VLM endpoint contract before merge. The best fix is either a narrow allowlist update for `MiniMax-M2.7` with regression tests and aligned docs, or a broader solution through #70422 that also resolves whether selected MiniMax model ids need to be forwarded to `/v1/coding_plan/vlm`.
