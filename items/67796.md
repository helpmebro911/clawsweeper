## Summary

Keep this PR open. Current main still has the Mattermost message-tool send path resolving media only from `params.media`, while the shared message tool schema and other channel handlers support `path`/`filePath` aliases. The PR is a focused implementation candidate, but it should handle blank `media` fallback correctly and add regression coverage before merge.

## Best Possible Solution

Keep the PR open as a plausible Mattermost message-tool bug fix. The best path is to revise the fallback to use normalized optional-string semantics for `media`, `filePath`, and `path`, add focused handleAction tests for both aliases and blank-media fallthrough, and coordinate separately with #52120 and #58439 for thread inheritance and mediaLocalRoots forwarding.

## Risks / Open Questions

- The PR's current fallback expression should be adjusted so each candidate is normalized independently; blank or whitespace `media` should fall through to `filePath` and then `path`.
- Regression tests should cover `filePath` and `path` through Mattermost `handleAction`, not only outbound `sendMedia`.
- The related Mattermost `threadId` and `mediaLocalRoots` work remains open in #52120 and #58439, so merging this PR alone would only solve the alias portion of #67784.
