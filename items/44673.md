## Summary

Keep open. Current main still lacks the PR's central user-facing guidance: the macOS VM docs only tell users to enable Remote Login before running the VM headlessly, and the LaunchAgent bootstrap error still points to signing into the desktop or using a custom LaunchDaemon without mentioning auto-login. The PR is stale against current docs layout because localized docs are no longer maintained here, but the underlying English docs/source change remains useful rather than obsolete.

## Best Possible Solution

Refresh PR #44673 rather than close it: add the auto-login guidance to `docs/install/macos-vm.md` in the post-setup/headless VM flow, add the same hint to `throwBootstrapGuiSessionError`, update the launchd guidance test, and drop the obsolete localized docs edit in favor of the current English-docs/i18n workflow.
