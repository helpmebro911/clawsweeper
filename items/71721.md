## Summary

Keep #71721 open. Current main still has direct OpenClaw LAN/tailnet `os.networkInterfaces()` call paths that propagate sandbox failures, and the reported `NetworkManager.assumeNetworkInterfaceNames` stack also matches the bundled Bonjour `@homebridge/ciao` dependency path. The linked PR #71950 is relevant prior work, but it is not landed on main and appears to cover only part of the crash surface.

## Best Possible Solution

Keep #71721 open. The best fix is a maintainer-reviewed gateway hardening change that makes restricted-interface enumeration non-fatal across both OpenClaw-owned LAN/tailnet helpers and the bundled Bonjour/ciao startup path, with regression tests that assert gateway startup/discovery gracefully degrades when `os.networkInterfaces()` throws `uv_interface_addresses` errors. PR #71950 can be reused if expanded or paired with a ciao/Bonjour guard.
