## Summary

Keep open. Current main still lets the reported `os.networkInterfaces()`/`uv_interface_addresses` failure propagate through the gateway LAN and tailnet discovery helpers, and the regression tests still assert thrown errors rather than graceful fallback. The linked PR #71950 is directly relevant, but the provided GitHub context shows it is open and unmerged, so it has not fixed the issue on main.

## Best Possible Solution

Keep #71721 open until PR #71950 or an equivalent maintainer-reviewed fix lands. The best code path is to make gateway LAN and tailnet discovery use `safeNetworkInterfaces()` or an equivalent safe wrapper, then update `src/gateway/net.test.ts` and `src/infra/tailnet.test.ts` to assert graceful fallback instead of thrown `uv_interface_addresses` errors.
