## Summary

Keep open. Current `main` at `d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132` still does not expose structured progress metrics on the requested hook events, so the issue remains a valid, actionable feature request rather than cleanup material.

## Evidence

- **Hook event types still lack metrics fields:** `PluginHookAgentEndEvent` only has `messages`, `success`, `error`, and `durationMs`; `PluginHookBeforeToolCallEvent` only has `toolName`, `params`, `runId`, and `toolCallId`; `PluginHookAfterToolCallEvent` only adds `result`, `error`, and `durationMs`. There is no `metrics` object or equivalent structured progress payload.
  - file: [src/plugins/hook-types.ts:204](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/src/plugins/hook-types.ts#L204)
