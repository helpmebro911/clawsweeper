## Summary

Current main still only partially satisfies the requested recovery guidance. Direct gateway startup invalid-config errors suggest `openclaw doctor --fix`, but they still do not mention the requested `openclaw.json.bak` rollback path. The daemon start/restart preflight paths still use generic `openclaw doctor` wording and omit backup recovery guidance.

## Best Possible Solution

Keep this issue open and make a narrow UX fix across the gateway invalid-config startup surfaces. Centralize the recovery hint, use it from direct gateway startup validation and daemon start/restart preflight, mention `openclaw doctor --fix` plus an explicit `openclaw.json.bak` fallback where appropriate, and extend the gateway recovery and daemon config-guard tests to assert both recovery steps. Add a concise changelog entry when the user-facing text changes.
