## Summary

Keep this issue open. Current main still only partially satisfies the requested invalid-config recovery guidance: direct gateway startup mentions `openclaw doctor --fix`, but it still omits an explicit `openclaw.json.bak` rollback hint, and daemon start/restart invalid-config preflight still recommends generic `openclaw doctor` without `--fix` or backup recovery guidance.

## Best Possible Solution

Keep this open and make a narrow UX fix across the invalid-config startup surfaces. Centralize the recovery hint if practical, update direct gateway startup validation plus daemon start/restart preflight to recommend `openclaw doctor --fix` and the appropriate backup fallback, and extend the existing gateway recovery and daemon config-guard tests to assert both recovery steps.
