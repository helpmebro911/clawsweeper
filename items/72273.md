## Summary

Keep #72273 open. Current main still hardcodes the Discord gateway startup READY timeout at 15s and runtime READY timeout at 30s, with no env/config override wired into the lifecycle path. The issue is recent, has a concrete multi-account reproduction, and the requested capability is not already covered by the existing Discord eventQueue or inboundWorker timeout knobs.

## Best Possible Solution

Add this as an extension-owned Discord configuration change, preferably under the existing `channels.discord` / `channels.discord.accounts.<accountId>` surface rather than `gateway.discord`: define validated `gateway.readyTimeoutMs` and `gateway.runtimeReadyTimeoutMs` (or similarly named Discord-local fields), normalize/clamp them in a small helper, thread them into `runDiscordGatewayLifecycle`, update lifecycle tests for configured/default behavior, and document the new knobs in the Discord channel and config reference docs. Consider raising the default startup timeout to account for `DISCORD_ACCOUNT_STARTUP_STAGGER_MS` when multiple enabled accounts are configured.
