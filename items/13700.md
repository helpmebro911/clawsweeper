## Summary

Keep #13700 open. Current main has adjacent compaction-checkpoint branch/restore and native subagent transcript forking, but it still lacks the requested arbitrary named session snapshot surface: no `/session save`, `/session load`, `/session restore`, named snapshot Gateway API, reusable template flow, or restore path for `.reset` / `.deleted` transcript archives. The latest provided issue comment also explicitly kept this item open, and the only commit after that reviewed SHA touches WhatsApp files, not session snapshot code.

## Best Possible Solution

Keep #13700 open until OpenClaw either implements a deliberate native session snapshot surface or splits it into canonical scoped issues. A strong path would add typed Gateway snapshot save/list/load/delete methods, slash/TUI/CLI controls, deterministic transcript/model/session metadata preservation where feasible, spawn-from-snapshot support for reusable batch work, and a safe restore path for `.reset` / `.deleted` transcript archives.
