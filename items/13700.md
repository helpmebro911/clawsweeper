## Summary

Keep #13700 open. Current main has adjacent partial capabilities, especially native `sessions_spawn` with `context:"fork"` and Gateway compaction checkpoint branch/restore, but it still lacks the requested user-facing native session snapshot surface: no `/session save`, `/session load`, named snapshot API, reusable session template, or restore path for `.reset`/`.deleted` transcript archives.

## Best Possible Solution

Keep #13700 open until OpenClaw implements a deliberate native session snapshot surface or splits the work into canonical follow-ups. A strong path is typed Gateway methods for named snapshot save/list/load/delete, slash/TUI controls, model/session metadata preservation where deterministic, and a safe restore path for `.reset`/`.deleted` transcript archives. Treat `sessions_spawn context:"fork"` and compaction checkpoint branch/restore as related partial capabilities, not closure of this issue.
