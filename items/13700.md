## Summary

Keep #13700 open. Current main still has adjacent compaction-checkpoint branch/restore and native subagent `context: "fork"`, but it does not expose the requested native `/session save`, `/session load`, `/session restore`, named arbitrary snapshot API, reusable session template, or restore path for `.reset` / `.deleted` transcript archives. The latest provided maintainer/Codex issue comment also explicitly kept this item open.

## Best Possible Solution

Keep #13700 open until OpenClaw either implements a deliberate native session snapshot surface or splits the request into canonical scoped follow-ups. A strong implementation path would add typed Gateway snapshot save/list/load/delete methods, slash/TUI controls, deterministic transcript/model/session metadata preservation where feasible, spawn-from-snapshot support for reusable batch work, and a safe restore path for `.reset` / `.deleted` transcript archives.
