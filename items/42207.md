## Summary

Keep open. Current `main` does not implement this hardening: `tools.fs.workspaceOnly` still defaults to `false`, current tests and schema text still codify that default, and the host `read` path is still created without any denylist guard when workspace restriction is off. The PR discussion also surfaced a real gap in the proposed patch: even if merged as written, it would still leave the standalone `read` tool unguarded in the `workspaceOnly=false` path.
