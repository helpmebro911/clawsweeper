## Summary

Keep open. Current main still matches the reported failure mode: a missed terminal chat/lifecycle event can leave Control UI busy on a stale chatRunId after embedded timeout/abort. The later #69747 fix helps lifecycle-error events that do arrive, but it does not synthesize a terminal event from the embedded timeout path or add the Control UI agent.wait watchdog tracked by #42213/#42258.

## Best Possible Solution

Keep this issue open, or explicitly consolidate it into #42213/#42258 if maintainers want one tracker. The robust fix should guarantee a terminal chat/lifecycle state for embedded timeout/abort even when `agent_end` is missed, and add a Control UI recovery path such as an `agent.wait`-backed stale-run watchdog so `chatRunId` cannot keep Stop and queued sends blocked indefinitely.
