## Summary

Current main still leaves the reported failure mode plausible: Control UI busy/Stop state is driven by a local chatRunId, that id is cleared only by terminal chat events or reconnect/reset paths, and the embedded timeout path still aborts the active session without directly synthesizing a terminal chat event if agent_end is missed. The merged #69747 fix registers chat.send runs so lifecycle errors that do arrive are surfaced, but it does not close the missing-terminal-event edge case described here.

## Best Possible Solution

Keep this issue open until maintainers either explicitly consolidate it into #42213/#42258 or land a fix that guarantees terminal chat/lifecycle state for embedded timeout/abort when agent_end is missed. The best implementation is a backend terminal-event guarantee plus a Control UI stale-run recovery path, likely using agent.wait before clearing chatRunId and flushing queued messages, with regression coverage for timeout/abort terminal emission and connected UI recovery.
