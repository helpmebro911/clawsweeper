## Summary

Keep this issue open. Current main still has only targeted Kimi Anthropic and OpenAI-compatible malformed tool-argument repair, while the reported supported custom/local `copilot-api` `anthropic-messages` path is not shown fixed or covered by tests.

## Best Possible Solution

Keep this issue open. Add a focused regression or live repro for a custom/local `anthropic-messages` provider streaming large fragmented `input_json_delta` tool arguments after long context, especially large CJK `write` content and heredoc-style `exec` commands. If confirmed, extend guarded argument repair or add explicit parse-failure diagnostics before empty arguments reach tool validation; treat #53591 and #53735 as related Kimi-specific fixes rather than the canonical resolution for this report.
