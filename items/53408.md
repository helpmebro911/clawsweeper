Keep this issue open. Current main has targeted Kimi and OpenAI-compatible malformed tool-argument repair, plus narrower client-tool string coercion, but I did not find evidence that the reported custom/local `copilot-api` `anthropic-messages` path is fixed or covered by tests.

## Best Possible Solution

Keep this issue open. Add a focused regression or live repro for a custom/local `anthropic-messages` provider streaming large fragmented `input_json_delta` tool arguments after long context, especially large CJK `write` content and heredoc-style `exec` commands.
