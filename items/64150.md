## Summary

Keep this PR open. Current main still lacks local `find`/`grep`/`ls` tool registration and still maps Anthropic OAuth Claude Code tool names only by case, so the reported `Glob` dispatch gap remains. The PR addresses a real path, but its implementation is not merge-ready because it adds filesystem discovery tools without the existing workspace/sandbox and tool-policy catalog integration.

## Best Possible Solution

Keep this PR open as a real but incomplete implementation candidate. The best path is a focused revision or successor that uses the dependency's current factory API, wraps `find`/`grep`/`ls` through OpenClaw's workspace/sandbox filesystem policy, updates `tool-catalog` and public tools docs, makes an explicit policy decision for any external `fd`/`rg` acquisition/execution path, and adds regression tests for Anthropic OAuth `Glob` -> `find`, `Grep` -> `grep`, `LS` -> `ls`, plus `group:fs` allow/deny behavior.
