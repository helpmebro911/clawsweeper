## Summary

Keep this PR open. Current main still does not implement the requested Anthropic OAuth `Glob`/`Grep`/`LS` dispatch path, and the cross-referenced OAuth compatibility PR was closed unmerged rather than superseding it. The submitted patch addresses a real gap, but it is not merge-ready: it imports stale/nonexistent dependency symbols and adds filesystem discovery tools without OpenClaw's workspace/sandbox and tool-policy integration.

## Best Possible Solution

Keep this PR open as a real but incomplete implementation candidate. The best path is a revised patch that uses the pinned dependency's factory API (`createFindTool`, `createGrepTool`, `createLsTool`), wraps these tools through OpenClaw's workspace/sandbox filesystem policy, adds them deliberately to the core tool catalog and docs if maintainers want them in `group:fs`, and adds regression tests for Anthropic OAuth `Glob` -> `find`, `Grep` -> `grep`, `LS` -> `ls`, plus allow/deny behavior for the new filesystem tools.
