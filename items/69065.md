## Summary

Keep open. Current main still has no Android/Termux gateway service adapter: gateway service resolution only supports macOS launchd, Linux systemd, and native Windows Scheduled Tasks, and unsupported platforms still throw. The foreground gateway command exists and Android docs now describe Android as a companion node rather than a Gateway host, but the docs do not resolve the Termux daemon/background-service request or give Android-specific fallback guidance.

## Best Possible Solution

Keep #69065 open until maintainers choose and implement the Android/Termux product path: either add a real Termux-compatible gateway service adapter, add a supported non-systemd daemon/foreground mode with clear CLI guidance, or explicitly document Android/Termux Gateway hosting as unsupported with an actionable `openclaw gateway run` workaround. Add regression coverage for `process.platform === "android"` so install/start/status produce the intended behavior instead of the generic unsupported-platform throw.
