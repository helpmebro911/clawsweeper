## Summary

Keep open. Current main partially recognizes `GOOGLE_CLOUD_API_KEY` for `google-vertex`, but it does not satisfy the PR's full user-visible and security scope: the key is not in the provider env-var/scrubbing registry, shell-env fallback derives from that registry, and the docs still only document Vertex ADC.

## Best Possible Solution

Keep the PR open or replace it with an equivalent maintainer patch that adds `GOOGLE_CLOUD_API_KEY` to the provider auth/env-var metadata used for scrubbing and shell fallback, updates the Vertex provider docs, and adjusts tests to cover detection plus registry-backed scrubbing/fallback behavior on current main.
