## Summary

Keep #20934 open. Current main still does not expose REST-native /v1/sessions reset/status/compact/model/reasoning endpoints for HTTP chat-completions integrations. The Gateway already has the underlying RPC/session operations, and adjacent HTTP kill/history plus tools/invoke surfaces exist, but they do not satisfy the requested standardized session-management REST API. Related PR #23941 was closed unmerged, and #29985 explicitly scoped REST endpoints out.

## Best Possible Solution

Keep #20934 open as the canonical REST session-management tracker. The best product/code path is to add authenticated Gateway HTTP endpoints for reset, status/list-or-resolve, compact, and model/reasoning patch operations that reuse sessions.reset, sessions.list/resolve, sessions.compact, and sessions.patch semantics, with route/auth/scope tests and public docs under the OpenAI-compatible HTTP API.
