## Summary

Keep open. Current `main` still does not expose the requested REST session-management endpoints for the OpenAI-compatible HTTP surface. Session control exists on the gateway RPC/WebSocket side (`sessions.patch`, `sessions.reset`, `sessions.compact`, etc.), but the documented and implemented HTTP compat surface is still limited to `/v1/chat/completions`, `/v1/models`, `/v1/embeddings`, and `/v1/responses`. The report remains actionable and not obsolete.
