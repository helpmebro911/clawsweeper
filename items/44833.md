## Summary

Current `main` still appears to have the reported gap. Telegram inbound handling builds a media placeholder for captionless photo messages, but mention matching only evaluates `messageTextParts.text` (plus audio transcript), so group `mentionPatterns` do not activate on photo-only messages. An existing test on current `main` already shows analogous media-only group input being dropped with `reason: "no-mention"`.

## Evidence

- **Media placeholder is built but not used for mention matching:** `resolveTelegramInboundBody` sets `rawBody` to the media placeholder when there is no text, but later computes `computedWasMentioned` from `messageTextParts.text` rather than `rawBody`/`bodyText`. That leaves captionless photo messages with no text signal for group mention gating.
  - file: [extensions/telegram/src/bot-message-context.body.ts:176](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/telegram/src/bot-message-context.body.ts#L176)
  - command: `nl -ba extensions/telegram/src/bot-message-context.body.ts | sed -n '170,275p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Mention helper short-circuits on empty text:** `matchesMentionWithExplicit` normalizes the message text and returns only the explicit-mention result when `textToCheck` is empty, so regexes such as `.*` are never applied to captionless photo messages.
  - file: [src/auto-reply/reply/mentions.ts:162](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/auto-reply/reply/mentions.ts#L162)

- **Current test suite already documents analogous no-mention drop:** The existing Telegram unit test verifies that a group voice-only message with `requireMention: true` and configured `mentionPatterns` is skipped and logged as `no-mention`, which matches the same gating behavior affecting photo-only group messages.
