## Summary

Keep open. Current `main` does not implement this PR's sandbox-skill refresh behavior: embedded sandbox runs still skip reloading skill entries when a snapshot already has `resolvedSkills`, and `resolveSkillsPromptForRun` still returns the cached snapshot prompt verbatim whenever it exists. For copied sandbox workspaces, that means the model can still see pre-sandbox skill paths instead of the synced workspace-local `SKILL.md` paths.
