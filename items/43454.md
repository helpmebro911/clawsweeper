## Summary

Current main still does not expose the requested `subagent:ended`, `agent:turnComplete`, or tool-call-threshold events through the workspace/internal `HOOK.md` hook system. Main has adjacent Plugin SDK hooks, but those are not the same public surface as the internal workspace hook API this issue is asking maintainers to expand.

## Best Possible Solution

Keep this issue open for maintainer API judgment. The concrete path is either to add a narrow internal-hook bridge for `subagent:ended` plus well-defined `agent:turnComplete` and tool-count lifecycle events, or to make an explicit product decision that these workflows should use Plugin SDK hooks such as `subagent_ended`, `after_tool_call`, and `agent_end`, with docs updated so workspace hook authors are not left guessing.

## Risks / Open Questions

- Expanding workspace/internal hooks overlaps the existing Plugin SDK hook surface, so maintainers should decide whether these lifecycle events belong in `HOOK.md` internal hooks, plugin hooks, or both.
- If implemented, payload shape, event ordering, mutation rights, and any message-injection behavior become user-visible contracts and need explicit API review.
