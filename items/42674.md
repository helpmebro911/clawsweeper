## Summary

Current `main` does not implement this PR’s inline `buffer`/`filename`/`contentType` path for Telegram sends. The shared message tool still advertises those send parameters, but the Telegram action/runtime path only consumes `media`/`mediaUrl`, and `sendMessageTelegram` still only loads outbound media from URL/path. This remains a live gap rather than conservative-close material.
