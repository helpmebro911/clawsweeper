## Summary

Keep open. Current `main` still retries compaction up to 3 times for each chunk with no cross-chunk circuit breaker or shared failure state, so the reported token-burn path remains plausible. The two linked PRs were closed, and the reviewed implementation never landed on `main`.

## Evidence

- **Current compaction loop still retries per chunk:** `summarizeChunks()` iterates every chunk and wraps each `generateSummary()` call in `retryAsync()` with `attempts: 3`. There is no circuit-breaker state, no consecutive-failure cap, and no early bailout between chunks.
