## Summary

Keep open. Current main still contains the gateway model-pricing refresh behavior implicated by #72208: gateway startup still starts the automatic pricing refresh, the OpenRouter and LiteLLM fetches only use timeout signals, the 24-hour refresh timer is not unref'd, and the stop callback only clears the scheduled timer rather than aborting an in-flight refresh. The cross-referenced #72306 is directly relevant and appears to be the right fix direction, but it is still open/unmerged in the provided context, so the reported regression is not implemented on main yet.

## Best Possible Solution

Keep #72208 open until a fix like #72306 lands on main and is verified. The best path is to unref the pricing refresh timer, abort in-flight LiteLLM/OpenRouter pricing fetches from the gateway stop callback, add deterministic unit coverage for both behaviors, and preferably verify Windows npm-global gateway startup, dashboard reachability, and Ctrl+C shutdown under pricing-fetch timeout conditions.
