## Summary

Keep this issue open. Current main still lacks the requested lightweight, label-scoped subagent instruction files: no workspace/user `subagents/<label>.md` lookup, no YAML frontmatter role metadata, no role-body injection before native `sessions_spawn`, and no ordinary pre-spawn plugin/API contract that can mutate child prompt, model, and tool policy together.

## Best Possible Solution

Keep this open for maintainer product/API design. The strongest path is either a narrow native resolver for workspace/user-scoped `subagents/<label>.md` files applied before native spawn, or a documented pre-spawn plugin hook that can mutate the pending child system context, model/thinking, and constrained tool policy. Add docs and focused tests for lookup precedence, explicit-param precedence, invalid/malicious frontmatter, and ordinary `sessions_spawn` behavior.

## Risks / Open Questions

- A plugin can approximate prompt text with `before_tool_call` or `before_prompt_build`, but that does not provide a documented label-file resolver, frontmatter schema, precedence rules, or hard tool-policy override for ordinary native spawns.
- The eventual design needs maintainer decisions on precedence between explicit spawn params, per-agent config, global subagent defaults, role-file frontmatter, plugin hooks, and `tools.subagents` security policy.
