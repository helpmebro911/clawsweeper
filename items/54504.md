## Summary

Keep this issue open. Current main still lacks persistent label-scoped markdown role files or frontmatter-driven model/tool policy for `sessions_spawn`; it only supports explicit per-spawn fields, full configured agents, attachments, and broader prompt/model hooks.

## Best Possible Solution

Keep this open for maintainer product/API design. The best path is either a narrow core resolver for workspace/user-scoped subagent role markdown files with documented precedence and frontmatter schema, or a new explicit spawn-time plugin API that can mutate prompt, model, thinking, and tool policy before the child run is launched.

## Risks / Open Questions

- Closing as ClawHub now would be premature because current hooks cannot safely apply label-scoped frontmatter and tool policy to a pending `sessions_spawn` call.
- Any implementation needs maintainer design for trust, precedence, cache stability, and tool-policy ordering between workspace-scoped files, user-scoped files, agent config, and explicit spawn overrides.
