## Summary

Keep open. Current main still does not fully guard the reported user-visible leak: the shared downgraded-tool-call sanitizer still returns bare OpenAI-style `{"tool_calls":[...]}` JSON unchanged, and the Telegram/direct partial streaming path still forwards parsed visible text to `onPartialReply` without a streaming-aware tool-call JSON sanitizer. The related first fix attempt (#71954) is closed unmerged, while #72011 is open and appears to be the active fix path for this issue.

## Best Possible Solution

Keep #70797 open until #72011 or an equivalent maintainer-approved fix lands. The fix should sanitize standalone OpenAI-style `tool_calls` JSON before user-visible partial preview delivery and final visible-text delivery, including incomplete-fragment handling for streaming, while preserving ordinary JSON and inline/fenced code examples. Regression coverage should include `src/shared/text/assistant-visible-text.test.ts` and the `handleMessageUpdate` partial-reply path in `src/agents/pi-embedded-subscribe.handlers.messages.test.ts`.
