## Summary

Issue #70797 should stay open. Current main still has the reported bare `{"tool_calls":[...]}` leak class: `stripDowngradedToolCallText()` returns early unless marker text such as `[Tool Call` or `[Historical context` is present, and the Telegram partial-preview path forwards streamed `cleanedText` to `onPartialReply` without a bare tool-call JSON sanitizer. The linked #71954 PR was closed unmerged, and #72011 is an active fix attempt, not shipped behavior.

## Best Possible Solution

Keep #70797 open as the canonical bug. The best current path is to land a focused fix like #72011: add a streaming-aware sanitizer before `onPartialReply` in `src/agents/pi-embedded-subscribe.handlers.messages.ts`, keep or add shared final sanitizer coverage for bare OpenAI-style `tool_calls` JSON, and include regression tests proving partial Telegram previews with incomplete and complete tool-call JSON never expose the raw payload while ordinary JSON remains intact.
