## Summary

Keep open. Current `main` and the latest shipped release still generate the Windows auto-update restart batch file with an unconditional `schtasks /End /TN ...` before any port polling or fallback, and the tests still lock that behavior in. I did not find evidence of a fix on `main` or in `v2026.4.23`.

## Evidence

- **Current main still emits unconditional `schtasks /End` in the update restart batch script:** The Windows branch of `prepareRestartScript()` writes `schtasks /End /TN "${taskName}"` directly into the generated `.bat`, then only afterwards enters the port-release polling loop. There is no task-state guard or timeout wrapper in this script path.
