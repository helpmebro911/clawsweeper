## Summary

Keep open. Current main still renders the Control UI Sessions table around the raw session key, optional custom label, and generic kind badge; it does not show a channel icon/name or derived channel label for Telegram/Feishu-style session keys. The related closed work covers chat dropdown/displayName behavior, not this Sessions list request.

## Best Possible Solution

Implement this in the Control UI Sessions page: derive a stable channel display from GatewaySessionRow.channel, origin.provider, and session-key parsing; show a compact channel name/icon or badge next to the key or in a dedicated Channel column; keep the existing editable Label field; and add focused ui/src/ui/views/sessions.test.ts coverage for Telegram and Feishu direct/group rows.
