## Summary

Current main still does not implement the requested Control UI Sessions list channel identity. The Sessions table renders Key, Label, Kind, Updated, Tokens, Compaction, Thinking, Fast, Verbose, and Reasoning, with the raw session key as the primary row text; backend session rows already expose channel/origin metadata, but the UI row type and websocket update allowlist still omit those fields. The related items in the provided context cover chat dropdown labels or the Updated date column, not this Sessions table channel-name/icon request.

## Best Possible Solution

Keep this issue open. Implement the feature in the Control UI Sessions view by exposing channel/origin metadata through ui/src/ui/types.ts and ui/src/ui/controllers/sessions.ts, then render a compact Channel column or key-cell badge using row.channel/origin with a session-key parsing fallback. Preserve the editable Label field and add focused Sessions view coverage for Telegram and Feishu direct/group rows.
