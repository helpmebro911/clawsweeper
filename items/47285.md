## Summary

Keep #47285 open. Current main still does not implement native Azure OpenAI handling in the bundled memory-lancedb plugin: the embedding client only passes apiKey/baseURL to the OpenAI SDK, and the plugin config/manifest still reject any Azure API-version setting. The PR is a focused change for a real missing surface, but it should not be merged as-is because review already identified unaddressed gaps around configurable api-version and Azure AI Foundry host detection.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up, but require a revised implementation before merge: add Azure endpoint detection for all accepted Azure OpenAI/Foundry host families, add an optional configurable embedding apiVersion through runtime config and manifest schema, preserve non-Azure OpenAI-compatible behavior, and cover the OpenAI client options/config parsing with focused memory-lancedb tests. If maintainers decide Azure-specific LanceDB memory should live outside the bundled plugin, close manually with a ClawHub/plugin direction after that product judgment.
