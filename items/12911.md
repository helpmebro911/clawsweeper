## Summary

Keep #12911 open. Current main has browser Talk in /chat, but it remains OpenAI Realtime/WebRTC-specific and does not implement the requested Grok/xAI Live Voice Mode. The issue discussion also has recent maintainer comments explicitly keeping the xAI/Grok scope open.

## Best Possible Solution

Keep #12911 open as the tracker for Grok/xAI Live Mode in /chat. The best path is to make browser Talk provider-aware, add xAI realtime voice/browser-session support if the upstream API supports it, wire it through the bundled xAI plugin or an explicit plugin API extension, and update Control UI docs/tests. If maintainers decide OpenAI Talk satisfies the product direction, close only after that explicit product decision.
