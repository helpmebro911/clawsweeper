## Summary

Keep #30389 open. Current main partially satisfies the request with a visible WebChat attachment button and a session model selector, but the upload path remains image-only and the requested composer-visible skill/capability menu is not implemented beyond typed slash-command completion. The related open document-upload trackers #30759 and #70438 still match the remaining WebChat file/document work.

## Best Possible Solution

Keep #30389 open unless maintainers deliberately split it. Preserve the shipped attachment button and model picker, then complete WebChat document/file upload end to end through UI accept/validation/previews, chat.send serialization with filenames and non-image payload shape, Gateway parsing/offload/extraction, provider/tool handling, docs, and tests. Separately define a composer-visible skill/capability picker that maps to skills, tools, agents, task profiles, or model profiles instead of relying only on typed slash commands.
