## Summary

Keep open. Current `main` partially covers the request: WebChat now has a paperclip attachment button and a session model selector, but attachment support is still image-only and there is no chat-integrated skill/capability menu in the composer/header. That is not strong enough for an `implemented_on_main` close.

## Evidence

- **WebChat composer has an attachment button and hidden file input:** The chat input renders a file input plus a paperclip button labeled `Attach file` in the composer.
  - file: [ui/src/ui/views/chat.ts:1205](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/views/chat.ts#L1205)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Attachment support is image-only:** The accepted attachment types are restricted to `image/*`, and MIME validation only allows `image/` types.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/chat/attachment-support.ts#L1)

## Risks / Open Questions

- The issue bundles three separate asks. Two are present in some form on `main`, so future triage should likely narrow the open work to non-image uploads and any desired chat-native skill/capability picker.
- Mobile chat controls appear to expose session and thinking controls in the gear menu, but not the chat model selector there; if mobile parity matters, that may be an additional gap beyond the original request.
