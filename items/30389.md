## Summary

Keep #30389 open. Current main implements a header model selector and a visible attachment button, but WebChat attachments remain image-only end to end and the requested composer-visible skill/capability menu is still only covered by typed slash-command completion. Related open issues #30759 and #70438 cover narrower non-image upload work, but they do not cover the full skill/capability picker request.

## Best Possible Solution

Keep #30389 open unless maintainers deliberately split it. Preserve the shipped attachment button and model selector, complete non-image WebChat uploads through UI accept/validation/previews, chat.send serialization with filenames/document payloads, gateway parsing/offload/extraction, provider/tool handling, docs, and tests, and separately define a composer-visible capability picker backed by commands.list, skills, tools, agents, task profiles, or model profiles. Coordinate the document upload portion with #30759 and #70438 rather than duplicating that work.
