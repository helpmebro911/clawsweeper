## Summary

Keep this issue open. Current main still forwards the configured compaction reserve budget into built-in Pi summarization without clamping it to the selected model output ceiling, and same-author PR #54392 remains open/unmerged as the implementation candidate for this exact bug.

## Best Possible Solution

Review and either land PR #54392 or replace it with an equivalent maintainer-approved fix in the compaction owner path. The fix should clamp built-in compaction summarization reserve/output budgeting against the selected model's maxTokens before calling Pi generateSummary, and add regression coverage proving an Anthropic Claude 4.6 1M-context setup with reserveTokensFloor=300000 cannot request max_tokens above 128000.
