## Summary

Keep open. Current main still forwards the configured compaction reserve into Pi summarization without clamping it to the selected model output cap, and same-author PR #54392 remains open and unmerged as the implementation candidate. Related #61852 was closed as a duplicate of this remaining work, not as implemented.

## Best Possible Solution

Keep this issue open and route the remaining work through review/update/landing of PR #54392 or an equivalent maintainer-approved patch. The implementation should clamp built-in compaction summarization reserve/output budgeting against the selected model's maxTokens before Pi generateSummary/completeSimple, preserve headers and safeguard paths, and add regression coverage proving an Anthropic Claude 4.6 1M-context setup with reserveTokensFloor=300000 cannot request max_tokens above 128000.
