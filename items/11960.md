## Summary

Keep #11960 open. Current main still implements only the secondary hook-config schema fix; the primary request remains unsolved because automatic daily/idle stale rollovers do not emit the internal command reset hook that bundled session-memory listens to. The issue is actively tracked by open PRs #71792 and #61675, and the existing maintainer/Codex comment already reached the same keep-open conclusion for this exact gap.

## Best Possible Solution

Keep #11960 open until the automatic daily/idle stale rollover path emits the existing reset hook behavior, or an equivalent documented `session:reset`/`before_reset` surface, with `previousSessionEntry` available to bundled session-memory. The narrow maintainer path is to land or replace PR #71792/#61675 with regression coverage for daily and idle rollovers, then update the session-memory metadata/docs if the shipped behavior changes.
