## Summary

Keep #11960 open. Current main still fixes only the secondary hook-config schema issue; the primary request remains unsolved because automatic daily/idle stale rollovers still do not emit the internal command reset hook that bundled session-memory listens to. The linked implementation PRs #71792 and #61675 are still open in the provided GitHub context, so the issue should remain as the canonical tracker until one lands or is replaced.

## Best Possible Solution

Keep #11960 open until PR #71792, PR #61675, or a replacement lands a narrow fix that fires the existing reset hook behavior, or an equivalent documented session reset surface, during automatic daily and idle stale rollover. The fix should pass previousSessionEntry to session-memory, preserve the typed session_end/session_start behavior, and add regression coverage for both daily and idle rollover so memory persistence cannot regress.
