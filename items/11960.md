## Summary

Current main still does not implement the requested `session:reset`/`session:before-reset` internal hook event, nor does the bundled `session-memory` hook persist on automatic daily/idle rollovers. The secondary hook-config schema mismatch is fixed, and there is adjacent plugin lifecycle/before-reset support plus open implementation PRs, but the core user problem remains unshipped.

## Best Possible Solution

Keep #11960 open until OpenClaw ships automatic daily/idle reset persistence for the bundled `session-memory` hook, either by adding a supported internal reset event/contract for stale rollovers or by making a maintainer-approved plugin `before_reset`/`session_end` path the documented replacement. Treat the hook-config schema portion as fixed, but do not close the issue while the auto-reset persistence path remains missing on main.
