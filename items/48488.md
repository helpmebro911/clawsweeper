## Summary

Keep #48488 open. Current main still awaits lane tasks directly with no task-level timeout or configurable taskTimeoutMs option, so a never-settling task can still keep an active slot forever. The linked PR #48690 appears to be the relevant pending implementation, but it is open and not present in the checked-out main, so the issue should not be closed as implemented.
