## Summary

Keep open. Current main still matches the reported failure mode: Control UI Usage requests `sessions.usage` and `usage.cost`, but list-mode session usage only scans gateway-visible configured agents, while `agents.delete` removes the agent config and purges session-store entries even when `deleteFiles=false`. The linked KeepMyClaw comment is a useful backup/workaround discussion, and PR #51327 directly targets this bug, but that PR is still open/unmerged in the provided context, so it is not closure evidence for #50553.

## Evidence

  - file: [ui/src/ui/controllers/usage.ts:180](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/ui/src/ui/controllers/usage.ts#L180)

- **sessions.usage only discovers configured agents:** `discoverAllSessionsForUsage` gets `listAgentsForGateway(params.config).agents` and calls `discoverAllSessions` for each returned agent. A deleted agent that was pruned from config is therefore skipped in list-mode usage discovery.
  - file: [src/gateway/server-methods/usage.ts:305](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/gateway/server-methods/usage.ts#L305)
