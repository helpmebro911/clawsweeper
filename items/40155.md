## Summary

Keep this issue open. Current main still has plausible WebUI-vs-Telegram divergence for skill-backed tool availability: Telegram passes channel/topic skill filters into the shared reply path, while WebUI chat.send uses the internal channel path without Telegram skillFilter. I did not find source, tests, changelog, history, or discussion evidence that the reported Telegram + vLLM + himalaya mismatch has been fixed.

## Best Possible Solution

Keep this issue open and reproduce on current main with the same agent/session, vLLM gpt-oss-20b model, and himalaya skill through both WebUI chat.send and Telegram. Capture the resolved skill snapshot, effective tool inventory, model compat, messageProvider/channel metadata, and replyOptions for both paths. If Telegram loses himalaya or exec availability, fix the Telegram context/skillFilter/tool-policy path and add a regression test around Telegram dispatch; if it no longer reproduces with that evidence, close later as cannot_reproduce or implemented_on_main.
