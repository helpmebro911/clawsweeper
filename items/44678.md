## Summary

Keep open. Current `main` does not implement explicit native Ollama `params.num_ctx` overrides. Native `/api/chat` requests still seed `options.num_ctx` from `model.contextWindow` in `extensions/ollama/src/stream.ts:626-659`, the Ollama wrapper only injects `num_ctx` for OpenAI-compatible Ollama models in `extensions/ollama/src/stream.ts:125-197`, and the generic extra-params wrapper in `src/agents/pi-embedded-runner/extra-params.ts:243-303` does not map `num_ctx` into native Ollama stream options.

## Evidence

There is no config-derived `params.num_ctx` plumbing here.
