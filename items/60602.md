## Summary

Keep this issue open. Current main still does not inject or configure Bedrock requestMetadata for per-agent attribution, even though the pinned pi-ai dependency supports forwarding it to Bedrock. The active session agent id is available in the runner path, but the current OpenClaw wrappers only pass other stream options, Bedrock region, guardrails, and cache-point behavior. The provided GitHub context also shows #60698 as an open, unmerged implementation candidate for this behavior, so this issue should not be closed before that PR or a successor lands.

## Best Possible Solution

Keep the issue open and have maintainers review #60698 or a successor patch. The best fix should live in the Bedrock-owned runner/provider path, derive agent id and display name from verified runtime context, add or thread provider-hook context if needed, merge cleanly with any future configurable metadata from #57360, enforce AWS metadata constraints, preserve non-Bedrock behavior, and add focused tests proving the final Bedrock request includes metadata only when appropriate.
