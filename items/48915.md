## Summary

Current main still does not implement the requested snake_case recognition. `isRawToolCallBlock` only accepts camelCase `toolCall`, `toolUse`, and `functionCall`; the sanitizer only drops invalid tool-call blocks when that predicate matches, so a broken `tool_use` block would still be treated as ordinary content and preserved. The related PR #50802 appears to track the right fix, but it is still open and unmerged, so the issue should stay open until the fix lands.
