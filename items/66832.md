## Summary

Current main still leaves the reported `logs.tail` redaction gap open, and the linked implementation PR remains open and unmerged. Because an open PR by a contributor specifically targets this issue with closing syntax, this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while maintainers review and either land PR #67041 or apply an equivalent patch on main. The fix should extend `src/logging/redact.ts` default patterns for Basic auth, generic JWTs, Pomerium assertion headers, and hyphenated security token headers, with focused regression coverage in `src/logging/redact.test.ts` and log-tail coverage if maintainers want end-to-end assurance.
