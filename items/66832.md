## Summary

Current main still leaves the central bug open: the active logs.tail path redacts returned log lines with src/logging/redact.ts defaults, and those defaults still miss Basic auth, generic JWTs, and x-pomerium-jwt-assertion-style headers. PR #67041 is an open unmerged implementation candidate with closing syntax for this issue, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open while maintainers review and either land PR #67041 or replace it with an equivalent patch on main. The fix should extend src/logging/redact.ts defaults for Basic auth, generic JWTs, and reported security assertion headers, add focused regression tests in src/logging/redact.test.ts, and only close this issue after the fix has merged.

## Risks / Open Questions

- A real security-sensitive behavior risk remains if Basic auth, generic JWTs, or Pomerium assertion headers reach the configured gateway log file and are returned through logs.tail to read-scoped clients.
- X-OpenClaw-Token appears already covered by the generic TOKEN assignment pattern, so the eventual fix should focus on the uncovered formats and avoid adding unnecessarily broad redaction patterns.
