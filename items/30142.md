## Summary

Keep open. Current `main` has only partial superseding work: the delivery layer now treats `sendPayload` as optional, and 2026.3.2 shipped shared `sendPayload` support for other adapters, but the adapters this PR still targets are not all implemented on `main` today.

## Risks / Open Questions

- The PR branch is stale against current outbound architecture; Google Chat and Mattermost now use attached-result plugin builders, and MS Teams uses runtime delegate forwarding, so this diff would need rework before merge.
- Closing as implemented would be incorrect: four targeted adapters still do not expose `sendPayload` on `main`.
- Closing as obsolete would also be too aggressive: channels like Mattermost and MS Teams still have presentation/channel-data related outbound surfaces, so the remaining gap may still be useful work.
