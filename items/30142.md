## Summary

Keep #30142 open. Current main still only partially supersedes the PR: Discord and Slack have shipped `sendPayload`, but Google Chat, Mattermost, MS Teams, and Synology Chat still do not expose `sendPayload` on their bundled outbound adapters. The PR should be rebased or narrowed, but the remaining adapter work is real.

## Best Possible Solution

Keep #30142 open for maintainer action, or replace it with a narrower PR targeting only Google Chat, Mattermost, MS Teams, and Synology Chat. Drop the already-shipped Discord/Slack portions, reuse the shared reply-payload helpers where they fit each adapter, and add adapter-level tests for text-only, media-only, mixed `mediaUrls`, and channelData/presentation behavior before landing.
