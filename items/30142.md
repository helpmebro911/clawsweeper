## Summary

Keep #30142 open. Current main still only partially supersedes it: Discord and Slack already have shipped sendPayload support, but Google Chat, Mattermost, MS Teams, and Synology Chat still lack sendPayload on their bundled outbound adapters. The PR should be rebased or narrowed before merge, but the remaining adapter work is real and is not fully replaced by the related closed items in the provided context.

## Best Possible Solution

Keep #30142 open for maintainer action, or replace it with a narrower PR that targets only Google Chat, Mattermost, MS Teams, and Synology Chat. Rebase away the already-shipped Discord/Slack portions, reuse the current shared reply-payload helpers where they match the adapter semantics, and add focused sendPayload contract tests before landing.
