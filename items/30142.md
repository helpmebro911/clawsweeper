## Summary

Keep this PR open. Current main still only has sendPayload for Discord and Slack among the six adapters named here; Google Chat, Mattermost, MS Teams, and Synology Chat still expose sendText/sendMedia without a sendPayload entry point. The PR is stale and would need rebasing/narrowing because Discord and Slack are already implemented differently on main, but the remaining adapter work is still real.

## Best Possible Solution

Keep this PR open for maintainer action, but require a rebase and narrowing before merge. The best path is to drop the already-shipped Discord and Slack changes, implement sendPayload only for Google Chat, Mattermost, MS Teams, and Synology Chat using current shared payload helper/contract-test patterns, and keep focused tests for text-only, media-only, mixed media URLs, empty or channelData-only payloads, chunk behavior, and delivery fallback compatibility.
