## Summary

Keep open. Current main has not implemented the Anthropic 1M GA migration: the active bundled Anthropic plugin still injects `context-1m-2025-08-07` for `context1m: true`, still strips that beta for OAuth-style tokens, and tests/docs still assert the pre-GA beta-header behavior. The PR also targets a now-removed core file path, so the useful work would need to be rebased onto the current plugin architecture rather than closed as implemented.

## Best Possible Solution

Keep the item open for maintainer/product review or contributor rework. The best path is to rebase the migration onto `extensions/anthropic/stream-wrappers.ts`, update the current tests and public docs, preserve `context1m` as an explicit context-accounting/backcompat opt-in if maintainers still want that cost guard, and retain safe OAuth behavior for explicitly configured legacy beta headers.
