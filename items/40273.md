## Summary

Current `main` already fixes ACP `sessions_spawn(..., runtime: "acp", mode: "run")` completion announce on the tool path, with matching docs and tests. This PR's `spawnAcpDirect()` registration is obsolete against the current implementation.

## Close Comment

Closing this from a Codex review because current `main` already implements the ACP completion-registration behavior for `sessions_spawn({ runtime: "acp" })`, just in a different place than this PR. The accepted ACP path now registers the child run in `src/agents/tools/sessions-spawn-tool.ts:282`, the ACP test coverage asserts that registration in `src/agents/tools/sessions-spawn-tool.test.ts:320`, and the docs describe parent-owned one-shot ACP sessions reporting completion through the internal announce path in `docs/tools/acp-agents.md:338`. Current registry behavior also keeps `cleanup: "keep"` runs from being archived (`src/agents/subagent-registry-run-manager.ts:303`, with proof in `src/agents/subagent-registry.archive.e2e.test.ts:86`). Since `main` already has the requested behavior, this PR is obsolete.
