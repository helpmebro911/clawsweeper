## Summary

Keep open. Current main has a related local `openclaw exec-policy` CLI and docs for syncing `tools.exec.*` with host approvals, but I did not find high-confidence evidence that the specific reported surfaces are fixed: `openclaw configure` including the security setting, npm update preserving the existing value, or update/configure warning users about the change.

## Evidence

- **Related implementation exists but is scoped differently:** PR #64050 says it added `openclaw exec-policy show|preset|set` to synchronize local `tools.exec.*` and host approvals, and explicitly notes it does not mutate remote gateway config or node-host approvals/config over RPC.
  - command: `web.open https://github.com/openclaw/openclaw/pull/64050`
  - sha: [fe0cd12](https://github.com/openclaw/openclaw/commit/fe0cd12)
- **Docs document exec-policy as a separate command:** The public docs describe `openclaw exec-policy` as a local convenience command for inspecting and syncing requested `tools.exec.*` config plus local `~/.openclaw/exec-approvals.json`, not as part of the configure wizard or update flow.
  - file: [docs/tools/exec-approvals.md:253](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/tools/exec-approvals.md#L253)
  - command: `web.open https://github.com/openclaw/openclaw/blob/main/docs/tools/exec-approvals.md`
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Docs still require two policy layers for no-approval host exec:** The docs say no-approval host exec requires both requested exec policy in OpenClaw config and host-local approvals policy, with `exec-policy preset yolo` as a local shortcut. That is useful mitigation, but it is not proof that update preserves config or configure exposes the setting.
  - file: [docs/tools/exec-approvals.md:317](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/tools/exec-approvals.md#L317)
