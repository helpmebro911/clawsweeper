## Summary

Keep open. Current `main` still does not expose `tools.exec.security` or `tools.exec.ask` anywhere in the `openclaw configure` wizard, so the reported configuration UX gap remains. The current docs/runtime defaults do show that unset gateway/node exec policy is now `security=full` and `ask=off`, which weakens the specific "reset causes approvals" symptom, but that is not enough to close because the wizard omission itself is still present and there is no evidence here of an update-time preservation/warning fix.
