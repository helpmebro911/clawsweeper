## Summary

Keep PR #71154 open. Current main still has the exact reported behavior: `evaluateShellAllowlist()` rejects any shell command containing backslash-newline before analysis, and both gateway/node-host policy still report `analysisOk=false` as `allowlist miss`. The linked issue #71153 being closed and related PR #71225 being closed unmerged do not make this PR obsolete; no equivalent implementation is present on main.

## Best Possible Solution

Keep the PR open and continue review. The likely best path is a focused implementation that normalizes POSIX backslash-newline before allowlist analysis with quote/heredoc/comment-aware parsing, preserves the existing command-substitution fail-closed guarantees, adds distinct `unsupported-shell-syntax` reporting in gateway and node-host paths, and includes regression coverage for LF/CRLF allowlisted commands, true allowlist misses, unparseable syntax, and heredoc edge cases.
