## Summary

Keep open. Current main still has the reported behavior: exec allowlist analysis hard-rejects commands containing POSIX backslash-newline continuations before shell analysis, and both gateway and node-host policy still collapse analysis failures into the existing allowlist-miss path. The PR is a focused implementation for the open linked bug #71153; the later related PR #71225 is closed and not merged, so it does not supersede this work.

## Best Possible Solution

Keep the PR open as the active implementation candidate for #71153. Review the durable-approval semantics, ensure the line-continuation normalization and distinct unsupported-syntax errors are covered in gateway and node-host tests, fix the changelog attribution if retained, then land or replace it with an equivalent patch.
