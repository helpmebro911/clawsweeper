## Summary

Keep this PR open. Current main still rejects POSIX backslash-newline before allowlist analysis and still reports analysis failures as allowlist misses. The PR remains a plausible implementation candidate for the linked exec allowlist bug, but it needs maintainer review on the durable-approval security semantics and changelog attribution before landing.

## Best Possible Solution

Keep this PR open and review it as the active implementation candidate. The best path is to land an adapted version that normalizes POSIX backslash-newline before allowlist analysis with quote/heredoc/comment-aware parsing, emits a distinct unsupported-shell-syntax denial through gateway and node-host paths, resolves the durable-approval semantics for unparseable commands, and includes focused regression coverage plus a properly attributed changelog entry.
