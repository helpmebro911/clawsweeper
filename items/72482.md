## Summary

Keep this PR open. Current main still has the Telegram non-streaming gap the PR targets: with `streamMode: "off"`, Telegram does not create an answer draft stream, but still passes `suppressDefaultToolProgressMessages: false`, so generic tool/progress messages can be delivered. The PR is a narrow candidate fix with a matching regression-test update and no unrelated security-sensitive file changes in the provided diff.

## Best Possible Solution

Review and, if maintainers agree with final-only Telegram non-streaming semantics, land this narrow PR with its regression test. Before merge, either confirm the reasoning-token portion of #72363 is already fixed by current reasoning-lane handling or adjust the closing relationship so any remaining reasoning-specific work stays tracked separately.
