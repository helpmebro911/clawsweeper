## Summary

Keep #7456 open. Current main still has only forward/cancel onboarding wizard flow: prompt types have no Back result, the Clack adapter does not inject Back handling, `runSetupWizard` remains a mostly sequential flow, and Gateway wizard RPC exposes only start/next/cancel/status. The related Back PR #7762 was closed unmerged, #5880 is a distinct interrupted-resume attempt, and the later dialog PR #18286 is not present on current main and would not by itself add onboarding Back navigation.

## Best Possible Solution

Keep #7456 open. The best implementation is a typed wizard navigation contract: add an explicit Back/previous-step result through `WizardPrompter`, the Clack adapter, Gateway wizard schemas/session handling, and client rendering; refactor onboarding into a state machine that preserves answers while moving backward and forward; and add CLI/RPC regression coverage. PR #7762 is useful prior art but should not be treated as shipped.
