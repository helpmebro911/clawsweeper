## Summary

Keep open. Current main still does not implement wizard-wide Back navigation for `openclaw onboard`; the classic onboarding path remains forward-linear, the prompt/protocol contracts have no back result or previous-step method, and the only current Back options are local submenu exits. The related implementation PR #7762 was closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Implement a typed wizard navigation contract rather than ad hoc menu entries: add an explicit Back result or previous-step operation to the prompter and gateway wizard APIs, refactor `runSetupWizard` into a state machine that preserves answers while moving backward and forward, and keep local submenu Back behavior aligned with the new general navigation model. PR #7762 is useful prior art but should not be treated as canonical because it was closed unmerged with type-safety and completeness concerns.
