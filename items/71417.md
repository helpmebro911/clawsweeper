## Summary

Keep #71417 open. Current main still reuses an existing fresh session for `openclaw agent --agent <id>` when `--session-id` is omitted: the CLI resolves the agent's main session key, then the session resolver reuses the stored `sessionId` if the row is fresh. Omitted delivery channel also still resolves through the delivery-plan `last` fallback. Docs partially describe that `--agent` uses the main session and `--channel` is delivery-only, but there is still no `--new-session`/`--fresh` flag and no resume banner/warning for scripted runs.

## Best Possible Solution

Keep the issue open. A conservative fix should add an explicit fresh-session path for `openclaw agent` and/or print a clear resume banner when a stored session id is reused, then update `docs/cli/agent.md` and `docs/tools/agent-send.md` and add focused tests for both Gateway and `--local` CLI paths.
