## Summary

Keep open. Current main still defaults omitted `openclaw agent --agent <id>` session selection to the agent main session and reuses a fresh stored `sessionId`; omitted delivery channel also still normalizes to `last`. I did not find a `--new-session`/`--fresh` flag, default-to-new behavior, or resume banner in the current CLI, Gateway, local embedded path, docs, or focused tests.

## Best Possible Solution

Keep this issue open and implement an explicit fresh-session path for `openclaw agent`: either change `--agent` without `--session-id` to create a fresh session by default, or add and document a `--new-session`/`--fresh` flag plus a clear stderr resume banner when an existing session is reused. Cover both Gateway and `--local` paths, update `docs/cli/agent.md` and `docs/tools/agent-send.md`, and add focused tests for omitted, blank, explicit, and fresh-session controls.
