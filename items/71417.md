## Summary

Keep #71417 open. Current main still routes `openclaw agent --agent <id>` without `--session-id` to that agent's main session key, then reuses the stored session id when the session is still fresh. The docs clarify that `--channel` affects delivery rather than session routing, but there is still no `--new-session`/`--fresh` flag, default-fresh behavior, or resume banner for this CLI path.

## Best Possible Solution

Keep this issue open and fix the CLI/session UX directly. A conservative implementation would add an explicit `openclaw agent --new-session` or `--fresh` option that forces a new session id for `--agent` runs, plus docs and a focused regression test. If maintainers prefer preserving default reuse, print a non-JSON stderr banner when resuming an existing session and document `/new` or the new flag as the scripted fresh-run path.
