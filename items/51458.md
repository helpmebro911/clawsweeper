## Summary

Current `main` still has the reported Slack message-action limitation. The `send` action converts `interactive`/`presentation` into Slack blocks, then rejects any request that also carries `media`, while the separate outbound reply path already uses a two-message media-then-blocks flow. The issue is not implemented on `main` and remains a valid bug/gap.
