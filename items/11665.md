## Summary

Keep #11665 open. Current main still routes direct and mapped hook agent actions into an isolated cron-style turn, and isolated turns still force a new session ID even when the hook sessionKey is stable. The config, schema, mapping, transform, and docs surfaces still expose sessionKey but no persistent hook-session mode or reuse switch, while the related fix PRs in the provided context are closed unmerged.

## Best Possible Solution

Keep the issue open and implement a backward-compatible persistent hook-session option in core, likely sessionMode: "isolated" | "persistent" or an equivalent reuse flag on direct /hooks/agent payloads, hook mappings, and transform results. Default to isolated, thread the option through hook normalization, mapping merge, config types/schema/help, dispatch, docs, and regression tests, then pass forceNew: false only for the persistent hook path.
