## Summary

Keep #11665 open. Current main still routes direct and mapped webhook agent actions through isolated cron-style turns, and that path still forces a new session ID whenever the hook job has `sessionTarget: "isolated"`. The hook config, payload, mapping, transform, schema, and docs surfaces expose `sessionKey` but no persistent/reuse mode, while the related fix PRs in the provided context are closed unmerged and the latest maintainer/Codex discussion already recommended keeping this open.

## Best Possible Solution

Keep the issue open and implement a backward-compatible persistent hook-session option in core, likely `sessionMode: "isolated" | "persistent"` or an equivalent reuse flag. Thread it through direct `/hooks/agent` payloads, hook mappings, transform results, config types/schema/help, dispatch, docs, and regression tests, then pass `forceNew: false` only for the persistent hook path.
