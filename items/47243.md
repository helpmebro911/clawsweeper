## Summary

Keep open. Current main still does not implement timestamp plus last-message preview in the Control UI session list. The backend/session-tool and TUI surfaces already support `includeLastMessage`/`lastMessagePreview`, so the request remains in scope, but PR #47243 needs refinement before it can safely land.

## Best Possible Solution

Keep the item open, or consolidate it with maintainer-selected canonical work for #47120. The best implementation is a focused Control UI patch that adds `lastMessagePreview` to UI types, requests `includeLastMessage` only on a bounded/capped session list, normalizes and truncates the preview before rendering it with the timestamp, removes unrelated dependency changes, and adds targeted controller/view coverage.
