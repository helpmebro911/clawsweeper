## Summary

Keep #11894 open. Current main has session/transcript cost reporting and auth-profile rotation plumbing, but it still does not expose the resolved auth profile for each completed model call, aggregate costs by auth profile, or emit a user-facing fallback/profile-rotation event. The related burn0 discussion remains a useful interim cost workaround, but it cannot provide OpenClaw auth-profile attribution without native metadata.

## Best Possible Solution

Keep #11894 open until OpenClaw records the resolved authProfileId for each completed model call in session-safe metadata, exposes it through transcript/usage/status or diagnostics, aggregates costs by auth profile, and emits a clear user-facing auth-profile rotation/fallback event. burn0/#55379 can remain an external cost workaround, but it is not a native replacement for OpenClaw auth-profile attribution.
