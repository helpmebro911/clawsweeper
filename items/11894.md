## Summary

Keep #11894 open. Current main has usage/cost summaries and internal auth-profile selection/rotation state, but it still does not expose the actual auth profile used per model call, aggregate cost by auth profile, or emit a user-visible profile fallback event. A same-day maintainer comment also explicitly kept the issue open, and the burn0 workaround only addresses network-level cost visibility, not OpenClaw auth-profile attribution.

## Best Possible Solution

Keep the issue open until OpenClaw records the actual auth profile used for each model call in transcript/session-safe metadata, exposes cost totals grouped by auth profile in /usage cost and/or session_status, and emits or stores a clear auth-profile rotation/fallback event for user-facing surfaces. The burn0 link can remain as an interim network-level cost workaround, but it does not satisfy native OpenClaw profile attribution.
