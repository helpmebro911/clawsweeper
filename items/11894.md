## Summary

Keep #11894 open. Current main has session/transcript cost reporting, provider/model cost aggregation, and internal auth-profile rotation plumbing, but it still does not expose the resolved auth profile for each completed model call, aggregate costs by auth profile, or emit a user-facing fallback/profile-rotation event. The related burn0/#55379 discussion remains a useful interim cost workaround, but it cannot provide OpenClaw auth-profile attribution without core metadata.

## Best Possible Solution

Keep #11894 open until OpenClaw records the resolved authProfileId for each completed model call in session-safe metadata, exposes it through transcript/usage/status or diagnostics, aggregates costs by auth profile, and emits a clear user-facing auth-profile rotation/fallback event. The likely implementation points are the embedded runner attempt metadata, transcript usage parsing, SessionCostSummary/gateway usage aggregation, status/session_status rendering, and model-call diagnostic/plugin hook types.
