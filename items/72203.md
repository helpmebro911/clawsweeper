## Summary

Keep open. This is a fresh, concrete v2026.4.24 Ubuntu QuickStart report with health-check and Control UI token-mismatch evidence, and current main does not contain a targeted post-release fix for the setup-daemon/dashboard token path. Existing code implements the intended generated-token and dashboard-fragment flow, but it also still has plausible local credential-precedence drift paths, so static intent is not enough to close the issue.

## Best Possible Solution

Keep the issue open and reproduce the v2026.4.24/current-main QuickStart install-daemon path on Ubuntu. Compare the generated `gateway.auth.token`, any `OPENCLAW_GATEWAY_TOKEN` in the shell and daemon service environment, the token used by setup health/`healthCommand`, and the token copied/opened by `openclaw dashboard`. If this is service-env or credential-precedence drift, fix the daemon install/health/dashboard path and add a regression test covering QuickStart token auth plus post-install health and Control UI connection.
