## Summary

Keep open. Current main has memory plugin registration and shared memory storage helpers, but the built-in file scan/watch/incremental/stale-cleanup pipeline is still owned inside memory-core and is not exposed as file-scan/file-change hooks for third-party memory plugins. This is a concrete plugin API gap, not just a ClawHub/plugin-distribution request.
