## Summary

Current `main` at `38caa6832d4e1d344736d635b0c5fdad37e4430b` does not implement this PR’s truncated-write protections. The host-side verification still checks only inode/path identity, the pinned write helper still accepts no expected-size argument and performs no rollback-on-size-mismatch logic, and the sandbox write path still uses the old five-argument protocol without byte-count enforcement.
