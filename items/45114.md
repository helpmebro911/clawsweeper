## Summary

Keep open. Current main still contains the exact `line-clamp: 3;` declaration that PR #45114 removes from `.chat-queue__text`, and the chat queue renderer still uses that class. The linked bug #45020 remains directly relevant, and the related later PR #45536 appears to be duplicate unresolved work rather than a merged or canonical superseding fix.

## Best Possible Solution

Keep this PR open or consolidate it with #45536. The code path still contains the noisy unprefixed `line-clamp: 3;`; the best product fix is a focused UI CSS change removing that declaration while preserving the existing `display: -webkit-box`, `-webkit-line-clamp: 3`, and `-webkit-box-orient: vertical`, with a quick Firefox chat-page verification.
