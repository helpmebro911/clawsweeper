## Summary

Keep #17215 open. Current main has adjacent update-safety pieces, including update dry-run, post-update doctor repair, plugin integrity drift checks, plugin compatibility notices, and ClawHub compatibility gates, but it still does not implement the requested native pre/post-update compatibility scanner, `doctor --upgrade-scan` surface, risk-scored JSON/human report, smoke recommendations, or rollback hints. The related PR #54354 is still an open, narrower plugin engine-compatibility proposal, not a shipped replacement for this issue.

## Best Possible Solution

Keep this issue open as a core update/doctor safety enhancement. The best path is a native `openclaw update` and/or `openclaw doctor --upgrade-scan` mode that produces both human and JSON reports for pre-update config/schema/deprecation checks, post-update health/plugin/runtime checks, smoke-test recommendations, risk classification, and rollback guidance, with docs and focused tests for the report contract. Treat https://github.com/openclaw/openclaw/pull/54354 as related plugin-specific compatibility work if it lands, not as a full superseding fix for this issue.
