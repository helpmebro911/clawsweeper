## Summary

Keep #17215 open. Current main has useful update safeguards (`openclaw update --dry-run`, `--json`, post-update doctor/plugin sync, rollback docs, compatibility registry), but I did not find the requested native pre/post upgrade compatibility scanner, risk score/report, smoke recommendation output, rollback recommendation logic, or `doctor --upgrade-scan`-style command. This is also not a good `clawhub` close: the request targets core update/doctor safety, which aligns with the project’s setup reliability and stability priorities.
