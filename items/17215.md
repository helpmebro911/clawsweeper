## Summary

Keep #17215 open. Current main still has adjacent update-safety pieces, but it does not implement the requested native pre/post-update compatibility scanner, `doctor --upgrade-scan` surface, risk-scored JSON/human report, smoke recommendations, or rollback hints. The existing maintainer Codex review on the issue also kept it open, and the only related PR in the provided context (#54354) is a narrower, still-open plugin engine-compatibility proposal rather than a shipped replacement.

## Best Possible Solution

Continue tracking this as a core update/doctor safety enhancement. The best path is a native `openclaw update` and/or `openclaw doctor` upgrade-scan mode that produces human and JSON reports for pre-update config/schema/deprecation checks, post-update health/plugin/runtime checks, smoke-test recommendations, risk classification, and rollback guidance, with docs and focused tests for the report contract. Treat PR #54354 as a related plugin-specific compatibility effort if it lands, not as a full superseding fix for this issue.
