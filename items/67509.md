## Summary

Keep this PR open. Current main still has no equivalent root/effective-root CLI guard, while the linked #67478 failure mode remains plausible through the modern entrypoint, legacy entrypoint, state-dir resolution, and Linux user-systemd service paths. The PR is still the active implementation candidate and has incorporated review feedback for tests, legacy entry coverage, container forwarding, narrowed bypass semantics, and effective UID handling.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent core fix that guards both src/entry.ts and src/index.ts before state/config side effects, checks real and effective UID, preserves intentional container forwarding, includes focused regression tests, and updates the DigitalOcean/install guidance or wrapper flow so root SSH users get a clear supported command path.
