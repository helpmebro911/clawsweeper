## Summary

Keep this PR open. Current main still does not implement reasoning-wrapped NO_REPLY suppression, the linked bug #66701 remains open, and #69394 is an open competing implementation rather than a merged superseder. The PR is focused and security-neutral, but it still has a material unresolved false-positive in the bare `think` regex that needs revision or maintainer judgment before landing.

## Best Possible Solution

Keep this PR open as an active implementation candidate for #66701. The best path is to revise the bare-marker regex so it cannot consume blank-line boundaries, add a regression for `think\n\n<user-facing text>.NO_REPLY`, then review and land the narrow `src/auto-reply/tokens.ts` behavior with tests. If maintainers prefer broad trailing-token suppression instead, choose #69394 or another canonical implementation first, then close competing PRs after that decision is resolved.

- Streaming/block delivery may need separate handling if reasoning content has already been dispatched before a final silent token arrives.
