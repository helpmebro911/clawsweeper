## Summary

Keep this PR open. Current main still has exact-token-only silent detection and still strips then delivers mixed-content trailing NO_REPLY payloads, so the linked reasoning-wrapper bug is not implemented on main. This PR is a focused code/test candidate, but its current bare `think` regex can still consume a blank-line boundary and suppress a substantive reply. Related PRs #69394 and #69472 are also open alternatives, not merged superseders.

## Best Possible Solution

Keep this PR open as an active implementation candidate for #66701. Revise the bare-marker regex so it cannot consume blank-line boundaries, add a regression for `think\n\n<user-facing text>.NO_REPLY`, and then choose a canonical policy among this PR, #69394, and #69472. Once a corrected implementation lands, close competing PRs as superseded.
