## Summary

Keep PR #6457 open. Current main has absorbed the auth-reply thread preservation slice, but the central group-scope Telegram command registration/cleanup is still missing, and the plugin-command "Command not found." fallback still drops topic thread routing. The latest PR discussion already called out the same remaining gaps; re-checking current main at b565e6e96310c03e81b32caf82894ce89f32fec5 confirms they are still present.

## Best Possible Solution

Keep this PR or a canonical follow-up open until Telegram native command sync registers and clears both default and `all_group_chats` scopes, and until the plugin-command not-found fallback sends with the same thread params used by other native command replies. The best code path is a small current-layout patch in `extensions/telegram/src/bot-native-command-menu.ts` and `extensions/telegram/src/bot-native-commands.ts`, with focused tests asserting both scopes and threaded not-found behavior.
