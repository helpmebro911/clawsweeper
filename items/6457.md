## Summary

Keep PR #6457 open. Current main partially covers the thread/auth-reply half of the PR, but it still does not register or clear Telegram command menus for the `all_group_chats` scope, and the plugin-command `Command not found.` fallback is still sent without topic/thread params.

## Best Possible Solution

Keep #6457 open or replace it with a focused Telegram plugin patch. The best fix is to register and clear both default and `all_group_chats` command scopes in `syncTelegramMenuCommands`, preserve topic params for the plugin-command not-found fallback, and add regression coverage for dual registration, dual cleanup, retry/cache behavior, and threaded fallback replies.
