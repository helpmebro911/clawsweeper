## Summary

Keep this PR open. Current main has absorbed part of the Telegram thread/auth reply behavior, but the central group-scope command registration and cleanup are still missing, and the plugin-command not-found fallback remains unthreaded. The PR is useful but stale against the current `extensions/telegram` layout and should be rebased or replaced with a focused patch.

## Best Possible Solution

Keep this PR open and rebase or replace it with a focused patch under `extensions/telegram`: register and clear both default and `all_group_chats` command scopes, cache the command hash only after all required scope syncs succeed, preserve thread params for the plugin-command not-found fallback using `resolveTelegramThreadSpec` and `buildTelegramThreadParams`, and add regression tests for dual registration, dual cleanup, retry/cache failure behavior, and threaded fallback replies including General-topic id 1 handling.
