## Summary

Keep this PR open. Current main has absorbed part of the thread/auth reply behavior, but the central Telegram group-scope command registration/cleanup is still missing, and the plugin-command not-found fallback still sends without topic thread params. The PR is stale against the current extensions/telegram layout, so the best path is a focused rebase or successor patch rather than closing it.

## Best Possible Solution

Keep this PR open until the remaining Telegram plugin behavior is implemented. The maintainer path is to rebase or replace it with a focused patch in extensions/telegram that registers and clears both default and all_group_chats command scopes in the current menu sync/native-disabled paths, preserves thread params for the plugin-command not-found fallback via resolveTelegramThreadSpec/buildTelegramThreadParams, and adds regression tests for dual registration, dual cleanup, retry/cache failure behavior, and threaded fallback replies.
