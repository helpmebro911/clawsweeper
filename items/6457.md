## Summary

Keep PR #6457 open. Current main partially overlaps the PR by threading native auth replies, but it still does not register or clear Telegram native command menus for the `all_group_chats` scope, and the plugin-command `Command not found.` fallback is still sent without topic/thread params. The prior community and maintainer comments remain accurate against current main `4a3030df9efa72c44bab567a4390ac0c4876a73f`.

## Best Possible Solution

Keep #6457 open, or replace it with a focused Telegram plugin patch. The best fix is to register and clear both default and `all_group_chats` Telegram command scopes in the current `syncTelegramMenuCommands` flow, preserve topic params for the plugin-command not-found fallback, and add regression coverage for dual registration, dual cleanup, cache/retry behavior, and threaded fallback replies.
