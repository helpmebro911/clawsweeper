## Summary

Keep #72081 open. Current main still prints a clean dashboard URL without `#token=` by design, while only passing a tokenized URL to clipboard/browser for non-SecretRef tokens. The report is fresh and concrete, and the no-browser/no-clipboard path can still leave a user with only the non-tokenized URL, so none of the conservative close reasons fit.

## Best Possible Solution

Keep the issue open, but steer the fix away from re-logging bearer tokens. A good maintainer path is to preserve clean terminal logs while improving the headless/WSL/no-clipboard fallback: give explicit auth recovery steps, make the copy/open success state clearer, and consider an explicit opt-in escape hatch only if it can avoid normal log capture and SecretRef exposure.
