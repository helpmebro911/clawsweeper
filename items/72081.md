## Summary

Keep #72081 open. Current main still prints a non-tokenized `Dashboard URL:` line, and the tests intentionally assert that terminal logs never contain `#token=` while clipboard/browser receive the tokenized URL for non-SecretRef tokens. That makes this a real product/UX decision rather than a safe cleanup close, especially for WSL/headless/clipboard-failure cases where the command can tell the user to use the printed URL even though the printed URL has no token.

## Best Possible Solution

Keep the issue open for a targeted UX/security decision. A likely fix belongs in `src/commands/dashboard.ts` and tests: preserve default token redaction from logs, but make the clipboard/headless failure path actionable, for example by printing explicit instructions to retrieve/paste the token, improving WSL clipboard handling, or adding a clearly named opt-in flow if maintainers accept the exposure risk.
