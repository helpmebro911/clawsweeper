## Summary

Keep #68648 open. Current main still waits for the final agent result for exec approval followups, so the PR is not implemented or obsolete. The PR targets a real narrow code path, but its current accepted-only change needs maintainer follow-up because the gateway emits an initial accepted ack before the run's later terminal success/error response, so returning on that ack can bypass the existing direct fallback path on post-accept failures.

## Best Possible Solution

Keep the PR open and revise the fix so exec approval followups do not wait for the full final model turn, while preserving failure/fallback behavior for dispatch or post-accept terminal errors. The likely product direction is a gateway/session-followup contract that resolves after the followup is safely accepted for dispatch, plus regression coverage for both the busy-lane timeout case and the post-accept failure fallback case.
