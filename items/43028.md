## Summary

Current main has cron backoff, failure alerts, retention pruning, and JSONL run history, but it still does not implement the issue's requested recurring-job circuit breaker, remove-time session cleanup contract, or automatic `.learnings/changelog.md` recording. This issue remains concrete and recent enough for maintainer/product work.

## Best Possible Solution

Keep this issue open. The best path is to split the remaining work into cron-owned changes: add a backward-compatible recurring execution circuit breaker in `src/cron/service/timer.ts` with config/types/docs/tests; design an explicit `cron.remove` cleanup contract across protocol, gateway, CLI, UI, and session/transcript ownership; and decide whether JSONL `cron.runs` replaces the requested `.learnings/changelog.md` audit sink or whether a new core writer should be implemented and documented.

## Risks / Open Questions

- The issue bundles three separable cron changes, so maintainers may want to split it into narrower implementation tasks instead of landing one broad patch.
- Remove-time cleanup needs a careful ownership contract to avoid deleting shared, custom, base, or still-referenced sessions/transcripts unexpectedly.
- The changelog request needs product resolution: either document JSONL `cron.runs` as the supported audit trail or add an explicit changelog writer if `.learnings/changelog.md` is intended to be core behavior.
