## Summary

Keep open. Current main still routes OpenRouter Anthropic model refs through the OpenAI-compatible streaming transport, while the embedded runner still applies a 120s chunk-idle watchdog. Native Anthropic fine-grained tool streaming exists, but I found no source, docs, or test evidence that the reported OpenRouter large tool-parameter path is fixed or covered.

## Best Possible Solution

Keep this issue open as the canonical tracker for OpenRouter/Anthropic large tool-parameter generation failures. Add a synthetic regression where an OpenAI-compatible stream withholds tool-call argument chunks longer than the default idle window, then either enable supported OpenRouter/Anthropic tool-argument streaming, adjust the LLM idle policy for silent tool-argument generation, or add a tool-loop strategy that avoids requiring one large `Write`-style payload.
