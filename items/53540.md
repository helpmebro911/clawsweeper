## Summary

Keep open. Current main has useful adjacent timeout handling and clearer idle-timeout messaging, but I did not find source, test, docs, or release evidence that the edited OpenRouter/Anthropic large tool-parameter failure now succeeds by default. The OpenRouter provider still runs through the OpenAI-compatible chat-completions path, and current fine-grained Anthropic tool streaming is wired for native Anthropic transport, not clearly for OpenRouter-routed Anthropic models.

## Best Possible Solution

Keep this issue open as the canonical tracker for the edited OpenRouter/Anthropic large tool-parameter failure. A maintainer fix should reproduce the no-intermediate-chunk tool-call-argument path with a synthetic delayed stream and, if possible, a live OpenRouter/Anthropic replay; then either make the LLM idle-timeout policy safe for slow tool-argument generation, enable provider-supported fine-grained tool-use streaming on the OpenRouter route, or add a core/tool-loop strategy that prevents large generated content from needing one silent tool-call parameter payload. Add regression coverage that proves the Write-style large-argument case succeeds or fails with the intended actionable timeout behavior.
