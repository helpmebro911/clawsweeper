## Summary

Keep this PR open. Current main still routes iMessage inbound message text through raw model-facing fields, so the hardening topic is not already implemented. The submitted patch is focused and security-relevant, but it is not merge-ready because it wraps only the legacy envelope body and does not protect the active `BodyForAgent`/prompt path or add regression coverage.

## Best Possible Solution

Revise this PR or replace it with a maintainer-reviewed iMessage hardening patch. The implementation should protect the actual model-bound path through `UntrustedContext` or another prompt-bound field, keep `CommandBody`/`RawBody` raw and deterministic for command parsing, preserve reply/history behavior intentionally, and add focused iMessage tests for prompt assembly, commands, reply context, history, and echo behavior.

## Risks / Open Questions

- Merging this PR as-is could create a false sense that iMessage inbound prompt-injection hardening is active while raw `BodyForAgent` still reaches prompt assembly.
- The patch adds randomized external-content markers to legacy envelope/history text, which may affect deterministic history, rendering, or echo-related behavior without securing the primary model input.
- No iMessage regression coverage is included for prompt content, command parsing, reply context, history, or echo behavior.
