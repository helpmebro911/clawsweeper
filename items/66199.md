## Summary

Keep open. Current main still routes iMessage inbound text to the model through raw `BodyForAgent`/`BodyStripped`, while this PR only wraps the legacy `Body` envelope produced by `formatInboundEnvelope(...)`. The submitted diff is security-relevant but ineffective as-is and needs revision or replacement rather than maintainer cleanup closure.

## Best Possible Solution

Keep this PR open for revision or replace it with a maintainer-reviewed iMessage hardening patch. The fix should make an explicit product/security decision about primary DM text, protect the actual model-bound path through `UntrustedContext` or an equivalent prompt-bound field, keep command parsing raw and deterministic via `CommandBody`/`RawBody`, and add focused iMessage tests for prompt content, command parsing, reply context, history, and echo behavior.
