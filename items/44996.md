## Summary

Current main does not implement `gateway.bind: "netbird"`, NetBird interface detection, or bind-mode-driven auto-TLS. The PR is a focused gateway feature with concrete verification and user demand in the discussion, and it is not safely closeable as implemented, duplicate, superseded, or ClawHub-only because current plugin APIs do not expose a gateway bind-mode/TLS extension point.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either to land a core gateway implementation with tests/docs for `gateway.bind: "netbird"`, NetBird interface detection, auto-TLS, CLI/onboarding/status/control-UI origin behavior, or first design a generic gateway bind-mode/plugin API that would let NetBird support live outside core without private runtime hooks.
