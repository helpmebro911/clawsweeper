## Summary

Keep this PR open. Current main still does not implement or document `gateway.bind: "netbird"`, and the current plugin surface does not expose a way to add a new gateway bind mode outside core. The PR is stale and security-sensitive, but it remains a plausible implementation candidate for the linked NetBird gateway request rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review and author refresh. If OpenClaw wants first-class NetBird support, implement it in core gateway/config/docs/tests because current plugin surfaces do not expose gateway bind-mode registration: update the bind enum/schema/help, NetBird interface detection, bind resolver, connection details, Control UI links, TLS runtime/fingerprint/status probes, onboarding/config flows, docs, and regression tests. If maintainers prefer not to add service-specific bind modes, open a deliberate gateway bind extension/API design instead of closing this focused implementation as obsolete.
