## Summary

Keep #70967 open. Current main still has the reported mismatch: the Control UI label depends on `thinkingDefault` from the active session row or `sessions.list.defaults`, but the gateway populates both with the model/profile fallback resolver instead of the config-aware `resolveThinkingDefault()`. Runtime chat/status paths do use the config-aware resolver, so the issue remains a real UI/API consistency bug. Related fix PR #70981 is open and unmerged, so the issue should stay open until a fix lands.

## Best Possible Solution

Keep the issue open and land a focused gateway/session fix: populate `sessions.list.defaults.thinkingDefault` and each `GatewaySessionRow.thinkingDefault` with the same config-aware resolution order used by runtime/status paths, including per-agent `agents.list[].thinkingDefault`, per-model `params.thinking`, and global `agents.defaults.thinkingDefault`. Add a regression test in `src/gateway/session-utils.test.ts` for a configured default that differs from the model fallback, then let the fixing PR close #70967 when merged.
