## Summary

Keep #12590 open. Current main still has the reported counter synchronization path: after a memory-flush-triggered compaction, `runMemoryFlushIfNeeded` can persist the post-increment `compactionCount` as `memoryFlushCompactionCount`, while `shouldRunMemoryFlush` skips whenever those counters are equal. Current tests still assert the synchronized state, and the supplied issue context points to open or closed-unmerged fix attempts rather than a shipped fix.

## Best Possible Solution

Keep #12590 open and fix the bookkeeping contract in `src/auto-reply/reply/agent-runner-memory.ts`: after a flush-triggered compaction, preserve the pre-increment compaction count as `memoryFlushCompactionCount` or replace the equality-based dedup state with a clearer marker that cannot skip the next eligible compaction cycle. Add regression coverage proving consecutive eligible auto-compaction cycles run `memoryFlush` each time, and update tests that currently assert synchronized counters. PR #51421 and the closed prior fix attempts are useful references, but the issue should remain open until a fix lands on main.
