## Summary

Keep open. The issue is recent, concrete, and still maps to current core Control UI/Gateway behavior: disconnects are still surfaced to the UI as user-visible `lastError` messages, and current main does not add a Control UI transport heartbeat/ping-pong configuration or a differentiated reconnect/banner flow that would resolve the reported frequent timeout popup. The supplied discussion also points to lost RPC responses during flapping and references PR #46472 as only an adjacent Feishu-specific heartbeat idea, not a merged or canonical fix for this Control UI issue.
