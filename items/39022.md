## Summary

Keep #39022 open. Current main still has persistent `/model` session overrides, but no `/model <name> --turns N`, warning threshold, remaining-turn accounting, automatic revert, or `models.limits` config contract.

## Best Possible Solution

Keep the issue open for maintainer product judgment and likely core implementation. A complete solution would need parser and native-command changes for `/model <ref> --turns N`, session-store fields for remaining turns/revert target/warning state, decrement and notification logic after completed turns, reset through the existing model override helper, strict config/docs for any accepted `models.limits` or per-model `maxTurns` shape, and focused tests for parsing, persistence, warning, auto-revert, and config validation. If maintainers prefer a plugin route, first define a supported plugin API for safely intercepting core model selection and per-turn session accounting.
