Keep #70280 open. The request is a concrete docs/attribution request, not a ClawHub/plugin capability, external administration task, stale bug, or duplicate of the memory provenance tracker. ## Best Possible Solution

A maintainer should compare the Nuwa commit/design claim against OpenClaw's Dreaming history and either add a short attribution/acknowledgment to the user-facing Dreaming docs or README, or reply with the project rationale for declining attribution. If accepted, the likely code path is a docs-only change around `docs/concepts/dreaming.md` plus any desired README/acknowledgments placement.
