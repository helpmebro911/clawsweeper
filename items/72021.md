## Summary

Current main still has the reported behavior: short-term promotion treats recall, daily, and grounded signals as one combined signal count for the `minRecallCount` gate and scoring. Daily/session ingestion writes `signalType: "daily"` into the same short-term recall store, and existing tests explicitly assert that daily-only entries can clear the promotion gates. The related title-search matches are about session reset/session-memory hooks and do not supersede this memory-core promotion scoring issue.

## Best Possible Solution

Keep the issue open for maintainer/product judgment and a targeted memory-core fix. The likely path is to separate real recall gating from reinforcement signals, or rename/configure the gate as signal-count based if daily/grounded promotion is intended. Any fix should update `rankShortTermPromotionCandidates`, `applyShortTermPromotions`, promote-explain diagnostics, the daily-only/grounded tests, and docs for the deep promotion thresholds.
