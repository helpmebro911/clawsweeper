## Summary

Keep #45449 open. Current main still sends Discord ack/status reactions on receipt, not on agent run start, and the Discord config/docs/tests do not expose or verify `ackReactionTiming`. The PR is stale against the current `extensions/discord` layout, but the requested behavior remains useful and not superseded; the provided PR discussion also shows the review concern about deferred lifecycle gating was addressed on the branch.

## Best Possible Solution

Keep the PR open and ask for a rebase/port onto current `extensions/discord`: add channel/account-scoped `ackReactionTiming`, schema/help/docs coverage, defer Discord status/plain ack startup from receipt to `onAgentRunStart` when configured, preserve scope gating and terminal cleanup ordering, and add focused regression tests in `extensions/discord/src/monitor/message-handler.process.test.ts`.
