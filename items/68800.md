## Summary

Current main does not implement PR #68800's production fix: `openclaw mcp serve` still reaches `loadConfig()` before any channel-server-specific `routeLogsToStderr()` call, so config warnings can still pollute the stdio MCP stream. The PR remains a focused, plausible fix for the linked open regression #68587, though its added test should be improved or renamed because the bot review correctly notes it uses `InMemoryTransport` rather than stdio.

## Best Possible Solution

Keep PR #68800 open and have a maintainer review or update it: add `routeLogsToStderr()` at the start of `serveOpenClawChannelMcp()` before `loadConfig()` can run, and pair it with a regression test that exercises the real stdio path or rename the existing in-memory test so it does not claim stdout isolation coverage.
