## Summary

Keep open. Current main still lacks the channel MCP stdout isolation fix: `openclaw mcp serve` reaches `loadConfig()` before any `routeLogsToStderr()` call, while config validation warnings can still be logged before the stdio transport starts. The PR is a focused, low-risk production fix, though its added test should be adjusted because the review comment correctly notes it uses `InMemoryTransport` and does not directly exercise stdio stdout isolation.

## Best Possible Solution

Keep this PR open and either land the focused production fix or supersede it with the same owner-surface change on current main. Before landing, adjust the test so it directly covers stderr routing before config load or real stdio stdout isolation; the existing tools/list capability check can remain if renamed to avoid implying it verifies stdout isolation.
