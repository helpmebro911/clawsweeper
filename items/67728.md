## Summary

Keep open. Current main has partial remote-CDP hardening, but not the requested auto-reconnect behavior for a WebSocket that drops after opening or while a CDP command is in flight. The related implementation PR was closed unmerged, and current tests still lock in hard failure for the core mid-call close case.

## Best Possible Solution

Keep this issue open and scope the fix in the browser plugin. Preserve the existing handshake retry, then add explicit recovery for recoverable post-open transport closes: evict stale cached CDP/Playwright connections for later actions, retry safe read/snapshot/list operations with bounded backoff, and avoid replaying ambiguous mutating commands unless they are proven idempotent or have operation-level dedupe. Add targeted tests for close-after-open, mid-command close, next-action recovery, retry exhaustion, and non-replay of unsafe mutations. The closed related PR #67748 can be used as background but should not be treated as shipped.
