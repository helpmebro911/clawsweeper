## Summary

Keep open. The linked issue #48314 was closed because current main now separates think effort from reasoning visibility, but this PR's narrower UX cleanup is still not implemented: /status still hides reasoning when it is off, prints raw values like `on`, and the runtime prompt still says /status only shows Reasoning when enabled. The PR is stale against the current file layout, but it still contains useful remaining work.

## Best Possible Solution

Keep the PR open or replace it with an equivalent fresh patch that ports the UX change to `src/status/status-message.ts`: always show reasoning status, map canonical values to `hidden`/`visible`/`stream`, update the runtime prompt copy and regression tests, and handle legacy unconstrained session strings safely.
