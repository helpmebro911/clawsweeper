## Summary

Keep open. Current main still has the TUI inline reasoning-tag gap for assistant string content: the formatter sanitizes renderability but does not strip or separately surface `<think>...</think>` from plain string payloads, and the stream assembler consumes that formatter for live and final TUI output. The PR is focused and relevant, but it needs adjustment before merge because the bot review flags a real role-gating regression in the proposed diff.

## Best Possible Solution

Keep this PR open for rebase/integration. The best fix is a narrow assistant-only TUI formatter/stream-assembler change that strips inline reasoning when thinking display is off, surfaces it separately when thinking display is on, preserves literal tags in user/command transcript text, and reuses existing shared reasoning-tag utilities where possible.
