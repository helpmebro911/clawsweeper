## Summary

Keep open. Current main still lacks the two job-level fork guards proposed by PR #70002, and both workflows still consume `OPENCLAW_DOCS_SYNC_TOKEN` for upstream-only docs publishing/dispatch work. The provided PR is a narrow CI fix, the related ClawSweeper matches do not supersede it, and there is no protected-label or maintainer-author reason to auto-close it.

## Best Possible Solution

Keep PR #70002 open for maintainer review and likely merge or directly apply the same two job-level guards. The best repository outcome is to make these upstream-only docs workflows skip in forks while preserving current behavior in `openclaw/openclaw`.
