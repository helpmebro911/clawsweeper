## Summary

Keep #70002 open. Current main still lacks the proposed fork guards on both docs workflows, and both jobs still consume OPENCLAW_DOCS_SYNC_TOKEN to act on openclaw/docs. The PR is a narrow two-line CI fix for a real fork-only failure path and is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land the two job-level `if: github.repository == 'openclaw/openclaw'` guards if maintainers agree that these docs publishing workflows should remain upstream-only; no core product code or plugin API work is involved.
