## Summary

Keep open. Current `main` still omits `cache_read_tokens` and `cache_write_tokens` when building cron run-log telemetry, while the run-log types/parser already support those fields. The PR's reported gap is still present and is not implemented on `main`.

## Evidence

- **Telemetry writer still drops cache fields:** `finalizeCronRun` builds `telemetryUsage` with only `input_tokens` and `output_tokens`, optionally `total_tokens`, then writes that object into the run session result. It stores cache counts on the session entry but does not add them to `telemetryUsage`.
