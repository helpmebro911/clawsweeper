## Summary

Keep open. Current main still has the telemetry gap this PR targets: cron isolated runs copy cacheRead/cacheWrite to the session entry, but the telemetry usage object that flows into the run-log JSONL only gets input/output/total token fields. The PR discussion and bot review identify the same narrow producer-side omission, and current types/parser/tests already support the cache fields when present.

## Best Possible Solution

Keep the PR open and refresh or port the narrow fix on top of current main: derive cacheRead/cacheWrite from usage in src/cron/isolated-agent/run.ts, set sessionEntry as today, and also set telemetryUsage.cache_read_tokens/cache_write_tokens when positive. Add a focused regression test around runCronIsolatedAgentTurn or its test harness so a cron run with cache usage returns telemetry usage containing the cache fields, not only a run-log parser round-trip test.
