## Summary

Keep open. Current `main` still drops `cache_read_tokens` and `cache_write_tokens` when cron isolated-agent runs build run-log telemetry, and the latest shipped release shows the same omission, so this PR is addressing a real unfixed gap.

## Evidence

- **Current writer still omits cache token telemetry:** `src/cron/isolated-agent/run.ts` builds `telemetryUsage` with `input_tokens` and `output_tokens`, optionally `total_tokens`, then stores `cacheRead`/`cacheWrite` only on `sessionEntry`. There is no assignment of `cache_read_tokens` or `cache_write_tokens` into the telemetry object written to the run log.
  - file: [src/cron/isolated-agent/run.ts:775](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/cron/isolated-agent/run.ts#L775)
  - command: `nl -ba src/cron/isolated-agent/run.ts | sed -n '760,795p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **Run-log contract already supports those fields:** The public cron usage type and run-log reader both already recognize `cache_read_tokens` and `cache_write_tokens`, which means the gap is in the writer path rather than the schema/parser.
  - file: [src/cron/run-log.ts:307](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/cron/run-log.ts#L307)
