## Summary

Keep open. Current `main` still rotates Telegram partial-mode previews onto a new message at assistant-message boundaries, so the PR's requested single-message coalescing across tool-call turns is not implemented on `main`. The existing tests also codify that split behavior.

## Evidence

- **Partial-mode assistant starts still rotate to a new preview:** `onAssistantMessageStart` unconditionally calls `rotateAnswerLaneForNewAssistantMessage()` unless a narrow skip path applies. That rotation materializes/archive-tracks the old preview and then calls `forceNewMessage()`, which starts a new message instead of continuing edits on the same one.
