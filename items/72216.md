## Summary

Keep PR #72216 open. Current main still has the Nostr setup-status delegation the PR targets: `channel.setup.ts` delegates status resolution to the full `setup-surface.js`, and that full surface imports `nostr-key-utils`, which imports `nostr-tools`. The linked #72182 repro remains a plausible current-main failure path for setup/status scans that do not defer status collection.

## Best Possible Solution

Keep PR #72216 open and land it, or apply an equivalent main change, so Nostr setup status stays on the lightweight `channel.setup.ts` surface while full Nostr configuration/runtime behavior remains delegated only for interactive setup work. Keep the regression test that mocks `setup-surface.js` to throw, and address the small relay-count brittleness before merge if practical.
