## Summary

Keep this PR open. Current main has implemented the configured-default part through merged #72324 and a UI follow-up, but it still does not implement the PR's catalog-backed session-row path: `sessions.list` does not load/pass a gateway model catalog, `listSessionsFromStore`/`buildGatewaySessionRow` have no catalog input, and `chat.history` still resolves defaults without the owning agent. Because the PR explicitly covers those remaining seams and tests, closing it as implemented or superseded would be premature.

## Best Possible Solution

Keep this PR open for maintainer handling, but narrow it against current main. The best path is either a small follow-up that adds a catalog-aware session-row/default resolver for `sessions.list` and any expected `chat.history` parity, or a maintainer decision that catalog-backed defaults are intentionally out of scope with a separate canonical issue. Preserve contributor attribution if this PR's tests or approach are reused.
