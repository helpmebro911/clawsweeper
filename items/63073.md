## Summary

Keep open. Current main and the latest release already use GET for MiniMax Coding Plan remains, so the original method bug from #63056 is resolved, but this PR's remaining host/baseUrl behavior is not implemented. The overlapping #65059 PR is still open and unmerged, so it is a candidate for consolidation rather than a shipped or clearly canonical superseder.

## Best Possible Solution

Keep this PR open for maintainer review or manual consolidation with #65054/#65059. The best fix is a MiniMax-owned usage host resolver that derives from the same provider/auth source as the usage credential, preserves /v1/api/openplatform/coding_plan/remains, supports global and CN baseUrls, avoids unsafe workspace-env routing, and adds regression tests for global, CN, invalid baseUrl, and mixed minimax/minimax-portal configurations.
