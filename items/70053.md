## Summary

Current `main` still sends `sessions_spawn` sub-agent runs through a gateway WebSocket before they reach the `subagent` queue lane, while the docs still describe a dedicated in-process lane. The pairing path also still refuses silent approval for `scope-upgrade`, so the reported gateway-pairing failure mode remains plausible on current code.
