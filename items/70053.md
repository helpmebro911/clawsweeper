## Summary

Keep open. Current main has mitigations around backend shared-token gateway calls, and the issueâ€™s downstream plugin-polling workaround is useful context, but the requested native subagent path still has no in-process dispatch shortcut: `sessions_spawn` still calls the gateway RPC/WebSocket path before the agent run reaches the `subagent` command lane. The docs also now describe the required backend shared-auth caller shape, which confirms the pairing surface still matters rather than being removed.

## Best Possible Solution

Keep the issue open and implement or explicitly reject the requested product boundary. Best code path: add a trusted in-process dispatch seam for native subagent spawns when the gateway runtime is in the same process, preserving the current `agent` handler semantics for scopes, model override authorization, idempotency, timeout/final response behavior, task tracking, and `CommandLane.Subagent` enqueueing. If maintainers choose to keep the WebSocket boundary, update `docs/tools/subagents.md` to say the lane is post-gateway, add focused coverage for `gateway.auth.mode="none"` and generated-token startup behavior, and document the supported server-side auth prerequisite.
