## Summary

Keep #40332 open. Current main still lacks declarative per-binding or per-agent ACPX permission overrides: the ACP runtime/default binding config surfaces only carry agent/backend/mode/cwd/label, while ACPX permissionMode and nonInteractivePermissions remain global plugin config plus a separate session-level permissionProfile control after a session exists.

## Best Possible Solution

Implement the requested declarative permission scoping as a core ACP/runtime contract change: extend AgentRuntimeAcpConfig and bindings[].acp schema/help/docs for permissionMode and nonInteractivePermissions, persist those fields in configured binding metadata, thread them through ACP session initialization or runtime ensure input, and make ACPX resolve precedence as bindings[].acp over agents.list[].runtime.acp over global plugins.entries.acpx.config. Add tests for validation, precedence, configured binding materialization, and ACPX runtime wiring.
