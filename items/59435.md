## Summary

Keep open. The report is recent, concrete, author is not a maintainer, has no protected labels, and describes a supported advanced setup: multiple gateways with unique config paths, state dirs, and ports. Current main still has service reinstall/regeneration paths that appear capable of dropping existing OPENCLAW_* service env values such as OPENCLAW_CONFIG_PATH and rebuilding from the current CLI/update process context, so there is not enough evidence for a conservative close.

## Evidence

- **Issue has actionable repro and affected surface:** Issue #59435 reports OpenClaw 2026.4.1, macOS LaunchAgent ai.openclaw.gateway.plist, three gateway instances with distinct OPENCLAW_CONFIG_PATH values and ports, and a specific failure after openclaw update where the main gateway plist is regenerated to point at another instance config path and port.
  - command: `provided GitHub context for openclaw/openclaw#59435`
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Multiple gateways are a documented advanced setup:** The gateway runbook documents that multiple gateways on one host are supported when each instance has a unique gateway.port, OPENCLAW_CONFIG_PATH, OPENCLAW_STATE_DIR, and workspace, matching the issue's deployment model rather than an incoherent or out-of-scope report.
  - file: [docs/gateway.md](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/gateway.md)
  - command: `web inspection of docs.openclaw.ai/gateway and GitHub docs/cli/gateway.md`

- **Installer path can rebuild service env from current process context:** runDaemonInstall reads the current config snapshot, resolves the port from the current config/options, reads existing service env only when loaded, merges it into the invocation env, and then calls buildGatewayInstallPlan with installEnv, port, and existingServiceEnv before service.install writes the resulting service definition.
  - file: [src/cli/daemon-cli/install.ts:798](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/src/cli/daemon-cli/install.ts#L798)

- **Existing OPENCLAW_* service env is not preserved:** collectPreservedExistingServiceEnvVars skips HOME, PATH, TMPDIR, and all OPENCLAW_* keys from the existing service environment. That means an existing OPENCLAW_CONFIG_PATH from the plist is not preserved by this helper; the regenerated environment comes from durable/current env plus generated service env.
  - file: [src/commands/daemon-install-helpers.ts:1186](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/src/commands/daemon-install-helpers.ts#L1186)

This does not prove the reported bug is fixed on main.
