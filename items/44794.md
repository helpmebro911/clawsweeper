## Summary

Current `main` does not implement config-driven `gateway.preStartCommand` or `gateway.wrapperScript` support. The managed gateway install path still builds a fixed `openclaw gateway --port …` command, and the documented install surface only exposes `--port`, `--runtime`, `--token`, `--force`, and `--json`. This remains a valid open feature request rather than something conservatively closable.

## Evidence

- **No matching config fields exist:** Repository search found no occurrences of `preStartCommand` or `wrapperScript` in current source/docs, so the requested config keys are not implemented.
