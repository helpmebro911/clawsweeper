## Summary

Keep open. The report has multiple confirmations, is not old enough for stale-insufficient-info, and current main still routes desktop Gemini CLI OAuth through the same localhost:8085 flow with only narrow fallback handling. I did not find source, docs, tests, changelog, or related-context evidence proving the reported AbortError/login failure is fixed or superseded.

## Best Possible Solution

Keep this issue open for the Google plugin owner. The likely maintainer path is to reproduce against current main or the latest release on macOS with a real Gemini CLI install, then harden extensions/google/oauth.ts and the shared callback helper as needed, for example by starting the localhost callback listener before opening the browser, adding clearer timeout/manual fallback behavior for AbortError during token exchange/project discovery, and adding a focused regression test for the desktop callback flow.
