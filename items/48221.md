## Summary

Keep PR #48221 open. Current main has superseded the exact implementation shape with provider-owned replay hooks, but it has not implemented the PR's central generic behavior for arbitrary non-native `anthropic-messages` proxies with custom model aliases. The PR itself should not be merged as-is because review comments identified an over-broad Bedrock/Anthropic-family condition, but the underlying custom-proxy issue is still plausible and not tracked by a clearer canonical item.

## Best Possible Solution

Keep the item open but require a rework against current main. The best path is a scoped replay-policy change in the current architecture: either add an explicit opt-in/metadata path for unowned `anthropic-messages` proxies that cannot preserve signatures, or add provider-owned `buildReplayPolicy` hooks for affected bundled proxy plugins such as LiteLLM if maintainers want provider-specific defaults. Regression coverage should include a custom alias without `claude`, native `anthropic`, Bedrock non-Claude, and modern Claude proxy behavior so the prompt-cache preservation tradeoff is deliberate.
