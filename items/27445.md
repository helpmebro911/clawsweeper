## Summary

Keep #27445 open. Current main has adjacent subagent announce routing work, nested parent delivery, and ACP-only `streamTo: "parent"`, but it still does not expose the requested native `announceTarget` default or per-`sessions_spawn` override. Top-level native completion announces still use the external delivery path when the requester is dormant, so the core parent-only orchestration policy remains unimplemented.

## Best Possible Solution

Keep #27445 open as a core subagent announce policy request. The likely implementation path is to add a public `announceTarget` policy to native `sessions_spawn` and `agents.defaults.subagents`, update the strict config schema and public docs, and add coverage for current channel delivery, top-level parent-only routing, active requester steering, nested subagent delivery, and interactions with the complementary #33478 callback/thread hardening work.
