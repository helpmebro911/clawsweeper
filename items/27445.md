## Summary

Keep #27445 open. Current main has adjacent subagent announce routing work, nested requester injection, and ACP-only `streamTo: "parent"`, but it still does not provide the requested native `announceTarget` default or per-`sessions_spawn` override for top-level parent-session orchestration.

## Best Possible Solution

Keep #27445 open as a core subagent announce-routing policy request. The likely product path is to add a public `announceTarget` policy with default `"channel"` and opt-in `"parent"` at both `sessions_spawn` and `agents.defaults.subagents`, update strict config/schema/generated docs, and add regression coverage for current channel delivery, parent-only top-level routing, active requester steering, nested subagent delivery, ACP `streamTo` separation, and interactions with #33478/#8299.
