## Summary

Keep open. Current main still deliberately blocks the behavior this PR is trying to add: the Control UI markdown renderer only emits `<img>` for base64 `data:image/...` markdown images, tests still assert remote https images are flattened to alt text, and the Control UI CSP still restricts `img-src` to self/data/blob. The PR remains a relevant implementation candidate for the linked open bug #40945, but the renderer-only change is incomplete against current main and carries an explicit remote-image privacy/security tradeoff that needs maintainer review.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require an updated end-to-end design before merge: renderer behavior, tests, CSP changes, public docs, and an explicit maintainer decision on privacy controls such as https-only support, `referrerpolicy="no-referrer"`, lazy loading, a user setting, or a domain allowlist. If maintainers decide the current no-remote-fetch policy should stand, close this PR only after recording that decision on linked #40945.

## Risks / Open Questions

- The PR's renderer-only patch is incomplete because current Control UI CSP blocks remote `http(s)` images from loading.
- Allowing arbitrary remote images would cause operator browsers to fetch attacker- or model-supplied URLs, exposing IP address, User-Agent, and read timing; the PR does not resolve that policy decision.
- The PR allows plain `http://` images, which may be undesirable for a local control surface without an explicit maintainer decision or allowlist/settings design.
