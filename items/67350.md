The PR should remain a rework/review candidate because the provided review discussion also identifies unresolved blocking functional and security/privacy issues at the PR head.

## Best Possible Solution

Keep this PR open as a rework candidate. The maintainer path is to rebase onto current main, split or narrow the cross-cutting work where practical, fix the undefined health helper and account-scoped Telegram direct-session protection gaps, justify any new plugin SDK runtime exports, and add focused regression coverage for routing, degraded health, structured locks, Telegram poller liveness, heartbeat/task delivery protection, and overflow compaction provenance.

## Risks / Open Questions

- The PR is broad and cross-cutting across routing, persistence, health/status, Telegram runtime, heartbeat/task delivery, and plugin SDK surfaces, so it needs maintainer review rather than cleanup closure.
