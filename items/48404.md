## Summary

Keep open. The report is a recent, concrete regression on ImmortalWrt/OpenWrt where executing the CLI segfaults while importing the entry module succeeds. Current main has generic startup and help fast paths plus low-memory/package-bloat fixes, but I did not find source, changelog, test, or related-item evidence that specifically resolves or supersedes this OpenWrt segmentation fault.

## Evidence

- **Concrete recent bug report:** Issue #48404 reports ImmortalWrt/OpenWrt x86_64 with Node v22.22.1, OpenClaw v2026.3.13, `openclaw --help` and `node dist/entry.js --help` both segfaulting, while importing `dist/entry.js` succeeds. The issue is less than 60 days old and has no protected maintainer author/label, but it is concrete enough to keep investigating.
- **No specific OpenWrt/segfault fix found:** A targeted search for ImmortalWrt/OpenWrt and segfault/segmentation references found only joke tagline text, not a changelog entry, test, or code path showing this platform-specific crash was fixed.
