## Summary

Keep open. Current `main` does not implement WebChat document uploads for PDF/Word/TXT. The WebChat/Control UI path is still image-only end to end: the file picker accepts only `image/*`, the client serializes every attachment as `type: "image"`, and the gateway attachment parser drops non-image payloads.

## Evidence

- **WebChat file picker is image-only:** The WebChat attachment accept filter is hard-coded to `image/*`, and supported MIME detection only returns true for `image/` types.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/chat/attachment-support.ts#L1)
  - command: `nl -ba ui/src/ui/chat/attachment-support.ts | sed -n '1,80p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **WebChat input handlers reject non-images:** Paste, file-select, and drag-drop handlers only keep files whose MIME passes the image-only filter; the preview renderer also assumes `<img src=...>` attachments.
  - file: [ui/src/ui/views/chat.ts:206](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/views/chat.ts#L206)

- **Client sends attachments as image payloads only:** `buildApiAttachments()` converts every queued attachment into `{ type: "image", mimeType, content }` before calling `chat.send`; there is no document/file variant on the WebChat send path.
  - file: [ui/src/ui/controllers/chat.ts:213](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/controllers/chat.ts#L213)
