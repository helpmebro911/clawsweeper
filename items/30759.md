## Summary

Keep #30759 open. Current main still implements WebChat/Control UI attachments as an image-only path, while this issue asks for document upload/extraction for PDF, Word/DOC/DOCX, and TXT. Existing PDF/document extraction support is available through other surfaces, but it is not wired into the WebChat `chat.send` upload path and does not cover Word/DOC/DOCX here. Related open items are overlapping but not a strong canonical duplicate for this specific request.

## Best Possible Solution

Keep #30759 open unless maintainers explicitly mark another issue as the canonical WebChat document-upload tracker. A complete fix belongs in the core WebChat/Control UI and Gateway `chat.send` contract: extend composer accept/preview/state with filenames and document metadata, serialize non-image attachments, add Gateway MIME/size validation and visible errors, reuse existing text/PDF extraction where appropriate, add Word/DOC/DOCX extraction or provider offload behavior, and cover the UI/Gateway paths with focused tests and docs.
