## Summary

Keep #30759 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still has the same image-only WebChat/Control UI upload path, and the requested PDF/Word/TXT document extraction workflow is not shipped. Active PR #70947 is useful related work for generic non-image WebChat attachments, but it is open/unmerged and does not fully cover document text extraction or Word/DOC/DOCX processing.

## Best Possible Solution

Keep #30759 open unless maintainers explicitly consolidate it into a canonical WebChat document-upload tracker or PR. A complete fix should extend Control UI attachment acceptance and preview metadata beyond images, preserve filenames/MIME through `chat.send`, add Gateway MIME/size validation and visible unsupported-file errors, route PDFs/text/Office documents into an agent-visible extraction or media-reference path, reuse the existing PDF/document-extractor surface where appropriate, add Word/DOC/DOCX extraction or a clear offload/provider behavior, and cover UI/Gateway/document cases with focused tests and docs.
