## Summary

Keep this PR open. Current main still ignores `replyToMode: "off"` for Google Chat’s implicit inbound thread context and typing placeholder, so the reported bug is not fixed. The PR is focused and includes useful live DM verification, but it needs revision before merge because its delivery-time guards would also flatten explicit reply directives that the shared reply-threading contract preserves.

## Best Possible Solution

Keep this PR open and request a focused revision: suppress implicit Google Chat inbound thread propagation in `extensions/googlechat/src/monitor.ts` for `ReplyToId`, `ReplyToIdFull`, and the typing placeholder when `account.config.replyToMode === "off"`; leave final delivery honoring `payload.replyToId` so explicit reply tags still work; rebase onto the current split delivery files; add a Google Chat monitor regression test for the off-mode DM path.
