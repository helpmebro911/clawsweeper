## Summary

Keep PR #47672 open. Current main still has the reported pre-doctor update UX gap: `openclaw doctor` prints a static update note and calls `runGatewayUpdate` without the existing progress controller. The PR is a narrow one-file fix, and while #47493 tracks the same area, #47672 is not clearly superseded because its patch is smaller and includes the cleanup/TTY safeguards noted in review.

## Best Possible Solution

Keep the capability open for maintainer review. The best path is to land one narrow `src/commands/doctor-update.ts` change that reuses `createUpdateProgress`, passes `progress` into `runGatewayUpdate`, keeps TTY-gated output, and guarantees spinner cleanup, then close the duplicate PR with a link to the landed one.
