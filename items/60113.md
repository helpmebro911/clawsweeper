## Summary

Keep open. Current `main` still contains the unguarded agent transport text sanitization path that can call `.replace()` on runtime-invalid values, the Bedrock/Anthropic serialization path still assumes `text`/`thinking` are always strings, and `embedded run agent end` still does not record `error.stack`. The linked fix PR is also still open rather than landed.

## Evidence
