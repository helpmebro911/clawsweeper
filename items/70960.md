## Summary

Keep open. Current main still does not fully satisfy the bundled report: the QClaw Windows Electron installer surface is not present in this checkout, one Control UI session/agent selector path still prefers identity name over `agents.list[].name`, and the topbar still lacks a token/cost usage display even though usage data is exposed elsewhere.

## Best Possible Solution

Keep this issue open or split it into focused follow-ups: route the QClaw Windows installer deletion report to the QClaw/Electron installer owner with the supplied log and registry details; in OpenClaw Control UI, make all agent/session selector surfaces consistently prefer `agents.list[].name` before identity-derived names; and either restore the topbar token/cost usage display or intentionally replace it with a clearly equivalent current-session usage surface backed by existing session usage fields.
