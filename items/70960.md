## Summary

Keep #70960 open. Current main partially addresses the agent-name surface because `agents.list` returns configured names and the Agents/session selectors can render them, but the broader report is not fully resolved: the assistant identity path still does not use `agents.list[].name` as the primary assistant display name, the requested Control UI top-right usage/cost display is not present, and the QClaw Electron installer path described by the reporter is not represented in this repository enough to close the whole multi-part issue as fixed or out of scope.

## Best Possible Solution

Keep the issue open, or split it into narrower maintainer-triaged items: route the QClaw Electron installer disappearance to the repository or owner that ships QClaw, update `resolveAssistantIdentity`/Control UI tests if `agents.list[].name` should be the canonical display-name source, and decide whether to restore a compact current-session token/cost indicator in the Control UI topbar using the existing session usage fields.
