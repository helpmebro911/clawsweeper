## Summary

Current main only partially addresses the bundled report. The Agents page selector now prefers `agents.list[].name`, but other Control UI agent display paths still prefer identity-derived names, the topbar token/cost/cache usage display is still absent, and the QClaw Windows Electron upgrade/delete path appears to belong outside this source tree rather than being fixed here.

## Best Possible Solution

Keep this issue open or split it into focused follow-ups. Route the QClaw Windows Electron upgrade deletion report to the QClaw installer owner with the supplied log and registry details. In OpenClaw Control UI, make all agent display surfaces consistently prefer `agents.list[].name` before identity-derived names where that is the intended contract, and restore or deliberately replace the topbar current-session token/cost/cache usage display with an equivalent visible usage surface backed by existing session/usage data.
