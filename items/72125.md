## Summary

Keep #72125 open. Current main still routes normal Feishu conversation replies through the Feishu-specific reply dispatcher, which sends text/card/media/streaming replies directly and does not emit plugin `message_sent` or internal `message:sent` hooks. The linked PR #72138 appears to be the right active fix path, but it is open/unmerged in the provided context, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #72125 open and complete review/landing of PR #72138 or an equivalent Feishu-owned fix. The fix should emit canonical plugin `message_sent` and internal `message:sent` for successful and failed normal Feishu reply branches, pass routed session and group mirror metadata from `extensions/feishu/src/bot.ts`, and add Feishu regression coverage for text, card, media, streaming, and failure paths.
