## Summary

Keep open. Current `main` at `cc0f3e0e40364023cc3a39227a411898ee68ce3b` still lacks the UI-side leaked-row filtering described in linked PR #67036, and the backend `SystemEvent` shape still has no audience/visibility discriminator. The reported Control UI transcript leak remains plausible and is still being actively tracked by open follow-up work rather than already fixed on `main`.

## Evidence

- **UI history filter on main is still narrow:** `shouldHideHistoryMessage` only hides assistant `NO_REPLY` messages and the synthetic transcript-repair tool result. It does not match leaked `System (untrusted)` / exec-status rows.
