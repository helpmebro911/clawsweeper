## Summary

Keep this issue open. Current main still has the explicit `models.providers.*.request.proxy.mode = "env-proxy"` workaround, but it does not implement the requested startup warning for proxy environment variables plus remote providers without request proxy configuration. The related implementation candidate, #68023, is closed and unmerged in the provided GitHub context.

## Best Possible Solution

Implement a non-breaking startup warning in or near `logGatewayStartup`: use the existing proxy-env helpers to detect configured proxy variables, inspect configured model providers for remote HTTP(S) targets, skip local/private/non-remote providers and providers with explicit `request.proxy`, and warn once with the `models.providers.<provider>.request.proxy.mode = "env-proxy"` remediation. Add focused startup-log tests for warn, no-env, all-configured, and local-provider cases, using closed #68023 as prior design context rather than as a shipped fix.
