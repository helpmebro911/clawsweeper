## Summary

Keep this PR open. Current main still has the reported nested skills-root bug: it redirects to `dir/skills` before validating that `dir` already contains loadable skills, so root-level skills under a managed/workspace/extra skills root can be skipped. The PR is a focused implementation candidate with a targeted regression test and no visible unrelated security-sensitive changes in the provided diff.

## Best Possible Solution

Keep this PR open and review or land a version of the fix that verifies the configured directory is already a loadable skills root before redirecting to its nested `skills` directory. Preserve the regression test for the root-plus-nested layout, and align the guard with the loader’s existing validity, candidate-limit, containment, and size-cap behavior.
