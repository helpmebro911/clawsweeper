## Summary

Keep #13583 open. Current main still has adjacent pre-tool blocking, one-pass final-answer revision, and outbound send cancellation hooks, but it does not implement the requested fail-closed response-gate contract for mandatory tool-call accounting, typed validators, retry/fallback policy, or audit records. The latest maintainer/Codex discussion also explicitly kept this issue open for that remaining core API gap.

## Best Possible Solution

Keep #13583 open as the canonical response-gate policy-as-code design item. The best maintainer path is a versioned core/plugin API for mandatory tool-call accounting, typed response validators, fail-closed final-response gating, bounded retry/fallback behavior, and audit records; domain-specific policy packs can then live in plugins or ClawHub.
