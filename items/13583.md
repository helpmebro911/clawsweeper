## Summary

Keep open. Current main has useful adjacent hooks: fail-closed `before_tool_call` execution blocking, outbound `message_sending` cancellation, and a Codex `Stop` relay into `before_agent_finalize`. It still does not provide the requested pre-response policy-as-code gate: no two-phase tool-only planning protocol, no `responseGate`/`require_tools` validator config, no mandatory-tool accounting, no fail-closed final-response validator layer, and no audit/retry/fallback contract. The issue discussion also distinguishes existing tool-call/plugin guardrail implementations from the missing core response-gate capability.

## Best Possible Solution

Keep this open as the canonical response-gate policy-as-code design item. Build a versioned core/plugin API for mandatory tool-call accounting, typed validators, fail-closed final-response gating, bounded retry/fallback semantics, and audit output. Once that mechanical response-gate contract exists, service-specific policies and policy packs can live in ClawHub plugins.
