## Summary

Keep #25493 open. Current main still hardcodes the requested Safety and OpenClaw CLI Quick Reference prompt sections, does not load proposed section override files such as SAFETY.md, and the existing alternatives are either whole-prompt replacement or narrow provider/runtime section overrides that do not cover Safety or CLI reference. The related PR #25938 was closed unmerged, and the latest issue discussion already records a Codex review keeping this open.

## Best Possible Solution

Keep #25493 open. The best maintainer path is to either add documented, tested section/template overrides for selected OpenClaw-owned prompt sections, with explicit safety and prompt-cache constraints, or make a deliberate product decision that Safety and other core sections are intentionally not section-editable and document systemPromptOverride as the supported full-replacement escape hatch.

## Risks / Open Questions

- Closing now would drop an actionable core prompt-builder/product-policy request that is not currently shipped.
- Implementing section editing for Safety needs maintainer judgment about safety semantics, prompt-cache stability, and which OpenClaw-owned sections should be user-editable.
- The existing whole-prompt override and provider/plugin prompt contributions are useful workarounds, but they require replacing the entire prompt or only cover narrow provider-owned sections, so they are not equivalent to the requested section-template feature.
