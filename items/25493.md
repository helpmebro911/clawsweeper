## Summary

Keep #25493 open. Current main still hardcodes the requested Safety and OpenClaw CLI Quick Reference sections, does not load proposed section override files like SAFETY.md, and the existing alternatives are whole-prompt replacement, hook-based full replacement, or narrow provider/runtime section overrides. The related PR #25938 was closed unmerged, and the latest issue discussion already recorded the same keep-open conclusion; the relevant files are unchanged since that review SHA.

## Best Possible Solution

Keep #25493 open. The best maintainer path is to either add documented, tested section/template overrides for selected OpenClaw-owned prompt sections such as Safety, system tips/tool-call style, and CLI reference, or make an explicit product decision that Safety and other core sections are intentionally not section-editable and document systemPromptOverride plus prompt hooks as the supported escape hatches.

## Risks / Open Questions

- Closing this now would drop an actionable core prompt-builder/product-policy request that current main still does not implement.
- Existing whole-prompt replacement and prompt hooks are useful workarounds, but they are not equivalent to documented section/template overrides because they require replacing or appending to the prompt rather than editing selected OpenClaw-owned sections.
- Making Safety editable needs explicit maintainer judgment around safety semantics, prompt-cache stability, and which core sections should be overrideable.
