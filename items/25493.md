## Summary

Keep #25493 open. Current main has some prompt customization surfaces, including a full `systemPromptOverride`, plugin `before_prompt_build` mutation, and provider-owned overrides for a small set of sections, but it does not implement the requested editable section/template workflow. The specific example in the issue, overriding the Safety section through a workspace file such as `SAFETY.md`, remains missing: Safety is still hardcoded in the core prompt builder, and provider section overrides are limited to `interaction_style`, `tool_call_style`, and `execution_bias`. The linked PR #25938 is relevant prior work and directly attempted this feature, but it was closed unmerged, so it does not supersede or implement the issue on main.
