## Summary

Keep open. Current main still has the reported Control UI Debug Snapshots behavior: disconnected refresh returns without setting any user-visible state, the Snapshots card only changes the Refresh button label while loading, and the debug render path still does not receive connection state or a disabled reason to display inline.

## Best Possible Solution

Keep the issue open. The maintainable fix belongs in the Control UI Debug surface: pass connection state or a refresh-disabled reason into `renderDebug()`, show an inline loading/progress state inside the Snapshots card while `debugLoading` is true, show a disconnected callout or disabled reason when refresh cannot run, and add focused tests for `loadDebug()` and `renderDebug()` behavior. PR #39806 should be reviewed or refreshed as the existing implementation attempt.
