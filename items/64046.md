## Summary

Keep open. Current main has meaningful partial coverage for log redaction and Control UI masking, but the broader sensitive-data request is not implemented: plaintext config remains supported, primary/included config writes still serialize JSON to disk, and normal config backups still copy secret-bearing content.

## Best Possible Solution

Keep this issue open as the umbrella until maintainers either split and link canonical sub-issues or land the remaining work: an at-rest secret strategy for config/auth artifacts/backups/service outputs, a completed redaction audit for diagnostic/raw sinks, a full Control UI sensitive-field inventory, and regression coverage across persistence, logs, API snapshots, diagnostics, and UI masking.
