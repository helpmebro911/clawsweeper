## Summary

Keep open. Current main has shipped meaningful redaction and UI masking improvements, but the central feature request is broader sensitive-data handling and remains only partially implemented: primary config writes, targeted config mutations, and generic backups still persist JSON/plain copies, and the docs still describe plaintext-at-rest audit findings for openclaw.json, auth-profiles.json, .env, and generated models.json.

## Best Possible Solution

Keep this issue open as the umbrella until maintainers either split and link canonical sub-issues or ship the remaining work: an at-rest secret strategy for primary config/auth artifacts/backups/service outputs, a completed redaction audit for remaining diagnostic/raw sinks, a full UI sensitive-field sweep, and regression coverage across config, logs, diagnostics, API snapshots, and Control UI.
