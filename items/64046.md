## Summary

Current main partially addresses #64046, especially Control UI masking and sink-level file log redaction, but the umbrella request is not complete. Plaintext config remains supported and written as JSON, config backups still copy the config file, session transcript persistence still appends raw messages without a redaction gate, and the issue discussion explicitly tracks remaining phases rather than treating the work as shipped.

## Best Possible Solution

Keep #64046 open as the umbrella until maintainers either finish or deliberately split and supersede the remaining work. The best path is to mark completed sub-work such as file-log redaction and UI masking, then track the still-missing pieces explicitly: at-rest config/backup strategy, transcript and diagnostics redaction parity, low-frequency sink audit, and regression coverage. If maintainers prefer narrower tracking, create/link canonical subissues for those slices before closing the umbrella as superseded.
