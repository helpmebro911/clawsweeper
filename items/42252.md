## Summary

Keep open. Current `main` partially addresses the report: doctor now audits and repairs gateway service entrypoint drift, and gateway-status docs explain runtime-vs-connectivity mismatches. But doctor still reports gateway health and daemon runtime as separate sequential steps, so mixed output like `Gateway not running.` followed by `Runtime: running ...` is still possible, and I found no doctor-side consolidated verdict, JSON diagnosis block, or Telegram 409 duplicate-instance hinting.
