## Summary

Keep this issue open. Current main has deterministic Discord native-command routing through channel routes, configured ACP bindings, thread bindings, and CommandTargetSessionKey, but it still does not implement explicit `/new @AgentName` or `/reset @AgentName` targeting for shared multi-agent Discord channels. The prior ClawSweeper comment already reached the same keep-open conclusion, and the current checkout still matches that code shape.

## Best Possible Solution

Keep the issue open and implement a focused Discord native-command enhancement: add a supported optional target-agent option or mention/name parser for `/new` and `/reset`, resolve it through existing Discord routing, agent config, bindings, and authorization rules, set the resulting CommandTargetSessionKey only for the selected agent/session, and add regression tests for shared guild channels, configured ACP bindings, thread-bound sessions, disabled/blocked channel parity, unauthorized target attempts, and the no-target default behavior. Treat highest-context or active-participant defaulting as a separate maintainer product decision if desired.
