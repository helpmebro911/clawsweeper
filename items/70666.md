## Summary

Keep open. Current main still appends the unscheduled-reminder note whenever a reminder-looking reply has `successfulCronAdds === 0` and no same-session OpenClaw cron job, and plugin-managed reminder writes do not increment that counter or have a public scheduling signal/config opt-out on main. PR #71170 is a relevant proposed fix, but it is open and unmerged in the provided GitHub context, so the issue is still actionable.

## Best Possible Solution

Keep #70666 open until PR #71170 or an equivalent fix lands on main. The minimal maintainer path is a documented `agents.defaults.reminderGuard` plus per-agent override with config/schema/docs/tests, or a reviewed plugin SDK signal that lets registered tools report a scheduled event into the same telemetry the guard uses.
