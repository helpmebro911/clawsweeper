## Summary

Keep open. Current main still only trims cron model and fallback strings, so the OpenRouter cron model-ID whitespace bug is not fixed on main. This PR is a plausible implementation candidate, but its blanket whitespace stripping would also rewrite supported model aliases that intentionally contain spaces, including persisted jobs normalized during load.

## Best Possible Solution

Keep this PR open for revision, or supersede it with a narrower maintainer patch. The fix should preserve aliases exactly after trim, repair accidental whitespace only for clear provider/model refs or after alias lookup fails, cover payload and top-level create/patch paths for both `model` and `fallbacks`, and add regression tests for the OpenRouter corruption case plus spaced primary and fallback aliases.

## Risks / Open Questions

- Blanket `.replace(/\s+/g, '')` would rewrite valid primary aliases such as `My Fast Model`, `DeepSeek R1`, or `Qwen3 8B` before alias lookup, causing invalid-model behavior.
- The same rewrite would break fallback aliases and can remove the intended fallback chain.
- Because persisted cron jobs are normalized during load, landing this PR as-is could break existing jobs that already use spaced aliases.
- Closing this PR as obsolete would lose a real bug report and candidate fix because current main still leaves the OpenRouter cron whitespace-corruption path unresolved.
