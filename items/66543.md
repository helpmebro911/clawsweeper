## Summary

Keep open. Current main still lacks the cron model-ID whitespace repair, but this PR's blanket whitespace stripping is not safe because supported model aliases can contain spaces. GitHub still shows this PR open and linked to the closed cron corruption bug. ([github.com](https://github.com/openclaw/openclaw/pull/66543))

## Best Possible Solution

Keep this PR open for revision, or supersede it with a narrower maintainer fix. The fix should preserve bare aliases, repair accidental whitespace only for clear provider/model refs or after alias lookup fails, cover payload and top-level create/patch paths for both `model` and `fallbacks`, and add regression tests for the OpenRouter corruption case plus spaced primary and fallback aliases.

## Risks / Open Questions

- Blanket internal-whitespace stripping can break valid primary model aliases such as `My Fast Model`, `DeepSeek R1`, or `Qwen3 8B`.
- The same stripping can break fallback aliases and remove the intended fallback chain.
- Because persisted cron jobs are normalized during load, existing jobs using spaced aliases could start failing after this PR.
- The linked OpenRouter model-ID corruption still appears unresolved on current main, so closing this PR as implemented would lose a real bug without a safe shipped fix.
