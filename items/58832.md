## Summary

Keep #58832 open. Current main still has an explicit Slack DM auto-thread path when `replyToMode === "all"`: top-level direct messages can use their own message timestamp as `autoThreadId`, and direct messages are not part of the `isRoomish` branch that suppresses top-level channel/group thread keys. That matches the reported failure mode rather than proving it fixed.

## Evidence

- **Current main Slack routing still auto-threads top-level DMs for replyToMode=all:** In `resolveSlackRoutingContext`, `autoThreadId` is set when `!isThreadReply && replyToMode === "all" && threadContext.messageTs`; `canonicalThreadId` then uses that `autoThreadId` for non-roomish direct messages, so a top-level DM can still produce a thread-scoped session key from its own message timestamp.
