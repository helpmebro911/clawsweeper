## Summary

Keep PR #70742 open. Current main still has the boolean timeout helper and still falls through to `flushPendingToolResults()` for timed-out waits unless `clearPendingOnTimeout` is set and a clear method exists. Existing production cleanup call sites pass `clearPendingOnTimeout: true`, so the shipped cleanup hardening covers teardown, but the PR’s broader helper contract and regression-test change are not implemented on main.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to decide the intended contract for `flushPendingToolResultsAfterIdle()`: if any idle-wait timeout should be treated as forced cleanup, land a version of the PR that keeps the structured timeout state and updates the regression test; if non-cleanup timeouts should continue flushing synthetic results, close only with an explicit maintainer explanation of that intended behavior.
