## Summary

Keep #72037 open. Current main has moved the bundled ACPX runtime to acpx 0.6.1, but the OpenClaw Codex ACP wrapper still defaults to spawning npm/npx for @zed-industries/codex-acp@^0.11.1, and the bundled plugin dependency graph only includes acpx itself. The related #60693 comment points at the same bundling idea but is a broad multi-patch issue, and #71549 was a closed/unmerged ACPX bump rather than a proven fix for this packaging behavior.

## Best Possible Solution

Keep this focused issue open. The best path is to make the Codex ACP adapter available without per-session npm/npx execution, preferably by fixing the published acpx/runtime packaging or OpenClaw's bundled ACPX plugin dependency staging, then updating the generated Codex wrapper to spawn the bundled/local codex-acp binary while preserving the isolated CODEX_HOME bridge. Add regression coverage around the default Codex agent command and packed runtime dependencies, and keep #60693 as related background rather than the canonical tracker for this specific bug.
