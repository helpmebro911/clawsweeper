## Summary

Keep open. Current main still implements session maintenance as a single global policy: the public config type/schema expose only one `session.maintenance.pruneAfter`/`maxEntries` set, and the save/load/cleanup paths apply that one resolved `pruneAfterMs` to every session entry. The KeepMyClaw comment is useful as a backup/retention workaround or complement, but it does not implement the requested in-repo per-pattern cleanup semantics, and the related per-agent/subagent retention items do not fully supersede this per-pattern request.
