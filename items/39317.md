## Summary

Keep #39317 open. Current main still only treats the direct CLI run as failed when the process exits non-zero or is killed; an exit-0 `codex-cli` JSONL run is parsed into text and returned as a successful CLI result without a semantic sandbox-denial/error signal. The prior focused fix PR #39460 was closed unmerged, and the related #66081 work is explicitly scoped to final-message artifacts rather than failure classification.

## Best Possible Solution

Keep the issue open and revive the substance of #39460 on current main: parse structured Codex JSONL records for sandbox-denied or failed command/tool/action/error/status signals after exit 0, fail the run before `buildCliRunResult`, add a local `sandbox_denied` failover/status/auth-profile policy if maintainers want that public contract, avoid assistant-message keyword matching, and add focused parser plus runner regression tests for real denials and false-positive discussion cases. Treat #66081 as related final-answer reliability work, not a replacement for semantic failure classification.
