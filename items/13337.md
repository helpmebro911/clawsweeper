## Summary

Keep #13337 open. Current main still does not implement a Vapi provider for the bundled Voice Call plugin. The Clawdia/Vapi bridge is useful adjacent external work, but it does not add `provider: "vapi"` to `@openclaw/voice-call`, and the current public plugin capability surface does not expose a telephony-provider registration seam that would make this exact request safely external-only.

## Best Possible Solution

Keep #13337 open for maintainer/product judgment or an implementation PR. A complete bundled fix would add a Vapi `VoiceCallProvider`, Vapi config/env validation, runtime provider selection, manifest UI/env metadata, focused provider/runtime tests, and Voice Call docs. If maintainers prefer external ownership, first document the supported Clawdia/Vapi bridge path or add a public telephony-provider registration seam so external plugins can satisfy `provider: "vapi"` without patching the bundled Voice Call plugin.
