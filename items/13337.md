## Summary

Keep open. Current main still does not contain a bundled Vapi provider for the Voice Call plugin, and the exact requested integration cannot currently be added to the existing Voice Call provider surface by an external plugin without either modifying `@openclaw/voice-call` or designing a new phone-provider extension seam. The showcased Clawdia/Vapi bridge is useful overlap, but it is not the requested first-class `voicecall` provider implementation.

## Best Possible Solution

Keep the issue open for maintainer/product judgment or an implementation PR. A complete bundled fix would add a Vapi provider class, config/schema and manifest metadata, runtime registration, tests, and Voice Call docs. If maintainers prefer external ownership, the better product path is to document/endorse the ClawHub or bridge approach, or add a public phone-provider extension seam before closing this specific bundled-provider request.
