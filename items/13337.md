## Summary

Current `main` still does not include a Vapi provider in the bundled Voice Call plugin. The adjacent Clawdia Vapi bridge is useful external work, but it does not implement `provider: "vapi"` for `@openclaw/voice-call`, and I did not find a public telephony-provider registration seam that would make this core issue safely external-only.

## Best Possible Solution

Keep #13337 open for maintainer/product judgment or an implementation PR. A bundled implementation should add a Vapi `VoiceCallProvider`, Vapi config schema/env resolution/validation, runtime provider selection, manifest UI/env metadata, focused provider/runtime tests, and Voice Call docs. If maintainainers prefer external ownership, first document the supported external bridge path or add a public telephony-provider registration capability for the Voice Call plugin.
