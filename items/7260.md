## Summary

Current main already implements the requested compaction continuity work: active-task preservation in merge instructions, verbatim preservation of recent turns, structured summary sections for pending asks/TODOs/identifiers, and a surfaced `recentTurnsPreserve` config knob. The changelog records these compaction continuity and safeguard structure changes in release `2026.3.7`.

## Evidence

- **Compaction merge instructions preserve active work:** The staged-summary merge prompt explicitly preserves active tasks/status, batch progress, the user's latest request, decisions, TODOs, and commitments.
  - file: [src/agents/compaction.ts:24](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/agents/compaction.ts#L24)
- **Recent turns are preserved verbatim outside the summary:** Safeguard compaction resolves `recentTurnsPreserve`, splits out recent messages, formats them under `## Recent turns preserved verbatim`, and appends that suffix after the main summary body.
  - file: [src/agents/pi-hooks/compaction-safeguard.ts:781](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/agents/pi-hooks/compaction-safeguard.ts#L781)
- **Structured summary format is enforced:** The safeguard requires exact headings `## Decisions`, `## Open TODOs`, `## Constraints/Rules`, `## Pending user asks`, and `## Exact identifiers`, and the fallback summary uses the same structure.
  - file: [src/agents/pi-hooks/compaction-safeguard-quality.ts:10](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/agents/pi-hooks/compaction-safeguard-quality.ts#L10)
- **Config surface exposes preserved recent turns:** `AgentCompactionConfig` includes `recentTurnsPreserve`, documented as preserving the most recent user/assistant turns verbatim; schema/help also validate and describe it.
  - file: [src/config/types.agent-defaults.ts:425](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/config/types.agent-defaults.ts#L425)
- **Regression tests cover the requested behavior:** Tests verify preserved recent turns, required headings/quality retries, and capped summaries that still retain split-turn and recent-turn context.
  - file: [src/agents/pi-hooks/compaction-safeguard.test.ts:616](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/agents/pi-hooks/compaction-safeguard.test.ts#L616)
- **Shipped release note:** The changelog for `2026.3.7` says compaction continuity was expanded to preserve active task status, latest user request, and follow-up commitments, and the safeguard was hardened around structured sections.
  - file: [CHANGELOG.md:3075](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/CHANGELOG.md#L3075)

## Risks / Open Questions

- The exact `memory/active_work.json` mechanism proposed in the issue is not present 1:1; current main addresses the continuity problem through structured safeguard summaries, preserved recent turns, memory flush, and post-compaction context reinjection instead.
