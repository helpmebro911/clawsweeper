## Summary

Keep open. Current main still routes the Dreaming 'Open Config' action to `config.openFile`, and the Windows opener still builds the reporter's failing `powershell.exe ... Start-Process -LiteralPath ...` command. The latest release tag `v2026.4.24` contains the same command, and current tests assert that invalid Windows shape instead of catching it.

## Best Possible Solution

Keep #69933 open and patch the Windows branch of `resolveConfigOpenCommand()` to use a valid literal-safe opener, for example `Invoke-Item -LiteralPath` or an equivalent Windows shell-execute path. Update `src/gateway/server-methods/config.test.ts` so the regression test rejects `Start-Process -LiteralPath` while still covering quoted and metacharacter-heavy config paths.
