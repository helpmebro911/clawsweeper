## Summary

Keep open. Current `main` still builds the Windows `config.openFile` command as `powershell.exe ... Start-Process -LiteralPath ...`, and local PowerShell rejects `-LiteralPath` for `Start-Process`, matching the reporter's failure. Official Microsoft docs for both Windows PowerShell 5.1 and PowerShell 7 also document `Start-Process` without a `LiteralPath` parameter. citeturn1view0turn1view1
