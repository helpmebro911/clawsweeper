## Summary

Keep open. Current main still uses a single shared GitHub Copilot API-token cache that is not bound to the selected GitHub token or auth profile, so profile rotation can still reuse a cached Copilot token from a different profile. The same author also has open, unmerged PR #43659 linked as the implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and resolve it through #43659 or an equivalent maintainer-owned patch. The implementation should bind the cached Copilot API token to the selected GitHub token/profile, preserve safe behavior for legacy cache files, and add regression tests proving that a rotated Copilot auth profile performs a fresh token exchange instead of reusing another profile's cached Copilot API token.

## Risks / Open Questions

- Leaving current main unchanged means multiple GitHub Copilot auth profiles can still share one cached Copilot API token, defeating quota rotation after one profile is rate-limited.
- The fix touches secret-adjacent token caching, so #43659 or any equivalent patch should be reviewed for token-material exposure, backwards-compatible legacy-cache handling, and focused regression coverage before landing.
