## Summary

Keep #71882 open. Current main still short-circuits memory status before asking the registered memory runtime when built-in memorySearch/default-store evidence is absent, so the reported runtime-only memory-plugin false negative is not implemented on main. The PR is a plausible core/plugin-contract fix attempt, but it should be revised before merge because the proposed patch hard-codes a third-party plugin id and only partially addresses console noise.

## Best Possible Solution

Keep the PR open and request a plugin-agnostic revision: remove the `memory-lancedb-pro` special case, let the status scan ask registered memory runtimes through an explicit generic capability/status contract when no built-in store exists, and add regression coverage for a runtime-only memory plugin. If JSON cleanliness remains in scope, handle raw `console.*` output during the probe or prove the existing JSON stderr routing is sufficient with tests.
