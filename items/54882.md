## Summary

Keep open. Current main still omits the `file` alias from the shared tool-display path resolver and read-start diagnostic, and the provided GitHub context shows #55121 is still open as the active implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review #55121, or land an equivalent focused fix. The implementation should add `file` to `resolvePathArg()`, reuse or align that alias handling in the read-start diagnostic, add regression coverage for `{ file: "..." }`, and update the shared display manifest if OpenClawKit/UI surfaces still depend on it.
