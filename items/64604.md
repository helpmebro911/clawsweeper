## Summary

Keep this PR open. Current main still logs unclassified compaction failures as the stable token `reason=unknown`, so the diagnostic problem remains. The PR is a relevant fix candidate, but it should be revised before landing because it keeps arbitrary failure text inside the classification token and also bundles unrelated README localization work.

## Best Possible Solution

Keep this PR open and request changes. Split the README localization into a separate docs PR, then land a focused compaction diagnostic fix that preserves `reason=unknown` as a stable classifier and adds a separate bounded, log-safe detail field for unknown failures, with regression tests for unrecognized errors, empty input, whitespace/control characters, and overly long details.
