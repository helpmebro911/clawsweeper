## Summary

Keep this PR open. Current main still does not expose `sessionKey` through the system-prompt runtime metadata or rendered `Runtime:` line, and the PR remains a narrow implementation candidate for the linked session-history recall problem. It needs maintainer product/security judgment before landing because it would make a raw session routing key model-visible.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, address the test constant cleanup, and explicitly choose between exposing raw `sessionKey` in Runtime metadata or improving the existing `sessionKey="current"` alias/tool guidance as the supported self-history path. Coordinate that decision with #40044 before landing either Runtime metadata field.
