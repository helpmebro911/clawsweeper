## Summary

Keep open. Current main still does not recognize the Groq error string described by this PR, so the sandbox-specific blocked-tool message is not reached for that provider format. The PR is small and on the right code path, but its added test is too weak because current main would still return the raw error containing the tool name without proving the regex was used.

## Best Possible Solution

Keep this PR open, rebase it if needed, and strengthen the regression test so it fails on current main and passes only when the Groq tool-name extraction drives the sandbox policy message. The implementation belongs in `formatAssistantErrorText` because that is the existing user-facing provider error formatter and the sandbox rewrite already hangs off the unknown-tool extraction path.
