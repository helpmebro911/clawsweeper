## Summary

Keep this issue open. Current main has a much stronger browser surface than the original report describes: the bundled browser plugin registers the browser tool, CLI, gateway method, managed profile, headless controls, custom launch args, remote CDP, and an optional sandbox browser image. But the central request is still not fully implemented because normal managed browser launch depends on an installed Chrome/Brave/Edge/Chromium executable, and Docker/browser binary installation is still opt-in or manual rather than a built-in Chromium runtime that works without external browser setup.

## Best Possible Solution

Keep this issue open and scope it to the remaining product decision: whether OpenClaw should ship a pinned Chromium runtime by default, provide an official prebuilt Docker/browser variant, or add installer/doctor flows that automatically install and persist Playwright Chromium. Reuse the existing bundled browser plugin, headless config, custom launch args, remote CDP, and sandbox browser image rather than duplicating the already-shipped browser-control surface.
