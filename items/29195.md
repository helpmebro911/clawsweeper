## Summary

Keep open. Current main has partial follow-up for this report: ACP now has `/acp status`, task-linked status output, and session-tool visibility defaults that let a requester inspect its own spawned session tree. But the core failure mode in the issue is still present: ACPX still defaults to `permissionMode=approve-reads` with `nonInteractivePermissions=fail`, and the docs explicitly warn that non-interactive ACP write/exec prompts can still abort with `AcpRuntimeError: Permission prompt unavailable in non-interactive mode`. The troubleshooting docs also still list post-completion ACP stalls as a known problem that may require manually killing stale `acpx` processes.
