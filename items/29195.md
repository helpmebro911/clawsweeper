## Summary

Keep #29195 open. Current main has added ACP task/status plumbing, timeout handling, and permission docs, but the core ACPX/Codex non-interactive reliability issue is still unresolved: the bundled ACPX default remains approve-reads/fail, same-parent cross-agent ACP child visibility is still blocked without global all visibility, and the docs still describe stale ACP process cleanup as manual.

## Best Possible Solution

Keep #29195 open as the canonical ACPX/Codex non-interactive reliability umbrella until the remaining behavior is addressed or split. The best path is a safe default or preflight for ACPX permissions, requester-facing remediation when permission/config failures occur, visibility that lets a spawning agent inspect its own cross-agent ACP child without global all visibility, and automatic timeout/reporting/cleanup for stale ACP harness processes while preserving the shipped task/status and docs improvements.
