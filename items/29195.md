## Summary

Keep #29195 open. Current main has improved ACP task tracking, status surfaces, timeout plumbing, and permission documentation, but the central ACPX/Codex reliability problem is still present: the embedded ACPX plugin still defaults to non-interactive write/exec failure, cross-agent ACP children remain blocked from default session-tool visibility, and stale ACP process cleanup is still documented as manual.

## Best Possible Solution

Keep #29195 open as the canonical ACPX/Codex non-interactive reliability umbrella until narrower issues exist. Best path: add a safe default or preflight for ACPX permissions, surface permission/config failures to the requester with actionable remediation, let a spawning agent inspect its own cross-agent ACP child without global visibility=all, and replace manual stale-process cleanup with automatic timeout/reporting semantics while preserving the shipped task/status and docs improvements.
