## Summary

Keep #70344 open. Current main covers the commentary-phase subset, but it does not fully implement the PR’s phaseless tool-use suppression or a safe sessions_send boundary fix. The PR is also not ready to close or merge as-is: its route-isolation approach conflicts with existing sessions_send A2A/announce behavior and route-preservation tests, and the PR has unresolved review findings.

## Best Possible Solution

Keep the item open for maintainer/product judgment and ask for a revised, narrower patch. Preserve the existing sessions_send A2A announce contract and persisted external route fields, suppress only the current turn's outward delivery when that is the proven bug, and add focused tests that show later announce/cron/system follow-ups still route to the original external chat. For the text leak side, extend suppression only for well-identified tool-use messages without phase metadata or coordinate with #70372 for provider thinking narration; avoid broad monologue post-processing unless it has much tighter signatures and regression coverage.
