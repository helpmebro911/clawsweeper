## Summary

Keep open. Current main has related commentary-phase suppression and route-preservation work, but this PR is not safely obsolete or implemented as-is. Its `sessions_send` changes conflict with current A2A announce behavior and route-preservation tests, and review comments flagged concrete regressions in the PR patch.

## Best Possible Solution

Do not auto-close or merge as-is. Maintainers should split the work: keep or add a narrow regression fix for phaseless tool-use/prelude leakage if still reproducible, preserve current `sessions_send` A2A announce semantics unless a product decision adds an explicit session-local/no-announce mode, and drop the broad sanitizer heuristic or replace it with stronger structured phase/tool metadata checks.
