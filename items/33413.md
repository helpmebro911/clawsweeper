## Summary

Keep #33413 open. Current main still does not update Slack `assistant.threads.setStatus` with tool-level labels during tool execution; it only sets the static reply-level status and routes tool activity to status reactions/draft preview progress. The related PR #33420 was closed unmerged, and #67129 is related customization work rather than a shipped or canonical replacement for this request.

## Best Possible Solution

Keep the issue open until Slack dispatch maps `onToolStart` events to `ctx.setSlackThreadStatus` for assistant-thread targets, with a small tool-name-to-label helper and focused Slack coverage asserting the extra `assistant.threads.setStatus` call. If maintainers choose draft-preview progress or configurable `loading_messages` as the replacement UX, they should document that product decision and link a canonical superseding issue or PR before closing #33413.
