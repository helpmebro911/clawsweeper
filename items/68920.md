## Summary

Keep this issue open. Current main still routes `/v1/chat/completions` through the full agent ingress/prompt assembly path, only supports the existing `x-openclaw-session-key`/OpenAI `user` session controls, and has no HTTP `lightContext`, `x-openclaw-light-context`, or stable `x-openclaw-session` support. The related PR #69060 is explicitly partial and remains an implementation candidate for only the session-warm/QMD subproblem.

## Best Possible Solution

Keep this issue open. Review and either land or close #69060 separately for the stable session/QMD warm-cache portion. For the remaining work, add an HTTP-specific light/voice mode in `src/gateway/openai-http.ts`/`src/gateway/http-utils.ts`, document the request contract, and thread an explicit prompt policy through `buildAgentCommandInput` and `agentCommandFromIngress` so the endpoint can intentionally reduce bootstrap context, tools, skills, and memory sections with regression tests in `src/gateway/openai-http.test.ts`. If maintainers prefer a split, keep this issue as the light-context tracker and let #69060 cover only the stable session header.
