## Summary

Current main still lacks the requested HTTP light-context/voice-mode contract, and the related stable-session PR only covers one partial subproblem. The issue should remain open as the tracker for reducing `/v1/chat/completions` prompt/context assembly latency.

## Best Possible Solution

Keep this issue open. Review and either land or close #69060 for the stable-session/QMD warm-cache subproblem, then add an explicit HTTP prompt policy in `src/gateway/openai-http.ts`/`src/gateway/http-utils.ts` and config/docs/tests so `/v1/chat/completions` can intentionally reduce bootstrap context, prompt sections, tools, skills, and memory behavior for low-latency voice/realtime callers.
