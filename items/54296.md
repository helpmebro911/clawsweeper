## Summary

Keep open. Current main has fixed the exec-approvals atomic write half, but the transcript memory-pressure half remains in core code paths: chat assistant transcript idempotency, delivery mirror transcript helpers, latest assistant lookup, and manual max-lines compaction still materialize full transcript files before scanning or trimming.

## Best Possible Solution

Keep this issue open. Treat exec-approvals persistence as already fixed, then land a focused core follow-up that introduces shared streaming or bounded transcript-scan helpers for the remaining transcript paths in src/gateway/server-methods/chat.ts, src/config/sessions/transcript.ts, and src/gateway/server-methods/sessions.ts, with regression coverage using large transcript fixtures and malformed-line cases.
