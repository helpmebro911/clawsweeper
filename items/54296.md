## Summary

Keep this issue open. Current main has the exec-approvals atomic-write fix, but the transcript memory-pressure half is still present in core code paths that read whole transcript files before scanning, deduping, or trimming.

## Best Possible Solution

Keep this issue open. Treat exec-approvals persistence as already fixed, then land a focused core follow-up that introduces shared streaming or bounded transcript-scan helpers for src/gateway/server-methods/chat.ts, src/config/sessions/transcript.ts, and src/gateway/server-methods/sessions.ts, with regression coverage using large transcript fixtures and malformed-line cases.
