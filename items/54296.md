## Summary

Keep open: current main has fixed the exec-approvals atomic-write half, but multiple core transcript paths still load whole transcript files for idempotency, latest-message, and manual compaction flows, so the central transcript memory-pressure report remains valid.

## Best Possible Solution

Keep this issue open. Treat exec-approvals persistence as already fixed, then land a focused core follow-up that replaces remaining whole-transcript idempotency, latest-message, delivery-mirror, and max-lines compaction scans with streaming or bounded helpers plus regression coverage for large transcripts.
