## Summary

Keep open. Current main still does not implement IPv6 LAN gateway binding: `gateway.bind=lan` resolves to the IPv4 wildcard `0.0.0.0`, and the supported `custom` bind path validates only canonical IPv4 addresses. The issue is a concrete core gateway feature gap rather than a ClawHub/plugin item, and the provided discussion correctly points at the missing ability to bind `::` or a specific IPv6 address.
