## Summary

Keep open. Current `main` still awaits `chat.history` without a client-side timeout or cancellation path. The recent `loadChatHistory` work on `main` only retries explicit startup `UNAVAILABLE` errors; it does not cover the PR's stalled-but-open RPC scenario, so the infinite loading report is not clearly obsolete or implemented.

## Evidence

- **`loadChatHistory` still waits indefinitely on `chat.history`:** `loadChatHistory` sets `chatLoading = true`, then awaits `state.client.request("chat.history", ...)` directly inside a retry loop. It only clears `chatLoading` in `finally` after the promise settles, and there is no timeout wrapper or abort path in this code.
