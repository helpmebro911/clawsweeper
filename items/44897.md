## Summary

Keep open. Current main and v2026.4.24 only retry explicit startup-gated `UNAVAILABLE` failures for `chat.history`; they still have no timeout or cancellation path for a `chat.history` RPC that stays pending on an open WebSocket. The PR is not safely obsolete or implemented, though it needs current-main review and tests around timeout, late responses, session switches, and live streams.

## Best Possible Solution

Keep the PR open and ask for an update against current main. The best fix is a bounded, tested `chat.history` startup-load escape path that clears `chatLoading` and surfaces a recoverable error when the request stays pending, while preventing stale or late history from overwriting a switched session, newer messages, newer errors, or an active stream. Ideally pair it with a gateway-client cancellation/removal mechanism for timed-out pending RPCs and regression tests in `ui/src/ui/controllers/chat.test.ts` plus any needed `ui/src/ui/gateway.ts` coverage.
