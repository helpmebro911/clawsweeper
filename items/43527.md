## Summary

Keep this issue open. Current main still renders the normal status agent summary as `no bootstrap files` when `bootstrapPendingCount` is zero, while that count is still derived only from `BOOTSTRAP.md` existence. The provided GitHub context also shows open PR #43614 with closing syntax for this issue, so the right maintainer path is to review/land or close that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and have a maintainer review/refresh PR #43614, or apply the equivalent narrow fix on main: change `buildStatusAgentsValue` to describe `bootstrapPendingCount` as active bootstrapping rather than file injection, and add a regression test for the mature-workspace zero-count case.
