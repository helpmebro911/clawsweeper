## Summary

Keep this issue open. Current main still has the reported misleading `no bootstrap files` wording on the normal status overview, the count behind it only checks whether `BOOTSTRAP.md` exists, and an open implementation PR (#43614) still links this issue with GitHub auto-close wording. The right maintainer path is to review, land, close, or supersede that PR rather than closing the issue now.

## Best Possible Solution

Keep this issue open and have a maintainer review, refresh, land, or supersede PR #43614. The narrow fix should update `buildStatusAgentsValue` so `bootstrapPendingCount` is described as active bootstrapping rather than bootstrap-file injection, and add a regression test for mature workspaces where `BOOTSTRAP.md` is absent while normal workspace bootstrap files are still injected.
