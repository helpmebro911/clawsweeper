## Summary

Current `main` does not already contain either change from PR #40248, and the thread-continuity change proposed in the PR conflicts with OpenClaw's current `replyToMode` contract. This is not strong evidence for closure; the item should stay open for maintainer follow-up or a revised patch.

## Risks / Open Questions

- Closing as `implemented_on_main` would be wrong: current `main` still returns only `{}` from the webhook path and ignores non-`MESSAGE` events.
- Closing as `cannot_reproduce` would be weak: the current code path still plausibly produces the chunk-thread split described in the PR when `typingIndicator` is `message` and `replyToMode` is `off`.
- Merging the PR as-is would likely regress threading semantics by forcing replies back into the inbound thread even when `channels.googlechat.replyToMode` is intentionally `off` or `first`.
