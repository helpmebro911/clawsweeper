## Summary

Current main still has the Mattermost DM threading bypass described by the report. The documented and tested contract says direct messages stay non-threaded, but the outbound Mattermost root resolver remains chat-kind unaware and can still forward a surviving payload.replyToId as root_id. The open PR #72305 is an implementation candidate, not shipped behavior; the broader configurable-DM-threading request belongs with #65729.

## Best Possible Solution

Keep this issue open and review or land PR #72305, or an equivalent Mattermost plugin patch. The fix should make outbound root-id resolution direct-chat aware so direct-message auto-replies do not send root_id when the current contract says DMs are non-threaded, while preserving channel/group threading and adding focused regression coverage for direct chats where payload.replyToId survives. Track configurable DM threading separately in #65729.
