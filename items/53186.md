## Summary

Keep open. Current main still has the reported Mattermost DM threading gap: direct messages are documented and resolved as non-threaded, but the outbound root resolver remains kind-agnostic and can still turn a reply payload id into Mattermost `root_id`. The later discussion and #65729 broaden this into configurable DM threading, but that is not shipped and does not replace the current documented flat-DM bug.

## Best Possible Solution

Keep this open and fix the Mattermost plugin. The conservative fix is to make the outbound root resolver or its call sites chat-kind aware so direct messages cannot convert fallback `payload.replyToId` into Mattermost `root_id` under the current docs. If maintainers prefer the later requested behavior, add an explicit Mattermost DM threading config surface with schema, docs, session/history handling, and regression tests rather than relying on the accidental fallback. Related #65729 can track the broader configurable DM-threading and thread-history product work, but it does not supersede this current bug yet.
