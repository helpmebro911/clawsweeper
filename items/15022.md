## Summary

Keep open. The related discussion correctly identified the desired fix in PR #16275, but that PR was closed unmerged, and current main still flushes block replies before tool execution instead of holding/coalescing text across tool calls. The current coalescer API also has no hold/resume mechanism, so the requested behavior is not implemented on main.
