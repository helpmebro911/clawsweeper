## Summary

Current main still does not populate `task_runs.agent_id` for native or ACP `sessions_spawn` task rows. The task registry and SQLite store support `agentId`, and both spawn paths resolve the target agent, but the task-run creation calls still omit it.

## Best Possible Solution

Keep this issue open and fix the core attribution plumbing. Add `agentId` to `RegisterSubagentRunParams`, pass native `targetAgentId` into `registerSubagentRun`, pass ACP `targetAgentId` into both `createRunningTaskRun` calls, and add regression coverage proving cross-agent `sessions_spawn` writes `TaskRecord.agentId` and SQLite `task_runs.agent_id`. Treat any historical backfill or doctor repair as a separate maintainer decision.
