## Summary

Current main still has the reported attribution gap. `sessions_spawn` resolves the target agent and encodes it in the child session key, and the task model/SQLite store can persist `agentId`, but the native subagent and ACP task creation paths do not pass the resolved target agent into `createRunningTaskRun`.

## Best Possible Solution

Keep this open and fix the core attribution plumbing. Add `agentId` to `RegisterSubagentRunParams`, pass `targetAgentId` from `spawnSubagentDirect`, include it in the registry manager's `createRunningTaskRun` call, and add regression coverage proving a cross-agent `sessions_spawn` writes `TaskRecord.agentId`/SQLite `agent_id`. In the same pass, pass the resolved ACP `targetAgentId` into ACP task creation and cover that path too.
