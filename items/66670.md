## Summary

Current `main` still creates `sessions_spawn` task-run records without setting `agentId`, and the task registry only persists/filters `agentId` when it is explicitly supplied. The workaround from the issue comment is real: `childSessionKey` carries the agent id, but OpenClaw core is not backfilling `task_runs.agent_id`, so the reported attribution gap remains.

## Evidence

- **Native subagent task rows omit agentId at creation:** `registerSubagentRun()` creates the `subagent` background task with `childSessionKey`, `runId`, `label`, and `task`, but does not pass `agentId` into `createRunningTaskRun(...)`.
