## Summary

Keep #66521 open. Current main still wires Signal typing through the normal reply pipeline and does not send an early typing indicator before `dispatchInboundMessage`, so the central UX request is not implemented. The PR is plausible but needs maintainer/code changes because its proposed raw pre-dispatch `sendTypingSignal` would run before the existing `sendPolicy: deny` suppression path that explicitly covers outbound indicators.

## Best Possible Solution

Keep the PR open for maintainer review, but request a policy-aware implementation. The best fix is to start Signal group typing earlier without bypassing `dispatch-from-config` delivery policy, probably by adding an early-typing hook/option that runs only after send policy and typing suppression are resolved, plus tests for group latency behavior and no typing emission under `sendPolicy: "deny"`.
