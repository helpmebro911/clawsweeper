## Summary

Current main still lacks the requested early Signal typing cue, so this PR is not obsolete. The patch is also not safe to close as implemented or merge as-is because it emits an outbound Signal typing RPC before the shared sendPolicy and typing suppression contracts are resolved.

## Best Possible Solution

Keep this PR open and rework it into a policy-aware implementation. The fix should start Signal typing earlier only after route/session sendPolicy and typingMode/suppressTyping have been resolved, or introduce an equivalent Signal-local guard that respects sendPolicy: "deny", typingMode: "never", suppressTyping, and the intended group-versus-DM scope. Add focused regression coverage for early group feedback and denied/disabled sessions emitting no typing indicator.

- The implementation scope is broader than the PR description: ctxPayload.To is present for both direct and group Signal targets.
- No regression test proves early Signal group feedback while denied or disabled sessions stay silent.
