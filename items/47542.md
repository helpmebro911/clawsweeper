## Summary

Keep open. Current main does not implement composing-aware inbound debounce for WhatsApp, and this PR is a concrete feature in the right bundled WhatsApp/plugin debounce surface. It still needs changes before merge because the provided review context identifies real blockers in the proposed branch: missing strict config-schema wiring, group key mismatch, repeated presence subscriptions, and an unbounded JID cache.

## Best Possible Solution

Keep the PR open but require revision. The best path is to implement composingDebounceMs end to end through WhatsApp config types, zod schema, generated/help/docs metadata, account resolution, and monitor options; add debouncer touch/touchByPrefix tests; wire Baileys presence.update with deduped presenceSubscribe, bounded JID/LID mapping, and direct/group debounce-key compatible matching; then add focused WhatsApp monitor tests before running the changed gate.
