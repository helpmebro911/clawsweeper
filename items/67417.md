## Summary

Keep open. Current main still routes `openclaw backup create` into a direct `tar.c(...)` call with no ENOENT-specific skip/retry path for files that disappear during archive creation, and the explicitly linked implementation PR #67472 remains open and unmerged. Broader live-backup race work in #72251 may supersede the narrow fix, but it is also still open.

## Best Possible Solution

Keep this issue open. Review and either land #67472, or close it in favor of a merged broader backup archiver fix such as #72251. The durable fix should live in `src/infra/backup-create.ts` and add a regression test proving backup creation succeeds when a planned session transcript disappears during archive creation.
