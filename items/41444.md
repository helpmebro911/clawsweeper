## Summary

Keep open. The issue is still paired with an open same-author implementation PR that uses closing syntax, and current main still shows the relevant Anthropic wrapper path is gated on extra params rather than model-level headers. This is not a stale cleanup close.

## Best Possible Solution

Review and resolve PR #41461 or replace it with an equivalent maintainer patch. The likely fix belongs in the Anthropic provider wrapper or transport header merge path: ensure OAuth-required betas are injected whenever an Anthropic OAuth token is used, even when the only configured anthropic-beta came from model/provider headers, and add a regression test for model-level anthropic-beta plus sk-ant-oat auth with no params.context1m. After the paired PR is merged or explicitly closed as superseded, this issue can be closed based on the actual landed behavior.
