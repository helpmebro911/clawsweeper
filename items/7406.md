## Summary

Keep #7406 open: current main still captures Telegram forum topic names for inbound/prompt context, but does not persist or surface them through session metadata, gateway session rows, or the Control UI session dropdown.

## Best Possible Solution

Keep #7406 open as the canonical tracker. The best fix is in the bundled Telegram/session metadata path: map cached Telegram TopicName into persisted session display metadata, keep label/displayName user overrides authoritative, ensure gateway sessions.list and the Control UI session selector show stable group/topic labels with key/id fallback, and add regression coverage for topic extraction, metadata persistence, gateway rows, and UI label resolution.
