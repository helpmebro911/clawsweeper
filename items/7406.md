## Summary

Keep #7406 open. Current main has Telegram topic-name cache/context plumbing, but it still does not wire the cached TopicName into the session display name used by the Control UI dropdown. The latest issue discussion also already reached the same conclusion, and the related PRs that attempted the full persistence/display behavior were closed unmerged.

## Best Possible Solution

Keep #7406 as the canonical tracker. The best fix is to wire cached Telegram TopicName into the generic session display path, likely by mapping it to a thread label/displayName during inbound session metadata recording, then add regression coverage that a Telegram forum topic session row and Control UI dropdown label include the human-readable topic name.
