## Summary

Keep #7406 open. Current main still only partially handles Telegram forum topic names: the Telegram plugin caches and exposes `TopicName` in inbound/prompt metadata, but the persisted session metadata, gateway `sessions.list` rows, and Control UI session selector still render from `label`/`displayName` paths that never receive the topic name.

## Best Possible Solution

Keep #7406 open as the canonical tracker. The best fix belongs in the bundled Telegram/session metadata path: map cached Telegram `TopicName` into persisted session display metadata, ensure gateway `sessions.list` and the Control UI session selector show stable human-readable group/topic labels with key/id fallback, and add regression coverage for Telegram context extraction, session metadata persistence, gateway rows, and UI label resolution.
