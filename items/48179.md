## Summary

Keep open. Current main has Telegram voice-note preflight transcription and recent audio fixes, but it does not implement the requested voice-priority lane, streaming transcription, concurrent text/voice processing, TTS caching, or streaming/parallel TTS response path. The related performance PR #39075 remains open/unmerged and is not a clear canonical replacement for this voice-specific request.

## Evidence

- **Telegram updates are still serialized by chat/topic, with no voice-specific priority lane:** The Telegram bot installs grammY sequentialization with getTelegramSequentialKey, and the key function routes normal messages by chat/topic. It has special control lanes for abort/status/approval/btw, but no branch for voice/audio messages, so a voice note in the same chat/topic does not bypass the standard lane.

I did not find a dedicated voice attachment queue or audio worker pool in the Telegram handler path.
