## Summary

Keep #7490 open. Current main still does not implement an optional per-agent `description` config field or expose equivalent specialization metadata through `agents_list`. The related implementation PR #7530 was closed unmerged, and the existing maintainer/Codex issue comment already recommends keeping this actionable core config/tool feature open.

## Best Possible Solution

Keep #7490 open until a maintainer accepts, implements, or explicitly rejects the core discovery surface. A complete implementation should add `description?: string` to the agent config type and strict runtime schema, regenerate config schema/help metadata, preserve the value through resolver/tool surfaces, include it in `agents_list`, update public sub-agent/config docs, and add focused tests for inclusion, omission, and strict rejection of unrelated unknown keys.
