## Summary

Keep #7490 open. Current main still does not add an optional per-agent `description` field to the agent config contract, runtime validation, generated config schema/help, resolver surface, or `agents_list` response. The related implementation PR #7530 was closed unmerged, and the existing maintainer/Codex comment already argues this remains an actionable core config/tool feature.

## Best Possible Solution

Keep #7490 open until a maintainer accepts, implements, or explicitly rejects this additive core discovery surface. A complete implementation should add `description?: string` to the agent config type and strict runtime schema, regenerate config schema/help metadata, preserve the value through resolver/tool surfaces, include it in `agents_list` for configured agents, update public docs such as https://docs.openclaw.ai/tools/subagents, and add focused tests for inclusion, omission, and strict rejection of unrelated unknown keys.
