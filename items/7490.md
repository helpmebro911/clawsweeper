## Summary

Keep #7490 open. Current main still does not support an optional per-agent `description` config field, and `agents_list` still cannot return agent capability descriptions. The related PR #7530 directly attempted the requested feature, but the provided GitHub context shows it was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep the issue open until OpenClaw implements the additive core config/tool behavior or a maintainer makes an explicit product decision. A complete implementation should add optional `description?: string` to `AgentConfig`, accept it in `AgentEntrySchema`, update generated config schema/help/docs as needed, include it in `agents_list` responses for configured agents, and add tests for inclusion and omission.
