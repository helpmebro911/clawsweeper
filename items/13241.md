## Summary

Keep #13241 open. Current main has useful supporting automation for stale cleanup, auto-close/candidate labels, issue intake, duplicate PR cleanup, and broad roadmap priorities, but it still does not implement the requested lightweight issue triage framework with explicit issue states, P0-P3 priority taxonomy, priority assignment rules, recurring cadence, visible current-cycle priorities, and documented stale-state transitions. The existing maintainer-authored Codex review comment also kept this issue open for the same remaining gap.

## Best Possible Solution

Keep the issue open for maintainer process judgment. A complete solution should define the smallest repo-owned issue state and priority taxonomy, clarify who can assign priority and how, document the regular triage cadence and stale-state transitions, and create or document a visible current-cycle priorities surface. Existing stale automation, intake templates, duplicate cleanup, label routing, and roadmap guidance should be reused as supporting infrastructure rather than treated as the completed framework.
