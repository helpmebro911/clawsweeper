## Summary

Keep #13241 open. Current main still has useful supporting automation for stale handling, auto-close/candidate labels, intake templates, duplicate cleanup, and broad roadmap priorities, but it does not implement the requested lightweight triage framework with explicit issue states, P0-P3 priority taxonomy, maintainer assignment rules, recurring cadence, visible current-cycle priority surface, and stale-state transitions.

## Best Possible Solution

Keep the issue open for maintainer process judgment. A complete solution should define the smallest repo-owned issue state and priority taxonomy, clarify who assigns priority and how, document the triage cadence and stale-state transitions, and create or document a visible current-cycle priorities surface. Existing stale automation, candidate labels, duplicate tooling, templates, and roadmap language should be treated as supporting infrastructure rather than the completed framework.
