## Summary

Keep #12208 open. Current main still does not implement a bounded inner scroll body for every large chat bubble; the shared SwiftUI chat UI and Android chat UI render bubble contents directly inside the outer transcript/list scroll, and the legacy `.large`/truncation symbols from the issue are not present in the current chat implementation.

## Best Possible Solution

Keep the issue open for a focused native chat UI/product decision. The likely implementation belongs in `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift` and, if parity is desired, the Android chat surface: define the current equivalent of a large bubble, wrap only the bounded body area in a vertical scroll container, preserve outer transcript scrolling, and add render/smoke coverage or documented manual mobile verification for tall-but-not-truncated content.
