## Summary

Keep #12208 open. Current main still does not provide an inner scroll body for all large or tall native chat bubbles: iOS renders bubble content in plain VStacks inside a transcript ScrollView, and Android renders bubble content in plain Columns inside a transcript LazyColumn. The existing maintainer/Codex comment on the issue also explicitly chose to keep this open, and I found no later implementation on current main.

## Best Possible Solution

Keep the issue open for a focused native chat UI fix. Define the current equivalent of a large or tall bubble, add a bounded inner vertical scroll container around the message body in apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift, add Android parity in apps/android/app/src/main/java/ai/openclaw/app/ui/chat/ChatMessageViews.kt, preserve transcript-level scrolling, and add focused render coverage or documented manual mobile verification for tall-but-not-truncated content.
