## Summary

Keep #12208 open. Current main still lacks an inner scroll body for all large/tall native chat bubbles; Swift and Android render message bodies as plain stacks/columns inside transcript-level scroll containers, and the provided maintainer/Codex discussion already reached the same conclusion on this issue.

## Best Possible Solution

Keep this open for a focused native chat UI fix. Define the current equivalent of a large or tall bubble, add a bounded inner vertical scroll container around the message body in apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMessageViews.swift, add Android parity in apps/android/app/src/main/java/ai/openclaw/app/ui/chat/ChatMessageViews.kt, preserve the existing transcript-level scrolling behavior, and add focused render coverage or documented manual mobile verification for tall-but-not-truncated content.
