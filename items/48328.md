## Summary

Keep open. Current main still has the provider/model predicate in the active memory-core FTS keyword path, so the PR's underlying bug is not implemented on main. The patch is stale because memory code now lives under extensions/memory-core, but the linked issue #48300 has concrete reproduction details and this PR still describes a plausible narrow fix plus regression coverage.

## Best Possible Solution

Keep the PR open or ask the contributor to port it to extensions/memory-core/src/memory. The likely fix is to remove providerModel only from searchKeyword/FTS keyword matching, keep providerModel filters in searchVector and listChunks, and add the cross-model FTS regression test under extensions/memory-core/src/memory/manager-search.test.ts.
