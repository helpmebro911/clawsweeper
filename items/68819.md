## Summary

Keep open. Current main still routes the bundled `claude-cli` backend through the child supervisor as bare `claude`; the child adapter only adds `.cmd` for npm/pnpm/yarn/npx and does not use the existing Windows wrapper unwrapping helper. The PR targets a real remaining Windows spawn gap, although the best final patch should likely reuse the current shared `windows-spawn` resolver and account for the review comments around PATH/PATHEXT behavior.

## Best Possible Solution

Keep this PR open or consolidate it into a canonical fix for #68788. The best implementation is to wire CLI backend child spawns through the existing `resolveWindowsSpawnProgram`/`materializeWindowsSpawnProgram` helper, using the actual child env and preserving PATH/PATHEXT semantics, then add targeted regression tests for `claude-cli` and generic CLI backend `.cmd` wrappers. After that lands, close the narrower duplicate PRs #68792 and #68849 as superseded.
