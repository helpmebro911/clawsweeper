## Summary

Current `main` does not implement provider-level `contextWindow` defaults for model providers, so there is not strong evidence to close this issue. The config schema only allows `contextWindow` on individual model entries, the merge path only preserves explicit per-model values, and current docs direct users to per-model `contextWindow` or `contextTokens` instead of a provider-level override.
