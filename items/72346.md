## Summary

Current main already implements the requested Bonjour/CIAO watchdog crash fix. The v2026.4.24 tag moved Bonjour into the bundled plugin without wiring the scoped process handlers into the plugin runtime; current main passes those handlers into the Bonjour advertiser, classifies `CIAO PROBING CANCELLED`/`CIAO ANNOUNCEMENT CANCELLED` as handled ciao cancellations, keeps the handlers registered across advertiser restarts, and disables Bonjour for the current process after repeated stuck restarts instead of crash-looping the gateway. No release tag newer than v2026.4.24 is present in this checkout, so the fix is verified on main but not marked as shipped.

## Best Possible Solution

Keep the current main implementation: scoped Bonjour/CIAO process handlers through the plugin runtime, lifecycle cleanup only on advertiser shutdown, bounded stuck-restart recovery, and documented disable options for multicast-hostile deployments. The remaining product action is to ship this in the next patch release; operators on v2026.4.24 should use `OPENCLAW_DISABLE_BONJOUR=1`, `discovery.mdns.mode="off"`, or `openclaw plugins disable bonjour` until upgraded.
