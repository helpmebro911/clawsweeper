## Summary

Keep this issue open. Current main still has the reported gap: message-channel TTS parses and strips `[[tts:...]]` directives, but the voice-call Twilio media-stream path forwards raw text to telephony synthesis, and the telephony TTS runtime/type boundary has no per-utterance directive override parameter.

## Best Possible Solution

Implement this in the voice-call owner path with a small additive telephony TTS contract change: parse and strip `[[tts:...]]` directives before Twilio media-stream synthesis using the existing `parseTtsDirectives` policy, pass provider override data or an equivalent per-utterance merged config into `textToSpeechTelephony`, forward provider overrides through telephony preparation/synthesis, and add focused tests for directive stripping plus ElevenLabs `voiceId`/`modelId` override propagation in telephony.
