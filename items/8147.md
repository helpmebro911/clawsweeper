## Summary

Keep #8147 open. Current main still hardcodes plain Enter-to-send in WebChat/Control UI, with Shift+Enter and IME-composition exemptions only. There is no Cmd/Ctrl+Enter mode, no chat send-shortcut prop or UI preference, and no `gateway.webchat` config/docs setting for send behavior. The existing issue discussion also already treated #8147 as the canonical tracker while the related #19859/#39821 items are overlapping rather than a shipped replacement.

## Best Possible Solution

Keep #8147 open as the canonical tracker. A complete fix should add a typed WebChat/Control UI send-shortcut preference or explicit product decision, persist it in UI settings if configurable, update composer placeholder/shortcut hints, document any new config or preference surface, and add focused tests for plain Enter, Shift+Enter, IME composition, and Cmd/Ctrl+Enter behavior.
