## Summary

Keep #8147 open. Current main still hardcodes plain Enter-to-send in the WebChat/Control UI composer, with only Shift+Enter and IME-composition exemptions. There is no Cmd/Ctrl+Enter mode, no send-shortcut prop or persisted UI preference, and no `gateway.webchat` config/docs setting for send behavior. The provided discussion also treats #8147 as the canonical tracker, while #19859 remains overlapping and #39821 was closed rather than shipped.

## Best Possible Solution

Keep #8147 open as the canonical tracker. A complete fix should add a typed WebChat/Control UI send-shortcut preference or explicit product decision, persist it in UI settings if configurable, update composer shortcut hints, document any new config or preference surface, and add focused coverage for plain Enter, Shift+Enter, IME composition, and Cmd/Ctrl+Enter behavior.
