## Summary

Keep #8147 open. Current main still sends WebChat/Control UI chat messages on plain Enter, with only Shift+Enter and IME-composition exceptions. There is no Cmd/Ctrl+Enter send mode in the chat render props, persisted UI settings, gateway WebChat config schema/types, or WebChat docs. The existing issue comment explicitly says to keep this open, and the provided related reviews for #38363 and #39821 also confirm the same shortcut work remains unshipped.

## Best Possible Solution

Keep #8147 open until OpenClaw implements or explicitly declines a core WebChat/Control UI send-shortcut setting. A complete fix likely belongs in the chat composer key handling plus either typed gateway WebChat config or persisted UI settings, with updated placeholder/help text, docs, and focused coverage for plain Enter, Shift+Enter, IME composition, and Cmd/Ctrl+Enter behavior.
