## Summary

Keep open. Current main does not provide high-confidence evidence that #50887 is fixed: byte-stream paths still decode chunks independently in ways that can create U+FFFD on split UTF-8 characters, and the related fix PR #51303 remains open/unmerged.

## Evidence

- **jsonl_socket_raw_decode:** `requestJsonlSocket` appends each TCP `data` chunk via `data.toString("utf8")` before JSON parsing, so a socket chunk split inside a multi-byte character can still corrupt the text before assembly.
  - file: [src/infra/jsonl-socket.ts:38](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/infra/jsonl-socket.ts#L38)
  - command: `nl -ba src/infra/jsonl-socket.ts | sed -n '1,90p'`
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **no_utf8_boundary_regression_test:** The current jsonl-socket tests cover malformed lines, half-closing, timeout, and socket errors, but do not include a split multi-byte UTF-8 regression for the reported U+FFFD behavior.
  - file: [src/infra/jsonl-socket.test.ts:28](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/infra/jsonl-socket.test.ts#L28)
  - command: `nl -ba src/infra/jsonl-socket.test.ts | sed -n '1,140p'`
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **child_adapter_raw_decode:** The process supervisor child adapter still decodes stdout and stderr chunks independently with `chunk.toString()`, matching the root-cause pattern described by the open related PR #51303.
  - file: [src/process/supervisor/adapters/child.ts:111](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/process/supervisor/adapters/child.ts#L111)
