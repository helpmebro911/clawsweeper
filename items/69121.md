## Summary

Keep #69121 open. Current main still has restart deferral before shutdown, including pending reply counts, but it does not implement the PR's second-stage bounded drain inside `server.close()` or active chat-run abort/cleanup before sockets and channels are closed. The provided PR discussion describes a focused, still-open contributor fix with targeted validation, not an obsolete or superseded PR.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a narrow gateway fix that passes the remaining restart drain budget into `server.close()`, drains pending reply dispatchers and active chat runs before teardown closes sockets/channels, aborts still-active chat runs when the bounded budget is exhausted, and keeps unrelated qa-lab changes out unless required for the regression scenario.
