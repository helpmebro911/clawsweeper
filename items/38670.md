## Summary

Keep PR #38670 open. Current main still does not implement agent-scoped internal hook policy overrides, and the adjacent shipped plugin hook and webhook agent routing controls do not supersede the proposed core internal-hook policy resolution.

## Best Possible Solution

Keep PR #38670 open for maintainer judgment. If OpenClaw wants this capability, implement it as a core config/runtime change with strict policy types and schemas for hooks.internal.events, agents.defaults.hooks, and agents.list[].hooks; pass config plus runtime agent identity into internal hook dispatch; apply agent overlays only to real agent-scoped events; document exact event matching; and add focused tests for precedence, named hook gating, non-agent hook behavior, and backward compatibility. If maintainers reject the product direction, close it explicitly with that rationale.
