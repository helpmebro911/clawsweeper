## Summary

Keep this issue open. Current main still implements `/usage full` as a per-response token/cache/cost footer plus session key only, while the requested cumulative context/window percentage, model metadata, and inherited `agents.defaults.responseUsage` config remain absent. The provided GitHub context also shows open PR #69618 as an unmerged implementation candidate for this same request, so the issue should stay open until that PR, or a replacement, is reviewed and landed or closed.

## Best Possible Solution

Keep this issue open while #69618 or a smaller replacement is reviewed. The implementation should enrich `/usage full` with model plus cumulative context/window/percentage data, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover the formatter, reply runner, `/usage` command, config schema/docs, and relevant reply-composition tests.
