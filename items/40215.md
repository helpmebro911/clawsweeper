## Summary

Keep this issue open. Current main still does not implement the requested `/usage full` cumulative context/window footer or an inherited `agents.defaults.responseUsage` default, and the provided context plus live issue page show an open implementation candidate linked to this issue rather than shipped behavior.

## Best Possible Solution

Keep this issue open while #69618, or a smaller replacement, is reviewed. The implementation should enrich `/usage full` with model plus cumulative context/window/percentage data, add documented inherited `agents.defaults.responseUsage` semantics while preserving session overrides, and cover the formatter, reply runner, `/usage` command, config schema/docs, TUI/session patch behavior, and reply-composition tests.
