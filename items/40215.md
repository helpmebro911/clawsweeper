## Summary

Keep open. Current `main` at `ffa84cdc025b3396d5044e322fbe6ff27cbeaf22` still implements `/usage` as a per-response token/cache/cost footer, with `full` adding only the session key. It does not include the active model plus cumulative context used/window/percent in the reply footer, and `agents.defaults.responseUsage` is not present in the agent defaults schema/types. The issue remains the canonical tracker: related #65487 was closed as a duplicate of this request, while #48635 and #69618 are still open/unmerged proposals in the provided GitHub context.

## Best Possible Solution

Keep #40215 open as the canonical request. The best implementation path is to enrich the shared response usage footer with active model and fresh context used/window/percent data, add an inherited `agents.defaults.responseUsage` config contract with schema/types/docs/tests, preserve session-level override semantics, and coordinate with #69618 or a narrower reviewed replacement before closing this issue.
