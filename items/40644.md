## Summary

Keep this issue open. Current main still renders the Control UI cron surface as a summary strip, filterable jobs list, run-history list, and edit form, with no cron calendar/timeline/day/week/heatmap or drag-to-reschedule UI. There is also an open, unmerged implementation candidate, #41892, whose body uses GitHub closing syntax for this issue, so the issue should remain open until that PR lands or maintainers choose a replacement scope.
