## Summary

Keep this issue open. Current main still renders the Control UI cron surface as a summary strip, filterable flat jobs list, run-history list, and edit form. The requested calendar/timeline modes, week/day grouping, heatmap, drag-to-reschedule path, and real cron-expression validation are not present on main. PR #41892 is an open unmerged implementation candidate with closing syntax for this issue, so the issue should stay open until that PR lands or maintainers choose a replacement plan.

## Best Possible Solution

Keep this issue open as the canonical Control UI cron calendar/timeline request. Review, land, shrink, or replace PR #41892 as the implementation path; close only after a merged mainline change ships a maintainer-approved cron timeline/calendar surface, or after maintainers create a narrower canonical follow-up for any remaining scope.
