## Summary

Keep open. Current `main` still uses the Unix-only `xargs` docs formatting pipeline, so the PowerShell compatibility request is not implemented on `main`, and the PR as submitted still appears Windows-fragile because it would invoke `oxfmt` without the Windows shim handling already used elsewhere in the repo.
