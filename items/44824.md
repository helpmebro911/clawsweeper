## Summary

Keep open. Current main at b979f2964c9609a4d2ce9e96d643ed020e3fc00e still routes `pnpm check:docs` through the Unix-only `git ls-files ... | xargs oxfmt` pipeline, and no equivalent `scripts/format-docs.mjs` helper has landed. The PR addresses a real unresolved Windows/PowerShell tooling problem, but the submitted branch still has a concrete review blocker around Windows npm `.cmd` shim resolution and silent spawn errors. Related open PRs exist, but the supplied context does not show a merged or clearly canonical superseding fix, so conservative cleanup should not close this PR automatically.

## Best Possible Solution

Keep the item open or ask the author to amend it. The best fix is a single canonical docs-format wrapper that removes the `xargs` dependency for `format:docs:check` and preferably `format:docs`, preserves the current tracked-file set, handles Windows npm/pnpm command shims via `pnpm exec`, `shell: true` on Windows, or an existing helper pattern, chunks arguments safely, and reports `result.error` when `git` or `oxfmt` cannot launch. After one corrected PR lands, close the remaining duplicate PRs against that merged fix and #44293.
