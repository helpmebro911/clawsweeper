## Summary

Keep open. Current main does not implement the requested first-class plugin hooks for model failover decisions or terminal all-models-failed outcomes. The runtime has structured failover logging and generic model-call telemetry hooks, but those do not expose the failover decision chain, next candidate, previous attempts, or terminal fallback-summary event to plugins. PR #70990 directly tracks this feature, but it is still open/unmerged, so the issue should remain available as the feature tracker until that or an equivalent maintainer-approved PR lands.

## Best Possible Solution

Keep #70976 open until PR #70990 or a maintainer-approved equivalent lands. The best implementation path is an additive, sanitized, fire-and-forget plugin hook pair wired through `src/plugins/hook-types.ts` and `src/plugins/hooks.ts`, emitted from the existing failover logging and terminal fallback-summary paths, documented in the public plugin hooks docs, and covered by focused hook-runner plus failover seam tests.
