## Summary

Keep this issue open. Current main still routes Feishu setup status through `probeFeishu`, and that probe still constructs the normal SDK HTTP client without any quiet/custom logger option. The provided discussion also shows same-author PR #67734 is open, unmerged, and uses closing syntax for this issue, so it should be reviewed or closed before resolving the issue.

## Best Possible Solution

Review #67734 as the implementation candidate. If correct, land a Feishu-owned change that gives setup/status probes a quiet or custom SDK logger while preserving `Feishu: connected as ...`, with focused regression coverage in Feishu client/probe/setup-surface tests. After that merges, close this issue as implemented.
