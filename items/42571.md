## Summary

Keep open. Current `main` does not include this fix, and the PR as described still leaves the Telegram target parser path incomplete. Bare `group:<numeric_chat_id>` delivery targets are still not normalized on `main`, so the cleanup is not already implemented and there is no strong basis to close the PR.

## Evidence

- **Current target normalization still rejects bare `group:` numeric chat IDs:** `normalizeTelegramChatId()` and `normalizeTelegramLookupTarget()` only call `stripTelegramInternalPrefixes()`, which strips `group:` only after a `telegram:`/`tg:` prefix. A bare `group:-100123` or `group:123456789` therefore remains unnormalized on current `main`.
  - file: [extensions/telegram/src/targets.ts:32](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/extensions/telegram/src/targets.ts#L32)
  - command: `nl -ba extensions/telegram/src/targets.ts | sed -n '1,120p'`
  - sha: [d29eaeafc1fc](https://github.com/openclaw/openclaw/commit/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132)
- **`parseTelegramTarget()` still misparses positive bare group-prefixed IDs:** `parseTelegramTarget()` uses `/^(.+):(\d+)$/` on the unnormalized string. For `group:123456789`, that splits to `chatId = "group"` and `messageThreadId = 123456789`, matching the Greptile concern. A read-only Node check returned `{"chatId":"group","messageThreadId":123456789}`.
  - file: [extensions/telegram/src/targets.ts:103](https://github.com/openclaw/openclaw/blob/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132/extensions/telegram/src/targets.ts#L103)

  - sha: [d29eaeafc1fc](https://github.com/openclaw/openclaw/commit/d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132)
- **Approval-button routing still depends on `resolveTelegramTargetChatType()` returning `group`:** `shouldInjectTelegramExecApprovalButtons()` only treats channel targets as eligible when `chatType === "group"`. Since `resolveTelegramTargetChatType()` delegates to `parseTelegramTarget()`, a bare `group:-100...` target can still fall through as non-group and suppress channel-only approval buttons.
