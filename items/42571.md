## Summary

Keep this PR open. The Telegram bare `group:<numeric>` delivery problem is still present in the checked-out current main and the latest local release tag, while the submitted PR is stale and not merge-ready because it patches obsolete `src/telegram/*` paths and does not update the shared parser entry point called by send and approval routing.

## Best Possible Solution

Keep this PR open as a stale but relevant implementation candidate until a current-layout fix lands. The maintainer path should be a narrow patch in `extensions/telegram/src/targets.ts` that normalizes bare `group:<numeric>` before topic/colon parsing while preserving explicit `telegram:group:` behavior, with regression coverage for strip/normalize/parse/chat-type behavior, send delivery, plugin target normalization/session announce routing, and exec approval button routing. After that lands, close or supersede this obsolete PR rather than merging it as-is.
