## Summary

Keep open. Current main still does not normalize bare Telegram `group:<numeric>` delivery targets in the shared Telegram target parser, and the submitted patch is stale against the current `extensions/telegram` layout and incomplete for the `parseTelegramTarget` entry point called by send and approval paths. Related #51688 was merged, but the checked-out main at `aac83e00cfe72e5395b4cdfe77ead1c534fe7ef8` still has source and tests preserving the bare `group:` prefix, so there is no implemented-on-main basis to close this PR.

## Best Possible Solution

Keep this open until a current Telegram fix lands. The best path is a narrow update in `extensions/telegram/src/targets.ts` that normalizes bare `group:<numeric>` before topic/colon parsing, preserves explicit `telegram:group:` handling, and adds regression coverage for `stripTelegramInternalPrefixes`, `normalizeTelegramChatId`, `normalizeTelegramLookupTarget`, `parseTelegramTarget`, `resolveTelegramTargetChatType`, `normalizeTelegramMessagingTarget`, and the send/approval paths.
