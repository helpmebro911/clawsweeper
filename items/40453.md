## Risks / Open Questions

- Because skill descriptions influence model selection, the new wording still needs maintainer copyedit for over-triggering or under-triggering. One example to review is messaging-platform wording that references Telegram as a respective skill even though current bundled skills include `slack` and `bluebubbles`, while Telegram appears to be handled through the message channel path.
- Longer descriptions can push the skills catalog into compact mode sooner, although current main has a tested compact fallback that omits descriptions before dropping skills.
- The broader open cleanup issue #43410 may be a better coordination point if maintainers want one sweep of bundled skill descriptions, but it is not a merged implementation and does not make this focused PR obsolete by itself.
