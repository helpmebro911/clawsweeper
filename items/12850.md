## Summary

Keep open. Current main still flattens the Mattermost inbound system-event preview before enqueueing it, while the full inbound body remains newline-preserving. I found no Mattermost config/schema/type/docs surface for the requested opt-in faithful preview mode, and the related discussion still treats #12850 as the remaining feature request.

## Best Possible Solution

Keep #12850 open. Implement this in the bundled Mattermost plugin as an opt-in account/top-level setting that preserves newlines for system-event previews while keeping today's compact one-line default. Add focused tests for default flattening and opt-in newline-preserving previews, and document the setting on docs.openclaw.ai/channels/mattermost.
