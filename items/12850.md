## Summary

Keep open. Current main still computes the Mattermost inbound system-event preview with `bodyText.replace(/\s+/g, " ").slice(0, 160)`, while the full inbound body is preserved separately. I found no Mattermost config, docs, or tests for the requested opt-in newline-preserving preview mode; the related Markdown newline work fixes ingestion/mention normalization, not this preview feature.

## Best Possible Solution

Keep the issue open. Add a Mattermost plugin setting, ideally top-level and per-account, that defaults to current compact one-line previews but can opt into a faithful preview helper that preserves `\n`, normalizes spaces/tabs within lines, and caps by max lines/chars. Document the option on the public Mattermost channel docs and add focused monitor coverage for both default flattening and opt-in newline-preserving system-event previews.
