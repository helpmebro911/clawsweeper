## Summary

Keep open. Current main still flattens Mattermost inbound system-event previews with `bodyText.replace(/\s+/g, " ")`, while the full inbound body remains newline-preserving. I found no Mattermost config/schema/type/docs/test surface for the requested opt-in faithful preview mode, and the related issue/PR context explicitly points to #12850 as the remaining feature request rather than a shipped fix.

## Best Possible Solution

Keep #12850 open. Implement the option in the bundled Mattermost plugin, likely as a top-level and per-account setting that defaults to today's compact one-line preview while allowing a faithful preview helper that preserves `\n`, normalizes spaces/tabs within each line, and caps output by max lines/chars. Add focused tests for default flattening and opt-in newline-preserving system-event previews, and document the setting on https://docs.openclaw.ai/channels/mattermost.
