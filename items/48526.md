## Summary

Keep open. Current main has partial recovery work for billing-disabled auth profiles, but it does not fully cover this issue's reported path: a billing-disabled Anthropic setup with same-provider fallback models can still skip without making a fresh provider request, and the onboarding auth path does not appear to clear stale usage lockouts the way `models auth login` does.
