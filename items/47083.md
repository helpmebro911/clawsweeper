## Summary

Current main partially addresses the Control UI freshness path, but the TUI path from PR #47083 is still not implemented. The TUI session-list type still does not carry `totalTokensFresh`, `applySessionInfo()` only updates `totalTokens` when a value is present and does not clear a previously cached footer value when the gateway omits stale totals, and the open PR still needs the fresh-null reset fix raised in review.

## Best Possible Solution

Keep the PR open, but rebase or split it against current main: add `totalTokensFresh` to the TUI session row/session info path, clear or ignore `totalTokens` when freshness is false or omitted by the gateway, preserve fresh `null` resets, and add a focused TUI session-actions regression test. Treat the current Control UI reducer/test behavior as already handled on main.
