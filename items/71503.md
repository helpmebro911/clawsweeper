## Summary

Keep PR #71503 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still uses the old LAN IPv4 selection behavior for gateway display, pairing setup codes, and the bundled device-pair plugin, so the PR is not obsolete or implemented on main.

## Best Possible Solution

Keep #71503 open for maintainer review and landing, or land an equivalent fix that updates all three affected surfaces: gateway/control UI LAN display, pairing setup-code URL resolution, and bundled device-pair LAN URL generation. The best implementation should preserve explicit bridge addresses as usable while deprioritizing automatic Docker/bridge/CNI candidates, include the PR's regression coverage, and either accept mirrored plugin logic for this narrow bug fix or create a deliberate public plugin-facing LAN-selection helper as a separate contract decision.
