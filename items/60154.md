## Summary

Keep open. Current `main` still stores Feishu inbound media in the shared global `~/.openclaw/media/inbound` tree, and the Feishu account config/docs do not expose any per-account `inboundDir` or equivalent isolation knob. The security concern in the issue is therefore still plausible and unimplemented.

## Evidence

- **Feishu account schema has no per-account inbound directory field:** `FeishuAccountConfigSchema` includes account credentials and shared channel settings, but no `inboundDir` or other account-scoped inbound media path option.
  - file: [extensions/feishu/src/config-schema.ts:177](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/feishu/src/config-schema.ts#L177)

- **Feishu docs show multi-account support but no inbound media isolation setting:** The user-facing Feishu docs document `defaultAccount` and `accounts.<id>` for multiple accounts, with no documented per-account inbound media directory setting.
  - file: [docs/channels/feishu.md:204](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/docs/channels/feishu.md#L204)

  - file: [extensions/feishu/src/bot-content.ts:370](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/feishu/src/bot-content.ts#L370)
