## Summary

Keep this PR open. Current main still rejects `timeoutSeconds: 0` in the Control UI and drops it from the UI-built payload, while the backend/protocol already accepts `0` and treats it as no timeout. The PR is a focused implementation candidate for the still-open paired issue #41272, so conservative cleanup should not close one half of the contributor issue/PR pair before review or merge.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #41272. The best path is to finish the focused UI fix in `ui/src/ui/controllers/cron.ts`, update controller tests and validation copy, confirm malformed/non-finite timeout input is still rejected, and decide whether CLI parity should be included here or tracked separately. After the corrected PR lands, the paired issue can close as implemented.
