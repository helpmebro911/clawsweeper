## Summary

Keep open. Current `main` still has the reported contract mismatch: `exec host=node` always invokes `system.run.prepare`, but the macOS node surface is still documented and exercised as advertising `system.run` without `system.run.prepare`, and gateway policy rejects undeclared node commands with the same error shape reported in the issue.

## Evidence

- **Node exec still requires prepare first:** `executeNodeHostCommand()` unconditionally sends `node.invoke` with `command: "system.run.prepare"` before any `system.run`, and aborts if no valid prepare payload comes back.
  - file: [src/agents/bash-tools.exec-host-node.ts:100](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/bash-tools.exec-host-node.ts#L100)
  - command: `nl -ba src/agents/bash-tools.exec-host-node.ts | sed -n '1,340p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Gateway rejects undeclared node commands:** `isNodeCommandAllowed()` returns `command not declared by node` when the command is allowlisted but missing from the node's declared command list, and `buildNodeCommandRejectionHint()` formats that into the exact `node command not allowed` error family described in the report.
  - file: [src/gateway/node-command-policy.ts:222](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/gateway/node-command-policy.ts#L222)

- **User-facing rejection text matches the report:** Gateway server methods map `command not declared by node` to `node command not allowed: the node (platform: ...) does not support "<command>"`. That is the same failure mode reported for `system.run.prepare`.
  - file: [src/gateway/server-methods/nodes.ts:1150](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/gateway/server-methods/nodes.ts#L1150)

  - file: [docs/nodes/index.md:321](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/nodes/index.md#L321)
