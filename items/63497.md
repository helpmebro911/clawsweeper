## Summary

Keep this PR open. Current main has shared media-reference handling and a direct image-tool test for `media://inbound`, but normal agent tool construction still passes `workspaceDir` into `createImageTool`; the image tool still resolves `media://inbound/<id>` as a workspace-relative path before `loadWebMedia` can hydrate the managed media URI. The PR remains relevant, but should be rebased/adapted to the current shared helpers rather than merged mechanically.

## Best Possible Solution

Keep this PR or a replacement fix open. Adapt the image tool to preserve or resolve `refInfo.isMediaStoreUrl` before workspace-relative path resolution, preferably by using `resolveMediaReferenceLocalPath` or by excluding media-store URLs from the `workspaceDir` branch. Add regression coverage through `createImageTool` or `createOpenClawTools` with `workspaceDir` and `fsPolicy: { workspaceOnly: true }`, covering both bare `media://inbound/<id>` and `MEDIA:media://inbound/<id>`, while retaining the shared inbound-media safety guards.
