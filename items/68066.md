## Summary

Keep this issue open. Current main still does not reconcile OpenRouter streamed usage with the authoritative generation `total_cost`, and the open closing PR #72598 is README-only, so it is not an implementation candidate that can safely close the billing bug.

## Best Possible Solution

Keep this issue open. Close or reject README-only PR #72598 separately, then implement the fix in the bundled OpenRouter provider/transport path: after a verified OpenRouter stream completes and `responseId` is known, fetch `/generation` with the resolved OpenRouter credential and persist the authoritative `total_cost` as either `usage.cost.total` or a versioned billed-cost field. Add regression coverage with mocked tier-priced OpenRouter streamed usage plus generation metadata, including lookup-failure fallback.
