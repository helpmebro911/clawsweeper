## Summary

Keep open. Current main does not implement the channel-prefixed Slack allowlist matching that the latest issue discussion identifies as the verified failure path; the runtime still only tries bare channel id case variants and optional name variants.

## Evidence

- **Slack channel resolver omits channel:<id> candidates:** resolveSlackChannelConfig builds candidates from channelId, lowercase channelId, uppercase channelId, and optional channel-name forms. It does not add channel:${channelId} or channel:${lowercaseId}, so a config key like channel:C0AJYR3BVTJ will not directly match a runtime channel id C0AJYR3BVTJ.
  - file: [extensions/slack/src/monitor/channel-config.ts:114](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/extensions/slack/src/monitor/channel-config.ts#L114)
  - command: `nl -ba extensions/slack/src/monitor/channel-config.ts | sed -n '88,155p'`
  - sha: [81a41fe5bee3](https://github.com/openclaw/openclaw/commit/81a41fe5bee350c6a035bfcc458eb8be380afe2c)
- **Shared candidate helper does not add prefixes:** buildChannelKeyCandidates only trims, deduplicates, and returns the keys passed by the caller. There is no hidden normalization that would make channel-prefixed Slack allowlist keys match bare runtime ids.
  - file: [src/channels/channel-config.ts:43](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/channels/channel-config.ts#L43)
  - command: `nl -ba src/channels/channel-config.ts | sed -n '1,120p'`
  - sha: [81a41fe5bee3](https://github.com/openclaw/openclaw/commit/81a41fe5bee350c6a035bfcc458eb8be380afe2c)
- **Runtime drops unmatched allowlisted channels:** Slack room gating calls resolveSlackChannelConfig and then isSlackChannelAllowedByPolicy. With groupPolicy=allowlist and a configured allowlist that does not match, the policy path returns false and the context rejects the inbound channel event before routing.
  - file: [extensions/slack/src/monitor/context.ts:333](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/extensions/slack/src/monitor/context.ts#L333)
  - command: `nl -ba extensions/slack/src/monitor/context.ts | sed -n '300,380p'`

- **Allowlist policy fails closed when route is not matched:** evaluateGroupRouteAccessForPolicy returns allowed:false with reason route_not_allowlisted for allowlist policy when routeMatched is false, which is the observable failure mode described in the issue follow-up.
  - file: [src/plugin-sdk/group-access.ts:80](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/plugin-sdk/group-access.ts#L80)
