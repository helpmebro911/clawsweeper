## Summary

Keep this issue open. Current main still routes Mattermost message-tool sends through the Mattermost plugin action handler, and that handler still ignores the requested `threadId`, `filePath`/`path`, and `mediaLocalRoots`/`mediaReadFile` inputs. The linked PR #67796 is open, unmerged, and only patches the media fallback portion, so it is an implementation candidate rather than grounds to close this issue.

## Best Possible Solution

Keep this issue open. Review and expand #67796, or land a replacement that updates `extensions/mattermost/src/channel.ts` so `handleAction` reads `threadId`, falls back across `media`/`mediaUrl`/`filePath`/`path`/`fileUrl`, forwards `mediaLocalRoots` and `mediaReadFile` to `sendMessageMattermost`, and adds focused Mattermost action tests for local file upload into a thread.
