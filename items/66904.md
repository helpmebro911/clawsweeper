## Summary

Keep open. Current main still does not subscribe to Baileys `messaging-history.set`, does not expose reconnect-history sync config, and its existing WhatsApp append-upsert path intentionally skips stale offline/history messages instead of adding them to agent-visible history.

## Best Possible Solution

Implement this in the WhatsApp plugin runtime: add a typed `messaging-history.set` handler, expose explicit opt-in reconnect-history configuration, normalize and dedupe synced messages through the existing inbound/history surfaces, and populate DM/group history or session context without auto-replying to stale backlog unless separately opted in. Update WhatsApp config types, strict zod schema, generated config metadata/docs, and regression tests for sync-type filtering, stale backlog handling, dedupe, and `historyLimit`/`dmHistoryLimit` interactions.
