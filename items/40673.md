## Summary

Current main has the low-level truthy-value guard needed for `OPENCLAW_RAW_STREAM=0`, but the maintainer-facing issue is still actionable. The docs still only describe enabling raw-stream logging, and the LaunchAgent/service-env path still has durable `.env` and config env sources that can re-enable logging after `launchctl unsetenv`. There is parser coverage for `0` being false, but no focused regression coverage for raw-stream append disable behavior or LaunchAgent restart/service-env precedence.

## Best Possible Solution

Keep this issue open. Resolve it with a narrow docs-and-regression change: document that only `1`, `true`, `yes`, and `on` enable raw-stream logging; document that unset, `0`, `false`, `off`, and other values disable it; explain LaunchAgent precedence with `launchctl setenv/unsetenv` versus durable `~/.openclaw/.env`, `$OPENCLAW_STATE_DIR/.env`, and config `env` entries; add focused tests for disabled `appendRawStream` values and daemon install/service-env planning. If a live macOS LaunchAgent still appends with the effective process env set to `0`, fix that restart/runtime path in the same issue.
