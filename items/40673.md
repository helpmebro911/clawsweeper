## Summary

Keep this open. Current main has a low-level raw-stream truthy guard that should disable writes when the effective process env is unset or set to `0`, but the user-facing LaunchAgent disable semantics are still not documented and there is no focused raw-stream/managed-service regression coverage proving post-disable appends stop. The issue remains actionable as a narrow docs-and-regression item, with runtime work if live macOS still reproduces under an effective env of `0` or absent.

## Best Possible Solution

Keep this issue open and resolve it with a narrow docs-and-regression change: document the exact enabling values for `OPENCLAW_RAW_STREAM`, document that unset, `0`, `false`, `off`, and other values disable logging, explain LaunchAgent/service precedence across `launchctl setenv/unsetenv`, state-dir `.env`, config `env`, and CLI flags, and add focused tests for disabled `appendRawStream` values plus managed-service/LaunchAgent env planning. If live macOS still appends when the effective gateway env is `0` or absent and no durable source enables raw stream, fix that runtime path in the same area.
