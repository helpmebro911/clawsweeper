## Summary

Keep open. Current main still lacks the requested public synchronous pre-LLM tool-result sanitization hook. OpenClaw has a real post-execution/pre-model tool-result middleware path, but it is restricted to bundled plugins; external plugin authors still only get pre-call params/blocking, post-call observation, and transcript persistence mutation. Related #34144 is closed, PR #14544 is closed/unmerged, and #62123 is related but not a canonical replacement for this narrower pre-LLM hook request.

## Best Possible Solution

Keep this issue open until OpenClaw either lands a documented, trust-gated public post-execution/pre-model tool-result mutation hook, or maintainers explicitly decide that this capability must remain bundled-only. A complete fix should define ordering relative to `before_tool_call`, bundled `registerAgentToolResultMiddleware`, `after_tool_call`, and `tool_result_persist`; specify sync/async and failure semantics; update public plugin docs; and add runtime tests proving the model receives only the sanitized result.
