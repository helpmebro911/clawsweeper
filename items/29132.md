## Summary

Keep open. Current main has a real post-execution/pre-model tool-result middleware path, but it remains a trusted bundled-plugin seam and is not exposed as the requested public synchronous `tool_result_before_model` hook for external plugin authors. `tool_result_persist` is still documented and wired as a transcript persistence transform, and the current public hook surface still lacks an equivalent result mutation hook before the model consumes the same-turn tool output.

## Best Possible Solution

Keep this issue open until OpenClaw either lands a documented, trust-gated public post-execution/pre-model tool-result mutation hook, or maintainers explicitly decide that this capability must remain bundled-only. A complete fix should define ordering relative to `before_tool_call`, bundled `registerAgentToolResultMiddleware`, `after_tool_call`, `tool_result_persist`, and `before_message_write`; specify sync/async and failure semantics; update public plugin docs; and add runtime tests proving the model receives only the sanitized result.
