## Summary

Keep open. Current `main` does not expose the requested `tool_result_before_model` hook or an equivalent mutable pre-LLM tool-result hook. The only tool-result mutation hook on `main` is `tool_result_persist`, and it is explicitly wired to transcript persistence, while live follow-up model requests use separate `context.messages` sanitization paths.

## Evidence

- **Requested hook name is absent:** A repo search for `tool_result_before_model` returned no matches, and the registered hook-name union only includes `tool_result_persist`/`before_message_write` around the tool-result persistence area.
  - file: [src/plugins/hook-types.ts:56](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/plugins/hook-types.ts#L56)

  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Existing tool-result hook is persistence-scoped:** `PluginHookToolResultPersistEvent`/`Result` only operate on a persisted `message`, and the session guard comments and wiring say the transform runs on toolResult messages before they are persisted to the session transcript.
  - file: [src/agents/session-tool-result-guard.ts:81](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/session-tool-result-guard.ts#L81)

  - file: [docs/concepts/agent-loop.md:101](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/concepts/agent-loop.md#L101)

## Risks / Open Questions

- Closing as implemented would be premature because the requested deterministic pre-LLM mutation boundary is still not part of the public plugin hook contract.
- `tool_result_persist` may help transcript hygiene, but the current codebase still distinguishes transcript persistence from live outbound `context.messages` handling during tool continuations.
