## Summary

Keep open. Current `main` has a trusted runtime-neutral tool-result middleware path that can rewrite OpenClaw-owned dynamic tool output before Pi/Codex feed it back to the model, but that seam is still bundled-only. The public plugin hook registry still lacks `tool_result_before_model` or `before_context_send`, and external plugins are explicitly rejected from the existing pre-model middleware path. The Agent14 comment usefully confirms this is a real plugin-author workflow gap, and the related #34144/#14544 context does not supersede this item because #34144 is closed and PR #14544 was closed unmerged.

## Best Possible Solution

Keep the issue open until OpenClaw either adds a documented, trust-gated public API for post-execution/pre-model tool-result rewriting, or maintainers explicitly decide this capability must remain bundled-only. A complete solution should define ordering relative to `before_tool_call`, `after_tool_call`, and `tool_result_persist`, specify fail-open/fail-closed behavior, and add runtime coverage proving the model only receives sanitized output.
