## Summary

Keep #32496 open. Current main still preserves arbitrary model params in config, but the core OpenAI/OpenAI-compatible transport path does not first-class forward `frequency_penalty`, `presence_penalty`, or `repetition_penalty`. The linked PR attempts are closed unmerged in the provided context, and #71350 is adjacent per-agent parameter work rather than a canonical replacement.

## Best Possible Solution

Keep #32496 open as the canonical sampling-penalty passthrough request. The best fix is targeted transport work: collect supported numeric snake_case and camelCase sampling params from resolved global, per-model, per-agent, and override params; inject them only into compatible OpenAI/OpenAI-compatible Responses, Chat Completions, Azure, and WebSocket request paths where supported; preserve existing payload hooks and plugin behavior; then add focused transport tests and user-facing docs for supported fields and scope.
