## Summary

Keep open. Current `main` still does not forward `frequency_penalty` or `presence_penalty` from model `params` into the OpenAI-compatible request paths inspected, so the feature request is not implemented on `main` as of `c6684af682bcb59bd1ee85cb49f72d4074007e2c`.

## Evidence

- **Embedded runner extra params wrapper only maps temperature/maxTokens:** `createStreamFnWithExtraParams` copies `temperature` and `maxTokens` into stream options, plus transport/cache-related fields, but there is no handling for `frequency_penalty`, `presence_penalty`, `top_p`, `top_k`, or `repetition_penalty`.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:233](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/pi-embedded-runner/extra-params.ts#L233)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **OpenAI completions payload builder only serializes maxTokens/temperature:** `buildOpenAICompletionsParams` adds `max_tokens` or `max_completion_tokens`, then `temperature`; it does not serialize `frequency_penalty` or `presence_penalty`.
  - file: [src/agents/openai-transport-stream.ts:1697](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/openai-transport-stream.ts#L1697)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **WebSocket request options do not include penalty params:** The WebSocket request path defines `WsOptions` with `temperature`, `maxTokens`, `topP`, `toolChoice`, and reasoning/text-verbosity fields. The builder then forwards `temperature`, `max_output_tokens`, `top_p`, and `tool_choice` only.
  - file: [src/agents/openai-ws-request.ts:17](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/openai-ws-request.ts#L17)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Current tests cover forwarded params but not penalties:** The WebSocket tests assert forwarding for `temperature`/`maxTokens` and separately for `topP`/`toolChoice`; there is no comparable coverage for `frequency_penalty` or `presence_penalty`, matching the missing implementation in code.
  - file: [src/agents/openai-ws-stream.test.ts:3043](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/openai-ws-stream.test.ts#L3043)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **No matching implementation found in repo search:** A repo-wide search for `frequency_penalty` / `presence_penalty` in `src`, `extensions`, `docs`, `test`, and `CHANGELOG.md` returned no current implementation hits for these params in the OpenAI-compatible transport paths.
  - command: `rg -n "frequency_penalty|presence_penalty" CHANGELOG.md docs src test -g '!**/dist/**'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)

## Risks / Open Questions

- A provider plugin could theoretically inject these fields through a custom `wrapStreamFn`, but the built-in core OpenAI-compatible paths inspected on current `main` do not implement them.
- `gh` inspection was unavailable in this environment because `GH_TOKEN` was not set, so this review relies on the checked-out `main` tree and local git history rather than live PR metadata.
