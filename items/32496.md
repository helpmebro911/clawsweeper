## Summary

Keep #32496 open. Current main still does not first-class forward `frequency_penalty`, `presence_penalty`, or `repetition_penalty` from model params. The new `extra_body` path is a useful workaround for `openai-completions` proxy payloads, but it does not satisfy the requested direct params behavior across OpenAI/OpenAI-compatible Responses, Chat Completions, and WebSocket paths. The linked PR attempts are closed unmerged, and #71350 is adjacent per-agent param work rather than a canonical replacement.

## Best Possible Solution

Keep #32496 open as the canonical sampling-penalty passthrough request. The best fix is targeted transport work: resolve numeric snake_case and camelCase sampling params from global, per-model, per-agent, and override params; inject only into compatible OpenAI/OpenAI-compatible Responses, Chat Completions, Azure, and WebSocket payload paths where supported; preserve existing payload hooks and proxy escape hatches; then add focused transport tests and user-facing docs for supported fields and scope.
