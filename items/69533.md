## Summary

Keep #69533 open. Current main still hardcodes the memory-core Dream Diary system prompt and prompt section labels, the memory-core config schema/resolver has no `dreaming.prompt` or template field, and the Dreams UI only patches `dreaming.enabled`. The provided related items are broader prompt issues or provider/plugin-boundary decisions, not a canonical duplicate for customizable Dreaming output.

## Best Possible Solution

Keep the issue open and implement it in the memory-core plugin surface: add a typed `plugins.entries.memory-core.config.dreaming.prompt` or narrower narrative-template field, validate supported placeholders, thread it through `resolveMemoryDreamingConfig` into `generateAndAppendDreamNarrative`, expose an editor in the Dreams settings UI, and update Dreaming docs plus focused memory-core/UI tests. Avoid a core-wide prompt override unless maintainers explicitly want Dreaming prompt control outside the memory plugin contract.
