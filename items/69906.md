## Summary

Keep PR #69906 open. Current main still has the WhatsApp multi-account `groups` inheritance behavior the PR is trying to change, while Telegram already has the matching `isMultiAccount` guard and regression tests. The PR also targets open maintainer-labeled issue #69874, and the discussion shows the Aisle security concern was acknowledged and narrowed to a migration/product-policy choice rather than proving the PR obsolete.

## Best Possible Solution

Keep the PR open for maintainer review or landing. The best path is to merge a focused WhatsApp guard that mirrors Telegram, includes the PR's regression tests and migration docs, and leaves any broader fail-closed group-policy hardening to a separate maintainer-owned Telegram+WhatsApp decision.

## Risks / Open Questions

- This is a breaking behavior change for multi-account WhatsApp users who relied on root `channels.whatsapp.groups`; maintainers should explicitly accept the documented migration path.
- The Aisle review identified a narrow `groupPolicy="open"` plus implicit groups-as-allowlist migration risk. The PR discussion argues doctor warnings and docs are sufficient, but a fail-closed policy would be a separate cross-channel product decision.
