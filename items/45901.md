## Summary

Keep open. Current main still creates managed session directories through plain recursive mkdir in the store, transcript, lock, setup/onboarding, probe, and agent-create paths. The PR is a focused security hardening attempt with substantive review discussion, not something already implemented or safely obsolete.

## Best Possible Solution

Keep the PR open or replace it with a smaller maintainer-owned hardening patch. The best implementation is a shared session-directory creation helper in src/config/sessions that applies 0700/private permissions for managed default and templated per-agent session roots, validates symlink/ancestor behavior before chmod or lock-time mkdir side effects, preserves plain mkdir for arbitrary custom session.store/transcript directories, and adds focused tests for setup, onboarding, agents.create, store writes, transcript writes, and lock acquisition.
