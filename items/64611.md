## Summary

Keep this PR open. Current main still treats a future heartbeat task last-run timestamp as not due, so the proposed guard and focused regression tests remain relevant. The provided PR diff is narrow and does not show a security-sensitive surface change.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this narrow behavior change, or an equivalent patch, by treating future heartbeat task `lastRunMs` values as due in `src/auto-reply/heartbeat.ts`, preserving focused `isTaskDue` regression coverage, then running the targeted heartbeat tests and `pnpm check:changed` before merge.
