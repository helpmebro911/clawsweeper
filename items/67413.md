## Summary

Keep open. Current main still only supports global memory-core dreaming configuration, automatic dreaming still fans out across all configured agent workspaces, and the proposed `agents.list[].dreaming` contract is not accepted by the strict config schema or exposed in the agent config type. There is also an open same-author implementation PR, #67420, with closing syntax for this issue, so the maintainer path is to review/land or close that PR rather than close this issue first.

## Best Possible Solution

Keep this issue open while #67420 is reviewed. The likely implementation path is to add a documented `agents.list[].dreaming` contract, at minimum `enabled`, wire it through the strict config schema/types and `resolveMemoryDreamingWorkspaces`, and add resolver/runtime/docs tests. If per-agent custom schedules remain in scope, decide explicitly whether memory-core should manage multiple scoped cron jobs or whether schedule flexibility should be tracked separately.
