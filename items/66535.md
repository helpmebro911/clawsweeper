## Summary

Current main still matches the reported cancellation gap. Manual `/compact` aborts any prior active run, then starts `compactEmbeddedPiSession` without passing an `abortSignal` and without registering the compaction in the active-run registry. The lower-level compaction code already supports abort signals, and the provided GitHub context shows open PR #66894 is still an unmerged implementation candidate with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, land, or replace PR #66894. The fix should wire manual `/compact` cancellation in `src/auto-reply/reply/commands-compact.ts` by creating an `AbortController`, registering a compacting active-run handle only after any previous run drains, passing `abortSignal` to `compactEmbeddedPiSession`, clearing the handle in `finally`, and adding regression coverage proving `/stop` or `abortEmbeddedPiRun(sessionId)` cancels an in-progress compaction. The gateway `sessions.compact` path should be addressed explicitly, either in the same patch or a linked follow-up.
