## Summary

Keep #66535 open. Current main still has the reported `/compact` cancellation gap: the command handler calls `compactEmbeddedPiSession` without passing an `abortSignal` and never registers the manual compaction in `ACTIVE_EMBEDDED_RUNS`, while the underlying compaction code and active-run registry already support the needed cancellation primitives. The related PR #66894 appears to be the right shape of follow-up from the provided context, but it is not merged into current main, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #66535 open as the canonical bug until PR #66894 or an equivalent fix lands. The fix should update `src/auto-reply/reply/commands-compact.ts` to create an AbortController, pass `abortSignal` to `compactEmbeddedPiSession`, register a compacting `EmbeddedPiQueueHandle` with `setActiveEmbeddedRun`, clear it in `finally`, and add regression coverage proving `/stop` or `abortEmbeddedPiRun(sessionId)` cancels an in-progress manual `/compact`.
