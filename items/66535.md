## Summary

Current main still does not wire cancellation into manual `/compact`. The lower-level compaction API supports `abortSignal`, and abort paths depend on active-run or chat-abort registration, but the `/compact` handler still calls `compactEmbeddedPiSession` without an abort signal or active-run handle. The provided context also shows #66894 is an open, unmerged closing PR, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Review and either land or replace #66894. The fix should wire cancellation in `src/auto-reply/reply/commands-compact.ts` by creating an `AbortController`, registering a compacting `EmbeddedPiQueueHandle` only after any previous run drains, passing `abortSignal` to `compactEmbeddedPiSession`, clearing the handle in `finally`, and adding regression coverage proving `/stop` or `abortEmbeddedPiRun(sessionId)` cancels an in-progress manual `/compact`. Consider the parallel `sessions.compact` gateway path explicitly.
