## Summary

Current main does not implement the requested behavior. A successful direct cron delivery can set delivered=true, but a fatal follow-up/tool payload still makes the isolated run status "error", and applyJobResult persists that error status to lastRunStatus/lastStatus and consecutiveErrors without treating delivered=true as effective success. The directly linked PR #50189 tracks the exact fix, but it is open and not present on main.

## Evidence

- **Current main persists raw error status despite delivered metadata:** applyJobResult writes result.status directly to job.state.lastRunStatus and lastStatus, records delivered state separately, then increments consecutiveErrors whenever result.status is "error". There is no delivered=true override in this path.
