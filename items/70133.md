## Summary

Keep open. Current `main` still serializes Feishu inbound work per chat, still awaits full reply dispatch before releasing that lane, and still has no timeout/release guard on the per-chat queue. The issue's reported behavior remains plausible on current code, and I did not find a shipped fix on `main` after `v2026.4.23`.

## Evidence

- **Per-chat lane still enforced in the Feishu inbound handler:** `createFeishuMessageReceiveHandler` builds a sequential key for each inbound message and awaits `enqueue(sequentialKey, task)`, so later messages for the same key cannot start until the earlier task settles.
