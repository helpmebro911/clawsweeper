## Summary

Keep this PR open. Current main still implements and tests the old default cron heartbeat wrapper that asks the model to relay the reminder to the user, while this PR is a focused implementation candidate for changing that active cron-event path to a neutral execution wrapper. The predecessor #64516 is closed and unmerged, so it does not supersede this PR.

## Best Possible Solution

Keep this PR open for maintainer review as the current focused implementation candidate. The likely path is to update the default deliver-to-user branch of src/infra/heartbeat-events-filter.ts to a neutral execution wrapper, preserve the internal-only delivery branch, keep HEARTBEAT_OK normalization safe for responsePrefix and short replies, and land with the focused heartbeat prompt and ghost-reminder regression tests once CI is acceptable.
