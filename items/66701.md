## Summary

Keep open. The checked-out `main` still treats `NO_REPLY` as silent only for the exact token or exact JSON envelope, and mixed text with a trailing `NO_REPLY` is intentionally normalized into visible output rather than suppressed. That matches the reported leak path instead of fixing it.

## Evidence

- **Silent detection is exact-match only:** `isSilentReplyText` matches only `^\s*NO_REPLY\s*$`, so prepended reasoning text does not qualify as a silent reply.
  - file: [src/auto-reply/tokens.ts:32](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/auto-reply/tokens.ts#L32)
  - command: `nl -ba src/auto-reply/tokens.ts | sed -n '1,220p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Directive parsing uses the exact-only helper:** `parseReplyDirectives` calls `isSilentReplyPayloadText(...)` and only blanks the text when that helper returns true, so a payload like `think ... NO_REPLY` stays non-silent through directive parsing.
  - file: [src/auto-reply/reply/reply-directives.ts:34](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/auto-reply/reply/reply-directives.ts#L34)
  - command: `nl -ba src/auto-reply/reply/reply-directives.ts | sed -n '1,260p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Mixed-content NO_REPLY is preserved as visible text:** `normalizeReplyPayload` strips a trailing `NO_REPLY` from mixed-content messages and keeps the remaining text if any content remains, which would expose prepended reasoning instead of suppressing the whole payload.
  - file: [src/auto-reply/reply/normalize-reply.ts:64](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/auto-reply/reply/normalize-reply.ts#L64)
  - command: `nl -ba src/auto-reply/reply/normalize-reply.ts | sed -n '1,180p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Tests lock in the mixed-content behavior:** Current tests assert that `"File's there. Not urgent.\n\nNO_REPLY"` remains a visible reply with the token removed, and outbound payload tests assert that only exact `NO_REPLY` payloads are suppressed while mixed text stays visible.
  - file: [src/auto-reply/reply/reply-utils.test.ts:122](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/auto-reply/reply/reply-utils.test.ts#L122)
