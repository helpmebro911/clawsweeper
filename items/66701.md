## Summary

Kept open because this is a real, still-actionable auto-reply bug with open implementation candidates explicitly linked by closing syntax; current main still does not suppress reasoning-prefaced or arbitrary trailing-token NO_REPLY payloads in the final reply normalization path.

## Best Possible Solution

Keep this issue open and review the active implementation PRs #66755 and #69394 rather than closing the issue now. Maintainers should pick the intended silent-token policy, add regression coverage for bare `think`, `<think>...</think>`, and trailing-token preamble cases in `src/auto-reply/tokens.ts` and/or `src/auto-reply/reply/normalize-reply.ts`, then close this issue after the chosen fix lands.
