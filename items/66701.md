## Summary

Keep this issue open. Current main still does not implement the requested reasoning-prefaced or glued trailing `NO_REPLY` suppression policy, and there are open unmerged PRs with closing syntax against this issue. The issue should remain the canonical tracker until maintainers choose and land the intended behavior.

## Best Possible Solution

Keep this issue open as the canonical tracker. Review and choose among the active implementation candidates, then land a fix with regression coverage for bare `think`, `<think>...</think>`, whitespace-delimited trailing tokens, glued final-token cases like `here.NO_REPLY`, and the relevant streaming/block delivery path.

## Risks / Open Questions

- Maintainers still need to choose the product policy: narrow suppression for explicit reasoning markers, or broader suppression for any trailing silent token.
- A final-payload normalizer-only fix may not cover streaming or block delivery if visible chunks are sent before the final silent token arrives.
- Broad trailing-token suppression would intentionally change behavior currently covered by the older mixed-content `NO_REPLY` regression tests.
