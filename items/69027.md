## Summary

Keep open. Current main still intentionally filters and suppresses `qwen3.6-plus` for Qwen Coding Plan endpoints, and docs/tests still encode that behavior. The linked PR #66367 is relevant and reportedly worked for one commenter, but it is represented as open/unmerged in the provided context, so the issue is not implemented on main or safely superseded.

## Best Possible Solution

Keep this open until a focused Qwen provider change lands on main. The likely fix belongs in `extensions/qwen`: remove or relax the Coding Plan-specific `qwen3.6-plus` catalog filter, config normalization, and explicit suppression; then update the Qwen provider tests and `docs/providers/qwen.md` to match the chosen product behavior. PR #66367 is relevant prior work but should not close this issue until merged or replaced by a canonical maintainer-tracked item.
