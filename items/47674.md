## Summary

Keep open. Current main still matches the reported failure path closely enough that this is not a safe cleanup close: Telegram polling still calls deleteWebhook before starting the runner, non-recoverable setup errors still escape that startup path, and the probe path still treats getMe 401 Unauthorized as a real failed auth probe. Later changes document/retry/bound Telegram startup operations, but I did not find a current-main fix for a valid-token deleteWebhook/getMe 401 startup failure.
