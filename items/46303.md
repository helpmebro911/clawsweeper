## Summary

Keep open. Current main still does not implement this PR's central restart-time drain behavior: inbound debounce buffers remain closure-local with only per-key flushing, followup queues have no all-queue drain waiter, and the SIGUSR1 restart path only gates/drains active command and embedded work before closing the server. The related shipped restart deferral/orphan-recovery work is narrower and does not supersede this PR.

## Best Possible Solution

Keep this PR open as an implementation candidate, but ask for a smaller maintainer-reviewed revision: add a controlled lifecycle-safe drain hook or registry for active inbound debouncers, keep the public plugin SDK backwards-compatible, flush debounce buffers at the right restart phase without losing late ingress, wait for active turns and then all followup queues including dropped-summary state, and cover the restart ordering and channel teardown races with focused tests.
