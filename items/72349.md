## Summary

Current main already implements the requested fix. Bedrock/AWS SDK auth, including `AWS_PROFILE`, resolves as `aws-sdk` auth without requiring a literal API key, and the embedded runner now seeds PI auth storage for aws-sdk/no-key runs so subagent `sessions_spawn` calls pass the upstream preflight while the AWS SDK credential chain signs the real Bedrock request. The fix appears to be on main after the latest release tag, so no fixed release is identifiable yet.

## Best Possible Solution

Keep the current main implementation and ship it in the next release; if the reporter can still reproduce on a build containing `b7533f5112aed3dd642b973964fc4b02d15e3239`, open a focused follow-up with fresh subagent logs from that build.
