## Summary

Keep open. Current main still lacks Feishu inbound sibling mention arbitration, and the PR remains a focused fix attempt for a real multi-account group-routing gap. It should not be closed as implemented, duplicate, stale cleanup, or ClawHub scope; it needs a corrected Feishu plugin implementation before merge.

## Best Possible Solution

Keep this PR open, or replace it with a corrected Feishu plugin implementation. The maintainer path should extract structured mentioned bot open_ids before per-account dispatch, derive the coordinator from Feishu default-account selection rather than a literal `main`, arbitrate specialist-only, coordinator-plus-specialist, and multi-specialist mentions once per event, and add regression coverage in the Feishu receive-handler/parser tests. Avoid substring-only implicit mention matching unless it is tokenized and covered.
