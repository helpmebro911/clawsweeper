## Summary

Current main still does not expose Slack Socket Mode ping/pong tuning. The requested SDK options exist upstream, users have current production-impact comments, and PR #41473 is still open and unmerged rather than shipped on main.

## Best Possible Solution

Keep #14248 open until PR #41473 or an equivalent patch lands. The fix should add a supported channels.slack.socketMode config object for clientPingTimeout, serverPingTimeout, and pingPongLoggingEnabled; validate it in Slack types/schema/metadata; preserve account inheritance; pass it into SocketModeReceiver or the underlying Slack socket client before start(); add focused config/runtime tests; and update the public Slack/config docs.
