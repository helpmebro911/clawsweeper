## Summary

Keep #14248 open. Current main at cf834e2a21eac7fd218989fb5b85e7e8dd4c3246 still has no supported Slack Socket Mode ping/pong tuning surface, the Slack config schema is strict and lacks the requested keys, and the runtime SocketModeReceiver construction does not pass clientPingTimeout, serverPingTimeout, or pingPongLoggingEnabled. The provided related context also identifies PR #41473 as the unmerged implementation path rather than evidence that the issue is fixed.

## Best Possible Solution

Keep the issue open until the Slack plugin adds a supported channels.slack.socketMode config surface or equivalent, validates it in Slack config types and strict zod schema, exposes UI/help/docs metadata, and passes clientPingTimeout, serverPingTimeout, and pingPongLoggingEnabled into SocketModeReceiver construction with focused tests. PR #41473 is the related starting point, but #14248 should close only after that PR or an equivalent current-main patch lands.
