## Summary

Keep #14248 open. Current main still lacks any supported Slack Socket Mode ping/pong timeout config, the locked Slack SDK version supports the requested options, users have current production-impact comments, and PR #41473 remains open/unmerged rather than shipped.

## Best Possible Solution

Keep #14248 open until PR #41473 or an equivalent patch lands. The implementation should add a supported `channels.slack.socketMode` config object for `clientPingTimeout`, `serverPingTimeout`, and `pingPongLoggingEnabled`; validate it in Slack types/schema/metadata; preserve root/account inheritance; pass it into `SocketModeReceiver` before `start()`; add focused config and runtime tests; and update the public Slack/config docs.
