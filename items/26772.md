## Summary

Keep open. Current main still uses full-reindex temp SQLite databases with atomic swap, and I did not find persisted checkpoint/resume state, resumable temp reuse, indexing-complete markers, or the proposed memorySearch.indexing configuration. The related #70567 is adjacent but explicitly distinct and remains open, so it does not supersede this feature request.

## Evidence
