## Summary

Keep open. Current `main` still defines agent avatars as any workspace-relative path, not `avatars/` specifically, and it has explicit Control UI auth-aware avatar fetching. But there is no strong evidence that the reported Control UI regression is fixed end-to-end, and the issue timeline already links a newer open follow-up (#70700) describing inconsistent workspace-relative avatar handling.
