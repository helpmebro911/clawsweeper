## Summary

Keep open. Current `main` has the broad runtime migration-before-validation flow, but it does not implement the specific `agents.defaults.memorySearch` cleanup reported here. The current strict `MemorySearchSchema` still rejects legacy flat keys like `chunkSize`, `chunkOverlap`, and flat `maxResults`; the runtime agent migration only moves top-level `memorySearch` into `agents.defaults.memorySearch`. Related PR #68685 is an open, unmerged fix attempt for exactly this remaining gap, so the issue is still a real tracked bug rather than implemented on main.

## Best Possible Solution

Keep #68664 open until #68685 or an equivalent maintainer-reviewed core fix lands. The fix should add a runtime/doctor migration for already-nested `agents.defaults.memorySearch` legacy keys before validation, decide whether to remap `chunkSize`/`chunkOverlap`/`maxResults` to `chunking.tokens`/`chunking.overlap`/`query.maxResults` or strip them intentionally, and add focused migration plus config-load regression tests.
