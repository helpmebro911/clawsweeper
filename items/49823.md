## Summary

Keep open. Current main does not implement a `maxCostPerRun` config option or equivalent pre-dispatch cost cap for embedded/heartbeat runs. The linked PR #49840 appears to track the proposed implementation, but it is still open and not present in the checked-out main, so this recent concrete core config request should not be auto-closed.
