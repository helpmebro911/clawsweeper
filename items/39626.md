## Summary

Keep open. Current main still lacks source, docs, or regression-test evidence that the reported bare JSON-ish pseudo tool-call leak, `exec{"command": ...}`, is prevented across shell/Telegram delivery. Existing hardening covers XML-style tool-call tags, MiniMax XML, model control tokens, memory scaffolding, and downgraded bracketed `[Tool Call: ...]` text; the OpenAI-compatible `tool_choice: "auto"` mitigation may reduce one provider root cause but does not prove this exact visible leak is fixed for the reported surfaces.

## Best Possible Solution

Keep tracking this as a behavior bug. The best fix is to prevent the affected provider transport from producing bare pseudo tool-call text and/or add a conservative assistant-visible sanitizer for bare JSON-ish pseudo calls such as `exec{...}`, with focused regression coverage for final reply payloads plus streaming/block delivery used by shell and Telegram. If the root cause is provider configuration, keep that transport fix but still ensure raw pseudo-call text cannot reach user-visible delivery paths.
