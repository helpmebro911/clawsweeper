## Summary

Keep this issue open. Current main still only supports local-gateway `sessions_send`/`sessions_spawn` routing, and the issue is paired with same-author open PR #43656 using closing syntax as the implementation candidate.

## Best Possible Solution

Keep this issue open as the canonical native cross-gateway `sessions_send` tracker. Maintainers should review, revise, land, or close PR #43656; if that PR is not the right shape, a replacement should add a documented secure peer/remote-gateway contract across config schema, gateway protocol, session tool schemas, auth/token resolution, tests, and user-facing docs.
