## Summary

Keep open. Current main appears to handle the runtime auto-enable/disable behavior safely, and generic plugin plus gateway docs cover pieces of the answer, but the requested BlueBubbles-specific documentation is still missing from the BlueBubbles guide.

## Best Possible Solution

Add a focused section to `docs/channels/bluebubbles.md` for disabling or safe local operation. It should compare `channels.bluebubbles.enabled=false` with `plugins.entries.bluebubbles.enabled=false`, include a JSON5 example using `gateway.mode="local"`, `gateway.bind="loopback"`, token auth, and both BlueBubbles disablement flags, and briefly state what startup can mutate or repair with links to the public plugin and gateway configuration docs.
