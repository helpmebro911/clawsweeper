## Summary

Keep this issue open. Current main has generic plugin disablement/runtime-dependency repair docs, generic gateway loopback/token docs, and source/tests showing channel-level disablement now blocks bundled plugin auto-enable. However, the requested BlueBubbles-specific documentation is still missing: the BlueBubbles page does not explain `channels.bluebubbles.enabled=false` versus `plugins.entries.bluebubbles.enabled=false`, does not include the composed safe local loopback/token example with both BlueBubbles layers disabled, and does not explain startup config mutation behavior in that BlueBubbles context.

## Best Possible Solution

Add a focused section to `docs/channels/bluebubbles.md` explaining `channels.bluebubbles.enabled=false` versus `plugins.entries.bluebubbles.enabled=false`, include a JSON5 safe local example with `gateway.mode="local"`, `gateway.bind="loopback"`, `gateway.auth.mode="token"`, `channels.bluebubbles.enabled=false`, and `plugins.entries.bluebubbles.enabled=false`, and link to the public plugin and gateway config docs while briefly stating what startup may persist, repair, or mutate.
