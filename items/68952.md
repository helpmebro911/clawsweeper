## Summary

Keep open. Current main still has the reported Ollama onboarding UX gap: local setup suggests bare `gemma4`, exact-name merging allows `gemma4` and `gemma4:latest` to both appear, and selected-model handling still says `Downloading`/`Downloaded` when it calls the Ollama pull path.

## Best Possible Solution

Keep the issue open and fix it in the Ollama bundled plugin: normalize Ollama bare names and `:latest` aliases for setup/model-picker de-duping and availability checks, prefer the discovered tagged name when present, and change pull progress copy to a truthful `Checking/Preparing/Using` flow unless actual layer download progress is observed. Add focused tests for `gemma4` selected with `/api/tags` returning `gemma4:latest`.
