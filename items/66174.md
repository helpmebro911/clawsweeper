## Summary

Keep this PR open. Current main still lacks save-time canonicalization for persisted session transcript paths, and this PR is a focused implementation candidate, but it needs revision because the proposed helper normalizes through the compatibility resolver instead of forcing the active sessions directory. Security review did not find supply-chain or CI-sensitive changes in the PR diff.

## Best Possible Solution

Keep this PR open and request a small revision: add save-time normalization in `src/config/sessions/store.ts`, derive the canonical target from `path.dirname(storePath)` plus a validated `sessionId` rather than `resolveSessionFilePath`, persist both `sessionFile` and the legacy `transcriptPath` alias if that alias remains supported, add coverage for structurally valid cross-root paths, and preserve the external-transcript deletion safeguards covered by the pruning tests.
