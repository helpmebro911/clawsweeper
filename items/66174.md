## Summary

Keep this PR open. Current main still lacks save-time canonicalization for persisted session transcript paths, and the PR is a focused implementation candidate, but the patch should be revised before merge because it routes canonicalization through the compatibility resolver that can preserve cross-root agent session paths.

## Best Possible Solution

Keep this PR open and request a small revision: add save-time normalization in `src/config/sessions/store.ts`, derive the canonical target directly from `path.dirname(storePath)` plus the validated `sessionId` instead of `resolveSessionFilePath`, persist both `sessionFile` and the compatibility `transcriptPath` alias if that alias remains supported, and keep the regression test while preserving external-transcript deletion safeguards.
