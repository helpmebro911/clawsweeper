## Summary

Keep #39857 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still returns positive context-token overrides before known model-window resolution, and the reply, session persistence, /status, and status-summary paths still feed those override values into the affected behavior. The related fix PR #39859 is open/unmerged in the provided GitHub context, so the central bug is not implemented on main.

## Best Possible Solution

Keep #39857 open until PR #39859 or an equivalent maintainer-approved fix lands. The fix should make positive configured and persisted context-token values act as ceilings when the selected model's smaller known window is available, preserve existing override behavior when the model window is unknown, align or remove the duplicate status-summary resolver behavior, and add focused regression coverage for the resolver, reply setup, /status, status summary, and session metadata persistence paths.
