## Summary

Keep #8635 open. Current main still does not expose raw structured token/cost usage through the agent-facing `session_status` tool. The tool renders a `/status`-style text card and returns only `ok`, `sessionKey`, `changedModel`, and `statusText` in details; structured per-session usage remains on separate Gateway RPC methods such as `sessions.usage`. The related #39400 is adjacent but narrower usage-window instability, not a canonical replacement for the raw session/last-turn token API requested here.

## Best Possible Solution

Keep the issue open until OpenClaw adds a documented structured agent-facing usage API. The likely path is to extend `session_status.details` with stable `usage.session`, `usage.lastTurn`, cost, budget/window availability, and source fields, or add an equivalent supported tool that reuses `sessions.usage` and transcript usage parsers while preserving current visibility checks.
