## Summary

Keep open. Current main still does not expose the requested structured per-session token usage through the `session_status` agent tool. The tool has only `sessionKey` and `model` inputs and returns a `/status`-style text card plus `statusText` metadata. There is a separate Gateway `sessions.usage` API with raw usage totals, but that is not the requested agent-facing `session_status` response shape.

## Best Possible Solution

Keep #8635 open until maintainers either add a structured `usage` payload to `session_status` or decide on a different agent-facing API. A conservative implementation would reuse the existing transcript/session usage parsers and Gateway usage types where possible, define the exact response contract for session totals, last turn, cost, and quota/budget fields, then add focused tests under `src/agents/openclaw-tools.session-status.test.ts` plus docs in `docs/concepts/session-tool.md` and `docs/reference/token-use.md`.
