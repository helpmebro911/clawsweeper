## Summary

Keep #8635 open. Current main at 4a3030df still does not expose raw structured token/cost usage through the agent-facing session_status tool. It renders a /status-style text card and can improve that card with transcript fallback, while structured per-session usage remains on separate Gateway methods rather than the minimal/default tool surface requested by the issue.

## Best Possible Solution

Keep the issue open until OpenClaw adds a documented structured agent-facing usage API. The likely implementation path is either extending session_status.details with stable usage.session, usage.lastTurn, cost, provider-window availability/source fields, and tests, or adding an equivalent supported tool that reuses Gateway sessions.usage and transcript usage parsers while preserving current visibility checks.
