## Summary

Keep #12429 open. Current main has minimal single-pod Kubernetes deployment support and CIDR-based first-time node auto-approval, but it still does not implement the requested Kubernetes ServiceAccount Trust path for autoscaled worker/node-host pods. The linked implementation PR #12434 is closed unmerged, and the broader distributed runtime RFC #42026 does not specifically replace the SA-token/TokenReview authentication request.

## Best Possible Solution

Keep #12429 open until OpenClaw implements an opt-in Kubernetes ServiceAccount Trust path across gateway config/schema, WebSocket connect protocol, GatewayClient/node-host credential handling, TokenReview validation, tests, docs, and deployment assets, or until maintainers explicitly designate a narrower canonical distributed-runtime/K8s-auth issue as the replacement. #42026 can inform the architecture, but it should not silently supersede this specific SA Trust request.
