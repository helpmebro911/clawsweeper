## Summary

Current main still does not implement the requested secure Kubernetes ServiceAccount Trust pairing path for autoscaled worker/node-host pods. The repo now has minimal single-pod Kubernetes deployment docs/manifests, but gateway auth, the WebSocket connect schema, the client payload, server auth handling, and node-host runtime remain token/password/device-token based. The referenced PR #12434 is closed unmerged, and the broader distributed-runtime RFC #42026 does not precisely supersede this narrow K8s pairing request.

## Best Possible Solution

Keep this issue open until OpenClaw either implements a secure, opt-in Kubernetes ServiceAccount Trust path across gateway config/schema, WebSocket connect protocol, TokenReview validation, node-host CLI/runtime support, tests, docs, and deployment assets, or a maintainer explicitly makes a narrower canonical design issue the source of truth. If the team wants this outside core, first define a documented gateway handshake/auth extension point; the current plugin/node runtime surfaces do not appear sufficient to replace connect-time authentication safely.
