## Summary

Keep #12429 open. Current main has a minimal single-pod Kubernetes/Kustomize install path and shared-token gateway auth, but it still lacks the requested opt-in Kubernetes ServiceAccount Trust flow for autoscaled node-host/worker pods. The linked implementation PR #12434 was closed unmerged, and the broader #42026 distributed runtime RFC does not specifically supersede the TokenReview-based auth request.

## Best Possible Solution

Keep #12429 open until OpenClaw implements an opt-in Kubernetes ServiceAccount Trust path, or maintainers explicitly nominate a narrower canonical K8s auth/distributed-runtime issue. The likely implementation should cover gateway config/types/schema, WebSocket ConnectParams and GatewayClient auth payloads, node-host credential handling, fail-closed TokenReview validation and allowlisting, pairing-skip policy for validated node identities, tests, docs, and deployment assets. Use #12434 as prior implementation evidence and #42026 as architecture input, but do not treat either as shipped replacement today.
