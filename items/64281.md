## Summary

Keep open. Current main at c2d82b87eee7a86bbd0e851169e0ca5559c44fe6 still does not implement `sessions_delegate` or `sessions_delegate_batch`, and the existing `sessions_spawn`/`sessions_send` tools are not equivalent to a fresh synchronous spawn-and-wait delegation primitive. This PR remains the implementation candidate, but it needs maintainer product/API judgment and cleanup for sanitizer coverage plus output-capture/cleanup ordering before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate for synchronous subagent delegation. Maintainers should decide the product/API direction, rebase it onto current main, add sanitizer coverage for delegate attachments, fix output-capture versus cleanup ordering including finally-release behavior, and land focused tests plus docs/display/policy updates if the new core tool surface is accepted.
