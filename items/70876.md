## Summary

Keep this PR open. Current main still omits configured MCP server tools from `/tools`, so the PR targets a real product bug, but the PR is not ready for cleanup closure or merge because it mixes the MCP inventory fix with unrelated plugin install dependency-resolution changes on a security-sensitive path.

## Best Possible Solution

Keep this PR open, but request a focused revision or supersede it with a narrow PR for the `/tools` MCP inventory bug. The fix should drop the installer changes, deliberately choose a lifecycle-safe way to obtain MCP inventory, materialize names the same way as the run path, apply the final bundled-tool policy, render an MCP servers group, and add compact plus verbose regression coverage.

- The latest PR approach described in the provided context only lists MCP tools when an MCP runtime already exists; that may not satisfy the linked reproduction where `/tools verbose` is run before asking the agent to use MCP tools.
- The PR needs regression tests for compact and verbose output, reserved-name collisions, `bundle-mcp` ownership, final effective policy filtering, and the chosen MCP runtime lifecycle behavior.
