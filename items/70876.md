## Summary

Current main still has the /tools inventory gap this PR targets: the command renders only core/plugin/channel inventory while the run path separately materializes and filters bundled MCP tools. The PR is not obsolete, but it should stay open for a focused revision because the provided PR context shows unrelated plugin installer changes and review-noted defects mixed into the branch.

## Best Possible Solution

Keep this PR open, but request a focused revision. The best path is to drop the installer changes, implement MCP inventory through a lifecycle-safe session MCP inventory helper, display materialized provider-safe MCP tool ids as an MCP servers group, apply the same final effective tool policy as the run path, and add compact plus verbose regression tests. If maintainers prefer cleaner history, supersede this with a fresh narrow PR linked to #70874's user-visible bug.
