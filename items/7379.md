## Summary

Keep open. Current `main` still implements `/usage` footers as tokens/cache/cost plus optional session key only. GitHub Copilot premium/chat quota data exists and is still surfaced in `/status`, but it is not wired into the per-reply `/usage` footer.

## Evidence

- **Reply footer formatter only renders token/cache/cost fields:** `formatResponseUsageLine` accepts usage tokens and optional cost config, then returns `Usage: <in> / <out>` with cache and estimated cost suffixes only. There is no provider-usage or quota input in this formatter.
  - file: [src/auto-reply/reply/agent-runner-usage-line.ts:9](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/auto-reply/reply/agent-runner-usage-line.ts#L9)
  - command: `nl -ba src/auto-reply/reply/agent-runner-usage-line.ts | sed -n '1,120p'`
  - sha: [7e16e3d07723](https://github.com/openclaw/openclaw/commit/7e16e3d07723cc3903f40ce94f53a3920c88b1fc)
- **Reply path never loads provider usage for `/usage` footer:** The response-footer path resolves `responseUsageMode`, computes API-key cost visibility, calls `formatResponseUsageLine(...)`, and optionally appends the session key for `full`. It does not call `loadProviderUsageSummary` or any provider-usage fetcher.
  - file: [src/auto-reply/reply/agent-runner.ts:1449](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/auto-reply/reply/agent-runner.ts#L1449)
