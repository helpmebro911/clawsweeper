## Summary

Current main still does not add GitHub Copilot Premium/Chat quota to the per-response `/usage` footer. The quota data exists and `/status` can render it, but the footer path still formats only token/cache/cost/session details. Related PR #43098 attempted the exact change but was closed unmerged, so it is useful prior work rather than a shipped fix.

## Best Possible Solution

Keep the issue open until the response footer path intentionally loads or reuses cached provider usage for GitHub Copilot, appends Premium/Chat quota in a way that preserves `/usage tokens` versus `/usage full` semantics, and adds a focused regression test around the per-response footer. PR #43098 is the canonical prior attempt to review if maintainers want a starting point.
