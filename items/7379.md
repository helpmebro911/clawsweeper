## Summary

Keep #7379 open. Current main still has Copilot Premium/Chat quota support only on the provider-usage/status path, while the per-response `/usage` footer formats turn token/cache/cost data plus the full-mode session key. The directly related PR #43098 was closed unmerged, and a maintainer comment already reached the same keep-open conclusion on April 26, 2026.

## Best Possible Solution

Keep #7379 open. The best path is a core reply-footer change that reuses the active provider's cached or timeout-bounded usage summary for GitHub Copilot OAuth sessions, appends formatted Premium/Chat quota when available, degrades silently on usage-fetch errors, and adds a focused regression test in the reply runner. PR #43098 is useful prior art, but it is closed unmerged and not shipped on current main.

## Risks / Open Questions

- A future implementation should not add an unbounded provider-usage network call to every reply; it should reuse cached data or enforce a short timeout so final response delivery is not delayed.
- The implementation should preserve `/usage tokens` versus `/usage full` semantics deliberately, rather than silently expanding one mode in a surprising way.
