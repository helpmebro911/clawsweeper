## Summary

Keep #7379 open. Current main still keeps Copilot Premium/Chat quota on the provider-usage/status path, while the per-response `/usage` footer only formats turn token/cache/cost data plus the full-mode session key. The directly related PR #43098 was closed unmerged, and the existing maintainer review comment also kept this issue open.

## Best Possible Solution

Keep #7379 open. The best path is a focused core reply-footer change that reuses the active provider's cached or timeout-bounded usage summary for GitHub Copilot OAuth sessions, appends formatted Premium/Chat quota when available, degrades silently on fetch errors, and adds a regression test in the reply runner. PR #43098 is useful prior art, but it is closed unmerged and not shipped on current main.

## Risks / Open Questions

- A future implementation should avoid adding an unbounded provider-usage network call to every reply; it should reuse cached data or enforce a short timeout so final response delivery is not delayed.
- The implementation should deliberately preserve `/usage tokens` versus `/usage full` semantics instead of silently expanding one mode in a surprising way.
