## Summary

Keep #70253 open. Current main still matches the reported Mattermost streaming/config bug, and the channel-level agent-to-agent behavior is not solved or superseded by the narrower related Mattermost block-streaming tracker #43020.

## Best Possible Solution

Keep this issue open or split it into two linked maintainable tasks: first, make the Mattermost main inbound path honor channels.mattermost.streaming and account.blockStreaming exactly like the slash/button paths, with a regression test for streaming off and blockStreaming true; second, define an explicit channel-level A2A contract for bot-authored messages, including Mattermost allowBots parity, mention behavior, route/fanout semantics, and loop guards. #43020 can remain the narrower block-streaming tracker, but it does not cover the A2A routing part of #70253.
