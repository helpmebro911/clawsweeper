## Summary

Keep #70253 open. Current main still matches the reporter's concrete Mattermost streaming root cause: the main inbound reply path unconditionally passes `disableBlockStreaming: true`, while the other Mattermost reply paths honor `account.blockStreaming`. The channel-level agent-to-agent part is also not implemented as reported: Mattermost has no `allowBots` config surface, filters self/system posts before routing, then resolves a single agent route and drops unmentioned group messages under the default mention gate. The issue is recent, specific, and not superseded by the related ClawSweeper context.

## Best Possible Solution

Keep the issue open. A maintainer fix should at minimum make the Mattermost main inbound path respect `account.blockStreaming`/canonical streaming config consistently with the slash/button/model-picker paths, and add regression coverage. The channel-level A2A part needs product/API judgment: either document that `tools.agentToAgent` only gates session tools, or add a Mattermost-compatible bot-message policy such as `allowBots` plus safe routing/mention-loop guards if shared-channel agent interaction is intended.
