## Summary

Keep open. Current main still does not surface WhatsApp forwarding metadata: the inbound type, monitor enrichment, context builder, and prompt renderer do not carry `contextInfo.isForwarded` or `contextInfo.forwardingScore`. The PR is a focused bundled WhatsApp channel improvement, and the pinned Baileys contract supports the requested source fields.

## Best Possible Solution

Keep the PR open. Rebase it onto current main, preserve the focused implementation path of extracting Baileys forwarding metadata in the WhatsApp inbound monitor, threading it through `WebInboundMessage` and `buildWhatsAppInboundContext`, rendering `forwarding_score` inside the existing untrusted forwarded metadata block, and adding focused contract tests for forwarded and non-forwarded WhatsApp messages.
