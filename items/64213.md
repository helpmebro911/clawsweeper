## Summary

Current main still does not implement the requested fallback from a gateway catalog miss to explicit `models.providers.<provider>.models[].input` metadata. The resolver only inspects the catalog plus narrow provider shims, all attachment entry points depend on that resolver result before parsing images, and the current tests still preserve fail-closed catalog-miss behavior. The related open PR #67766 is an implementation candidate for the same runtime class via #67759, but it remains unmerged in the provided context, so this issue should stay open.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and either land #67766 or replace it with a focused maintainer patch that lets `resolveGatewayModelSupportsImages()` consult normalized explicit provider model metadata on catalog miss. Coverage should include explicit provider refs, unique provider inference, catalog miss plus config hit, text-only config, catalog-hit precedence, and the chat, agent RPC, and node-event attachment paths.
