## Summary

Keep open. Current main still builds and renders context-breakdown file labels from `file.name` directly, while hook-mutated bootstrap files are only sanitized for a usable `path`. A virtual bootstrap file with `path` but no runtime `name` can still surface as `undefined`; the linked PR #48021 tracks a plausible fix but is open and unmerged, so the issue remains a valid bug report rather than cleanup-ready.

## Evidence
