## Summary

Keep open. Current main still has the exact fallback gap described by #40088: memory sync defaults to no interval, interval sync would not mark memory dirty, and the chokidar watcher has no error handler. The related PR #42042 directly proposed the right kind of fix, but the provided GitHub context says it was closed unmerged, and those changes are not present in current main.

## Evidence

- **Default interval remains disabled:** The resolved memory search sync config still defaults `intervalMinutes` to `0` when no override is provided, so default users still have no periodic fallback if watcher events stop.
  - file: [src/agents/memory-search.ts:362](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/agents/memory-search.ts#L362)
