## Summary

Keep open. Current main has related cron delivery improvements, but it does not implement the requested deterministic multi-target cron output routing. The public cron schema still models `delivery` as `none | announce | webhook` only, with no `output` field or file target; main-session cron only queues the configured system-event input; isolated cron delivery routes final output to chat/webhook paths, not arbitrary files or named session injection.

## Evidence
