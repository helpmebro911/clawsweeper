## Summary

Current main still does not clearly fix the central in-place source update/A2UI stale artifact failure. Adjacent update hardening and the Telegram SDK-helper mismatch are covered, but the reported source checkout build collision remains plausible: the git update path still builds in the existing checkout, the clean-worktree gate misses ignored generated files, and the A2UI bundler still runs the vendor lit TypeScript build without first clearing the implicated `dist` output.

## Best Possible Solution

Keep this open and fix the source-update/A2UI edge case directly. The best path is to clean or explicitly reject stale `vendor/a2ui/renderers/lit/dist` before `canvas:a2ui:bundle` during in-place source updates, add regression coverage for ignored generated A2UI output causing TS5055-style collisions, and document a safe recovery command for source checkouts when updater/build state is dirty.
