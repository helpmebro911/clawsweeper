## Summary

Keep open. Current `main` documents explicit default-model selection plus ordered failover models, and it supports provider-owned router defaults like `openrouter/auto` and `kilocode/kilo/auto`, but I did not find an OpenClaw-owned feature that selects models based on inferred task complexity.

## Evidence

- **Quickstart still expects an explicit default model:** The model quickstart tells users to pick a provider and set `agents.defaults.model.primary` directly; there is no documented task-complexity router in the basic model flow.
  - file: [docs/providers/models.md:11](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/providers/models.md#L11)

- **Config surface is primary plus fallbacks:** The config reference describes `model` as either a string or `{ primary, fallbacks }`. That is failover configuration, not complexity-driven selection logic.
  - file: [docs/gateway/configuration-reference.md:1169](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/gateway/configuration-reference.md#L1169)
