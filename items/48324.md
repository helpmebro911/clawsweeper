## Summary

Keep open. Current main does not implement PR #48324’s central behavior: persisted skill snapshots still invalidate on watcher version and skill-filter changes, not runtime eligibility/config/env/bin changes. The PR is also not clearly merge-ready because later review comments identify incomplete eligibility inputs and incomplete cache-path coverage, so this needs revision or a broader maintainer-owned fix rather than cleanup closure.

## Best Possible Solution

Keep PR #48324 open for maintainer review or author revision. A complete fix should derive a deterministic eligibility signature from the same effective inputs used by shouldIncludeSkill/evaluateRuntimeEligibility, persist it on SessionSkillSnapshot, and compare it in every persisted snapshot reuse path with focused regression coverage for env, config, binary, remote eligibility, and skill-filter/version interactions.
