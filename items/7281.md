## Summary

Current main still does not implement configurable fallbackChannel/fallbackChannels for hook mappings. Hook mapping config, schema, mapping normalization, dispatch payloads, cron delivery types, target resolution, and outbound delivery still model one primary channel/to target, with failureDestination serving failure notifications rather than alternate reply delivery. The existing issue discussion also already kept this open, and the linked closed Slack availability issue does not supersede the broader feature request.

## Best Possible Solution

Keep #7281 open for maintainer/product handling. A complete fix should add typed fallbackChannel or ordered fallbackChannels support to hook mapping config, schema/help/docs, transforms, replay/idempotency scope, hook dispatch payloads, cron delivery planning/types, and outbound delivery; retry alternates only after primary delivery failure; distinguish this from channel="last", bestEffort delivery, and failureDestination; and add focused tests for primary failure followed by fallback success plus all-targets-failed reporting.
