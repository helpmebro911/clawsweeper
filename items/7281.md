## Summary

Current main still does not implement configurable hook delivery fallback chains. Hook mappings, schema, dispatch, cron delivery types, docs, and outbound delivery still model one primary channel/to target; existing fallbackChannel references are unrelated session/channel-selection fallbacks, and the issue discussion already kept this open for the same reason.

## Best Possible Solution

Keep #7281 open for maintainer/product handling. A complete implementation should add typed fallbackChannel or ordered fallbackChannels support to hook mapping config, schema/help/docs, transform outputs, dispatch payloads, CronDelivery planning/types, and outbound delivery; retry alternates only after primary delivery failure; keep it distinct from channel="last", bestEffort delivery, and failureDestination; and add focused tests for primary failure followed by fallback success plus all-targets-failed reporting.
