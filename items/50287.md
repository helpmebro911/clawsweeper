## Summary

Keep open. Current main already has a schema-level guardrail that restricts `models.providers.*.models[*].input` to `text` or `image`, and the generated public schema mirrors that. However, I did not find the issue’s explicit follow-through: a targeted invalid `audio` modality regression test or docs that state the validation expectation. The issue is recent, clear, and actionable, and the broader related #50285 is still open, so conservative cleanup should not auto-close it.
