## Summary

Current main does not implement the PR’s bound-DM-topic guard. Telegram runtime bindings can rewrite a route to a final bound ACP session key, but the current inbound and native-command Telegram paths still append the DM-topic `:thread:<chatId>:<topicId>` suffix unconditionally. The PR remains a plausible focused bug fix, although it needs rebasing from the old `src/telegram/*` paths to `extensions/telegram/src/*`.

## Best Possible Solution

Keep #44900 open or supersede it only with a canonical maintainer-approved fix that preserves the exact bound ACP session key for Telegram DM topics. The likely implementation is to carry an explicit bound-route/final-session flag out of `resolveTelegramConversationRoute()` and skip DM-topic suffixing in `bot-message-context.ts`, `bot-native-commands.ts`, and session-state/debounce paths while preserving unbound DM-topic isolation. Add regression tests for bound DM-topic inbound messages and native commands, plus an unbound DM-topic control case.
