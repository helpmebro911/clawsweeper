## Summary

Keep #70014 open. Current main has multi-workspace dreaming fanout, but `resolveMemoryDreamingWorkspaces` still derives candidates only from `agents.list` and adds the default agent only when that list is empty. That leaves the reported primary workspace skip plausible when AGI/sub-agent workspaces are listed but the primary `agents.defaults.workspace` / `main` workspace is not an explicit `agents.list` entry. The related #65653 cleanup was about orphaned narrative sessions, not this enumeration gap.

## Best Possible Solution

Keep the issue open and fix memory dreaming workspace discovery. The likely path is to update `resolveMemoryDreamingWorkspaces` to include the primary/default workspace in addition to explicit agent workspaces, dedupe normalized paths, preserve correct `agentIds`, and add regression coverage for `agents.defaults.workspace` plus AGI/sub-agent entries where `main` is not listed. Also verify `dreaming-phases` session ingestion still maps agent ids correctly after the resolver change.
