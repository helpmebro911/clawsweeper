## Summary

Keep open. Current main fans dreaming out across configured `agents.list` workspaces, but the helper still only falls back to the implicit/default primary agent workspace when `agents.list` is empty. That leaves the reported “primary workspace plus configured sub-agents” case plausibly unprocessed, and no current test covers it.

## Best Possible Solution

Update the memory dreaming workspace resolver to include the implicit/default primary workspace when `agents.list` contains only sub-agents, dedupe by normalized workspace path, and add a regression test for `agents.defaults.workspace` plus multiple configured sub-agent workspaces. Keep #70014 open until that behavior is implemented or maintainer product direction says primary workspaces must be explicitly listed.
