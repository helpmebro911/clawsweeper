## Summary

Keep this issue open. Current main still enumerates configured agent workspaces first and only falls back to the runtime primary workspace when that configured list is empty, so a config with AGI sub-agent workspaces can still skip the primary workspace reported here.

## Best Possible Solution

Fix memory-core dreaming workspace enumeration so scheduled dreaming processes the runtime primary workspace plus configured agent workspaces, deduped by normalized path. The likely implementation belongs in the runtime dreaming path that already receives `ctx.workspaceDir`, with matching doctor/status handling and regression coverage for a non-empty `agents.list` of sub-agents while the primary workspace is only available from runtime/status context.
