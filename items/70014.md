## Summary

Keep #70014 open. Current main still appears to skip the primary/default workspace when `agents.list` is non-empty and does not include `main`; dreaming enumerates only the configured agent entries, matching the reported four-AGI-workspace symptom. The related #65653 is a different, closed session-cleanup/session-directory issue rather than a canonical fix for this enumeration bug.

## Best Possible Solution

Keep the issue open and fix the resolver: make `resolveMemoryDreamingWorkspaces` include the default agent workspace in addition to configured `agents.list` entries, dedupe by normalized workspace path, preserve all relevant `agentIds`, and add a regression test with `agents.defaults.workspace` plus only `agi-*` entries in `agents.list` so the sweep processes the primary workspace too.

## Evidence

- **Dreaming workspace resolver omits default when agents are configured:** `resolveMemoryDreamingWorkspaces` builds `agentIds` from `cfg.agents.list`; it only adds `resolveDefaultAgentId(cfg)` when that list yields no IDs. A config with only `agi-ceo`, `agi-cdo`, `agi-cmo`, and `agi-cso` will not enumerate the default/main workspace separately.

So a non-empty subagent list prevents fallback to the primary workspace.
