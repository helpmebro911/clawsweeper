## Summary

Keep this issue open. Current main still resolves dreaming workspaces from configured agents first and only falls back to the runtime/primary workspace when no configured workspaces exist, so a setup with configured AGI sub-agent workspaces can still skip the primary `/root/.openclaw/workspace`. The prior ClawSweeper comment identified the same root cause, and the relevant code remains unchanged in current main.

## Best Possible Solution

Keep this issue open and fix memory-core dreaming workspace enumeration so scheduled dreaming processes the runtime/primary workspace plus configured agent workspaces, deduped by normalized path. The likely change belongs at the runtime call sites that have `ctx.workspaceDir` or memory manager status workspace available, plus regression coverage for a non-empty `agents.list` containing sub-agents while the primary workspace only comes from runtime/status context.
