## Summary

Keep open. Current main still permits the reported insecure combination: `resolveGatewayRuntimeConfig` rejects `tailscaleMode === "funnel" && authMode !== "password"`, but it does not reject `tailscaleMode === "serve" && authMode === "none"`. Because `tailscale serve` keeps the gateway on loopback, the later non-loopback/no-auth guard does not fire, and existing tests/docs still describe or accept that path.

## Evidence

Since Serve stays on loopback, this path passes current startup validation.
