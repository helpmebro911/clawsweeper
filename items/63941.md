## Summary

Keep this PR open. Current main still rejects admin-configured self-hosted Firecrawl baseUrl values, the linked bug remains open, and this PR is a focused implementation candidate with recent discussion addressing both the strict SSRF guard and SecretRef test-coverage concerns.

## Best Possible Solution

Keep this PR open for maintainer review. Land this PR or a maintainer-owned equivalent that permits admin-configured self-hosted Firecrawl endpoints, keeps public hosts on HTTPS and the strict guard, allows cleartext HTTP only for private/loopback targets, routes trusted private endpoints through the existing trusted web-tools guard, preserves SecretRef coverage, adds endpoint/SSRF regression tests, updates the Firecrawl and web_fetch docs, then close linked bug #63877 as implemented and supersede or close overlapping #59666 as appropriate.
