## Summary

Keep open. Current main still has a simple Control UI browser-dictation mic: click toggles recording, final transcripts append to the draft, and sending remains a separate Enter/send-button action. The requested short-click auto-send, 3-second hold-to-continuous mode, separate continuous recording state, restart-on-pause behavior, and mode-specific tooltip/status copy are not implemented. The related closed reports cover basic voice input and separate Realtime Talk, not this narrower mic-button UX.

## Best Possible Solution

Keep the issue open for maintainer product/design judgment. If accepted, implement it in the Control UI source rather than generated dist output: add explicit dictation modes, expose a startStt option for one-shot versus continuous behavior, auto-send final transcripts carefully, avoid globals for recognition restart state, update tooltip/aria copy, and add targeted chat view/STT tests plus Control UI docs if the behavior ships.
