## Summary

Keep open. Current main has session-delta throttling and a default embedding cache, but changed session transcripts still build a full session entry and go through `indexFile`; that path clears existing chunk/vector/FTS rows before rewriting them. The requested append-only/incremental session indexing remains unresolved, and related PR #59577 is still an implementation candidate rather than a reason to close the issue.

## Best Possible Solution

Keep this issue open and land a reviewed incremental session-sync implementation in the current `extensions/memory-core` plus `src/memory-host-sdk/host` layout. The fix should reuse existing embedding cache behavior but add durable append/offset or chunk metadata, overlap-safe reprocessing, full-reindex fallback on truncation/config/model/parser changes, and regression tests for append growth, unchanged transcripts, compaction, and failure recovery. PR #59577 is a useful candidate or design reference, but it needs review/porting before it can close the issue.
