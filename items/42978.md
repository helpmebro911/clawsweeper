## Summary

Current main still does not implement the requested `bailian` / `BAILIAN_API_KEY` compatibility, but this PR is not merge-ready as-is because it targets the old static env-map shape and bundles unrelated cron and channel-log changes.

## Best Possible Solution

Keep this PR open for maintainer judgment, but ask for a split and rebase before merge. The auth fix should be rebuilt against the current manifest-driven provider auth surface by deciding whether `bailian` is a compatibility alias for Qwen/Alibaba and whether `BAILIAN_API_KEY` belongs in bundled plugin metadata or setup descriptors. The cron warning should be handled against #42196, and the channel-log fallback should stay separate from the auth work and be reconciled with the current rolling logger behavior.
