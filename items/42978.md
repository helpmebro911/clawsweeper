## Summary

Keep open. Current main still has no `bailian` / `BAILIAN_API_KEY` auth surface, but this PR is not merge-ready because it targets the old static env-map shape and bundles unrelated cron/log changes.

## Best Possible Solution

Keep this PR open for maintainer judgment, but ask for a split and rebase before merge. The best path is one auth-only patch for #42858 against current provider manifest/setup/alias metadata, with an explicit decision on whether `bailian` and `BAILIAN_API_KEY` are supported compatibility names. The cron warning should stay with #42196, and the channel-log fallback should stay separate because #42875 is already closed.
