## Summary

Keep #12507 open. Current main still lacks a native trusted integrity/signature contract for SKILL.md skills: ClawHub skill metadata and lock/origin state do not carry digest/signature/publisher-key fields, native skill install downloads and extracts archives without comparing them to trusted version metadata, runtime loading reads local SKILL.md files into skill entries without verification status, and the adjacent ClawHub plugin archive verification explicitly routes skill-family packages away from the plugin installer. A same-day Codex review comment by steipete also kept this issue open as canonical native skill integrity work, and I found no current-main implementation that supersedes that conclusion.

## Best Possible Solution

Keep #12507 open as the canonical native skill integrity work. A complete maintainer-approved fix should add trusted skill-version digest/signature/publisher metadata to the ClawHub skill API, verify native skill archives before extraction, persist expected integrity/signature state in .clawhub origin/lock metadata, reverify SKILL.md or full skill contents during skill snapshot/load, expose verification state on SkillEntry/status/UI/CLI surfaces, and define docs/tests/policy for unsigned or unverified skills and capability restrictions.
