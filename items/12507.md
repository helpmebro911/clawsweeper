## Summary

Keep #12507 open. Current main still lacks a native trusted integrity/signature contract for SKILL.md skills: native ClawHub skill metadata has no digest/signature fields, install downloads and extracts skill archives without comparing them to trusted version metadata, origin/lock state persists only version timestamps, and runtime/status surfaces carry no verification state. Adjacent ClawHub plugin archive verification does not satisfy this because skill-family packages are routed away from the plugin installer.

## Best Possible Solution

Keep #12507 open as the canonical native skill integrity work. A complete fix should add trusted digest/signature/publisher metadata to the ClawHub skill API, verify native skill archives before extraction, persist expected integrity/signature state in .clawhub origin/lock metadata, reverify SKILL.md or full skill contents during skill snapshot/load, expose verification state in SkillEntry/status/UI/CLI surfaces, and document/test policy for unsigned or unverified skills and capability restrictions.
