## Summary

Keep open. Current main does not provide enough evidence to close this Google LLM proxy regression: the Google LLM stream path uses `buildGuardedModelFetch`, and the guarded fetch source/tests still intentionally ignore ambient env proxy settings in strict mode unless an explicit trusted/env proxy request override is configured. The reporter's plain Docker `HTTP_PROXY`/`HTTPS_PROXY` setup therefore remains a plausible bug/regression, even though `models.providers.google.request.proxy.mode: "env-proxy"` appears to be an available workaround path.

## Best Possible Solution

Keep #72057 open and triage it as a Google LLM/model transport proxy regression or documentation/config gap. The likely code path is `extensions/google/transport-stream.ts` -> `src/agents/provider-transport-fetch.ts` -> `src/infra/net/fetch-guard.ts`; a fix should either safely auto-select trusted env proxy mode for provider model requests when env proxy applies, or explicitly document and surface the required `models.providers.google.request.proxy.mode: "env-proxy"` configuration with a targeted regression test for Google LLM streams.
