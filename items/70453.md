## Summary

Keep open. Current main has proxy-aware guarded fetch machinery, but the direct Google/Gemini LLM stream path still calls the generic provider transport fetch without the trusted env-proxy auto-upgrade used by provider HTTP/media paths. In the reported Fake-IP setup, that means the model request can still do local DNS pinning before the proxy path and trip the same SSRF/private-IP block instead of routing through `http_proxy`/`https_proxy`.

## Best Possible Solution

Keep this issue open and fix the LLM provider transport path, likely by applying the same conservative trusted env-proxy gating used in `src/media-understanding/shared.ts` to `buildGuardedModelFetch`: honor `HTTP_PROXY`/`HTTPS_PROXY` for trusted provider endpoints, keep `ALL_PROXY` and `NO_PROXY` safeguards, avoid overriding explicit dispatcher policy, and add a regression test for `google/gemini-*` LLM streaming through env proxy plus a user-visible error/timeout path.
