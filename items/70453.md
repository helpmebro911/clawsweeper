## Summary

Keep open. Current main has better generic proxy and error plumbing, but it does not implement the issue's central request: Google/Gemini model calls do not automatically switch model-provider guarded fetches into trusted env-proxy mode when http_proxy/https_proxy is set. The path still uses strict SSRF-pinned guarded fetch unless the provider is explicitly configured with a request proxy override.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the model-provider HTTP path: either have buildGuardedModelFetch opt into TRUSTED_ENV_PROXY when shouldUseEnvHttpProxyForUrl(target) is true, or document and enforce an explicit models.providers.google.request.proxy mode if automatic env proxy is intentionally disallowed. Add focused regression coverage for google-generative-ai with http_proxy/https_proxy plus NO_PROXY behavior, and verify CLI/WebUI surface the resulting provider/proxy errors.
