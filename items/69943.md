## Summary

Keep open. Current main still has the reported session-memory write path: transcript text is extracted as raw strings, appended as `user:`/`assistant:` lines, then written into `<workspace>/memory/*.md` without a sanitizer for chat-template tokens, raw `<tool_call>` XML, role markers, or `NO_REPLY`. The linked PR #69961 appears to target the right fix, but it is still open in the provided GitHub context and the checked-out main does not contain its sanitizer module or tests.

## Best Possible Solution

Keep the issue open and land a focused session-memory sanitizer fix, likely via PR #69961 or equivalent. The fix should sanitize or elide raw chat-template tokens, plain-text tool-call XML, orphaned role markers, and silent `NO_REPLY`-only turns before persisting memory, with regression tests at the transcript read path and handler write path. Maintainers should also decide whether startup-context should defensively sanitize already-existing memory files or document quarantine guidance for poisoned files.
