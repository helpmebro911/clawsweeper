## Summary

Keep open. Current `main` still persists raw `user:`/`assistant:` transcript text into session-memory files without any sanitization pass, so the reported poisoning path remains plausible and is not implemented on `main`. The issue discussion also already has a linked follow-up PR (#69961), which is still only cross-referenced rather than landed.

## Evidence
