## Summary

Keep open. Current main still has the CJK underestimation path this PR targets: compaction and preemptive compaction call upstream `estimateTokens()` directly, while the existing CJK-aware helper is only used in other paths such as context pruning. The PR is a plausible focused fix for #70052, received maintainer feedback that it is worthwhile, and the author pushed a follow-up commit addressing lint, toolCall arguments, and regression tests; it should not be closed as obsolete.

## Best Possible Solution

Keep #70112 open. The best path is to finish the PR rather than close it: keep the CJK-aware wrapper and regression tests, resolve or explicitly justify the preemptive truncation threshold review comment, then run the changed agents checks before merging and closing #70052.
