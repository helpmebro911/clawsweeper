## Summary

Keep open. Current main still does not implement this PR's core behavior: inbound image messages with a text-only reply model do not switch the main agent run to `agents.defaults.imageModel`. Main currently uses `imageModel` for media-understanding preprocessing, then continues to `runPreparedReply` with the resolved reply model. The PR remains relevant, though its implementation needs maintainer review for the bot-raised correctness concerns.

## Best Possible Solution

Keep #38881 open until maintainers either land a refined implementation or explicitly document that media-understanding preprocessing is the intended product boundary. A strong fix belongs around `src/auto-reply/reply/get-reply.ts`, should decide image-model switching before or in coordination with media understanding, thread the real alias index, respect heartbeat/session/channel overrides, detect actual images including `MediaUrl(s)`, honor imageModel fallbacks, require positive evidence that the current model lacks vision, and add focused `getReplyFromConfig` regression coverage for the final `runPreparedReply` model.
