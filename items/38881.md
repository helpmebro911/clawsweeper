## Summary

Keep #38881 open. Current main still does not auto-switch the final agent reply run from a text-only selected/default model to `agents.defaults.imageModel` when image input is present. The PR targets a real documented gap, but the submitted patch needs follow-up for the unresolved review concerns around detection, sequencing, capability certainty, URL-only media, and fallback-only config.

## Best Possible Solution

Keep #38881 open for a focused production fix in the reply/model-selection path. The implementation should detect actual inbound image inputs after normalization, including inline webchat images, offloaded `media://` refs, `MediaPath(s)`, `MediaUrl(s)`, and image MIME/type data; resolve `agents.defaults.imageModel.primary` and fallbacks through the real alias index/model catalog; switch the final `runPreparedReply` model only when the active model is positively known not to support images; respect heartbeat/session/channel overrides; avoid duplicate media-understanding work; and add regression coverage proving the final agent run receives the configured vision model.
