## Summary

Current main still only displays an existing paired-device `displayName`; it does not provide the requested user-facing paired-device rename workflow in the CLI, gateway protocol, server handlers, Control UI, or docs. The existing `node.rename` implementation is a separate paired-node surface and does not label paired device records.

## Best Possible Solution

Keep #13870 open until OpenClaw adds paired-device label management end to end: a gateway `device.rename` method/schema, an authorized storage update path for paired device records, `openclaw devices rename --device <id> --name <displayName>`, a Control UI rename action, docs at `https://docs.openclaw.ai/cli/devices`, and focused CLI/protocol/UI tests. Keep `node.rename` as a separate paired-node label surface.
