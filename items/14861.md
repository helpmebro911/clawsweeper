## Summary

Keep #14861 open. Current `main` still has targeted startup logs, startup tracing, and readiness gating, but it does not emit the requested aggregate gateway boot summary or surface the `hooks.enabled=true` plus missing `hooks.gmail.account` Gmail watcher skip outcome.

## Best Possible Solution

Keep #14861 open. The best maintainer path is to add a structured startup outcome collector in `startGatewaySidecars` or `startGatewayPostAttachRuntime`, record started/skipped/failed outcomes for Gmail watcher, internal hooks, channels, plugin services, memory, heartbeat, cron, delivery recovery, and related sidecars, then emit a concise normal boot summary after startup settles. Add regression coverage for the aggregate summary and for `hooks.enabled=true` with missing `hooks.gmail.account`.
