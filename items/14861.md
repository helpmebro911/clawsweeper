## Summary

Keep #14861 open. Current main still has targeted startup trace/readiness diagnostics, but it does not emit the requested aggregate Gateway boot summary, and the Gmail watcher missing-account path remains skipped or suppressed during normal startup.

## Best Possible Solution

Keep #14861 open. The maintainer path is to add a structured startup outcome collector around startGatewaySidecars/startGatewayPostAttachRuntime, record started/skipped/failed results for Gmail watcher, internal hooks, channels, plugin services, memory, heartbeat/cron/recovery sidecars, and emit a concise normal boot summary after sidecars settle. Add regression coverage for the aggregate summary and for hooks.enabled=true with missing hooks.gmail.account.
