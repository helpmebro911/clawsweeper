## Summary

Keep open. Current main still exposes the reported Kimi Coding provider path, still runs the Kimi tagged-tool-call stream wrapper, and does not contain a merged provider-specific fix or regression test for repeated identical tool calls after tool results. The reporter's proposed root cause remains overbroad because normal text-only final answers are not rewritten, but the reported loop is recent, detailed, and not safely resolved by current main or by the related Moonshot OpenAI-compatible fix.

## Best Possible Solution

Keep this issue open. Reproduce on current main with a captured Kimi Coding Anthropic Messages transcript around the tool-result turn. If Kimi re-emits identical tagged tool-call sections after results, fix `extensions/kimi-coding/stream.ts` or the provider replay path and add a focused regression in `extensions/kimi-coding/stream.test.ts`. If the transcript shows text-only final output paired with `toolUse` or stale tool replay, fix the embedded runner's stop-reason/tool-call normalization instead. Treat #71274 as useful prior investigation, not a mergeable fix as-is.
