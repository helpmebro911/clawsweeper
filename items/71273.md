## Summary

Keep open. Current main does not contain a merged fix for #71273: the Kimi Coding provider still registers the Kimi stream wrapper, and the wrapper still rewrites parsed tagged tool-call messages from `stop` to `toolUse`. The cross-referenced PR #71274 is closed and unmerged, while #70030 is a separate Moonshot/OpenAI-compatible replay-ID fix, not the Kimi Coding `anthropic-messages` stream path reported here.

## Best Possible Solution

Keep #71273 open. The best next step is a targeted Kimi Coding regression that captures the failing `anthropic-messages` sequence after tool results, proves the final answer ends with `stop` and does not reissue identical tool calls, and then fixes the verified layer in the provider wrapper or embedded runner. Do not close it via #71274 unless that PR is reopened/merged or superseded by a canonical merged fix; keep #70030 treated as separate Moonshot/OpenAI-compatible replay work.
