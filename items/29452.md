## Summary

Keep open. Current `main` implements tool-progress preview updates for Telegram and Discord, but the Feishu surface in this issue is not implemented: the shared streaming docs exclude Feishu, and the Feishu reply dispatcher only wires partial/reasoning streaming callbacks, not tool-progress callbacks.

## Evidence

- **Shared streaming docs exclude Feishu from tool-progress support:** `docs/concepts/streaming.md` says tool-progress preview updates are supported on Discord, Slack, Telegram, and Mattermost; Feishu is not listed.
  - file: [docs/concepts/streaming.md:172](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/concepts/streaming.md#L172)

- **Telegram has explicit tool-progress preview support:** Telegram docs expose `streaming.preview.toolProgress`, and the Telegram dispatcher pushes tool/progress/plan/approval updates into the preview stream.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:969](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/telegram/src/bot-message-dispatch.ts#L969)
