## Summary

Keep open. Current main still routes the primary `sessions_send` target run through the gateway with a nested lane but without `cleanupBundleMcpOnRunEnd`; the gateway handler only forwards cleanup when the request explicitly sets the flag. The embedded runner cleanup remains gated on that flag. There is partial cleanup for follow-up `runAgentStep` calls, and open PR #70465 tracks the exact gateway-side fix, but that fix is not present on current `main`.

## Best Possible Solution

Keep #70364 open until #70465 or an equivalent maintainer-reviewed fix lands. The likely fix belongs in the gateway agent ingress path: default `cleanupBundleMcpOnRunEnd` to true for `isNestedAgentLane(request.lane)` when the caller has not explicitly opted out, and add regression coverage for bare `nested`, `nested:*`, non-nested, unset lane, and explicit false cases. Also consider whether the primary `sessions_send` path should retire by session key after terminal completion as a belt-and-suspenders guard.
