## Summary

Keep this issue open. Current main still gates config writes only by channel and account, while the requested DM/group/topic/conversation-specific policy is not represented in the authorization scope, write targets, bundled channel schemas, docs, or tests.

## Best Possible Solution

Keep this issue open. The best path is a generic core/plugin contract change: extend the config-write scope and target model with normalized conversation identity, add backward-compatible schema/types for context-scoped configWrites, carry that identity through /config and /allowlist authorization, expose narrow SDK helpers for plugins, add contract tests for inheritance and denial behavior, then update bundled multi-context channel schemas/docs as they adopt it.
