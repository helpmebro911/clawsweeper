## Summary

Current main still does not implement per-context configWrites. The authorization model, command path, schemas, docs, and tests all remain channel/account scoped, and the requested DM/group conversation-level config is not accepted or enforced.

## Best Possible Solution

Keep this issue open. The best path is a generic core/plugin contract change: extend config schema/types for context-scoped write permissions, carry normalized conversation context into config-write authorization, update SDK policy helpers and allowlist/config edit targets, add contract tests for inheritance and denial messages, then update channel docs as each multi-context plugin adopts the contract.
