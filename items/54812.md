## Summary

Keep this issue open. Current main still only supports channel/account-scoped configWrites; it does not expose or enforce per-DM, per-group, per-topic, or generic per-conversation config-write permissions. The existing issue discussion already reached the same conclusion, and the code, schema, SDK helper, tests, and docs still match that channel/account-only contract.

## Best Possible Solution

Keep this issue open. The best implementation path is a generic core/plugin contract change: extend config schema/types for context-scoped write permissions, carry normalized conversation identity into config-write authorization, update SDK policy helpers and allowlist/config edit targets, add contract tests for inheritance and denial behavior, then update channel docs as each multi-context plugin adopts the contract.
