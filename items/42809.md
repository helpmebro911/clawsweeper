## Summary

Keep this PR open. Current main still routes Feishu card-mode table content through markdown-only card elements, so the native Feishu table rendering problem is not implemented or superseded. The PR is a relevant implementation candidate, but it needs a current-main rework and remaining review fixes before it is merge-ready.

## Best Possible Solution

Keep this PR open as a useful Feishu card-table implementation candidate. Ask for a current-main rework that shares parsing with the markdown IR where practical, converts card tables for both markdown and structured card paths, preserves backtick and tilde fenced code blocks, extends quoted-card extraction or stores a markdown fallback for native table elements, coordinates with #43690/#50429/#70651, and adds focused regression tests for card rendering and reply-context extraction.
