## Summary

Keep open. Current `main` still does not implement native Feishu table components for cards, and the submitted PR scope shown in the GitHub context appears incomplete for the main structured-card send paths and quoted-card parsing.

## Evidence

- **Current main still sends markdown cards as one raw markdown element:** `buildMarkdownCard()` on `main` still returns a schema 2.0 card whose body is a single `{ tag: "markdown", content: text }` element; there is no table parsing/conversion in the shipped code.
