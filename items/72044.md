## Summary

Keep open. Current main still fills missing DeepSeek V4 assistant tool-call `reasoning_content` with an empty string, and the DeepSeek tests explicitly lock that behavior. The issue reports that this is insufficient for longer same-provider thinking/tool-call chains, and I did not find a current main implementation that replays the last real DeepSeek reasoning content across those tool-only turns.

## Best Possible Solution

Keep #72044 open and fix the DeepSeek V4 thinking replay path. The likely fix belongs in the OpenAI-compatible transcript conversion or the DeepSeek V4 payload wrapper: preserve and replay actual non-empty `reasoning_content` for same-DeepSeek assistant tool-call turns across multi-turn `sessions_spawn`/tool loops, while keeping cross-provider legacy placeholder behavior separate if it is still needed. Add a regression covering at least three tool-call turns or a subagent path, then update the DeepSeek docs to match the precise behavior.
