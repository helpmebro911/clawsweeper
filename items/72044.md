## Summary

Keep #72044 open. Current main has DeepSeek V4 replay handling, but the remaining code path still backfills missing assistant `reasoning_content` with an empty string, and existing tests explicitly assert that placeholder behavior. The issue is recent, specific, unprotected, and reports a multi-turn/subagent failure not covered by the current short follow-up replay tests.

## Best Possible Solution

Keep the issue open and fix the DeepSeek V4 replay contract in the provider wrapper and/or upstream pi-ai: carry the last non-empty DeepSeek `reasoning_content` into later assistant tool-call replay messages that lack their own thinking block, then add a regression covering multi-turn `sessions_spawn` or at least three consecutive tool-call follow-up turns with thinking enabled. Existing placeholder behavior should remain only for legacy/provider-switch history if live DeepSeek accepts that case.
