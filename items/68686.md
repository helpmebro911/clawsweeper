## Summary

Keep PR #68686 open. Current main still drops the reusable CLI session ID on `auth-profile` and same-version `auth-epoch` mismatches, so the PR's core behavior is not implemented. Main has related improvements for stable OAuth token-refresh epochs and MCP/prompt resume hashing, but those do not cover the remaining auth-only invalidation path this PR targets.

## Best Possible Solution

Keep the PR open and adapt it to current main: preserve the existing CLI `sessionId` for auth-only `auth-profile` and valid same-version `auth-epoch` mismatches, keep system-prompt and MCP resume-hash invalidations as fresh-session resets with higher priority, retain the current auth epoch version upgrade behavior, and add regression tests for auth-only reuse plus simultaneous auth+config changes.
