## Summary

Keep open. Current main still does not expose Slack `unfurl_links` / `unfurl_media` as config fields or pass them through to outbound Slack `chat.postMessage` calls. The issue is a concrete bundled Slack plugin config gap, not a duplicate of the SecretRef discussion linked in the comment.

## Evidence

- **Slack config schema has no unfurl fields:** `SlackAccountSchema` enumerates Slack account/channel config keys such as `streaming`, `mediaMaxMb`, `replyToMode`, `actions`, `responsePrefix`, `ackReaction`, and `typingReaction`, then applies `.strict()`. There are no `unfurlLinks`, `unfurlMedia`, `unfurl_links`, or `unfurl_media` fields to accept the requested config.
