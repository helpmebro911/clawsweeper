## Summary

Keep this issue open. The report is recent, concrete, and current main still shows generic WhatsApp reconnect/watchdog behavior without a group-specific recovery path for the reported state where DMs continue while group upserts stop after repeated 408 reconnects. The related open PR #72621 explicitly excludes the group-reliability work tracked here and in #7433.

## Best Possible Solution

Keep this open as a WhatsApp plugin reliability bug. Add a focused regression harness for repeated 408 close/reopen cycles with both DM and group traffic, then implement the fix in `extensions/whatsapp`: likely group-aware liveness/recovery plus validated Baileys group reliability hooks such as socket-connected group metadata, retry message lookup, and group metadata event handling.
