## Summary

Keep this issue open. The report is fresh and concrete, and current main still supports the reported failure mode: npm installs can warn that the npm global bin directory is missing from the user's original PATH, then still print a successful install message while default post-install verification is disabled.

## Best Possible Solution

Keep the issue open and fix the installer path behavior. The best maintainer direction is to make default installer success contingent on `openclaw` being discoverable for the user's next command, or to avoid unconditional success when the original shell PATH lacks the npm global bin directory. Add focused coverage where npm's global bin is absent from the original PATH and no post-test PATH fixup or entry-file fallback is allowed; update public install troubleshooting only if opening a new shell or sourcing rc files remains an intentional requirement.
