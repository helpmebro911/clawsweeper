## Summary

Keep this issue open. Current main has the Korean Control UI locale and tests, and that UI portion is present in the latest release, but the requested agent-side language persistence is still not implemented in the default agent system prompt. The paired PR #53346 was closed unmerged, so it is not a merged implementation candidate anymore.

## Best Possible Solution

Keep the issue open but narrow the remaining work. Preserve the shipped Korean Control UI locale, then implement and test agent-side primary-language persistence in `src/agents/system-prompt.ts` and related prompt tests. If maintainers want Korean search localization, add explicit config-aware or prompt/tool behavior for providers that support it instead of assuming one global `search_lang` field works everywhere.
