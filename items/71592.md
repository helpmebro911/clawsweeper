## Summary

Keep this issue open. Current main still advertises shared TUI slash commands such as `/status` and `/compact`, but the TUI-local command switch has no local handler for either command and still falls through to `sendMessage(raw)`. In `--local`, that fallback uses `EmbeddedTuiBackend`, which calls `agentCommandFromIngress()` directly rather than the Gateway `chat.send` path that builds a message context and runs shared auto-reply command handlers.

## Best Possible Solution

Fix the TUI local embedded command boundary. Known shared commands advertised in TUI autocomplete, especially `/status` and `/compact`, should either execute through a shared local command adapter or render an explicit local unsupported/system response in `opts.local` mode, but they should not fall through to ordinary model text. Add focused local-mode regression tests for both commands and preserve gateway-connected forwarding behavior.
