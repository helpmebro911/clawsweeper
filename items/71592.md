## Summary

Keep #71592 open. Current main still advertises shared `/status` and `/compact` commands in the TUI slash surface, but `createCommandHandlers().handleCommand()` has no `status` or `compact` cases and its default branch still forwards raw slash text through `sendMessage(raw)`. In `openclaw tui --local`, that send path uses `EmbeddedTuiBackend`, which starts an embedded agent turn via `agentCommandFromIngress`, so the reported local embedded fallthrough remains present. The related items in the issue context are adjacent or older cross-surface reports, not a canonical fix for this local-mode execution bug.

## Best Possible Solution

Keep the issue open and fix the TUI local embedded command boundary. Recognized shared commands advertised by the TUI, especially `/status` and `/compact`, should either execute through a shared local command adapter or return an explicit local unsupported/system response in `opts.local` mode, but must not fall through to `sendMessage(raw)`. Add focused regression tests proving local `/status` and `/compact` do not reach `sendChat` / `agentCommandFromIngress`, while preserving gateway-connected forwarding for commands intentionally handled by `chat.send`.
