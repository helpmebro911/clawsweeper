## Summary

Keep #71592 open. Current main still advertises shared `/status` and `/compact` in TUI slash command discovery, but the TUI command switch has no `status` or `compact` cases. Its default path adds the raw slash text as a user message and calls `sendChat`; in `--local`, that client is `EmbeddedTuiBackend`, which calls `agentCommandFromIngress` directly rather than the gateway `chat.send` command-dispatch path. The supplied related items remain distinct and do not supersede this local embedded TUI execution bug.

## Best Possible Solution

Keep the issue open and fix the TUI local embedded command boundary. Recognized shared commands such as `/status` and `/compact` should either execute through a shared local command adapter or return an explicit local unsupported/system response, but they should not fall through to `sendMessage(raw)` in `opts.local` mode. Preserve gateway-connected forwarding for commands intentionally handled by `chat.send`, and add regression tests proving local `/status` and `/compact` do not reach `sendChat` or `agentCommandFromIngress`.
