## Summary

Keep open. Current main has sandboxed execution with per-session scope and a separate sandbox workspace, but it does not implement the requested per-run ephemeral/tmpfs workspace option. Sandbox workspaces are created under persistent state, reused by scope, and cleaned only by explicit recreate or idle/max-age pruning. This is core sandbox lifecycle/API behavior rather than a ClawHub-only plugin request.
