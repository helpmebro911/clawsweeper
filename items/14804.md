## Summary

Keep #14804 open. Current main still lacks a first-class ephemeral or tmpfs-backed workspace mode for isolated sessions. The existing sandbox system has per-scope workspaces, Docker tmpfs mounts for temporary container paths, manual recreate, and idle/age pruning, but not a documented per-run disposable filesystem lifecycle or cleanup-on-completion option.

## Best Possible Solution

Keep #14804 open until core sandbox lifecycle/config adds a documented disposable workspace mode. A strong fix should add explicit AgentSandboxConfig and schema fields, create a fresh per-run workspace where supported, clean it on completion, define Docker/SSH/OpenShell behavior including whether tmpfs is required or best-effort, update sandbox lifecycle tests, and update public sandbox/config docs.
