## Summary

Keep #68618 open. Current main has the Feishu sender-name lookup toggle and contact-API resolution path, but it does not implement a native sender alias map or fallback mapping for `open_id` values. The request is recent, unprotected by labels/author association, and overlaps a still-open Feishu permission-scope problem rather than being obsolete.

## Best Possible Solution

Keep the issue open and implement this in the Feishu plugin rather than OpenClaw core: add a typed `senderNameMap` or similarly named top-level/account-level Feishu config field, apply it before or after the Feishu API lookup according to documented precedence, cover permission-failure fallback and override behavior in Feishu tests, and update the Feishu channel docs/config reference.
