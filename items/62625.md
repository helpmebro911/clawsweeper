## Summary

Keep #62625 open. Current main at dcfd5913fd24a7fce8fe804bf687f4d4856a6652 still has the clean-start/missed-event gap described: session sync is event/dirty-state gated, no startup reconciliation marks changed session files dirty, and non-force CLI/session-start paths still skip session indexing when the in-process dirty set is empty.

## Evidence

- **Session-start/watch gate still skips clean incremental session sync:** `shouldSyncSessionsForReindex` returns false for `session-start` and `watch` before falling through to `sessionsDirty && dirtySessionFileCount > 0`, so a restarted process with no in-memory dirty files will not sync sessions.
  - file: [extensions/memory-core/src/memory/manager-session-reindex.ts:24](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/extensions/memory-core/src/memory/manager-session-reindex.ts#L24)
  - command: `nl -ba extensions/memory-core/src/memory/manager-session-reindex.ts | sed -n '1,240p'`
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Manager startup wires listeners but no session catch-up scan:** The non-status constructor branch only calls `ensureWatcher()`, `ensureSessionListener()`, and `ensureIntervalSync()`. The subsequent initial dirty calculation is for the `memory` source, not a durable comparison of `sessions/` files against indexed rows.
  - file: [extensions/memory-core/src/memory/manager.ts:224](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/extensions/memory-core/src/memory/manager.ts#L224)
  - command: `nl -ba extensions/memory-core/src/memory/manager.ts | sed -n '1,360p'`
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Session dirty state still depends on the in-process transcript event path:** `ensureSessionListener` subscribes to `onSessionTranscriptUpdate`; only processed pending events add entries to `sessionsDirtyFiles` and set `sessionsDirty = true`. Missed events or events from a prior gateway process are not reconstructed here.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:427](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/extensions/memory-core/src/memory/manager-sync-ops.ts#L427)
  - command: `nl -ba extensions/memory-core/src/memory/manager-sync-ops.ts | sed -n '370,760p'`
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Manual non-force CLI indexing does not force session reconciliation:** `openclaw memory index` calls `sync` with `reason: "cli"` and `force: Boolean(opts.force)`. With `--force` absent and no in-process session dirty state, the reindex gate remains false for sessions.
  - file: [extensions/memory-core/src/cli.runtime.ts:1110](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/extensions/memory-core/src/cli.runtime.ts#L1110)
