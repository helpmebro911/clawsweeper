## Summary

Keep #71452 open. Current main still has the exact generic 25-row formatter cap reported by the issue, and the human-readable `message read`, `message pins`, and Discord search output paths still flow through that capped formatter. Teams has partial provider-side pagination for pinned messages and channel lists, but that does not solve the shared formatter/action-contract problem.

## Best Possible Solution

Keep this open and implement a narrow message formatting/action-contract fix: pass the requested limit or pagination metadata into `renderMessageList`, remove or make configurable the hardcoded 25-row cap, ensure numeric CLI/tool `limit` values are normalized consistently, and add regression tests for more than 25 `read`, `list-pins`, and search rows while preserving provider-specific Teams pagination coverage.
