## Summary

Keep #71452 open. Current main at 5c4c33c still contains the exact generic 25-row formatter cap, and the CLI text paths for read, list-pins, and search still route through that formatter. Some provider-side pagination/limit work exists, especially for Discord reads and Teams pins/channel lists, but it does not solve the user-visible hard cap in non-JSON message output.

## Best Possible Solution

Keep the issue open and implement a narrow formatter/action-contract fix: pass the requested limit or pagination metadata into message-format rendering, remove or make configurable the hardcoded 25-row cap, add regression tests for more than 25 read/list-pins/search rows, and preserve existing provider-side pagination behavior for Discord and Teams.
