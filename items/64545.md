## Summary

Keep this issue open. Current main still derives the Mattermost interaction signing secret from the bot token using the public hardcoded HMAC key, and the same-author PR #64546 references this issue with closing syntax but remains open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open as the Mattermost security tracker. Review and land PR #64546 or a successor that adds a persistent per-account interaction signing secret separate from the bot token, wires it through the Mattermost account/config/credential contract, updates docs and tests, and closes this only after the reviewed fix merges and ships.
