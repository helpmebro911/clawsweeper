## Summary

Keep open. Current main still has the core Moonshot/Kimi 429 billing-misclassification path: generic 429 handling wins over billing evidence, Moonshot has no provider-owned classifier override, and adjacent tests do not cover a Moonshot 429 insufficient-balance payload.

## Best Possible Solution

Keep this open. Add a regression test for a Moonshot/Kimi HTTP 429 payload whose code or message clearly indicates insufficient balance/credits, then change failover and user-facing formatting precedence so billing evidence, or a Moonshot provider-owned classifyFailoverReason hook, can override the broad 429 rate-limit rule. Verify the surfaced message is billing-specific and auth-profile state uses billing disabled/backoff rather than short rate-limit cooldown.
