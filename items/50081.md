## Summary

Keep open. Current main still has the reported current-run path: webchat/RPC attachments are parsed into inline prompt images, HEIC/HEIF can remain `image/heic` or `image/heif`, and the embedded runner returns `existingImages` unchanged when the prompt has no textual image refs before passing them to `activeSession.prompt`. The linked PR #50320 is useful related work and explicitly claims to fix #50081, but it is open/unmerged in the provided context and its head commit is not present in this checkout.
