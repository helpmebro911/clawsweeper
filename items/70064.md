## Summary

Keep open. Current `main` shows the Telegram audio path wiring transcripts into `BodyForAgent` and the pre-agent hook pipeline, but there is no current-main fix or regression test that specifically addresses the reported downstream stale/incorrect-reply behavior after `message:preprocessed`. The issue is also only a few days old, so there is not enough evidence for a conservative close.

## Evidence

## Risks / Open Questions

- The report may still reflect a real end-to-end Telegram bug; this review did not run live Telegram reproduction traffic.
- Direct Telegram conversations are intentionally stateful, so some prior-turn influence is expected; a maintainer reproducing this should distinguish normal session memory from true stale-turn reuse.
- `gh` inspection was blocked by missing `GH_TOKEN`, but the provided issue context had no comments or timeline, so there was no extra maintainer discussion to resolve the report either way.
