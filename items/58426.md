## Summary

Current main still has the CLI bug this PR targets: unversioned `clawhub:` and bare-name ClawHub installs are passed to ClawHub unpinned, but the persisted install record is rewritten with the resolved version. Because `plugins update` reuses that recorded spec, the PR remains a real core CLI fix rather than external ClawHub/plugin-discovery work.

## Best Possible Solution

Keep PR #58426 open and finish it as a focused core CLI bug fix: persist `raw` for direct `clawhub:` installs, persist `preferredClawHubSpec` for bare-name ClawHub fallback, keep explicit `clawhub:pkg@version` installs pinned, remove stale test mock code, drop or separately justify the unrelated legacy-config guard change, then run the targeted CLI/update tests and build in a writable validation pass.
