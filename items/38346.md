## Summary

Keep #38346 open. Current main still has no Telegram mode that observes or ingests every allowed-group message while separately restricting agent-triggering/replies to allowlisted senders. The existing policy remains binary: `groupPolicy: "open"` lets any sender through, while `groupPolicy: "allowlist"` drops unauthorized senders before message context/history/ingest handling.

## Best Possible Solution

Keep this open for bundled Telegram plugin work. Add an explicit observe/ingest path for configured groups/topics, then apply a separate sender trigger allowlist before agent turns, command handling, and replies. Update Telegram config types/schema/docs and add regression tests for unauthorized unmentioned ingest without reply, unauthorized mentioned no reply, and allowlisted mentioned reply.

## Risks / Open Questions

- Telegram Bot API privacy mode may prevent receiving all group messages unless privacy is disabled or the bot is made a group admin; docs already note this transport prerequisite.
- Observe-only ingestion of unauthorized group traffic has privacy and retention implications and should be explicit opt-in with tests proving those messages never start agent turns or replies.
