## Summary

Keep open. Current main still does not expose optional TUI chat message timestamps: TUI message components are created from text only, history timestamps are used only for pending-message reconciliation, and there is no TUI config/CLI/slash-command surface for a timestamp toggle. The directly related PR #49069 remains open, and its referenced commits are not ancestors of current main.
