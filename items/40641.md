## Summary

Current main still contains the reported unquoted bracket-notation config examples in the exec, config, and nodes docs. The CLI/parser side still accepts bracket notation once the argument reaches OpenClaw, so the remaining bug is a real documentation/shell-quoting regression. Related PR #40642 proposed the right docs fix but was closed unmerged.

## Best Possible Solution

Keep this issue open as a concrete docs regression. Update the English source docs, especially docs/cli/config.md, docs/tools/exec.md, and docs/nodes/index.md, to quote every `openclaw config` example using bracket notation or switch those examples to shell-safe dot-index paths such as `agents.list.0.tools.exec.node`; add a short note in the config path docs that bracket paths containing `[]` must be quoted in shells that treat brackets as glob patterns. Let localized docs flow through the normal docs/i18n publishing path rather than patching generated locale output directly.
