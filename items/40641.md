## Summary

Current main still documents the affected unquoted bracket-notation config paths, including the exact per-agent exec node command from the report. The CLI supports bracket paths once the shell passes argv through, so the remaining actionable problem is the docs regression, and the related docs PR #40642 was closed unmerged.

## Best Possible Solution

Keep this issue open as a concrete docs regression. Quote every config CLI example that uses bracket notation, or switch the examples to dot-index form such as `agents.list.0.tools.exec.node`, and add a short note in the config docs explaining that paths containing `[]` should be quoted in shells like zsh. The likely source files are `docs/cli/config.md`, `docs/tools/exec.md`, and `docs/nodes/index.md`; localized pages should be regenerated through the docs pipeline.
