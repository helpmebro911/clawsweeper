## Summary

Keep this issue open. Current main has partial local transcript-token estimation for compaction/forking paths, but it still does not provide the requested shared estimated context snapshot for MiniMax-style missing/all-zero usage in `/status`, `/usage`, session rows, or context notices. The related reports in the provided context cover adjacent usage dashboards, local pruning, MiniMax auth/usage quota, and context-window discovery, but none is a canonical open/merged fix for this specific fallback.

## Best Possible Solution

Implement a shared estimated context snapshot path for sessions whose provider usage is missing or all-zero. Reuse `readSessionMessages` plus `estimateMessagesTokens` or the existing prompt estimator, add a source/estimated flag to session store or Gateway session rows/events, render the estimate clearly in `/status`, `/usage`, and context notices, and let compaction thresholds consume the estimate without overwriting fresh provider snapshots. Add regression coverage for MiniMax-style all-zero usage and transcripts that contain messages but no assistant usage metadata.

## Risks / Open Questions

- Character/token estimates will miss provider tokenization, hidden prompts, tool schema overhead, and formatting overhead, so any compaction threshold using them should keep conservative headroom.
