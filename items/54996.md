## Summary

Keep this issue open. Current main has partial transcript-message token estimation for compaction preflight and session forking, but it still does not implement the requested estimated context snapshot for missing/all-zero provider usage in `/status`, `/usage`, or the session usage row surfaces.

## Best Possible Solution

Implement a shared estimated context snapshot path for sessions with missing or all-zero provider usage. Reuse `readSessionMessages` plus `estimateMessagesTokens` or the existing character/token estimator, carry an explicit estimated source through session rows, render it in `/status` and `/usage`, and feed compaction thresholds without overwriting fresh provider snapshots. Add regression coverage for MiniMax-style all-zero usage and transcripts with messages but no assistant usage metadata.
