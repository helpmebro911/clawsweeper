## Summary

Keep #11473 open. Current main still has no token-free `staticMessage` cron payload: the cron type contract, Gateway schema, runtime validation/execution, CLI, Control UI, agent cron tool, tests, and docs are still built around `systemEvent` and `agentTurn`. The existing maintainer/Codex discussion also explicitly kept this open, and related #18160 is broader direct-exec work that remains open rather than a shipped superseder.

## Best Possible Solution

Keep #11473 open until OpenClaw either implements an additive token-free static message cron payload or explicitly supersedes it with a broader non-agent cron payload design that covers fixed text-to-channel delivery. A complete implementation should update `CronPayload`, Gateway TypeBox schemas, normalization/validation, runtime direct outbound delivery with run-log/audit/failure reporting, CLI and Control UI create/edit flows, the agent cron tool surface, docs, and focused tests. Cross-reference #18160 for broader direct-exec design, but do not close this narrower request yet.
