## Summary

Keep #11473 open. Current main still has no token-free static cron message payload: cron payloads are limited to `systemEvent` and `agentTurn`, Gateway/API schemas do not accept `staticMessage`, CLI/UI creation paths only build those two variants, and runtime execution still routes main jobs through heartbeat/system-event processing or detached jobs through isolated agent execution. The provided maintainer discussion also explicitly says to keep this issue open.

## Best Possible Solution

Keep #11473 open until maintainers either implement a first-class no-AI static channel-send cron payload or explicitly fold static text delivery into a broader non-agent cron payload design. The likely implementation should update the core cron type/normalization/validation contract, Gateway protocol schema, CLI and Control UI creation/editing surfaces, docs, and tests for direct channel delivery, failure reporting, and token-free execution.
