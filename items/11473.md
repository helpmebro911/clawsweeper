## Summary

Keep #11473 open. Current main still does not implement a token-free static cron message payload: cron payload contracts, Gateway validation, runtime execution, CLI, Control UI, macOS models, docs, and tests are still centered on `systemEvent` and `agentTurn`. Related #18160 is broader direct-exec cron work and remains open rather than a shipped superseder.

## Best Possible Solution

Keep #11473 open until OpenClaw either implements an additive token-free static message cron payload or explicitly supersedes it with a broader non-agent cron payload design that covers fixed text-to-channel delivery. A complete implementation should update `CronPayload`, Gateway TypeBox schemas, normalization/validation, runtime direct outbound delivery with run-log/audit/failure reporting, CLI, Control UI, macOS models, the agent cron tool surface, public docs, and focused tests. Cross-reference open #18160 for broader direct-exec design, but do not close this narrower static-message request yet.
