## Summary

Keep #9658 open. Current main still does not provide a streamed, file-backed, or payload-reference mechanism for the affected core tool arguments; exec, process write/paste, write, apply_patch, and direct HTTP tool invocation continue to take inline JSON/string arguments. The existing maintainer-thread Codex comment also explicitly kept the item open for the same core product/API gap.

## Best Possible Solution

Keep the issue open for maintainer product/API design. The likely path is a backward-compatible payload-reference or streamed/file-backed argument mechanism for exec command/stdin, process write/paste data, write content, and apply_patch input, plus documented limits/workarounds and regression coverage for large and special-character payloads. If maintainers reject the feature, document the explicit limits and recommended chunking/file workflows instead.
