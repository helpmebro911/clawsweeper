## Summary

Keep #9658 open. Current main still exposes the affected core tool payloads as inline string/object arguments and has no streamed, file-backed, or payload-reference mechanism for exec, process stdin/paste, write, or apply_patch. The only issue comment is also a maintainer-thread Codex review saying to keep this open for the same product/API gap.

## Best Possible Solution

Keep the issue open for maintainer product/API design. A complete solution should add a backward-compatible streamed or file-backed payload/reference mechanism for exec command/stdin, process write/paste data/text, write content, and apply_patch input; document supported limits and safe workarounds; and add regression coverage for large and special-character payloads. If maintainers reject the feature, document the explicit limits and recommended chunking/file workflows instead.
