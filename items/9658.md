## Summary

Keep #9658 open. Current main still does not provide a streaming or file-backed argument path for the affected core tools: exec takes one JSON string command, process stdin/paste uses inline string fields, write requires inline content, and apply_patch takes one inline string input. The newer provider-side streaming/repair code is bounded JSON accumulation/repair, not a transport that bypasses tool-call JSON serialization. The existing maintainer/Codex comment on the issue also explicitly kept this open, and the mentioned #9000 is a closed memory_search/qmd bug rather than a canonical tracker for this feature.

## Best Possible Solution

Keep the issue open for maintainer product/API design. The likely path is a backwards-compatible core mechanism for large tool payloads, such as file-backed or streamed stdin/content handles for exec, process write/paste, write, and possibly apply_patch, with updated schemas, docs, and regression tests for large/special-character payloads. If maintainers choose not to add such a transport, document the supported limits and safer workarounds explicitly.
