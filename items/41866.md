## Summary

Keep this PR open. Current main still does not implement the specific leading `to=cron ... {json}` user-facing sanitizer the PR targets, and the linked bug describes a real user-facing leak plus a deeper failed tool-execution path. The PR remains potentially useful, but it needs rebase/adaptation to the current sanitizer module and tighter matching before it is merge-ready. Security review found no supply-chain, CI, dependency, lockfile, or permission changes in the provided diff; it only touches agent sanitizer code and its regression test.

## Best Possible Solution

Rebase the PR onto current main and move the implementation into `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` or the shared assistant-visible sanitizer. Tighten detection to a conservative Codex-style leading `to=<registered tool> ... {args}` shape, add regression tests for both the Discord `to=cron` leak and start-of-message prose false positives, and keep separate handling for the underlying case where the provider emits tool calls as text instead of structured invocations.
