## Summary

Keep open. Current main does not implement opt-in MCPS/message-level signing for OpenClaw's MCP stdio client path: stdio messages are still serialized and written as plain JSON-RPC, the resolved stdio server config only carries command/args/env/cwd, and the public MCP config type has no signing/passport/key fields. The reporter's linked razashariff/mcps-openclaw package is useful evidence of a possible implementation path, but this is security/core transport hardening and needs maintainer product/security judgment rather than conservative cleanup closure.
