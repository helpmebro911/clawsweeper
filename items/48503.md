## Summary

Keep open. Current main still exposes `before_tool_call` with only tool name, params, run id, and tool call id in the event; action classification and input provenance are not surfaced to hook handlers. The repo does contain input provenance tracking and now supports plugin approval decisions, but those do not implement this issue’s requested hook enrichment. Related items cover adjacent pieces, not a clear canonical replacement for this exact API gap.
