## Summary

Keep this PR open. Current main already has baseline bundled NVIDIA API-key provider support, but it still lacks the PR's dynamic NIM `/models` discovery path, expanded NVIDIA model catalog, multimodal/reasoning metadata, and docs cleanup. The PR is therefore not obsolete, but it also should not merge as-is because the current diff still has integration issues around single-provider auth shape and NVIDIA model-id normalization, plus the new credentialed discovery path needs focused tests.

## Best Possible Solution

Keep this PR open and request changes. Rebase on current main, remove or align the already-shipped baseline auth wiring, use the single-provider auth option shape, keep the useful dynamic discovery/catalog/docs work, normalize NVIDIA model ids consistently for parsed and catalog ids, add focused tests for discovery fallback and multimodal metadata, and verify the credentialed `/models` fetch remains bounded and does not persist secrets.

## Risks / Open Questions

- Closing this PR as implemented would discard still-unshipped NVIDIA dynamic discovery, expanded catalog, multimodal metadata, and docs cleanup work.
- Merging the current PR as-is risks provider registration/type breakage because the single-provider helper expects auth option objects, not built `ProviderAuthMethod` objects.
- The current PR dynamic model lookup can miss catalog entries after normal model-ref parsing strips the provider prefix, which can drop metadata or send a bare model id.
- The new credentialed `/models` discovery path needs tests for fallback behavior, timeout handling, metadata preservation, and secret non-persistence before maintainer merge review.
