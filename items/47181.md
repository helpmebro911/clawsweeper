## Summary

Keep PR #47181 open. Current main has adjacent custom-provider and Microsoft Foundry work, but it still does not implement the PR's Azure Claude onboarding path, `anthropic-azure-api-key` auth choice, Azure Claude env vars, or Anthropic `/anthropic` routing. The PR likely needs redesign/rebase into the current plugin-owned provider auth architecture, but it is not obsolete enough for conservative cleanup closure.

## Best Possible Solution

Keep the PR open but request a rebase/redesign against current main: implement Azure Claude onboarding through the plugin-owned provider auth surface, validate `.services.ai.azure.com/anthropic` endpoints and resource names, preserve SecretRef/API-key storage behavior, add focused tests for interactive and non-interactive setup, and update user-facing provider docs. If maintainers decide Azure Claude should be separate from bundled Foundry, split it into an appropriate provider plugin rather than core onboarding flags.
