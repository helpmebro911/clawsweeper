## Summary

Current `main` still fails closed on unknown config keys during config load. The tolerant behavior requested in #40268 is not implemented on the read path; unknown-key stripping exists only in the doctor repair flow and docs still describe validation failure until removal.

## Evidence

- **Raw config validation still uses the strict schema result directly:** `validateConfigObjectRaw` calls `OpenClawSchema.safeParse(raw)` and returns schema issues on failure; there is no read-path `.strip()` or unknown-key filtering in this validator.
