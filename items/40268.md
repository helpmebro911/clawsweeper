## Summary

Current main still fails closed on unknown config keys during normal config reads. Last-known-good recovery and doctor repair paths exist, and #42201 is an open implementation candidate, but the requested tolerant read-path strip-and-warn behavior has not landed on main.

## Best Possible Solution

Keep this issue open. The best path is to review, harden, and either land or close #42201; if landed, close this as implemented with tests proving unknown-key startup/hot-reload tolerance while strict write/config-validate behavior remains intact. If #42201 is abandoned, implement the same behavior in the config read path with explicit regression coverage for root unknown keys, nested misplaced keys, invalid known values, legacy keys, plugin-local invalidity, last-known-good recovery, and doctor repair.
