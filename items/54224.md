## Summary

Keep this issue open. Current main covers Slack channel history and thread reads through the `read`/`readMessages` path with user-token read preference, but Slack message search is still absent from discovery, dispatch, runtime exports, implementation, CLI docs, and the CLI search command remains Discord-only.

## Best Possible Solution

Keep this issue open, but narrow it to the missing Slack message search capability. The maintainer path is to add a Slack-owned `search` message action backed by the user token and `search:read`, advertise it only when usable credentials exist, preserve `userTokenReadOnly=true` read-only semantics, make `openclaw message search` work for Slack without Discord-only `--guild-id`, and cover Slack action discovery, dispatch/runtime behavior, CLI, and docs. History and thread reads can remain on the current `read`/`readMessages` implementation.
