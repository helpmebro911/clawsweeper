## Summary

Keep #69297 open. Current main does not implement the requested WhatsApp per-group admin privacy control: the config type/schema do not accept `channels.whatsapp.groups.*.admin`, the inbound policy and group ingress paths have no configured-admin exception, and docs still describe group allowlist behavior without an admin override. The PR discussion is relevant and positive: bot review found concrete edge cases, and the author pushed follow-up fixes for schema support, wildcard fallback, sender-aware activation, and allowlist ingress handling.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to land a focused WhatsApp extension change that adds `groups.<jid>.admin` to the config contract/schema/generated metadata/docs, applies the admin exception consistently in mention gating, command authorization, and real group allowlist ingress, preserves `groupPolicy: "disabled"`, covers wildcard and exact-group fallback semantics, and either removes or separates the unrelated agent compaction test stabilization changes.
