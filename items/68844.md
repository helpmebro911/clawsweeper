## Summary

Keep open. Current main still has the newline-mode behavior this PR is trying to fix: blank-line-separated paragraphs are emitted one paragraph per chunk instead of being packed up to the limit, and the existing tests still encode that oversplitting behavior.

## Best Possible Solution

Keep #68844 open and review it as a focused bug fix. The best path is to land or rework the paragraph-packing change in `src/auto-reply/chunk.ts`, update the current chunking and outbound delivery regressions to expect packed short paragraphs, and verify the markdown fence split guardrail still holds.
