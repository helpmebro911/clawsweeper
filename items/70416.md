## Summary

Keep open. Current main appears to have fixed the missing/nonzero Opus 4.7 catalog entry through the pinned pi-ai dependency, so the original near-zero five-order symptom is partially obsolete. But the issue's long-retention cost requirement is still not implemented: OpenClaw emits Anthropic 1h cache markers, then collapses all cache creation tokens into one cacheWrite count and prices it with one flat cacheWrite rate instead of the provider's 5m vs 1h breakdown. The linked #70417/#70418 discussion covers cache divergence and warming, not a canonical replacement for this pricing fix.

## Best Possible Solution

Keep the issue open, narrow it to the remaining pricing bug, and implement a focused Anthropic cost-accounting fix: preserve/read the provider's cache_creation 5m and 1h token breakdown, compute cost.cacheWrite with separate 1.25x and 2x input multipliers for claude-opus-4-7 and other Anthropic models, keep usage-cost aggregation consistent with transcript costs, and add a regression test for cacheRetention: "long" on claude-opus-4-7.
