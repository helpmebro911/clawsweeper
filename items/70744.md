## Summary

Keep open. Current main has partial recovery paths for context overflow and Telegram dispatch failures, but the exact reported failure mode is not fully covered: explicit overflow compaction can still hold the message handler for the default 900-second compaction timeout before any reset/error path runs, and I found no targeted Telegram direct-session regression proving that a hung auto-compaction is promptly surfaced or auto-reset.

## Best Possible Solution

Keep the issue open and implement a targeted recovery path for channel sessions whose overflow compaction does not complete promptly: shorten or channel-scope the compaction timeout, emit a visible Telegram/Discord-safe failure before the typing window expires, and automatically rotate/quarantine the poisoned session after repeated overflow/compaction failure. Add an end-to-end or focused integration regression for a Telegram direct session where compaction hangs or times out, proving that the user sees a recovery message and the next turn uses a fresh session.
