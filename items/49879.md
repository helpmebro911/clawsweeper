## Summary

Keep open. Current main does not implement a global `agents.defaults.responseUsage` default: the agent defaults schema/types omit the key, `/usage` still persists only to the session entry, and reply footer rendering only reads `responseUsage` from the active session store. The provided related items are adjacent `agents.defaults.*` requests or config-validation reports, not a canonical duplicate for this usage-footer default.
