## Summary

Keep open. Current `main` still gates wiki recall on contiguous `includes(queryLower)` matches in both the digest candidate pass and the full page scorer, so the reported long multi-term query failure remains plausible. The `memory_search(corpus=wiki)` path also does not forward `minScore` into the wiki supplement search, so lowering `minScore` cannot recover candidates there. I did not find regression coverage for multi-term/token-overlap wiki queries, and the same query code is still present in the latest release tag `v2026.4.23`.

## Evidence

- **Digest candidate gating is still full-string only:** `buildDigestCandidatePaths` lowercases the full query and drops pages unless `buildDigestPageSearchText(...).includes(queryLower)` is true. That preserves the exact choke point described in the issue.
