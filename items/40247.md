## Summary

Keep open. Current `main` still lacks the PR’s behavior: non-loopback gateway binds listen only on the configured address, while internal gateway clients still default to loopback, so the reported `tailnet/custom` breakage remains plausible on supported configs.

## Evidence

- **Non-loopback listen hosts are still passthrough-only:** `resolveGatewayListenHosts()` returns `[bindHost]` for every bind host except `127.0.0.1`; there is no branch that adds `127.0.0.1` for `tailnet` or custom IP binds.
  - file: [src/gateway/net.ts:398](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/gateway/net.ts#L398)
