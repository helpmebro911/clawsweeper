## Summary

Keep open. Current main still retains and evaluates the full per-session toolCallHistory across heartbeat intervals, and heartbeats still use the main session by default unless isolatedSession is enabled. The hook checks for loops before recording the next call, so the fix needs to prune or ignore stale history before detection, not only while appending. Related PRs #40217 and #40798 are open/unmerged in the provided context, while earlier attempts were closed unmerged.

## Best Possible Solution

Keep #40144 open until a focused mainline fix lands. The likely fix belongs around src/agents/tool-loop-detection.ts and the before-tool-call path: prune or ignore toolCallHistory entries older than a conservative inactivity gap before detectToolCallLoop evaluates the current call, and keep recordToolCall/recordToolCallOutcome consistent with that pruning. Add regression coverage that simulates repeated heartbeat cycles separated by the gap, plus a control test proving continuous in-turn repeats still warn or block.
