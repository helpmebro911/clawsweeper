## Summary

Keep open. Current `main` still models Discord/Slack channels and threads as separate session keys, and the session config/schema does not expose any selective cross-channel grouping feature like the requested `session.channelGroups`. Existing session-sharing knobs only cover DMs (`dmScope`) and linked identities (`identityLinks`), not shared channel/thread sessions.

## Risks / Open Questions

- `session.scope: "global"` and `session.identityLinks` could be mistaken for a fix, but the docs/config describe them for shared channel participants and linked DM identities, not selective cross-channel/thread session merging.
- The issue also raises sandbox/trust-boundary concerns for trusted vs untrusted channels; current docs still frame non-DM/group sessions as non-main isolated sessions, so closing as obsolete would be misleading.
