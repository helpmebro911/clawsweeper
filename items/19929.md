## Summary

Keep #19929 open. Current main at 4a3030df still lacks selective shared-session routing for chosen Discord/Slack group, channel, or thread chats. Non-direct peers still resolve to isolated channel/group keys, route resolution only threads DM scoping and identity links into session-key construction, and strict config/types/docs expose no `channelGroups`, `groupScope`, `includeThreads`, or equivalent grouping option. The related PR #20078 is closed and unmerged, while #7524 is an open but narrower group-to-main request rather than a replacement for selective cross-channel/thread grouping.

## Best Possible Solution

Keep #19929 open as the concrete tracker for selective shared sessions across chosen group/channel/thread chats. A good implementation belongs in core routing/config: add a backward-compatible grouping config, thread it through `buildAgentSessionKey`/`resolveAgentRoute` and channel thread/outbound paths, update strict schemas/types/help/docs/generated metadata, and add regression tests for grouped routing, ungrouped compatibility, reply delivery, queue serialization, thread inheritance, and sandbox/trust-boundary behavior. Reuse any useful design from closed PR #20078 only after re-reviewing it against current main.
