## Summary

Keep open. Current main still does not expose `xhigh` for the specific requested `openai-codex/gpt-5.2` model. The OpenAI Codex provider has moved to provider-owned thinking profiles, but its xhigh list includes newer/base models and codex-suffixed 5.2, not base `gpt-5.2`; the matching helper will not make `gpt-5.2` match `gpt-5.2-codex`. Related PRs #6013 and #43054 proposed this exact fix but were closed unmerged, so there is no shipped or canonical superseding item to close against.

## Best Possible Solution

Keep #6820 open until the OpenAI Codex provider explicitly adds `gpt-5.2` to its xhigh thinking profile, with focused regression coverage for `openai-codex/gpt-5.2`, or until maintainers decide and document that base Codex GPT-5.2 should remain high-only.
