## Summary

Current main still does not expose `xhigh` for `openai-codex/gpt-5.2`. The OpenAI Codex provider recognizes base `gpt-5.2` as a modern model, but the xhigh allowlist omits it, the prefix helper does not make `gpt-5.2` match `gpt-5.2-codex`, and current tests only cover nearby OpenAI Codex 5.4/5.3 cases. The related fix PRs were closed unmerged, while RFC #37804 remains a broader design discussion rather than a canonical replacement for this concrete gap.

## Best Possible Solution

Keep #6820 open until the OpenAI Codex provider either adds base `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage for `listThinkingLevels("openai-codex", "gpt-5.2")` or `resolveThinkingProfile`, or maintainers explicitly document that base OpenAI Codex GPT-5.2 is high-only while `gpt-5.2-codex` is xhigh-capable. Keep RFC #37804 as the broader capability-drift design thread, not as a replacement for this concrete missing model case.
