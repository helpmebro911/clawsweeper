## Summary

Current main still does not satisfy #6820. The OpenAI Codex provider recognizes base `gpt-5.2` as a modern model, but its `OPENAI_CODEX_XHIGH_MODEL_IDS` list omits `gpt-5.2`, and `resolveThinkingProfile` only exposes `xhigh` when the model matches that list. The related one-off fix PRs were closed unmerged, while RFC #37804 is broader design work rather than a shipped replacement for this concrete model gap.

## Best Possible Solution

Keep #6820 open until the OpenAI Codex provider either adds base `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage for `listThinkingLevels("openai-codex", "gpt-5.2")` or `resolveThinkingProfile`, or maintainers explicitly document that base OpenAI Codex GPT-5.2 is high-only while `gpt-5.2-codex` is xhigh-capable. RFC #37804 should remain the broader capability-drift discussion, not a replacement for this concrete missing model case.
