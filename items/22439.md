## Summary

Keep PR #22439 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still has binary session-based bootstrap filtering plus separate lightweight context controls, but it does not implement schema-backed `agents.defaults.bootstrapTier`, `BootstrapTier`, `resolveBootstrapTier`, or `minimal`/`standard`/`full` filtering. The PR also has active same-day author follow-up and a maintainer-side Codex comment explicitly keeping it open with canonical issue #22438.

## Best Possible Solution

Keep #22439 open for maintainer product review alongside #22438. If maintainers accept the direction, land a current-main implementation that adds schema-backed `agents.defaults.bootstrapTier`, a tier resolver, tier-aware bootstrap filtering, wiring through `resolveBootstrapFilesForRun` and the `bootstrap-extra-files` hook, generated config/help/docs updates, and focused tests for default behavior plus `minimal`, `standard`, and `full`. If maintainers prefer the existing `lightContext`/`contextInjection` design instead, close it manually after that explicit product decision.
