## Summary

Keep PR #22439 open. Current main still has binary session-based bootstrap filtering and does not implement schema-backed `agents.defaults.bootstrapTier`, `BootstrapTier`, `resolveBootstrapTier`, or `minimal`/`standard`/`full` tier behavior. The linked feature issue #22438 is open, the PR has active same-day author follow-up, and the discussion includes a maintainer-side Codex comment explicitly keeping it open for product review.

## Best Possible Solution

Keep #22439 open for maintainer product review alongside #22438. If maintainers accept the direction, land a current-main implementation that adds schema-backed `agents.defaults.bootstrapTier`, a tier resolver, tier-aware bootstrap filtering, wiring through `resolveBootstrapFilesForRun` and the `bootstrap-extra-files` hook, generated config/help/docs updates, and focused tests for default behavior plus `minimal`, `standard`, and `full`. If maintainers prefer the existing `contextInjection`/`lightContext` model instead, close it manually after that explicit product decision.
