## Summary

Current main already provides no-key, terminal-style `web_search` through bundled key-free providers plus setup and runtime fallback support, so this feature request is satisfied.

## Evidence

- **`web_search` is already the lightweight terminal-side path:** The web-search docs describe `web_search` as a lightweight HTTP tool, not browser automation, matching the request for terminal-based search rather than a browser-driven flow.
  - file: [docs/tools/web.md:12](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/tools/web.md#L12)
  - sha: [7e16e3d07723](https://github.com/openclaw/openclaw/commit/7e16e3d07723cc3903f40ce94f53a3920c88b1fc)
- **Key-free providers are documented on main:** The current docs list DuckDuckGo, Ollama Web Search, and SearXNG as key-free fallbacks, explicitly removing the need for a Brave API key.
  - file: [docs/tools/web.md:178](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/tools/web.md#L178)
  - sha: [7e16e3d07723](https://github.com/openclaw/openclaw/commit/7e16e3d07723cc3903f40ce94f53a3920c88b1fc)
- **DuckDuckGo provider is explicitly no-key:** The bundled DuckDuckGo provider is documented as requiring no API key or account, with direct setup instructions through `openclaw configure --section web`.
  - file: [docs/tools/duckduckgo-search.md:10](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/tools/duckduckgo-search.md#L10)
