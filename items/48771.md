## Summary

Keep open. Current main fixes the `schtasks` `Last Result` parser half of #48771, but the restart health check still does not accept the reported `1008` close reasons such as `device identity required`, plain `pairing required`, or `connect failed`. The linked PR #48801 is useful related work, but it is open/unmerged in the provided GitHub context, so the shipped behavior is not resolved yet.

## Evidence
