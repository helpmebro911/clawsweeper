Keep this PR open. Current main still does not support `session.reset.mode = "off"`, and the PR remains a plausible implementation candidate, but it needs focused cleanup before maintainer merge.

The best maintainer path is a focused update against current main that adds `session.reset.mode: "off"` consistently across exported types, Zod validation, generated schema/help/docs, and `evaluateSessionFreshness`; adds a direct unit test proving stale daily/idle timestamps remain fresh when mode is off; regenerates `docs/.generated/config-baseline.sha256`; updates user-facing session/config docs; and removes or splits the unrelated SDK runtime-logger cleanup.
