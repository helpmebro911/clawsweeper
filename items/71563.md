## Summary

Keep this PR open. Current main still does not route Telegram `/new` or `/reset` through the control lane, while the PR is a focused implementation candidate with tests for the missing sequential-key behavior.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is a scoped Telegram change that routes only registered `/new` and `/reset` session lifecycle commands to the control lane, with the existing sequential-key unit coverage plus an active-run Telegram middleware regression test showing the commands bypass a busy lane without reviving stale reply/finalization races.
