## Summary

Keep #71563 open. Current main still routes Telegram `/new` and `/reset` through the normal chat/topic sequential key, so they can queue behind an active turn. The PR is a focused fix with tests, and the later PR commits addressed the review concern and removed `/clear` because it is not a registered command.

## Best Possible Solution

Keep the PR open and review or land a scoped Telegram fix that routes registered session lifecycle commands (`new`, `reset`) to the Telegram control lane, shares command-resolution logic with status routing, and adds regression coverage for bare and bot-username-suffixed commands. Do not include `/clear` unless the command registry gains that command.
