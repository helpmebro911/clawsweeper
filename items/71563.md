## Summary

Keep #71563 open. Current main has the Telegram sequentializer installed before command handling, and the key resolver only routes abort/status/btw/approval special cases to separate lanes. `/new` and `/reset` are registered session commands, but they currently fall through to the normal chat/topic sequential key, so the PR is still a plausible focused fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for maintainer review. The likely best product/code path is a scoped Telegram fix in `extensions/telegram/src/sequential-key.ts` that routes registered `/new` and `/reset` session commands to the control lane, with regression coverage in `extensions/telegram/src/sequential-key.test.ts` and any needed Telegram command-flow test for active-run bypass behavior. Do not close as implemented or duplicate unless a later main commit lands equivalent session-command control-lane routing.
