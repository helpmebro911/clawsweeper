## Summary

Keep open. Current main still implements Control UI/WebChat attachments as image-only: the picker accepts only image MIME types, select/drop handlers skip non-images, the UI serializes accepted attachments as `type: "image"`, and the gateway parser drops payloads it cannot verify as images. Related #30759 and #70438 overlap but remain open and do not fully cover this issue's broader documents, archives, generic binaries, workspace staging, and config scope.

## Best Possible Solution

Keep this open or explicitly consolidate it into a broader canonical WebChat attachment tracker. A complete fix likely belongs in the Control UI composer and `chat.send` gateway attachment path: broaden the picker/filter/preview model beyond images, stage non-image files under a controlled workspace or media/uploads directory, inject path/size/MIME references into the user turn, add configurable size and MIME policy, and cover documents, text, archives, and generic binaries with focused UI/gateway/security tests and docs.
