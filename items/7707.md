## Summary

Keep #7707 open. Current main has adjacent source labels, session visibility filtering, untrusted Active Memory prompt framing, and Memory Wiki provenance/claim concepts, but it still does not implement persistent memory trust metadata, trust-policy config, trust-tier expiration/review gates, or trust-weighted retrieval/downranking. The issue discussion also includes useful AKF prior art and a prior maintainer Codex review explicitly keeping this open; the related continuity PR was closed unmerged.

## Best Possible Solution

Keep #7707 open for maintainer/product/security design. A complete solution should add structured source trust metadata at memory ingestion, persist it through builtin/QMD/session-memory schemas and SDK contracts, document trust-policy config, enforce retrieval-time filtering/downweighting/review gates, support expiration by trust tier, preserve provenance through compaction and Memory Wiki flows, expose trust state where useful in UI/docs, and add regression coverage across builtin, QMD, Active Memory, session-memory, and compaction paths. HMAKT99's AKF comment is useful prior art, not shipped behavior.
