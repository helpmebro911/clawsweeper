## Summary

Keep #7707 open. Current main has adjacent memory provenance, active-memory, and Memory Wiki work, plus general untrusted-content guidance, but it still does not implement source-based memory trust tagging, trust-tier expiration/review policy, or trust-weighted retrieval. The latest issue discussion also already records a Codex review keeping it open, and the related continuity PR #39049 is closed unmerged in the provided context.

## Best Possible Solution

Keep this issue open for maintainer/product design. A complete fix likely belongs in the memory/context pipeline: classify memory origin at ingestion, persist structured trust metadata, expose documented config for trust policy, enforce review/expiration/filtering/downweighting at retrieval, and add docs/tests. HMAKT99's AKF comment and the existing Memory Wiki provenance/confidence model are useful prior art, but neither currently closes the core request.
