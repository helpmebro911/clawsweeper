## Summary

Keep PR #70596 open. Current main at 4a3030df still only warms QMD memory at gateway startup and explicitly skips builtin memory backends, so the central explicit-local builtin memory cold-start issue is not implemented on main. The PR is also not safe to close as obsolete: it is a focused recent bug-fix PR with linked performance context from #70460, but the existing review comment identifies a real cached-failure hazard that needs author or maintainer handling before merge.

## Best Possible Solution

Keep the PR open and request changes. The best path is to preserve the narrow explicit-local builtin prewarm goal, but implement it without poisoning the cached default manager on startup failure, for example by using a non-cached/status-style prewarm path that still warms the reusable provider safely, or by making provider initialization retryable after failure. Add regression coverage for explicit-local prewarm, non-local skip behavior, warning-only failures, and retry after a failed startup prewarm.
