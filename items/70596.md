## Summary

Keep #70596 open. Current main still only initializes QMD memory during gateway startup, while the PR targets a real remaining builtin local-memory cold-start gap linked to open issue #70460. The PR is recent, focused, and includes tests, but it still needs maintainer/author handling for the unresolved review concern rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for normal review. The best path is to either adjust the PR so builtin local prewarm cannot poison cached manager state after a transient failure, or add source-backed tests proving failed prewarm retries safely; then land it against the open #70460 cold-start problem if CI and maintainer review pass.

## Risks / Open Questions

- The PR needs resolution of the existing review concern around cached default builtin managers versus safe retry behavior after a failed startup probe.
- Prewarming explicit local embeddings intentionally moves local model initialization into post-attach startup, so maintainers should keep the PR's scope limited to `memory.backend=builtin` plus explicit `memorySearch.provider = "local"` and avoid remote or `auto` provider side effects.
