## Summary

Keep open. Current main still does not implement `openclaw models status --probe --all`; the CLI surfaces do not accept or forward a status-level `--all` flag, the status command has no `probeAll` option, and the probe planner still selects one model per provider. The linked PR #63259 remains the right implementation candidate, but it is open and unmerged in the provided context.

## Best Possible Solution

Keep this issue open and review #63259 or a successor PR. The fix should add `models status --probe --all` to both CLI paths, thread a `probeAll` option through `modelsStatusCommand` and `runAuthProbes`, expand targets across every configured primary/fallback/image candidate while preserving current default behavior and provider/profile filters, then add focused target-builder, CLI/JSON, and docs coverage before closing as implemented.
