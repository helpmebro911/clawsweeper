## Summary

Keep open. Current `main` does not implement the PR's fail-closed behavior for malformed Kimi tool-call JSON at `toolcall_end`/final result. The wrapper only rewrites end/final snapshots when a repair succeeded; if the final payload stays malformed and no repair is cached, those snapshots are left untouched.

## Evidence

- **`toolcall_end` only applies repaired args:** In `src/agents/pi-embedded-runner/run/attempt.tool-call-argument-repair.ts:262-278`, the `toolcall_end` branch mutates `event.toolCall`, `event.partial`, and `event.message` only inside `if (repairedArgs)`. There is no fail-closed path that clears malformed final arguments when `repairedArgs` is absent.
