## Summary

Keep open. Current `main` still lacks the PR's fail-closed handling for malformed final Kimi/Anthropic tool-call JSON: the extracted helper repairs cached args, but it does not clear malformed `toolcall_end`/final-result snapshots when no valid repair survives. The open PR's core logic is not present on `main`, and the current tests do not cover those end-of-stream failure cases.

## Evidence

There is no `clearedArgsIndices`-style state or malformed-final branch that clears dispatch/result snapshots when final JSON stays invalid.
