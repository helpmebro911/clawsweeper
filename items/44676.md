## Summary

Keep PR #44676 open. Current main has Kimi/tool-call argument repair and a later merged #54491 hardening, but it does not implement this PR's specific fail-closed behavior for malformed final JSON. The current code preserves meaningful preexisting argument objects and lacks a final-result clearing path for malformed raw payloads, so the PR still represents potentially useful work after rebase and maintainer review.

## Best Possible Solution

Keep the PR open, but rework it onto the extracted repair module. The best path is to add complete raw JSON validation at `toolcall_end`, clear dispatch and final snapshots only when the accumulated raw payload is truly malformed and unrepaired, and add regression tests for non-empty partial/stream/end/final snapshots plus unterminated-string payloads. During review, reconcile that with #54491's preserve-valid behavior so valid complete JSON is not accidentally dropped.
