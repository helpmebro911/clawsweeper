## Summary

Keep #10944 open. Current main still hard-codes Telegram outbound formatting to Bot API HTML mode across normal sends, captions, edits, and streaming previews, while the strict Telegram config schema/types/generated metadata expose related delivery knobs but no channel-level or per-account parseMode, MarkdownV2, or plain-text setting. The issue discussion is still actionable: it already corrected the outdated Markdown claim to current HTML behavior, and the latest maintainer-side Codex review also kept it open.

## Best Possible Solution

Keep #10944 open and implement this in the bundled Telegram plugin/config surface. The likely path is a typed channels.telegram.parseMode setting, with per-account override if maintainers want that granularity, wired through normal sends, media captions, edits, draft/stream preview materialization, and Telegram action delivery. Tests should cover HTML, MarkdownV2 or supported Markdown mode, and plain/no-parse-mode sends, with docs and generated config metadata updated.
