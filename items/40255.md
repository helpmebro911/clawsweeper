## Summary

Keep #40255 open. Current main still resolves configured heartbeat prompts and then unconditionally runs the resulting prompt through the workspace HEARTBEAT.md hint appender, so a custom prompt mentioning HEARTBEAT.md is still mutated before it is sent as the heartbeat user message. The docs still promise custom heartbeat bodies are sent verbatim, and the tests cover default prompt hinting but not the reported custom-prompt regression. Related PR #40756 remains the active open fix attempt, while earlier duplicate fix PRs were closed unmerged.

## Best Possible Solution

Keep the issue open and fix resolveHeartbeatRunPrompt so appendHeartbeatWorkspacePathHint is applied only to the built-in/default or internally generated heartbeat prompts where the workspace disambiguation is intended, not to user-configured heartbeat.prompt text. Add a regression test that configures a custom heartbeat prompt containing HEARTBEAT.md and asserts the Body passed to getReplyFromConfig remains unchanged, while preserving the existing default-prompt hint test.
