## Summary

Current main still mutates user-configured heartbeat prompts that mention HEARTBEAT.md, and PR #40756 is an open implementation candidate with closing syntax for the same regression. This issue should remain open until that PR or an equivalent patch lands.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing PR #40756 or an equivalent maintainer patch. The implementation should distinguish user-authored heartbeat prompt config from built-in/internal heartbeat prompts, skip `appendHeartbeatWorkspacePathHint` for configured custom prompt text, preserve the workspace-path hint for default/internal prompts, and add a regression test for a custom prompt mentioning `HEARTBEAT.md`.
