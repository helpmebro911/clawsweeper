## Summary

Keep open. Current main still has the exact callback gate this PR is trying to remove: reasoning streaming is disabled unless `onReasoningStream` exists, while the webchat gateway path does not supply that callback. The existing implementation can broadcast `stream: "thinking"` once reached, but the gateway-only/no-callback path is still blocked and not covered by current tests.

## Best Possible Solution

Keep the PR open and have maintainers either land a minimal version of this change or request updates: make `streamReasoning` depend on `reasoningMode === "stream"`, emit the `thinking` agent event independently from the optional channel callback, preserve existing callback behavior, and add a regression test for `reasoningMode: "stream"` with no `onReasoningStream`.
