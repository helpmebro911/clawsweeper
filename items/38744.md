## Summary

Keep PR #38744 open. Current main has ElevenLabs realtime transcription for the Voice Call plugin and browser OpenAI realtime Talk, but it does not implement this PR's iOS Talk Mode ElevenLabs realtime WebSocket STT path or STT settings. The PR also still has unresolved reliability concerns from review, so no conservative auto-close reason applies.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If this feature proceeds, either finish the iOS Talk Mode ElevenLabs STT implementation with robust WebSocket readiness, committed-transcript waiting, and Apple Speech fallback recovery, or redirect it into a broader gateway/plugin-backed realtime Talk architecture so iOS does not own provider-specific realtime STT logic directly.
