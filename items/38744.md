## Summary

Keep PR #38744 open. Current main does not implement the PR's native iOS ElevenLabs realtime WebSocket STT path: iOS Talk Mode still uses Apple SFSpeechRecognizer for continuous and push-to-talk capture. Main does include ElevenLabs Scribe v2 Realtime, but that implementation is a Gateway/plugin realtime-transcription provider used by Voice Call streaming, not iOS Talk Mode. The PR also has unresolved reliability review findings, so it is not safe to close as implemented or superseded.

## Best Possible Solution

Keep the PR open for maintainer/product judgment. If the feature proceeds, rebase it onto current main and decide whether iOS Talk Mode should own a native ElevenLabs STT client or use a new gateway/native realtime transcription protocol. Before merge, address the existing WebSocket readiness, committed-transcript waiting, fallback permission/recovery, settings persistence, docs, and focused iOS tests.
