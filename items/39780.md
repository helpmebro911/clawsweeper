## Summary

Current main still does not implement the requested guided Control UI recommendation flow. The insecure HTTP Overview hint remains manual text plus documentation links, while before/after diff and Save/Apply behavior are still scoped to Config and Quick Settings draft flows, not Overview recommendation callouts.

## Best Possible Solution

Keep #39780 open until Control UI recommendation callouts can stage a narrow, base-hash guarded config change, show a before/after diff, and let the user accept or dismiss it with confirmation after save/apply. The likely implementation should reuse the existing Config form diff/apply primitives or `config.patch` flow, wire config mutation callbacks into the relevant Overview recommendation path, add focused Overview/render/controller tests for `gateway.controlUi.allowInsecureAuth`, and update the public Control UI docs.
