## Summary

Current main still lacks the requested `<notts>...</notts>` visual-only TTS behavior, and this PR is not merge-ready because it targets stale TTS ownership while also mixing unrelated gateway, macOS, media, MIME, agent-media, script, and docs changes.

## Best Possible Solution

Keep this PR open for maintainer/author rework or replace it with a focused PR. The implementation should target `extensions/speech-core/src/tts.ts`, update `src/tts/directives.ts` only if the syntax becomes part of the directive contract, add focused tests in the current TTS/directive suites, and update `docs/tools/tts.md`. Split or drop the unrelated gateway authorization, macOS speech, media conversion, MIME, agent media, and helper-script changes.
