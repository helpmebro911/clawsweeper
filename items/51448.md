## Summary

Keep open. Current `main` still does not forward configured `tool_choice`/`toolChoice` into embedded-runner stream options, so the reported bug remains real. The submitted PR targets the right gap, but the posted patch would need rebasing to current test layout and should preserve alias-precedence semantics instead of resolving from the already-merged params object.

## Evidence

- **Current stream wrapper omits toolChoice forwarding:** `createStreamFnWithExtraParams()` forwards `temperature`, `maxTokens`, `transport`, `openaiWsWarmup`, and cached content, but there is no branch that maps `extraParams.tool_choice` or `extraParams.toolChoice` onto `streamParams.toolChoice`.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:289](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/agents/pi-embedded-runner/extra-params.ts#L289)
  - command: `nl -ba src/agents/pi-embedded-runner/extra-params.ts | sed -n '279,340p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Downstream OpenAI request builders only emit tool_choice from stream options:** The OpenAI HTTP transport sets `params.tool_choice` only when `options.toolChoice` is present, and the WS request builder does the same. Without the missing forwarding step above, config-layer `tool_choice` never reaches these request payloads.
  - file: [src/agents/openai-transport-stream.ts:1742](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/agents/openai-transport-stream.ts#L1742)

## Risks / Open Questions

- The open bug still affects OpenAI/OpenAI-compatible runs that rely on config-layer `tool_choice` to require tool use.
- The PR diff in the GitHub context references `src/agents/pi-embedded-runner/extra-params.openai.test.ts`, but current `main` uses a different test layout, so it is not an as-is close candidate.
- A naive forward-only rebase can introduce an alias-precedence regression if both `tool_choice` and `toolChoice` are present at different config precedence levels.
