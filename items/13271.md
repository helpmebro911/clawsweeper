## Summary

Keep #13271 open. Current main has adjacent pieces: per-channel allow/deny tool policy, optional loop detection, plugin `before_tool_call` blocking, and explicit `sessions_spawn`; it still does not implement a per-channel, per-turn tool-call budget with `warn`, `block`, and `auto-delegate` overflow behavior.

## Best Possible Solution

Keep #13271 open until OpenClaw adds a typed product/API path for per-channel, per-turn tool-call budgets. A complete implementation should add documented channel or agent/channel-scoped config, pass reliable channel/request context into enforcement, define `warn`, `block`, and `auto-delegate` semantics, and cover Discord plus at least one non-Discord channel in tests. If maintainers prefer plugin ownership, add the missing hook/runtime contract first instead of closing this as ClawHub-only.
