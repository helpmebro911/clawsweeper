## Summary

Keep #13271 open. Current main has adjacent pieces, including channel tool allow/deny policy, opt-in loop detection, before_tool_call blocking, plugin hooks, and manual subagent spawning, but it still does not implement the requested per-channel, per-turn tool-call budget with warn/block/auto-delegate overflow semantics.

## Best Possible Solution

Keep #13271 open until OpenClaw has a documented channel-scoped or agent/channel-scoped budget contract. A complete solution should add typed config/schema/help/docs, carry reliable channel/request context into tool-call enforcement, define warn/block/auto-delegate overflow semantics, implement enforcement in the tool-call runtime, and add tests covering Discord plus at least one non-Discord channel. If maintainers want plugin ownership, first add the missing hook/runtime contract needed for a plugin to implement this cleanly.
