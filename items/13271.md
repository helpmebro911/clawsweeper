## Summary

Keep #13271 open. Current main still has adjacent pieces, including per-channel allow/deny policy, opt-in loop detection, before_tool_call blocking, and explicit subagent APIs, but it does not implement a per-channel, per-turn tool-call budget with warn/block/auto-delegate overflow semantics. The existing Apr 26, 2026 Codex review comment also explicitly kept this item open, and related #9912/#14373 do not supersede the channel-scoped request.

## Best Possible Solution

Keep #13271 open until OpenClaw has a documented channel or agent/channel-scoped budget contract. A complete solution should add typed config/schema/help/docs, pass reliable channel/request context into enforcement, define warn/block/auto-delegate overflow semantics, implement enforcement in the tool-call runtime, and add tests covering Discord plus at least one non-Discord channel. If maintainers want plugin ownership, first add the missing hook/runtime contract needed for a plugin to implement this cleanly.
