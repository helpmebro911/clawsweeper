## Summary

Keep open. Current main still leaves the core audit problem unresolved for bare `$VAR` values, and the auth-profile audit path still can fall through to `PLAINTEXT_FOUND` after detecting an inline ref. The configure/apply type-mismatch path also still throws, matching the reported OAuth/static-profile abort. #54141 is the open implementation candidate with closing intent, so this issue should stay open until that PR lands or is closed.

## Best Possible Solution

Keep this issue open while #54141 is reviewed and either landed or closed. The best maintainer path is to land a focused core fix with regression coverage for bare `$VAR` and `${VAR}` env references in both config and auth-profile audit paths, then separately decide how to handle OAuth/static auth-profile type mismatches, post-migration scrub behavior, and rotation guidance.
