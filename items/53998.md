## Summary

Keep open. Current main still has the reported secrets-audit false-positive paths: bare `$VAR` is not parsed as a SecretRef, auth-profile inline refs still fall through to `PLAINTEXT_FOUND`, and `.env` audit still treats any non-empty known secret env assignment as plaintext. There is also an open unmerged PR, #54141, whose body uses GitHub closing syntax for this issue, so this issue should stay open until that PR is reviewed/landed or rejected.

## Best Possible Solution

Keep the issue open and treat #54141 as the current implementation candidate for the core `$VAR` and auth-profile false-positive path. If maintainers land it, they should verify the remaining `.env` collector and configure/OAuth behavior with tests or split those leftovers into narrower follow-up issues before closing this issue.
