## Summary

Keep #10737 open. Current main still does not implement channel-layer auto-acknowledgment or native typing for the legacy `channels.imessage` path; BlueBubbles has typing support, but that is a separate recommended iMessage channel and does not satisfy the requested `channels.imessage.autoAck` behavior.

## Best Possible Solution

Keep the issue open until maintainers either implement channel-level feedback for the bundled legacy iMessage plugin or explicitly decline it. A concrete implementation would likely live in `extensions/imessage`: add typed config/schema/docs for `autoAck`, wire the inbound monitor to send a pre-dispatch text ack or feature-detected native `imsg` typing signal, preserve a configurable fallback, and add regression tests for DM-only behavior and no duplicate final replies. BlueBubbles should remain documented as the recommended iMessage path, but it should not close this legacy `channels.imessage` request.
