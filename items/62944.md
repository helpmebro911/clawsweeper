## Summary

Keep open. Current main still forwards fixed 30,000 ms timeouts from the agent image tool instead of resolving the configured media image timeout, and the provided GitHub context shows open, unmerged PRs with closing syntax for this issue. Maintainers should review or close those PRs before this issue is closed.

## Best Possible Solution

Keep this issue open and review the linked open PRs #63046 and #63483. The best fix belongs in src/agents/tools/image-tool.ts: resolve the configured image timeout once, pass it to every describeImage/describeImages call, preserve the intended default when unset, and add regression coverage for single-image and multi-image paths.
