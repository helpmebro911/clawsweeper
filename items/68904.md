## Summary

Keep open. Current main still uses `groundedCount` for the Daily Log Review summary and staged section, and there are open PRs specifically attempting to fix this issue. The issue should stay open until one of those implementation PRs lands or is superseded.

## Best Possible Solution

Keep this issue open and review/land or supersede the open UI fix work. The best product fix is to make the Daily Log Review summary, staged section, and origin badges consistently use `dailyCount > 0`, unless maintainers intentionally rename that surface to grounded replay instead; add a regression test with daily-only and grounded-only entries so the two counters cannot be conflated again.
