## Summary

Keep this PR open. Current main still lacks the proposed `feishu_chat` chat-list action and still routes `feishu_chat` through the first enabled account instead of the calling agent's bound account, so the work is not obsolete. The PR needs rebase/security cleanup before merge because main has since added `member_info`, changed the TypeBox import style, and the PR's explicit `accountId` path needs to reject disabled accounts.

## Best Possible Solution

Keep the PR open for owner review. The best path is to rebase it onto current main, preserve `member_info` and current `typebox` imports, implement `list` plus per-agent routing using the existing Feishu tool-account seam, add execution-time checks for both `resolvedAccount.enabled` and `tools.chat`, and keep focused tests for list pagination, per-agent routing, explicit override, disabled-account rejection, and disabled-tool rejection.
