## Summary

Keep #23926 open. Current main has partial blocking `before_install` support for gateway-managed skill dependency installers, but native ClawHub skill install/update and the UI ClawHub install path still copy downloaded skill folders without that policy hook, and there is still no post-install observational lifecycle hook.

## Best Possible Solution

Keep #23926 open as the concrete tracker for skill install lifecycle hook/API work. The best path is to extend the documented `before_install` API or add equivalent skill lifecycle hooks so native ClawHub skill install/update/sync/UI/gateway paths run a blocking policy point before files are copied, then add a documented post-install observational hook with installedAt/success/failure metadata or record an explicit maintainer decision that post-install observation belongs elsewhere. Tests should cover CLI, gateway, UI ClawHub installs and updates, blocking behavior, warning/findings propagation, and post-install semantics.
