## Summary

Keep #23926 open. Current main has partial `before_install` support for gateway-managed skill dependency installs, but it does not implement the requested skill install lifecycle surface across native ClawHub skill install/update paths and still has no post-install observational hook/API.

## Best Possible Solution

Keep #23926 open as the concrete tracker for skill install lifecycle hook/API work. The best path is to extend the documented `before_install` API or add equivalent skill lifecycle hooks so native OpenClaw ClawHub skill install/update/sync/UI/gateway paths run a blocking policy point before files are copied, then add a documented post-install observational hook or record an explicit maintainer decision that post-install observation belongs elsewhere. Tests should cover CLI, Gateway, UI ClawHub installs and updates, blocking behavior, warning/findings propagation, and post-install semantics.
