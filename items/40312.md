## Summary

Keep open. Current `main` still appears to silently rewrite unknown web UI routes like `/tasks` to the chat tab, so the requested behavior is not implemented in-repo yet.

## Evidence

- **Current route sync still falls back unknown paths to chat:** `syncTabWithLocation` resolves `tabFromPath(window.location.pathname, host.basePath) ?? "chat"`, then immediately calls `syncUrlWithTab`, which rewrites the browser pathname to the tab path. That means an unknown path such as `/tasks` still normalizes to `chat` and gets rewritten.
  - file: [ui/src/ui/app-settings.ts:491](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/ui/src/ui/app-settings.ts#L491)
  - command: `nl -ba ui/src/ui/app-settings.ts | sed -n '488,503p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Unknown paths are not recognized as valid tabs:** `tabFromPath` returns `PATH_TO_TAB.get(normalized) ?? null` for non-root paths, so `/tasks` is an unknown route unless explicitly mapped.
  - file: [ui/src/ui/navigation.ts:116](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/ui/src/ui/navigation.ts#L116)
  - command: `nl -ba ui/src/ui/navigation.ts | sed -n '116,133p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **URL sync mutates the pathname to the resolved tab route:** `syncUrlWithTab` compares `window.location.pathname` to `pathForTab(tab, host.basePath)` and assigns `url.pathname = targetPath` when they differ, which is the silent rewrite behavior the issue describes.
  - file: [ui/src/ui/app-settings.ts:561](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/ui/src/ui/app-settings.ts#L561)
  - command: `nl -ba ui/src/ui/app-settings.ts | sed -n '561,580p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Proposed fix was never merged:** PR #50997 (`fix(web): stop silently rewriting /tasks route`) changed this exact fallback and added a regression test, but GitHub shows `merged_at: null` and the PR was auto-closed for the author's active-PR limit, not because the fix landed.
  - command: `curl -s https://api.github.com/repos/openclaw/openclaw/pulls/50997 ; curl -s https://api.github.com/repos/openclaw/openclaw/pulls/50997/files`
