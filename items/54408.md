## Summary

Keep open. Current main still routes the pre-compaction memory flush as a normal embedded prompt with `silentExpected: true`, but without `transcriptPrompt: ""` or another internal-only transcript boundary. The runner treats an undefined `transcriptPrompt` as the effective user prompt and submits it through the normal session prompt path, while docs still describe the flush as silent. Related #51421/#12590 address flush cadence, not this transcript-visibility leak.

## Best Possible Solution

Keep this open as the core memory-flush transcript leak tracker. The best fix is to make pre-compaction memory flush a truly internal turn, likely by passing `transcriptPrompt: ""` or adding an explicit internal-run API in `src/auto-reply/reply/agent-runner-memory.ts`, then add regression coverage proving the session JSONL/UI history contains no user message with the flush prompt. The fix should also ignore or sanitize historically leaked `Pre-compaction memory flush` user prompts during compaction so already-polluted sessions stop re-triggering the safeguard loop. Treat #51421/#12590 as the separate cadence-counter bug unless that PR is expanded to cover transcript visibility.
