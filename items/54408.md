## Summary

Keep open. Current main still submits the pre-compaction memory flush as a normal embedded prompt with `silentExpected: true`, but without `transcriptPrompt: ""` or another internal-only transcript boundary. The docs still promise a silent turn, and the related #51421/#12590 work is cadence/counter handling rather than transcript invisibility or cleanup of already polluted sessions.

## Best Possible Solution

Keep this open as the core memory-flush transcript leak tracker. The best fix is to make pre-compaction memory flush a truly internal turn, likely by passing `transcriptPrompt: ""` from `src/auto-reply/reply/agent-runner-memory.ts` or adding an explicit internal-run API, then add regression coverage proving session JSONL and `chat.history` never contain a user message with the flush prompt. Also add compaction-side cleanup for sessions that already contain leaked memory-flush prompts. Treat #51421/#12590 as separate cadence work unless that PR is expanded to cover transcript visibility.
