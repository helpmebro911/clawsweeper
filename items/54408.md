## Summary

Current main still has the reported bug path: the pre-compaction memory flush is marked as silent for assistant/output handling, but its prompt is still submitted to the embedded active session as the visible prompt because no hidden transcript prompt or internal-only run marker is passed. The related #51421/#12590 work is about memory-flush cadence, not transcript invisibility or cleanup of polluted sessions.

## Best Possible Solution

Keep this open as the core memory-flush transcript leak tracker. The likely fix belongs in `src/auto-reply/reply/agent-runner-memory.ts` or the embedded-runner API: make pre-compaction memory flush a true internal turn, for example by passing `transcriptPrompt: ""` or adding an explicit internal-run contract, then add regression coverage proving session JSONL and `chat.history` never contain a user message with the flush prompt. Keep #51421/#12590 focused on cadence/counter behavior unless that work is expanded to cover transcript invisibility and polluted-session cleanup.
