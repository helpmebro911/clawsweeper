## Summary

Keep open. Current main does not implement the requested two-phase macOS LaunchAgent restart. `restartLaunchAgent` still performs stale-port cleanup, enables the service, then calls `launchctl kickstart -k`; the normal-path unit test explicitly asserts there is no `bootout` or `bootstrap`. The referenced PR #43961 is closed and unmerged, so it is not a shipped canonical fix.

## Best Possible Solution

Keep #70801 open for maintainer judgment and a targeted macOS gateway lifecycle change. The best implementation would either add the requested `bootout -> wait previous LaunchAgent pid/port gone -> bootstrap + kickstart` ordering in `src/daemon/launchd.ts`, or document and test an equivalent pre-start wait that maintainers accept as safer than bootout. It should preserve the detached handoff path and add explicit ordering tests plus macOS launchd smoke coverage.
