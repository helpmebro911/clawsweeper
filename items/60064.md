## Summary

Keep open. Current `main` still has the protocol/UI mismatch this PR fixes: gateway exec-approvals validators reject allowlist entries carrying `source: "allow-always"`, while runtime/store code already preserves that metadata.

## Evidence

- **Gateway schema still omits `source`:** `src/gateway/protocol/schema/exec-approvals.ts:4` defines `ExecApprovalsAllowlistEntrySchema` with `id`, `pattern`, `argPattern`, `lastUsedAt`, `lastUsedCommand`, and `lastResolvedPath`, and `additionalProperties: false` at line 13. There is no `source` field, so a payload containing `source: "allow-always"` is still invalid on current `main`.
