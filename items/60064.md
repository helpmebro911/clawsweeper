## Summary

Keep open. The PR is still a plausible, scoped fix: current main's gateway exec approvals allowlist schema accepts `argPattern` but not `source: "allow-always"`, while `additionalProperties: false` would reject that metadata in `exec.approvals.set` and `exec.approvals.node.set` payloads. Related PRtags comments identify duplicate open PRs, but the current main surface does not show the requested change landed.
