## Summary

Keep #71961 open. Current main still lacks the PR's central behavior: compaction lifecycle events are not logged at normal info level for Gateway watch/log-follow visibility, the start handler does not receive event reason metadata, docs do not list Gateway logs as an observable compaction surface, and the current handler test only covers compaction-count reconciliation. The PR is recent, focused, and the provided review context shows the only bot concern was a test-organization nit that the author addressed.

## Best Possible Solution

Keep the PR open for maintainer review and likely land #71961 or an equivalent patch that logs embedded compaction start/end at info level, carries the compaction reason through the handler, updates the compaction docs, and keeps the focused regression test.
