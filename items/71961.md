## Summary

Keep #71961 open. Current main does not contain the PR’s requested normal Gateway visibility for embedded compaction lifecycle events: compaction start is still debug-only, completion has no info-level lifecycle log, and the start event reason is not passed into the handler. The PR is a focused, recent bug-fix/docs/test change, and the provided review discussion shows only a style concern that the author addressed.

## Best Possible Solution

Keep the PR open and review it as the likely implementation path: add optional `info` support to the embedded subscribe logger type, pass compaction events into the handler, emit start/complete/incomplete lifecycle logs at info level with structured metadata, and keep the docs plus focused handler regression test. No current-main implementation or canonical replacement was found.
