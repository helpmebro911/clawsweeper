## Summary

Current `main` does not match the reported `server.mjs` device-approval timeout flow. The Docker/runtime paths now start the gateway directly, health checks use `/healthz`, pairing-required/auth-closed responses are treated as healthy reachability, and I found no `server.mjs` file or matching timeout log string in the checkout.

## Close Comment

Closing from a Codex review because I could not reproduce this report against current `main`. The current Docker/runtime paths no longer use a `server.mjs` wrapper: Compose starts `node dist/index.js gateway` (`docker-compose.yml:29`) and the image default command is `node openclaw.mjs gateway --allow-unconfigured` with `/healthz`-based health checks (`Dockerfile:280`). Current gateway restart health logic explicitly treats an auth-closed / pairing-required gateway as healthy reachability (`src/cli/daemon-cli/restart-health.ts:58`, covered by `src/cli/daemon-cli/restart-health.test.ts:242`), and the docs describe pairing-required as a recoverable approval state rather than a crash (`docs/gateway/troubleshooting.md:343`, `docs/cli/devices.md:53`). I also found no `server.mjs` file or matching timeout log string in the repo. If this still happens on a current image, please open a new issue with the exact image tag, container command, and startup/shutdown logs from `openclaw-gateway`.
