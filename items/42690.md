## Summary

Keep open. Current main has adjacent runtime/fallback model reporting fixes, but the shared status card still has no generic non-fallback `Active:` line when the selected/default model and active routed model differ. The current tests explicitly cover fallback display and assert that `Active:` is absent, leaving the requested routed-active-model split unimplemented.

## Best Possible Solution

Keep this open. Implement the fix in the shared status rendering path: when `resolveSelectedAndActiveModel(...).activeDiffers` is true and `resolveActiveFallbackState` is not active, render a separate `↪️ Active: provider/model` line while keeping the selected/default `🧠 Model:` line. Add focused regression coverage in `src/auto-reply/status.test.ts` and `src/agents/openclaw-tools.session-status.test.ts` for a selected/default model plus `sessionEntry.modelProvider/model` from a non-fallback router.
