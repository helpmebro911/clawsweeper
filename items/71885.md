## Summary

Keep open. Current main has not implemented the PR’s SQLite normalization, and the underlying SQLite bind failure is real. However, the PR should receive maintainer review rather than cleanup closure because `TaskRecord.task` is still a string contract in current core APIs and several task surfaces call `.trim()`, so a store-only fix may be narrower than the full structured-task contract question.

## Best Possible Solution

Keep the PR open for maintainer review. The best code direction is to decide whether structured task values are a supported runtime input; if yes, normalize them at the task creation/store boundary and update the affected task contracts/tests, or if no, add explicit validation before SQLite so invalid structured values fail earlier with a clear error. The current PR is a plausible minimal guard but should be reviewed against the string-based `TaskRecord.task` contract before merge.
