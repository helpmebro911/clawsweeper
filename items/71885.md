## Summary

PR #71885 should remain open. Current main still binds `TaskRecord.task` directly into the SQLite `task_runs.task` TEXT column, and an in-memory `node:sqlite` check confirms object values cannot be bound. The PR is a narrow bug-fix candidate with a targeted regression test in the supplied diff, and no conservative close reason applies.

## Best Possible Solution

Keep PR #71885 open for maintainer review. The best path is a small persistence-boundary normalization fix with regression coverage, while preserving existing string task values and the current read path that returns `task` as a string.
