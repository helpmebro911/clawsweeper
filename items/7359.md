## Summary

Current main still does not provide a default unified view of an agent's own Slack channel activity from a DM. Slack DMs and channels remain separate session buckets; outbound sends are now mirrored into the destination session, but default session-tool visibility is still tree-scoped and does not let a DM session read an unrelated Slack channel session unless broader visibility is explicitly configured. The related closed Slack issues cover thread context and queueing, not this broader cross-context self-activity recall request.

## Best Possible Solution

Keep #7359 open and implement an explicit same-agent Slack self-activity recall path. The likely direction is a scoped outbound activity ledger or session-history bridge keyed by agent, Slack account, target channel/thread, and sent message id, exposed to the DM context without broadly granting all cross-session history. Add regression coverage for: send to Slack channel from a DM, ask from the DM whether it was sent, and verify the agent can cite its own sent channel message while unrelated user/channel history remains gated.
