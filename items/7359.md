## Summary

Keep #7359 open. Current main still separates Slack DM/main sessions from Slack channel sessions, and outbound `message` sends are mirrored only into the destination Slack session transcript. The default session-history tool visibility is `tree`, so a DM/main session cannot read an unrelated same-agent Slack channel session unless the operator enables broad visibility, which is not the scoped self-activity recall requested here. The provided issue discussion also includes a same-day maintainer/Codex comment explicitly keeping this exact gap open, and there have been no relevant source changes since that reviewed SHA.

## Best Possible Solution

Keep #7359 open and implement a scoped Slack self-activity recall path. The clean product shape is an outbound activity ledger or session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, exposed from DM/main context only for the agent's own outbound Slack sends. Add regression coverage for a Slack DM/main turn that posts to a Slack channel via the `message` tool, then later recalls that sent message without enabling broad cross-session transcript visibility.
