## Summary

Keep #7359 open. Current main still separates Slack DM/main sessions from Slack channel sessions, and the message-tool outbound mirror is written to the destination Slack session rather than a scoped self-activity record visible from the originating DM/main session. The provided issue discussion includes a same-day maintainer/Codex comment explicitly keeping this gap open, and there are no relevant file changes from that reviewed SHA to current main.

## Best Possible Solution

Keep #7359 open and implement a scoped Slack self-activity recall path. The clean product shape is an outbound activity ledger or session-history bridge keyed by agent id, Slack team/account, destination channel/thread, and Slack message id/timestamp, exposed from DM/main context only for the agent's own outbound Slack sends. Add regression coverage for a Slack DM/main turn that posts to a Slack channel via the message tool, then later recalls that sent message without enabling broad cross-session transcript visibility.
