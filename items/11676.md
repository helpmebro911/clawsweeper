## Summary

Issue #11676 should stay open. Current main still requires a Node-hosted OpenClaw runtime on normal OS environments and does not provide a stable runtime-agnostic agent/core protocol for bare-metal, C/Rust/WASM, or ESP32-class ports. The plugin SDK is a useful extension surface, but it is still TypeScript/Node/Gateway-oriented rather than a portable conformance spec. The prior steipete Codex review comment also explicitly kept this issue open, and the related closed #45496 is narrower browser/WASM work, not a canonical embedded/no-Node plan.

## Best Possible Solution

Keep #11676 open for explicit maintainer product judgment. A strong path would define a stable runtime-agnostic agent/core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, and platform adapters, then add conformance tests or a small reference implementation proving the protocol can run outside the Node-hosted OpenClaw runtime. MimiClaw should remain linked as useful external prior art.
