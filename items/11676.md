## Summary

Issue #11676 should stay open. Current main still requires a Node-hosted OpenClaw runtime and does not expose a stable runtime-agnostic agent/core protocol suitable for bare-metal, C/Rust/WASM, or ESP32-class ports. The existing plugin SDK and agent harness are useful extension points, but they remain TypeScript/Node-oriented and documented as experimental trusted native plugin surfaces, not a portable conformance spec.

## Best Possible Solution

Keep the issue open for explicit maintainer product judgment. A strong path would define a stable runtime-agnostic agent/core protocol with schemas for tool calls, memory/session state, streaming/events, auth/model selection, and platform adapters, then add conformance tests or a small reference implementation proving the protocol can run outside the Node-hosted OpenClaw runtime. MimiClaw should remain linked as useful external prior art.
