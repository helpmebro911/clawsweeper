## Summary

Keep open: current main still has the auto-reply CLI path call `runCliAgent` directly and return the result without `persistCliTurnTranscript`, while the direct CLI command path does persist CLI turns. The PR remains a plausible focused bug fix, and the provided review context shows the threadId feedback was addressed in the PR head.

## Best Possible Solution

Keep the PR open for maintainer review. Rebase or recreate the small fix on current main: in `runAgentTurnWithFallback`'s successful CLI branch, best-effort persist the turn with `persistCliTurnTranscript`, pass `originatingThreadId`, use the transcript-safe prompt body, add regression tests for success/failure/threadId, and consider parity with the direct CLI compaction lifecycle.
