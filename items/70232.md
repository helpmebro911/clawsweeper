## Summary

Keep open. Current `main` added the Bonjour disable controls and Ciao classification helpers, but the production Bonjour plugin still does not wire the unhandled-rejection handler that would suppress `CIAO PROBING CANCELLED`, so the reported gateway crash path is not clearly fixed on `main`.

## Evidence

- **Unhandled-rejection suppression is optional inside the advertiser:** `startGatewayBonjourAdvertiser` classifies Ciao cancellations and would suppress them, but it only registers that handler when `deps.registerUnhandledRejectionHandler` is provided.
  - file: [extensions/bonjour/src/advertiser.ts:178](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/extensions/bonjour/src/advertiser.ts#L178)
  - command: `nl -ba extensions/bonjour/src/advertiser.ts | sed -n '162,260p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **Production Bonjour entry does not pass the handler:** The bundled Bonjour plugin calls `startGatewayBonjourAdvertiser(..., { logger: api.logger })` and does not provide `registerUnhandledRejectionHandler`, so the suppression path above is not active in normal runtime.
  - file: [extensions/bonjour/index.ts:23](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/extensions/bonjour/index.ts#L23)

  - file: [extensions/bonjour/src/advertiser.test.ts:75](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/extensions/bonjour/src/advertiser.test.ts#L75)
