Keep open. Current main still does not implement a TUI startup recent-session summary, and #71719 was closed as superseded by this PR, making #71856 the canonical implementation discussion. The PR is plausible because the Gateway/TUI session list API already exposes derived titles and last-message previews, but the branch still needs maintainer review and fixes for reconnect duplication plus focused regression coverage.
