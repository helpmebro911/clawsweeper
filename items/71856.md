Keep open. Current main does not implement the requested TUI startup recent-session summary, and this PR is the canonical implementation track for closed issue #71719. It is not conservative cleanup material; it still needs maintainer review because the PR diff runs inside every `onConnected` callback and the reconnection-duplication concern remains valid.

## Best Possible Solution

Keep the PR open for maintainer review. Revise the TUI startup implementation to run once after initial non-local history load, use the existing bounded `sessions.list` fields, keep failures non-blocking, and add targeted TUI tests before merge.
