## Summary

Keep open. Current main still resolves `before_model_resolve` inside each embedded attempt reached by the model fallback loop, so the core boundary described in #41487 remains meaningful. The related #63139 is useful evidence and tracks an adjacent inverse symptom, but it is not a clear enough superseding issue to close this broader report.

## Evidence

- **Fallback loop still owns per-candidate attempts:** `agentCommandInternal` calls `runWithModelFallback` and invokes `runAgentAttempt` for each fallback candidate, deriving `isFallbackRetry` locally. The model hook is not hoisted outside this fallback loop.
