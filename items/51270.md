## Summary

Keep open. Current `main` still mishandles `/model list` and `/model status` in the TUI, while docs and the shared gateway command path both treat them as supported subcommands, so this PR is not obsolete.

## Evidence

- **TUI still parses `/model <args>` as a direct model set:** In current `main`, the TUI `case "model"` only opens the picker when there are no args; otherwise it calls `client.patchSession({ model: args })`. That means `/model list` and `/model status` are still treated as literal model ids instead of subcommands.
  - file: [src/tui/tui-command-handlers.ts:342](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/tui/tui-command-handlers.ts#L342)
  - command: `nl -ba src/tui/tui-command-handlers.ts | sed -n '342,356p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Known `model` commands never fall through to the shared gateway handler:** Only the `default` branch forwards raw slash commands via `sendMessage(raw)`. Because `model` is handled locally, `/model status` cannot reach the shared gateway `/model status` implementation from the TUI today.
  - file: [src/tui/tui-command-handlers.ts:566](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/tui/tui-command-handlers.ts#L566)
  - command: `nl -ba src/tui/tui-command-handlers.ts | sed -n '560,568p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Shared `/model` directive handling already supports both `list` and rich `status` output:** `maybeHandleModelDirectiveInfo` recognizes `directive === "list"` and `directive === "status"`. The status path builds a detailed multi-line reply including current vs active model, default model, agent, auth file, and per-provider auth information.
  - file: [src/auto-reply/reply/directive-handling.model.ts:214](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/auto-reply/reply/directive-handling.model.ts#L214)
  - command: `nl -ba src/auto-reply/reply/directive-handling.model.ts | sed -n '212,357p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Docs promise `/model list` and `/model status` as supported flows:** The docs explicitly say `/model` and `/model list` open the compact picker, and `/model status` is the detailed view. Current TUI behavior does not match that documented contract.
  - file: [docs/concepts/models.md:121](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/concepts/models.md#L121)
  - command: `nl -ba docs/concepts/models.md | sed -n '121,142p'`
