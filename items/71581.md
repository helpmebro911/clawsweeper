## Summary

Keep this issue open. Current main still routes exec-event and cron-event heartbeat wakes through the explicit same-agent session key, and the single-owner session.dmScope="main" helper is still limited to inbound DM channel paths. The closed PR #72160 is unrelated and was not merged.

## Best Possible Solution

Keep this issue open and implement a narrow core routing helper for internal system-event wakes. When session.dmScope="main" and the same single-owner allowlist semantics used by inbound DM routing apply, normalize same-agent direct-message session keys to resolveAgentMainSessionKey before exec/cron system-event enqueue, heartbeat wake targeting, preflight peek, and event consumption. Add regression coverage for exec notify-on-exit, explicit exec system events, and cron next-heartbeat events, with negative cases for multi-owner or wildcard allowlists, non-main scopes, subagent keys, isolated heartbeats, and non-DM explicit sessions.
