## Summary

Keep #71581 open. Current main still accepts a forced exec/cron heartbeat session key whenever it belongs to the same agent, and the forced-session path does not consult `session.dmScope` or the single-owner allowlist helper. The reported behavior is a real core routing gap, not superseded by the unrelated closed PR #72160.

## Best Possible Solution

Implement a core heartbeat/session-routing fix gated by effective `session.dmScope === "main"` plus the same single-owner allowlist precondition used for inbound DMs. The fix should redirect same-agent direct-message exec-event and cron-event wake processing to `agent:<id>:main` while preserving original delivery context, and add focused regression tests for background exec and cron system events from a per-channel direct-session key.
