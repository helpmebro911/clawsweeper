## Summary

Keep open. Current main still has the original Control UI polling cadence, so PR #45930 is not implemented or obsolete. The PR is also not merge-ready as-is because the posted review identifies a concrete logs-tail data-loss risk: the UI still fetches only 500 lines / 250 KB per poll, and the gateway log-tail implementation resets to the newest maxBytes window when backlog exceeds that byte limit.

## Best Possible Solution

Keep the PR open and request a narrow revision. Nodes/debug polling may be reasonable to slow down, but logs need a safer design first: preserve the shorter logs cadence, widen or adapt the `logs.tail` window, add a push/streaming path, or add overflow recovery that makes data loss explicit. Add a regression test around high-volume `logs.tail` cursor overflow before considering the logs interval change merge-ready.
