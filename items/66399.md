## Summary

Current main still has the reported supervisor behavior: timeout/manual/scoped cancellation goes through `cancelAdapter`, which immediately calls `adapter.kill("SIGKILL")`. Adjacent process-tree and wait-fallback safeguards exist, but they do not implement the requested supervisor-level SIGTERM-first grace period for the direct supervised child, and the tests still assert immediate SIGKILL on the relevant timeout paths.

## Best Possible Solution

Keep this issue open and implement a narrow supervisor-level cancellation change: on manual cancellation, scoped replacement, overall timeout, and no-output timeout, send SIGTERM first, start an unref'ed grace timer, then escalate to SIGKILL if the run has not settled. Clear the escalation timer on settle/dispose, preserve existing child/PTY wait fallbacks and process-tree protections, and update supervisor/adapter tests to assert SIGTERM-first behavior plus hard-kill escalation. Treat #71710 as the separate agent-side SIGTERM handling follow-up.
