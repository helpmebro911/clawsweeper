## Summary

Current main still does not implement the central request. Supervisor cancellation for overall/no-output timeouts and manual replacement still calls `adapter.kill("SIGKILL")` directly, and supervisor tests still assert immediate SIGKILL behavior. There are adjacent safeguards in the adapters and `killProcessTree`, including a SIGKILL wait fallback and process-tree SIGTERM-before-SIGKILL behavior, but they do not provide the requested supervisor-level SIGTERM grace window before hard kill for the direct supervised child. The related discussion is recent, includes production impact, and the prior PR mentioned in the thread is closed without merge, so this should remain open.

## Best Possible Solution

Keep this issue open and implement a narrow supervisor change: on cancellation, overall timeout, and no-output timeout, send SIGTERM first, start a documented bounded grace timer, then escalate to SIGKILL while preserving the existing process-tree and wait-fallback guarantees. Update supervisor and adapter tests to assert SIGTERM-first behavior, SIGKILL escalation, and no regression in child/PTY drain handling. Coordinate but do not merge this into #71710, because that issue covers the agent process responding to SIGTERM rather than the supervisor's signal escalation policy.
