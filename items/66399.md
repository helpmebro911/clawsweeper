## Summary

Keep open. Current main still cancels supervised exec runs with immediate SIGKILL on timeout/cancel paths, and tests still assert that behavior. There are adjacent improvements around process-tree killing and SIGKILL wait fallback, but they do not satisfy the requested SIGTERM-first supervisor escalation and bounded drain behavior for the exec tool.

## Best Possible Solution

Keep this issue open and implement a focused process-supervisor fix: introduce a documented SIGTERM grace window before SIGKILL for supervised exec cancellation/timeouts, preserve a hard kill guarantee and cross-platform wait fallback, close or destroy stdin/stdout/stderr only where safe, and update supervisor/adapter/exec tests so SIGTERM-first and SIGKILL escalation are covered. Coordinate with #71710 for agent SIGTERM handling, but do not treat that as superseding this supervisor-side fix.
