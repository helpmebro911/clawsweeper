## Summary

Current main still cancels supervised exec runs by calling adapter.kill("SIGKILL") directly on no-output timeout, overall timeout, and scoped-run replacement. Adjacent process-tree and wait-fallback work exists, but it does not implement the requested supervisor-level SIGTERM-first grace window before hard kill, and the current regression tests still assert immediate SIGKILL behavior.

## Best Possible Solution

Keep this issue open and implement a focused process-supervisor change: send SIGTERM first for supervised exec cancellation/timeouts, use a documented bounded grace period before SIGKILL, preserve the hard-kill and wait-fallback guarantees across child and PTY adapters, safely handle stdio cleanup only where appropriate, and update supervisor/adapter tests to assert SIGTERM-first plus forced escalation. Coordinate with #71710 for agent-side SIGTERM handling, but do not treat it as superseding this supervisor-side fix.
