## Summary

Keep open. Current main still ships the default-enabled bundled Bonjour plugin with unpatched @homebridge/ciao@1.3.6, and upstream ciao 1.3.6 still runs the Windows arp polling command without an options object, so #71633 remains a narrow current-main fix for the linked Windows console-flicker report rather than maintainer cleanup.

## Best Possible Solution

Keep #71633 open for maintainer review. The best path is to land this narrow @homebridge/ciao@1.3.6 pnpm patch, or an equivalent upstream version bump if one becomes available, with lockfile consistency and either Windows manual verification or a focused regression check showing the ciao arp polling exec receives `windowsHide: true`.
