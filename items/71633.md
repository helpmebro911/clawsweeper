## Summary

Keep #71633 open. Current main still ships the default-enabled Bonjour plugin through unpatched @homebridge/ciao@1.3.6, and upstream ciao still calls the Windows arp polling command without an options object, leaving Node's windowsHide default at false. The PR is a narrow current-main fix for the linked Windows console-flicker report, not conservative cleanup material.

## Best Possible Solution

Keep #71633 open for maintainer review. The best path is to land this narrow @homebridge/ciao@1.3.6 pnpm patch, or an equivalent upstream version bump if one becomes available, with package/lockfile consistency, a user-facing changelog entry, and either Windows manual verification or a focused regression check proving the ciao arp polling exec receives windowsHide: true.
