## Summary

Keep this PR open. Current main still ships the bundled Bonjour plugin through unpatched @homebridge/ciao@1.3.6, and upstream ciao's Windows arp polling exec still lacks a windowsHide options object. The linked Windows report #71492 gives a matching process trace and workaround, and the PR diff is a narrow patch that directly targets that dependency path. The broader #72332 remains open and is not a merged canonical superseder.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this narrow @homebridge/ciao@1.3.6 pnpm patch, or replace it with an upstream ciao version bump that demonstrably adds windowsHide to the Windows arp polling exec. If maintainers prefer the broader #72332 route, mark this PR superseded only after that broader fix is reviewed or merged.
