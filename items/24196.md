## Summary

Keep #24196 open. Current main still does not implement native multi-file Discord uploads for ordinary message actions or outbound reply delivery: those paths resolve one media source, pass one `mediaUrl`, and build a Discord payload with a single uploaded file. The existing maintainer-side comment also explicitly keeps the issue open, and the commits since that comment touched WhatsApp/release/doc surfaces rather than Discord attachment sending.

## Best Possible Solution

Keep the issue open and implement native multi-file sending in the bundled Discord plugin: add a backward-compatible ordered multi-attachment input shape for message actions and outbound payloads, preserve existing single `media`/`path`/`filePath` behavior, validate and split by Discord count/size limits, send eligible batches as one Discord `files` array, and update focused Discord tests and docs.
