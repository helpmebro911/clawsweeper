## Summary

Keep #24196 open. Current main still does not implement native multi-file Discord uploads for normal message actions or outbound replies: the Discord plugin accepts a single scalar media source, `sendMessageDiscord` accepts one `mediaUrl`, `sendDiscordMedia` constructs a one-entry `files` payload, and shared multi-media reply delivery still fans out into separate sends. This also matches the maintainer-side issue comment already posted on #24196.

## Best Possible Solution

Keep the issue open and implement native multi-file sending in the bundled Discord plugin: add a backward-compatible ordered multi-attachment input shape for message actions and outbound payloads, preserve existing single `media`/`path`/`filePath` behavior, validate and split by Discord count/size limits, send eligible batches as one Discord `files` array, and update focused Discord tests and docs.
