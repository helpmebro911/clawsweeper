## Summary

Keep #24196 open. Current main still exposes Discord outbound message media as a single `media`/`path`/`filePath` value and constructs one Discord upload file per send; component file uploads also explicitly reject multiple attachment references. The issue is clear, non-maintainer-authored, has no protected labels, and the requested native Discord multi-file send capability remains meaningful work.
