## Summary

Keep open. Current `main` still computes `usage.cost` from a single `loadCostUsageSummary(...)` call with no per-agent fan-out, while `sessions.usage` already discovers sessions across all gateway agents. That means the requested behavior is not implemented on `main`, and the linked mismatch remains plausible. The PR also has unresolved review concerns about all-or-nothing `Promise.all` failure behavior and is currently dirty against base, so it should stay open for maintainer judgment rather than cleanup-closing.
