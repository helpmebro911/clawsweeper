## Summary

Current main still does not show the session key or higher-level requester context in the macOS exec approval popup. The issue is recent, concrete, non-maintainer-authored, unprotected, and an existing maintainer-thread Codex review already kept it open; the checked-out main code still matches that finding.

## Best Possible Solution

Keep #9876 open. The narrow implementation should add a trimmed non-empty Session detail row to ExecApprovalsPromptPresenter and cover both socket/node and gateway prompt paths. The fuller product fix should extend the macOS prompt request model to carry safe requester context already present in the gateway approval payload, such as turnSourceChannel, turnSourceTo/account/thread or a derived sender label, then add focused macOS prompt/model tests so the UI parity does not regress.
