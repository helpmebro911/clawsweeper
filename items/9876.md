## Summary

Keep #9876 open. Current main still passes `sessionKey` into the macOS exec approval prompt paths and carries richer turn-source metadata in the gateway/TypeScript approval request, but the native Swift prompt request/presenter does not render a Session row or decode/display Channel/Sender/Thread context. The Control UI still shows the Session row, so the parity gap described by the issue remains. The existing Apr 26 Codex review comment from steipete also explicitly kept this item open; related #52368 appears to have solved session propagation, not the macOS display gap.

## Best Possible Solution

Keep #9876 open. The narrow fix belongs in `apps/macos/Sources/OpenClaw/ExecApprovalsSocket.swift`: trim `request.sessionKey` and render a non-empty `Session` detail row in `ExecApprovalsPromptPresenter.buildAccessoryView`. The fuller fix should extend the native prompt request model to decode safe turn-source/requester metadata from gateway approvals and display readable Channel/Sender/Thread context where available, with focused macOS coverage for gateway and node prompt paths.
