## Summary

Keep #9876 open. Current main still passes `sessionKey` into the macOS exec approval prompt model and uses it for prompt eligibility, but `ExecApprovalsPromptPresenter.buildAccessoryView` does not render a Session row. The gateway/TypeScript approval payload also carries turn-source fields, but the Swift prompt request model does not decode or display them, so the requested requester/channel context remains missing.

## Best Possible Solution

Keep #9876 open. The narrow fix belongs in `ExecApprovalsPromptPresenter.buildAccessoryView`: trim `request.sessionKey` and render a non-empty `Session` detail row. The fuller fix should extend `ExecApprovalPromptRequest` to decode safe turn-source/requester metadata from the gateway payload, display readable Channel/Sender/Thread context where available, and add focused macOS coverage for the gateway and node prompt paths.
