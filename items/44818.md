## Summary

Keep open. Current `main` still routes Gemini web search and Gemini memory embeddings through different HTTP guard modes: web search explicitly enables trusted env-proxy mode, while Gemini embeddings still use the default strict guarded-fetch path that ignores `HTTP_PROXY`/`HTTPS_PROXY` unless trusted env-proxy mode is selected. The same embedding path is present in release `v2026.4.22`, so there is not strong evidence this issue is already fixed on shipped code.

## Evidence

- **Gemini embedding requests still go through the generic memory remote HTTP helper:** `fetchGeminiEmbeddingPayload()` calls `withRemoteHttpResponse()` for Gemini embedding requests.
  - file: [extensions/google/embedding-provider.ts:165](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/google/embedding-provider.ts#L165)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **The memory remote HTTP helper does not enable env-proxy mode:** `withRemoteHttpResponse()` forwards to `fetchWithSsrFGuard()` with URL/init/policy only; it does not set `mode: TRUSTED_ENV_PROXY` or use any env-proxy wrapper.
  - file: [src/memory-host-sdk/host/remote-http.ts:6](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/memory-host-sdk/host/remote-http.ts#L6)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Guarded fetch defaults to strict mode and only uses env proxy in trusted env-proxy mode:** `resolveGuardedFetchMode()` falls back to `STRICT`, and dispatcher creation only uses `createHttp1EnvHttpProxyAgent()` when `mode === TRUSTED_ENV_PROXY` and proxy env vars are configured.
  - file: [src/infra/net/fetch-guard.ts:113](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/infra/net/fetch-guard.ts#L113)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Current tests codify that strict mode ignores env proxy by default:** `fetch-guard.ssrf.test.ts` has an explicit regression assertion: strict mode ignores env proxy, while trusted env-proxy mode routes through it.
  - file: [src/infra/net/fetch-guard.ssrf.test.ts:1007](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/infra/net/fetch-guard.ssrf.test.ts#L1007)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Gemini web search uses the trusted env-proxy path:** Gemini web search calls `withTrustedWebSearchEndpoint()`, which flows through `withTrustedWebToolsEndpoint()` and sets `useEnvProxy: true`, selecting trusted env-proxy mode.
  - file: [extensions/google/src/gemini-web-search-provider.runtime.ts:66](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/google/src/gemini-web-search-provider.runtime.ts#L66)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)

  - file: [extensions/google/embedding-provider.ts:165](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/google/embedding-provider.ts#L165)
