## Summary

Current main still does not treat whitespace-separated repeated silent-token payloads such as `NO_REPLY\n\nNO_REPLY` as silent. The shared predicate remains exact-token-or-JSON-envelope only, normalization strips only one trailing token and can leave a visible `NO_REPLY`, and plugin/runtime delivery paths inherit that shared predicate. The related Quiubo plugin guard is a useful workaround, but it does not fix OpenClaw's shared suppression behavior.

## Best Possible Solution

Keep this issue open and fix the shared silent-token predicate or normalization path in `src/auto-reply/tokens.ts` and `src/auto-reply/reply/normalize-reply.ts`. The implementation should classify token-only repeated sequences such as `NO_REPLY\n\nNO_REPLY` and `HEARTBEAT_OK\n\nHEARTBEAT_OK` as silent, flow through `isSilentReplyPayloadText` and runtime delivery plans, add regression tests, and preserve #19537 mixed-content semantics.
