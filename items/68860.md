## Summary

Keep open. Current main still drops embedded Pi `turn_start` / `turn_end` events in the subscription bridge, while the PR targets that exact missing event pipeline path. The work is not implemented, not superseded by the broader plugin-hook issue, and there is no missing core API blocker because `AgentEventStream` already permits additional stream names.

## Best Possible Solution

Keep #68860 open. The best path is to add a small turn handler in `src/agents/pi-embedded-subscribe.handlers.turn.ts`, wire `turn_start` and `turn_end` into `createEmbeddedPiSessionEventHandler`, emit both global `emitAgentEvent` and per-run `onAgentEvent` payloads with stable `stream: "turn"` data, include useful timestamps consistently, and add focused tests for callback and global bus delivery.
