## Summary

Keep open. Current main still drops embedded Pi `turn_start` and `turn_end` events at the OpenClaw subscription layer, and the PR targets a real observability gap. The PR is not mergeable as-is because its latest head partially applied the timestamp suggestion and left malformed TypeScript in the new turn handler.

## Best Possible Solution

Keep this PR open but request changes. The best path is a small repaired patch in `src/agents/pi-embedded-subscribe.handlers.ts` plus a turn handler that forwards `turn_start` and `turn_end` through both `emitAgentEvent` and `params.onAgentEvent`, with stable payloads such as `phase`, `startedAt`/`endedAt`, and `toolResultsCount`, plus focused regression tests for both delivery paths. No security-sensitive PR content needs removal.
