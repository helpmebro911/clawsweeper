## Summary

Keep open. Current main still lacks #63793’s central ACP lifecycle fixes: `/acp spawn` does not probe the freshly initialized runtime before reporting success, and stale `Gateway disconnected` failures are not treated as recoverable in close, cancel, active-turn cancel, or early-turn retry paths. The related `/acp` command-routing work is present on main, but that is a separate surface and does not supersede this PR. The latest PR discussion also leaves a real review concern around resolved unhealthy status results, so this needs maintainer review/iteration rather than cleanup closure.

## Best Possible Solution

Keep #63793 open for ACP/Gateway maintainer review. The best path is to land a narrowly scoped fix that adds a direct post-init spawn liveness/status check with cleanup, handles resolved unhealthy status results, and applies an explicit stale gateway-disconnect close-code allowlist across close, passive cancel, active-turn cancel, and early-turn retry, with the focused tests proposed in the PR.
