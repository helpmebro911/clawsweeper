## Summary

Keep open. Current main still lacks the requested per-channel or broader tool-output visibility policy, and the existing global suppression path still does not suppress mutating tool failures such as `message send` from chat delivery. Related PR #51091 remains open/unmerged and would address only the global `messages.suppressToolErrors` behavior, not the per-channel visibility control requested here.

## Best Possible Solution

Keep the issue open for a core runtime/config design: define a tool-output visibility policy with global defaults and per-channel overrides, apply it in embedded-run payload shaping and streaming chat delivery, keep tool errors available to the agent, update schema/docs, and add focused tests for mutating `message`/`exec` failures plus Telegram/Discord/Slack streaming delivery. Coordinate with #51091 rather than closing this broader request as fully tracked there.
