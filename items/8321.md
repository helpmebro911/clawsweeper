## Summary

Keep #8321 open. Current main still has no per-job `disableCache` or `contextPruning` field for cron jobs, and isolated cron runs still derive pruning/cache behavior from global/per-agent agent defaults. Related PR #8341 was closed unmerged, and #24597 tracks a broader direct script execution feature rather than this per-job cache override.

## Best Possible Solution

Keep #8321 open and implement an additive per-job `contextPruning` override, likely reusing `AgentContextPruningConfig` with `mode: "off" | "cache-ttl"`, then thread it through cron create/update/list schemas, CLI/UI surfaces, normalization, and isolated-run config construction. Add tests proving a job-level `mode: "off"` overrides global `agents.defaults.contextPruning.mode: "cache-ttl"`, and keep #24597 as a separate direct-script-execution track.

## Risks / Open Questions

- Users may confuse existing `--light-context` with cache disablement; current code only uses it to reduce bootstrap context for embedded isolated runs.
- A correct implementation needs coordinated updates across cron types, TypeBox protocol schemas, CLI/UI clients, store normalization/doctor behavior if needed, runner config merging, and regression tests.
