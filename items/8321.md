## Summary

Keep #8321 open. Current main still does not provide a per-job `disableCache` or `contextPruning` override for cron jobs. The cron job type, gateway validation, CLI/UI/macOS creation surfaces, and isolated runner all still use global/per-agent agent defaults for context pruning. Related PR #8341 attempted this but was closed unmerged, and #24597's direct script/no-LLM path is a separate cost-reduction idea rather than this job-level cache/pruning control.

## Best Possible Solution

Keep #8321 open and implement an additive per-job cron `contextPruning?: AgentContextPruningConfig` override using the existing `mode: "off" | "cache-ttl"` contract. Thread it through cron create/update/list schemas, persistence normalization, CLI, Control UI/macOS protocol/editor surfaces, docs, and agent-tool interfaces, then merge it into the effective `cfgWithAgentDefaults` before isolated cron invokes the embedded/CLI runner. Add regression coverage proving a job-level `mode: "off"` overrides global or per-agent `agents.defaults.contextPruning.mode: "cache-ttl"`. Track direct script/no-LLM cron execution separately.
