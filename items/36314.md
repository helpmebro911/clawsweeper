## Summary

Keep #36314 open. Current main has several related robustness mechanisms, but it still lacks the requested cohesive Gateway process watchdog, user-configurable global error recovery strategy map, richer health counters, and deliberate degraded-startup mode.

## Best Possible Solution

Keep the umbrella issue open or split it into focused maintainer-owned tasks: define a first-class `gateway.watchdog` config/schema/docs surface, specify Gateway process restart-loop detection and notification/auto-stop behavior above platform supervisors, decide whether user-configurable global recovery strategies are wanted, enrich privacy-safe Gateway health metrics, and design degraded startup behavior that composes with last-known-good config and existing channel health recovery.
