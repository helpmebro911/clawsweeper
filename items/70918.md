## Summary

Keep PR #70918 open. Current main still has the zh-Hans navigation drift the PR describes: English navigation has the nested Memory group with seven pages, while the zh-Hans overlay only exposes the Memory overview page. The source repo intentionally owns `docs/.i18n/zh-Hans-navigation.json` and the docs sync script reads that overlay for zh-Hans, so this is actionable in `openclaw/openclaw` rather than obsolete or external.

## Best Possible Solution

Keep the PR open and either merge a current, validated version of #70918 or apply the same scoped update directly: mirror the English nested Memory group in `docs/.i18n/zh-Hans-navigation.json`, keep the localized group label `记忆`, and run the docs/i18n validation lane before landing.
