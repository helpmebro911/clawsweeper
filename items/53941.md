## Summary

Current main still has the LiteLLM provider docs gap described by this issue: the manual setup command remains an unpinned `pip install 'litellm[proxy]'`, and there is no advisory/remediation text for the reported v1.82.7/v1.82.8 PyPI compromise. The bundled OpenClaw plugin does not vendor the PyPI LiteLLM package, so the codebase is not directly affected, but the requested user-facing docs update is still absent.

## Best Possible Solution

Keep this issue open and land or adapt the contributor's LiteLLM docs advisory in `docs/providers/litellm.md`, which publishes to `https://docs.openclaw.ai/providers/litellm`: replace the unpinned install example with current safe guidance, add a dated notice for the v1.82.7/v1.82.8 LiteLLM PyPI compromise, and include downgrade, key-rotation, and server-audit steps for users who ran an affected proxy.
