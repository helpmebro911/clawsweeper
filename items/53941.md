## Summary

Current main still has the exact gap reported by this issue: the LiteLLM provider docs show an unpinned `pip install 'litellm[proxy]'` command and there is no in-repo advisory or remediation text for the reported LiteLLM PyPI compromise. The bundled LiteLLM plugin is still only a network/proxy integration, so the right fix is documentation, not a runtime dependency change.

## Best Possible Solution

Keep this issue open. Land or adapt the contributor's requested docs change in `docs/providers/litellm.md`: replace the unpinned LiteLLM proxy install example with current safe guidance, add a dated notice for the reported v1.82.7/v1.82.8 PyPI compromise, and include downgrade, key-rotation, and server-audit steps for users who ran an affected self-hosted proxy. The public docs target is `https://docs.openclaw.ai/providers/litellm`.
