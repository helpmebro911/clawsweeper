## Summary

Keep #7661 open. Current main has generic `channel-create` vocabulary and Discord-owned channel creation plumbing, but the Slack-centered request and proposed `openclaw message channel create` CLI remain missing. Slack discovery does not advertise `channel-create`, Slack dispatch/runtime have no creation branch or `conversations.create` wrapper, Slack setup/docs do not include `channels:manage`, and CLI/docs still list only channel info/list.

## Best Possible Solution

Keep #7661 open. Implement Slack-owned `channel-create` support in `extensions/slack` with a documented `channels:manage` setup path, action gating/discovery, a `conversations.create` runtime wrapper, tests for dispatch/runtime/scopes, and an `openclaw message channel create` CLI mapping through the existing message-action runner. Preserve Discord's existing provider-specific channel management path.
