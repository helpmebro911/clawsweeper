## Summary

Keep open. Current main still exposes inline attachment fields on the shared `message` tool, but the `send` execution path does not hydrate `buffer`/`filename` and the Telegram plugin only forwards `mediaUrl`/`media` to `sendMessageTelegram`. Related #63578 remains an open implementation candidate, while earlier PRs for this bug are closed unmerged.

## Best Possible Solution

Keep this issue open until the shared send path and Telegram plugin either actually upload `buffer`/`filename`/`mimeType` attachments or explicitly reject those fields for `action="send"`. The likely code path is `src/infra/outbound/message-action-params.ts`, `src/infra/outbound/message-action-runner.ts`, `extensions/telegram/src/action-runtime.ts`, and `extensions/telegram/src/send.ts`, with focused regression tests for direct buffer sends and for safe media access boundaries. Review, update, and land or replace #63578 rather than closing the bug first.
