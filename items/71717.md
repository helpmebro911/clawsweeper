## Summary

Keep #71717 open. It is a fresh Windows-native bug report for a core exec failure (`spawn EPERM` on every command), and current main does not show a shipped/source-backed fix for that symptom. The current host exec path still resolves Windows exec through PowerShell and spawns it via the supervisor; existing Windows coverage verifies PowerShell selection, .cmd/npm shim handling, and close/exit behavior, but not an all-commands EPERM spawn failure.

## Best Possible Solution

Keep the issue open. Maintainers should request full exec/gateway logs, the exact command attempted, whether the gateway runs interactively or as a Windows Scheduled Task/service, the resolved PowerShell path, and any AppLocker/Defender/ACL details. A likely code path to investigate is Windows host exec shell resolution and supervisor child spawn error handling, adding targeted diagnostics or a conservative fallback only after reproducing the EPERM root cause.
