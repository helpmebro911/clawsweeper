## Summary

Keep #71717 open. It is a fresh Windows 11 report against 2026.4.23 where every agent exec call from Feishu or WebChat fails with `spawn EPERM`. Current main has Windows exec improvements for other failure modes, but I did not find source, tests, release notes, or a canonical related item showing this specific all-command EPERM spawn failure is fixed or superseded.

## Best Possible Solution

Keep the issue open and ask for a minimal Windows log bundle: exact command, full `spawn EPERM` stack/error object, resolved shell path, `openclaw config get tools.exec.host/security/ask`, gateway logs around the failed exec, and whether `pwsh.exe` or `powershell.exe -NoProfile -NonInteractive -Command "echo ok"` can be spawned by a small Node script. If reproduced, the likely fix belongs in `src/agents/shell-utils.ts`, `src/process/supervisor/adapters/child.ts`, or `src/process/spawn-utils.ts` with a Windows regression test for EPERM diagnostics/fallback rather than closing under the older Windows exec fixes.
