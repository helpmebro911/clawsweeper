## Summary

Keep #14344 open. Current main still does not implement WhatsApp message delete/revoke: WhatsApp advertises reactions and polls only, dispatch accepts only `react`, the runtime has no delete branch, and the active WhatsApp send API exposes send/poll/reaction/typing but no delete/revoke helper. The related #38723 is a closed duplicate-like report, not a shipped canonical fix.

## Best Possible Solution

Keep #14344 open as the canonical WhatsApp delete/revoke feature request. The likely implementation belongs in the bundled WhatsApp plugin: advertise `delete`, allow it in `supportsAction`, route it through the gateway-owned WhatsApp action runtime, add an active-listener delete/revoke helper using the Baileys socket, validate bot-owned/fromMe deletion and group participant keys, then add focused action/send API tests and update `docs/cli/message.md` plus `docs/channels/whatsapp.md`. Treat #38723 as the duplicate if maintainers want one tracker.
