## Summary

Keep #14344 open. Current main still does not expose or execute WhatsApp message delete/revoke: discovery advertises only react/poll, the WhatsApp action adapter accepts only react, the runtime handles only react, the active listener send API has no delete/revoke helper, and docs still exclude WhatsApp from the shared delete action. The related #38723 was closed as a duplicate of this canonical request, not as an implementation.

## Best Possible Solution

Keep #14344 open as the canonical bundled WhatsApp plugin feature request. Implement delete in WhatsApp message-action discovery and supportsAction, route it through the gateway-owned action runtime, add an active-listener delete/revoke helper using the Baileys message key, enforce bot-owned/fromMe deletion with group participant handling, then add focused runtime/send tests and update docs/cli/message.md plus docs/channels/whatsapp.md.
