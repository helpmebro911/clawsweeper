## Summary

Keep #14344 open. Current main still does not expose WhatsApp message delete/revoke through the bundled WhatsApp plugin: discovery advertises only `react`/`poll`, the adapter only supports `react` message actions, the runtime only handles `react`, the active listener send API has no delete/revoke method, and user-facing docs still exclude WhatsApp from shared `delete`. The related #38723 was closed as a duplicate of this canonical request, not as an implementation.

## Best Possible Solution

Keep #14344 open as the canonical bundled WhatsApp plugin feature request. The implementation should add `delete` to WhatsApp message-action discovery and `supportsAction`, route it through the gateway-owned action runtime, add an active-listener delete/revoke helper using the Baileys message key, enforce bot-owned/fromMe deletion with group participant handling, then cover the runtime/send API with focused tests and update `docs/cli/message.md` plus `docs/channels/whatsapp.md`.
