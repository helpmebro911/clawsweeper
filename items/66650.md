## Summary

Keep this PR open. Current main still has the reported `sessions_history` to `chat.history` limit mismatch: the tool accepts and forwards oversized limits, while the gateway rejects `chat.history.limit > 1000` before its internal clamp can run. The PR is a narrow implementation candidate for #66573, and the related alternative #66615 is closed unmerged rather than superseding it.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for #66573. The maintainer path is to request or add the missing execute-level runtime-clamp regression test, then review/land the narrow schema-plus-runtime clamp if CI is otherwise healthy. Do not close it as duplicate or obsolete while the linked bug remains open and #66615 is closed unmerged.
