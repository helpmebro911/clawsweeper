## Summary

Keep open. Current main still returns raw, unrecognized assistant error strings from `formatAssistantErrorText`, and that formatter output is sent into user-facing error replies. The PR is a plausible fix for the linked raw-error leak reports, but it should be refreshed before merge because its warning-log preview still needs stronger redaction and log sanitization.

## Best Possible Solution

Keep this PR open and ask for a small refresh against current main: preserve the generic user-safe catch-all for unrecognized assistant errors, keep actionable recognized-error branches ahead of it, redact before truncating the operator preview, pass the preview through `sanitizeForLog`, and keep the new orphaned `tool_result`/long-string regression tests.
