## Summary

Keep open. Current `main` and the latest shipped release still return the updated skill entry from `skills.update` without redacting secret-bearing fields, so the reported leak is still real and this PR is not safe to auto-close.

## Risks / Open Questions

- `skills.update` can still echo plaintext `apiKey` and secret `env` values back over the gateway response path.
- Arbitrary per-skill `config` data can include secret-like values, so any fix needs to handle more than just `apiKey`.
- There is no merged regression coverage today for redaction of the `skills.update` response payload.
