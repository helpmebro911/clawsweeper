## Summary

Keep #37661 open. Current main still has tool-call loop detection plus exact duplicate/event suppression, but I did not find an output-level guard for repeated assistant text fragments/ngrams or a per-turn outbound flood cap that would stop the reported pure text/channel spam. The Apr 11 comment is important because it confirms the issue is not limited to subagents and is distinct from #34574's tool-call loop. Closed #39536 is supporting evidence for the same missing output circuit breaker, not a canonical replacement.

## Best Possible Solution

Keep this issue open. Implement an output-level repetition guard in the assistant streaming/final reply pipeline, plus a per-turn outbound duplicate/message cap in shared delivery planning. Add focused tests for repeated deltas, repeated sentence/ngram fragments within one assistant message, repeated block-stream payloads, and repeated channel sends. Keep #34574 separate because it tracks tool-call loops, while #37661 tracks pure assistant text/output flooding.

- A text repetition guard can false-positive on legitimate repeated logs, code blocks, tables, or quoted text; thresholds should be conservative and configurable.
