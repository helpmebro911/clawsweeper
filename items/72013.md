## Summary

Keep open. Current main still reconciles any persisted ACP session whose identity is pending and has a stable ACPX session id, with no terminal one-shot or closed-session skip before runtime ensure/reconcile. The linked PR #72023 appears to be the right active fix candidate, but it is open/unmerged, so the original issue remains the canonical bug report until that or an equivalent fix lands.

## Best Possible Solution

Review and land PR #72023 or an equivalent narrow fix in `AcpSessionManager.reconcilePendingSessionIdentities`: skip terminal one-shot and closed ACP records before counting/checking runtime identity, and keep focused regression tests proving runtime ensure/status is not called for those records. The issue should close via the fixing PR once merged, not by ClawSweeper cleanup now.
