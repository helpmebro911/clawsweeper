## Summary

Keep open. Current `main` still reconciles every persisted ACP session whose identity is pending and has a stable ACP/agent session id; it does not skip terminal `mode: "oneshot"` or `state: "closed"` records before incrementing `checked` and potentially logging the startup warning. The related PR #72023 appears to target this exact bug, but it is still open/unmerged in the provided GitHub context.

## Best Possible Solution

Keep #72013 open until the narrow ACP manager fix lands on `main`: skip `session.acp.mode === "oneshot"` and `session.acp.state === "closed"` before pending-identity reconciliation, and add regression tests proving terminal one-shot and closed records return `{ checked: 0, resolved: 0, failed: 0 }` without invoking the runtime.
