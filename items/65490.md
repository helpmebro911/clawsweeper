## Summary

Keep this issue open. Current main has useful partial work for fallback notices, /status fallback display, cron delivery previews, and cron CLI syntax, but it still does not implement the central production-operations requests: fail/mark-degraded startup primary-model reachability before accepting turns, a compact agent-readable runtime model/fallback/last-error diagnostic surface, or cron add dry-run/preflight that fails closed on unresolved announce delivery before persistence.

## Best Possible Solution

Keep this open but narrow the maintainer path: add a configurable startup/provider preflight that live-probes the configured primary model and either blocks startup or marks the gateway degraded; add a safe agent-visible diagnostics RPC/tool exposing selected model, active responding model, fallback state, and last provider/fallback error category/code; and add cron add dry-run/preflight plus fail-closed announce target validation before job persistence. Treat #56852, #58303, #63900, #56996, #57045, and #57046 as related completed subwork, not a substitute for the remaining production execution-discipline work.
