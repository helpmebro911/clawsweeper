## Summary

Keep this issue open. Current main has meaningful partial coverage for fallback visibility and cron delivery previews, but the central production operations requests remain: startup primary-model reachability is still warning-only, there is no compact agent-readable runtime model/fallback/last-provider-error diagnostic surface, and cron add still lacks dry-run/preflight plus fail-closed announce-target validation before persistence.

## Best Possible Solution

Keep this open but narrow the maintainer path: add a configurable startup/provider preflight that can live-probe the configured primary model and either block startup or loudly mark the gateway degraded; add a safe agent-visible diagnostics RPC/tool with selected model, active/responding model, fallback state, and last provider/fallback error category/code; and add `cron add` dry-run/preflight plus fail-closed announce delivery target validation before job persistence. Treat closed related fallback and cron issues as completed subwork, not proof that this production execution-discipline tracker is done.
