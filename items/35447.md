## Summary

Keep #35447 open. Current main still records cron ownership metadata and stamps new jobs from the agent cron tool, but the cron service contract, service ops, gateway protocol, gateway handlers, and cron tool calls still do not carry caller identity or enforce per-agent/session filtering for list/update/remove/run/enqueueRun. The related implementation PRs #36517 and #49174 are closed unmerged, while #49175 is only a narrower follow-up about cron.runs job-name leakage rather than a canonical replacement for the full isolation contract.

## Best Possible Solution

Keep #35447 open and implement a core cron caller-context contract across the agent cron tool, gateway protocol/schema, gateway server handlers, CronService contract, and cron service ops. Add focused regression tests for list/listPage filtering plus update/remove/run/enqueueRun denial across agent/session boundaries, include cron.runs job-name visibility, and document legacy-job plus admin override behavior.
