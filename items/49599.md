## Summary

Keep #49599 open. Current main does not implement custom HTTP headers for CLI Gateway WebSocket connections: there is no `--header` option, no `OPENCLAW_GATEWAY_HEADERS` handling, no `gateway.remote.headers` config surface for Gateway remote settings, and `GatewayClient` still constructs `ws` client options without user-supplied headers. The related PR #49646 was closed unmerged, so it does not supersede the issue.
