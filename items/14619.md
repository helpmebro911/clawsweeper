## Summary

Keep #14619 open. Current main still builds a human-readable Tooling section from runtime tool names while separately serializing callable tool definitions with descriptions and parameters. I found no scoped skipToolListText-style config, and the existing full system prompt override or promptMode none are broader than the requested optimization. Related #14785 is broader tool-schema overhead work, not a canonical replacement for this narrower request.

## Best Possible Solution

Keep the issue open until OpenClaw adds a scoped agent/default config, such as agents.defaults.skipToolListText, that suppresses only the human-readable Tooling list while preserving structured callable tool schemas. The implementation should update prompt builder/run plumbing, config types/zod/generated schema/help/docs, /context reporting, and focused tests proving both prompt omission and continued tool schema availability.

## Risks / Open Questions

- systemPromptOverride can remove Tooling only by replacing all generated prompt guidance, so it is not equivalent to the requested scoped option.
- promptMode none/minimal are runtime prompt-shape controls and either keep Tooling or remove far more than the requested tool-list text.
- The current context report path still prints a Tool list line while buildSystemPromptReport hardcodes listChars to 0, so a real fix should align prompt generation and /context accounting.
