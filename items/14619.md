## Summary

Keep #14619 open. Current main still builds an in-band Tooling section with human-readable tool names and summaries while also sending structured tool definitions with descriptions and parameter schemas. The only nearby config is a full `systemPromptOverride`, not a scoped option to omit just the tool list text; the provided April 26 maintainer-side Codex comment already kept this exact issue open and current HEAD still matches that finding.

## Best Possible Solution

Keep #14619 open until OpenClaw adds a scoped agent/default config such as `skipToolListText` that omits only the human-readable Tooling list while preserving callable structured tool schemas. A complete fix should update the prompt builder and embedded runner plumbing, config types/zod/generated schema/help/docs, `/context` reporting if needed, and focused tests proving schemas remain available while the in-band list is omitted. If maintainers decide `systemPromptOverride` is the intended boundary, that should be stated explicitly on the issue.
