## Summary

Keep open. Current main still implements inbound audio as media-understanding/STT that produces transcript text for the reply pipeline; it does not expose raw audio as a native LLM prompt modality or have a model capability contract for audio input. The related PR #48900 is also described in the provided context as still open because native audio/video forwarding is not implemented on main.
