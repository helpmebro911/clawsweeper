## Summary

Keep this issue open. Current main still enqueues a 160-character inbound-body preview into trusted system events for Slack, MS Teams, and Mattermost while separately delivering the full inbound body to the agent envelope. Open PR #67761 by the same author still references this issue with closing syntax, so the maintainer path is to review and either land or close that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and review PR #67761. The likely product fix belongs in the Slack, MS Teams, and Mattermost channel handlers: keep the full inbound content in the normal agent envelope, remove message-body previews from notification-only system events, and add focused regression tests proving long inbound bodies are not duplicated into trusted `System:` lines.
