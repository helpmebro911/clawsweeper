## Summary

Keep PR #27771 open. Current main still does not implement `channels.signal.configPath`, and the requested `signal-cli --config <path>` behavior remains concrete and actionable for the bundled Signal plugin. The existing PR is stale and targets old `src/signal/*` paths, but the contributor explicitly said maintainers may take it over.

## Best Possible Solution

Keep the PR open for maintainer takeover or replacement. Implement optional `channels.signal.configPath` in the current bundled Signal plugin layout: add it to the exported config type, strict schema, generated metadata/UI hints, docs, and account configured detection; trim and omit blank values; propagate it from resolved account config through `monitorSignalProvider` into `spawnSignalDaemon`; emit `--config <path>` before daemon startup args; and add focused Signal schema/autostart tests for set, blank, and unset cases.
