## Summary

Keep PR #27771 open. Current main still does not implement `channels.signal.configPath` or pass `signal-cli --config <path>` during Signal daemon startup. The PR is stale against the current `extensions/signal/*` layout, but the requested bundled Signal plugin behavior remains concrete and the author explicitly invited maintainer takeover.

## Best Possible Solution

Keep the PR open for maintainer takeover or replacement. Port the feature to the current bundled Signal plugin: add optional `channels.signal.configPath` to the exported config type, strict schema, config UI hints/help/docs, account configured detection, and autostart tests; trim and omit blank values; propagate the value through `monitorSignalProvider` into `spawnSignalDaemon`; and emit `--config <path>` before the daemon subcommand.
