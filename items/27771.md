## Summary

Keep open. Current `main` at `1252da325f2b5afa354cfb8ff94ae61783005060` still does not implement `channels.signal.configPath` or pass `--config` to Signal daemon startup. The PR is stale against the current bundled plugin layout, but the requested behavior is valid and the latest PR discussion says the contributor cannot continue, so this is takeover/adaptation work rather than cleanup closure.

## Best Possible Solution

Keep the PR open or let a maintainer take it over. The best implementation is to add optional `channels.signal.configPath` to the current Signal config type, strict zod schema, help/labels/generated channel metadata, and docs; trim and omit blank values; propagate the value through `monitorSignalProvider` into `spawnSignalDaemon`; emit `signal-cli --config <path>` before daemon startup arguments; and add focused Signal autostart tests for set, blank, and unset values.
