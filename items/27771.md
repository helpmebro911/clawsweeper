## Summary

Keep this PR open. Current main still lacks `channels.signal.configPath` in the bundled Signal plugin config contract, schema, docs, and daemon startup path. The submitted branch is stale because it targets the old `src/signal/*` layout, but the feature remains concrete and the author explicitly invited maintainer takeover.

## Best Possible Solution

Keep this PR open for maintainer takeover or a replacement patch. Port the feature to the current bundled Signal plugin by adding optional `channels.signal.configPath` to the Signal config type/schema and discoverability surfaces, propagating it through account resolution and `monitorSignalProvider` into `spawnSignalDaemon`, and adding focused tests that verify `--config <path>` is emitted as separate argv elements only for nonblank values.
