## Summary

Keep PR #27771 open. Current main still does not implement `channels.signal.configPath` for Signal: the strict Signal config schema and type omit the key, daemon startup has no `configPath` option and never emits `--config`, monitor does not propagate a config path, and the Signal docs/UI surface do not list the setting. The existing PR is useful but targets the removed pre-pluginized `src/signal` layout; the author has also invited maintainer takeover.

## Best Possible Solution

Keep the PR open for maintainer takeover or a replacement PR. Port the feature to `extensions/signal`: add optional `channels.signal.configPath` to the current Signal config type/schema and metadata/UI hints, document it on the public Signal docs page, include it in configured-state detection, trim and omit blank values, propagate it through `monitorSignalProvider` into `spawnSignalDaemon`, emit `--config <path>` before daemon startup arguments, and add focused tests for set, blank, unset, schema acceptance, and daemon argv behavior.
