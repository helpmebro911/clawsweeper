## Summary

Keep this PR open. Current main still lacks a heartbeat-specific skill allowlist: the strict heartbeat schema and exported config type do not expose `allowSkills`, `runHeartbeatOnce` does not pass a heartbeat-derived `skillFilter`, and the current docs/inventory do not list the key. The existing reply path already supports `skillFilter`, so the PR is a focused additive implementation candidate rather than obsolete cleanup. Security pass: the provided PR diff is limited to docs, config schema/help/labels, generated schema, tests, and heartbeat runner plumbing; it does not introduce dependency, workflow, package, lockfile, artifact download, or secret/permission changes.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this PR or an equivalent small additive change that adds `agents.defaults.heartbeat.allowSkills` and `agents.list[].heartbeat.allowSkills`, preserves unset versus empty-array behavior, passes the merged heartbeat value into `getReplyFromConfig` as `skillFilter`, and keeps the Zod schema, exported types, generated schema/help/labels, docs, config-honor inventory, and targeted regression tests aligned.
