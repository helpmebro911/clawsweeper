## Summary

Keep this PR open. Current main still does not implement a heartbeat-specific skill allowlist: heartbeat config rejects `allowSkills`, the TypeScript config surface lacks it, and the heartbeat runner does not pass a `skillFilter` into reply runs. The provided PR context and Greptile review show a focused additive implementation candidate with docs, schema, runtime threading, and tests, so maintainer review/rebase is the right cleanup path.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is a small additive config surface for `agents.defaults.heartbeat.allowSkills` and `agents.list[].heartbeat.allowSkills`, threaded into `getReplyFromConfig` as `skillFilter` while preserving unset versus `[]`, with Zod schema, exported types, generated config help/labels/schema, docs, config-honor inventory, and regression tests kept aligned. Close only after this PR or a newer equivalent implementation lands.
