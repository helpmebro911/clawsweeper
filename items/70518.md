## Summary

Keep PR #70518 open. Current main still has no heartbeat-specific skill allowlist: the strict heartbeat schema, TypeScript config type, user-facing docs, config-honor inventory, runtime reply options, and regression tests all omit `agents.defaults.heartbeat.allowSkills` / `agents.list[].heartbeat.allowSkills`. The PR is a focused additive core config/runtime change, and the existing reply path already has a `skillFilter` merge contract that makes the proposed implementation shape plausible rather than blocked by a missing core API.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent narrow patch: add `allowSkills?: string[]` to the heartbeat schema/type/help/labels/generated config metadata and docs, thread merged defaults/per-agent heartbeat `allowSkills` into `getReplyFromConfig` as `skillFilter`, preserve unset behavior, preserve `[]` as disabling heartbeat skills, and keep focused schema/runtime/inventory tests covering defaults and per-agent overrides.
