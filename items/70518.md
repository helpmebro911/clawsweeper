## Summary

Keep open. Current main does not implement `agents.defaults.heartbeat.allowSkills` or `agents.list[].heartbeat.allowSkills`, and the existing heartbeat runner still calls `getReplyFromConfig` without a heartbeat-specific `skillFilter`. The PR targets a real existing seam because the reply layer already supports `skillFilter`, including empty arrays for no skills.

## Best Possible Solution

Keep the PR open or land an equivalent narrow change: add `allowSkills?: string[]` to heartbeat config schema/types/docs/generated metadata, thread the resolved heartbeat allowlist into `getReplyFromConfig` as `skillFilter`, and keep regression coverage for unset, non-empty, and empty allowlists.
