## Summary

Keep PR #70518 open. Current main still lacks a heartbeat-specific skill allowlist: the heartbeat runner does not pass a `skillFilter`, the strict heartbeat config schema/types/generated schema do not accept `allowSkills`, and the docs/tests/inventory do not document or lock in that field. The existing `agents.defaults.skills` / `agents.list[].skills` mechanism is only the normal agent skill filter, so it does not solve the PR's narrower heartbeat-only prompt-size problem.

## Best Possible Solution

Keep #70518 open and review or land a scoped version that adds `agents.defaults.heartbeat.allowSkills` and `agents.list[].heartbeat.allowSkills` through the config type, strict schema, generated schema/help/labels, docs, and heartbeat config-honor inventory, then passes the merged heartbeat allowlist as `skillFilter` into `getReplyFromConfig`. Preserve current behavior when unset and cover both non-empty and empty allowlists in runner/schema tests.
