## Summary

Keep open. Current main still does not expose `--thread-id` on `openclaw cron add/create` or `openclaw cron edit`; those commands only accept `--channel`, `--to`, and `--account` for delivery targeting. The underlying cron/delivery path can already carry a parsed `threadId` from `:topic:` targets, and docs mention that manual target-string workaround, but the requested CLI flag remains missing on main. The related PRs #51581 and #60890 directly track the same fix, but they are still open/unmerged, so this issue remains the canonical user-facing request rather than cleanup-ready.
