## Summary

Keep this issue open. Current main still lacks first-class `--thread-id` support on `openclaw cron add/create` and `openclaw cron edit`; Telegram topic delivery is only available through manual `:topic:` target encoding, while `openclaw message send` already exposes the flag. PR #60890 is an open, unmerged implementation candidate with closing syntax for this issue, so maintainers should review, land, or close that PR before closing the issue.

## Best Possible Solution

Keep this issue open while maintainers review #60890. The best path is a focused cron CLI change that adds `--thread-id` to `cron add/create` and `cron edit`, composes Telegram topic targets as `<chatId>:topic:<threadId>`, preserves existing manual topic targets, rejects invalid combinations early, and adds regression coverage in `src/cli/cron-cli.test.ts`.
