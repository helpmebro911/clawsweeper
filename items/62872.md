## Summary

Keep this issue open. Current main only skips `sessions_send` A2A for a parent sending to its own one-shot ACP child; persistent ACP sessions and normal peer/persistent sessions still run the ping-pong plus announce path, and the tool still has no per-call `maxPingPongTurns` or disable option. The merged #69817 work is relevant but intentionally narrower, while #62814 was closed as a duplicate of this issue.

## Best Possible Solution

Keep this issue open as the canonical tracker for persistent-session `sessions_send` duplicate behavior. The likely fix belongs in `src/agents/tools/sessions-send-tool.ts` and `src/agents/tools/sessions-send-tool.a2a.ts`: either make A2A context-aware for persistent peer-session sends or add an explicit per-call ping-pong max/disable option, while preserving the #69817 parent-owned one-shot ACP child skip and adding regression coverage for persistent peer-session duplicates.

## Risks / Open Questions

- A global `session.agentToAgent.maxPingPongTurns=0` setting exists, but it disables ping-pong chaining everywhere rather than solving persistent peer-session sends per call or per context.
- A broad skip could break legitimate peer A2A follow-up and visible announce workflows, so the fix should distinguish persistent peer-session sends from the already-shipped parent-owned one-shot ACP child skip.
