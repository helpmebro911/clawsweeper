## Summary

Current main still leaves the reported persistent-session `sessions_send` ping-pong behavior unresolved. The merged ACP-child fix only skips A2A for parent-owned one-shot ACP child sessions, while peer and persistent targets still enter the generic A2A follow-up path. The tool schema still has no per-call ping-pong disable/max control, and the documented knob is only the global `session.agentToAgent.maxPingPongTurns` setting.

## Best Possible Solution

Keep this issue open as the canonical tracker for persistent-session `sessions_send` A2A behavior. The likely fix belongs in `src/agents/tools/sessions-send-tool.ts` and `src/agents/tools/sessions-send-tool.a2a.ts`: add a per-call ping-pong max/disable option or make A2A context-aware so persistent peer-session sends return the direct inline reply without framework-generated announce/ping-pong turns by default, while preserving the existing parent-owned one-shot ACP child skip and adding regression tests for peer persistent sessions.
