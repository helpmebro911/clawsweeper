Keep this PR open. Current main still lacks the Feishu in-place card update behavior, and the PR remains a plausible implementation candidate, though it needs a narrower review/rebase because the patch also carries unrelated Feishu reply-dispatcher and monitor comment test changes.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best maintainer path is to ask for a focused rebase that only wires Feishu card update payloads through card-interaction/card-ux-shared, parses card-action message_id in monitor.account, calls updateCardFeishu from card-action with failure isolation, and adds targeted tests for confirm, cancel, quick actions, malformed uc, and missing message_id behavior. If maintainers prefer not to use this branch, open or land a narrower replacement before closing it.
