## Summary

Keep open. Current main still builds the gateway service PATH from a curated/minimal service-path heuristic, not from the current login-shell PATH, and the issue is recent, concrete, unlabeled, and author association is NONE. The discussion correctly narrows the likely root cause to buildMinimalServicePath rather than getShellPathFromLoginShell, and the reporter’s workaround remains relevant.

## Evidence

- **Current service PATH is heuristic/minimal:** src/daemon/service-env.ts defines addCommonUserBinDirs, resolveDarwinUserBinDirs, getMinimalServicePathParts, and buildMinimalServicePath. The macOS service PATH is assembled from env-configured roots, common user bin directories, version-manager defaults, and system dirs; it is not sourced from /bin/zsh -l -c 'echo $PATH'. The common user dirs include ~/bin but not the reported ~/.bin or ~/.cargo/bin.
