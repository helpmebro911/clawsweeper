## Summary

Keep open. Current `main` still contains both reported paths: Dreaming narrative cleanup still routes through admin-gated `sessions.delete`, and the managed Dreaming cron reconciler still reuses a startup-time cron getter while the current tests intentionally preserve the runtime `cron service unavailable` warning. The issue is also recent and was reconfirmed in follow-up comments, so there is not enough evidence for a conservative close.
