## Summary

Keep this PR open. Current main still skips bundled pre-install resolution for scoped npm specs, and the existing test suite explicitly asserts that old behavior. The PR remains a focused implementation candidate, with a small product/docs question around whether exact official scoped specs should enable bundled plugins instead of installing npm copies.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or adapt the narrow helper change if maintainers want exact bundled npmSpec matches to win before npm: trim the raw spec, keep plugin-id lookup for bare specs, use exact `npmSpec` lookup for scoped/non-bare specs, let non-matches continue to ClawHub/npm, update the helper tests, and adjust PR/docs wording to match current duplicate-precedence policy.

## Risks / Open Questions

- Landing this changes the default for scoped specs that exactly match a bundled plugin npmSpec; maintainers should confirm that `openclaw plugins install @openclaw/<plugin>` should enable the bundled copy instead of installing the npm package in that exact-match case.
- The PR body's runtime-precedence explanation appears stale against current main, where explicit installed globals can beat bundled duplicates even though auto-discovered globals do not.
- Docs may need a small update, especially around the distinction between a different scoped npm package and an official scoped spec that exactly matches bundled metadata.
