## Summary

Keep #9443 open. Current main still does not publish official prebuilt Android APKs in GitHub releases: public Android docs explicitly say the app has not been publicly released, Android release tooling builds signed AAB bundles rather than release APK assets, CI only validates/debug-builds the Play APK, and there is no Android release workflow. The discussion’s community APK/client links are useful workarounds, but they do not satisfy the request for official OpenClaw APK releases.

## Best Possible Solution

Keep #9443 open as the canonical official Android APK release request. A complete fix should define the signed APK/flavor and distribution policy, add maintainer-approved release automation that uploads official APK assets to GitHub releases or an explicitly chosen Android distribution channel, update https://docs.openclaw.ai/platforms/android with the official install path, and continue redirecting duplicate APK-release requests here until that ships or maintainers explicitly decide against official APK distribution.
