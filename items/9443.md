## Summary

Keep #9443 open. Current main still does not provide official prebuilt Android APK release downloads: the public Android docs say the app has not been publicly released, Android release tooling builds signed AAB bundles, CI only validates a Play debug APK build, and the latest GitHub release assets contain only macOS artifacts. Community APK/client links in the discussion are useful workarounds, but they do not satisfy the request for official OpenClaw APK releases.

## Best Possible Solution

Keep #9443 open as the canonical official Android APK release request. A complete fix should define the supported signed APK/flavor and distribution policy, add maintainer-approved release automation or an explicitly chosen distribution channel, upload official Android install assets where users are told to find them, and update https://docs.openclaw.ai/platforms/android with the official install path. Continue redirecting duplicate APK-release requests here until that ships or maintainers explicitly decide not to distribute APKs.
