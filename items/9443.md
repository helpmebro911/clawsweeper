## Summary

Keep open. Current main still does not provide official prebuilt Android APK release assets. The Android docs now warn that the app has not been publicly released, CI only validates a scoped Play debug APK build, release helper code produces signed AAB bundles rather than GitHub APK release assets, and the latest public release assets for v2026.4.24 contain no APK. Related community work and open/closed APK workflow PRs do not supersede this older canonical request.

## Best Possible Solution

Keep #9443 open as the canonical request until OpenClaw either ships an official Android APK release asset workflow or a maintainer explicitly decides not to distribute APKs. A complete implementation likely belongs in release infrastructure, with signed flavor policy, release asset upload, docs updates, and release checks aligned with the Android app readiness path.
