## Summary

Keep PR #66542 open. Current main still does not implement bundle MCP reconnect/retry/jitter/dead-server tracking or parallel startup, and the open PR is a focused attempt at that missing behavior. It should not be auto-closed, but it also should not merge as-is while the review comments on reconnect concurrency and JSON-RPC error classification remain unresolved.

## Best Possible Solution

Keep the PR open and ask for a revised implementation or replacement PR that adds bounded MCP startup concurrency, explicit retry/backoff/jitter, dead-state resurrection, and in-flight reconnect joining, while using a narrow transport-error classifier and secret-safe catalog/log metadata. Add focused regression tests for startup retry, `callTool()` reconnect, concurrent callers during reconnect, JSON-RPC server errors that must not reconnect, and disposal/resource cleanup.
