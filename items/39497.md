## Summary

Keep #39497 open. Current main still has the reported `/agent` read-guard gap in the sandboxed read path when the workspace-root guard is active: the guard maps only `containerWorkdir`, the read-tool wiring passes only that option, and current regression coverage still expects `/agent/secret.txt` to fail with `Path escapes sandbox root`.

## Best Possible Solution

Keep the issue open. The best fix belongs in the sandboxed file-tool guard/wiring: validate through the sandbox mount table, or pass an explicit `/agent` mapping only for the `workspaceAccess: "ro"` agent workspace mount. Add regression coverage proving `/agent/...` reads resolve in the intended read-only path while write/edit/apply_patch remain blocked. Treat `/tmp` reads as a separate policy decision.
