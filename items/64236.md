## Summary

Keep this issue open. Current main still does not define, validate, document, or consume a per-model `reasoningDefault` under `agents.defaults.models[...]`. The code has per-agent `reasoningDefault`, global/per-model thinking defaults, and a catalog-derived reasoning visibility fallback, but that is not the requested per-model reasoning visibility configuration. The related open PR #71817 is only a partial Telegram/Feishu per-agent fix and does not supersede this request.

## Best Possible Solution

Add a first-class per-model reasoning visibility config, likely `reasoningDefault?: "on" | "off" | "stream"` on `AgentModelEntryConfig` / `agents.defaults.models[modelRef]`, then update Zod, generated schema/help/docs, and reply/model-selection resolution precedence. Maintainers should also decide whether the currently allowlisted but unsupported `agents.defaults.reasoningDefault` should become a real global default or be removed from the gateway-tool allowlist.
