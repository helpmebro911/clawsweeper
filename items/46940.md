## Summary

Keep PR #46940 open. Current main still does not preserve target-derived `toolContext` through exec-approval forwarded outbound delivery, and the PR addresses a real remaining gap. The branch likely needs refresh/completion rather than closure, especially for queued delivery replay.

## Best Possible Solution

Keep the PR open and refresh it against current main. A complete fix should add `toolContext?: ChannelThreadingToolContext` to the current outbound adapter contract, thread it through `DeliverOutboundPayloadsParams` / `ChannelHandlerParams` / `createChannelOutboundContextBase(...)`, derive it from each exec-approval forward target, preserve it through queued delivery storage and recovery replay, and add regression tests for direct forwarded request/resolved deliveries plus queue replay.
