## Summary

Keep open. Current main has explicit Discord bound-thread webhook echo suppression, but it still does not implement the reported narrow PluralKit original/proxy collapse: suppression is keyed to matching bound webhook identity, generic inbound dedupe is still message-ID based, and the paired implementation PR #52007 is closed unmerged.

## Best Possible Solution

Keep this issue open. The best fix is in the Discord plugin preflight/process path: add a regression that simulates a bound thread receiving the original Discord message plus a PluralKit proxy copy with different message ids and missing or nonmatching webhook metadata, then collapse that pair before dispatch. Prefer an explicit PluralKit/Discord canonical signal early in preflight; if that is unavailable, use a narrow short-lived bound-thread/account/agent-session/content fallback and ensure suppressed duplicates do not start or keep typing indicators alive.

## Evidence

It returns false when the webhook id is missing or differs, which leaves the PluralKit original/proxy pair shape described in this issue uncollapsed unless the webhook metadata happens to match.
