## Summary

Keep open. Current main still does not expand `${VAR}` or `$VAR` in the shared user-path resolver, and an open implementation PR with closing syntax is already linked for this issue.

## Best Possible Solution

Keep this issue open. Review and either land or close PR #58281, reconciling duplicate PR #53718 as needed. The likely fix belongs in the shared user-path resolution path with regression tests for `${VAR}` and `$VAR` expansion so ClawHub skill installs do not write under literal env-placeholder directories.

## Evidence
