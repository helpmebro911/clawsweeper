## Summary

Keep open. Current main still contains the exact hard-coded transport-stream error throws this PR changes, and the existing tests do not cover preserving `output.errorMessage` on the errored finalization path. The related closed issues/fixes do not supersede this narrow transport-message propagation gap.

## Best Possible Solution

Keep the PR open for maintainer review. The likely path is to rebase it onto current main, preserve `output.errorMessage` in the shared finalizer and both Responses variants, keep the fallback for missing messages, add the proposed unit coverage, and consider whether native Google/OpenAI-compatible finish-reason propagation needs a companion fix before merge.
