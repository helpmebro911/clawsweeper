## Summary

Keep open. Current main partially addresses the original raw-read concern with a bespoke `readLocalBuffer` path, but it has not landed the PR's shared `readLocalFileSafely` integration and still writes attachment temp files without an explicit `0o600` mode. No canonical related item in the provided context supersedes those remaining changes.

## Best Possible Solution

Keep the PR open or ask the contributor to rebase it onto current main. The best product/code path is to either route the attachment cache through `readLocalFileSafely` while preserving local-root checks and `MediaUnderstandingSkipError` mapping, or deliberately extend the current `readLocalBuffer` implementation to match the shared helper's identity checks. The temp-file write in `getPath()` should also use `{ mode: 0o600 }`, with focused tests for temp permissions and any oversized-file error mapping.
