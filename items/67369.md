## Summary

Keep open. Current main partially covers the request with agent scoping, `--active <minutes>`, and JSON output, but it still lacks the requested duration-style stale/newer filters, size-threshold filter, and list-row byte-size metadata needed to implement `--larger-than`.

## Best Possible Solution

Keep this issue open and implement the remaining core CLI/protocol work deliberately: add duration parser-backed `--older-than` and `--newer-than` filters, define whether `--larger-than` means transcript bytes, session-store bytes, or token count, expose that metadata consistently in CLI JSON, gateway `sessions.list`, and `sessions_list`, then update `docs/cli/sessions.md` plus focused parser, command, protocol, and tool tests.
