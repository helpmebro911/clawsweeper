## Summary

Keep #10659 open. Current main has useful partial hardening, including SecretRef storage/resolution, model-auth marker persistence, host exec environment sanitization, configurable redaction heuristics, and plugin hook seams that support the linked workaround/plugin approaches. It still does not ship the requested native masked-secrets feature: no `{{secret:NAME}}` command substitution, no `secrets.mask` or `maskedSecrets` config contract, no exact mask-list redaction of arbitrary exec/process output before it reaches the model, and no generic command/file policy preventing an agent from reading otherwise-readable env files. The latest provided maintainer-associated issue comment also explicitly kept this issue open.

## Best Possible Solution

Keep #10659 open until OpenClaw ships a native masked-secret or credential-provider path for agent-executed commands and process output, or maintainers explicitly supersede it with a canonical design such as #59165. Continue acknowledging `jmkritt/openclaw-secrets-hardening` and `openclaw-credential-vault` as useful mitigations that can run today, not as closure evidence for the core request.
