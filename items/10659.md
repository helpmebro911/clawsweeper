## Summary

Keep #10659 open. Current main has useful SecretRef storage/resolution, model-auth markers, security audit/scrub paths, host exec env sanitization, and plugin hook seams that support the linked `jmkritt/openclaw-secrets-hardening` workaround and `openclaw-credential-vault` plugin approach. It still does not ship the native masked-secrets feature requested here: no `{{secret:NAME}}` command substitution, no `secrets.mask`/`maskedSecrets` config, no native mask-list redaction of arbitrary exec/process output before it reaches the model, and no generic block preventing an agent from reading otherwise-readable `.env` files or arbitrary environment variables. The latest provided maintainer-associated comment also explicitly kept this issue open and treated those linked mitigations as useful but not closure evidence.

## Best Possible Solution

Keep #10659 open until OpenClaw ships a native masked-secret/credential-provider path for agent-executed commands and process output, or maintainers explicitly supersede it with a canonical design such as #59165. Continue acknowledging `jmkritt/openclaw-secrets-hardening` and `openclaw-credential-vault` as useful mitigations that can run today, not as closure evidence for the core request.
