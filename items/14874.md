## Summary

Keep #14874 open. Current main still lacks an optional SIE warn/strict verification gate at the skill loader boundary, and the directly related implementation PR #14893 was closed unmerged. Existing protections cover path containment, skill install scanning, ClawHub archive metadata, and plugin install hooks, but none decide unsigned/valid-signed/invalid-signed skill status before skills become prompt-, command-, snapshot-, or sandbox-visible.

## Best Possible Solution

Keep #14874 open. The best path is a disabled-by-default core loader verification policy, or a first-class loader-time verification hook/API, that produces deterministic warn/strict outcomes for unsigned, valid-signed, and invalid-signed skills before they are exposed to prompts, skill commands, snapshots, sandbox sync, or runtime use. Existing install and ClawHub protections should remain adjacent safeguards rather than being treated as this feature.
