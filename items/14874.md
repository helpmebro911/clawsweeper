## Summary

Keep #14874 open. Current main still does not provide an optional SIE warn/strict verification gate in the skill loader, the directly related implementation PR #14893 was closed unmerged, and an owner comment on the issue already reached the same conclusion on April 26, 2026. The relevant skill loading, workspace exposure, type, and config surfaces still load and expose skills without signed/unsigned/invalid-signed status or enforcement.

## Best Possible Solution

Keep #14874 open. The best path is a disabled-by-default core skill-loader verification policy, or a first-class loader-time verification hook/API, that makes deterministic warn/strict decisions for unsigned, valid-signed, and invalid-signed skills before they are exposed to prompts, commands, snapshots, sandbox sync, or runtime use. The closed PR #14893 can be used as prior art, but the current repo still needs an implementation aligned with OpenClaw's TypeScript loader/config surfaces and maintainer security design.
