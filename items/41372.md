## Summary

Keep this issue open. Current main has addressed several sub-areas from the field report, especially config preflight/reload handling and much of the cron CLI syntax, but the broad report is not fully implemented or clearly superseded. One concrete reported docs mismatch still exists: the scheduled-tasks guide says `openclaw cron list` lists all jobs, while the CLI only includes disabled jobs when `--all` is passed.

## Best Possible Solution

Keep this issue open for maintainer triage, then split the remaining actionable pieces into focused follow-ups. The smallest concrete current fix is to update the cron docs so `openclaw cron list` is documented as enabled-only by default and `openclaw cron list --all` is shown for disabled jobs. Maintainers should then decide whether remaining requests such as cron upsert/resource warnings, isolated web-search reproduction coverage, and update-process detachment need separate core issues or can be closed as already handled by current config, Discord, and onboarding hardening.
