## Summary

PR #70681 should stay open. Current main still contains the Unix `lsof -Fpc` command-name filter that the PR is trying to remove, and the cleanup path still uses that parser directly, so the macOS gateway PID mismatch is not implemented or superseded on main.

## Best Possible Solution

Keep this PR open, or land an equivalent maintainer patch, because current main still has the Unix lsof command-field bug. Before merge, address the review comments around poll-path verifier timeouts, update the stale PID tests to cover macOS-style `cnode` plus ps argv verification, and run the targeted infra/CLI restart tests described in the PR.
