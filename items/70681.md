## Summary

Keep PR #70681 open. Current main still contains the lsof `c`/command-name filter that the PR is trying to replace, so the macOS `p_comm` vs rewritten argv[0] bug described in #70664 is not implemented on main. The PR is a focused bug fix with concrete reproduction evidence, but the bot review notes small follow-up corrections before merge.

## Best Possible Solution

Keep this PR open and ask the contributor to address the bot review: add Unix argv verification for listener PIDs, pass `POLL_SPAWN_TIMEOUT_MS` on the polling path, update the stale-pid tests away from lsof command-field classification, and then land it as the canonical fix for #70664 if CI and maintainer review pass.
