## Summary

Keep open. Current main still has the Unix stale-gateway PID bug described by PR #70681/#70664: it trusts the `lsof -Fpc` command field to contain `openclaw`, so a macOS gateway reported as `cnode` is dropped before any argv-based verification can happen. The PR is a focused, useful fix, but the submitted patch still needs the review feedback addressed before merge.

## Best Possible Solution

Keep PR #70681 open. The best path is to revise it rather than close it: remove Unix gateway classification from `parsePidsFromLsofOutput`, verify candidate PIDs via argv/`ps` before stale cleanup and poll decisions, pass `POLL_SPAWN_TIMEOUT_MS` on the poll path, optionally switch `lsof` to `-Fp`, then run the targeted infra and gateway CLI tests described by the PR.
