## Summary

Keep open. Current `main` still short-circuits `--agent` requests to the agent main-session key before considering `--to`, so the PR’s reported bug is not implemented on `main`. The touched surface has also been refactored from `src/commands/agent/session.ts` to `src/agents/command/session.ts`, so the PR likely needs a rebase rather than cleanup-closing.

With `--agent mybot --to cw_111`, this still resolves to the agent main session instead of an agent-scoped peer key.
