## Summary

Keep this PR open. Current main still resolves an explicit agent id to that agent's main-session key before considering `--to`, so the reported `openclaw agent --agent <id> --to <target>` routing bug remains. The PR is still useful, but it needs a rebase because the implementation moved from the old command path into the agents command module.

## Best Possible Solution

Rebase this PR onto current main, port the selected-agent `--to` derivation into `src/agents/command/session.ts`, preserve explicit `--session-key` and `--session-id` precedence, and land focused regression coverage for `--agent + --to` plus the reviewed guardrails.
