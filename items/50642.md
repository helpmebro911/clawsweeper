## Summary

Keep open. Current main still matches the reported macOS node first-use TLS TOFU behavior: a fresh wss:// endpoint gets allowTOFU=true, the TLS delegate stores and accepts the first certificate before SecTrustEvaluateWithError, and the connected node session can dispatch node.invoke.request to the macOS runtime including system.run. The linked PR #50643 is open and unmerged, so it does not supersede the issue for cleanup closure.
