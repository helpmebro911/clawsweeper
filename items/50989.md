## Summary

Keep #50989 open. Current main has a phase-aware suppression path that drops assistant output marked as `commentary`, and that partially addresses the related discussion. But the requested generic suppression for Slack/group-channel pre-tool narration is not implemented: unphased text before tool execution is still intentionally flushed, and the Anthropic transport used by the reported Claude setup does not attach commentary/final-answer phase metadata.

## Evidence

- **Issue context:** #50989 reports Slack multi-agent group-channel noise with Claude and Gemini fallback, and asks for a `suppressIntermediateText`-style option that drops text before tool calls while still delivering final text. The KeepMyClaw comment supports suppression as the source fix but describes only an operational recovery workaround, not an OpenClaw implementation.
