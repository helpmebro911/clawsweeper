## Summary

Keep open. Current `main` computes and uses the effective context limit internally, including the `agents.defaults.contextTokens` cap, but the public `llm_input`/`llm_output` hook payloads still do not expose that effective limit to plugins.

## Evidence

- **Hook event types do not expose an effective context limit:** `PluginHookLlmInputEvent` and `PluginHookLlmOutputEvent` only define run/session/provider/model plus prompt/history or output/usage fields. There is no `contextTokens`, `contextWindowTokensOverride`, or equivalent effective-limit field in the hook contract.
  - file: [src/plugins/hook-types.ts:179](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/plugins/hook-types.ts#L179)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '145,220p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Current main resolves the effective limit from config:** `resolveContextWindowInfo()` caps the native model context with `cfg?.agents?.defaults?.contextTokens` and returns `source: "agentContextTokens"` when that cap is lower than the model limit.
  - file: [src/agents/context-window-guard.ts:23](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/context-window-guard.ts#L23)
  - command: `nl -ba src/agents/context-window-guard.ts | sed -n '1,90p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **PI runner uses the effective limit internally:** `resolveEffectiveRuntimeModel()` applies the resolved `ctxInfo.tokens` cap to the runtime model context window, and the main runner passes `ctxInfo.tokens` into the attempt as `contextTokenBudget`. So the effective limit exists internally on current `main`.
  - file: [src/agents/pi-embedded-runner/run/setup.ts:128](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/pi-embedded-runner/run/setup.ts#L128)
  - command: `nl -ba src/agents/pi-embedded-runner/run/setup.ts | sed -n '119,146p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Hook emission still omits that internal budget:** The `llm_input` emission includes prompt/history/images only, while `llm_output` includes resolved ref, harness id, assistant text, and usage. Neither emission includes the effective context budget even though the attempt already has `contextTokenBudget`.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2276](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/pi-embedded-runner/run/attempt.ts#L2276)

  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Published docs still describe these hooks as prompt/history and output observers only:** The public hook docs describe `llm_input` as observing provider input and `llm_output` as observing provider output, with no documented effective-context field for plugin consumers.
