## Summary

Keep PR #46552 open. Current main already has basic queue mode docs, but it still lacks the PR’s dedicated steer + partial/block streaming explanation and the specific troubleshooting note about session queue overrides. The runtime and channel docs support the core behavior the PR is trying to document, and the provided PR discussion shows earlier review concerns were addressed rather than leaving unresolved bot feedback.

## Best Possible Solution

Keep the PR open and either merge it after normal docs review/rebase or manually port the small `docs/concepts/queue.md` addition. The best docs path is to add a focused steer/streaming section that distinguishes `partial` preview finalization from `block` chunk emission, explains non-streaming steer fallback to followup, and adds troubleshooting for `/queue` session overrides taking precedence over config.
