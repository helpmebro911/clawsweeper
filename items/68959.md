## Summary

Keep #68959 open. Current main has improved session grouping and the separate Agents management tab, and related #70610 covered a narrower raw-label bug, but the reported workflow is still not implemented: the chat session selector is driven by existing session rows, not all configured agents, and the chat UI still has no visible control to start/switch to any configured agent that lacks an existing session.

## Best Possible Solution

Keep the issue open and implement a chat-visible configured-agent switcher. Populate it from agents.list, show configured display/identity names, switch to an existing agent main session when present, and otherwise call sessions.create with agentId/key main before loading chat history. Add focused Control UI tests covering multiple configured agents where a non-default agent has no existing sessions.list row.
