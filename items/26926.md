## Summary

Keep #26926 open. Current main still only handles text inbound from Synology Chat; the existing media support is outbound URL-based file sending and does not cover user-uploaded image/CSV/PDF attachments from Synology Chat. The prior issue discussion already includes a maintainer-side Codex review keeping this open for the same inbound attachment gap, and no Synology Chat code/docs changed between that reviewed SHA and current main.

## Best Possible Solution

Keep #26926 open and implement inbound attachment handling in the bundled Synology Chat plugin. Confirm Synology Chat's attachment webhook/API shape, extend the typed payload/parser, download or stage accepted images/CSVs/PDFs through existing media-safe helpers, expose media metadata in the inbound context, update https://docs.openclaw.ai/channels/synology-chat, and add focused webhook plus inbound-context regression tests.
