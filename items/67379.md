## Summary

Current main still drops `agent:<id>:subagent:<uuid>` QMD session keys during scope normalization, so the reported default-deny path remains actionable.

## Best Possible Solution

Keep this issue open and fix the QMD scope contract for spawned subagent sessions. The narrow code path is to make `agent:<id>:subagent:<uuid>` inherit the parent session's QMD classification or receive an explicit documented subagent classification, then add regression coverage in `src/memory-host-sdk/host/qmd-scope.test.ts` and the mirrored package test.
