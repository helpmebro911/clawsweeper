## Summary

Keep open. Current main still gates gateway agent delivery on `request.deliver === true`, so an omitted `deliver` on `agent:*:telegram:group:*` sessions remains session-only despite the session key carrying a Telegram group/topic destination. I found no current helper or test coverage that defaults Telegram group-bound agent sessions to visible delivery while preserving `deliver:false`.

## Best Possible Solution

Implement the narrow gateway behavior requested in the issue: add a well-tested helper that recognizes `agent:<agent>:telegram:group:<chatId>` and `agent:<agent>:telegram:group:<chatId>:topic:<topicId>` session keys, treat omitted `deliver` as visible delivery only for those sessions, preserve `deliver:false`, and keep the existing internal-channel suppression. The likely touch points are `src/gateway/server-methods/agent.ts` plus focused tests in `src/gateway/server-methods/agent.test.ts` or `src/infra/outbound/agent-delivery.test.ts`.
