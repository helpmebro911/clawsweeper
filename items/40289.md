## Summary

Keep this PR open. Current main still drops Discord card/component text before ordinary button/select fallback events reach the agent, so the reported behavior remains real. The submitted fix is directionally useful, but it targets the removed `src/discord` layout and should be ported into the current `extensions/discord` plugin path with focused tests.

## Best Possible Solution

Keep this PR open and ask for a rebase/port into the current Discord plugin. The best patch would add a structured card-context field to `DiscordComponentEntry`, collect `spec.text`, fallback text, and visible text/section block content in `extensions/discord/src/components.ts`, preserve it through the component registry, include it in ordinary button/select fallback event text from `extensions/discord/src/monitor/agent-components.ts`, and update `extensions/discord/src/components.test.ts` plus `extensions/discord/src/monitor/monitor.test.ts`. The old system-prompt hunk should be reconciled with the current Discord `messageToolHints` path instead of copied directly.
