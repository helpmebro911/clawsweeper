## Summary

Keep #7909 open. Current main still exposes a single Control UI assistant-message copy action labeled “Copy as markdown”; it writes the supplied markdown text directly to the clipboard and there is no Markdown/plain-text selector, preference, or test/doc evidence that plain-text copy shipped. The Apr 26 maintainer-side Codex comment in the issue also explicitly kept this open, and the Android copy/selection related items are separate mobile surfaces, not a superseding fix for Control UI.

## Best Possible Solution

Keep #7909 open. The best implementation belongs in the Control UI assistant-message action path near ui/src/ui/chat/copy-as-markdown.ts and ui/src/ui/chat/grouped-render.ts: add a compact Markdown vs plain-text copy menu or a user preference, reuse or adapt an existing markdown-to-text helper, and add focused UI/clipboard tests proving both formats write the expected clipboard text.
