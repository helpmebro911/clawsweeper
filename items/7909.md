## Summary

Keep #7909 open. Current main still has a single Control UI assistant-message copy action labeled “Copy as markdown” that writes the markdown string directly to the clipboard. There is no Markdown/plain-text selector, default preference, or focused test/doc evidence that plain-text assistant-message copy has shipped. The latest issue discussion also contains a maintainer-side Codex review explicitly keeping the item open.

## Best Possible Solution

Keep #7909 open. The best implementation belongs in the Control UI assistant-message action path around ui/src/ui/chat/copy-as-markdown.ts and ui/src/ui/chat/grouped-render.ts: add a compact Markdown vs plain-text copy menu or a user preference, reuse or adapt an existing markdown-to-text helper, and add focused UI/clipboard tests proving both formats write the expected clipboard text.
