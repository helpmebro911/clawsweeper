## Summary

Keep open. Current `main` still intentionally preserves explicit session `modelOverride` values across `/new` and `/reset`, persists them in the session store, and continues to resolve them ahead of config defaults. `/status` also does not label session-level model overrides, so the issueâ€™s core behavior remains present.
