## Summary

Keep #71849 open. Current main still routes Voice Call realtime context questions through the shared full `openclaw_agent_consult` path, with no `realtime.fastContext` schema/docs/manifest surface, no hard-deadline direct memory/session lookup in the realtime handler, and no partial-transcript accumulation. The maintainer comment already called this out as a real gap, and the relevant current-main code still matches that assessment.

## Best Possible Solution

Keep the issue open and implement the fast path in the bundled Voice Call plugin: add a default-off `realtime.fastContext` schema/manifest/docs surface, accumulate and normalize partial transcripts in `RealtimeCallHandler`, use the existing memory search manager surface for a hard-deadline direct memory/session lookup, inject snippets or an explicit miss into the realtime session, and preserve `openclaw_agent_consult` as the deeper fallback. Add focused tests for config parsing, partial transcript handling, repeated lookups in one call, timeout/miss behavior, and consult fallback.
