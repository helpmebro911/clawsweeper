## Summary

Current `main` still exposes only turn-level `llm_output` hooks with aggregated usage. The requested per-LLM-call event or separate `llm_call` hook is not implemented in the current hook contract or embedded runner, so this item should remain open.

## Evidence

- **Hook contract still has no per-call hook:** `PluginHookName`, `PLUGIN_HOOK_NAMES`, and `PluginHookLlmOutputEvent` only define `llm_input`, `llm_output`, and `agent_end`; there is no `llm_call` hook or per-iteration flag/shape in the public plugin hook surface.
