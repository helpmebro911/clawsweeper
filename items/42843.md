## Summary

Current main still has the finite WhatsApp reconnect cap and a terminal max-attempt stop path this PR is trying to remove. The PR should stay open as a valid implementation candidate, but it needs to be refreshed for the current WhatsApp plugin layout and config contract before merge.

## Best Possible Solution

Keep this PR open and require a refresh against the current WhatsApp plugin architecture. The likely maintainer path is to update `extensions/whatsapp/src/reconnect.ts`, `extensions/whatsapp/src/connection-controller.ts`, and `extensions/whatsapp/src/auto-reply/monitor.ts` so retryable disconnects never permanently kill the monitor, while logged-out and session-conflict closes remain terminal. Add tests for default unlimited retries, bounded degraded-mode retry, non-retryable closes, and auth-unstable behavior, then reconcile `web.reconnect.maxAttempts` schema/help/docs/generated metadata before merge.
