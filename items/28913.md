## Summary

Keep #28913 open. Current main still has trigger-based memory-lancedb autoCapture with no captureFilter/excludeWrapped config, manifest schema, or wrapper/source-aware pre-filter. External untrusted-content markers exist in core/security docs, but the memory auto-capture path does not consume them.

## Best Possible Solution

Keep the issue open. Implement a backward-compatible memory-lancedb option such as memory.captureFilter.excludeWrapped, add it to typed config parsing and openclaw.plugin.json schema/UI hints, apply a robust agent_end pre-filter before shouldCapture, and add regression tests with wrapped EXTERNAL_UNTRUSTED_CONTENT containing trigger words like always while preserving normal direct preference capture.
