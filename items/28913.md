## Summary

Keep #28913 open. Current main still lacks source- or wrapper-aware filtering for memory-lancedb autoCapture: the requested config is not parsed, the manifest schema rejects unknown keys, shouldCapture remains trigger-based, and agent_end still stores user-message text after only shouldCapture filtering.

## Best Possible Solution

Keep the issue open. Implement a backward-compatible memory-lancedb option such as memory.captureFilter.excludeWrapped, add it to typed config parsing and openclaw.plugin.json schema/UI hints, apply an agent_end pre-filter before shouldCapture, and add regression tests with wrapped EXTERNAL_UNTRUSTED_CONTENT containing trigger words like always while preserving direct user preference capture.
