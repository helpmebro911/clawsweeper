## Summary

Keep #44346 open. Current main supports on-demand skill reference files, but it does not implement the PR's proposed `metadata.openclaw.references.autoLoad`, `SkillEntry.referencesContent`, or automatic `references/` concatenation. This is a core skills loader/API behavior decision rather than a ClawHub/new-skill cleanup item, and the PR includes focused implementation, docs, tests, and follow-up commits addressing some review feedback.

## Best Possible Solution

Keep the PR open for explicit maintainer judgment. If OpenClaw wants `references/` auto-load as a core skill API, update the loader so both sandbox and non-sandbox skill reads receive the merged content, preserve path and aggregate-size guards, and add focused tests/docs. If maintainers prefer the current progressive-disclosure model, close or rework the PR with that rationale rather than auto-closing it as already implemented.
