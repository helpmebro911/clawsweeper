## Summary

Do not auto-close PR #48285. Current main has superseded part of the reported internal-service symptom with a narrower backend/operator shared-secret loopback path, but it has not implemented the PR’s node-role device-less loopback exemption or the auth.mode=none backend pairing skip. The PR also has discussion evidence that the original root-cause claim was incomplete, so it should not be merged as-is or closed as already implemented without maintainer judgment.

## Best Possible Solution

Keep the PR open for maintainer judgment, but do not merge it as-is. The best path is a rebased, narrower PR or canonical issue that targets any still-reproducible `auth.mode=none` backend/internal-service pairing failure on current main, with tests against `shouldSkipLocalBackendSelfPairing` and the real `callGateway()` operator/backend path. The node-role device-less loopback exemption should remain out unless maintainers explicitly decide to change the documented gateway security contract.
