## Summary

Keep open. Current main still has a synchronous/serial provider catalog path during model catalog warmup, and the requested lazy/parallel/optional startup validation behavior is not implemented. The related fork commit positively confirms the same issue was worked around out-of-tree by deferring prewarm, but that patch is not present in OpenClaw main.

## Evidence
