## Summary

Current main still promotes source-derived daily-memory snippets directly into MEMORY.md. The ranking and apply path filters some contaminated dreaming artifacts and rehydrates live source text, but it does not summarize, distill, or expose a distilled-promotion mode. The related metadata-contamination work in #67442/#67601 does not supersede this request.

## Best Possible Solution

Keep this issue open for memory-core maintainer triage. Decide whether deep promotion should remain raw-by-design or gain a distilled-promotion mode; if distilled output is desired, implement it in the memory-core ranking/apply path, preserve source provenance, add regression tests for raw versus distilled output, update config/schema only if configurable, and clarify the public Dreaming docs.
