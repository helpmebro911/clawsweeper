## Summary

Current main still promotes rehydrated daily-memory snippet text directly into MEMORY.md. The code has filtering for contaminated dreaming artifacts, but there is no distillation/summarization step, no distilled-promotion config option, and the Dreaming docs still describe append-style promotion rather than curated insight generation. The related metadata-contamination issue/PR is narrower and does not supersede this issue.

## Best Possible Solution

Keep this issue open for memory-core maintainer work. Decide whether deep promotion should remain raw-by-design or gain a distilled-promotion mode, then update `extensions/memory-core/src/short-term-promotion.ts` or the apply pipeline, preserve source provenance, add focused regression tests for distilled versus raw output, and clarify the public Dreaming docs.
