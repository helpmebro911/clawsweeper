## Summary

Current main still has the reported mismatch: doctor warns about missing recent session transcripts but suggests a dry-run cleanup command without `--fix-missing`, while the cleanup implementation only previews missing-transcript pruning when `fixMissing` is true. The two linked fix PRs were closed unmerged in the provided GitHub context, so the fix is not present on main.

## Best Possible Solution

Keep this issue open and land the narrow doctor hint fix: update `src/commands/doctor-state-integrity.ts` so the missing-transcript `Preview cleanup impact` command includes `--dry-run --fix-missing`, then update `src/commands/doctor-state-integrity.test.ts` to require that exact preview behavior. The existing cleanup command behavior can remain flag-gated because the CLI already supports and documents the correct preview mode.
