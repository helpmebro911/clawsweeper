## Summary

Keep open. Current main has a runtime-dispatcher FormData normalizer for some guarded-fetch paths, but the proxy-fetch wrappers named in #48554 still pass `init.body` directly into `undici.fetch`, and the audio runner still injects `resolveProxyFetchFromEnv()`'s proxy fetch into providers when proxy env vars are set. That caller-supplied proxy fetch bypasses the runtime normalization path. PR #48556 remains open/unmerged, and the latest maintainer comment explicitly left this narrower proxy-wrapper gap open.

## Best Possible Solution

Keep #48554 open. The best fix is to add a focused regression test for `makeProxyFetch()` and `resolveProxyFetchFromEnv()` with a `globalThis.FormData` multipart body under an undici proxy dispatcher, then either normalize FormData inside `proxy-fetch.ts` or route these proxy wrappers through the existing runtime FormData normalization without breaking plugin SDK proxy semantics. PR #48556 can be used as the candidate implementation if it is refreshed against current main.
