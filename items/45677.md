## Summary

Keep PR #45677 open. Current main still drops RuntimeLogger metadata, so the PR is not obsolete, but the submitted fix likely needs adjustment because OpenClaw’s tslog-backed structured logging path uses metadata as the first logging argument.

## Best Possible Solution

Keep the item open and request a corrected, narrow fix: adapt RuntimeLogger’s `(message, meta?)` surface to the tslog structured convention by calling the underlying logger with `(meta, message)` for non-empty metadata and `(message)` otherwise, then add focused regression coverage for `createRuntimeLogging()` that proves metadata becomes structured log attributes.
