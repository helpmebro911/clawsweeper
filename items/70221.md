## Summary

Keep #70221 open. Current main still supports only height for `[embed]`/canvas previews: the parser maps `height` to `preferredHeight`, preview types and sidebar content have no width field, and the iframe renderer/CSS still render the frame at `width: 100%`. The request is a concrete core Control UI/rich-output feature request, not a ClawHub/plugin cleanup item, and no provided related item supersedes it.

## Best Possible Solution

Keep the issue open and implement width support as a core Control UI/rich-output contract: add a bounded `preferredWidth`/`width` field to the canvas preview model, parse it from `[embed width="..."]`, carry it through normalized/stored blocks and sidebar content, render it with responsive max-width/overflow behavior, and update docs plus focused parser/render tests.
