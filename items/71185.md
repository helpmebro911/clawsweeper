## Summary

Current main still has the pre-PR null-unsafe support-capture path: `buildSkillsCapture` trusts every `resolvedSkills` entry and `redactPathForSupport` immediately calls string methods. This PR is recent, focused, contributor-authored, and not superseded by the unrelated related-item matches in the provided context.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or refine the focused hardening in `src/trajectory/metadata.ts` and `src/logging/diagnostic-support-redaction.ts` with the proposed regression tests in `src/trajectory/metadata.test.ts`; any producer-side cleanup in `src/agents/skills/workspace.ts` can be handled as a separate follow-up if maintainers want to prevent partial entries at the source.
