## Summary

Keep open. Current main still has the pre-PR support-capture behavior: `buildSkillsCapture` maps every `resolvedSkills` entry without validating `name`, `filePath`, or `baseDir`, and then sorts with `left.name.localeCompare(...)`; `redactPathForSupport` still requires a string and immediately calls `startsWith`. The PR remains a focused defensive fix with targeted regression coverage, and I found no evidence that it has already landed or been superseded.

## Best Possible Solution

Keep the PR open for maintainer review and either merge/adapt its defensive consumer fix or request a producer-side companion change. The best code path is to make support capture resilient to malformed `resolvedSkills` entries, preserve fallback inventory from `skillsSnapshot.skills`, and add the proposed regression tests around partial entries and nullish path redaction.
