## Summary

Keep #72228 open. The report is concrete and current main still has a plausible gap: the core logger imports `tslog`, while bundled runtime dependency revalidation only computes missing specs from the active plugin package dependencies, not from retained/install-root specs such as a previously staged `tslog`. If `tslog` is removed from the external runtime deps root while plugin-local deps remain present, current code can still skip reinstalling retained deps before mirrored dist code imports logging.

## Best Possible Solution

Keep the issue open and fix the bundled runtime dependency repair path so the install root validates every spec it intends to retain/install, not only the current plugin package deps. A robust fix should ensure `tslog` is always resolvable for mirrored core dist logging, either by including/verifying it in the external runtime root or by keeping core logging resolution anchored to the packaged OpenClaw root, plus a regression test that removes `node_modules/tslog` from an external `plugin-runtime-deps/openclaw-*` root and verifies startup repairs or fails with a clear pre-import error.
