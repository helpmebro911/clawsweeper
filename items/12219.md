## Summary

Keep #12219 open. Current main still does not implement the requested `skill.yaml`/`skill.yml` permission manifest standard, install-time permission review for `openclaw skills install`, `openclaw skills audit`, signing/attestation, or trust-grade support. The useful external work mentioned in the thread (`@ellistevo/openclaw-secure`, ClawHub `shell-security-ultimate`, and `globalcaos/tinkerclaw`) should be treated as references, not shipped replacements for the core skill permission contract.

## Best Possible Solution

Keep #12219 open for maintainer product/security design. Define a versioned skill permission manifest, validate it during ClawHub and direct skill install/update flows, show declared permissions before `openclaw skills install`, add an audit/trust report or equivalent, and connect declarations to `before_install` and `before_tool_call` enforcement where appropriate. Use `@ellistevo/openclaw-secure`, ClawHub `shell-security-ultimate`, and `globalcaos/tinkerclaw` as reference implementations while coordinating with broader supply-chain work such as #10827 and the ClawHub ecosystem thread #50090.
