## Summary

Keep #12219 open. Current main has adjacent safety work for plugin install hooks, tool-call hooks, and skill code-safety scanning, but it still does not implement a `skill.yaml` permission manifest standard, `openclaw skills audit`, install-time permission review for direct ClawHub skill installs, signing/attestation, or trust-score presentation. The latest maintainer-side discussion also explicitly kept this issue open and treated the linked external projects as useful references rather than shipped OpenClaw behavior.

## Best Possible Solution

Keep #12219 open for maintainer product/security design. A strong implementation path is to define a versioned skill permission manifest, validate it during direct ClawHub skill install/update and relevant plugin/bundle install flows, show permissions before `openclaw skills install`, add a user-facing `openclaw skills audit` or equivalent trust report, and connect declarations to `before_install` and `before_tool_call` enforcement where the existing hook APIs are sufficient. Treat `@ellistevo/openclaw-secure`, `shell-security-ultimate`, and `globalcaos/tinkerclaw` as reference implementations, not close reasons.
