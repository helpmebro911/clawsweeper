## Summary

Keep #12219 open. Current main still has no `skill.yaml` permission manifest standard and no install-time permission review, `openclaw skills audit`, signing/attestation, or trust-score display. The issue discussion also includes an April 26, 2026 maintainer-side Codex review explicitly keeping this item open, treating `@ellistevo/openclaw-secure`, the ClawHub `shell-security-ultimate` skill, and globalcaos/tinkerclaw as useful reference implementations rather than shipped OpenClaw behavior.

## Best Possible Solution

Keep #12219 open for maintainer product/security design. Define a versioned skill permission manifest, validate it during ClawHub/direct skill install and updates, show permissions before `openclaw skills install`, add a real `openclaw skills audit` or equivalent trust report, and connect declarations to install/runtime enforcement hooks where the API is sufficient. Treat `@ellistevo/openclaw-secure`, the ClawHub `shell-security-ultimate` skill, and globalcaos/tinkerclaw as useful reference implementations, not as a substitute for shipped OpenClaw behavior.
