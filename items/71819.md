## Summary

Keep #71819 open. Current main still has the reported behavior: the dispatch catch path releases the inbound dedupe claim for any error, and there is no separate poisoned/committed terminal state for failures after an externally visible reply. The PR is a focused, recent contributor fix for open bug #69303 with tests and review follow-up, not obsolete on main.

## Best Possible Solution

Keep the PR open for maintainer review and likely landing after normal CI/touched-surface validation. The best code direction is the PR's third dedupe terminal state: preserve safe retry for failures before any visible output, but stamp/poison inbound dedupe after externally visible dispatch so replayed provider `message_id`s cannot duplicate user-visible progress or side effects. Track the tool-result-only exclusion as a follow-up if maintainers want broader idempotency.
