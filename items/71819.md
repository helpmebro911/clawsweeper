## Summary

Keep #71819 open. Current main still releases the inbound dedupe claim on every dispatch error and has no poisoned/committed terminal state for errors after externally visible output. The PR is a focused fix for still-open bug #69303, with matching implementation and regression coverage in the provided PR context.

## Best Possible Solution

Keep #71819 open for maintainer review and likely landing after normal validation. The best code direction is the PR's third dedupe terminal state: release for failures before any visible output, but stamp/poison the inbound key after externally visible dispatch so replayed provider `message_id`s cannot duplicate user-visible progress or side effects. Track the tool-result-only exclusion as a narrower follow-up if maintainers want broader idempotency.
