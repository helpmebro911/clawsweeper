## Summary

Keep open. Current main still has the CLI/runtime mismatch described by the PR: the full memory-core entry registers publicArtifacts, but the CLI metadata entry only registers CLI commands, and the CLI metadata loader still does not wire registerMemoryCapability. The linked issue #70181 remains a real product bug rather than an obsolete or duplicate PR.

## Best Possible Solution

Keep #70185 open and either merge or adapt its fix: register memory-core publicArtifacts from cli-metadata, wire registerMemoryCapability in loadOpenClawPluginCliRegistry with the same kind/slot guards as full registration, restore memory state on CLI metadata register failure, and add focused CLI-metadata or wiki-bridge regression coverage.
