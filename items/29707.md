## Summary

Keep #29707 open. Current main still excludes regular Amazon Nova Bedrock model IDs from cache-retention eligibility and the bundled Amazon Bedrock plugin still wraps direct Nova calls with `cacheRetention: "none"`. Recent Bedrock cache work on main appears focused on Anthropic Claude and opaque application inference profiles, not native Nova prompt caching.

## Best Possible Solution

Keep the issue open for the bundled `amazon-bedrock` plugin. The likely fix is to add explicit Nova-family cache eligibility, preserve forced `cacheRetention: "none"` for unsupported non-Anthropic Bedrock models, shape or inject Bedrock Nova `cachePoint` metadata according to the current AWS Converse contract, and update the Bedrock regression tests plus the prompt-caching docs at https://docs.openclaw.ai/reference/prompt-caching.
