## Summary

Keep #29707 open. Current main still forces regular Amazon Nova Bedrock calls onto the non-Anthropic no-cache path, while tests and docs explicitly codify that Nova/Mistral cache settings are forced to `cacheRetention: "none"`. The issue's requested Nova prompt-caching path is not implemented on main.

## Best Possible Solution

Keep the issue open for the bundled `amazon-bedrock` plugin. The best fix is to add explicit Nova-family cache eligibility and runtime pass-through, make cachePoint insertion conditional on the current AWS Nova Converse contract, keep `cacheRetention: "none"` for unsupported non-Anthropic Bedrock models, and update the Bedrock/plugin regression tests plus the public prompt-caching docs at https://docs.openclaw.ai/reference/prompt-caching.
