## Summary

Keep open. Current `main` still allows isolated cron announce delivery to use fatal error payload text, then separately sends failure notifications for non-`bestEffort` jobs. The raw-JSON rendering appears partially mitigated upstream, but the requested behavior change (do not announce failed runs to user-facing channels) is not implemented.
