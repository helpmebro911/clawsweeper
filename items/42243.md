## Summary

Keep open. Current main still converts fatal isolated cron outcomes into `isError: true` delivery payloads and passes them through normal announce delivery; the related attempted fix #46236 is closed unmerged. Some provider/raw JSON text is now sanitized in adjacent agent paths, but the requested no-announce-on-error behavior is not implemented.

## Best Possible Solution

Keep this issue open. Implement an isolated-cron guard so fatal/error run outcomes skip normal `announce` fallback delivery and remain visible in cron run history. If maintainers want user-visible failure notices, send them only through the explicit failure-notification path with generic, sanitized text, and add regression coverage for `delivery.mode: "announce"`, `bestEffort: false`, provider/raw error payload, `lastRunStatus: "error"`, and no normal outbound delivery attempt.
