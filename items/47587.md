## Summary

Keep open. Current main has improved part of the reported Telegram album path flow: media groups are collected into a single allMedia list and MediaPaths is populated for all current attachments, and image descriptions no longer suppress image media-note entries. However, the central requested behavior is still not fully implemented on main: media understanding still selects only the first matching image by default unless tools.media.image.attachments is explicitly set to all, and the media-note formatter still renders both path and url, which duplicates the path when Telegram sets MediaUrl equal to MediaPath. That leaves the issue's all-images-transcribed and path-once expectations unresolved.
