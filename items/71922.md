## Summary

Keep open. Current main still collapses the three reported model-fallback cases into `unknown`: the runtime classifier returns `unknown` for an empty signal and for its final catchall, the exact `Unknown error (no error details in response)` path is intentionally classified as failover reason `unknown`, and the exported failover reason/protocol schemas do not contain `empty_response`, `no_error_details`, or `unclassified`. The related ClawSweeper item #49190 is about cron delivery warning status and does not supersede this model-fallback reason split.

## Best Possible Solution

Implement this in core by extending the failover reason/runtime failure contracts with `empty_response`, `no_error_details`, and `unclassified`; classify the three paths separately in `src/agents/pi-embedded-helpers/errors.ts`; update cooldown/probe policy to preserve the current transient behavior where intended; update cron protocol/schema/UI/log consumers; and add regression coverage for empty signal, exact no-details response, and catchall-with-preview behavior.
