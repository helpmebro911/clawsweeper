## Summary

Keep open. Current main still collapses the reported empty response, explicit no-details response, and unclassified catchall paths to `unknown`; the requested reason labels are not present in the failover reason type, cooldown policy, or tests.

## Best Possible Solution

Keep the issue open and implement the split in the core failover path: extend `FailoverReason`, classify empty/no-status as `empty_response`, exact no-details responses as `no_error_details`, and unmatched message fallthroughs as `unclassified`; then update cooldown policy, fallback observation/error preview behavior, tests, and user-facing docs or CLI status buckets as needed.
