## Summary

Keep open. Current main routes proactive send through the Feishu plugin and supports semantic `presentation` cards, but it still does not detect raw Feishu interactive-card JSON supplied in the `message` string. That string falls through the normal Feishu text path, which sends `msgType: "post"`, not `"interactive"`. Related closed items are adjacent Feishu validation/card-delivery work, not a canonical fix for this regression.

## Best Possible Solution

Keep this issue open for Feishu-owned maintainer handling. The narrow fix would be either a conservative compatibility detector in `extensions/feishu/src/channel.ts` that recognizes valid Feishu interactive-card JSON in `message` and routes it to `sendCardFeishu`, with a regression test for proactive `message(action=send)`, or an explicit docs/product decision that raw provider-native card JSON is no longer supported and users must migrate to `presentation`.
