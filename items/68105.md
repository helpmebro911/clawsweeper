## Summary

Keep open. Current main still keeps RTL RLI/PDI isolation local to the TUI formatter, while shared assistant-visible text, gateway live chat projection, and outbound delivery paths still pass Hebrew/Arabic text without a shared bidi-isolation stage. The prior ClawSweeper comment’s conclusion remains valid against current main, and the related closed TUI/Telegram-preview reports do not supersede this outbound boundary bug.

## Best Possible Solution

Keep this issue open. Implement a shared RTL/bidi isolation helper under src/shared/text, reuse it from the TUI formatter, and apply it on the canonical user-visible outbound path before channel chunking/send. Add focused regression tests for the shared helper, outbound/plain-text delivery, gateway/live projection if that surface is intended to be covered, and no-double-wrap behavior for existing bidi controls.
