## Summary

Keep open. Current main still silently filters `sessions_list` through the configured session visibility guard, then returns only `count` and `sessions` without a visibility/scope note. The gateway `sessions.list` result contract also has no scope metadata. Docs describe the default `tree` visibility, but the issue asks for in-output transparency, which remains missing. Ryce’s comment about KeepMyClaw and #50621 is related operator context, not a duplicate or an implementation in OpenClaw core.
