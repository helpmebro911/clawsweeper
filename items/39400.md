## Summary

Keep #39400 open. Current main still exposes provider quota in `session_status` as an optional human-readable status-card line, not as stable structured usage fields with explicit availability, source, or reset metadata. The prior ClawSweeper keep-open comment remains accurate on the current main SHA.

## Best Possible Solution

Keep the issue open. If cron/monitor use of `session_status` is supported, add a structured `usage` object to tool `details` with provider/source, explicit availability or unavailable reason, normalized 5h/day windows with percent and resetAt, and reset explanation where known. Preserve the current human-readable status line. Add focused coverage for no auth, provider error, empty windows, timeout, and OpenAI Codex 5h/Day/Week mapping. If maintainers prefer not to expand `session_status`, create and link a dedicated agent-facing usage tool instead.
