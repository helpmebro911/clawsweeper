## Summary

Keep open. Current main still treats `session_status` provider quota as an optional human-readable status line, not stable structured usage data. It omits usage-window output on provider errors, empty windows, and timeouts, and the tool result details do not expose `usage.window5h`, `usage.windowDay`, source, reset reason, or an explicit unavailable state. The issue is recent, author is not protected, and the only provided label is `stale`, so no auto-close reason applies.
