## Summary

Current main still has the reported product gap: media-understanding outputs carry provider/model telemetry, but the inbound body formatter sent to BodyForAgent still renders only the media section text, so the assistant cannot answer from current-turn telemetry.

## Best Possible Solution

Keep this issue open. Implement a narrow formatter/apply-path change in src/media-understanding/format.ts and src/media-understanding/apply.ts so successful current-turn media-understanding outputs render an Analysis model line from output.provider/output.model when available. Add focused coverage in src/media-understanding/format.test.ts and an apply-path regression in src/media-understanding/apply.test.ts proving BodyForAgent receives actual telemetry while unavailable or synthetic telemetry is omitted or clearly labeled.
