## Summary

Current main still keeps media-understanding provider/model telemetry internally and exposes it in status output, but the prompt-visible inbound media body still omits the actual analysis provider/model. The issue remains actionable upstream work.

## Best Possible Solution

Keep this issue open and implement a narrow formatter change in `src/media-understanding/format.ts`: include a prompt-visible current-turn analysis provider/model line for each successful media output when known, then add focused coverage in `src/media-understanding/format.test.ts` and an apply-path regression proving `BodyForAgent` receives it. If provider/model is unavailable, the formatter should avoid implying a guessed model and either omit the line or render an explicit unavailable value based on maintainer product preference.
