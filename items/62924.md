## Summary

Current main still carries media-understanding provider/model telemetry internally, but does not render it into the assistant-visible inbound body. The earlier ClawSweeper comment and reporter validation remain current: a local `Analysis model:` formatter patch would address the product gap, but that behavior has not landed on main.

## Best Possible Solution

Keep this issue open. Implement a narrow media-understanding formatter/apply-path change that renders an `Analysis model:` line from `output.provider` and `output.model` for successful, non-synthetic current-turn outputs when telemetry is available. Add focused coverage in `src/media-understanding/format.test.ts` and an apply-path regression in `src/media-understanding/apply.test.ts` proving `BodyForAgent` receives the actual telemetry while missing or synthetic telemetry is not misrepresented.
