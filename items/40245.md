## Summary

Keep open. Current `main` still enforces workspace-root-only bootstrap file loading, explicitly rejects outside-workspace traversal/aliases, and does not expose any `sharedDir`/`sharedFiles`/`commonFiles` config that would satisfy this request.

## Evidence

- **Bootstrap reads are still workspace-root guarded:** `readWorkspaceFileWithGuards()` opens files with `rootPath: params.workspaceDir` and `boundaryLabel: "workspace root"`, so bootstrap/context reads remain constrained to the workspace boundary.
  - file: [src/agents/workspace.ts:60](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/agents/workspace.ts#L60)
