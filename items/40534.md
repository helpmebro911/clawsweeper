## Summary

Keep this PR open. Current main still has the core cron announce gap: the embedded runner creates reasoning payloads, but `buildEmbeddedRunPayloads` does not propagate `isReasoning` into returned payloads, and cron outcome selection still filters only around `isError`. The PR is useful but needs rebasing/adaptation to the current `resolveCronPayloadOutcome` path.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, propagate `isReasoning` from `buildEmbeddedRunPayloads`, keep the run-result and delivery payload types aligned, filter reasoning in `resolveCronPayloadOutcome` and helper selection/fallback paths, exclude reasoning from post-error recovery detection, preserve the current `finalAssistantVisibleText` policy for Telegram/Discord, and add current-path regression tests for mixed reasoning/final text, reasoning-only, and error-then-reasoning-only cron announce cases.
