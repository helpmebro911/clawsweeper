## Summary

Keep this PR open. Current main still has the cron announce reasoning-filter gap, and this PR remains a relevant implementation candidate, but it needs rebasing/adapting to the current centralized `resolveCronPayloadOutcome` path before maintainer review or merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, propagate `isReasoning` from `buildEmbeddedRunPayloads`, include `isReasoning` in the cron `DeliveryPayload` surface, skip reasoning in summary/output/deliverable selection and the `firstText` fallback inside `resolveCronPayloadOutcome`, exclude reasoning from post-error recovery detection, and add regression tests for mixed reasoning plus final text, reasoning-only, and error-then-reasoning-only cron announce cases.
