## Summary

Keep PR #68339 open. Current main already has the basic NVIDIA auth-choice metadata and a test for it, so that part of the PR is partially overtaken, but the PR is not fully implemented or safely obsolete: current main still lacks the provider-local onboarding config/default-model applier, the NIM/on-prem docs section, and the requested `reasoning: true` catalog change for the default Nemotron model.

## Best Possible Solution

The already-landed auth-choice manifest/runtime metadata should be reconciled rather than duplicated.
