## Summary

Keep #31583 open. Current main still blocks the reported `GOG_KEYRING_PASSWORD` case: configured skill env keys are copied into pending overrides, but sensitive-looking names only pass when declared by skill metadata, and the bundled `gog` skill still does not declare the key.

## Best Possible Solution

Keep #31583 open until #50960 or an equivalent maintainer-reviewed fix lands. The narrow fix belongs in `src/agents/skills/env-overrides.ts`: allow explicitly configured `skills.entries.<skill>.env` keys that match sensitive patterns, such as `GOG_KEYRING_PASSWORD`, through the skill env sanitizer while preserving the dangerous host-env blocklist. Add regression coverage for configured undeclared `_PASSWORD`/`_TOKEN` keys and verify the host `exec` child-process path inherits the applied env.
