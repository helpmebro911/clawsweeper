## Summary

Keep open. Current main still treats `*.trajectory.jsonl` as primary session transcripts, so the doctor false-positive targeted by #70812 remains present. The PR is recent, active, and addresses a real bug linked to #70680; the related #72047 was closed unmerged, so it does not supersede this PR.

## Best Possible Solution

Keep #70812 open. The best path is to land or refine a focused fix that excludes true trajectory sidecars from doctor orphan transcript scans while preserving valid session IDs ending in `.trajectory`, with regression coverage in `src/config/sessions/artifacts.test.ts` and `src/commands/doctor-state-integrity.test.ts`; also review whether disk-budget and usage scan call sites need explicit sidecar policy.
