## Summary

Keep open. Current main still has the reported behavior: trajectory sidecars are documented as `<session>.trajectory.jsonl`, but the shared transcript classifier still treats any non-archived `.jsonl` file as a primary transcript, and doctor uses that classifier for orphan archival candidates. PR #70812 is a plausible active fix for open issue #70680, but it should get maintainer review for the disk-budget/sidecar retention edge before merge rather than being auto-closed.

## Best Possible Solution

Keep #70812 open and finish it as the canonical fix for #70680. The best patch should classify trajectory sidecars only with directory context and a matching primary transcript, update doctor and usage scans to pass that context, preserve valid primary transcripts such as `debug.trajectory.jsonl`, and explicitly decide disk-budget behavior for sidecars with regression coverage for doctor orphan detection, usage discovery, valid `.trajectory` session IDs, and budget cleanup.

## Risks / Open Questions

- Merging a helper-only exclusion without a sidecar cleanup policy may make large trajectory sidecars count toward total session disk usage while no longer being eligible for budget removal.
- A filename-only exclusion can regress valid user-provided session IDs ending in `.trajectory`; the PR's later direction to require a matching primary transcript is the right shape, but should stay covered by tests.
