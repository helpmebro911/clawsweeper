## Summary

Keep open. Current main still treats ordinary `*.trajectory.jsonl` sidecars as primary/usage-counted session transcripts, so the doctor orphan false-positive reported by #70680 remains possible. This PR is the active implementation candidate; related #72047 was closed unmerged, and the PR discussion shows useful follow-up on dotted `.trajectory` session IDs plus an unresolved disk-budget/security-sensitive review concern.

## Best Possible Solution

Keep this PR open as the active fix candidate for #70680. Maintainers should review and land the session-artifact classifier/doctor regression fix, preserve the dotted `.trajectory` session-id behavior, add or document explicit disk-budget sidecar handling, and split the unrelated node exec workdir changes unless they are intentionally part of the same landing decision.
