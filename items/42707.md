## Summary

Keep open. Current `main` still lacks Telegram subagent lifecycle hook registration and still does not hydrate persisted Telegram delivery context on `--session-id` resumes, so the reported behavior is not implemented on `main`. The PR also has unresolved review feedback on its proposed target normalization logic, so it is not a safe conservative close.

## Evidence

- **Telegram entry does not register subagent lifecycle hooks:** `extensions/telegram/index.ts` declares the bundled Telegram channel entry, but unlike Discord/Matrix/Feishu it has no `registerFull(...)` hook wiring and no Telegram subagent hook registration path.
  - file: [extensions/telegram/index.ts:3](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/extensions/telegram/index.ts#L3)

With no Telegram hook registration on `main`, the PR's reported failure path still exists.
