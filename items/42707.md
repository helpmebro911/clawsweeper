## Summary

Keep this PR open. Current main still does not implement Telegram subagent lifecycle hooks, and the agent resume path still does not hydrate requester routing from the persisted session entry. Parts of the PR are now superseded by main, especially shared session-binding and Telegram thread-binding state, so the PR should be rebased rather than merged as-is. The existing review comments are useful: the implementation direction is relevant, and the target-prefix normalization concern remains material.

## Best Possible Solution

Keep this PR open as a stale but relevant implementation candidate. Rebase it onto current main; implement Telegram subagent_spawning, subagent_delivery_target, and subagent_ended via a Telegram registerFull/subagent-hooks-api.ts pattern; use plugin SDK facades and existing Telegram target/topic helpers; update the hook boundary invariant allowlists; preserve the already-shipped global adapter/thread-binding state; hydrate resolveAgentRunContext from the resolved SessionEntry on --session-id resumes; and add focused tests for Telegram topic-bound sessions_spawn(thread=true, mode="session"), resume routing hydration, unbind cleanup, and legacy telegram:group:/tg: target forms.
