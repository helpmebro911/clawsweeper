## Summary

Keep PR #47162 open. Current main has not implemented the requested Python/shell scanner coverage: `src/security/skill-scanner.ts` still only treats JS/TS-family files as scannable, and the colocated tests only assert those extensions. The requested change is also in a real core security path because gateway-backed skill dependency installs call the built-in scanner before running installer metadata. The PR is not a conservative close candidate: it addresses a real gap, and the provided review context shows earlier feedback was partially addressed while later concrete review comments remain unresolved rather than superseding the work.

## Best Possible Solution

Keep PR #47162 open for maintainer review or author revision. The best path is to update the core scanner in `src/security/skill-scanner.ts` with Python and shell coverage, include tests for `.py`, `.sh`, `.bash`, extensionless shebang shell entrypoints, non-literal Python command execution, and false-positive guards, then coordinate with the broader open scanner PRs if maintainers prefer one canonical security-scanner change set.
