## Summary

Keep this PR open. Current main still has the dispatch-level empty-output path the PR targets: an undefined resolver result becomes `replies = []`, and if no callbacks queued tool/block/final output, `dispatchReplyFromConfig` records completion and returns `{ queuedFinal: false, counts: { final: 0, ... } }` without a user-visible fallback.

## Best Possible Solution

Keep the PR open and review/update it against current main. The maintainer path is a narrow fallback after dispatch counts are known, only for direct-style user delivery with no routed origin, no suppressed delivery, no queued tool/block/final output, no block-stream output, and no intentional handled/no-text branch, plus regression tests for the direct empty-output case and the channel/group no-op case.
