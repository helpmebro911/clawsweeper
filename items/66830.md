## Summary

Keep this issue open. Current main still computes the preflight compaction and memoryFlush thresholds from `reserveTokensFloor`, while the config/docs/Pi settings path treat `reserveTokensFloor` as a minimum floor under the user-facing `reserveTokens`. The provided GitHub context also shows PR #69379 is open, unmerged, and uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review or land PR #69379, or an equivalent core fix that resolves an effective reserve for both preflight compaction and memoryFlush gates from configured `reserveTokens` plus `reserveTokensFloor` using the same semantics as `applyPiCompactionSettingsFromConfig`. Add regression coverage for `reserveTokens > reserveTokensFloor` and preserve the deliberate `threshold <= 0` behavior for small-window models.
