## Summary

Keep open. Current main still computes the preflight compaction and memoryFlush token gates from `reserveTokensFloor`, not the user-configured `reserveTokens`, while Pi settings and docs treat `reserveTokensFloor` as a floor under `reserveTokens`. The provided GitHub context also shows open PR #69379 with closing syntax for this issue, so this should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review or land PR #69379, or an equivalent core fix that resolves an effective reserve as `max(configured reserveTokens, reserveTokensFloor)` for both preflight compaction and memoryFlush gates, with regression coverage for `reserveTokens > reserveTokensFloor`. After that merges, this can close as implemented.
