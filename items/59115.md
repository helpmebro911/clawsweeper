## Summary

Keep open. Current `main` still drops Slack forwarded/share attachments from thread-history context, so this PR is not implemented on `main` and still addresses a real user-facing gap. The PR discussion also shows one remaining edge-case review concern after the follow-up commit.

## Evidence

- **Current thread-history code still ignores Slack attachments:** `resolveSlackThreadHistory` on current `main` only retains messages with trimmed `text` or `files`, and the page message type does not include `attachments`, so forwarded/share payloads are still omitted from thread history.
  - file: [extensions/slack/src/monitor/thread.ts:109](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/slack/src/monitor/thread.ts#L109)
