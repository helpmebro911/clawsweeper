## Summary

Keep #66498 open. Current main still explicitly marks Gemini web_search freshness as unsupported, short-circuits freshness/date filters before making the Gemini request, and documents Gemini search as query/count-only. The request is recent, concrete, and not superseded by the broader native passthrough issue #17925, which remains open for other provider passthrough work.

## Best Possible Solution

Keep #66498 open. The likely product/code path is to add provider-owned Gemini freshness support in `extensions/google/src/gemini-web-search-provider.runtime.ts`, mapping OpenClaw freshness values (`day/week/month/year` and Brave shortcuts if accepted through shared normalization) to Gemini `google_search.timeRangeFilter` start/end timestamps, update the tool schema/docs, include freshness in the cache key, and add focused tests for accepted and rejected freshness/date-filter combinations.
