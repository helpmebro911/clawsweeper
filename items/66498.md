## Summary

Current main still does not implement Gemini `web_search` freshness. The bundled Google plugin explicitly marks `freshness`, `date_after`, and `date_before` unsupported, the runtime returns the shared unsupported-filter response before constructing the Gemini request, and the request body still sends only `tools: [{ google_search: {} }]`. The related open issue #17925 is broader native-search passthrough work, not a canonical replacement for this narrower Gemini freshness gap.

## Best Possible Solution

Keep this issue open. The best path is a Google-plugin-owned fix or maintainer decision: verify the Gemini grounding API contract, then either map OpenClaw `freshness`/date filters into a supported native request field if one exists, or deliberately implement a documented query/date normalization strategy if native freshness is unavailable. Any change should update `extensions/google/src/gemini-web-search-provider.ts`, `extensions/google/src/gemini-web-search-provider.runtime.ts`, cache-key construction, focused Google web-search tests, and `docs/tools/gemini-search.md`.
