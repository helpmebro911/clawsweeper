## Summary

Keep open. Current main still rejects Gemini web_search freshness before making a Gemini request: the Google plugin advertises freshness/date filters as unsupported, the runtime calls the shared unsupported-filter guard up front, and the Gemini request still sends only an empty google_search grounding tool. The prior ClawSweeper comment remains accurate against current main; the broader open native-search passthrough issue is adjacent but not a canonical replacement for this narrower Gemini freshness gap.

## Best Possible Solution

Keep this issue open. The likely fix belongs in the Google plugin: verify the current Gemini grounding contract, map OpenClaw freshness/date_after/date_before into Gemini google_search timeRangeFilter if supported, remove or narrow the unsupported-filter guard for Gemini, include the normalized filter in the Gemini cache key, add focused tests for request construction/errors/cache separation, and update docs/tools/gemini-search.md and docs/tools/web.md.
