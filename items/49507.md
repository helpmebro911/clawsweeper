## Summary

Keep open. Current main partially addresses the issue by falling back to an agent/channel binding when no account is available, but it does not implement the requested `agentId` -> `accountId` inference and it still gives a same-channel session `lastAccountId` precedence over the agent binding. That matches the wrong-account failure mode described in the issue comments, so there is not enough evidence to close this as implemented.
