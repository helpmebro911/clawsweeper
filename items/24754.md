## Summary

Keep #24754 open. Current main has browser WebChat/Control UI chat, device pairing, Tailscale/HTTPS access, and PWA/web push pieces, but I did not find native browser-chat E2EE, WebRTC/WebCrypto chat transport, or per-device/per-agent chat keys that satisfy the requested capability. A ClawHub/plugin path may be plausible, but not high-confidence enough to auto-close because the request touches first-run web UX, transport/security semantics, and possibly core web/gateway surfaces.
