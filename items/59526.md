## Summary

Keep open. The request is a plausible feature request for account/profile selection directly inside the `/models` picker/buttons. Current documented behavior shows text-based per-session profile routing via `/model ...@<profileId>`, while the Discord `/model`/`/models` picker is documented as provider/model/runtime selection with Submit, not account/auth-profile selection.

## Evidence

- **Issue request:** #59526 asks for choosing the backing GPT/Claude account directly from the `/models` buttons/menu, with current account visibility and no raw provider/account model string editing.
- **Current picker surface:** The Discord docs describe `/model` and `/models` as opening an interactive picker with provider, model, and compatible runtime dropdowns plus Submit; this does not cover an account/auth-profile selector.
  - file: [docs/channels/discord.md:453](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/channels/discord.md#L453)
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Existing profile workaround is text-based:** OAuth docs say multiple profiles in one agent can be selected globally via `auth.order` or per-session via `/model ...@<profileId>`, with example `/model Opus@anthropic:work`. That addresses account routing, but not the requested `/models` button/menu UX.
  - file: [docs/concepts/oauth.md:183](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/concepts/oauth.md#L183)
