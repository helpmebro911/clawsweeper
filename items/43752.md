## Summary

Current main still does not implement the PR’s behavior: the effective gateway port reaches lock acquisition, but the JSON lock payload written by `acquireGatewayLock` omits it. The PR remains a plausible focused bug fix, with a small rebase/schema adjustment needed for current main.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, add `port?: number` to the Zod `LockPayloadSchema` as well as the payload type/write path, retain the 1-65535 guard, and land it with focused `gateway-lock` tests for provided-port persistence and absent-port compatibility.
