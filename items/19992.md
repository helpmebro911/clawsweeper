## Summary

Current main addresses parts of #19992 by blocking gateway start/restart on invalid config, validating OpenClaw-owned config writes before commit, keeping rotating config backups, and restoring invalid startup configs from last-known-good backups. But the specific requested behavior in the issue title/body is not fully implemented: the restart path still does not auto-run `doctor`, and there is no `config restore --last` style restore command on main.

## Evidence

## Risks / Open Questions

- Closing this would overstate what shipped: the mainline implementation blocks invalid restarts and preserves backups, but it does not yet provide the issue's explicit auto-doctor-before-restart workflow.
- The requested restore UX remains unimplemented on main; operators still rely on last-known-good auto-recovery, rejected payloads, or manual backup/doctor flows instead of a dedicated `config restore --last` command.
