## Summary

Keep open. Current `main` contains only part of the proposed fix: rollover now clears inherited `sessionFile`, but isolated cron still re-derives transcript paths with `resolveSessionTranscriptPath(...)` instead of the store-aware `resolveSessionFilePath(...)` flow described in the PR. The extra coverage mentioned in the PR discussion is also not present on `main`, so there is not strong evidence this was already implemented elsewhere.

## Evidence
