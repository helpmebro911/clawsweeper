## Summary

Keep PR #71860 open. Current main still treats successful structured exec completions as user-relayable heartbeat events when a delivery target exists, and current tests still encode that behavior. The PR is a focused fix with the mixed-batch review concern addressed in the provided discussion; related #72218 is adjacent final HEARTBEAT_OK payload work, and #72253 overlaps but is not clearly canonical enough to auto-close this useful PR.

## Best Possible Solution

Keep #71860 open for maintainer review. The best path is to decide the heartbeat exec-completion contract, then either land/refine this narrow successful-code-0 suppression with its regression tests or explicitly consolidate it into a chosen canonical PR such as #72253. Do not close it as cleanup until that canonical path is settled or an equivalent main-branch fix lands.
