## Summary

Keep #71860 open. Current main still treats structured successful exec completions as relayable exec-event output, and the newer related #72253 is broader but not a complete superseder of this PR's stronger user-delivery suppression behavior.

## Best Possible Solution

Keep #71860 open for maintainer review or fold its stronger suppressUserDelivery behavior into the broader #72253 approach. The best final fix should classify only trusted/local structured exec completions as internal, filter internal-only success output from mixed batches, suppress user delivery for benign completions even if the model returns visible text, preserve real failure relay, and cover these paths with focused heartbeat prompt and runner tests.
