## Summary

Keep this issue open. Current main at aac83e00cfe72e5395b4cdfe77ead1c534fe7ef8 still matches the reported macOS screen.snapshot behavior: malformed params are swallowed into defaults, snapshot data is base64-encoded into JSON without a size guard, and the same author has an open unmerged fix PR, #68186, that is the implementation candidate.

## Best Possible Solution

Keep this issue open and review or land #68186, or apply an equivalent macOS runtime fix: distinguish omitted params from malformed params, return INVALID_REQUEST before invoking snapshotScreen on malformed input, enforce the gateway payload budget before base64 JSON emission, and add focused MacNodeRuntimeTests coverage. Close only after the implementation merges.
