## Summary

Current main still matches the reported bug: the generic_repeat detector is warning-only, warning outcomes do not block tool execution, and process action=list is not treated as a known polling tool. The issue remains actionable, and the related implementation PR #60248 is still only an open candidate in the provided context rather than shipped behavior.

## Best Possible Solution

Keep this issue open until main changes. The best maintainer path is a narrow runtime fix in src/agents/tool-loop-detection.ts that evaluates genericRepeat against criticalThreshold before the warning branch, updates unit and before-tool-call e2e coverage, and aligns docs/config-help. Maintainers should review or replace PR #60248, and separately decide whether process action=list belongs in known poll/no-progress detection.
