## Summary

Keep #68701 open. Current main still preserves `to` for `channel: "webchat"` in the shared delivery-context normalizer, so the PR's stated normalization change has not landed. The PR also has an unresolved review concern: its current structured-prefix heuristic would keep `telegram:*`/`channel:*` values on webchat contexts, so the useful next step is a narrower fix rather than closing it as implemented or obsolete.

## Best Possible Solution

Keep the PR open and ask for a narrower implementation: webchat normalization should preserve legitimate webchat-owned direct-session targets such as `webchat:*`, but strip phone-number-shaped values and structured cross-channel targets such as `telegram:*`, `channel:*`, or other non-webchat prefixes. Add regression coverage for both the preserved webchat target and stripped cross-channel structured target paths, then run the changed test lane.
