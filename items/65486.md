## Summary

Current main still has the reported restart/resume gap. Exec approvals are process-local, stale or missing approval IDs still resolve to INVALID_REQUEST, transcript replay only strips tool-result details rather than expiring approval-pending results, and main-session restart recovery generically resumes tool-result tails without approval-aware repair. The prior ClawSweeper keep-open comment remains consistent with current main; no merged fix or canonical superseding item was found in local history/context.

## Best Possible Solution

Keep this issue open. The fix should live in session replay/restart recovery or in a deliberate approval-manager persistence contract: when resumed history contains an exec tool result with status approval-pending whose ID is not present in the live manager, rewrite it to an expired/error tool result so the agent re-issues exec, or securely persist and re-register the pending approval with original command binding metadata. Add a regression test that simulates gateway restart with an approval-pending transcript tail and proves the resumed turn does not keep resolving the stale ID.
