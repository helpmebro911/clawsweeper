## Summary

Keep open. Current main does not implement the requested resolved outbound target context for the `message_sending` hook. The hook event type and delivery call sites still expose raw destination/content plus routing metadata, while target resolution and cross-context detection remain internal outbound/channel logic. The related items in the provided GitHub context are adjacent security-hook proposals, not a canonical replacement for this focused SDK/API request.

## Evidence

- **Hook event type has no targetContext fields:** `PluginHookMessageSendingEvent` currently contains `to`, `content`, optional `replyToId`, optional `threadId`, and optional `metadata`; it does not expose `targetContext`, `isExternal`, `isCrossContext`, `targetType`, `channel`, or `isSameAccount`.
