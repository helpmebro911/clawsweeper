## Summary

Keep #37878 open. Current main still drops numeric-string costs, still admits boolean and non-finite numeric costs in the model-usage helper paths, and has no regression coverage for the requested edge cases. The linked implementation PR #37877 is closed and unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Keep #37878 open and implement the narrow fix from PR #37877: add a shared finite cost parser in `skills/model-usage/scripts/model_usage.py` that accepts finite numeric strings, rejects booleans, rejects `NaN`/`Infinity`, and is used by `aggregate_costs`, `pick_current_model`, and `latest_day_cost`. Add focused regression tests in `skills/model-usage/scripts/test_model_usage.py` for numeric-string aggregation/current/latest-day behavior and for bool/non-finite values being ignored.
