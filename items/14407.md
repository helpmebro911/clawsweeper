## Summary

Keep #14407 open. Current main has partial Control UI reveal controls for gateway auth/config secrets, but the requested cross-platform password visibility pattern is still missing from Skills API-key UI and multiple native macOS, iOS, and Android credential fields. The linked PR #7124 is closed and unmerged in the provided GitHub context, and no relevant app/UI files changed since the prior keep-open review.

## Best Possible Solution

Keep the issue open and implement a reusable cross-platform sensitive-input reveal pattern: extend the existing Control UI reveal controls to Skills/API-key fields, add SwiftUI reveal wrappers for macOS and iOS SecureField credential entries, add Android Compose password fields with PasswordVisualTransformation plus trailing visibility icons, and cover the touched credential surfaces with focused view/UI tests where feasible.
