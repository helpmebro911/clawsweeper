## Summary

Keep #14407 open. Current main has partial web reveal controls for gateway credentials and generic config secrets, but the requested cross-platform password visibility toggle is still missing from Skills API-key entry, iOS/macOS Swift secure fields, and Android Compose gateway credential fields.

## Best Possible Solution

Keep the issue open until OpenClaw either implements a reusable sensitive-input reveal control across web Skills/config/login/overview plus iOS, macOS, and Android credential fields, or a maintainer explicitly narrows the requested platform scope. The likely path is adding reveal state and icon controls to `ui/src/ui/views/skills.ts`, Swift secure-field wrappers for iOS/macOS credential and API-key fields, Compose trailing-icon password fields for Android, and focused UI tests for each touched surface.
