## Summary

Keep PR #45979 open. Current main still has the shrinking loop-bound bug in `src/gateway/server-maintenance.ts`: the loop condition reads `params.dedupe.size - DEDUPE_MAX` while deleting from the same map. The PR’s proposed precomputed `excess` fix is not present in the checkout, and the existing maintenance tests do not cover dedupe overflow trimming.

## Best Possible Solution

Keep the PR open and either merge the minimal stable-bound fix or recreate it on main with a regression test in `src/gateway/server-maintenance.test.ts` that seeds more than `DEDUPE_MAX` dedupe entries, advances the maintenance timer, and asserts the map trims to exactly the cap while removing the oldest entries.
