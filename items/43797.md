## Summary

Keep this issue open. Current main at c2d82b87eee7a86bbd0e851169e0ca5559c44fe6 still prunes sandbox runtimes and registry entries without deleting the local sandbox workspace directory, and the provided GitHub context shows #43835 remains an open, unmerged PR using closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review #43835 as the current implementation candidate. The preferred fix belongs in `src/agents/sandbox/prune.ts`: after successful runtime removal for non-shared sandbox registry entries, derive the workspace path from `cfg.workspaceRoot` plus the registry `sessionKey`, remove it recursively with safe error handling, preserve shared and still-in-use workspaces, and add focused regression tests for session, agent, and shared scope behavior.
