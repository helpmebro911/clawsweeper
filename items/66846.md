## Summary

Keep open. Current main still does not implement `agents.defaults.compaction.memoryFlush.model`: the config type/schema/docs do not expose it, and the memory flush runtime still gates, sizes, and executes the flush with the session provider/model. The PR remains a focused implementation candidate for linked feature request #66802 rather than conservative cleanup.

## Best Possible Solution

Keep this PR open and review it as the implementation path for #66802. The best code path is to add `agents.defaults.compaction.memoryFlush.model` consistently across config types, strict zod schema, generated schema/help/labels, human compaction docs, and runtime resolution, with tests covering provider/model overrides, model-only overrides, session fallback, threshold sizing against the resolved flush model, CLI-provider gating against the resolved flush provider, and provider-scoped auth behavior.
