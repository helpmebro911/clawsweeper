## Summary

Keep this PR open. Current main still does not implement `agents.defaults.compaction.memoryFlush.model`: the public config type/schema/help/generated/docs surfaces omit it, and the memory-flush runtime still gates and runs the flush using the session provider/model. The linked feature request #66802 remains open, and this PR is a focused implementation candidate rather than conservative cleanup. Security review found no workflow, dependency, lockfile, lifecycle-script, or package-resolution changes in the PR patch; the security-sensitive part is the intended model/provider routing of memory-flush context, which needs maintainer review before merge.

## Best Possible Solution

Keep this PR open and review it as the implementation path for #66802. The best product/code path is to add `agents.defaults.compaction.memoryFlush.model` consistently across config types, strict zod schema, generated schema/help/labels, user docs, and runtime resolution; ensure threshold sizing, CLI-provider gating, fallback options, and provider-scoped auth all use the resolved flush provider/model; add focused tests for provider/model override, model-only override, session fallback, smaller-context models, CLI-provider skips, and provider/auth profile scoping; then rebase and run the changed validation lanes.
