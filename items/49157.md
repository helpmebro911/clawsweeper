## Summary

Keep open. Current main has useful mitigations for session lock cleanup, including an in-process watchdog and exit/signal cleanup, but it still preserves the core risk described in the issue discussion: live-PID lock files remain valid until the 30-minute stale window, and long or no-timeout runs can set an effectively unbounded max hold window. I do not see enough source-backed evidence that the reported Gateway/session deadlock is fixed on main.
