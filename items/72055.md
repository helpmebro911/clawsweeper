## Summary

Keep PR #72055 open. Current main still lacks the requested `plugins.entries.bonjour.config.instanceName` path: the bundled Bonjour plugin always derives the advertised instance name from `ctx.machineDisplayName`, while the advertiser already supports an `opts.instanceName` override. The PR is a focused plugin-owned config/schema/docs/test change for the already-reported multi-gateway mDNS name collision in #54467, and the current plugin SDK/loader surface is sufficient for that implementation.

## Best Possible Solution

Keep the PR open for maintainer review and CI. The best path is to land the plugin-owned config implementation in the Bonjour plugin: add `instanceName` to `extensions/bonjour/openclaw.plugin.json`, read and defensively normalize `api.pluginConfig.instanceName` in `extensions/bonjour/index.ts`, preserve the existing suffix behavior, and keep the docs/tests/changelog coverage from the PR. No new core `gateway.bonjour.*` config seam appears necessary.
