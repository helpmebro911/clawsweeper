## Summary

Keep open. Current main still does not implement the installer no-op skip or `--force` override, and #44700 was closed specifically because this PR is the canonical implementation thread. The PR should not be closed as implemented or obsolete; it needs revision for the unresolved review concern that the skip check must verify the npm-managed install, not any matching `openclaw` found on PATH.

## Best Possible Solution

Keep #44986 open, but require a revision before merge: the installer skip should compare the requested registry version against the npm-managed OpenClaw package itself, for example by reading `$(npm root -g)/openclaw/package.json` or otherwise proving the resolved binary belongs to the active npm global bin. Preserve the `--force` / `OPENCLAW_FORCE_INSTALL=1` override and keep finalization diagnostics such as PATH warnings on the skipped path.
