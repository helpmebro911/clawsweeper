## Summary

Current main has implemented the restart-side LaunchAgent hardening from merged PR #72616, including re-bootstrap recovery and detached restart handoff. But this issue also explicitly reports the `gateway stop` path, and current source/tests still keep the 10 x 100 ms stop confirmation window and bootout fallback that warns it left the service unloaded. That remaining stop behavior is not fully implemented-on-main.

## Best Possible Solution

Keep this issue open and narrow it to the remaining macOS `gateway stop` semantics. Maintainers should either implement the remaining stop-side fix by extending the stop confirmation budget or making bootout fallback restore a recoverable disabled-but-loaded state, or explicitly document and close that stop bootout is intentional while relying on the already-merged restart/re-bootstrap recovery paths.
