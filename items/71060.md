## Summary

Keep open. Current main has the earlier macOS LaunchAgent restart hardening from related reports, but the specific stop-path behavior reported here is still present: `stopLaunchAgent` waits only 10 x 100 ms before falling back to `launchctl bootout`, and the warning/test coverage still codify that fallback as leaving the service unloaded. The restart/self-restart part is partially mitigated by kickstart-first and detached handoff logic, but that does not remove the still-present unload-on-slow-stop path.

## Best Possible Solution

Keep this open for a targeted macOS LaunchAgent lifecycle fix. The likely product/code path is to either lengthen `waitForLaunchAgentStopped` to match the gateway drain budget, avoid bootout for ordinary slow drains, or re-bootstrap into a disabled-but-registered state after any bootout fallback, with regression tests that prove `gateway stop` remains persistent while the LaunchAgent is not left unregistered. The existing restart kickstart-first and detached-handoff behavior should be kept.
