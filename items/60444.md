## Summary

Keep open. Current main still uses `structuredClone` in the session-store cache read/write path, `loadSessionStore`, auth-profile store cloning, and several gateway/secrets/config clone paths that this PR targets. The PR discussion also includes a concrete linked bug (#45438), benchmark evidence, and a review note that the PR author addressed with a follow-up JSDoc commit, so this is a plausible focused bug fix rather than cleanup noise.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a narrowly validated JSON-safe clone helper for the proven hot paths, preserve cache mutation-isolation behavior with tests, and decide separately whether the expanded gateway/secrets/config clone replacements should land in this PR or a smaller follow-up.
