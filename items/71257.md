## Summary

Keep open. Current `main` still reads raw session transcript text and writes it directly into the bundled `session-memory` Markdown artifact, so the PR is not obsolete or implemented. The linked bug #71031 remains a live core issue, and the PR review discussion identifies useful but still-unresolved implementation details rather than a reason to close it.

## Best Possible Solution

Keep #71257 open and consolidate it with #71031 and the overlapping #69961 work. The best fix belongs in the bundled `session-memory` transcript/persistence path, preferably by reusing existing shared sanitizers for assistant-visible text, directive tags, and `MEDIA:` parsing, plus targeted regression tests proving memory files do not persist `NO_REPLY`, chat-template tokens, tool-call XML, or real OpenClaw directive/media markers.
