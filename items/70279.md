## Summary

Keep open. Current main has addressed several Claude CLI setup pitfalls and now emits model-fallback skip diagnostics, but the issue’s remaining maintainer requests are still actionable in core/docs: root/systemd Claude CLI guidance is absent, there is no CLI-backend-specific skip log, and fallback auto-pinning in session state still appears intentional for non-user overrides.

## Best Possible Solution

Keep this issue open and narrow it to remaining core/docs work: add a CLI-backend-facing skip diagnostic or document where operators should look for `model-fallback/decision`, add public CLI backend docs for root-managed systemd Claude CLI behavior with a supported non-root or `IS_SANDBOX=1` stance, and decide whether auto fallback persistence should be suppressed when the configured primary was skipped before dispatch.
