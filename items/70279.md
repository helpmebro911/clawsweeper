## Summary

Keep open. Current `main` has better fallback diagnostics than the original report, but the core item is not implemented end-to-end: docs still do not cover the root/systemd Claude CLI constraint, the bundled `claude-cli` path still auto-injects `--permission-mode bypassPermissions` without any root-aware guard or supported `allowRoot` switch, and fallback winners are still intentionally persisted into `sessions.json` as auto model overrides.

## Evidence

- **CLI backend docs still omit the root/systemd Claude CLI caveat:** The current CLI backend docs explain PATH issues for launchd/systemd, Claude login, and the automatic `--permission-mode bypassPermissions` mapping, but they do not mention running the gateway as `root`, the Claude CLI root restriction, or any `IS_SANDBOX=1` workaround.
