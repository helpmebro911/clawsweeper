## Summary

Keep open. Current `main` still processes the full `agent_end` message history on every successful turn in `extensions/memory-lancedb`, so the reported inefficiency is not implemented away. The PR discussion also surfaced unresolved correctness problems in the proposed cursor approach around failure retries, compaction/reset, and async hook overlap, so there is not strong evidence for a conservative close.

## Evidence

- **Current main still rescans full history:** `extensions/memory-lancedb/index.ts` still iterates `event.messages` directly inside the `agent_end` hook and then applies `toCapture.slice(0, 3)`. There is no per-session cursor, dedupe state, or reset/compaction handling on current `main`.
  - file: [extensions/memory-lancedb/index.ts:613](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/extensions/memory-lancedb/index.ts#L613)
