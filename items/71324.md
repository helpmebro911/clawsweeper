## Summary

Do not close #71324. Current main still has the narrower #71229-era history sanitizer: it strips unquoted internal runtime context and hides empty/heartbeat rows, but it does not implement the PR's remaining legacy projection cases for quote-wrapped internal-context rows, background-task status prefixes, gateway restart status blocks, or startup/async runtime prompt rows. The PR is an active, focused Gateway/WebChat bug fix with concrete regression-test scope and should stay open for maintainer review rather than being treated as implemented or superseded.

## Best Possible Solution

Keep #71324 open as the active Gateway/WebChat follow-up. The best path is to land a narrowly scoped history-projection fix in `src/gateway/chat-sanitize.ts`/`src/gateway/session-history-state.ts` with regression tests for quote-wrapped internal context, background-task statuses, gateway restart blocks, startup/runtime prompt rows, and mixed rows that preserve real user text. Reviewer comments about over-stripping and assistant-role rewriting should be resolved or explicitly accepted before merge.
