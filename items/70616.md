## Summary

Keep #70616 open. Current main has partial internal `execution` normalization for memory dreaming, but the public `memory-core` plugin schema still rejects `dreaming.execution`, the documented user-facing dreaming config only exposes `enabled` and `frequency`, and the dream narrative subagent path still runs without a model or fallback override. The existing cron subsystem already has a model/fallback payload contract that could be reused, but memory-core dreaming does not yet expose or wire it.

## Best Possible Solution

Keep the issue open and implement this in the bundled `memory-core` plugin contract: add a documented `dreaming.execution` shape with `model` and ordered fallbacks (or intentionally reconcile it with the existing `execution.defaults` helper), update the strict plugin manifest schema and tests, and wire the resolved values into managed dreaming agent-turn/narrative execution using the existing cron model/fallback contract where applicable.
