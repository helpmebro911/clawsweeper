## Summary

Keep #70616 open. Current main has a partial internal `dreaming.execution.defaults` normalizer, but the user-facing memory-core plugin schema still rejects `dreaming.execution`, docs do not list it, and the runtime does not pass model or fallback settings into dreaming narrative subagent runs. The requested per-dreaming model/fallback configuration remains actionable in this repo.

## Best Possible Solution

Keep the issue open. A complete fix should extend the memory-core plugin schema and docs for a supported `dreaming.execution` shape, decide whether fallback models are per-run or mapped through existing agent fallback mechanics, propagate the selected model/fallbacks into dreaming narrative subagent runs, and add schema/runtime tests proving the requested config is accepted and used.
