## Summary

Issue #51509 targets the removed Chrome extension relay/browser-relay options flow. Current `main` has no browser relay subsystem to extend; OpenClaw migrated to Chrome MCP `existing-session`/named browser profiles instead.

## Close Comment

Closing after a Codex review because this request no longer matches current `main`. OpenClaw removed the legacy Chrome extension relay/browser-relay path in `2026.3.22` (`CHANGELOG.md:2211`), and the runtime now keeps `ensureExtensionRelayForProfiles()` only as an intentional no-op (`extensions/browser/src/browser/server-lifecycle.ts:9`). Current migration/docs move old `driver: "extension"` configs to Chrome MCP `existing-session` and remove `browser.relayBindHost` (`src/commands/doctor/shared/legacy-config-core-normalizers.ts:27`, `docs/gateway/doctor.md:205`). The supported browser-control model is now named browser profiles via `--browser-profile` / `?profile=<name>` (`docs/tools/browser.md:370`), not relay-extension connection profiles. Since the reported browser-relay options flow is no longer part of OpenClaw core, there is no current mainline path here to reproduce or land this request against.
