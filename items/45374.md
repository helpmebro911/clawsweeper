## Summary

Keep open. Current main has adjacent doctor cleanup for stale `plugins.allow`/`plugins.entries` and bundled `plugins.load.paths` aliases, but it does not implement the PR's service version/state-dir drift reporting in `gateway status`/doctor. The PR is not cleanly obsolete; it should be rebased or split against the newer doctor/plugin-repair architecture and related open follow-ups.

## Best Possible Solution

Keep this PR open for maintainer judgment. The best path is to rebase and split it: drop or adapt the parts already covered by current stale allow/entry and bundled load-path repairs, then separately evaluate a focused service runtime drift patch for `gateway status`/doctor, coordinated with #42252 and the dangling channel cleanup in #65293.
