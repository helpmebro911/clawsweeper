## Summary

Keep #38066 open. Current main still has the internal Telegram draft-stream knobs, but it does not expose `channels.telegram.streamThrottleMs` or `minInitialChars` through config types, strict schema, docs, or the dispatcher. The related PR #38255 is the active implementation path, but it is open/unmerged, so the request is not implemented on main.

## Best Possible Solution

Keep #38066 open until PR #38255 or an equivalent maintainer-approved change lands. The fix should add Telegram preview cadence config to the exported Telegram config type, strict Zod schema, generated config metadata, and public docs; resolve top-level/per-account values through the existing Telegram account merge path; pass them into `createTelegramDraftStream`; and cover defaults, validation/clamping, fallback behavior, and account overrides with focused tests.
