## Summary

Keep #14593 open. Current main at cf834e2 still does not provide an official Docker/Linuxbrew path for brew-backed skill installs, and the reported `openai-whisper` path remains brew-only. The latest issue discussion also includes an explicit maintainer Codex review from steipete saying to keep this open while focused Docker/skills fixes remain candidates.

## Best Possible Solution

Keep #14593 open. The best maintainer path is a focused Docker/skills fix: add a maintainer-approved opt-in Linuxbrew build/setup/docs path for the non-root Docker gateway image, add non-brew installers or OS/container scoping for bundled brew-only skills, or change onboarding so the official Docker flow does not offer impossible brew installs. The active related PRs from the issue discussion can remain implementation candidates until one lands and is verified against Dockerfile/setup/docs plus onboarding and skill-install behavior.
