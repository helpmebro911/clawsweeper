## Summary

Keep #14593 open. Current main still does not provide an official Docker/Linuxbrew install path, the Docker setup flow only passes apt/plugin/sandbox build args, `openai-whisper` remains brew-only, and onboarding still offers a brew-backed install that resolves to the `brew not installed` failure when no brew executable exists. The latest maintainer/Codex issue comment also explicitly says to keep this open while fix PRs remain candidates.

## Best Possible Solution

Keep #14593 open. The best maintainer path is a focused Docker/skills fix: either add an official opt-in Linuxbrew setup that works for the non-root Docker image and is documented/tested, add non-brew or OS-scoped installers for bundled brew-only skills, or change onboarding so the official Docker flow does not offer installs that are impossible in the shipped container. Active PR candidates such as #57553/#55940 should remain implementation candidates until one lands and is verified against Dockerfile/setup/docs plus onboarding and skill-install behavior.
