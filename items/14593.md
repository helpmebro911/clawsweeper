Keep #14593 open. Current main still lacks an official Docker/Linuxbrew path, the Docker setup script does not pass or persist a brew install option, the reported bundled `openai-whisper` skill is still brew-only, and the installer still returns `brew not installed` when no brew executable is found. The issue also has a same-day maintainer/Codex keep-open comment and active unresolved fix-PR references, so conservative cleanup should not close it.

## Best Possible Solution

Keep #14593 open until a focused Docker/skills fix lands. The likely path is an official opt-in Linuxbrew setup for the non-root Docker image, OS-scoped or non-brew installer metadata for bundled brew-only skills such as `openai-whisper`, or onboarding/status behavior that does not offer impossible brew installs in the shipped Docker flow. Use the active fix PR discussion (#57553/#42303/#55940) as input and validate across `Dockerfile`, `scripts/docker/setup.sh`, `docs/install/docker.md`, bundled skill metadata, `src/agents/skills-install.ts`, and `src/commands/onboard-skills.ts` tests.
