## Summary

Keep PR #66478 open. Current main still sends the raw trimmed `emoji` value into Feishu reaction add/remove/list paths, while the Feishu helper and local constants are built around Feishu `emoji_type` strings such as `THUMBSUP`. The linked bug report and PR discussion provide concrete reproduction/error evidence, and the PR is a focused fix rather than obsolete or already implemented.

## Best Possible Solution

Keep the PR open and finish it as the canonical fix for #66406: normalize agent-provided unicode emoji and case-insensitive Feishu type strings before Feishu add/remove filtering calls, keep unknown values surfaced to the API, add focused regression coverage for `👍` and known type pass-through, and address the remaining minor review note by hoisting the known-type `Set` if the latest PR branch has not already done so.
