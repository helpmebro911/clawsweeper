## Summary

Keep this PR open. Current main still lacks the proposed `agents.defaults.reminderGuard` and per-agent `reminderGuard` opt-out, and the shipped reminder guard still only recognizes core `cron.add` success or an existing enabled core cron job for the session. The linked plugin-tool false-positive remains a real product gap unless maintainers land this PR or replace it with an equivalent core/plugin contract.

## Best Possible Solution

Keep this PR open for maintainer review, then land or adapt the config-based `reminderGuard` opt-out. A longer-term plugin SDK signal for successful external scheduling may be cleaner, but that should be a separate CODEOWNER-reviewed API proposal; current main still needs a shipped mitigation for plugin-managed reminder tools.
