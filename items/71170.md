## Summary

Keep #71170 open. Current main still has the reminder guard hard-wired to core cron evidence only, current config/schema surfaces do not expose `reminderGuard`, and external plugin tools do not have a documented public way to tell the guard that a reminder was scheduled. The PR is a focused, recent core bug-fix/config opt-out for #70666 rather than stale or duplicate cleanup.

## Best Possible Solution

Keep #71170 open for maintainer review as the current focused fix for #70666. The best path is to rebase/validate the PR against current main and either merge the additive `reminderGuard` opt-out or explicitly redirect the work to a narrower SDK scheduling-signal proposal; it should not be auto-closed because current main still lacks an equivalent fix.
