## Summary

Keep open. Current main still lacks `agents.defaults.reminderGuard`, any per-agent `reminderGuard` override, or any plugin-tool scheduling signal that would suppress the false reminder note for externally scheduled reminders. #70666 was closed as superseded by this PR, so #71170 remains the canonical open tracker rather than an obsolete or ClawHub-only item.

## Best Possible Solution

Keep #71170 open for maintainer review. Rebase it onto current main, fix the changelog attribution, then either land the minimal additive `agents.defaults.reminderGuard` plus per-agent override or explicitly supersede it with a maintainer-approved plugin SDK scheduling-signal design. Treat adjacent Telegram outbound sanitization work, such as #66732, as a separate surface rather than a reason to close this PR.
