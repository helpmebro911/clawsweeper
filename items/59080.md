## Summary

Keep open. Current `main` still routes uncategorized assistant error strings straight to user-facing payloads, so a plain JavaScript `JSON.parse` message would still leak if it reaches `lastAssistant.errorMessage`. The malformed tool-call repair on `main` is also scoped to `kimi`/`anthropic-messages` and `openai-completions`, not generic `anthropic-messages` sessions like the reported `nex` path. There is partial mitigation for replayed Anthropic history, but not a landed end-to-end fix for this issue.
