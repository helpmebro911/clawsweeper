## Summary

Keep #47505 open. Current main has picked up a few overlapping accessibility improvements, but it still lacks several central parts of this PR: skip-link/sr-only infrastructure, raw JSON editor VoiceOver mitigation, generated config-form label/help wiring, chat rotor headings, semantic pinned-message list, command-palette roles, and sessions table sort semantics. The PR is a concrete implementation candidate and its own review thread shows the author addressed the main early ARIA/UI review findings rather than abandoning them.

## Best Possible Solution

Keep the PR open and have the author or a maintainer refresh it against current main, preserving the still-missing accessibility fixes while dropping or reconciling pieces that main has already implemented. The likely landing path is a scoped web-UI accessibility patch with tests for generated config form associations, command palette/session table roles, and chat semantics, plus documented manual VoiceOver/Safari verification for behaviors that are hard to automate.
