## Summary

Current main still has the active data-loss path described in the later issue comments: unclosed reasoning tags are stripped in strict mode before Telegram delivery. Raw free-form Telegram text is HTML-escaped before parse_mode=HTML, so the original parse-mode explanation is partly obsolete, but the issue remains actionable because the current strict-mode stripper still drops the tail and existing tests codify that behavior. Related PRs #49337 and #49996 are open/unmerged follow-ups rather than a shipped fix.
