## Summary

Keep open. Current main still has partial provider/model attribution for fallback attempts, but the terminal exhausted-fallback wrapper does not carry structured session_id/sessionKey/lane metadata. The prior ClawSweeper comment correctly identified the remaining gap, and the current main SHA still shows the same product behavior.

## Best Possible Solution

Keep this issue open and implement a narrow logging/fallback attribution fix: pass sessionId/sessionKey or lane into runWithModelFallback from agent-command and auto-reply paths, carry it on FallbackSummaryError or the final wrapper log context, emit attempted providers/models plus normalized session_id/sessionKey in structured JSONL fields, and add focused tests around model_fallback_decision and the terminal `Embedded agent failed before reply` log path.

## Risks / Open Questions

- A fix should preserve existing provider/model attempt observability while adding normalized session attribution; changing only the human-readable error string would not satisfy structured log processors that depend on top-level fields.
