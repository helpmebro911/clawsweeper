## Summary

Keep this issue open. The discussion includes a maintainer update explicitly keeping the runtime traversal problem open, and current main only documents the QMD limitation/workaround; it does not add symlink pruning, realpath cycle detection, or QMD exclusion controls before `qmd update` traverses configured collections.

## Best Possible Solution

Keep this issue open as a focused QMD traversal robustness item. Add a controlled regression around a workspace-visible temp monorepo symlink loop, then make traversal cycle-safe/no-follow upstream or add an OpenClaw-side QMD exclusion/pruning policy before `qmd update` can recurse. The current docs workaround should remain, but it is not the product fix.
