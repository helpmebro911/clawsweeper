## Summary

Keep #45421 open. Current main still lacks Slack `ackReactionTiming` / `run-start` support in config, schema, docs, and runtime. The PR addresses a real remaining behavior gap; its review finding about scope gating was already handled in the PR by ce7bce913, but that work is not present on main.

## Best Possible Solution

Keep #45421 open and either rebase/adapt the PR or land an equivalent Slack-scoped implementation. The implementation should add `ackReactionTiming` to Slack config types, zod schema, labels/help, and docs; preserve `received` as the default; defer the initial ack/status reaction until `onAgentRunStart` only when configured; continue honoring `ackReactionScope`; and cover both default and gated deferred behavior in Slack tests.
