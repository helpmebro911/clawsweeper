## Summary

Keep open. Current `main` still strips `:thread:` from cron delivery inference before parsing the session key, so Telegram direct-message topic context is still lost. The PR remains a relevant, scoped bug fix rather than something already implemented or obsolete.

## Evidence

- **Current main still strips thread context:** `inferDeliveryFromSessionKey()` still calls `parseAgentSessionKey(stripThreadSuffixFromSessionKey(rawSessionKey))`, and `stripThreadSuffixFromSessionKey()` removes everything after the last `:thread:` marker. For a key like `agent:velizar:telegram:direct:834275911:thread:834275911:197918`, that leaves plain peer id `834275911` rather than a Telegram topic target.
