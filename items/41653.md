## Summary

Keep open. Current main still has the unguarded `WebSocketTaskBox.sendPing()` continuation resume that this PR targets, and the keepalive loop still calls that path. The functional fix is useful, but this PR also adds an unrelated Swift CI workflow even though current `ci.yml` already has a scoped `macos-swift` lane, so the best maintainer path is to keep/review the crash fix while dropping or reworking the workflow change.

## Best Possible Solution

Keep the PR open, ask the contributor or a maintainer to split/remove the new workflow, and land a focused `GatewayChannel.swift` fix that gates `sendPing()` continuation resumption exactly once. If there is a preferred duplicate PR for the same crash, pick one canonical implementation and close the others only after that path is explicit or merged.
