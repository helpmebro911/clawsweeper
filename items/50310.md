## Summary

Keep open. Current main has partial mitigation for archived Discord threads and a gateway-level `sessions.delete`, but the requested stale/orphan cleanup is still not fully implemented: deleted/inaccessible Discord threads are only unbound from thread bindings, there is no Discord channel-existence/session GC sweep or `gcIdleHours` config, and there is no agent-facing `sessions_delete` or `sessions_gc` tool. Related #49259 tracks broader orphan-session pruning, but it does not fully supersede this Discord-specific agent/thread cleanup request.

## Evidence

- **Archived-thread handling is partial:** `DiscordThreadUpdateListener` only handles `THREAD_UPDATE` payloads that report `isThreadArchived(data)` and then calls `closeDiscordThreadSessions`; there is no corresponding delete listener path here.
  - file: [extensions/discord/src/monitor/listeners.ts:785](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/extensions/discord/src/monitor/listeners.ts#L785)
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **Deleted/inaccessible thread sweep does not clean sessions:** When the Discord thread-binding sweep sees a gone-thread error, it calls `manager.unbindThread({ reason: "thread-delete" })`; this removes the binding record but does not call the session close/delete helper.
  - file: [extensions/discord/src/monitor/thread-bindings.manager.ts:305](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/extensions/discord/src/monitor/thread-bindings.manager.ts#L305)

  - file: [extensions/discord/src/monitor/thread-session-close.ts:46](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/extensions/discord/src/monitor/thread-session-close.ts#L46)
