## Summary

Keep #11487 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still does not expose a canonical optional `name`, `label`, or `description` field for channel group config entries; the relevant schemas, TypeScript types, generated metadata, docs, and tests remain strict and behavior-only.

## Best Possible Solution

Keep the issue open. The best implementation is to choose one canonical optional string metadata field, add it consistently to core and bundled channel group-entry types and Zod schemas, preserve it through config load/write while ignoring it functionally at runtime, regenerate config metadata, update public group/channel docs, and add validation tests proving the field is accepted without changing group behavior.
