## Summary

Keep #11487 open. Current main still does not provide a canonical optional `name`, `label`, or `description` field for channel group config entries. The prior Codex review comment correctly identified the gap, and the current 4a3030df checkout still has strict behavior-only group schemas, generated metadata that forbids extra group-entry fields, docs that list only behavior fields, and tests that reject unknown group-entry keys.

## Best Possible Solution

Keep the issue open. The best path is to choose one canonical optional string metadata field, add it consistently to core and bundled channel group-entry types and Zod schemas, preserve it through config load/write while ignoring it functionally at runtime, regenerate config metadata, update public group/channel docs, and add validation tests proving the field is accepted without changing group behavior.
