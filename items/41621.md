## Summary

Current main still shares one `skillsFilter` between the Agents Skills panel and the top-level Skills tab, so the PR's core state-isolation change has not been implemented or superseded. The diff is narrowly scoped UI state wiring and does not introduce a security-sensitive surface, but it should be rebased and ideally covered by a small regression test before landing.

## Best Possible Solution

Keep this PR open or replace it with an equivalent maintainer patch. The best fix is to add an isolated `agentSkillsFilter` state for the Agents Skills panel, keep the top-level Skills tab on `skillsFilter`, update the view-state type, and add focused regression coverage or manual verification that filtering in one surface does not affect the other.
