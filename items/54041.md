## Summary

Keep open. Current main still only partially supports Anthropic request shaping: model params can add raw `anthropicBeta` values and `context1m` headers, but the requested Anthropic `context_management` payload support, server compaction/context editing behavior, and `memory_20250818` native tool path are not implemented or documented. The discussion includes a concrete production +1, and the related 1M-context trackers do not supersede this broader request.

## Best Possible Solution

Keep this open for maintainer/product design in the bundled Anthropic provider. A good implementation would add documented Anthropic model params for beta/header handling, validated `context_management` body shaping, tests for header and payload mutation, replay/stream handling for provider compaction blocks and stop reasons, and a clear decision on whether `memory_20250818` can be wrapped over existing memory tools or needs a new provider-defined-tool SDK seam.
