## Summary

Keep open. Current main still does not implement the requested Anthropic context-management beta support: raw `anthropicBeta` headers and `context1m` handling exist, but there is no first-class Anthropic `context_management` payload shaping, no compaction/context-editing stream and replay handling, no `pause_after_compaction` behavior, and no `memory_20250818` provider-defined tool path. The related open issues #10213 and #48558 cover narrower slices, but they do not supersede this broader request.

## Best Possible Solution

Keep this issue open for maintainer/product design in the bundled Anthropic provider. A complete fix should add documented Anthropic model params for beta/header handling, validated `context_management` payload shaping for compaction and context editing, stream and replay support for compaction blocks and `pause_after_compaction`, request coverage for direct Anthropic transport and provider wrappers, and a clear decision on whether `memory_20250818` is provider-owned Anthropic behavior or needs a generic provider-defined-tool SDK seam.
