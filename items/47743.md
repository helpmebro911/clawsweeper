## Summary

Keep open. Current main still does not implement an explicit plugin trust-tier field, resolver, CLI display, or trust-tier docs. The PR is a plausible core plugin/security metadata change rather than ClawHub/plugin-discovery work, but it needs the existing review concern fixed before merge.

## Best Possible Solution

Keep the PR open and ask for a rebase plus the review fix. If maintainers want this model, implement trust tier as core plugin metadata derived from the effective plugin format, surface it consistently in list/info/JSON output, document the content/native boundary and reserved sandboxed tier, and add regression tests for bundle, native, and undefined-format fallback cases.
