## Summary

Keep this issue open. #63096 is still open and unmerged, uses closing syntax for this issue, and current main still forwards completion-wake gateway runs with only the caller-provided extra system prompt instead of rebuilding the persisted session Inbound/Group context that normal chat assembles. The image-history cache churn raised in the comments also remains a separate unresolved concern rather than a proven fixed behavior.

## Best Possible Solution

Keep this issue open while maintainers review #63096. The best path is to land or reject that PR on its merits, require regression or live cache-trace proof for same-session prompt/cache stability across normal chat, heartbeat, and completion wakes, and split the image-history cache churn into a narrower follow-up if it is not fixed by the same work.
