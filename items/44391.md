## Summary

Keep open. Current `main` still contains the reported followup dedupe bug, and this PR also bundles unrelated behavior changes with plausible edge-case regressions raised in review.

## Evidence

- **Current main still dedupes followup payloads before any cross-target check:** `resolveFollowupDeliveryPayloads()` unconditionally runs `filterMessagingToolDuplicates()` and `filterMessagingToolMediaDuplicates()` on all followup payloads, then separately checks `shouldSuppressMessagingToolReplies()`. There is no guard limiting dedupe to same-target sends, so matching text/media from a cross-target messaging tool send can still be removed from the origin reply path.
