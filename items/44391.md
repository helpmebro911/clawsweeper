## Summary

Do not close #44391. Current main still has the followup-specific cross-target dedupe gap: followup payloads are routed through `resolveFollowupDeliveryPayloads`, which unconditionally removes text/media matching messaging-tool sends before checking whether the tool sent to the same origin target. The normal agent reply path already has the cross-target guard and tests, but the followup path does not. The PR should be reworked rather than closed or merged as-is because the discussion shows unresolved review concerns around bundled, unrelated transient-retry and delivery-serialization changes.

## Best Possible Solution

Keep #44391 open and ask for a focused rebase against current main: port the cross-target dedupe guard into `resolveFollowupDeliveryPayloads`, preserve legacy dedupe when target metadata is absent, add followup delivery/runner tests for cross-target text and media, and split or drop the unrelated transient-retry, delivery-serializer, and Mattermost dependency changes unless they are handled in separate reviewed PRs.
