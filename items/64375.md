## Summary

Current `main` still only honors `deleteAfterRun` for `kind: "at"`, so this PR has not been implemented and the reported behavior remains plausible for `every`/`cron` jobs. The PR also adds missing regression coverage for those schedule kinds, and the review discussion does not show a new blocker beyond a pre-existing persistence/audit tradeoff that already applies to one-shot jobs.
