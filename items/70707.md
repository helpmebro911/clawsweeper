## Summary

Keep PR #70707 open. Current main still has the race shape described in the PR: device removal/token rotate/token revoke respond first, then defer socket disconnect with queueMicrotask; per-RPC dispatch only rechecks shared gateway auth generation, not device-token invalidation. The PR’s proposed `invalidated` client flag, `invalidateClientsForDevice` context hook, dispatch guard, and regression tests are not present on main. The provided Greptile review also supports the PR as targeted and well-tested rather than obsolete.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a focused fix equivalent to PR #70707: add synchronous per-device client invalidation in the gateway request context, mark affected clients before responding in `device.pair.remove`, `device.token.rotate`, and `device.token.revoke`, reject invalidated clients at the top of per-RPC dispatch, and keep regression tests for the response-ordering invariant and context behavior.
