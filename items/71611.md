## Summary

Keep #71611 open. Current main still uses direct `fs.rename()` in the memory atomic-reindex swap, so the linked Windows EBUSY bug is not implemented on main. The PR is a relevant fix attempt but remains unmergeable as submitted because its patch breaks template literals, sidecar suffix handling, and backup-path uniqueness.

## Best Possible Solution

Keep the PR open for revision or replace it with a clean maintainer patch. The fix should port the cron `renameWithRetry` behavior into `extensions/memory-core/src/memory/manager-atomic-reindex.ts`, preserve suffix handling for the base SQLite file plus `-wal` and `-shm`, keep the `randomUUID()` backup path, and add targeted tests for EBUSY retry plus EPERM/EEXIST fallback before closing #64187.
