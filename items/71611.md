## Summary

Keep this PR open. Current main still has the Windows-sensitive direct rename path in memory atomic reindex, so the intended bug fix is not implemented on main. The PR targets the right owner file and mirrors an existing cron-store pattern, but the submitted patch is not mergeable as supplied because the provided diff drops template literals, sidecar suffix handling, and the unique backup UUID.

## Best Possible Solution

Keep this PR open for revision or replace it with a clean maintainer patch. Port the existing cron-store renameWithRetry behavior into extensions/memory-core/src/memory/manager-atomic-reindex.ts, preserve base/-wal/-shm suffix handling and the randomUUID backup path, add focused atomic-reindex tests for retry/fallback behavior, and keep #64187 as the canonical bug until the corrected implementation lands.
