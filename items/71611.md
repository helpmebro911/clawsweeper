## Summary

Keep #71611 open. Current main still lacks the requested Windows EBUSY retry/copy fallback in memory atomic reindex, and the linked Windows bug remains concrete. The PR is relevant but currently unmergeable because its submitted diff breaks template literals and sidecar/backup path handling.

## Best Possible Solution

Keep the PR open for revision or replace it with a clean maintainer patch: port the cron `renameWithRetry` pattern into `extensions/memory-core/src/memory/manager-atomic-reindex.ts`, preserve suffix handling and the `randomUUID()` backup path, add tests for EBUSY retry plus EPERM/EEXIST fallback across SQLite sidecar files, and keep #64187 as the canonical bug until the corrected implementation lands.
