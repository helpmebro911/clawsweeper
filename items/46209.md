## Summary

Keep #46209 open. Current main appears to have fixed the original Control UI heartbeat metadata overwrite by treating heartbeat/cron/exec turns as system events and preserving existing display/session route metadata, but the PR's specific label-before-transcript derived-title behavior is still not implemented. Several non-Control-UI consumers still prefer `derivedTitle` ahead of `displayName`/`label`, so the PR may still contain useful work after rebasing/narrowing.

## Best Possible Solution

Keep the PR open, but ask for a current-main rebase and narrow it to the remaining derived-title precedence issue: add `entry.label` and `entry.origin?.label` before transcript-derived titles in the existing `deriveSessionTitle` path, preserve the existing truncation/session-id tests, and add focused regression coverage for label/origin-label precedence. The already-shipped system-event metadata preservation should remain the canonical fix for the original Control UI heartbeat overwrite.
