## Summary

Keep open. Current main has absorbed some Telegram fixes, but this PR is not fully implemented or safely obsolete: getFile failures still return null into the placeholder path, mention-gating drops still lack the requested verbose log, command-menu sync still hashes only the command list, and media-group handling still drops non-recoverable item failures. The batch should be split and ported rather than merged as-is.

## Best Possible Solution

Keep this PR open for maintainer handling, but do not merge the batch as-is. Split and port the still-relevant Telegram fixes into the current extensions/telegram layout, starting with the getFile/placeholder failure path, command-menu cache invalidation, mention-gating verbose logging, and media-group item handling. Preserve the safer current retry and approval authorization boundaries, and leave already-absorbed pieces such as ACP placement, reply parameters, proxy handling, and word-boundary chunking on main.

## Risks / Open Questions

- The PR targets the older src/telegram layout while current main uses extensions/telegram, so merging the batch as-is would be high-conflict and likely to regress newer plugin-layout code.
- The approval callback changes are security-sensitive because unauthorized Telegram users should remain silent unless they are explicitly configured approvers.
- Retry, media download, proxy, and media-group behavior affect visible message delivery and network routing; each remaining fix needs a focused port and regression test.
- Several linked issues are closed, but current source still shows relevant behavior, so issue closure alone is not enough to treat this PR as implemented.
