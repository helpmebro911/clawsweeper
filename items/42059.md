## Summary

Keep this PR open. Current main has absorbed some Telegram fixes, but the batch is not fully implemented or safely obsolete; several core fixes remain missing in the current `extensions/telegram` plugin, and the PR targets the older `src/telegram` layout.

## Best Possible Solution

Keep this PR open for maintainer handling, but do not merge it as one large stale batch. Split and port the still-relevant fixes into `extensions/telegram`, prioritizing getFile failure surfacing, command-menu cache invalidation, mention-gating verbose logging, media-group item handling, draft-stream stop races, and Telegram outbound metadata stripping. Preserve already-absorbed current behavior for reply parameters, ACP placement, rate-limit retry handling, proxy transport, and approval callback routing.
