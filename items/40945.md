## Summary

Keep open. Current main still flattens remote markdown image URLs to alt text in the Control UI renderer, and the Control UI CSP/docs still intentionally block remote http(s) image loads. The provided GitHub context also shows open PR #41025 references this issue with closing syntax and remains unmerged, so the issue should stay open while maintainers review, revise, or close that PR.

## Best Possible Solution

Keep this issue open and review #41025 as the implementation candidate. The best fix, if maintainers want remote images, should address both the markdown renderer and the Control UI image-loading policy, likely via an explicit allowlist, proxy/managed-media path, or documented opt-in instead of only allowing http(s) in the renderer.

## Risks / Open Questions

- Allowing remote markdown images would make the operator browser fetch model-supplied third-party URLs, which has tracking and privacy implications and conflicts with the current CSP/docs unless maintainers intentionally change that policy.
- A renderer-only fix would not fully solve the observable behavior on current main because the Control UI CSP would still block remote http(s) image loads.
