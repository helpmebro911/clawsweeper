## Summary

Keep #17810 open. Current main still sorts SOUL.md, IDENTITY.md, USER.md, and MEMORY.md ahead of generic files but renders them as ordinary files under `# Project Context`; there is no `context.elevation`/section config surface, and the linked implementation PR #18110 is closed unmerged. The only current-main commit after the prior Codex review is unrelated WhatsApp work, so the previous keep-open conclusion still matches the checked-out code.

## Best Possible Solution

Continue tracking #17810 for maintainer product direction. If accepted in core, add a documented config/schema surface for deterministic context-file elevation before Project Context, preserve current behavior when unset, exclude elevated files from the flat block unless explicitly retained, and cover prompt rendering plus config/docs tests. If maintainers prefer externalization, document the intended context-engine/plugin pattern clearly enough that contributors can implement it without brittle full system-prompt replacement.
