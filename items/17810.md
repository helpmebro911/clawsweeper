## Summary

Keep #17810 open. Current main still injects SOUL.md, IDENTITY.md, USER.md, and MEMORY.md as ordinary Project Context files, with only deterministic ordering plus a SOUL.md hint. There is no `context.elevation` config/schema/docs/test coverage, and the related implementation PR #18110 is closed unmerged.

## Best Possible Solution

Continue tracking #17810 for maintainer product direction. If accepted in core, add a documented config/schema surface for deterministic context-file elevation before Project Context, preserve current behavior when unset, exclude elevated files from the flat block unless explicitly retained, and cover prompt rendering plus config/docs tests. If maintainers prefer externalization, document the intended plugin/context-engine pattern clearly enough that contributors can implement it without brittle prompt replacement.
