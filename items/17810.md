## Summary

Keep open. Current main still does not implement configurable context-file elevation for SOUL.md, IDENTITY.md, USER.md, or MEMORY.md. The prompt builder orders known bootstrap files and adds a fixed SOUL guidance sentence, but it still renders all stable context files as ordinary `## <path>` entries under `# Project Context`; config/schema/docs expose bootstrap injection and size controls, not `context.elevation` or an `includeInProjectContext` surface. The related PR #18110 is closed unmerged in the provided GitHub context, and the latest maintainer/Codex issue discussion already explicitly kept this open.

## Best Possible Solution

Continue tracking #17810 for maintainer product direction. If accepted, implement this in the agent prompt/bootstrap path with a documented config/schema surface, deterministic matching/order, elevated sections before Project Context, exclusion from flat context unless explicitly retained, and focused system-prompt plus bootstrap/CLI tests and docs. If maintainers prefer externalization, close only after a concrete plugin/API or ClawHub path exists that can restructure bootstrap prompt sections without replacing the whole system prompt.
