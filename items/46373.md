## Summary

Keep open. Current main does not contain the PR's two central changes: Control UI still starts async locale loading without a readiness promise or first-render gate, and agent error formatting/sanitization still lacks a specific tool_use/tool_result mismatch recovery message. The PR is not merge-ready as-is because the existing review comments identify concrete issues in its regex and locale ordering, but it remains a focused bug-fix PR rather than something to auto-close.

## Best Possible Solution

Keep the PR open for maintainer or contributor revision. The best path is to rebase/split if useful, add a public `i18n.ready` startup contract, gate the first Control UI render until the effective locale is loaded with settings taking precedence deterministically, and add a narrowly-scoped tool_use/tool_result mismatch classifier with negative tests for unrelated `tool_use_id` validation errors. Then run the relevant changed UI and agent test/type lanes.
