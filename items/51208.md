## Summary

Keep open. Current `main` still routes hook-mapped agent runs through the isolated cron path, forces a fresh session for those runs, and explicitly withholds stored CLI session IDs on fresh runs. That matches the reported behavior: `cliSessionIds.codex-cli` can be persisted after the first run, but later hook events still will not call the resume path.
