## Summary

Keep this issue open. Current main still has no typed or documented config-level task notify default, and the ACP, subagent, and Task Flow creation paths still fall back to `done_only` unless a per-task policy is passed.

## Best Possible Solution

Implement this as a core config/schema change, likely under `agents.defaults.tasks`: resolve an effective default notify policy before ACP, subagent, and Task Flow records are created; preserve current behavior when unset; keep cron, CLI, and media silent defaults intact; document the config and add focused regression tests for ACP, subagent, Task Flow, cron, media, and per-task overrides.
