## Summary

Keep this issue open. Current main still does not implement an enforced DM-level `toolsBySender` policy for allowlisted DM senders. The shared tool policy resolver remains group/channel scoped, Signal only exposes `toolsBySender` under `channels.signal.groups`, and Telegram's `direct.*.toolsBySender` schema is not wired into the effective tool filtering path for direct sessions.

## Best Possible Solution

Implement a generic direct-message sender tool-policy contract and wire it into effective tool filtering before tools are exposed. A good maintainer path would add typed config/schema/docs for `channels.<channel>.toolsBySender` or a clearly documented per-DM equivalent, resolve direct sender identity consistently for Signal and Telegram, apply the policy alongside global/agent/group policies, and add regression tests showing one allowlisted DM sender can keep full tools while another allowlisted sender is denied tool access.
