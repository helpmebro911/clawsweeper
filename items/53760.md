## Summary

Keep open. Current main still does not enforce DM-level sender tool policies for Telegram, Signal, or the generic tool-policy pipeline. Telegram has typed/schema support for `direct.*.tools` and `direct.*.toolsBySender`, but the active filtering path still resolves only group/channel policies. Signal still documents and types `toolsBySender` only for group config.

## Best Possible Solution

Implement a generic direct-message sender tool-policy contract, then wire it through the same enforced filtering paths as group policy. Add shared typed config/schema/docs for DM-level `tools` and `toolsBySender`, resolve verified direct sender identity for Signal and Telegram, feed the resolved DM policy into `createOpenClawCodingTools`, `applyFinalEffectiveToolPolicy`, gateway scoped tool resolution, and media/tool helpers, and add regression tests proving one allowlisted DM sender can retain tools while another is denied them.
