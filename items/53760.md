## Summary

Keep this issue open. Current main still only applies `toolsBySender` through group/channel tool-policy resolution, not direct-message sender policy. Telegram has a direct-config type/schema field, but the active agent tool filtering never consults it; Signal only exposes `toolsBySender` under group config.

## Best Possible Solution

Implement a generic direct-message sender tool-policy contract rather than a channel-specific workaround. The likely maintainer path is to add typed config/schema/docs for DM-level `toolsBySender`, resolve direct sender identity consistently for Signal and Telegram, feed that policy into both `createOpenClawCodingTools` and `applyFinalEffectiveToolPolicy`, and add regression tests proving one allowlisted DM sender can retain tools while another allowlisted sender is denied them.
