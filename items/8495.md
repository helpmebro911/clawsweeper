## Summary

Keep #8495 open. Current main still blocks unauthorized WhatsApp group traffic but does not leave the group, expose a WhatsApp group-join hook/event, or provide a WhatsApp leave-group message action. The broader related #11489 remains useful context, but it does not supersede the exact WhatsApp auto-leave/manual-leave request, and an existing maintainer-side Codex comment already kept this item open for the same product gap.

## Best Possible Solution

Keep #8495 open. The likely implementation belongs in the bundled WhatsApp plugin: add an explicit auto-leave config/schema/docs path, listen for WhatsApp group membership/add events or reconcile new group membership from current Baileys runtime data, call the platform leave-group API only under an operator-controlled policy, and add focused tests. A separate additive hook/event or broader group pairing flow can be coordinated with #11489, but it should not replace the concrete WhatsApp auto-leave/manual-leave request unless maintainers explicitly choose that product direction.
