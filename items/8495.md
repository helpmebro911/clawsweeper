## Summary

Keep #8495 open. Current main still blocks or skips unauthorized WhatsApp group traffic, but it does not auto-leave unauthorized groups, expose a WhatsApp group-join hook/event, or support a WhatsApp leave-group message action. The related #11489 is broader and still open, so it is not a shipped replacement for this WhatsApp-specific auto-leave request.

## Best Possible Solution

Keep #8495 open for a deliberate bundled WhatsApp plugin feature. The likely path is to add opt-in config/schema/docs for unauthorized-group auto-leave, implement a verified Baileys/runtime group leave operation with account and target authorization, add a group participant/update listener or equivalent join detection, cover allowlist and denied-group behavior with tests, and coordinate the owner notification/group-join event shape with the broader open #11489 design rather than treating #11489 as a replacement.
