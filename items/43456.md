## Summary

Keep this issue open. Current main still does not wake Mattermost `oncall` channel/group sessions for unmentioned replies to the bot's own posts; `root_id` is used for thread/session routing only, and the mention gate has no reply-to-bot implicit mention input.

## Best Possible Solution

Implement this in the bundled Mattermost plugin: resolve the replied-to/root post when needed, verify its `user_id` matches the Mattermost bot user id, feed a `reply_to_bot` implicit mention fact into the Mattermost/shared mention gate, and add regression tests for unmentioned threaded and inline replies in default `oncall` mode.
