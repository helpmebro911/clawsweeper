## Summary

Keep open. Current main still has the heartbeat responsePrefix interpolation gap: heartbeat resolves the raw prefix, does not pass onModelSelected into getReplyFromConfig, and normalizes replies with the uninterpolated prefix. This PR remains a plausible implementation candidate for same-author issue #43064; related PRs #43133 and #43158 were closed unmerged.

## Best Possible Solution

Keep this PR open for maintainer review. The right code path is to rebase or apply the equivalent focused fix on current main: use createReplyPrefixContext in runHeartbeatOnce, pass onModelSelected through getReplyFromConfig, resolve the raw responsePrefix with the populated prefixContext after the LLM call, and add focused heartbeat regression coverage without deep extension internals where feasible.
