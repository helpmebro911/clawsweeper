## Summary

Keep this issue open. Current main has meaningful partial progress, but the bundle still contains active core reliability/security/product requests that are not fully implemented and not fully superseded by focused follow-ups.

## Best Possible Solution

Keep this issue open for maintainer triage, or replace it with focused canonical issues before closure. Split the remaining work by surface: fallback visibility and fallback bootstrap/context re-injection, doctor dry-run/diff preview, per-path filesystem policy for native and MCP file tools, repeated subagent failure circuit breaking, prompt-injection content scanning policy, spend/budget gates, and scheduled model health routing. Close only slices that are already shipped or fully covered by focused follow-ups such as #70976/#70990 for failover hooks and the existing tool-hook work for quality gates.
