## Summary

Keep this umbrella issue open. The linked failover follow-up is narrow and does not supersede the broader issue.

## Best Possible Solution

Keep this issue open for maintainer triage, or replace it with focused canonical issues before closing the umbrella. Split remaining work by surface: fallback visibility and fallback-specific bootstrap/context re-injection, config-write health rollback, graduated crash recovery, doctor dry-run/diff preview, per-path filesystem policy across native and MCP tools, repeated sub-agent failure circuit breaking, prompt-injection scanning policy, spend/budget gates, and scheduled model-health routing. Treat #70976/#70990 as covering only the failover hook slice, not the broader fallback and reliability requests.
