## Summary

Keep open. Current main has useful partial coverage, but the umbrella still tracks several unresolved core reliability/security/product gaps, and the related failover follow-up is explicitly narrower than the remaining fallback visibility and context reinjection requests.

## Best Possible Solution

Keep this issue open for maintainer triage, or replace it with focused canonical issues before closing the umbrella. Treat #70976/#70990 as covering only the failover hook slice unless merged code/docs explicitly cover non-verbose fallback visibility and fallback-specific bootstrap/context reinjection. Split remaining work by surface: fallback visibility/context reinjection, config-write health rollback, crash recovery, doctor preview/diff, per-path filesystem policy, repeated sub-agent failure circuit breaking, prompt-injection scanning policy, spend/budget gates, and scheduled model-health routing.
