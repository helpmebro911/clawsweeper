## Summary

Keep #46660 open. Current main does not implement `agent_to_agent_turn` or an equivalent A2A turn-level plugin seam. Intermediate ping-pong replies are still internal `runAgentStep` calls with `deliver:false`, while plugin-visible message delivery only happens at the final announce send. This is plausible core plugin API work, not a ClawHub-only plugin, because the needed turn number, speaker/listener, flow, and delivery-target context exists only inside the core A2A orchestration path.

## Best Possible Solution

Keep the PR open for maintainer/design review. The best path is a first-class core A2A turn hook emitted from `src/agents/tools/sessions-send-tool.a2a.ts`, registered through the typed plugin hook surface, with tests and docs for turn 0, stable flow IDs, speaker/listener metadata, delivery target including account/thread data, bounded ordered delivery, and conversation-access policy.

## Risks / Open Questions

- Because the proposed event carries reply text, the implementation should explicitly fit the plugin conversation-access/security policy rather than becoming an unguarded content exposure path.
