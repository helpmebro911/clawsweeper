## Summary

Keep open. Current main has safeguard compaction, recent-turn preservation, memory flush, and summary quality audits, but it does not implement the requested loop-aware saturation guard: no `agents.defaults.compaction.guard` config, transcript-tail detector/scorer, guarded instruction augmentation, post-compaction validator, or recommend-reset escalation is present on main. The issue is also not superseded: the linked implementation PR stack remains open/draft in the provided GitHub context.
