## Summary

Keep #68982 open. Current main still does not implement the PR's central behavior: live chat stream state is text-only, gateway chat delta/final payloads are built from text buffers only, and the streaming UI renders a synthetic text message with reasoning disabled. The PR discussion also describes active contributor follow-up and clean/mergeable status, with review comments pointing to concrete fixable plumbing gaps rather than a cleanup-close reason.

## Best Possible Solution

Keep #68982 open for maintainer review or contributor follow-up. The best implementation path is to plumb structured assistant stream payloads end to end: gateway chat delta/final should preserve thinking and text blocks with the existing suppression rules, the UI controller/state and ChatItem types should carry structured messages without collapsing them to text, renderStreamingGroup should honor the same showReasoning visibility policy as finalized history, and focused gateway/controller/UI regressions should cover thinking-only, thinking+text, hidden-reasoning, heartbeat/silent, and final-resync cases. Maintainers can then decide whether #68982 or the broader #59086 is the canonical landing vehicle.
