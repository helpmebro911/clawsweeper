## Summary

Keep PR #71141 open. Current main still lacks the central behavior from the PR: reply normalization does not parse `[[react:...]]` / `[[react_to_current:...]]`, Telegram delivery rejects empty reaction-only replies, and the outbound adapter does not execute `channelData.telegram.reaction`. The linked issue #71140 provides a concrete Telegram symptom and live-validation context, so this is a plausible focused bug fix rather than cleanup-close material.

## Best Possible Solution

Keep the PR open or implement an equivalent fix on main: parse reaction directives during shared reply normalization, preserve them as structured Telegram channel data, allow reaction-only Telegram replies when a target message is available, call the existing `reactMessageTelegram` transport from reply/outbound delivery, and add focused normalization plus Telegram adapter/delivery regression coverage.
