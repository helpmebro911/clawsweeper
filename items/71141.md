## Summary

Keep #71141 open. Current main still does not implement the requested Telegram outbound reaction-directive path: reply directive parsing only handles media/reply/audio/silent fields, outbound normalization does not synthesize `channelData.telegram.reaction`, Telegram reply delivery rejects no-text/no-media replies before handling channel data, and the Telegram outbound adapter only sends text/media plus buttons/quotes. The PR is a focused recent draft tied to open bug #71140, so it remains useful for maintainer review rather than cleanup.

## Best Possible Solution

Keep the PR open for maintainer review or land an equivalent fix: parse `[[react:...]]` / `[[react_to_current:...]]` into structured Telegram channel data, preserve it through outbound normalization, allow reaction-only Telegram replies when a valid reply target exists, call `reactMessageTelegram` before optional text/media sends, and add focused parser plus Telegram delivery/adapter tests.
