## Summary

Keep #11623 open. Current main has adjacent macOS menu-bar status, HoverHUD, WebChat, Canvas, and Talk overlay code, but it still does not implement the requested Clawi feature: multiple configurable floating per-agent bubbles with persisted positions, mouse-follow/anchor behavior, and per-agent chat launch/routing. The provided issue discussion already includes an April 26, 2026 Codex review keeping it open for the same product/API gap, and current main still matches that assessment.

## Best Possible Solution

Keep #11623 open. The best path is either a native macOS implementation under apps/macos/Sources/OpenClaw that derives bubble state from agent/session activity, persists positions, supports drag/anchor or mouse-follow behavior, and routes chat to the selected agent, or a maintainer-designed native macOS UI/window plugin API that would make an external ClawHub implementation feasible.
