## Summary

Current main still does not implement per-agent floating Clawi bubbles for macOS. Existing macOS surfaces are adjacent, but they are menu-bar status, Hover HUD, WebChat, Canvas, and a single Talk Mode orb, not named draggable per-agent bubbles with inline chat and mouse-follow/anchor behavior. The plugin/ClawHub path is also not sufficient yet because the documented plugin API does not expose a native macOS SwiftUI/AppKit window extension point.

## Best Possible Solution

Keep #11623 open. The best path is either a native macOS implementation under apps/macos/Sources/OpenClaw/ that connects bubble state to agent/session activity, persisted positions, and chat routing, or an explicit maintainer-led design for a native macOS UI extension API before moving this feature to plugins/ClawHub.
