## Summary

Keep #11623 open. Current main has adjacent macOS HoverHUD, WebChat, Canvas, and Talk orb surfaces, but it still does not implement multiple configurable floating Clawi agent bubbles with persisted positions, drag/anchor or mouse-follow behavior, or per-agent chat launch/routing. The existing issue discussion also already contains a steipete/Codex review keeping this open, and the current plugin SDK does not expose a native macOS SwiftUI/AppKit window extension point that would make a ClawHub close appropriate.

## Best Possible Solution

Keep #11623 open. The best path is a native macOS implementation under apps/macos/Sources/OpenClaw that derives bubble state from agent/session activity, persists positions, supports drag/anchor or mouse-follow behavior, and routes chat to the selected agent. If maintainers want this outside core, first design and document a native macOS SwiftUI/AppKit window extension point, then move the implementation to a plugin/ClawHub track.
