## Summary

Keep this PR open. Current main has shipped a related `logging.file` config fix for #61295, but it has not implemented this PR's root `--log-file` / `OPENCLAW_LOG_FILE` override path, and the PR branch itself still has unresolved review feedback around routed-command fast paths and daemon status env handling.

## Best Possible Solution

Keep the PR open as a useful implementation candidate, but require a revision before merge: add root `--log-file` parsing for both Commander and routed fast paths, add an explicit `OPENCLAW_LOG_FILE` resolver with correct precedence, make daemon status resolve against the inspected service environment when applicable, and keep the existing `logging.file` config fix from main as the baseline.
