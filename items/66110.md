## Summary

Keep this PR open. Current main still lacks the root `--log-file` option and `OPENCLAW_LOG_FILE` file-sink override, so the PR is not obsolete. The provided PR diff remains a useful implementation candidate, but review comments correctly identify remaining route-first and daemon-status environment gaps that should be fixed before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate, but require revisions before merge: add root `--log-file` to parser/help, promote it before both Commander and route-first execution paths, implement an explicit `OPENCLAW_LOG_FILE` resolver with precedence over `logging.file`, thread an env parameter through log-path resolution so daemon status can use the inspected service environment, and retain focused tests/docs for those boundaries.
