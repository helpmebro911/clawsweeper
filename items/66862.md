## Summary

Keep open. Current main has not implemented the requested no-flag 1M behavior for Anthropic Opus/Sonnet 4.6, and the linked bug #66766 remains open. The PR addresses a real user problem, but as written it is still incomplete because it changes the context resolver without aligning the Anthropic beta-header capability path, docs, and header tests.

## Best Possible Solution

Keep the PR open for maintainer review or rework. The best fix is a coordinated change for #66766 that keeps `resolveContextTokensForModel()`, Anthropic beta-header injection, OAuth/legacy-token fallback behavior, status/session limits, tests, and Anthropic provider docs in agreement. If product direction is no-flag 1M for Opus/Sonnet 4.6 API-key auth, the request wrapper should send the 1M beta for that path while preserving a clear 200k fallback for OAuth/legacy-token auth.
