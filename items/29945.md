## Summary

Keep #29945 open. Current main still hardcodes branded pairing replies for newly created unpaired DM pairing requests, has no `unpairedResponse` config contract, and the direct maintainer PR #39065 is still identified in the provided GitHub context as open/unmerged with `Closes #29945`.

## Best Possible Solution

Keep #29945 open until #39065 or an equivalent maintainer-approved change lands. The fix should add an explicit unpaired-DM response policy, wire it through shared pairing helpers and affected channel/plugin call sites, update config schema/generated metadata and public docs, and add regression coverage for silent, code-only, and branded behavior across representative channels.
