## Summary

Keep #8972 open. Current main still supports exact Discord guild channel keys plus a literal `*` catch-all and parent-channel fallback for threads, but not glob/prefix patterns like `ws-*`. The category-based alternative is also not implemented in the Discord config schema. The related #60582 context covers category-level channel config specifically, so it does not supersede the main glob/wildcard request.

## Best Possible Solution

Keep the issue open until Discord channel allowlist matching supports a documented pattern form such as glob/prefix entries, or until maintainers intentionally split the category alternative into the already-related #60582 path. A complete fix should update the Discord matcher, schema/config docs, and regression tests for `ws-*` matching a dynamic channel name without requiring a per-channel config entry.
