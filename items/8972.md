## Summary

Keep #8972 open. Current main still resolves Discord guild channel allowlists by exact id/slug/name, normalized exact match, thread-parent fallback, and the literal `*` wildcard only. A configured `ws-*` entry remains a literal key and will not match `ws-task-1`; the category alternative remains separately tracked by open issue #60582.

## Best Possible Solution

Keep #8972 open. Implement a documented, opt-in Discord guild channel pattern form such as `ws-*`, likely in the shared channel matcher or Discord resolver, with exact matches taking precedence, existing thread-parent behavior preserved, literal `*` remaining the final fallback, schema/types/docs/status/security-audit updates, and regression coverage proving dynamically created matching Discord channels are allowed without per-channel config edits. Track category inheritance through #60582 unless maintainers decide to merge both designs.
