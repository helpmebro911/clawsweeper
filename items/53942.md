## Summary

Keep this issue open. Current main still does not provide user-level targeted overrides or a trust-level setting for the cited Safety, Tool Call Style, or self-update permission sections. The available mechanisms are whole-prompt replacement, limited provider/model overlays for selected sections, and plugin hook mutation, which do not satisfy the proposed typed user config.

## Best Possible Solution

Keep this issue open for core agent configuration design. The likely path is a documented, schema-backed user-level and per-agent prompt policy surface for selected sections or an explicit trust policy, wired into `src/agents/system-prompt.ts`, generated config schema/help/docs, and tests proving Safety, Tool Call Style, and self-update permission text can be selectively replaced or relaxed without requiring full `systemPromptOverride`. Coordinate with the broader #25493 prompt-template discussion without losing this narrower section-level safety-policy requirement.
