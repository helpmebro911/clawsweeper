## Summary

Keep this issue open. Current main still has whole-prompt replacement, GPT-5 personality overlay controls, and provider/runtime-owned section overrides for a small set of sections, but it does not provide the requested user-level targeted overrides or trust policy for Safety, Tool Call Style, or OpenClaw Self-Update/update-permission guidance.

## Best Possible Solution

Keep this open for core agent configuration design. A conservative implementation would add a documented, schema-backed user-level and per-agent prompt policy surface for selected sections, or an explicit trust policy, wire it through `src/agents/system-prompt.ts`, config schema/help/generated docs, and gateway config mutation guards, and add tests proving Safety, Tool Call Style, and OpenClaw Self-Update guidance can be selectively replaced or relaxed without requiring full `systemPromptOverride`. Coordinate with #25493 while preserving this issue's narrower safety-policy requirement.

## Risks / Open Questions

- Relaxing Safety and self-update guidance changes OpenClaw's operator trust boundary and should get explicit maintainer product/security judgment.
- Whole-prompt replacement exists today, but using it as the answer would force users to fork and maintain the entire OpenClaw prompt rather than selectively override the cited sections.
- Plugin hooks can mutate prompt text, but that is plugin-owned/compatibility behavior rather than a documented, schema-backed user-level override for these core sections.
