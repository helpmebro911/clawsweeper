## Summary

Keep open. Current main has adjacent model failover and auth-profile rotation/backoff controls, but it does not implement the requested configurable main LLM retry policy with max attempts, custom backoff array, retryOn classes, and total wall-clock cap. The linked PR #49800 also does not supersede the issue because its own scope says it adds only hardcoded same-profile overloaded retries and no new config surface, and that PR is still open in the provided context.

## Evidence
