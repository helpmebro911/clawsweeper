## Summary

Keep open. Current main still matches the reported core scheduling gap: bootstrap maintenance runs through the foreground/default path, deferred debt execution is only enabled for background maintenance, and the only automatic background scheduling path is turn maintenance. The hot-cache policy details appear to live in the external lossless-claw context-engine implementation, but OpenClaw’s host scheduling/API behavior still needs a core fix or clearer contract before this can be closed.

## Best Possible Solution

Keep this issue open. A narrow core fix should make bootstrap/reconcile-created deferred compaction debt schedule or run an allowed background maintenance pass, or extend the context-engine maintenance contract so engines can explicitly request debt-consuming follow-up or report that policy intentionally superseded the debt. Add regression coverage for the bootstrap path, then coordinate the hot-cache policy semantics with lossless-claw so OpenClaw can either pay down debt or surface a non-sticky policy reason.
