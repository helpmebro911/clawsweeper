## Summary

Current main still appears to have the reported OpenClaw host scheduling gap. Bootstrap maintenance reaches `runContextEngineMaintenance` without `executionMode`, defaults to foreground, and only turn-triggered maintenance can be deferred into the background path that enables `allowDeferredCompactionExecution`. The hot-cache policy details look plugin-owned, but the host still lacks a bootstrap/reconcile debt-consuming follow-up path.

## Best Possible Solution

Keep this issue open. The best path is a narrow OpenClaw host change that lets bootstrap/reconcile-created deferred compaction debt schedule or run an allowed background maintenance pass, or an additive context-engine maintenance contract for engines to request a debt-consuming follow-up or explicitly report that policy superseded debt. Add a bootstrap-path regression test and coordinate the hot-cache behavior with the Lossless Claw plugin.
