## Summary

Keep #39923 open. Current main still implements numbered config backup rotation, the active config write paths still call that maintenance module, and tests still assert `.bak.1` through `.bak.4` while treating non-ring `.bak.*` suffixes as orphans. The related implementation PRs in the provided GitHub context remain unmerged or closed unmerged, so the requested datetime-suffix behavior has not shipped.

## Best Possible Solution

Keep #39923 open until a datetime-suffix implementation lands or maintainers explicitly decide to keep the numbered backup ring. A complete fix belongs in `src/config/backup-rotation.ts` with matching updates in `src/config/config.backup-rotation.test.ts`, covering bounded pruning, chmod discovery, collision behavior, and deliberate handling of legacy `.bak.N` files. The open implementation PRs #39982 and #40392 are the current candidate follow-ups rather than a reason to close the issue.
