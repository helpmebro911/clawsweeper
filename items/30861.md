## Summary

Keep open. Current main partially improves multi-agent safety by making the active session selector persist in the chat header, grouping sessions by agent, and persisting the selected session across switches/reloads, but it does not implement the requested persistent active-agent plus workspace indicator or switch confirmation feedback.

## Evidence

- **Chat header still only renders the session/model/thinking controls:** The chat page header delegates to renderChatSessionSelect(state), and that renderer outputs only the session <select> plus model/thinking controls; there is no dedicated active-agent/workspace context block in the header UI.
  - file: [ui/src/ui/app-render.ts:1414](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/app-render.ts#L1414)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Session selector shows agent-aware labels, but only inside the picker:** Current main groups sessions by parsed agent id and prefixes duplicate labels so the collapsed native select keeps the agent visible. This is a partial mitigation, not the requested always-visible agent/workspace badge.
  - file: [ui/src/ui/chat/session-controls.ts:475](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/chat/session-controls.ts#L475)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Workspace is not surfaced in chat-header code:** A targeted search for "workspace" across the chat-header implementation files returned no matches, so the requested persistent workspace label is not implemented in the dashboard chat header.
  - command: `rg -n "workspace" ui/src/ui/chat/session-controls.ts ui/src/ui/views/chat.ts ui/src/ui/app-render.ts ui/src/ui/app-render.helpers.ts`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Workspace is still shown only on the Agents overview surface:** The existing workspace display lives in the Agents panel overview card, not in the chat/dashboard header. Users must navigate away from chat to see it.
  - file: [ui/src/ui/views/agents-panels-overview.ts:108](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/ui/src/ui/views/agents-panels-overview.ts#L108)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **No switch-confirmation toast implementation found:** A targeted UI search found no toast or "Switched to ..." implementation on current main, so the requested switch feedback is still missing.
  - command: `rg -n "toast|Switched to|switched to" ui/src/ui ui/src/styles --glob '!**/*.test.ts'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)

## Risks / Open Questions

- The existing session-picker improvements reduce ambiguity somewhat, so maintainers could mistakenly treat this as done even though the requested workspace visibility and explicit switch feedback remain absent.
- Session persistence across reloads is already wired through settings/URL sync, which means a future implementation likely needs only UI/header work rather than backend/session plumbing.
