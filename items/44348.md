## Summary

Keep open. Current `main` does not implement the PR's requested API change: `before_agent_start` and `agent_end` event payloads still omit `sessionKey` and `agentId`. Those values are available today only on the second `ctx` argument, so there is a workaround, but the additive event-surface change proposed by #44348 has not landed.

## Risks / Open Questions

- The PR likely needs a rebase: `main` now routes legacy prompt-build behavior through `src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts`, and hook type definitions are split across `hook-before-agent-start.types.ts` and `hook-types.ts`, so the March patch shape no longer matches current file layout exactly.
- There is already a viable workaround via `ctx.agentId` and `ctx.sessionKey`, so maintainer judgment is still needed on whether duplicating those fields onto event payloads is worth expanding the public plugin contract.
