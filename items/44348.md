## Summary

Current main has not implemented PR #44348's requested event-payload fields. The agent/session identity is available on the second hook context argument, but `before_agent_start` and `agent_end` event objects still omit `sessionKey` and `agentId`, so this remains a real public plugin API decision rather than a conservative auto-close.

## Best Possible Solution

Keep the PR open for maintainer judgment. If accepted, implement the additive optional fields in the current hook contract files, thread them conditionally into `before_agent_start` and `agent_end` event payloads without emitting undefined properties, and add focused regression tests for both populated and omitted cases. If maintainers prefer the existing context-only contract, close manually with docs or a comment telling plugin authors to read `ctx.sessionKey` and `ctx.agentId`.
