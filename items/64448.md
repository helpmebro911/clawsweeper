## Summary

Keep open. Current `main` still sends Feishu media as separate upload-plus-send calls without any visible idempotency key, request UUID, or in-flight dedupe on the normal image/file outbound path, so the reported duplicate-send bug is not strongly disproven or obviously fixed.
