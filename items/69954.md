## Summary

Keep PR #69954 open. Current main still trusts a structurally valid but stale `sessionFile` path during `sessions cleanup --fix-missing` and prunes the entry when that path is missing, without checking the canonical `<sessionId>.jsonl` fallback described in #50248. The PR is a focused fix for a real remaining cleanup-safety bug, though the reviewed branch should also address the repair-only JSON accounting gap noted in review comments before merge.

## Best Possible Solution

Keep the PR open and revise it into a complete cleanup fix: when the persisted `sessionFile` is missing, check the sessionId-derived canonical transcript before pruning, repair the stored path when that canonical file exists, surface repair-only changes in dry-run and applied JSON (`wouldMutate` included), and add focused regression tests for stale-path repair, true missing transcripts, and normal valid `sessionFile` behavior. Consider aligning doctor’s missing-transcript check with the same fallback because #50248 includes doctor output as part of the unsafe workflow.
