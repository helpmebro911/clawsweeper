## Summary

Keep PR #69954 open. Current main still deletes entries when the stored `sessionFile` path is structurally valid but missing, without checking the canonical `<sessionId>.jsonl` transcript. The PR addresses a real linked bug (#50248), but its review feedback about repair-only `wouldMutate` accounting should be handled before landing.

## Best Possible Solution

Keep #69954 open and land the focused cleanup safety fix after updating `pruneMissingTranscriptEntries` to report both pruned and repaired entries. Tests should cover dry-run keep/repair behavior, applied repair behavior, true missing transcript pruning, and repair-only `wouldMutate` reporting.
