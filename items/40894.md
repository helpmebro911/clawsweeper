## Summary

Keep open. Current main still does not implement the central media-staging request: the 5 MB media store/server/sandbox staging cap is still hardcoded and there is no global `tools.media.maxBytes` config contract. The PR contains useful implementation work for that bug, but it also bundles unrelated assistant-error transcript hygiene and cross-agent workspace changes; the workspace portion is already present on main, while the assistant-error collapse is not. The best maintainer path is to rebase/split the PR and review the media fix for #40880 separately from the transcript hygiene work.

## Best Possible Solution

Keep this PR open as a potentially useful implementation candidate, but require a rebase and split before merge: land the `tools.media.maxBytes` media store/server/sandbox staging fix for #40880 with schema/help/tests, drop the cross-agent workspace slice that current main already covers, and move the assistant-error transcript hygiene into a separate PR tied to its own bug and replay-history tests.
