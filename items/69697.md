## Summary

Keep PR #69697 open. Current main still has the reported undefined-params crash path: `hashToolCall()` hashes `digestStable(params)`, `stableStringify(undefined)` returns the JavaScript value `undefined`, and `createHash().update(serialized)` then throws `ERR_INVALID_ARG_TYPE`. The current tests cover string/number/null params but not undefined, while runtime hook entry points accept `unknown` params and pass them into loop-history hashing.

## Best Possible Solution

Keep the PR open and either merge a small version of its fix or ask the contributor to add a focused regression test in `src/agents/tool-loop-detection.test.ts` for undefined params. The implementation should make the stable serialization contract honest (`string | undefined`) and ensure the value passed to `crypto.update()` is always a string, preserving deterministic hashing for undefined params.

## Evidence
