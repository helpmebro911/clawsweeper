## Summary

Keep open. Current main still does not implement `openclaw doctor --dry-run` or an equivalent doctor-wide no-write validation mode. The existing `--dry-run` support is scoped to other commands such as reset/uninstall/config/update, while doctor can still prompt/apply config repairs and write config during repair paths.

## Best Possible Solution

Keep #13362 open until OpenClaw implements a doctor-specific `--dry-run` mode or maintainers explicitly reject the product direction. The likely fix spans the doctor CLI registration, `DoctorOptions`, config repair/finalization flow, write-config contribution and other doctor repair write paths, exit-code behavior, docs, and focused tests proving recommendations still run while config/state writes do not.
