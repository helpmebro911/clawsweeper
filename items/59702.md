## Summary

Keep open. The report is recent, concrete, and salvageable, with an additional commenter expanding the affected surface beyond plain sessions_spawn. Current main appears to contain the narrower #59555 subagent scope-upgrade fix, but the linked active RFC #69066 treats #59702 as part of a broader unresolved internal service identity/auth coupling that includes subagents, cron/heartbeat, browser/tooling, exec approvals, and Windows token persistence questions.

## Evidence

- **Issue reports a broader internal RPC failure, not only sessions_spawn:** The issue body says sessions_spawn, config.patch/config.get gateway tools, and any internal agent-to-gateway RPC fail with WebSocket close 1008 pairing required on Windows Server 2025. The Apr 15 comment adds heartbeat spawns and custom skill spawns at 100% failure rate, and argues channel-level DM pairing is being applied to gateway-originated internal RPCs without sender identity.
  - command: `provided GitHub context for https://github.com/openclaw/openclaw/issues/59702`
- **Existing fix is narrower than the open report:** PR #59555 fixed the v2026.4.1 sessions_spawn subagent scope-upgrade failure by changing callSubagentGateway in src/agents/subagent-spawn.ts to use operator.admin for admin-only lifecycle calls. The PR discussion explicitly says message-handler.ts, handshake-auth-helpers.ts, call.ts, and method-scopes.ts were not changed, so it does not prove the broader internal service identity/channel-policy problem is fixed.
  - file: [src/agents/subagent-spawn.ts:148](https://github.com/openclaw/openclaw/blob/23d2a8e/src/agents/subagent-spawn.ts#L148)
  - command: `https://github.com/openclaw/openclaw/pull/59555`
  - sha: [23d2a8e](https://github.com/openclaw/openclaw/commit/23d2a8e)
- **Related maintainer discussion remains open:** Issue #69066 is an open RFC, cross-referenced from #59702, that explicitly lists #59702 among March-April 2026 gateway auth regressions and describes the unresolved coupling between user auth, internal service auth, and local operator auth in authorizeGatewayConnect. Its Phase 1 plan still proposes first-class service identity for subagent, browser tool, cron, exec-approvals, and Telegram native approvals, with Windows token-storage questions still open.
  - command: `https://github.com/openclaw/openclaw/issues/69066`
- **Release notes show a partial shipped fix, not a full close condition:** Release v2026.4.2 notes include the #59555 fix: admin-only subagent gateway calls are pinned to operator.admin so sessions_spawn no longer dies on loopback scope-upgrade pairing with close(1008). That supports partial progress, but does not cover the issue's config RPC, heartbeat, custom-skill, and channel-policy claims.
  - command: `https://sourceforge.net/projects/openclaw.mirror/files/v2026.4.2/`
