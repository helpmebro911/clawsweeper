## Summary

Keep open. Current main has general model fallback, but I did not find a high-confidence fix for this specific main-session stability report. Network/connection errors are still classified as timeout-class failures, and timeout handling can still rotate through remaining auth profiles before cross-provider model fallback, which matches the repeated-error/delayed-fallback shape described in #62847.
