## Summary

Current `main` has partial support for ambient GitHub-token auth, but it does not implement the requested non-interactive Copilot onboarding flow. The built-in `github-copilot` provider is still device-login/TTY-only, and non-interactive `openclaw onboard --auth-choice github-copilot` would fail because the provider does not expose `runNonInteractive` or any Copilot-specific onboard flag.

## Evidence

- **Copilot docs still describe the built-in flow as interactive-only:** `docs/providers/github-copilot.md` says `openclaw models auth login-github-copilot` requires an interactive TTY and should not be run in a non-interactive script or CI pipeline.
  - file: [docs/providers/github-copilot.md:77](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/providers/github-copilot.md#L77)
  - command: `nl -ba docs/providers/github-copilot.md | sed -n '1,180p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Provider only registers a device-login auth method:** `extensions/github-copilot/index.ts` registers `auth` with a single `device` method using `run`, and no `runNonInteractive` implementation is present for Copilot onboarding.
  - file: [extensions/github-copilot/index.ts:95](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/github-copilot/index.ts#L95)
