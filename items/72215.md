## Summary

Keep #72215 open. Current main still lacks `cron.enabled` and `agents.defaults.compaction.mode` in the agent-facing gateway config mutation allowlist, so the PR is not already implemented or obsolete. The requested paths are real OpenClaw config surfaces, the PR is small and directly tied to open request #72166, and the existing PR comments are supportive. Because this is an explicit model-to-operator trust-boundary allowlist change, a maintainer should review and decide whether to land this PR or an equivalent patch.

## Best Possible Solution

Have a maintainer review the trust-boundary implications and, if acceptable, land #72215 or an equivalent patch that adds exactly `cron.enabled` and `agents.defaults.compaction.mode` to `ALLOWED_GATEWAY_CONFIG_PATHS` with the proposed positive guard tests. Keep #72166 open as the canonical request until the implementation lands on main.

## Risks / Open Questions

- Allowing `cron.enabled` gives owner-triggered agent automation the ability to pause or resume all scheduled cron jobs, so the trust-boundary decision should be explicit.
- Allowing `agents.defaults.compaction.mode` lets agent automation switch compaction strategy globally; that appears aligned with existing config semantics but still deserves maintainer review because it affects long-session behavior.
