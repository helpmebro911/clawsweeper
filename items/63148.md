## Summary

Keep this PR open. Current main still treats Feishu DM topic messages as flat per-sender sessions, while the PR addresses a real bundled Feishu channel gap linked to #35478. The work is not obsolete or a ClawHub cleanup candidate, but it needs to be rebased or ported onto the current `monitor.message-handler.ts` and `sequential-key.ts` receive path, with the thread_id-only debounce issue addressed before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase or port it onto current main by adding DM topic-scoped `peerId`, `parentPeer`, reply-in-thread handling, and `MessageThreadId` behavior in `extensions/feishu/src/bot.ts`; extending `extensions/feishu/src/sequential-key.ts` and `extensions/feishu/src/monitor.message-handler.ts` to partition DM topics by `root_id` or `thread_id`; preserving main-surface DM and existing group behavior; and adding regression tests for root_id, thread_id-only, dynamic-agent, sequential queue, and debounce cases.
