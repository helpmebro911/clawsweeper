## Summary

Current main still does not implement automatic inline rendering for a bare image URL typed into the Control UI chat composer. The existing implementation supports uploaded/pasted image attachments, structured image blocks, explicit assistant media directives, data-image markdown, and managed assistant image storage, but plain user text such as `https://example.com/image.png` remains text/link content rather than being promoted to an image preview.

## Best Possible Solution

Keep this feature request open for maintainer product/security design in the Control UI chat stack. The likely implementation should add an explicit safe preview pipeline for plain chat text image URLs, probably by routing through same-origin managed media or a guarded proxy, and include tests across composer/send behavior, message normalization, grouped rendering, markdown sanitization, managed image handling, and Control UI CSP.

## Risks / Open Questions

- A direct regex-to-`<img>` change would conflict with the current Control UI CSP and could make operator browsers fetch arbitrary remote URLs from chat content.
- Remote image previews need explicit product/security design around proxying, allowlists, managed blobs, size/type validation, and privacy behavior rather than simple markdown or DOM rendering changes.
