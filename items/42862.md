## Summary

Keep open. Current main supports structured image blocks, `MEDIA:` attachment directives, and markdown-image extraction in specific outbound paths, but it does not auto-promote a plain chat text URL such as `https://example.com/image.png` into an inline image. The current Control UI markdown and CSP behavior also intentionally avoids arbitrary remote image fetches, so this needs explicit product/security design rather than stale cleanup.

## Best Possible Solution

Keep this feature request open for maintainer product/security design. A good implementation would add a deliberate chat-rendering path for safe image previews, with allowlist/CSP behavior and regression tests around `ui/src/ui/markdown.test.ts`, `ui/src/ui/chat/grouped-render.test.ts`, and `src/media/parse.test.ts`, rather than converting arbitrary plain text URLs directly to `<img>` tags.
