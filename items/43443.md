## Summary

Keep this PR open. Current main has changed the main restart path to kickstart-first behavior, so the patch is stale as-is, but the central bootstrap EIO recovery is still not implemented across install, doctor repair, restart fallback, and detached handoff bootstrap paths.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main and rescope it around a shared bounded EIO-only `launchctl bootstrap` retry used by `bootstrapLaunchAgentOrThrow`, doctor repair, restart fallback, and detached handoff bootstrap paths. Separately decide whether PID SIGKILL escalation is still needed under the current kickstart-first restart model; if retained, strengthen PID identity guarding and cover transient EIO retry, errno 125/non-EIO non-retry behavior, repair recovery, install activation, and handoff/fallback paths with focused tests.
