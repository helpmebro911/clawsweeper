## Summary

Keep this PR open. Current main still lacks the PR's central bounded EIO-only `launchctl bootstrap` retry, and the remaining launchd bootstrap surfaces can still fail after a transient `Bootstrap failed: 5: Input/output error`. The PR is stale against current restart behavior and needs rebase/rescope, but it remains a plausible focused bug fix rather than a cleanup-close candidate.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main and rescope it around a shared bounded EIO-only `launchctl bootstrap` retry used by `bootstrapLaunchAgentOrThrow`, `repairLaunchAgentBootstrap`, install activation, restart fallback recovery, and detached handoff bootstrap paths. Separately decide whether PID SIGKILL escalation is still needed under the current restart model; if retained, add stronger process-identity protection and focused tests for retry success/exhaustion, errno 125 and non-EIO non-retry behavior, doctor repair, install activation, restart fallback, and handoff scripts.
