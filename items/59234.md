## Summary

Keep open. Current main appears to avoid the exact `qmd status` vector probe by making QMD vector/status probes local, but the QMD boot/update path still invokes `qmd embed` even when `searchMode` is `search`, so the underlying ARM/CPU-only vector-runtime work is still plausibly unresolved. The issue is recent and has multiple independent confirmations plus a related open issue, so it is not a conservative cleanup close.

## Evidence

- **Current main still runs QMD embed from boot/update path:** `initialize()` starts `runUpdate("boot", true)` when QMD boot updates are enabled. `runUpdate()` runs `qmd update` and then calls `runQmd(["embed"])` when `shouldRunEmbed()` is true. `shouldRunEmbed()` is explicitly documented to keep embeddings current regardless of active retrieval mode, and returns true on first run because `lastEmbedAt === null`.
  - file: [extensions/memory-core/src/memory/qmd-manager.ts](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/extensions/memory-core/src/memory/qmd-manager.ts)
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Current main no longer proves the requested full behavior:** `status()` reports QMD vector availability as true and `probeVectorAvailability()` returns true without a `qmd status` shellout, which may address the exact status-probe symptom. However, this does not satisfy the issue's broader `searchMode: "search" should skip vector work` request because `qmd embed` is still scheduled independently.
  - file: [extensions/memory-core/src/memory/qmd-manager.ts](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/extensions/memory-core/src/memory/qmd-manager.ts)
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Issue discussion remains active and independently reproduced:** The issue has confirmations on ARM Linux, Raspberry Pi 4/5, and WSL, with the latest provided comment on 2026-04-21. Commenters also linked upstream QMD issue tobi/qmd#491, upstream PR tobi/qmd#530, and related OpenClaw issue #67113.
