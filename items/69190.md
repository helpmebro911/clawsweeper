## Summary

Keep open. Current main has real hard tool-policy enforcement, but the report combines two still-not-fully-disposable concerns: disabling `web_fetch` alone does not disable `exec`/`curl`, and related open issue #47487 still covers an exec/profile restriction gap; the browser-timeout half lacks the target URL/logs/config needed to prove that current unreleased browser fallback and timeout work resolves this WSL/Docker scenario.

## Best Possible Solution

Keep #69190 open for maintainer triage. The likely cleanup path is to split guidance: document or respond that strict browser-only runs require hard policy such as allowing only `browser` and denying `exec`/`web_fetch`, track the exec/profile bypass under #47487, and ask for a current-main browser repro with target URL, browser profile/CDP config, WSL reachability checks, logs, and whether the failure is open/navigate/snapshot/act.
