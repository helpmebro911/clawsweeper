## Summary

Keep open. Current main still enables Node's compile cache without OpenClaw install/version scoping in both bootstrap paths, and this PR remains a relevant implementation candidate. It is not ready to close as implemented or obsolete because the maintainer discussion explicitly keeps it useful but asks for refresh, CI, changelog credit, explicit NODE_COMPILE_CACHE semantics, and either a stronger stale-bytecode repro or preventative framing.

## Best Possible Solution

Keep this PR open as the concrete implementation candidate, but require the author or maintainer to refresh it on current main, use a Node-22-compatible `enableCompileCache` call, document the intentional `NODE_COMPILE_CACHE` base-directory reinterpretation, add the requested changelog credit if it ships as user-facing cache hardening, and either prove stale bytecode reuse or describe the change as preventative install/version-scoped cache isolation. After that, review and land or close based on fresh CI and root-cause evidence.
