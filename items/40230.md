## Summary

Keep this PR open. Current main still lacks the requested symlink-following path for fixed workspace bootstrap files, and this PR is a focused implementation candidate for an open user-facing bug with related open reports.

## Best Possible Solution

Keep this PR open for maintainer/security review, or replace it with a maintainer-authored patch in `src/agents/workspace.ts`. The best fix should support symlinked fixed top-level bootstrap files only, preserve boundary-reader safety properties as closely as possible, keep extra/glob bootstrap paths constrained to the workspace, and include regression tests for external symlink success, symlink-to-hardlink rejection, dangling/non-regular/oversized targets, and extra-bootstrap rejection.
