## Summary

Keep this issue open. Current main still does not expose or forward a `memorySearch.local.maxThreads` setting for the built-in local node-llama-cpp embedding provider. The shipped local tuning surface covers `contextSize`, not CPU thread count, and the strict config schema would reject the requested key today.

## Best Possible Solution

Add a validated optional positive-integer `agents.defaults.memorySearch.local.maxThreads` setting, merge it through default/per-agent memory config resolution, carry it through the memory embedding provider option types and local adapter path, update the node-llama-cpp shim to accept it, and pass it to `getLlama` only when configured. Add focused tests for schema acceptance/rejection, default and override merging, runtime forwarding/omission, and update generated config help/schema plus the public memory config docs.
