## Summary

Current main still does not implement `memorySearch.local.maxThreads` for local node-llama-cpp embeddings. The prior Codex review comment correctly identified the gap, and the same config/schema/runtime path remains unchanged at `348728c28c1ea9ae7c5824350baba35ee08a275e`: local config carries `modelPath`, `modelCacheDir`, and `contextSize`, while `getLlama` is still called with only `logLevel`.

## Best Possible Solution

Keep this issue open. The best implementation is a validated positive integer `agents.defaults.memorySearch.local.maxThreads` plus per-agent override, wired through config types, strict Zod schema, schema help/labels/generated schema, resolved memory-search config, memory embedding provider create options, the node-llama wrapper type, and `createLocalEmbeddingProvider`, then passed to `getLlama` only when configured. Add focused tests for default omission, configured forwarding, and config validation, and update the memory config docs.
