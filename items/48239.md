## Summary

Keep open. Current main has meaningful mitigations: Telegram native exec approvals apply agent/session filters, bind origin routing through turn-source/session metadata, and notify the origin chat when DM-only delivery routes elsewhere. But the reported safety concern is not fully closed: `target: "dm"` still delivers by approver Telegram user ID rather than an agent/chat-bound surface, the visible approval prompt does not include the owning agent/session, and typed `/approve` authorization is sender/account based rather than checked against the request's origin session/chat.
