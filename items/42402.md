## Summary

Keep this PR open. Current `main` still has the crash path the PR targets: values from `env.vars` are collected with only a falsy check, then later passed to `containsEnvVarReference`, which assumes a string and calls `.includes()`. The submitted patch is stale against the current file layout, but it remains a focused valid bug-fix candidate rather than obsolete or implemented.

## Best Possible Solution

Keep this PR open as a focused bug-fix candidate. Rebase the one-line non-string guard into `src/config/config-env-vars.ts`, move the regression coverage into `src/config/config.env-vars.test.ts`, and decide whether skipped malformed `env.vars` entries should produce a warning before running the relevant changed/config test lane.
