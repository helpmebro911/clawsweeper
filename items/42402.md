## Summary

Keep this PR open. Current main still lacks the specific `env.vars` runtime guard this PR proposes, and the read path applies `config.env` before schema validation, so truthy non-string values from parsed config can still reach `containsEnvVarReference` and crash on `.includes()`. The PR is narrow and useful, but it needs to be rebased onto the current file layout.

## Best Possible Solution

Keep this PR open as a valid focused bug-fix candidate. Rebase it onto current main by applying the non-string guard in `src/config/config-env-vars.ts`, move the regression test into `src/config/config.env-vars.test.ts`, and consider whether skipped non-string `env.vars` should emit a diagnostic before running the relevant config env-var test lane or changed gate.
