## Summary

Keep #38349 open. Current main still implements group activation as `mention` or `always` only, with no `contextualActivation` config, peeking/engaged state machine, decision-model call path, contextual decision logs, or `openclaw gateway decisions/logs` feature for this purpose. The related implementation PR #38350 is closed unmerged, and #1721 is only an older closed predecessor, so there is no shipped or canonical replacement for the remaining work.

## Best Possible Solution

Keep #38349 open as the canonical product/API issue. The best path is either to implement a first-class contextual activation contract across config schemas, session persistence, Gateway protocol, WhatsApp and Telegram pre-dispatch gating, observability/CLI/docs/tests, or first add a typed plugin/channel activation seam that lets a plugin make the peeking/engaged decision before bundled channel runtimes drop unmentioned group messages. PR #38350 and #1721 remain useful design context, but neither is shipped on current main.
