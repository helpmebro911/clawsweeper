## Summary

Keep PR #41624 open. Current main still has the narrower Dockerfile ownership command that only chowns `/home/node/.cache/ms-playwright`, while the PR proposes chowning `/home/node/.cache` so the runtime `node` user can write the parent cache directory. The related #43846 was closed as a duplicate child and does not supersede this parent PR.

## Best Possible Solution

Keep the PR open as a plausible focused Docker bug fix. The best path is to land the one-line parent-directory ownership change, ideally with a regression assertion in `src/dockerfile.test.ts` that the browser install path chowns `/home/node/.cache`, plus maintainer verification of the `OPENCLAW_INSTALL_BROWSER=1` Docker build path.
