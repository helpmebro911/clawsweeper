## Summary

Keep this PR open. Current main still leaves `/home/node/.cache` owned by root in the optional Docker browser-install path, while this PR’s one-line diff widens ownership to the parent cache directory. The related duplicate was closed unmerged, and current tests/CI do not appear to exercise or assert the `OPENCLAW_INSTALL_BROWSER=1` cache ownership behavior. Security review found the diff is narrowly scoped to Dockerfile ownership and does not add dependency, workflow, secret, package-resolution, or downloaded-code changes, but it is still Docker infrastructure and should get maintainer validation before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best maintainer path is to review the focused Docker ownership change, add or request targeted proof for `OPENCLAW_INSTALL_BROWSER=1` cache ownership under `USER node`, and land it if that validation passes.
