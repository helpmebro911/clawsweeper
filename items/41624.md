Current main still creates `/home/node/.cache/ms-playwright` as root in the optional Docker browser-install path and only changes ownership of that child directory before switching to the non-root `node` user. The PR remains a focused one-line Dockerfile fix for a plausible permissions bug, and the related duplicate attempt #43846 was closed unmerged rather than superseding it.
