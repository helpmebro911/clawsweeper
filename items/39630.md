## Summary

Keep PR #39630 open. Current main still does not expose the requested `tools.web.fetch.allowPrivateNetwork` opt-in for `web_fetch`; it only exposes the narrower `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` path. The linked feature issue #39604 remains open, and the PR discussion still has unresolved security/product policy questions around metadata endpoints and configurable exceptions, so this PR is not obsolete or safely closeable.

## Best Possible Solution

Keep the PR open for maintainer direction or revision. A strong path would add a documented `web_fetch` private-network opt-in aligned with the current `ssrfPolicy` config surface, thread it through strict schema, generated config metadata, public types, runtime, cache partitioning, tests, docs, and changelog, and explicitly resolve cloud metadata endpoint policy before merge. Continue treating #39604 as the canonical feature request until the product decision is made.
