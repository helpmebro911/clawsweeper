## Summary

Keep this issue open. Current main still has the reported DAVE-disabled Discord voice pre-Ready failure surface: `/vc join` waits for `@discordjs/voice` to reach `Ready`, destroys the connection on the fixed 15s wait failure, and only registers a session after that point. Existing recovery is post-Ready receive recovery, not a fix for a voice WebSocket close after Hello before UDP handshaking.

## Best Possible Solution

Keep this issue open until the intended non-DAVE Discord voice behavior is decided. The likely implementation belongs in `extensions/discord/src/voice/manager.ts`: add explicit pre-Ready state/close diagnostics and regression coverage around `entersState` failure before `sessions.set`, then either implement bounded retry or DAVE-enabled fallback, or reject/deprecate `channels.discord.voice.daveEncryption=false` with clear operator guidance if Discord now requires DAVE. If #63098 becomes canonical, preserve this issue's trace there before closing it as duplicate.
