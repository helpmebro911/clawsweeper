## Summary

Keep this issue open. Current main still does not implement a `brave.http` diagnostics flag or Brave-specific request/status/timing/cache hit/miss logging, and the request remains actionable inside the bundled Brave plugin using existing diagnostic SDK and web-search provider context surfaces. The prior ClawSweeper comment remains directionally correct on current main at c2d82b87eee7a86bbd0e851169e0ca5559c44fe6.

## Best Possible Solution

Keep this issue open. Implement the feature in `extensions/brave` by passing the OpenClaw config or a precomputed diagnostics-enabled flag into the Brave runtime, gating logs with `isDiagnosticFlagEnabled("brave.http", config)`, logging sanitized outbound URL/query/params plus response status and duration, and logging cache hit/miss around the existing cache read/write paths. Add focused Brave provider tests and document the new flag in `docs/diagnostics/flags.md` and `docs/tools/brave-search.md`.
