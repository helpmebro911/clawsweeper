## Summary

Keep this issue open. Current main still does not implement a `brave.http` diagnostics flag or Brave-specific request/status/timing/cache logging, while the existing plugin SDK and Telegram implementation show this is actionable inside the bundled Brave plugin without a new core API.

## Best Possible Solution

Keep this issue open. Implement the feature in `extensions/brave` by passing full config or a precomputed diagnostics-enabled state into the Brave search runtime, gating logs on `brave.http` via `openclaw/plugin-sdk/diagnostic-runtime`, logging sanitized outbound URL/query/params plus response status and duration, and logging cache hit/miss around the existing cache path. Add focused Brave provider tests and document the flag in `docs/diagnostics/flags.md` and `docs/tools/brave-search.md`. No new core API appears necessary.
