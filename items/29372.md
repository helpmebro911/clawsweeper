## Summary

Keep #29372 open. Current main still does not implement `agents.list[].subagents.workspace`: the per-agent config type, strict runtime schema, generated schema, docs, and same-agent/default spawn path still only support existing subagent controls. The related #40171/#40176 work fixed cross-agent target workspace inheritance, but it does not provide the requested configurable workspace override for omitted-agentId/default child subagents.

## Best Possible Solution

Keep #29372 open for a core config/runtime implementation if maintainers want this isolation behavior. A complete fix would add and document `agents.list[].subagents.workspace`, update TypeScript types, zod/runtime/generated schemas, path normalization/help docs, and spawn workspace resolution, with tests for omitted `agentId`, explicit same-agent spawns, `requireAgentId`, cross-agent preservation, and bootstrap isolation.
