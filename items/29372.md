## Summary

Keep #29372 open. Current main still does not implement `agents.list[].subagents.workspace`; the config type, strict runtime schema, generated schema, docs, and same-agent/default subagent spawn path all preserve the caller/requester workspace rather than resolving a configured child workspace override. The related #40171 cross-agent inheritance fix is real, but it only covers explicit `agentId` cross-agent spawns and the current tests explicitly preserve same-agent workspace inheritance.

## Best Possible Solution

Keep #29372 open for a core config/runtime implementation if maintainers want this isolation behavior. A complete fix should add and document `agents.list[].subagents.workspace`, update TypeScript types, zod/runtime/generated schemas, path normalization/help docs, and spawn workspace resolution, with focused tests for omitted `agentId`, explicit same-agent spawns, `requireAgentId`, cross-agent preservation, and bootstrap isolation.
