## Summary

Keep open. Current `main` fixes related cross-agent workspace inheritance, but it does not implement the requested `agents.list[].subagents.workspace` override for default same-agent subagents. The current behavior still preserves the caller/inherited workspace for same-agent spawns, which is the gap this issue asks to solve.

## Evidence

- **Per-agent subagents config has no workspace field:** `AgentConfig.subagents` only defines `allowAgents`, `model`, and `requireAgentId`; there is no `workspace` override under `agents.list[].subagents`.
  - file: [src/config/types.agents.ts:102](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/config/types.agents.ts#L102)
  - command: `nl -ba src/config/types.agents.ts | sed -n '90,120p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Spawn workspace resolution has no per-agent subagents.workspace hook:** `resolveSpawnedWorkspaceInheritance()` accepts only `targetAgentId`, `requesterSessionKey`, and an explicit runtime override. Without an explicit override, it resolves to the target agent workspace or requester session agent workspace.
  - file: [src/agents/spawned-context.ts:53](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/spawned-context.ts#L53)
  - command: `nl -ba src/agents/spawned-context.ts | sed -n '53,69p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Same-agent subagents still inherit requester workspace:** The current test suite explicitly asserts that same-agent spawns preserve the inherited requester workspace rather than using a separate configured child workspace.
  - file: [src/agents/subagent-spawn.workspace.test.ts:158](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/subagent-spawn.workspace.test.ts#L158)
  - command: `nl -ba src/agents/subagent-spawn.workspace.test.ts | sed -n '118,165p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Docs only describe cross-agent target-workspace inheritance:** Current docs say spawning to a different ACP agent without `--cwd` inherits the target agent workspace. They do not document a per-agent `subagents.workspace` default for same-agent child spawns.
  - file: [docs/tools/acp-agents.md:117](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/tools/acp-agents.md#L117)

## Risks / Open Questions

- A recent April 23, 2026 change on `main` addressed the related cross-agent workspace bug, so future triage should avoid treating that adjacent fix as closing this narrower same-agent/default-subagent request.
- Operators can partly work around the issue today with `requireAgentId` plus dedicated target agent profiles, but that still conflicts with the issue's stated goal of avoiding separate profiles for default child agents.
