## Summary

Current main still has the reported Volcengine/BytePlus catalog mismatch. The shared Kimi K2.5 and Kimi coding rows remain `reasoning: false` with `maxTokens: 4096`, and those rows are still consumed by both bundled provider plugins, while the Together catalog continues to expose Kimi K2.5 as reasoning-capable with a 32K output limit.

## Best Possible Solution

Keep this issue open. The best fix belongs in the bundled Volcengine/BytePlus provider-owned catalog path, primarily `src/plugin-sdk/volc-model-catalog-shared.ts`, with provider docs updated if output limits or reasoning flags change. Maintainers should verify current Volcengine and BytePlus ModelArk limits and thinking controls, then add focused tests that cover Kimi K2.5 and Kimi K2 Thinking `reasoning` and `maxTokens` propagation for both provider plugins.
