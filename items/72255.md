## Summary

Keep open. Current main still treats the per-agent auth profile store as a runtime credential source even when a matching profile declaration is absent from openclaw.json, so the reported Pi orphan-credential path is not clearly fixed or superseded.

## Best Possible Solution

Treat auth.profiles declarations as authoritative for runtime selection, or add an explicit declared-profile allowlist to Pi/model auth resolution. At minimum, add doctor/secrets audit coverage that identifies stored auth-profiles.json entries with no matching openclaw.json declaration and emits a visible runtime warning if such a credential is selected.
