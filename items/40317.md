## Summary

Keep open. Current `main` does not implement the proposed configurable strict/tolerant config-validation mode. Reads still fail closed on validation errors, writes still reject invalid keys outright, the top-level config schema remains strict, and there is no `validation.writeMode` / `validation.readMode` / `unknownKeyLogLevel` surface in code or docs.

## Evidence

- **Gateway startup still hard-fails invalid config:** `loadConfig()` validates with `validateConfigObjectWithPlugins(...)`; on failure it records an invalid snapshot and immediately calls `throwInvalidConfig(...)` instead of stripping unknown keys or continuing in tolerant mode.
  - file: [src/config/io.ts:1256](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/config/io.ts#L1256)

  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Write path is strict-only:** `writeConfigFile()` always runs `validateConfigObjectRawWithPlugins(...)` before persist and throws on the first validation issue. There is no mode switch for tolerant writes.
  - file: [src/config/io.ts:1643](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/config/io.ts#L1643)
