## Summary

Keep #69054 open. Current main already treats `delivery.accountId` as a first-class routing field, but it does not implement the requested `openclaw cron lint` command, startup warning/fail path, or add/update validation requiring `delivery.accountId` for announce-mode cron jobs targeting multi-account channels. The linked #69055 is a separate open forensic logging request, not a duplicate or shipped replacement.

## Best Possible Solution

Implement this in core cron rather than closing it: add a reusable cron lint helper that inspects stored jobs plus resolved channel account IDs, expose `openclaw cron lint [--warn-only]`, document the migration playbook, and wire the same rule into gateway startup plus cron add/update validation after the chosen deprecation window. Tests should cover single-account no-op, multi-account findings, CLI exit codes, and gateway add/update rejection.
