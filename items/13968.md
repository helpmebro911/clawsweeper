## Summary

Current main still does not implement strict on-demand HEARTBEAT.md loading. The loader always includes and reads HEARTBEAT.md before run-kind filtering/cache effects, and current normal-run behavior still injects HEARTBEAT.md when heartbeat guidance is enabled for the default agent. The related #55311 path fixed heartbeat lightContext forwarding, but it does not satisfy this issue's requested default non-heartbeat omission before disk reads/cache population.

## Best Possible Solution

Keep #13968 open. Implement a run-kind-aware bootstrap loading/cache path that excludes HEARTBEAT.md for default non-heartbeat runs before disk reads and session snapshot cache population, preserves HEARTBEAT.md for actual heartbeat runs and lightweight heartbeat context, adds focused tests for enabled default normal-run omission plus disabled/lightweight regressions, and updates docs/gateway/heartbeat.md to describe the new contract.
