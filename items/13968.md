## Summary

Keep #13968 open. Current main still does not implement strict on-demand HEARTBEAT.md loading: default non-heartbeat runs can still load and inject HEARTBEAT.md when heartbeat guidance is enabled, while the related #55311 lightContext forwarding fix is separate.

## Best Possible Solution

Implement an explicit run-kind/isHeartbeat-aware bootstrap loading path so default non-heartbeat runs omit HEARTBEAT.md before disk read, cache, hook processing, and prompt injection, while actual heartbeat runs still retain HEARTBEAT.md and lightweight heartbeat mode remains heartbeat-only. Add focused tests for default omission, heartbeat retention, disabled cadence/prompt-section behavior, and update token-use/system-prompt docs if behavior changes.
