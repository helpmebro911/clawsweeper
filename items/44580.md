## Summary

Keep open. Current `main` still does not implement the wrapped-Mistral part of the report: replay policy resolution uses the owning provider plugin only, `openrouter` only declares Gemini passthrough replay hooks, and Mistral's `strict9` replay policy remains scoped to the `mistral` plugin itself. That means an `openrouter` model id like `mistralai/...` still does not obviously resolve `strict9` transcript sanitization on current `main`.

## Evidence

- **Replay policy is owner-only:** `resolveTranscriptPolicy` reads `runtimePlugin?.buildReplayPolicy` and returns that policy directly when present, so an OpenRouter-owned model does not pick up another provider's replay policy.
  - file: [src/agents/transcript-policy.ts:197](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/transcript-policy.ts#L197)

  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **OpenRouter only wires Gemini passthrough replay hooks:** The OpenRouter plugin spreads `PASSTHROUGH_GEMINI_REPLAY_HOOKS` and has no Mistral-specific replay-policy branch or `strict9` handling.
  - file: [extensions/openrouter/index.ts:140](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/openrouter/index.ts#L140)
