## Summary

Keep #71469 open. Current main still opens the Gemini CLI OAuth URL before printing any fallback URL, so a silently successful-but-ineffective browser opener can leave the user waiting on the localhost callback with no visible auth URL. The PR is a focused fix for the linked #70964 report, and the provided maintainer discussion explicitly says it is valid on current main and worth landing.

## Best Possible Solution

Keep the PR open and land this scoped Google plugin OAuth fix, or apply an equivalent change that logs the Gemini authorization URL before the best-effort browser opener and keeps the focused local-flow regression test.
