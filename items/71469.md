## Summary

Keep open. Current `main` still has the exact silent-browser-launch failure mode described in #70964, and this PR is a minimal, targeted fix with new regression coverage rather than a stale or superseded item.

## Evidence

- **Current main still only prints the OAuth URL on thrown launch errors:** `loginGeminiCliOAuth` updates progress, then calls `ctx.openUrl(authUrl)` inside a `try/catch`, and only logs the URL inside the `catch`. If browser launch resolves without actually opening a browser, users still get no visible URL and the local callback path waits indefinitely, which matches the report.
  - file: [extensions/google/oauth.ts:43](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/extensions/google/oauth.ts#L43)
