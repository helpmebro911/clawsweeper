## Summary

Keep this PR open. Current main still routes bare numeric Discord allowlist entries through channel lookup before any guild lookup, and the current resolver test explicitly records that a matching guild ID like `999` remains unresolved unless it is written as `guild:999`. The PR addresses a real, still-present bug, but likely needs to be rebased or ported to the current `extensions/discord` path.

## Best Possible Solution

Keep the PR or equivalent work open, port the guild-first bare-numeric behavior into `extensions/discord/src/resolve-channels.ts`, preserve explicit `channel:` and `guild:` semantics plus channel fallback, and update `extensions/discord/src/resolve-channels.test.ts` to cover both guild-first resolution and non-guild numeric channel fallback before merging.
