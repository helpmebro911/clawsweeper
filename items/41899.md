## Summary

Keep open. Current main has plugin hook error handling, plugin load/status reporting, manual plugin enable/disable, gateway diagnostics, and a few subsystem-specific circuit breakers, but it does not implement the requested framework-level plugin-scoped circuit breaker with rolling failure/timeout/lock-overrun accounting, persisted breaker state, hook skipping, half-open recovery, forced-open quarantine, status visibility, or reset commands. This is core plugin/gateway runtime stability work, not a ClawHub-only plugin idea.

## Best Possible Solution

Keep this issue open and treat it as core plugin/gateway stability work. The likely path is an additive plugin-runtime breaker layer around hook/service/tool execution, with structured telemetry for plugin id + hook + session, persisted breaker state, status/diagnostics output, operator reset/quarantine commands, and focused tests proving one failing plugin is skipped without degrading unrelated plugins.
