## Summary

Current main still matches the reported failure mode. `resolveSingleAccountKeysToMove()` returns all movable static keys when named accounts exist and no `namedAccountPromotionKeys` surface is present, and the only bundled channels currently declaring that surface are Telegram and Matrix. Slack still relies on top-level `allowFrom` inheritance for named accounts, so moving that key into `accounts.default` would still change behavior for named accounts.
