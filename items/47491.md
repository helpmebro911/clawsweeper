## Summary

Keep PR #47491 open. Current main already covers the reset-hook and direct compaction-hook agent-id fixes, but the embedded subscribe compaction path still invokes before_compaction/after_compaction with only sessionKey, even though the embedded context and plugin hook context types already carry agentId and sessionId. The PR remains a concrete, narrow core plugin-hook fix rather than cleanup that can be closed.

## Best Possible Solution

Keep the PR open and refresh it against current main as a narrow fix for the remaining embedded compaction hook context gap. The best implementation is to forward ctx.params.agentId and ctx.params.sessionId alongside sessionKey in src/agents/pi-embedded-subscribe.handlers.compaction.ts and add/update regression coverage in src/plugins/wired-hooks-compaction.test.ts or the embedded compaction handler tests. The reset/direct compaction pieces should not be reworked unless needed for consistency because current main already covers them.
