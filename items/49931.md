## Summary

Keep open. Current main still does not implement a `tools.exec.shell` override: the config type/schema expose exec host/security/ask/node/path/safe-bin settings but no shell field, gateway exec still calls `getShellConfig()` directly, and Windows shell selection remains hard-coded to PowerShell. The node-host path also remains hard-coded to `cmd.exe` on Windows and `/bin/sh -lc` elsewhere. This is a core built-in exec behavior request with approval/safe-bin/security implications, not a clean ClawHub/plugin close.
