## Summary

Current `main` at `e672b61417af5c45b0431df6d9109a1f4b618ef5` still does not support WhatsApp Newsletter/Channel `@newsletter` targets in the message tool. The WhatsApp plugin normalizes only E.164/user JIDs and group JIDs, rejects other `@` targets before send, documents WhatsApp targets as E.164/group-only, and has no newsletter route/chat type model. The related PR #13424 is closed unmerged, and #62697 tracks inbound newsletter processing rather than this outbound send request.

## Best Possible Solution

Keep #13417 open as the outbound WhatsApp Newsletter/Channel message-tool tracker for the bundled WhatsApp plugin. A good fix should add a newsletter JID recognizer/normalizer, define newsletter target policy semantics, represent newsletter/channel routing separately from direct/group chats where needed, route sends through the active WhatsApp listener, add focused normalization/resolver/session/send tests, and update the message plus WhatsApp docs. Treat #62697's inbound newsletter work and optional metadata/follow/react operations as related but separate unless maintainers choose a broader WhatsApp Channels project.
