## Summary

Current main still does not implement outbound WhatsApp Newsletter/Channel targets for the message tool. The existing bundled WhatsApp plugin only accepts direct E.164/user JIDs and group JIDs, rejects other `@` forms such as `@newsletter`, documents WhatsApp targets as E.164/group-only, and has no newsletter/channel routing model. The existing maintainer Codex review also explicitly kept this item open, and the related PR #13424 was closed unmerged while #62697 tracks inbound newsletter handling rather than this outbound send case.

## Best Possible Solution

Keep #13417 open as the outbound WhatsApp Newsletter/Channel message-tool tracker for the bundled WhatsApp plugin. A good fix would add a newsletter JID recognizer/normalizer, define default-target and allowlist semantics for newsletter channels, model newsletter/channel routing separately from direct/group sessions where needed, route sends through the active listener safely, add focused normalization/resolver/session/send tests, and update the message plus WhatsApp docs. Treat #62697's inbound newsletter handling and optional metadata/follow/react operations as separate follow-up work unless maintainers choose one broader WhatsApp Channels project.
