## Summary

Keep open. Current main still does not expose the requested lightweight REST endpoints for `sessions.list`, `agents.list`, combined status, or `sessions.get`. The gateway HTTP router only wires existing built-in HTTP surfaces such as `/v1/models`, `/v1/embeddings`, `/v1/chat/completions`, `/v1/responses`, `/tools/invoke`, `/sessions/:key/kill`, `/sessions/:key/history`, plugin routes, Control UI, and health/readiness probes. The related PR #47863 directly attempted this feature but is closed and unmerged, so it does not supersede the issue.
