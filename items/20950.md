## Summary

Keep #20950 open. Current main still supports a single cron definitions store file (`jobs.json`) plus the derived `jobs-state.json` runtime sidecar, but not a `jobs.d`/directory-backed one-file-per-job store. The related PR #41206 is an open, unmerged implementation candidate; #70531 was closed into #20950, and #53581 only handled the complementary runtime-state split.

## Best Possible Solution

Keep #20950 open as the canonical product issue until maintainers land #41206 or a replacement. The eventual implementation should add an additive, documented directory-backed cron definitions store with deterministic per-file loading, compatibility with existing `cron.store`/`jobs.json`, preservation of the current `jobs-state.json` runtime sidecar behavior, config schema/help/docs/doctor updates, and focused load/save/gateway tests.
