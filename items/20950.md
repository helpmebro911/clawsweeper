## Summary

Current `main` partially addresses the GitOps pain behind #20950 by moving mutable cron runtime state into `jobs-state.json`, but it does not implement the requested `jobs.d/` or per-job file loading. The code, config schema, and docs still support a single `cron.store` file for job definitions, so there is not strong evidence to close this as implemented.

## Evidence

- **Cron store still uses a single definitions file plus derived sidecar state file:** `resolveDefaultCronStorePath()` returns `jobs.json`, `resolveStatePath()` derives `-state.json`, and `resolveCronStorePath()` accepts only one optional store path string. `loadCronStore()` / `saveCronStore()` read and write that single store file plus the sidecar state file; there is no directory scan or per-file job loader.
  - file: [src/cron/store.ts:30](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/cron/store.ts#L30)
