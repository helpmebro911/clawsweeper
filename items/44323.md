## Summary

Current `main` has better `doctor` messaging for memory-search readiness, but it still does not perform the config-load validation this PR is trying to add. The PR discussion is still actionable, and the branch also carries unrelated cron/agent-lifecycle changes, so this is not a conservative auto-close candidate.
