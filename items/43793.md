## Summary

Keep open. Current main still lacks the PR's central TUI reconnect UX: generic gateway disconnects still resolve to idle, there is no Ctrl+R editor hook, and neither the TUI backend nor GatewayChatClient exposes a manual reconnect method. The PR discussion also records useful follow-up fixes on the branch, including auth/pairing hint suppression and a macOS remote-probe side-effect fix, so the work is not obsolete even though it should be rebased and narrowed.

## Best Possible Solution

Keep this PR open for maintainer review, but ask the contributor to rebase and narrow it. The best path is a focused TUI/gateway change that exposes a safe manual reconnect API, wires a visible retry shortcut or command, shows reconnect/restart guidance only when auto-reconnect is actually active, preserves auth/pairing-specific guidance, and includes regression tests. The macOS remote-probe coordinator fix should either be split or reviewed as a separate surface.
