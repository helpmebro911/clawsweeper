## Summary

Keep open. Current main still contains the reported compaction-end path that zeroes every assistant message usage snapshot, without the timestamp/index guard described in the issue and linked PR #50845. The safer timestamp-gated implementation exists in replay-history sanitization, but it has not been applied to the live compaction handler.
