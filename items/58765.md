## Summary

Keep open. Current `main` still does not apply `memorySearch.outputDimensionality` to the local GGUF embedding path, and the local provider cache key does not include a dimension override for reindex detection. The report is recent and specific, so this is not a stale/insufficient-info close.

## Evidence

- **Local GGUF provider ignores dimension overrides:** `createLocalEmbeddingProvider` reads `local.modelPath`, `local.modelCacheDir`, and `local.contextSize`, then returns the full `embedding.vector` after normalization. There is no use of `options.outputDimensionality` and no truncation step before returning embeddings.
  - file: [src/memory-host-sdk/host/embeddings.ts:23](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/memory-host-sdk/host/embeddings.ts#L23)
  - command: `nl -ba src/memory-host-sdk/host/embeddings.ts | sed -n '1,120p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Local provider reindex key omits dimensions:** The builtin local memory adapter only stores `{ provider: "local", model: provider.model }` in `cacheKeyData`. Unlike dimension-aware providers, a local `outputDimensionality` change would not change the provider cache key used for full reindex detection.
  - file: [extensions/memory-core/src/memory/provider-adapters.ts:83](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/memory-core/src/memory/provider-adapters.ts#L83)
  - command: `nl -ba extensions/memory-core/src/memory/provider-adapters.ts | sed -n '70,110p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Dimension-aware providers already wire this through:** Gemini includes `outputDimensionality` in its runtime cache key, and Bedrock includes `dimensions` in its runtime cache key. That shows current `main` supports provider-specific dimension overrides for some providers, but not for `local`.
  - file: [extensions/google/memory-embedding-adapter.ts:42](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/google/memory-embedding-adapter.ts#L42)
  - command: `nl -ba extensions/google/memory-embedding-adapter.ts | sed -n '34,70p'; nl -ba extensions/amazon-bedrock/memory-embedding-adapter.ts | sed -n '1,80p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Full reindex logic depends on provider cache key changes:** The memory reindex tests explicitly assert that a provider cache-key change forces a full reindex. Because the local adapter does not include dimensions in its cache key, local dimensional truncation is not implemented with the requested reindex behavior.
  - file: [extensions/memory-core/src/memory/manager-reindex-state.test.ts:62](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/memory-core/src/memory/manager-reindex-state.test.ts#L62)
  - command: `nl -ba extensions/memory-core/src/memory/manager-reindex-state.test.ts | sed -n '48,72p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Docs still scope output dimensionality away from local embeddings:** The memory config docs document `outputDimensionality` under Gemini-specific and Bedrock sections, while the local embedding section only documents `local.modelPath`, `local.modelCacheDir`, and `local.contextSize`. Schema help likewise describes `outputDimensionality` as provider-specific for Gemini and Bedrock families.
  - file: [docs/reference/memory-config.md:108](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/reference/memory-config.md#L108)
