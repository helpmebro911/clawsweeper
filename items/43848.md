## Summary

Current main only appears to have fixed the Telegram forum-topic threadId routing part. The ANNOUNCE_SKIP visible-delivery fallback and requester transcript pollution/ping-pong behavior described in this issue still remain in the sessions_send A2A path.

## Best Possible Solution

Keep this issue open. A maintainer fix should preserve the current threadId routing, add a visible-delivery fallback for Telegram topic handoffs where the announce step returns ANNOUNCE_SKIP after a real target reply, and suppress or redesign same-agent Telegram topic-to-topic ping-pong before it records fresh inter_session turns in the requester session. Closed PRs #43869 and #44064 are useful prior-art references.
