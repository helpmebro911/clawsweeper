## Summary

Current main only fixes the Telegram forum-topic threadId part. The ANNOUNCE_SKIP visible-delivery fallback and requester transcript pollution paths described by this issue are still present, and the related fix PRs #43869 and #44064 are closed unmerged prior art.

## Best Possible Solution

Keep this issue open and use #43869 and #44064 as prior-art references. A maintainer fix should preserve the current threadId routing, add regression coverage for Telegram topic handoffs where announce returns ANNOUNCE_SKIP, deliver the target reply when visible delivery would otherwise disappear, and suppress or redesign same-agent Telegram topic-to-topic ping-pong before it writes fresh inter_session turns into the requester session.
