## Summary

Current `main` at `c6684af682bcb59bd1ee85cb49f72d4074007e2c` does not contain the trigger-based voice wake routing feature from PR #30354. The gateway still exposes only `voicewake.get`/`voicewake.set`, the agent protocol still lacks `voiceWakeTrigger`, macOS forwarding still sends no trigger metadata, and docs still describe only the global trigger list. This PR is not implemented on `main`, so it should stay open.
