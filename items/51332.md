## Summary

Keep open. Current `main` adds strong Ollama docs about `/v1` OpenAI-compatible mode causing raw tool JSON/plain-text pseudo-calls, but I did not find a normal runtime diagnostic that explicitly flags "tool-call-shaped text with no executed tool call" during agent runs. The issue looks only partially addressed through docs and setup guidance, not closed out in product behavior.

## Evidence

- **Docs warn about the exact Ollama failure mode:** The Ollama provider docs now explicitly warn that using `http://host:11434/v1` breaks tool calling and can make models emit raw tool JSON as plain text, and they steer users to native `/api/chat` instead.
  - file: [docs/providers/ollama.md:13](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/providers/ollama.md#L13)

- **Docs also call OpenAI-compatible Ollama tool calling unreliable:** The same doc repeats that OpenAI-compatible mode is not reliable for tool calling and that native `/api/chat` is the supported path for streaming plus tool use.
  - file: [docs/providers/ollama.md:298](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/providers/ollama.md#L298)
