## Summary

Keep #71486 open. Current main still matches the reported LM Studio @iq* truncation bug: the shared auth-profile parser only protects @q* and bit-depth quant suffixes, and provider ownership/model matching still strips from the first @. The PR addresses both paths, and the maintainer discussion explicitly says this is a valid bug and the best base, with requested broader regression coverage before merge. Related PRs #71488 and #71490 are closed unmerged, so they do not supersede this open patch.
