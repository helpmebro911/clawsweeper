## Summary

Current `main` does show a `💵 Cost:` line in `/status`, but that line is an estimated cost for the latest reply, not a cumulative session-cost total. The requested behavior in #64334 is still missing; current OpenClaw exposes cumulative session spend through `/usage cost` instead.

## Evidence

- **`/status` cost is computed from current status token fields, not transcript/session totals:** `buildStatusMessage()` decides whether to show cost, estimates it from `inputTokens` and `outputTokens`, and renders `💵 Cost:` inline with the token line. There is no transcript walk or session-total aggregation in this path.
  - file: [src/status/status-message.ts:783](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/status/status-message.ts#L783)

  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Docs define `/status` as last-reply cost, not session cost:** User-facing docs say `/status` shows the session model, context usage, and last response tokens, and only adds estimated cost for the last reply when using API-key auth.
  - file: [docs/reference/api-usage-costs.md:17](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/docs/reference/api-usage-costs.md#L17)
