## Summary

Keep this issue open. Current main still renders only a token-count-derived estimated `Cost:` line in `/status` and `session_status`; it does not render a cumulative transcript-backed `Session cost` line. An open, unmerged PR (#72517) references this issue with closing syntax, so the issue should remain open until that PR is reviewed, landed, or closed.

## Best Possible Solution

Keep the issue open. Review PR #72517 as the active closing candidate; if it remains README-only, request code changes or close it. The product fix should add a cumulative transcript-backed session cost line in the shared `/status` and `session_status` renderer, reuse existing transcript/session cost aggregation, preserve current cost visibility rules, and add focused tests for multi-turn totals and OAuth/token hiding.
