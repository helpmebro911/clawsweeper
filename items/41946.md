## Summary

Keep open. Current main still does not address the Feishu underscore-in-URL rendering bug, and this PR is a relevant but incomplete implementation candidate. The submitted patch is not mergeable as-is: it imports a non-exported helper, only handles markdown-link URLs rather than the bare URL shown in the linked repro, and its regex can corrupt links containing parentheses. Security review found no CI, dependency, lockfile, secret, publishing, or supply-chain surface in the diff; the blocking concerns are functional and type-safety issues.

## Best Possible Solution

Keep this PR open for rework or replacement. The fix should stay in the Feishu plugin send/rendering path, handle both bare URLs and markdown-link URLs without corrupting valid parentheses or escaped characters, add focused regression tests for send/edit payloads and parsed post links, and then close the linked Feishu bug when the corrected implementation lands.

## Risks / Open Questions

- Merging the PR as submitted would introduce a missing named export from `extensions/feishu/src/post.ts` to `extensions/feishu/src/send.ts`.
- The proposed regex stops URL capture at the first `)`, which can regress valid markdown links containing parentheses.
- The linked bug repro uses a bare URL; the PR helper only rewrites `[label](url)` markdown links and would not fix that exact path.
- No regression test is included for Feishu send/edit payloads containing underscore-heavy URLs.
