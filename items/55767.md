## Summary

Keep open. Current main at 62869c85025b4a77b3a06af4f72a9dff5caf84e3 still accepts hook request, default, and mapping session keys after only request-policy and allowlist-prefix checks; it does not reject reserved internal `subagent:`, `acp:`, or `cron:` namespaces. The PR remains a focused gateway security fix, and the supplied review thread shows the author addressed both the static mapping fail-fast gap and the whitespace unwrap bypass.

## Best Possible Solution

Keep PR #55767 open for maintainer security review and either merge/adapt its reserved namespace guard or land an equivalent fix on main. The best implementation should validate `hooks.defaultSessionKey`, static mapping `sessionKey`, request keys, and templated mapping results against recursively unwrapped, trimmed `agent:<id>:` session segments, with regression coverage for request, mapping, default config, nested agent wrappers, and whitespace bypass cases.
