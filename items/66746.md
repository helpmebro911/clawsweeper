## Summary

Keep this issue open. Current main still has the same unresolved Windows sqlite-vec loader/readiness gap: extension loading is enabled and there is an in-process cosine fallback, but the bundled sqlite-vec loader still has no Windows .dll suffix retry and no post-load vec0 usability probe before marking vectors available. The related Windows tracking issue and proposed PR are still open in the provided context, so this is not implemented or safely superseded on main.

## Best Possible Solution

Keep this issue open unless maintainers explicitly consolidate it into #65704 or a merged successor to #69059. The likely fix belongs in both sqlite-vec loader copies plus memory-core readiness: validate usable vec0 support after load, decide and test the Windows .dll suffix retry, preserve the in-process cosine fallback, and add focused Windows or mocked regression coverage for failed-load and load-success-without-usable-vec0 cases.
