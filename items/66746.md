## Summary

Keep this issue open. Current main mitigates degraded vector recall with stored embeddings and in-process cosine fallback, but I did not find evidence that the reported Windows node:sqlite/sqlite-vec loader failure and missing chunks_vec creation are fixed.

## Best Possible Solution

Keep this open unless maintainers explicitly consolidate it into #65704 or #69059. The best fix belongs in src/memory-host-sdk/host/sqlite-vec.ts and packages/memory-host-sdk/src/host/sqlite-vec.ts, with memory-core readiness updated to validate usable vec0 support after load, decide whether to land the Windows .dll suffix retry, preserve the in-process cosine fallback, and add focused Windows or mocked regression coverage for suffix retry and load-success/no-usable-vec0 cases.
