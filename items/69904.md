## Summary

Keep PR #69904 open. Current main still lacks the proposed stale transient async-notice filtering, so it is not already implemented or superseded. The PR may be useful, and the Greptile review supported the basic stalePolicy approach, but the later review concern remains material: the proposed exec-event short-circuit can consume valid non-completion exec notices such as Exec started and Exec denied.

## Best Possible Solution

Keep the PR open for maintainer/author follow-up. The best path is to preserve the useful stalePolicy/drop-on-session-advance idea, but adjust the heartbeat exec-event handling so it only skips and consumes events that were intentionally stale-filtered, while preserving non-completion exec lifecycle notices and mixed cron queues. Add regression coverage for Exec started/Exec denied on exec-event wakes plus stale completion and mixed cron cases before merging.
