## Summary

Keep PR #69904 open. Current main does not implement the proposed stale transient system-event policy, and the PR targets a real remaining gap in async/heartbeat notices. The PR also is not safe to close as obsolete or implemented: the provided review discussion flags a remaining merge-risk around exec-event wakes that can still enqueue non-completion notices, so the useful path is to revise the PR rather than close it.

## Best Possible Solution

Keep the PR open and revise it against current main: add a narrow stale-policy field for transient success/completion notices, discard only those entries when session `updatedAt` is newer, preserve durable blocked/error/timeout notices, and update heartbeat exec-event handling so filtering stale completions never consumes cron reminders or non-completion exec notices. The regression suite should cover stale completion drops, durable failure preservation, mixed stale-completion-plus-cron queues, and exec started/denied queues.
