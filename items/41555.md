## Summary

Keep open. Current main improves nearby thinking-control behavior, but it does not prove the reported recovery problem is fixed: `/new` intentionally preserves user-selected model and thinking overrides, and Claude 4.6 still defaults to adaptive thinking. The Kimi follow-up has a partial current mitigation because Kimi thinking defaults off unless explicitly enabled, but the central Opus `/new` recovery concern remains plausible and untracked by a canonical merged fix in the provided related context.

## Best Possible Solution

Keep this issue open for a maintainer/product decision. A good fix would be a targeted recovery path rather than silently changing all `/new` semantics: for example a `/new` recovery mode or stuck-run detector that can clear thinking/model runtime state, abort active reasoning runs, or offer a channel-safe prompt to reset thinking to off when a reasoning loop is detected. If maintainers decide the intended behavior is only “fresh transcript, same selected model/thinking,” the issue should be closed with docs or UX guidance explaining that `/think off` or model switch is the supported recovery path.
