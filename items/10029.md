## Summary

Keep #10029 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still renders WebChat code blocks as escaped monospace text with language labels, copy-button chrome, and JSON collapse behavior, but it does not tokenize code or apply syntax-highlighting markup or theme styles. The related copy-button work is separate and already present, but it does not satisfy the syntax-highlighting request.

## Best Possible Solution

Keep #10029 open until Control UI/WebChat integrates a sanitization-compatible syntax highlighter in ui/src/ui/markdown.ts or an adjacent renderer layer, adds matching theme CSS, and adds focused tests proving highlighted fenced code while preserving copy, language label, JSON collapse, cache, and sanitization behavior.
