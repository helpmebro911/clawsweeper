## Summary

Keep #10029 open. Current main still renders WebChat fenced and indented code as escaped monospace text with language labels, copy-button chrome, and JSON collapse behavior, but it does not tokenize code or apply syntax-highlighting markup/theme. The related closed #30390 copy-button work is separate and does not satisfy this syntax-highlighting request.

## Best Possible Solution

Keep #10029 open until Control UI/WebChat integrates a sanitization-compatible syntax highlighter in `ui/src/ui/markdown.ts` or an adjacent renderer layer, adds matching theme CSS, and adds focused tests proving highlighted fenced code while preserving copy, language label, JSON collapse, cache, and sanitization behavior.
