## Summary

Keep this PR open. Current main only covers part of the problem: it still writes delivery-mirror assistant entries and filters transcript-only OpenClaw assistants from model replay, but `chat.history` and `sessions.get` still consume raw transcript messages without the PR's requested provider/model filter. The provided discussion also includes a maintainer comment explicitly keeping this PR as the canonical Track B delivery-mirror consumer-path candidate under #69208.

## Best Possible Solution

Keep this PR or an equivalent narrow patch alive under #69208 Track B. The best implementation is to centralize the transcript-only OpenClaw assistant predicate and apply it consistently to `chat.history`, `sessions.get`, and any intended history consumer surface before pagination and byte budgeting, while preserving the existing model-replay filter and adding regression coverage for the visible/API paths.
