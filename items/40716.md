## Summary

Keep this PR open. Current main only partially addresses the delivery-mirror problem: replay/model context now drops transcript-only OpenClaw assistant artifacts, and the write path dedupes some redundant mirrors, but `chat.history` and `sessions.get` still read and return/project raw transcript messages without a shared delivery-mirror/gateway-injected filter. A maintainer also explicitly kept this PR as the canonical Track B candidate under #69208.

## Best Possible Solution

Keep this PR open or replace it with an equivalent narrow rebased patch under #69208 Track B. The best implementation is to centralize a shared transcript-only OpenClaw assistant artifact predicate covering at least `delivery-mirror` and `gateway-injected`, apply it consistently before pagination and byte budgeting in the intended visible/API consumer paths such as `chat.history` and `sessions.get`, preserve transcript persistence as the audit trail, and add focused regression tests for those consumer paths.

## Risks / Open Questions

- The PR is based on older context: current main already has broader replay filtering for both `delivery-mirror` and `gateway-injected`, so the final patch should not add a delivery-mirror-only parallel concept that misses newer transcript-only artifacts.
- Merging the PR as-is may be incomplete because `chat.history`, embedded chat-history stubs, `sessions.get`, and session-history HTTP/SSE surfaces now share adjacent projection paths that need a maintainer decision on the intended visible/API contract.
- The current write-path redundant-mirror check reduces some new duplicate persistence but does not clean or hide historical transcript entries, so consumer-side filtering is still needed if the Track B product direction remains to hide transcript-only artifacts from visible/API paths.
