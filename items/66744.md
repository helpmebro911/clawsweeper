## Summary

Keep open. Current `main` and the latest release still contain the exact unsafe `provider?.models?.some(...)` access, and the merge-mode `models.json` path can still carry forward malformed provider entries from an existing file. That means the reported `TypeError` is not clearly fixed on shipped or current code.
