## Summary

Keep open. Current checked-out main still contains the unguarded `provider?.models?.some(...)` call that can throw the reported TypeError for non-array runtime provider data, the focused test file lacks a non-array regression case, and the provided GitHub context identifies open PR #66761 as the unmerged closing implementation candidate.

## Best Possible Solution

Keep this issue open and review PR #66761 as the canonical implementation candidate. The likely fix remains a narrow `Array.isArray(provider?.models)` guard in `src/agents/models-config.providers.policy.lookup.ts` with focused regression tests in `src/agents/models-config.providers.policy.lookup.test.ts`; after that PR merges, close as implemented. A separate follow-up can address stricter upstream provider-shape normalization if maintainers want that broader contract hardening.
