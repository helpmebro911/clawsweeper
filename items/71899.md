## Summary

Keep #71899 open. Current main still resolves a non-default agent workspace by appending the agent id to any configured `agents.defaults.workspace`, while setup stores the main workspace in that same default field. The `agents add` wizard uses that resolver as the prompt initial value, so the documented peer-level `workspace-<agentId>` behavior is still missing on main. The PR is an active, focused fix for #71889 with relevant reviewer/reporter feedback already addressed in its discussion.

## Best Possible Solution

Keep #71899 open and continue normal maintainer review. The desired mainline fix should make setup-created `agents.defaults.workspace` values that point at the managed main workspace fall through to peer-level `workspace-<agentId>` defaults, while preserving explicit per-agent workspaces and intentional custom shared workspace bases. The PR's added resolver coverage, wizard-default coverage, docs note, and changelog entry are the right surfaces to review before merge.
