## Summary

Keep open. Current main still sends Feishu streaming replies as CardKit `card_id` interactive references, while the Feishu read paths still do not request raw CardKit content and the interactive-card parser still has no `json_card` or `card_id` resolution path. The related implementation PR #55466 was closed unmerged, so this remains an actionable Feishu plugin bug.

## Best Possible Solution

Keep this issue open. The fix should stay in the Feishu plugin read path: request validated raw CardKit content for both `getMessageFeishu()` and `listFeishuThreadMessages()`, parse `json_card` summary/body text before legacy element/post fallback, and add regression tests for quoted-message and thread-history reads of streaming CardKit cards. #55466 is useful prior art but needs rework or revival against current Feishu/Lark API behavior.
