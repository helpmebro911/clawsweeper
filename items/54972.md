## Summary

Keep open. Current main still sends Feishu streaming replies as CardKit `card_id` interactive messages, while quoted-message and thread-history reads do not request raw CardKit content or parse `json_card`, so the reported `[Interactive Card]` fallback remains actionable. The related fix PR #55466 was closed unmerged.

## Best Possible Solution

Keep this issue open and rework or review the #55466 approach against current main: request raw CardKit content from both `im.message.get` and thread `im.message.list`, parse `json_card` summary/body text before the legacy element/post fallback, and add quoted-message plus thread-history regression tests for streaming CardKit cards.
