## Summary

Keep open. Current main still rejects a root-level `paperclip` property on `agent` params: `AgentParamsSchema` does not include the field and remains `additionalProperties: false`, while the gateway handler validates with that schema before dispatch. The linked Paperclip adapter source for `@paperclipai/adapter-openclaw-gateway@2026.416.0` unconditionally appends `agentParams.paperclip`, matching the issue and live triage comments. The prior wake-payload relaxation does not cover `req agent`.

## Best Possible Solution

Keep the PR open and land a narrow additive protocol fix: accept an optional opaque `paperclip` metadata record in `AgentParamsSchema`, keep the regression test for `req agent`, and remove any unrelated test assertion churn before merge. If maintainers prefer not to special-case Paperclip in core, the alternative must be a coordinated Paperclip adapter/docs change to move metadata onto an already-supported agent params path; current main does not satisfy the documented adapter behavior.
