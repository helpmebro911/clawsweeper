## Summary

Keep #34400 open. Current main still only partially satisfies recursive memory search: default discovery recursively walks `memory/`, but nested dated paths still miss temporal decay, dirty searches still run sync in the background, watcher targets still include chokidar glob paths for extra/QMD directories, and several user-facing strings still advertise flat `memory/*.md`. The related fix PRs #34585 and #35691 were closed unmerged, so there is no landed implementation or canonical replacement to close against.

## Best Possible Solution

Keep #34400 open and land a focused memory-core fix or explicit maintainer decision. The likely implementation path is to support nested dated memory paths in temporal decay, replace remaining chokidar glob watch targets with directory targets plus filtering for extra paths and QMD collections, decide whether dirty `memory_search` should await sync for first-call freshness, update tool/prompt/config/docs copy to `memory/**/*.md`, and add regression tests for nested dated files plus watcher target construction. Reuse the concrete analysis from the closed unmerged PRs #34585 and #35691 rather than closing this issue as complete.
