Current main still enforces and documents the 0-5 `session.agentToAgent.maxPingPongTurns` cap. This PR remains a focused implementation candidate for the linked #43972 request, but it needs a rebase, generated/help/docs updates, boundary tests, and maintainer review before merge.

## Best Possible Solution

Rebase it onto current main, raise both the Zod schema maximum and runtime clamp to 20, update current public docs, schema help, TypeScript comments, and generated schema metadata, fix the existing review comments, and add focused boundary coverage proving 20 is accepted while values above 20 remain bounded before maintainer review.
