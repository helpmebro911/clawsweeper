## Summary

Current main still treats custom model-provider headers as literal static config. The Fireworks prompt-cache and LiteLLM/Langfuse use cases from #16387 remain unimplemented: there is no resolver for `{{session.key}}`, `{{agent.id}}`, or `{{run.id}}` in `models.providers.*.headers` or `models.providers.*.request.headers`, and the bundled Fireworks plugin does not add `x-session-affinity`.

## Best Possible Solution

Keep #16387 open. The best maintainer path is a narrow, documented runtime header-template resolver for `models.providers.*.headers` and `models.providers.*.request.headers`, evaluated at request dispatch with explicit placeholders for session key, agent identity, and per-turn/run identity. Add regression coverage for Fireworks `x-session-affinity`, LiteLLM/Langfuse-style headers, main turns, compaction, retries, and OpenAI-compatible transports. If maintainers prefer provider-owned behavior only, document that product decision and the plugin/API path that replaces custom-provider config templating.
