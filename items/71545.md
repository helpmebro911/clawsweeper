## Summary

Keep this PR open. Current main still supports streamable-http MCP servers, but the session runtime caches the original client and forwards `callTool()` directly with no lost-session detection, reconnect, metadata refresh, or retry path. The PR branch is a focused implementation candidate, and the provided discussion shows the early review defects were addressed in a maintainer hardening commit. Related #68187 is SSE-specific and does not supersede this streamable-http PR.

## Best Possible Solution

Keep this PR open for review. The best path is to validate and, if CI passes, land the PR branch’s streamable-http reconnect implementation in `src/agents/pi-bundle-mcp-runtime.ts` with its regression coverage in `src/agents/pi-bundle-mcp-runtime.test.ts` and `src/agents/pi-bundle-mcp-test-harness.ts`, preserving the custom schema validator, single in-flight reconnect, narrow lost-session detection, metadata refresh, one retry, and retryability after failed reconnect attempts.
