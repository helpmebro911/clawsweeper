## Summary

Keep #71545 open. Current main still caches one streamable-http MCP client per session runtime and forwards tool calls directly through the cached client without detecting lost session errors, rebuilding the transport/client, refreshing tool metadata, or retrying. The PR is a focused, recent core runtime fix with regression tests and active review context, including maintainer-updated hardening for the bot-review concerns, so it is not obsolete or safely closeable.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a hardened version of the streamable-http session recovery in `src/agents/pi-bundle-mcp-runtime.ts`, preserving the custom MCP JSON schema validator, sharing concurrent reconnects, refreshing catalog/tool metadata after reconnect, keeping failed reconnects retryable, and retaining the focused streamable-http regression tests from the PR.
