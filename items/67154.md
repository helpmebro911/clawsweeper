## Summary

Keep this issue open. Current main still stores and mutates runtime usage metadata inside `exec-approvals.json`; the related corruption class is mitigated by atomic writes, but the requested separate `exec-runtime.json` runtime store does not exist.

## Best Possible Solution

Keep this open as a focused core hardening feature. A maintainer-quality implementation should add a backward-compatible `exec-runtime.json` usage store, stop ordinary gateway/node/macOS exec from writing runtime metadata to `exec-approvals.json`, merge legacy and new runtime metadata for UI/CLI display, preserve existing allowlist reads, and add tests/docs proving a read-only approvals file still works while runtime tracking remains writable.
