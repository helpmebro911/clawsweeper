## Summary

Keep this issue open. Current main still has no opt-in TUI observe or mirror mode, and the TUI remains scoped to the active session for history, chat events, and BTW/side-result events. The broader same-author #41548 is still open and does not clearly supersede this narrower read-only TUI visibility request.

## Best Possible Solution

Keep this open as a core TUI/Gateway UX feature. Implement an explicit `openclaw tui --observe` or `tui.mirrorChannels` mode that preserves default `/new` isolation, aggregates recent same-agent activity through gateway/session history surfaces, relaxes exact-session event filtering only in observe mode, labels channel/session origin, and enforces read-only or clearly scoped interactive behavior. Add focused tests for same-agent different-session events, channel-originated final and side-result events, history aggregation, and read-only behavior; coordinate with #41548 only if maintainers choose it as the canonical umbrella.
