## Summary

Keep open. Current main has adjacent Telegram polling watchdogs, per-session nested lanes, and channel health-monitor recovery, but I did not find source, tests, docs, changelog, or related-item evidence proving that nested `sessions_send` activity can no longer freeze all Telegram pollers and gateway timers. The recent “Affects me” comment is current field signal, so conservative cleanup should not close this yet.

## Best Possible Solution

Keep this issue open until maintainers either reproduce it on current main with multiple Telegram bot accounts and nested `sessions_send`, or land/link a targeted fix with a regression proving Telegram polling and health-monitor timers continue after coordinator-to-subagent completion. The most useful next step is an integration/liveness test plus diagnostics from an affected deployment, using the gateway diagnostics export path documented at https://docs.openclaw.ai/gateway/health.
