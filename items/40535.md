## Summary

Current `main` already fixes the underlying allow-always persistence problem by falling back to hashed exact-command durable approvals when no reusable allowlist pattern can be persisted, and by reusing those approvals on later runs. That makes this PR's reject-on-failure approach obsolete.

## Close Comment

Closing after Codex review: current `main` already addresses the repeated-prompt problem this PR targets, but with a different implementation. Instead of rejecting `allow-always`, OpenClaw now falls back to a hashed exact-command durable approval when no reusable allowlist pattern can be persisted. The gateway path does this in `src/agents/bash-tools.exec-host-gateway.ts:324`, the node-host path does the same in `src/node-host/invoke-system-run.ts:626`, and the durable-match logic lives in `src/infra/exec-approvals.ts:792`. Coverage also exists for the rerun behavior in `src/agents/bash-tools.exec.approval-id.test.ts:712` and for exact-command durable trust in `src/infra/exec-approvals-policy.test.ts:164`. Because the underlying bug is already fixed on `main`, this PR is obsolete.
