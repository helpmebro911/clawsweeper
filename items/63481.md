## Summary

Keep this PR open. Current main still starts Feishu accounts through the gateway concurrently, the Feishu gateway still calls the per-account monitor path, and that path still fetches bot identity directly without a shared startup probe queue. The PR remains a focused implementation candidate for the linked Feishu multi-account startup timeout report; the later open PR #67466 is related but not merged and appears to pursue a different startup-latency strategy, so it does not safely supersede this one yet.

## Best Possible Solution

Keep this PR open as an active implementation candidate for the linked Feishu multi-account startup timeout bug. Maintainers should review or rebase it against current main, preserve the abort-while-waiting behavior from the follow-up commit, keep regression coverage for concurrent `fetchBotIdentityForMonitor` callers, and decide how it should interact with #67466's broader startup-latency work before merging.
