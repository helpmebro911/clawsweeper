## Summary

Keep open. Current `main` still has the reported user-scope-only systemd behavior in both the lightweight restart helper and the daemon lifecycle code, so this is not implemented on main. The PR also carries an unrelated Docker mount-whitespace change that conflicts with current validation.

## Evidence

- **Restart helper still user-first only:** `triggerOpenClawRestart()` on current `main` still runs `systemctl --user restart` first, then falls back to `systemctl restart`, with no scope detection or detected-scope reporting.
  - file: [src/infra/restart.ts:353](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/infra/restart.ts#L353)

- **CLI lifecycle code still targets user scope only:** `runSystemdServiceAction()` calls `execSystemctlUser(env, [params.action, unitName])` directly, and `restartSystemdService()` still returns plain `{ outcome: "completed" }`, so the main daemon/service path does not detect or report system scope.
  - file: [src/daemon/systemd.ts:604](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/daemon/systemd.ts#L604)
