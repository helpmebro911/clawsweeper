## Summary

Keep open. Current main has not implemented this PR's requested systemd scope detection, and the broader systemd lifecycle path still targets user-scope systemd only. The PR also should not land as-is because the provided review context flags an unrelated Docker validation change and a real active-vs-enabled priority bug in the proposed detection logic.

## Best Possible Solution

Keep the PR open or ask the author to rework it. The maintainer-quality path is a narrow systemd lifecycle fix in `src/daemon/systemd.ts` plus any needed `src/infra/restart.ts` parity, with tests for system-scope restart/status/stop, active-scope preference, no unrelated Docker changes, and docs clarifying supported system-service lifecycle behavior.
