## Summary

Keep open. Current `main` still has the underlying Linux service-scope gap on a documented supported surface, and this PR is not safe to auto-close because it mixes an unrelated Docker change with a partial restart fix that still has review-noted correctness issues.

There is no enabled/active scope detection or reporting in the shipped code.
