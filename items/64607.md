## Summary

Keep open. Current main has meaningful partial inline media support, but the full request is still not implemented: structured MEDIA attachments and managed/generated images render inline, while web/link previews are absent, remote markdown images remain intentionally flattened, direct tool-result image shapes are incomplete, and persisted raw audio blocks are redacted rather than normalized into durable playable attachments.

## Best Possible Solution

Keep this issue open as the umbrella unless maintainers split it. Preserve the shipped MEDIA directive, managed-image, and attachment player support, then implement or track server-side link previews with an explicit CSP/privacy/fetch policy, settle the remote markdown image policy alongside #41025/#40945, finish direct tool-result media rendering for #50779/#52025-style payloads, and normalize generated audio/video into durable renderable attachment or managed-media references.
