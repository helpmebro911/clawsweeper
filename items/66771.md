## Summary

Keep this issue open. Current main still has the plausible malformed mixed-key path: the MSTeams handler mutates the routed session key in place, the thread helper appends blindly, and the shared route cache can retain the mutated route object before a later thread appends a second suffix.

## Best Possible Solution

Fix MSTeams thread routing so cached base routes remain immutable. The narrow path is to compute a separate thread-qualified session key from a clean base route without mutating the object returned by resolveAgentRoute(), pass that key through MSTeams context/session dispatch, and add a regression test for two channel-thread replies in the same channel with different thread ids asserting the second key is base:thread:new, never base:thread:old:thread:new.
