## Summary

Keep #71537 open. Current main has partial session-memory reset recovery, but it still lacks the PR's archive-only `findPreviousSessionFile` fallbacks when only `<sessionId>.jsonl.reset.*Z` files remain, and the bundled `session-logs` skill still documents only active `*.jsonl` searches while mentioning deleted archives only as a tip. The related #60409 is complementary gateway history work, not a superseding fix for the memory hook or skill docs.

## Best Possible Solution

Keep the PR open for normal review. The best path is a scoped patch that lands the remaining archive-only session-memory fallbacks plus the `session-logs` archived-transcript guidance, with the now-redundant lockfile change removed if current main already has it.
