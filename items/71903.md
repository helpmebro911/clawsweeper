## Summary

Keep open. Current main still has the self-PID/no-starttime reclamation behavior this PR targets, and the PR is a focused core-session-lock fix tied to concrete lost-response reports. It is not obsolete or safely closable, but it needs normal maintainer review because the proposed test change skips an old behavior test rather than replacing it with explicit coverage for the new conservative behavior.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to refine the patch, add focused regression coverage for the live self-PID/no-starttime case, preserve PID-recycling/dead-PID reclamation tests, and then land it if the session-lock behavior matches the linked lost-response reports. Current main does not contain an equivalent fix.
