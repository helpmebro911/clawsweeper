## Summary

Keep open. Current main still contains the self-PID/no-starttime lock reclamation behavior that PR #71903 targets, and the related report is a plausible core transcript/session-lock corruption path rather than a cleanup-close category.

## Best Possible Solution

Keep #71903 open for maintainer review. The best path is to rebase it on current main, preserve the conservative live-PID/starttime handling if reproduction confirms it, replace the skipped test with explicit live self-PID/no-starttime and recycled-PID coverage, and coordinate with #44510 for the broader transcript corruption guard work.
