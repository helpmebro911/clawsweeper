## Summary

Keep open. Current main still does not clearly define or document Telegram channel_post behavior when per-chat sender-scoped allowFrom is configured, and the related PR #49063 is open/unmerged and explicitly left this semantic question out of scope. The code still relies on message sender identity for group allowFrom checks, while Telegram channel_post does not provide a normal human sender identity.
