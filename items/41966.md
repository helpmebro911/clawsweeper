## Summary

Keep this issue open. Current main still intentionally skips MEDIA directives inside fenced Markdown code blocks, the behavior is covered by an in-tree test, and the related implementation PR is closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open. The likely fix belongs in src/media/parse.ts and its tests: either extract valid own-line MEDIA directives from fenced blocks while preserving validation, or intentionally keep skipping fenced directives but surface a warning and update assistant/docs guidance so the failure is not silent. Review or replace the closed #41980 implementation rather than closing this issue before a fix lands.
