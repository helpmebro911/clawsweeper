## Summary

Keep this issue open. Current main still explicitly skips `MEDIA:` extraction inside markdown fences, tests still assert that fenced `MEDIA:` lines remain visible text, and the only directly related implementation PR is closed unmerged.

## Best Possible Solution

Keep this issue open. The best fix belongs in `src/media/parse.ts`: either extract own-line valid `MEDIA:` directives from fenced blocks or surface a visible warning while stripping/retaining text deliberately. Add focused tests in `src/media/parse.test.ts` for backtick and tilde fences, invalid/prose examples, and valid own-line directives, and update assistant/docs guidance if fenced directives remain unsupported.
