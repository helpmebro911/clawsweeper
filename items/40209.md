## Summary

Keep this PR open. Current main still stores system-event queues under the raw trimmed session key, while heartbeat forced-session resolution canonicalizes agent-scoped session keys to lowercase before peeking or consuming events. The PR remains a focused implementation candidate for the linked channel-session heartbeat bug; it is not implemented on main and is not obsolete.

## Best Possible Solution

Review and either land this focused PR with the mixed-case lookup and cross-case dedupe regression tests, or adjust it to canonicalize only recognized agent-scoped session keys before merge. Current main still exhibits the linked channel-session heartbeat/system-event mismatch, so closing this PR now would drop a plausible fix.
