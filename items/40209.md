## Summary

Keep this PR open. Current main still stores system events under the exact session-key casing supplied by the caller, while the heartbeat path canonicalizes targeted agent session keys to lowercase before peeking and draining system events. The PR remains a focused implementation candidate for the channel-session heartbeat miss described in the linked discussion.

## Best Possible Solution

Review and either land this focused PR with the mixed-case lookup and cross-case dedupe regression tests, or adjust the implementation to canonicalize only recognized agent-scoped session keys before merge. Closing this PR now would discard a plausible fix for a still-present heartbeat/system-event key mismatch.
