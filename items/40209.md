## Summary

Keep this PR open. Current main still stores system events under the raw trimmed session key, while heartbeat routing canonicalizes agent-scoped session keys to lowercase before peeking/consuming system events. The PR remains a focused implementation candidate for the linked channel-session heartbeat failure.

## Best Possible Solution

Keep this PR open for maintainer review or landing. The best path is to align system-event session-key canonicalization with heartbeat/session routing and keep the mixed-case channel lookup plus cross-case dedupe regression tests; if broad lowercasing is too wide, adjust the implementation to canonicalize only agent-scoped session keys.
