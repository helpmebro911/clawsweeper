## Summary

Keep open. Current main still has the case-sensitive system-event queue that PR #40209 is trying to fix: system events store and look up the raw trimmed session key, while heartbeat/session routing canonicalizes agent-scoped session keys to lowercase before peeking pending events. The linked issue #34338 was closed stale, but the underlying code path remains present and the PR's regression tests are not on main.

## Best Possible Solution

Keep PR #40209 open for maintainer review or landing. The best code direction is the scoped fix proposed here: normalize `requireSessionKey()` to the lowercase canonical form used by session routing, and add focused regression tests for mixed-case channel session lookup and cross-case deduplication. If maintainers prefer a broader heartbeat/session-key cleanup, fold this PR into that work rather than closing it as stale.
