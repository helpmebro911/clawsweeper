## Summary

Keep open. Current main still has the backup planning implementation in core and a broad `openclaw/plugin-sdk/runtime` backup-archive helper, but it does not expose the requested `openclaw/plugin-sdk/backup` subpath or public SDK exports for `BackupPlan`, `resolveBackupPlanFromDisk`, or archive-name helpers. The PR is stale and needs revision, but the work is still plausibly useful SDK surface rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open, but require a current-main revision. If maintainers want backup planning as a supported plugin API, add a narrow `src/plugin-sdk/backup.ts` subpath, update `scripts/lib/plugin-sdk-entrypoints.json`, `package.json` exports, plugin SDK docs, and API/export checks, then review whether the public surface should expose raw `BackupPlan` or a narrower plugin-facing backup-plan adapter.
