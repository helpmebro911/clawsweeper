## Summary

Keep PR #71738 open. Current main still has the Matrix thread isolation and reply-placement gaps described by the PR: Matrix detects thread roots for routing, but room history/watermarks and readMessages remain room-flat, and thread sends/edits still emit reply fallback metadata that can place threaded content in the main timeline. The PR is not implemented or superseded, but it should be revised before merge to address the existing review findings about stale thread watermarks and pagination correctness.

## Best Possible Solution

Keep the PR open and ask for a revision that preserves the core direction: add threadRootId-scoped Matrix room-history queues and watermarks, add thread-aware readMessages behavior, and remove unintended Matrix fallback reply metadata. Before merge, address the existing review findings with targeted tests for thread sub-queue eviction/watermark races and pagination correctness, then run the Matrix plugin tests and extension typecheck.
