## Summary

Keep this issue open. Current main has useful exact-duplicate stream guards, commentary-phase suppression, empty-response retries, and prompt guidance to reduce narration, but I did not find source, tests, docs, or history proving that the reported WebChat/Omlx-Qwen semantic repetition across separate tool and approval steps is fixed.

## Best Possible Solution

Keep this issue open and ask for a current-main WebChat reproduction with the raw transcript, gateway `chat` events, agent events, provider/model config, and whether tools or approvals were involved. If the duplicate is UI/history rendering, add a focused regression around `ui/src/ui/controllers/chat.ts` or `ui/src/ui/app-gateway.ts`; if it is duplicate event delivery, fix the gateway/subscriber idempotency path; if it is generated by the Omlx/Qwen model, tune the WebChat/system prompt or provider guidance and cover it with a regression for repeated visible preambles around tool and approval steps.
