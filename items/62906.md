## Summary

Keep open. Current main has several narrow duplicate guards for exact stream/final payloads, WebChat history refresh artifacts, gateway idempotency, and inbound redelivery, but the attached report is broader: repeated assistant preambles and repeated intent statements across tool and approval steps in WebChat with a Qwen/Omlx setup. I did not find source, docs, tests, or related issue evidence that current main fixes that observed behavior well enough to close this issue.

## Best Possible Solution

Keep this issue open and capture a current-main reproduction with one raw session transcript plus matching chat and agent gateway events for the same WebChat run. If the duplicate is UI/history rendering, add a regression around ui/src/ui/controllers/chat.ts or ui/src/ui/app-gateway.ts. If it is duplicate delivery from the embedded subscriber, add run/idempotent suppression in the subscriber or delivery path. If it is model-generated repetitive narration around tools/approvals, tune the WebChat/system prompt or add provider-specific prompt guidance for the affected Qwen/Omlx-compatible path rather than closing as implemented.
