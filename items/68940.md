## Summary

Keep open. Current main still has the docs-vs-CLI mismatch at the center of #68940: the CLI Backends quick start still documents `openclaw agent --message "hi" --model codex-cli/gpt-5.5`, but the top-level `openclaw agent` Commander registration and CLI reference still do not expose `--model`. The lower Gateway/agent layers now have provider/model override seams and the OpenAI plugin registers a default `codex-cli` backend, so the issue is actionable in this repo as CLI/docs plumbing rather than a stale or external report.

## Best Possible Solution

Keep #68940 open and fix the mismatch directly. Either add `--model <provider/model>` to `openclaw agent`, extend `AgentCliOpts`, parse/provider-split as needed, pass authorized provider/model overrides through both Gateway and `--local` embedded paths, and add parser plus routing tests; or update `docs/gateway/cli-backends.md` and other examples to use the existing `openclaw capability model run --model ... --prompt ...` command and clarify the bundled `codex-cli` registration prerequisites.
