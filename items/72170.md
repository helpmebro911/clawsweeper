## Summary

PR #72170 should stay open. Current main has a generic Kubernetes guide and an Azure Linux VM guide, but it still lacks AKS-specific deployment instructions, ACR integration, or an `az aks` workflow. The PR is a focused docs contribution in an established hosting-docs surface; the remaining concerns are reviewable doc correctness issues, not a conservative cleanup close reason.

## Best Possible Solution

Keep the PR open. The best path is maintainer review of a concise AKS deployment guide, preserving the non-loopback gateway bind requirement for ingress, replacing globally unique Azure resource names with placeholders, keeping cost/security claims conservative, and deciding whether AKS belongs in `docs/install/kubernetes.md` or a separate install page.
