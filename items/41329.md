## Summary

Keep open. Current main has broad replay hygiene that should mitigate Anthropic request rejection, but I do not see a heartbeat-specific containment/pruning fix for #41329. Default heartbeats still run in the parent session unless isolatedSession is enabled, and the current heartbeat transcript test explicitly locks in append-only behavior for meaningful heartbeat output. The unmerged #46987 discussion remains useful but is not proof that this issue was fixed on main; #54138 also tracks a related remaining heartbeat leakage symptom.
