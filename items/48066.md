## Summary

Keep open. Current main does not show a strong fix for the reported Feishu WebSocket startup/handshake crash path: the Feishu monitor still starts the SDK WebSocket client as a floating promise with only a synchronous try/catch, the global unhandled-rejection filter still exits on unclassified non-transient rejections, and there is no Feishu-specific regression coverage for handshake/start failures. The related #48125 discussion is useful context but is closed unmerged and scoped to webhook payload null checks, not this WebSocket load-time failure.
