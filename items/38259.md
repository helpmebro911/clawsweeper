## Summary

Current main still does not expand `$VAR` or `${VAR}` references while loading `.env` files. The dotenv loader parses values and assigns them unchanged, existing tests cover precedence/security but not expansion, and the linked fix PR #38254 is closed unmerged rather than superseding the issue.

## Best Possible Solution

Keep the issue open and implement dotenv-level expansion in `src/infra/dotenv.ts` before assigning values to `process.env`. Support both `${VAR}` and `$VAR` using process env plus earlier loaded dotenv entries, preserve shell/CWD/global/gateway precedence, and add focused regression tests in `src/infra/dotenv.test.ts` for shell-backed aliases, same-file aliases, chained aliases, missing references, workspace `.env`, global/state-dir fallback, and gateway.env fallback behavior. Reuse PR #38254 as prior art after rechecking it against current workspace `.env` security constraints.
