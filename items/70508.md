## Summary

Keep #70508 open. Current main and the latest release still use the Mattermost draft preview path that resets only `lastPartialText` at assistant/reasoning boundaries and writes tool status into the active draft post, so the reported overwrite behavior is not already fixed. The PR addresses a real unshipped Mattermost bug, but the current PR discussion shows unresolved race/test concerns that should be handled before merge.

## Best Possible Solution

Keep the PR open and use it as the active Mattermost fix path. The best mergeable version should rotate and flush the Mattermost draft stream before assistant/reasoning/tool boundaries, serialize or otherwise guard callback ordering so item and tool events cannot duplicate or overwrite each other, preserve the intended tool-status UX, and add Mattermost-owned regression tests mirroring the Telegram/Discord boundary cases. Update Mattermost streaming docs if the final user-visible behavior changes from one preview post to preserved boundary posts.
