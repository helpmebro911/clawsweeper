## Summary

Keep open. Current main still has the reported bug: `resolveProvidersForModelsJsonWithDeps` always calls implicit provider discovery and merges those providers before `models.mode` is applied. The Codex bundled provider still exposes a `catalog` hook, docs/schema still say `replace` should use configured providers only, and the related PR #68978 was closed unmerged rather than shipped.

## Best Possible Solution

Implement the missing branch in `src/agents/models-config.plan.ts` before calling `resolveImplicitProviders`: when `cfg.models?.mode === "replace"`, skip implicit discovery and return/normalize only explicit `models.providers`. Add a focused regression test proving default and `merge` still call implicit discovery while `replace` does not, and keep #66957/#48483-style model catalog follow-ups separate if they cover different UI/RPC surfaces.
