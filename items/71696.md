## Summary

Keep #71696 open. Current main still lacks Telegram chat-type-based reply mode and still has the reply-threading/plugin-context gaps this PR targets. The PR is recent, focused, and supersedes the earlier closed #70579 rather than being superseded by it.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase or retarget it onto current main, keep the runtime `replyToMode` fallback behavior, add Telegram `replyToModeByChatType` consistently across schema/types/generated metadata/docs, resolve reply mode per direct/group chat in message and native-command delivery, pass the triggering Telegram message id through outbound planning, expose provider message ids to plugin command handlers, and run the relevant Telegram/config/plugin/outbound checks.
