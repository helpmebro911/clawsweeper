## Summary

Keep open. Current main still does not reject `announce + feishu + missing delivery.to` at cron creation time, and the related fix PRs in the provided context are still open/unmerged rather than implemented on main.

## Evidence

- **Gateway create path still accepts announce delivery without target:** `cron.add` calls `createJob`, pushes the returned job, and persists it. `createJob` validates only the generic cron job spec, main-session agent id, delivery support, and failure destination support before scheduling; there is no Feishu/Lark missing-target check in this add path.
  - file: [src/cron/service/ops.ts:313](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/cron/service/ops.ts#L313)

It does not require `delivery.to` for `mode: "announce"` and `channel: "feishu"`.
  - file: [src/cron/service/jobs.ts:196](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/cron/service/jobs.ts#L196)
