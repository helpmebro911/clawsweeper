## Summary

Keep open. Current `main` still lacks any iMessage-side fallback to recover inbound `reply_to_*` context from `chat.db` when `imsg` omits it, so the bug this PR targets is not implemented on `main`. The PR discussion also shows the proposed fix was iterated for real review concerns rather than disproven, but the branch now targets the older `src/imessage/**` layout and would need a rebase onto the current `extensions/imessage/**` plugin structure plus fresh verification.
