## Summary

Keep open. Current `main` does not implement the requested per-spawn tool override for `sessions_spawn`: the tool schema and docs still expose model/thinking/sandbox/context controls but no `tools` parameter, and the documented controls remain global/per-agent policy (`tools.profile`, `tools.allow/deny`, `tools.subagents.tools.*`) rather than per call. The plugin-tool part of the report is partly outdated on `main`: subagent tool resolution now carries explicit plugin allowlists and supports gateway-bindable plugin registry reuse, so the remaining gap is the per-spawn override itself.
