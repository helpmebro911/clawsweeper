## Summary

Keep #23906 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still lacks a public per-spawn tool selector for `sessions_spawn`: the tool schema, native spawn params, and gateway agent RPC do not expose or transport `tools`, `disableTools`, `toolsAllow`, or equivalent fields. The related #15032 overlaps on restricted subagent tools but remains open and does not fully cover this issue's local-model tool-less spawn plus plugin/MCP inheritance details.

## Best Possible Solution

Keep #23906 open until maintainers implement or explicitly reject a backward-compatible per-spawn tool selection contract. A concrete implementation should add `sessions_spawn` support for `tools: "none" | string[]` or explicit `disableTools`/`toolsAllow` fields, validate and document it, carry it through `SpawnSubagentParams` and the gateway agent RPC, map tool-less spawns to runner `disableTools: true`, map subsets to `toolsAllow`, and add tests for core tools, plugin/MCP tools, global subagent policy precedence, sandbox/security behavior, and small local-model tool-less spawns. Keep #15032 related unless maintainers designate one canonical issue for both security isolation and local-model/plugin inheritance requirements.
