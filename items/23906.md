## Summary

Keep #23906 open. Current main still does not expose a per-call `sessions_spawn` tool selector or tool-less mode, even though lower-level embedded-runner primitives support `disableTools` and `toolsAllow`. Main has global/configured subagent tool policy, but that is not the requested backward-compatible per-spawn override, and the issue has a recent maintainer review explicitly saying to keep it open.

## Best Possible Solution

Keep #23906 open until maintainers implement or explicitly reject a backward-compatible per-spawn tool selection contract. A concrete fix should add a public `sessions_spawn` field such as `tools: "none" | string[]` or `disableTools` / `toolsAllow`, validate and document it, carry it through `SpawnSubagentParams` and the gateway `agent` RPC, map tool-less spawns to runner `disableTools: true`, map subsets to runner `toolsAllow`, and test core tools, plugin/MCP tool names, global subagent policy precedence, sandbox/security behavior, and local-model tool-less spawns. Keep #15032 related unless maintainers designate it as the canonical umbrella for both security isolation and local-model/plugin inheritance.
