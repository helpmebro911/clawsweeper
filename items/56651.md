## Summary

Keep open. Current main has partial mitigations for adjacent timeout paths, but it does not implement the requested gateway-level `announceTimeoutMs`, `connectionTimeoutMs`, or `pluginInitTimeoutMs` config surface. The remaining connection/plugin-init portions are not fully superseded by the related narrower items.

## Best Possible Solution

Keep this issue open and implement a deliberate gateway lifecycle timeout policy: add typed/schema/help/docs-backed gateway config for provider/channel connection and plugin initialization timeout budgets, decide whether announce timeouts belong under gateway or agent/subagent defaults, wrap the relevant async startup/announce paths with structured timeout errors, and cover the behavior with config validation plus gateway/channel/plugin lifecycle tests. Link or split narrower follow-ups for #51282, #59339, and #65415 if maintainers prefer separate workstreams.
