## Summary

Current main does not implement the requested cleanup on the user-facing outbound sanitizer path. The shared XML stripper exists, but `sanitizeUserFacingText()` still reaches only `stripFinalTagsFromText()`, which removes `<final>` tags and does not call `stripToolCallXmlTags()`. The auto-reply delivery normalization path still calls this sanitizer for outbound reply text, so the reported messaging-channel leak remains a meaningful open bug. Related PR #62883 targeted this exact fix but was closed unmerged in the provided GitHub context.
