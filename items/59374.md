## Summary

Keep open. The request is a concrete, recent feature request for Tavily API-key failover. The provided issue has no protected labels and no maintainer author association, but it is only 23 days old and describes actionable expected behavior. Public current Tavily docs still document a single `plugins.entries.tavily.config.webSearch.apiKey` / `TAVILY_API_KEY` configuration path, not multiple-key rotation, so there is not strong evidence that current main already implements the requested behavior. It also belongs in the Tavily plugin/provider surface rather than being incoherent or stale.
