## Summary

Current `main` does not give strong enough evidence to close this. The macOS app is intentionally a menu-bar-only app (`LSUIElement`) with no normal main window, which could explain a user seeing “no response” after double-clicking, but that does not prove the reported 2026.3.7 DMG install was healthy. I did not find a repo-local fix tied to this issue or a shipped macOS-launch regression fix that clearly resolves the report, and this environment cannot reproduce the DMG launch path on macOS.
