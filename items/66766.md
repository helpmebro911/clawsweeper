## Summary

Keep open. Current main still preserves the reported 200k fallback for Anthropic Opus/Sonnet 4.6 without `params.context1m: true`, and open PR #66862 is an implementation candidate that references this issue with closing syntax. #66790 is closed/unmerged and superseded by #66862, so there is no merged fix yet.

## Best Possible Solution

Keep this issue open while maintainers review PR #66862 or an equivalent patch. The fix should align `resolveContextTokensForModel()`, session/status persistence, Control UI behavior, Anthropic beta-header/auth semantics, docs, and regression coverage for Opus/Sonnet 4.6 without `context1m`, including an explicit opt-out or legacy-auth case.
