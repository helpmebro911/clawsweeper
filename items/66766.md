## Summary

Current `main` still appears to have the reported bug. The Anthropic 1M override for Opus/Sonnet 4 remains gated on `params.context1m === true`, and the read-only status/session code paths still call the resolver with `allowAsyncLoad: false`, which does not perform discovery. The existing unit test also still expects `anthropic/claude-opus-4-6` without `context1m` to resolve to `200_000`, so this item should stay open.
