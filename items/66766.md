## Summary

Keep this issue open. Current main still preserves the reported Anthropic Opus/Sonnet 4.6 no-`context1m` path as 200k, and open PR #66862 is an unmerged implementation candidate that references this issue with closing syntax.

## Best Possible Solution

Keep this issue open while maintainers review PR #66862 or an equivalent patch. The fix should align the context resolver, session/status persistence, Control UI behavior, Anthropic beta-header/auth semantics, docs, and regression coverage for Opus/Sonnet 4.6 without `context1m`, including any needed explicit opt-out behavior.
