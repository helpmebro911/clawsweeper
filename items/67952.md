## Summary

Keep this PR open. Current main still implements `ltm list` as a count-only command duplicating `ltm stats`, so the PR addresses a real remaining memory-lancedb CLI behavior gap rather than obsolete or already-shipped work.

Keep this PR open or replace it with an equivalent maintainer patch: add a bounded `MemoryDB.list` path that returns non-vector records, make `ltm list` print JSON entries with validated `--limit` and optional newest-first ordering, keep `ltm stats` as the count-only command, and add focused memory-lancedb CLI regression tests before merge.
