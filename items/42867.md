## Summary

Keep this issue open. Current main has the heartbeat-aware daily/idle freshness half, but it still lacks a configurable turn-count TTL or equivalent max-turn rollover for sessions. The same-author #45390 overlaps broader session TTL/max-lifetime work, but it remains open and is not a maintainer-designated canonical replacement for this narrower max-turn reset request.

## Best Possible Solution

Keep this open as a core session lifecycle feature unless maintainers explicitly consolidate it into #45390. A complete implementation should add a documented and schema-validated max-turn reset policy, define which turns count versus heartbeat/system events, wire the check into automatic session rollover before reuse, and add regression coverage for daily, idle, heartbeat, and max-turn interactions.
