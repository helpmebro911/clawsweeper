## Summary

Keep this issue open. Current main has the heartbeat-aware staleness half, but it still does not provide a configurable turn-count TTL or any equivalent automatic rollover based on session turns. The open same-author #45390 overlaps the broader TTL/max-lifetime theme, but it is not a safe automatic duplicate close for this narrower request.

## Best Possible Solution

Keep this open as a core session lifecycle feature unless a maintainer explicitly consolidates it into #45390. A complete fix should add schema-validated and documented turn-count reset policy, define which interactive turns count versus heartbeat/system events, wire the policy into automatic session rollover before reuse, and add regression coverage for daily, idle, heartbeat, and max-turn interactions.
