## Summary

Keep open. Current main still lacks a doctor warning for missing `memorySearch.extraPaths`: `noteMemorySearchHealth` does not inspect `resolved.extraPaths`, the doctor tests have no extraPaths coverage, and the builtin memory indexer silently ignores missing extra paths. PR #46837 is a focused unmerged fix for a real gap, but it should be revised to align path resolution with the runtime/backends and cover the reviewer-raised `~` behavior.

## Best Possible Solution

Keep #46837 open and revise it into a corrected small fix: add a doctor diagnostic for the resolved default-agent `memorySearch.extraPaths`, share or mirror the runtime path resolver for builtin and QMD behavior, warn on stale configured paths without suppressing existing provider diagnostics, and add focused tests for absolute, workspace-relative, missing, existing, and `~` cases according to the chosen contract.
