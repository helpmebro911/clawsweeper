## Summary

Keep #13610 open. Current main implements the external SecretRef baseline, but native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager sources are still absent; the issue discussion explicitly keeps this as the umbrella after #26155.

## Best Possible Solution

Keep #13610 open as the native secrets-manager provider umbrella. The best product path is to either implement first-class AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager providers across types, schema/protocol, resolver dispatch, CLI/docs, and tests, or make an explicit maintainer decision that exec wrappers or a future plugin/ClawHub provider API are the supported route.
