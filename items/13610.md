## Summary

Keep #13610 open. Current main has the shipped SecretRef baseline for env/file/exec providers, but it still does not implement native AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, or Google Cloud Secret Manager providers. The issue discussion explicitly preserves #13610 as the native-provider umbrella after #26155 shipped the baseline.

## Best Possible Solution

Keep #13610 open as the native secrets-manager provider umbrella. The maintainer path is either to add first-class AWS Secrets Manager, HashiCorp Vault, Azure Key Vault, and Google Cloud Secret Manager support across SecretRef types, config schema, resolver dispatch, CLI configure/audit/apply flows, docs, and tests, or make an explicit product/API decision that exec wrappers or a future public secret-provider plugin/ClawHub API are the supported route.

## Risks / Open Questions

- Closing this now would remove the explicit umbrella for first-class secrets-manager provider work that the issue discussion preserved after #26155 merged the env/file/exec baseline.
