## Summary

Current main still matches the reported bug: OpenClaw docs recommend the static Browserbase WebSocket URL, the browser plugin connects configured CDP URLs directly without Browserbase session creation, and current Browserbase docs require creating a session and using the returned connectUrl.

## Best Possible Solution

Keep this issue open. The narrow fix is to update docs/tools/browser.md so it no longer recommends the obsolete static Browserbase root URL, then either document the manual create-session/connectUrl workflow or add Browserbase session creation/release support in the bundled browser plugin. If maintainers want this vendor-neutral, add a typed remote-CDP session-provider seam in the browser plugin/SDK and implement Browserbase through that seam rather than hard-coding provider lifecycle behavior in shared core.
