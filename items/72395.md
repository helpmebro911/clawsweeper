## Summary

Current main already fixes the Windows `ERR_UNSUPPORTED_ESM_URL_SCHEME` path used by this Discord/browser report: bundled plugin Jiti native loading is disabled on Windows, Windows absolute import specifiers are normalized to `file://` URLs, and regression coverage locks the Windows channel/plugin loader behavior. This is the same root-cause family already closed in related reports such as #72198 and #72231.

## Best Possible Solution

Close this as implemented on current main, keep the shared Windows-safe plugin loader behavior, and ship it in the next release. If a fresh report appears after a release containing this code, it should include the current OpenClaw version, Node version, full gateway/plugin logs, and which bundled plugin ids still hit a raw Windows path.
