## Summary

Keep this issue open. Current main has partial protections for Gemini reasoning, including native Google `thought: true` parsing, tagged reasoning cleanup, final-tag enforcement, and exact `Reasoning:` prefix handling. It still does not implement the broader reported fix for Gemini reasoning emitted as ordinary text, especially appended or multilingual reasoning preambles before channel delivery.

## Best Possible Solution

Keep this issue open as the umbrella for Gemini plain-text reasoning leakage. The best path is to review or replace #60551 with a narrow shared delivery sanitizer and provider/channel tests that cover untagged planning prose, appended reasoning blocks, multilingual reasoning headers, tagged fragments, and normal prose pass-through. Keep #48593 scoped to final-tag fragment cleanup unless it is expanded and merged with broader coverage.

## Evidence

- **Native Google transport only separates explicitly marked thoughts:** The direct Gemini transport maps parts with `thought: true` into thinking blocks, but parts with ordinary `text` are appended to visible text. ## Risks / Open Questions

- A broad preamble sanitizer can create false positives in normal prose, code blocks, quoted text, or multilingual answers, so the fix needs targeted tests and conservative matching.
- Some assistant stream paths are already protected by final-tag enforcement, but message-tool, block-reply, and channel delivery surfaces can still receive unflagged visible text; the final fix should cover the actual delivery boundary without relying only on prompts.
