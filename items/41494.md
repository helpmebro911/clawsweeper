## Summary

Current main still only hides Gemini reasoning when it arrives as structured native thinking, recognized reasoning tags, or payloads explicitly marked `isReasoning: true`. The reported plain-text Gemini leak shape, including multilingual reasoning headers and untagged planning prose, is not covered, and the supplied discussion points to an open candidate fix rather than shipped behavior.

## Best Possible Solution

Keep this issue open as the umbrella for Gemini plain-text reasoning leakage. The best path is to review and land #60551 or a replacement that adds a conservative outbound sanitizer at the delivery boundary, with provider/channel regression tests for untagged planning prose, appended reasoning blocks, multilingual reasoning headers, tagged fragments, and normal prose pass-through.

## Evidence

It also distinguishes the broader plain-text leak from narrower final-tag fixes and cross-references open candidate PR #60551.

## Risks / Open Questions

- A broad plain-text sanitizer can false-positive legitimate prose, quoted examples, code blocks, or multilingual answers if it is not tightly scoped.
- Some structured reasoning paths already behave correctly, so a fix should not regress native `isReasoning` suppression, tagged reasoning stripping, or intentional `/reasoning on` display.
