## Summary

Keep this issue open. Current main still accepts `skills.update` config-mode payloads with `apiKey` and `env`, writes them into `skills.entries.<skillKey>`, and returns `config: current` directly in the success response. There is also an open unmerged implementation candidate, #69998, that uses closing syntax for this issue, so this issue should remain open until that PR is reviewed and merged or closed.

## Best Possible Solution

Review and land #69998 if it is clean, or apply the same narrow fix directly in `src/gateway/server-methods/skills.ts`: import/use `redactConfigObject()` for the config-mode `skills.update` response and add a regression test proving `apiKey` and secret-like `env` values are redacted in the response while full values are still written to config.
