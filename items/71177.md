## Summary

Keep open. Current main has useful mitigations for heartbeat context growth, especially optional `isolatedSession` and `lightContext`, but it does not implement the requested automatic heartbeat session rotation policy. Non-isolated heartbeat runs still use the existing session, and the docs/config surface explicitly make isolated heartbeat sessions opt-in rather than default rotation.

## Best Possible Solution

Keep this open or explicitly consolidate it into #45390 after maintainer review. The concrete core path is to add a heartbeat-owned lifecycle policy, likely under `agents.defaults.heartbeat`, with clear defaults, archive behavior, tests around non-isolated heartbeat sessions, and docs showing how it interacts with existing `isolatedSession`, `lightContext`, daily/idle reset, and the related #71031 session-memory poisoning report.
