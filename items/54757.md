## Summary

Keep this issue open. The prior ClawSweeper comment correctly identified the remaining gap, and current main still lets final tool-error warning payloads reach the WhatsApp reply path. The latest related history only narrows some acknowledged failed-action warnings and Telegram preview behavior; it does not add a WhatsApp external-channel suppression policy.

## Best Possible Solution

Implement an external-recipient-safe tool-error policy for WhatsApp and similar channels. The narrow code path is to pass suppressToolErrorWarnings, or a channel-scoped equivalent, into WhatsApp inbound auto-reply runs and add regression tests proving final read/edit/write/message failure warnings are not delivered to WhatsApp while normal final replies and intentional media/tool artifacts still deliver.
