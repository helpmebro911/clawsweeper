## Summary

Keep this issue open. Current main still lets final tool-error warning payloads reach WhatsApp delivery, and the existing global suppression does not cover mutating tool failures like message/edit/write failures. The prior ClawSweeper comment is still accurate against the newer c2d82b87 main checkout.

## Best Possible Solution

Implement an external-recipient-safe tool-error policy for WhatsApp and similar channels. The likely narrow fix is to pass suppressToolErrorWarnings, or a channel-scoped equivalent, into WhatsApp inbound auto-reply runs and add regression coverage proving final read/edit/write/message failure warnings are not delivered to WhatsApp while normal final replies and intentional media/tool artifacts still deliver.
