## Summary

Keep open. Current main still intercepts WebUI `/model <args>` locally and sends any non-empty argument through `sessions.patch`, so `/model status`, `/model list`, and `/model info` are not implemented in the WebUI local handler. The PR addresses a real WebUI inconsistency, but its blanket `status|list|info` reservation still needs maintainer/product judgment because the shared `/model` contract treats the argument as a free-form model id, and only `status`/`list` are clearly documented/shared today.

## Best Possible Solution

Keep the PR open and steer it toward a safer WebUI fix: align Control UI `/model list` and `/model status` with the shared model directive/list/status behavior, preferably reusing that shared path or adding an explicit documented disambiguation rule. Add WebUI regression tests proving `status`/`list` do not call `sessions.patch`, normal model selection still does, and any maintainer-approved behavior for exact reserved-word model IDs and `/model info` is locked down.
