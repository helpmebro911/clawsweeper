## Summary

Keep #71795 open. Current main still queues a trusted Mattermost inbound-preview system event containing the user message text immediately before dispatching the real inbound reply, and the shared reply path formats queued events as `System:` lines prepended to the prompt. That matches the reported `prompt.submitted` first-line behavior. Discord's normal message path uses clean `BodyForAgent` plus untrusted metadata and does not add the same inbound-preview system event. The related ClawSweeper matches are adjacent channel/system-event reports but do not supersede this Mattermost-specific bug.

## Best Possible Solution

Keep the issue open and fix the Mattermost normal inbound path so user message text is not echoed into a trusted `System:` event. Sender/channel metadata should stay in the inbound envelope or an explicit untrusted metadata/context field, while `BodyForAgent` remains the clean user message. Add a Mattermost regression test that drives a normal inbound post through dispatch and asserts the submitted prompt does not start with `System: ... Mattermost message ...` containing the user body.
