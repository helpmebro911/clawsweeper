## Summary

Keep #71795 open. Current main still matches the reported prompt contamination path: Mattermost enqueues a system event for every accepted inbound post using the user-message preview, and the shared reply prompt builder drains those queued events into `System:` lines prepended to the model prompt. Mattermost also sets `BodyForAgent` to the clean message body, so the issue wording is slightly imprecise, but the observable `prompt.submitted` first-line `System: ... Mattermost message ...` behavior remains current.

## Best Possible Solution

Keep the issue open. The likely fix belongs in the Mattermost plugin: stop using prompt-drained `enqueueSystemEvent` for ordinary inbound user-message previews, or route that preview to logging/activity UI only. Preserve the existing clean `BodyForAgent`/`BodyForCommands` behavior, and add a Mattermost regression test proving a normal accepted post does not prepend `System: Mattermost message ...` to the model prompt while real Mattermost/system/interactions events still use the system-event path when appropriate.
