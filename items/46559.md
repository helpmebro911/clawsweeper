## Summary

Keep open. Current main still has the core leak path this PR is trying to address: assistant chat deltas, throttled flushes, and final/error chat payloads are forwarded to session/node subscribers without checking `/verbose off`. The PR discussion also identifies real gaps in the PR implementation, so the right maintainer path is a narrower clean fix, not an auto-close as implemented.

## Best Possible Solution

Keep the item open or replace it with a clean PR tied to #25592. The fix should gate session/node delivery of assistant chat deltas, throttled flushes, and final/error chat payloads when resolved verbose is `off`, while preserving WS/control UI broadcasts. Add regression tests for verbose-off assistant deltas, pre-tool flushes, lifecycle final/error payloads, and any intended in-flight verbose setting behavior. Drop the unrelated plugin/docs/SDK changes from this PR.
