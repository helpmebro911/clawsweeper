## Summary

Review of https://github.com/openclaw/openclaw/pull/67706: keep open. Current main still lacks compaction lifecycle event emission from embedded-runner overflow and timeout recovery, and this PR targets a real Control UI observability gap. It is not implemented or superseded on main, but the patch should be revised before merge to align completion semantics and protect memory compaction accounting.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. Best path: update src/agents/pi-embedded-runner/run.ts so overflow and timeout recovery emit compaction lifecycle events consistent with handleCompactionEnd, adjust memory accounting to only count completed compactions, and add targeted overflow and timeout recovery tests for both the global event bus and params.onAgentEvent before merge.

## Risks / Open Questions

- The PR as written computes completed as ok && compacted, while the canonical subscribe handler uses result presence and abort state; that mismatch should be resolved before merge.
- Because agent-runner-memory currently treats any compaction phase:end as completed, failed recovery end events could incorrectly increment memory compaction state unless the consumer or emission contract is tightened.
- Focused regression coverage is still missing for both emitAgentEvent and params.onAgentEvent compaction start/end payloads in successful and failed overflow/timeout recovery paths.
