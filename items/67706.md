## Summary

Current main still lacks compaction event emission in the embedded runner's overflow and timeout recovery helpers, so this PR targets a real remaining behavior gap. It should stay open, but the patch needs semantic hardening before merge because failed recovery end events can be miscounted by an existing memory consumer unless that consumer or the emitted event contract is tightened.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best path is to update the recovery helpers on current main to emit compaction start/end events to both `emitAgentEvent` and `params.onAgentEvent`, align `completed` semantics with `handleCompactionEnd` or explicitly justify the difference, harden `agent-runner-memory.ts` to require `completed === true` before counting a compaction, and add focused overflow/timeout recovery tests for successful and failed start/end payloads.
