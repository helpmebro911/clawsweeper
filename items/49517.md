## Summary

Keep open. Current main still treats bare `/new` as a reset on the gateway/web chat path: the agent handler matches `/new` and `/reset`, calls `performGatewaySessionReset`, and the reset service overwrites the same session store entry with a new session id. There is a TUI-specific improvement that gives `/new` a unique session key, but the central web/gateway/channel behavior reported in #49517 is still not implemented on main, and the related PR #49595 was closed unmerged in the provided GitHub context.

The nearby test explicitly describes bare `/new` as resetting the same session.
