## Summary

Current main still does not implement the requested Telegram `groupPolicy: "members"` all-members allowlist policy. The config contract, generated metadata, Telegram runtime, docs, and tests still model only `open`, `disabled`, and `allowlist`; the related chat-only-trigger plugin is useful prior art but not the requested bundled Telegram policy.

## Best Possible Solution

Keep #20321 open for explicit bundled Telegram plugin and shared config-contract work. A complete fix should add `groupPolicy: "members"` to exported config types, strict schema, generated metadata/help, docs, and Telegram runtime; define fail-closed all-current-members semantics; implement bounded Telegram membership/count verification with cache invalidation on join/leave events; and add focused schema, runtime, security, and docs tests. Treat the closed PRs and `openclaw-chat-only-trigger` as useful prior art, not as a close basis.
