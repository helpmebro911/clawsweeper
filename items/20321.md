## Summary

Current main still does not implement the requested Telegram `groupPolicy: "members"`. The shared config type/schema, generated config metadata, Telegram runtime authorization path, docs, and current tests all still model only `open`, `disabled`, and `allowlist`. The linked `openclaw-chat-only-trigger` plugin is useful related prior art, but it is a chat-only gating workaround, not the requested bundled Telegram all-members allowlist policy.

## Best Possible Solution

Keep #20321 open for explicit bundled Telegram plugin and shared config-contract work. The likely implementation belongs in the Telegram plugin plus shared config/schema/docs: add `groupPolicy: "members"` to the exported type and strict schema, define fail-closed all-current-members semantics, implement Telegram membership/count verification with bounded caching and join/leave invalidation, update security/doctor/config docs, and add focused runtime/schema tests. Treat the closed PRs and `openclaw-chat-only-trigger` as useful prior art rather than a close basis.
