## Summary

Keep this PR open. Current main still normalizes the iMessage sent-message echo text key with CRLF normalization plus trim only, so NUL-prefixed echoed text will not match the outbound text cache. The PR is a narrow implementation candidate for the linked iMessage echo bug and is not superseded by a merged fix.

## Best Possible Solution

Review and land this focused iMessage echo-cache normalization PR, or land an equivalent patch that strips NUL-prefixed echoed text and adds regression coverage for single and repeated leading NUL bytes. Keep broader U+FFFD or all-control-character normalization as a separate maintainer decision unless intentionally combined.
