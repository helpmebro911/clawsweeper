## Summary

Keep open. Current main still preserves QMD diff-format snippets through search and short-term recall tracking, then promotion rehydration still tries to match that stored diff snippet against normalized live daily-note text. A ranked QMD-backed candidate can therefore still fail promotion exactly as reported.

## Best Possible Solution

Fix memory-core so QMD diff snippets are converted to promotion-matchable raw text before they enter short-term recall state, or make rehydration tolerant of QMD diff headers while preserving the stale/deleted-source guard. Add a regression test where a QMD-style '@@ -x,y' snippet is ranked and successfully promotes from the corresponding daily memory file.
