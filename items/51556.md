## Summary

Keep open. Current `main` at `608c08fc54da2f86338366f8eeafd7e2a02713aa` still swallows non-`ENOENT` errors in the sync pairing allowlist reader, so this is not implemented on main. The PR also needs maintainer follow-up because its `readFileSync` change no longer matches the current async path: async `stat` errors are propagated, but async `readFile` errors are still converted to fallback empty state via `readJsonFileWithFallback`.
