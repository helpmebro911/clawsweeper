## Summary

Keep open. Current main still does not trigger the bundled session-memory hook for automatic daily/idle session resets. The hook is registered only for command:new and command:reset, the handler ignores non-command events, and the lazy stale-session path archives and emits session_end/session_start plugin hooks without calling the internal reset hook path. PR #61675 appears to target this exact bug, but it is open/unmerged in the provided context and its head SHA is not an ancestor of current main.
