## Summary

Keep #7234 open. Current main still exposes Discord `actions.messages` as a single boolean, validates only that boolean shape, advertises read/edit/delete together behind the same gate, and blocks send/read/edit/delete through `isActionEnabled("messages")`. The 2026-04-26 prior Codex/maintainer comment keeping this open still matches current main at e672b61417af5c45b0431df6d9109a1f4b618ef5.

## Best Possible Solution

Keep #7234 open. Implement a backward-compatible Discord gate shape such as `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }` or equivalent separate top-level gates, then update config types, strict zod schema, generated metadata/docs, account gate resolution, discovery, runtime checks, and focused Discord tests. Preserve existing boolean behavior while allowing deployments to disable destructive deletes without disabling sends or reads.
