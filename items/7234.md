## Summary

Keep open. Current main still models Discord `channels.discord.actions.messages` as one boolean gate, and Discord discovery/runtime still use that one gate for read/fetch, send, edit, and delete. The request is concrete, not protected by author/label rules, and the provided related items do not supersede this missing Discord permission granularity.

## Best Possible Solution

Continue tracking this issue until Discord supports backward-compatible granular message gates, likely `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }`, with aligned config types, zod schema/generated metadata, account action-gate handling, discovery, runtime checks, docs, and focused tests. If maintainers prefer a different permission model, capture that product decision here rather than closing the concrete request as cleanup.
