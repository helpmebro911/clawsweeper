## Summary

Keep #7234 open. Current main still models Discord message gates as one coarse boolean and still routes send/read/edit/delete through that same `messages` gate, so the requested independent controls are not implemented.

## Best Possible Solution

Keep #7234 open. Implement a backward-compatible Discord gate shape such as `channels.discord.actions.messages: boolean | { send?: boolean; read?: boolean; edit?: boolean; delete?: boolean }` or equivalent top-level gates, then update config types, strict zod schema, generated metadata, account gate resolution, discovery, runtime checks, docs, and focused Discord tests. Preserve existing boolean behavior while allowing deployments to disable destructive delete without disabling send/read/edit/read history.
