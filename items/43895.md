## Summary

Keep this PR open. Current main still builds the aggregate status session-default display from one global configured model, even though per-agent `agents.list[].model` overrides are supported and can make effective defaults differ by agent. The PR still addresses a real remaining status/status JSON/TUI summary gap, but it should be rebased or adapted to current status runtime seams.

## Best Possible Solution

Keep this PR open or replace it with a narrow maintainer patch. Compute `sessions.defaults` from each listed agent's effective status-safe default model, keep a concrete model only when all agent defaults match, expose an optional variation marker through `StatusSummary` and `GatewayStatusSummary`, render `varies by agent` in CLI and TUI status surfaces, and add divergent/uniform per-agent regression tests.
