## Summary

Keep open. Current `main` still only protects the immediate active session during store maintenance, not child subagent sessions, and this PR's proposed helper was already flagged for reading only in-memory runs instead of the persisted cross-process snapshot used elsewhere on `main`. That means the item is neither implemented on `main` nor strong enough to close as obsolete.

## Evidence

- **Current maintenance path preserves only the caller's active session key:** `saveSessionStoreUnlocked()` builds `preserveSessionKeys` from `opts.activeSessionKey` alone, then passes that set into `pruneStaleEntries()` and `capEntryCount()`. There is no current-main lookup for active subagent child session keys before pruning.
  - file: [src/config/sessions/store.ts:284](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/config/sessions/store.ts#L284)
  - command: `nl -ba src/config/sessions/store.ts | sed -n '250,320p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Prune/cap helpers can preserve keys, but only if the caller supplies them:** `pruneStaleEntries()` and `capEntryCount()` support `preserveKeys`, but they do not discover subagent-backed sessions themselves. On current `main`, they rely entirely on the caller-provided set.
  - file: [src/config/sessions/store-maintenance.ts:156](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/config/sessions/store-maintenance.ts#L156)
  - command: `nl -ba src/config/sessions/store-maintenance.ts | sed -n '150,320p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Cross-process subagent reads on main are supposed to use the persisted snapshot:** `getSubagentRunsSnapshotForRead()` merges disk-backed registry state with in-memory runs so other workers can observe active runs. Any protection helper for active child sessions needs this snapshot behavior, not just the local `subagentRuns` map.
  - file: [src/agents/subagent-registry-state.ts:37](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/subagent-registry-state.ts#L37)
  - command: `nl -ba src/agents/subagent-registry-state.ts | sed -n '1,180p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Main has an explicit regression test proving persisted active runs matter:** `session-utils.subagent.test.ts` verifies the system should treat a child session as running when the local worker only has terminal snapshots but the persisted registry still has an active run. That matches the Codex review concern on this PR.
  - file: [src/gateway/session-utils.subagent.test.ts:595](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/gateway/session-utils.subagent.test.ts#L595)
