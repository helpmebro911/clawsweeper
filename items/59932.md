## Summary

Keep open. Current main still lacks the active-subagent session protection described by PR #59932: session pruning/capping do not accept or honor protected subagent keys, store save maintenance does not query the subagent registry, disk-budget enforcement only protects the single activeSessionKey, and the subagent registry has no getActiveSubagentSessionKeys helper. The PR also has an unresolved Codex P1 review concern about reading only in-memory runs rather than the persisted run snapshot, so it should not be closed as already implemented.
