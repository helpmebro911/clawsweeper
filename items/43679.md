## Summary

Keep this PR open. Current main still lacks the PR's shared suspicious-orchestration guard in the outbound reply normalization path; existing code handles exact/JSON/mixed NO_REPLY and other sanitizer families, but not the route-marker plus tool-JSON leakage shape this PR targets.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase it onto current main, then either land the shared normalizeReplyPayload guard with its regression tests or adapt the same narrow route-marker/tool-JSON protection into the current sanitizer stack without weakening existing NO_REPLY, media/channelData, and explanatory-prose behavior.
