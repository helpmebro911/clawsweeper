## Summary

Keep open. Current main still uses Feishu open_id-only mention matching, and this PR is a plausible implementation candidate for the still-open linked bug, but it needs a refreshed, complete patch before maintainer cleanup can close it.

## Best Possible Solution

Keep this PR open and ask for a refreshed implementation tied to #40768. The best path is to first verify the Feishu/Lark bot identity contract, then thread the chosen stable identifier through `FeishuProbeResult`, monitor identity state, event dispatch, `parseFeishuMessageEvent`, `checkBotMentioned`, mention stripping, mention-forward extraction, debounce mention merging, and sequential-key parsing. Add focused regression coverage for a multi-bot group where the mention open_id differs but the stable identifier matches, including `/model` stripping and a debounced message batch.
