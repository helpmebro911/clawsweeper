## Summary

Keep this PR open. Current main still does not preserve under-cap non-alpha PNGs in the shared web media optimization path, and JPEG filename normalization still only applies to HEIC sources. The PR is stale against the current `src/media/web-media.ts` layout and should address the existing validation concern before merge, but the underlying behavior remains useful and unresolved on main.

## Best Possible Solution

Keep the PR open, ask for a rebase onto `src/media/web-media.ts` and `src/media/web-media.test.ts`, and adjust the implementation so under-cap PNG preservation only bypasses JPEG conversion after PNG bytes are validated. Preserve the useful filename fix by normalizing every actual JPEG output to `.jpg`, then add focused regression tests for both behaviors.
