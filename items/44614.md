## Summary

Keep open. Current `main` does not implement this Windows detached gateway-update flow, and the PR still carries unresolved correctness risks around restart semantics and Windows supervision assumptions.

## Evidence

- **Current main still uses in-process gateway restart for update.run:** `update.run` writes the restart sentinel and, on success, always calls `scheduleGatewaySigusr1Restart(...)` before responding. There is no `detachedResultPath` handling or forced process exit path on current `main`.
  - file: [src/gateway/server-methods/update.ts:104](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/gateway/server-methods/update.ts#L104)
  - command: `nl -ba src/gateway/server-methods/update.ts | sed -n '1,220p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Current main still performs global update in-process:** `runGatewayUpdate` builds `globalInstallEnv`, runs `globalInstallArgs(...)` directly, and preserves the npm `--omit=optional` retry. There is no Windows-only detached helper branch on current `main`, and `src/infra/update-detached-win32.ts` is absent from the checkout.
  - file: [src/infra/update-runner.ts:1301](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/infra/update-runner.ts#L1301)
  - command: `nl -ba src/infra/update-runner.ts | sed -n '1288,1355p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Windows restart behavior still has a supervised-vs-fallback split:** Repo docs say native Windows prefers Scheduled Tasks but falls back to a Startup-folder login item when task creation is denied. Core respawn logic also explicitly disables detached win32 respawn when no Scheduled Task markers are present. That means a schtasks-only forced-exit update path is not obviously safe for every supported Windows install mode.
  - file: [src/infra/process-respawn.ts:30](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/infra/process-respawn.ts#L30)

The PR review context flagged that the detached helper path did not propagate that prepared env, which is a real correctness concern for this specific change rather than evidence the PR is obsolete.
