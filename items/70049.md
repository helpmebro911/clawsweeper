## Summary

Current `main` does not implement a callable `memory_add` surface. `memory-core` registers only the read/search tools, and the existing write paths are pre-compaction flush plus CLI/dreaming promotion flows. The provided issue context also points to PR #70540 as still open, so there is no shipped fix on `main` to justify closure.

## Evidence
