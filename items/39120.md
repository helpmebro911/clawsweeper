## Summary

Keep #39120 open. Current main still restricts `bootstrap-extra-files` to recognized bootstrap basenames, has no first-class custom bootstrap-file config key, and docs/tests still describe and assert the allowlist. Related PR #39174 was closed unmerged, and #39635 is an overlapping narrower config request rather than a shipped or clearly canonical replacement.

## Best Possible Solution

Keep #39120 open for maintainer/product decision. A direct fix should either widen the bundled `bootstrap-extra-files` contract or add a typed first-class custom bootstrap-file config key, while preserving workspace realpath checks, per-file and total bootstrap budgets, deterministic ordering, and subagent/lightweight-context behavior. Coordinate with #39635 if maintainers choose the config route, then close one item only after the canonical design ships or one issue is explicitly marked as the tracker.
