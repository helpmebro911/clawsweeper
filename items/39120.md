## Summary

Keep #39120 open. Current main still rejects arbitrary workspace files for `bootstrap-extra-files`: the loader keeps a hardcoded recognized-basename allowlist, returns `invalid-bootstrap-filename` for anything else, and the public docs still document that only recognized bootstrap basenames are loaded. The linked PR #39174 directly attempted the requested change, but the provided GitHub context says it was closed unmerged, so it does not supersede or implement the issue.
