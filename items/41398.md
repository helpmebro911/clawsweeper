## Summary

Keep open. The PR is stale and needs rework for current main, but the underlying Google prefix-stripping work is not fully implemented on main: current code has partial duplicate-prefix cleanup for stored overrides, while Google helper/manifest, OpenAI-compatible transport params, and native PDF URL construction still leave plausible paths where a prefixed Google model id can reach a provider call.

## Best Possible Solution

Keep this PR open, but request a current-main rework: implement Google-owned prefix cleanup through the current `extensions/google` model-id/manifest/runtime seams, add regression coverage for resolved Google chat/OpenAI-compatible and native PDF paths, preserve nested model ids for non-Google providers, and remove the unrelated `.secrets.baseline` refresh.
