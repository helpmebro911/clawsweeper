## Summary

Keep this PR open. Current main still lacks Amazon Bedrock `service_tier` forwarding, and the PR remains the focused implementation candidate for the linked Bedrock feature request rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to keep the behavior owned by the Amazon Bedrock plugin, forward `serviceTier`/`service_tier` through the existing `extraParams` and `wrapStreamFn` seam, inject top-level `{ serviceTier: { type } }` only for Bedrock Converse stream requests, preserve existing payload fields, decide the public SDK export/baseline question, and run the relevant changed gates before merge.
