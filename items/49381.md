## Summary

Keep open. Current main still does not provide the stronger per-dispatch single-final guard requested by #49381, and the linked #59771 is still an open candidate fix rather than a merged/superseding resolution. The Feishu dispatcher still closes the active streaming card from `onError`, which matches #59771's stated root cause, and existing tests only cover exact duplicate final text while explicitly preserving distinct non-streaming final payloads.
