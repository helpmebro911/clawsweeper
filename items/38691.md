## Summary

Keep #38691 open. Current main still hard-disables the Android Voice Tab manual mic path when `SpeechRecognizer.isRecognitionAvailable(context)` is false, and there is still no generic configured OEM recognizer fallback. The PR has a concrete Xiaomi reproduction and a plausible Android API path, but it needs revision for the retry-loop review concern and the current source path.

## Best Possible Solution

Keep the PR open for revision. The best path is to rebase onto the current Android app package, add a generic configured OEM recognizer fallback for the manual Voice Tab mic path using `Settings.Secure.getString(..., "voice_recognition_service")`, `ComponentName.unflattenFromString`, and `SpeechRecognizer.createSpeechRecognizer(context, component)`, while preserving a clean disable/no-retry path when no backend is usable. Add focused Android/Robolectric coverage if the recognizer creation seam can be tested without brittle platform mocking.
