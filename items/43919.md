## Summary

Keep this PR open. Current main has shipped the HTTP_PROXY/HTTPS_PROXY EnvHttpProxyAgent bootstrap path, but it still intentionally excludes ALL_PROXY/all_proxy and SOCKS fallback behavior. That remaining scope is exactly what this PR attempts to add, so it is not obsolete. It should not merge as-is without rebase and security cleanup.

## Best Possible Solution

Keep this PR open for maintainer review, but require a rebase onto current main and a narrower implementation through the existing src/infra/net/proxy-env.ts and src/infra/net/undici-global-dispatcher.ts helpers. Add explicit tests for ALL_PROXY/all_proxy, socks4/4a/4h/5/5h normalization, lowercase-empty precedence, malformed and authenticated proxy URL redaction, NO_PROXY wildcard and loopback behavior, dispatcher replacement, and SSRF-preserving strict paths. If maintainers decide OpenClaw should intentionally keep Undici's HTTP_PROXY/HTTPS_PROXY-only semantics, close this only with that explicit product/security rationale and leave #43821 or a narrower follow-up as the canonical record.

## Risks / Open Questions

- Closing as implemented would drop the still-missing ALL_PROXY/all_proxy and SOCKS fallback behavior without an explicit maintainer decision to reject that scope.
- Merging as-is is risky because it changes process-wide globalThis.fetch routing and overlaps current centralized undici proxy/bootstrap helpers.
- The PR’s SOCKS-to-HTTP rewrite assumes local proxy tools accept HTTP CONNECT on the same listener; SOCKS-only endpoints may fail differently.
- Review comments flagged possible credential leakage when logging authenticated proxy URLs; proxy URL userinfo needs redaction before merge.
- Any ALL_PROXY support must preserve current SSRF, NO_PROXY, lowercase precedence, and loopback-direct contracts.
