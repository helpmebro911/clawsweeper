## Summary

Keep #9865 open. Current main still lacks first-class provider Batch API routing for background LLM turns in cron, heartbeat, or subagent execution. Existing batch support is scoped to memory embeddings and media-style work, and the current provider/plugin seams are StreamFn-based rather than a durable submit/poll/reconcile batch-turn contract. #9865 also appears to be the canonical tracker, with related #49326 already proposed as a duplicate of it.

## Best Possible Solution

Keep #9865 open as the canonical product/API tracker. The likely path is a maintainer-approved provider/runtime batch-turn contract with submit, poll, cancel, reconcile, and delivery semantics, then explicit config/tool surfaces for cron jobs, heartbeat runs, and sessions_spawn/subagent work, plus docs and regression coverage. Until that contract exists, this should not be closed as implemented or ClawHub-only.
