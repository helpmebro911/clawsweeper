## Summary

Keep #9865 open. Current main still lacks first-class OpenAI/Anthropic Batch API routing for delayed background LLM turns; the batch surfaces present today are embeddings, media/STT, message coalescing, or config batching rather than provider batch submit/poll/cancel/result handling for cron, heartbeat, sessions.send, or sessions_spawn/subagent runs.

## Best Possible Solution

Keep #9865 open as the canonical product/API tracker. The best path is an additive provider/runtime batch-turn contract with submit, poll, cancel, timeout, failure, usage, and transcript reconciliation semantics, followed by explicit opt-in fields for cron agentTurn jobs, heartbeat/background runs, sessions.send, and sessions_spawn/subagents, with OpenAI/Anthropic plugin implementations plus docs and focused regression coverage.
