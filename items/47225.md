## Summary

Keep PR #47225 open. Current main has related prerequisite plumbing (`availableTools` names reach context-engine `assemble()`), but it still does not implement the PR's central behavior: an `AssembleResult` return field that filters the tool schemas sent to the model for a given turn. The PR discussion also identifies unresolved contract details, especially whether an explicit empty override should suppress all schemas.

## Best Possible Solution

Keep this PR open or replace it with a maintainer-owned follow-up that defines a context-engine schema-selection contract, forwards enough live tool information for engines to make that selection, honors `[]` explicitly if empty means no schemas, applies the projection consistently across runner surfaces, and adds focused helper/runner tests plus context-engine docs.
