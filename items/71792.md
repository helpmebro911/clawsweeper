## Summary

Keep #71792 open. Current main still rolls stale daily/idle sessions into a new session and emits plugin session lifecycle hooks, but it does not emit the internal `command:reset` event that the bundled `session-memory` hook requires. The PR remains a focused, active fix for a real current-main gap, and the related context does not show a landed canonical replacement.

## Best Possible Solution

Keep #71792 open and review it as the focused fix for auto daily/idle rollover memory persistence. The best path is to emit the existing internal `command:reset` event for non-command stale rollovers with `commandSource` distinguishing `auto:daily` and `auto:idle`, pass the resolved agent workspace, and keep the new daily/idle/fresh-session regression coverage unless maintainers intentionally redesign session reset lifecycle hooks more broadly.
