## Summary

Keep #71792 open. Current main still rolls stale daily/idle sessions into a new session and emits plugin session lifecycle hooks, but it does not emit the internal `command:reset` event that bundled `session-memory` listens for. The PR is active and focused, and the provided review context shows the earlier missing idle coverage was added; related #11960 remains open and #43533 was closed unmerged, so there is no strong close reason.

## Best Possible Solution

Keep #71792 open and review it as the focused fix for auto daily/idle rollover memory persistence. The likely best code path is to emit the existing internal `command:reset` event for non-command stale rollovers with `commandSource` distinguishing `auto:daily` and `auto:idle`, while preserving the existing `session-memory` command-event contract unless maintainers choose a broader lifecycle-hook redesign.
