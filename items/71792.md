## Summary

Keep #71792 open. Current main still rolls stale daily/idle sessions into a new session and emits plugin lifecycle hooks, but it does not emit the internal `command:reset` event that bundled `session-memory` listens for, so the PR still addresses a real memory persistence gap.

## Best Possible Solution

Keep #71792 open and review it as the focused fix for daily/idle rollover memory persistence. The likely best path is to emit the existing internal `command:reset` event for non-command stale rollovers with `commandSource` values that distinguish `auto:daily` and `auto:idle`, while preserving the existing bundled `session-memory` command-event contract unless maintainers choose a broader reset lifecycle hook redesign.
