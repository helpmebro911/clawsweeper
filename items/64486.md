## Summary

Keep this issue open. Current main still implements an after-the-fact unscheduled-reminder note, while the requested pre-send forced choice or revision gate is not implemented.

## Best Possible Solution

Keep this issue open. The best implementation is a focused pre-send gate in the agent reply/finalization pipeline that reuses the existing commitment detector, `successfulCronAdds`, and session-cron coverage check, then requests one revision before dispatch: either remove unsupported commitment language or create a concrete scheduled action. If maintainers prefer this as plugin policy, first expose documented per-turn scheduling/task/heartbeat evidence on the affected reply path.
