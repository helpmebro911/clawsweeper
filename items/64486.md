## Summary

Keep open. Current `main` still implements this as a post-send disclaimer, not a pre-send gate, and I found no shipped config or alternate path that enforces the requested behavior.

## Evidence

- **Core path still appends a note after reply generation:** `runReplyAgent` checks for reminder-commitment text after reply payloads already exist, then swaps in `appendUnscheduledReminderNote(...)` when no `cron.add` succeeded and no existing session cron covers it.

  - sha: [e68b2269b999](https://github.com/openclaw/openclaw/commit/e68b2269b99949851a9ebce30a040f8bf2c9be13)
