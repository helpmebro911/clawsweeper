## Summary

Keep PR #40314 open. Current main still emits `message:received` and `message:sent` internal hooks without draining `event.messages`, and the PR is an unmerged focused fix for open bug #39688. A newer open PR #41375 looks like a possible successor for the replyable-surface part, but it is also unmerged and intentionally scopes out `message:sent`, so this item is not safe to auto-close as implemented or clearly superseded.

## Best Possible Solution

Keep this PR or an explicit successor open until #39688 is resolved. The maintainer path should consolidate #40314 with #41375, decide the intended `message:sent` reply contract, drain internal hook messages only where there is a safe reply target, preserve session context, add loop-suppression or queue-persistence if outbound hook replies are emitted, and update regression tests plus the public hooks docs to match the final contract.
