## Summary

Keep open. Current main still lacks a `chat.send` message body byte cap, and this PR remains a focused gateway hardening candidate rather than stale cleanup. It needs revision for the byte-limit semantics called out in review before merge.

## Best Possible Solution

Keep this PR open as a useful gateway hardening candidate. Ask the author or a maintainer to revise it so the 512 KiB policy is defined consistently, ideally with clear raw-vs-normalized byte semantics, no misleading schema character-count guard, updated protocol docs, and focused ASCII boundary plus multi-byte tests; then run the relevant changed gateway/protocol checks and merge if green.
