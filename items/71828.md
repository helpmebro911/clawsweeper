## Summary

Keep PR #71828 open. Current main still refreshes the run-side typing TTL only when `startTypingLoop()`/text-entry paths are invoked, while keepalive ticks call `triggerTyping()` without extending that controller TTL. The PR's one-line production change and regression test still target a real gap, and the related broader presence work is not landed on main.

## Best Possible Solution

Keep PR #71828 open for maintainer review. The best path is to land this targeted controller TTL refresh and regression test, or explicitly fold the same fix into the broader #65919 active-run presence work while preserving cleanup after run completion, dispatch idle, and sealed-controller behavior.
