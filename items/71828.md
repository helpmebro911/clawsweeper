## Summary

Keep PR #71828 open. Current main still has the run-side typing TTL gap this PR targets: keepalive ticks invoke the typing callback, but `triggerTyping()` does not refresh `createTypingController`'s own TTL, so the shared controller can clean up and stop Discord typing during an otherwise active long run. The broader #65919 issue remains open and #66365 is closed unmerged, so this small PR is not obsolete or safely superseded.

## Best Possible Solution

Keep PR #71828 open for maintainer review. The best path is to land the targeted keepalive TTL refresh with its regression test, or explicitly fold the same behavior into the broader #65919 active-run presence work. The implementation should preserve cleanup after run completion and dispatch idle, and should not extend typing after the controller is sealed.
