## Summary

Keep PR #71828 open. Current main still has the high-level typing TTL expiry that this PR targets: keepalive ticks call the channel typing callback, but they do not refresh `createTypingController`'s own TTL, so the controller can clean up and stop Discord typing while a long run is still active. The PR's one-line production change and regression test directly address that gap, and the related #65919/#66365 context supports that this area still has unresolved long-run presence behavior rather than being obsolete on main.

## Best Possible Solution

Keep PR #71828 open for maintainer review. The best path is to land this targeted controller TTL refresh plus the regression test, or fold the same fix into a broader #65919 active-run presence lease if maintainers want one consolidated implementation. The fix should preserve cleanup after run completion plus dispatch idle and avoid extending typing after the controller is sealed.
