## Summary

Keep this PR open. Current main still has the DuckDuckGo onboarding metadata gap: the setup flow filters web-search providers to entries with `onboardingScopes` containing `"text-inference"`, while DuckDuckGo's shared provider metadata still omits that field and both runtime and contract exports inherit the shared base. The PR is also paired with the same author's open bug report #65862, so conservative cleanup should not close only this PR; it should be reviewed or rebased against the current shared metadata shape.

## Best Possible Solution

Review or rebase this PR together with #65862. On current main, the minimal fix should add `onboardingScopes: ["text-inference"]` to DuckDuckGo's shared web-search provider metadata, preserving both runtime and contract/public-artifact paths, and add or update a focused contract/setup assertion so DuckDuckGo remains visible in the web-search setup picker.
