## Summary

Keep this PR open. Current main still filters setup-visible web-search providers by `onboardingScopes`, DuckDuckGo's current shared provider metadata still omits that field, and the same author's linked bug #65862 remains open. The PR likely needs a rebase/update for the current shared metadata shape, but the underlying fix is still valid.

## Best Possible Solution

Review or rebase this PR together with #65862. The current-main fix should add `onboardingScopes: ["text-inference"]` to DuckDuckGo's shared web-search provider metadata so both runtime and contract paths stay aligned, then add a focused DuckDuckGo provider/contract/setup assertion to prevent the wizard visibility regression from returning.
