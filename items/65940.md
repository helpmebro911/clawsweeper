## Summary

Keep this PR open. Current main still filters setup web-search providers by `onboardingScopes`, while DuckDuckGo's shared provider metadata still omits that field, so the setup wizard bug remains. The PR is also paired with the same author's still-open bug report #65862; the related alternate PR #70170 is closed unmerged, so there is no merged superseding fix.

## Best Possible Solution

Review or rebase this PR together with #65862. On current main, the narrow fix should add `onboardingScopes: ["text-inference"]` to DuckDuckGo's shared web-search provider metadata in `extensions/duckduckgo/src/ddg-search-provider.shared.ts`, preserving both runtime and public contract paths, and add or update a focused DuckDuckGo provider/contract/setup assertion so the wizard visibility does not regress.
