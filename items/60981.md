## Summary

Keep this PR open. Current main still implements only the binary `tools.fs.workspaceOnly` guard, while this PR is an active, security-sensitive implementation candidate for PathGuard-style `allowedPaths` / `denyPaths`, canonical path policy, media/apply_patch parity, and spawned-subagent filesystem ceilings.

## Best Possible Solution

Keep this PR open for maintainer handling. The best path is to request a current-main rebase and focused security/design review, then either land a narrowed core implementation under `tools.fs` or replace it with maintainer-owned work that adds canonicalized deny-wins `allowedPaths` / `denyPaths`, structured policy denials, file/media/apply_patch/prompt-image coverage, schema/docs updates, and monotonic spawned-subagent filesystem ceilings. Coordinate the remaining filesystem scope with #7722 and broader security-profile work in #8719.
