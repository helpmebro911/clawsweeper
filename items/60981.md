## Summary

Keep this PR open. Current main still only exposes a binary `tools.fs.workspaceOnly` filesystem guard and has not shipped the requested PathGuard ACL layer with canonicalized `allowedPaths` / `denyPaths`, structured policy denials, apply_patch/media parity, or persisted spawned-subagent filesystem ceilings. The PR is a plausible core security implementation candidate, but it remains large, security-sensitive, and has concrete unresolved review concerns that need maintainer review or a narrower replacement implementation.

## Best Possible Solution

Keep this PR open for maintainer handling. The best path is to request a current-main rebase and a focused security/design pass, then either land a narrowed core implementation under `tools.fs` or replace it with maintainer-owned work that adds canonicalized deny-wins `allowedPaths` / `denyPaths`, structured policy denials, file/media/apply_patch/prompt-image coverage, schema/docs updates, and monotonic spawned-subagent filesystem ceilings. Coordinate scope with #7722 and the broader security profile in #8719.
