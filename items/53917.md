## Summary

Keep open. Current main still has the Feishu reply-media gap described here: reply attachments flow through createFeishuReplyDispatcher, but that dispatcher neither accepts nor forwards mediaLocalRoots into sendMediaFeishu. Direct Feishu outbound/action sends have partial local-root plumbing, so this is a narrow remaining reply-dispatcher bug rather than cleanup material.

## Best Possible Solution

Keep this issue open and fix it in Feishu-owned code. The best path is to thread agent-scoped outbound media roots through both Feishu bot reply-dispatcher call sites, add a typed mediaLocalRoots or media-access field to CreateFeishuReplyDispatcherParams, pass it through sendMediaReplies into sendMediaFeishu, and add focused reply-dispatcher regression coverage for local reply attachments. Coordinate with #49082, #51375, #58591, and #39523 so the reply-path fix and any Feishu channel/account config surface have clear semantics.
