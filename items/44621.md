## Summary

Keep open. Current main still strips mixed-message `/model` text but clears the directive state before model selection, so `/model <provider/model> <message>` is not applied as a one-shot runtime override. The PR addresses a real remaining core behavior gap, but the submitted patch has unresolved review feedback around stored session overrides and auth-profile handling, so it needs revision rather than conservative closure.

## Best Possible Solution

Keep PR #44621 open. The product/code direction is to implement mixed-message `/model <provider/model> <message>` as a non-persistent per-message override in the core reply directive pipeline, add regression coverage for prior session model/auth-profile overrides, and address the existing review comments before maintainer merge consideration.
