## Summary

Keep open. Current `main` does not implement this PR’s one-shot mixed-message `/model` behavior: mixed-message directives are still stripped and then cleared before reply execution, there is no per-message model/auth-profile handoff in the reply pipeline, and the docs/tests on `main` do not show the requested semantics as shipped.

## Evidence

- **Mixed-message `/model` is discarded before execution:** `resolveReplyDirectives()` still clears inline directives when the stripped message has remaining user text. That path calls `clearInlineDirectives(parsedDirectives.cleaned)`, which removes `hasModelDirective` and `rawModelDirective`, so `/model <provider/model> <message>` does not survive into reply execution on current `main`.
