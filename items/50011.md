## Summary

Keep open. Current main still has `gateway.nodes.denyCommands` typed and schema-validated only as `string[]`, while the custom invalid-command detection remains in `security audit`, not in `openclaw config set`. The linked PRs in the provided timeline all attempt to close #50011 but are still open/unmerged, so there is no landed canonical fix on main.
