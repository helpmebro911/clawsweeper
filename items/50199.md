## Summary

Keep open. Current main has deterministic skill source precedence and per-agent allowlists, but it does not implement user-configurable skill priority, fallback ordering, a `~/.openclaw/skills-config/priority.json` contract, or `openclaw skills priority` commands. The issue is coherent and the follow-up comments clarified the intended overlap cases. This is not a ClawHub-only request because it targets core skill loading/selection policy rather than adding a specific optional skill.
