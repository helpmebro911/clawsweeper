## Summary

Keep this PR open. Current main still has the reported SDK seam gap: channel agent-tool factories are typed and called with cfg only, while adjacent tool paths already receive trusted requester sender context. The PR remains a narrow additive implementation candidate for the closed linked issue #58806, ideally with a focused regression test added before landing.

## Best Possible Solution

Keep this PR open or land an equivalent core SDK change. The best path is to extend ChannelAgentToolFactory params with optional requesterSenderId and senderIsOwner, thread options.senderId/options.senderIsOwner through listChannelAgentTools, and add a narrow regression test around a fake channel plugin agentTools factory.
