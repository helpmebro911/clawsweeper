## Summary

Keep this PR open. Current main still has the reported plugin SDK seam gap: channel agent-tool factories receive only config, while the adjacent core/message-tool path already receives trusted sender context. The PR is a narrow additive candidate for channel plugins such as DMWork to implement sender-aware access control, and it is not obsolete, implemented, or ClawHub-only.

## Best Possible Solution

Review and land this PR, or an equivalent core SDK change: extend `ChannelAgentToolFactory` and `listChannelAgentTools` params with optional `requesterSenderId` and `senderIsOwner`, thread `options.senderId` and `options.senderIsOwner` from `createOpenClawCodingTools`, and add a narrow regression test around a fake channel plugin `agentTools` factory. This belongs in core because third-party channel plugins need a generic trusted sender-context seam.
