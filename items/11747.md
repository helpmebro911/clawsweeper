## Summary

Keep #11747 open. Current main still exposes passive reaction notification modes and the Discord, Telegram, and Slack reaction paths enqueue system events for later prompt context rather than waking or dispatching an immediate agent turn. Related #38714 is a narrower open Discord plugin-hook request, while closed #60805 documents a blocked plugin workaround and does not ship the requested cross-channel reaction wake behavior.

## Best Possible Solution

Keep #11747 open for maintainer product/API design. The best path is a documented cross-channel reaction wake option, such as reactionAutoRespond or reactionTriggersTurn, that preserves existing passive system-event behavior while allowing authorized reactions to trigger an immediate routed agent turn, optionally scoped by emoji/policy. If maintainers prefer plugin-first automation, core still needs a reaction event hook and/or safe wake/dispatch API before this can move cleanly to ClawHub; #38714 can cover the narrower Discord hook piece.
