## Summary

Keep #11747 open. Current main still implements Discord, Telegram, and Slack reactions as passive reaction notification system events, with no reactionAutoRespond/reactionTriggersTurn-style config, no immediate wake/agent dispatch in the reaction paths, and no public reaction hook for plugins. Related #38714 is a narrower open Discord hook request; closed #60805 and merged #34045 do not supersede the cross-channel reaction-wakes-agent request.

## Best Possible Solution

Keep #11747 open for maintainer product/API design. The best path is a documented cross-channel reaction wake option, such as reactionAutoRespond or reactionTriggersTurn, that preserves existing passive system-event behavior while allowing authorized reactions to trigger an immediate routed agent turn, optionally scoped by emoji/policy. If maintainers prefer plugin-first automation, core still needs a reaction event hook and/or safe wake/dispatch API before this can move cleanly to ClawHub; #38714 can track the narrower Discord hook piece.
