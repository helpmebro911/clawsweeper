## Summary

Keep #11747 open. Current main still receives Discord, Telegram, and Slack reactions as routed session system events, but there is no reactionAutoRespond/reactionTriggersTurn-style config and no channel handler path that wakes an immediate agent turn. The related Slack PR #34045 only fixed routing of Slack system events, and related #52658 tracks plugin event forwarding rather than the cross-channel agent wake behavior requested here.

## Best Possible Solution

Keep #11747 open for a maintainer-chosen cross-channel reaction wake contract. A good product path is a documented option such as reactionAutoRespond or reactionTriggersTurn that promotes authorized reactions into an immediate routed agent turn for Discord, Telegram, and Slack, with tests preserving existing system-event behavior. If maintainers prefer plugin-first automation, the core work should be a reaction event and wake API, with #52658 or a broader canonical issue tracking that plugin surface.
