## Summary

Keep this PR open. Current main still lacks the MiniMax manifest auth aliases that the PR adds, while MiniMax runtime still exposes the CN hook aliases and current auth resolution depends on manifest-backed providerAuthAliases before resolving env/config/profile credentials. It is also a same-author implementation PR paired with open #63823 and uses closing intent, so the maintainer path is to review and land this patch or apply an equivalent fix.

## Best Possible Solution

Review and land this PR, or apply an equivalent maintainer patch that adds providerAuthAliases for minimax-cn and minimax-portal-cn in extensions/minimax/openclaw.plugin.json plus focused manifest regression coverage in extensions/minimax/index.test.ts. After the fix merges, close #63823 as implemented.
