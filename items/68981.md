## Summary

Keep #68981 open. Current main has meaningful partial coverage: session rows carry label/displayName/derived-title/status data, Control UI can edit labels, the chat session selector uses labels/display names, and `sessions_spawn cleanup: "delete"` is implemented/documented. The broader requested UX is still not complete: there is no explicit `/session rename` or model-generated auto-title workflow, session keys are not renamed, Control UI filtering is limited to active/limit/global/unknown plus text search rather than channel/kind/status dropdowns or channel/kind grouping, and the archive timer behavior remains best-effort with related #41530 still open.

## Best Possible Solution

Keep the issue open, but narrow the remaining product work: preserve existing label/displayName support and `cleanup: "delete"`; add a user-facing session rename/auto-title workflow if desired; add Control UI channel/kind/status filters, grouping, and status indicators backed by the session list contract; and track the `archiveAfterMinutes` reliability/default behavior through #41530 or an explicitly linked subtask.
