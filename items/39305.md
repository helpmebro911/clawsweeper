## Summary

Keep open. Current main still only exposes wall-clock `runTimeoutSeconds` for sub-agent runs, not the requested tool-call inactivity nudge/kill recovery. The closed PR #43017 is useful prior work but was not merged, and the open related issues #39127/#39141 track broader activity/watchdog infrastructure rather than fully superseding this specific request.

## Evidence

- **Sub-agent config lacks stall recovery fields:** The strict `agents.defaults.subagents` schema accepts existing defaults such as `runTimeoutSeconds`, `announceTimeoutMs`, and `requireAgentId`; it does not define `stallNudgeSeconds`, `stallTimeoutSeconds`, or equivalent nudge/kill settings.
