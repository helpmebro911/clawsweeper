## Summary

Keep open. Current main still lacks the requested activity-based sub-agent stall recovery: no stall config fields, no sessions_spawn overrides, no per-run last-tool-call/nudge state, and no nudge-then-kill monitor. Related PR #43017 was closed unmerged, and #39141 is broader watchdog/alert infrastructure rather than this concrete recovery behavior.

## Best Possible Solution

Keep #39305 open until maintainers either implement activity-based sub-agent stall recovery or explicitly consolidate it into a canonical session activity/watchdog design. A likely implementation should add strict config and sessions_spawn override fields, track deterministic child tool/run activity, inject an auditable nudge into the child session, terminate through the existing subagent kill/cancellation path after the kill threshold, and cover config/runtime/docs with focused tests.
