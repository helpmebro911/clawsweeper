## Summary

Keep this draft PR open. Current main has backend-based sandboxing and exec approval hardening, but it does not implement the PR's macOS sandbox-exec wrapper or sandboxProfile exec parameter. The PR is also paired with same-author open feature issue #58730 and is security-sensitive enough to require maintainer product/security review rather than conservative cleanup.

## Best Possible Solution

Keep this PR open for maintainer review alongside #58730. If OpenClaw wants native macOS process isolation, design it as a sandbox backend or backend capability behind the existing agents.defaults.sandbox / SandboxBackendHandle.buildExecSpec model, with config docs and regression coverage. If maintainers decide macOS Seatbelt support should not live in core, close this PR and the paired issue together after that explicit decision.
