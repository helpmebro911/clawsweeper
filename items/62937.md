## Risks / Open Questions

- The PR changes a security boundary for host process execution and should not be auto-closed or merged without explicit maintainer product/security review.
- The patch appears stale against the current backend-oriented sandbox architecture; if accepted, it should likely be redesigned as a sandbox backend or backend capability instead of a one-off exec parameter.
- The PR body notes `sandbox-exec` is deprecated on macOS, and the proposed `permissive` profile allows broader access, so policy semantics need careful review against OpenClaw's sandbox and approval contracts.
- The provided tests focus on profile-string generation and platform accessibility; they do not prove real macOS process isolation, failure behavior, docs/config alignment, or interaction with the existing approval model.
