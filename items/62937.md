## Summary

Keep open. Current main has optional backend-based sandboxing and exec approvals, but it does not implement this PR's macOS `sandbox-exec` wrapper or `sandboxProfile` exec parameter. The PR is also an open draft explicitly tied to open same-author issue #58730, so conservative cleanup should leave it for maintainer review rather than closing one half of the pair. citeturn0view0

## Best Possible Solution

Keep this PR open for maintainer review alongside #58730. If OpenClaw wants native macOS process isolation, redesign it as a sandbox backend or backend capability behind the existing `agents.defaults.sandbox` / `BashSandboxConfig.buildExecSpec` model, with docs, config coverage, and regression tests. If maintainers decide macOS Seatbelt support should live outside core, close both the PR and the paired issue explicitly after that product decision.
