## Summary

Keep #10957 open. Current main has Android companion-node functionality, pairing documentation, capability coverage, Play/third-party flavor split, and signed AAB build mechanics, but it still explicitly says the Android app is not publicly released and does not document Android beta access, tester enrollment, or a public distribution path for an existing-Gateway phone node. The existing issue discussion also already kept this open for the same repo-side gap.

## Best Possible Solution

Keep #10957 open until maintainers either provide and document an Android beta/public distribution path for existing-Gateway Android phone nodes, or explicitly route Android beta-access requests to a support/auto-response process with clear public instructions. Continue Android node capability, flavor split, signed bundle, and release-hardening work separately in the Android app and release docs.
