## Summary

Keep #10957 open. Current main implements the Android companion-node topology, pairing docs, capability registry, tests, Play/third-party flavor split, and signed AAB build mechanics, but it still does not provide or document Android beta/public distribution or tester access for an existing-Gateway phone node. The issue discussion also already contains an April 26 Codex review posted by steipete that kept this open for the same remaining access/documentation gap.

## Best Possible Solution

Keep #10957 open until maintainers either publish and document an Android beta/public distribution path for existing-Gateway Android phone nodes, or explicitly route Android beta-access requests to a support/auto-response process with clear public instructions. Continue Android capability, flavor split, signed bundle, and release-hardening work in the Android app/release surfaces separately.
