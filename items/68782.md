## Summary

Keep this issue open. Current main has shipped important partial work: bundled plugin runtime dependencies are no longer eagerly installed by default, and startup/doctor repairs are scoped to active/default-enabled bundled plugins. But the central request remains broader than that: install-time selection for bundled plugins/channels/skills, a strict runtime discovery mode, and setup UX for selecting channel/plugin sets are not implemented on current main.

## Best Possible Solution

Keep the issue open, but narrow it into concrete maintainer-owned work: define whether OpenClaw should ship a slim core package plus ClawHub/on-demand bundled plugin installs, add a config/runtime `strict` discovery mode only if needed, and separately track setup/channel selection UX. Treat the current runtime-deps scoping and the closed #71818 loop fix as partial progress, not completion.
