## Summary

Keep #68782 open. Current main substantially reduces the reported pain through cold plugin registry work, allowlist-gated runtime loading, non-eager bundled runtime dependency installs, and skill filtering/on-demand skill reads. But the central request for true selective installation is not fully implemented: the npm package still ships the full `dist/` payload and bundled `skills/`, optional bundled clusters remain a build-time/default-included choice, and there is no documented install/setup surface like `setup --channels` or `plugins.strictMode` that skips non-selected bundled artifacts. The related #71818 runtime-deps loop appears addressed by current runtime-deps verification, but it is only a narrower bug fix and does not supersede this broader product request.

## Best Possible Solution

Keep the issue open and narrow it to the remaining product decision: whether OpenClaw wants true install-time selection for bundled plugins/channels/skills, or whether the current cold-registry plus on-demand runtime-deps model is the intended endpoint. If implemented, the likely path is a documented setup/install surface that persists selected plugin/channel IDs, aligns `plugins.allow`/plugin index/doctor behavior, skips or externalizes non-selected bundled artifacts and dependencies, and defines how bundled skills move from `skills.allowBundled` filtering to actual selective install/update semantics. Treat #71818 as a fixed related runtime-deps bug, not the canonical tracker for this broader request.
