## Summary

PowerMem is a reasonable memory backend idea, but current main explicitly routes optional memory backends through the memory plugin slot and community plugin/ClawHub distribution rather than OpenClaw core.

## Evidence

## Close Comment

Closing from a ClawSweeper/Codex review. This looks like a useful integration, but on current `main` it fits the existing memory-plugin/ClawHub path rather than OpenClaw core. `VISION.md:52`-`64` says optional capability should usually ship as plugins, be hosted in the maintainer's own repo, and notes that memory is already a dedicated plugin slot. The current plugin contract already supports alternate memory backends through `plugins.slots.memory` and `api.registerMemoryCapability(...)` (`src/config/types.plugins.ts:24`-`29`, `docs/plugins/sdk-overview.md:173`-`195`, `docs/plugins/manifest.md:676`), and the community-plugin docs send this class of integration to ClawHub/npm (`docs/plugins/community.md:9`-`22`). There is also already an install-on-demand memory backend precedent in `memory-lancedb` (`docs/tools/plugin.md:109`-`112`, `extensions/memory-lancedb/package.json:14`-`32`). The issue thread now points to a separate `openclaw-extension-powermem` repo, which is the right shape for this feature. If PowerMem needs a missing SDK seam, please open a focused issue for that contract gap instead.
