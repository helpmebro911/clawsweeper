## Summary

Keep this PR open. Current main still has the linked `/compact` cancellation gap: the manual compact command calls `compactEmbeddedPiSession` without an abort signal and without registering the compaction in the active-run registry. The PR is a relevant implementation candidate, but it needs maintainer review because it bundles unrelated launchd/profile behavior and lacks focused cancellation regression coverage.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #66535. The best path is to revise/review it so queued manual compaction registers an abortable active-run handle when the task actually starts, forwards an abort signal into compaction, cascades abort to any active reply operation, cleans up in `finally`, adds focused regression coverage, and removes or splits the unrelated launchd/profile change.
