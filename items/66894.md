## Summary

Keep open. Current `main` still does not wire `/compact` into the embedded-run abort path, so the PR is addressing a real gap rather than duplicating an implementation already on `main`. The review discussion also surfaced real edge cases that the contributor iterated on, which makes this an active, useful PR rather than stale cleanup material.

## Evidence

- **`/compact` does not pass an abort signal on current `main`:** `handleCompactCommand` aborts any prior active run and then calls `runtime.compactEmbeddedPiSession(...)`, but it does not provide the existing optional `abortSignal` field or any replacement cancellation hook to that compaction call.
  - file: [src/auto-reply/reply/commands-compact.ts:100](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/auto-reply/reply/commands-compact.ts#L100)
  - command: `nl -ba src/auto-reply/reply/commands-compact.ts | sed -n '88,150p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Compaction API already supports cancellation, but the command path is not using it:** `CompactEmbeddedPiSessionParams` includes `abortSignal?: AbortSignal`, and the compaction implementation ultimately passes `params.abortSignal` into `compactWithSafetyTimeout(...)`, which means the missing wiring is specifically at the `/compact` command boundary.
  - file: [src/agents/pi-embedded-runner/compact.types.ts:70](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/pi-embedded-runner/compact.types.ts#L70)
  - command: `nl -ba src/agents/pi-embedded-runner/compact.types.ts | sed -n '1,120p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Queued compaction is not registered in `ACTIVE_EMBEDDED_RUNS` on current `main`:** `compactEmbeddedPiSession` enqueues work on the session/global lanes but this file does not call `setActiveEmbeddedRun` or `clearActiveEmbeddedRun`. By contrast, normal embedded runs register their queue handle in `run/attempt.ts`, so compaction currently has no active-run handle for abort routing.
  - file: [src/agents/pi-embedded-runner/compact.queued.ts:99](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/pi-embedded-runner/compact.queued.ts#L99)
  - command: `nl -ba src/agents/pi-embedded-runner/compact.queued.ts | sed -n '1,160p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Abort surfaces rely on `abortEmbeddedPiRun`, which only sees registered active runs or reply operations:** `abortEmbeddedPiRun(sessionId)` first checks `ACTIVE_EMBEDDED_RUNS` and only falls back to `abortReplyRunBySessionId(sessionId)` when no embedded handle exists. `/stop`-style command handling calls into this path, so an unregistered compaction run is not conservatively proven cancellable on current `main`. The PR discussion usefully identified queue-registration and abort-cascade edge cases, and the contributor added follow-up commits for those cases rather than the PR becoming obsolete.
