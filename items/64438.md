## Summary

Keep open. Current main still has remote memory embedding endpoint support and local/QMD reranking paths, but no configurable remote reranker endpoint contract for builtin or QMD memory search. The prior ClawSweeper comment reached the same keep-open conclusion, and the newer main SHA still shows the same missing surface.

## Best Possible Solution

Keep this open for a focused memory-search design. The best path is an additive remote reranker contract covering config schema/help/docs, secret handling, SSRF/timeout behavior, a Cohere-compatible HTTP reranker, builtin vector-only fallback on failure, tests, and a QMD integration story. If maintainers want this outside core, first add a documented reranker provider API so provider-specific adapters can live in plugins without replacing the whole memory backend.
