## Summary

Keep open. Current `main` supports remote memory endpoints only for embeddings, while reranking is still documented and wired through QMD/local model paths rather than a remote reranker config. The issue is coherent, recent, and not already covered by a linked plugin or workaround in the discussion.

## Evidence

- **Remote memory config is embedding-only:** The public memory config docs define `memorySearch.remote` as a custom endpoint override for embedding providers, with keys `remote.baseUrl`, `remote.apiKey`, and `remote.headers`, and the example uses `provider: "openai"` with `model: "text-embedding-3-small"`. There is no reranker endpoint block.
