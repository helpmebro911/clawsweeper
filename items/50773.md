## Summary

Keep open. Current main still only skips heartbeat runs for the main lane and the resolved target session lane, and the wake retry loop still only treats `requests-in-flight` as retryable. I found no current-main `cron-in-progress`, `lanes-busy`, or `heartbeat.skipWhenBusy` implementation, while cron work is still queued on the separate `CommandLane.Cron` lane.

## Evidence

- **Heartbeat runner does not check cron lane:** `runHeartbeatOnce` checks `getQueueSize(CommandLane.Main)` and later the resolved session lane; there is no `CommandLane.Cron` check in the heartbeat skip path.
  - file: [src/infra/heartbeat-runner.ts:758](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/infra/heartbeat-runner.ts#L758)
