## Summary

Keep open. Current main still parses the model-supplied exec `security` argument and combines it with configured exec security via `minSecurity`, so the reported downgrade from configured `full` to requested `allowlist` is still present. The PR is a focused implementation candidate, but it should be reviewed and cleaned up before merge.

## Best Possible Solution

Keep this PR open and review it as a focused security/policy bug fix. The best landing shape is a minimal change in `src/agents/bash-tools.exec.ts` that makes configured/session exec security authoritative for model tool calls, preserves the explicit elevated-full override, adds regression coverage for configured `full` with model `allowlist` and configured `allowlist`/`deny` with model attempts to change policy, and removes or documents the now-ignored model-call `security` parameter plus any unused `maxSecurity` helper.

## Risks / Open Questions

- PR head still adds an exported `maxSecurity` helper and tests even though the final implementation no longer uses it.
