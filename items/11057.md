## Summary

Keep #11057 open. Current main still does not provide native Claude Max/browser-session 1M support for `anthropic/claude-opus-4-6`: OpenClaw only adds the 1M Anthropic beta when `params.context1m: true` is configured, then strips that beta for `sk-ant-oat` OAuth/subscription tokens and falls back to the standard window. The custom-provider workaround in the discussion is useful, but it does not close the built-in Anthropic provider request.

## Best Possible Solution

Keep this issue open until OpenClaw either implements native Claude Max/browser-session 1M support for Opus 4.6, documents a maintainer-approved decision that 1M remains API-key/custom-provider only, or supersedes this with a narrower canonical Anthropic OAuth long-context issue. Likely surfaces are `extensions/anthropic/stream-wrappers.ts`, `src/agents/context.ts`, model listing/status context reporting, Anthropic provider docs, and regression tests for OAuth and API-key behavior.
