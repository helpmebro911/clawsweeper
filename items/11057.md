## Summary

Keep #11057 open. Current main still does not provide native 1M Opus 4.6 support for Claude Max/browser-session auth: Opus/Sonnet 4.x 1M is config-gated by `params.context1m`, and OAuth/setup-token credentials strip the 1M beta header. Current main adds/defaults Opus 4.7 1M behavior, but that does not resolve the specific Opus 4.6 Claude Max request.

## Best Possible Solution

Keep this issue open until OpenClaw either implements native Claude Max/browser-session 1M support for Opus/Sonnet 4.6 in the built-in Anthropic provider, documents a maintainer-approved API-key/custom-provider-only decision, or links a narrower canonical follow-up. The likely path spans `extensions/anthropic/stream-wrappers.ts`, `src/agents/context.ts`, model listing/status behavior, Anthropic provider docs, and OAuth/API-key regression coverage.
