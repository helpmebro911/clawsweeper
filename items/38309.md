## Summary

Keep #38309 open. Current main still accepts only plain string values for `sandbox.docker.env`; SecretRef objects are rejected by schema/types and are not collected or materialized by the secrets runtime before Docker launch. The related PR #38310 is useful implementation context, but it is closed unmerged and does not supersede the issue.

## Best Possible Solution

Implement SecretRef support end-to-end for Docker sandbox env values: accept `SecretInputSchema` in `SandboxDockerSchema`, type `SandboxDockerSettings.env` as `Record<string, SecretInput>` while keeping resolved `SandboxDockerConfig.env` string-only, add secrets runtime collectors for defaults and per-agent Docker env entries, register the target paths for audit/plan/configure, regenerate schema/docs, and add focused schema/runtime/Docker-arg regression tests. PR #38310 remains useful design context but should not close the issue unless its equivalent changes land on main.
