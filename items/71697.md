## Summary

Keep #71697 open. Current main still supports direct Ollama Cloud native configuration, but the agent/Gateway native `api: "ollama"` path does not normalize replay history into strict user/assistant alternation before sending `/api/chat`. The existing replay hook still covers OpenAI-compatible routes only, and current tests explicitly expect no native Ollama replay policy.

## Best Possible Solution

Keep the issue open and fix the native Ollama provider path. The likely maintainer path is an Ollama-owned replay policy or converter-level normalization for `api: "ollama"` that merges or reshapes non-alternating user and assistant turns while preserving supported native Ollama tool-call behavior, plus regression tests for the agent/Gateway replay history that currently reaches `/api/chat` as non-alternating roles.
