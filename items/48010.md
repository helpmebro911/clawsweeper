## Summary

Keep open. Current main partially addresses the Ollama thinking-control subcase by mapping resolved OpenClaw thinking state to Ollama's top-level `think` request field, and users can set per-model `agents.defaults.models["ollama/<model>"].params.thinking` to influence that default. But the requested first-class per-model Ollama `parameters`/`options` passthrough is still missing from `models.providers.ollama.models[]`: the strict model schema does not accept `parameters` or `params`, and the native Ollama stream builder only constructs `options` from `contextWindow`, runtime `temperature`, and runtime `maxTokens`, not arbitrary model/provider options such as `top_p`. The directly related implementation PR #48023 was closed unmerged, while #69464 remains open and is not proof of mainline implementation.
