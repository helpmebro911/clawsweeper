## Summary

Keep open. Current main partially addresses the report by carrying `runId` through `before_agent_start`/`agent_end` and adding a diagnostic trace carrier, but the central plugin-hook observability request is not implemented end to end: agent hook context still lacks sender/message correlation fields and top-level span hierarchy fields, `message_sending` still lacks correlation in its event and common outbound context, and subagent hooks still do not expose both parent/requester and child run ids. The related #70924 work is a useful foundation, but its own context says runtime hook/exporter wiring is follow-up work, so it does not supersede this issue.
