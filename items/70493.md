## Summary

Keep this issue open. Current main still has the Playwright-unavailable route for non-existing-session browser actions, the browser tool still preserves distinct sandbox/session routing for sub-agent contexts, and the available tests/docs I found cover tool policy or cleanup rather than the requested isolated-session navigate/snapshot/act regression. A maintainer comment in the issue explicitly says not to close until that isolated-session coverage exists.

## Best Possible Solution

Keep this issue open. The best fix is to make isolated sub-agent browser actions use a Playwright-capable runtime consistently, either by packaging/loading the browser plugin runtime in the isolated gateway or by explicitly delegating to the host/node browser owner when that is the intended architecture. Add a regression test that exercises browser navigate or snapshot through the real isolated sub-agent path and proves it does not return the Playwright-unavailable error when the main gateway or browser node supports Playwright.
