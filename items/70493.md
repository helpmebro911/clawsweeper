## Summary

Keep open. Current main has related browser runtime dependency diagnostics, node proxy plumbing, and docs for enabling the browser tool in subagents, but I did not find source or test evidence that isolated sub-agent browser calls no longer hit the Playwright-unavailable route. The maintainer comment also explicitly says not to close until isolated-session coverage exists.

## Best Possible Solution

Keep this open and fix it with an isolated-session regression test that runs browser snapshot or navigate through sessions_spawn/subagent execution when the main gateway/browser node has Playwright available. If the failure reproduces, route isolated browser calls explicitly to the host/node browser proxy or ensure the isolated runtime loads the same bundled browser plugin runtime dependencies before declaring the issue fixed.
