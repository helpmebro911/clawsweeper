## Summary

Keep #70493 open. Current main has browser target routing, sandbox bridge plumbing, node browser proxy support, and diagnostics for missing Playwright, but I did not find source or test evidence that the isolated sub-agent browser path is proven to support navigate/snapshot/act consistently when the main session browser works. A maintainer comment explicitly asked not to close until isolated-session coverage exists.

## Best Possible Solution

Keep the issue open until OpenClaw has a source-backed fix or regression test for the isolated sub-agent browser route, ideally exercising `browser({ action: "navigate" })` or `browser({ action: "snapshot" })` through the same isolated session path and asserting it does not return the Playwright-unavailable failure when the main gateway/browser node supports browser automation.
