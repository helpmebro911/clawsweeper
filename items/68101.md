## Summary

Current main still has the reported Lobster behavior: unresolved workflow-file-looking pipeline inputs are swallowed by workflow detection and then retried as inline pipelines. The issue also has an open unmerged PR with closing syntax, so it should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Review and land #68106, or an equivalent Lobster plugin change in `extensions/lobster/src/lobster-runner.ts`, that rethrows workflow resolution errors for workflow-file-like inputs, keeps normal inline pipeline parsing unchanged, and adds regression coverage for missing relative workflow files.

## Evidence

- **Current main still swallows workflow resolution errors:** `detectWorkflowFile()` trims the candidate, calls `resolveWorkflowFile()`, catches all errors from `stat()`/extension validation, and returns `null`. A missing relative `.lobster` path therefore does not surface as a workflow path error.
