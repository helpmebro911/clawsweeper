## Summary

Keep open. Current main already moved setup and regular `status` compatibility checks onto the snapshot path, but the specific `status --all` path targeted by this PR still calls the full diagnostics-backed compatibility notice builder. The PR also has an unresolved correctness concern around hook-derived compatibility notices on `loadModules: false` snapshots, so it should be narrowed or fixed rather than closed as shipped.

## Best Possible Solution

Keep the PR open, but require changes before merge: either implement a metadata-backed compatibility snapshot that faithfully reports `hook-only` and `before_agent_start` without loading full plugin runtimes, or keep `status --all` on diagnostics until that contract exists. The setup part is already shipped on main via `2f81c5f5`; the remaining useful work is a focused, well-tested status-all optimization or a narrower follow-up that addresses the snapshot fidelity gap.
