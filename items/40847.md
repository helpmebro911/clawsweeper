## Summary

Keep open. Current main has partial generic diagnostics for failed media-understanding decisions, but it does not cover the PR’s broader selected-image zero-output goal, and the PR itself still has review-identified logic gaps for no-provider and all-skipped paths. The diff is narrowly scoped to runtime logging and shows no obvious security or supply-chain-sensitive change.

## Best Possible Solution

Keep this PR open for maintainer review or contributor follow-up. The best implementation would amend src/media-understanding/runner.ts to use the existing logging path, remove or move the unreachable no-provider warning before the early return, decide whether all-skipped image attempts should warn, include attempted provider/model/reason details without leaking secrets, and add targeted Vitest coverage for failed, no-provider, and all-skipped selected-image zero-output cases.
