## Summary

Keep open. Current main still does not implement the PR’s central behavior: provider-shaped overflow counts like `requested: N`, MiniMax-style `prompt is too long: N, model maximum context length: M`, and additive `A + B > limit` messages are recognized as overflow in some classifiers but are not extracted as observed token counts for compaction. Current main also still omits `currentTokenCount` entirely when overflow is confirmed but the provider text has no parseable count, so the PR remains a plausible focused bug fix rather than obsolete or superseded.

## Best Possible Solution

Keep PR #70473 open. The best path is to review it as a focused agents bug fix: verify the new extraction patterns, keep the synthetic `ctxInfo.tokens + 1` fallback if it avoids confirmed-overflow compaction dead ends, retain the regression tests/docs, and handle the separate session-preservation behavior through #70479 or its successor.
