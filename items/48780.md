## Summary

Keep #48780 open. Current main still lacks the specific defensive normalization for `</arg_value>>` appended inside string-valued `exec`/`read` parameters. The linked PR #48792 is directly relevant and useful, but it is still open/unmerged, so the issue remains the canonical bug report for now.
