## Summary

Keep PR #47255 open. Current main still does not expose an `async` parameter on `sessions_send`; the existing fire-and-forget behavior is only available through `timeoutSeconds: 0`, which is now documented but does not implement the PR's requested schema/DX alias. The PR discussion also shows the one reviewer-identified conflict case was addressed in the PR head commit.

## Best Possible Solution

Keep the PR open for maintainer review or a refreshed patch. If accepted, land a current-main implementation that adds `async` to `SessionsSendToolSchema`, maps `async: true` to `timeoutSeconds: 0`, rejects conflicting non-zero `timeoutSeconds`, and adds focused schema/runtime tests plus any needed docs/tool-description updates.
