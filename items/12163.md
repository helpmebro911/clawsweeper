## Summary

Keep #12163 open. Current main already forwards OpenAI TTS speed from provider config and Talk/provider overrides into the OpenAI speech request, but the reported `[[tts:speed=1.5]]` directive path is still missing because the bundled OpenAI speech provider only parses voice/model directive tokens. The related PR attempts in the provided context, #12177 and #22618, were closed unmerged and do not supersede the issue.

## Best Possible Solution

Keep the issue open. Implement provider-owned OpenAI speed directive handling in `extensions/openai/speech-provider.ts`: handle `speed` only when `policy.allowVoiceSettings` permits it, parse a finite number, validate OpenAI's `0.25..4.0` range, return `{ speed }` as provider overrides, and add focused tests for preferred OpenAI `[[tts:speed=1.5]]`, explicit `[[tts:provider=openai speed=1.5]]`, disabled voice settings, invalid values, and the final OpenAI request body.
