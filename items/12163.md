## Summary

Keep #12163 open. Current main supports OpenAI TTS speed through config/Talk overrides and forwards speed to the OpenAI speech request, but the reported `[[tts:speed=1.5]]` directive path is still not handled by the bundled OpenAI speech provider.

## Best Possible Solution

Keep the issue open. Implement provider-owned OpenAI directive handling in `extensions/openai/speech-provider.ts`: add a `speed` case gated by `policy.allowVoiceSettings`, parse finite numeric values, validate `0.25..4.0`, return `{ speed }` provider overrides, and add focused tests proving the directive reaches the OpenAI request body without regressing existing config and Talk speed support.

## Evidence

- A future fix should validate OpenAI directive speed against OpenAI's `0.25..4.0` range rather than copying ElevenLabs' narrower `0.5..2.0` range.
