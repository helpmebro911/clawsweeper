## Summary

Keep #12163 open. Current main has partial OpenAI TTS speed plumbing for config/Talk overrides and the request body, but the reported per-reply directive path still cannot set `providerOverrides.openai.speed` because the OpenAI speech provider only parses voice/model directive tokens.

## Best Possible Solution

Implement the missing OpenAI directive handler in `extensions/openai/speech-provider.ts`: parse `speed` only when `allowVoiceSettings` is enabled, validate OpenAI's documented `0.25..4.0` range, return `{ speed }`, and add focused tests proving `[[tts:speed=1.5]]` and `[[tts:provider=openai speed=1.5]]` route into `providerOverrides.openai` and reach the OpenAI request body. Keep the existing config/Talk/request-body plumbing. citeturn0search0
