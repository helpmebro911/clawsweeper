## Summary

Keep PR #70589 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still routes the local `openclaw agent --local --json` envelope through `runtime.log(JSON.stringify(...))`, while JSON mode deliberately routes console logs to stderr. The PR is a focused fix for a real remaining bug, with regression coverage, and the related #62440 issue remains open; the earlier related PR #62453 is closed and unmerged in the provided context.

## Best Possible Solution

Keep #70589 open and review it as the active focused fix for #62440. The best code path is to import and use `writeRuntimeJson()` in `src/agents/command/delivery.ts` for local JSON envelopes, keep the envelope shape unchanged, and land the added regression test that asserts `deliverAgentCommandResult` calls `runtime.writeJson()` rather than `runtime.log()` when a structured runtime is available.
