## Summary

Keep PR #70589 open. Current main still has the reported local `openclaw agent --local --json` stdout/stderr bug: the local delivery path writes the JSON envelope via `runtime.log`, while JSON-mode console output is routed to stderr. The PR is a focused bug fix for open issue #62440 and uses the existing `writeRuntimeJson` stdout path already used by the gateway agent command.

## Best Possible Solution

Keep #70589 open for maintainer review and CI. The best code path is to land or adapt its minimal `writeRuntimeJson(runtime, buildOutboundResultEnvelope(...))` change in `src/agents/command/delivery.ts` with the focused regression test in `src/agents/command/delivery.test.ts`; if consolidating duplicates, prefer the version that uses the established helper and keeps the test organized around `deliverAgentCommandResult`.
