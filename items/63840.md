## Summary

Keep open. Current main still does not preserve explicit Slack assistant-DM root thread context for non-`all` reply modes, so the PR is not obsolete. The branch is directionally useful, but it still needs revision because it keys on raw `channel_type` and includes an unrelated agents test-harness edit.

## Best Possible Solution

Keep open and revise. Preserve Slack Agents & Assistants DM root thread context using normalized direct-message state, not raw event `channel_type`; keep non-DM self-thread roots from forcing `MessageThreadId`; prove the subagent completion path gets per-thread identity; add focused tests for raw `im`, D-prefix DMs with missing/wrong `channel_type`, non-DM self-thread roots, and concurrent assistant-DM roots; split the unrelated agents harness edit.
