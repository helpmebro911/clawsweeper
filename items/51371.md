## Summary

Keep open. Current `main` does not contain PR #51371's one-shot continuation guard or an equivalent fix: `WebSocketTaskBox.sendPing()` still resumes the same `CheckedContinuation` directly, `ThrowingContinuationSupport` is a plain single resume helper, and the keepalive loop still calls `task.sendPing()` on the live gateway path.
