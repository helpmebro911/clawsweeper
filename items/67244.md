## Summary

Keep open. Current main still has the reported source-level gaps: ACP backend lookup is still process-local/registry-only unless plugin services have been started in that process, and the explicit agent JSON path still returns the raw ACP/embedded result without a sessions_yield descendant-final reconciliation step or a completed liveness state.

## Best Possible Solution

Keep this issue open and patch source. Add a narrow, policy-aware ACP backend ensure/activation path before explicit embedded ACP availability checks and manager backend lookup, so a healthy bundled acpx backend can be registered in the current process without bypassing plugin policy. Separately, reconcile explicit openclaw agent --json results after sessions_yield by draining descendant ACP/subagent completion, refreshing final assistant text, clearing replayInvalid only after proven terminal completion, and defining/documenting the terminal liveness JSON contract. Add regression coverage for runtime: "acp", sessions_yield, no initial assistant-visible text, descendant completion, final JSON metadata cleanup, and the outside-workspace ACP write smoke described here.
