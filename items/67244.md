## Summary

Keep open. Current main still has both reported source-level gaps: ACP spawn/manager paths only consult the current process-local backend registry, while bundled acpx registration occurs when the plugin service starts; and embedded run metadata still has no completed liveness state, with normal terminal and sessions_yield paths resolving to working and being returned directly in agent --json output.

## Best Possible Solution

Keep this issue open and patch the source. Add a narrow ACP runtime ensure/registration seam before explicit embedded ACP availability checks and manager backend lookup, without hard-coding broader core policy beyond the bundled acpx registration path. Separately, reconcile explicit `agent --json` results after `sessions_yield`: wait for descendant ACP completion when metadata is still working or replay-invalid, populate final assistant text when available, clear stale `replayInvalid`, and define the terminal liveness contract with regression tests for `openclaw agent --json`, `runtime: "acp"`, `sessions_yield`, and descendant completion cleanup.
