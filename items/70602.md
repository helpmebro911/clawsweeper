## Summary

Keep PR #70602 open. Current main does not implement the heartbeat keep-owner-tools behavior, and the PR still addresses a real core-agent cache-alignment problem. However, the submitted patch is not clearly sufficient as written because the current main core tool path filters `cron`, `gateway`, and `nodes` inside `createOpenClawCodingTools()` before the bundled-tool-only `applyFinalEffectiveToolPolicy()` pass runs; this matches the existing review comment that heartbeat cache alignment is still not restored for core owner-only tools.

## Best Possible Solution

Keep the PR open for maintainer review/rework. The likely best path is to thread an explicit heartbeat run policy into the core `createOpenClawCodingTools()` owner-only filtering path so heartbeat runs keep the same core tool names as owner conversation turns while wrapping privileged execution with a guard. Add regression coverage comparing heartbeat vs conversation tool name arrays for core tools, and keep #70418 separate as the cache-warmer design discussion.
