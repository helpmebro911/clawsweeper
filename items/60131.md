## Summary

Keep open. Current `main` only prioritizes the declared TTS provider; it does not restrict parsing to that provider, does not expose a selected-provider field in the parse context, and still silently lets overlapping keys fall through to other providers.

## Evidence

- **Parser reprioritizes but still iterates every provider:** `parseTtsDirectives` scans `provider=` first, then calls `prioritizeProvider(...)`, but each non-provider token still walks all ordered providers until one returns `handled`. There is no warning when no provider matches.
  - file: [src/tts/directives.ts:93](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/tts/directives.ts#L93)
  - command: `nl -ba src/tts/directives.ts | sed -n '70,180p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Public parse context still lacks selected provider:** `SpeechDirectiveTokenParseContext` contains `key`, `value`, `policy`, `providerConfig`, and `currentOverrides`, but no `selectedProvider` field, so providers cannot see the explicit provider choice through the shared parse contract.
  - file: [src/tts/provider-types.ts:95](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/tts/provider-types.ts#L95)

  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)

  - file: [src/tts/directives.test.ts:126](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/tts/directives.test.ts#L126)

  - file: [extensions/openai/speech-provider.ts:165](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/openai/speech-provider.ts#L165)
