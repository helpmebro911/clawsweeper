## Summary

Current main still does not implement compaction-local same-provider auth-profile rotation/failover. Direct and delegated compaction still resolve a single effective target/profile and make one summarization call, while normal agent turns use a separate auth-controller candidate list and `advanceAuthProfile()` path. The discussion's idempotency concern remains relevant: a correct fix needs retries against one stable compaction snapshot with at-most-one committed summary/transcript update. The related title-search matches do not supersede this issue; they cover UI compaction, config backup rotation, or compaction logging rather than same-provider auth-profile failover.

## Best Possible Solution

Keep this issue open. The best fix is to share or extract auth-profile candidate ordering, cooldown skipping, failure classification, and failure marking for compaction; snapshot the transcript once; retry eligible same-provider profiles against that exact snapshot; and commit exactly one successful summary/checkpoint/transcript rotation. Add focused regression tests for manual compaction, preflight budget compaction, overflow-triggered compaction, timeout-triggered compaction, context-engine delegation, and the partial-failure idempotency case.

## Risks / Open Questions

- A naive retry loop could double-compact or apply a summary to a mutated transcript; the fix needs one captured compaction snapshot or attempt id and an at-most-one commit boundary.
- Manual `/compact`, preflight budget compaction, overflow recovery, timeout recovery, and context-engine-owned compaction enter through different paths, so a partial fix could leave the reported lockup in place on some surfaces.
