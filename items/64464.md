## Summary

Keep open. Current `main` still launches managed Chrome with inherited env plus `HOME` only, so the missing-`DISPLAY` WSL2 headed-launch path described in #64464 is still plausible. The discussion was useful: it linked troubleshooting docs and several candidate fix PRs, but no landed fix is present on `main` yet.
