## Summary

Keep open. Current main still treats Android as unsupported for gateway service installs, and the user-facing Android docs explicitly define Android as a companion node that does not host the Gateway. The AndroidForClaw comment is a useful external workaround, but it does not prove OpenClaw core/Termux gateway support is implemented or that the request can be cleanly closed as ClawHub/plugin work through a documented extension surface.

## Evidence

- **Issue context:** #48855 asks for running the OpenClaw gateway service directly on Android/Termux and reports the current error: `Gateway service install not supported on android`. The only comment links AndroidForClaw as an external full-stack Android implementation/workaround, not an OpenClaw core fix.
