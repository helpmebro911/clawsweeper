## Summary

Keep open. Current main has adjacent bundled-runtime-deps and restart hardening, but I do not see a source-backed fix that specifically covers the reported cold-cache browser staging failure plus service-manager shutdown/SIGKILL race. The related #72058/#72084 work tracks the broader v2026.4.24 runtime-deps regression and stale postinstall behavior, but it does not clearly supersede this narrower shutdown-race report.

## Best Possible Solution

Keep #72114 open as a narrower runtime-deps/update restart-race report, or explicitly link it under #72058 only after a maintainer confirms that the broad tracker owns this shutdown/SIGKILL failure mode. The likely product fix belongs in core: make bundled runtime-deps staging observable to restart/shutdown deferral, avoid service-manager restarts while a staging npm subprocess is active, and add regression coverage for cold-cache browser runtime-deps staging during an update-driven gateway restart. Also clarify the log wording so `failed after N ms` is not mistaken for a configured staging timeout.
