## Summary

Keep #68909 open. Current main still only probes the systemd user scope for gateway runtime and still classifies non-owned gateway listeners as stale without cgroup attribution, so the reported headless system-level systemd false positive is not implemented on main. The PR is a plausible focused bug fix, but the provided review context includes unresolved P1/P2 correctness concerns that should be addressed before merge rather than auto-closing the PR.

## Best Possible Solution

Keep the PR open and either have the author update it or let a maintainer extract the fix: add a system-bus runtime fallback for known gateway system units, keep node units as auxiliary context only, and add conservative cgroup-aware stale PID filtering limited to known safe OpenClaw sibling service combinations with regression tests for the reviewed edge cases.
