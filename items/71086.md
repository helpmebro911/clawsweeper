## Summary

Keep PR #71086 open. Current main still always surfaces mutating tool failures, so the reported user-facing problem is not implemented on main. The PR is focused and useful, but it should be revised before merge because the proposed suppression is keyed only to `hasUserFacingReply`, which can hide a real mutating-tool failure on failed or interrupted turns that already emitted assistant text. The existing review discussion also notes a stale comment that should be updated.

## Best Possible Solution

Keep the PR open and revise it: suppress stale mutating-tool warnings only when the turn has a successful terminal outcome plus a user-facing reply, preserve warnings for error/interrupted turns, update the stale policy comment, and add regression tests for both successful recovery and partial-output failure cases.
