## Summary

Do not close PR #71086. Current main has not implemented the proposed suppression, and the PR still represents a real behavior change, but the existing review finding is source-backed: suppressing solely on `hasUserFacingReply` would also hide mutating-tool failures after partial/error turns.

## Best Possible Solution

Keep the PR open and ask for a narrower implementation: suppress stale mutating-tool warnings only when the turn has a successful terminal state or the failed mutating action was actually recovered, while preserving warnings for unresolved mutating failures after error/interrupted turns. Add regression coverage for both the recovered-success case and the partial/error case, and update the stale payloads.ts comment noted by the review bot.
