## Summary

Keep PR #71086 open. Current main still always surfaces mutating-tool failures, including after assistant output, so the requested behavior is not implemented. The PR is a focused attempt with a targeted test change, but the open review concern is valid: suppressing based only on any user-facing reply can hide unresolved mutating-tool failures from partial/error turns.

## Best Possible Solution

Keep the PR open and ask for a narrow revision: suppress stale mutating-tool warnings only when the turn reached a successful final completion, preserve warnings for error/interrupted or unresolved mutating failures, update the stale block comment noted by Greptile, and add regression coverage for both successful recovery and partial/error-output cases. Coordinate with #39406 if maintainers want the broader configurable delivery policy handled separately.
