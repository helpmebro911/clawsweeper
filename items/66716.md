## Summary

Keep this PR open. Current main still has the preflight compaction early return for fresh persisted token totals, so the reported cached-token auto-compaction bug remains plausible. The PR is a focused implementation candidate, but it overlaps with #64384 and #65600 and needs maintainer consolidation/rebase rather than auto-close.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The maintainer path is to consolidate it with #64384 and #65600, then land one rebased fix in `src/auto-reply/reply/agent-runner-memory.ts` that checks explicitly fresh persisted totals against the preflight threshold, falls back to transcript estimation for stale or unknown freshness, preserves transcript-byte compaction, and adds regression coverage for high cached Anthropic totals, below-threshold fresh totals, stale/legacy fallback, and heartbeat/CLI skips.
