## Summary

Keep open. Current `main` still has the preflight-compaction fast path that returns early whenever persisted totals are present and `totalTokensFresh` is not `false`, so fresh cached token totals never reach the threshold check described in the PR.

## Evidence

- **Early return still present on main:** `runPreflightCompactionIfNeeded` computes `shouldUseTranscriptFallback`, and when that is false it immediately returns without calling `shouldRunPreflightCompaction`. For `totalTokensFresh === true` with a persisted total, that skips compaction gating entirely.

## Risks / Open Questions

- Closing this PR would leave `main` with a known logic hole where fresh cached totals can bypass preflight compaction entirely.
- The current test suite on `main` does not cover the fresh-token threshold path, so this regression is not protected without a change like the one proposed here.
