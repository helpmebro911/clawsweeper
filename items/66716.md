## Summary

Current main still has the fresh-token early return this PR targets, and the PR remains a plausible implementation candidate rather than something already shipped or clearly superseded. It overlaps with #64384 and #65600, but the latest PR discussion/commit also addresses the `totalTokensFresh: undefined` legacy-session edge, so closing it now would discard useful review-resolved work.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best maintainer path is to consolidate it with #64384 and #65600, land one tested fix that evaluates explicitly fresh persisted totals against the preflight token threshold, falls back to transcript estimation for stale or unknown freshness, and preserves the current active-transcript byte trigger.
