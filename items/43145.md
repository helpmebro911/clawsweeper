## Summary

Keep this PR open. Current main still does not implement the PR's central retained run-trace timeline store or `agent.timeline` gateway RPC. Some sub-fixes are already present on main, but the remaining trace feature is not obsolete and the PR diff has unresolved security-sensitive review findings plus broad unrelated scope.

## Best Possible Solution

Keep this PR open for explicit maintainer handling, but do not land it as-is. The useful path is to split the already-shipped sub-fixes away, then replace the remaining trace work with a focused PR aligned with #68204: a canonical run trace schema, safe gateway/API exposure, ownership checks, redaction, bounded retention/span caps, and targeted tests. Unrelated sandbox, cron, plan-search, simulation, and roadmap work should be separate PRs or dropped from this branch.
