## Summary

Current main does not implement the requested generic custom-suffix stripping. Model refs with `:med`/`:high` remain literal model ids through alias resolution, and provider params are looked up by exact `provider/modelId`, so the requested “unique config key, canonical API model id, variant-specific params” behavior is still missing.
