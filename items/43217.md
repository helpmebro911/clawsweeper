## Summary

Keep open. Current main still does not implement the PR's central behavior: the shared Google thinking sanitizer receives only `modelId` and `thinkingLevel`, and the generic negative `thinkingBudget` path only deletes the invalid budget. The PR is stale because the sanitizer has moved into the shared provider stream helper, and its proposed direct `thinkingLevel` injection may need to become Gemini-2.5 budget mapping, but it remains plausible bug-fix work rather than a cleanup close.

## Best Possible Solution

Keep this PR open and ask for a rebase onto the shared Google thinking wrapper. The maintainer path is to adapt `src/plugin-sdk/provider-stream-shared.ts` so the negative-budget sanitizer can use resolved model capability or the provider thinking profile, preserve Gemini 3/Gemma/adaptive behavior, repair fixed-level Gemini 2.5 payloads with the model-family-correct budget mapping, add a `gemini-2.5-flash` fixed-level regression test plus a non-reasoning negative case, and update the stale Gemini-3.1-only comment.
