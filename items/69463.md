## Summary

Keep open. Current main does not implement the PR's `diagnostics.sessionTracing` config or dynamic `x-session-id` / `x-session-name` request-header injection. Existing main supports static provider headers, provider-owned stream wrappers, native transport turn-state hooks, and model-call diagnostics, but those do not fully replace this PR's all-provider, opt-in run/session correlation header behavior. The supplied PR discussion is also positive: Greptile characterized the implementation as safe to merge with only P2 cleanup suggestions, so there is no strong conservative close reason.

## Best Possible Solution

Keep this PR open for maintainer product/API review. The best path is either to land a revised, documented, opt-in core or SDK-backed mechanism for dynamic LLM request correlation headers, or to first extend the plugin/provider hook context so a ClawHub observability plugin can attach run/session headers across providers without one-off core behavior.
