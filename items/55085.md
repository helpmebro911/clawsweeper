## Summary

Keep open. Current main has adjacent GUI/onboarding surfaces, including Gateway wizard RPC and config.schema/config.schema.lookup, but it still does not include this PR's requested stable read-only setup discovery export (`openclaw configure surface --json-out ...`) or an equivalent provider/channel setup-surface document. Existing `plugins list --json` is plugin inventory, and `configure` remains documented and wired as an interactive command.

## Best Possible Solution

Keep the PR open for maintainer/product review. The best path is to define one stable setup-discovery contract for external UIs, likely reusing the existing manifest/setup metadata and aligning it with current Gateway/config-schema surfaces, then update the PR to address the review comments before considering merge.
