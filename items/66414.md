## Summary

Keep PR #66414 open. Current main still lacks both central fixes proposed by the PR: provider-qualified catalog ids are not matched by `resolveGatewayModelSupportsImages()`, and Control UI chat still reads pasted/selected/dropped image files asynchronously without tracking pending reads or blocking send. The PR is a focused bug fix with useful discussion, but review comments identify stale-props/state-update problems that should be addressed before merge rather than closing the item as obsolete.

## Best Possible Solution

Keep the PR open, but require the author or maintainer to revise it so attachment read state is updated against live app state and attachment additions merge into the latest array. The gateway provider-qualified catalog regression can be kept, and the Control UI fix should include focused tests for concurrent reads, send blocking, and override commands such as `/new` or `/compact`.
