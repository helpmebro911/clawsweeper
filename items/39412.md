## Summary

Keep PR #39412 open. Current main already covers the daemon gateway-token persistence part with config-backed, SecretRef-aware install behavior, but it still does not implement the PR's ambient provider `*_FILE` environment-variable convention. The PR is still useful as a focused provider-secret follow-up, but it should be rebased or narrowed before merge because its gateway-token edits overlap newer main behavior and the review thread has unresolved branch-specific findings.

## Best Possible Solution

Keep the PR open as useful stale work, but do not merge the branch as-is. Rebase or rewrite it against current main, drop the obsolete daemon gateway-token persistence edits, and implement only the still-missing provider `*_FILE` env-var behavior if maintainers want that ambient convention. The implementation should integrate with `src/agents/model-auth-env.ts`, `src/agents/live-auth-keys.ts`, and provider usage paths; preserve separators for multi-key files; report unreadable file misconfiguration clearly; and add focused tests for direct-env precedence, missing/unreadable files, single-key vars, prefixed vars, list vars, and provider usage auth.
