## Summary

Keep PR #70056 open. Current main still has the reported gateway stop-timeout bug: the timeout branch records `running: true`, returns before `store.aborts`/`store.tasks` cleanup, and the existing test still asserts that stale behavior. The PR targets a real bug linked to #70024, but the supplied review discussion identifies a real race in the proposed patch, so the right outcome is revision or an equivalent fixed implementation, not closure.

## Best Possible Solution

Keep the PR open and request a small revision: the timeout path should stop reporting `running: true`, record stop metadata, and unblock recovery, but it must also prevent the timed-out task's later `finally`/restart chain from overwriting a replacement task. Add regression coverage for both restart-after-timeout and a late-settling old task.
