## Summary

Keep this PR open. Current main still has the gateway stop-timeout stale-store bug that the PR targets, and the paired open bug remains unresolved. The submitted patch is relevant but should be revised before merge because the review comments identify a real stale timed-out-task race that current main's lifecycle code would allow if the PR simply deletes the task reservation on timeout.

## Best Possible Solution

Keep the PR open and ask for a revision in `src/gateway/server-channels.ts` that makes timeout state honest without letting a stale timed-out task mutate runtime or restart state for a replacement lifecycle. Add a regression test where the timed-out task settles after a successful restart, then land the revised PR or an equivalent maintainer patch to resolve #70024.
