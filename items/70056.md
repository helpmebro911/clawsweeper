## Summary

Keep open. The linked gateway bug is still present on current main, and this PR is a plausible targeted fix but is not safe to close or mark implemented because the proposed patch has an unresolved race called out in review: deleting the task on timeout can let a replacement start while the old task's finally/restart chain later mutates runtime state.

## Best Possible Solution

Keep the item open and revise the gateway fix so stop timeouts no longer lie as running:true or permanently block restarts, while also preventing a timed-out old task from clobbering a replacement. A robust patch should guard post-settlement runtime/restart cleanup by task identity or preserve a reservation until the original task settles, then add regression coverage for both restart-after-timeout and slow eventual settlement after a restart.
