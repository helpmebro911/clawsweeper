## Summary

Keep open. Current main still builds `sessions_spawn` child delivery from the tool context only, so the parent session's stored `deliveryContext.to` / `threadId` is not recovered before the child `agent` call. The related Slack routing PRs/issues address adjacent paths, but this PR remains the focused tracker for parent delivery-context backfill in `sessions_spawn` and ACP spawn.

## Best Possible Solution

Keep PR #69201 open and either merge it after maintainer review/CI or integrate an equivalent fix: backfill missing delivery fields in `spawnSubagentDirect` and `resolveAcpSpawnRequesterState` from the parent session's stored delivery context, preserve the gateway `agent` method as the single persistence writer, keep stale-thread guards for explicit root-level targets, and retain targeted regression coverage for the persisted child session entry.
