## Summary

Current `main` already implements voice calling through the bundled `voice-call` plugin, including outbound/inbound calls, multi-turn conversations, realtime transcription/voice provider hooks, a CLI surface, and an agent tool. The request is therefore already satisfied on `main`, even though the shipped shape is a plugin (`voice-call`) rather than a built-in `channels.voice` core channel.

## Close Comment

Closing from a Codex review because current `main` already has this capability through the `voice-call` plugin. The shipped implementation is documented in `docs/plugins/voice-call.md:9` (outbound + inbound multi-turn calls), exposed on the CLI in `docs/cli/voicecall.md:9`, and registered in runtime code in `extensions/voice-call/index.ts:399` and `extensions/voice-call/index.ts:422` (`voicecall.*` gateway methods plus the `voice_call` tool). The changelog also shows the Voice Call plugin shipped in the `2026.1.11` section at `CHANGELOG.md:6507` and `CHANGELOG.md:6525`. This is implemented on `main`, although it ships as a plugin rather than a built-in core `channels.voice` entry.
