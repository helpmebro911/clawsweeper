## Summary

Keep open. Current main has the newer pending tool-media queue, but it still does not implement this PR's specific fallback from inline read image results to the original read tool path. The PR remains useful, though it needs to be rebased onto current main and should address the existing review comment about blank path shadowing file_path.

## Best Possible Solution

Keep the PR open for changes. The best fix is to rebase onto current main, pass normalized read start args into the trusted tool-result media extraction path, fall back to trimmed path then file_path only when inline image content exists and no structured media, MEDIA text, or details.path is present, and add regression coverage for both the no-details.path read image case and the blank path plus file_path case.
