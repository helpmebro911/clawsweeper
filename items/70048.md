## Summary

Keep open. Current `main` already has explicit setup flows for vLLM, LM Studio, and a generic custom OpenAI/Anthropic-compatible endpoint, so the issue body overstates the current gap. But the bundled LiteLLM onboarding path still only asks for an API key and then writes the default `http://localhost:4000` base URL, which leaves a real silent-misconfiguration path for non-default LiteLLM proxy deployments.

## Evidence
