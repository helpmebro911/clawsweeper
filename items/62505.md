## Summary

Keep open. The issue has a concrete remaining reproduction path from the discussion: async exec completions can still be skipped when heartbeat cadence is disabled with `agents.defaults.heartbeat.every = "0m"`. Current main enqueues exec-event wakes, but the heartbeat runner does not keep any agents scheduled when the interval resolves to null, then returns `disabled` before dispatching targeted exec-event wakes. The related PR #67913 was reported as cancelled in the issue context, and I did not find an equivalent disabled-heartbeat exec-event bypass on main.
