## Summary

Keep #47452 open. Current main still lacks the requested qiniu/Kimi anthropic-messages replay behavior: the generic Anthropic fallback only enables dropThinkingBlocks for Claude model IDs, while same-model signed thinking blocks are preserved and serialized back to Anthropic message payloads as type: "thinking". The discussion correctly identified a qiniu/Moonshot-style provider capability fix, but the related PR #47459 is closed unmerged and no qiniu provider or dropThinkingBlockModelHints implementation exists on current main.

## Evidence

- **Generic Anthropic fallback only drops Claude thinking:** For unowned anthropic-messages providers, the replay policy preserves signatures and only sets dropThinkingBlocks when the normalized model id contains "claude". A custom qiniu model id like moonshotai/kimi-k2.5 therefore gets no dropThinkingBlocks policy from this path.
