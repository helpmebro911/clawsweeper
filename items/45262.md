## Summary

Keep #45262 open. Current main still does not implement the requested Slack `slashCommand.contextAware` behavior: the single configured Slack slash command still rejects an empty text argument with `Message required.`, and the public Slack config type/schema/docs do not expose `contextAware`. The PR is also not safe to treat as already solved or closeable because the provided review comments identify real implementation issues around unused context wiring and raw channel-history prompt/directive injection.

## Best Possible Solution

Keep the PR open for maintainer/product judgment, but require a reworked implementation on the current `extensions/slack` layout: add the typed config/schema/docs/tests, preserve the current `Message required` default when the option is false, fetch recent history only for an authorized empty room-ish slash invocation, format/filter history using the existing Slack supplemental-context patterns, and pass it through a non-command context surface such as `ThreadHistoryBody`/`UntrustedContext` or an equivalent safe field so old messages cannot be parsed as inline directives.
