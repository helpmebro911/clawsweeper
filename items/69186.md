## Summary

Keep #69186 open. Current main has adjacent pieces, including documented `agent_end`/`subagent_ended` plugin hooks and newly added generic Web Push plumbing, but it still does not implement the requested Control UI completion/success sound or a browser-local setting/custom sound URL for agent-turn completion. The related #59199 cleanup correctly covers the lower-level post-turn hook, but this issue asks for an actual WebChat/Control UI audible cue for background-tab users, and that remains missing.

## Best Possible Solution

Keep the issue open as a focused Control UI feature request. A good implementation would add a browser-local setting for completion cues, hook it into terminal chat events for the active session/run, avoid firing on aborted/error states unless explicitly configured, and include focused UI/controller tests. Optional custom sound URL support should be handled conservatively, likely browser-local only, with autoplay/permission behavior accounted for. If maintainers want this outside core, the missing piece is a documented Control UI/frontend extension API rather than just the existing backend `agent_end` hook.
