## Summary

Keep open. Current `main` still implements the sandbox browser CDP path with a port-offset Chromium listener behind a `socat` relay, and recent runtime/tests/docs reinforce that design rather than replacing it. The related `#62873` work fixed startup-hang behavior, but it did not simplify the CDP path to a direct Chromium port.

## Evidence

- **Entrypoint still uses port offset plus socat relay:** The sandbox browser entrypoint derives `CHROME_CDP_PORT` from `OPENCLAW_BROWSER_CDP_PORT`, waits on Chromium readiness at the derived port, then starts `socat` to relay from the public CDP port to Chromium. This is the exact proxy layer the issue asks to evaluate removing.
