## Summary

Keep open. Current main has an Ollama native streaming path that addresses the likely NDJSON streaming mismatch discussed in the issue, but the issue is not fully closable: llm-task still concatenates provider and model without stripping an already-qualified provider/model model parameter, and the qwen-style thinking-only path still does not become normal llm-task JSON text.

## Evidence

- **llm-task still accepts model override raw and concatenates provider/model:** The tool resolves provider and model separately, uses the raw params.model string unchanged, then builds modelKey as `${provider}/${model}`. Passing provider=`anthropic` and model=`anthropic/claude-sonnet-4-6` still becomes `anthropic/anthropic/claude-sonnet-4-6`.
