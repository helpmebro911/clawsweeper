## Summary

Keep open. Current main still fail-fasts startup-gated `chat.history` and `models.list` requests with retryable `UNAVAILABLE`; it does not implement the PR's server-side startup barrier. Later shipped work retries TUI `chat.history`, but that is narrower than this PR's cached/pre-retry Control UI and `models.list` case, so the PR is not obsolete.

## Best Possible Solution

Keep this PR open for maintainer review or a rebased successor. The best path is a gateway-owned startup barrier or equivalent server-side queue for startup-gated methods, with tests covering dispatch after sidecars, timeout fallback, non-gated bypass, and legacy no-barrier behavior. Client retries should remain defense in depth, not the only fix for cached or older clients.
