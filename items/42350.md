## Summary

Keep open. Current `main` still lacks iOS/macOS inline Adaptive Card rendering, so this is not implemented or obsolete. At the same time, the proposed approach appears to hard-code a plugin-specific `<!--adaptive-card-->` text marker into shared chat UI, which conflicts with the repo's extension-agnostic and documented-seam guidance; that argues against merging this exact design, but not strongly enough for an auto-close reason.
