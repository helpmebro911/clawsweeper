## Summary

Keep this PR open. Current main still does not implement inline Adaptive Card rendering in the shared Swift chat UI used by iOS/macOS, and this contributor PR is paired with the same author’s still-open Android PR #42304. The PR is also not merge-ready as-is: review comments identify functional gaps in the proposed Swift parser/view, and the security-sensitive URL/image rendering path needs explicit maintainer review rather than automatic closure.

## Best Possible Solution

Keep the PR open for maintainer product review. If native iOS/macOS Adaptive Card rendering is wanted, update the branch against current main, resolve the parser/layout/caching/image fallback review comments, add focused Swift parser/view coverage or an app smoke path, explicitly review URL/image loading behavior, and coordinate the decision with the open Android companion PR #42304. If maintainers prefer not to support native client renderers, close the platform PR set together with a product-scope comment and leave the external `openclaw-adaptive-cards` plugin as the marker-producing integration.
