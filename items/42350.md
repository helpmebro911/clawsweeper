## Summary

Keep open. Current `main` does not implement inline Adaptive Card rendering for the iOS/macOS chat UI, so this PR is not obsolete. It also mixes in a docs/community plugin listing change that conflicts with current ClawHub listing guidance, and the proposed marker-based rendering path still needs product/architecture review against OpenClaw's generic `presentation` model rather than a cleanup close.
