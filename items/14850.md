## Summary

Keep open. Current main only implements a narrower per-binding `session.dmScope` override for direct-message routing. The requested capability in #14850 is to force a guild/channel binding to share a main or aliased session across Discord channel and Telegram DM surfaces, and current routing still keeps Discord guild channels on `agent:<agentId>:discord:channel:<channelId>` session keys. Related PR #71750 partially addressed direct-peer scoping only; related PR #50875 is still open and explicitly scoped away from group/channel peers; PR #39049 was closed unmerged.
