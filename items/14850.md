## Summary

Keep #14850 open. Current main now has binding-level `session.dmScope`, but it still does not implement a binding-level `sessionKey`, `sessionAlias`, or main-session target for non-direct Discord guild/channel bindings. `dmScope` is applied only to direct-message peers, so the reported Discord channel to Telegram DM session unification remains missing.

## Best Possible Solution

Keep this open until maintainers either add a documented binding-level session target for non-direct route bindings or explicitly reject channel-to-main session unification. The likely implementation path is `AgentRouteBinding`, `RouteBindingSchema`, config help/docs, `resolveAgentRoute`, and `buildAgentSessionKey`/`buildAgentPeerSessionKey`, with tests proving Discord channel bindings can intentionally share an agent main or aliased session while defaults remain isolated.
