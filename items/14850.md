## Summary

Keep #14850 open. Current main still implements only binding-level `session.dmScope` for direct-message routing and still has no binding-level fixed `sessionKey`, alias, or main-session target for Discord guild/channel bindings to share a Telegram DM/main session.

## Best Possible Solution

Keep the issue open until maintainers either add and document an explicit binding-level session target for non-direct route bindings, or make an explicit product decision to reject channel-to-main session unification. The likely implementation path is `AgentRouteBinding`, `RouteBindingSchema`, `resolveAgentRoute`, and `buildAgentPeerSessionKey`, with regression tests proving Discord guild/channel bindings can intentionally share an agent main or aliased session while preserving isolated defaults.
