## Summary

Keep this issue open. Current main still bundles Telegram TTS text into the media caption for normal-length voice-note sends, only splitting text on caption overflow or video-note cases, and there is no `messages.tts.delivery` or equivalent typed/configured option.

## Best Possible Solution

Implement a Telegram-owned TTS delivery mode, or a shared `messages.tts.delivery: "bundled" | "separate"` if maintainers want cross-channel semantics. The implementation should send visible text as a normal Telegram message and then send the voice note without a caption when the effective mode is `separate`, with focused tests for auto-TTS voice delivery, short-caption behavior, overflow compatibility, reply/thread routing, and docs on the public TTS or Telegram pages.
