## Summary

Keep #9120 open. Current main still has timer-based inbound debouncing, active-run queue collection, and continuation-skip context injection, but not the requested user-controlled continuation marker workflow that buffers marked fragments until an unmarked final message arrives.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. A complete path likely belongs in shared inbound handling or a documented plugin/API seam: add typed messages.continuationMarker and messages.continuationTimeout config/schema/docs, buffer marked text-only inbound messages before agent invocation, strip the marker from model-visible text, release on the first unmarked message or timeout, define media/control-command behavior, and add focused tests across inbound helpers, channel routing, queue interaction, and typing policy.
