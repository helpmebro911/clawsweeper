## Summary

Keep #9120 open. Current main still has timer-based inbound debouncing and active-run queue collection, but it does not implement the requested user-controlled continuation marker, continuation timeout, marker stripping, or continuation-aware typing mode.

## Best Possible Solution

Keep the issue open for maintainer/product judgment. A complete path likely belongs in shared inbound handling or a documented plugin/API seam: add typed messages.continuationMarker and messages.continuationTimeout config/schema/docs, buffer marked text-only inbound messages before agent invocation, strip the marker from model-visible text, release on the first unmarked message or timeout, define media/control-command behavior, and add focused coverage across inbound helpers, channel routing, queue interaction, and typing policy.
