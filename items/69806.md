## Summary

Keep PR #69806 open. Current main does not contain the central test-stabilization changes: the npm execpath missing-case test is still host-env sensitive, and the config validation regression/allowed-values tests still reach the doctor contract registry path without local mocks. The PR is useful but needs changes before merge because the proposed migration mock shape is inconsistent with current main and the root markdown-it-task-lists dependency addition needs justification or removal.

## Best Possible Solution

Keep the PR open and request a small revision: rebase onto current main, either omit applyPluginDoctorCompatibilityMigrations from these validation-test mocks or return { config: cfg, changes: [] }, remove or clearly justify the root markdown-it-task-lists dependency addition, and then run the relevant changed/test lanes before maintainer merge.
