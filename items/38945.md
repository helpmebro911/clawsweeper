## Summary

Keep #38945 open. Current main at cf834e2a21eac7fd218989fb5b85e7e8dd4c3246 still lacks the PR's central fix: memory-core MMR tokenization is ASCII plus CJK-specific, while FTS query planning still omits combining marks and NFC normalization. Related shipped work improved CJK chunking, CJK MMR, and CJK/trigram FTS, but it does not cover non-CJK Unicode token sets, combining-mark scripts, or NFD/NFC matching.

## Best Possible Solution

Keep the PR open and port the narrow tokenizer fix to current memory-core: use NFC normalization plus a Unicode token regex that includes letters, marks, numbers, and underscores for MMR, direct FTS query building, and shared keyword planning where applicable. Add focused regression tests for accented Latin/Finnish, Cyrillic, Arabic/Hebrew, Thai, Hindi, and NFD-to-NFC matching. Treat this as a real bundled memory-core bug fix, not ClawHub or a duplicate of the CJK-only work.
