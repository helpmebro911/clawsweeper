## Summary

Keep open. Current `main` still lacks a distinct `google-vertex` provider/runtime path with Vertex-specific ADC and endpoint routing, so the regression is not clearly fixed in core.

## Evidence

- **No dedicated google-vertex provider on main:** The bundled Google plugin manifest still only exposes `google` and `google-gemini-cli`; `google-vertex` is not registered as its own provider surface.
  - file: [extensions/google/openclaw.plugin.json:4](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/google/openclaw.plugin.json#L4)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **google-vertex is still handled as a Google hook alias:** The provider policy still special-cases `google-vertex` inside Google normalization logic instead of giving it an independent provider implementation.
  - file: [extensions/google/provider-policy.ts:99](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/google/provider-policy.ts#L99)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Transport code still builds Gemini API model URLs, not Vertex publisher/project paths:** Current Google transport builds requests as `<baseUrl>/models/<id>:streamGenerateContent` and contains no Vertex AI `/projects/.../locations/.../publishers/google/models/...` routing.
  - file: [extensions/google/transport-stream.ts:182](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/google/transport-stream.ts#L182)

  - file: [extensions/google/api.test.ts:109](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/google/api.test.ts#L109)

- **Issue comments show workaround, not a shipped resolution:** The follow-up comment reports success after setting `GOOGLE_CLOUD_LOCATION=global`, which supports a config workaround but does not show that current main restored a separate Vertex transport/auth path.
