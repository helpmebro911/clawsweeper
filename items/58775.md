## Summary

Keep open. Current main has partial google-vertex metadata and model-id normalization, but it still lacks native Google Vertex chat transport/auth: google-vertex is routed through Google hooks, the Google transport builds Google Generative AI model URLs, and ADC marker auth is not exchanged for a Vertex bearer token. The supplied GitHub context also shows PR #60860 is open and explicitly intended to close this issue, so the issue should remain open until that PR or a maintainer-owned equivalent lands.

## Best Possible Solution

Keep this issue open while PR #60860, or a maintainer-owned equivalent, is reviewed. The best implementation is a native google-vertex path in the Google plugin that keeps ADC auth, project/location resolution, and Vertex URL construction separate from Google AI Studio while sharing only safe model-id normalization and adding focused regression tests for URL/auth behavior.
