## Summary

Keep open. This is a recent, concrete feature request from a non-maintainer, and current main only shows adjacent session persistence, session send/create/reset, and compaction checkpoint branching. I do not see strong evidence that the requested open-agent-sdk-style fork/resume/continue API is fully implemented or obsolete.

## Evidence

- **Issue is eligible but not stale:** #59109 was opened 2026-04-01 by Zsmboom with author association NONE, no protected labels, and concrete requested APIs: forkSession(id), resume by sessionId, and continue most recent session. It is under 60 days old, so stale_insufficient_info does not apply.
  - command: `provided GitHub context for #59109`
- **Current gateway method surface has adjacent session operations, not the requested named API:** The gateway exposes sessions.list, sessions.compaction.branch/restore, sessions.create, sessions.send, sessions.abort, sessions.patch, sessions.reset, sessions.delete, and sessions.compact. That supports session management, but does not prove an open-agent-sdk-style forkSession(id), resume option, or continue-most-recent API exists.
  - file: [src/gateway/server-methods-list.ts:72](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/src/gateway/server-methods-list.ts#L72)
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Current docs describe sessionKey to sessionId persistence rather than arbitrary API resume/fork semantics:** The session-management docs describe each sessionKey pointing at a current sessionId, with resets/daily/idle expiry creating new sessionIds, and a thread parent fork guard. This is related behavior but not equivalent to the requested public fork/resume/continue API.
  - file: [docs/reference/session-management-compaction.md:351](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/reference/session-management-compaction.md#L351)

- **Existing UI controller branches only from compaction checkpoints:** The Control UI can request sessions.compaction.branch from a selected checkpoint and sessions.compaction.restore, which is narrower than forking any existing session ID as requested.
  - file: [ui/src/ui/controllers/sessions.ts:2125](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/ui/src/ui/controllers/sessions.ts#L2125)
