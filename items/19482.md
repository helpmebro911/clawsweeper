## Summary

Keep #19482 open. Current main still unconditionally recreates `HEARTBEAT.md` during workspace bootstrap, while the heartbeat runner, docs, and tests preserve a meaningful difference between a missing file and an empty heartbeat file. The PR is still a focused fix for current behavior, with the main improvement being a tighter negative regression fixture.

## Best Possible Solution

Keep the PR open and either land it after tightening the negative regression fixture, or apply the same code change directly on main: only seed `HEARTBEAT.md` for brand-new workspaces while preserving missing-file heartbeat semantics for existing workspaces.
