## Summary

Keep open. Current `main` still recreates `HEARTBEAT.md` for any workspace missing the file, so the PR's requested behavior is not implemented on `main` or the latest release.

## Evidence

- **Current code still writes HEARTBEAT unconditionally:** `ensureAgentWorkspace()` computes `isBrandNewWorkspace`, but then always calls `writeFileIfMissing(heartbeatPath, heartbeatTemplate)` with no guard. A workspace that already exists but has no `HEARTBEAT.md` will have it recreated.
  - file: [src/agents/workspace.ts:373](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/agents/workspace.ts#L373)

- **Brand-new guard is used elsewhere, not for HEARTBEAT:** `ensureGitRepo()` already exits early when `isBrandNewWorkspace` is false, which shows the codebase distinguishes fresh-workspace-only seeding paths. `HEARTBEAT.md` is not following that pattern on current `main`.
  - file: [src/agents/workspace.ts:325](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/agents/workspace.ts#L325)
