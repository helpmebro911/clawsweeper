## Summary

Keep #19482 open. Current main still recreates `HEARTBEAT.md` unconditionally during workspace bootstrap, while the heartbeat runner, docs, and tests distinguish missing files from empty heartbeat files. The PR remains a focused fix for current behavior, with the main remaining improvement being a tighter negative regression fixture.

## Best Possible Solution

Keep the PR open and either land it after tightening the negative regression fixture, or apply the same code change directly on main: only seed `HEARTBEAT.md` for brand-new workspaces while preserving missing-file heartbeat semantics for existing workspaces.
