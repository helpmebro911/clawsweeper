## Summary

Keep open. Current main still recreates HEARTBEAT.md whenever it is missing, while the heartbeat runner and docs preserve a semantic difference between a missing file and an empty/comment-only file. This PR remains a focused implementation candidate, but it needs a rebase onto the current test layout and a tighter negative fixture before landing.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, move the regression coverage into src/agents/workspace.test.ts, tighten the negative fixture to create all canonical workspace files except HEARTBEAT.md, then land this guard or apply the same guarded write directly on main.
