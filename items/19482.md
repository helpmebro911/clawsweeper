## Summary

Keep open. Current main still unconditionally recreates `HEARTBEAT.md` during workspace bootstrap, while heartbeat runtime and docs intentionally distinguish a missing file from an empty template file. This PR remains a focused implementation candidate, though it should be rebased and its negative test tightened.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, move the regression coverage into the current workspace test layout, tighten the negative fixture to represent a fully initialized workspace with only `HEARTBEAT.md` removed, then land this guard or apply the same change directly on main.
