## Summary

Keep this PR open. Current main still has the stale Google Gemini CLI `flash` alias that the PR changes: `flash` resolves to `gemini-3.1-flash-preview`, while the same backend default and current docs already use `google-gemini-cli/gemini-3-flash-preview`. The CLI runner passes backend aliases directly into the `--model` argument, so this is still a real execution path rather than an obsolete diff.

## Best Possible Solution

Keep the PR open, review it as a focused bundled Google plugin bug fix, add or request a small regression test for Gemini CLI alias resolution, and verify the remaining preview aliases before merging or asking for a tiny follow-up adjustment.
