## Summary

Keep open. Current main has independently addressed the config.patch restart/hot-reload part, but it still does not implement the MiniMax M2.5 XML tool-call promotion this composite PR also targets. The current MiniMax path strips XML out of assistant text instead of converting it into structured toolUse blocks, so the PR is only partially obsolete.

## Best Possible Solution

Keep the item open or ask the author to rebase and narrow it to the missing MiniMax behavior: parse MiniMax XML tool invocations before extractAssistantText strips them, emit structured toolUse blocks, decode basic XML entities, accept attributes in any order, and add focused regression tests. Treat the config.patch hot-reload part as already handled on main unless a maintainer wants a follow-up for sentinel/response-shape details.
