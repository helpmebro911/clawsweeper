## Summary

Keep open. Current `main` still routes cron-triggered embedded work onto the shared `nested` lane while Gateway concurrency only special-cases `cron`, `main`, and `subagent`, so the PR's core fix is not implemented on `main`. The contributor's later `cron-nested` revision also directly answers the review concern about widening interactive nested throughput, although it now needs a rebase onto the newer file layout from `#71413`.
