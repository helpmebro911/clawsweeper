## Summary

Keep #38248 open. Current main has cost metadata, usage/cost reporting, provider quota snapshots, and improved failover handling for provider-returned billing/spend-limit errors, but it still lacks the requested native request-level rolling spend ceiling for model fallback chains. The prior discussionā€™s external mitigations such as `npx vibe-billing setup`, Kalibr routing, and LiteLLM virtual keys are useful workarounds, not an OpenClaw core guard.

## Best Possible Solution

Keep the issue open until OpenClaw either adds a native cost guard in the model request/failover path or explicitly designs a blocking plugin/API seam for rolling spend enforcement. A strong fix would add documented config, hourly/daily accounting, pre-attempt estimated-cost checks, post-response reconciliation from provider usage/cost data, pause/reject behavior, owner notification, and regression coverage around fallback chains.
