## Summary

Current main still does not implement the requested opt-in auto-forwarding of Node `agent.request` image attachments into outbound IM delivery. The gateway parses attachments for agent/model input, but the protocol schema, node-event handler, command options, and delivery path have no preserved auto-forward flag or delivery-ready original attachment payloads to inject when the agent replies with only text or no media.

## Best Possible Solution

Keep this issue open. The best implementation is an additive gateway protocol/node-event change that accepts an explicit auto-forward flag, preserves sanitized delivery-ready attachment references through `AgentCommandOpts`, and injects them into outbound delivery only under narrow conditions such as `deliver: true`, a resolved external target, supported media types/channels, and no agent-supplied media. Add regression tests for text-only replies, no-reply runs, agent-media override, unsupported channels, and size/type enforcement.
