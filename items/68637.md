## Summary

Keep open. Current `main` still rejects empty or whitespace-only `write.content`: `REQUIRED_PARAM_GROUPS.write` requires `content` without `allowEmpty`, and `assertRequiredParams` only accepts string params with `trim().length > 0` unless that flag is set. The PR is a focused fix for the live issue it links (#68636), uses an existing validation mechanism, and adds targeted regression coverage for the accepted and still-rejected cases.

## Best Possible Solution

Keep the PR open or land an equivalent fix on `main`: set `allowEmpty: true` for `REQUIRED_PARAM_GROUPS.write` content while preserving rejection for missing, null/undefined, and non-string content, and add the targeted regression tests proposed by the PR.
