## Summary

Keep open. Current `main` has the generic `resolveInboundLastRouteSessionKey` helper and coverage, but the affected inbound DM handlers named in PR #40233 still mostly write last-route updates to `route.mainSessionKey` instead of the isolated session key, so the requested behavior is not implemented on `main`.
