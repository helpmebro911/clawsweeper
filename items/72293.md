## Summary

Keep open. Current main still has the reported abort-classification gap: a final assistant turn with `stopReason: "aborted"` is not propagated from the embedded lifecycle producer into session lifecycle, `agent.wait`, or subagent completion outcome, so the parent can still receive an apparently successful no-output completion instead of an abort/failure. The existing Codex review comment on this issue is consistent with current main.

## Best Possible Solution

Keep this issue open and patch the embedded subagent terminal path. Propagate final assistant `stopReason: "aborted"` and `errorMessage` from `handleAgentEnd` into lifecycle events, teach `agent.wait` and subagent registry completion to classify it as killed/failed instead of ok, persist `abortedLastRun: true`, and add regression coverage for an empty-content aborted assistant turn producing a parent-visible failure/no-output announcement.
