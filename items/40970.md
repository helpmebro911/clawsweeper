## Summary

Keep open. Current main only partially covers the requested behavior: the file writer can internally roll an `openclaw-YYYY-MM-DD.log`-shaped path to today’s dated file, but `getResolvedLoggerSettings()`, gateway startup logging, and `logs.tail` still see the configured literal placeholder. The PR is a focused fix candidate, with a small deterministic-test cleanup still needed.

## Best Possible Solution

Keep this PR, or land an equivalent focused fix, after making placeholder resolution a shared logging-file helper used by resolved settings, startup output, runtime writes, and log tail. Preserve the current daily rollover behavior, add deterministic fake-timer coverage for `logging.file` placeholder substitution and log tail, and update the logging/config reference if arbitrary `YYYY-MM-DD` replacement is intended.
