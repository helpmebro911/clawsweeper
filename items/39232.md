## Summary

Keep #39232 open. Current main still lacks a universal pre-filter raw inbound plugin hook: the typed hook catalog has no `on_raw_inbound`/`onRawInbound`/`before_channel_filter`, unknown typed hooks are ignored, and WhatsApp `message_received` still fires only after access control plus group mention gating. The related PR #39240 was closed unmerged, and current Telegram/Signal `ingest` support is only channel-specific internal mention-skip ingest, not the requested cross-channel typed plugin seam.

## Best Possible Solution

Keep the issue open for maintainer/API design. A good path would add an observe-only, privacy-aware pre-filter inbound contract to the typed plugin/channel SDK, wire participating channel plugins before access control and mention gating only behind explicit opt-in, document the privacy model in public plugin/channel docs, and add contract plus channel-specific ordering tests. If maintainers decide raw ambient observation is not supported, document that policy and point users to narrower post-gating alternatives such as current opt-in `message_received` or channel-specific internal ingest behavior.
