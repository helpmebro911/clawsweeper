## Summary

Keep #70005 open. Current main has useful mitigations for generated transcript noise and the literal `assistant` tag, but the requested semantic-importance material selection is still not implemented. REM still builds reflections from concept-tag recurrence and candidate confidence from frequency/relevance/consolidation-style signals, and the narrative layer only writes from whatever fragments REM/light selected. The provided related items cover other dreaming configuration/sprawl work, not this specific semantic selection gap.

## Best Possible Solution

Keep this issue open for memory-core product/implementation work. The likely fix belongs in the pre-REM material-selection path: add a deterministic or model-backed semantic density/importance pass that down-ranks routine role labels and boilerplate, promotes decisions/preferences/conclusions/unusual events, and adds focused regression coverage using a fixture shaped like the reported Persona/DQ/SRW session versus high-frequency `assistant` noise. The existing noise filters and `assistant` tag blacklist should be kept as supporting safeguards, not treated as the complete fix.
