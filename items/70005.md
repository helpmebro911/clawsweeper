## Summary

Keep #70005 open. Current main has useful partial mitigations for dreaming noise, including transcript sanitization, generated-cron/heartbeat filtering, contaminated dreaming-snippet rejection, and a small REM tag blacklist. But the requested semantic-importance material selection is still not implemented: REM reflections remain concept-tag occurrence driven, light dreaming still selects by recency/recall count, candidate truths use recall-style confidence, and there is no `dreaming.promptStyle` or material-selection strategy knob.

## Best Possible Solution

Keep the issue open for memory-core product work. The likely fix belongs in `extensions/memory-core/src/dreaming-phases.ts` and/or the short-term recall pipeline: add a semantic importance or grounded event extraction pass before REM and narrative inputs, prioritize decisions/preferences/conclusions/unusual events over repeated role/scaffolding tokens, and add regression tests where a low-frequency meaningful user decision beats high-frequency generic transcript scaffolding. Only add a public `dreaming.promptStyle` or strategy config if maintainers want this behavior tunable rather than default.
