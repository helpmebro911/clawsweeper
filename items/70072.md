## Summary

Keep open. Current main has adjacent diagnostics (`memory promote`, `memory promote-explain`, Gateway Dreams status) and #68882 was closed as the zero-promotion fix, but #70072 is a distinct additive request. The scheduled deep report still writes only count lines, verbose scoring still goes to the logger rather than a workspace artifact, and there is no `openclaw memory dreaming doctor` command or aggregate gate-reachability diagnosis.

## Best Possible Solution

Keep #70072 open and implement it in `memory-core`: add a structured deep-run explanation path that records passing and rejected candidates with threshold pass/fail reasons, render a concise candidate table plus thresholds into `memory/dreaming/deep/YYYY-MM-DD.md`, optionally emit verbose JSONL under `memory/.dreams/deep/`, and expose an aggregate gate-reachability diagnostic through a CLI command or existing memory status/doctor surface.
