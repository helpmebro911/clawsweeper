## Summary

Keep open. Current main has adjacent Z.AI thinking controls, GLM catalog/docs updates, model-special-token cleanup, and generic empty OpenAI-compatible retry work, but I found no source, test, docs, or history evidence that the reported GLM-4.7 garbled visible-output/session-corruption path is fixed, especially for the reporter's Cloudflare AI Gateway to GLM routing chain.

## Best Possible Solution

Keep this issue open for maintainer investigation. Reproduce on current main with direct `zai/glm-4.7` and separately with the reporter's Cloudflare AI Gateway to GLM setup; capture outgoing request payloads, raw stream chunks, affected session JSONL, and exact model/provider config. If direct Z.AI fails, fix the Z.AI replay/stream/thinking path with regression coverage. If only the Cloudflare route fails, document that GLM should use the direct `zai` provider or add explicit supported handling for that route. Consolidate with #64262 only after the same root cause is confirmed.
