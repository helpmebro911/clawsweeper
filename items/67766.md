## Summary

Keep this PR open. Current main still resolves image support only from the gateway model catalog plus narrow provider shims, so catalog-miss models with explicit `models.providers.*.models[].input: ["text", "image"]` are still treated as not image-capable. The linked bug reports remain open, and the related closed Qwen-only PR was superseded by this broader explicit-config approach rather than replacing it.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate for the explicit provider-config image-capability fallback. The best path is maintainer review of the override semantics, then either land a focused gateway fix with the existing regression tests or request a narrower variant that still respects deliberate user `input` metadata when the selected provider/model is missing from the gateway catalog.

## Risks / Open Questions

- The main product decision is whether raw explicit provider config should override catalog absence when a provider plugin intentionally suppresses a model for a specific endpoint. Maintainers should confirm that behavior before landing, especially for Qwen Coding Plan URLs.
- The fallback should preserve catalog-hit precedence and provider alias normalization; the PR appears to add tests for those cases, but those are the key review points before merge.
