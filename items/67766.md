## Summary

This PR should remain open. Current main still resolves gateway image capability from the gateway model catalog plus narrow Foundry/claude-cli shims, with no fallback to explicit `models.providers.*.models[].input` metadata when the selected provider/model is missing from the catalog. The PR remains a focused implementation candidate for the linked WebChat/custom-provider image-drop bug, though maintainers should reconcile it with the overlapping broader work in #70824 before landing.

## Best Possible Solution

Keep this PR open as a focused implementation candidate for selected-provider explicit config fallback. The maintainer path is to rebase it on current main, preserve catalog-hit precedence and alias-aware provider matching, add or keep targeted regression coverage for explicit `models.providers` image metadata, and reconcile it with #70824 and the linked open issues before landing or closing one path as superseded.

## Risks / Open Questions

- This PR may not fully fix the Qwen Coding Plan reproduction if the normalized runtime config snapshot has already removed `qwen3.6-plus`; maintainers should verify the source/runtime config materialization path before landing against #67759.
- There is active overlap with #70824. Maintainers should choose one resolver semantic rather than landing competing fallback paths.
- The fallback trusts user-authored `input` metadata. A misdeclared text-only model would likely fail later with a provider API error instead of silently dropping or offloading the image.
