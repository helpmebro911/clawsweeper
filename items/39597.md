## Summary

Keep open. Current main already transports client metadata on Gateway connect, and Mac native/browser clients use distinguishable IDs, but that metadata is not exposed as trusted agent runtime context such as `client=macapp`. The agent runtime line still only includes channel/capabilities, while chat dispatch maps the caller ID into sender metadata instead.
