## Summary

Current `main` does not give strong evidence for auto-close. The reported doubled bootstrap path is still plausible in code when the UI is loaded from `/__openclaw__/`, and the two linked fix PRs were closed unmerged for queue-limit reasons rather than because the issue was already resolved. Docs now point users to `/` or a configured `gateway.controlUi.basePath`, so this may also be an unsupported-route/UX problem, but that ambiguity is exactly why this should stay open for maintainer judgment instead of cleanup-closing.
