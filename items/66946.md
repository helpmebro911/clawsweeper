## Summary

Current main still contains the Control UI bootstrap URL composition that can produce the reported duplicated `__openclaw__` request, and the related fix PRs were closed unmerged. This remains a concrete, recent behavior bug rather than a close candidate.

## Best Possible Solution

Keep this issue open and land a focused core Control UI fix. The likely path is to make the bootstrap config endpoint basePath-relative, update the gateway contract, UI bootstrap loader, Vite dev stub, gateway/UI tests, and public Control UI docs, with explicit coverage for `gateway.controlUi.basePath: "/__openclaw__"` and a short-lived compatibility route if needed.
