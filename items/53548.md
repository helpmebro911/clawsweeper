## Summary

Current main still has the requested coupling: unthreaded `mode="session"` is rejected in both native subagent and ACP spawn paths, tests assert that rejection, and the public docs still document `thread: true` as required. The discussion describes a real remaining product gap, not an obsolete or already-shipped behavior.

## Best Possible Solution

Keep this open. The best fix is to decouple session persistence from thread binding in `src/agents/subagent-spawn.ts` and `src/agents/acp-spawn.ts`: allow `mode="session"` without `thread`, create persistent native/ACP child sessions, route initial and completion output back through the parent/announce path when there is no thread binding, return stable `childSessionKey` and label metadata for `sessions_send`, preserve existing `thread=true` behavior, and update regression tests plus the public ACP/subagents docs.
