## Summary

Current main still rejects explicit `mode="session"` without `thread: true` in both native subagent and ACP spawn paths. The current tests and public docs still encode that requirement, and the merged diagnostics work only made the error more actionable. The open #39372 PR remains an implementation candidate, but the requested behavior has not landed on main.

## Best Possible Solution

Keep this issue open. The best path is to review and harden #39372 or a replacement PR that decouples persistence from thread binding in `src/agents/subagent-spawn.ts` and `src/agents/acp-spawn.ts`, allows `mode="session"` without `thread`, creates persistent native and ACP child sessions, returns stable `childSessionKey`/label metadata for `sessions_send`, delivers initial/completion output through the parent or originating conversation when no thread binding exists, preserves current `thread=true` behavior, and updates tests plus `docs/tools/subagents.md` and `docs/tools/acp-agents.md`.
