## Summary

Keep open. Current main has partial `google-vertex` metadata and auth-marker detection, but it still lacks the PR's central dedicated Vertex runtime provider, ADC-to-Bearer token exchange, and Vertex project/location request URL routing. The PR is also paired with open related work, including #60736 and #58775, so it remains an implementation candidate rather than conservative cleanup-close material.

## Best Possible Solution

Keep this PR open for normal maintainer review, or replace it with a maintainer-owned Google plugin implementation that gives `google-vertex` exact provider ownership, removes the legacy generic-Google alias collision, resolves project/location/baseUrl deterministically, exchanges supported ADC credentials into Bearer auth with clear unsupported-credential errors, builds the Vertex `/v1/projects/{project}/locations/{location}/publishers/google/models/...` URL, preserves the already-shipped Gemini thought-signature replay behavior, and adds focused Google Vertex auth/routing tests plus docs for the supported setup path. After merge, let #60736 and #58775 close through the implementation.
