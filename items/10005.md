## Summary

Keep #10005 open. Current main still has no `session.statusCard`, `agents.defaults.statusCard`, JSON-only, or equivalent suppression setting for chat `/status` or the `session_status` tool. The related #13450 CLI `openclaw status --json` work is a separate CLI diagnostics surface, and #69445 was closed as superseded by this issue, so #10005 remains the canonical tracker.

## Best Possible Solution

Keep #10005 open until OpenClaw adds an explicit, documented status-card visibility control for chat `/status` and `session_status`, or a maintainer decides not to support it. A complete implementation should preserve current rendering by default, add schema/types/help/docs for the chosen key, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, and add focused tests for both chat-command and agent-tool paths. The CLI `openclaw status --json` surface should remain documented as separate from chat/tool status-card suppression.
