## Summary

Keep #10005 open. Current main still has no `session.statusCard`, `agents.defaults.statusCard`, JSON-only, or equivalent suppression setting for chat `/status` or the `session_status` tool. The related closed #13450 CLI `openclaw status --json` surface is implemented, but it does not cover the requested chat/tool status-card control.

## Best Possible Solution

Keep #10005 open until OpenClaw adds an explicit status-card visibility control for chat `/status` and `session_status`, or a maintainer decides not to support it. A complete fix should preserve current card rendering by default, add schema/types/help/docs for the chosen key, thread the setting through `buildStatusReply`, `buildStatusText`, `buildStatusMessage`, and `session_status`, and add focused tests for both chat-command and agent-tool paths while documenting CLI `openclaw status --json` as a separate surface.
