## Summary

Keep open. Current `main` supports Discord native reply threading via `message_reference`, but the requested `channels.discord.autoMentionReply` setting and automatic `<@userId>` injection are not present in config, docs, or outbound send logic.

## Evidence

- **Docs only describe native reply references:** Discord docs list `channels.discord.replyToMode` (`off|first|all|batched`) and describe attaching native reply references; there is no documented auto-mention reply option.
  - file: [docs/channels/discord.md:519](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/channels/discord.md#L519)

- **Config schema has no autoMentionReply field:** The Discord config schema includes `replyToMode` and adjacent Discord settings, but no `autoMentionReply` or similar outbound reply-mention toggle.
  - file: [src/config/zod-schema.providers-core.ts:566](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/config/zod-schema.providers-core.ts#L566)

## Risks / Open Questions

- Discord already supports native reply references, so maintainers may decide explicit visible `@mention` injection is a product choice rather than a bug fix.
- Implementing this would require resolving the replied-to author at send time and defining when to suppress injection for DMs, existing mentions, multi-chunk replies, and explicit `[[reply_to:*]]` cases.
