## Summary

Keep open: current `main` still lacks the proposed remote-node-host guard. `openclaw models auth login` persists auth to the local agent/config state, while current docs say remote-mode auth profiles live on the gateway host and node hosts are only peripherals for forwarded execution. The PR is addressing a still-plausible bug and is not implemented on `main` yet.
