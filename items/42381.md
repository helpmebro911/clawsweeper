## Summary

Keep this PR open. Current main still runs provider auth and persists returned OAuth profiles into the resolved local agent auth store, with no guard for a node host pointed at a remote gateway. The linked bug remains open, and this PR is still a focused implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep this PR open, rebase it onto current main, and review/land a focused remote node-host guard in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`. Prefer the shared `src/gateway/net.ts:isLoopbackHost` helper and confirm the intended auth-method scope; after merge, #42291 can close as implemented.
