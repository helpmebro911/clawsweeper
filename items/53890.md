## Summary

Keep this issue open. Current main still has inbound per-topic Telegram agent routing and explicit outbound topic targeting, but it does not implement a config-level per-agent default outbound topic/thread binding. The prior ClawSweeper keep-open review remains accurate against c2d82b87eee7a86bbd0e851169e0ca5559c44fe6, and the related reports cover different Telegram command, account default, Discord thread, or ACP media-binding work rather than this feature.

## Best Possible Solution

Keep this issue open. If maintainers want the feature, implement it in the bundled Telegram plugin/config surface with a typed outbound-default topic binding, zod schema and generated config metadata/help/docs updates, and focused tests. The conservative design should keep the behavior Telegram-owned and define precedence as explicit threadId or :topic: target first, current conversation/tool context next, then configured per-agent outbound default.
