## Summary

Keep this issue open. Current main still supports Telegram per-topic inbound agent routing, explicit `threadId` or `:topic:` outbound targets, and current-context auto-threading, but it does not provide a config-level per-agent default outbound topic/thread binding. The prior ClawSweeper comment is still accurate against current main.

## Best Possible Solution

Keep this open and implement the feature in the Telegram plugin/config surface if maintainers want it. A conservative implementation should add typed config/schema/docs/tests for per-agent outbound topic defaults, define precedence such as explicit `threadId` or `:topic:` target first, then current conversation context, then configured per-agent default, and wire it through Telegram-owned outbound routing without adding Telegram-specific policy to generic core.
