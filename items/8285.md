## Summary

Keep #8285 open. Current main still has reaction and typing/status acknowledgment paths, but no supported configurable pre-processing text acknowledgment that sends a visible text reply and then continues normal agent processing across the requested channels. The issue discussion is concrete and has a second Feishu user report, plus a same-day maintainer/Codex review already concluded this should remain open.

## Best Possible Solution

Keep tracking this as a core/product or plugin-API feature request. The best path is to add a supported pre-processing text acknowledgment contract, either as `messages.ackText`/scope with channel/account overrides or as a documented non-terminal plugin hook, then update config types, zod schema, docs, and channel tests so Slack, WhatsApp, webchat, and Feishu send the acknowledgment immediately while preserving normal agent processing and final replies.
