## Summary

Keep #8285 open. Current main still does not provide a supported configurable text acknowledgement before normal agent processing. It has reaction acknowledgements, status/typing indicators, and terminal hook paths, but no `messages.ackText`-style config or public non-terminal send-and-continue plugin hook.

## Best Possible Solution

Keep #8285 open as a core/product or plugin-API feature request. The best path is to add a supported non-terminal pre-processing acknowledgement contract, likely `messages.ackText` with scope and per-channel/account overrides, or a documented plugin/channel hook that can send interim text and then continue normal model dispatch. Update config types, zod schema, generated schema/help/labels, public docs, and channel tests for affected messaging surfaces.
