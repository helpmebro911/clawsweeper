## Summary

Keep #39013 open. Current main still lets hook-mutated bootstrap files shaped like `{ path, content, virtual: true }` pass through with no synthesized `name`, then `buildBootstrapInjectionStats` returns `name: file.name` and `/context list` renders that field directly. The earlier display-fix PRs referenced from the issue were closed unmerged, and the current regression test coverage still only exercises bootstrap files with explicit names.

## Best Possible Solution

Keep the issue open. First fix `buildBootstrapInjectionStats` to derive a stable label from `file.name`, then `path.basename(file.path)`, then `file.path`, then an explicit placeholder, and add a regression test for a hook-injected virtual bootstrap file with `path` and no `name`. After that, recheck session-memory indexing/accumulation and either exclude injected bootstrap/system content from transcript recall or split that behavior into a narrower follow-up.
