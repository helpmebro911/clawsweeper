## Summary

Keep this PR open. Current main still lacks a top-level `auth` reload rule, so `auth.profiles.*` changes still fall through to the conservative unknown-path restart behavior. The PR is a focused implementation candidate paired with open bug report #68258, and its diff is limited to the gateway reload planner plus regression tests.

## Best Possible Solution

Review and land this PR or an equivalent patch on main: add a top-level `{ prefix: "auth", kind: "none" }` reload rule, keep existing `gateway.auth.*` restart behavior, and add regression tests for `auth.profiles.*` no-op handling plus mixed `gateway.auth`/`auth.profiles` batches. After merge, #68258 can close as implemented.
