## Summary

Keep open. Current main still preserves the two behaviors implicated by the report: wake-triggered heartbeats can prefer an originating forced session before `heartbeat.session`, and default Telegram DMs still route to `agent:main:main`. The provided GitHub context also lists #56092 as an open, unmerged PR with GitHub closing syntax for this issue, so the issue should remain open while that implementation candidate is reviewed or closed explicitly.

## Best Possible Solution

Keep this issue open while maintainers review #56092 and either land it or close it with explicit rationale. Separately decide the Telegram product behavior: either keep and document default DM-to-main routing as intentional, or implement Telegram DM isolation in the Telegram plugin with matching tests and channel-routing docs.
