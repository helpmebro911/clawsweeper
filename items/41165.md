## Summary

Keep open. Current main still contains the heartbeat forced-session precedence and Telegram default DM-to-main behavior implicated by the report, and an open unmerged PR (#56092) still explicitly targets this issue.

## Best Possible Solution

Keep this issue open while maintainers review/land #56092 or close that PR with an explicit rationale. The narrow code path to settle first is heartbeat session resolution: either heartbeat.session should outrank an originating forced session for wake-triggered runs, or the current precedence should be documented. If maintainers also want Telegram DMs isolated by default, update the Telegram routing code, tests, and channel-routing docs together.
