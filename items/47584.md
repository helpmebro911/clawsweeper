## Summary

Keep open. Current main has adjacent iOS voice wake, deep-link, and `/hooks/agent` plumbing, and the iOS target links AppIntents.framework, but there is no native Swift AppIntent/AppShortcutsProvider/ProvidesDialog implementation for Siri to accept a question and return a spoken OpenClaw reply. The related open PR #40874 overlaps with Action Button/App Intent launch-to-Talk-Mode work, but it is unmerged and does not clearly supersede this issue's direct Siri question/reply request.
