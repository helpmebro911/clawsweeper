## Summary

PR #71645 should stay open. Current main still sends Mattermost tool-start status through the active draft stream without flushing pending partial assistant text or rotating to a new post, so the reported overwrite path remains present and is not implemented or superseded on main.

## Best Possible Solution

Keep PR #71645 open for maintainer review. The best path is to preserve Mattermost streamed assistant text across tool-start events using the existing draft-stream lifecycle, add a focused regression test for the partial-text-then-tool-start sequence, and update the Mattermost preview-streaming docs if maintainers accept the separate status-post behavior.
