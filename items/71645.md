## Summary

PR #71645 is a focused Mattermost bug fix that is not implemented on current main. The current Mattermost tool-start handler still updates the active draft preview with tool status without flushing streamed assistant text or rotating to a new post, so the reported overwrite path remains plausible and source-backed.

## Best Possible Solution

Keep PR #71645 open for maintainer review. The best path is to preserve Mattermost streamed assistant text across tool-start events using the existing draft stream lifecycle, add a focused regression test for partial text followed by tool start, and update the Mattermost preview-streaming docs if the accepted behavior intentionally creates a separate status post.
