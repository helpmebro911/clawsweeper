## Summary

Keep open. Current main still hardcodes the leading agent identity line and does not expose `agents.defaults.preamble`, `agents.list[].preamble`, `identityLine`, or an equivalent narrow preamble setting. The available `systemPromptOverride` is a full prompt replacement, not the requested configurable first line. Related PR #44497 was closed unmerged, and open PR #48442 is only an implementation candidate with unresolved schema/runtime wiring concerns.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and repair #48442 or implement a narrower core change that lets `agents.defaults` and `agents.list[]` override only the leading identity line/preamble while preserving the assembled OpenClaw prompt. The implementation should update config types, strict Zod schemas, generated schema/help docs, embedded runner, compaction, CLI runner, command-system-prompt call sites, and focused tests for SOUL.md persona interaction plus per-agent/default precedence.
