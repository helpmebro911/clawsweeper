## Summary

Keep PR #50483 open. Current main still has the shared iOS/macOS chat streaming surfaces that the PR targets: streaming parser segments use fresh UUID identities, agent events emitted by the gateway can include sessionKey but the Swift agent-event model drops it, `ChatViewModel.handleAgentEvent` still filters only by sessionId/runId shape, and streaming scroll updates fire on every token. The related PR #40985 is closed unmerged and the provided PR discussion says #50483 supersedes it with broader session-flow/layout fixes, so this is not an obsolete duplicate.

## Best Possible Solution

Keep #50483 open for maintainer review. The best path is to reconcile the PR's shared `OpenClawChatUI` changes against current main, preserve the targeted parser/view-model tests, and explicitly settle the pending-run/session-key concurrency behavior before merge. If maintainers prefer a narrower landing, split the clearly independent stable parser IDs, scroll debounce, bootstrap/session-switch cleanup, and agent-event sessionKey decoding into smaller reviewed PRs.
