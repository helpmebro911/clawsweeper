## Summary

Keep open. Current main still lacks the PR's core shared chat UI changes: agent stream events are filtered by run/session id without a sessionKey fallback, parser segment IDs are still UUID-based, and streaming scroll updates are still immediate. The PR is the active follow-up to older closed flicker PRs, not superseded by them. It also still needs maintainer/contributor follow-through because the latest head has unresolved session/run scoping concerns and an apparent stale regression test after the final gating change.

## Best Possible Solution

Keep #50483 open as the active focused bug-fix PR. The best path is to clarify the desired live-stream scoping contract for concurrent clients and session switches, update shouldAcceptAgentEvent and the regression tests to match that contract, then rerun the shared OpenClawKit Swift tests and iOS/macOS build validation before merge. If maintainers want a smaller merge, split stable parser IDs and scroll debouncing from the riskier session-scoping changes.
