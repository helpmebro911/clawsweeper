## Summary

Current main still does not implement the PR's central ACP behavior: `sessions_spawn(runtime="acp")` dispatches the initial task to the agent call, but there is no prompt-only transcript seed before the first ACP turn completes. The PR remains a plausible implementation candidate for #43518, especially given the linked empirical reproduction, but it should be rebased and narrowed because it also carries unrelated CI/workflow changes that need separate security/maintainer review.

## Best Possible Solution

Keep this PR open as a real implementation candidate, but ask the contributor to rebase and narrow it to the ACP transcript behavior: seed the initial `sessions_spawn(runtime="acp")` task into the child transcript immediately, pass provenance consistently, dedupe only the seeded replay, and keep focused regression tests. Split or drop the CI/workflow/labeler changes into a separate maintainer-reviewed PR.
