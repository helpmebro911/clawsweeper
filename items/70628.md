## Summary

Keep open. Current main still implements the reported Telegram DM no-visible-response path by synthesizing a `NO_REPLY` fallback and sending the projected payload as a visible, non-silent delivery. The existing regression test still expects that behavior, and an open PR (#70630) is the active fix path rather than shipped code.

## Best Possible Solution

Keep #70628 open and review or land #70630, or an equivalent patch, that treats successful Telegram DM no-visible-response turns as handled silence: do not synthesize a transport-visible `NO_REPLY`, keep real error/non-silent delivery failure fallback behavior intact, finalize status as a normal handled turn, and flip the Telegram dispatch regression test to assert no chat message is sent.
