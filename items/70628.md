## Summary

Keep open. Current main still synthesizes and delivers a Telegram direct-message `NO_REPLY` fallback for successful no-visible-response turns, the Telegram regression test still asserts the unwanted visible fallback behavior, and open PR #70630 from the same author remains unmerged with a targeted fix for this issue.

## Best Possible Solution

Keep this issue open and review or land PR #70630, or apply an equivalent narrow Telegram adapter fix. The implementation should treat successful no-visible-response Telegram direct turns as handled silent no-ops, avoid synthesizing a transport-visible `NO_REPLY`, preserve the real dispatch-error and non-silent-delivery-failure fallback paths, and update the Telegram dispatch regression test to assert no DM delivery for this case.
