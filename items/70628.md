## Summary

Keep open. Current main still matches the reported bug: Telegram no-visible-response turns synthesize `NO_REPLY`, project it through direct-chat silent-reply rewrite policy, and deliver the visible fallback. The linked PR #70630 appears to be the right active fix path, and maintainer discussion already confirms the bug remains on main.

## Best Possible Solution

Keep #70628 open and land PR #70630 or an equivalent Telegram adapter patch that treats successful direct-message no-visible-response turns as handled silent no-ops, preserves the real error fallback path, and flips the current regression test to assert no chat message is sent.
