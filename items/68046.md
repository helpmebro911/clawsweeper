## Summary

Keep this PR open. Current main still strips non-text MCP tool-result fields in the HTTP MCP gateway, and this PR is a focused implementation candidate for the still-open linked bug. It should be revised to validate known MCP content block shapes before passthrough.

## Best Possible Solution

Keep this PR open and treat it as the implementation candidate for #68037. The best path is to update `normalizeToolCallContent` so it preserves only valid MCP text/image/audio/resource/resource_link blocks, falls back for malformed entries, and adds focused regression coverage on the HTTP `tools/call` path before maintainer review and merge.
