## Summary

Keep #8959 open. Current main has a shared single-select presentation block, plus adjacent poll/wizard/provider-native multiselect support, but it still lacks the requested portable message-tool multi-select contract with min/max values and one structured selected-values callback across Slack, Discord, Telegram, and fallback text paths.

## Best Possible Solution

Keep the issue open until OpenClaw adds an additive portable batch-selection contract in the shared message presentation/message tool surface, exports it through openclaw/plugin-sdk/interactive-runtime, documents it at docs.openclaw.ai/plugins/message-presentation, and implements focused render/callback handling for Slack, Discord, Telegram, and fallback text paths.
