## Summary

Keep this open. Current main still routes Mattermost preview streaming through one draft post, disables core block-streaming delivery for that run, and only clears local partial text at assistant-message boundaries instead of finalizing or rotating the Mattermost draft post. The direct fix attempts in the provided context, #43041 and #46330, are closed unmerged.

## Best Possible Solution

Keep the issue open and fix it in the Mattermost plugin path. At assistant-message/tool boundaries, the preview streaming path should preserve the completed streamed segment by flushing/finalizing or rotating the draft post before the next segment updates the preview, while still suppressing duplicate final payloads. Add a focused Mattermost regression test that simulates text, a tool boundary/new assistant message, more text, and final delivery.
