## Summary

Keep open. Current main still has the generic pairing-store diagnostics this PR tries to improve, and the related batch PR #47409 was closed unmerged rather than superseding or shipping the change. The PR likely needs a rebase/adaptation because the key-sanitization helpers now live in `src/pairing/allow-from-store-file.ts`, and the existing review comment about `JSON.stringify` in error construction should be addressed before merge.

## Best Possible Solution
