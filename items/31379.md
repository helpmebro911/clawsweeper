## Summary

Current main still has the reported Control UI behavior: `renderChannels()` renders exactly the ids returned by `resolveChannelOrder()`, and `resolveChannelOrder()` returns custom-only `channelMeta` or `channelOrder` without appending the built-in channel cards. The focused fix PR #44165 is still open in the provided GitHub context, while earlier PR #31350 was closed unmerged, so this issue should remain open.

## Best Possible Solution

Keep #31379 open until #44165 or an equivalent focused Control UI fix lands. The likely implementation belongs in `ui/src/ui/views/channels.ts`: merge snapshot-provided custom ids with the built-in channel id list, preserve snapshot/custom order first, de-dupe ids, keep the existing enabled-channel promotion sort, and add regression coverage in `ui/src/ui/views/channels.test.ts`.
