## Summary

Current main still has the reported Control UI behavior: channel cards are rendered from `channelMeta` or `channelOrder` exactly, and the backend builds those arrays from loaded channel plugins only. The provided related context shows focused PR #44165 is still the open fix attempt, while PR #31350 was closed unmerged, so this issue should remain open until a fix lands.

## Best Possible Solution

Keep #31379 open until #44165 or an equivalent focused Control UI fix lands. The likely fix belongs in `ui/src/ui/views/channels.ts`: merge snapshot-provided custom ids with the built-in channel id list, de-dupe while preserving snapshot/custom order first, keep the existing enabled-channel promotion sort, and add regression coverage in `ui/src/ui/views/channels.test.ts` for custom-only `channelMeta`, custom-only `channelOrder`, and the no-snapshot fallback.
