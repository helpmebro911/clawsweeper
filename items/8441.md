## Summary

Keep #8441 open. Current main still does not support `skills.entries.<skill>.thinking` or `.model` as per-skill turn overrides; the schema rejects those top-level keys, command specs do not carry them, and skill invocation does not apply them. The related implementation PR #8455 is closed unmerged, and recent comments show the request is still active.

## Best Possible Solution

Keep tracking this issue. A good implementation would extend the public `SkillConfig` type and strict Zod schema, carry optional `model` and `thinking` defaults through `SkillCommandSpec` generation, apply them during skill invocation as defaults that respect explicit user directives, and update focused config/runtime tests plus `docs/tools/skills-config.md`.
