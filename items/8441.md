## Summary

Keep #8441 open. Current main still does not implement top-level `skills.entries.<skill>.thinking` or `skills.entries.<skill>.model`: the typed config and strict schema omit those fields, tests still enforce rejection of unknown top-level skill-entry keys, docs only document `enabled`/`env`/`apiKey` plus nested custom `config`, and the skill command invocation path has no model/thinking override plumbing. The prior PR #8455 was closed unmerged, and the latest maintainer-thread comment also kept this open.

## Best Possible Solution

Keep #8441 open. Implement this in core skills config and invocation plumbing: extend `SkillConfig` and `SkillEntrySchema` with validated optional `model` and `thinking`, carry those values through `SkillCommandSpec` or equivalent invocation context, apply them as per-skill defaults below explicit user/session/agent overrides, add focused schema and invocation tests, and update the public skills config docs.
