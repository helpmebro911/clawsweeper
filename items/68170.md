## Summary

Keep open. Current main still implements the reported broad sibling-home scan: Doctor enumerates `/home` on Linux and `/Users` on macOS, probes each non-dot sibling for `.openclaw`, and prints discovered paths in the multiple-state-dir warning. I found no current-user scoping, backup-name filtering, redaction for non-current-home paths, or env/config opt-out in the relevant source, tests, or Doctor docs. The related #68752 is closed and corroborates the same noisy multi-user-host behavior, but it does not supersede this still-reproducible issue.

## Best Possible Solution

Keep this issue open and patch the Doctor state integrity check in `src/commands/doctor-state-integrity.ts`. A conservative fix should constrain discovery to the current effective home/service user or remove broad sibling-home enumeration, avoid printing non-current-user state paths verbatim, add an explicit env/config suppression if maintainers want the diagnostic to remain optional, and add focused tests for `/home/alice`/`/home/bob` plus backup-sibling directory cases. Update the public Doctor docs if scan semantics or operator controls change.
