## Summary

Keep #48320 open. The current shared Windows spawn helper still omits `windowsHide` on the direct `.exe` and shell-fallback outcomes that this PR targeted, and downstream ACP/Codex/Docker callers forward that value into `spawn()`. The PR's ACPX test path is stale after refactors, but the core behavior remains unresolved rather than implemented or superseded.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main. The core patch should now target `src/plugin-sdk/windows-spawn.ts` plus `src/plugin-sdk/windows-spawn.test.ts`, adding explicit coverage for Windows direct `.exe` resolution and deciding/locking the shell-fallback `windowsHide: true` behavior. ACPX-specific test changes should be refreshed against the current ACPX wrapper layout or dropped if no longer relevant.
