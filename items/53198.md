## Summary

Keep this issue open. Current main at c2d82b87eee7a86bbd0e851169e0ca5559c44fe6 still does not implement the documented Discord elevated fallback from `channels.discord.allowFrom`, `sandbox explain` still computes elevated allowlists only from `tools.elevated.allowFrom.<channel>`, and #53259 remains an open implementation candidate rather than a merged fix.

## Best Possible Solution

Keep this issue open and review, revise, land, or close #53259. The right fix belongs in the Discord/channel-owned elevated fallback path plus the shared diagnostics path: wire `channels.discord.allowFrom` as the fallback when `tools.elevated.allowFrom.discord` is omitted, make `sandbox explain` use the same effective allowlist logic, and add regression coverage for omitted fallback, explicit empty override, and diagnostics consistency.
