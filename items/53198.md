## Summary

Keep this issue open. Current main still does not implement the documented Discord elevated fallback from `channels.discord.allowFrom`, `sandbox explain` still computes elevated authorization only from `tools.elevated.allowFrom.<channel>`, and open PR #53259 is an unmerged implementation candidate rather than a reason to close before merge.

## Best Possible Solution

Review and either land, revise, or close #53259. The fix should add a Discord-owned `channels.discord.allowFrom` fallback to the elevated runtime path, make `sandbox explain` use the same effective allowlist resolution, and add regression coverage for omitted `tools.elevated.allowFrom.discord`, explicit empty override behavior, and diagnostics consistency.
