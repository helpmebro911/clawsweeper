## Summary

Keep open. Current main has newer ACP task-completion plumbing, but the specific `streamTo="parent"` direct-chat final-output behavior in this PR is still not implemented: the normal completion-announce path explicitly excludes `streamTo="parent"`, child inline delivery is disabled for that path, and the parent relay still emits internal system events rather than routing the child’s final visible answer back to a non-threaded direct chat. The PR likely needs a current-main rework, especially around the existing task notifier and the whitespace-delta review concern, but it is not obsolete enough to close.

## Best Possible Solution

Keep the PR open, but ask for a current-main rework: implement final visible output delivery for `streamTo="parent"` non-threaded direct chats through the current ACP/task completion architecture, preserve existing threaded behavior, avoid duplicate background-task notices, retain best-effort failure semantics, and add regression tests for direct-chat success, threaded suppression, errors, truncation, and whitespace-only delta preservation.
