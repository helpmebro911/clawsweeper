## Summary

Keep this issue open. Current main still plans generated `models.json` by resolving plugin-owned implicit provider catalogs before applying `models.mode`, and the Codex and Arcee bundled provider paths called out by the report still exist. The narrower #66957 tracks replace-mode implicit discovery and startup delay, but this issue also asks for a broader maintainer decision about whether bundled/provider-owned catalogs should appear without onboarding or explicit config.

## Best Possible Solution

Keep this issue open for maintainer judgment on the generated `models.json` contract. A concrete implementation path is to short-circuit `models.mode === "replace"` before `resolveImplicitProviders`, add regression coverage proving replace mode does not run implicit discovery, and separately decide whether merge/default mode should document provider-owned bundled catalogs as intentional or gate Codex/Arcee catalog hooks on explicit activation, onboarding, auth, or config intent.
