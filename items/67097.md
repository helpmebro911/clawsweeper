## Summary

Keep this issue open. Current main still resolves implicit plugin provider catalogs before `models.mode` handling, and the reported Codex and Arcee catalog paths still exist. The related #66957 is only the narrower `models.mode: "replace"` delay/implicit-discovery bug; this issue also asks for a broader maintainer decision about whether bundled/provider-owned catalogs should appear in generated `models.json` without explicit onboarding or config.

## Best Possible Solution

Keep this open for maintainer judgment on the generated `models.json` contract. A concrete fix path is to short-circuit `models.mode: "replace"` in `src/agents/models-config.plan.ts` before `resolveImplicitProviders`, add regression coverage for that behavior, and separately decide whether merge/default mode should document provider-owned bundled catalogs as intentional or gate Codex/Arcee catalog hooks on explicit activation, onboarding, or auth/config intent.
