## Summary

Current main still leaves the reported heartbeat delivery path unresolved: `buildReplyPayloads` passes heartbeat payloads through unchanged, and the shared sanitizers/tests cover XML and downgraded `[Tool Call: ...]` text rather than uppercase `[TOOL_CALL]...[/TOOL_CALL]` / `[TOOL_RESULT]...[/TOOL_RESULT]` blocks. This issue also has an open same-author implementation PR with closing syntax, so it should stay open until that PR is reviewed and landed or replaced.

## Best Possible Solution

Keep this issue open and review PR #54165, or land an equivalent fix in `src/auto-reply/reply/agent-runner-payloads.ts` with focused tests in `src/auto-reply/reply/agent-runner-payloads.test.ts` proving heartbeat replies strip uppercase `[TOOL_CALL]` and `[TOOL_RESULT]` bracket-format blocks. If maintainers confirm the interactive/XML reports are a broader stream handoff regression, track that separately instead of closing this heartbeat-specific issue early.
