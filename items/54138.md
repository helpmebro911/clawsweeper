## Summary

Current main still does not prove the heartbeat-specific bracket-block leak is fixed. The heartbeat branch in `buildReplyPayloads` still passes payload text through, the final delivery path still calls it with `isHeartbeat`, and the focused payload tests have no `[TOOL_CALL]` / `[TOOL_RESULT]` heartbeat regression case. The same-author PR #54165 remains the implementation candidate from the provided context, so this issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/land PR #54165, or land an equivalent current-main fix in `src/auto-reply/reply/agent-runner-payloads.ts`. The fix should strip uppercase `[TOOL_CALL]...[/TOOL_CALL]` and `[TOOL_RESULT]...[/TOOL_RESULT]` blocks from heartbeat replies before channel delivery, with focused coverage in `src/auto-reply/reply/agent-runner-payloads.test.ts`.
