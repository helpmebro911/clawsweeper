## Summary

Keep open. Current main has adjacent mitigations for Codex/provider error sanitization and fallback signaling, but the repo does not provide strong evidence that the reported raw tool-payload leak is fully fixed end-to-end. The Control UI tool-card path still renders tool output largely verbatim, and web-fetch tests still expect `<<<EXTERNAL_UNTRUSTED_CONTENT ...>>>` wrappers in tool payload text, so the reported leakage remains plausible.
