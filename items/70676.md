## Summary

Keep #70676 open. Current main still has the exact silent revocation failure paths the PR describes: shared-auth eviction and device-removal eviction both catch `socket.close()` failures and ignore them without logging or `terminate()` fallback. The PR discussion also shows a concrete review concern was addressed in the branch, so this remains plausible useful work rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open. The best path is to land or reapply a narrow gateway fix that logs revocation close failures, force-terminates the affected WebSocket as a fallback, threads the gateway logger through all shared-auth revocation call sites, and adds unit coverage for throwing `socket.close()` on device-removal and shared-auth revocation paths.
