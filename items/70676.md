## Summary

Keep #70676 open. Current main still has the exact silent gateway revocation paths described by the PR: shared-auth stale/all disconnects and device removal call `socket.close(...)` inside bare `catch {}` blocks with no warning and no `terminate()` fallback. The proposed PR remains a focused, plausibly useful gateway auth hardening change rather than an obsolete or already-implemented patch.

## Best Possible Solution

Keep the PR open or port its focused fix into main: add log-then-terminate fallback for shared-auth and device-token revocation close failures, thread bound logger callbacks through all existing call sites, and keep the proposed regression tests for throwing `socket.close()` and throwing `terminate()` so the eviction loop continues.
