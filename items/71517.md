## Summary

Keep PR #71517 open. Current main still has split, limited filename decoding in remote media and Feishu paths, and the PR is a recent, focused implementation attempt for open related bugs rather than obsolete or superseded cleanup.

## Best Possible Solution

Keep the PR open for maintainer review or iteration. The right path is to land a shared `src/media/filename` utility with focused tests, wire `fetchRemoteMedia` and Feishu inbound filenames through it, and leave any remaining cross-channel migration to #48788 or a narrower follow-up.
