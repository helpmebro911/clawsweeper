## Summary

Keep PR #71517 open. Current main still lacks the shared filename decoder, multi-encoding Content-Disposition handling, and Feishu mojibake recovery the PR proposes. The linked issues #48388 and #48788 are still open and describe the same remaining user-visible problem, so this PR is not obsolete, implemented, or superseded.

## Best Possible Solution

Keep PR #71517 open and review or port it as a focused bug fix/refactor: land the shared filename decoder plus Feishu recovery once the SDK surface and tests are acceptable, while leaving #48788 as the broader tracker for migrating remaining channel/plugin filename paths.
