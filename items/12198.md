## Summary

Keep #12198 open. Current main still has path, size, and install-code safety controls, plus web-fetch sanitization, but it does not implement a shared Markdown validation contract for prompt-bearing files such as workspace bootstrap Markdown, extra bootstrap files, or SKILL.md content before those files can influence prompts or model-visible skill reads.

## Best Possible Solution

Keep the issue open until OpenClaw has a shared prompt-bearing Markdown validation contract, with tests and docs, wired into workspace bootstrap ingestion, extra bootstrap files, SKILL.md discovery and model-visible skill read paths, plus native ClawHub skill install or publish policy where applicable. Existing realpath, size, external-content, and dangerous-code scanners can inform the implementation but do not replace this validator.
