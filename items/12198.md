## Summary

Keep #12198 open. Current main still has adjacent prompt-surface hardening, but not the requested shared Markdown validation for prompt files: workspace bootstrap Markdown is read and injected after boundary/size/truncation guards only, SKILL.md discovery parses frontmatter only, ClawHub skill install only requires a SKILL.md marker before copying, and Skill Workshop's scanner is limited to generated proposals and lacks an HTML-comment rule. The existing maintainer/Codex issue comment also explicitly kept this item open for the same missing cross-surface validation.

## Best Possible Solution

Keep this open until OpenClaw has a shared prompt-Markdown validation contract with tests and docs, wired into workspace bootstrap ingestion, extra bootstrap hooks, SKILL.md discovery/on-demand loading, skill install/ClawHub policy, and any plugin API needed for third-party enforcement before prompt Markdown reaches the model. Start with high-confidence blockers such as HTML comments and define warn/block/customization behavior explicitly.
