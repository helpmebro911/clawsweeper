## Summary

Keep #12198 open. Current main still lacks a shared Markdown validation contract for prompt-bearing Markdown. The existing code has path, size, install-code, Skill Workshop generated-content, and web/external-content controls, but those do not validate workspace bootstrap Markdown, extra bootstrap files, ordinary SKILL.md discovery/read paths, or native ClawHub skill installs. The existing Apr 26 Codex review comment posted by steipete reaches the same conclusion and remains consistent with the current checkout.

## Best Possible Solution

Keep the issue open until OpenClaw has a shared prompt-bearing Markdown validation contract with tests and docs. Wire it into workspace bootstrap ingestion, extra bootstrap files, SKILL.md discovery and model-visible read paths, and local ClawHub skill install/update handling; coordinate any publish-side ClawHub policy separately. Existing path guards, size limits, dangerous-code scans, Skill Workshop proposal scanning, and external-content wrapping should inform the design but do not replace this validator.
