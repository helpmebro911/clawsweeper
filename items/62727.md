## Summary

Keep open. Current main still preserves the entire Avatar value from IDENTITY.md, so a descriptive avatar line with a trailing path would be passed downstream as one invalid local path rather than extracting the intended avatar file. The PR is focused and tests the missing behavior, but its current extraction approach needs maintainer review or revision because bot review comments identified path-truncation regressions for legitimate hyphenated avatar filenames.

## Best Possible Solution

Keep the PR open and ask for a small revision: implement descriptive avatar trailing-path extraction in src/agents/identity-file.ts while preserving direct avatar paths/URLs/data URIs exactly, and add regression tests for the new descriptive form plus direct paths containing spaced hyphens and en/em dashes. The best final shape is a narrow parser helper with tests in src/agents/identity-file.test.ts and, if the CLI import behavior is considered user-facing, one agents set-identity regression test that proves the parsed avatar reaches agents.list[].identity.avatar.
