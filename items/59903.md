## Summary

Current `main` does not implement this PR's `/status` behavior. `/status` still renders recent successful tasks as `recently finished` and still hides agent-local recent failures when there are no active tasks, so the PR remains relevant rather than closable. The PR discussion is useful: the author addressed the review-noted regression on the branch (`b4dbcf2c582da6c467b7f210889077dae5d84c9d`), but that branch change is not present on `main` at `a155b84cdadef068de50274318952032cd867788`.

## Evidence

- **Current `/status` path still shows completed-success output:** `buildStatusText()` still uses `formatSessionTaskLine()` for `/status`. That formatter emits `"recently finished"` whenever there are no active tasks and no recent failures, which is exactly the behavior this PR aims to remove.
  - file: [src/status/status-text.ts:91](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/status/status-text.ts#L91)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Agent-local fallback still drops recent failures:** When the current session has no task line, `/status` falls back to `formatAgentTaskCountsLine()`, but that helper only reports active counts and returns nothing unless `totalCount > 0`; it does not preserve the PR's reviewed fix for same-agent recent failures with zero active tasks.
  - file: [src/status/status-text.ts:127](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/status/status-text.ts#L127)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Snapshot logic can still focus a recent successful task:** `buildTaskStatusSnapshot()` uses the first recent terminal task as focus when there are no active tasks and no failures, so a recent successful task remains eligible to populate `/status` detail text on current `main`. That matches the original issue rather than resolving it.
  - file: [src/tasks/task-status.ts:169](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/tasks/task-status.ts#L169)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Current tests do not cover the PR's missing behavior:** The `/status` tests on `main` cover stale completed tasks, current-session failures, and active agent-local fallback, but there is still no test asserting that a recent completed-success task is suppressed or that agent-local recent failures remain visible. The PR discussion added exactly that missing coverage and noted a follow-up fix in commit `b4dbcf2c582da6c467b7f210889077dae5d84c9d`.
  - file: [src/auto-reply/reply/commands-status.test.ts:274](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/auto-reply/reply/commands-status.test.ts#L274)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
