## Summary

Keep PR #40443 open. Current main still does not implement the PR's explicit reload/discard semantics: config reload has no discard option, snapshot application still preserves dirty form state, explicit reload call sites still use the passive load path, and the Agents model select remains driven by config form state. The linked bug #40352 remains unsolved on main, and the provided PR discussion shows a focused fix with review feedback addressed rather than obsolete work.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, preserve the explicit reload versus passive refresh split in `ui/src/ui/controllers/config.ts`, wire Agents/Config/Channels explicit reload actions to discard pending edits, and keep focused regression coverage for form-mode dirty state, raw-mode dirty state, config hash-conflict recovery, and the Agents model dropdown rerender.
