## Summary

Keep open. Current `main` supports Perplexity only as a bundled web-search plugin, not as a first-class LLM/model provider with provider auth choice, model catalog, or model-picker integration.

## Evidence

- **Perplexity manifest only declares web-search capability:** The bundled Perplexity plugin manifest exposes `webSearchProviders` only; there is no model-provider contract in the plugin metadata.
  - file: [extensions/perplexity/openclaw.plugin.json:22](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/extensions/perplexity/openclaw.plugin.json#L22)
  - sha: [7e16e3d07723](https://github.com/openclaw/openclaw/commit/7e16e3d07723cc3903f40ce94f53a3920c88b1fc)
- **Plugin entry only registers a web-search provider:** The plugin runtime calls `api.registerWebSearchProvider(...)` and does not register any model/provider surface.
  - file: [extensions/perplexity/index.ts:8](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/extensions/perplexity/index.ts#L8)
  - sha: [7e16e3d07723](https://github.com/openclaw/openclaw/commit/7e16e3d07723cc3903f40ce94f53a3920c88b1fc)
- **User-facing docs explicitly say this is not a model provider:** The Perplexity docs describe the integration as a web-search provider and explicitly label it `not a model provider`. Setup is via `openclaw configure --section web`.
  - file: [docs/providers/perplexity-provider.md:21](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/providers/perplexity-provider.md#L21)
  - sha: [7e16e3d07723](https://github.com/openclaw/openclaw/commit/7e16e3d07723cc3903f40ce94f53a3920c88b1fc)
- **Provider directory still lists Perplexity only under web search:** The provider directory entry is `Perplexity (web search)`, while the page otherwise catalogs LLM/model providers.
  - file: [docs/providers/index.md:60](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/providers/index.md#L60)
  - sha: [7e16e3d07723](https://github.com/openclaw/openclaw/commit/7e16e3d07723cc3903f40ce94f53a3920c88b1fc)
- **Onboarding coverage is scoped to web search, not model selection:** Current setup docs place Perplexity in the web-search step, and the setup test asserts it configures `tools.web.search.provider = "perplexity"`, not a model-provider selection.
  - file: [docs/start/wizard.md:36](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/start/wizard.md#L36)
  - sha: [7e16e3d07723](https://github.com/openclaw/openclaw/commit/7e16e3d07723cc3903f40ce94f53a3920c88b1fc)
- **Quickstart test shows Perplexity writes web-search config only:** The onboarding test for Perplexity verifies `result.tools?.web?.search?.provider` becomes `perplexity`; there is no corresponding model-provider auth/model-picker path in this plugin.
  - file: [src/commands/onboard-search.test.ts:404](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/src/commands/onboard-search.test.ts#L404)
