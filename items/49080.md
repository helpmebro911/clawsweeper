## Summary

Keep open. Current main still keeps durable bootstrap files in a sessionKey-scoped cache and reuses that cache on later session turns, so the reported stale USER.md/MEMORY.md policy drift is not fixed on main. The discussion’s claimed fix direction is useful, but it does not match the checked-out implementation or current tests.
