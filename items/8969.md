## Summary

Keep #8969 open. Current main implements the first requested quick win as `openclaw skills check`, but the remaining two requested operator diagnostics are still not present on the requested core surfaces: `openclaw status` does not surface manifest-registry load diagnostics, and the non-tool `/skill` path still rewrites the prompt without an operator-visible warning.

## Best Possible Solution

Keep this open or split the remaining work into focused follow-ups. The best product path is to add a status section or JSON field for manifest-registry load diagnostics and emit an operator-visible warning/diagnostic when a non-tool `/skill` invocation rewrites the prompt; after that lands with tests/docs, this umbrella issue can close as implemented.
