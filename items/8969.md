## Summary

Keep #8969 open. Current main has shipped the first requested quick win as `openclaw skills check`, but the two remaining operator diagnostics are still missing from the requested `status` and non-tool skill-command prompt rewrite surfaces. The issue also already has a maintainer-authored Codex review comment saying the same remaining scope should stay open.

## Best Possible Solution

Keep #8969 open or split it into focused follow-ups. Preserve `openclaw skills check` as the shipped solution for the first quick win, then add manifest-registry diagnostics to `openclaw status`, `openclaw status --json`, and `openclaw status --all`, plus an operator-visible warning/log when a non-tool skill command rewrites the request for model-directed execution. Cover both with focused status and skill-command tests plus docs updates.
