## Summary

Keep #8969 open. Current main implements the first quick win as `openclaw skills check`, but the remaining requested status-surface plugin registry diagnostics and operator-visible warning for non-tool skill command prompt rewrites are still not implemented on the requested surfaces.

## Best Possible Solution

Keep #8969 open, or split it into focused follow-ups. Preserve `openclaw skills check` as the shipped solution for the first quick win. Add manifest-registry diagnostics to `openclaw status`, `openclaw status --json`, and `openclaw status --all`; then add an operator-visible diagnostic when a non-tool skill command rewrites the request for model-directed execution. Cover both with focused status and skill-command tests plus public docs updates.
