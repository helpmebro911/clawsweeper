## Summary

Keep #70191 open. Current main does not implement `/health/deep` or an equivalent model-contract probe: the core HTTP probe map only handles `/health`, `/healthz`, `/ready`, and `/readyz`; readiness details expose only `ready`, `failing`, and `uptimeMs`; docs and tests describe the same shallow/live readiness surfaces. The issue is recent, contributor-authored, has useful overlay prior art, and asks concrete maintainer design questions about a native Gateway endpoint rather than a stale/duplicate cleanup item.

## Best Possible Solution

Leave the issue open for maintainer product/design judgment. If accepted, implement a native Gateway deep-health/model-contract probe in the core HTTP probe path, reuse the `/ready` detail authorization policy, derive the active primary/fallback model state from the existing config/session model resolvers, and wire bounded routing/error telemetry through the agent model-call path. If maintainers prefer externalization, first document the plugin route/hooks approach and any missing SDK/auth helper needed to match the proposed `/ready`-style policy.
