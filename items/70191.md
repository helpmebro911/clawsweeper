## Summary

Keep open. Current main does not ship a native /health/deep or equivalent HTTP probe that returns the active provider/model/fallback contract. Existing probe routes cover shallow liveness/readiness, while model status JSON exists only as CLI/config introspection and is not the requested authenticated Gateway HTTP health surface. The issue is recent, concrete, and asks for upstream design direction before a PR, so it needs maintainer product/API judgment rather than conservative cleanup.

## Best Possible Solution

Leave the issue open for maintainer design review. The best path is to either define a core Gateway probe contract, likely near src/gateway/server-http.ts with tests in src/gateway/server-http.probe.test.ts and docs under docs/gateway/health.md, or have a maintainer explicitly redirect it to a ClawHub/plugin design with any missing SDK seams documented. If implemented in core, reuse or deliberately replace canRevealReadinessDetails(), define the stable JSON shape, and add tests for auth, no-inference behavior, redaction, and model/fallback resolution.
