## Summary

Current main still does not implement the requested native auth-gated `/health/deep` no-inference model-contract endpoint. The issue is recent, contributor-authored, and asks for maintainer API/security direction before a PR, so it should remain open.

## Best Possible Solution

Keep this issue open for maintainer product/API/security direction. The likely path is an additive gateway probe or authenticated readiness-detail extension backed by existing configured model/fallback resolution plus bounded routing/error telemetry hooks, with tests for local versus remote auth, unauthorized no-body/no-leak behavior, no-inference operation, redaction, status semantics, and public Gateway health/model-contract docs. If maintainers prefer a plugin route, first document or add the missing SDK/runtime helper and auth contract needed to implement it outside core.
