## Summary

Keep this issue open. Current main still only has shallow HTTP liveness/readiness probes plus separate CLI/RPC/model-list surfaces; it does not provide the proposed auth-gated no-inference HTTP endpoint that reports the active backend model/provider/fallback contract and routing/error telemetry. The issue is recent, concrete, contributor-authored, and asks for maintainer API/security direction before a PR.

## Best Possible Solution

Keep this issue open for maintainer product/API direction. The likely path is an additive Gateway probe, either /health/deep or an authenticated extension of /ready details, backed by existing model/fallback resolution plus bounded routing/error telemetry hooks. It should include tests for local versus remote auth, unauthenticated leakage boundaries, no-inference behavior, path redaction, and docs on the public Gateway health/model pages.
