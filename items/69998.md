## Summary

Keep this PR open. Current main still returns the updated `skills.update` config object directly, so the linked credential-echo bug is not fixed on main. The PR is a focused implementation candidate, and the related duplicate PRs are closed unmerged rather than superseding it.

## Best Possible Solution

Keep this PR open. The best path is to update the PR so `skills.update` either restores/keeps existing secret values when redacted sentinels are submitted, or stops returning writable secret-bearing config in the success payload. Then land the focused production change plus regression test and let it close #66769.
