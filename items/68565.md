## Summary

Keep open. Current main still does not implement the PR's central `transformTransportMessages` behavior: signed and redacted thinking blocks are still preserved only for same-model replay. The PR also should not be merged as-is because the discussion surfaced a real OpenAI Responses compatibility risk: preserving every opaque Anthropic `thinkingSignature` across targets would feed non-JSON signatures into OpenAI's JSON parser. The useful path is a narrower provider-compatible fix with regression coverage, not maintainer cleanup closure.

## Best Possible Solution

Keep the PR open or explicitly supersede it with a narrower canonical replay fix. The best implementation should preserve immutable Anthropic signed/redacted thinking only when replaying to Anthropic-compatible providers that can accept those opaque blocks, continue downgrading or stripping them for incompatible targets such as OpenAI Responses, and add regression tests for signed Anthropic cross-model Anthropic replay, redacted-only replay, unsigned downgrade/drop behavior, and the OpenAI Responses negative case.
