## Summary

Keep open. Current main still allows prompt-visible plugin hook fields to flow unbounded from `before_prompt_build` and legacy `before_agent_start` into embedded, CLI, and harness prompt assembly. The PR is not superseded by the related length-limit items and remains a plausible core hardening change, though it needs refresh against the current shared prompt-hook resolver and should cover all prompt-visible hook fields.

## Best Possible Solution

Keep the PR open and refresh it against current main. The best implementation is a shared prompt-visible hook string clamp applied after per-hook merging and after phase-plus-legacy joins, used by embedded, CLI, and harness prompt preparation, with warning logs and regression tests for modern hooks, legacy hooks, combined overflow, all prompt-visible fields, and normal-length passthrough.
