## Summary

Keep this PR open. Current main still has the unconditional heredoc approval gate in allowlist mode, so the linked regression is not implemented on main. The PR is a plausible fix direction, but its current head is not merge-ready: the final raw-text regex misses unquoted strip-tab heredocs (`<<-EOF`), and the added tests reference an undefined `warnings` variable.

## Best Possible Solution

Keep the PR open but require a narrow rework before merge: derive heredoc approval from the existing parser semantics or add a robust helper that treats quoted and backslash-quoted delimiters as literal while still detecting unquoted `<<EOF` and `<<-EOF`; fix the test warning capture; add direct `processGatewayAllowlist` coverage for quoted, backslash-quoted, unquoted, and strip-tab heredoc cases; then run the targeted agents/infra tests for this approval path.
