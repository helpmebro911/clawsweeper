## Summary

Keep #68854 open. Current main still forces explicit approval for every heredoc segment in gateway allowlist mode, so the quoted-heredoc regression is not implemented on main. The PR is a relevant focused fix attempt, but it still needs maintainer review because the latest PR review notes a remaining `<<-EOF` classification gap, and the related PRs #68754/#68824 are open duplicates rather than merged canonical replacements.

## Best Possible Solution

Keep the PR open for maintainer triage. The best fix is a single canonical gateway change that removes the blanket approval for quoted/literal heredocs, handles `<<`, `<<-`, single-quoted, double-quoted, and backslash-quoted delimiters from raw shell text or shared parser output, preserves the existing unquoted-expansion protections, and adds direct `processGatewayAllowlist` regression tests for quoted auto-approval and unquoted approval-required behavior.
