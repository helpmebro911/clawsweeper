## Summary

Close as `clawhub`: current `main` already routes alternative context/compaction strategies into the plugin ecosystem via context-engine plugins, and project scope explicitly prefers optional capability outside core.

## Risks / Open Questions

- If maintainers later decide OpenClaw should ship a first-party default lossless context engine, that narrower product decision could justify a new core issue.
- Community plugin implementations can evolve independently, so closing this does not guarantee any specific third-party plugin will remain maintained.

## Close Comment

Closing this after a ClawSweeper/Codex review because the idea fits OpenClaw’s plugin ecosystem, not OpenClaw core. Current `main` already provides a supported context-engine extension path for alternative context assembly/compaction strategies: see `docs/concepts/context-engine.md`, `docs/tools/plugin.md`, `docs/plugins/sdk-overview.md`, and `src/plugin-sdk/index.ts` (`registerContextEngine`, `delegateCompactionToRuntime`). Project scope also explicitly says optional capability should usually ship as plugins and that ClawHub/community distribution is the preferred path (`VISION.md`). There is already a documented community plugin route for this category, including `Lossless Claw (LCM)` in `docs/plugins/community.md`. Given that direction, this RLM-style delegation approach belongs as a ClawHub/community context-engine plugin rather than a new OpenClaw core issue.
