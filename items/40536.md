## Summary

Current main does not implement the requested per-skill model override. `skills.entries.*` is still limited to enablement, env/API-key injection, and a generic `config` bag; unknown top-level skill fields such as `model` are rejected. Skill slash commands currently rewrite the prompt or dispatch a tool, but the run model is already resolved before that inline skill handling. Plugin hooks can override a model for an entire run, but they do not provide the requested core `skills.entries.<skill>.model` behavior or a reliable model switch tied to model-invoked skill use, so this remains a coherent open enhancement.
