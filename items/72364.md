## Summary

Current main implements the central compaction lifecycle fix. The old in-place JSONL truncation path has been removed; `truncateAfterCompaction` now creates a compacted successor transcript, leaves the original transcript untouched, emits post-compaction transcript updates for the successor, and propagates the new session id/file through manual, context-engine, overflow, timeout, main-runner, followup, memory-flush, and gateway compaction paths. The implementation does not synthesize `/compact`, but it solves the reported mid-conversation active-transcript rewrite problem well enough to close as implemented.

## Best Possible Solution

Close this as implemented on main. Keep the successor-transcript rotation design rather than adding a synthetic `/compact` dispatch path; if live UI still needs extra affordances, track that as a narrower follow-up around notifications or history refresh behavior.
