## Summary

Keep open. Current `main` still contains the reported gateway-thinking bug: `src/agents/pi-embedded-subscribe.ts` only enables `streamReasoning` when `onReasoningStream` is present and `emitReasoningStream()` still returns early when that callback is absent, so WS-only clients never reach the `emitAgentEvent({ stream: "thinking" ... })` path.

## Evidence

- **Callback remains optional:** The subscribe params still define `onReasoningStream` as optional, so WS/webchat sessions can legally omit it.
  - file: [src/agents/pi-embedded-subscribe.types.ts:23](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/agents/pi-embedded-subscribe.types.ts#L23)
  - command: `nl -ba src/agents/pi-embedded-subscribe.types.ts | sed -n '1,40p'`
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Streaming state still depends on callback presence:** `streamReasoning` is still initialized as `reasoningMode === "stream" && typeof params.onReasoningStream === "function"`, which disables reasoning streaming for sessions without that callback.
  - file: [src/agents/pi-embedded-subscribe.ts:90](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/agents/pi-embedded-subscribe.ts#L90)
  - command: `nl -ba src/agents/pi-embedded-subscribe.ts | sed -n '70,110p'`

- **Early return still blocks thinking event emission:** `emitReasoningStream()` still returns on `!state.streamReasoning || !params.onReasoningStream` before the `emitAgentEvent({ stream: "thinking" ... })` call, so the WS broadcast is unreachable without the callback.
  - file: [src/agents/pi-embedded-subscribe.ts:710](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/agents/pi-embedded-subscribe.ts#L710)

  - file: [src/agents/pi-embedded-subscribe.handlers.messages.ts:415](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/agents/pi-embedded-subscribe.handlers.messages.ts#L415)
