## Summary

Current main still has the reported callback coupling, so webchat/WS clients without `onReasoningStream` cannot receive live `stream: "thinking"` events. The issue is concrete, source-backed, not maintainer-authored, has no protected label, and the related fix PRs in the provided context are closed unmerged.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `src/agents/pi-embedded-subscribe.ts`: initialize `streamReasoning` from `reasoningMode === "stream" && canShowReasoning`, remove the callback from the early-return condition, emit the `stream: "thinking"` agent event before the optional callback call, and add a focused regression test for callback-absent WS delivery. Then update reasoning-stream docs if webchat/WS live thinking is intentionally supported.
