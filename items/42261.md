## Summary

Current main still matches the reported gateway/WebSocket live-thinking bug. The subscribe layer only enables reasoning streaming when an optional `onReasoningStream` callback exists, and `emitAgentEvent({ stream: "thinking" })` is still behind that callback guard. Webchat `chat.send` does not provide the callback, and the internal-webchat typing policy suppresses the synthesized typing callback path, so the agent event bus never receives live thinking events for callback-less WS clients.

## Best Possible Solution

Keep this issue open. The fix likely belongs in `src/agents/pi-embedded-subscribe.ts`: initialize `streamReasoning` from stream mode plus existing reasoning-visibility gates rather than callback presence, remove `params.onReasoningStream` from the early-return condition, emit `stream: "thinking"` before optionally invoking the surface callback, and add a callback-absent regression test proving the agent event bus receives live thinking for WS clients.
