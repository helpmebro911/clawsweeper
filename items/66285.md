## Summary

Keep this PR open. Current main still lacks explicit Active Memory chat-type support in the manifest schema, TypeScript config types, normalization filter, runtime chat-type resolver, docs, and tests. The PR is a focused implementation candidate for the closed linked bug, but the resolver change should parse structured explicit session keys before broad marker checks before it lands.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Update the resolver to parse structured agent:<agent>:explicit:<sessionId> keys before broad marker checks, then land the schema, type, normalization, docs, and regression-test changes for explicit Active Memory sessions.
