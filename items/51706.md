## Summary

Keep this PR open. The original linked runtime-model display bug is already shipped on main via #55425, but the PR now carries a still-relevant fallback-status follow-up: current main still has only one selected model path for session_status, so showing the runtime model can collapse selected-vs-active fallback semantics. The PR head is not mergeable as-is because its fallback preservation hardcodes the configured default instead of the stored fallback-selected ref, and it also needs rebasing onto the current src/status module split.

## Best Possible Solution

Keep this PR open or replace it with a focused follow-up. Preserve the shipped #55425 runtime-model behavior for ordinary sessions, add a display-only runtime model path through the current src/status status-text/status-message interfaces, keep selected-model semantics intact during active fallback, derive fallback selected refs from fallbackNoticeSelectedModel or the actual routed selection rather than the global default, and add tests for runtime-only display, unknown-provider preservation, active fallback visibility, and channel/group-selected fallback sessions. Do not merge the current PR head without addressing the fallback-selected bug and rebasing onto the current status module layout.
