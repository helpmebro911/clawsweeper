## Summary

Keep this PR open. Current main already shipped the original runtime-model display fix via #55425, but the remaining review concern is still real: `session_status` feeds the persisted runtime model through the single selected-model path, so active fallback state can be collapsed when no explicit session override exists. The PR is also stale against the current status module split and should be rebased or replaced with a narrower display-only runtime-model change.

## Best Possible Solution

Keep this PR open or replace it with a focused follow-up. Preserve the shipped #55425 behavior for ordinary runtime-model display, add a display-only runtime model path through the current `src/status` status-text/status-message interfaces, keep selected-model semantics sourced from explicit overrides, channel/group selection, or persisted `fallbackNoticeSelectedModel`, and add seam tests for runtime-only display, unknown-provider preservation, active fallback visibility, and channel/group-selected fallback sessions.
