## Summary

Keep open. Current main still lacks a CLI root guard or clear DigitalOcean 1-Click root-execution workaround, and the provided GitHub context shows open, unmerged PR #67509 as the direct implementation candidate for this issue.

## Best Possible Solution

Keep this issue open and review/land PR #67509 or an equivalent maintainer-approved fix. The likely implementation path is an early CLI guard before state/config resolution, with an OPENCLAW_ALLOW_ROOT-style escape hatch, tests for root help/version behavior, and optional DigitalOcean 1-Click recovery or wrapper guidance if maintainers want image-specific mitigation.
