## Summary

Keep #13620 open. Current main still does not implement outbound Telegram media groups/albums: there is no `sendMediaGroup`/`InputMedia` implementation in the checkout, reply delivery loops over `mediaUrls` and calls single-item Telegram media APIs, direct Telegram sends accept one `mediaUrl`, the outbound adapter intentionally sequences multiple media through repeated `sendMessageTelegram` calls, and tests/docs still describe or assert sequential media behavior. The related PRs in the provided GitHub context (#21309, #51449, #51457) are useful prior art but closed unmerged, and their referenced commits are not ancestors of current `main`.

## Best Possible Solution

Keep #13620 open and implement outbound album support in the bundled Telegram plugin. The likely path is a `sendMediaGroup` flow for 2-10 groupable photos/videos across reply delivery, direct Telegram sends, and outbound payload delivery, preserving single-item fallback for one item, GIFs, audio, documents, caption overflow, buttons, reply/thread/silent/pin behavior, and non-Telegram channels. Add focused Telegram tests and user-facing docs for `mediaUrls`/album semantics.
