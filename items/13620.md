## Summary

Keep #13620 open. Current main still does not implement outbound Telegram media groups/albums: the Telegram plugin has no `sendMediaGroup`/`InputMedia` send path, reply delivery iterates `mediaUrls` one item at a time, direct Telegram sends accept a single `mediaUrl`, the outbound adapter sequences multiple media through repeated `sendMessageTelegram` calls, tests assert that sequential behavior, and docs only document a single `mediaUrl`. The related #14027 discussion and PRs #21309, #51449, and #51457 are useful prior art, but the provided GitHub context shows those PRs closed unmerged and the requested behavior is still absent on current `main`.

## Best Possible Solution

Keep #13620 open and implement outbound album support in the bundled Telegram plugin. The likely path is a `sendMediaGroup` flow for 2-10 groupable photos/videos across reply delivery, direct Telegram sends, and outbound payload delivery, preserving current single-item fallback for one item, GIFs, audio, documents, caption overflow, buttons, reply/thread/silent/pin behavior, and non-Telegram channels. Use the closed PRs #21309/#51449/#51457 as prior art, then add focused Telegram tests and user-facing docs for the resulting `mediaUrls` or album semantics.
