## Summary

Keep #13620 open. Current main still does not implement outbound Telegram media groups/albums: there is no `sendMediaGroup`/`InputMedia` outbound path, reply delivery iterates media one at a time, the Telegram message action/direct send surface accepts only one `mediaUrl`, the outbound adapter sequences `mediaUrls` through repeated single sends, and docs only advertise single-media Telegram sending. The related in-repo album issues/PRs are closed or unmerged, so they do not supersede this tracker.

## Best Possible Solution

Keep #13620 open and implement outbound album support in the bundled Telegram plugin. The likely path is to add a `sendMediaGroup` flow for 2-10 groupable photo/video items across reply delivery, direct Telegram send/action, and outbound payload delivery, while preserving single-item and non-groupable fallbacks, first-item caption handling, caption-overflow follow-up text, inline-button follow-up behavior, and thread/reply/silent/pin semantics. Add focused Telegram tests and update `docs/channels/telegram.md` plus any generic message-tool docs if a user-facing multi-media parameter is added.
