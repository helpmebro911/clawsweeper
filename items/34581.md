## Summary

Keep PR #34581 open. Current main still uses strict `express.json()` for the Microsoft Teams Bot Framework webhook, has no raw-body JSON escape repair path, and does not convert malformed authenticated JSON into a 200 acknowledgement to prevent Azure Bot Service backoff. The PR describes a concrete Teams deployment failure and contains useful focused work, though its repair regex still needs maintainer review before landing.

## Best Possible Solution

Keep #34581 open for Microsoft Teams maintainer review. The best path is a focused webhook fix in `extensions/msteams/src/monitor.ts` that preserves the pre-parse auth/JWT ordering, parses authenticated Bot Framework activities from raw bytes or an equivalent safe seam, repairs only truly bare invalid JSON escapes without corrupting valid escaped backslashes, acknowledges unrecoverable authenticated payloads with HTTP 200 to avoid Azure Bot Service backoff, and adds regression coverage for repair behavior, unrecoverable JSON acknowledgement, middleware ordering, and body-size limits.
