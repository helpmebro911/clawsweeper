## Summary

Keep open. Current main has partially improved the hash calculation and the checked-in bundle currently looks deduped, but the A2UI generator path is still install-layout-sensitive: the Rolldown config still prefers `ui/node_modules` over root `node_modules`, still lacks a `signal-polyfill` singleton alias, and current tests do not prove byte-stable generated artifacts with and without `ui/node_modules`.

## Best Possible Solution

Fix the generator rather than only refreshing generated artifacts: make A2UI dependency resolution canonical across install layouts, or alias the full singleton dependency set including `signal-polyfill` and `signal-polyfill/`; then add regression coverage that proves generated bundle/hash stability with and without `ui/node_modules`. Coordinate with open PR #68934 if maintainers want that patch as the implementation vehicle.
