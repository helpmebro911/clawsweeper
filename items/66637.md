## Summary

Keep open. Current `main` still exposes `browser.proxy` from the macOS app node, but that path forwards to a local loopback browser-control server on `127.0.0.1:<gatewayPort+2>`. The macOS remote-mode docs and tunnel code show remote mode does not start a local gateway and only forwards the gateway control port, which matches the report that non-browser node commands work while browser proxy calls fail with connection-refused semantics.
