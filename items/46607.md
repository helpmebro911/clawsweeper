## Summary

Keep open. Current main does not implement inline Google/Gemini audio @file refs in headless payload text, and the linked issue #46601 remains a real unshipped capability gap. PR #46607 still looks relevant, but it needs the existing review hardening addressed before it can be considered merge-ready.

## Best Possible Solution

Keep the PR and linked issue open. The best path is to rebase the PR onto current main, put the conversion in the Google headless/request path or provider wrapper, restrict rewriting to current user text parts, enforce workspace/sandbox path boundaries, sniff/normalize audio MIME with shared media helpers, keep a 20 MB inline-data guard, preserve unresolved refs, and add focused regression tests for success, missing files, oversized files, assistant-history refs, and common `.m4a` brands.
