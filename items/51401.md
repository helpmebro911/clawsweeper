## Summary

Keep open. Current `main` still derives package-install status from the installed package's `package.json.packageManager`, and `update status` renders that value directly. Because the published package currently declares `pnpm`, an npm global install would still be reported as `pnpm`, matching the issue report rather than resolving it.

## Evidence

- **Detection still trusts repo metadata first:** `detectPackageManager()` reads `<root>/package.json`, splits `packageManager`, and returns `pnpm`/`bun`/`npm` before checking lockfiles or any install-path markers.
  - file: [src/infra/detect-package-manager.ts:6](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/infra/detect-package-manager.ts#L6)
