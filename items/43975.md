## Summary

Keep this PR open. Current main still generates Windows gateway task scripts without a UTF-8 bootstrap and still would parse a future bootstrap line as the command, so the PR remains a focused implementation candidate for the linked Chinese-username Windows gateway failure.

## Best Possible Solution

Keep this PR open for maintainer review, or replace it with an equivalent main-branch fix. The fix belongs in `src/daemon/schtasks.ts`: emit a fixed UTF-8 code-page bootstrap in generated Windows `gateway.cmd`, teach `readScheduledTaskCommand` to ignore that exact bootstrap during read-back, and keep the Chinese-path regression coverage. Do not auto-close this PR unless an equivalent implementation lands on main or a maintainer chooses a different Windows encoding strategy.
