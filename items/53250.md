## Summary

Current main still emits generic exec-approval unavailable guidance and does not include the requested Control UI bootstrap command, approvals inspection command, or concrete setup hints in the shared timeout/unavailable message. The directly linked implementation PR #53347 was closed unmerged, so there is no shipped fix to close against.

## Best Possible Solution

Keep this open. Apply a small core UX change in `src/infra/exec-approval-reply.ts` that preserves native-channel setup guidance while adding concrete recovery steps for `openclaw dashboard --no-open`, `openclaw approvals get`, and the relevant `channels.<channel>.execApprovals.enabled` setup path, then add focused tests in `src/infra/exec-approval-reply.test.ts`.
