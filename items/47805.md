## Summary

Keep open. Current main still backs the native session menu from registered session-store entries, not from arbitrary `.jsonl` transcript files copied into the sessions directory. Related usage-session discovery exists, including archived/reset transcript handling from other cleanup work, but that discovery is wired to `sessions.usage`, not `sessions.list` or the session menu.

## Evidence

  - file: [src/config/sessions/combined-store-gateway.ts:73](https://github.com/openclaw/openclaw/blob/d228463120a5057f7ccf9a5c31af62fe4a922088/src/config/sessions/combined-store-gateway.ts#L73)
