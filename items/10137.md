## Summary

Keep #10137 open. Current main has partial Responses compatibility coverage: generic `api: "openai-responses"` payloads already use array `input` and `stream: true`, but the requested stable normalization knobs are still missing for generic proxy/relay endpoints, especially opt-in top-level fallback `instructions` and optional omission of `max_output_tokens`. The provided discussion also identifies #10140 as the closed duplicate, so #10137 remains the canonical tracker.

## Best Possible Solution

Keep #10137 open as the canonical tracker. A solid fix would add documented stable provider/model compat options around `ModelCompatConfig`/`ModelCompatSchema` and `OpenAIResponsesPayloadPolicy`, covering generic top-level fallback `instructions` and optional `max_output_tokens` omission, with focused tests for proxy-like `api: "openai-responses"` providers. Treat #10140 as the closed duplicate copy.
