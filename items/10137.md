## Summary

Current main still only partially covers #10137. The Responses transport already sends array `input` and `stream: true`, but generic `api: "openai-responses"` providers still lack stable compatibility controls for fallback top-level `instructions` and for omitting `max_output_tokens`. #10140 is the closed duplicate, so #10137 remains the canonical open tracker.

## Best Possible Solution

Keep #10137 open as the canonical tracker. Add opt-in Responses proxy compatibility controls in the transport/payload policy and stable model/provider config, especially fallback/default top-level `instructions` for generic `api: "openai-responses"` relays and optional omission of `max_output_tokens`, with focused regression tests and docs. Treat #10140 as the closed duplicate copy.
