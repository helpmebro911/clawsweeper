## Summary

Keep #10137 open. Current main still only partially covers the report: `openai-responses` requests already use array `input` and `stream: true`, but generic Responses-compatible proxy models still do not have stable config/payload controls for fallback top-level `instructions` or omitting `max_output_tokens`. #10140 is the closed duplicate, so #10137 remains the canonical tracker.

## Best Possible Solution

Keep #10137 open as the canonical tracker. Add opt-in Responses proxy compatibility controls in the transport/payload policy and stable provider/model config, especially fallback/default top-level `instructions` for generic `api: "openai-responses"` proxies and optional omission of `max_output_tokens`, with focused regression tests and docs. Treat #10140 as the closed duplicate copy.
