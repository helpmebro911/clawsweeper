## Summary

Keep open. Current main still persists `allow-always` approvals by mutating a previously loaded `ExecApprovalsFile` object and then rewriting the whole `exec-approvals.json`, so the reported last-write-wins race is still plausible in both gateway-host and node-host paths. I did not find any current concurrency-specific regression coverage proving a merge/lock fix landed.

## Evidence

- **Stale whole-file rewrite remains in store helper:** `addAllowlistEntry` appends to the passed-in `approvals` object and immediately calls `saveExecApprovals(approvals)`. `saveExecApprovals` serializes the full file and replaces it on disk, but does not reload or merge current contents before writing.
  - file: [src/infra/exec-approvals.ts:481](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/infra/exec-approvals.ts#L481)
  - command: `nl -ba src/infra/exec-approvals.ts | sed -n '440,560p'; nl -ba src/infra/exec-approvals.ts | sed -n '906,997p'`
- **Gateway allow-always path still uses stale approval snapshot:** The gateway exec approval flow persists `allow-always` entries through `persistAllowAlwaysPatterns({ approvals: approvals.file, ... })` and falls back to `addDurableCommandApproval(approvals.file, ...)`, both using the approval file object loaded earlier in the request lifecycle.
  - file: [src/agents/bash-tools.exec-host-gateway.ts:327](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/bash-tools.exec-host-gateway.ts#L327)
  - command: `nl -ba src/agents/bash-tools.exec-host-gateway.ts | sed -n '200,340p'`
- **Node-host allow-always path still uses stale approval snapshot:** The node-host `system.run` flow does the same thing: `persistAllowAlwaysPatterns({ approvals: phase.approvals.file, ... })` or `addDurableCommandApproval(phase.approvals.file, ...)`, again persisting from the existing in-memory snapshot rather than a fresh read.
  - file: [src/node-host/invoke-system-run.ts:626](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/node-host/invoke-system-run.ts#L626)
  - command: `nl -ba src/node-host/invoke-system-run.ts | sed -n '620,645p'`
- **Tests cover persistence shape, not concurrent writes:** Current store and node-host tests assert single-flow allow-always persistence and metadata preservation, but I did not find a test covering simultaneous approvals or merge-on-write behavior.
  - file: [src/infra/exec-approvals-store.test.ts:203](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/infra/exec-approvals-store.test.ts#L203)
  - command: `rg -n "concurrent|parallel|simultaneous|allow-always" src/node-host/invoke-system-run.test.ts src/agents/bash-tools.exec-host-gateway.test.ts src/infra/exec-approvals-store.test.ts; nl -ba src/infra/exec-approvals-store.test.ts | sed -n '203,312p'; nl -ba src/node-host/invoke-system-run.test.ts | sed -n '1270,1360p'`
