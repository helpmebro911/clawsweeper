## Summary

Keep #68926 open. Current main already has chat-header session, model, and thinking selectors, but it still lacks the PR's central dedicated agent selector in chat controls. The PR is a focused web UI feature and is not implemented, duplicated, or out of repo scope, though it needs rebase/rework against current main and the unresolved bot review concerns.

## Best Possible Solution

Keep the PR open and ask for a rebase onto current main. The best product path is a narrow chat-header agent selector added beside the existing session/model/thinking controls, preferably in `ui/src/ui/chat/session-controls.ts` or a small shared helper, with tests for multi-agent main-session switching, legacy/non-agent session fallback, and preservation of session-specific thinking options.
