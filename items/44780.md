## Summary

Keep open. Current `main` does not implement a configurable speech-recognition locale for iOS Talk Mode, so this draft PR is not obsolete. The specific bug rationale in the PR body is weaker than stated because Apple documents `SFSpeechRecognizer.init()` as using the user’s default language settings rather than hardcoding `en-US`, but that does not make the proposed feature already implemented or safe to close.
