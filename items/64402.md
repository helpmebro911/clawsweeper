## Summary

Keep open. Current `main` still has the same core mismatch the issue describes: Discord advertises `channel-create`/`channel-edit`/`channel-delete`/`channel-move` and the Discord action handler implements them, but the CLI registration only exposes `message channel info` and `message channel list`. The docs mirror that limited surface. The agent-runtime concern is also not clearly resolved, because the embedded message-provider allowlist for `node` still omits the `message` tool entirely.
