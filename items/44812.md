## Summary

Keep open. Current `main` partially mitigates session-dir bloat by pruning old `.deleted.*` and `.reset.*` archives and by optionally evicting archive artifacts under a disk-budget policy, but it does not fully implement the issue as reported: normal maintenance still leaves `.bak.*` files in place, and there is no session-maintenance cleanup path for `*.tmp` files.

## Evidence

- **Normal maintenance only deletes `deleted` and `reset` archives:** `saveSessionStoreUnlocked` calls `cleanupArchivedSessionTranscripts` only for `reason: "deleted"` and `reason: "reset"`. There is no matching cleanup call for `bak` or `tmp` artifacts.
  - file: [src/config/sessions/store.ts:317](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/config/sessions/store.ts#L317)
  - command: `nl -ba src/config/sessions/store.ts | sed -n '280,332p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Archive cleanup helper cannot remove `*.tmp` and only removes one archive reason per pass:** `cleanupArchivedSessionTranscripts` scans names through `parseSessionArchiveTimestamp(entry, reason)`, so it only targets timestamped archive suffixes like `.deleted.<ts>` or `.reset.<ts>`. It has no logic for `*.tmp`.
  - file: [src/gateway/session-transcript-files.fs.ts:232](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/gateway/session-transcript-files.fs.ts#L232)
  - command: `nl -ba src/gateway/session-transcript-files.fs.ts | sed -n '232,268p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Integration test proves ordinary cleanup leaves `.bak.*` behind:** The pruning integration test writes old `.deleted.*`, recent `.deleted.*`, and old `.bak.*` files, runs `saveSessionStore`, then expects the old `.deleted.*` to be removed while the `.bak.*` file still exists.
  - file: [src/config/sessions/store.pruning.integration.test.ts:151](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/config/sessions/store.pruning.integration.test.ts#L151)
  - command: `nl -ba src/config/sessions/store.pruning.integration.test.ts | sed -n '151,185p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **`.bak.*` removal is only tied to disk-budget pressure:** `enforceSessionDiskBudget` includes archive artifacts in its removable queue, but that logic only runs when `session.maintenance.maxDiskBytes` / `highWaterBytes` are configured and the sessions dir is already over budget.
  - file: [src/config/sessions/disk-budget.ts:258](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/config/sessions/disk-budget.ts#L258)

  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **No session-cleanup path mentions `*.tmp` handling:** A repo search across the session maintenance docs and implementation found no `*.tmp` cleanup logic in the session cleanup paths; the only `.tmp` hit in `src/config/sessions` was a test helper temp-dir prefix.
  - command: `rg -n '\.tmp' docs/cli/sessions.md src/config/sessions src/commands/doctor-state-integrity.ts src/gateway/session-transcript-files.fs.ts src/config/sessions/disk-budget.ts`

## Risks / Open Questions

- The issue may now be less severe than on `2026.3.11` because current `main` does prune old `.deleted.*` / `.reset.*` artifacts by default, but the reported `.bak.*` and `.tmp` accumulation is not fully covered.
- If maintainers close this now, users can still hit unreclaimed session-dir growth in long-lived installs that never exceed the configured disk budget or that accumulate leftover `*.tmp` files.
