## Summary

Keep open. Current main has after-failure context overflow detection with /reset or /new guidance, preflight compaction/recovery when the estimated prompt already exceeds the effective budget, and a Control UI-only context notice. I did not find the requested session-scoped proactive warning sent into persistent chat channels such as Telegram before overflow, nor a chat-channel /context or /session usage report that satisfies the optional UX. The related #50065 remains open and tracks compaction drift; it does not fully supersede this proactive warning request.
