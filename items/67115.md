## Summary

Keep this PR open. Current main still short-circuits macOS sessions with lingering SSH_* environment variables before reaching the Darwin `open` path, so the linked dashboard bug is not implemented on main. The PR is narrow and has no obvious security or supply-chain concern, but it should get a focused regression test before landing.

## Best Possible Solution

Review and land this PR or an equivalent narrow patch, ideally with a focused `resolveBrowserOpenCommand()` regression test proving Darwin with SSH variables and no display variables resolves to `open`, while Linux without a display still returns `ssh-no-display` or `no-display` as intended.
