## Summary

Current main still does not implement a config-backed default for `openclaw tui --deliver`. The TUI CLI only exposes an explicit `--deliver` flag with default `false`, `runTui` still computes `deliverDefault` from CLI options with a `false` fallback, the send path forwards that fixed value, and the config type/schema surfaces still have no `tui.deliver` key. The related PRs #33198 and #33208 are useful prior attempts but were closed unmerged; #66991 is related impact evidence for the delivery-off failure mode, not a shipped or canonical replacement for this persisted-default request.

## Best Possible Solution

Keep #33102 open until a maintainer-approved change adds a documented persisted delivery default, likely `tui.deliver`, wires it through `OpenClawConfig`, strict zod/generated config schema and config help/docs, defines precedence for config versus explicit `--deliver` and `--no-deliver`, and adds focused tests proving config default, CLI override, and false fallback behavior. Treat #33198, #33208, and the arcabot workaround linked from #66991 as prior art rather than shipped fixes.
