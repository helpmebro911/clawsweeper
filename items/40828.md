## Summary

Keep this PR open. Current main still does not implement the requested `/total_usage` chat/native command, Telegram range UI, or cross-session model breakdown in the `/usage` command path. The PR is also the active refresh of the older closed/unmerged #11901 rather than an obsolete duplicate, but it needs maintainer review and likely a rebase because main now has related provider+model session-usage aggregation and the PR discussion flags functional regressions to resolve before merge.

## Best Possible Solution

Keep this PR open and ask for an update against current main. The best path is to implement `/total_usage` as a first-class chat/native command only if maintainers still want that product surface, preferably reusing or aligning with the existing `sessions.usage` provider+model aggregates, preserving transcript prefiltering, keeping provider+model partitioning, retaining authorization checks, and adding focused command/infra tests plus public slash-command and token-use docs.
