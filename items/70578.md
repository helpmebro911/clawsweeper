## Summary

Keep #70578 open. Current main has partially mitigated the silent data-loss path by preserving source keys during write projection and rejecting suspicious destructive writes, but the issue is not fully obsolete: the root config schema is still strict, tests still expect unknown config fields to be rejected, and bundled plugin CLI surfaces are still gated by `plugins.allow`. The related #67557 fix only covered the gateway restart/hash-guard loop mentioned in the report, not the broader schema/round-trip and CLI allowlist concerns.

## Best Possible Solution

Keep the issue open, narrow it into actionable follow-ups, and decide the schema contract explicitly: either preserve inert unknown top-level fields with regression coverage for `plugins install` round-trips, or document/confirm the fail-closed strict-schema policy and ensure config writers never clobber invalid/unknown data. Address the bundled CLI allowlist question separately by either exempting OpenClaw-owned bundled command surfaces or documenting that restrictive `plugins.allow` intentionally gates them.
