## Summary

Keep this PR open. Current main fixed the original #42982 email-redaction symptom in one transcript path, but it still does not implement the additive `logging.redactPatterns` behavior this PR targets, and the PR needs revision for newer transcript redaction paths plus docs/schema contract alignment before landing or closing.

## Best Possible Solution

Keep this PR open for maintainer review or contributor revision. The best path is to decide the public `logging.redactPatterns` contract, implement one shared config-derived pattern resolver used by `resolveRedactOptions`, `redactToolDetail`, and persisted transcript redaction, preserve explicit caller-supplied pattern semantics if intended, then update docs/schema help and add tests proving custom config patterns do not drop default token/PEM redaction across affected sinks.
