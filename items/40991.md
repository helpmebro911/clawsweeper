## Summary

Keep open. Current main still has the reported single-message Telegram path: getFile failures are converted to null, the user-visible failure reply only runs for thrown media errors, and body construction can still emit a media placeholder when no attachment was resolved.

## Evidence

- **getFile failures still degrade to null:** resolveTelegramFileWithRetry catches Telegram getFile errors, including the file-too-big case, logs them, and returns null. resolveMedia then returns null when no file object is available.
