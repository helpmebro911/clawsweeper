## Summary

Keep PR #69725 open. Current main still unconditionally sends `system.run.prepare` before `system.run` for `exec host=node`, while the Swift macOS companion node still advertises and handles `system.run` without defining or handling `system.run.prepare`. The provided related context shows #37591 and #38781 remain open and #54970 was closed unmerged, so this PR is not implemented on main and is not conservatively superseded.

## Best Possible Solution

Keep PR #69725 open for maintainer review. The best product path is either to merge a complete Swift macOS `system.run.prepare` implementation, or to merge this backwards-compatible agent-side local-plan fallback for legacy nodes that advertise `system.run` without prepare. In either path, preserve the canonical `systemRunPlan` approval contract and include regression coverage for the macOS command list that lacks `system.run.prepare`.
