## Summary

Keep open. Current `main` no longer matches the original root-cause claim that the macOS app lacks `system.which`/`system.run` support, but the gateway still performs the remote bin probe immediately on node connect with the same default 15s timeout and no co-located/loopback or app-mode skip/defer path. That means the reported timeout behavior remains plausible and is not strongly shown to be fixed on `main`.
