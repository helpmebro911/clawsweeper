## Summary

Keep open. Current main still does not canonicalize an automatic session auth-profile override from a legacy `provider:default` OAuth profile to a preferred OAuth alias, while this PR is a focused auth-routing fix with regression coverage and no unrelated supply-chain surface.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, adapt the regression tests to the current mocked `session-override.test.ts` harness, align the identity-matching logic with existing auth-profile metadata semantics, then run the targeted auth-profile test lane and changed gate before maintainer review or landing.
