## Summary

Keep open. Current `main` still computes skill setup eligibility from process env plus `skills.entries.<skill>.env/apiKey` in OpenClaw config; it does not detect external credential-file fallbacks or allow a skill-defined setup-status hook, so the reported false positive is not implemented away.

## Evidence

- **Status check only consults env and OpenClaw skill config:** `buildSkillStatus()` marks an env requirement satisfied only when the variable exists in `process.env`, `skills.entries.<skill>.env`, or `skills.entries.<skill>.apiKey` mapped via `primaryEnv`. There is no external file probe.
  - file: [src/agents/skills-status.ts:177](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/skills-status.ts#L177)
  - command: `nl -ba src/agents/skills-status.ts | sed -n '170,189p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Requirement evaluation has no custom setup-status callback:** Entry requirement evaluation forwards only `isEnvSatisfied` and `isConfigSatisfied` into `evaluateRequirementsFromMetadataWithRemote()`. No skill-owned status reporter or arbitrary credential resolver exists here.
  - file: [src/shared/entry-status.ts:23](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/shared/entry-status.ts#L23)
  - command: `nl -ba src/shared/entry-status.ts | sed -n '20,46p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Gateway/UI uses the same workspace skill status report directly:** `skills.status` returns `buildWorkspaceSkillStatus(...)` to clients, so the UI's skill page inherits the same eligibility logic rather than performing a richer check.
  - file: [src/gateway/server-methods/skills.ts:69](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/gateway/server-methods/skills.ts#L69)
  - command: `nl -ba src/gateway/server-methods/skills.ts | sed -n '68,98p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Docs only promise env vars or OpenClaw config, not external credential files:** The public skills docs define `requires.env` as satisfied when the env var exists or is provided in config, and `requires.config` as truthy `openclaw.json` paths. That does not cover `~/.config/...` fallback files.
  - file: [docs/tools/skills.md:197](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/tools/skills.md#L197)
