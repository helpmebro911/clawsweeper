## Summary

Keep open. Current main still returns keyword-search snippets by truncating the beginning of the matched FTS row text, so the user-visible bug reported in #47730 is not implemented away. The PR is stale against the moved memory-core path, but the central fix remains useful and should be adapted rather than closed.

## Best Possible Solution

Keep the PR open and ask for a rebase/adaptation onto extensions/memory-core/src/memory/manager-search.ts. The best fix is to select an FTS5 snippet for real MATCH queries, keep full text available where fallback scoring or substring-only searches need it, preserve the explicit 64-token clamp and ellipsis markers from the PR v2 feedback, and add a regression test where a deep keyword match appears in the returned snippet.
