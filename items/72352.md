## Summary

Keep open. Current main still does not implement PR #72352's configured-model/provider scoping for non-live agent provider discovery. The later provider-discovery refactor made runtime discovery explicit and bounded some unscoped fallback work, but it still does not derive an allowlist from configured models/providers. This PR remains a plausible core agents performance fix, with a small test-quality issue and a rebase needed.

## Best Possible Solution

Keep PR #72352 open. Rebase it onto current main, adapt it to the explicit runtime discovery API, fix the fallback regression test so it exercises an unresolved configured model owner, then run the changed agents/plugin checks or build gate expected for provider-discovery runtime changes. If it stays green, this belongs in core because it optimizes the existing agents implicit provider discovery path using existing provider/plugin ownership and `onlyPluginIds` seams.
