## Summary

Keep #48150 open. Current main still does not expose or query-match Feishu directory peer email fields: live lookup only matches `open_id`/`name` and returns `id`/`name`, and `channel-list` passes that peer payload through. The PR is a focused bundled Feishu plugin improvement, with one outstanding review request for explicit `email` query coverage.

## Best Possible Solution

Refresh and land a narrow Feishu plugin patch on current main: add optional `email` and `enterprise_email` to Feishu live peer results, include both fields in normalized query matching, verify `channel-list`/JSON propagation, and add tests for both `enterprise_email` and an email-only query branch. If maintainers want typed cross-channel email fields, extend `ChannelDirectoryEntry` deliberately; otherwise keep the change plugin-local and additive.
