## Summary

Keep this open. Current main already has the gateway/browser OpenAI Realtime Talk session path, but the native macOS Talk integration requested here is still missing, and maintainer discussion explicitly narrowed the remaining work to macOS consuming the existing realtime session flow rather than adding a parallel backend PCM RPC path.

## Best Possible Solution

Keep this open as the canonical native macOS Talk realtime follow-up. The likely implementation is to add a `GatewayConnection` wrapper for `talk.realtime.session`, add a native realtime audio/WebRTC session object modeled on `ui/src/ui/chat/realtime-talk.ts`, wire `TalkModeRuntime` and overlay state/cancellation into that session, and preserve the existing `SFSpeechRecognizer` -> `chat.send` -> `talk.speak` path as fallback.
