## Summary

Keep this issue open. Current main has the shared gateway/browser OpenAI Realtime Talk session path, but the requested native macOS Talk integration is still missing, and maintainer discussion explicitly narrowed the remaining work to macOS client support rather than closing it.

## Best Possible Solution

Keep this issue open as the canonical native macOS Talk Realtime follow-up. The likely implementation is to add a macOS `GatewayConnection` wrapper for `talk.realtime.session`, build a native realtime audio/WebRTC session modeled on `ui/src/ui/chat/realtime-talk.ts`, wire it into `TalkModeRuntime` and overlay phase/cancellation, and preserve the current `SFSpeechRecognizer` -> `chat.send` -> `talk.speak` path as fallback. Only add backend PCM streaming RPCs if maintainers decide native macOS must use provider-agnostic backend bridging instead of the existing WebRTC session flow.
