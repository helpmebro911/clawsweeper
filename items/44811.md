## Summary

Keep open. Current `main` still lacks the PR's core change: `writeConfigFile` projects runtime writes onto `runtimeConfigSourceSnapshot` directly, so newer disk-only keys can still be dropped during runtime snapshot writeback. The focused stale-`session.dmScope` regression test from the PR is also not present on `main`.

## Evidence

- **Current write path still uses stale runtime source snapshot:** `writeConfigFile` on current `main` computes a merge patch from the runtime snapshot, then reapplies it onto `runtimeConfigSourceSnapshot` without first refreshing from disk.
