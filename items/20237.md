## Summary

Keep open. Current `main` has landed part of the request: the Control UI already supports inline cron management, exposes `deleteAfterRun`, and shows chat-level context/compaction indicators. But the core asks in #20237 are still missing: there is no native WebUI toast/notification system for cron results, no snooze/reschedule popup actions, and no UI action for a context-warning flush/save flow.

## Evidence

- **Cron management is already in the Control UI:** The docs describe shipped Control UI cron capabilities as list/add/edit/run/enable/disable plus run history, and note advanced controls including delete-after-run.
  - file: [docs/web/control-ui.md:113](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/docs/web/control-ui.md#L113)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Inline cron edit/toggle/run/remove actions exist:** The cron view renders direct Edit, Clone, Enable/Disable, Run, Run if due, History, and Remove buttons for each job.
  - file: [ui/src/ui/views/cron.ts:1481](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/ui/src/ui/views/cron.ts#L1481)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Delete-after-run is exposed in the UI:** The cron form includes a `deleteAfterRun` checkbox, and the quick-create flow has a `Run once` preset that sets `deleteAfterRun = true`.
  - file: [ui/src/ui/views/cron.ts:1003](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/ui/src/ui/views/cron.ts#L1003)
