## Summary

Keep #13593 open. Current main has cron run state, run-history persistence, failure alerts/backoff, and generic gateway liveness/readiness probes, but the issue's central observability work is still missing: structured per-attempt success/skipped/error logging, cron-specific health reporting, and `openclaw cron list` warnings for overdue or repeatedly failing jobs. The related PR #46501 is closed unmerged in the provided context, so it remains useful prior art rather than shipped behavior.

## Best Possible Solution

Keep #13593 open. The best product path is to add structured per-attempt cron execution logging across scheduled, startup catch-up, and manual execution paths; warn on skipped outcomes; add cron health semantics either as `/health/cron` or an explicit cron component in the existing readiness/health model; and teach `openclaw cron list` to annotate overdue and repeated-failure jobs with focused service, CLI, probe, and documentation coverage. PR #46501 remains useful prior art, but it was closed unmerged and is not present on current main.
