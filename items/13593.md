## Summary

Keep #13593 open. Current main has useful partial cron observability, including durable finished-run JSONL records, failure alert state, generic health/readiness probes, and some UI notices, but it still does not implement the issue’s requested per-attempt execution logging, skipped-run warnings, cron-specific health endpoint/component, or `openclaw cron list` health annotations. The related PR #46501 is relevant prior art, but the provided GitHub context shows it was closed unmerged and its head commits are not on current main.

## Best Possible Solution

Keep #13593 open. The best maintainer path is to add structured per-attempt cron execution logging across scheduled timer runs, startup catch-up, and manual runs; warn on skipped and non-throwing failure outcomes; expose cron health through `/health/cron` or an explicit cron component in the existing health/readiness model; and teach `openclaw cron list` to annotate overdue and repeatedly failing jobs, with focused service, CLI, probe, docs, and regression-test coverage. PR #46501 can be mined as prior art but should not be treated as shipped.
