## Summary

Keep #13593 open. Current main has partial cron observability, including finished-run JSONL history, failure-alert state, generic `/health`/`/readyz` probes, and UI attention notices, but it still does not implement the issue’s central requests: structured per-attempt cron execution logging, warnings for skipped/non-throwing failures, a cron-specific health surface, or `openclaw cron list` health annotations. PR #46501 is relevant prior art, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep #13593 open. The maintainer path is to add structured per-attempt cron execution logging across scheduled timer runs, startup catch-up, and manual runs; warn on skipped and non-throwing failure outcomes; expose cron health through `/health/cron` or an explicit cron component in the existing readiness model; and add `openclaw cron list` annotations for overdue and repeatedly failing jobs, with focused service, CLI, probe, docs, and regression-test coverage. PR #46501 can be mined as prior art but should not be treated as shipped.
