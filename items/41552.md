## Summary

Keep this PR open. Current main still clears the Overview Gateway Token whenever the WebSocket URL input changes, so the PR is not implemented or obsolete. The linked bug remains open, and the PR is a focused implementation candidate with regression coverage, but its exact behavior needs maintainer review because preserving the bearer token across arbitrary gateway URL changes can send a token scoped for one gateway to another endpoint.

## Best Possible Solution

Review or rework the Control UI fix instead of closing it. A safer product path is to stop losing a token during harmless editing while preserving the per-gateway credential boundary, for example by using a draft URL field, requiring confirmation before applying a different endpoint, and only carrying a token to the new gateway when the token was explicitly supplied for that gateway or the endpoint is proven equivalent.
