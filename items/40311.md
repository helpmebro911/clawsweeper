## Summary

Keep open. Current main still does not expose a Brave Goggles parameter in the active provider-owned Brave web_search implementation, and this PR remains a useful feature candidate. The PR discussion is useful: earlier review feedback caught runtime-test and cache-key edge cases, and the author updated the branch, but the current diff still targets the older core web-search path rather than the current Brave plugin boundary.

## Best Possible Solution

Keep this PR open as a useful feature candidate, but ask for a rebase/port into the active Brave provider. Add optional goggles support in extensions/brave schema/shared schema/runtime, pass it through both Brave web and llm-context URL builders, include it in the cache key without lossy normalization or missing-value collisions, update docs/tools/brave-search.md and docs/tools/web.md, and add focused Brave provider tests. No new core/plugin API appears necessary.
