## Summary

Keep this PR open. Current main at aac83e00 still lets IRC inherit the global block-streaming default when `channels.irc.blockStreaming` is unset, which is the behavior this PR is trying to change for the linked IRC cut-off report. The proposed diff is narrow and still appears useful; it touches only IRC inbound policy and its colocated test, with no observed security or supply-chain-sensitive changes.

## Best Possible Solution

Keep this PR open as an implementation candidate. The best maintainer path is to rebase or recreate the same IRC-owned change on current main, keep the unset/false/true regression tests, run the IRC targeted tests plus the changed gate, and merge it or close it only after an equivalent fix lands.
