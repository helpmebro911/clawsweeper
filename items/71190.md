## Summary

Keep #71190 open. Current main still has the core orphan-merge behavior that can re-merge an accumulated queued block, and I did not find a regression test or implementation that strips prior queued markers/abort notes while preserving legitimate cross-channel orphan text.

## Best Possible Solution

Keep the issue open and fix the core transcript-repair helper. A conservative patch should preserve legitimate orphaned cross-channel text, but normalize the orphan body before merging by removing prior queued-message markers and abort-note prefixes, then add regression coverage for repeated orphan repairs so the prompt cannot grow across turns.

## Evidence

There is no cleanup of an existing queued marker or `Note: The previous agent run was aborted...` prefix before re-merging.
