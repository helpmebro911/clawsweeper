## Summary

Keep open. Current `main` still exposes WhatsApp reaction sending, but not inbound reaction events or a way to identify which message received a reaction, so the core gap described in the issue remains. The report is messy and mislabeled as a crash, but it is still actionable enough to preserve as a plugin-API/channel-surface gap rather than auto-close as ClawHub or incoherent.
