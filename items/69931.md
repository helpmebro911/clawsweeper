## Summary

Keep #69931 open. Current main supports WhatsApp outbound/ack reactions, but I did not find a WhatsApp inbound reaction event path or plugin API that can tell a skill/plugin which historical message was reacted to. This is not a ClawHub-only close because the requested skill would need channel/runtime data that OpenClaw does not currently surface. The related #47167 is adjacent plugin-ingress work, but it does not cover WhatsApp reaction events specifically.

## Best Possible Solution

Keep open. The best path is an additive WhatsApp/channel plugin API design: ingest WhatsApp reaction events, normalize them into a typed reaction notification/system event with emoji, add/remove action, reactor identity, target message id, conversation/account/session context, and expose it through a documented opt-in plugin hook/config surface with tests and docs. A ClawHub skill can then consume that API without patching the bundled WhatsApp plugin.

## Risks / Open Questions

- Implementation needs an upstream Baileys contract check before choosing the exact event/source payload shape for WhatsApp reactions.
- WhatsApp reaction events can expose personal identifiers and group/message metadata, so the fix should preserve the existing privacy opt-in/access-control pattern used for WhatsApp message hooks.
