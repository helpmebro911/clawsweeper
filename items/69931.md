## Summary

Keep open. Current main supports WhatsApp outbound/ack reactions, but I could not find a plugin-visible inbound WhatsApp reaction event or payload that would let a skill identify which historical message received which emoji. The report is recent, non-maintainer-authored, and not covered by the provided related items; it points at a real channel/API gap rather than a ClawHub-only skill request.

## Best Possible Solution

Keep this open as a WhatsApp channel/API gap. The best path is plugin-owned WhatsApp work that captures Baileys reaction updates, normalizes the target message key/emoji/reacting sender/add-remove semantics, applies existing WhatsApp access-control and routing rules, and exposes it through a typed plugin-visible hook or reaction-notification surface with docs and focused tests. Once that exists, a ClawHub skill/plugin can consume reactions without needing private channel internals.

## Evidence

There is no reaction-specific extraction path before plugin hooks are emitted.
