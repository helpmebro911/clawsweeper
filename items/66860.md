## Summary

Keep open. Current `main` still classifies `missing-session-entry` as an orphaned subagent run and prunes it during both restore and resume paths, and the tests explicitly lock in that pruning behavior. I did not find any degraded-preservation or explicit warning path matching the issue's requested behavior.
