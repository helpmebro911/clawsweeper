## Summary

Current `main` still treats parent-session handoff as successful completion delivery and finalizes the subagent task before the parent reply has actually been delivered to the user, so the reported lifecycle bug remains plausible and supported by code/tests.

## Evidence

- **Subagent task is terminalized before announce cleanup starts:** `completeSubagentRun()` persists the outcome, then immediately calls `safeFinalizeSubagentTaskRun()` before browser cleanup and before `startSubagentAnnounceCleanupFlow()`. That means the detached subagent task is marked succeeded/failed before the parent-owned announce path runs.
  - file: [src/agents/subagent-registry-lifecycle.ts:673](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/subagent-registry-lifecycle.ts#L673)
  - command: `nl -ba src/agents/subagent-registry-lifecycle.ts | sed -n '647,735p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Delivery success is recorded only later in cleanup:** `finalizeSubagentCleanup()` sets detached task `deliveryStatus: "delivered"` only when `didAnnounce` is true, and otherwise falls into retry/give-up logic. This separation shows completion and delivery are still modeled as different moments, but completion happens first.
  - file: [src/agents/subagent-registry-lifecycle.ts:457](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/subagent-registry-lifecycle.ts#L457)
  - command: `nl -ba src/agents/subagent-registry-lifecycle.ts | sed -n '445,540p'`
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Active requester-session wake is counted as delivered:** For completion-mode announces, if the requester session is active and `queueEmbeddedPiMessage()` succeeds, `deliverSubagentAnnouncement()` returns `{ delivered: true, path: "steered" }` without waiting for any user-visible outbound send.
  - file: [src/agents/subagent-announce-delivery.ts:649](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/subagent-announce-delivery.ts#L649)
  - command: `nl -ba src/agents/subagent-announce-delivery.ts | sed -n '648,770p'`

- **Tests lock in session-internal handoff as delivered:** The completion-delivery test explicitly expects `delivered: true, path: "steered"` and also asserts `callGateway` was not called, so current main intentionally treats an internal requester wake as successful delivery.
  - file: [src/agents/subagent-announce-delivery.test.ts:141](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/subagent-announce-delivery.test.ts#L141)
