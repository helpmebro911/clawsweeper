## Summary

Keep #41164 open. Current main still lacks a config-level default delivery channel/target for new cron jobs, and the observable default path can still resolve to `last`. The existing `cron.failureDestination` only covers failure notifications, not primary cron output routing.

## Evidence

- **Cron config type has no primary delivery defaults:** `CronConfig` exposes scheduler, retry, webhook, retention, run-log, failure-alert, and failure-destination settings, but no `cron.delivery.defaults` or equivalent primary delivery default.
  - file: [src/config/types.cron.ts:30](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/config/types.cron.ts#L30)
  - command: `nl -ba src/config/types.cron.ts | sed -n '1,240p'`
  - sha: [81a41fe5bee3](https://github.com/openclaw/openclaw/commit/81a41fe5bee350c6a035bfcc458eb8be380afe2c)
- **Strict config schema would reject a new cron delivery defaults object:** The cron zod schema is strict and only allows the current cron keys, including `failureDestination`; no primary delivery defaults object is accepted.
  - file: [src/config/zod-schema.ts:568](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/src/config/zod-schema.ts#L568)
