## Summary

Keep open. Current `main` still resolves the TUI gateway target from the current process config/env only, so a gateway started in another terminal with `--port 48789` is not shown to have any path that updates `openclaw tui` away from the default `18789` unless the user also passes `--url` or exports matching env/config.

## Evidence

- **TUI connection path does not inspect daemon/runtime state:** `resolveGatewayConnection()` loads config, validates explicit auth, then builds the URL from `buildGatewayConnectionDetails({ config, ...(urlOverride ? { url: urlOverride } : {}) })`. There is no call here to daemon status, service env, or any runtime-port sync helper before choosing the URL.
  - file: [src/tui/gateway-chat.ts:267](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/tui/gateway-chat.ts#L267)
  - command: `nl -ba src/tui/gateway-chat.ts | sed -n '249,275p'`
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Shared gateway URL builder only uses current-process config/env:** `buildGatewayConnectionDetailsWithResolvers()` computes `localPort` from `resolveGatewayPort(config, process.env)` and then builds `ws://127.0.0.1:${localPort}`. That means the fallback local URL comes only from this process's config/env inputs.
  - file: [src/gateway/connection-details.ts:37](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/gateway/connection-details.ts#L37)
  - command: `nl -ba src/gateway/connection-details.ts | sed -n '29,46p'`
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Port resolver still falls back to 18789 when this process lacks override data:** `resolveGatewayPort()` checks only `env.OPENCLAW_GATEWAY_PORT`, then `cfg.gateway.port`, then returns `DEFAULT_GATEWAY_PORT`. A separate `openclaw gateway --port 48789` process does not populate this new TUI process by itself.
  - file: [src/config/paths.ts:289](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/config/paths.ts#L289)
  - command: `nl -ba src/config/paths.ts | sed -n '285,300p'`
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Daemon/service runtime port data exists elsewhere but is not wired into TUI:** `gatherDaemonStatus()` reads the service command and merges `command.environment` into `serviceEnv`, which is exactly where an active daemon-managed `OPENCLAW_GATEWAY_PORT` can be observed. The TUI path above does not use this mechanism.
