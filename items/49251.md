## Summary

Keep open. Current main has adjacent safeguards: model fallback/cooldown handling, visible rate-limit/billing failure messages, in-process follow-up queues for busy sessions, restart continuation delivery, and main-session restart recovery. It still does not implement the requested durable prompt queue/resume behavior when all models are unavailable because of API limits; the exhausted fallback path surfaces an error asking the user to retry instead of persisting and automatically replaying the prompt when capacity returns.
