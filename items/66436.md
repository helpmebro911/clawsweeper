## Summary

Keep open. Current origin/main still matches the reported macOS failure mode: the generated LaunchAgent uses KeepAlive=true with ThrottleInterval=1, and a missing config still exits with EX_CONFIG before the gateway starts. Existing invalid-config recovery and systemd RestartPreventExitStatus handling do not cover launchd respawn storms for a missing config file.

## Best Possible Solution

Keep #66436 open. The best fix is a launchd-specific config-error storm guard that preserves intentional fast restarts: for example, add a supervised wrapper/backoff or conditional retry suppression for EXIT_CONFIG_ERROR/missing config, or make missing config enter a wait-for-config/degraded mode. Update LaunchAgent tests, daemon/service audit if the plist contract changes, and user-facing gateway troubleshooting docs.
