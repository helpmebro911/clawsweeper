## Summary

Keep this issue open. Current main still emits a macOS LaunchAgent with `KeepAlive=true` and `ThrottleInterval=1`, and the missing-config path exits with sysexits config error 78 before the gateway loop can apply any in-process retry behavior. The adjacent #60885 work is related launchd supervision cleanup, but it does not cover repeated missing-config respawns or log storms.

## Best Possible Solution

Keep this issue open for a macOS LaunchAgent/gateway startup fix that prevents repeated missing-config or `EXIT_CONFIG_ERROR` respawns while preserving intentional restarts. A good solution would add launchd-specific suppression, a sentinel or bounded backoff, a max-retry stop path, or a degraded wait-for-config mode, then update LaunchAgent/gateway regression tests and troubleshooting docs. Coordinate with #60885, but only close as duplicate if that work explicitly covers the missing-config crash-loop/log-storm case.
