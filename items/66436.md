## Summary

Keep this issue open. Current main still has the reported macOS launchd failure shape: the LaunchAgent template keeps `KeepAlive=true` with `ThrottleInterval=1`, while missing gateway config exits with `EX_CONFIG` before the gateway loop starts. The related #60885 work is adjacent launchd supervision/restart recovery, not a canonical fix for missing-config respawn storms.

## Best Possible Solution

Keep this issue open for a macOS launchd-specific storm guard at the LaunchAgent/gateway startup boundary. Preserve fast intentional restarts, but stop repeated missing-config/`EXIT_CONFIG_ERROR` respawns via supervised backoff, max retry, config-error suppression, or a degraded wait-for-config mode. Update LaunchAgent tests and gateway troubleshooting docs, and coordinate with #60885 without treating it as a duplicate unless a canonical issue explicitly covers this missing-config failure mode.

## Risks / Open Questions

- A blanket `ThrottleInterval` increase could regress intentional fast macOS gateway restarts; the safer fix should distinguish repeated config-error exits from normal restart requests.
- Because launchd lacks systemd's `RestartPreventExitStatus=78`, the fix likely needs a launchd-specific wrapper, sentinel, disabled-state handoff, bounded backoff, or degraded wait-for-config mode.
