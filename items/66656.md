## Summary

Keep open. Current main still does not implement this PR's session-preview fix: the transcript tail scanner returns the latest user/assistant-shaped message without checking the entry type, so a `diagnostic.heartbeat` entry with assistant content can still become `lastMessagePreview`. The related #66544 PR addresses a separate display-name fallback path, not this transcript-preview path.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a focused fix in `readLastMessagePreviewFromOpenTranscript` with the included regression coverage, while explicitly choosing whether the filter should cover only `diagnostic.heartbeat` or every `diagnostic.*` entry. Coordinate separately with #66544 because that PR addresses display-name/origin-label leakage, not transcript-preview derivation.
