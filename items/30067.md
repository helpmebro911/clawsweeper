## Summary

Current `main` does not show strong evidence that issue #30067 is already implemented. The onboarding docs and CLI/macOS setup flows include security/auth/dashboard guidance and a dedicated onboarding session, and runtime code improves the message after rate limits happen, but I did not find a proactive first-run heads-up that warns users temporary setup-time background work may trigger API rate-limit errors before chat starts.
