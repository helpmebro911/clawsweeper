## Summary

Keep open. Current main still has the sub-agent documentation drift this PR tries to address, but the PR also needs revision before it is mergeable: the provided review comments about ACP-only `streamTo` and denied-tool wording are source-backed, and current runtime behavior now shows the `announceTimeoutMs` default is 120000 ms rather than the PR body's 90000 ms.

## Best Possible Solution

Keep the PR open for maintainer or contributor follow-up. The best path is to rebase the docs change onto the current `docs/tools/subagents.md` structure, keep the useful fixes from #47558, and correct the source-backed caveats: `streamTo` is ACP-only, depth 2 can be an orchestrator when `maxSpawnDepth >= 3`, the default deny policy is broader than four tools, and `announceTimeoutMs` should document the current runtime default. If a newer PR becomes the maintainer-selected canonical fix, this PR can then be closed as superseded with that link.
