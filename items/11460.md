## Summary

Keep open. Current main still supports WhatsApp outbound reaction writes only: the WhatsApp action adapter advertises/accepts `react`, but not `reactions`, the CLI docs list `reactions` support only for Discord/Google Chat/Slack/Matrix, and the inbound WhatsApp monitor only handles `messages.upsert` message bodies/media rather than reaction events or historical reaction storage. The related PR #13641 was closed unmerged and only attempted inbound event surfacing, so it does not satisfy the query/history request.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin implements a complete reaction read path: retrieve or persist WhatsApp reaction history, advertise and handle the generic `reactions` message action for WhatsApp, surface inbound reaction add/remove events with correct account/conversation routing and access checks, and add focused tests plus docs for CLI/tool behavior.
