## Summary

Keep #11460 open. Current main still supports only outbound WhatsApp `react`; it does not advertise, dispatch, persist, retrieve, or document WhatsApp `reactions` queries, and the related implementation attempts are not merged/shipped.

## Best Possible Solution

Keep #11460 open until the bundled WhatsApp plugin implements a complete reaction read path: advertise and handle the generic `reactions` message action, retrieve or persist historical reactions where feasible, surface inbound add/remove reaction events through normal routing and access checks, and add focused WhatsApp tests plus docs. PR #48570 can be used as implementation input, but it is not a shipped superseding fix.
