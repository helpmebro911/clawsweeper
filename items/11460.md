## Summary

Keep #11460 open. Current main still only supports outbound WhatsApp `react`; it does not implement the requested WhatsApp `message reactions` query path, historical reaction retrieval, or inbound reaction event surfacing. The related PRs in the provided GitHub context are not merged/shipped fixes, and current main does not contain their `messages.reaction`/`reactionMessage` handling.

## Best Possible Solution

Keep #11460 open until the bundled WhatsApp plugin implements a complete reaction read path: advertise and handle the generic `reactions` message action, retrieve or persist historical reactions where feasible, surface inbound add/remove reaction events through normal routing and access checks, and add focused WhatsApp tests plus docs. PR #48570 can remain implementation input, but it is not a shipped superseding fix.
