## Summary

Keep open. On current `main` (`a155b84cdadef068de50274318952032cd867788`), gateway-host `exec` still probes a login shell for PATH, and `openclaw gateway start` still funnels through a service restart path. That means the original restart chain described in #60087 is still present in code; I did not find evidence that it was fixed on `main`.

## Evidence

- **Gateway exec still probes login-shell PATH:** Gateway-host exec calls `getShellPathFromLoginShell(...)` whenever no explicit `PATH` override is provided.
  - file: [src/agents/bash-tools.exec.ts:1590](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/bash-tools.exec.ts#L1590)
  - command: `nl -ba src/agents/bash-tools.exec.ts | sed -n '1586,1606p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Login-shell probe still uses `-l -c`:** `getShellPathFromLoginShell` ultimately runs the configured shell as `shell -l -c 'env -0'`, which sources login-shell startup files such as `~/.profile`.
  - file: [src/infra/shell-env.ts:81](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/infra/shell-env.ts#L81)

  - file: [src/agents/bash-tools.exec.path.test.ts:146](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/bash-tools.exec.path.test.ts#L146)
