## Summary

Current main already resolves bare model aliases and bare model ids through the configured alias index / unique-provider inference, so the reported fallback to `minimax/k2p5` does not match current behavior. The proposed UI patch would also change the option value to a full key while the `<select>` is still bound to the raw alias string, which would leave alias-backed configs without a matching selected option.

## Close Comment

Closing after a Codex review of current `main`. I could not verify the reported failure on the current code path: bare aliases and bare model ids are already resolved through the configured alias index / unique-provider inference in `src/agents/model-selection-shared.ts:378` and `src/agents/model-selection-shared.ts:603`, and the existing tests in `src/agents/model-selection.test.ts:802` and `src/agents/model-selection.test.ts:931` cover that behavior. On that basis, this PR does not look actionable against current `main`, so I’m closing it as `cannot_reproduce`.
