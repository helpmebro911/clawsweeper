## Summary

Current `main` does not contain this fix. `buildExecApprovalPendingReplyPayload` still emits fenced code blocks for approval commands, BlueBubbles still runs those messages through `stripMarkdown` before send, and `stripMarkdown` still only removes single-backtick inline code. A local reproduction against the checked-out `main` leaves visible backtick fence fragments in the rendered plain-text output. The PR also has an unresolved review concern: the proposed fenced-block regex can drop same-line triple-backtick inline spans (`Run ```npm i``` now`), so this is not strong enough to close as already-implemented or obsolete.
