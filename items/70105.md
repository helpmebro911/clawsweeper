## Summary

Keep open. Current main still lacks the PR's debug event-log sticky-bottom behavior: gateway events are prepended into the shared buffer, the Debug view maps that array directly, there is no debug-event-log scroll state/handler/scheduler, and tests only cover chat/log scroll behavior. The related #69520 is about chat auto-follow, not the Debug page event log, so it does not supersede this PR.

## Best Possible Solution

Keep the PR open for maintainer review or implementation. The best product path is to add a debug-event-log scroll state and scheduler analogous to logs auto-follow, render the Debug event log inside its own bounded scroll container, preserve manual scroll position when users inspect history, and add focused tests in `ui/src/ui/app-scroll.test.ts` plus any lifecycle coverage needed for `eventLog` updates.
