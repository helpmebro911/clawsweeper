## Summary

Keep this PR open. Current main still accepts the first embedded-agent payload text as the slug source, ignores the payload error signal, strips edge dashes before truncation, and then persists any non-null slug into the session-memory filename. The related #38161 work is still open and broader, while the merged #66455 timeout fix does not implement these safeguards.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. The best fix belongs in `src/hooks/llm-slug-generator.ts`: extract and test pure slugification, reject error-marked or non-slug-like embedded-agent payloads before filename construction, and strip leading/trailing dashes after truncation. Close only after this PR or an equivalent merged change ships those safeguards, with coordination against #38161.

## Risks / Open Questions

- The proposed helper appears threshold-based and should also explicitly reject payloads where `isError === true`; a short provider error such as `quota exceeded` could otherwise still look slug-like.
- The word and character thresholds are product-sensitive, though the existing prompt asks for a 1-2 word slug and the timestamp fallback is already supported.
- #38161 overlaps in the broader slug-hardening area, so maintainers should coordinate final behavior if both PRs continue.
