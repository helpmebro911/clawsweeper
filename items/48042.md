## Summary

Keep open. Current main still resolves the reported config to the implicit `user` Chrome MCP existing-session profile when `browser.cdpUrl` and `defaultProfile: "user"` are both set without an explicit `profiles.user`, so the browser path can still reach the DevToolsActivePort-based attach flow instead of the configured CDP URL. The related PR #48068 described the right targeted fix but was closed unmerged, and I do not see that behavior in current main.

## Evidence
