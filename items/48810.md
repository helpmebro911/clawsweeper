## Summary

Keep #48810 open. Current main still has a plausible unresolved compaction-timeout/retry integrity gap: OpenClaw wraps compaction in a Promise.race timeout and releases the session write lock after timeout, but the losing compaction promise is not awaited. If the underlying pi-coding-agent compaction has already appended or continues despite abort, a later retry can still append another compaction event. I found no current repair or regression test that enforces a linear parentId chain for duplicate compaction siblings.
