## Summary

Keep open. The reported failure mode still exists on current main: plugin-local schema errors become invalid config, gateway startup throws instead of quarantining one plugin, and registry loader paths still request fatal load-error behavior. The related PR #70394 is the right shape of follow-up but is provided as open/unmerged and its proposed change is not present in this checkout.

## Best Possible Solution

Keep #70371 open and land a narrow plugin-local degradation path. Invalid `plugins.entries.<id>.config` should mark only that plugin entry as failed/disabled in the in-memory registry, skip its registration/tool exposure, keep gateway and unrelated plugins/channels booting, and make `openclaw doctor` report and repair or quarantine the bad entry. Regression tests should cover startup continuing with one invalid configured plugin, loader metadata/ensure paths not throwing for that isolated failure, and doctor surfacing the exact config path and schema rule.
