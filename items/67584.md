## Risks / Open Questions

- Maintainers still need to choose whether this narrow plugin-owned templating design should land or whether #63829 should drive a broader per-agent memory-wiki configuration model.
- The PR changes filesystem path resolution and memory-wiki data boundaries, so review should focus on fail-closed handling for missing/unknown placeholders, `..` normalization, and write paths such as `fs.mkdir(..., { recursive: true })`.
- The branch intentionally skips memory prompt/corpus supplements when `vault.path` is templated; that is a plausible minimum-scope fix but changes `memory_search`/`memory_get` availability for templated setups and needs maintainer acceptance.
- Docs and PR wording should avoid implying hostile multi-tenant security isolation on one shared gateway; current security docs define a trusted personal-assistant boundary.
