## Summary

Keep #38364 open. Current main still lacks the central requested path: structured reply-to metadata from Control UI/chat.send into the agent MsgContext. The no-output side is partially mitigated by NO_REPLY/history filtering, but that does not implement the reply metadata request.

## Best Possible Solution

Keep #38364 open. Best path: add an additive chat.send reply metadata field, add Control UI composer/thread state to capture the selected parent message, validate/map it in the Gateway into MsgContext ReplyTo* fields plus optional parent-content summary, then cover the protocol, Gateway mapping, and Control UI request behavior with focused tests. Preserve the current NO_REPLY/no-output filtering as the partial fix already on main.
