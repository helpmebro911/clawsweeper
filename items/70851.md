## Summary

Keep PR #70851 open. Current main still adds every listed session row to the chat picker except global/unknown and optionally cron rows; there is no internal-session visibility filter for subagent, ACP, or node-backed rows. The gateway already carries lineage metadata such as `spawnedBy` and `spawnDepth`, but the UI picker does not consume an internal classification, and the UI-side row type is not aligned with the gateway metadata. Related PR #69552 was closed unmerged, so it is related context rather than a shipped canonical fix.

## Best Possible Solution

Keep this open and revise it toward the stated end-state: add a stable gateway/session classification for internal runtime rows, or at minimum a shared parser-backed internal-session helper; have the chat picker hide internal rows by default while keeping the selected internal row visible; keep admin/debug session surfaces able to request/show internals; and add focused coverage for subagent, ACP, node-backed, spawnDepth-only fallback, active-session exception, and false-positive keys.
