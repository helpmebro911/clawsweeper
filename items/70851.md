## Summary

Keep PR #70851 open. Current main still does not hide subagent/ACP/node-backed internal sessions from the chat picker, and the related #69552 was closed/unmerged as a narrower earlier attempt rather than a shipped canonical fix. The PR remains a plausible focused fix, though it should be reviewed/reworked around precise internal-session classification rather than broad substring matching.

## Best Possible Solution

Keep this PR open for maintainer review rather than auto-closing it. The best fix is to add an explicit internal-session classification at the gateway/session source, expose it through the session row contract, and have the normal chat picker hide internal rows by default while preserving the currently selected internal session. If maintainers accept an interim UI mitigation, it should use precise parsed helpers for subagent/ACP/node-backed keys and include tests for active internal sessions and any `spawnDepth` fallback.
