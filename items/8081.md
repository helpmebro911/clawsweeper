## Summary

Keep #8081 open. Current main has useful lower-level controls, but it still documents a single trusted-operator gateway model and does not implement shared-gateway admin/user/guest RBAC, per-user credential isolation, or audit semantics for sensitive configuration and credential operations. The latest issue discussion also has a maintainer-authored Codex review by steipete explicitly keeping this tracker open.

## Best Possible Solution

Keep #8081 open as the broad product/security tracker. A complete solution should define gateway and channel caller identity, role- or scope-based authorization for sensitive config and credential operations, per-user or per-tenant secret isolation, audit logging, channel/group identity mapping, documented trust boundaries, and regression coverage across gateway auth, config read/write paths, HTTP APIs, and channel command surfaces. If maintainers choose an external-wrapper or ClawHub-first direction instead, document that explicitly and identify any missing plugin/API surfaces needed to support it outside core.
