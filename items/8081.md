## Summary

Keep #8081 open. Current main has partial controls, but it still documents a single trusted-operator gateway model and does not implement shared-gateway admin/user/guest RBAC, per-user credential isolation, or audit semantics. The latest maintainer-authored issue comment also explicitly keeps this tracker open.

## Best Possible Solution

Keep #8081 open as the broad product/security tracker. A complete solution should define gateway/channel caller identity, role- or scope-based authorization for sensitive config and credential operations, per-user or per-tenant secret isolation, audit logging, channel/group identity mapping, documented trust boundaries, and regression coverage across gateway auth, config read/write paths, HTTP APIs, and channel command surfaces. If maintainers choose an external-wrapper or ClawHub-first direction, document that explicitly and identify any missing plugin/API surfaces needed to support it outside core.

## Risks / Open Questions

- Partial controls such as trusted-proxy allowUsers, operator scopes, owner-only surfaces, sender-scoped tool policies, SecretRef handling, and per-agent sandboxing can be mistaken for full multi-user RBAC.
