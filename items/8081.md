## Summary

Current main still does not implement the requested multi-user RBAC/auth/secrets-isolation system. It has useful hardening around gateway auth, config redaction, DM/session isolation, tool policy, and multi-user warnings, but the documented trust model remains one trusted operator boundary per gateway, not per-user tenant roles with admin/user/guest permissions. A maintainer review comment on the issue also explicitly kept this open on 2026-04-26, and the provided context shows active related work and external wrappers rather than a shipped core fix.

## Best Possible Solution

Keep #8081 open as the broad product/security tracking issue until maintainers choose a direction. A core solution would need an explicit identity and authorization model for gateway/API/channel callers, role/scoped permissions, sensitive config and credential isolation/redaction semantics, audit logging, and docs/tests that update the current personal-assistant trust model. If maintainers decide this belongs outside core, close only after documenting the external-wrapper/plugin path and confirming no missing core extension API blocks it.
