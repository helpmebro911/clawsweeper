## Summary

Keep open. Current main partially addresses the RPC path: `sessions.patch` now emits `session:patch`, and the Control UI `/model` command calls that RPC. But the channel `/model` directive path and the `session_status` tool still mutate session model fields directly through session-store writes without creating or triggering a `session:patch` hook event. That leaves a meaningful part of the issue's requested behavior unresolved.
