## Summary

Keep open. Current main has prompt-cache telemetry/context-engine plumbing, but it does not implement the requested opt-in keep-warm ping lifecycle: no proposed config surface, no session activity timestamp fields, and no gateway/agent timer that sends invisible max_tokens=1 Anthropic requests between user turns. The issue's cited related work (#62179) is reflected in current runtimeContext.promptCache support, but that is foundational telemetry rather than the feature requested here.
