## Summary

Keep PR #70968 open. Current main still embeds managed durable/auth env values into the gateway install environment, launchd renders those values into the plist, and runtime dotenv loading skips keys already present in process.env. The PR is not implemented on main or superseded by local history, but its current patch likely needs refinement for the PATH/service-owned-env edge case raised in review.

## Best Possible Solution

Keep the PR open and ask for a small revision: remove literal managed dotenv/config/auth secret values from the LaunchAgent install environment, preserve generated service-owned variables such as PATH/HOME/TMPDIR and required OPENCLAW_* service metadata, and add regression coverage that managed secret literals are absent while the curated service PATH remains present. A macOS-specific analogue to the existing systemd env-file handling could also be considered, but the minimum fix belongs in `buildGatewayInstallEnvironment`/LaunchAgent install coverage.
