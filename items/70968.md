## Summary

Keep PR #70968 open. Current main still includes literal managed `.env`/config/auth-profile values in the gateway install-plan environment and the LaunchAgent writer will serialize them into the plist; `loadRuntimeDotEnvFile` then skips fresh `.env` values when the key already exists in `process.env`. The PR targets a real remaining bug from #70612, but its current implementation needs maintainer review or a small adjustment for the PATH/service-owned-variable concern raised in review before it should land.

## Best Possible Solution

Keep the PR open and revise it into a narrow gateway install-plan fix: drop literal values for managed secret/config/auth-profile keys before writing LaunchAgent environment, preserve generated service-owned variables such as PATH/HOME/TMPDIR and required OpenClaw service metadata, and update `daemon-install-helpers` regression tests to prove secrets are omitted while service PATH/port behavior remains intact.
