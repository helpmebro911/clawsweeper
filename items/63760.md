## Summary

Keep open. Current main still runs subagent/ACP completion announces as ordinary requester-session agent calls with no announce-specific model override or no-history/light-context policy; existing knobs cover child run model/light context and announce timeout only.

## Best Possible Solution

Keep this issue open and implement a schema-backed announce policy, likely under agents.defaults.subagents.announce with per-agent inheritance where subagent defaults already exist. Wire it into direct, queued, and descendant-wake announce agent calls so completion handoff can use an allowed cheaper model and light/no-history context without changing spawned child model behavior. Add focused tests for native subagents, ACP completions, nested requester sessions, queue fallback, and unauthorized model overrides, then document the knobs on the public Sub-agents and ACP agents pages.
