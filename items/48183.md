## Summary

Keep open. Current main still contains the reported Feishu HTTP server cleanup race: `server.close()` is called and `httpServers` entries are deleted or cleared immediately, with no `closeAllConnections()` or awaited close callback. The linked Feishu cleanup PRs are evidence of active attempted fixes, but they are open or closed unmerged; the broader ResourceManager issues are follow-up architecture work, not a shipped fix for this bug.

## Evidence

- **Current Feishu monitor state cleanup still deletes immediately after close:** `stopFeishuMonitorState(accountId)` is synchronous and calls `server.close()` followed immediately by `httpServers.delete(accountId)`. The global cleanup loop similarly calls `server.close()` for each server and then immediately clears the map.
