## Summary

Keep this issue open. Current main still does not discover, diagnose, or repair the reported layout where a complete legacy-named config remains beside a missing or skeletal canonical config in the default OpenClaw state directory. Later crash-loop and last-known-good recovery work narrows related symptoms, but it does not cover the first-upgrade sibling-file case.

## Best Possible Solution

Keep this open for a narrow config-recovery fix. Do not restore broad legacy runtime auto-detection; instead add a startup or doctor diagnostic for a legacy-named config in the default OpenClaw state directory when canonical openclaw.json is missing, empty, or skeletal, offer a safe doctor --fix copy or merge path when feasible, and update the public migration/troubleshooting guidance.
