## Summary

Keep open. Current main still does not discover, diagnose, or repair the specific `~/.openclaw/moltbot.json` layout reported here. Main has improved adjacent crash-loop diagnostics and last-known-good recovery, but those paths depend on the canonical `openclaw.json` path or an existing backup and do not inspect a legacy `moltbot.json` already inside `~/.openclaw/`.

## Best Possible Solution

Keep this open for a narrow maintainer fix. The best path is to avoid reviving broad load-time legacy auto-detection, but add an explicit Doctor/startup diagnostic for `~/.openclaw/moltbot.json` when canonical `openclaw.json` is missing or skeletal, offer a safe `doctor --fix` copy or merge path when feasible, and update the public migration/changelog wording so users with `moltbot.json` already under `~/.openclaw/` know exactly what to do.
