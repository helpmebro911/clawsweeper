## Summary

Keep open. Current main has implemented several Mattermost parity pieces from the umbrella, including reactions, inbound reaction events, interactive buttons, threading, typing indicators, markdown-table handling, and media upload. The provided discussion also correctly credits PR #19957 for the rich-interactions portion. However, the issue still asks for read receipts / presence, and current Mattermost support only exposes send/react message actions with no Mattermost presence or read-receipt implementation/docs. A meaningful requested capability remains missing, so conservative maintainer cleanup should not close #15634.
