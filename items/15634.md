## Summary

Keep #15634 open. Current main has substantial Mattermost parity work, including threading, markdown-preserving delivery, reactions, typing indicators, media, and interactive buttons, but the umbrella still explicitly asks for read receipts / presence and current Mattermost source/docs do not implement or document that scope. The issue discussion also already has a recent maintainer-side Codex review saying to keep the umbrella open for that remaining gap.

## Best Possible Solution

Keep #15634 open as the Mattermost parity umbrella for now, or split the unresolved read receipts / presence work into a narrower canonical Mattermost issue and close this umbrella only after that remaining scope is implemented, documented, or explicitly descoped by a maintainer. Keep #19957 as canonical evidence for the completed rich-interactions/buttons portion.
