## Summary

Keep open. Current main still implements `POST /hooks/agent` as an async fire-and-forget trigger that returns only `{ ok, runId }`; it does not parse `blocking`, wait for completion, return response text, or expose blocking error semantics. This PR remains a concrete implementation candidate for the still-open result-return request tracked by #49019, with review feedback and follow-up fixes rather than obsolete or already-shipped work.

## Best Possible Solution

Keep this PR open as the concrete implementation path.
