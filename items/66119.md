## Summary

Keep this issue open. The report is recent, detailed, and destructive, and current main still auto-starts the bundled Signal plugin's `signal-cli daemon` path without any preflight for a local account file with `registered:false`, automatic backup, or fail-closed recovery warning. I found no current implementation, docs, tests, changelog, history, or related-item evidence that fixes or supersedes this exact failure mode.

## Best Possible Solution

Keep the issue open and investigate in the bundled Signal plugin. The best path is a Signal-owned, non-mutating account-state preflight before daemon auto-start where feasible, fail-closed warning behavior for detected `registered:false`, mocked regression coverage around startup, and user-facing backup/recovery docs. If the destructive transition is ultimately upstream signal-cli behavior, OpenClaw should still avoid auto-starting the daemon when it can safely detect an obviously invalid local account state.
