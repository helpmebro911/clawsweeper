## Summary

Current main still persists `models auth login` credentials into the resolved local agent auth store and has no remote node-host guard or gateway-side auth-profile mutation path. The provided GitHub context also shows open PR #42381 with closing syntax for this issue, so the issue should remain open until that implementation candidate is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #42381. If maintainers accept the blocking approach, land a focused guard in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts`; if they prefer credential sync, first design an explicit authenticated gateway auth-profile mutation API.
