## Summary

Keep open. Current main still writes `openclaw models auth login` results to the locally resolved agent auth store, with no remote node-host guard or gateway credential-sync path. PR #42381 remains open with closing syntax and should be reviewed as the implementation candidate before this issue closes.

## Best Possible Solution

Keep this issue open and review PR #42381. The conservative fix belongs in `src/commands/models/auth.ts` with regression coverage in `src/commands/models/auth.test.ts` to block or clearly warn on OAuth login from remote node hosts; if direct node-host OAuth should work, first design a secure gateway-side auth-profile mutation API and scopes.
