## Summary

Keep open. Current main still lacks Feishu lifecycle/status reporting into the gateway runtime snapshot, so this PR is not obsolete. The PR also should not be cleanup-closed as implemented or superseded: the supplied PR file list does not include the Feishu channel start path or `monitor.account.ts`, and current gateway health policy keys stale-socket detection on `lastTransportActivityAt`, not app-level `lastEventAt`.

## Best Possible Solution

Keep this PR open and ask for a rebase/revision against current main. The best implementation should pass `ctx.setStatus` from `extensions/feishu/src/channel.ts` through `monitorFeishuProvider`, `monitorSingleAccount`, event handlers, and websocket/webhook transports; publish `connected`, `mode`, `lastConnectedAt`, disconnect/error state, and meaningful `lastTransportActivityAt` updates; and add focused Feishu lifecycle tests that prove the gateway health policy sees the new snapshot fields.
