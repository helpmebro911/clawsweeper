## Summary

Keep #70529 open. Current main still defaults Chrome MCP existing-session attachment to spawning `npx -y chrome-devtools-mcp@latest`, and the docs still describe `profile="user"` as a supported path that requires the Gateway/node host to spawn that command. The maintainer comment explicitly calls this out as a Desktop-specific packaged-app failure separate from the broader open #62288 and #63926 follow-ups.

## Best Possible Solution

Keep the issue open and fix the Desktop/browser runtime so existing-session attach does not depend on ambient `npx` in packaged apps. Best path is to bundle or explicitly resolve the Chrome DevTools MCP server for Desktop, or degrade with a structured diagnostic that recommends the managed `openclaw` profile and points login-handoff UX to #63926 while leaving broader existing-session robustness under #62288.
