## Summary

Keep #70529 open. Current main still defaults the built-in `user` / existing-session browser profile to spawning ambient `npx chrome-devtools-mcp@latest`, while the Desktop-specific report says the packaged app ships `node` without `npm`/`npx`. A maintainer comment explicitly says to keep this open as the Desktop-specific variant rather than closing it into #62288 or #63926.

## Best Possible Solution

Keep the issue open and fix the Desktop/browser existing-session runtime so packaged Desktop does not depend on ambient `npx`. The likely product direction is to bundle or explicitly resolve the Chrome DevTools MCP driver for Desktop, or degrade with a structured diagnostic that names the packaged-app limitation and recommends the managed `openclaw` profile or a configured `mcpCommand`. Do not close it as duplicate of #62288 because maintainer triage called #70529 the Desktop-specific `npx` packaging variant.
