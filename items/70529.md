## Summary

Keep #70529 open. Current main still routes the built-in `user` profile through the Chrome MCP existing-session driver, and that driver still defaults to spawning `npx -y chrome-devtools-mcp@latest`; I found no Desktop-safe bundled driver resolution or `spawn npx ENOENT` diagnostic on main. The existing maintainer comment also explicitly asks to keep this as the Desktop-specific variant rather than close it under the broader existing-session attach issues.

## Best Possible Solution

Keep #70529 open as the Desktop-specific existing-session bug. The best fix is to make the Desktop/browser runtime independent of ambient `npx` for Chrome MCP attach, either by bundling/resolving a packaged `chrome-devtools-mcp` executable or by adding a Desktop-aware command resolution path, plus an explicit diagnostic for missing `npx` that recommends the managed `openclaw` profile/login handoff. Coordinate with #62288 for general existing-session brittleness and #63926 for visible login handoff UX, but do not close this as a duplicate because the packaged-app `npx` failure remains distinct and unshipped.
