## Summary

Keep open. Current main still has the WhatsApp auth-dir path behavior described in the report, and the linked implementation candidate #64563 is still open and unmerged. The issue should remain open until that PR, or an equivalent focused fix, lands.

## Best Possible Solution

Keep this issue open and resolve it through #64563 or an equivalent focused patch: make the WhatsApp default auth dir lazy or explicitly env/state-dir backed, align `listWhatsAppAuthDirs()` and default/legacy auth-dir helpers with the same OAuth root, and add an `OPENCLAW_STATE_DIR` profile-isolation regression test. Doctor/docs guidance for `OPENCLAW_OAUTH_DIR` can follow after the core credential isolation bug is fixed.
