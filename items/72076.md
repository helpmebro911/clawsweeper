## Summary

Keep this PR open. Current main has a Codex-mode Computer Use setup path, but it does not implement the PR's broader all-harness `computer` plugin. The discussion also has active maintainer product/security input that a plain plugin is not sufficient and that a computer-use SDK plus security/permissions model is under consideration, so this is not a conservative auto-close candidate.

## Best Possible Solution

Keep the PR open for maintainer product and security review. The best path is to settle the generic computer-use extension SDK, permission, and security model first, then decide whether the cua-driver implementation should be revised against that contract, published externally through ClawHub/npm, or promoted as a bundled plugin with explicit maintainer ownership.
