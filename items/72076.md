## Summary

Keep #72076 open. It is a plausible computer-use plugin contribution, but not a conservative auto-close: current main does not include `extensions/computer`, and the PR discussion has explicit maintainer signal that a simple plugin is insufficient while OpenClaw works through a computer-use SDK plus security and permissions model. Existing docs generally prefer external plugins/ClawHub, but this specific desktop-control surface needs maintainer product/security judgment and the PR also has unresolved implementation review findings.

## Best Possible Solution

Keep the PR open for maintainer handling. The likely path is to first define the computer-use extension SDK and permission/security model, including how desktop-control tools are approved, audited, scoped to macOS/node-host permissions, and exposed across harnesses. After that contract exists, the cua-driver work can either be revised against it or published externally through ClawHub/npm if core promotion is not warranted. The current PR should not be merged as-is until the reported schema, error-handling, import, validation, and screenshot behavior defects are fixed.
