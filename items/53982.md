## Summary

Keep this issue open. Current main still documents and displays the incomplete Nextcloud Talk bot install command with only `--feature reaction`, while outbound 401 errors still say only to check the bot secret. Upstream Nextcloud Talk still gates bot message responses behind `Bot::FEATURE_RESPONSE`, so the reported user-visible failure remains actionable in the Nextcloud Talk plugin/docs surface.

## Best Possible Solution

Keep this issue open and apply a narrow Nextcloud Talk plugin/docs fix: update the English Nextcloud Talk setup command and setup wizard to include `--feature webhook --feature response --feature reaction`, add a short troubleshooting note for HTTP 401 replies when the bot was registered without response permission, improve `extensions/nextcloud-talk/src/send.ts` to mention re-registering with `--feature response`, and cover the docs/wizard or send-error behavior with focused tests. Do not edit generated localized docs directly.
