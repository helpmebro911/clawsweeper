## Summary

Keep this issue open. Current main still omits `--feature response` from the Nextcloud Talk setup docs and setup wizard, and outbound send failures still map HTTP 401 only to a bot-secret diagnostic. Upstream Nextcloud Talk stable32 confirms outgoing bot messages are filtered through `FEATURE_RESPONSE`, and the related PR attempts were closed unmerged rather than shipped.

## Best Possible Solution

Apply a narrow Nextcloud Talk plugin/docs fix: update the public setup command and setup wizard intro to use `--feature webhook --feature response --feature reaction`, add a short 401 troubleshooting note for bots registered without `response`, and improve the send-path 401 error to mention re-registering with `--feature response` while preserving the shared-secret mismatch possibility. Add focused coverage for the setup text or send-error behavior.
