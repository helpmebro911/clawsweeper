## Summary

Keep open. Current main has path/line citations, short-term promotion source markers, event logs, and a separate provenance-rich memory-wiki layer, but it still does not implement the requested durable-memory provenance model: required sourceRefs, multiple supporting sources, merge/edit provenance retention, or promotion blocking for source-less durable memories.

## Best Possible Solution

Keep this issue open and implement it with the adjacent durable record and write-pipeline work in #42646 and #42648: add a typed provenance/sourceRefs model to durable memory records, require at least one source before promotion, preserve provenance through create/merge/edit/conflict outcomes, and expose the refs consistently in retrieval and review flows. Memory Wiki should consume or mirror that core model rather than substitute for it.

## Risks / Open Questions

- Closing this now would treat path/line citations and promotion markers as equivalent to required durable provenance, leaving sourceRefs, multiple supporting sources, merge/edit history, and source-less promotion gating untracked.
