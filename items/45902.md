## Summary

Keep PR #45902 open. Current main does not implement the PR's central behavior: setting `/model` inside a Telegram topic/thread does not persist a parent-chat future-thread default for newly created Telegram threads. Main only has generic parent model override reads when a parent already has `providerOverride`/`modelOverride`, while `/model` still writes the active session entry only. The PR also carries unresolved review concerns and broad unrelated changes, so it should not be closed as implemented or obsolete.

## Best Possible Solution

Keep the PR open as a useful but stale implementation attempt. The best path is a focused replacement or heavily rebased PR that only changes Telegram thread model-default behavior, stores a scoped parent-chat future-thread default without cross-chat leakage, handles `direct_messages_topic` routing if still required, and adds regression tests for forum topics, DM topic threads, existing-thread non-mutation, parent write races, and main-scoped DM isolation.
