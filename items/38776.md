## Summary

Keep #38776 open. Current main still only partially parses Feishu quoted interactive-card content, and the PR remains useful because the requested header/i18n/template/nested-card extraction is not implemented or superseded by a landed replacement. There are overlapping open Feishu card parser items, but no clear merged or maintainer-selected canonical successor for this focused PR.

## Best Possible Solution

Keep #38776 open unless maintainers explicitly choose a canonical successor. The best path is a focused Feishu plugin parser update in `extensions/feishu/src/send.ts` with regression tests for header titles, non-empty fallback across `elements`/`body.elements`/`i18n_elements`, primitive template variables, nested card elements, and raw/post/card-reference shapes returned by the current Feishu message read APIs. If a broader PR such as #56795, #42218, or another maintainer-selected implementation becomes canonical, close #38776 only after linking that replacement.
