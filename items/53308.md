## Summary

Current main still has an actionable bundled Signal group-ingress bug. Signal group messages are authorized through sender-based `groupAllowFrom` before per-group Signal `groups` config is consulted, so a configured group ID alone cannot admit the message. The `requireMention` default still falls through to `true`, but docs currently describe mention gating as the default, so that part needs maintainer policy judgment rather than an auto-close.

## Best Possible Solution

Keep this issue open and fix the bundled Signal plugin. The likely path is to make Signal group admission consult `channels.signal.groups`/`resolveChannelGroupPolicy` for the Signal group ID before sender authorization can drop the event, while preserving `groupAllowFrom` as sender authorization inside allowed groups. Add regression tests for a configured Signal group ID with no sender `groupAllowFrom`, for sender allowlist rejection/acceptance inside configured groups, and for the explicit `requireMention:false` workaround. Decide separately whether an explicit group entry without `requireMention` should default to false or whether docs/config should continue requiring an explicit override.
