## Summary

Keep open. Current main still matches the reported dropped Signal group path: Signal group admission is evaluated through the sender allowlist helper, while Signal group IDs in `channels.signal.groups` are not consulted before the handler blocks. The mention helper also still defaults to `true` when a group entry exists without an explicit `requireMention`, so the no-mention half of the report remains an actionable product/code decision rather than a close candidate.

## Best Possible Solution

Keep this issue open and fix the bundled Signal plugin. Add a group-id admission check using `channels.signal.groups` under `groupPolicy: "allowlist"` before sender authorization blocks the message, while preserving `groupAllowFrom` as sender authorization. Then decide whether Signal-specific group entries should default to `requireMention: false` or whether docs must require an explicit override, and add regression tests for specific Signal group ids under allowlist policy.
