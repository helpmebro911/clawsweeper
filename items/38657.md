## Summary

Current main still leaves the broader #38657 behavior unresolved. The related discussion/PR narrowed one stale provider-baseUrl case, but the checked-in merge path still preserves existing non-empty `models.json` `apiKey` and same-surface `baseUrl` values over newly generated provider config, and tests/docs explicitly lock that preservation behavior.

## Best Possible Solution

Keep #38657 open. The best fix is an explicit Onboard/provider-auth update path that distinguishes deliberate same-provider edits from omitted fields: new apiKey and baseUrl values should persist through `openclaw.json`, regenerated `agents/*/agent/models.json`, and auth-profile handling, while existing runtime values are preserved only when the new config truly omits those fields or SecretRef/auth-profile ownership requires it. Add regression coverage for Moonshot CN/global baseUrl edits, plaintext key replacement, same API-surface baseUrl changes, SecretRef-managed credentials, and auth-profile precedence.
