## Summary

Keep open. Current main still implements the issue's reported skill-onboarding path as a prompt that only shows the Homebrew install command, then proceeds to `skills.install`, where a brew-backed install fails when no brew executable is available. The top-level macOS installer does install Homebrew, but that separate `scripts/install.sh` behavior does not cover the onboarding skill dependency flow described here.

## Best Possible Solution

Keep this issue open and address the skill dependency onboarding path directly. A conservative implementation would add an explicit macOS-only confirmation path that can install Homebrew before running brew-backed `skills.install`, reusing the admin and failure-handling expectations from `scripts/install.sh`. If maintainers do not want onboarding to install Homebrew, change the prompt and docs so users are not led to expect automatic installation. Coordinate Docker/Linux behavior with #14593 and skill feedback/fallback behavior with #57555.
