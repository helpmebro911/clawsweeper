## Summary

Keep open. Current main ships the external SecretRef foundation for env/file/exec and documents a 1Password CLI recipe through exec, but the requested native keyring/Keychain and native 1Password provider sources are still absent from the source contract, config schema, resolver dispatch, Gateway protocol, CLI/setup UX, and plugin setup helpers. The issue discussion already includes a maintainer Codex review keeping this open for that remaining scope.

## Best Possible Solution

Keep #17311 open but narrow it to the remaining native providers: keyring/Keychain and 1Password. A complete fix would add those sources across `SecretRefSource`, Zod/config schemas, resolver runtime, Gateway protocol schema, CLI/config/setup flows, plugin setup helpers, docs, and tests. If maintainers prefer env/file/exec plus documented exec recipes as the final design, record that decision and open narrower follow-ups before closing this broad issue.
