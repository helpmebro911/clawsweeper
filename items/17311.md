## Summary

Keep #17311 open. Current main implements SecretRefs for env, file, and exec and documents 1Password only as an exec recipe, but native keyring/Keychain and native 1Password provider sources are still absent from the type contract, config schema, resolver dispatch, Gateway protocol, CLI setup flow, and plugin-facing setup helpers. The latest issue discussion also contains a maintainer-authored Codex review explicitly keeping this open for the same remaining scope.

## Best Possible Solution

Keep the issue open and narrow the maintainer path to the missing native providers: add keyring/Keychain and 1Password SecretRef sources across types, schemas, resolver runtime, Gateway protocol, CLI/setup UX, docs, and tests; or have a maintainer explicitly decide that env/file/exec plus documented exec recipes are the final product direction and link focused follow-up issues for any remaining macOS/keyring needs.
