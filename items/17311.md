## Summary

Keep this issue open. Current main partially shipped the SecretRef foundation with env/file/exec providers and docs for 1Password through an exec recipe, but it still does not implement native keyring/Keychain, native 1Password, or a plugin-owned SecretRef provider/source seam. Same-author PR #72548 is open and directly addresses the remaining architecture/keyring work, so this issue should stay open until that PR is reviewed, landed, or declined.

## Best Possible Solution

Review PR #72548 as the active implementation candidate. If it lands, close this issue as implemented for the env/keyring/plugin-seam portion and split a focused native 1Password follow-up only if maintainers still want it bundled. If it is declined, narrow this issue to the accepted native keyring/Keychain, native 1Password, or plugin SecretRef provider API work, with cloud-provider follow-up left to #13610 or dedicated provider issues.
