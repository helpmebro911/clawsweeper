## Summary

Keep this issue open. Current main has the env/file/exec SecretRef foundation and documents 1Password through an exec-provider recipe, but it still does not have native keyring/Keychain, native 1Password, or a plugin SecretRef provider seam. Same-author PR #72548 is the active implementation candidate, so this issue should not be closed before that PR is reviewed, landed, or declined.

## Best Possible Solution

Review #72548 as the active implementation candidate. If it lands, close this issue as implemented or split a focused native 1Password follow-up if maintainers still want it bundled. If it is declined, document the product decision and narrow this issue to the accepted native keyring/Keychain, native 1Password, or plugin SecretRef provider API work, with broader cloud-provider tracking left to #13610 or dedicated provider issues.
