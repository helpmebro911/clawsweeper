## Summary

Keep #39142 open. Current main still sanitizes model fallback exhaustion into fixed billing/rate-limit/generic copy, but it does not expose a supported configurable `messages.modelExhausted` or equivalent user-facing exhaustion-message setting.

## Best Possible Solution

Keep the issue open until core adds a supported configurable model-exhaustion reply surface or maintainers explicitly choose a different product direction. A complete fix likely belongs in `MessagesConfig`, `MessagesSchema`, generated config help/labels/docs, and the fallback failure branch in `runAgentTurnWithFallback`, with focused tests for pure billing exhaustion, pure rate-limit exhaustion, and mixed billing/rate-limit behavior.
