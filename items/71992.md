## Summary

Current main does not implement the requested Control UI webchat fix. The `final` and `aborted` handlers still mutate `chatMessages` before clearing `chatStream`/`chatRunId`, which matches the reported Lit rendering race. The related PR #72004 is open and appears to be the right focused path, but the fix is not in `main` yet.

## Best Possible Solution

Keep #71992 open until a focused fix such as PR #72004 lands on `main`, with regression coverage that proves `chatStream`/`chatRunId` are cleared before `chatMessages` is mutated in both `final` and `aborted` paths. Keep broader persisted-transcript duplication coordinated with #69208 or a narrower backend issue instead of closing this report prematurely.
