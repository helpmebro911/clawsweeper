## Summary

Current main already implements the PR's central behavior: `channels.status` probe work is bounded-concurrent while preserving deterministic result order, and the normal route-first CLI path uses a 30000ms probe default when `--timeout` is omitted. The PR is now superseded as an implementation candidate; only a narrow Commander-fallback cleanup remains as optional hardening.

## Best Possible Solution

Close this PR as already implemented and shipped. Keep the current bounded-concurrency route-first implementation; if maintainers want additional hardening, open a smaller follow-up for the Commander fallback default or extra CLI fallback regression tests.
