## Summary

Keep this PR open. Current main still lacks a shared native Swift copy affordance for assistant messages, and the existing WebChat copy-as-markdown button is a separate TypeScript UI surface. The PR is plausible but should not merge as-is because it copies raw primaryText rather than the same visible assistant text produced by AssistantTextParser, which can include hidden thinking tags/content when traces are off.

## Best Possible Solution

Keep this PR open and request revisions. The copy payload should be derived from the same AssistantTextParser.segments(from:includeThinking:) path used by ChatAssistantTextBody, then joined as visible text for the current trace setting. Keep the assistant-role guard and platform pasteboard handling, add a focused parser/helper test for the copied visible text, and verify the shared native chat UI on macOS and iOS.
