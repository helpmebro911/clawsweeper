## Summary

Keep this PR open. Current main still lacks a native shared Swift assistant-message copy affordance, so the PR is not obsolete, but the proposed implementation should not merge as-is because it copies raw `primaryText` instead of the same visible assistant text produced by `AssistantTextParser`. That can put hidden thinking tags/content on the clipboard when traces are disabled.

## Best Possible Solution

Keep this PR open and request revisions: derive the clipboard payload from the same `AssistantTextParser.segments(from:includeThinking:)` path used by `ChatAssistantTextBody`, preserve the assistant-role guard and platform pasteboard handling, add a focused parser/helper test for visible copied text, and verify the shared native chat UI on both macOS and iOS.
