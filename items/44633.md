## Summary

Current `main` already routes Discord auto-TTS replies through the native voice-note path, and the same behavior is present in release `v2026.4.22`. The issue request is therefore already implemented.

## Close Comment

Codex review: this is already implemented on current `main` and shipped in `v2026.4.22`. The current TTS routing includes Discord in `extensions/speech-core/src/tts.ts`, auto-TTS sets `audioAsVoice` for supported voice-note channels, and `extensions/speech-core/src/tts.test.ts` has an explicit regression test for Discord native voice messages. The shipped changelog for `v2026.4.22` also calls this out directly: Discord/TTS now routes auto voice replies through the native voice-note path so Discord receives Opus voice messages instead of regular audio attachments. On that basis, this issue looks resolved. If you still reproduce file attachments on `v2026.4.22` or newer, please open a fresh issue with exact version, Discord channel config, and any `ffmpeg`/`ffprobe` or voice-message delivery logs.
