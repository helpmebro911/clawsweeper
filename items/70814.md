## Summary

Keep open. Current main still lets a raw `delivery.channel` string flow into cron delivery resolution, and arbitrary normalized strings can survive as deliverable channels before failing later as `Unsupported channel`. PR #70814 is a plausible focused bug fix with regression coverage, but it needs maintainer/security review because the proposed recovery infers `to` from `channel` and a bot raised an allowlist-bypass concern.

## Best Possible Solution

Keep #70814 open for maintainer review. The best path is to settle the product/security boundary first: either land a narrow compatibility fix that preserves allowlist semantics and keeps the resolver regressions, or reject invalid `delivery.channel` values and move legacy/misfiled-job repair into doctor/gateway validation. The unrelated macOS formatting suppressions should be split out or explicitly accepted before merge.
