## Summary

Keep this PR open. Current main still omits Docker Desktop and OrbStack user bin directories from the macOS LaunchAgent service PATH, and the PR is a narrow core daemon fix for the sandbox `docker` lookup failure described in #41263.

## Best Possible Solution

Keep the PR open and review or land a rebased version of this narrow additive fix with service-env regression coverage. This belongs in core daemon service PATH handling because `openclaw gateway install` owns the generated LaunchAgent environment; it is not a ClawHub/plugin cleanup item.
