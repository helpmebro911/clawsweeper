## Summary

Keep this PR open. Current main still lacks the requested grouped Control UI text path, group-level empty suppression, and per-line system-event/internal-instruction metadata filtering, so the PR is not obsolete. It is a plausible focused fix, but it still needs revision against current main and review of the latest functional concerns before merge.

## Best Possible Solution

Keep this PR open and ask for a focused rebase/revision: preserve current `processMessageText` internal-runtime-context stripping, preserve `normalizeMessage` assistant directive/media cleanup, route grouped visible text through the shared stripping contract, suppress all-empty rendered groups at the group level, extend both `stripInboundMetadata` and `stripLeadingInboundMetadata` with narrow system-event/internal-instruction filtering that handles continuation lines without deleting legitimate user-authored `System:` text, and keep regression tests for Control UI grouped bubbles, TUI leading-only formatting, assistant `<think>/<final>` scaffolding, and multiline system events.
