## Summary

Do not close. Current `main` partially mitigates the report by merging/inheriting main-agent auth for subagents that have no local store, but `openclaw onboard` still writes the Anthropic setup-token only to the selected `agentDir`, and stale same-id subagent token/cooldown state can still override the main agent's fresh credential. The code and docs are also internally inconsistent about whether auth is shared or inherited across agents, so there is not strong evidence that #44571 is fully implemented or obsolete.
