## Summary

Keep #72087 open. The issue is new, concrete, and reports a narrow regression boundary: `dist/entry.js`/CLI main invocation fails while direct `runCli()` and provider calls succeed. Current `main` still has distinct entry-main side effects before `runCli()`, and I did not find source, test, docs, or post-release history evidence that this exact Linux/Codex OAuth image-generation entrypoint failure has been fixed or superseded.

## Best Possible Solution

Keep the issue open. A maintainer fix should reproduce the exact `dist/entry.js`/`openclaw.mjs` main invocation on Linux, isolate the interaction among entry respawn, argv/profile handling, env normalization, warning filtering, compile cache, and Undici/fetch network state, then add a focused regression test or smoke that exercises the entry-main path rather than only direct `runCli()` or provider calls.

## Evidence

That remains a different path from direct `runCli(argv)`, matching the reporter's narrowed failure boundary.
