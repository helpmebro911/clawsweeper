## Summary

Keep open. Current main still routes MiniMax through the generic hybrid Anthropic/OpenAI replay hooks, which produce strict Anthropic replay policy with tool-call ID sanitization enabled. The provider-local opt-out proposed by this PR is not present, current MiniMax tests still assert the old behavior, and the provided PR discussion remains concrete and relevant to MiniMax error 2013. The PR diff is narrow and does not introduce an observable security or supply-chain-sensitive change.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this provider-local wrapper, or an equivalent MiniMax-owned replay hook, so minimax and minimax-portal disable tool-call ID sanitization only for anthropic-messages while preserving strict sanitization for OpenAI-compatible paths. A shared hybrid-family sanitizeToolCallIds option can be considered later if more providers need the same knob.
