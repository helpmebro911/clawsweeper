## Summary

Current `main` still eagerly aggregates every discovered agent `sessions.json` into a combined store and still clones stores on both cache write and return, so the reported OOM pattern is not implemented away on `main`. The issue should remain open.

## Evidence

- **Combined gateway load still reads every agent store:** `loadCombinedSessionStoreForGateway` calls `resolveAllAgentSessionStoreTargetsSync(cfg)` and then loops every target, calling `loadSessionStore(storePath)` for each before merging entries into one in-memory map.

  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **sessions.list still uses the combined all-agent loader:** The `sessions.list` gateway handler loads `const { storePath, store } = loadCombinedSessionStoreForGateway(cfg);` before listing sessions, so the main listing path still materializes the full combined store.
  - file: [src/gateway/server-methods/sessions.ts:579](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/gateway/server-methods/sessions.ts#L579)

  - file: [src/gateway/sessions-resolve.ts:150](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/gateway/sessions-resolve.ts#L150)
