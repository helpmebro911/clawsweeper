## Summary

Keep this issue open. Current main still has a single resolved memory embedding provider/model/store per agent, model-filtered rows inside one SQLite index, and fallback behavior that switches provider then performs a full reindex. It does not implement first-class multi-index memory, `indexes[]`/`activeIndex`, per-index ingestion state, active/fallback index routing, or the requested observability.

## Best Possible Solution

Keep the issue open for maintainer design. The likely path is memory-core plus public memory/plugin contract work: add backward-compatible `indexes[]`/`activeIndex` config, keep a canonical chunk source, build per-index ingestion/backfill state, route reads to one selected index at a time with controlled fallback, expose per-index status/metrics, and preserve current single-index configs and full-reindex fallback semantics.
