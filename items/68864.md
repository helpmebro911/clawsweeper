## Summary

Keep open. Current main still serves Web UI checkpoint details from `SessionEntry.compactionCheckpoints` metadata only; it does not rebuild or discover checkpoint `.jsonl` files for historical sessions whose store metadata is empty or missing. The linked fix PR #68883 directly targeted this bug but was closed unmerged, so the user-visible report remains actionable.

## Best Possible Solution

Keep the issue open and implement the missing gateway behavior. The likely fix belongs around `src/gateway/session-compaction-checkpoints.ts` and `src/gateway/server-methods/sessions.ts`: when `compactionCheckpoints` is absent or empty, discover `{sessionId}.checkpoint.{checkpointId}.jsonl` files in the session directory using the existing filename parser, filter them to the target session, validate file existence, and return safe checkpoint entries. Apply the same source of truth to `sessions.list` count/latest plus `compaction.list/get/branch/restore`, and add a regression test where the store entry has no `compactionCheckpoints` but checkpoint files exist.
