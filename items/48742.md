## Summary

Keep #48742 open. Current main still has the #44460 behavior: when `cron.webhook` is unset and a stored cron job has `notify: true`, doctor only emits a warning and leaves `notify` in place. The linked PR is a plausible fix path, though it should be rebased/scoped because it also carries unrelated provider and test changes.

## Best Possible Solution

Keep the PR open, but require a rebase and a focused patch for `src/commands/doctor-cron.ts` plus regression tests covering `notify: true` with unset `cron.webhook` for undefined/none/webhook/announce or unknown delivery modes. If maintainers prefer not to merge the contributor branch because of unrelated changes, land the same targeted doctor behavior separately and then close both #48742 and #44460 as superseded/implemented.
