## Summary

Keep this PR open. Current main at 348728c28c1ea9ae7c5824350baba35ee08a275e still lacks a WhatsApp channel/account read-only config surface and still has transport-level WhatsApp outbound paths without a read-only gate. The PR remains a plausible implementation candidate, but the discussion identifies concrete follow-up fixes before merge.

## Best Possible Solution

Keep this PR open and finish the WhatsApp transport-level read-only implementation here or in a direct replacement patch. The shipped path should add channel/account readOnly config, resolve it per account, hard-gate WhatsApp outbound messages, media, polls, reactions, composing/presence, read receipts, pairing replies, and inbound reply/send closures, while still creating pairing requests for operator approval without sending WhatsApp replies.
