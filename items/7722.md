## Summary

Keep #7722 open. Current main only implements the narrower `tools.fs.workspaceOnly` boolean guard; it still rejects the requested `tools.fileAccess`, `mode`, `allowedPaths`, and `denyPaths` config surface, and exec has no filesystem allow/deny enforcement. The latest issue discussion also explicitly preserves this broader scope, with #5948 and PR #60981 treated as partial/contextual rather than complete.

## Best Possible Solution

Keep #7722 open until OpenClaw implements a versioned filesystem access policy surface, likely under `tools.fs` or a successor config contract, with deny-wins allow/deny path rules enforced across read/write/edit/apply_patch/local media and a defensible exec strategy, or until a maintainer explicitly narrows or rejects that product direction. Treat `tools.fs.workspaceOnly`, #5948, #8719, and PR #60981 as related partial/contextual work rather than completion.
