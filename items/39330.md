## Summary

Keep #39330 open. Current main still appends the internal unscheduled-reminder note into reply payloads and returns those guarded payloads through final delivery; the existing tests still assert that user-visible text includes the note in the affected no-cron scenarios. The PR remains useful but should address the still-open review concern to strip only text that OpenClaw injected, not legitimate model output that happens to end with the same sentence.

## Best Possible Solution

Keep #39330 open for maintainer review. The best fix is a narrow change in src/auto-reply/reply/agent-runner.ts and src/auto-reply/reply/agent-runner-reminder-guard.ts that prevents internally injected reminder-guard notes from reaching final delivery while preserving legitimate model output containing the same sentence. Gate sanitization to payloads modified by the reminder guard, or restructure the guard so it records the condition without appending user-visible text, then update the existing reminder-guard tests for cron.add success, existing active cron coverage, unrelated cron jobs, disabled cron jobs, missing sessionKey, and cron-store read failure.
