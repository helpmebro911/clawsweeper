## Summary

Keep this PR open. Current main still archives the planned backup sources through `tar.c(...)` with no archive-time ENOENT/ENOTDIR skip path, while this PR is a focused implementation candidate for the open backup race in #67417. It should be revised or superseded rather than closed, because review comments identify real manifest/verification and regression-test gaps.

## Best Possible Solution

Keep this PR open and request a revision, or land an equivalent/superseding fix after review. The best implementation should handle archive-time ENOENT/ENOTDIR in `src/infra/backup-create.ts`, keep manifest/result/verify semantics consistent for skipped payloads, add a regression test that simulates a session transcript disappearing during packing, and coordinate explicitly with #67417 and #72251 before either PR is closed.

## Risks / Open Questions

- Merging the submitted patch unchanged could produce a backup whose embedded manifest still declares payloads skipped during packing, causing `backup verify` or restore tooling to reject or misrepresent the archive.
- The proposed `MockPack` currently propagates ENOENT to the write stream after invoking Pack error listeners, which blocks a faithful regression test for the primary skip behavior.
- A lower-level `tar.Pack` replacement must preserve gzip, portable mode, preservePaths, path remapping, temporary archive cleanup, and atomic publish behavior from the current `tar.c` path.
- #72251 may supersede this PR if maintainers prefer a broader volatile-file filter/retry design, but it is still open and should be coordinated rather than used to close this PR now.
