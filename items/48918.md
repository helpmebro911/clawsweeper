## Summary

Keep #48918 open. Current main still does not implement a standardized user-level per-skill preferences/conventions file that is auto-loaded when a skill is triggered. The existing skill system discovers and loads `SKILL.md` skill roots, supports whole-skill precedence/overrides and per-skill env/apiKey config, and prompts the model to read the skill's `SKILL.md`; it does not merge a lower-precedence `preferences.md` or `conventions.md` alongside a system/bundled skill. The related ClawSweeper matches are adjacent skill config/routing requests, not canonical duplicates of this specific preference-loading ask.

## Evidence
