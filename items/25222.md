## Summary

Keep open. Current main has queue modes, typing keepalive/status surfaces, `/status`, and `/stop`, but it does not implement the requested automatic, immediate user-facing session-busy reply when a normal follow-up arrives during an active run. The active-run path enqueues/collects follow-ups and returns no reply to the channel.
