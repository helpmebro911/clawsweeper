## Summary

Keep #37626 open. Current main still implements Feishu wiki node listing as a single `spaceNode.list` call with only `parent_node_token`, so `feishu_wiki` still inherits Feishu's first-page behavior. The related fix PR #37641 directly targeted this issue and included tests, but it was closed unmerged and its changes are not present on current main or the latest release tag.

## Best Possible Solution

Keep the issue open and fix it in the bundled Feishu plugin. The likely change belongs in `extensions/feishu/src/wiki.ts`: request a bounded `page_size`, continue while Feishu returns `has_more` with a usable `page_token`, accumulate all node items, and add a focused `extensions/feishu/src/wiki.test.ts` regression test mocking multiple `spaceNode.list` pages. Exposing manual pagination fields is a separate product choice; automatic full pagination solves the central data-loss problem without changing the user workflow.
