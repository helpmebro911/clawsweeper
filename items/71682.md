## Summary

Keep #71682 open. Current main partially addresses the report: `agents.defaults.llm.idleTimeoutSeconds` exists, is schema-backed, documented, and writable through the CLI, and `openclaw models set` supports explicit model selection. The issue is still not safely closeable because agent-facing `gateway config.patch/apply` cannot edit the LLM idle-timeout key, timeout-profile cooldown/faster fallback remains open in #51057, and the local Ollama slowdown/model-selection symptoms have not been reproduced or ruled out on current main.

## Best Possible Solution

Keep open for maintainer triage. Document the immediate workaround (`openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>` or `0` to disable), decide whether the owner-only gateway tool should allow bounded edits to that key, and keep timeout-profile cooldown/faster fallback scoped to #51057 unless #71682 adds a distinct repro. The Ollama-specific model-selection/performance symptoms should be reproduced or narrowed before closing.
