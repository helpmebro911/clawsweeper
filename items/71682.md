## Summary

Keep open. Current main has an operator-facing LLM idle timeout knob for slow local models, but it does not fully resolve the reported user problem because agent-facing `gateway config.patch/apply` still fail closed to a narrow allowlist that excludes `agents.defaults.llm.idleTimeoutSeconds`, and timeout cooldown/faster fallback behavior remains separately open in #51057.

## Best Possible Solution

Keep open for maintainer triage. The practical path is to document and support the operator command `openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>` for slow local models, then decide whether owner-only agent-facing `gateway config.patch/apply` should permit bounded edits to this key. Keep timeout auth-profile cooldown and faster fallback policy aligned with #51057 unless this issue adds a distinct reproduction that needs separate handling.
