## Summary

Keep #71682 open. Current main partially addresses the report with a supported `agents.defaults.llm.idleTimeoutSeconds` knob, CLI write coverage, docs, timeout error guidance, and a recent adjacent live model-switch fallback fix. It is still not safely closeable because the issue also asks for the LLM/agent to be able to change the timeout, while the agent-facing `gateway config.patch/apply` allowlist still excludes `agents.defaults.llm.idleTimeoutSeconds`; the timeout-driven profile cooldown/faster fallback behavior is also still unresolved and tracked by open #51057.

## Best Possible Solution

Keep open for maintainer triage. The practical path is to document the operator command `openclaw config set agents.defaults.llm.idleTimeoutSeconds <seconds>` for slow local models, then decide whether the owner-only agent-facing `gateway config.patch/apply` allowlist should permit bounded edits to `agents.defaults.llm.idleTimeoutSeconds`. Keep timeout auth-profile cooldown and faster fallback policy aligned with the existing open #51057 thread unless #71682 adds a distinct repro that warrants a separate fix.
