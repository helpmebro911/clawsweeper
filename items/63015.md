## Summary

Keep this PR open. Current main still normalizes loose outbound reply payloads from only `mediaUrl`, so final auto-reply objects using `filePath`, `path`, `media`, or `fileUrl` still lose attachments before channel delivery. The PR remains a focused implementation candidate, but maintainers should settle alias precedence and likely add `fileUrl` parity before merge.

## Best Possible Solution

Keep this PR open and finish it as a small Plugin SDK outbound normalization fix: decide the canonical alias precedence, update `normalizeOutboundReplyPayload` to support the chosen alias set, include regression tests for `media`, `mediaUrl`, `filePath`, `path`, and preferably `fileUrl`, then run the relevant changed-lane checks before landing.
