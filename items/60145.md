## Summary

Keep open. Current `main` still has the boundary mismatch from #60145: the workspace-skill loader enforces containment inside the configured skills root, but `openclaw security audit` still treats only the broader workspace root as the escape boundary.

## Evidence

- **Audit still checks against workspace root:** `collectWorkspaceSkillSymlinkEscapeFindings` resolves `workspaceRealPath` from the workspace dir and accepts a skill when `isPathInside(workspaceRealPath, skillRealPath)` is true. The emitted finding/title/remediation also describe the boundary as the workspace root.
  - file: [src/security/audit-extra.async.ts:487](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/security/audit-extra.async.ts#L487)

  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)

In workspace discovery, `baseDirRealPath` becomes the resolved skill root and child `skillDir` / `SKILL.md` entries are then checked against that narrower root, not the broader workspace root.
