## Summary

Keep open. Current `main` still allows Mattermost DM replies to pick up `payload.replyToId` and send it as `root_id`, which contradicts the published DM threading behavior and means the PR's fix is not already implemented on `main`. The provided PR discussion is supportive, not obsolete: the only review comment summarizes the same missing guard and reports a focused regression test addition, with no contrary evidence.
