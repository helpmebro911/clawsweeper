## Summary

Keep this issue open. Current main still has the non-string stream-content risk described in the report, only parses fallback reasoning fields when they are strings, and `mistral/mistral-small-latest` reasoning is still a bundled documented path. An open PR, #67203, also references this issue with closing syntax, so the issue should remain open until that candidate is merged, rejected, or superseded by a landed fix.

## Best Possible Solution

Keep this issue open and have maintainers review #67203 or land an equivalent focused fix in `src/agents/openai-transport-stream.ts`: guard streamed text content as string-only, normalize Mistral `reasoning_content` array/object block shapes into thinking/text deltas, and add regression coverage for non-string Mistral stream chunks.
