## Summary

Keep open. Current `main` still drops Telegram group-history media when the bot is not mentioned: skipped history entries store only text metadata, the shared `HistoryEntry` type has no media fields, and later prompt assembly only exposes current-turn `MediaPaths`, not media from pending group history.

## Evidence

- **Skipped Telegram group messages are recorded as text-only history entries:** In the no-mention group path, `recordPendingHistoryEntryIfEnabled()` is called with an entry containing only `sender`, `body`, `timestamp`, and `messageId`; no media paths or media types are preserved.
  - file: [extensions/telegram/src/bot-message-context.body.ts:294](https://github.com/openclaw/openclaw/blob/171322644dece55402569c52b26370fd81769ca6/extensions/telegram/src/bot-message-context.body.ts#L294)
  - command: `nl -ba extensions/telegram/src/bot-message-context.body.ts | sed -n '266,307p'`
  - sha: [171322644dec](https://github.com/openclaw/openclaw/commit/171322644dece55402569c52b26370fd81769ca6)
- **Shared history contract cannot carry media:** `HistoryEntry` on current main only defines `sender`, `body`, `timestamp`, and `messageId`.
  - file: [src/auto-reply/reply/history.ts:30](https://github.com/openclaw/openclaw/blob/171322644dece55402569c52b26370fd81769ca6/src/auto-reply/reply/history.ts#L30)
  - command: `nl -ba src/auto-reply/reply/history.ts | sed -n '30,35p'`
  - sha: [171322644dec](https://github.com/openclaw/openclaw/commit/171322644dece55402569c52b26370fd81769ca6)
- **Later Telegram context assembly still strips history down to text and only attaches current-turn media:** `InboundHistory` is populated from pending history with only `sender/body/timestamp`, while `ctxPayload.MediaPath(s)` comes from `currentMediaForContext` plus `replyMedia`; pending group-history media is never merged back in.
  - file: [extensions/telegram/src/bot-message-context.session.ts:312](https://github.com/openclaw/openclaw/blob/171322644dece55402569c52b26370fd81769ca6/extensions/telegram/src/bot-message-context.session.ts#L312)
  - command: `nl -ba extensions/telegram/src/bot-message-context.session.ts | sed -n '312,374p'`
  - sha: [171322644dec](https://github.com/openclaw/openclaw/commit/171322644dece55402569c52b26370fd81769ca6)
- **Prompt media notes depend on `ctx.MediaPaths`, not text history:** `buildReplyPromptBodies()` adds media notes from `buildInboundMediaNote(ctx)`, and `buildInboundMediaNote()` reads only `MediaPath/MediaPaths/MediaUrl/MediaUrls/MediaTypes`. `InboundHistory` entries are separately typed as text-only `{ sender, body, timestamp }`.
  - file: [src/auto-reply/media-note.ts:103](https://github.com/openclaw/openclaw/blob/171322644dece55402569c52b26370fd81769ca6/src/auto-reply/media-note.ts#L103)
