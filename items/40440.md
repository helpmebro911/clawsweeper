## Summary

Keep open. Current main still records skipped mention-gated Telegram group messages as text-only history entries, while later Telegram context construction builds media payloads only from the current message and reply media. The related fix attempts #40470 and #40490 were closed unmerged, and current source has no equivalent history-media persistence or replay path.

## Best Possible Solution

Keep this issue open and implement a focused Telegram plugin fix: extend the shared reply-history and `MsgContext.InboundHistory` contracts with media references, preserve or refresh downloaded media for skipped group messages beyond the current inbound media TTL, merge history media into the later `MediaPaths`/`MediaTypes` payload when the bot is mentioned, update prompt metadata as needed, and add a regression test for image-without-mention followed by an @mention asking about that image.
