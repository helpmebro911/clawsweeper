## Summary

Current main still appears to have the reported Telegram bug: skipped mention-gated group messages can carry downloaded media in `allMedia`, but the pending history entry only records text metadata, and later Telegram context construction only forwards current-turn and reply media. The related fix PRs #40470 and #40490 are closed unmerged, and there is no current source, test, or changelog evidence that the history-media replay path has shipped.

## Best Possible Solution

Keep this issue open and implement the focused Telegram/shared-history fix: extend the reply-history and `MsgContext.InboundHistory` contracts with optional media references, persist or refresh media for skipped mention-gated group messages, merge replayed history media into later Telegram `MediaPaths`/`MediaTypes`, update prompt metadata if useful, and add a regression test for image-without-mention followed by an @mention asking about that image.

## Risks / Open Questions

- No live Telegram group reproduction was run in this read-only review, but the current source path directly matches the reported failure mode.
- The eventual fix needs an explicit media lifetime policy so history entries do not point at expired inbound files or retain group media longer than intended.
- Extending history media support affects plugin-SDK-visible reply history and `MsgContext.InboundHistory`, so the change should be backwards-compatible and covered by focused contract and Telegram regression tests.
