Keep this PR open. Current main still has no opt-in Linuxbrew support in the primary Docker image/setup path, and the linked Docker skill-install bug remains unresolved. The PR is a relevant implementation attempt, but it needs maintainer review and rework before it could be merged safely.

## Best Possible Solution

Keep this PR open for maintainer judgment. The best path is to resolve #14593 through one canonical Docker-safe approach: rework this PR against current main with opt-in Linuxbrew support, docs, focused setup tests, preserved runtime-image behavior, conditional environment handling, and an accepted installer supply-chain policy; or intentionally choose and land a replacement such as #55940 or #57553. Close this PR only after a replacement ships or maintainers explicitly reject Docker-bundled Linuxbrew.
