## Summary

Keep open. Current `main` still lacks Docker Linuxbrew support, so the underlying bug/request is not implemented. The PR discussion also surfaced concrete correctness problems in the proposed patch, which means this is not a safe conservative cleanup close.

## Evidence

- **Current Dockerfile has no brew install path:** `main` only exposes optional Docker build args for extra apt packages, browser install, and Docker CLI install. There is no `OPENCLAW_INSTALL_BREW` arg or Linuxbrew/Homebrew install block in the runtime image.
  - file: [Dockerfile:199](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/Dockerfile#L199)

  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Current Docker setup script does not wire brew through build or .env:** `scripts/docker/setup.sh` exports and persists `OPENCLAW_DOCKER_APT_PACKAGES`, `OPENCLAW_EXTENSIONS`, and `OPENCLAW_INSTALL_DOCKER_CLI`, then passes only those build args to `docker build`. There is no `OPENCLAW_INSTALL_BREW` handling on `main`.
  - file: [scripts/docker/setup.sh:279](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/scripts/docker/setup.sh#L279)
