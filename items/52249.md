## Summary

Keep this issue open. Current main has removed the obsolete direct-resume ACP relay theory, but ACP parent completion still depends on session-scoped system events plus heartbeat wake delivery. The heartbeat-disabled path and newer native-spawn comment remain plausible against current code, and same-author PR #60402 is still an open implementation candidate rather than a reason to close the issue early.

## Best Possible Solution

Keep this issue open. Review, rebase, and either land or close PR #60402, or replace it with a shared yielded-parent completion wake primitive. The accepted fix should add regression coverage proving ACP completion wakes a yielded parent when heartbeat is disabled or absent, and should evaluate the newer native-spawn/mirror report so the solution is not narrowly ACP-only if the bug is shared.
