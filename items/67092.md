## Summary

Current main still preserves text before an orphan closing reasoning tag and then appends the final-answer tail, so the malformed `reasoning </think> final` shape can still leak through shared visible-text/history sanitization and the stateful streaming path. The linked fix attempts were closed unmerged, so they do not implement or supersede this issue.

## Best Possible Solution

Keep this issue open. Land a focused core text-sanitization fix that treats an orphan closing reasoning tag outside an active reasoning block as a privacy boundary when answer text follows it, while preserving literal/code examples. Apply the rule consistently in the shared sanitizer and the stateful streaming sanitizer, then add regression tests for `reasoning </think> final` in shared text, assistant visible/history extraction, embedded streaming/block replies, and UI extraction.

## Risks / Open Questions

- A fix that blindly drops everything before any literal closing reasoning tag could erase legitimate documentation or code examples; keep the existing fenced/inline code protections and add regression cases.
- There are at least two relevant stripping paths: the shared sanitizer and `stripBlockTags` in the streaming/block-reply path. Updating only one would leave inconsistent delivery, preview, or history behavior.
