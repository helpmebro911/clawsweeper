## Summary

Keep this PR open. Current main at 348728c28c1ea9ae7c5824350baba35ee08a275e still does not implement an OpenAI/Azure OpenAI provider-level concurrency limiter or an LLM_MAX_CONCURRENT configuration surface. The PR is a plausible implementation candidate, but the review comments identify real merge blockers: chat completions are not wrapped, formatting is broken in the PR diff, queued aborts are not handled, invalid env values can disable throttling, and wrapping only stream creation would release the limiter slot before streamed responses finish.

## Best Possible Solution

Keep this PR open for author or maintainer revision. A mergeable solution should add a tested provider-request limiter covering OpenAI Responses, Azure OpenAI Responses, and OpenAI chat completions; hold the slot until the stream is fully consumed or cancelled; reject queued work promptly on AbortSignal; safely parse invalid/empty concurrency configuration; and document the chosen config surface. If maintainers implement that separately, this PR can then be closed as superseded.
