## Summary

Keep open. Current main still lacks the Docker image change needed for fresh named volumes: the runtime stage switches to the non-root `node` user without first creating `/home/node/.openclaw` with `node:node` ownership, while the Compose/docs surface mounts that directory and config writes still create `openclaw.json.<pid>.<uuid>.tmp` there. The provided timeline also shows open implementation candidates #63959 and #72662, so the issue should remain open until one lands or is replaced.

## Best Possible Solution

Review and land #72662 or another narrow Dockerfile fix that creates `/home/node/.openclaw` before `USER node` with `node:node` ownership and private permissions, preserving attribution for #63959/#48072 as appropriate. Add Docker named-volume smoke proof or equivalent maintainer validation, and document recovery for already-created root-owned volumes. After the fix merges, this issue can close as implemented.
