## Summary

Keep PR #48604 open. Current main has partially superseded the schema/validation part by accepting `agents.list[].contextTokens`, but the central runtime behavior requested by the PR is still not implemented consistently: `resolveContextWindowInfo` has no `agentId` input, the embedded runner setup still resolves only model/default/global caps, and compaction plus extension-factory paths still cannot honor a per-agent cap. The existing PR review comments correctly identify gaps that remain relevant.

## Best Possible Solution

Keep the PR or replace it with a focused follow-up that implements `agents.list[].contextTokens` as an authoritative per-agent runtime override. The implementation should add an `agentId`-aware helper or extend `resolveContextWindowInfo`, apply it in the main embedded runner, `/compact`, queued/context-engine compaction, context-pruning, compaction-safeguard setup, memory/status paths as needed, and add regression tests for lower-than-default, higher-than-default, no-agent fallback, and compaction/extension behavior.
