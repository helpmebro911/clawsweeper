## Summary

Keep #27482 open. Current main still supports image-only direct chat attachments in the Control UI and Gateway path, while video support exists as separate media-understanding/video-generation surfaces rather than the requested direct message upload to an LLM.

## Best Possible Solution

Keep #27482 open until OpenClaw adds first-class direct video chat attachments in the Control UI and Gateway path: accept and preview MP4/WebM/MOV, add a video-capable attachment/content model, route to provider-native video input or the existing media-understanding runtime with capability and size checks, and cover UI, Gateway, provider dispatch, transcript, and docs behavior with focused tests.
