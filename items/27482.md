## Summary

Keep #27482 open. Current main has video media-understanding plumbing and a CLI `video describe` path, but the requested direct video upload through the chat/message upload surface is still not implemented: the Control UI and gateway attachment path remain image-oriented and drop or coerce non-image attachments.

## Evidence

- **Control UI attachment accept/type guard is image-only:** The chat attachment support constant is `image/*`, and `isSupportedChatAttachmentMimeType` only accepts MIME types starting with `image/`.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/ui/src/ui/chat/attachment-support.ts#L1)

  - file: [ui/src/ui/views/chat.ts:208](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/ui/src/ui/views/chat.ts#L208)
