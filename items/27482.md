## Summary

Keep #27482 open. Current main still has image-only direct chat attachments in the Control UI and Gateway `chat.send` path. Video understanding exists as separate media-understanding/CLI/plugin runtime surfaces, but the requested direct MP4/WebM/MOV upload-to-LLM chat workflow is not implemented.

## Best Possible Solution

Keep #27482 open until OpenClaw adds first-class direct video chat attachments across the core UI and Gateway path: accept and preview MP4/WebM/MOV, add a video-capable attachment/content model, route provider-native video input or media-understanding summaries by model/provider capability and size policy, persist transcript metadata, and cover UI, Gateway parsing, provider dispatch, and docs with focused tests.
