## Summary

Keep #9544 open. Current main still directly uses Baileys `useMultiFileAuthState` for WhatsApp auth and only adds `creds.json` atomic-write/backup hardening; it has not replaced the multi-file credential and signal-key store with a SQL or consolidated production auth store. The related PR #66863 proposed that broader replacement but was closed unmerged, and the prior maintainer/Codex review comment also correctly kept this open.

## Best Possible Solution

Keep #9544 open until the bundled WhatsApp plugin replaces `useMultiFileAuthState` with a production auth store, migrates existing Baileys multi-file credentials and signal keys safely, updates docs/config/auth presence/logout behavior, and adds focused fresh-install, migration, corruption, and key-persistence tests.
