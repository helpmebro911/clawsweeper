## Summary

Keep #9544 open. Current main still uses Baileys `useMultiFileAuthState(authDir)` for WhatsApp auth and only hardens `creds.json` writes; it has not replaced the multi-file signal-key store with a SQL/consolidated production auth store. The related PR #66863 proposed that broader replacement but was closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #9544 open until the bundled WhatsApp plugin replaces `useMultiFileAuthState` with a production auth store, safely migrates existing Baileys multi-file credentials and signal keys, updates auth docs/config/status/logout behavior, and adds focused fresh-install, migration, corruption, and key-persistence tests. The closed PR #66863 is useful prior art but is not merged and should not supersede the remaining work.
