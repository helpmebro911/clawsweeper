## Summary

Keep PR #38650 open. Current main still treats a workspace bootstrap file symlink that resolves outside the workspace as a validation failure and then injects it as missing, so the linked user bug is not implemented on main. PR #40230 is a related open candidate, but it is not merged or clearly maintainer-designated as the canonical replacement.

## Best Possible Solution

Keep this PR open until a maintainer either lands a current-main fix or explicitly closes it in favor of a canonical successor such as #40230. The best implementation path is a targeted workspace bootstrap reader policy in readWorkspaceFileWithGuards/openBoundaryFile, with tests for shared outside-workspace bootstrap symlinks, in-workspace symlink aliases, symlink loops, hardlink rejection, max-size enforcement, and Gateway agents.files behavior.
