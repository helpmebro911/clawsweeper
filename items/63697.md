## Summary

Keep this PR open. Current main still intercepts the native chat reset/compact slash commands before they can reach `chat.send`, while the PR is a plausible implementation candidate but remains broad, unmerged, and has unresolved functional and security-sensitive review concerns.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to re-scope or split it into a focused native Chat UI change that forwards slash commands through `chat.send`, preserves reset/compact transcript semantics with Swift regression tests, and separately reviews the broader UI/model-picker/exec-approval work. Before merge, preserve the current `operator.approvals` compatibility gate and coordinate any APNs-related client expectations with #67493.
