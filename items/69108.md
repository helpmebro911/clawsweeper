## Summary

Keep #69108 open. Current main already implements the `--agent` + `--session-id` precedence fix, but it does not implement the PR's `--new-session` CLI flag or docs. The remaining feature is still tracked by open issue #54864, and this draft PR is a focused, recent implementation rather than an obsolete or duplicate-only change.

## Best Possible Solution

Keep the PR open for maintainer review or ask the author to rebase/narrow it to the remaining `--new-session` feature, since main already has the session-id isolation fix. A good final shape would add the Commander flag, mutual exclusion with `--session-id`, generated UUID plumbing shared by gateway and local fallback paths, JSON/stdout surfacing of the generated id, docs, changelog, and focused CLI/session tests.
