## Summary

Keep PR #66546 open. Current main does not implement `transcriptRotateBytes`, `transcriptMaxLines`, or an equivalent per-transcript `.jsonl` rotation/cap. Existing maintenance rotates only `sessions.json`, prunes/caps session entries, cleans reset/deleted archives, and enforces an aggregate sessions-directory budget by deleting artifacts or whole sessions. The linked bug #66360 remains a real core storage-maintenance gap, and this PR is a plausible fix that needs maintainer review rather than cleanup closure.

## Best Possible Solution

Keep PR #66546 open for maintainer review, or replace it with a narrower core patch that adds explicit transcript `.jsonl` rotation/capping under `session.maintenance`, wires it only into safe hot-path and explicit cleanup paths, updates schema/docs, and keeps focused regression coverage for header preservation, permissions, rollback, concurrent recreation, and bulk cleanup behavior.
