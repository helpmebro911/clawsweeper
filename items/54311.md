## Summary

Keep this issue open. Current main still has the reported bundled `gh-issues` frontmatter key, the skill remains a single oversized `SKILL.md`, and the skill-creator validator/packager still reject frontmatter keys outside their narrow allowlist. The closed related PR #54360 was not merged, so it did not resolve the issue on main.

## Best Possible Solution

Keep this issue open and fix it with a focused bundled-skills cleanup: align the public skill frontmatter contract across runtime, docs, skill-creator guidance, validator tests, and bundled `gh-issues`. Since runtime defaults `user-invocable` to true, the narrow immediate fix is likely to remove the redundant field from `skills/gh-issues/SKILL.md`; then split the large orchestration material into `references/` files and tighten the top-level description to be trigger-oriented.
