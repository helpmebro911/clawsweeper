## Summary

Keep #40252 open. Current main still implements the reported iMessage behavior: media-only sends synthesize `<media:image>` / `<media:${kind}>`, then pass that value as the JSON-RPC `text` field with the file attachment. The related fix PRs #40341 and #46343 were closed unmerged per the provided GitHub context, and current tests still encode placeholder-based media-only delivery/cache behavior.

## Best Possible Solution

Keep the issue open and fix the iMessage plugin send path in `extensions/imessage/src/send.ts` so media-only sends pass an empty `text` field plus `file`, while preserving text+media captions, reply IDs, the empty-payload guard, markdown/directive cleanup for real text, and echo-cache behavior. Add focused regression coverage around `sendMessageIMessage()` and delivery of media-only replies asserting `params.text` is empty for media-only image/audio/video sends.
