## Summary

Keep open. Current main still renders `displayName` directly for session selector labels when no explicit `label` is present, so a display name like `webchat:g-agent-control-tower-main` can still reach the Control UI selector. The PR is relevant, but the recorded review comment identifies a real correctness issue in the proposed patch: it filters user-authored `label` values as well as transport-generated `displayName` values.

## Best Possible Solution

Keep the PR open and revise it against the current `ui/src/ui/chat/session-controls.ts` implementation: filter synthetic transport names from `displayName` in the selector/display-name path, preserve explicit `label` as authoritative, and add focused tests for displayName-only main/subagent rows plus an explicit-label preservation case.
