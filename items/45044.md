## Summary

Do not close this PR. Current main still has the broad `!hasAssistant` reset path in `prepareSessionManagerForRun`, so a persisted session containing a user message but no assistant message can still be truncated when this initializer runs. Main has adjacent restart-drain work, but it does not implement the PR's session-file guard or regression coverage.

## Best Possible Solution

Keep the PR or a maintainer-owned successor open. The best fix is to narrow `prepareSessionManagerForRun` so only truly header-only pre-created files are reset, preserve any file that already contains a message, add focused regression coverage for `[header]`, `[header,user]`, and `[header,user,assistant]`, and make the tests portable across CI platforms. Consider whether defensive archival before truncating remains desirable, but the core product fix belongs in `src/agents/pi-embedded-runner/session-manager-init.ts`.
