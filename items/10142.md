## Summary

Keep #10142 open. Current main still lacks an internal `session:end` hook event; the adjacent `session_end` surface is typed plugin-only, and the docs still distinguish plugin hooks from operator-installed internal `HOOK.md` hooks. The related direct implementation PRs are closed/unmerged or still open and not landed.

## Best Possible Solution

Keep #10142 open until maintainers either implement the internal `session:end` hook contract or explicitly document typed plugin `session_end` as the supported replacement for this workflow-integration use case. A complete internal-hook implementation should add a typed event/guard, emit it from the relevant gateway, cron, reset, and subagent completion paths, and update tests plus https://docs.openclaw.ai/automation/hooks.
