## Summary

Keep #10142 open. Current main still does not implement or document the requested internal `session:end` hook event. The repo has adjacent typed plugin `session_end` lifecycle hooks, but docs distinguish those from operator-installed internal `HOOK.md` hooks, and the direct/internal implementation attempts referenced in the discussion are not merged.

## Best Possible Solution

Keep #10142 open until maintainers either implement the internal `session:end` hook contract or explicitly document typed plugin `session_end` as the supported replacement for this workflow-integration use case. A complete internal-hook implementation would add a typed `SessionEndHookEvent`/guard, emit it from the relevant gateway, cron, reset, and subagent completion paths, add regression tests, and update https://docs.openclaw.ai/automation/hooks.
