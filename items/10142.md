## Summary

Keep open. Current main still does not implement or document an internal `session:end` hook event; it only has adjacent plugin lifecycle hooks such as `session_end`, `agent_end`, and `subagent_ended`. The issue asks for the internal `HOOK.md`/`triggerInternalHook` surface, and the latest maintainer review comment also explicitly kept it open.

## Best Possible Solution

Keep #10142 open until maintainers either implement the internal `session:end` hook contract or make an explicit product decision that plugin `session_end`/`agent_end` is the supported replacement. A complete fix should define the event context/type guard, emit `createInternalHookEvent("session", "end", ...)` from the relevant completion paths, add regression coverage, and update `docs/automation/hooks.md`; if plugin-only is intended, document that migration path in the public hooks docs.
