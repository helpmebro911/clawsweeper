## Summary

Keep open. Current main still allows Google Gemini API-key setup to select the Gemini default model, and heartbeat runs still inherit the default model unless a heartbeat-specific model is configured. The existing empty/comment-only HEARTBEAT.md skip is only a partial mitigation; actionable HEARTBEAT.md content still runs, and main-session heartbeat remains the default when isolatedSession is not enabled.

## Best Possible Solution

Keep this issue open and fix the boundary explicitly. The best maintainer path is to preserve the existing primary model during Google/provider setup unless the user explicitly confirms switching defaults, or require explicit confirmation/disablement before heartbeat can inherit a newly configured paid external provider. Add regression coverage for an existing actionable HEARTBEAT.md plus Google API-key setup so paid background usage cannot begin silently.

- The current heartbeat model override is a manual mitigation, not a setup-time confirmation or default-preservation boundary for paid provider auth.
