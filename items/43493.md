## Summary

Keep this PR open. Current main still lacks an interactive custom-provider metadata step during `openclaw configure --section model`; the custom-provider flow configures endpoint/auth/model identity and writes default metadata, while docs/schema only support manual configuration. The related custom-provider context-window UX issue #48338 is still open, so this PR remains a plausible implementation candidate rather than cleanup fodder.

## Best Possible Solution

Keep this PR open as a plausible implementation path for the still-open custom-provider metadata UX work. The best path is to rebase it onto current main, align the prompt with provider-level metadata and Azure defaults, confirm it handles the current `ModelDefinitionConfig` surface, and then either land it with targeted configure/model-metadata tests or supersede it with an equivalent maintainer-owned implementation for #48338.
