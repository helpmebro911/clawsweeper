## Summary

Keep PR #71857 open. Current main still rejects owner-prefixed ClawHub skill slugs before registry lookup, while the CLI and gateway pass the raw user-provided slug into that shared skills path. The PR addresses the concrete #71767 copy-paste failure and is not superseded by the related ASCII hardening or archive filename fixes.

## Best Possible Solution

Keep this PR open for maintainer review. The right fix is a narrow skills parser change that accepts optional <owner>/<slug> only for requested ClawHub skill install/update, validates both segments with the existing ASCII slug guard, strips the owner before registry/install-dir/lockfile use, preserves tracked legacy behavior, and keeps focused regression tests plus the changelog entry.
