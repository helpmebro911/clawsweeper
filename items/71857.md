## Summary

Keep this PR open. Current main still rejects owner-prefixed ClawHub skill slugs before any registry lookup, while the live ClawHub page publishes that exact install command and the ClawHub API resolves the bare slug. The PR is a focused implementation candidate for the linked bug, not something already implemented, duplicated, or superseded on main.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow shared ClawHub skill slug parser in src/agents/skills-clawhub.ts that accepts optional <owner>/<slug> only for requested install/update inputs, validates both segments under the existing ASCII and hyphen-edge rules, canonicalizes to the bare slug before registry/download/install-dir/lockfile use, preserves legacy tracked-slug update compatibility, and ships with focused regression tests plus the changelog entry. If the branch is currently unmergeable, ask the contributor to rebase and rerun the changed gate rather than closing the work.
