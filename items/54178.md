## Summary

Keep open. Current main still renders the Control UI Agents selector as a native `<select>` driven only by `change`, latest release v2026.4.24 has the same selector implementation, and I found no current source, test, docs, history, or canonical duplicate evidence that the reported browser-automation click behavior is fixed or intentionally unsupported.

## Best Possible Solution

Keep this issue open until current main is verified with the supported browser automation path. If pointer-based automation still cannot operate the Agents selector, replace the native select with an accessible automation-friendly combobox/listbox or visible agent list and add focused browser/UI coverage. If maintainers decide programmatic native-select popup opening is not supportable, document the supported automation path explicitly, preferably while coordinating with the broader #57067 Agents-page dropdown UX tracker.
