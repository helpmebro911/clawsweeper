## Summary

Keep open. Current main still has the WebChat attachment path implemented as image-only in the Control UI and gateway parser, while PR #70947 is a focused, recent attempt to fix the still-open bug tracked by #69447 and supersede the older #57707 approach. The PR also has substantive unresolved review findings, so the best maintainer action is review/iterate rather than close as cleanup.

## Best Possible Solution

Keep #70947 open, address the P1/P2 review comments, and continue it as the current fix path for #69447. The likely mergeable solution should broaden Control UI attachment acceptance/rendering, serialize generic files with filenames, persist non-image webchat attachments through the gateway/media path with pre-decode validation, and preserve existing image sniff/offload behavior with regression tests.
