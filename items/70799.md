## Summary

Keep open. Current main still lacks the PR's outer guards on BlueBubbles typing/read-receipt helpers, and there is no canonical duplicate or shipped implementation in the provided context. The PR is a focused, recent bug-fix candidate with a concrete reproduction, though maintainer review should validate the exact root cause because the current monitor path appears to pass the same account id to the inner guard.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to add a regression test for a Private-API-disabled BlueBubbles server across the cold-cache/account-id path, then either land a verified guard that prevents optional typing/read-receipt calls from surfacing failures or replace it with a root-cause fix that ensures the Private API status is known before those optional endpoints run.
