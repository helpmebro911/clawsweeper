## Summary

Current main has not implemented this PR's defensive outer guards. The BlueBubbles read/typing helpers still only check the cached Private API status inside the shared request path after client/account resolution, while the inbound webhook flow still calls those helpers and logs read/typing failures. The PR is a narrow, plausible bug fix with concrete reproduction details and a supportive bot review, so it should not be closed as implemented, duplicate, stale, or out of scope.

## Best Possible Solution

Keep the PR open. Maintainers should either land the narrow outer guards with a regression test that covers the opts.accountId/client.accountId cache mismatch or replace it with a deeper fix that prevents Private-API-only read/typing calls once the server has reported `private_api: false`.
