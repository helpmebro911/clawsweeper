## Summary

Keep PR #48406 open. Current main does not contain the saturated-session recovery guide, the `agents.defaults.compaction.guard` config surface, or the loop-aware guard/runtime/validator/recommend-reset implementation that the proposed docs describe. The linked `neural-bridge` skill comment is related to session management, but it is not an equivalent core operator guide or a replacement for the missing OpenClaw compaction guard docs.

## Best Possible Solution

Keep the PR or replace it with a docs-only follow-up after the loop-aware compaction guard stack is either landed or explicitly redesigned. The docs should describe only shipped behavior, link from the compaction/session docs and agent config docs, and keep `recommend-reset` framed as diagnostic unless a separate reviewed runtime path changes that contract.
