## Summary

Keep open. Current main still lacks the stale `pluginDebugEntries` turn-start cleanup, the `updateSessionStoreEntry()` preserve-activity option proposed by this PR, and the verbose-mode suppression path that prevents a failed clear from rehydrating stale debug banners. The PR remains a focused, plausible bug fix with targeted regression coverage and an addressed bot-review note, not an auto-close candidate.

## Best Possible Solution

Keep this PR open and review it as a focused runtime/session-store bug fix. The best path is to land or revise the proposed generic turn-start cleanup: clear `pluginDebugEntries` at reply-turn start with preserve-activity semantics, make cleanup best-effort, suppress disk rehydration for the rest of the turn when clearing fails, and keep Active Memory's writer behavior extension-owned. The proposed tests in `src/config/sessions.test.ts` and `src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts` are the right guardrails.
