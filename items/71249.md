## Summary

Keep PR #71249 open. Current main still has the stale `pluginDebugEntries` behavior the PR targets: verbose reply assembly reloads the persisted session entry and appends plugin debug lines, while there is no turn-start clear helper, no suppression path after a clear failure, and `updateSessionStoreEntry()` still uses the activity-touching merge path. The PR discussion is also substantive: the bot review found the targeted fix sound, and the one P2 mutation note was addressed in `fb267ad48e22299ae851a6629341e8b77f11d53c`.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land or recreate the focused fix on main: clear turn-scoped `pluginDebugEntries` at reply-turn start, preserve `updatedAt` for metadata-only cleanup, make cleanup best-effort, suppress stale debug rehydration for that turn after a persisted clear failure, and keep the proposed regression coverage in `src/config/sessions.test.ts` and `src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts`.
