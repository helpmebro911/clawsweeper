## Summary

Keep this PR open. Current main still lacks the proposed generic turn-start cleanup for stale plugin debug banners, the preserve-activity option on `updateSessionStoreEntry()`, and the verbose-refresh suppression path after a cleanup failure. The PR remains a focused runtime/session-store bug-fix candidate rather than obsolete, duplicate, or out of scope.

## Best Possible Solution

Keep this PR open and review it as a focused runtime/session-store bug fix. The best path is to land or revise the generic turn-start cleanup in `src/auto-reply/reply/agent-runner.ts`, add opt-in preserve-activity support to `updateSessionStoreEntry()` in `src/config/sessions/store.ts`, suppress stale disk rehydration after a cleanup failure for that turn, and keep Active Memory's writer behavior plugin-owned. The proposed regression coverage in `src/config/sessions.test.ts` and `src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts` is the right guardrail set.
