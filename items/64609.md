## Summary

Current main still matches the reported Telegram multi-account bug: channel-level group/topic prompt config is dropped from the merged account snapshot in multi-account mode, while the mention-policy path still falls back to channel-level groups. The linked fix PR #64650 is closed and unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open. The best fix is a Telegram-owned resolver change, likely in `extensions/telegram/src/bot-core.ts`, that uses account-level `groups` first and falls back to `channels.telegram.groups` only for prompt/skill/topic context resolution. Keep account routing inheritance unchanged, and add a regression test in `extensions/telegram/src/bot.create-telegram-bot.test.ts` for two configured accounts with channel-level group/topic `systemPrompt` and `skills`. If maintainers reject fallback semantics, add a doctor warning and public docs clarification instead of leaving the behavior silent.
