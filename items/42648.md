## Summary

Current main has memory search, append-only compaction flushes, thresholded short-term promotion into MEMORY.md, and the memory-wiki companion, but it still does not implement the requested resolver-backed durable write pipeline with type classification, required source attachment, near-duplicate/conflict lookup, create/merge/update/conflict outcomes, and revision history.

## Best Possible Solution

Keep this issue open. The best path is to implement a resolver-backed durable write pipeline in memory-core on top of the canonical schema/provenance/review work: typed candidates with required source refs, entity normalization, duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and non-destructive revision/provenance history. Coordinate with #42646, #42647, and #42650, and treat #49108 as useful classifier/entity prior art rather than a replacement for the core write contract.
