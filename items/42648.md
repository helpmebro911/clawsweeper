## Summary

Keep open. Current main has adjacent memory work, but the requested durable write resolver is still missing: writes are still append-oriented or promotion-oriented, not a typed classify/dedupe/merge/conflict pipeline with provenance and revision history.

## Best Possible Solution

Keep this open and implement a resolver-backed durable write pipeline in memory-core/active memory after or alongside the canonical schema/provenance/review work. The concrete path is a typed candidate model that requires source refs, normalizes entities, searches current durable memory for near duplicates and contradictions, routes outcomes to create, merge/update, or conflict-for-review, and records non-destructive revision/provenance history.
