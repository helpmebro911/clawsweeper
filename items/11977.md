## Summary

Keep open. Current main still publishes Docker release images only for linux/amd64 and linux/arm64, and the manifest job only combines those two digests. There is no riscv64 release job, no riscv64 manifest path, and Docker docs do not document a prebuilt riscv64 image. Related PR #12033 is useful prior art, but it was closed unmerged and does not supersede the issue; the existing maintainer/Codex issue comment also explicitly kept #11977 open.

## Best Possible Solution

Keep #11977 open until a maintainer lands a Docker release workflow change that adds a non-blocking linux/riscv64 build, publishes the riscv64 digest into the multi-arch manifest when available, handles Dockerfile prerequisites such as Bun/native addon behavior on riscv64, and updates Docker docs or release validation for the supported image behavior. PR #12033 can be used as prior art, but it closed unmerged and needs rebasing/review or a fresh implementation.
