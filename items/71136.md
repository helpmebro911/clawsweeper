## Summary

Keep #71136 open. Current main has a run-time context-window guard that partially covers the high-level safety goal for `agents.defaults.contextTokens`, but the requested resolver-wide minimum is not implemented for all advertised inputs. Explicit agent/session context token overrides still short-circuit model lookup in key session/status paths, `agents.list[].contextTokens` is accepted but not part of the run-time minimum calculation, and `maxTokens` remains transport/priority-specific rather than a shared minimum across request/config/model limits.

## Best Possible Solution

Keep the issue open and implement a single token-limit resolver that computes the conservative minimum across explicit agent/session caps, configured provider model metadata, discovered model metadata, and defaults. Wire it into embedded run budgeting, session persistence/status, and per-agent `agents.list[].contextTokens`; add focused tests for the 500000 vs 131072 example and for `maxTokens` clamping across OpenAI/Anthropic-style transports or shared stream option preparation.
