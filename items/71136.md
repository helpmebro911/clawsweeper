## Summary

Keep this issue open. Current main still has multiple strict-priority token-resolution paths and no shared minimum resolver for contextTokens and maxTokens. The prior ClawSweeper comment correctly identified partial context-window safety work, but current main still leaves the requested cross-source minimum rule unfinished.

## Best Possible Solution

Implement a shared token-limit resolver that computes the minimum of valid agent default/per-agent config, configured provider model metadata, discovered runtime model metadata/cache values, and safe defaults. Wire it through agent run setup, auto-reply, session/status defaults, session persistence, and transport stream-parameter construction. Add focused regression tests for high agents.defaults.contextTokens, high agents.list[].contextTokens, configured model contextWindow/contextTokens below the agent cap, discovery-cache limits below config, and requested maxTokens above the selected model limit.
