## Summary

Keep open. Current main still has the Discord inbound worker serializing same-session messages before they reach the reply pipeline, while the documented `steer` mode requires mid-run injection. The related #60604 context is Telegram-specific and open; it does not prove the Discord report is fixed on main. The existing Discord queue test also asserts the second same-session message is not processed until the first run completes, which matches the reported failure mode rather than resolving it.
