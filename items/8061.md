## Summary

Keep #8061 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still has only a global subagent concurrency cap at agents.defaults.subagents.maxConcurrent; agents.list[] has no typed or validated per-agent maxConcurrent, and both native and ACP sessions_spawn paths still dispatch through the shared subagent lane.

## Best Possible Solution

Keep #8061 open. Add a validated per-target-agent subagent concurrency setting, preferably agents.list[].subagents.maxConcurrent unless maintainers choose another public shape; update types, strict schema, generated schema/help, and docs; then enforce it alongside the existing global CommandLane.Subagent cap in both native and ACP sessions_spawn paths. Add focused tests proving a capped target such as tts-agent serializes while uncapped targets can still run concurrently.
