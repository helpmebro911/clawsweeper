## Summary

Current main still only supports a global subagent concurrency lane through agents.defaults.subagents.maxConcurrent. The requested per-target-agent cap in agents.list[] is not present in the schema, generated config schema, typed config, subagent spawn path, or lane queue setup.

## Best Possible Solution

Keep #8061 open until core config and runtime support per-target-agent subagent concurrency. A likely implementation would add a validated per-agent field to AgentConfig/AgentEntrySchema and generated docs, then enforce it in the subagent run scheduling path by target agent id in addition to the existing global subagent lane cap, with tests covering one capped agent and other uncapped agents running concurrently.
