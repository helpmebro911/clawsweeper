## Summary

Keep open. Current main still has no explicit top-level auth/auth.profiles reload rule, so auth.profiles.* remains unmatched and still falls into the conservative restart path. The issue is also paired with open PR #68257 by the same author, so the maintainer path is to review and land or reject that PR rather than close this issue early.

## Best Possible Solution

Keep this issue open and route maintainer attention to PR #68257 or an equivalent gateway reload planner fix. The fix should explicitly classify auth.profiles.* OAuth/profile-refresh diffs as no-op/dynamic-read changes, preserve gateway.auth.* as restart-triggering gateway configuration, and add regression tests for both behaviors.
