## Summary

Keep this PR open. Current main still does not expose a unified provider billing schema in `openclaw models status --json`, does not include OpenRouter metered account/key usage, and does not wire OpenRouter into the current provider-usage surface. The PR remains a plausible implementation candidate, but it is not ready to close or merge because it needs rebase/design work around the provider-owned usage hook boundary plus explicit handling for sensitive billing output, non-2xx/partial failures, and fetch gating.

## Best Possible Solution

Keep this PR open as the implementation candidate. Rebase it onto current main, add only the generic core schema/output changes needed for a reviewed windowed/metered usage shape, and move OpenRouter-specific usage auth, HTTP requests, and parsing into `extensions/openrouter` via `resolveUsageAuth` and `fetchUsageSnapshot`. Fetch billing data only for output modes that render it, mark non-2xx and partial key failures accurately, add focused command/fetcher tests, and update public docs for the new JSON fields.

## Risks / Open Questions

- The current PR shape puts OpenRouter-specific auth/HTTP/parsing in core `src/infra`, which conflicts with the current provider-owned usage hook boundary and extension ownership rules.
- The proposed JSON output exposes account credits, per-key usage, limits, and remaining spend-like data; this needs explicit maintainer review for sensitivity and user expectations.
- The diff still performs OpenRouter billing fetches for default non-plain rich status even though the OpenRouter data is only consumed in JSON output, adding unnecessary latency and secret-bearing network calls.
- Non-2xx credits/key responses are collapsed to `null`, and partial or all-null key results can still look healthy; revoked keys or endpoint failures may be reported inaccurately.
- The command diff appears to type `openrouterUsage` as `ReturnType<typeof loadOpenRouterMeteredUsage>` while assigning an awaited value and reading fields, so the branch needs rebase/typecheck cleanup.
