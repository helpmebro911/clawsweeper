## Summary

Keep #70667 open. Current main still contains the stale `openclaw devices approve --latest` recovery guidance in both PR-touched docs files, while the CLI and canonical devices docs already make `--latest` preview-only and require a second explicit `requestId` approval.

## Best Possible Solution

Keep the PR open and merge or recreate its narrow docs correction on current main: update `apps/android/README.md` and `skills/node-connect/SKILL.md` to document the two-step flow, using `openclaw devices approve --latest` only as a preview/inspect step and `openclaw devices approve <requestId>` as the token-minting approval step.
