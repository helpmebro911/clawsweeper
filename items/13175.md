## Summary

Keep #13175 open. Current main has Feishu interactive card plumbing, callback handling, and hard-coded quick-action/approval cards, but it still does not implement generic Feishu inline-button rendering/routing for shared presentation button blocks or the requested Telegram-like inline button flow. The existing discussion also correctly distinguishes open PR #47891 as narrower `sendPayload`/channelData card work, not this full feature.

## Best Possible Solution

Keep #13175 open and implement native Feishu rendering for shared `MessagePresentation` button blocks in the bundled Feishu plugin. Add `presentationCapabilities`/`renderPresentation` or equivalent outbound support, map button values into structured Feishu card interaction envelopes, and add tests proving generated Feishu `action`/`button` elements route clicked values through `handleFeishuCardAction`. Treat PR #47891 as separate narrower `sendPayload` work unless maintainers explicitly make it the canonical tracker for this broader feature.
