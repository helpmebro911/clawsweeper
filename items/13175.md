## Summary

Keep #13175 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still does not implement generic Feishu inline/native interactive card buttons: Feishu presentation buttons render as markdown fallback text, and the Feishu outbound adapter/registration do not expose native presentation or payload hooks for clickable card buttons. Existing Feishu card-action handling is useful for structured quick/approval flows, but it is not a shipped implementation of generic `[label](btn:action)` or shared `MessagePresentation` button rendering. Related PR #47891 is open and narrower sendPayload/native-card payload work, not a completed fix for this issue.

## Best Possible Solution

Keep #13175 open. Implement Feishu-owned native rendering for shared presentation buttons: add Feishu outbound presentationCapabilities plus renderPresentation or equivalent payload delivery, render Schema 2.0 action/button elements for value and URL buttons, wrap callback values in structured Feishu interaction envelopes, route clicks through handleFeishuCardAction, preserve fallback text for unsupported surfaces, and add tests for clickable buttons, URL fallback, unsupported fallback, and callback dispatch. Treat #47891 as related narrower sendPayload/native-card payload work unless maintainers explicitly broaden it.
