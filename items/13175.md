## Summary

Keep #13175 open. Current main still does not implement generic Feishu inline/native card buttons: shared presentation button blocks are rendered as markdown fallback text, and the Feishu outbound adapter does not expose the native presentation or payload hooks needed for clickable Feishu card buttons. Existing Feishu card-action handling is useful, but it is wired to structured quick-action and approval flows, not generic `[label](btn:action)` or `MessagePresentation` button rendering. Related PR #47891 remains open and narrower sendPayload work rather than a shipped fix for this issue.

## Best Possible Solution

Keep #13175 open. Implement Feishu-owned native rendering for shared presentation buttons: add Feishu outbound presentationCapabilities plus renderPresentation or equivalent payload delivery, render Schema 2.0 action/button elements for value and URL buttons, wrap callback values in structured Feishu interaction envelopes, route clicks through handleFeishuCardAction, preserve markdown fallback for unsupported surfaces, and add tests for clickable buttons, URL fallback, disabled/unsupported fallback, and callback dispatch. Treat #47891 as related narrower sendPayload/native-card payload work unless maintainers explicitly broaden it.
