## Summary

Keep #39307 open. Current main still preserves silent-by-default compaction behavior, with only opt-in brief channel notices and a transient Control UI indicator. Post-compaction recovery is internal agent guidance, not a reliable user-facing disclosure or offer to reload context. The related issues in the provided context do not supersede this request.

## Best Possible Solution

Keep this open for maintainer/product work. A complete fix should define the default compaction/reset visibility contract across supported chat surfaces, decide which surfaces get durable or hard-to-miss events, add safe assistant-side disclosure and re-grounding behavior after compaction, and update docs/tests around `agents.defaults.compaction.notifyUser`, post-compaction recovery semantics, and related suppression controls such as #72043.
