## Summary

Keep PR #48476 open. Current main still does not implement the PR’s launchd bootstrap retry or `launchctl load -w` fallback, and the related macOS gateway install failure remains observable in source/tests/docs. The PR also still carries substantive review concerns around legacy `launchctl load` semantics and sudo/root targeting, so it should be revised rather than closed as implemented or obsolete.

## Best Possible Solution

Keep the PR open and rework it into a narrow current-main patch: add a well-classified bootstrap duplicate/retry path, decide explicitly whether any `launchctl load -w` fallback is safe for non-sudo user LaunchAgents, verify registration before reporting success, preserve existing GUI-session guidance for unsupported headless cases, and add focused tests for #8619/#46466-style errors plus non-retriable plist/permission failures.
