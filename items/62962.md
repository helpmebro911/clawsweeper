## Summary

Keep open. Current main still lacks the requested schema-driven per-tool security classification contract. OpenClaw has adjacent controls such as owner-only gating, allow/deny policy, plugin before_tool_call approval hooks, and provider-level parallel_tool_calls transport settings, but no public metadata surface for isReadOnly, isDestructive, isConcurrencySafe, interruptBehavior, isOpenWorld, or toAutoClassifierInput.

## Best Possible Solution

Keep this open. The best implementation is a backwards-compatible AgentToolSecurityProps-style metadata contract on AnyAgentTool and the public plugin SDK, preserved through registration, wrappers, adapters, inventories, and runtime definitions. Annotate core and bundled plugin tools incrementally, add conformance checks, and wire policy/UI/scheduling consumers behind conservative fallbacks while keeping existing heuristic and approval paths as compatibility fallback.

## Risks / Open Questions

- This is a core/public SDK contract change, so defaults and optionality need to preserve existing third-party plugin behavior.
- Wrappers, runtime adapters, effective tool inventories, MCP/export surfaces, provider bridges, and hook-wrapped tools need explicit preserve-or-drop rules for any new metadata.
- Concurrency metadata must stay conservative and must not weaken sandboxing, owner-only gating, allow/deny policy, approval hooks, or provider safety controls.
- toAutoClassifierInput needs redaction rules for secrets, SecretRef values, media references, and sensitive tool arguments before it is sent to any classifier prompt.
