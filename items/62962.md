## Summary

Keep open. Current main still lacks the requested schema-driven per-tool security classification contract. OpenClaw has related safety mechanisms, including owner-only filtering, allow/deny policy, plugin before_tool_call approvals, ACP readonly heuristics, and provider-level parallel_tool_calls, but none is the requested standard metadata surface for read-only, destructive, concurrency, interrupt, open-world, or classifier behavior.

## Best Possible Solution

Keep this open. The best path is a backwards-compatible metadata contract on AnyAgentTool and the public plugin SDK, preserved through registration, wrappers, adapters, inventories, and conformance checks. Core tools can then be annotated incrementally, with policy/UI/scheduling consumers added behind conservative fallbacks.

## Risks / Open Questions

- This is a core/public SDK contract change, so defaults and optionality need to preserve compatibility for existing third-party plugins.
- Wrappers, inventories, runtime adapters, hooks, trajectory/export surfaces, and provider bridges need a deliberate preserve-or-drop rule for any new metadata.
- Concurrency metadata should remain conservative and must not weaken sandbox, owner-only, allow/deny, approval, or provider safety controls.
- toAutoClassifierInput needs careful redaction and SecretRef/media handling so classifier prompts do not leak sensitive arguments.
