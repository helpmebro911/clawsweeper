## Summary

Keep this PR open. Current main has absorbed some Teams media pieces, but it still lacks the PR's channel /chats fallback and still has unresolved DM media behavior tracked by #65329. The PR should not merge as-is because its tenant-token path calls an undeclared token-provider overload.

## Best Possible Solution

Keep this PR open for maintainer cleanup: rebase against current main, preserve the shipped exact SMBA auth behavior, use readResponseWithLimit for hosted-content $value responses, add or explicitly reject the channel /chats fallback with tests, and coordinate remaining DM fallback behavior with #65329. The tenant-token overload should be replaced with a typed provider capability before any merge.
