## Summary

Keep open. Current `main` still does not give the Google/Gemini providers a catalog path into implicit `models.json`, and the fallback path still lands on `DEFAULT_CONTEXT_TOKENS = 200_000` when discovery/config metadata is absent.

## Evidence

- **Direct Google provider still has no catalog/staticCatalog:** `buildGoogleProvider()` registers auth, transport normalization, and `resolveDynamicModel`, but no `catalog`, `discovery`, or `staticCatalog` field.
  - file: [extensions/google/provider-registration.ts:14](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/google/provider-registration.ts#L14)

  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Gemini CLI provider still has no catalog/staticCatalog:** `buildGoogleGeminiCliProvider()` likewise exposes auth and `resolveDynamicModel` only; there is still no discovery catalog for `google-gemini-cli`.
  - file: [extensions/google/gemini-cli-provider.ts:33](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/google/gemini-cli-provider.ts#L33)

  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Implicit provider discovery only includes providers with catalog hooks:** `resolvePluginDiscoveryProviders()` filters providers down to ones with `catalog`/`discovery`/`staticCatalog`, and `resolvePluginImplicitProviders()` only materializes results from `runProviderCatalog()`. Providers without those hooks never seed implicit `models.json` entries.
  - file: [src/plugins/provider-discovery.ts:31](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/plugins/provider-discovery.ts#L31)

  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Google dynamic resolution still depends on existing template models:** `resolveGoogleGeminiForwardCompatModel()` clones from template ids already present in `ctx`; if no matching template exists, it returns `undefined`. An empty Google discovery cache therefore still leaves no Google template to clone.
  - file: [extensions/google/provider-models.ts:44](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/google/provider-models.ts#L44)

  - file: [src/agents/pi-embedded-runner/model.ts:540](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/pi-embedded-runner/model.ts#L540)
