## Summary

Current main does not show a modality-aware hybrid merge. Multimodal files are indexed with minimal label text, while hybrid ranking still applies the default BM25 weight to every result, so the reported image/audio suppression remains plausible and is not clearly fixed on `main`.

## Evidence

- **Hybrid merge is still a uniform weighted sum:** `mergeHybridResults` computes `vectorWeight * vectorScore + textWeight * textScore` for every entry with no modality check or bypass for image/audio chunks.
  - file: [extensions/memory-core/src/memory/hybrid.ts:127](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/memory-core/src/memory/hybrid.ts#L127)
  - command: `nl -ba extensions/memory-core/src/memory/hybrid.ts | sed -n '1,240p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Multimodal chunks still get only a path-style text label:** Multimodal indexing builds labels like `Image file: <path>` / `Audio file: <path>` and stores that as `contentText`, which is the only text BM25 can rank against.
  - file: [src/memory-host-sdk/host/multimodal.ts:73](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/memory-host-sdk/host/multimodal.ts#L73)

  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Multimodal file records are created from label text plus inline media:** `buildFileEntry` sets `contentText` from the modality label and returns a `kind: "multimodal"` entry, but the indexed text remains the synthetic label/path string.
  - file: [src/memory-host-sdk/host/internal.ts:245](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/memory-host-sdk/host/internal.ts#L245)

  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Search result plumbing does not carry modality metadata:** The search row/result types and SQL select only `id`, `path`, `start/end`, `text`, `source`, and scores, so current search/merge code has no modality signal available to special-case BM25 at ranking time.
