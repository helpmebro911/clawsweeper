## Summary

This issue should remain open. Current main still routes the default agent workspace fallback through resolvers that do not read OPENCLAW_WORKSPACE_DIR, while the system prompt prints the resolved fallback workspace. A related fix PR was opened with closing syntax for this issue but was closed unmerged, so there is no implemented-on-main or superseding shipped fix to close against.

## Best Possible Solution

Keep this issue open and review or reimplement the closed PR #66831 behavior on current main: make resolveDefaultAgentWorkspaceDir prefer a non-empty OPENCLAW_WORKSPACE_DIR before OPENCLAW_PROFILE/OPENCLAW_HOME fallback logic, preserve explicit per-agent workspace and agents.defaults.workspace precedence, and add focused regression coverage in the workspace resolver, agent-scope fallback, and run-workspace paths so prompt and runtime workspace resolution stay aligned.
