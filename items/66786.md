## Summary

Keep this issue open. Current main at 646a268d2710bc54e478349ef5690f3c8803df4f still does not route the default agent workspace fallback through OPENCLAW_WORKSPACE_DIR, while the system prompt continues to display the workspace value passed through that fallback path. The linked fix PR #66831 was closed unmerged, and the current agent tests still cover OPENCLAW_HOME rather than the reported OPENCLAW_WORKSPACE_DIR override.

## Best Possible Solution

Implement the closed PR #66831 behavior on current main: make resolveDefaultAgentWorkspaceDir prefer a non-empty OPENCLAW_WORKSPACE_DIR before OPENCLAW_PROFILE/OPENCLAW_HOME default fallback, keep explicit per-agent workspace and agents.defaults.workspace precedence unchanged, and add focused regression tests for the resolver plus resolveAgentWorkspaceDir / resolveRunWorkspaceDir so the system prompt and runtime workspace stay aligned.
