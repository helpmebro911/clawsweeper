## Summary

Current `main` now covers both parts of this report: legacy `tools.web.search.*` provider-owned config is normalized before startup validation, and the later `skills.update` write-back regression described in comments is addressed by the new source-based config persistence path added in commit `4d1ee3a73ed19063d16ac627d3071b7637fda82d`. I could not tie that final write-path fix to a tagged release yet.

## Evidence
