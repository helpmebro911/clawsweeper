## Summary

Keep #7540 open. Current main still does not subscribe the bundled WhatsApp plugin to Baileys call events, define WhatsApp call tracking config, or expose sanitized WhatsApp call activity through hooks/tools/session metadata. The maintainer discussion says the request remains valid and should be implemented in the WhatsApp plugin rather than the PSTN voice-call plugin.

## Best Possible Solution

Implement the feature in the bundled WhatsApp plugin: add an opt-in channel/account setting if maintainers want one, subscribe to Baileys call events, normalize and sanitize call payloads, apply existing account, DM/group, pairing, self-chat, and allowlist boundaries, then expose contact activity through a documented session-event, tool, or plugin-hook surface with focused tests and docs.
