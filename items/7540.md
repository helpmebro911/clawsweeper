## Summary

Keep #7540 open. Current main and the latest release still do not subscribe the WhatsApp/Baileys socket to `call`, and there is no call-tracking config, docs, or tests. The Baileys version used by the WhatsApp plugin does expose a `call: WACallEvent[]` event, so this remains an actionable WhatsApp plugin enhancement.

## Best Possible Solution

Keep the issue open and implement it in the bundled WhatsApp plugin: add an opt-in per-channel/per-account call-tracking setting if maintainers want the default off, subscribe to Baileys `call`, apply the same account and access-control boundaries used for inbound messages, expose sanitized call/contact activity through a documented session-event or tool surface, and add focused config/schema/docs/tests for the new behavior.
