## Summary

Keep #7540 open. Current main still does not subscribe the bundled WhatsApp plugin to Baileys `call` events, does not define WhatsApp call tracking config, and does not expose sanitized WhatsApp call activity through hooks/tools/session metadata. The maintainer discussion explicitly says the request remains valid and belongs in the WhatsApp plugin rather than the PSTN `voice-call` plugin, and the relevant WhatsApp files have not changed since that review.

## Best Possible Solution

Keep the issue open and implement it inside the bundled WhatsApp plugin: add an opt-in channel/account setting if maintainers want one, subscribe to Baileys `call`, normalize and sanitize call payloads, apply existing account, DM/group, pairing, and allowlist boundaries, then expose contact activity through a documented session-event, tool, or plugin-hook surface with focused tests and docs.
