## Summary

Keep open. Current `main` still has the reported Android TLS shape: the app opts into a domain-specific network security config for `openclaw.local` and `ts.net`, while both gateway TLS trust managers are anonymous `X509TrustManager` implementations that only override the 2-argument trust methods. I did not find any `X509ExtendedTrustManager` or other hostname-aware trust path on `main`, and Android onboarding/tests explicitly treat `*.ts.net` as a supported gateway host.
