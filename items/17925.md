## Summary

Keep #17925 open. Current main implements the Gemini side as a bundled `gemini` web_search provider using Google Search grounding, but the ZAI/GLM side is still missing: the bundled ZAI plugin has no `webSearchProviders` contract, no `registerWebSearchProvider(...)`, no web-search public artifact, and no docs/config/test coverage for ZAI search. The related Zhipu/ZAI search PRs were closed unmerged, so there is no canonical shipped fix to supersede this issue.

## Best Possible Solution

Keep #17925 open until the bundled ZAI plugin either registers a ZAI/GLM `web_search` provider through the existing plugin web-search contract with manifest metadata, config/docs, and tests, or adds provider-owned native `tools: [{ type: "web_search" }]` passthrough for ZAI model calls. Gemini appears implemented, so maintainers may split any remaining automatic-fallback semantics into a narrower follow-up if desired.
