## Summary

Keep #17925 open. Current main implements the Gemini side as a bundled `gemini` web_search provider using Google Search grounding, but the ZAI/GLM native `web_search` passthrough or bundled ZAI search provider is still missing. The related ZAI PRs in the provided context are closed and unmerged, so there is no canonical shipped fix to supersede this issue.

## Best Possible Solution

Keep #17925 open until the bundled ZAI plugin either registers a ZAI/GLM `web_search` provider through the existing plugin web-search contract with manifest metadata, config/docs, and tests, or adds provider-owned native `tools: [{ type: "web_search" }]` passthrough for ZAI model calls. Gemini appears implemented, so maintainers may split any remaining automatic-fallback semantics into a narrower follow-up if desired.
