## Summary

Keep open. Current main still lacks the requested github-copilot/claude-opus-4.7 adaptive-effort clamp: the Anthropic Messages transport maps `minimal`/`low` to `output_config.effort: "low"` for Opus 4.7, and then emits that value without a provider/model-specific compatibility guard. The related PR #70584 is the right shape of fix but is still open/unshipped in the provided GitHub context.

## Best Possible Solution

Keep #70322 open until a fix like #70584 lands: clamp `minimal`/`low` to `medium` for `github-copilot/claude-opus-4.7` on the adaptive Anthropic Messages path, add regression coverage for low and minimal, and separately decide whether `invalid_reasoning_effort`/format schema rejections should avoid profile cooldown.
