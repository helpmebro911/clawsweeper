## Summary

Keep open. Current main mitigates the bundled/default GitHub Copilot Claude model metadata by setting Copilot model definitions to `reasoning: false`, but the underlying Anthropic Messages adaptive-thinking transport still maps `minimal` and `low` to `output_config.effort: "low"` for Opus 4.7 whenever the resolved model is reasoning-capable. That leaves the reported configurable/metadata path unfixed, and the provided GitHub context already has an open focused fix PR (#70584) plus an earlier closed-unmerged attempt (#70376).

## Best Possible Solution

Keep #70322 open and resolve it through #70584 or an equivalent maintainer patch: add a provider-scoped Anthropic adaptive-effort clamp for `github-copilot` + `claude-opus-4.7` so `minimal`/`low` become `medium`, preserve non-Copilot behavior, and add regression tests for Copilot Opus 4.7 low/minimal plus unaffected models. Separately consider whether deterministic provider schema/format rejections should avoid auth-profile cooldown or use a narrower cooldown classification.
