## Summary

Keep #15032 open. Current main still has only global/config-derived sub-agent tool policy; `sessions_spawn` does not accept, persist, or enforce a per-spawn `tools`/allow/deny policy. The related implementation PR #23506 is closed and unmerged in the provided GitHub context, and #23906 is an open related/narrower request rather than a shipped replacement.

## Best Possible Solution

Keep #15032 open as the canonical core API/security gap. Implement an additive `sessions_spawn` tool-policy parameter, likely `tools: { allow?: string[]; deny?: string[] }` and possibly `tools: "none"` to cover #23906, then thread it through the tool schema, `SpawnSubagentParams`, child-session metadata, and `resolveSubagentToolPolicyForSession`. Enforce deny-wins and narrow-only semantics, add focused unit/e2e coverage, and update https://docs.openclaw.ai/tools/subagents.
