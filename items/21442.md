## Summary

Keep open. Current `main` has Groq documentation and a bundled Groq media-understanding plugin, but it does not show strong evidence of a native Groq LLM provider plugin or configure/onboarding integration equivalent to other bundled model providers.

## Evidence

- **Groq docs describe OpenAI-compatible setup, not a dedicated onboard flow:** The Groq provider page says OpenClaw connects to Groq through its OpenAI-compatible API and shows only `GROQ_API_KEY` plus a `groq/...` model ref example.
  - file: [docs/providers/groq.md:9](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/docs/providers/groq.md#L9)
  - command: `nl -ba docs/providers/groq.md | sed -n '1,220p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Bundled Groq extension is media-understanding only:** The Groq plugin registers `groqMediaUnderstandingProvider` and its manifest declares only `mediaUnderstandingProviders`, not `providers`.
  - file: [extensions/groq/index.ts:4](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/extensions/groq/index.ts#L4)
  - command: `nl -ba extensions/groq/index.ts | sed -n '1,200p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Groq manifest lacks provider/auth-choice metadata used by native model providers:** `extensions/groq/openclaw.plugin.json` contains `providerAuthEnvVars` and media-understanding metadata, but no provider contract or wizard/auth-choice entries.
  - file: [extensions/groq/openclaw.plugin.json:2](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/extensions/groq/openclaw.plugin.json#L2)

- **Contract test fixture treats Groq as media-understanding only:** `pluginRegistrationContractCases.groq` lists `mediaUnderstandingProviderIds: ["groq"]` and no `providerIds`, unlike native model providers.
  - file: [test/helpers/plugins/plugin-registration-contract-cases.ts:64](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/test/helpers/plugins/plugin-registration-contract-cases.ts#L64)
