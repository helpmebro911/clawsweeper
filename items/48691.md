## Summary

Keep open. Current main partially mitigates the old healthcheck behavior by delegating version checks to `openclaw update status`, but it does not implement the requested Homebrew-aware recommendation. The status/updater stack still models package installs as npm/pnpm/bun, can render `npm update <version>`, and the package update path invokes npm/pnpm/bun rather than `brew upgrade openclaw`.
