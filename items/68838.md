## Summary

Keep open. Current main still has the reported key-only `FOLLOWUP_QUEUES.delete(key)` path, and there is an open same-author PR with closing syntax that is the implementation candidate. This issue should remain open until that PR is reviewed and merged or explicitly closed.

## Best Possible Solution

Review PR #68839 as the canonical implementation candidate. If it is correct, land the identity guard and regression test, then let GitHub close this issue from the merged closing PR. If it is not correct, keep this issue open and ask for a revised fix that guards late drain cleanup against deleting a replacement queue.
