## Summary

Keep #70953 open. Current main still implements exec approval as an asynchronous two-phase flow: approval is registered, a background waiter/follow-up is started, and the exec tool returns `status: "approval-pending"` immediately. Docs and tests explicitly cover that behavior, so the requested foreground pause-until-resolution behavior remains unimplemented and is not covered by the related exec-approval reports.

## Best Possible Solution

Keep the issue open. The best implementation path is to add a foreground interactive exec approval mode that awaits `exec.approval.waitDecision` before returning from the tool call, then either runs the command and returns the actual command result, returns a denial/timeout result, or exits clearly on abort. Preserve the existing async follow-up behavior for explicit background/non-interactive paths, and update `src/agents/bash-tools.exec-host-gateway.ts`, `src/agents/bash-tools.exec-host-node.ts`, focused approval tests, and `docs/tools/exec.md` / `docs/tools/exec-approvals-advanced.md` together.
