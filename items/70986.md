## Summary

Keep #70986 open. Current main still has both reported root causes: wikilink extraction is regex-based and not Markdown-aware, and lint resolution only accepts path-derived targets, not frontmatter titles. The related #64696 overlaps reply-tag noise but does not cover the frontmatter-title resolver gap, so it is not a clean duplicate.

## Best Possible Solution

Keep the issue open and fix it in the memory-wiki plugin: make `extractWikiLinks` Markdown-aware or token-based, resolve wikilinks against normalized path, basename, frontmatter title, slugified title, and any alias fields, and add regression tests for fenced/inline-code literals, reply-tag transcript literals, natural-title `[[...]]` links, and relative Markdown links. Treat #64696 as related context, not a replacement for this issue.
