## Summary

Keep open. Current main still has both reported memory-wiki lint gaps: `extractWikiLinks` applies raw regex extraction after only removing the managed related block, and `collectBrokenLinkIssues` validates only page relative paths and basenames, not frontmatter titles or normalized title slugs. The only local broken-wikilink test asserts one missing-page warning and does not cover fenced/inline-code suppression, directive-tag false positives, relative target normalization, or title-based resolution. Related #64696 remains narrower and open, so it does not supersede this broader report.

## Best Possible Solution

Keep #70986 open as the canonical broader memory-wiki lint bug. The maintainer fix should update `extensions/memory-wiki/src/markdown.ts` to make wikilink extraction markdown-aware, update `extensions/memory-wiki/src/lint.ts` to resolve normalized page titles, ids, slugs, basenames, and relative paths consistently, and add focused regression tests in `extensions/memory-wiki/src/markdown.test.ts` and `extensions/memory-wiki/src/lint.test.ts` for fenced code, inline code, frontmatter/comment skips, `[[reply_to_current]]` transcript literals, natural frontmatter-title links, and relative markdown links.
