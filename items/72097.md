## Summary

Current main still does not implement the requested active-session plugin tool refresh. Plugin install paths persist and refresh registry metadata but still tell users to restart the gateway, and embedded Pi runs materialize a fixed tool array/customTools allowlist for the agent session. I found no source, docs, tests, or related-item evidence of a plugin-install toolset delta broadcast or per-turn tool re-query for already-active sessions.

## Best Possible Solution

Keep this issue open. The likely core fix is a generic plugin-registry/tool inventory invalidation path: after successful plugin install/update/enable that introduces tool registrations, refresh the active gateway registry as needed and make active sessions rebuild or merge tool definitions before the next LLM turn. Cover CLI and `/plugins install` paths, gateway sessions, embedded Pi runs, and regression tests proving a newly installed tool is visible without a manual gateway restart. Coordinate with #44401 only if restart-system-event delivery becomes part of the chosen design; it does not supersede this tool refresh request.
