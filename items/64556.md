## Summary

Current main still has the reported wake-hook routing mismatch: wake mappings can carry agentId/sessionKey in config/docs/schema, but the wake action drops those fields and dispatchWakeHook always enqueues to the resolved main session. The provided GitHub context also shows open PR #70268 with closing syntax for this issue, so this issue should remain open until that implementation candidate is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #70268 as the active implementation candidate. The fix should thread agentId/sessionKey through wake HookAction mapping and dispatch, reuse the existing hook agent/session policy helpers, preserve main-session fallback when routing is omitted, and add regression coverage for explicit sessionKey, agentId-derived main-session routing, templated session-key guardrails, and direct /hooks/wake behavior if that API broadening is accepted.
