## Summary

Keep #17876 open. The issue asks for a staged model migration workflow, but current main still only exposes immediate primary setting, manual fallback list management, OpenRouter-specific scanning, and auth/status probes. The provided issue discussion also includes an explicit maintainer comment saying to keep it open.

## Best Possible Solution

Keep #17876 open for a core `openclaw models` workflow. Build on the existing status/probe, fallback, auth, and scan primitives: add a candidate as fallback, test an arbitrary provider/model ref, promote only after verification while moving the old primary into fallbacks, persist enough prior state for `openclaw models rollback`, and cover the config transitions with focused CLI tests and docs.
