## Summary

Keep #17876 open. Current main has model status/list/set/scan, auth probing, and manual fallback management, but it does not implement the requested staged migration workflow with arbitrary model testing, promote-to-primary behavior, automatic old-primary fallback placement, or rollback. This is core model/config UX rather than a clear ClawHub-only item.

## Evidence

- **Current models CLI command surface:** `openclaw models` registers list, status, set, set-image, aliases, fallbacks, image-fallbacks, scan, and auth subcommands. There is no `test`, `migrate`, `promote`, or `rollback` command registered for staged migration.
