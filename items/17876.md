## Summary

Keep #17876 open. Current main still has immediate model setting, manual fallback list management, status/auth probes, and an OpenRouter-specific scan flow, but no staged model migration workflow with arbitrary model testing, promotion, old-primary fallback insertion, or rollback. The supplied issue context also includes an explicit maintainer comment saying to keep this open.

## Best Possible Solution

Keep #17876 open for a core `openclaw models` workflow. Build on the existing status/probe, fallback, auth, and OpenRouter scan primitives: add a candidate as fallback, test an arbitrary provider/model ref, promote only after verification while moving the old primary into fallbacks, persist enough previous config for `openclaw models rollback`, and cover the config transitions with focused CLI tests and docs.
