## Summary

Keep open. Current `main` still only supports boolean Discord mention gating plus separate user/role allowlists; there is no sender-specific `requireMentionFromIds`-style control. The request is coherent, in scope for the bundled Discord plugin/config surface, and not strong enough to close as implemented, cannot reproduce, ClawHub, incoherent, or stale-insufficient-info.

## Evidence

- **Discord mention gating is boolean-only:** `extensions/discord/src/group-policy.ts` resolves group mention behavior by checking only `channelEntry.requireMention`, then `guildEntry.requireMention`, then defaulting to `true`. There is no sender-specific mention list in this path.
  - file: [extensions/discord/src/group-policy.ts:93](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/extensions/discord/src/group-policy.ts#L93)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Config schema has no `requireMentionFromIds` field:** `src/config/zod-schema.providers-core.ts` defines Discord guild/channel config with `requireMention`, `users`, and `roles`, but no per-sender mention-gating field.
  - file: [src/config/zod-schema.providers-core.ts:448](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/config/zod-schema.providers-core.ts#L448)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Public Discord config types match the schema:** `src/config/types.discord.ts` exposes `requireMention?: boolean` and separate `users?: string[]` / `roles?: string[]` on Discord guild/channel entries. No `requireMentionFromIds`-style property exists in the public contract.
  - file: [src/config/types.discord.ts:39](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/config/types.discord.ts#L39)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **User allowlists are separate from mention gating:** `extensions/discord/src/monitor/allow-list.ts` enforces guild/channel `users` and `roles` as access restrictions. This is distinct from the mention-gating resolver, so the requested behavior is not already covered by current allowlist handling.
  - file: [extensions/discord/src/monitor/allow-list.ts:221](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/extensions/discord/src/monitor/allow-list.ts#L221)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)

It also notes other surfaces currently ignore `/activation`, so there is no documented Discord workaround for per-sender mention requirements.
  - file: [docs/channels/discord.md:195](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/docs/channels/discord.md#L195)
