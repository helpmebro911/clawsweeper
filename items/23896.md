## Summary

Keep #23896 open. Current main still lacks Discord per-sender mention gating: the requested `requireMentionFromIds` field is absent from config types, strict schema, docs, tests, and runtime mention resolution, and the prior issue discussion already reached the same keep-open conclusion.

## Best Possible Solution

Keep #23896 open. Best path is an additive Discord plugin/config change: add a typed sender-scoped mention policy such as `requireMentionFromIds` at guild/channel level, extend the strict zod schema plus generated metadata/docs, make the Discord mention-requirement resolver sender-aware after PluralKit/sender resolution, and add preflight tests proving listed human senders must mention while unlisted allowed bridge bots can trigger without mentions.
