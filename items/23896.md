## Summary

Keep #23896 open. Current main still does not support Discord per-sender mention gating: the config types, strict schema, runtime resolver, preflight path, tests, and docs expose guild/channel-level `requireMention` plus sender allowlists/bot modes, but no `requireMentionFromIds` or equivalent sender-scoped mention policy.

## Best Possible Solution

Keep #23896 open. Best path is an additive Discord plugin/config change: add a typed sender-scoped mention policy such as `requireMentionFromIds` at guild/channel level, extend strict zod schema plus generated metadata/docs, make the mention-requirement resolver sender-aware after Discord/PluralKit sender resolution, and add preflight tests proving listed human senders must mention while unlisted allowed bridge bots can trigger without mentions.
