## Summary

Keep #13676 open. Current main still does not have a first-class Project model, Projects tab/sidebar, project-to-home-session binding, project workspace list, or project-scoped skill/tool/runtime constraints. Existing agents, sessions, workspace skills, and per-agent sandbox/tool policies are useful adjacent primitives but do not satisfy the broader dashboard/product request.

## Best Possible Solution

Continue tracking #13676 as the canonical core/product dashboard request. A complete implementation should add explicit Project storage and Gateway protocol methods, expose Projects in Control UI, bind each project to a home session plus optional workstreams, associate one or more workspace paths, and surface project-scoped tool/skill/runtime constraints while reusing existing agent, session, workspace, skill-policy, and sandbox primitives where they fit.
