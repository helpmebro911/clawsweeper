## Summary

Keep open. The issue asks for `openclaw doctor` to detect stale cached `skillsSnapshot` metadata containing old runtime-root paths and suggest safe invalidation. Current `main` has related session and skills maintenance, including config-write snapshot invalidation, but I found no doctor check that scans persisted session `skillsSnapshot.prompt` or `resolvedSkills` for inactive runtime roots, deleted temp paths, or bundled-skill paths that disagree with the active runtime root. The reporter’s follow-up with a verified manual remediation remains relevant, and the related ClawSweeper reports/PRs cover adjacent stale-snapshot refresh behavior rather than this doctor diagnostic.
