## Summary

Current main still does not fix or document the reported intermittent `gog` Gmail auth failure under non-interactive macOS scheduled runs. OpenClaw has visible `gog` and Gmail automation surfaces, but auth resolution is delegated to the external `gog` CLI, and the current skill/docs do not cover cron-safe keyring setup, debug logging, or recovery/preflight behavior for this failure mode.

## Best Possible Solution

Keep this issue open. The best maintainer path is to document a cron-safe `gog` setup in the bundled `gog` skill and public Gmail Pub/Sub/scheduled automation docs, and/or add an OpenClaw doctor/preflight warning for non-interactive macOS `gog` auth. The fix should cover stable `HOME`/`PATH`/`GOG_ACCOUNT`/`GOG_CLIENT`, `gog auth keyring file`, `GOG_KEYRING_BACKEND=file`, `GOG_KEYRING_PASSWORD`, and an auth-resolution debug/logging path before deciding whether an upstream `gogcli` bug is also needed.
