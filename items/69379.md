## Summary

Keep this PR open. Current main still computes both preflight compaction and memory-flush thresholds from `reserveTokensFloor` only, while the documented/configured semantics and Pi settings path support the PR's `max(reserveTokens, reserveTokensFloor)` direction. The PR remains the active candidate for #66830 after #67297 closed unmerged.

## Best Possible Solution

Keep this PR open as the canonical implementation path for #66830: land the effective reserve helper or equivalent `max(reserveTokens, reserveTokensFloor)` calculation, thread it through preflight compaction and memory-flush gates, keep focused regression coverage and the changelog entry, then close #66830 after merge.
