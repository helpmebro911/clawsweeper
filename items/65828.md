## Summary

Keep this PR open. Current main still has the MiniMax web-search metadata/onboarding mismatch this PR targets, and the PR is a focused contributor fix rather than obsolete cleanup. It still needs maintainer judgment on whether a generic `MINIMAX_API_KEY` should be auto-detected as search-capable when MiniMax search requires Token Plan access.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to preserve the useful MiniMax metadata, onboarding, and docs alignment, then explicitly decide how `MINIMAX_API_KEY` should participate in web-search auto-detection: accept the Token Plan false-positive risk, require explicit MiniMax Search configuration, lower/constrain the env auto-detect path, or add a search-capability check if MiniMax exposes one. Add focused tests for setup visibility and the selected auto-detection behavior before merge.
