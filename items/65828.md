## Summary

Keep this PR open. Current main still has the MiniMax web-search metadata/onboarding mismatch the PR targets: runtime can use MINIMAX_API_KEY, but provider metadata, setup visibility, docs, and tests still describe Coding Plan env vars only. The PR is focused and security scope looks narrow, but the unresolved Token Plan false-positive auto-detection concern is a real product tradeoff to resolve before merge.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. Preserve the useful MiniMax metadata/docs/onboarding alignment, but explicitly resolve the Token Plan auto-detection tradeoff before merge: for example, support MINIMAX_API_KEY when explicitly configured under plugins.entries.minimax.config.webSearch.apiKey, add a search-capability check if MiniMax exposes one, lower or constrain blind env auto-detection, or get maintainer approval that the false-positive risk is acceptable. Add focused coverage for MiniMax Search setup visibility and auto-detection once the intended behavior is chosen.
