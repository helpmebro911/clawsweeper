## Summary

Keep #20756 open. Current main still does not auto-select the sole enabled account for omitted-account `message` sends; the shared runner and WhatsApp outbound path still fall back through explicit/default/binding/default-account behavior, and the related fix PR #20801 is closed unmerged.

## Best Possible Solution

Keep #20756 open. Implement shared outbound account resolution after channel selection in `runMessageAction`: preserve explicit params, tool defaults, and agent bindings; use a configured default only when usable/enabled; auto-select exactly one enabled account when no usable default exists; return clear zero-or-multiple-enabled-account errors. Add focused regression tests for the shared message action path and the WhatsApp disabled-default/single-enabled-account case, then update multi-account message/channel docs.
