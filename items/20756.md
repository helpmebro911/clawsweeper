## Summary

Keep #20756 open. Current main still does not auto-select the only enabled account for omitted-account `message` sends. The shared message action path only uses an explicit `accountId`, the tool/default account id, or the first agent binding; channel selection only resolves the channel, not the account; and WhatsApp still falls through to its default-account helper when no account is provided. The related fix PR #20801 was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep #20756 open. Implement shared outbound account resolution in runMessageAction after channel selection: preserve explicit params, tool defaults, and agent bindings; use a configured default only when enabled/configured; auto-select exactly one enabled account when no usable default exists; and return clear zero-or-multiple-enabled-account errors. Add focused regression tests for the shared message action path plus the WhatsApp disabled-default/single-enabled-account case, then update multi-account/channel-routing docs.
