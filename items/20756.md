## Summary

Keep open. Current `main` still defaults omitted `accountId` to the channel default path, and for WhatsApp that path still resolves to `default` whenever that account exists, without checking whether it is disabled or whether another single enabled account should be auto-selected.

## Evidence

- **Message tool does not add single-enabled-account fallback:** The tool only sets `params.accountId` from an explicit `accountId` parameter or the bound agent account. There is no logic to inspect channel accounts and choose the only enabled one when `accountId` is omitted.
  - file: [src/agents/tools/message-tool.ts:763](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/agents/tools/message-tool.ts#L763)
  - command: `nl -ba src/agents/tools/message-tool.ts | sed -n '740,820p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Outbound runner preserves omitted accountId unless caller provided one:** `runMessageAction()` resolves `accountId` from params or `defaultAccountId`, then optionally from agent bindings. It does not scan configured accounts for a sole enabled account.
  - file: [src/infra/outbound/message-action-runner.ts:849](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/infra/outbound/message-action-runner.ts#L849)
  - command: `nl -ba src/infra/outbound/message-action-runner.ts | sed -n '844,872p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Shared default-account helper still prefers literal `default`:** `resolveListedDefaultAccountId()` returns `default` whenever that id is present in the account list, before considering any other fallback. It does not consider account enabled/disabled state.
  - file: [src/channels/plugins/account-helpers.ts:115](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/channels/plugins/account-helpers.ts#L115)
  - command: `nl -ba src/channels/plugins/account-helpers.ts | sed -n '106,126p'`
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **WhatsApp send path still falls back to default-account resolution:** When outbound WhatsApp send has no explicit `accountId`, `resolveOutboundWhatsAppAccountId()` calls `resolveDefaultWhatsAppAccountId()`, and the error still reports `No active WhatsApp Web listener (account: <resolved>)` for that resolved account.
  - file: [extensions/whatsapp/src/send.ts:22](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/extensions/whatsapp/src/send.ts#L22)

There is no test for auto-selecting a sole enabled non-default account.
