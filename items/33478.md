## Summary

Keep #33478 open. Current main still lacks the structured `sessions_send` action/override contract and still relies on exact `REPLY_SKIP` / `ANNOUNCE_SKIP` model text for A2A control flow. The thread-routing portion is partially fixed and tested, but a maintainer comment explicitly asks to keep this issue as acceptance criteria for #63789 until structured skip/no-reply outcomes and thread inheritance are covered.

## Best Possible Solution

Keep #33478 open as acceptance criteria for first-party A2A handoff work, likely coordinated with #63789 and adjacent routing work such as #27445. The best fix is a structured `sessions_send` outcome/action contract covering skip, no-reply, announce suppression, explicit routing overrides, legacy token compatibility, and regression tests for both token chatter and threaded announce routing.
