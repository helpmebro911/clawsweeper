## Summary

Keep open. Current main has newer overlapping gateway-status work, including lightweight connect/auth probes and `--require-rpc`, but it does not fully contain this PR's explicit-auth boundary for CLI URL overrides or its restart-health timeout/budget change. The PR likely needs rebase/splitting, not cleanup closure.

## Best Possible Solution

Keep the PR open, but require a rebase or replacement PR that splits the still-useful work into current-main terms: preserve the connect/auth-capability default status probe, add tests and implementation for explicit CLI credentials on `--url` daemon status probes, and separately decide whether restart health should adopt a bounded longer local probe timeout.
