## Summary

Keep #70523 open. Current main only mitigates the common case by defaulting dreaming storage to separate reports; the explicit inline storage path still exists, is schema-accepted and tested, and still writes raw light/REM phase bodies into `memory/YYYY-MM-DD.md` before request-scoped narrative fallback. Main does not contain the PR's fallback result plumbing, `inlineBodyLines`, or managed-block removal path, and the related open #70332 is overlapping but not a merged or clearly canonical superseder.

## Best Possible Solution

Keep #70523 open for maintainer review. The best path is either to land a scoped version that preserves separate/default behavior while suppressing raw inline light/REM dumps after successful request-scoped fallback, or to make an explicit maintainer product decision to remove/deprecate inline dreaming storage instead of leaving the supported inline path noisy. If maintainers prefer #70332 as the canonical diary-hardening vehicle, they should say so and close #70523 with that explicit superseding link.
