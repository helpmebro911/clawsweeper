## Summary

Keep open. Current `main` still has the same thread-scoping path the report describes: ACP/subagent completion delivery can resolve to the bound Discord thread, send the parent-side completion turn back through `agent` ingress with that `threadId`, and `initSessionState` still resolves/touches conversation bindings for that inter-session turn because it only suppresses retargeting for `heartbeat`/`cron-event`/`exec-event` providers.

## Evidence
