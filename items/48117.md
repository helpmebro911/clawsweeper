## Summary

Keep open. The issue reports the Windows PowerShell installer failing when the spawned npm install hits `UNABLE_TO_VERIFY_LEAF_SIGNATURE`. Current main still runs a single `npm.cmd install -g` from `scripts/install.ps1` and exits on any non-zero npm result, with no TLS-specific detection, `NODE_EXTRA_CA_CERTS` setup, cafile guidance, or one-shot retry. The related Windows PR #48416 proposed a fix but was closed unmerged; the later landed TLS work is Linux/nvm gateway startup handling and does not cover this Windows installer path.
