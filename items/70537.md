## Summary

Keep #70537 open. Current main has adjacent hardening for malformed tool-call argument streams, but I did not find enough source, test, changelog, or landed-history evidence that the reported direct Anthropic array-field JSON corruption is fixed. The linked fork PRs are useful signal, but they are not evidence of a landed OpenClaw core fix.

## Best Possible Solution

Keep the issue open until maintainers either land or reject a targeted fix. The best next step is a focused regression test for direct Anthropic `anthropic-messages` tool-call streaming with array-of-object arguments, plus a source fix in the gateway/embedded-runner serialization path if that test reproduces the malformed JSON. If the external fork PRs contain the right approach, port the minimal fix into OpenClaw main and cite the shipped commit/release back to #70537.
