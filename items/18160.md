## Summary

Keep #18160 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still has no direct exec/shell cron payload: cron types, Gateway schema, validation, runtime dispatch, CLI, UI, and docs remain limited to `systemEvent` and `agentTurn`. The related implementation PR #51276 was closed unmerged, while duplicate shell/exec cron requests have been closed in favor of this issue as the canonical core tracker.

## Best Possible Solution

Keep #18160 open as the canonical core cron direct-exec tracker. The best path is an additive `exec` or `shell` payload contract across cron types, Gateway schema/validators, normalization, runtime dispatch, CLI add/edit, Control UI, docs, and focused tests, using a controlled process runner and documented security, logging, timeout, retry, and delivery semantics. Broader direct-tool or conditional pipeline ideas should remain separate unless maintainers intentionally fold them into this core cron payload design.
