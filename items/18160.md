## Summary

Keep #18160 open. Current main still has no direct exec/shell cron payload: the type, Gateway schema, validator, runtime dispatch, CLI, UI, docs, and agent cron tool surfaces all remain built around `systemEvent` and `agentTurn`. The related implementation PR #51276 was closed unmerged, and the latest maintainer comment explicitly kept #18160 open as the core tracker.

## Best Possible Solution

Keep #18160 open as the canonical core cron direct-exec tracker. The best implementation path is an additive `exec` or `shell` payload contract across cron types, Gateway schema/validators, normalization, runtime dispatch, CLI add/edit, Control UI, docs, and focused tests, with a controlled process runner and documented security, logging, timeout, and delivery semantics. Broader direct-tool or conditional pipeline ideas should stay separate unless maintainers explicitly fold them into this core cron payload design.
