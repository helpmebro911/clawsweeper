## Summary

Keep open. Current `main` still rejects workspace bootstrap files that symlink outside the workspace root, and there is no strong evidence that PR #51277's sibling-directory allowance is already implemented or obsolete.

## Evidence

- **Symlink hops still require the resolved target to stay inside the workspace root:** `applyResolvedSymlinkHop()` on current `main` throws immediately when `linkCanonical` is outside `rootCanonicalPath`; there is no fallback that permits a sibling under the parent `~/.openclaw/` root.

  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Canonical-alias enforcement is still workspace-root-only:** `assertLexicalBoundaryOrCanonicalAlias()` returns only when the canonical path is inside `rootCanonicalPath`, then throws. No parent-root exception exists on current `main`.
  - file: [src/infra/boundary-path.ts:635](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/infra/boundary-path.ts#L635)
