## Summary

Keep open. Current main improves cron auditability with task records, finished-run logs, finished webhook delivery, and failure destinations, but it still does not provide the requested native cron start/completion webhook, telemetry-event surface, or typed plugin cron lifecycle hook. The related implementation PR #8855 was closed unmerged, and the latest maintainer review comment already kept this item open for the same missing behavior.

## Best Possible Solution

Keep tracking #8209 until a maintainer chooses the product direction. The most conservative implementation path is a generic cron lifecycle event surface, preferably a typed plugin hook or subscription API that emits start and finish events for both agentTurn and systemEvent jobs, with optional docs/examples for posting to external systems. If core webhooks are chosen instead, keep them generic, reuse the existing task ledger/run-log data, apply the existing SSRF/token handling, and add focused tests for start, success, failure, timeout, and disabled/misconfigured webhook cases.
