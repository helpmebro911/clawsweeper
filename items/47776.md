## Summary

Keep #47776 open. The submitted PR is not merge-ready, but the central Linux/systemd npm-global problem is not implemented on current main: service-mode startup still calls only the existing stale cleanup, and the Unix cleanup path still filters `lsof` output by command name before any argv verification, so a gateway shown by `lsof` as `node` can still be missed.

## Best Possible Solution

Keep the item open, but do not merge the current PR branch unchanged. The best path is a focused gateway cleanup patch: make Unix listener discovery collect PIDs independently of the lsof command name, verify each candidate through `isGatewayArgv` using `/proc/<pid>/cmdline` or `ps`, preserve existing self/ancestor protections, and add a regression test for the npm-global `node` command-name case in the service-mode startup cleanup path.
