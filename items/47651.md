## Summary

Keep open. Current main partially implements Gemini multimodal memory indexing, but only for opt-in image/audio files under configured extraPaths. #47651 asks for broader raw media recall, including video, PDFs, and media sent to an agent and marked worth remembering. Current config/schema/docs and tests still exclude video/PDF and do not provide a direct remembered-inbound-media path.

## Best Possible Solution

Keep this open and build on the existing memory-core/Gemini structured-input path: add explicit video and PDF modalities plus format/size limits, support the current Gemini Embedding 2 model name alongside any needed preview compatibility, and design a deliberate path for user-approved inbound media to be persisted and indexed rather than only indexing configured extraPath files.
