## Summary

Current `main` already contains Control UI chat-state handling that prevents duplicate assistant rendering by clearing streaming artifacts on history reload and deferring `session.message` reloads until the active run finishes; the behavior is covered by focused regression tests added in commit `51186d2725439646f7fea0e59bb466f957404c46`.
