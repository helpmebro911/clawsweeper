## Summary

Keep #72240 open. Current main still has only coarse exec termination reporting: it can classify OpenClaw timeouts versus a generic signal, but it does not expose the requested SIGKILL cause metadata, supervisor-vs-OS distinction, RSS/process-tree data, stdout/stderr tail as structured failure metadata, or a gateway log/run pointer. The related open issues cover adjacent supervisor/SIGKILL problems, but none fully supersedes this macOS diagnostics request.

## Best Possible Solution

Keep the issue open and implement this in core exec/process, not ClawHub: extend the supervisor/run exit contract and process session records with structured termination origin metadata, timeout settings, run/session identifiers, and preserved output tail; emit the same fields in exec.process.completed diagnostics; surface them in foreground exec and process poll/log details; add supervisor/runtime tests for supervisor timeout SIGKILL, manual cancel, and externally observed SIGKILL on macOS/POSIX. Broad-scan guardrails over large OpenClaw state/log paths can be a follow-up or separate scoped issue if maintainers want product behavior beyond diagnostics.
