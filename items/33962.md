## Summary

Keep #33962 open. Current main still generates session-memory slugs with the default agent's resolved primary model, runs that work through the normal embedded-runner lanes, and only falls back to the timestamp slug after the LLM helper returns. The generic timeout configuration mitigates one symptom, but the requested lightweight slug model, early fallback, or debounce/decoupled lane behavior is still absent.

## Best Possible Solution

Keep #33962 open for a focused core fix. The best path is to add a slug-specific lightweight model override to the config contract, wire it through src/hooks/llm-slug-generator.ts, preserve the existing llmSlug: false opt-out, and add regression coverage for model selection, timeout/fallback behavior, and lane impact. A short heuristic fallback or debounce can be considered alongside the model override, but the central fix is decoupling slug generation from the user-facing primary model.
