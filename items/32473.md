## Summary

Keep open. Current `main` still intentionally rejects non-localhost HTTP Control UI sessions without device identity, so the issue's reported behavior still matches shipping code. Recent UI/docs work improves the explanation and workaround guidance, but it does not implement the requested homelab/non-127.0.0.1 insecure flow.
