## Summary

This issue should stay open. Current main still does not implement Slack reply_broadcast support: outbound Slack send/action/generic message-tool paths only plumb threadTs/thread_ts, and inbound thread_broadcast events are accepted without surfacing an agent-visible broadcast/mirroring flag. The related PR #64365 remains open as an implementation candidate, not shipped behavior.

## Best Possible Solution

Keep this issue open and review/land PR #64365 or a successor Slack-plugin patch. The best fix belongs in the Slack plugin: add typed replyBroadcast plumbing through Slack send/action/generic message-tool paths, pass Slack reply_broadcast only for threaded replies, expose inbound thread_broadcast visibility in agent/tool context so mirroring is possible, and add focused regression tests plus Slack docs.
