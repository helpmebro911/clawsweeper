## Summary

Current `main` still has the exact alias gap this PR fixes. `handleToolExecutionStart` for the `read` tool checks `path` and `file_path` only, while the shared `resolvePathArg` helper already supports `path`, `file_path`, and `filePath`. The existing tests cover `file_path` but not `filePath`, and local history inspection found no merged equivalent fix. This PR still appears relevant and should remain open.
