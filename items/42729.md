## Summary

Keep this PR open. Current main still has the cron restart duplicate-execution gap for an already-executed stale past-due slot: startup catch-up uses the runnable scan before the maintenance-only `lastRunAtMs >= nextRunAtMs` repair can advance that stale slot. The PR is a narrow implementation candidate with a matching regression test and no apparent security-sensitive diff scope.

## Best Possible Solution

Keep this PR open and review or land it, or port the same guard into current main's `isRunnableJob`/startup catch-up runnable path with the PR's regression test. The linked bug report is closed, but current main still appears to need this narrow scheduler fix.
