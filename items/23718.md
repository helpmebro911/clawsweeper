## Summary

Keep open. Current `main` still implements `heartbeat.model` as a string-only override; the requested `{ primary, fallbacks }` chain support is not present in the config types, schema, runtime path, docs, or tests.

## Evidence

- **Heartbeat config type is string-only:** `agents.defaults.heartbeat.model` is declared as `model?: string`, unlike the shared `AgentModelConfig` used for normal agent/subagent model selection.
  - file: [src/config/types.agent-defaults.ts:345](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/config/types.agent-defaults.ts#L345)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Schema only validates a string:** `HeartbeatSchema` accepts `model: z.string().optional()`, so `{ primary, fallbacks }` would not validate on current main.
  - file: [src/config/zod-schema.agent-runtime.ts:30](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/config/zod-schema.agent-runtime.ts#L30)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Runtime forwards one normalized string override:** The heartbeat runner reads `heartbeat?.model` with `normalizeOptionalString(...)` and passes a single `heartbeatModelOverride`; there is no fallback-chain parsing or propagation.
  - file: [src/infra/heartbeat-runner.ts:1064](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/infra/heartbeat-runner.ts#L1064)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Tests cover only string heartbeat overrides:** Current heartbeat override tests only assert string values such as `ollama/llama3.2:1b`, which matches the current implementation surface.
  - file: [src/infra/heartbeat-runner.model-override.test.ts:168](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/infra/heartbeat-runner.model-override.test.ts#L168)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Docs still document heartbeat.model as provider/model string:** The heartbeat docs describe `model` as an optional `provider/model` override, with no object-form fallback support documented.
  - file: [docs/gateway/heartbeat.md:224](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/docs/gateway/heartbeat.md#L224)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Comparable subagent surface already supports object form:** Subagent model config is explicitly typed as `AgentModelConfig` / `string or {primary,fallbacks}`, which highlights that heartbeat has not been upgraded to the same contract.
  - file: [src/config/types.agent-defaults.ts:400](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/config/types.agent-defaults.ts#L400)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
