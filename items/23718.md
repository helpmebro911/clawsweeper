## Summary

Keep open. The request is concrete and still meaningful on current main: `heartbeat.model` remains string-only in schema/types and the heartbeat runtime only forwards a string `heartbeatModelOverride`. Regular agent/subagent model config already supports `{ primary, fallbacks }`, so the requested parity is not implemented on main.
