## Summary

Keep this issue open. Current main still ships the broad mandatory `memory_search` prompt/tool wording reported by the user, the bundled memory plugin actively registers that surface, and existing tests preserve the `run memory_search` guidance without a current-system-state exception.

## Best Possible Solution

Update `extensions/memory-core/src/prompt-section.ts` and `extensions/memory-core/src/tools.ts` so memory recall is framed for historical prior work, decisions, people, preferences, and todos, while current system state/configuration/time/service questions are directed to live verification tools first. Add focused regression coverage in `extensions/memory-core/index.test.ts`; if prompt precedence remains ambiguous, add a `buildAgentSystemPrompt` composition test around the Execution Bias and Memory Recall sections.
