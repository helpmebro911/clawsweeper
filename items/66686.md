## Summary

Keep open. Current main still hard-codes `memory_search` as a mandatory pre-answer step for broad categories including `people`, while the main system prompt and CLI docs separately direct agents to verify current runtime/config state with live status commands. That conflict matches the report rather than disproving it.

## Evidence

That is the exact over-broad rule the issue reports, and it remains present in both the search+get and search-only branches.
