## Summary

Keep this issue open. Current main still ships the reported broad mandatory memory-search guidance in both the memory prompt section and the `memory_search` tool description. Separate live-check guidance exists, but it does not carve current system state, configuration, service, or time questions out of the memory-specific mandatory wording.

## Best Possible Solution

Update `extensions/memory-core/src/prompt-section.ts` and `extensions/memory-core/src/tools.ts` so memory recall is framed for historical prior work, decisions, dates, people, preferences, and todos, while current system state, configuration, time, and service-status questions are directed to live verification tools first. Add focused regression coverage in `extensions/memory-core/index.test.ts`; if prompt precedence remains ambiguous, add a `buildAgentSystemPrompt` composition test covering Execution Bias plus Memory Recall.
