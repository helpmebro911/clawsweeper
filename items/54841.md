## Summary

Keep open. Current main still reads and records the configured Feishu webhookPath but does not compare incoming request paths before body parsing, signature validation, and dispatch. PR #54904 remains open and unmerged with closing syntax for this issue, so it should be reviewed/landed or closed before this issue is resolved.

## Best Possible Solution

Keep this issue open while PR #54904 remains open. The best path is maintainer review of that PR, or an equivalent Feishu-owned fix in extensions/feishu/src/monitor.transport.ts that normalizes the configured webhookPath, rejects mismatched request pathnames before body parsing/signature validation/dispatch, preserves query-string behavior on the configured path, and adds focused regression coverage.
