## Summary

Keep this issue open. Current main still reads the configured Feishu webhookPath but does not compare the incoming request URL/path before body parsing, signature validation, and dispatch. The provided GitHub context also shows PR #54904 is open, unmerged, and uses closing syntax for this issue, so the issue should remain open until that implementation candidate is merged or rejected.

## Best Possible Solution

Keep the issue open while PR #54904 is open. The best path is maintainer review of that PR, or an equivalent Feishu-owned fix in extensions/feishu/src/monitor.transport.ts that normalizes the configured webhookPath, rejects mismatched URL pathnames with 404 before body parsing/signature work/dispatch, preserves query-string behavior on the configured path, and adds focused regression coverage.
