## Summary

Keep open. Current main still lacks the requested reusable sub-agent template feature: there is no `subagents.templates`/`promptFile` config surface, no `sessions_spawn(template=...)` parameter, and the spawn path builds the child prompt directly from the provided `task`. The cross-referenced AIpexForge/agent-swarm PR is useful evidence of the orchestration use case, but it is external context rather than an OpenClaw implementation or canonical superseding issue.

## Evidence

- **sessions_spawn schema has no template parameter:** The tool schema exposes `task`, `label`, `runtime`, `agentId`, `model`, `thinking`, `runTimeoutSeconds`, context/thread/attachment controls, and ACP-only fields, but no `template` or `promptFile` field. The execute path reads and forwards those explicit fields directly.
  - file: [src/agents/tools/sessions-spawn-tool.ts:103](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/agents/tools/sessions-spawn-tool.ts#L103)
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **Native subagent spawn params lack template support:** `SpawnSubagentParams` includes `task`, `label`, `agentId`, `model`, `thinking`, `runTimeoutSeconds`, and related controls, but no template identifier. Later, the child task message is assembled as `[Subagent Task]: ${task}`, with no prompt-file lookup or template prepending step.
  - file: [src/agents/subagent-spawn.ts:110](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/agents/subagent-spawn.ts#L110)
