## Summary

Keep this PR open. Current main still does not implement `model.tool` or split model routing, and the PR remains a plausible implementation candidate. It should not be closed as implemented, duplicate, or ClawHub-only because the current plugin hooks cannot perform per-tool-continuation model/transport swaps, while the PR itself still needs maintainer review for correctness and auth/transport boundary issues.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to decide whether split model routing belongs in the core `agents.defaults.model` contract or in a new plugin/API seam. If it stays core, rework it so the effective per-call model controls tool enablement, model resolution, auth profile and API key selection, transport dispatch, transcript policy, provider wrappers, context budgeting, fallbacks, diagnostics, docs, generated config baselines, and tests. If maintainers prefer the plugin route, use #13225 as the design thread for the missing per-tool routing seam instead of closing this PR as already handled.

## Risks / Open Questions

- The proposed implementation may not satisfy its advertised local-primary plus API-tool workflow because the first tool-emitting turn and tool availability are still governed by the primary model.
- Cross-provider routing can reuse primary-provider transport, auth profile, API key injection, transcript policy, provider text transforms, and provider-specific stream wrappers for tool-model turns.
- Context budgeting, fallback behavior, diagnostics, cache traces, and trajectory metadata are primary-model-bound in current main and need a clear per-call/effective-model contract before this lands.
- Accepting `model.tool` as core config would require generated config docs/baselines and public docs updates.
- The provided PR test plan still lacks manual Ollama plus Anthropic integration proof and full repo validation.
