## Summary

Keep this PR open. Current main has not implemented `model.tool` or split model routing, and the existing plugin hooks do not provide an equivalent per-tool-continuation routing contract. The PR is also not ready to close as obsolete: the review discussion identifies real cross-provider defects in the proposed implementation, but that means it needs maintainer review/changes rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or author changes. The best path is to rework the design so the effective per-call model drives tool support, provider transport/auth resolution, transcript policy, and provider-specific stream wrappers; update the config schema/docs/generated baseline if a core `model.tool` surface is accepted; and add focused tests plus at least one cross-provider integration proof for the advertised local-primary/API-tool workflow. If maintainers prefer a plugin API instead of core config, use #13225 as the canonical design discussion rather than closing this as already handled.
