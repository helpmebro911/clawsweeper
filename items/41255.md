## Summary

Keep open. Current main and the latest release still reject `sessions_send` calls that provide both `sessionKey` and `label`, matching the bug described by linked #41199. This PR is a narrow implementation candidate for a real live behavior, not obsolete or superseded by a merged fix.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best maintainer path is to review one canonical `sessions_send` fix, require or add a focused regression test proving `sessionKey` wins when both selectors are provided, merge it, then close overlapping PRs and the `sessions_send` portion of #41199 as implemented.
