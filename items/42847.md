## Summary

Keep open. Current `main` does not implement this PR, and recently-added Slack plugin code, tests, and docs all codify the opposite behavior: Slack reads still prefer the user token regardless of `userTokenReadOnly`. The PR also has unresolved review feedback about bot-present-but-denied read regressions, so conservative cleanup should not auto-close it as implemented/obsolete.
