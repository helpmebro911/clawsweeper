## Summary

Keep this issue open. Current main only gives Feishu abort text such as `/stop` a dedicated control lane; `/status` and `/new` still use the normal per-chat queue, and open PR #43512 remains an unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and have maintainers review, land, or replace PR #43512 with a current-main Feishu patch that routes the intended text control commands to a non-blocked queue lane while preserving normal per-chat serialization. The patch should add regression coverage proving `/stop`, `/status`, and `/new` execute while a same-chat Feishu task is active or queued, then this issue can close after merge.
