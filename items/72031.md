## Summary

Keep #72031 open. Current main at c9c0ab3a44904a2d2bc815799c1deabb270c5fdb still has the reported Bedrock aws-sdk media-understanding auth regression: image, MiniMax fallback, and audio/video execution auth still require a static API key before the provider call, while Bedrock docs and other runtime paths support aws-sdk credential-chain auth without apiKey. The related PR #72092 is the right canonical fix candidate but is open/unmerged in the provided context.

## Best Possible Solution

Keep #72031 open and land or adapt PR #72092: add a narrow missing-api-key allowance for auth.mode === "aws-sdk" in resolveImageRuntime, resolveMinimaxVlmFallbackRuntime, and resolveProviderExecutionAuth; avoid storing empty runtime keys; and add regression coverage for amazon-bedrock aws-sdk media-understanding calls with no static apiKey.
