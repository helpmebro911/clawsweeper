## Summary

Keep #72031 open. Current main still has the reported bug: media-understanding image and audio/video auth paths call `requireApiKey` unconditionally, while `requireApiKey` still throws for empty `apiKey` even when auth mode is `aws-sdk`. The chat/simple-completion path already has an allowlist for missing keys in `aws-sdk` mode, and the Bedrock docs describe `auth: "aws-sdk"` as requiring no `apiKey`. The linked PR #72092 appears to be the right pending fix, but it is open/unmerged in the provided GitHub context, so the issue should remain open until that or equivalent code lands on main.

## Best Possible Solution

Keep #72031 open and land or adapt PR #72092: add a narrow `aws-sdk` missing-api-key allowance to the media-understanding image, MiniMax fallback, and audio/video execution auth paths; avoid storing empty runtime keys; and keep regression tests for Bedrock `aws-sdk` auth with no static API key. Close the issue only after that behavior is present on main.
