## Summary

Keep #72031 open. Current main still has the exact media-understanding auth bug: the image and media runner paths call `requireApiKey` unconditionally, while `requireApiKey` still throws for empty `apiKey` even when `auth.mode` is `aws-sdk`. The linked #72092 is an open fix PR, not shipped on main, so closing the reporter issue before that lands would be premature.

## Best Possible Solution

Keep the issue open until #72092 or an equivalent main-branch patch lands. The best fix is to mirror the simple-completion auth behavior in media-understanding: allow an empty API key only for `aws-sdk` mode, avoid storing empty runtime keys, preserve existing errors for other auth modes, and add regression tests for the Bedrock image path plus the audio/video execution auth path.
