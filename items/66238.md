## Summary

Current main still only partially addresses this bug. The lean `status --json` path avoids plugin preload, but it intentionally disables `channelSummary`. The `tasks list --json` and `tasks audit --json` paths still have no route-first lightweight implementation and still load through the shared status/health/sessions/tasks registrar, whose top-level imports pull in unrelated command modules and task audit/maintenance code. The related #71739/#71746 work is present for `agents list --json`, but it is scoped to that command and does not supersede the tasks/status surfaces here.

## Best Possible Solution

Keep this issue open. The best fix is to add lightweight route-first JSON paths for `openclaw tasks list --json` and `openclaw tasks audit --json` or split the tasks registrar/command modules so those JSON paths do not import status/health/sessions or task maintenance/audit work unnecessarily. Separately, implement and test a lightweight `status --json` channelSummary strategy that avoids full plugin/runtime preload while preserving the documented JSON surface. Add import/load or startup-bench regression coverage for the tasks JSON paths.
