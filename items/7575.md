## Summary

Keep #7575 open. Current main still has no Sysbox/sysbox-runc implementation, no supported `agents.defaults.sandbox.docker.runtime` selector, and the documented Docker sandbox deployment remains Docker-socket based. The related issue discussion explicitly treats this as the canonical Sysbox/runtime architecture tracker rather than completed by the existing sandbox-common or Docker-socket path.

## Best Possible Solution

Keep #7575 open as the canonical Sysbox/runtime-architecture tracker. A good resolution would either add a conservative Docker/OCI runtime selector with types, strict schema, config resolution, Docker create `--runtime` wiring, docs, and tests, or explicitly move Sysbox to a plugin-backed sandbox backend built on `registerSandboxBackend` with public docs, ClawHub tracking, and any missing SDK seams proposed separately.
