## Summary

Keep open. Current main has stronger sandbox infrastructure and plugin-backed sandbox backends, but it still does not implement the requested Sysbox/sysbox-runc architecture, socketless Docker-in-Docker gateway mode, or a Docker runtime selector. The latest issue discussion also contains an explicit maintainer keep-open signal for this exact distinction.

## Best Possible Solution

Keep #7575 open as the canonical tracker for Sysbox/sysbox-runc socketless sandbox architecture until a maintainer decides the product direction. A good fix would either add a conservative Docker runtime option with schema/types/docs/tests and documented host setup, or publish/endorse a Sysbox sandbox backend plugin through ClawHub if the existing backend API is sufficient. Do not close it based only on sandbox-common or DooD improvements.
