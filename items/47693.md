## Summary

Keep PR #47693 open. Current main still lacks the PR's Slack DM drop-gate verbose logging, still leaves the batched `appMentionRetryKeys` cleanup limited to the last flushed entry, and still returns silently on empty resolved Slack content. The PR is not ready as-is because the provided review comment about guarding the empty-content `logVerbose()` call remains valid, and the branch likely needs a rebase onto the current plugin SDK import style.

## Best Possible Solution

Keep the PR open and ask for a small rework against current main: use the current plugin SDK runtime-env import path, add guarded verbose trace points for the Slack DM handler drop gates, clean up `appMentionRetryKeys` for every entry in a successfully dispatched batch while preserving the null-prepare retry behavior, and keep the proposed regression tests plus a guard test for verbose-off behavior.
