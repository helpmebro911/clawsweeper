## Summary

Keep #70630 open. Current main still has the Telegram no-visible-response path that synthesizes a `NO_REPLY` outbound plan and delivers the projected silent-reply fallback when it becomes visible. The current regression test still asserts that buggy DM behavior, while the PR’s provided diff changes that test to expect true silence and status reaction completion. The linked bug #70628 remains the same unresolved behavior rather than a duplicate/superseded item.

## Best Possible Solution

Keep the PR open for maintainer review or landing. The likely right product/code path is the one this PR proposes: suppress synthetic silent-reply fallback for Telegram direct-message no-visible-response turns, mark that path as a handled silent turn so status reactions finish as done, preserve fallback delivery for real failures or group cases where a projected fallback is non-empty, and update the Telegram dispatch regression test accordingly. If maintainers prefer not to merge this exact branch, they should apply an equivalent fix to `extensions/telegram/src/bot-message-dispatch.ts` and close #70628 with the resulting regression coverage.
