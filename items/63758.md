## Summary

Current main still leaves the reported gap open. The prior Codex review comment accurately identified that cron task rows are created and finalized, but the cron producer paths do not seed or update `progressSummary`; direct inspection against current main SHA `c2d82b87eee7a86bbd0e851169e0ca5559c44fe6` shows the same behavior remains.

## Best Possible Solution

Keep this issue open. Add cron-owned coarse progress summaries when scheduled, startup catch-up, and manual cron task runs are created; update them at meaningful execution phases through `recordTaskRunProgressByRunId`; and add focused regression coverage proving active cron work has live detail in `/tasks`, `/status`, or `session_status` before `terminalSummary` is written.
