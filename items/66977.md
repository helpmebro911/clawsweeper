## Summary

Keep open. Current main improves degraded behavior by storing embeddings and falling back to in-process cosine ranking, but the builtin sqlite-vec acceleration path still depends on node:sqlite loadable-extension APIs. I did not find a macOS-specific repair, alternate SQLite binding, or docs/workaround that resolves the reported native sqlite-vec load failure. The /opt/homebrew/bin workaround in the discussion is useful for QMD/PATH-based setups, but it does not prove builtin memory now loads sqlite-vec when node:sqlite cannot enable extension loading.

## Best Possible Solution

Keep this issue open until maintainers either validate and document a supported macOS Node/runtime combination where builtin sqlite-vec loads successfully, or change builtin memory to use a SQLite runtime/binding that supports loadable extensions on macOS. If native acceleration remains unsupported by design, document the macOS limitation and the Homebrew/QMD workaround while keeping the cosine path labeled as degraded fallback behavior.
