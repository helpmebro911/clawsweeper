## Summary

Keep open. Current main still requires `after_block_id` for `feishu_doc` insert and does not expose root-child `index` insertion, while this PR is a focused Feishu plugin implementation candidate with a small validation fix still needed.

## Best Possible Solution

Keep this PR open, or land an equivalent maintainer patch in `extensions/feishu`: make `after_block_id` optional only for `action: "insert"`, add `index: Type.Optional(Type.Integer({ minimum: 0 }))`, default missing `after_block_id` to root insertion at `index ?? 0`, preserve the existing after-block branch, pass the index through normal and batched Descendant insertion paths, and add targeted Feishu doc tool tests plus skill docs for the new mode.
