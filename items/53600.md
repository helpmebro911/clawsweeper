## Summary

Current main has broader diagnostics and partial mechanics for several named surfaces, but it still does not ship the requested native-VPS reasoning-latency analysis or an instrumentation plan covering every layer in the report.

## Best Possible Solution

Keep this open and scope it into a bounded native-Linux gateway performance investigation. Capture one trace/runbook covering bootstrap/context assembly timing, prompt/token estimation, memory query embedding/search, compaction trigger and model-call duration, queue waits, subagent spawn and announce-back, Telegram preview/final delivery, hook execution, auth fallback decisions, and process RSS/heap/GC. Then publish quick-win config guidance and split narrower optimization tickets for measured hot spots.
