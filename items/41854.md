## Summary

Current main has Groq metadata, docs, and a bundled Groq media-understanding plugin, but it does not yet provide the requested Groq text-inference provider catalog/onboarding path. The PR should not be closed as implemented, and it also should not be merged as-is because it is stale against the current plugin-owned provider architecture and includes security-sensitive baseline/allowlist churn unrelated to the feature.

## Best Possible Solution

Keep this PR open for maintainer/contributor rework, but do not merge the current diff. The best path is to rebuild it against current main as plugin-owned Groq provider work: add a Groq model provider catalog under extensions/groq, register it with defineSingleProviderPluginEntry or the equivalent provider plugin APIs, add providerAuthChoices/CLI flag metadata to extensions/groq/openclaw.plugin.json, cover interactive and non-interactive onboarding through the existing plugin auth-choice tests, and drop the unrelated .secrets.baseline/allowlist churn unless a separate secret-scanning fix is explicitly needed.
