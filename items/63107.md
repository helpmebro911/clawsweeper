## Summary

Keep open. Current main still lacks the requested pre-RPC CLI warning for shell-looking text passed through main-session cron system events, and #63112 is an open, unmerged same-author implementation candidate that explicitly links this issue with closing syntax.

## Best Possible Solution

Keep this issue open and review #63112. The best fix is a small shared cron CLI helper used by both cron add and cron edit to emit a non-blocking stderr warning before cron.add/cron.update when the resolved session target is main, the payload kind is systemEvent, and the text looks like a shell command, with focused CLI tests for warning and non-warning cases.
