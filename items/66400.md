## Summary

Keep this PR open. Current main still drops same-session TUI agent tool events when they arrive before the first chat event registers the run, and the PR is a focused implementation candidate whose second commit addresses the earlier review regressions.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate. The best path is to land this approach or an equivalent TUI fix that uses `evt.sessionKey` to accept same-session early agent events, preserves the `handleChatEvent` bookkeeping for `activeChatRunId`, `isLocalBtwRunId`, `pendingOptimisticUserMessage`, and `noteLocalRunId`, updates the `AgentEvent` type, and adds focused regression coverage for the gateway tool-first ordering.
