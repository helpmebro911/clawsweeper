## Summary

Keep PR #66400 open. Current main has a local embedded-backend mitigation for tool-first runs, but the central TUI handler path still drops unknown `agent` events before any `chat` event registers the run, and the TUI `AgentEvent` type still does not model the `sessionKey` that gateway agent events already carry.

## Best Possible Solution

Keep the PR open and land an equivalent handler-level fix if tests confirm the gateway/TUI race: add `sessionKey` to the TUI `AgentEvent`, pre-register same-session early agent events before the known-run filter, preserve the BTW guard and optimistic local-run bookkeeping from `handleChatEvent`, and add focused tests for same-session accept, cross-session reject, BTW non-hijack, and pending optimistic state cleanup.
