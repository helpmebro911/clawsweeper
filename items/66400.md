## Summary

Current main still has the reported TUI ordering bug: `handleAgentEvent` rejects unknown run IDs before consulting `sessionKey`, while only `handleChatEvent` registers same-session run IDs. The PR remains a focused implementation candidate, and its latest diff addresses the earlier bot-review concerns around BTW runs and optimistic pending state.

## Best Possible Solution

Keep this PR open as the implementation candidate. The best path is to land this approach or an equivalent TUI fix that adds `sessionKey` to `AgentEvent`, adopts same-session early agent events before the known-run filter, mirrors `handleChatEvent` bookkeeping for `activeChatRunId`, `isLocalBtwRunId`, `pendingOptimisticUserMessage`, and `noteLocalRunId`, and adds focused TUI handler regression tests.
