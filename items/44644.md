## Summary

Current `main` does not implement this PR's requested behavior. Provider onboarding still writes API-key auth profiles to exactly one `agentDir`, while sibling-agent sync exists only for OAuth helpers. The PR is not obsolete and should remain open for maintainer review or replacement.

## Evidence

- **Interactive plugin-provider onboarding writes one auth store:** `runProviderPluginAuthMethod` iterates returned profiles and calls `upsertAuthProfile` once per profile with the resolved `agentDir`; there is no sibling-agent expansion or `syncSiblingAgents` option in this path.
