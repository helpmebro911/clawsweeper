## Summary

Keep open. Current `main` has replaced the PR’s old onboarding files with plugin-based auth flows, so this patch is stale as written, but the underlying behavior is still not implemented: OAuth setup supports sibling-agent sync, while API-key onboarding still writes to only one agent directory in both interactive and non-interactive paths.
