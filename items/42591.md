## Summary

Keep this issue open. Current main still ships the macOS/Linux installer as one large `scripts/install.sh` entrypoint, and the requested `scripts/install/main.sh` plus `lib/`/`vendor/` split is not present. The issue is concrete and recent, with no protected-label or maintainer-author auto-close constraint.

## Best Possible Solution

Keep this issue open for maintainer-reviewed installer cleanup. A good implementation would preserve `scripts/install.sh` as the public curl-pipe entrypoint while extracting narrowly testable shell libraries for UI, download, platform detection, npm/git install, and build-tool logic, then update installer tests and docs to cover the new layout.
