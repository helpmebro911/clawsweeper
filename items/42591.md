## Summary

Keep open. Current main still has a monolithic `scripts/install.sh`, with the requested downloader, gum/UI, platform/build-tool, npm, git, and main-flow responsibilities inline. Docs, tests, and installer smoke workflow still treat `install.sh` as the public installer surface, so the requested maintainability refactor has not shipped.

## Best Possible Solution

Keep this issue open for a conservative installer refactor. Preserve `scripts/install.sh` as the public entrypoint, extract narrowly scoped sourced modules for UI/logging, downloads/checksums, gum bootstrap, platform/package-manager detection, build tools, npm install/repair, git install, and main argument flow, then update installer docs plus the direct shell/unit/smoke coverage around the new layout.
