## Summary

Keep #17424 open. Current main still does not implement the requested `agents.defaults.memorySearch.dimensions` public config key, and OpenAI/OpenAI-compatible memory embedding requests still omit the `dimensions` field. The existing issue discussion already identified this gap, and the current checkout at e672b614 still matches that assessment.

## Best Possible Solution

Keep #17424 open. Maintainers should choose whether the public config should be `dimensions`, `outputDimensionality`, or an alias pair, then update schema/types/help/docs, defaults and per-agent merge logic, OpenAI-compatible inline and batch request bodies, provider cache keys, and reindex identity. Add focused schema, resolver, request-body, batch, and cache/reindex tests.

## Risks / Open Questions

- Some OpenAI-compatible embedding providers may reject a `dimensions` field, so the implementation should validate or gate it by provider/model capability.
- Changing embedding dimensions must participate in provider cache keys and reindex identity to avoid mixing vectors with incompatible sizes.
