## Summary

Keep #17424 open. Current main still only supports provider-specific `outputDimensionality` plumbing for Gemini/Bedrock-style providers; it does not support the requested public `agents.defaults.memorySearch.dimensions` key, and OpenAI/OpenAI-compatible inline and batch embedding requests still omit the `dimensions` request field. The existing April 26, 2026 issue comment already identified this gap, and main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still matches that assessment.

## Best Possible Solution

Keep #17424 open. Maintainers should decide whether the public config should be `dimensions`, `outputDimensionality`, or an alias pair, then thread the resolved value through schema/types, defaults and per-agent overrides, OpenAI/OpenAI-compatible inline requests, OpenAI batch JSONL bodies, runtime cache keys, and reindex identity. Add focused schema, resolver, request-body, batch, and cache/reindex tests.
