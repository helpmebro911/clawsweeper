## Summary

Keep open. Current main still skips top-level symlinked plugin directories during auto-discovery, and the PR discussion contains a concrete user confirmation plus proposed regression tests. The PR should not be landed as-is because its head includes unrelated files, but the underlying bug remains real and unshipped.

## Best Possible Solution

Keep the PR open and ask for a cleaned revision limited to src/plugins/discovery.ts plus focused regression tests in src/plugins/discovery.test.ts for symlinked plugin directories and broken symlinks. Alternatively, maintainers can cherry-pick the narrow fix and Dithilli's proposed tests into a canonical PR, then explicitly supersede this one after that lands.
