## Summary

Keep #11489 open. Current main still has DM pairing only; group policy remains open/disabled/allowlist, Telegram and WhatsApp group paths block or allow messages without an owner approval queue, and the pairing CLI has no group chat approve/reject semantics. Related #8495 and #42400 are narrower open items and do not supersede the requested cross-channel group pairing flow.

## Best Possible Solution

Keep #11489 open for maintainer/product direction. The likely path is an additive channel/core contract change: extend group policy/config/schema/docs, add pending group approval state keyed by channel/account/group ID, add CLI approve/reject semantics, teach Telegram/WhatsApp join or first-seen group paths to notify the owner, and decide whether pending read-only observation should be implemented via the #42400-style no-reply work or a separate restricted runtime route.
