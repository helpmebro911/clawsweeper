## Summary

Keep #70440 open. Current main has adjacent incomplete-turn suppressions, but it still does not provide the requested global or per-channel policy to silently swallow empty-turn fallback errors for direct/TTS channels. The maintainer discussion explicitly kept this request open after the #70396/#70623 false-positive fix, and source/docs confirm the shipped behavior is narrower: committed messaging-tool delivery is suppressed, and clean empty/reasoning-only turns can be treated as silent only for group chats where silent replies are allowed. Direct chats, including the reported Signal/TTS shape, still stay on the failed-turn/error path by design.

## Best Possible Solution

Keep #70440 open as a core configuration/product request. The likely path is an explicit, documented empty/incomplete-turn delivery policy under agent defaults and/or per channel/surface, defaulting to current `error` behavior, with a `silent` mode that logs WARN server-side and suppresses channel delivery for direct TTS/voice-note contexts. The implementation should update config types, Zod schema/help/docs, the embedded runner fallback path, Signal or generic TTS routing tests, and preserve the existing #70396 committed-delivery suppression.
