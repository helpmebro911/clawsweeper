## Summary

Keep this issue open. Current main still lacks a global or per-channel incomplete-turn delivery policy; direct/TTS-capable channels can still receive the visible "Agent couldn't generate a response" fallback for genuine empty turns.

## Best Possible Solution

Keep the issue open and implement a first-class empty-turn delivery policy in the embedded runner/reply-run configuration path. It should default to current visible error behavior, support global and per-surface or per-channel silent overrides for direct/TTS surfaces such as Signal, preserve WARN/server logging, and add regression coverage for direct chats, groups, genuine empty turns, reasoning-only turns, message-tool side effects, and existing NO_REPLY semantics.
