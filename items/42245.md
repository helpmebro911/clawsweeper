## Summary

Keep this PR open. Current main still accepts `openclaw cron add --message ...` without `--agent`, forwards `agentId: undefined`, and has no missing-agent warning or focused regression coverage. The PR remains a narrow implementation candidate for the linked cron UX issue; the adjacent cron-tool agent inference work is a different tool-created-job path and does not replace the CLI warning.

## Best Possible Solution

Keep this PR open as a focused implementation candidate. Refresh it on current main, add the warning in `src/cli/cron-cli/register.cron-add.ts` after payload classification so it only applies to `agentTurn` jobs, emit on stderr to preserve JSON stdout, keep wording accurate for configured default agents, and add focused tests in `src/cli/cron-cli.test.ts` for missing, blank, explicit-agent, session-key, and system-event cases.
