## Summary

Keep this PR open. Current main still does not warn when `openclaw cron add --message` omits `--agent`, and the runtime still falls back to the configured default agent. The PR is a focused candidate for #42196, not superseded by the adjacent session-agent inference work in #72326.

## Best Possible Solution

Keep this PR open as a candidate fix for #42196. Refresh it on current main, keep the warning scoped to `cron add` agent-turn jobs, emit it on stderr so JSON stdout stays parseable, add regression coverage for missing, blank, explicit-agent, session-key, and system-event cases, and make the warning wording consistent with configured-default-agent behavior.
