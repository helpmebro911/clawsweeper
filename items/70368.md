## Summary

Keep this PR open. Current main still has the reported behavior, so the PR is not obsolete or implemented. The current PR head is also not clearly safe to close as low-signal: it is a real implementation candidate for #70357, but its latest effective-bind classification needs maintainer review because `auto` and `custom` can fall back to non-loopback runtime binds.

## Best Possible Solution

Keep this PR open and request a corrected, maintainer-reviewed implementation. The safest path is to downgrade only definitely local-only exposure, fail closed for ambiguous `auto` or fallback-prone `custom` binds, keep sibling gateway audit checks semantically coherent, add regression coverage for loopback, lan, custom loopback, custom exposed, and auto/container or fallback behavior, and update the public audit-check docs if the severity taxonomy changes.
