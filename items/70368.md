## Summary

Keep open. Current main does not implement the requested audit behavior: `gateway.trusted_proxies_missing` is still emitted only for literal `gateway.bind="loopback"` and still at `warn`, while the regression test and audit-check docs still encode that behavior. The PR also is not safely obsolete or ready to close as duplicate: the duplicate PR #70377 is closed unmerged, the linked issue #70357 is closed, but the current main code path remains unchanged. The latest PR discussion raises real audit-classification concerns around effective bind handling, especially `auto`, so the best maintainer path is to keep the PR open for a corrected/focused implementation rather than auto-close it.

## Best Possible Solution

Keep the PR open for maintainer review and ask for a narrower, fail-closed implementation in `src/security/audit-gateway-config.ts`: derive severity from an effective-bind classifier that matches current runtime/config validation, treats uncertain `auto` cases conservatively, preserves critical checks for exposed listeners, and adds regression tests for loopback info, custom loopback, lan/tailnet/custom warn, and auto/container or fallback behavior. Update the public audit-check docs once the intended severity taxonomy is accepted.
