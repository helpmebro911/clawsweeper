## Summary

Keep #70368 open. Current main still emits `gateway.trusted_proxies_missing` as `warn` for literal loopback binds, the tests and docs still lock in that behavior, and the PR discussion shows unresolved security-scope questions in the latest revision rather than a clean obsolete change.

## Best Possible Solution

Keep the PR open and ask for a revised, security-conservative implementation. The best path is to base `gateway.trusted_proxies_missing` on effective gateway exposure without hiding `bind_no_auth` or Control UI origin findings, handle `loopback`, `lan`, custom loopback, custom fallback, and `auto` container/fallback cases explicitly, and add regression tests for each branch. Splitting the sibling audit-check classifier changes from the narrow severity downgrade may make maintainer review easier.
