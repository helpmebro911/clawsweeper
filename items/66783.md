## Summary

Keep open. Current main already fixes the immediate Talk mode microphone regression by allowing same-origin microphone in the default gateway Permissions-Policy header, but it does not implement this PR's main capability: configurable `gateway.http.securityHeaders.permissionsPolicy` with override and explicit-disable behavior.

## Best Possible Solution

Keep this PR open for maintainer review or contributor follow-up. The best path is to rebase onto current main, keep or explicitly re-decide the shipped `camera=(), microphone=(self), geolocation=()` default, add `gateway.http.securityHeaders.permissionsPolicy?: string | false` through config types, zod/generated schema metadata, docs, runtime resolution, and `createGatewayHttpServer`, and keep focused tests for default, override, and explicit-disable behavior. Remove or split any unrelated auth-resolution or schema metadata changes before merge.
