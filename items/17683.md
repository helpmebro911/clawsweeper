## Summary

Keep open. Current main has useful building blocks for this request, including isolated cron runs, lightweight context, per-job tool allowlists, global tool policy, run usage logging, and documented LiteLLM spend-limit workarounds. But the requested core capability still has meaningful missing pieces: native per-job token caps, daily spend ceilings, graceful budget aborts, an admin-defined script-only mode, and dry-run execution for scoped jobs are not implemented as a first-class mode.
