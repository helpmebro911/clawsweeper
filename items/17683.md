## Summary

Keep #17683 open. Current main still has only partial controls for scoped automation: cron/hooks support isolated runs, light context, timeouts, tool allow-lists, model overrides, and post-run telemetry, but not the requested first-class script-limited mode, per-job/daily budget enforcement, budget aborts, dry-run cost previews, or planned-vs-actual run logs.

## Best Possible Solution

Keep #17683 open as the product/API tracker. The best implementation is a native scoped automation mode spanning cron, /hooks/agent, gateway protocol/schema, CLI, embedded runner accounting, docs, and tests: per-job token caps, daily spend ceilings, graceful budget aborts, admin-defined script-only execution, dry-run cost previews, and structured planned-vs-actual run logs, with #38248 treated as a narrower cost-guard component.

## Risks / Open Questions

- Existing toolsAllow, isolated sessions, lightContext, timeouts, contextTokens, and run telemetry may satisfy some narrow cost-reduction workflows, but they do not satisfy the requested native scoped/script-limited mode.
- Related #38248 is an open, narrower spending-ceiling issue; it may cover part of the budget-enforcement work but does not supersede script-only execution, cron-first deterministic runs, dry-run previews, or planned-vs-actual logging.
