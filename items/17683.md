## Summary

Keep #17683 open. Current main still only has partial controls for cron/hooks/agent turns: isolated sessions, light context, timeouts, tool allow-lists, and post-run usage/cost telemetry. It does not provide the requested first-class scoped/script-limited mode with per-job token caps, daily spend ceilings, budget aborts, admin-defined script-only execution, dry-run cost previews, or planned-vs-actual run logs.

## Best Possible Solution

Keep #17683 open as the product/API tracker. The best path is a native scoped automation mode covering cron, hooks, gateway protocol, CLI, embedded runner accounting, docs, and tests: per-job token caps, daily spend ceilings, graceful budget aborts, admin-defined script-only execution, dry-run/plan cost preview, and structured planned-vs-actual run logs.

## Risks / Open Questions

- Existing toolsAllow, isolated sessions, lightContext, timeouts, contextTokens, maxTokens, and usage telemetry reduce scope or improve visibility, but they do not satisfy the native scoped/script-limited mode requested here.
- Related cost-guard work such as #38248 may cover part of the spend-ceiling problem, but #17683 is broader because it also asks for cron-first deterministic runs, script-only execution, dry-run previews, and planned-vs-actual logging.
- A complete fix likely crosses cron schema, hooks, gateway protocol, CLI, embedded runner accounting, docs, and tests; splitting budget enforcement, script-only execution, and preview/logging into narrower implementation tasks would reduce risk.
