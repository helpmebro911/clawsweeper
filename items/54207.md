## Summary

Keep open. Current main still lets the Docker sandbox bind source be the gateway-container path for the default sandbox workspace in Docker-out-of-Docker setups. The docs now warn about host-path parity, but the bundled Compose/setup path still maps host storage to `/home/node/.openclaw`, defaults sandbox workspaces under that container-visible state dir, and passes the resulting `workspaceDir` directly to `docker create` without host-path translation.

## Best Possible Solution

Keep this open and fix the Docker gateway sandbox path contract. The likely maintainer path is to translate container-visible sandbox/workspace paths back to host-visible `OPENCLAW_CONFIG_DIR`/`OPENCLAW_WORKSPACE_DIR` before `docker create`, or have Docker setup configure a host-visible `agents.defaults.sandbox.workspaceRoot` with an identical gateway-visible parity mount. Add regression coverage that Docker gateway sandbox creation does not emit `/home/node/.openclaw/sandboxes/...` as a host Docker bind source for Docker Desktop macOS.
