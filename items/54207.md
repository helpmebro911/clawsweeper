## Summary

Keep this issue open. Current main still has the reported Docker-out-of-Docker path mismatch: the Docker gateway maps host state into `/home/node/.openclaw`, the sandbox workspace root defaults under the gateway-visible state dir, and the Docker backend passes that resolved container path directly as the bind source. The newer sandbox docs warn about host-path parity, but the Docker setup/runtime path still does not translate `/home/node/.openclaw/sandboxes/...` back to `OPENCLAW_CONFIG_DIR` or configure a default identical host/container mount. The related closed #40692 does not supersede this because that report involved an unsupported named-volume setup, while this issue follows the supported host-bind Docker Compose flow.

## Best Possible Solution

Keep the issue open and fix the Docker gateway sandbox path contract. The likely maintainer path is to either translate container-visible state paths back to the host `OPENCLAW_CONFIG_DIR`/workspace paths before `docker create`, or update Docker setup to configure a host-visible `agents.defaults.sandbox.workspaceRoot` plus an identical bind mount that the gateway can access. Add regression coverage for the Docker gateway flow so `/home/node/.openclaw/sandboxes/...` is not emitted as a host Docker bind source on Docker Desktop macOS.
