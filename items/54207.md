## Summary

Current main still appears to have the Docker-out-of-Docker sandbox bind-source bug. The official Docker gateway path maps host storage to `/home/node/.openclaw`, the default sandbox workspace root resolves under the gateway container home, and the Docker sandbox creation path passes that resolved `workspaceDir` directly as the `docker create -v` source without translating it back to a host-visible path. The prior ClawSweeper comment reached the same conclusion, and no current source, test, or docs change on main proves a runtime fix.

## Best Possible Solution

Keep this issue open and fix the Docker gateway sandbox path contract. The likely maintainer fix is to either translate container-visible sandbox/workspace paths back to host-visible `OPENCLAW_CONFIG_DIR`/`OPENCLAW_WORKSPACE_DIR` before Docker creation, or have Docker setup configure a host-visible `agents.defaults.sandbox.workspaceRoot` plus an identical gateway-visible parity mount. Add regression coverage proving Docker gateway sandbox creation does not emit `/home/node/.openclaw/sandboxes/...` as the Docker daemon bind source for the default Docker Desktop/macOS setup.
