## Summary

Current `main` does not implement the requested connection-level `health` fields. The `health` payload is still service/channel/session-oriented, and the only in-repo connection bit for UI clients is local WebSocket state, not RPC-returned `connected`/`rttMs`/`lastHeartbeatAt`.

## Evidence

- **HealthSummary shape lacks requested fields:** `HealthSummary` defines `ok`, `ts`, `durationMs`, channel data, agent data, and session summaries only; there is no top-level `connected`, `rttMs`, or `lastHeartbeatAt`.
  - file: [src/commands/health.types.ts:23](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/commands/health.types.ts#L23)
  - command: `nl -ba src/commands/health.types.ts | sed -n '1,220p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **Health snapshot builder does not compute connection metrics:** `getHealthSnapshot()` assembles channel probes, heartbeat cadence, agents, and sessions, then returns the summary without any connection-level transport fields.
  - file: [src/commands/health.ts:376](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/commands/health.ts#L376)
  - command: `nl -ba src/commands/health.ts | sed -n '215,430p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **Public docs still describe health as service-level summary:** Gateway health docs say the snapshot includes `ok`, `ts`, `durationMs`, per-channel status, agent availability, and session-store summary. The protocol docs describe `health` generically as the gateway health snapshot; neither doc mentions connection RTT/heartbeat reachability fields.
  - file: [docs/gateway/health.md:61](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/docs/gateway/health.md#L61)
  - command: `nl -ba docs/gateway/health.md | sed -n '1,160p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **UI connection state is client-local, not part of health RPC:** The browser Gateway client exposes `connected` by checking `WebSocket.OPEN`. That supports the issue's claim that a UI currently needs separate connection-state handling instead of getting it from `health`.
  - file: [ui/src/ui/gateway.ts:323](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/ui/src/ui/gateway.ts#L323)
