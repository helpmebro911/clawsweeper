## Summary

Keep this PR open. Current main still lacks the bundled-channel-entry-specific diagnostic and field-level bundled-entry validation logging, so the PR is not obsolete or implemented on main. The branch should not land as-is because the provided PR diff/review context shows it removes the existing debug module-shape diagnostic for ordinary missing-register plugin failures.

## Best Possible Solution

Keep the PR open and request a narrow revision: preserve `formatMissingPluginRegisterError` for normal plugin missing-register failures, detect `bundled-channel-entry` using the already-resolved record kind or a small type guard, keep the bundled-entry missing-field logging improvement, and add regression tests for both the bundled-channel diagnostic and the existing debug module-shape behavior.
