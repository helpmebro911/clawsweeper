## Summary

Keep open. Current main does not already implement this PR's bundled-channel-entry-specific diagnostics: malformed bundled channel entries still collapse to a generic bundled contract warning, and the legacy plugin loader still reports the generic missing register/activate error. The PR is also not cleanly superseded by #67380, which is closed but unmerged. However, the branch as described still needs maintainer review because it would drop the documented OPENCLAW_PLUGIN_LOAD_DEBUG module-shape details for ordinary missing-register plugin failures.

## Best Possible Solution

Keep the PR open for a narrow revision: add bundled-channel-entry-specific diagnostics in both full and CLI registry loader paths, preserve formatMissingPluginRegisterError(mod, env) for ordinary plugins, use the already-normalized record.kind/hasKind path instead of re-reading resolved.definition, and add focused regression coverage for the wrong-loader bundled-channel-entry case plus field-specific bundled entry validation logging.
