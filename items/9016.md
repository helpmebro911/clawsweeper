## Summary

Keep #9016 open. Current main has estimated cost surfaces and transcript aggregation, but it still does not capture OpenRouter's provider-reported `usage.cost`, `cost_details.upstream_inference_cost`, or `prompt_tokens_details.cache_write_tokens` from OpenAI-compatible responses and carry that exact per-message cost into agent runtime/session metadata.

## Best Possible Solution

Keep #9016 open. The best implementation is to capture OpenRouter raw usage at the shared OpenAI-compatible response/stream boundary, preserve provider-reported cost separately from OpenClaw's local pricing estimate, include cache-write tokens and typed provider cost details in transcript/session usage and `EmbeddedPiAgentMeta`, then expose per-turn and cumulative values through `session_status` or a dedicated usage/cost tool. Telegram and other reply footers can render exact OpenRouter cost when available and fall back to existing estimates otherwise.
