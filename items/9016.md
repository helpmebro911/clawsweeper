## Summary

Keep open. Current main has estimated cost reporting, transcript-backed cost aggregation, and `/usage`/`session_status` cost surfaces, but it still does not capture OpenRouter's provider-reported `usage.cost` / `usage.cost_details` from streaming responses or expose that exact per-message cost to the agent runtime. The ClawRouter comment is a useful external workaround, and #7006/#9696 are related, but neither supersedes this exact OpenRouter provider-reported cost request.

## Best Possible Solution

Keep #9016 open until OpenClaw captures OpenRouter `usage.cost` and `usage.cost_details` at the OpenAI-compatible streaming transport boundary, persists that provider-reported cost in normalized usage/transcripts, and exposes it through a structured agent-runtime surface such as `session_status` details or a dedicated usage/cost tool. The existing estimated-cost and Control UI analytics work should remain, but should not be treated as closing the exact provider-reported OpenRouter cost request.
