## Summary

Keep this issue open. Current main still rejects `sk-ant-api03-...` on the reported `openclaw models auth paste-token --provider anthropic` path, and the linked implementation candidate PR #72124 is open and unmerged.

## Best Possible Solution

Review and land PR #72124 or an equivalent patch that widens the Anthropic paste-token validator to accept both setup tokens and standard Anthropic API keys, adds focused validator/CLI regression tests for both prefixes, and updates CLI/docs wording if maintainers decide `paste-token` should stay token-only.
