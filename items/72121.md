## Summary

Keep open. Current main still rejects `sk-ant-api03-...` on the reported `models auth paste-token --provider anthropic` path, and #72124 is an open implementation candidate with closing syntax rather than a merged fix.

## Best Possible Solution

Review and land #72124 or an equivalent patch that widens the shared Anthropic token validator for the paste-token path to accept both `sk-ant-oat01-` setup tokens and `sk-ant-api03-` API keys, with focused regression tests against the real validator/call path. If maintainers want API keys to remain on the separate `api-key` auth method, update the CLI error/help/docs to make that split explicit instead.
