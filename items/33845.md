## Summary

Keep PR #33845 open. Current main still keeps Slack thread participation in a process-global in-memory dedupe cache, while Slack mention gating still relies on that cache to permit non-mention replies in bot-participated threads. The PR is open/unmerged in the provided GitHub context, and the related open PRs do not provide a landed canonical replacement.

## Best Possible Solution

Keep #33845 open and request maintainer-directed revisions or supersede it only after a canonical landed replacement exists. The likely implementation path is to migrate Slack thread participation to the existing `openclaw/plugin-sdk/persistent-dedupe` helper, or to the accepted persistent keyed-store helper if #70626 becomes the chosen contract, with regression tests for restart round-trip, TTL expiry, missing/corrupt state, clear semantics, max-entry behavior, and split module instances.
