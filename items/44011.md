## Summary

Keep this PR open. Current main partially overlaps by exposing sessionKey and other correlation fields on typed message hooks, but it still does not expose agentId in PluginHookMessageContext, does not derive or pass inbound agentId into message_received context, and does not thread outbound mirror/session identity into message_sending or message_sent typed plugin context.

## Best Possible Solution

Keep this PR open as a valid implementation candidate. The best path is to update it against current main by adding optional agentId to the current message hook type module and canonical mapper types, deriving/passing inbound agentId from the resolved session, resolving outbound mirror/session identity once, threading sessionKey and agentId into both message_sending and message_sent typed contexts, adding session-only and mirror-wins tests, and documenting ctx.agentId in the public plugin hook docs.
