## Summary

Keep open. Current main does not implement a per-call `tools` parameter for `sessions_spawn`; child tool scope is still derived from config-level subagent policy and agent policy, so the request remains a live enhancement rather than something already fixed or obsolete.

## Evidence

- **sessions_spawn schema has no tools field:** `SessionsSpawnToolSchema` defines `task`, `label`, `runtime`, `agentId`, `resumeSessionId`, `model`, `thinking`, `cwd`, timeout fields, `thread`, `mode`, `cleanup`, `sandbox`, `context`, `streamTo`, `lightContext`, `attachments`, and `attachAs`. There is no `tools` property in the current tool input schema.
