## Summary

Keep this PR open. Current main still uses and tests numeric config backup rotation, while this PR is a focused implementation candidate paired with open same-author feature issue #39923. The overlapping open PR #39982 is not merged or clearly maintainer-selected as the canonical replacement, and this PR still needs maintainer judgment on legacy `.bak.N` preservation/migration and collision-safe timestamp names before it should merge or close.

## Best Possible Solution

Keep this PR open for maintainer review alongside #39923 and #39982. ## Risks / Open Questions

- The PR's proposed cleanup treats legacy numeric `.bak.N` files as orphans, which can delete existing rollback snapshots on the first upgraded config write before the datetime backup set is fully populated.
- The PR's timestamp format uses millisecond precision but still has no collision fallback; same-millisecond writes can overwrite or drop one backup slot depending on platform rename behavior.
- There are two open implementation candidates for #39923, so maintainers still need to choose a canonical direction or request a combined revision before closing either contributor path.
