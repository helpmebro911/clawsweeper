## Summary

Keep this PR open. Current main still uses numeric config backup rotation and tests that behavior as the contract. This PR is a focused implementation candidate paired with open same-author issue #39923, while overlapping PR #39982 is also open and not maintainer-selected as canonical. Security review found no supply-chain or infrastructure surface in the provided diff, but functional review still leaves legacy backup migration and timestamp collision behavior for maintainer review.

## Best Possible Solution

Keep this PR open for maintainer review alongside #39923 and #39982. The best implementation path is to select one datetime-backup approach in src/config/backup-rotation.ts, preserve or safely migrate legacy .bak.N files until retention is established, add collision-safe timestamp destinations, keep bounded retention, update src/config/config.backup-rotation.test.ts, and add docs or changelog only if the selected behavior ships.

## Risks / Open Questions

- The PR would delete existing legacy numeric .bak.N rollback snapshots on the first upgraded config write unless migration/preservation policy changes before merge.
- The PR documents but does not prevent same-millisecond timestamp collisions, so burst writes can overwrite or drop one historical backup slot.
- There are two open implementation candidates for #39923, so maintainers still need to choose a canonical direction or request a combined revision.
