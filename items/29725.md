## Summary

Current `main` (c6684af682bcb59bd1ee85cb49f72d4074007e2c) does have Discord presence mutation logic, but it is exposed through the Discord message-action path (`set-presence`) and internal gateway runtime, not through a public plugin SDK surface like `api.runtime.channel.discord.updatePresence()`. The public `api.runtime.channel` contract still only documents and types generic helpers such as `mentions`, so the issue's requested plugin-runtime seam is not implemented on `main`.

## Evidence

- **Public runtime docs lack a Discord-specific channel helper:** `docs/plugins/sdk-runtime.md` documents `api.runtime.channel` as shared channel helpers and only lists `mentions`; there is no documented `discord` namespace or `updatePresence()` helper.
  - file: [docs/plugins/sdk-runtime.md:345](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/plugins/sdk-runtime.md#L345)

- **Public runtime type is intentionally generic:** `PluginRuntimeChannel` is explicitly described as generic core helpers only, and its exported shape contains `text`, `reply`, `routing`, `mentions`, etc., but no `discord` slot for presence control.
