## Summary

Keep open. Current main still builds the webchat session dropdown from stored session rows plus the active session key, while `agents.list` is only used for labeling already-present agent groups. The PR is stale against a refactored UI path and has a real global-scope correctness issue to address, but it remains a focused implementation candidate with useful tests rather than conservative close material.

## Best Possible Solution

Keep the PR open or ask the contributor to refresh it against `ui/src/ui/chat/session-controls.ts`. The best implementation is to merge configured `agents.list` entries into `resolveSessionOptionGroups()`, synthesize only session keys that the gateway can route correctly for the reported scope/main-key contract, dedupe against stored rows and the active key, keep labels stable, and add regression tests for session-less configured agents, custom main keys, and global-scope behavior. Maintainers may consolidate this with #48671 if they prefer one refreshed PR, but the linked product bug #41933 is not implemented on current main.
