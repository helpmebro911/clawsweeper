## Summary

Keep #41955 open. Current main still does not implement the requested Control UI behavior: the chat session dropdown is built from persisted session rows plus the active session, while `agents.list` is only used to label existing agent groups. The PR is stale against a refactored code path and has a known global-session-scope correctness concern, but the linked user problem in #41933 remains real and unshipped, so this is not conservative close material.

## Best Possible Solution

Keep the PR open or ask the contributor to refresh it against `ui/src/ui/chat/session-controls.ts`. The best fix is to merge configured `agents.list` entries into `resolveSessionOptionGroups()` in a scope-aware way, dedupe against real session rows, preserve stable agent labels, and add regression coverage for configured agents without sessions plus global/custom main-key cases.
