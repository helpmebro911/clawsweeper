## Summary

Keep this PR open. Current main still does not add configured agents that lack session-store rows to the webchat session dropdown, and the linked bug remains unsolved. The submitted patch is stale because the dropdown resolver moved to the grouped session-control path, but the PR is still a focused implementation candidate or can be consolidated with related PR #48671.

## Best Possible Solution

Keep this PR open or consolidate it with #48671. The best fix is to update `resolveSessionOptionGroups()` to merge `state.agentsList.agents` into the grouped dropdown with scope-aware key synthesis, dedupe against stored rows and the active key, preserve configured labels, and add regression tests for session-less configured agents, custom `mainKey`, and global-scope behavior.
