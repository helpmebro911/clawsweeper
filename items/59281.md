## Summary

Keep open. The issue is recent, author is not protected, labels are not protected, and the current loader code still contains the Windows native-jiti guard that keeps Windows off the native import path discussed in the report. The report is concrete and has follow-on related issue references, so it is not suitable for stale, incoherent, cannot-reproduce, clawhub, or implemented-on-main closure.

## Evidence

- **Current main still disables native jiti on Windows:** `supportsNativeJitiRuntime()` returns false when `process.platform === "win32"`, and `shouldPreferNativeJiti()` immediately returns false when that helper is false. This directly matches the reported Windows path staying on the non-native jiti loader path. Source observed via GitHub main view. ([github.com](https://github.com/openclaw/openclaw/blob/main/src/plugins/sdk-alias.ts))
  - file: [src/plugins/sdk-alias.ts:2899](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/src/plugins/sdk-alias.ts#L2899)
  - command: `web open/find github.com/openclaw/openclaw/blob/main/src/plugins/sdk-alias.ts supportsNativeJitiRuntime`
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Issue has concrete reproduction and measured impact:** The issue body lists affected production call sites and Windows timings, including multi-minute jiti imports for anthropic and memory-core versus sub-second to low-second native-import timings. ([github.com](https://github.com/openclaw/openclaw/issues/59281))
  - command: `provided GitHub context; web open issue #59281`
- **Comment reinforces unresolved Windows loader behavior:** A commenter proposed removing the Windows guard from `supportsNativeJitiRuntime()` so Windows can use native import mode, and reported bundled plugin loading dropping from 70-90s to 18s with that local change. This is useful supporting discussion, not evidence that main already fixed it.
  - command: `provided GitHub context for issue #59281 comment 4211474008`
- **Related open reports suggest active loader fallout:** The timeline links later open issues #69103, #71233, and #71495. At least #71233 also concerns jiti plugin loader behavior on recent Linux releases, which makes a conservative maintainer close inappropriate without a fix or reproduction proof.
