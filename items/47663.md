## Summary

Current main does not implement the PR's requested opt-out. The reminder guard still appends `UNSCHEDULED_REMINDER_NOTE` when the commitment regex matches, no cron add occurred, and no enabled same-session cron exists. The accepted config contract also lacks `messages.unscheduledReminderNote`, so the proposed user-facing flag is not shipped under that name or an equivalent alias. The linked issue #30787 was stale-closed, but the underlying opt-out request remains unresolved and this PR is still a plausible implementation path.

## Best Possible Solution

Keep the PR open for maintainer review because the central user problem remains on main. The best path is to decide the product shape for the annotation first, then either merge an opt-out flag or a more distinct system-annotation delivery; if using the flag, update the full config contract and docs, add a direct wiring test for `runReplyAgent`, and retain the PR's short-circuit optimization for cron-store reads when disabled.
