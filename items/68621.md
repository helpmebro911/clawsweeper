## Summary

Keep open. The PR is still useful because current main does not expose `authProfileOverride` on `before_model_resolve` / legacy `before_agent_start` hook results and does not feed any hook-selected auth profile into embedded-runner auth-profile ordering. The PR discussion also describes a concrete same-provider multi-account routing use case plus local/live validation, and the related items are adjacent auth-profile rotation/discovery requests rather than a canonical replacement for this hook API change.

## Best Possible Solution

Keep the PR open for maintainer review and likely merge after normal validation. The best path is a small core hook API extension: add `authProfileOverride` to the hook result contract, merge it with the same priority semantics as provider/model overrides, pass it into existing auth-profile ordering only when the current profile was not explicitly user-pinned, and keep docs/schema/tests aligned.
