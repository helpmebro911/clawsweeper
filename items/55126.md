## Summary

Current main still does not thread `memorySearch.outputDimensionality` into OpenAI embedding `dimensions` for inline or batch requests. The related implementation candidate #55507 remains open and unmerged with closing syntax for this issue, so this issue should stay open until that PR or a replacement patch lands.

## Best Possible Solution

Keep this issue open. Review and update #55507, or replace it with a current-main patch that copies `memorySearch.outputDimensionality` into the OpenAI embedding client, sends OpenAI `dimensions` for both inline and batch `/v1/embeddings` requests when supported, includes the dimension in provider cache/reindex identity, and adds regression tests for inline request bodies, batch JSONL bodies, and cache-key behavior.
