## Summary

Current main still has the reported OpenAI memory embeddings gap. The config and memory-core provider request carry `memorySearch.outputDimensionality`, but the bundled OpenAI provider/client and shared remote request path do not turn it into OpenAI's `dimensions` field; the OpenAI batch path also omits it. #55507 is an open, unmerged implementation candidate with closing syntax for this issue, so the issue should remain open until that PR is reviewed and landed or closed.

## Best Possible Solution

Keep this issue open and review #55507 as the implementation candidate. The fix should preserve `outputDimensionality` on the OpenAI embedding client, send it as OpenAI `dimensions` for inline and batch embeddings when supported, include it in OpenAI runtime cache-key data so reindex decisions change when dimensions change, and add regression tests covering request bodies and cache-key behavior.
