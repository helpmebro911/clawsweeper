## Summary

Keep #45784 open. The reported bug is still real on current main, but the PR's current implementation is not safe to close as implemented or merge as-is because it changes the established OPENCLAW_HOME contract for general home roots.

## Best Possible Solution

Keep the PR open pending author or maintainer revision. The best fix is a narrow resolver change, shared by src/utils.ts and src/config/paths.ts or centralized in the home/path helper, that uses OPENCLAW_HOME directly only when the resolved value already denotes the .openclaw state/config directory, preserves ${OPENCLAW_HOME}/.openclaw for ordinary home roots, and adds regression tests for both cases plus affected config/.env/plugin-root consumers.

## Evidence

Since resolveRequiredHomeDir treats OPENCLAW_HOME as the effective home, OPENCLAW_HOME=/home/user/.openclaw still resolves to /home/user/.openclaw/.openclaw on current main.
