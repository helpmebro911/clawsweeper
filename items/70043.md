## Summary

Keep #70043 open. Current main still defaults an unset Telegram `reactionLevel` to `minimal`, passes that through the Telegram channel prompt adapter, and injects the exact `Reactions are enabled for Telegram in MINIMAL mode.` system-prompt text. This is not implemented, obsolete, or superseded by the provided related items.

## Best Possible Solution

Keep the issue open and make an explicit product decision for unset Telegram `reactionLevel`. If existing sessions should not receive new reaction guidance without opt-in, distinguish missing from explicit `minimal` in the Telegram resolver or `agentPrompt.reactionGuidance`, avoid injecting the Reactions section for legacy unset configs, and align docs, type comments, generated metadata, tests, and changelog around the chosen default/migration behavior.
