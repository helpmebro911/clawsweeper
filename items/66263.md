## Summary

Current main still does not satisfy the native Codex compaction accounting request. The Codex native compaction path reports success without a post-compaction token snapshot, and both session-list and status fallbacks can still promote transcript-derived pre-compaction usage as fresh-looking current context. The merged related #64669 work fixes cumulative token usage projection, but it does not implement the compaction boundary or unknown/stale post-compaction state requested here.

## Best Possible Solution

Keep this issue open. Implement native Codex compaction accounting end to end: confirm the app-server token snapshot contract, propagate tokensAfter when reliable, persist an explicit unknown/stale state when it is missing, add or reuse a compaction boundary that transcript and status fallback honor, and add regression coverage for known tokensAfter, missing tokensAfter, and post-compaction session/status fallback. Treat #64669 as already-merged cumulative token projection hardening, not the canonical fix for this issue.
