## Summary

Keep open. Current main contains the narrower #64669 cumulative-usage fix, but the Codex-native compaction path still does not satisfy #66263: it completes from native compaction signals without capturing a post-compaction token count, and session compaction accounting can preserve an old total when `tokensAfter` is absent. Status surfaces freshness separately, but it can still compute percentage/remaining values from stale preserved totals.

## Best Possible Solution

Keep #66263 open and implement a focused Codex-native compaction accounting fix: parse and persist `tokensAfter` when native app-server events provide it; when absent, clear or mark `totalTokens` stale during compaction-count updates; add a compaction boundary so transcript fallback cannot reuse pre-compaction totals; adjust status/context reporting to avoid percent/remaining calculations for unknown stale usage; and cover known/unknown post-compaction token cases in Codex compact/projector plus session/status tests.
