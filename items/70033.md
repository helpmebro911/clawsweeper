## Summary

Keep #70033 open. Current main has a malformed streamed tool-call argument repair path, but it is only wired for Kimi anthropic-messages and openai-completions, not Amazon Bedrock's bedrock-converse-stream path reported here. The Bedrock plugin wraps cache/guardrail/region behavior only, and I did not find Bedrock-specific coverage for large write/exec arguments becoming `{}`.

## Best Possible Solution

Keep the issue open and fix it in the Bedrock/embedded-runner streaming path. A maintainer-quality fix should either extend malformed tool-call argument repair to `amazon-bedrock`/`bedrock-converse-stream` when safe, or add a Bedrock-specific guard that detects empty required arguments for streamed tool calls and stops the retry loop with a clear model/provider error. Add focused regression coverage for a Bedrock Converse stream where large `write.content` or `exec.command` deltas currently leave the tool call with `{}`.
