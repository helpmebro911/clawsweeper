## Summary

Keep open. Current main does not include the PR's compaction guard scorer, transcript-tail detector, guard config surface, action mapping, or recommend-reset scaffolding. Existing compaction-safeguard code covers adjacent behavior such as recent-turn preservation, tool-failure summaries, and quality audits, but it does not implement the deterministic loop-aware scorer proposed in PR #48312. The related context also points to an unfinished PR stack and an unresolved design concern about snapshot/restore recovery, so this needs maintainer product/code review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review or restack it under the broader loop-aware compaction guard work in #48238. The best product path is to review the scorer together with the detector, runtime hook, validator, recommend-reset diagnostics, and the snapshot/restore recovery gap before deciding whether to land, redesign, or manually close the stack.
