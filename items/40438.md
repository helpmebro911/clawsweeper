## Summary

Keep this PR open. Current main still does not provide a configurable directory exclusion contract for builtin memory indexing, so the central feature remains useful. The PR is not merge-ready as-is because the memory implementation moved, the PR includes an unrelated executable network preloader, and the proposed walker still needs excluded-root coverage.

## Best Possible Solution

Keep this PR open and salvage the focused feature: rebase onto current main, remove ipv4-fix.cjs, add an additive memorySearch.excludeDirs or broader exclude-pattern contract across schema/types/help/generated docs, thread it through resolveMemorySearchConfig, packages/memory-host-sdk/src/host/internal.ts, and extensions/memory-core sync/status/watch paths, skip excluded roots as well as descendants, and add tests for default exclusions, user-added exclusions, default memory-dir exclusions, excluded extraPaths roots, watcher consistency, and QMD/builtin scope.

## Risks / Open Questions

- The PR cannot be merged as-is because it targets the old src/memory/* implementation; current main has memory discovery in packages/memory-host-sdk/src/host and manager logic in extensions/memory-core/src/memory.
- The unrelated ipv4-fix.cjs broadens executable/network behavior outside the stated feature and should be removed before any functional review continues.
- The proposed child-directory check still needs to skip an excluded extraPaths root itself, such as an extraPaths entry directly pointing at node_modules or .venv.
- QMD behavior remains an open design point because current main maps memorySearch.extraPaths to QMD collections separately from the builtin scanner.
