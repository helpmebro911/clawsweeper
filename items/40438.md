## Summary

Keep this PR open. Current main still lacks a configurable directory-exclusion contract for memory indexing, so the central feature remains useful, but the PR should not merge as-is because it is stale against the relocated memory implementation and includes an unrelated network preloader file.

## Best Possible Solution

Keep this PR open as a useful implementation candidate, but require a focused rebase onto current main: remove `ipv4-fix.cjs`, add an additive `memorySearch.excludeDirs` or broader exclude-pattern contract across schema/types/help/generated docs, thread it through `resolveMemorySearchConfig`, `packages/memory-host-sdk`/`src/memory-host-sdk` file discovery, and `extensions/memory-core` sync/status/watch/reindex paths, skip excluded roots as well as descendants, and either implement equivalent QMD semantics or explicitly document QMD scope with tests.
