## Summary

Current main does not implement `memorySearch.excludeDirs`. The memory file walker in `src/memory-host-sdk/host/internal.ts` still recurses through extra-path directories without basename filtering, and the config/docs surface only exposes `extraPaths`. This PR’s goal remains unshipped, though the branch is stale against the current memory-host-sdk layout and includes an unrelated `ipv4-fix.cjs` file in the GitHub context.

## Risks / Open Questions

- The PR targets pre-refactor files from the old memory path (`src/memory/internal.ts`, `src/memory/manager*.ts`), while current main routes this behavior through `src/memory-host-sdk/host/internal.ts`; it will need a rebase/adaptation rather than direct merge.
- The GitHub context shows an unrelated added file, `ipv4-fix.cjs`, so the branch is not merge-ready as-is even if the feature remains desirable.
- Because the feature is still absent on main, users who point `memorySearch.extraPaths` at project roots can still index junk markdown from directories like `node_modules` or `.git` unless they avoid those roots manually.
