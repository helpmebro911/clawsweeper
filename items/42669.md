## Summary

Keep this PR open. Current main still does not implement the requested `skills.priority` config or priority-aware prompt truncation, and this active contributor PR remains a plausible implementation candidate. It needs a focused refresh before merge because the supplied diff also touches persisted skill snapshot reuse, which is correctness- and security-sensitive.

## Best Possible Solution

Keep this PR open and request a focused refresh against current main: add `skills.priority` to the public config type, strict schema, generated schema, config help/docs, and tests; apply exact names and `metadata.openclaw.skillKey` aliases before every prompt count/character limit while preserving deterministic non-priority ordering; keep persisted snapshot metadata aligned with prompt order; invalidate reused snapshots for all snapshot-affecting skills config, not only priority; and separate or justify any unrelated agent-command/session changes before normal review and changed-lane validation.
