Keep this PR open. Current main still does not implement `skills.priority`, the PR is still an open contributor implementation candidate, and the latest PR head still has correctness and snapshot-cache security concerns that should be resolved in review rather than closing the work as obsolete.

Keep this PR open and ask for a focused refresh: add `skills.priority` across the public config type, strict schema, generated schema, docs, and tests; apply exact-name and `skillKey` priority ordering before every prompt limit with no later alphabetical re-sort; preserve deterministic ordering for non-priority skills with a fixed locale; align persisted snapshot metadata with prompt order; and invalidate reused snapshots when all snapshot-affecting skills config changes.
