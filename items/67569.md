## Summary

Keep this PR open. Current main still lacks the gateway-catalog cache population and session-row fallback proposed here, so the linked Control UI context meter bug is not clearly implemented on main. The PR remains a plausible fix candidate, but it needs revision before merge because review comments identify correctness risks around slash-containing model IDs, global cache reset behavior, and unconditional 200k fallbacks.

## Best Possible Solution

Keep this PR open and request a targeted revision: resolve session-row context windows from precise catalog or persisted runtime data instead of an unconditional DEFAULT_CONTEXT_TOKENS fallback, avoid mixing raw slash model IDs with synthetic provider-qualified cache keys, reset any new context-cache side effects in test helpers, and keep regression coverage for Anthropic plus alias/slash-provider cases.
