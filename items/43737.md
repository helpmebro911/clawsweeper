## Summary

Keep this issue open. Current main still does not guarantee the resolved main session appears in the chat session dropdown when the active session list only contains heartbeat, and open PR #43984 remains an unmerged implementation candidate for this exact issue.

## Best Possible Solution

Review and land PR #43984 or supersede it with an equivalent narrow fix. The shipped behavior should guarantee the resolved main session key remains selectable when sessions.list with activeMinutes returns only heartbeat, ideally with regression coverage for stale main plus active heartbeat and the current selection set to heartbeat.
