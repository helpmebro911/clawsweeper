## Summary

Keep open. Current main has shipped several Codex harness fixes, but this tracker still has unresolved native compaction/context-accounting scope, including open child #66263 and #68209, and current docs/source still describe native Codex compaction as start/completion-only without post-compaction token metadata.

## Best Possible Solution

Keep this tracker open until maintainers resolve or explicitly supersede the remaining Codex compaction/context-accounting follow-ups. The likely implementation path is in extensions/codex/src/app-server/compact.ts plus session/status accounting: capture tokensAfter if Codex app-server exposes it, otherwise persist an explicit unknown/stale post-compaction state, prevent transcript fallback from treating pre-compaction usage as fresh, and update focused tests/docs.
