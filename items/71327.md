## Summary

Keep this PR open. Current main still lacks the LINE schema guard proposed here, while LINE runtime treats `dmPolicy: "open"` as accepting every DM sender. The PR is a focused LINE-owned hardening change with tests and favorable review context, not a conservative cleanup-close candidate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a LINE-owned schema hardening change that reuses `requireOpenAllowFrom`, includes focused LINE config-schema tests for top-level and account-level open-DM cases, aligns account validation with LINE's inherited config merge behavior, and adds release-note or migration guidance for configs that must switch to `allowFrom: ["*"]` or `dmPolicy: "allowlist"`.
