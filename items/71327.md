## Summary

Keep PR #71327 open. Current main still has the LINE schema gap: `dmPolicy: "open"` is valid with missing or specific `allowFrom`, while the runtime treats `open` as accepting every DM sender. The generic docs, security audit, doctor repair path, and LINE setup wizard already point toward explicit `allowFrom: ["*"]`; the PR is a focused fix for that remaining schema validation gap.

## Best Possible Solution

Keep the PR open for maintainer review and merge an equivalent fix: add the LINE top-level and account-level `requireOpenAllowFrom` schema refinement with account-specific messages, keep the focused regression tests for missing/specific/wildcard `allowFrom`, and update docs only if maintainers want the LINE channel page to repeat the existing generic wildcard rule.
