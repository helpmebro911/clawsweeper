## Summary

Keep PR #71327 open. Current main still accepts LINE `dmPolicy: "open"` with missing or specific `allowFrom` because the LINE schema has no `requireOpenAllowFrom`/`superRefine` guard, while the LINE runtime treats `dmPolicy: "open"` as allowing all DM senders. The PR is a focused schema/test hardening change, and the later PR context shows it addressed the bot's initial coverage/message findings.

## Best Possible Solution

Keep the PR open and review it as a focused LINE schema hardening fix. The likely maintainer path is to land the schema guard plus regression tests, with any requested docs/changelog migration note for `dmPolicy: "open"` requiring `allowFrom: ["*"]`.
