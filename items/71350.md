## Summary

Keep open. Current main has already implemented the issue's first-order per-agent thinking default and flat per-agent model params via `agents.list[].thinkingDefault` and `agents.list[].params`, but it has not implemented the requested `agents.list[].models` per-agent per-model catalog shape or any rules/escalation/AGENTS.md-driven `thinkingStrategy` behavior. Related #64792 was closed into this issue, and PR #44126 is still only a related open implementation discussion, not a shipped replacement for the whole request.

## Best Possible Solution

Keep the issue open as the canonical remaining product/design item. The best path is to acknowledge that `agents.list[].thinkingDefault` and flat `agents.list[].params` already ship, then decide whether OpenClaw wants a new `agents.list[].models` merge layer or only docs pointing users to `agents.list[].params`. For the adaptive part, implement a deliberate core agent-loop strategy surface such as `agents.list[].thinkingStrategy` with rule matching and failure escalation semantics, update config schema/docs/generated help, and add focused tests around resolution order, provider-supported thinking levels, and run-time escalation behavior. PR #44126 can inform the adaptive-controls design, while #32496 remains separate provider-param passthrough work.
