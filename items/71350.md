## Summary

Keep #71350 open. Current main already covers part of the request with `agents.list[].thinkingDefault` and per-agent `agents.list[].params`, but the broader requested capability is still not implemented: there is no `agents.list[].models` per-agent per-model override surface, no `thinkingStrategy` config, and no rules/failure-based thinking escalation. Related #64792 was closed as duplicate/superseded by this issue, and #44126 remains an open implementation discussion for adaptive thinking controls rather than shipped main behavior.

## Best Possible Solution

Keep #71350 open as the canonical product/design issue. The best path is to preserve the shipped `agents.list[].thinkingDefault` and `agents.list[].params` surfaces, then decide whether the remaining per-agent per-model override should be a new `agents.list[].models` schema or an extension of `params`. Adaptive thinking should be designed deliberately in core agent-loop/session policy or through an approved agent tool/API, with focused tests for rules mode, failure escalation, provider-supported `adaptive`, and interactions with the open #32496 sampling-param passthrough and #44126 adaptive-control work.
