## Summary

Current main only partially satisfies this feature. Per-agent `thinkingDefault` and flat `agents.list[].params` are implemented and documented, but the requested per-agent per-model `agents.list[].models` map and adaptive `thinkingStrategy` rules/escalation/AGENTS.md policy surface are still missing. Related #32496 and #44126 are relevant, but neither supersedes all remaining work here.

## Best Possible Solution

Keep this issue open as the canonical product/API discussion, or split it into narrower follow-ups. Preserve the shipped `agents.list[].thinkingDefault` and flat `agents.list[].params` behavior, then decide whether core should add `agents.list[].models` for per-agent per-model params and a `thinkingStrategy` API for message rules, failure escalation, and AGENTS.md-sourced policy. Coordinate provider parameter forwarding with #32496 and adaptive thinking controls with #44126.
