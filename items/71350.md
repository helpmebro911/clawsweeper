## Summary

Keep #71350 open. Current main already covers adjacent P0 pieces through `agents.list[].thinkingDefault` and flat `agents.list[].params` per-agent stream params, but the issue remains broader: there is no `agents.list[].models` per-agent per-model params map, no `thinkingStrategy` schema, and no rules/escalation adaptive thinking implementation on main. Related context also points to remaining work rather than closure: #32496 is still open for broader provider parameter pass-through, and the ClawSweeper report for PR #44126 kept adaptive thinking controls open rather than treating them as shipped.

## Best Possible Solution

Keep the issue open but narrow it. Treat `agents.list[].thinkingDefault` and flat `agents.list[].params` as already implemented, then decide whether `agents.list[].models` is still needed for per-agent per-model overrides. Track rules/escalation adaptive thinking through PR #44126 or a focused follow-up design, and coordinate provider-specific parameter expansion with the open #32496 surface.

## Risks / Open Questions

- The request bundles already-shipped config pieces with unimplemented adaptive behavior; closing it would likely lose the remaining strategy/per-model override work.
- Flat `agents.list[].params` may not satisfy agents that switch across multiple selected/fallback models and need model-specific params per agent.
- Adaptive thinking strategies would affect runtime/session/UI semantics and need explicit schema, docs, and focused tests before being treated as implemented.
