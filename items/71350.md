## Summary

Keep this issue open. Current main implements the per-agent `thinkingDefault` schema and a flat per-agent `agents.list[].params` override surface for values such as `temperature` and `maxTokens`, but it still does not implement the requested per-agent per-model `agents.list[].models` map or any `thinkingStrategy` rules, failure escalation, or AGENTS.md-sourced thinking policy. The prior ClawSweeper comment correctly recognized this as a remaining product/API discussion, with #44126 and #32496 related but not a shipped replacement for the full request.

## Best Possible Solution

Keep this issue open as the canonical product/API discussion or split it deliberately. Preserve the shipped `agents.list[].thinkingDefault` and flat `agents.list[].params` behavior, then decide whether core should add a per-agent per-model `models` override map and a `thinkingStrategy` API for rule matching, failure escalation, and AGENTS.md-sourced policies. Coordinate broader parameter forwarding with #32496 and adaptive-thinking work with #44126 rather than closing this issue as fully implemented.
