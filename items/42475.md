## Summary

Current main still has cost visibility, provider spend-limit error classification, and external/provider-level workaround docs, but not per-agent daily/monthly cost budget enforcement before gateway/model dispatch. The request remains actionable, and the discussion's ATXP, nornr, and vibe-billing comments are useful prior art rather than a shipped OpenClaw implementation or ClawHub plugin that closes the core guardrail.

## Best Possible Solution

Keep this issue open as a concrete core/gateway guardrail request. The best implementation path is a documented agents.defaults and agents.list[] budget contract, a request-time per-agent ledger/check before every model attempt and fallback, explicit warn/block/pause behavior, status and doctor visibility, and regression coverage for gateway sessions, cron, heartbeat/subagent paths, concurrency, missing cost metadata, and provider fallback chains. If maintainers decide this should be plugin-first, first add a supported blocking pre-model-call budget API rather than closing it as ClawHub-ready today.

## Risks / Open Questions

- Budget enforcement has product/security semantics that need maintainer design: approximate pricing, OAuth/no-dollar-cost cases, cache pricing drift, provider/tool/media spend outside core chat, and UTC vs local reset boundaries.
- Adjacent open budget/limit issues such as #38248, #58826, and #13615 overlap; maintainers may want to consolidate, but none currently supersedes this specific per-agent daily/monthly gateway cap request.
- A transcript-scan-only implementation would race concurrent runs, retries, fallback attempts, and delayed transcript writes; a durable ledger or reservation-style check is likely needed for hard caps.
