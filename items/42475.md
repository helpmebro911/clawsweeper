## Summary

Current main still has cost visibility, provider quota reporting, and post-hoc transcript/session cost aggregation, but it does not implement config-driven per-agent daily/monthly cost budget enforcement before gateway or model dispatch. The external ATXP, nornr, LiteLLM, and vibe-billing references are useful prior art/workarounds, not a shipped OpenClaw core feature or a sufficient ClawHub close path for the requested universal gateway guardrail.

## Best Possible Solution

Keep this open as a core/gateway guardrail request. The best implementation path is a documented agents.defaults and agents.list[] budget config contract, a durable per-agent ledger or reservation check before every model attempt and fallback, explicit warn/block/pause behavior, status and doctor visibility, docs updates, and regression coverage for gateway sessions, cron/heartbeat/subagent paths, concurrency, missing cost metadata, and provider fallback chains. If maintainers choose plugin-first, first add a supported blocking pre-model-call budget API rather than treating the current plugin hooks as sufficient.

## Risks / Open Questions

- A transcript-scan-only cap would race concurrent runs, fallback retries, delayed transcript writes, cron, heartbeat, and subagent paths; hard enforcement likely needs a durable ledger or reservation before each model attempt.
- Maintainers still need product and security decisions for approximate pricing, zero-dollar or subscription providers, media/tool spend, provider pricing drift, and UTC versus local reset boundaries.
- Closing as ClawHub-ready would skip the missing universal blocking budget API plus status/doctor/config documentation requested by the issue.
