## Summary

Keep open. Current main has a real failure-class-aware foundation and auth-profile cooldown/skip behavior, which addresses part of the 401-vs-429 concern raised in #47910 and the related #61328 discussion. But the specific request is not fully implemented: generic auth failures still use short profile cooldowns, format/invalid-request failures are still treated as failover-worthy, and operator visibility appears centered on `models status` auth-profile state rather than the requested provider-health `/status` surface.
