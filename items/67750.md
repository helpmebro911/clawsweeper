## Summary

Keep this issue open. Current main has timeout-triggered compaction and explicit idle-timeout payloads, but the external chat failure path still collapses mixed fallback exhaustion into the generic `/new` message when verbose failure details are off. That leaves the reported successful-compaction, retried-timeout, unavailable-fallback UX unresolved.

## Best Possible Solution

Keep this open and update `src/auto-reply/reply/agent-runner-execution.ts` so external chat failures after successful compaction preserve structured cause context: compaction attempted/succeeded, retried model hit the LLM idle timeout, and fallback candidates were unavailable or exhausted. Add regression coverage for successful compaction followed by timeout plus mixed fallback exhaustion on an external chat channel.
