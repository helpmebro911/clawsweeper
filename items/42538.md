## Summary

Current main still matches the reported bug. The gateway health path refreshes `getHealthSnapshot()` without any channel runtime snapshot, while `channels.status` explicitly reads `context.getRuntimeSnapshot()` and passes per-account runtime into channel status summary building. The open implementation candidate #42586 still uses closing syntax for this issue and is unmerged, so this issue should stay open until that PR is reviewed and landed or closed.

## Best Possible Solution

Keep this issue open until #42586 is reviewed and either landed or closed. The likely fix belongs in the gateway health path: thread the channel manager runtime snapshot into `refreshGatewayHealthSnapshot()` / `getHealthSnapshot()`, merge per-channel/per-account runtime fields before `buildChannelSummary`, and add a regression test proving health and `channels.status` agree on `running` and `connected`.
