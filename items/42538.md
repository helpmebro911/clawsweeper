## Summary

Current `main` still appears to have the reported mismatch. The `health` RPC path builds channel summaries from config/read-only inspection without gateway runtime state, while `channels.status` explicitly merges `getRuntimeSnapshot()`. WhatsApp's summary code falls back to `running: false` and `connected: false` when those runtime fields are absent, so the issue remains plausible on `main` at `7d9172d5e093435d37ed3e00e476a4eaf84a252b`.
