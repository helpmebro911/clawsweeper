## Summary

Keep this PR open. Current main still does not reset the runReplyAgent session after all fallback models time out. The PR targets a real remaining gap from #20910, but it needs rebase onto the current test split and a tighter implementation that classifies only definitive all-model timeout exhaustion, handles timeout error payloads, and avoids session rotation for heartbeat or transient provider failures.

## Best Possible Solution

Keep this PR open as the current implementation candidate. Rebase it onto current main, move the tests into the current runReplyAgent/execution test split, add a timeout reset callback or equivalent session-rotation path in runAgentTurnWithFallback, classify only structured all-model timeout exhaustion from fallback attempts, handle timeout isError payloads as well as thrown/meta.error failures, preserve heartbeat and transient HTTP guards, then run the changed auto-reply/agent test lane before maintainer review.
