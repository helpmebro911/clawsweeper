## Summary

Current main still does not implement the PR's requested runReplyAgent session reset after definitive all-model timeout exhaustion. Main has related timeout-recovery compaction for high-context attempts, but timeout meta/errors and timeout isError payloads still do not rotate the reply session. This PR remains a plausible implementation candidate, but it needs rebase and a safer timeout classifier before maintainer review.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a rebase onto current main. The best fix belongs in the auto-reply runner/fallback boundary: detect definitive all-model timeout exhaustion from structured FallbackSummaryError attempts and timeout isError/meta/error surfaces, preserve the existing high-context timeout-compaction and transient HTTP retry paths, skip heartbeat turns, rotate the session through resetReplyRunSession only after that proof, and port coverage into the current agent-runner-execution/runReplyAgent test split.
