## Summary

Keep #17211 open. Current main has a useful compaction-checkpoint subsystem, including list/get/branch/restore for pre-compaction snapshots, but the requested native task-boundary checkpoint engine still has meaningful missing pieces: generic create/resume/diff APIs, deterministic replay packet generation, and the proposed done/saved/dropped/next contract validation model.
