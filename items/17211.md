## Summary

Keep #17211 open. Current main still implements compaction checkpoint snapshot recovery only, not the requested native checkpoint engine with checkpoint.create/list/resume/diff, deterministic replay packets, structured task-boundary contracts, diffing, or validation hooks. The issue discussion also already contains a maintainer/Codex review explicitly keeping this umbrella open.

## Best Possible Solution

Keep #17211 open as the native checkpoint/replay umbrella. A good implementation should add versioned Gateway/session APIs for checkpoint.create/list/resume/diff, persist a structured task contract with done/saved/dropped/next plus validation state, generate deterministic replay packets, expose diff/audit data, update docs/UI, and add protocol/runtime/UI contract tests. If maintainers prefer plugin ownership, first add an explicit checkpoint/context-engine API so a plugin can implement it without reaching into core internals.
