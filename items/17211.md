## Summary

Keep #17211 open. Current main still ships compaction checkpoint recovery around pre-compaction transcript snapshots, not the requested native checkpoint engine with checkpoint.create/list/resume/diff, deterministic replay packets, task-boundary contracts, diffing, or validation hooks. The existing issue discussion also explicitly says to keep this umbrella open.

## Best Possible Solution

Keep #17211 open as the umbrella for native checkpoint/replay design. A good implementation should add versioned Gateway/session APIs for checkpoint.create/list/resume/diff, persist a structured task contract with done/saved/dropped/next plus validation state, generate deterministic replay packets, expose diff/audit data, update docs/UI, and add protocol/runtime/UI contract tests. If maintainers prefer a plugin-owned direction, first add an explicit checkpoint/context-engine API instead of relying on the current compaction-only context-engine surface.
