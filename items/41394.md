## Summary

Keep open. Current main has strong write-side validation and last-known-good/doctor recovery, so the linked crash-loop bugs are partly addressed, but the requested configurable strict/tolerant validation mode is not implemented and the read path still does not strip unknown keys by default. The related read-path item #40268 is still open in the provided GitHub context.
