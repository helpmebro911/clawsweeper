## Summary

Keep #9912 open. Current main at e672b614 still lacks a global or per-agent hard budget for max turns, max tool calls, or timeout-after-first-tool-call behavior. Existing loop detection and timeouts are related mitigations, and OpenResponses accepts `max_tool_calls`, but that field is documented as currently ignored and none of these implement the requested config contract or forced final-response behavior. The related #46140 and #47175 are closed variants, while #13271 remains a separate per-channel budget/auto-delegation request.

## Best Possible Solution

Keep #9912 open as the canonical global/per-agent hard iteration budget request. A complete fix should add typed config, strict schema/help, generated config docs, and public docs for max model-tool turns and/or total tool calls at `agents.defaults` and per-agent scope; define timeout-after-first-tool-call semantics; enforce the limits in the embedded/tool-call runtime; add focused tests for limit exhaustion and final-response behavior; and keep #13271 separate for per-channel budgets and auto-delegation.
