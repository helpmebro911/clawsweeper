## Summary

Keep open. Current main still has useful opt-in tool-loop detection, LLM idle timeout, and overall agent timeout controls, but it does not implement the requested agent-level hard `maxTurns` / `maxToolCalls` / post-first-tool timeout budget or force-final-response behavior. Related #13271 remains a narrower per-channel budget request rather than a canonical replacement for this global request.

## Best Possible Solution

Keep #9912 open until OpenClaw either implements a global or per-agent hard budget around the model-tool loop, or a maintainer explicitly narrows the product direction to existing loop detection. A complete fix should add typed config/schema/docs for total turns/tool calls and timeout behavior, enforce it in the agent runtime before additional tool execution, define the forced-final/last-tool-result behavior, and keep #13271 as the separate per-channel overflow/delegation track.
