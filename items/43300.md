## Summary

Current main still fails closed for trusted-proxy loopback/internal gateway self-connections, and open implementation PRs are explicitly linked as candidates. This issue should remain open until maintainers choose and merge a security-reviewed fix or close the candidate PRs as superseded.

## Best Possible Solution

Keep this issue open. Maintainers should review and either land or explicitly supersede #63379 and #64122, using #69066 as the longer-term service-identity direction. The chosen fix should preserve fail-closed trusted-proxy defaults for external traffic and add focused regression coverage for internal loopback/self-connection behavior.
