## Summary

Keep open. Current main still does not resolve Feishu media from quoted/replied messages: it resolves media from the triggering message, builds the agent media payload before fetching the parent message, and fetched parent messages do not expose the raw JSON needed to extract image_key/file_key. The PR remains a focused fix attempt, not an obsolete or superseded item.

## Best Possible Solution

Keep this PR open or replace it with an equivalent focused fix. The best current path is to rebase/adapt it onto main, preserve rawContent for fetched Feishu messages, resolve quoted/replied media only after the quoted message passes context-visibility checks, build the final media payload after all media is collected, preserve current Feishu routing/thread behavior, and add a quoted-media regression test.
