## Summary

Keep open. Current `main` still does not include quoted/replied Feishu media in agent context: it builds the media payload from the inbound message before fetching the quoted parent, and fetched parent messages discard the raw JSON needed to recover `image_key`/`file_key`. The existing Feishu tests cover quoted text visibility, not quoted media resolution.

## Evidence

- **Media payload only uses the current inbound message:** `handleFeishuMessage` resolves media from `event.message.content` and immediately builds `mediaPayload` before any quoted message fetch. Later it fetches `ctx.parentId` only for `quotedContent`, so quoted attachments never enter `mediaList`.
  - file: [extensions/feishu/src/bot.ts:712](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/extensions/feishu/src/bot.ts#L712)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '706,770p'`
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Quoted message fetch loses raw media keys:** `parseFeishuMessageItem` reads `item.body?.content` into local `rawContent` but returns only parsed `content` and `contentType`; `FeishuMessageInfo` has no `rawContent` field on `main`. For image/file replies, `parseFeishuMessageContent` collapses the message to placeholders like `[image message]`, which is insufficient for `resolveFeishuMediaList`.
