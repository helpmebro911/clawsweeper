## Summary

Keep open. Current `main` still does not resolve media from quoted/replied Feishu messages: the inbound media payload is finalized before quoted-message lookup, and fetched quoted messages only retain rendered text/placeholder content rather than the raw JSON needed to extract `image_key`/`file_key`.

## Evidence

- **Quoted-message path never adds quoted media:** `handleFeishuMessage` resolves media for the inbound message, immediately builds `mediaPayload`, and only afterwards fetches the quoted message. The quoted-message branch sets `quotedContent` only; it never calls `resolveFeishuMediaList` for the parent message or mutates `mediaList`.
  - file: [extensions/feishu/src/bot.ts:717](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/extensions/feishu/src/bot.ts#L717)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '700,790p'`
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Fetched Feishu messages discard raw attachment JSON:** `parseFeishuMessageItem` reads `item.body?.content` into a local `rawContent`, but `FeishuMessageInfo` only exposes parsed `content`/`contentType`. For non-text payloads, `parseFeishuMessageContent` falls back to placeholders like `[file message]`, so the quoted-message fetch path loses the original attachment keys.
  - file: [extensions/feishu/src/send.ts:219](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/extensions/feishu/src/send.ts#L219)
  - command: `nl -ba extensions/feishu/src/send.ts | sed -n '219,285p'`
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Current type shape has no field for raw quoted content:** `FeishuMessageInfo` on `main` has `content` and `contentType` but no `rawContent`, so current callers cannot pass the original quoted message JSON into media resolution.
  - file: [extensions/feishu/src/types.ts:69](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/extensions/feishu/src/types.ts#L69)
  - command: `nl -ba extensions/feishu/src/types.ts | sed -n '69,84p'`
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Media resolver still requires raw message content with keys:** `resolveFeishuMediaList` parses the supplied `content` to extract media keys via `parseMediaKeys`. If the quoted-message fetch path only has placeholder text like `[file message]`, this function cannot recover the attachment.
  - file: [extensions/feishu/src/bot-content.ts:336](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/extensions/feishu/src/bot-content.ts#L336)
