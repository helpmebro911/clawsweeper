## Summary

Keep open. Current main has Nextcloud Talk as a bundled plugin, but the requested typing indicator is not implemented: inbound replies use the generic dispatch helper without typing callbacks, the Nextcloud Talk send layer only covers message and reaction endpoints, and the channel schema/types/docs do not expose a typingIndicator option. The existing channel plugin typing hooks mean this should be a focused Nextcloud Talk plugin enhancement, not a stale or duplicate cleanup close.

## Best Possible Solution

Keep the issue open and implement it as a focused Nextcloud Talk bundled-plugin enhancement: add a typed sendTypingNextcloudTalk helper using the existing account resolution, SSRF guard, and API credentials; wire start/stop into the inbound reply lifecycle with createTypingCallbacks or an additive shared dispatch option; add schema/types/docs for a conservative typingIndicator toggle/default; and cover success, cleanup-on-error, disabled-config, and missing-credential cases in Nextcloud Talk tests.
