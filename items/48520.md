## Summary

Keep PR #48520 open. Current main does not contain the SEA build surface proposed by the PR, and the existing Docker packaging path only partially overlaps the stated cold-start motivation. The PR also has unresolved review blockers, so it is not safely mergeable as-is, but it remains a concrete core packaging proposal rather than a duplicate, ClawHub item, or repo-external task.

## Best Possible Solution

Keep the PR open for maintainer judgment, but require a substantial rework before merge: decide whether SEA is a supported core packaging target or whether the shipped Docker/GHCR image is the intended cold-start solution; if SEA remains desired, update the build to the supported Node 22/24 SEA flow or explicitly raise the runtime requirement, remove generated `sea-config.json` from source control, verify downloaded Node artifacts, delete or wire unused scripts, add docs under the existing Docker/install surface, and add a lightweight validation path for the generated artifact.
