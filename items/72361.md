## Summary

Keep this PR open. Current main still has the reported `/allowlist --store` policy gap: the store-only path skips the existing `resolveConfigWriteDeniedText` guard and writes to the pairing allowlist store directly. The PR is a focused two-file fix paired with open issue #72360 by the same author, so the right maintainer action is review/merge or supersede it with an equivalent narrow fix.

## Best Possible Solution

Keep this PR open and review/merge it, or supersede it with an equivalent narrow fix on main: apply `resolveConfigWriteDeniedText` to store-targeted `/allowlist add|remove --store` mutations before `updatePairingStoreAllowlist`, using the explicit channel/account config-write target, and keep regression tests for channel-level false, account-level false, and allowed store edits. Because this PR is paired with open issue #72360 by the same author, close the pair only after this implementation or an equivalent maintainer-approved fix lands.
