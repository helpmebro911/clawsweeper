## Summary

Keep this PR open. Current main still has the reported `/allowlist --store` policy gap: the config-write authorization check only runs in the config-edit branch, while the store-only branch can call `updatePairingStoreAllowlist` directly. The PR is paired with open issue #72360 and uses closing syntax for it, so it should be reviewed and landed or superseded by an equivalent fix, not closed during cleanup.

## Best Possible Solution

Keep this PR open and review/merge it, or manually port the same narrow fix: apply `resolveConfigWriteDeniedText` to store-targeted `/allowlist add|remove --store` mutations before `updatePairingStoreAllowlist`, using the explicit channel/account config-write target, and keep the channel-level false, account-level false, and allowed regression tests. Because the PR is paired with open issue #72360 via closing syntax, close both only after the implementation lands or an equivalent maintainer-approved fix supersedes it.
