## Summary

Keep #69608 open. Current main has partial startup-context support for slugged daily notes, but it still lacks the shared daily-memory/session-summary model and several concrete caller changes from the PR. The related #69617 is a separate narrow gateway log-suppression PR and does not supersede this memory/startup fix.

## Best Possible Solution

Keep the PR open and ask for maintainer review/rebase rather than closing it. The best product/code path is a focused implementation that centralizes daily-memory filename parsing and session-summary classification in the host/runtime-file contract, then updates startup context, session-memory persistence, doctor, dreaming ingestion, REM backfill, and short-term promotion to consume the same rules with regression tests for mixed canonical/slugged notes, local/UTC boundaries, missing bootstrap placeholders, lightweight slug generation, and cross-platform legacy paths.
