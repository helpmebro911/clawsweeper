## Summary

Keep this PR open. Current main still does not honor `--custom-base-url` for LiteLLM non-interactive onboarding because LiteLLM uses the generic single-provider API-key helper, whose non-interactive config callback cannot see CLI onboarding options. The PR remains a plausible focused fix candidate, but it should not be closed as obsolete or implemented on main; it needs maintainer review because it also adds `/v1/models` discovery behavior and has no regression test in the submitted diff.

## Best Possible Solution

Keep this PR open as a valid bug-fix candidate. The best maintainer path is to rebase it onto current main, add focused regression coverage for LiteLLM non-interactive `--custom-base-url`, and then decide whether to keep or narrow the added `/v1/models` discovery behavior after resolving the security/logging and capability-metadata review points.
