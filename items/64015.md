## Summary

Keep open. Current main still preserves the central diagnostics inconsistency: `status --deep --json` embeds gateway health, but health builds channel summaries from a minimal account snapshot that lacks runtime and credential-source fields, while `channels.status` uses the richer runtime snapshot path. Doctor also still warns that a SecretRef-managed gateway token is unavailable from a synchronous raw-config path instead of resolving the SecretRef or sharing the secrets-audit resolvability result. Some subpoints look improved, but the Discord health/status and gateway SecretRef false positives remain actionable.

## Best Possible Solution

Keep this issue open. Fix `health` and `status --deep` to reuse the same channel runtime/account projection contract as `channels.status`, preserving `running`, `connected`, `tokenSource`, probe, and audit fields. Then update doctor gateway auth to resolve applicable `gateway.auth.token` SecretRefs through the async gateway SecretRef credential path or consume the same resolvability result as `secrets audit` before warning. Add focused regressions for a running configured Discord account and a resolvable gateway auth SecretRef; split already-fixed subpoints into follow-up notes if maintainers prefer narrower tracking.
