## Summary

Current main still does not implement the requested Private/Secret/Incognito mode. The gateway session protocol has no per-session or per-channel private/ephemeral/persistence-off flag, session metadata and transcript helpers still persist to disk, workspace bootstrap files are still loaded into context by default, and the provided related discussion makes #7403 the canonical open tracker rather than a duplicate.

## Best Possible Solution

Keep #7403 open as the canonical product request. A complete fix should add an explicit private/ephemeral session mode across the Gateway protocol, session config/store, transcript writing, workspace bootstrap and memory loading, Mission Control/channel controls, and tests proving that no session history or workspace memory files are read or written while the mode is active.
