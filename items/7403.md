## Summary

Keep #7403 open. Current main still lacks an enforceable per-session Private/Secret/Incognito mode: there is no Gateway/session/private persistence control, session creation still creates a transcript file, normal turn persistence still writes JSONL, and the related discussion already treats this issue as the canonical tracker after #12242 and the closed-unmerged #52478.

## Best Possible Solution

Keep #7403 open as the canonical product request. A complete implementation should add an explicit private/ephemeral session mode through the Gateway protocol and session state, enforce no transcript/media/session-history writes, suppress workspace bootstrap and memory reads/writes, expose Mission Control and per-channel controls, and add regression tests proving private sessions neither read workspace/memory context nor write persistent history.

## Risks / Open Questions

- The request crosses protocol, session store/transcripts, workspace bootstrap, memory/context-engine behavior, media persistence, channel commands, and Mission Control UI; a partial implementation could create a false privacy guarantee.
- There is a global contextInjection: never knob for bootstrap suppression, but treating that as Private Mode would leave transcript and other persistence paths untouched.
