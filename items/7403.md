## Summary

Keep #7403 open. Current main still does not provide an enforceable Private/Secret/Incognito session mode, and the related context makes this issue the canonical tracker rather than a cleanup-close candidate.

## Best Possible Solution

Keep #7403 open as the canonical product request. A complete fix should add an explicit private/ephemeral session mode through the Gateway protocol and session/config state, enforce no transcript/media/session-history writes, suppress workspace bootstrap and memory reads/writes, expose Mission Control and per-channel controls, and add regression tests proving private sessions do not read workspace/memory context or write persistent history.
