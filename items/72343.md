## Summary

Keep open. Current main still routes bare `openclaw` to Crestodian with no config-snapshot gate for fresh or empty installs, and the tests, Docker smoke, and docs still encode that existing behavior. The PR is active, focused, contributor-authored, has no protected labels, and the provided discussion shows the author addressed the Greptile smoke-coverage concern rather than abandoning the work.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, the right implementation path is the PR's scoped CLI route: before the existing bare-root Crestodian branch, read the config snapshot, route missing or valid-empty authored config to classic onboarding, and preserve Crestodian for configured installs and invalid configs that need repair. Keep the paired unit tests, Docker smoke expectation, and docs updates so the first-run contract is explicit.
