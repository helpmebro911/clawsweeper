## Summary

Keep PR #38658 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still has the Slack pending-only channel-history behavior: resolved `requireMention` is not carried into dispatch, processed `requireMention=false` room messages are not recorded into the sliding history window, dispatch clears room history after both no-reply and delivered-reply turns, and an existing regression test asserts that the second always-on channel message does not receive the first message as history. The PR remains a focused plausible fix, with the no-reply review gap addressed in later PR commits, but the ordering/product-semantics concern still needs maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep PR #38658 open for maintainer review. If always-on Slack channel context is desired, update `extensions/slack/src/monitor/message-handler/*` so prepared messages carry the resolved mention policy into dispatch, `requireMention=false` processed room messages are retained in both reply and no-reply paths, retained history order is deterministic, prompt/history regression coverage is updated, and the Slack/group-history docs describe the behavior. If maintainers prefer pending-only group history, close the PR with that explicit product rationale.
