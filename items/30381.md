## Summary

Keep #30381 open. Current main still validates the OpenAI-compatible request body `model` before dispatching `/v1/chat/completions`, even when `x-openclaw-agent-id` selects the target agent. The requested compatibility behavior is not implemented, the docs/tests still describe and enforce the stricter OpenClaw agent-target model contract, and the issue discussion remains active and specific.

## Best Possible Solution

Implement a header-aware bypass in the OpenAI-compatible model validation path: when `x-openclaw-agent-id` resolves to a valid agent, ignore the request body `model` for validation and agent/model selection, while continuing to validate explicit `x-openclaw-model` backend overrides. Add a regression test for `model: "gpt-4"` plus `x-openclaw-agent-id` on `/v1/chat/completions`, consider the same case for `/v1/responses`, and update https://docs.openclaw.ai/gateway/openai-http-api if this becomes the supported compatibility contract.
