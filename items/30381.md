## Summary

Current `main` still rejects non-`openclaw` request `model` values before `x-openclaw-agent-id` can relax that check, so the reported behavior is not implemented on `main` and the issue should stay open.

## Evidence

- **chatCompletions still validates request model after resolving header-selected agent:** `src/gateway/openai-http.ts:530` reads the body `model`, `src/gateway/openai-http.ts:533` resolves agent/session context from header or model, and `src/gateway/openai-http.ts:541` immediately calls the shared validator; if it returns an error, `src/gateway/openai-http.ts:546` sends HTTP 400.
  - file: [src/gateway/openai-http.ts:530](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/gateway/openai-http.ts#L530)
  - command: `nl -ba src/gateway/openai-http.ts | sed -n '526,550p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **shared validator rejects any non-agent-style model string regardless of header:** `src/gateway/http-utils.ts:308-313` returns `Invalid \`model\`. Use \`openclaw\` or \`openclaw/<agentId>\`.` whenever `params.model` is present but does not parse as an OpenClaw agent target. That branch runs before any `x-openclaw-model` handling and has no exception for `x-openclaw-agent-id`.
  - file: [src/gateway/http-utils.ts:308](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/gateway/http-utils.ts#L308)
  - command: `nl -ba src/gateway/http-utils.ts | sed -n '303,318p'`
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **header does select agent, but only for request context:** `src/gateway/http-utils.ts:350-352` gives `x-openclaw-agent-id` precedence when selecting the agent, and `src/gateway/http-utils.ts:383-396` uses that for session routing. That means the agent header is honored for routing, but not for skipping request-model validation.
  - file: [src/gateway/http-utils.ts:350](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/gateway/http-utils.ts#L350)

- **tests cover invalid model rejection but not the requested header override behavior:** `src/gateway/openai-http.test.ts:306-318` asserts a 400 with `Invalid \`model\`` for an invalid body model, while the existing header test at `src/gateway/openai-http.test.ts:187-191` only covers `x-openclaw-agent-id` with a valid `model: "openclaw"`. There is no test showing invalid body model + agent header succeeds.
  - file: [src/gateway/openai-http.test.ts:306](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/gateway/openai-http.test.ts#L306)
