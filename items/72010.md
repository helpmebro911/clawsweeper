## Summary

Keep open. Current `main` still has the old stuck-session diagnostic semantics: session activity is refreshed only by queue/state changes, there is no `markDiagnosticSessionProgress` API, ACP runtime events are projected without a progress callback, and docs/schema help still describe a simple age threshold. The PR is an active, concrete bug fix; prior #71969 was closed unmerged by the too-many-PRs guard, not shipped. Some ancillary rebase/test cleanup from the PR already appears present on `main`, so the best path is normal maintainer review/rebase rather than auto-close.

## Best Possible Solution

Keep #72010 open for normal maintainer review. The still-missing product fix belongs in core diagnostics and reply/ACP dispatch: add a diagnostics-gated session-progress marker, call it for real reply/tool/status/block and ACP runtime progress while excluding typing keepalives, reset/back off stuck warnings, and update the generated config docs. Drop or rebase away ancillary changes that current `main` already contains.
