## Summary

Keep open. Current main still has age-based stuck-session warnings with no progress marker API, no warning backoff state, and no dispatch/ACP wiring that resets diagnostic activity for active reply/tool/status/block/ACP events. PR #72010's head patch still carries the missing diagnostic implementation and addressed the earlier diagnostics.enabled review; a few sidecar rebased fixes are already present on main, but the central behavior remains unmerged.

## Best Possible Solution

Keep the PR open and rebase/narrow it around the still-missing diagnostic behavior: add the progress marker/backoff state, wire real reply/tool/status/block and ACP runtime events as progress, keep typing keepalive excluded, and preserve the diagnostics.enabled guard. Sidecar fixes that already landed on main can be removed from the PR during rebase.
