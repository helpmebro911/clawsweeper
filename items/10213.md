## Summary

Keep #10213 open. Current main at cf834e2a21eac7fd218989fb5b85e7e8dd4c3246 still lacks Anthropic-native server-side compaction config, request shaping, stream/block handling, docs, and tests. The closed PR #10214 was not merged, and broader issue #65287 overlaps but does not clearly supersede this focused tracker.

## Best Possible Solution

Keep #10213 open as the focused tracker for Anthropic server-side compaction. A good fix should add an explicit opt-in provider/model or compaction config surface, validate and document it, inject the correct Anthropic beta header and `context_management.edits` payload without clobbering existing headers/payload hooks, preserve returned `compaction` blocks across history, handle `compaction_delta` and `stop_reason: "compaction"`, add verbose/logging behavior, and add focused tests alongside the existing OpenAI Responses server-side compaction coverage. Treat #65287 as related broader cache/TTL design work unless maintainers explicitly choose it as canonical.
