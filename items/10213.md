## Summary

Keep #10213 open. Current main still does not implement Anthropic-native server-side compaction: the config/schema surface has no server-side Anthropic compaction option, the bundled Anthropic provider wrapper only handles existing beta/service-tier/fast-mode behavior, and the native Anthropic transport does not inject `context_management.edits` or round-trip provider compaction blocks. The related PR #10214 is closed unmerged in the provided GitHub context, and open #65287 is broader native-compaction/cache-TTL design rather than a clear canonical replacement for this focused request.

## Best Possible Solution

Keep #10213 open as the focused Anthropic server-side compaction tracker. A complete fix should add an explicit opt-in config or provider/model param, validate and document it, inject Anthropic native compaction beta headers and `context_management.edits` only for direct Anthropic-compatible requests, preserve streamed/stored compaction blocks for replay, handle provider compaction stop behavior, add logging, and cover the Anthropic wrapper/transport paths with focused tests. Treat #65287 as related broader prompt-cache/native-compaction design unless maintainers explicitly mark it canonical.
