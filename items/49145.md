## Summary

Keep PR #49145 open. Current main has only partial Signal quote handling: it exposes quoted body/sender in inbound context, but the PR's central remaining work is not implemented on main: Signal `ReplyToId` extraction, reaction-triggered heartbeat wake, and outbound Signal quote parameters for native quoted replies.

## Best Possible Solution

Keep the PR open and review the current head against main. The best path is a focused Signal-channel implementation with tests for `dataMessage.quote.id` to `ReplyToId`, visibility gating, reaction wake, outbound `quote-timestamp`/`quote-author`, and first-chunk-only quote behavior across formatted text, media, monitor replies, and any `sendPayload` path. Close later only if those behaviors are separately merged or the PR is superseded by a canonical replacement.
