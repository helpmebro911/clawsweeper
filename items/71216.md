## Summary

Keep #71216 open. Current main has adjacent surfaces, especially `agents.defaults.sandbox.mode`, multi-agent bindings, profile-based multiple gateways, node command allow/deny, and `tools.fs.workspaceOnly`, but it does not implement the requested `routing.rules`, `instances`, or `gateway.nodes.denyPaths` schema/runtime enforcement. The issue is recent, author association is NONE, has no protected labels, and the related ClawSweeper matches are adjacent schema/config work rather than canonical duplicates.

## Best Possible Solution

Keep the issue open for maintainer design. A good path is to acknowledge that sandbox mode already exists as `agents.defaults.sandbox.mode`, then split or refine the remaining work: decide whether tag-based provider routing belongs in agent/model routing, decide whether `instances.*` should replace or complement profile-based multi-gateway setup, and implement any path deny list with TypeScript types, Zod/generated schema, docs, doctor validation, and runtime enforcement tests.
