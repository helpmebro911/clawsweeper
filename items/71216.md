## Summary

Keep this issue open. Current main still has adjacent sandbox, model, binding, multi-agent, and node command-policy features, but it does not implement the requested first-class config/runtime surface: no root `sandbox`, no `routing.rules`, no `instances`, and no `gateway.nodes.denyPaths`. The existing ClawSweeper comment remains consistent with the current checkout at 348728c28c1ea9ae7c5824350baba35ee08a275e.

## Best Possible Solution

Keep this issue open and split the work deliberately: decide whether root sandbox policy should alias or wrap `agents.defaults.sandbox`; design tag-aware provider/model routing in the agent/model-selection layer with explicit metadata propagation; decide whether multi-instance declarations belong in config or external gateway/profile management; and implement `gateway.nodes.denyPaths` only with schema/types/docs plus real runtime path-enforcement and security/doctor test coverage.

## Risks / Open Questions

- The issue bundles four separable product/API surfaces; implementing all in one schema change would be hard to review safely.
- `gateway.nodes.denyPaths` needs runtime enforcement across actual file/tool/node access chokepoints, not just a schema field.
- `routing.rules` and `instances.*` need maintainer direction because they affect model-selection policy, metadata propagation, gateway/profile management, and trust boundaries.
