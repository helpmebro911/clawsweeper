## Summary

Keep #71216 open. Current main has adjacent `agents.defaults.sandbox.mode` and route bindings, but it does not implement the requested config contract: there is no top-level `sandbox`, `routing.rules`, or `instances`; gateway node policy still exposes only command allow/deny lists; and the generated schema/help expose `gateway.nodes.denyCommands` but not `gateway.nodes.denyPaths`. The issue is recent and asks for core schema plus gateway enforcement/doctor validation, so it remains actionable rather than stale or ClawHub-only.

## Best Possible Solution

Keep this open as a core config/security design item. The best path is to split or sequence the work: decide whether sandbox remains under `agents.*.sandbox` or gains a top-level alias, design tag-based provider routing against the current agent/model selection contracts, add `gateway.nodes.denyPaths` only with actual read/write chokepoint enforcement plus tests, and separately scope multi-instance declaration/validation because it affects gateway lifecycle and config ownership.
