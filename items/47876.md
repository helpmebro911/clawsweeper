## Summary

Keep open. Current main has partial health/status/session surfaces and a clear ClawHub-first policy for community-tool discovery, but it does not include the proposed `/gateway/monitoring` page, does not reference ClawMonitor, and does not expose the requested session-centric last-in/last-out/work-state summary. The linked PR #47877 remains the direct docs/showcase work item, so this issue is not safely implemented or cleanly closable as ClawHub-only cleanup.
