## Summary

Keep open. Current `main` does not implement first-class Excel or generic binary `input_file` handling for Gateway/OpenResponses. The path still defaults to text/HTML/CSV/JSON/PDF only, injects decoded file content ephemerally into prompt context, and has no workspace materialization or stable binary receipt/path for tools.

## Evidence

- **Default file MIME allowlist excludes Excel/binary:** `DEFAULT_INPUT_FILE_MIMES` only includes `text/plain`, `text/markdown`, `text/html`, `text/csv`, `application/json`, and `application/pdf`. `resolveInputFileLimits` uses that list as the fallback allowlist.

  - command: `nl -ba src/media/input-files.ts | sed -n '90,210p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Extractor only returns text or PDF-rendered images:** `InputFileExtractResult` has only `filename`, optional `text`, and optional `images`. `extractFileContentFromSource` rejects unsupported MIME types, special-cases PDF, and otherwise decodes the buffer as text; it does not persist binary bytes or return a workspace path/receipt.
  - file: [src/media/input-files.ts:16](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/media/input-files.ts#L16)
