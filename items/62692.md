## Summary

Keep open. Current main registers managed `web_search` and the bundled SearXNG provider, and it has later mitigations for local-model prompt pressure and compaction loops. However, the reported llama-server failure mode still has meaningful remaining work: the OpenAI Completions stream path only turns real `choice.delta.tool_calls` into executable tool calls, while plain text JSON/tool-call content is treated as assistant text. The issue's proxy workaround is useful evidence, but this is a core/provider transport compatibility issue rather than a ClawHub-only item.
