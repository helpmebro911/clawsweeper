## Summary

Keep #38716 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still does not implement Slack RTS via assistant action tokens: Slack subscribes to normal message/app/reaction/member/channel/pin events only, does not model or store `action_token`, does not call `assistant.search.context`, and does not expose Slack `search` in the message action surface. The related `search.messages` PR context is a different user-token Web API path and does not supersede assistant-thread `action_token` RTS support.

## Best Possible Solution

Keep the issue open. The best implementation path is in the bundled Slack plugin unless maintainers explicitly decide to externalize it: subscribe to `assistant_thread_started` and `assistant_thread_context_changed`, capture and scope `action_token` per assistant thread, add an `assistant.search.context` helper exposed through Slack `search` or a dedicated tool, gate setup on the required Slack app settings/scopes, and add focused tests plus Slack setup/docs updates. If maintainers prefer ClawHub, first open a narrower plugin/API design item for how an external plugin can access the existing Slack app event stream and assistant action tokens without replacing the bundled Slack channel.
