## Summary

Keep this issue open. Current main still normalizes `google/gemini-3.1-pro` to the preview id and still uses `gemini-3-pro-preview` as the Gemini 3.1 Pro forward-compat template, with tests documenting that preview-derived behavior. The related fix PRs in the provided context were closed unmerged, and the later stale-catalog issue/PR is adjacent but does not fix this Google alias/template problem.

## Best Possible Solution

Keep the issue open and fix the Google plugin/model-selection behavior in main: stop rewriting `google/gemini-3.1-pro` to the preview id where stable ids should remain picker-visible, update Gemini 3.1 Pro forward-compat template precedence away from `gemini-3-pro-preview`, and add focused regression coverage in `extensions/google/provider-models.test.ts`, `extensions/google/model-id.test.ts`, and the model-selection path that currently asserts preview normalization.
