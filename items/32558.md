## Summary

Keep PR #32558 open. Current main still does not implement Microsoft Graph upload-session fallback for Microsoft Teams OneDrive/SharePoint uploads, while the PR is a focused channel reliability fix with related predecessor PRs closed unmerged. The active send and reply paths still route group/channel file sends through direct `/content` upload helpers, so the requested behavior remains missing.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. The best path is a focused `extensions/msteams/src/graph-upload.ts` implementation that preserves the simple upload path for small files, adds Graph upload-session fallback for large OneDrive and SharePoint uploads, handles `202 nextExpectedRanges` plus final `200/201`, uses Graph-valid chunk sizing/order, retries transient upload-session failures without infinite stalls, fixes the reviewed off-by-one, and lands with focused OneDrive and SharePoint regression tests.
