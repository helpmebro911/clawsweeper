## Risks / Open Questions

- The PR predates current buildUserAgent/User-Agent handling in graph-upload.ts, so a rebase must preserve User-Agent headers on Graph session creation and simple uploads.
- The PR review discussion identifies an off-by-one in UPLOAD_SESSION_MAX_STALLS: the strict > guard allows six stalled responses when the constant says five.
- The hardcoded 4 MiB fallback threshold should be confirmed against the observed Teams/Graph failure mode before merge.
- Chunk PUTs use a preauthenticated uploadUrl returned by Graph; the implementation should keep Authorization omitted for that URL and avoid letting any non-Graph-controlled source feed it.
