## Summary

Keep PR #32558 open. Current main still has no Microsoft Graph upload-session path for the active Microsoft Teams OneDrive/SharePoint upload helpers, and the PR remains a focused Teams reliability change. Related PRs #26668 and #26712 were closed unmerged, and #49126 was a maintainer tracker rather than a shipped implementation.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. The best path is a focused `extensions/msteams/src/graph-upload.ts` change that adds a Graph upload-session path for Teams OneDrive and SharePoint uploads, handles `202 nextExpectedRanges` plus final `200/201`, uses valid chunk sizing/order per Graph docs, retries/resumes transient upload-session failures, fixes the reviewed stall-count off-by-one, and lands with focused OneDrive and SharePoint regression tests.
