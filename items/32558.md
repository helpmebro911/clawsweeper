## Summary

Keep open. Current `main` does not include the large-file Graph upload-session fallback from #32558, and the earlier PR #26668 was explicitly closed as a duplicate of #32558 rather than as an already-landed implementation.

## Evidence

- **OneDrive upload still uses only the simple `/content` endpoint:** `uploadToOneDrive` unconditionally does a single `PUT` to `https://graph.microsoft.com/v1.0/me/drive/root:/OpenClawShared/<file>:/content` with the full buffer. There is no size threshold, `createUploadSession`, chunk loop, or `nextExpectedRanges` handling in current `main`.
  - file: [extensions/msteams/src/graph-upload.ts:29](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/msteams/src/graph-upload.ts#L29)

  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
