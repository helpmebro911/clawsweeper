## Summary

Keep open. Current main has a stronger persistence-time cap for oversized toolResult.details, but it still counts hidden details in the preemptive context estimator. The PR is not fully obsolete because the false-overflow class can still exist when detail payloads contribute to aggregate context size without any single tool result crossing the per-result truncation threshold.

## Best Possible Solution

Keep the item open and adapt the PR to current main: remove hidden toolResult.details from model-context estimation, retain the existing persistence-time details sanitizer, and add a focused regression test for aggregate hidden details that would currently cross the preemptive overflow threshold while visible content remains within budget.
