## Summary

Keep open. Current `main` still contains the reported `tryRepairWritableBits` behavior, and the touched file is unchanged between the latest release `v2026.4.23` and `main`, so this is not implemented on `main` despite the follow-up comment claiming it was.

## Evidence

- **Current main still returns false when permissions are already tightened:** In `tryRepairWritableBits`, the branch `if (typeof st.mode !== "number" || (st.mode & 0o022) === 0) { return false; }` is still present. That means an already-secure dir is treated as unrepaired instead of re-validated, which matches the race described in the issue.
  - file: [src/infra/tmp-openclaw-dir.ts:108](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/infra/tmp-openclaw-dir.ts#L108)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Current main still throws from the fallback path on that false return:** `ensureTrustedFallbackDir` still throws `Unsafe fallback OpenClaw temp dir` when `resolveDirState(...)` is `invalid` and `tryRepairWritableBits(...)` returns false, so the reported failure mode is still reachable in source.
  - file: [src/infra/tmp-openclaw-dir.ts:125](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/infra/tmp-openclaw-dir.ts#L125)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **The error can still happen during logger initialization before subcommands run:** `resolveDefaultLogDir()` calls `resolvePreferredOpenClawTmpDir()` at module initialization time via `DEFAULT_LOG_DIR`, which is consistent with the issue's stack trace and 'fails before any subcommand runs' report.
  - file: [src/logging/logger.ts:43](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/logging/logger.ts#L43)
  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **Latest release and current main have no diff for the affected file:** `git diff --no-ext-diff --unified=3 a9797214338ba31c52c796adbb75afb16e0684a9 cc0f3e0e40364023cc3a39227a411898ee68ce3b -- src/infra/tmp-openclaw-dir.ts` produced no output. The latest release and current `main` therefore carry the same implementation here.
  - file: [src/infra/tmp-openclaw-dir.ts](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/infra/tmp-openclaw-dir.ts)
