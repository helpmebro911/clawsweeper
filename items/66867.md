Keep this issue open. Current main still has the reported non-idempotent temp-dir repair branch, and the linked fix remains an open, unmerged implementation candidate rather than shipped behavior.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review, refine, and land or supersede PR #67108 with a narrow change in `src/infra/tmp-openclaw-dir.ts` that treats expected concurrent permission repair as success by re-checking `resolveDirState`, including fallback and already-secure owned-directory cases, plus targeted regression coverage in `src/infra/tmp-openclaw-dir.test.ts`.
