## Summary

Keep open. Current `main` and the latest shipped release still return the rotated plaintext device token in the `device.token.rotate` RPC success payload, and the linked fix PR (#66794) was closed unmerged after review identified a self-rotation lockout regression.

## Risks / Open Questions

- The secret-exposure behavior remains present on current `main` and in the latest release.
- Closing as implemented would be incorrect because no fix is merged or shipped.
- A safe fix likely needs a different design than naive response redaction, because self-rotation currently depends on receiving the new token before the server disconnects the old session.
