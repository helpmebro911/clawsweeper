## Summary

Keep #38138 open. Current main still resolves the Discord route from static account/guild/role/peer/parentPeer inputs before mention detection, and Discord preflight only uses the selected effective route's agentId to build mention regexes and gate the already selected route. The recent multi-agent docs rewrite documents parentPeer as a routing tier but does not add a mention override, and the touched Discord/routing files are unchanged since the prior keep-open review.

## Best Possible Solution

Keep the issue open. Implement a Discord mention-based routing override before mention gating, either as a highest-priority route tier with explicit mention inputs or as a documented Discord preflight override that maps native bot identity and per-agent mentionPatterns to an agent. Add regression coverage for a Discord thread inheriting a parentPeer binding while explicitly mentioning another configured agent, then update the Discord and multi-agent docs to state the final precedence rules.
