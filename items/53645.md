## Summary

Keep open. Current main still appears to drop Feishu message-tool `replyTo` on the plugin `send` path: the generic outbound layer resolves `replyTo`, and lower-level Feishu delivery can use it, but `extensions/feishu/src/channel.ts` only maps it for `thread-reply`, not ordinary `send`.

## Best Possible Solution

Keep this issue open and fix it in the Feishu plugin owner module. `extensions/feishu/src/channel.ts` should treat explicit `replyTo`/`reply_to` on `action: "send"` as a normal inline `replyToMessageId`, while preserving `thread-reply` as the explicit `reply_in_thread` path. Add focused tests for Feishu message-tool `send` with explicit `replyTo` across text, presentation/card, and media sends, plus a guard that the card/presentation schema remains optional for media-only and text sends.
