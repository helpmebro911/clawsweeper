## Summary

Keep this PR open. Current main still lacks peer-aware outbound account selection in runMessageAction, and the PR is a focused fix attempt for a plausible multi-account Slack routing bug. It should not be merged as-is because the submitted lookup scans raw bindings, can include ACP bindings, and can misroute when multiple route bindings share the same peer under different agents/accounts.

## Best Possible Solution

Keep this PR open and request a revised implementation. The best path is to add or reuse a typed outbound account resolver in src/infra/outbound/message-action-runner.ts that preserves explicit --account/defaultAccountId precedence, derives peer id/kind from the normalized outbound target, filters route bindings via listRouteBindings, keeps resolved agent/account scoping, and adds regression tests for the reported Slack multi-account peer binding plus ACP-binding and same-peer collision cases.
