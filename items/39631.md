## Summary

Current main still surfaces failed mutating tool warnings as user-facing reply payloads. `edit` is treated as mutating, `messages.suppressToolErrors` only suppresses non-mutating failures, and Telegram still delivers `isError` payloads instead of suppressing them. The related fix PRs #39636 and #39717 were closed unmerged in the provided GitHub context, so they do not supersede the issue.

## Best Possible Solution

Keep the issue open until OpenClaw deliberately changes the agent reply payload policy. The likely fix belongs around `resolveToolErrorWarningPolicy` and `buildEmbeddedRunPayloads`: suppress or normalize failed mutating-tool warning payloads when a user-facing assistant reply already exists, retain a safe fallback warning when no assistant reply exists, keep full details in logs/debug surfaces, and update payload plus Telegram dispatch coverage for the channel-visible behavior.
