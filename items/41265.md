## Summary

Keep open. Current main still constructs Discord upload Blobs without forwarding the detected media content type, and the shared web-media loader still optimizes every non-GIF image through JPEG/PNG, so the PR's central WebP/MIME bug is not implemented on main. The PR is a focused bug-fix attempt for open issue #41256, but it still needs maintainer review and should address the existing review concerns around animation detection, APNG/WebP/AVIF preservation, and Discord media cap handling before merge.

## Best Possible Solution

Keep the PR open and have maintainers either finish this branch or supersede it with a focused Discord/media fix: propagate `media.contentType` into every Discord upload Blob path, add a narrow preservation contract for Discord WebP/animated formats without regressing other channels, and cover WebP MIME preservation plus actual animated-image detection with tests. The linked bug #41256 should remain the canonical user-facing tracker until that lands.
