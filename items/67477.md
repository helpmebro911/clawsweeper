## Summary

Keep this PR open. Current main already implements the `message_sending` cancel/transform part for inbound auto-replies, shipped in 2026.4.24 via #70118, but the WhatsApp auto-reply path still does not emit plugin `message_sent` or internal `message:sent` hooks. The PR therefore still contains unique behavior, and its diff has unresolved correctness and public SDK-surface review concerns, so it is not a conservative cleanup close.

## Best Possible Solution

Keep this PR open, but ask the author or a maintainer to rebase and narrow it around the still-missing WhatsApp auto-reply sent-hook parity. The `message_sending` portion should reuse the shipped shared dispatcher behavior from #70118; the remaining implementation should emit plugin `message_sent` and internal `message:sent` only with accurate delivered content, success/failure state, message IDs where available, and session/account/conversation/group metadata, with focused tests for cancellation, rewrite, no-op, media fallback, and failure paths.
