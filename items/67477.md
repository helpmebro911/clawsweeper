## Summary

Keep this PR open. Current main has shipped the shared inbound `message_sending` before-delivery path, but WhatsApp auto-reply delivery still does not emit plugin `message_sent` or internal `message:sent` hooks, which is unique remaining work from this PR.

## Best Possible Solution

Keep this PR open, but rebase and narrow it around the still-missing WhatsApp auto-reply sent-hook parity. The already-shipped `message_sending` behavior should rely on the shared dispatcher path from #70118. The remaining implementation should emit plugin `message_sent` and internal `message:sent` only from accurate delivered outcomes, ideally by reusing or extracting the existing canonical sent-hook emitter, with focused tests for cancellation, rewrites, blanked content, chunking, media fallback, and failure paths.
