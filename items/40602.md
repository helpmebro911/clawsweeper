## Summary

Keep this PR open. Current main still deletes new Feishu Bitable placeholder rows only when `fields` is missing or has zero keys, while this PR targets the still-missing case where Feishu returns placeholder records whose fields are present but contain default empty values. The PR is narrow and not superseded by the related Feishu Bitable delete/permission work.

## Best Possible Solution

Keep this PR open as a plausible focused Feishu plugin bug fix. The best maintainer path is to add a small regression test around the `feishu_bitable_create_app` cleanup path or a narrow cleanup helper, covering present-but-empty Feishu default values plus meaningful counterexamples such as non-empty text, links, files, users, locations, booleans, and numbers, then merge the conservative predicate if the test proves it only deletes placeholder rows.
