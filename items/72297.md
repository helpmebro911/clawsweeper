## Summary

Keep this PR open. Current main still does not solve the linked QMD name-mismatch failure when `qmd collection list --json` exposes only collection names, and this PR is a focused implementation candidate for open #71783. The current PR patch is useful but should be revised before merge because it removes a QMD collection based on an unverified name parsed from stderr.

## Best Possible Solution

Keep this PR or the linked issue active, but revise the implementation before merge: only automatically rebind a different-named QMD collection after OpenClaw can verify through structured QMD metadata that the stale collection owns the same managed path and pattern. If QMD cannot expose that metadata, keep refusing automatic removal, surface a clearer warning/manual repair, and add regression coverage for both the #71783 `Name: life-main` conflict and refusal to remove an unverified stderr-only name.
