## Summary

Keep this PR open. Current main still relies on structured QMD list metadata for destructive rebinds and preserves the name-only skip path, while this PR is a recent active implementation candidate linked to open #71783. The latest PR revision removed the earlier stderr-only removal risk, but it still needs maintainer review because the verified-metadata fallback may be redundant and may not solve the original name-only QMD case.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to either revise it so OpenClaw can obtain authoritative QMD metadata for the `Name:` conflict before removing anything, or explicitly accept verified-metadata-only behavior and keep #71783 open for the remaining name-only QMD limitation/manual repair path. Do not close this as cleanup while it is an active focused candidate for the linked memory-core bug.
