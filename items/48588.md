## Summary

Keep open. Current `main` still has the Feishu HTTP cleanup pattern this PR targets: both `stopFeishuMonitorState()` and `monitorWebhook()` call `server.close()` and immediately clear shared state without awaiting the close callback. The PR remains relevant to linked issue #48183, but maintainers should require the remaining shutdown API/test coverage concerns to be addressed before landing.

## Best Possible Solution

Keep #48588 open as the active focused fix for #48183. Refresh it against current `main`, keep the shared awaited HTTP close helper for both Feishu webhook cleanup paths, update the public stop wrapper if maintainers want an awaitable restart contract, and add a targeted regression test for webhook server cleanup/restart ordering.
