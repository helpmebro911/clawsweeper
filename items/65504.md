## Summary

Keep this issue open. Current main includes partial mitigations for auth-profile timeout poisoning and deferred post-turn maintenance, but the live gateway and `agent.wait` contract still flatten results into `ok|error|timeout` without queue/preflight/provider/post-turn/gateway-draining attribution.

## Best Possible Solution

Keep this open as the OpenClaw tracking item for an additive timeout-attribution contract. The best maintainer path is to split the useful pieces from the closed draft into focused PRs that add phase-aware timing/timeout metadata to embedded-run telemetry, lifecycle events, gateway dedupe snapshots, `agent.wait` responses, protocol schema/docs, caller types, and regression tests, while leaving lossless-claw scheduling work to #54488 or the external lossless-claw PR.
