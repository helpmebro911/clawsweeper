## Summary

Keep this issue open. Current main has useful partial fixes for auth-profile timeout poisoning and deferred post-turn maintenance, but the central live gateway contract remains flat: `agent.wait`, lifecycle snapshots, dedupe snapshots, protocol docs, and caller types still expose `ok|error|timeout` without queue/preflight/provider/post-turn/gateway-draining attribution.

## Best Possible Solution

Keep this issue open as the OpenClaw tracking item for an additive timeout-attribution contract. The best path is to rework the useful OpenClaw pieces from #65511 into smaller PRs that add phase-aware timing/timeout metadata to embedded-run telemetry, lifecycle events, gateway dedupe snapshots, `agent.wait` responses, protocol schema/docs, caller types, and focused tests, while leaving owner-specific lossless-claw scheduling work to #54488 or the external lossless-claw PR.
