## Summary

Keep this issue open. Current `main` still leaves the central PTY/background exec lifecycle problem unresolved: supervisor ownership is in-memory, `reconcileOrphans` is still a no-op, background process visibility is backed by in-memory bash session maps, and sessions are documented as lost on process restart. The related #66001 PR is open, unmerged, authored by the same reporter, and uses closing syntax for this issue, so it should be reviewed or closed before this issue is resolved.

## Best Possible Solution

Review #66001 as the active implementation candidate. Maintainers should decide whether the desired contract is in-process orphan reconciliation, durable startup reaping or re-registration for stale PTY exec trees, explicit scope replacement/watchdogs for long-lived PTY background runs, or a combination, then land focused regression coverage for PTY exec ownership, process-list visibility, and lost-session cleanup.
