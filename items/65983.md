## Summary

Keep this issue open. An open same-author implementation PR, #66001, explicitly uses closing syntax for this issue and is not merged. Current main still matches the reported ownership gap: process-supervisor ownership is in memory, `reconcileOrphans()` remains a deliberate no-op, `process list` is driven from the in-memory bash process registry, and background PTY exec can run without an effective timeout when background timeout bypass applies.

## Best Possible Solution

Review #66001 as the implementation candidate. Maintainers should either land a reconciliation fix, broaden it to durable restart cleanup if that is the intended lifecycle contract, or close the PR with an explicit product decision explaining why untracked background PTY exec process trees are acceptable. Only after that should this issue be closed.
