## Summary

Keep open. Current main still has the reported lifecycle gap: supervisor orphan reconciliation is a deliberate no-op, exec/process session ownership is in-memory, and the generic PTY exec path can background without a timeout or watchdog. The same author has an open implementation candidate, #66001, that uses GitHub closing syntax for this issue, so this should remain open until that PR is reviewed, merged, or closed.

## Best Possible Solution

Review #66001 as the active implementation candidate. Maintainers should decide whether OpenClaw wants durable cold-start orphan reaping for PTY exec runs or the smaller in-process reconciliation contract, then land the chosen fix with focused regression coverage around lost session ownership, PTY background runs, and process-list visibility.
