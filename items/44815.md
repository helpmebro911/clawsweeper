## Summary

Keep open. Current `main` does not contain this Feishu placeholder fix. `parseMessageContent` still returns raw JSON for top-level non-text media messages, and that content is still propagated into the agent-facing body while the saved media path is attached separately. The underlying mismatch described by the PR is therefore not implemented on `main`, and the broader media-text gap noted in review comments also remains.

## Evidence

- **Top-level Feishu parser still falls through to raw JSON:** On `main`, `parseMessageContent` only special-cases `post`, `text`, `share_chat`, and `merge_forward`.
