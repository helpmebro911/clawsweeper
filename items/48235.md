## Summary

Keep #48235 open. Current main still does not implement the scheduled daily/idle reset continuity hint, and the live PR remains a relevant but incomplete implementation rather than something already shipped or clearly superseded.

## Best Possible Solution

Keep the PR open for maintainer or author follow-up. The best implementation belongs in the core session rollover path: capture a bounded, sanitized continuity note only for scheduled daily/idle resets, never explicit `/new` or `/reset`; inject it exactly once into the prompt path that actually reaches immediate and queued runs; clear it only after delivery; and add regression tests for daily, idle, explicit reset, multiline assistant text, and queue modes.
