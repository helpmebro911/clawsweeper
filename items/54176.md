## Summary

Current main still does not classify legacy Word/OLE MIME types as binary in the media-understanding file auto-embed path, and open PR #54380 is the unmerged implementation candidate that says it fixes this issue. This should remain open until that PR, or an equivalent scoped fix with regression tests, lands.

## Best Possible Solution

Review and land #54380, or an equivalent scoped patch, that adds `application/x-cfb` and `application/msword` to `src/media-understanding/apply.ts` and adds focused regression coverage in `src/media-understanding/apply.test.ts`. After that merges, this issue can close as implemented.
