## Summary

Current main still does not treat legacy Word/OLE MIME types as binary in the media-understanding auto-embed path, and the provided GitHub context shows open PR #54380 is an unmerged implementation candidate with closing syntax for this issue. The issue should remain open until that PR or an equivalent scoped fix lands with regression coverage.

## Best Possible Solution

Keep this issue open and review/land #54380, or an equivalent scoped patch, that adds `application/x-cfb` and `application/msword` to the media-understanding binary MIME guard in `src/media-understanding/apply.ts` and adds focused regression tests in `src/media-understanding/apply.test.ts`. After the PR merges, close this issue as implemented by the landed commit.
