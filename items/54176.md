## Summary

Keep this issue open. Current main still does not treat `application/x-cfb` or `application/msword` as binary in the media-understanding auto-embed path, and the provided GitHub context shows open PR #54380 as the unmerged implementation candidate for this issue.

## Best Possible Solution

Review #54380, ensure the media-understanding binary guard treats both `application/x-cfb` and `application/msword` as binary attachments, add focused regression tests in `src/media-understanding/apply.test.ts`, then land it or close it with maintainer rationale. After that lands, this issue can close as implemented.
