## Summary

Current main still has the prompt-contract mismatch reported here: the system prompt emits `## Current Date & Time` but only includes `Time zone: ...`. The separate `session_status` hint and gateway message timestamps help temporal grounding, but they do not rename or clarify the misleading section. The existing ClawSweeper comment on this issue remains consistent with current main.

## Best Possible Solution

Keep this issue open and fix the narrow prompt-contract mismatch directly. Preserve cache-stable timezone-only behavior, but rename the system prompt section to `Time Zone` or add adjacent text saying current date/time is not included and should be obtained through the appropriate runtime status path. Update `src/agents/system-prompt.test.ts`, `docs/concepts/system-prompt.md`, `docs/date-time.md`, and stale timezone wording so implementation, tests, and docs agree.
