## Summary

Keep open. The linked Control UI bug remains present on current main: the chat refresh button is still enabled during active turns because its disabled binding only checks loading/disconnected state. The PR’s intended one-line UI fix is still useful, but the branch also contains unrelated Slack provider behavior and a new Slack test with unresolved review concerns, so the best maintainer path is to split or revise rather than close as already handled.

## Best Possible Solution

Keep the item open and ask for a narrow revision: land the Control UI disabled guard for the refresh button, preferably with a small render test asserting the button is disabled for `chatSending`, `chatRunId`, and `chatStream`, and move the unrelated Slack provider changes into a separate PR that addresses the runtime status/test concerns.
