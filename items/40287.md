## Summary

Keep this PR open. Current main still lacks the requested Slack `file_shared` Socket Mode fallback, setup/docs subscription entry, and focused tests. The PR is stale because it targets the old `src/slack` layout, but the central behavior remains missing and the PR is the canonical remaining tracker after related items were closed as superseded by it.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate, but require a rebase or replacement port into `extensions/slack`: add a Slack `file_shared` event handler under `extensions/slack/src/monitor/events`, register it from `extensions/slack/src/monitor/events.ts`, hydrate the matching Slack message safely, dedupe against normal message delivery and retries, preserve existing team/account/channel/user authorization and media SSRF protections, update the generated setup manifest and `docs/channels/slack.md`, and add focused Slack plugin tests for large-file and multi-file fallback behavior.
