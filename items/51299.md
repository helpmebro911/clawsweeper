## Summary

Keep open. Current main still implements the legacy iMessage path as a direct headless `imsg rpc` child process, and the docs still describe the same LaunchAgent/headless permission model the report says fails on Sequoia. The newer macOS TCC-safe path exists for `system.run` through the GUI app, but the iMessage plugin does not use that path. No iMessage runtime code changed between `v2026.4.22` and current `main`.

## Evidence

- **iMessage docs still describe headless `imsg` execution and same-context prompts:** The current iMessage docs say the gateway "spawns `imsg rpc`" and instruct headless LaunchAgent/SSH users to run `imsg chats` or `imsg send` in that same context to trigger TCC prompts.
