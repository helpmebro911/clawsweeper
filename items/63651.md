## Summary

Keep this PR open. Current main still has the Windows schtasks restart stdout writes, and the stale-PID retry path can call the same restart implementation again. The PR addresses a plausible real bug, but it is stale/mixed and should be narrowed before merge.

## Best Possible Solution

Keep this PR open, but require a focused gateway-only update: rebase onto current main, drop or split the browser CDP/SSRF changes, preserve exactly one human completed-restart success message, de-duplicate stdout across stale-PID retries, and add a targeted regression test for the retry/output behavior.
