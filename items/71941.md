## Summary

Current main still has the reported `agents add` wizard behavior: the workspace prompt is seeded from `resolveAgentWorkspaceDir`, and that resolver nests non-default agents under `agents.defaults.workspace` when a fallback workspace is configured. That conflicts with the multi-agent docs' peer-level `workspace-<agentId>` convention. PR #71941 is open, unmerged, and directly addresses the linked bug #71889, so it is not a conservative close candidate.

## Best Possible Solution

Keep PR #71941 open and ask for a small update: use the peer-level suggestion only for brand-new agents in the documented default-install case, preserve existing-agent and custom shared-root behavior, fix the `agents.add.test.ts` environment stub, correct the changelog attribution, then run the relevant changed checks and close #71889 through the merged PR.
