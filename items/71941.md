## Summary

Keep PR #71941 open. Current main still seeds the interactive `agents add` workspace prompt from `resolveAgentWorkspaceDir(cfg, agentId)`, and that resolver still appends a non-default agent id under `agents.defaults.workspace`, which conflicts with the documented peer-level `workspace-<agentId>` layout. The PR is a focused fix for a real remaining behavior, but it needs the existing review comments addressed before merge.

## Best Possible Solution

Keep the PR open, ask the contributor to address the test portability and changelog review comments, then land an equivalent narrow wizard-prefill fix that preserves existing runtime workspace resolution and existing-agent update behavior while adding regression tests for the documented default, custom shared roots, state-dir isolation, and update flow.
