## Summary

Keep this issue open. Current main still routes direct Google Gemini through tagged reasoning and still injects the reported <think>/<final> prompt without a native tool-call exception, while the Google transport already sends functionDeclarations and parses functionCall parts. The central prompt/tool-call interference is therefore not implemented or superseded on main.

## Best Possible Solution

Keep this open and fix the Google provider reasoning-output path. The maintainer direction should be either to stop using tagged reasoning for direct google-generative-ai when native reasoning/tool calls are available, or to make the tagged prompt explicitly state that provider function calls are outside the <think>/<final> text contract and must be emitted through native function calling. Add focused regression coverage for Google resolveReasoningOutputMode/system-prompt composition with tools, while preserving transport tests proving functionDeclarations are sent and functionCall parts are parsed.
