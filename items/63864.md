## Summary

Keep open. Current main still hard-codes new lanes to concurrency 1, and session lanes wrap embedded agent turns, so the requested capability is not implemented on main. The PR is a relevant implementation candidate for the session-concurrency part of #53023, but it is not safe to close or merge as-is because the public config key is missing from the strict schema/config artifacts and the diff removes unrelated gateway reload, SDK export, and public type surface behavior.

## Best Possible Solution

Keep this PR open for contributor revision or maintainer reimplementation. The narrow path is to rebase on current main, preserve existing gateway reload/security code and SDK/type exports, add a settled session-lane concurrency key to the config type, strict zod schema, defaults/resolver, generated config metadata/help/docs, and tests, then wire startup and hot reload through a setter that updates existing session:* lanes. Leave #53023 open for auto-yield and lane-wait-threshold work unless those are implemented separately.
