## Summary

Keep this PR open. Current main still limits the bundled session-memory hook to explicit command:new and command:reset events, while daily/idle automatic rollover is handled through newer session lifecycle plumbing that is not wired to this bundled internal hook. The PR remains a plausible fix, but it needs a rebase/adaptation to current main and should address the unresolved ACP duplicate-write review concern before merge.

## Best Possible Solution

Keep this PR open for maintainer/contributor follow-up. Rebase it onto current main, decide whether bundled session-memory should consume the existing session_end lifecycle payload or a new typed internal session:rollover event, then update handler metadata/docs/tests so daily and idle rollover snapshots are persisted without double-writing explicit ACP reset flows.
