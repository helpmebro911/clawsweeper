## Summary

Keep this issue open. Current main still exposes Discord scheduled-event list/create only; event-edit and event-delete are absent from the shared message-tool contract, Discord discovery, API mapping, runtime dispatch, REST helpers, CLI surface, docs, and tests. The reporter's follow-up accurately identifies the bundled Discord implementation path and the remaining work.

## Best Possible Solution

Keep this issue open and implement event-edit/event-delete as coordinated shared message-tool and bundled Discord plugin work: add the actions to the shared action list, target metadata, display metadata, schema, CLI, and docs; add eventId to the event parameter schema; expose the actions from Discord discovery under the existing events gate; add Discord API mapping and runtime dispatch; add helpers for scheduled-event fetch/update/delete using discord-api-types patch/delete contracts; preserve cover-image handling; and add focused unit tests for parser/runtime/REST payload behavior.
