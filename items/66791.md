## Summary

Keep this issue open. Current main still only exposes Discord scheduled-event list/create, and the shared message-tool contract still lacks `event-edit`, `event-delete`, and `eventId` support. The reporter's follow-up accurately identifies the existing Discord implementation path and the missing lifecycle actions.

## Best Possible Solution

Keep this issue open and implement the missing lifecycle actions as a coordinated shared-contract plus bundled Discord plugin change: add `event-edit` and `event-delete` to the shared message action list, schema, display/target metadata, and Discord discovery; add Discord API mapping and runtime actions; add REST helpers for scheduled-event fetch/update/delete using Discord PATCH/DELETE endpoints; preserve existing event fields safely before PATCH; add focused Discord tests; and update the public Discord docs at https://docs.openclaw.ai/channels/discord.
