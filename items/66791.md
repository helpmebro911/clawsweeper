## Summary

Keep open. Current `main` still supports Discord scheduled events only for `event-list` and `event-create`; there is no `event-edit` or `event-delete` path in the Discord action registry, parser, runtime dispatcher, REST helpers, CLI, or docs. The reporter’s follow-up comment matches the current extension structure, so this is a coherent enhancement request, not an obsolete or ClawHub-scoped item.

## Risks / Open Questions

- Closing this would hide a real gap in the existing Discord event lifecycle: agents can create and list scheduled events, but cannot update or remove them on current `main`.
- A proper fix is additive across multiple existing Discord-plugin surfaces (action registry, parser, runtime, REST helpers, CLI/docs, and likely tests), so treating it as already handled would be misleading.
