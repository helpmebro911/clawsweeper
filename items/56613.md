## Summary

Keep open. Current main implements the per-agent TTS half through `agents.list[].tts`, but the Talk/Voice session-selection half remains missing on the mobile Talk/Voice path. Android Voice still has no agent/session picker and both manual mic and Talk Mode route through the resolved main/default session key; the iOS Voice/Talk path shows the same main-session model. A maintainer comment in the issue also explicitly kept this open for the remaining Talk/Voice UI session-selection work.

## Best Possible Solution

Keep this issue open for the remaining Talk/Voice agent/session-selection work. The likely implementation is to add selected voice session/agent state to Android and iOS/macOS Voice/Talk UI, wire that selected key into `TalkModeManager` and manual mic sends instead of only the resolved main/default key, reuse or adapt the existing Chat session picker/filtering pattern, add focused mobile routing/UI tests, and preserve the shipped `agents.list[].tts` per-agent TTS configuration rather than adding the proposed legacy `talk.voiceId` shape.
