## Summary

Keep open. Current `main` does not implement bearer-token auth for the standard `amazon-bedrock` Converse provider requested here. `AWS_BEARER_TOKEN_BEDROCK` is recognized for auth-source/discovery metadata, but the actual Bedrock model-call path still uses the AWS SDK default credential chain. Bearer-token execution is implemented for the separate `amazon-bedrock-mantle` provider instead.

## Evidence

- **Standard Bedrock docs still say no API-key auth:** `docs/providers/bedrock.md:9`-`18` says the bundled `amazon-bedrock` provider uses pi-ai's Bedrock Converse path and that auth is the AWS SDK default credential chain, 'not an API key.'
  - file: [docs/providers/bedrock.md:9](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/providers/bedrock.md#L9)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Docs explicitly say runtime Bedrock calls still use AWS SDK auth:** `docs/providers/bedrock.md:149`-`155` states that even when `AWS_BEARER_TOKEN_BEDROCK` is resolved early, 'The actual model-call auth path still uses the AWS SDK default chain.'
  - file: [docs/providers/bedrock.md:149](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/providers/bedrock.md#L149)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Model-auth code treats bearer token as AWS-SDK source metadata:** `src/agents/model-auth.ts:356`-`388` maps `AWS_BEARER_TOKEN_BEDROCK` to `mode: "aws-sdk"` and a source label; it does not resolve a bearer token into a direct Bedrock request auth mode for the standard provider.
  - file: [src/agents/model-auth.ts:356](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/model-auth.ts#L356)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Bearer-token implementation is isolated to Mantle, not standard Bedrock:** `docs/providers/bedrock-mantle.md:16`-`19` documents explicit `AWS_BEARER_TOKEN_BEDROCK` auth for `amazon-bedrock-mantle`, and `extensions/amazon-bedrock-mantle/package.json:7`-`10` includes `@aws/bedrock-token-generator` while `extensions/amazon-bedrock/package.json:7`-`10` does not.
  - file: [docs/providers/bedrock-mantle.md:16](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/providers/bedrock-mantle.md#L16)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
- **Changelog only claims bearer-token work for Mantle:** `CHANGELOG.md:1166` and `CHANGELOG.md:1181` record Mantle support and Mantle IAM bearer-token generation; there is no corresponding shipped changelog entry for bearer-token auth on the standard `amazon-bedrock` Converse provider.
  - file: [CHANGELOG.md:1166](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/CHANGELOG.md#L1166)
  - sha: [c6684af682bc](https://github.com/openclaw/openclaw/commit/c6684af682bcb59bd1ee85cb49f72d4074007e2c)
