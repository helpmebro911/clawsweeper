## Summary

Keep #38808 open. Current main still preserves Feishu `root_id` and `thread_id` on inbound events, but the reply-routing behavior remains group-only: direct/P2P messages force `replyInThread=false`, do not select `root_id` as the thread reply target, set `threadReply=false`, and strip reply metadata before sending. The linked bug #38806 is still open, and related replacement PRs #38814 and #38996 are closed unmerged, so this PR is not obsolete.

## Best Possible Solution

Keep #38808 open and either rebase/merge it or reimplement the same narrow Feishu plugin fix on current main: detect `isDirect && Boolean(ctx.threadId)`, use `ctx.rootId` for P2P thread reply targeting, set `replyInThread` and `threadReply`, preserve reply metadata by not setting `skipReplyToInMessages` for that direct-thread case, keep plain P2P quote replies unchanged, and land the focused regression tests. Treat #38808 as the canonical fix for #38806 unless a maintainer lands a newer replacement.
