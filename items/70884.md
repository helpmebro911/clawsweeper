## Summary

Keep this PR open. Current main still has the documentation gap the PR targets: trusted-proxy WebSocket sessions without device identity can connect but lose self-declared scopes, while the trusted-proxy and protocol docs do not explain that consequence or clearly separate HTTP `x-openclaw-scopes` behavior from WebSocket handshake scopes. The PR discussion supports the docs direction, and the later PR commits address the important review feedback around break-glass wording and the local `allowInsecureAuth` exception.

## Best Possible Solution

Keep this PR open and review or land it, or land an equivalent docs update in `docs/gateway/trusted-proxy-auth.md` and `docs/gateway/protocol.md` that explicitly separates HTTP `x-openclaw-scopes` behavior from WebSocket handshake scopes, documents scope clearing without device identity, preserves the local `allowInsecureAuth` exception, and frames `dangerouslyDisableDeviceAuth` as break-glass only.
