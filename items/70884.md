## Summary

Keep PR #70884 open. Current main still has the trusted-proxy docs gap: the docs say Control UI trusted-proxy sessions can connect without device identity and describe `x-openclaw-scopes`, but they do not explain that the header is HTTP-only or that device-less WebSocket scopes are cleared. The runtime and tests support the PR’s clarification, and the provided PR context shows review feedback was addressed in later PR commits rather than leaving this as stale bot churn.

## Best Possible Solution

Keep PR #70884 open for maintainer review/merge. The best docs path is to add the HTTP-only `x-openclaw-scopes` boundary, document WebSocket scope clearing without device identity, include the local `allowInsecureAuth` exception, and keep `dangerouslyDisableDeviceAuth` framed as a critical break-glass downgrade.
