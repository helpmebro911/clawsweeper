## Summary

Keep this PR open. Current main still has the docs gap: the trusted-proxy docs describe `x-openclaw-scopes` as an HTTP trusted-proxy header without clearly steering WebSocket readers away from it, and the WebSocket protocol docs list device-less exceptions without explaining the scope-clearing consequence. The PR is a focused docs-only correction for real gateway behavior and is not superseded by current main.

## Best Possible Solution

Review and land this PR, or land an equivalent docs update in `docs/gateway/trusted-proxy-auth.md` and `docs/gateway/protocol.md` that clearly separates HTTP `x-openclaw-scopes` behavior from WebSocket handshake scopes, documents scope clearing without device identity, preserves the local `allowInsecureAuth` token/password exception, and keeps `dangerouslyDisableDeviceAuth` framed as break-glass only.
