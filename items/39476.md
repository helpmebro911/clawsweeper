## Summary

Keep #39476 open. Current main still exposes requester-session context to A2A target/reply/announce prompts, still runs the normal reply-back ping-pong plus announce flow for peer `sessions_send`, and still has no per-run hard guard that removes or denies `sessions_send` during nested A2A turns. The existing ACP parent-owned-child skip is narrower than this report, and #44309 is a broader dispatch-only feature request rather than a shipped fix for reverse-call duplicate delivery.

## Best Possible Solution

Keep the issue open and fix the core A2A `sessions_send` path. Add a framework-level guard that prevents A2A target/reply/announce turns from calling `sessions_send` back to the requester, narrow or remove requester-session prompt exposure where feasible, preserve the existing parent-owned ACP-child skip, keep #44309 as the broader dispatch-only semantics discussion, and add regression coverage for normal peer A2A duplicate visible delivery.
