## Summary

Keep #12398 open. Current main still lacks a first-class cron postcondition contract, deterministic post-run verifier, durable verifier result state, or a cron-owned way for a clean isolated `agentTurn` to be converted into retry/escalation based on an external check. The prior issue discussion also explicitly kept this open, and the only commit since that review is WhatsApp-specific, not cron-related.

## Best Possible Solution

Keep #12398 open for maintainer/product design and implementation. The likely core path is an additive cron job contract, gateway/CLI/schema/docs/UI updates, deterministic verifier execution after isolated agent completion and before `applyJobResult()`, durable verifier state and run-log fields, bounded retry prompt injection with checker output, and integration with cron retry/failure-alert delivery. If maintainers prefer plugin ownership, first add a supported cron post-run hook/API that can fail a run and request cron-owned retry/escalation.
