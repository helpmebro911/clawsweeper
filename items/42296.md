## Summary

Keep open. Current `main` still leaks raw Feishu media JSON into agent-visible fields, so the underlying bug is not implemented on `main`; however, PR #42296's approach would regress rich-text `post` messages by overwriting parsed human text with placeholders only.

## Evidence

- **Raw media JSON still passes through parseMessageContent:** `parseMessageContent()` only special-cases `post`, `text`, `share_chat`, and `merge_forward`. For `image`/`audio`/`video`/`file`, it falls through and returns the original JSON payload string.
