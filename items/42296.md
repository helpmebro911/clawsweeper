## Summary

Current `main` still feeds raw Feishu media JSON into agent-visible body fields, so the underlying bug is not implemented away. PR #42296 should stay open because its proposed overwrite of `ctx.content` would regress rich-text `post` messages by dropping their human text whenever embedded media is present.

## Evidence

- **Current main still returns raw JSON for non-post Feishu media messages:** `parseMessageContent` only special-cases `post`, `text`, `share_chat`, and `merge_forward`; every other message type falls through to `return content`, so `image`/`audio`/`video`/`media` bodies still surface raw JSON like `{\"image_key\":...}` on current `main`.
  - file: [extensions/feishu/src/bot-content.ts:132](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/extensions/feishu/src/bot-content.ts#L132)
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Agent-visible Feishu fields still use `ctx.content` directly:** `buildFeishuAgentBody` starts from `ctx.content`, and `finalizeInboundContext` passes that same `ctx.content` into `RawBody` and `CommandBody`, so any raw media JSON parsed earlier still reaches the model-facing context on current `main`.
  - file: [extensions/feishu/src/bot.ts:160](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/extensions/feishu/src/bot.ts#L160)
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Rich-text `post` messages intentionally preserve text plus media markers:** `parsePostContent` renders embedded images/media as `![image]` / `[media]` while also preserving surrounding paragraph text and title into `textContent`. Replacing `ctx.content` wholesale with joined placeholders would discard user instructions from mixed text+media posts.
  - file: [extensions/feishu/src/post.ts:152](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/extensions/feishu/src/post.ts#L152)
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **The PR already has an unresolved correctness review:** The existing review on PR #42296 points out that overwriting `ctx.content` after media resolution drops textual content for `post` messages, e.g. a chart image plus the user's instruction becomes only `<media:image>`. That concern matches current `post` parsing behavior in `main`.
  - command: `curl -L -s https://api.github.com/repos/openclaw/openclaw/pulls/42296/comments`
  - sha: [b346dabd3a0e](https://github.com/openclaw/openclaw/commit/b346dabd3a0e9dbcdb0a19f409595c13913bb323)
- **Main already computes media placeholders separately:** `resolveFeishuMediaList` stores a `placeholder` for each downloaded Feishu media item, so there is a clear narrower path for fixing raw JSON leakage without clobbering `post` text. That makes this PR's current implementation look incomplete rather than obsolete.
  - file: [extensions/feishu/src/bot-content.ts:450](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/extensions/feishu/src/bot-content.ts#L450)
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
