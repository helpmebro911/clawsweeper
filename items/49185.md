## Summary

Current main has partial failover mitigations, but it does not implement the requested configurable SDK retry count per provider or per fallback candidate. The docs still state that OpenClaw lets provider SDKs handle normal short retries, and the implemented knobs only cap long Retry-After waits or limit same-provider auth-profile rotations after an error has surfaced.

## Evidence

- **Model retry docs keep SDK short retries:** The retry policy explicitly says model providers still let provider SDKs handle normal short retries; the exposed environment knob only caps long SDK Retry-After waits for Stainless-based SDKs.

This is not a per-provider maxRetries setting.
