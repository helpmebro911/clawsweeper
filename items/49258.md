## Summary

Keep open. Current main covers the normal successful isolated cron delivery case, but the reported contradictory state is still possible because cron persists execution status and delivery status independently: an isolated run can return `status: "error"` while still returning `delivered: true`, and `applyJobResult` will then increment `consecutiveErrors` instead of reconciling the state. The simple repro may need the condition called out in the issue discussion: a message is delivered, but the agent/run also emits an error payload.

## Evidence
