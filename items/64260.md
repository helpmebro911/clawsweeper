## Summary

Keep open. Current `main` still defines and emits `llm_output` without any `contextWindow`, effective `contextTokens`, or `maxTokens` fields, so the requested plugin hook payload is not implemented on core surfaces yet.

## Evidence

- **Hook type omits requested fields:** `PluginHookLlmOutputEvent` only exposes `runId`, `sessionId`, `provider`, `model`, optional `resolvedRef`/`harnessId`, `assistantTexts`, `lastAssistant`, and token usage. There is no context-window or max-token metadata in the public hook contract.

- **PI harness emission still lacks context metadata:** The main PI runner emits `llm_output` with `runId`, `sessionId`, `provider`, `model`, `resolvedRef`, optional `harnessId`, `assistantTexts`, `lastAssistant`, and `usage` only. No context limit or model `maxTokens` is threaded into the event.
