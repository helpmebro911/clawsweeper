## Summary

Keep open. Current main still implements the reported path: `openclaw update` refreshes a loaded gateway service by forcing `gateway install`, that install plan reads proxy variables from the invocation environment, and systemd rendering writes those non-empty variables as `Environment=` lines. The issue has no protected maintainer author/label, but the requested fix is not implemented on main.
