## Summary

Keep #25592 open. Current main has commentary/final_answer hardening, but it still allows ordinary unphased assistant text before or between tool calls to buffer, flush through block-reply delivery, and enter final payload assembly. The issue discussion also has recent reproduction reports and an Apr 26 maintainer review explicitly keeping this as the broader canonical bug.

## Best Possible Solution

Keep #25592 open as the canonical product bug until OpenClaw implements or explicitly rejects a channel-safe policy such as final-only channel delivery, configurable enforceFinalTag for channel sessions, or an opt-in replyMode/tool-only explicit-send mode. Add regression coverage for ordinary unphased assistant text before/between tool calls in both block-reply delivery and final payload assembly.
