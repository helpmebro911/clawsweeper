## Summary

Keep this issue open. Current main still backs up the whole OpenClaw state directory without exclude flags, smart defaults, size warnings, or backup progress for multi-GB state. Atomic temp-file cleanup fixes related partial-output symptoms, but not the large-installation archive scope problem. The linked ClawHub plugin `keepmyclaw` is a useful workaround, and open PR #44288 remains the implementation candidate for the built-in backup command.

## Best Possible Solution

Keep the issue open and use #44288 as the current implementation candidate. Review its exclude design, protected-path behavior, manifest compatibility, dependency impact, and tests. If #44288 is closed instead of landed, the core fix should still live in `src/infra/backup-create.ts`, `src/commands/backup-shared.ts`, `src/cli/program/register.backup.ts`, and `docs/cli/backup.md` as conservative exclude support for ephemeral state directories plus clearer progress or failure reporting for large archives. The `keepmyclaw` ClawHub plugin remains a useful external workaround, not a replacement for the built-in backup command fix.
