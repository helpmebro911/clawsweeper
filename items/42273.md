## Summary

Keep this issue open. Current main still does not provide built-in state-subdirectory exclusions, size/progress feedback, or a large-installation backup policy for `openclaw backup create`. The issue also has an open implementation candidate, #44288, whose PR body uses closing syntax for this issue, so the issue should remain open until that PR lands or maintainers choose a different fix.

## Best Possible Solution

Keep this issue open and review #44288 as the current implementation candidate. The core fix should live in the backup CLI/planner/archive path, adding conservative exclusion support for large ephemeral state directories plus clear progress or failure reporting. If #44288 is closed instead of landed, maintainers should still implement a focused built-in fix in `src/cli/program/register.backup.ts`, `src/commands/backup-shared.ts`, `src/infra/backup-create.ts`, and `docs/cli/backup.md`. The `keepmyclaw` ClawHub plugin is a useful workaround, not a replacement for fixing the built-in backup command.
