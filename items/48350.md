## Summary

Keep PR #48350 open. Current main does not include the proposed post-compaction validator or the prerequisite loop-aware guard config/scorer stack. The closest shipped behavior is the existing safeguard summary quality audit, but that runs before returning the compaction result and only checks required headings, identifier retention, and latest-user-ask overlap. It does not validate post-compaction counters, unresolved-item retention, stale-system promotion, repeated-failure collapse, or reset recommendation. The related umbrella #48238 and follow-up #48368 remain open rather than superseding this PR, and Ryce's snapshot/rollback comment is a useful design concern, not evidence that the PR is obsolete.

## Best Possible Solution

Keep the PR open or fold its validator into the broader #48238 loop-aware compaction guard plan. The best implementation path is to first settle the guard config/scorer/runtime hook stack, then add a post-compaction validator with tests for goal retention, unresolved items, stale-system promotion, repeated-failure collapse, compaction-effect evidence, and conservative reset recommendation, while separately deciding whether snapshot rollback is required before runtime adoption.
