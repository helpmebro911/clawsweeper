## Summary

Keep PR #71235 open. Current main does not implement the conditional `python` -> `python3` shim in any of the four Dockerfiles, while the relevant images still either install `python3` directly or allow it through the documented Docker apt package hook. The related ClawSweeper items cover optional dependency installation, apt retry hardening, workspace aliases, or missing-python guidance; none supersedes this focused Dockerfile shim PR.

## Best Possible Solution

Keep the PR open and have maintainers review or refresh it against current main. The best implementation is the PR's v2 shape: add the conditional shim to all four Dockerfiles using `command -v python3` for the target and `! command -v python` for the guard, then add lightweight Docker smoke coverage where practical, especially for sandbox-common when `python3` is installed.
