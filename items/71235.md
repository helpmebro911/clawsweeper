## Summary

Keep open. Current main still ships Debian bookworm Docker images that install or allow `python3` without adding an unversioned `python` command, while PR #71235 is a narrow active Dockerfile fix and the author already incorporated the review suggestions about `command -v python3` and pre-existing `python` commands.

## Best Possible Solution

Keep #71235 open for maintainer review. The best path is a narrow Dockerfile compatibility patch in the shape of the PR's v2 guard, with Docker smoke proof if feasible; this belongs in the bundled Docker/sandbox surfaces rather than ClawHub because it changes base image ergonomics for OpenClaw's maintained container images.
