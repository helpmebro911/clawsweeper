## Summary

Keep #71235 open. Current main still installs or documents Python only as `python3` in the relevant Docker image paths and has no `python` shim in the root gateway Dockerfile or sandbox Dockerfiles. The PR is a focused Docker/runtime usability change, and the provided review context shows the author already addressed the robustness nits by using `command -v python3` for the symlink target and `command -v python` for the skip check.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to merge a narrowly scoped shim in the four Dockerfiles, ideally with a small Dockerfile text test or CI smoke assertion that verifies the guard uses `command -v python` and links to the resolved `python3` path without forcing Python into slim/no-Python variants.
