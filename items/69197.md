## Summary

Keep #69197 open. Current main still lacks a poll-active or equivalent suppression path, and the PR addresses a real exec/poll race that is not implemented elsewhere in the checked-out code. The PR should be revised rather than closed: the latest review concern about a boolean `pollActive` being racy for overlapping polls is valid enough to require a counter/token-style fix before merge.

## Best Possible Solution

Keep the PR open and ask for a small revision: replace the boolean active-poll flag with a counter/token on `ProcessSession`, suppress `maybeNotifyOnExit()` while the active poll count is greater than zero, mark `exitNotified` when poll consumes the terminal state, preserve normal notification when no poll is active, and add regression tests for the normal poll race, overlapping polls, poll timeout, and kill success/failure behavior.
