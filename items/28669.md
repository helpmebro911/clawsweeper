## Summary

Keep #28669 open. Current main e672b614 still lacks the requested `audio/webm`, `video/webm`, and `video/x-matroska` MIME-to-extension mappings, while the same helper is still used by saved-media IDs, outbound attachment filename inference, local media loading, and plugin-facing media helpers. The PR remains a narrow core bug fix, not a duplicate, ClawHub item, or obsolete change.

## Best Possible Solution

Keep the PR open. Rebase or recreate the narrow patch on current main, add `audio/webm -> .webm`, `video/webm -> .webm`, and `video/x-matroska -> .mkv` in `src/media/mime.ts`, and add focused regression coverage in `src/media/mime.test.ts`. A saved-media or outbound attachment filename test would also protect the user-visible behavior described in the PR.
