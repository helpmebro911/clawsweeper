## Summary

Keep #28669 open. Current main still does not include the requested WebM/Matroska MIME-to-extension mappings, and the affected helper is still used by saved-media and outbound filename inference paths. The PR remains a narrow core bug fix rather than a duplicate, ClawHub item, or obsolete change.

## Best Possible Solution

Keep the PR open. Rebase or recreate the narrow patch on current main, add audio/webm -> .webm, video/webm -> .webm, and video/x-matroska -> .mkv in src/media/mime.ts, then add focused regression coverage in src/media/mime.test.ts. A saved-media or outbound filename test would also protect the reported user-visible behavior.
