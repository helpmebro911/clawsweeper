## Summary

Keep this PR open for maintainer rework. Current main already fixes the generic `openclaw logs --follow` / `logs.tail` cross-date path, but the stated `openclaw channels logs` surface still reads the configured log file directly and lacks cross-date regression coverage. The PR also bundles unrelated cron warning work that is already represented by open #42245 for #42196.

## Best Possible Solution

Keep this PR open, but require rework before merge: split the cron warning into the existing #42196/#42245 path, then either close the logging portion as superseded for generic `openclaw logs --follow` or replace it with a focused `openclaw channels logs` patch that reuses a shared exact rolling-log resolver and adds regression tests for missing current-day files, current-day precedence, custom `logging.file`, and channel filtering.
