## Summary

Current main still has the TUI display-layer behavior reported here: the footer formatter renders a missing total token snapshot as `?` even when the context window is known, and the footer passes nullable `sessionInfo.totalTokens` directly into that formatter. Session listing can still produce the relevant state, because zero or missing totals are dropped while context tokens can be resolved from the entry, transcript, or model defaults. The related fixing PR #43044 is closed and unmerged, and the latest release tag has the same formatter behavior.

## Best Possible Solution

Keep this issue open. The best fix is to revive or reimplement the focused PR #43044 behavior in `src/tui/tui-formatters.ts`: when `context` is known and `total` is null or undefined, display `0/<ctx> (0%)` instead of `?/<ctx>`, then add a small regression test in `src/tui/tui-formatters.test.ts` and, if needed, a TUI session-actions test proving defaults plus missing totals render predictably.
