## Summary

Keep PR #44363 open. Current main already has partial cron observability and reliability behavior, but the PR’s central doctor-side reliability diagnostics and stuck/failed-job reporting are not implemented on main. The PR is also not safely closeable as duplicate/superseded: related issue #13593 remains open for broader cron execution logging/warnings, and this stale PR still contains potentially useful prior art even though it needs a major rebase/split before maintainer review.

## Best Possible Solution

Keep the PR open as useful prior art, but ask for a narrow cron-only rework: remove unrelated memory/agent-watchdog changes, rebase onto current main, implement doctor/CLI diagnostics using existing `runningAtMs`, `lastRunStatus`, `lastError`, run-log, and task-ledger data, add focused tests for stuck/failed-job reporting, and coordinate any remaining broad observability scope with #13593.
