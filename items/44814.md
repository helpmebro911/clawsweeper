## Summary

Keep open. Current main still has the reported hardcoded 5 MB WebChat/Control UI attachment limits, and there is no `gateway.webui.attachmentMaxBytes` or equivalent schema/type/docs support. The PR is stale relative to current main and likely needs expansion, but it is still a focused fix for a real remaining product issue; linked #44757 being closed does not make this PR obsolete.

## Best Possible Solution

Keep the PR open and ask for a rebase/current-main update: add a maintainer-approved public config key for WebChat/Control UI attachment size, wire all current gateway attachment call sites through one resolver/default, update schema/types/help/generated config docs, and add targeted regression tests proving larger configured uploads no longer fail on both `chat.send` and `agent.request`.
