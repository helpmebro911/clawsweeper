## Summary

Current main has prompt-cache retention, cache-ttl pruning, and heartbeat keep-warm guidance, but not the dedicated Cache TTL Warmer proposed in #70418. The related heartbeat divergence work addresses a different heartbeat behavior, while this issue asks for a distinct opt-in subsystem with its own config, minimal provider request, idle-skip logic, and no session/channel side effects.

## Best Possible Solution

Keep #70418 open for maintainer product/design judgment. If accepted, the likely path is a core agent/provider-runtime feature or a deliberately added extension API: opt-in per-agent scheduling, exact conversation-prefix reuse, configurable idle skip and cost controls, minimal provider request, discarded response, no session/channel delivery, and tests around Anthropic request shape and cache-control stability.
