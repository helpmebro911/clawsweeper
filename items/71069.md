## Summary

Keep this issue open. Current main still resets stored session model overrides that are outside the configured model allowlist, and the reset system event still does not name the rejected model ref. The exact reported Gemma/GGUF identifier is not documented or recognized in-tree; supported Gemma examples use provider-specific refs such as `google/gemma-4-26b-a4b-it`, `ollama/gemma4`, or `inferrs/google/gemma-4-E2B-it`.

## Best Possible Solution

Keep this issue open. Ask the reporter for the exact model/provider configuration and intended backend, then either improve the reset diagnostic to include the rejected ref and allowed/provider-specific refs, document the required Gemma/GGUF setup more directly, or add a canonical alias only if `Gemma4-26b-a4-it-gguf` is meant to map to a supported provider model.
