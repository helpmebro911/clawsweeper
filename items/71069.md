## Summary

Keep open. Current main still has the exact stored-override reset path that can emit the reported generic fallback message, while docs/tests show Gemma 4 support is provider-specific and exact-key/allowlist based. The issue is recent and salvageable as either a clearer UX/error-message fix or a provider-specific model alias/config clarification.

## Best Possible Solution

Keep this open as a focused model-selection UX/config issue. Best maintainer path: update the stored session override reset message to include the rejected normalized key and `/models` or `agents.defaults.models` guidance, then add a regression test for a disallowed stored override. If OpenClaw wants to support this exact Gemma GGUF spelling, add provider-specific documentation or alias/config guidance for the appropriate local provider instead of silently bypassing the allowlist.
