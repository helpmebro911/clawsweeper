## Summary

Keep open. Current main still has the reported reset behavior for persisted disallowed model overrides, and the issue is recent with a concrete message. The correct fix is likely clearer validation/error text, not simply accepting the exact `Gemma4-26b-a4-it-gguf` string.

## Best Possible Solution

Keep the issue open and tighten the disallowed persisted-model override path. A conservative fix would include the rejected model ref in the reset/system message, point users to `/models` or the allowlist docs, and add a regression test. Only add alias normalization for `Gemma4-26b-a4-it-gguf` if a real supported provider catalog maps that exact ID to a valid provider/model ref.
