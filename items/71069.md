## Summary

Keep #71069 open. Current main still emits the reported generic reset message when a stored model override is outside the agent allowlist, and the exact reported ref `Gemma4-26b-a4-it-gguf` is not a documented canonical Gemma 4 ref. The issue is recent and asks either for acceptance or a clearer rejection, so it is not a conservative auto-close.

## Best Possible Solution

Keep the issue open. Ask the reporter for the model/provider config and session override fields, plus whether the target is Google, Ollama, LM Studio, inferrs, or another GGUF backend. Then either improve the reset message to name the rejected ref and allowed refs, document the required provider/model setup more directly, or add a canonical alias/normalizer only if `Gemma4-26b-a4-it-gguf` is intended to map to a supported provider model.
