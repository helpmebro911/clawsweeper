## Summary

Keep open. Current main still hardcodes the bundled memory flush write target to `memory/${YYYY-MM-DD}.md`, the supported `agents.defaults.compaction.memoryFlush` config has no `pathTemplate` or equivalent field, and the memory-flush write wrapper enforces the resolved path at tool execution time. The issue is recent, non-maintainer-authored, unlabelled, and actionable as a core/bundled memory-plugin config request rather than a stale or duplicate cleanup candidate.

## Evidence
