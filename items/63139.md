## Summary

Keep open. Current main still dispatches `before_model_resolve` from inside each embedded fallback candidate attempt. Main has the partial `runId`/model context mitigation, but it still lacks public `attempt`/`isFallbackRetry` hook context and docs/tests do not establish a safe per-candidate contract for routing plugins.

## Best Possible Solution

Keep this issue open until OpenClaw chooses and ships a clear hook contract. The strongest fix is to run `before_model_resolve` once at the logical run boundary and feed that choice into `runWithModelFallback` as the primary candidate. A smaller compatible fix is to expose `attempt` and `isFallbackRetry` on `PluginHookAgentContext`, wire them from the fallback loop through embedded attempts, update public docs/types to state the hook is per fallback candidate, and add regression coverage with a routing plugin that preserves failover intentionally.
