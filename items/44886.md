## Summary

Current `main` still shows the mismatch reported in #44886: Telegram DM config accepts `direct.<peer>.tools`, and DM `direct` config is used for prompt/skill overrides, but the runtime tool-policy path still resolves only `group`/`channel` contexts and `groups` config, not Telegram `direct` config. This is not strong close evidence.

## Evidence

- **Telegram DM config still advertises per-DM tool policy:** `TelegramDirectConfig` still includes `tools` and `toolsBySender`, and the Telegram channel schema still accepts `direct` entries.
  - file: [src/config/types.telegram.ts:283](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/config/types.telegram.ts#L283)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Telegram runtime still loads DM `direct` config for inbound DM handling:** DM chats resolve `freshTelegramCfg.direct[chatId] ?? direct["*"]` and return that as `groupConfig`/`topicConfig`.
  - file: [extensions/telegram/src/bot-core.ts:544](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/telegram/src/bot-core.ts#L544)

- **DM prompt overrides still propagate:** Inbound Telegram context still derives `groupSystemPrompt` from `groupConfig`/`topicConfig` and writes it into `GroupSystemPrompt` even for non-group chats.
  - file: [extensions/telegram/src/bot-message-context.session.ts:305](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/telegram/src/bot-message-context.session.ts#L305)

  - file: [src/sessions/session-key-utils.ts:136](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/sessions/session-key-utils.ts#L136)

  - file: [src/agents/pi-tools.policy.ts:384](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/pi-tools.policy.ts#L384)
