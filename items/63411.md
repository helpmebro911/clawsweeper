## Summary

Current main still does not implement the requested streaming text/thinking repetition breaker or loop-specific context sanitization. The idle timeout is chunk-activity based, the shipped loop detector is tool-call scoped, and aborted partial assistant output is still explicitly retained for transcript visibility. The related open PR #39961 remains the implementation candidate rather than a reason to close this issue now.

## Best Possible Solution

Keep this issue open. Review or adapt #39961, or implement an equivalent core stream guard near the assistant stream subscription path: detect repeated text/thinking patterns while the stream is active, abort with explicit loop metadata, prevent buffered repetitive content from being drained as a final reply, and quarantine loop-aborted partial assistant output from future model replay while keeping transcript visibility intact.
