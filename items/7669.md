## Summary

Keep #7669 open. Current main still does not implement the requested npm `dev` dist-tag publish path: release automation exposes only `beta`/`latest`, real publishes are release-tag based, publish helpers and release-plan types only model stable/beta, tests cover stable/beta behavior, and the live npm registry has no `dev` dist-tag. The existing maintainer-side Codex review comment also reached the same keep-open conclusion.

## Best Possible Solution

Keep #7669 open until maintainers either implement a first-class main-snapshot npm `dev` publish path, including workflow input/trigger support, release-plan validation, publish-helper support, post-publish verification, and docs; or explicitly retire npm dev snapshots and update release-channel, updater, installer, and release docs so `dev` is documented as git-only.
