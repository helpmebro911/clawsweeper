## Summary

Keep #7669 open. Current main still does not provide a dev npm dist-tag publish path: the public OpenClaw NPM Release workflow only accepts beta/latest, the publish helper maps every non-latest request to beta, the release plan type excludes dev, and the live npm registry still has no openclaw@dev tag. The existing dev channel remains a git-main update path, so the issue's npm CI/CD acceptance criteria are still unmet.

## Best Possible Solution

Keep the issue open until release managers either implement a supported main-snapshot npm publish path for `openclaw@dev` with validation/postpublish checks, or intentionally retire the npm dev tag and update `https://docs.openclaw.ai/install/development-channels`, `docs/cli/update.md`, and release docs to say dev is git-only.
