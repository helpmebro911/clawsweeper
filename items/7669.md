## Summary

Keep #7669 open. Current main still has no first-class npm `dev` dist-tag release path: the public npm release workflow only exposes `beta`/`latest`, real publishes require release tags rather than moving `main`, the publish helper maps any non-`latest` request to `beta`, release-plan types only model stable/beta, and the live npm registry currently has no `dev` dist-tag.

## Best Possible Solution

Keep #7669 open until maintainers either implement a first-class main-snapshot npm `dev` publish path, including workflow input/trigger support, release-plan validation, publish-helper support, post-publish verification, and docs; or explicitly retire npm dev snapshots and update release-channel, update, installer, and release docs so dev is documented as git-only.
