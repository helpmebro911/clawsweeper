## Summary

Keep this issue open. Current main still has the reported BOM failure path: the shared frontmatter parser checks for `---` before stripping a leading UTF-8 BOM, the skill parser delegates directly to it, and the local skill loader drops parsed skills when `description` is missing. The related closed BOM/write-tool report #45432 does not cover this skill-discovery path.

## Best Possible Solution

Fix the shared markdown frontmatter parser by removing a leading UTF-8 BOM before checking for the opening `---` delimiter. Add a focused `parseFrontmatterBlock` regression test for BOM-prefixed content and a skill loading or `skills list` path test proving a valid BOM-prefixed `SKILL.md` remains discoverable. Keep #45432 closed as a separate write/edit-tool BOM report; it does not supersede this skill-discovery bug.

## Evidence
