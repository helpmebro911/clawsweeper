## Summary

Keep open. Current main already has session switching through a chat header select and a separate Sessions table, but the requested UX is specifically an expandable/dropdown conversation list under Chat in the left sidebar. The sidebar still renders only the Chat tab for the chat nav group, with no session children, active conversation row, timestamps, or new-conversation affordance there.

## Best Possible Solution

Keep this open as a first-party Control UI navigation enhancement. The likely implementation belongs in the sidebar rendering path: reuse the existing session-list data and session-switch behavior, add an expandable conversation list under Chat with active-state styling and a new-conversation action, and cover expanded/collapsed/mobile behavior with focused UI tests.
