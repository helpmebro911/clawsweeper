## Summary

Keep #71553 open. Current main still builds WhatsApp ack reaction options from `sender.jid` only, while the identity pipeline can represent LID-only group senders as `sender.lid` with no `sender.jid`. The downstream reaction sender already preserves LID participants when one is supplied, so the PR is a focused remaining bug fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and have maintainers land this behavior, either by rebasing #71553 or applying the equivalent current-main change: derive a `participant` from `sender.jid ?? sender.lid` for WhatsApp group ack reactions, keep direct-message behavior unchanged, and add the LID-only group ack regression test in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.test.ts`.
