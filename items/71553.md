## Summary

Keep this PR open. Current main still constructs WhatsApp ack-reaction options from `sender.jid` only, while current inbound identity normalization can represent a group participant as `sender.lid` with no `sender.jid`. The existing send layer and related tests already support LID participants when one is supplied, and the PR discussion plus bot review identify the same narrow missing fallback and regression test.

## Best Possible Solution

Keep this PR open and ask for a current-main rebase/adaptation: set the ack reaction participant from `sender.jid ?? sender.lid` in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.ts`, preserve direct-message behavior and JID precedence, add the LID-only group ack regression in `extensions/whatsapp/src/auto-reply/monitor/ack-reaction.test.ts`, then run the targeted WhatsApp extension format, lint, type, and test lanes before merge.
