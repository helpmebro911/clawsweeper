## Summary

Keep #71553 open. Current main still drops LID-only WhatsApp group senders when constructing automatic ack reaction participants, while the PR is a focused one-line fallback plus a targeted regression test. The provided PR discussion and Greptile review support the same narrow root cause, and the related items are adjacent WhatsApp LID/mention reports rather than a canonical superseding fix.

## Best Possible Solution

Keep the PR open for maintainer review and CI. The best code path is the PR's current direction: change the ack reaction participant construction to prefer `sender.jid` and fall back to `sender.lid`, with the added unit regression covering a group `sender.lid`/no-`sender.jid` message. No broader core or plugin API work is needed for this narrow bug.
