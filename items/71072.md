## Summary

Keep open. Current main preserves explicit empty plugin-scope semantics, but it has not implemented this PR's requested O(1) empty-scope fast path: `loadOpenClawPlugins({ onlyPluginIds: [] })` still proceeds into the normal loader path and reaches discovery/manifest work before returning an empty registry.

## Best Possible Solution

Keep the PR open for maintainer review. The useful direction is a narrow loader fast path for explicit empty scopes, placed so reentry protection and activation state clearing remain correct, plus a focused `src/plugins/loader.test.ts` regression that proves `discoverOpenClawPlugins` and `loadPluginManifestRegistry` are not invoked for `onlyPluginIds: []`.
