## Summary

Keep open. Current main still uses and documents direct routing of bound ACP follow-ups to the bound child session, while this PR proposes a parent-orchestrated follow-up interpretation rule and explicitly leaves the deeper runtime behavior change for future work. There is no strong close reason: it is not implemented on main, not stale, and not superseded by the related thread-bound fallback docs work.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main and decide whether to land this as explicit agent/orchestrator guidance, or pair it with the runtime/session-continuation change that makes parent-orchestrated follow-up rewriting the default for bound ACP sessions while preserving explicit verbatim relay.
