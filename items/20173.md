## Summary

Keep #20173 open. Current main still handles Discord inbound user turns through the bundled MessageCreate listener only; there is no MESSAGE_UPDATE/messageUpdate listener, no edit-handling config/schema/docs surface, and the inbound replay key is still based on account/channel/message id so an edited message with the same Discord id would be suppressed by the existing dedupe path.

## Best Possible Solution

Keep #20173 open and implement it in the bundled Discord plugin. Add opt-in config/type/schema/generated metadata/docs/UI-hint coverage, register a Carbon MessageUpdateListener or equivalent MESSAGE_UPDATE path, normalize update payloads through the existing Discord preflight and inbound worker pipeline, preserve current bot/self/access/mention filters, make replay dedupe edit-aware, define debounce/rate-limit and in-flight cancellation behavior, and add focused regression tests for listener registration, config validation, and edited-message reprocessing.
