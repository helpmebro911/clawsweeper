## Summary

Keep #25621 open. Current main still throws from `resolveGatewayService()` on unsupported platforms such as OpenBSD, and both `openclaw status` and `openclaw gateway status` still reach that resolver before they can degrade to read-only diagnostic output. The linked PR #25690 is useful prior art, but it was closed unmerged and its unsupported-platform stub is not present on main.

## Best Possible Solution

Keep the issue open until OpenClaw adds a graceful unsupported-platform GatewayService adapter or guards read-only status paths so `openclaw status` and `openclaw gateway status` work on OpenBSD and other unsupported platforms. PR #25690 remains useful prior art for the stub shape, but current main still needs the actual implementation and regression coverage.
