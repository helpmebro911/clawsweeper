## Summary

Keep open. Current main still restricts managed Gateway services to macOS, Linux, and Windows, and unsupported platforms still throw `Gateway service install not supported on ${process.platform}`. The reported `openclaw status` and `openclaw gateway status` paths still resolve the Gateway service before their later read-status fallbacks can apply, so the OpenBSD crash remains actionable. The related PR #25690 directly targeted this issue, but the provided GitHub context shows it was closed unmerged.

## Best Possible Solution

Keep #25621 open until OpenClaw either adds a graceful unsupported-platform `GatewayService` adapter/fallback, guards read-only status resolver calls so `openclaw status` and `openclaw gateway status` work on unsupported platforms, or makes an explicit documented product decision that these platforms must run the gateway manually outside managed service flows. The likely code path is `src/daemon/service.ts` plus regression coverage for unsupported `process.platform` in the status paths; PR #25690 remains useful prior art but was closed unmerged.
