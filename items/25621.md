## Summary

Keep open. Current main still limits managed Gateway services to darwin, linux, and win32, and unsupported platforms still throw `Gateway service install not supported on ${process.platform}`. The read-only `openclaw status` and `openclaw gateway status` paths still resolve that service before their later status fallbacks can degrade gracefully, so the OpenBSD report remains actionable. The related PR #25690 was useful prior art for an unsupported-platform stub, but the provided context shows it was closed unmerged and current main does not contain that behavior.

## Best Possible Solution

Keep #25621 open until OpenClaw either adds a graceful unsupported-platform `GatewayService` adapter/stub or guards read-only status resolver calls so `openclaw status` and `openclaw gateway status` work on OpenBSD and other unsupported platforms. Lifecycle commands can still reject clearly, but they should point users to manual foreground/service-wrapper guidance if managed service installation is intentionally unsupported. PR #25690 remains useful prior art, but it was closed unmerged.
