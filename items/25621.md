## Summary

Keep #25621 open. Current main still limits managed Gateway service adapters to macOS, Linux, and Windows, and unsupported platforms still throw from `resolveGatewayService()`. The read-only `openclaw status` and `openclaw gateway status` paths still reach that resolver before their fallback/catch handling can degrade, so the OpenBSD failure remains actionable. The related PR #25690 is useful prior art, but the provided GitHub context shows it was closed unmerged and its stub behavior is not present on main.

## Best Possible Solution

Keep the issue open until OpenClaw adds a graceful unsupported-platform GatewayService adapter or guards read-only status paths so `openclaw status` and `openclaw gateway status` work on OpenBSD and similar platforms. Reuse #25690 as prior art for the stub shape, but add current-main regression coverage for unsupported `process.platform` status behavior and keep mutating lifecycle operations clear and actionable, ideally with manual supervisor guidance or a generic shell-service template.
