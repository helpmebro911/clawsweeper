## Summary

Keep this PR open. Current main still does not strip malformed non-array `required` values from the shared tool schema normalization path, so the Bailian/strict-provider failure described by #40568 is not implemented on main. The PR remains a plausible focused fix, but it needs rebasing onto the newer `normalizeToolParameterSchema` structure and should keep the added regression coverage.

## Best Possible Solution

Keep this PR open and review a rebase onto current main. The best implementation is to strip own non-array `required` fields in the shared `normalizeToolParameterSchema` path before provider-specific cleaning, preserve valid `required` arrays, and add focused regression tests for the OpenAI-compatible strict-provider top-level path plus nested Gemini schemas. Treat the Aisle prototype-pollution note as hardening work unless the rebased patch already touches that copy loop.
