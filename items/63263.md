## Summary

Keep this PR open. Current main still lacks most of the infer CLI reliability behavior proposed here: local model runs still use the normal local agent path without a per-run infer session key, image describe commands still do not register or thread a prompt override, structured web-search error payloads still stop fallback, and openai-codex image understanding metadata still has no autoPriority. The PR is a plausible implementation candidate, but it needs rebase and review fixes before landing.

## Best Possible Solution

Keep this PR open as an implementation candidate. The maintainer path is to rebase it onto current main, preserve the current agentDir behavior, add a per-run infer sessionKey for local model runs, register and thread image --prompt with override precedence above per-entry prompts, narrow structured web-search fallback to availability/auth failures, move openai-codex image autoPriority into the current manifest/runtime provider surfaces only if that provider should participate in auto image fallback, and rerun the targeted CLI, web-search, and media-understanding tests before review/landing.

## Risks / Open Questions

- The PR's structured web-search fallback currently treats every top-level string error as provider unavailability, which can silently ignore invalid user constraints by falling through to a different provider.
- The image --prompt wiring needs runner-level precedence above per-model entry.prompt; otherwise the new CLI flag can be ignored for configured model entries.
- The PR was written against an older media defaults layout and current main already has some drift, including agentDir passing and manifest-backed media metadata, so it should be rebased and adjusted before review.
