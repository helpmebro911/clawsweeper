## Summary

Keep this PR open. Current main still has the older WhatsApp retry window and still does not surface auto-reply delivery failures as agent-visible system events, so the PR remains a plausible bug-fix candidate. It cannot be closed as implemented or obsolete, but it needs rebase/adaptation from old `src/web` paths to the current `extensions/whatsapp` plugin layout and should address the review concern about which reply kinds emit system events.

## Best Possible Solution

Keep the PR open as a useful WhatsApp bug-fix candidate. The best maintainer path is to rebase it onto the current `extensions/whatsapp` plugin implementation, update the current retry helper/defaults to cover the reconnect window, wire WhatsApp dispatcher delivery failures into `enqueueSystemEvent` through the existing plugin SDK/runtime seam, scope the event to the intended reply kind(s), and add regression tests for the widened retry window and agent-visible delivery-failure signal.
