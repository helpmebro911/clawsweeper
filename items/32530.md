## Summary

Keep #32530 open. Current main still treats agent definitions as explicit `agents.list` entries, not discoverable workspace-local config files, and the plugin SDK exposes agent runtime helpers and harness registration but no stable agent-definition discovery/contribution contract. TinyTb's #27706 hot-reload comment is useful related zero-restart context, but it does not supersede the requested external workspace auto-discovery surface.

## Best Possible Solution

Keep #32530 open and narrow it around external workspace agent discovery. A good maintainer path is either a core `agents.autoDiscover` config schema or a documented plugin-backed agent-discovery API that runs before normal agent enumeration, with trusted roots, accepted filenames, schema validation, deterministic scan order, explicit precedence against `agents.list`, hot-reload/watch behavior, docs, and regression tests. Link #27706 only as related zero-restart context.
