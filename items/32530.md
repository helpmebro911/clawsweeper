## Summary

Keep #32530 open. Current main still models agents as explicit `agents.list` records and exposes manual create/update/delete/list paths, but it does not support `agents.autoDiscover`, workspace-local `agent.json`/`openclaw-agent.json` discovery, or a plugin API for contributing agent definitions from external workspaces. TinyTb's #27706 comment is useful zero-restart context, but #27706 is closed and does not supersede the distinct external workspace discovery request.

## Best Possible Solution

Keep #32530 open and narrow it around external workspace agent discovery. The best maintainer path is either a first-class `agents.autoDiscover` config contract or a documented plugin-backed agent-definition discovery API that runs before normal agent enumeration, with trusted roots, accepted filenames, schema validation, deterministic scan order, explicit precedence against `agents.list`, watch/hot-reload behavior, docs, and regression tests. Link #27706 only as related zero-restart context.

## Risks / Open Questions

- A script or trusted plugin can approximate this today by writing `agents.list`, but that is an out-of-band workaround and does not settle trusted roots, schema validation, deterministic scan order, precedence, or watch/hot-reload behavior.
- Plugin discovery exists for plugins and workspace skills, but current plugin surfaces do not provide an agent-definition discovery API; closing this as ClawHub would push contributors toward unsupported config mutation.
