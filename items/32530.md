## Summary

Keep open. Current `main` does not implement auto-discovery of agent configurations from external workspaces. The config surface still exposes only explicit `agents.defaults` and `agents.list` entries, agent workspace enumeration is derived from those configured agents only, and the docs still describe manual agent creation with `openclaw agents add`. Current `main` does appear to have improved the related runtime-management pain point from the comment: `agents.list` changes are treated as hot reloads rather than mandatory full gateway restarts, and config-patch guard coverage explicitly allows adding a new agent entry.
