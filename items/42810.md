## Summary

Keep this PR open. Current main still does not implement ACP control-plane audit logging, and the branch is a plausible but not merge-ready implementation: it adds the requested audit module, but still needs a rebase, a real production sink, best-effort audit emission, tighter storage/retention semantics, and preservation of current ACP lifecycle behavior. The open gateway tool-call audit work in #63557 is a different surface, not a replacement for ACP session lifecycle audit logging.

## Best Possible Solution

Keep this PR open, but require a rebased and narrower ACP audit implementation before merge. The maintainable path is to preserve current AcpSessionManager lifecycle semantics, wire a real audit sink under the normal OpenClaw state/log directory with restrictive permissions, make audit emission best-effort, cover session init/close/cancel/run/runtime-control events with focused tests, harden buffering/query/retention behavior, and split unrelated config/schema edits out. Treat #63557 as separate gateway tool-call audit work.

## Risks / Open Questions

- Merging the branch as-is could regress current initializeSession runtime option propagation for model/thinking/cwd.
- Audit logging is security-sensitive and should be best-effort; awaiting an unhealthy audit sink inside lifecycle paths can turn successful session init/close operations into failures.
- The proposed file sink still has audit-log integrity and availability gaps around buffer caps, query/flush races, retention enforcement, directory creation, and restrictive file permissions.
- The advertised persistent audit trail is not wired into the normal AcpSessionManager singleton path because the PR defaults to NullAuditLogger unless a caller manually injects another logger.
- The PR mixes the ACP audit feature with unrelated config/schema churn, increasing review and security surface even though it does not add new third-party dependencies, workflows, package lifecycle hooks, or action refs.
