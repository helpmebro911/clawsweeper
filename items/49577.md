## Summary

Keep open. Current main has related gateway status/readiness/process-health improvements, but the central Feishu false-healthy inbound-delivery gap is still present: the Feishu gateway runtime does not publish inbound/transport activity into channel status, and the shared health policy treats channels without transport-liveness reporting as healthy rather than degraded.

## Evidence

- **Issue context:** The report is specifically about Feishu on OpenClaw 2026.3.13 showing running/WebSocket/gateway liveness while inbound messages did not reach sessions or gateway dispatch logs. The only issue comment correctly frames this as an observability gap and suggests last-inbound/dispatch counters plus stale process checks.
- **Feishu status snapshot lacks inbound/transport fields:** Feishu's computed status snapshot exposes account identity plus app/domain/port, and `startAccount` only patches `port` before handing off to the monitor. It does not publish `connected`, `lastInboundAt`, `lastEventAt`, or `lastTransportActivityAt`.
