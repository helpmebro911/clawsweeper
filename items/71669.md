## Summary

Keep open. Current main still rejects non-localhost insecure HTTP Control UI sessions without device identity, and the docs/UI now explain that policy, but the requested gateway startup warning is not implemented. The related #32473 discussion confirms this remains a real VPS/Docker/homelab support path with known HTTPS, SSH tunnel, Tailscale, Traefik, and break-glass workarounds; #56830 is an open copy/UI-protocol clarification and does not supersede this server-log warning request.

## Best Possible Solution

Keep #71669 open for a small core gateway change. Add a startup warning in `src/gateway/server-startup-log.ts` using the existing `cfg`, `bindHost`/`bindHosts`, and `tlsEnabled` inputs when Control UI is enabled by default or config, device auth is not disabled, and the gateway is exposed over non-secure non-loopback HTTP. Cover it in `src/gateway/server-startup-log.test.ts`, keep wording aligned with `docs/web/control-ui.md` and `docs.openclaw.ai/gateway/security`, and coordinate it with the broader #32473 policy/support thread plus the narrower #56830 copy clarification.
