## Summary

Keep open. Current main still has the documented and tested remote plain-HTTP Control UI rejection, plus UI/docs guidance, but it does not emit the requested server-side startup warning when operators are likely to hit that rejection. The related #56830 PR is scoped to protocol/UI copy for #32473 and does not supersede this startup-log request.

## Best Possible Solution

Keep this issue open and add a targeted gateway startup warning after runtime config, bind hosts, TLS state, Tailscale mode, and Control UI config are resolved. The likely implementation point is src/gateway/server-startup-log.ts or the post-attach/startup runtime path, with coverage in src/gateway/server-startup-log.test.ts. The message should point operators to HTTPS/Tailscale Serve, localhost or SSH tunnel access, reverse proxy setup, and the break-glass gateway.controlUi.dangerouslyDisableDeviceAuth option while preserving the current auth policy.

## Risks / Open Questions

- The eventual warning predicate needs careful wording to avoid false positives for HTTPS terminated by Tailscale Serve or a reverse proxy while the local gateway listener is still HTTP.
- The fix should preserve the current remote plain-HTTP security policy and only add operator guidance unless maintainers decide to change the broader behavior tracked by #32473.
