## Summary

Keep #48194 open. Current `main` still implements cron delivery as a single primary target plus optional `failureDestination`; it does not ship multi-channel `delivery.additionalTargets`, and the PR discussion already identified unresolved implementation gaps that need maintainer review rather than automated cleanup closure.

## Best Possible Solution

Keep the PR open for maintainer review or replacement by a complete implementation. The right product/code path is a reviewed core change that adds `delivery.additionalTargets` across types, gateway schema, normalization, patch merge, isolated-run fan-out, CLI/UI/docs, and tests, while resolving the existing review concerns around dispatch, patch preservation, and optional target failure handling.
