## Summary

Keep this issue open. Current main still has useful external-content wrapping, input-level provenance, plugin tool hooks, outbound hooks, and exec approval controls, but it does not implement the requested persistent taint model, default prohibition on content-derived execution, general exec risk classes, or provenance-aware secret/outbound enforcement. The linked related items are narrower security threads, not a single merged canonical replacement for this umbrella.

## Best Possible Solution

Keep this issue open as the architecture umbrella unless maintainers explicitly split it into canonical implementation issues. A defensible path is to carry provenance metadata through external content, tool results, summaries, persistence, and future reads; enrich `before_tool_call` with stable action/risk/provenance fields; and enforce content-derived execution, secret access, and outbound-action policy in the core tool/approval paths while linking narrower follow-ups such as #54373, #48519, #48503, #41140, #47492, and #53185.
