## Summary

Keep this open. Current main has live transcript event plumbing for Feishu-backed sessions, but it does not satisfy the reporter's bidirectional exact-sync expectation: the Control UI connects in WebChat mode, dashboard sends use `deliver: false`, WebChat clients intentionally do not inherit external delivery routes, and the UI only reloads the currently active session when `session.message` arrives.

## Best Possible Solution

Keep this open for maintainer triage. Reproduce on current main with Feishu DM and group sessions, recording the active Control UI session key, Feishu route/session scope, gateway logs, and whether `session.message` arrives. If true bidirectional mirroring is desired, add an explicit opt-in Control UI delivery/mirror flow for Feishu-backed sessions plus tests for Feishu inbound-to-active-session reload and explicit dashboard-to-Feishu delivery. If the intended behavior is that Control UI is a session viewer/editor rather than an external chat mirror, update the WebChat/Feishu docs to make that boundary explicit before closing.
