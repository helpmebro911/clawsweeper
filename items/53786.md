## Summary

Current main still does not implement exact real-time bidirectional mirroring between the Feishu bot channel and the Local Dashboard. The dashboard connects as WebChat, sends with internal/session-local delivery, and WebChat clients are explicitly prevented from inheriting external channel routes. Feishu inbound messages are routed into the resolved Feishu session and can refresh the dashboard only when that same session is active.

## Best Possible Solution

Keep this open for maintainer product triage. Reproduce current main with Feishu DM and group/thread sessions, record the active Control UI session key, resolved Feishu route/session key, gateway logs, and session.message events. If bidirectional mirroring is desired, add an explicit opt-in Control UI mirror/delivery flow for Feishu-backed sessions with tests for Feishu inbound-to-active-session refresh and dashboard-to-Feishu delivery. If mirroring is intentionally out of scope, document that boundary in Control UI, WebChat, and Feishu docs before closing.
