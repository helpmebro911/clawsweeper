## Summary

Keep open. The Discord plugin on current main still wires inbound agent turns through `MessageCreateListener` only, while the reported Discord `MESSAGE_UPDATE` edit-to-mention path remains unimplemented. The issue and follow-up comment describe a concrete bundled Discord-channel workflow gap, not a ClawHub/off-repo cleanup item.
