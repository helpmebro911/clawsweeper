## Summary

Keep open. Current main still persists raw transcript text into bundled session-memory artifacts, and startup context then loads date-prefixed memory files back into new sessions. PR #69961 addresses a live core bug reported in #69943 and corroborated by affected-user comments; it is not implemented on main or superseded. The PR still has non-blocking but real review concerns to resolve before merge.

## Best Possible Solution

Keep the PR open and land a scoped session-memory fix after addressing the review comments. The best path is to sanitize per-turn before memory persistence, add a handler-level defense-in-depth pass, preferably reuse or align with `src/shared/text/assistant-visible-text.ts` for special-token/tool-call cleanup, add explicit NO_REPLY/garbage elision tests in `src/hooks/bundled/session-memory/handler.test.ts` or a colocated sanitizer test, then close #69943 when merged.
