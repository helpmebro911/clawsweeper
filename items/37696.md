## Summary

Keep #37696 open. Current main still implements and tests the reported strict-mode behavior: an unclosed `<think>` leaves the sanitizer inside a thinking block, so strict mode omits the remaining text. The shared delivery profile and channel preview paths still call that strict sanitizer, and message-end delivery still skips empty cleaned text unless media or an explicit messaging-tool fallback exists. The related fix attempts in the provided discussion are closed unmerged except #51915, which remains an open unshipped fix candidate rather than a shipped replacement for this issue.

## Best Possible Solution

Keep #37696 open until a focused fix lands. The likely path is to update `stripReasoningTagsFromText` strict-mode behavior and regression tests so malformed or unclosed reasoning tags cannot erase all deliverable assistant text, then verify assistant extraction and channel delivery/preview paths. #51915 is the current open candidate to merge, revise, or supersede; if maintainers intentionally want strict-mode discard for privacy, document that contract and add a delivery-level fallback so users do not get silent non-delivery.
