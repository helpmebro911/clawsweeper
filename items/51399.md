## Summary

Keep open. Current `main` still does not contain the Slack-specific attachment-vision reference this PR proposes, so it is not `implemented_on_main`. At the same time, parts of the proposed doc are stale against current `main`: Slack file-only thread starters are now supported, PDF attachment extraction still does not forward rendered PDF images into the model, and the correct operator-facing image-model key is `agents.defaults.imageModel.*`. This needs a rebase/rewrite, not cleanup closure.

## Evidence

- **Slack docs still only have a brief inbound-attachments note:** `docs/channels/slack.md` currently documents Slack inbound attachments only inside the `Media, chunking, and delivery` accordion. It mentions private-URL download and `channels.slack.mediaMaxMb`, but there is no dedicated `Attachment vision reference` section covering supported media types, pipeline details, or known limits.
  - file: [docs/channels/slack.md:605](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/channels/slack.md#L605)
