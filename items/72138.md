## Summary

Keep PR #72138 open. Current main still sends Feishu normal conversation replies through the Feishu-owned dispatcher without emitting plugin `message_sent` or internal `message:sent`, while the generic outbound path already has that hook behavior. The PR remains a relevant fix for the reported Feishu hook gap, with review comments to resolve before merge rather than a cleanup-close reason.

## Best Possible Solution

Keep PR #72138 open, or land an equivalent Feishu-owned fix. The maintainable path is to emit canonical plugin `message_sent` and internal `message:sent` from `extensions/feishu/src/reply-dispatcher.ts`, pass routed session key and group mirror metadata from `extensions/feishu/src/bot.ts`, preserve correct success/failure semantics for streaming plus media, include message IDs where available, and add Feishu tests for text, card, media, streaming finalization, and failure branches.
