## Summary

Keep PR #72138 open. Current main still lacks the Feishu normal-reply `message_sent` / internal `message:sent` emission that the PR targets, while the generic outbound path already emits those hooks. The PR remains relevant to open bug #72125 and should be reviewed or replaced by an equivalent Feishu-owned fix, not cleaned up as obsolete.

## Best Possible Solution

Keep the PR open or land an equivalent Feishu-owned fix. The durable path is to emit canonical plugin message_sent and internal message:sent from extensions/feishu/src/reply-dispatcher.ts, pass routed session key and group mirror metadata from extensions/feishu/src/bot.ts, include streaming-card message identity where available, avoid false failure hooks for partially delivered streaming text, and cover text, card, media, streaming finalization, and failure paths in Feishu tests.
