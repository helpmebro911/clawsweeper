## Summary

Current main partially addresses this issue: `agents.defaults.thinkingDefault` is implemented in the defaults type/schema and thinking docs, but the issue body also requests `agents.defaults.reasoningDefault`. That global reasoning default is still not implemented: config types/schema/generated help only define `reasoningDefault` for `agents.list[]`, runtime resolution falls back from session to per-agent `reasoningDefault` and then `off`/model default, and docs describe reasoning visibility as per-agent only. The related open PRs are implementation candidates or partial channel fixes, not shipped main behavior, so this issue should stay open.

## Best Possible Solution

Keep this issue open until maintainers either implement and document `agents.defaults.reasoningDefault` across the config type, strict schema, generated schema/help, directive resolution, status/session surfaces, and regression tests, or explicitly decide reasoning visibility should remain per-agent/session-only and remove the unsupported global key from tool/docs surfaces. The open global-reasoning-default PRs should be reviewed as implementation candidates before duplicating work.
