## Summary

Keep #71658 open. Current main still has the same deterministic gaps: the actionable prompt gate is still a narrow allowlist that misses the reported imperative prompt, and the planning-only detector still requires future-tense promise wording or structured planning format, so present-continuous narration like "I'm running it now" does not trigger the strict-agentic retry path.

## Best Possible Solution

Keep the issue open and patch `src/agents/pi-embedded-runner/run/incomplete-turn.ts`. Add focused regression tests for the reported imperative prompt and for accepted ack/action prompts followed by present-continuous no-tool narration, then broaden the actionable-prompt and planning-only narration detection without retrying casual chatter or completed-result summaries.
