## Summary

Current main still has the reported macOS false `No GUI detected` path. `resolveBrowserOpenCommand()` returns `ssh-no-display` for any non-Windows platform with SSH env vars and no DISPLAY/WAYLAND_DISPLAY before the Darwin `open` branch can run, and `openclaw dashboard` uses that result to print the SSH tunnel hint. This issue is also paired with open implementation PR #67115 using closing syntax, so it should remain open until that PR or an equivalent fix is reviewed and merged or closed.

## Best Possible Solution

Keep this issue open and review PR #67115, or replace it with an equivalent narrow fix in `src/infra/browser-open.ts` that excludes `darwin` from the SSH/no-display early return or narrows that guard to Linux. Add regression tests proving macOS resolves to `open` with lingering SSH_* env vars and no DISPLAY/WAYLAND_DISPLAY, while Linux SSH/no-display behavior remains intact.
