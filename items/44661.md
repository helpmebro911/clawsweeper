## Summary

Current main already ships a supported agent/session hooks system. Users can add custom hooks, enable them with `openclaw hooks`, and use the `agent:bootstrap` lifecycle event to modify bootstrap files before they are injected into the session context.

## Risks / Open Questions

- The issue proposed a YAML + shell-script API, while current main implements the capability through discovered hook directories and JS/TS handlers. This close assumes the requested capability matters more than that exact configuration shape.

## Close Comment

Closing as implemented on current main after Codex review. OpenClaw already ships user-manageable agent hooks via `openclaw hooks` (`docs/cli/hooks.md`), including the `agent:bootstrap` lifecycle event (`docs/automation/hooks.md`). Custom hooks are loaded from workspace/managed hook directories (`src/hooks/loader.ts`), and `src/agents/bootstrap-hooks.ts` plus `src/agents/bootstrap-files.ts` show that an `agent:bootstrap` hook can mutate `context.bootstrapFiles` before those files are injected into the run context. That behavior is also covered by `src/agents/bootstrap-hooks.test.ts`. This capability is present in the shipped `v2026.4.22` release (`a94763ee224e858e29dd177cd6f7f4daa438c2f1`). The exact YAML/shell-script design from the issue was not adopted, but the requested session-start hook capability already exists on the supported hooks surface.
