## Summary

Keep open. Current main still wires `streamTo="parent"` for `runtime="acp"`, but the parent relay only surfaces assistant text and lifecycle events. Existing agent event streams already include tool and thinking events, and the ACP bridge translates tool events for ACP clients, but `startAcpSpawnParentStreamRelay()` explicitly returns for every non-lifecycle stream after handling assistant deltas, so tool calls/results/thinking are not visible in the parent session. The related #67590 channel-output request is broader and closed; it does not supersede this ACP parent-relay gap.
