## Summary

Keep open. Current main still runs `runMemoryFlushIfNeeded()` only in the pre-turn path, while runtime compaction events during the embedded turn are only counted and later persisted as compactions; I did not find a post/in-turn memory flush retry tied to those events.

## Evidence

- **Pre-turn memory flush call:** `runReplyAgent` runs `runPreflightCompactionIfNeeded()` and then `runMemoryFlushIfNeeded()` before constructing the follow-up runner and starting the embedded turn.
  - file: [src/auto-reply/reply/agent-runner.ts:1062](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/auto-reply/reply/agent-runner.ts#L1062)
  - command: `nl -ba src/auto-reply/reply/agent-runner.ts | sed -n '1010,1115p'`
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **In-turn compaction handling does not flush:** The embedded-turn `onAgentEvent` handler tracks `compaction` start/end events and increments `attemptCompactionCount` when `completed === true`, but it does not call `runMemoryFlushIfNeeded()` or another memory flush path.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:1288](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/auto-reply/reply/agent-runner-execution.ts#L1288)

- **Post-turn accounting only increments compaction count:** After a turn with `autoCompactionCount > 0`, OpenClaw calls `incrementRunCompactionCount()` and injects post-compaction context; no memory flush/audit is run here.
  - file: [src/auto-reply/reply/agent-runner.ts:1527](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/auto-reply/reply/agent-runner.ts#L1527)
