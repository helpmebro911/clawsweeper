## Summary

Current main still does not implement the requested native subagent cascade circuit breaker. The spawn path has depth, active-child, allowlist, sandbox, timeout, and setup checks, but no persisted consecutive-failure state, tripped/cooldown precheck, forced override, manual reset, or notification. Registry/task state records individual runs and terminal outcomes only. The config schema, tool parameters, CLI/docs, and Control UI-visible task surfaces also lack the requested circuitBreaker block or breaker status.

## Best Possible Solution

Keep this issue open and implement the native subagent cascade breaker in the core spawn and registry path. Persist breaker state keyed by requester plus target agent and/or normalized task signature, update it on terminal subagent completion, reset on success, check it before dispatch, expose explicit force/manual reset, add strict config/schema/docs, and surface state through subagent/task status plus Control UI. Treat provider billing/rate-limit breakers as related but separate follow-up unless maintainers intentionally combine the scopes.

## Risks / Open Questions

- Provider billing/rate-limit damage from the discussion is adjacent scope and likely needs coordinated provider-level cooldown/spend-cap work rather than being folded entirely into the subagent breaker.
- Breaker keying needs maintainer judgment: target-agent-only can over-block unrelated work, while task-signature-only can miss broken agent configuration.
- The implementation should make blocked spawns observable in CLI/UI and support explicit operator override/reset so reliability guardrails do not become silent failures.
