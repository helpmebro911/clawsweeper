## Summary

Keep this PR open. Current main still does not expose effective sandbox mounts from `openclaw sandbox explain` in JSON or text output, and this PR remains the active implementation candidate. The PR discussion also identifies real follow-up requirements, so it should be revised rather than closed as obsolete.

## Best Possible Solution

Keep this PR open as the active implementation candidate. The best path is to update `sandbox explain` with a backend-aware effective-mounts helper, reuse `resolveSandboxRuntimeStatus` for canonical sandboxed/direct decisions, emit an empty mounts array or explicit direct-runtime hint when not sandboxed, cover JSON and text output plus custom binds, agent workspace overrides, main-session aliases, and SSH/OpenShell remote roots, and drop unrelated test-file churn before review.
