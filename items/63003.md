## Summary

Current main still does not implement this PR's requested `sandbox.mounts` JSON field or `Effective mounts:` text output, and the paired issue was already closed as superseded by this PR. The PR remains useful but should stay open because its current diff has correctness risks around canonical session status and backend-specific roots, plus unrelated test-file churn.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Revise it to compute sandboxed status through the canonical runtime-status path, derive effective mounts from the selected sandbox backend, retain the requested JSON and text output, add focused tests for JSON, text, Docker binds, direct runtime, aliases, agent workspace overrides, and remote roots, and drop unrelated test-file churn before maintainer review.
