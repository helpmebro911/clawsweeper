## Summary

Keep PR #71063 open. Current main still has the unguarded initial `child.stdin.write()` / `end()` path in `createChildAdapter`, and the regression test proposed by the PR is not present. The PR is a recent, focused bug fix with concrete reproduction context around `google-gemini-cli` / missing `gemini` binary behavior, so it is not conservative-cleanup closeable.

## Best Possible Solution

Keep the PR open for maintainer review. The best code direction is to harden `src/process/supervisor/adapters/child.ts` around the initial stdin write/end path for failed or already-closing child processes, keep the regression coverage in `src/process/supervisor/adapters/child.test.ts`, and run the normal changed gate before merge.
