## Summary

Keep #69668 open. Current main supports isolated multi-agent routing and the chat header groups session options by agent, but it does not implement the requested team/project parent hierarchy, team management, or configurable team/group metadata. The main Sessions view is still a flat table of session rows, and the agent/session config types only expose agent-level identity/workspace/session fields, not team/project grouping.

## Best Possible Solution

Treat this as a Control UI plus config/product feature request. A plausible implementation would add an explicit, documented grouping field such as agents.list[].team or agents.list[].project, expose it through the agent/session list APIs, and render the chat/session list with collapsible project/team parents while preserving the existing per-agent isolation model. If maintainers do not want a new core product concept, the next best path is to document the rejected scope and provide a narrower grouping-by-prefix or grouping-by-agent follow-up, but current main is not enough to close this item.
