## Summary

Keep #38302 open. Current main still does not support per-account Slack native command prefixes or account-level native-name mappings for built-in slash commands. Slack config exposes only the existing slashCommand fields, strict schema/metadata reject proposed naming fields, the native command registry resolves names by provider only, and Slack native registration/dispatch does not pass account naming data. The issue also has a second user confirming the multi-app Slack collision, while the related implementation attempts #38326 and #38391 are closed and unmerged in the provided GitHub context.

## Best Possible Solution

Keep the issue open and implement this in the bundled Slack plugin plus core native-command contract. A maintainer-grade fix should add a validated account-level Slack native command naming option, either nativePrefix-derived names or an explicit nativeNames map; thread that resolved account config through native spec generation, Slack registration, argument-menu values, and findCommandByNativeName dispatch; preserve current unprefixed behavior and the /status to /agentstatus override; update Slack setup/docs; and add focused multi-account tests proving prefixed native commands route to the intended built-in commands.
