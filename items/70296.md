## Summary

Keep #70296 open. Current main partially addresses the Control UI model read-back concern, but the restart-sentinel channel-noise request is not implemented: restart sentinel messages are still formatted with summary/doctor-hint text and delivered to resolved channel routes, with no suppress config flag found. BOOT.md also still appends the canonical silent-token footer unconditionally, so any requested prompt/footer consolidation remains unresolved.

## Best Possible Solution

Keep the issue open or split it. The concrete remaining path is to add a documented, typed config setting for suppressing restart-sentinel channel notices, wire it through sentinel handling so session recovery/continuation still works, and cover the behavior with restart-sentinel tests. Separately, align BOOT.md prompt construction with the documented `NO_REPLY` protocol, either by keeping one canonical footer or letting BOOT.md fully govern silence behavior. The Control UI model read-back piece should be rechecked against current main and closed only as part of this issue after the remaining behavior is resolved or split.
