## Summary

Keep this issue open. Current main appears to have fixed or covered the Control UI model read-back complaint, but the restart-sentinel channel notification and BOOT.md silent-token conflict are still present in source. The sentinel path still formats a restart notice and delivers it to resolved channel routes with no documented suppression config key, and BOOT.md execution still appends a hard-coded `NO_REPLY` footer after workspace content.

## Best Possible Solution

Keep this issue open and split the remaining work into targeted fixes: add a documented restart-sentinel channel-notification policy/config key and guard the outbound notice path in `src/gateway/server-restart-sentinel.ts`, then decide and test the BOOT.md silent-token contract in `src/gateway/boot.ts`. Preserve the current Control UI effective-model read-back behavior and tests unless a fresh reproduction shows a remaining UI regression.
