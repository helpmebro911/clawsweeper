## Summary

Keep #70296 open. Current main appears to handle the Control UI model read-back path for string and object-shaped model values, but the issue bundles two restart/boot behaviors that are still present: restart sentinel notices are still delivered to routed channels with no suppress config, and BOOT.md prompts still get a hardcoded NO_REPLY footer appended after the workspace file content.

## Best Possible Solution

Keep the issue open or split it into narrower trackers. The restart-sentinel path likely needs an explicit config/API decision that suppresses end-user channel notices while preserving internal system events and restart continuations, with schema/docs/tests. The BOOT.md path needs a product decision to either remove the injected footer or align it with the documented silent-token contract instead of overriding workspace instructions. The Control UI model display portion can be separately verified and closed if maintainers agree current main covers it.
