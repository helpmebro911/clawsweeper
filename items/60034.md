## Summary

Current `main` still hardcodes `session-memory` output under `memory/YYYY-MM-DD-slug.md`, and the only documented/configured option is `messages`. The issue’s specific enhancement request is not implemented. The report’s AGENTS.md rationale is overstated relative to OpenClaw’s own templates, but that is not strong evidence to close a fresh enhancement request.

## Risks / Open Questions

- The issue body frames this as AGENTS.md non-compliance, but OpenClaw’s own templates and memory docs still center `memory/YYYY-MM-DD.md` style files, so the motivation should probably be narrowed to configurability/workspace organization rather than spec violation.
- There is a workaround via a separate managed/workspace hook, but it is not a true drop-in override because bundled `session-memory` cannot be overridden by name.
