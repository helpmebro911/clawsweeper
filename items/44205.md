## Summary

Current `main` still implements the pre-PR behavior: descendant fallback replies are aggregated from up to four child runs and joined together, and the colocated regression test still asserts that multi-descendant join behavior. Because `delivery-dispatch` consumes that helper directly for completed-descendant fallback delivery, this PR is not already implemented on `main`. The PR discussion also includes unresolved review concerns about dropping valid multi-child output and about the broad interim-message filter potentially suppressing real descendant results, so there is not strong enough evidence for a conservative auto-close.
