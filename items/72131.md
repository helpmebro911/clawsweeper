## Summary

Keep open. Current main still leaves Windows node platform defaults at the minimal system-command set, while this PR is a focused implementation of the MEMBER-authored #71876 request to allow safe declared Windows companion commands by default. The allowlist is used in active pairing/reconnect and invocation paths, so the PR is not obsolete cleanup material.

## Best Possible Solution

Keep #72131 open for maintainer review and normal gateway changed-lane validation. If maintainers accept Windows companion parity, land the focused allowlist/test change; separately handle or track the gateway configuration documentation note from #71876 about `allowCommands`, `denyCommands`, and re-pairing after command-list changes.
