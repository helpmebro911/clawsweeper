## Summary

Keep open. Current main still has the reported ACP capacity behavior: startup identity reconciliation does not release cached oneshot handles, and `acp.maxConcurrentSessions` enforcement still throws directly from `runtimeCache.size()` after only normal TTL-based idle eviction. The PR is a plausible focused fix, but it is not implemented on main and the provided review context includes follow-up correctness concerns that should be resolved before landing.

## Best Possible Solution

Keep the PR open and either revive it or reimplement the same targeted behavior on main: release cached oneshot handles after startup identity reconciliation, add a pressure-specific dormant oneshot reclaim path before throwing `ACP max concurrent sessions reached`, and preserve actor-queue serialization, hard timeouts, idleness revalidation, capacity rechecks, observability counters, and focused regression tests.
