## Summary

Keep open. Current main does not implement transcript archiving or model-handoff creation/consumption on default model config changes, and the narrower related PR #41907 was closed unmerged rather than shipped. This PR is stale and has unresolved functional review concerns, but it is still aimed at core gateway/session behavior rather than ClawHub or external administration.

## Best Possible Solution

Keep this PR open for maintainer judgment or replacement. The best path is a reviewed core gateway/session implementation that handles default model config changes by clearing or rotating stale persisted model state, archives old transcripts when needed, preserves explicit user model overrides, and only injects sanitized one-shot handoff context after the run is actually going to proceed, with focused tests for hot reload, restart reload, multi-session handoff retention, and status/context token refresh.
