## Summary

Keep open. Current `main` still rejects Claude-style flat alias params, and the PR discussion surfaced a real implementation gap: the embedded-runner adapter path does not forward `prepareArguments`, so this PR is not an auto-close candidate as implemented or obsolete.

## Evidence

- **Embedded runner drops `prepareArguments`:** `toToolDefinitions()` rebuilds each tool definition with `name`, `label`, `description`, `parameters`, and `execute`, then calls `tool.execute(...)` directly. There is no forwarding or invocation of `tool.prepareArguments` on this path.
  - file: [src/agents/pi-tool-definition-adapter.ts:216](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/pi-tool-definition-adapter.ts#L216)
  - command: `nl -ba src/agents/pi-tool-definition-adapter.ts | sed -n '210,270p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Production embedded path always goes through the adapter:** `splitSdkTools()` always returns `customTools: toToolDefinitions(tools)`, so OpenClaw's embedded runner path passes through the adapter that drops `prepareArguments`.
  - file: [src/agents/pi-embedded-runner/tool-split.ts:8](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/pi-embedded-runner/tool-split.ts#L8)
