## Summary

Keep #70541 open. Current main still has the reported bug in the auto-compaction completion path: after `autoCompactionCount > 0`, `agent-runner.ts` reads post-compaction context from `process.cwd()` instead of the configured run workspace, while the preflight compaction path already uses `followupRun.run.workspaceDir`. The issue is recent, specific, reproducible, unprotected, and not superseded by the related adjacent post-compaction items.

## Best Possible Solution

Keep the issue open. The maintainer fix should update `src/auto-reply/reply/agent-runner.ts` to pass the configured run workspace to `readPostCompactionContext` in the `autoCompactionCount > 0` path, matching `agent-runner-memory.ts`. Add a focused regression test where cwd and `followupRun.run.workspaceDir` differ, auto-compaction completes, and the queued post-compaction system event contains the workspace `AGENTS.md` content.
