## Summary

Current main does not implement the requested watchdog change. The resume watchdog still has a 180_000 ms default cap, the resolver still clamps configured maxMs to at least 1_000 ms rather than supporting 0/null as disabled, and the config schema still requires maxMs to be at least 1_000. Related PR #41022 appears to track the same fix but remains open/unmerged; PR #46050 is closed/unmerged, so the issue is not implemented or superseded by landed work.
