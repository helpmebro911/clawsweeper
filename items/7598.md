## Summary

Current `main` already provides the requested backup-layer behavior well enough: session transcripts are persisted as on-disk JSONL, compaction/context pruning does not rewrite that disk history, optional `memory_search` session indexing exists, session retention controls exist, and `openclaw backup create` archives sessions as part of local state backup.

## Evidence

- **Session transcripts already persist on disk:** Docs state that OpenClaw stores session transcripts under `~/.openclaw/agents/<agentId>/sessions/*.jsonl` for session continuity and optional session-memory indexing.
  - file: [docs/gateway/security/index.md:382](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/gateway/security/index.md#L382)
- **Compaction does not delete on-disk history:** Config reference says context pruning only trims in-memory context and does not modify session history on disk.
  - file: [docs/gateway/configuration-reference.md:1443](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/gateway/configuration-reference.md#L1443)
- **Exact transcript is available from disk:** Session tool docs say `sessions_history` is bounded/safety-filtered, and that the exact byte-for-byte transcript should be inspected from the transcript file on disk.
  - file: [docs/concepts/session-tool.md:63](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/concepts/session-tool.md#L63)
- **Session transcript search is already implemented:** Memory config documents `memorySearch.experimental.sessionMemory` plus `sources: ["sessions"]` to index session transcripts and surface them through `memory_search`.
  - file: [docs/reference/memory-config.md:342](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/reference/memory-config.md#L342)
- **Retention/cleanup knobs already exist:** Session maintenance docs include retention and cleanup controls such as `pruneAfter`, `resetArchiveRetention`, and disk-budget enforcement.
  - file: [docs/gateway/configuration-reference.md:2036](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/gateway/configuration-reference.md#L2036)
- **First-class backup command includes sessions:** `openclaw backup create` is documented as backing up local OpenClaw state including sessions, and the changelog records that backup feature in release `2026.3.8`.
  - file: [docs/cli/backup.md:11](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/docs/cli/backup.md#L11)
- **Shipped release evidence for backup + transcript search:** Changelog entries show `openclaw backup create` in `2026.3.8` and experimental session-transcript indexing for `memory_search` in `2026.1.16-1`.
  - file: [CHANGELOG.md:2783](https://github.com/openclaw/openclaw/blob/7e16e3d07723cc3903f40ce94f53a3920c88b1fc/CHANGELOG.md#L2783)

## Close Comment

Closing this as implemented on current `main` after a ClawSweeper/Codex review. OpenClaw already keeps session transcripts as on-disk JSONL under `~/.openclaw/agents/<agentId>/sessions/*.jsonl` (`docs/gateway/security/index.md:382`), and compaction/context pruning explicitly does not rewrite that disk history (`docs/gateway/configuration-reference.md:1443`). The exact transcript is available from disk (`docs/concepts/session-tool.md:63`), `memory_search` can already opt into session-transcript indexing via `memorySearch.experimental.sessionMemory` (`docs/reference/memory-config.md:342`), session retention controls exist under `session.maintenance` (`docs/gateway/configuration-reference.md:2036`), and `openclaw backup create` archives local state including sessions (`docs/cli/backup.md:11`; changelog entry in `CHANGELOG.md:2783`).
