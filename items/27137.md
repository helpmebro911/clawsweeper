## Summary

Keep #27137 open. Current main has stronger shared DM/group policy primitives and several channel-local ingress helpers, but it still does not implement the requested unified core `authorizeIngress()`/`IngressContext` gate across channel, account, event kind, sender identity, chat context, allowlists, pairing-store reads, command authorization, and standardized reason codes.

## Best Possible Solution

Keep #27137 open for maintainer design. The best path is a narrow core/security or SDK-backed ingress authorization helper that accepts normalized adapter inputs for channel, account, event kind, sender identity, chat context, and command surface; centralizes DM/group policy, allowlist merging, pairing-store reads, command authorization, and reason codes; then migrates representative message, reaction, pin, interaction, callback, slash, and voice paths with contract tests proving new ingress surfaces cannot bypass the shared gate.
