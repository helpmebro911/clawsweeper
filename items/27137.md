## Summary

Keep #27137 open. Current main has stronger shared DM/group policy, command-gating, and direct-DM helpers, but it still does not provide the requested unified `authorizeIngress()`/`IngressContext` gate across channel, account, event kind, sender identity, chat context, allowlists, pairing-store reads, command authorization, and standardized reason codes. The only issue discussion also includes a maintainer-posted Codex review explicitly keeping this item open.

## Best Possible Solution

Keep #27137 open for maintainer design. The best path is a narrow core/security or SDK-backed ingress authorization contract that accepts normalized adapter inputs for channel, account, event kind, sender identity, chat context, allowlists/store policy, and command surface; centralizes DM/group policy, allowlist merging, pairing-store reads, command authorization, and reason codes; then migrates representative message, reaction, pin, interaction, callback, slash, and voice paths with contract tests proving new ingress surfaces cannot bypass the shared gate.
