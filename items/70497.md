## Summary

Keep open. Current main still does not strip JSON Schema conditional keywords `if`, `then`, and `else` from Gemini tool schemas, and the PR’s regression test is not present. The related #69964 is the closed, unmerged predecessor of this same work, not a shipped fix.

## Best Possible Solution

Keep #70497 open and integrate the focused fix after normal PR validation: add `if`, `then`, and `else` to `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS`, keep the targeted regression test, and treat #69964 only as the closed predecessor while #70497 remains the active implementation PR.
