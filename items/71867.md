## Summary

Keep #71867 open. Current main (4a3030df9efa72c44bab567a4390ac0c4876a73f) still runs doctor auth health against the implicit/default auth store and emits generic auth note titles without agent attribution. The related #71923 is an open candidate fix, but the behavior is not implemented on main yet.

## Best Possible Solution

Keep the issue open and land a focused doctor-auth fix, likely via #71923 or an equivalent patch. The fix should label `Model auth`, cooldown, and OAuth refresh diagnostics with the agent id, resolve each configured/relevant agentDir read-only for diagnostics, avoid duplicating inherited-only stores, and add regression coverage for divergent multi-agent auth state.
