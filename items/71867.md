## Summary

Keep #71867 open. Current main still runs doctor auth health against one implicit/default auth store and emits generic `Auth profile cooldowns`, `OAuth refresh errors`, and `Model auth` note titles, so the requested per-agent attribution is not implemented. The related #71923 PR is the focused active fix, but it is not on current main yet.

## Best Possible Solution

Keep #71867 open until #71923 or an equivalent fix lands. The best implementation is to label the checked agent store in doctor auth health, enumerate relevant configured agent auth stores or call out divergent state when multiple stores exist, and add focused regression coverage for multi-agent doctor output.
