## Summary

Current `main` still hides same-agent group/channel sessions from `sessions_list` under the default `tools.sessions.visibility="tree"` policy. The CLI can still enumerate those sessions directly from the store, so the reported mismatch between CLI/store visibility and tool visibility remains plausible and is not clearly fixed on `main`.

## Evidence

- **Default visibility is still `tree`:** The session-tools visibility resolver still defaults to `tree`, not `agent`, so same-agent sessions are not broadly visible unless config is changed.
  - file: [src/agents/tools/sessions-access.ts:27](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/tools/sessions-access.ts#L27)
  - command: `nl -ba src/agents/tools/sessions-access.ts | sed -n '27,35p'`
- **`tree` mode excludes non-spawned same-agent sessions:** The visibility guard only allows the current session plus spawned children in `tree` mode; other same-agent sessions are rejected before listing.
  - file: [src/agents/tools/sessions-access.ts:192](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/tools/sessions-access.ts#L192)
  - command: `nl -ba src/agents/tools/sessions-access.ts | sed -n '192,255p'`
- **`sessions_list` filters by visibility before `kinds`:** `sessions_list` fetches gateway sessions, runs `visibilityGuard.check(key)`, and skips disallowed rows before kind filtering, so `kinds:["group"]` cannot surface hidden same-agent group sessions under default `tree` visibility.
  - file: [src/agents/tools/sessions-list-tool.ts:114](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/tools/sessions-list-tool.ts#L114)
  - command: `nl -ba src/agents/tools/sessions-list-tool.ts | sed -n '114,164p'`
- **CLI/tool mismatch still exists by design:** The CLI `openclaw sessions` path reads session-store targets directly and emits all matching rows, which matches the commenter's observation that the CLI can see sessions the tool cannot.
  - file: [src/commands/sessions.ts:146](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/commands/sessions.ts#L146)
  - command: `nl -ba src/commands/sessions.ts | sed -n '146,199p'`
- **Docs still describe this restricted default:** Current docs/config help say `tools.sessions.visibility` defaults to `tree`, while `agent` is the mode that exposes any session in the current agent id.
  - file: [docs/gateway/config-tools.md:255](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/docs/gateway/config-tools.md#L255)
  - command: `nl -ba docs/gateway/config-tools.md | sed -n '255,276p'`
