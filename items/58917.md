## Summary

Current `main` still shows the response-footer input count from raw normalized `usage.input`, while OpenClaw's usage normalization explicitly strips cached prompt tokens from OpenAI-style prompt totals. That means the PR's core bug is still real on `main`, and there is no release/changelog evidence that the footer fix already shipped. The PR should stay open, though its current branch also carries unrelated fixes that likely need to be split or cleaned up before merge.

## Evidence

It also still uses the same raw `input` for cost estimation.
