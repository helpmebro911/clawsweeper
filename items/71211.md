## Summary

Keep open. Current main still returns exec stdout/stderr-derived output through foreground results, background process polling/logging, command-output events, notification/followup paths, and node-host formatting without applying the existing secret redaction helper at those exec-owned boundaries. The prior Codex review comment remains accurate against current main SHA 348728c28c1ea9ae7c5824350baba35ee08a275e.

## Best Possible Solution

Keep this issue open and patch the exec-owned output boundary. Apply redactSensitiveText consistently before stdout/stderr-derived text or details leave exec: foreground buildExecForegroundResult, runtime outcome construction and updates, process poll/log/list details, command-output events, notify-on-exit/followup summaries, and node-host formatNodeRunToolResult. Add focused regression tests with env-style secrets and bearer/API-key output so redaction is proven before exec results reach the agent-facing surfaces, while preserving existing output caps and persisted transcript safeguards.
