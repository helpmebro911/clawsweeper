## Summary

Current main still has the reported bug: shell commands containing backslash-newline are rejected before allowlist analysis, and both gateway and node-host paths still surface analysis failures as allowlist misses. The linked fix PR #71154 is open, while the related PR #71225 is closed but not merged per the provided GitHub context, so this issue should remain open until an equivalent fix lands.

## Best Possible Solution

Keep #71153 open and land or revise PR #71154, or an equivalent focused fix, so POSIX backslash-newline continuations are normalized before allowlist analysis and genuinely unparseable shell syntax gets a distinct gateway/node-host denial reason with regression coverage.
