## Summary

Keep open. Current main still treats global cron failure alerts as opt-in: `resolveFailureAlert` returns `null` when a job has no per-job `failureAlert` and `cron.failureAlert.enabled` is not explicitly `true`. The PR's central behavior change and its default-unset regression test are not present on main, and the related ClawSweeper matches cover different cron failure-alert capabilities rather than superseding this PR. The provided PR discussion is positive: Greptile identified the missing default-path test, Codex flagged the per-job override regression, and the PR's second commit/context says both concerns were addressed in the PR branch.

## Best Possible Solution

Keep the PR open for maintainer review. If OpenClaw wants recurring cron failure alerts to be opt-out, merge or adapt the PR's guard change and default-unset regression test while preserving per-job overrides when global alerts are disabled; also update the user-facing configuration docs to reflect the new default.
