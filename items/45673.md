## Summary

Keep open. Current main already implements the central Feishu session-account-before-defaultAccount priority, but this PR’s later review discussion added and addressed a narrower safety fix: only let inherited session account context win for Feishu-origin sessions. Current main does not pass `messageChannel` into Feishu tool account resolution, so non-Feishu sessions with colliding account IDs can still select a Feishu account instead of the configured Feishu default.

## Best Possible Solution

Keep the item open and narrow it to the remaining channel-scoping fix: thread `ctx.messageChannel` through Feishu tool account resolution for all shared Feishu tool-client paths, prefer inherited account context only for Feishu-origin sessions, preserve explicit `accountId` as highest priority, and add tests for Feishu session routing, non-Feishu account-ID collision fallback, unknown account fallback, and doc/bitable/drive/perm call sites as needed.
