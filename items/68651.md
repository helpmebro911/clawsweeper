## Summary

Keep #68651 open. Current main still has no `models.pricing.enabled` config surface and `startGatewayModelPricingRefresh` still schedules the pricing refresh whenever gateway runtime services start. The linked issue #53639 remains a real, concrete request for an operator disable switch; current main only changed the pricing cache shape and timeout behavior, not the requested opt-out.

## Best Possible Solution

Keep the PR open and have a maintainer or contributor rebase it onto current main, preserving the central behavior: a documented, schema-backed `models.pricing.enabled: false` option that skips the entire pricing bootstrap and scheduled refresh path without changing the default enabled behavior. The updated patch should cover current main's OpenRouter plus LiteLLM pricing fetch path, generated config metadata, public configuration docs, and tests that account for the current deferred bootstrap behavior.
