## Summary

Keep #40011 open. Current main still exposes only the global OTLP endpoint in OpenClaw config and the diagnostics-otel service still derives trace, metric, and log URLs from that single endpoint. The issue is actionable, not protected by author/labels, not stale, and the supplied discussion only adds a cc without a superseding implementation or plugin workaround.
