## Summary

Keep #13601 open. Current main has partial durable cron run history via per-job JSONL files, `cron.runs`, RPC/UI filters for status, delivery, query, and sort order, plus automatic size/line pruning, but the requested CLI log/history surface, time-window filters, `cron list` success/failure aggregates, and manual prune/vacuum workflow remain materially incomplete.

## Best Possible Solution

Keep #13601 open until maintainers either implement or deliberately narrow the remaining history product surface. A complete path would add timestamp filters through `ReadCronRunLogPageOptions`, `cron.runs`, protocol schemas, UI state, CLI options, and docs; expose a clearer `openclaw cron logs` or `openclaw cron history` command with status/time/follow/export behavior; add success-rate and failure aggregates to `cron list` or a dedicated summary; and decide whether manual prune/vacuum/history-retention-days belongs in core or should be explicitly documented out of scope.
