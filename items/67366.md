## Summary

Keep open. Current main still contains the unsafe `.trim()` calls on the Telegram credential replacement path, and the related fixing PRs in the provided GitHub context are closed unmerged rather than shipped.

## Best Possible Solution

Keep this issue open. Patch the shared wizard text handling so `WizardPrompter.text` results are normalized before any `.trim()` call, at least in `promptSingleChannelToken` and the generic setup-wizard text-input path. Add a regression test that mocks `undefined` from the Telegram token replacement prompt after choosing not to keep the existing token, and verify onboarding or `openclaw channels add` returns a validation/default path instead of throwing.

## Risks / Open Questions

- The exact terminal condition that makes `prompter.text` resolve `undefined` still needs a focused regression test, but current source still contains the unguarded dereference that would throw in that condition.
- A fix should preserve cancel handling: Clack cancel symbols should still raise `WizardCancelledError`, while nullish text results should be normalized or rejected before string methods run.
- Because this path handles secrets, regression tests should use fake tokens and should not print or snapshot real token values.
