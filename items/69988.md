## Summary

Keep open. Current main still supports compaction only under `agents.defaults.compaction`; `agents.list[]` has no compaction field in the TypeScript type or strict Zod schema, and the Pi runtime paths still read defaults-only compaction settings. #69987 is closed and superseded by #69988, while older overlapping #52751 is still open but less complete because it does not cover the runtime compaction handoff this PR targets.

## Best Possible Solution

Keep #69988 open for maintainer review, or land an equivalent core change: add `agents.list[].compaction` to `AgentConfig` and `AgentEntrySchema`, share the defaults compaction schema safely, merge per-agent compaction over `agents.defaults.compaction` in `resolveAgentConfig()` with nested `qualityGuard`/`memoryFlush` inheritance, thread the resolved agent id into Pi settings, safeguard extension setup, run, and direct `/compact` paths, and update generated schema/docs plus focused tests. Treat #69987 as superseded by #69988; #52751 can be handled separately as overlapping but not a complete replacement.
