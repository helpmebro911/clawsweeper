## Summary

Keep open. Current main has related Moonshot mitigations for native Kimi tool_call IDs and `thinking.keep`, but I did not find a Moonshot replay/history fix that preserves or synthesizes `reasoning_content` for assistant tool-call turns after LCM compaction or cross-model fallback. The linked lossless-claw issue is useful companion evidence, not a superseding OpenClaw issue.

## Best Possible Solution

Keep #71491 open. The likely OpenClaw-side path is a Moonshot-owned replay/stream wrapper or `sanitizeReplayHistory` hook that preserves existing `reasoning_content` and, when Kimi thinking is enabled, safely synthesizes the provider-required field for replayed assistant tool-call turns that lost it during compaction or cross-model fallback. Add focused Moonshot tests mirroring the DeepSeek cross-provider missing-`reasoning_content` coverage, and coordinate separately with lossless-claw so LCM preserves OpenAI-style top-level `reasoning_content` through compaction.
