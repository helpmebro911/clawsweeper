## Summary

Keep #69030 open. Current main still implements `openclaw infer model run` as an agent-runtime turn, with no `--raw` flag or alternate raw/direct capability wired into the CLI. The source, tests, and docs all match the reporter's root cause: local execution calls `agentCommand` for the default agent, gateway execution calls the `agent` RPC, and docs explicitly say `model run` reuses the agent runtime.

## Best Possible Solution

Keep the issue open. Implement an explicit raw/direct inference mode for `openclaw infer model run` or a new `model.raw` capability that prepares the selected provider/model without loading agent identity files, AGENTS.md/SOUL.md/USER.md, skills, MCP tools, or agent session state. Add CLI metadata/schema/docs and focused tests proving raw mode does not call `agentCommand` or the gateway `agent` RPC, while preserving the existing agent-wrapped path for backward compatibility. Treat #63700 and #67306 as related but separate inference-surface enhancements, not duplicates of this raw-runtime request.
