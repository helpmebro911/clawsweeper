## Summary

Keep open. Current main still has the reported gateway channel stop-timeout behavior: the timeout branch marks the channel running, returns before clearing the task/abort maps, and an existing test currently asserts that a restart after timeout is skipped. The linked PR #70056 is the right active fix path, but it is open and unmerged, so the issue is not implemented on main yet.

## Best Possible Solution

Keep #70024 open until #70056 or an equivalent gateway fix lands. The fix should clean `store.aborts` and `store.tasks` on stop timeout, set `running: false`, preserve useful stop/error metadata, and update `server-channels.test.ts` so restart after timeout succeeds instead of being skipped.
