## Summary

Keep open. Current `main` still contains the reported stop-timeout path, the restart gate still blocks on stale `store.tasks`, and a current regression test explicitly expects the timeout to leave the account `running: true` with no second start. There is no evidence on `main` that issue #70024 is fixed.

## Evidence

- **Stop-timeout branch still reports running and skips cleanup:** In the current `stopChannel` timeout branch, `waitForChannelStopGracefully(...)` returning false logs a warning, sets runtime to `running: true`, `restartPending: false`, and returns immediately. The cleanup of `store.aborts.delete(id)` and `store.tasks.delete(id)` only happens on the clean-stop path below that early return.
