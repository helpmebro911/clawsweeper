## Summary

Keep open. Current main still lacks the PR's central behavior: `models auth paste-token` has no non-interactive `--token` option/path, and invalid auth-profile warnings still omit accepted credential types. The linked bug is already closed as superseded by this PR, so this remains the canonical implementation candidate rather than a cleanup-close candidate.

## Best Possible Solution

The best path is to land this narrow fix, or replace it with an equivalent maintainer patch, with focused coverage for CLI forwarding, non-interactive token handling, empty/Anthropic token validation, invalid-type warning diagnostics, and a docs/help update for the new CLI usage plus secret-exposure guidance.
