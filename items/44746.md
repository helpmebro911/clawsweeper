## Summary

Current `main` does not match the reported failure mode. OpenClaw’s supported plugin contract already provides workspace resolution on the hook context (`ctx.workspaceDir`) and via `api.runtime.agent.resolveAgentWorkspaceDir(...)`; `before_prompt_build` is invoked with `workspaceDir` on the embedded runner and CLI preparation paths, and the embedded run params require a `workspaceDir` input. I could not find an in-tree `memory-continuity-probe`, so the report appears tied to an external plugin reading a non-contract `api.runtime.workspaceDir` field rather than a current OpenClaw core regression.
