## Summary

Keep open. Current main has model fallback runtime support and `openclaw models status --probe` auth probes, but it still does not provide `/models test-fallback` or an equivalent command that intentionally skips the primary model and verifies the configured fallback chain. The directly related PR #43342 attempted this feature and was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Continue tracking #6599 until maintainers either implement a first-class fallback-chain probe or explicitly decline the product surface. A good implementation would add `/models test-fallback` and possibly a CLI equivalent, reuse the real fallback resolution/run path with an intentional primary skip, support first-fallback and all-fallback modes, include focused slash/CLI tests, and document how it differs from `models status --probe`.
