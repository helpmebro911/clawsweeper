## Summary

Keep #6599 open. Current main still has model fallback runtime and `openclaw models status --probe`, but it does not expose `/models test-fallback` or an equivalent CLI command that intentionally skips the primary model and exercises the configured fallback chain. The only direct implementation attempt in the provided context, PR #43342, was closed unmerged, and the latest maintainer/Codex discussion already kept this issue open.

## Best Possible Solution

Keep #6599 open until maintainers implement or explicitly decline a first-class fallback-chain probe. A strong solution would add `/models test-fallback` and likely `openclaw models test-fallback`, reuse the real fallback resolution/run path with an intentional primary skip, support first-fallback and all-fallback modes, respect active agent/session auth context, add focused slash/CLI tests, and document how it differs from `models status --probe`.

## Risks / Open Questions

- A real fallback-chain probe will make live model calls, so UX should warn about token use and possible rate-limit effects.
- The implementation should avoid mutating session fallback overrides, auth-profile pinning, transcript state, or cooldown state unexpectedly.
- `models status --probe` overlaps conceptually, so docs should explain that it checks auth/model reachability while `test-fallback` exercises the fallback chain.
