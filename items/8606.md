## Summary

Keep #8606 open. Current main still prunes context by rewriting the in-memory message list, but it does not emit or document a `session:prune`/`context:prune` hook or return pruning metadata that a hook consumer could index. The earlier linked #9761 was closed unmerged, and the direct implementation PR #12581 remains open rather than shipped.

## Best Possible Solution

Keep #8606 open until #12581 or a replacement maintainer-approved implementation lands. The likely path is to extend `src/agents/pi-hooks/context-pruning/*` so pruning collects safe metadata, emits an internal `session:prune` or equivalent lifecycle event after successful pruning, adds focused tests, and documents the event in `docs/automation/hooks.md` and `docs/concepts/session-pruning.md`.
