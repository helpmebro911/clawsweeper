## Summary

Keep open. Current `main` still does not expose a context-pruning hook, and the directly related implementation PR #12581 remains open and unmerged. The earlier linked PR #9761 was closed without merging, so it does not prove the feature shipped.

## Best Possible Solution

Keep #8606 open until PR #12581 or a replacement maintainer-approved implementation lands. The likely path is an additive pruning lifecycle event around `src/agents/pi-hooks/context-pruning/*`, tests that verify when the event fires and what metadata/content is exposed, and docs updates for the public hook surface at `docs/automation/hooks.md`.
