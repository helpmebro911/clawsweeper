## Summary

Keep #8606 open. Current main still does not expose a `session:prune`/`context:prune` hook or pruning metadata for hook consumers: the pruning extension just rewrites messages via `pruneContextMessages(...)`, the pruner returns only `AgentMessage[]`, and the hook type catalog/docs do not list any pruning lifecycle event. The direct implementation PR #12581 is still open, while the earlier #9761 was closed unmerged.

## Best Possible Solution

Keep #8606 open until #12581 or a replacement maintainer-approved implementation lands. The likely implementation belongs in `src/agents/pi-hooks/context-pruning/*`: collect safe pruning metadata, emit a `session:prune` or equivalent lifecycle event only after successful pruning, cover it with focused tests, and document the new event in `docs/automation/hooks.md` and `docs/concepts/session-pruning.md`.
