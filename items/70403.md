## Summary

Keep #70403 open. Current main still implements the per-phase Dream Diary behavior that this PR is trying to change: light and REM phases each call `generateAndAppendDreamNarrative`, docs still say a diary entry is appended after each phase, and tests still assert light/REM narrative writes. The related PRs cover adjacent dreaming cleanup, but none is a closed/merged canonical replacement for the one-entry-per-sweep behavior.

## Best Possible Solution

Keep the PR open for maintainer review or land a cleaned variant that enforces one Dream Diary append per sweep. The likely code path is to remove light/REM calls to `generateAndAppendDreamNarrative`, keep deep or a sweep-level narrative as the single append point, update `docs/concepts/dreaming.md`, and replace current light/REM narrative tests with assertions that those phases do not write `DREAMS.md`. If applying this PR, also address the bot review's type-only import cleanup in `extensions/memory-core/src/dreaming-phases.ts`.
