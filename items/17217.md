## Summary

Current main still does not implement the requested wall-clock TTL pruning of normal conversation history or a manual prune-context/session-prune API. The existing behavior documents and implements session resets, manual compaction, and cache-TTL pruning of old tool results only, and the supplied issue discussion already includes a maintainer-authored Codex review explicitly keeping this item open.

## Best Possible Solution

Keep the issue open for maintainer product work. Decide whether `contextPruning.ttl` should expire or summarize normal conversation history, or whether the current tool-result/cache-cadence behavior is intentional. If full TTL pruning is desired, add an explicit gateway/protocol session-prune surface and implement age-based removal or summarization in the Pi/context-engine path with tests and docs. If it is intentionally out of scope, update user-facing docs to make that limitation unambiguous and point users to `/compact`, `/new`, and `/reset` as the supported controls.
