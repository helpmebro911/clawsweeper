## Summary

Keep #17217 open. Current main documents session lifecycle and provides `/new`, `/reset`, `sessions.reset`, and `/compact`/`sessions.compact`, but it still does not implement the requested wall-clock TTL pruning of normal conversation history or a manual prune-context API. The existing cache-TTL path is scoped to in-memory old tool-result pruning and only runs during context handling after the TTL window has elapsed.

## Best Possible Solution

Keep the issue open for maintainer product work. Decide whether `contextPruning.ttl` should expire or summarize normal conversation history, or whether current tool-result/cache-cadence behavior is intentional. If full TTL pruning is desired, add an explicit gateway/protocol session-prune surface and implement age-based removal or summarization in the Pi/context-engine path with tests and docs. If it is intentionally out of scope, update user-facing docs to make that limitation unambiguous and point users to `/compact`, `/new`, and `/reset` as the supported controls.
