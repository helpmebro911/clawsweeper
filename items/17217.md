## Summary

Keep open. Current main has documented session lifecycle and manual reset support, but the remaining requested capability is not implemented: cache-TTL pruning is in-memory tool-result trimming, not age-based removal of all messages older than the TTL, and there is no explicit prune-context session control endpoint documented or implemented in the gateway session surface.

## Evidence

- **Session lifecycle documented:** Current docs define session routing/lifecycle and say sessions reset via daily reset, idle reset, or manual `/new`/`/reset`. This addresses only the documentation/reset part of the request.
  - file: [docs/concepts/session.md:57](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/docs/concepts/session.md#L57)

  - file: [docs/concepts/session-pruning.md:9](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/docs/concepts/session-pruning.md#L9)
