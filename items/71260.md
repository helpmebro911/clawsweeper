## Summary

Keep open. Current main still has only `touch-activity` and `preserve-activity` session merge policies, `mergeSessionEntryWithPolicy()` still shallow-spreads patches, and there is no typed `pluginMetadata` session field or `sessions.patch` payload field. This is a concrete plugin/API gap RFC, not a ClawHub-only request or stale support report.

## Best Possible Solution

Keep #71260 open. The best path is an additive, typed session metadata contract: add a namespaced `pluginMetadata` shape or equivalent SDK-owned field, implement an explicit merge policy with deletion semantics, adopt it only at call sites that ingest plugin-owned metadata, and add focused unit tests for preservation, overwrite, and removal. Coordinate with #71426/#71428 so the sessions.patch and plugin-permission seams share one coherent plugin-state model.
