## Summary

Keep open. Current main still has no Telegram same-group cross-agent symmetry rule: Telegram group sessions are agent-scoped, `sessions_list` filters through the generic session visibility guard, and `sessions_send` applies the same cross-agent visibility/agent-to-agent policy. The related #59933 remains distinct because it covers same-agent group listing, not this cross-agent one-way failure.

## Best Possible Solution

Keep this issue open. Reproduce with two configured agents in the same Telegram group using explicit `tools.sessions.visibility="all"` and enabled, allowlisted `tools.agentToAgent`; if it reproduces, add regression coverage that each agent sees both `agent:<agent>:telegram:group:<id>` keys and can `sessions_send` both directions, then fix the gateway/session-store/visibility path. Track #59933 separately because it is same-agent group listing, not this cross-agent mismatch.
