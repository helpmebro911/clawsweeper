## Summary

Keep open. Current main has partial channel-side status reaction work, but it does not implement the requested generic queued-message feedback hook and the reported Feishu path still relies on reply-start typing, which is not invoked when the active-run branch only enqueues a follow-up. Related PR #39676 proposed an onQueued callback for this exact gap but was closed unmerged.

## Evidence

- **Generic queue branch still has no user-visible queued callback:** When an active run resolves to enqueue-followup, current main enqueues the follow-up, optionally kicks the follow-up runner, touches session state, cleans up typing, and returns undefined. There is no call to a queued-feedback callback before returning.
