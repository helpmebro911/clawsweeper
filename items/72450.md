## Summary

Keep this same-day regression report open. Current main still uses the 2026.4.24 Logs tab path (`logs.tail` -> configured file-log tail -> UI entries), and I found no post-release functional fix, duplicate, or merged PR that resolves the reported empty Web UI logs after upgrading to 2026.4.24.

## Best Possible Solution

Keep this issue open and investigate the 2026.4.24 Logs tab path end to end. First reproduce or ask the reporter for the `logs.tail` response, browser console/network error, install method, `logging.file` config, and whether `openclaw logs --follow` or `openclaw gateway call logs.tail --params '{"limit":50}'` returns lines on the same host. The likely fix belongs in the generic logging/UI path: either correct configured log-file discovery/cursor behavior in `src/logging/log-tail.ts` or surface a clearer UI error/diagnostic in `ui/src/ui/controllers/logs.ts` and `ui/src/ui/views/logs.ts`, with a regression test for the confirmed failure mode.
