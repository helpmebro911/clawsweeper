## Summary

Keep open. Current main still lacks `privacy.redactPII` and still emits raw human-identifying inbound metadata into the LLM-visible user prompt context, so the PR is not obsolete or implemented elsewhere.

## Best Possible Solution

Keep the PR open and rebase/rework it against current main. The likely product path is to add the `privacy.redactPII` config surface, thread it through `runPreparedReply()` into inbound prompt metadata rendering, pseudonymize or strip only OpenClaw-injected metadata, and preserve raw values for internal auth/routing paths. Add focused tests for chat IDs, sender metadata, reply/forward metadata, history senders, group/thread labels, and the forwarded-from case-sensitivity gap before merge.
