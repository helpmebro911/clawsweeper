## Summary

Keep this PR open. Current main still lacks the Swift `.gitignore` generalization, the proposed large-file guard, the proposed unjustified-exec guard, the package scripts, and the boundary-runner wiring. The PR is still actionable repo hygiene, but it needs a rebase onto the current `scripts/run-additional-boundary-checks.mjs` structure and fixes for the review-commented parser gaps before it should become a required CI gate.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, wire the new checks through `scripts/run-additional-boundary-checks.mjs`, update allowlists for the current exec/shell surfaces, add regression coverage for the parser gaps noted in review, and then rerun the targeted script tests plus the relevant changed CI checks.

- The proposed `check:no-unjustified-exec` guard should be parser-hardened before it becomes a required CI check, especially for semicolonless imports and `shell: true` inside comments or string literals.

- The exec and `shell: true` allowlists may need adjustment against current main before the checks can pass reliably.
