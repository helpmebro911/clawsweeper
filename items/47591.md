## Summary

Keep open. Current main has partial work in the same area, including configured typing keepalive through run completion/dispatch idle and write-ahead outbound delivery recovery, but it does not implement the requested gateway-level silence watchdog or per-turn response delivery confirmation with no-response alerts, same-turn retry, response IDs, and duplicate suppression.

## Evidence

- **partial_presence_keepalive:** The typing controller keeps configured typing alive during a run, refreshes TTL during long tool executions, and only stops once both run completion and dispatch idle have occurred. This is narrower than the requested generic silence watchdog message after a period with no operator-visible output.
