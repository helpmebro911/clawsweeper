## Summary

Keep this issue open. Current main still has the Kimi thinking/subagent failure shape: `sessions_spawn` can carry a thinking override into the child native subagent run, but the bundled Kimi provider only toggles top-level `thinking` and does not preserve or synthesize `reasoning_content` for replayed assistant tool-call messages. The related DeepSeek/Venice/Z.AI fixes do not cover Kimi Coding.

## Best Possible Solution

Keep this issue open and fix the provider-owned replay path for Kimi Coding, or add a documented generic provider-compat seam if multiple providers need the same behavior. The likely fix belongs in `extensions/kimi-coding`: when Kimi thinking is enabled, preserve existing `reasoning_content` or add a Kimi-accepted placeholder for replayed assistant tool-call messages, with focused regression coverage for `sessions_spawn` plus tool-call continuation under Kimi thinking. Cross-check the still-open #50178 provider-required `reasoning_content` gap, but do not close this Kimi-specific report as implemented or duplicated yet.
