## Summary

Keep this issue open. Current main still has the reported Kimi Coding failure shape: native `sessions_spawn` can pass a thinking override into the child run, but `extensions/kimi-coding` only patches the top-level `thinking` field and does not preserve or synthesize `reasoning_content` on replayed assistant tool-call messages. The prior Codex comment and related #50178 are useful context, but #50178 is broader/Qianfan-oriented and the closed #35023 replay fix does not cover this Kimi-specific provider path.

## Best Possible Solution

Keep this issue open and fix the provider-owned replay path in `extensions/kimi-coding`. When Kimi thinking is enabled, preserve existing provider reasoning metadata or add a Kimi-accepted `reasoning_content` placeholder for replayed assistant tool-call messages, then add focused regression coverage for thinking-enabled `sessions_spawn` plus Kimi tool-call continuation. Cross-check #50178 for a reusable provider seam, but do not close this Kimi-specific report as implemented or duplicated yet.
