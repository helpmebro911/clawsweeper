## Summary

Keep #70563 open. Current main has the broader #35023 latest-thinking preservation fix, but I do not see a Kimi Coding/subagent-specific fix for provider requests that require `reasoning_content` on assistant tool-call replay when thinking is enabled. Kimi only patches top-level `thinking`, while the blank `reasoning_content` placeholder behavior exists for providers like DeepSeek/Venice and is not exposed as a generic compat option.

## Best Possible Solution

Keep #70563 open and fix it in the Kimi provider or a documented generic compat seam. The likely implementation is a Kimi/Moonshot-style replay or payload wrapper that preserves or fills `reasoning_content` on assistant tool-call replay when thinking is enabled, plus focused tests for `sessions_spawn`/tool-call continuation with Kimi thinking. If maintainers prefer a shared solution, align it with the open #50178 class of provider-required `reasoning_content` replay gaps.
