## Summary

Keep #70563 open. Current main has generic thinking-block replay hardening and provider-specific `reasoning_content` fixes for DeepSeek/Venice/Z.AI-like paths, but I do not see an equivalent Kimi Coding fix for the reported `sessions_spawn` + thinking-enabled replay shape. The bundled Kimi wrapper still only toggles top-level `thinking` and strips generic reasoning params; it does not add or preserve `reasoning_content` on assistant tool-call replay messages.

## Best Possible Solution

Keep the issue open and add a Kimi-specific replay/payload fix. The likely direction is a provider-owned Kimi replay sanitizer or stream payload wrapper that, when Kimi thinking is enabled, preserves/translates prior reasoning or adds provider-accepted `reasoning_content` placeholders on assistant tool-call replay messages. Add focused coverage for a thinking-enabled parent turn that calls `sessions_spawn` and then continues without provider rejection.
