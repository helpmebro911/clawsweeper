## Summary

Keep #40016 open. Current main has session-label storage and a Gateway `sessions.patch` backend path, but the requested first-class `openclaw sessions rename` / `set-subject` CLI command is still absent, and the `openclaw sessions` table/JSON row model still omits `label`, `displayName`, and `subject`, so setting a label would not solve the reported readability problem in that CLI surface.

## Best Possible Solution

Keep the issue open and implement a focused `openclaw sessions rename <session-key> <label>` command, with `set-subject` or `set-label` as an alias if maintainers want to match the issue wording. Wire it to the existing `sessions.patch` label validation path, add a readable name column or otherwise surface label/displayName/subject in `openclaw sessions` text and JSON output, and cover the new command and output behavior in CLI tests and `docs/cli/sessions.md`.
