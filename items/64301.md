## Summary

Keep this PR open. Current main still lacks the allowlist-backed reverse lookup for hex-suffixed, underscore-stripped, dot-prefixed, and dropped-leading-character malformed tool names, while the open PR is a focused implementation candidate for the active agent-runtime normalization path.

## Best Possible Solution

Keep this PR open as an implementation candidate. Maintainers should review or rework the fallback in src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts, keep every recovery path allowlist-backed and fail-closed on ambiguity, and add targeted tests for hex-suffixed names, functionsread5f-style names, underscore-stripped and dot-prefixed variants, dropped-leading-character collisions, blacklist1a2b3c, readfeed, and foo_bar/foobar collisions before landing.
