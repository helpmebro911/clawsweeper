## Summary

Keep open. Current main already covers the gateway-wide fetch bootstrap portion, but it does not implement the PR’s remaining Discord-specific env-proxy WebSocket behavior or the claimed proxy tunnel reuse hardening. The PR is not ready as-is because the discussion flagged real design issues, including a core-src import from a bundled plugin and missing NO_PROXY handling, but the underlying Discord proxy gap is still actionable in this repo.

## Best Possible Solution

Keep the PR open, but ask for a narrower rework: leave the already-shipped gateway global env-proxy bootstrap alone, implement Discord WebSocket env-proxy support through the plugin SDK ambient proxy seam or another exported SDK helper, preserve `NO_PROXY` behavior, avoid extension imports from core `src/**`, and add focused Discord tests for `HTTPS_PROXY`, `NO_PROXY`, explicit `channels.discord.proxy`, and the Undici proxy tunnel option actually used to prevent stale CONNECT reuse.
