## Summary

Keep open. Current main still reads only `connectAuth.password` in gateway password mode, so a valid token-only `connectAuth.token` payload still falls through to `password_missing`. The PR remains a focused fix for the Rabbit R1 / Tailscale Funnel path described in #51953, and the provided PR context shows it also added the edge-case regression tests requested by Greptile.

## Best Possible Solution

Keep PR #69638 open and have a maintainer review or rebase it. The best code path is the narrow password-mode fallback to `connectAuth.token` when `connectAuth.password` is absent, with regression coverage for Rabbit token-only success, mismatched token rejection, and password-over-token precedence. The longer-term device-specific bootstrap-token follow-up should remain separate from this compatibility fix.
