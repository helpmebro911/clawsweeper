## Summary

Keep open. Current main has related 429 handling through provider SDK short retries, auth-profile cooldowns, and model/profile fallback, but it does not implement the requested run-level paused-for-rate-limit state with persisted retry metadata and scheduled automatic resume of the pending model call.

## Evidence
