## Summary

Current `main` still does not expose any standalone macOS node `audio.*` or `microphone.*` command family. Microphone access is modeled as a permission and as an optional audio track on existing `camera.clip` / `screen.record` flows, so there is not enough evidence to conservatively close this issue as fixed or obsolete.

## Evidence

- **macOS node command list on current main:** `MacNodeModeCoordinator.currentCommands` advertises canvas, screen, system, optional browser, optional camera, and optional location commands. When camera is enabled it adds only `camera.list`, `camera.snap`, and `camera.clip`; there is no standalone audio/microphone command.
