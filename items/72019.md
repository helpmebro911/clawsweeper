## Summary

Current main still matches the reported bug: native `sessions_spawn` puts the same `task` text into the subagent system prompt and the first child user message. The related PRs #72026, #72028, and #72053 are candidate fixes, but they are still open in the provided context and not present on current main. The ClawHub skill mentioned in the issue comment is a useful workaround/pattern resource, but this remains a core prompt-assembly bug in OpenClaw.

## Best Possible Solution

Keep #72019 open as the canonical bug. Land one narrow fix that keeps the full task in exactly one prompt role, preserves multiline task fidelity, and adds regression coverage around the assembled subagent system prompt plus first user message. PR #72053 appears closest in scope from the provided context, but maintainers should choose the best implementation after review.
