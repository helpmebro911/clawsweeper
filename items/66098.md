## Summary

Keep this PR open. Current main still does not expose or consume an effective reasoning default for Web UI chat rendering, so the user-visible mismatch remains. The PR is a plausible implementation candidate, but it needs rework against current main because thinking defaults are now partly handled through session rows/defaults and the proposed history-derived cache can become stale.

## Best Possible Solution

Keep this PR open for maintainer review and rework. The best path is a backward-compatible gateway/UI contract for effective reasoning state that preserves the current session-row/default thinking behavior, keeps fresh session updates authoritative after model or session patches, adds focused gateway and UI regression tests for fresh sessions with reasoningDefault, and leaves the broader config.patch gap tracked by related #64830 unless maintainers deliberately expand scope.
