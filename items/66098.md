## Summary

Keep this PR open. Current main has since implemented the thinking-default display path through session rows, but it still does not expose or consume an effective reasoning default for Web UI chat history. The PR is therefore partly stale but not obsolete; it remains a plausible implementation candidate for the reasoning-visibility mismatch described in related #64830, with maintainer review needed for the stale-cache concerns already raised in review comments.

## Best Possible Solution

Keep this PR open for maintainer review/rework. The best path is to add a backward-compatible gateway/UI contract for effective reasoning state that matches the current session row/session-controls architecture, keep the already-shipped thinking-default row behavior, address stale-cache precedence after session/model patches, and track the broader config.patch gap in related #64830 separately unless this PR is expanded to cover it.
