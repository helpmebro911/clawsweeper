## Summary

Keep open. Current `main` still routes Active Memory through the same embedded-agent path the report describes, `lightweight` mode only removes bootstrap-file injection rather than the broader startup work, and there is no shipped changelog evidence of a bootstrap-latency fix after the report. I could not do a live repro in this checkout because `pnpm` is unavailable here, so this is a source-and-history review rather than a benchmark confirmation.

## Evidence

- **Active Memory still uses embedded runner with lightweight/off flags:** The bundled Active Memory plugin still calls `api.runtime.agent.runEmbeddedPiAgent(...)` with `bootstrapContextMode: "lightweight"`, `disableMessageTool: true`, `verboseLevel: "off"`, `reasoningLevel: "off"`, and a blocking `timeoutMs`, matching the issue's reported path rather than a newer fast path.

- **Lightweight mode only skips bootstrap-file injection:** `applyContextModeFilter()` returns an empty bootstrap-file set for default/cron `lightweight` runs, but that optimization is scoped to bootstrap context files only; it does not indicate a separate minimal startup path for plugin loading, hooks, auth, or model resolution.
