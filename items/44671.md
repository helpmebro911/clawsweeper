## Summary

Keep open. Current main still rejects whitespace in Docker config/workspace paths and extra mount specs, so PR #44671 is not obsolete or implemented. The PR is a plausible focused fix for #44599, but the provided review context also identifies YAML quoting edge cases that should be addressed before merge.

## Best Possible Solution

Keep PR #44671 open or replace it with an equivalent maintainer patch. The best fix is in `scripts/docker/setup.sh`: allow spaces in config/workspace and valid mount paths, emit all generated compose mount strings through a safe YAML scalar helper, cover the sandbox overlay too, and add Docker setup e2e tests for spaced Windows-like paths plus apostrophes/backslashes/control-character rejection.
