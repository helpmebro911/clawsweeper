## Summary

Keep PR #46377 open. Current main still has the reported native PDF provider issues: Anthropic PDF calls use raw fetch with a caller-controlled base URL and API-key header, and Gemini PDF calls still put the API key in the generateContent query string. The PR is not implemented on main, not out of scope, and not safely obsolete, though its strict-guard approach needs maintainer review because it may regress proxy-only deployments.

## Best Possible Solution

Keep the item open and replace or update the PR with a narrow native PDF provider fix: route both Anthropic and Gemini PDF requests through the shared SSRF guard, move Gemini auth to `x-goog-api-key`, release guarded dispatchers correctly, add focused regression tests for private/internal base URLs and no query-string API key, and preserve configured provider proxy behavior instead of forcing strict direct networking for all deployments.
