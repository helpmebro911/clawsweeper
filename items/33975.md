## Summary

Keep #33975 open. Current main covers the model-attribution half through `messages.responsePrefix` template variables, but it still does not implement an opt-in approval gate before automatic model fallback.

## Best Possible Solution

Keep this open, or split it so #33975 can become the canonical fallback-approval tracker. The implementation should add an opt-in fallback approval policy in the core model fallback path, with config schema/types, docs, tests, and channel-appropriate approval UX. Treat `messages.responsePrefix` with `{modelFull}` / `{provider}` as the current solution for model attribution.
