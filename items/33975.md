## Summary

Keep open. Current main covers the model-attribution half through responsePrefix templates such as {modelFull}, but the requested opt-in approval gate before model fallback execution is still missing from schema, runtime flow, docs, and tests.

## Best Possible Solution

Keep this open as the fallback-approval tracker, while treating model attribution as already satisfied by messages.responsePrefix templates. Implement an opt-in per-agent fallback approval policy in the core model fallback path, with schema/types, docs, and tests for approval, denial, timeout, proceed/fail behavior, cancellation, and delivery through existing approval or notification surfaces where appropriate.
