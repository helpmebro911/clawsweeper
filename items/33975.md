## Summary

Keep this issue open. Current main covers the model-attribution half through `messages.responsePrefix` templates, but it still does not implement an opt-in approval gate before model fallback execution.

## Best Possible Solution

Keep this issue open, or split it so this issue becomes the canonical fallback-approval tracker. Treat `messages.responsePrefix` with `{modelFull}` and `{provider}` as the shipped attribution solution, then implement the remaining feature as an opt-in per-agent fallback approval policy in the core model fallback path with schema/types, docs, and tests for approval, timeout, fail-fast/proceed, cancellation, and channel delivery behavior.
