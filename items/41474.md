## Summary

Keep open. Current main has useful guardrails around stable appcast packaging and config-clobber auditing, but it does not fully solve the reported macOS companion behavior: the signed app still defaults Sparkle automatic checks/downloads on, and the config-clobber path records diagnostics/preserves the clobbered file rather than preventing or restoring the lost config. The reporter also explicitly said the suggested recovery workaround was not helpful.

## Best Possible Solution

Keep this open and fix it in the macOS companion/update owner area. The best path is to make automatic companion downloads/install-on-quit opt-in or prompt-only, ensure beta appcast/update selection cannot happen unless explicitly selected, and add release/test coverage for that policy. Separately, harden config preservation by preventing update-channel-only or reinstall/onboarding clobbers, restoring from last-known-good/backup when safe, or surfacing a clear recovery path in the app/doctor with regression coverage based on the existing macOS config-clobber test.
