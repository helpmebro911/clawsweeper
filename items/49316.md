## Summary

Keep open. Current main still does not parse mlx-vlm's `input_tokens`/`output_tokens` fields on the `openai-completions` streaming usage path. The broader usage normalizer supports those aliases, but the actual Chat Completions stream parser bypasses it and reads only `prompt_tokens`/`completion_tokens`, so the reported 0% context symptom can still occur for the payload shown in #49316. The discussion usefully links upstream work and two closed OpenClaw PRs, but neither PR is merged and there is no canonical open replacement item in the provided context.
