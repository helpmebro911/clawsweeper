## Summary

Current `main` has related Anthropic/Claude CLI hardening and cross-agent OAuth mirroring, but I did not find strong evidence that the specific stale `~/.claude/.credentials.json` versus per-agent `auth-profiles.json` desync reported in #44616 is fixed for the legacy `anthropic` OAuth path. The code now prefers `claude-cli/*` synthetic auth, yet direct Anthropic OAuth still resolves through stored auth profiles, and there is no Anthropic external-CLI sync implementation to make `.claude/.credentials.json` the source of truth automatically.

## Evidence

- **Anthropic plugin prefers Claude CLI synthetic auth, not external auth-profile sync:** The Anthropic manifest declares `syntheticAuthRefs` for `claude-cli`, and the runtime resolves live Claude CLI credentials only when the provider is `claude-cli`.
  - file: [extensions/anthropic/openclaw.plugin.json:8](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/anthropic/openclaw.plugin.json#L8)

- **Legacy/direct Anthropic OAuth still goes through auth-store OAuth resolution:** `buildAnthropicProvider()` still uses `ctx.resolveOAuthToken()` for usage auth on the Anthropic provider, which means stored OAuth profiles remain in the path for direct `anthropic/*` usage.
  - file: [extensions/anthropic/register.runtime.ts:602](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/anthropic/register.runtime.ts#L602)
