## Summary

Keep #14601 open. Current main still does not add `openrouter/moonshotai/kimi-k2.5` to the bundled OpenRouter onboarding/model-picker catalog. The prior maintainer/Codex comment remains accurate: K2.6 is present, dynamic/manual OpenRouter refs can work, but the requested selector row is still missing.

## Best Possible Solution

Keep the issue open. Either add `moonshotai/kimi-k2.5` to `extensions/openrouter/provider-catalog.ts` with current OpenRouter capability, context, token, and cost metadata, update `docs/providers/openrouter.md`, and add focused catalog/onboarding coverage; or have a maintainer explicitly document that K2.6 supersedes K2.5 for the OpenRouter onboard selector.
