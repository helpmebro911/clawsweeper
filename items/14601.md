## Summary

Current main still does not add `openrouter/moonshotai/kimi-k2.5` / `moonshotai/kimi-k2.5` to the bundled OpenRouter onboarding/model-picker catalog. Dynamic/manual OpenRouter refs work, and the live OpenRouter catalog still exposes Kimi K2.5, but those do not satisfy the specific selector/catalog request. The prior Codex review comment reached the same result, and the current checkout has not changed that surface.

## Best Possible Solution

Keep #14601 open until maintainers either add `openrouter/moonshotai/kimi-k2.5` to the bundled OpenRouter provider catalog with matching docs/tests, or explicitly decide and document/comment that K2.6 supersedes K2.5 for OpenRouter onboarding/model selection. Runtime dynamic refs and manual entry remain useful workarounds but are not a complete fix for the requested selector row.
