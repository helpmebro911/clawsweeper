## Summary

Keep open. Current `main` does not implement Discord channel-delete session cleanup. The shipped Discord monitor only registers message, reaction, reaction-remove, thread-update, and optional presence listeners, and the only session cleanup path is for archived threads, where matching sessions are merely marked stale (`updatedAt = 0`) rather than removed. Generic session maintenance still defaults to pruning after 30 days, so deleted-channel sessions can still linger for a long time. The PR is also stale against the current plugin layout (`extensions/discord/*`), so it needs a port rather than closure.
