## Summary

Keep #19362 open. Current main still does not register or implement Discord CHANNEL_DELETE session cleanup, while Discord guild-channel sessions remain persisted as isolated `agent:<agentId>:discord:channel:<channelId>` entries. The PR is still useful, but it needs a refresh for the current `extensions/discord` layout and should address the transcript archival/cleanup issues already raised in review.

## Best Possible Solution

Keep #19362 open and refresh it against current main. Add and register a Carbon channel-delete listener under `extensions/discord/src/monitor`, validate the deleted channel id, remove matching `agent:<agentId>:discord:channel:<channelId>` session entries through the current session-store/runtime seam, archive and clean transcripts with existing `sessions.delete` semantics, and add focused tests for registration, cleanup, invalid ids, multi-agent stores, and transcript handling.
