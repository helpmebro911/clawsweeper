## Summary

Keep open. Current main still has no Discord CHANNEL_DELETE listener or deleted-channel session cleanup, while Discord channel sessions are still persisted. The PR is a real implementation candidate, but it cannot land as-is because it targets the removed src/discord layout and needs a refresh to the current extensions/discord plugin boundary plus careful session/transcript safety.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a refresh against current main: port the listener and tests to extensions/discord/src/monitor, register it through registerDiscordMonitorListeners, delete exact agent:<agentId>:discord:channel:<channelId> entries for the affected channel, gather the deleted session entries under the updateSessionStore lock, archive and clean up transcripts through a plugin-safe SDK/runtime seam or core-owned helper with store-directory restrictions, then run the Discord plugin tests and changed gate before merge.
