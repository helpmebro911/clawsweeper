## Summary

Keep #19362 open. Current main still does not register or implement a Discord channel-delete listener, and the linked bug remains a real Discord plugin gap. The PR should be refreshed for the current `extensions/discord` layout and corrected for the transcript archival issues raised in review before merge.

## Best Possible Solution

Keep the PR open and refresh it against current `extensions/discord`. The best fix is to add a Discord `CHANNEL_DELETE` listener in the Discord plugin startup path, validate the deleted channel ID, remove matching `agent:<agentId>:discord:channel:<channelId>` session entries across the relevant stores using lock-aware session-store helpers, archive and clean up transcripts with the same semantics as existing session deletion, and add focused regression tests for multi-agent cleanup and transcript retention.
