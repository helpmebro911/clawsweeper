## Summary

Current main still does not implement Discord CHANNEL_DELETE session cleanup. This PR remains a useful implementation candidate, but it targets the removed `src/discord` layout and needs to be ported to the current bundled Discord plugin under `extensions/discord` before maintainer review or landing.

## Best Possible Solution

Keep this PR open as an implementation candidate, but require a refresh against current main: port the listener/tests to `extensions/discord/src/monitor`, register it in `registerDiscordMonitorListeners`, validate the Carbon channel-delete payload ID, use a plugin-safe way to target all relevant agent session stores, delete only exact `agent:<agentId>:discord:channel:<channelId>` entries, archive the deleted entries' transcripts under the update lock with current cleanup/restriction semantics, then run the Discord plugin tests and changed gate when available.
