## Summary

Keep open. Current main still lacks the requested subscribeEmbeddedPiSession-level markdown-table integration test, the relevant chunking implementation remains fence-aware rather than table-aware, and open PR #66344 is an unmerged implementation candidate that explicitly declares it will close this issue and #66614.

## Best Possible Solution

Keep this issue open while #66344 is reviewed. The best path is to land or deliberately close that PR; main should then include table-aware protection in EmbeddedBlockChunker and chunkByParagraph, plus subscribeEmbeddedPiSession-level coverage proving Intro, the full markdown table, and Outro are emitted as separate block replies without splitting the table.
