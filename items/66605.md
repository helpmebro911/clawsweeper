## Summary

Keep open. Current main still lacks table-aware block streaming chunking and also lacks the requested subscribeEmbeddedPiSession integration regression test. The linked bug #66614 is still open, and the linked PR #66344 is an open, unmerged attempt rather than shipped main behavior; its provided context describes unit-level tests, not the specific integration test requested here.

## Best Possible Solution

Keep #66605 open or fold it explicitly into the active table-splitting fix: implement table-safe break detection for EmbeddedBlockChunker/chunkByParagraph, then add the requested subscribeEmbeddedPiSession integration case in the streams-soft-chunks-paragraph-preference suite using createParagraphChunkedBlockReplyHarness and the Intro/table/Outro payload.
