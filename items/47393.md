## Summary

Keep #47393 open. Current main still lacks the central behavior from the PR: Discord preflight only resolves the current account route, has no route-owner arbitration across Discord accounts, and still drops bot identity to undefined when fetchUser("@me") fails. The PR is also not a clean obsolete close candidate because review comments identify real edge cases that should be addressed before merge.

## Best Possible Solution

Keep the PR open and ask for a refresh against current main. The best path is to implement route-owner arbitration in the Discord plugin preflight, preserve applicationId as the botUserId fallback in fetchDiscordBotIdentity, thread any needed bot-name mention fallback through the current handler types, and add regression tests for owner precedence, explicit cross-bot mention bypass, fetchUser failure, DMs, and disabled/stale accounts.
