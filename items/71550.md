Keep this PR open. Current main still lacks a `channels.slack.logLevel` schema/type/runtime path, and the PR is a focused implementation candidate for the linked Slack socket-mode log-noise request. The latest PR patch fixes the earlier uppercase/lowercase review concern, but PR review should verify the setting is applied to the actual socket-mode receiver/client path, not only Bolt App logging.
