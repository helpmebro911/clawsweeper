## Summary

Keep #48032 open. Current main still schedules GatewayClient reconnects with an unref'd timer, which matches the reported node-host exit condition. The linked fix PRs are useful and directly relevant, but they are still unmerged, so the bug is not implemented on main.

## Evidence

- **Current reconnect timer remains unref'd:** `scheduleReconnect()` still backs off and schedules `setTimeout(() => this.start(), delay).unref();`, so the reconnect attempt does not keep the Node event loop alive after a gateway close.
  - file: [src/gateway/client.ts:838](https://github.com/openclaw/openclaw/blob/d228463120a5057f7ccf9a5c31af62fe4a922088/src/gateway/client.ts#L838)
