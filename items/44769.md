## Summary

Current `main` already routes Telegram media downloads through the same proxy-aware transport path as Bot API calls, and the changelog records this fix in version `2026.3.13`. The issue describes behavior on `2026.3.12`, which is older than the shipped fix.

## Close Comment

Closing as implemented on `main` after a Codex review. Current code now threads Telegram media downloads through the same proxy-aware transport path as the bot API: `extensions/telegram/src/bot-core.ts:169` resolves a shared `telegramTransport` from `proxyFetch`, and `extensions/telegram/src/bot/delivery.resolve-media.ts:232` uses that transport’s `sourceFetch` for `/file/bot...` downloads. Regression coverage also exists in `extensions/telegram/src/bot.media.downloads-media-file-path-no-file-download.e2e.test.ts:134`, which asserts media downloads prefer `proxyFetch` over global fetch. The changelog records this fix in `CHANGELOG.md:2508` under version `2026.3.13`, while this report was filed against `2026.3.12`. If you still hit this on a current build, please open a new issue with the exact current version, proxy mode, and fresh logs.
