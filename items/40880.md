## Summary

Keep open. Current main still hardcodes the 5 MB media store limit and sandbox staging still copies inbound media with that constant, so the reported regression is not implemented on main. The provided GitHub context shows open PR #40950 explicitly closes this issue and proposes the correct narrow fix, so this issue should remain open until that PR or an updated equivalent lands.

## Best Possible Solution

Keep this issue open and review/land PR #40950 or an updated equivalent. The fix should add a bounded `media.maxBytes` config field, wire it through the media store, media server, and sandbox staging paths, preserve the 5 MB default, document the field, and add regression coverage for configured values and unset-config reset behavior.
