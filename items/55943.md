## Summary

Keep #55943 open. Current main still documents and implements the bundled `session-memory` hook only for `/new` and `/reset` internal command events. Subagent completion currently emits typed plugin lifecycle hooks such as `subagent_ended`, but there is no source-backed bridge from a completed subagent run/session to the bundled session-memory writer, and the provided related ClawSweeper context does not identify a canonical open item that supersedes this specific gap.

## Best Possible Solution

Keep the issue open and implement a narrow lifecycle bridge if maintainers want this behavior: when a subagent run/session reaches a terminal state, emit a memory-writing path with the child session key, workspace, session entry/transcript, and a clear reason such as `subagent-complete`, or teach the bundled `session-memory` hook to subscribe to an appropriate session/subagent lifecycle event. Add a regression test that spawns a subagent, completes it, and verifies the expected `memory/YYYY-MM-DD-*.md` file is created without duplicating main-session `/new` or `/reset` behavior.
