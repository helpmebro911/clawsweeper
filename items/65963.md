## Summary

Keep this issue open. Current main partially mitigates the original session-sprawl symptom by running one managed dreaming sweep, deleting narrative subagent sessions, and scrubbing stale dreaming artifacts, but the central operator controls remain missing: no public dreaming model override, no retain-days or burst cap, no `sessions purge --kind dreaming`, and no dreaming-specific session picker filter/collapse. The follow-up comment correctly identifies `dreaming.model` as the highest-leverage remaining gap, and related #65589 is closed unmerged while #69320 addressed a separate cron-timeout problem.

## Best Possible Solution

Keep this issue open for a memory-core-owned product fix: expose and document a supported dreaming execution/model override, wire it into the subagent/runtime path, add bounded retention or purge-by-kind semantics for generated dreaming sessions/transcripts, and add a targeted UI/CLI way to hide or purge dreaming artifacts. Reuse the existing narrative cleanup and scrubber as the baseline rather than replacing it wholesale.
