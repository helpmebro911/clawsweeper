## Summary

Keep this issue open. Current main has implemented the most important missing piece, `dreaming.model`, and has future-session cleanup for Dream Diary subagent sessions, but the issue still contains unresolved operator cleanup/UX asks: no dreaming retention knob, no `sessions purge --kind dreaming`, no sessions CLI/UI include/hide/collapse control for lingering dreaming artifacts, and no explicit burst/concurrency cap beyond existing phase limits. The fixing commit also says `Refs #65963`, not `Fixes`, which matches a partial implementation rather than a complete close.

## Best Possible Solution

Keep this issue open, but narrow the remaining product work: keep the shipped `dreaming.model` and narrative-session cleanup, then either add a memory-core-owned retention/purge policy for generated dreaming sessions/transcripts plus a sessions CLI/UI filter, or split those leftovers into smaller canonical follow-up issues and close this one once those are linked.
