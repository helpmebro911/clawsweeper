## Summary

Keep open. The prior Codex review remains correct on current main: ordinary non-timeout exec failures are still hidden at default verbosity, and success-looking assistant text can remain the only user-visible payload after the failed exec result. The related open PR is a companion destructive-command catalog, not a merged fix for fail-closed post-exec reply behavior.

## Best Possible Solution

Keep this issue open. The fix likely belongs in src/agents/pi-embedded-runner/run/payloads.ts and adjacent embedded-run coverage: treat exec/bash failures as mutating user-visible failures at default verbosity, surface the exact failed exec result or a clear failed-exec summary, and suppress or override success-looking assistant text when the matching exec action failed. Add a regression for exit 127 / python: command not found in local embedded mode. Keep #60507 separate unless it grows into the actual post-exec fail-closed implementation.
