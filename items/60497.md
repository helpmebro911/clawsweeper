## Summary

Keep open. Current main still carries the exec failure signal through the runner, but ordinary non-timeout `exec`/`bash` failures are intentionally suppressed from default user-visible payloads. Assistant text is added before that warning policy runs, so a success-looking final answer can still be the only visible output after `python: command not found`.

## Best Possible Solution

Keep this issue open. The likely fix belongs in `src/agents/pi-embedded-runner/run/payloads.ts` with adjacent embedded-run tests: treat failed `exec`/`bash` results as user-visible mutating failures at default verbosity, add a concise warning for exit 127 / command-not-found, and suppress or override success-looking assistant text when the matching exec action failed. Keep #60507 separate unless it changes scope into the actual post-exec fail-closed implementation.
