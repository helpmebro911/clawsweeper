Keep this PR open for maintainer follow-up or explicit superseding. Current main covers the duplicate-webhook problem through shared inbound dedupe, but the PR's remaining behavior is not fully implemented on main, and the submitted diff has security/privacy and regression concerns that make it unsafe to merge as-is.
