## Risks / Open Questions

- Merging this PR as-is could replace mature google-auth-library verification with custom JWT/OAuth handling in a security-sensitive webhook path.
- The PR head mixes dedupe, reply defaults, lifecycle parsing, access policy, auth/token caching, and config helper refactors, making review and regression isolation poor.
- The PR head's Google Chat-local dedupe is weaker than current main's shared scoped dedupe and can suppress retries or cross-account deliveries incorrectly.
- The PR head appears to log raw or sensitive Google Chat webhook/auth data in production paths.
- Current main still lacks at least part of the requested product behavior, so closing this PR now would require a clearer canonical follow-up for the replyToMode default decision.
