## Summary

Current main still does not implement Telegram waiting-tip messages or the proposed `messages.waitingTips` config. This PR is not obsolete or externally owned; it remains a plausible implementation candidate for Telegram wait feedback, but it needs maintainer/product direction and code fixes before merge.

## Best Possible Solution

Keep this PR open for maintainer/author follow-up. Rework it against `extensions/telegram/src/bot-message-dispatch.ts`, decide whether it should satisfy the narrower #6946 processing-indicator request or remain a separate product-approved bilingual tip feature, fix the send/cancel cleanup race, remove or fully implement `customTipsPath`, and add focused Telegram dispatch tests for fast-stream cancellation, successful cleanup, cleanup failure, forum topics, and superseded turns.
