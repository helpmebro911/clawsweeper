## Summary

Keep this PR open. Current main still lacks Telegram waiting-tip delivery and the `messages.waitingTips` config surface, while the submitted patch is stale and not merge-ready rather than obsolete or implemented.

## Best Possible Solution

Keep this PR open for maintainer or author follow-up. The useful path is to rebase onto `extensions/telegram/src/bot-message-dispatch.ts`, decide whether the scope should be the narrow processing indicator tracked in #6946 or a product-approved bilingual tips feature, fix the send/cancel cleanup race, remove or fully implement `customTipsPath`, and add focused Telegram dispatch coverage for fast-stream cancellation, successful cleanup, cleanup failure, forum-topic thread params, and superseded turns. If maintainers decide bilingual tips are not a core Telegram UX, close after directing the contributor toward a plugin/API discussion rather than treating current main as implemented.

## Risks / Open Questions

- Merging as submitted would not apply cleanly to current main because it targets the removed `src/telegram` layout instead of `extensions/telegram`.
- The proposed waiting-tip controller can leave an orphan Telegram tip message when cleanup/cancel races an in-flight send.
- The proposed `customTipsPath` config is advertised in type/schema/help/UI labels but has no runtime file-loading implementation, creating a misleading no-op setting.
- The bilingual learning-tip UX is broader than the narrower Telegram processing-indicator request in #6946 and needs maintainer product judgment before becoming bundled core behavior.
- Future custom-tip file loading would need careful path scope, error handling, and security review before merge.
