## Summary

Keep open. Current `main` has configurable agent/LLM timeout aborts and manual status/cancel surfaces, but it does not implement the requested main-session timeout monitor that pauses the run and asks the user to continue, cancel, or check status.

## Evidence

- **Agent timeout today is a hard abort, not a confirmation prompt:** `docs/concepts/agent-loop.md` documents `agents.defaults.timeoutSeconds` as an abort timer for the agent runtime, with early-end cases limited to abort/cancel/disconnect/wait timeout.
  - file: [docs/concepts/agent-loop.md:159](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/docs/concepts/agent-loop.md#L159)

  - file: [src/agents/pi-embedded-runner/run/attempt.ts:1835](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/src/agents/pi-embedded-runner/run/attempt.ts#L1835)
