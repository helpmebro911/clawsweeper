## Summary

Keep open. Current main still keys WebChat heartbeat chat suppression to `showOk`, so the reported `showAlerts=false` case is not implemented. The PR also should not be closed as implemented or merged as-is: the one-line swap to `showAlerts` would conflate OK acknowledgments with alert content and can regress OK-only visibility.

## Best Possible Solution

Keep the PR open, but ask for a refined implementation: WebChat chat filtering should classify the heartbeat reply after `stripHeartbeatToken`/ack normalization, suppress OK-only output when `showOk=false`, suppress non-OK alert content when `showAlerts=false`, and add regression coverage for at least `showOk=false/showAlerts=true`, `showOk=true/showAlerts=false`, and default visibility.
