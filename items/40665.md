## Summary

Keep open. Current main at f0b327cf686e3a957fef8c94985ee6509953a3b7 still wires Discord voice autoJoin only through READY, not RESUMED, and the join/cleanup paths still call VoiceConnection.destroy() directly without an idempotent stale-connection guard. The provided discussion also shows same-author PR #40676 is open, unmerged, and uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open as the canonical Discord voice reliability bug. Review and either land PR #40676 or replace it with an equivalent Discord plugin fix that registers autoJoin for RESUMED as well as READY, makes stale VoiceConnection cleanup idempotent when the dependency reports Destroyed, scopes cleanup safely for same-guild multi-account setups, and adds regression coverage for RESUMED autoJoin plus already-destroyed stale cleanup.
