## Summary

Keep open. Current main still registers Discord voice autoJoin only on READY, not RESUMED, and the voice teardown path still calls @discordjs/voice VoiceConnection.destroy() without a destroyed-state guard. The provided GitHub context also shows an open, unmerged same-author PR with closing syntax for this bug, so this issue should remain open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open as the canonical Discord voice reliability bug. Review, revise, and land PR #40676 or an equivalent Discord plugin fix that registers autoJoin for RESUMED as well as READY, makes VoiceConnection teardown idempotent when @discordjs/voice reports Destroyed, and adds regression coverage for RESUMED autoJoin, already-destroyed stale cleanup, and multi-account same-guild safety.
