## Summary

Keep open. Current main appears to have repaired the latest published appcast entry: `appcast.xml` advertises `2026.4.24`, and that zip URL returns HTTP 200. But the issue discussion is about the recurring macOS release/update pipeline class, not only one stale appcast row, and the remaining guardrail is not implemented in this repo: the public macOS workflow is still validation-only with private/manual asset and appcast handoff, `release-check` only validates Sparkle build numbers, and `make_appcast.sh` constructs the release asset URL without checking that the GitHub asset exists. The linked follow-up #71576 was closed as not planned, so it is not an open canonical tracker.
