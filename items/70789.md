## Summary

Keep open. Current main still uses reverse newest-first result pairing in `extractToolCards`, so the sequential same-name tool-call bug described by PR #70789 is not implemented on main. The PR is a focused UI bug fix linked to closed issue #70746, and the remaining maintainer path is review/merge or port with a stronger regression test for call-call-result-result ordering.

## Best Possible Solution

Keep PR #70789 open for maintainer review. The best path is to land or port the exact-ID-priority plus earliest-unmatched fallback fix in `ui/src/ui/chat/tool-cards.ts`, and add a true regression test in `ui/src/ui/chat/tool-cards.node.test.ts` for call-call-result-result ordering with repeated tool names.
