## Summary

Keep open. Current main still treats Linux service management as a systemd user-unit path, and the doctor repair/install path can still install or repair that user unit without first recognizing that a same-name system unit under /etc/systemd/system owns the gateway. The issue discussion is concrete and also notes related doctor fallout in #62763, but no current-main fix or superseding canonical issue is evident.

## Evidence
