## Summary

Keep open. Current main still exits 0 for supervised gateway restarts, so this PR is not implemented on main and remains potentially useful. The caveat is that the PR rationale overstates current default Linux service behavior because OpenClaw’s canonical systemd unit and docs now use Restart=always, not Restart=on-failure.

## Best Possible Solution

Keep the PR open for maintainer review. If accepted, split the spawned and supervised branches so spawned restarts continue to exit 0, non-launchd supervised restarts exit non-zero for Restart=on-failure recovery, launchd keeps its delay semantics, and a focused Linux/non-launchd supervised restart test is added. Also adjust the PR rationale or docs wording to reflect that current canonical OpenClaw systemd units use Restart=always.
