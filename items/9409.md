## Summary

Keep #9409 open. Current main still returns generic context-overflow user messages across the embedded runner, precheck path, shared formatter/sanitizer, and auto-reply fallbacks. The requested channel/session label, context size, model limit, model identity, and overage diagnostics are not shipped; related PR #15749 attempted the exact change but is closed unmerged, and the other linked PRs are also closed unmerged or cover adjacent context hardening rather than this user-facing diagnostic request.

## Best Possible Solution

Keep #9409 open. The best maintainer path is a shared context-overflow diagnostic formatter that accepts a safe session/channel label, provider/model identity, effective context budget, observed or estimated token count, computed overage, and concise recovery actions, then wires it through `run.ts`, precheck/attempt handling, `formatAssistantErrorText`, sanitizer normalization, and auto-reply fallbacks with focused regression tests.
