## Summary

Keep #9409 open. Current main still surfaces generic context-overflow replies in the embedded runner, precheck path, shared formatter/sanitizer, and auto-reply fallbacks. The requested session/channel label, context size, model limit, overage, and recovery-specific diagnostics are not shipped, and the related diagnostic PRs in the provided GitHub context are closed unmerged.

## Best Possible Solution

Keep #9409 open. The best maintainer path is a shared context-overflow diagnostic formatter that accepts a safe session/channel label, provider/model identity, effective context budget, observed or estimated token count, computed overage, and concise recovery actions, then wires it through `run.ts`, precheck/attempt handling, `formatAssistantErrorText`, sanitizer normalization, and auto-reply fallbacks with focused regression tests.
