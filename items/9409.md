## Summary

Keep #9409 open. Current main still surfaces generic context-overflow text and does not show the requested session/channel, token count, model limit, or overage diagnostics. The related PR #15749 was closed unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep the issue open. The best implementation is a shared context-overflow formatter that receives safe session/channel labeling, provider/model, effective context budget, observed prompt/session token count, and overage, then wires it through the embedded runner terminal path, `formatAssistantErrorText`, sanitizer/fallback delivery paths, and regression tests. It should avoid leaking sensitive raw session identifiers unless maintainer policy allows them.
