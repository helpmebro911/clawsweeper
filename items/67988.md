## Summary

Keep this issue open. Current main still routes the Agents/Control UI primary-model dropdown through a label builder that prefers configured aliases or raw ids, while the friendlier catalog display-name helper is only used by the chat model picker. The Anthropic runtime recognizes Claude Opus 4.7 and Claude Sonnet 4.6, but that does not fix the Control UI dropdown label path reported here.

## Best Possible Solution

Keep this issue open and fix the Agents/Control UI model option builder. The likely implementation belongs in `ui/src/ui/views/agents-utils.ts`: resolve configured model values against the qualified model catalog before falling back to aliases/raw ids, share the catalog display-name logic used by the chat picker, and add focused tests for configured `anthropic/claude-opus-4-7` and `anthropic/claude-sonnet-4-6` rendering as full Claude display names. If `anthropic/claude-opus-4-7` is still absent from the upstream catalog, add provider-owned catalog metadata or augmentation for that Anthropic row rather than hard-coding it in generic core UI code.
