## Summary

Keep open. Current `main` does not implement this PR’s behavior, and the proposed guard would not fire on the real runtime path because `gateway` `config.schema.lookup` is still classified as mutating.

## Evidence

- **Current payload policy still warns on mutating tool errors:** `resolveToolErrorWarningPolicy` shows warnings whenever `lastToolError.mutatingAction` is truthy, and there is no current `gateway`/`config.schema.lookup` exception on `main`. That means the reported warning noise is still possible today.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:107](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/agents/pi-embedded-runner/run/payloads.ts#L107)
  - command: `nl -ba src/agents/pi-embedded-runner/run/payloads.ts | sed -n '93,118p'`
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **`config.schema.lookup` is still treated as mutating:** `isMutatingToolCall()` treats `gateway` actions as mutating unless the normalized action is in `READ_ONLY_ACTIONS`. `READ_ONLY_ACTIONS` contains generic verbs like `get`, `list`, `inspect`, and `check`, but not `config.schema.lookup`, so this action resolves as mutating.
  - file: [src/agents/tool-mutation.ts:24](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/agents/tool-mutation.ts#L24)
  - command: `nl -ba src/agents/tool-mutation.ts | sed -n '112,145p'`
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Runtime propagates that mutating classification into tool errors:** The embedded subscribe path builds `ToolCallSummary` from `buildToolMutationState()` and stores `mutatingAction` on the summary used for later tool-error rendering. So the runtime path feeding payload warnings inherits the current mutating classification for `gateway config.schema.lookup`.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:93](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/agents/pi-embedded-subscribe.handlers.tools.ts#L93)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.tools.ts | sed -n '90,100p'`
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Review discussion already flagged the implementation mismatch:** The PR review context notes that the patch checks `mutatingAction === false`, but the runtime currently marks `gateway` `config.schema.lookup` as mutating because it is not in the gateway read-only allowlist. That means the proposed suppression would not address the reported behavior as submitted.
  - sha: [1441c3377578](https://github.com/openclaw/openclaw/commit/1441c3377578dd37c392ab492ebe33f2a46478b3)

## Risks / Open Questions

- Merging this PR as-is would likely leave issue #42856 unresolved because the suppression path depends on `mutatingAction === false`, while current runtime classification produces `true` for `gateway` `config.schema.lookup`.
- The PR also lacks the boundary tests called out in review, so even if the classification changed later, the intended narrow suppression scope would remain under-specified.
