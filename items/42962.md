## Summary

Keep this PR open, but do not land it unchanged. Current main still classifies `gateway` `config.schema.lookup` as mutating, so this PR's `mutatingAction === false` payload suppression would not run on the reported production path. A related open PR, #42898, targets the classifier root cause, but it is not merged and does not by itself make this PR obsolete. Security review found only agent payload/test changes, with no CI, dependency, lockfile, publishing, secrets, permissions, or external code-execution surface.

## Best Possible Solution

Revise this PR or coordinate it with #42898: fix `src/agents/tool-mutation.ts` so `gateway` `config.get` and `config.schema.lookup` are classified as read-only, keep a narrow payload-layer suppression for the exact benign `config schema path not found` lookup miss if needed, and add tests for the real runtime classifier path plus inverse mutating gateway failures. Once one canonical PR contains those pieces and passes the relevant agent tests, land that and let #42856 close as implemented.
