## Summary

Current main still does not implement the requested suppression. This PR is relevant to the open #42856 bug, but as submitted it is functionally incomplete because the real runtime path still marks `gateway` `config.schema.lookup` as mutating, so the PR's strict `mutatingAction === false` guard likely will not fire.

## Best Possible Solution

Keep this PR open, but revise it or coordinate it with #42898 so the shipped fix both classifies compound read-only gateway actions correctly and suppresses only the exact benign `config.schema.lookup` path-not-found miss. Add tests for the real path from `buildToolMutationState` into `lastToolError` and payload rendering, plus inverse tests proving mutating gateway failures and other gateway errors still surface.
