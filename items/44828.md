## Summary

Current `main` does not cleanly justify closing this PR. The generic reply pipeline already strips exact `NO_REPLY` payloads before Feishu channel delivery, so the PR's `reply-dispatcher.ts` change is largely redundant. But Feishu direct outbound paths still forward raw text to Feishu send helpers without any silent-token guard, and this PR also misses `sendMedia` mixed-payload paths and lacks matching tests. Conservatively, this should stay open rather than be closed as already implemented or obsolete.
