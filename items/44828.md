## Summary

Keep open. Current main still does not implement Feishu NO_REPLY suppression, and this PR is not safely obsolete: review feedback identifies real missing paths in the patch, while related PR #56167 is open, narrower, not merged, and also has an unresolved fallback gap.

## Best Possible Solution

Keep this PR open for maintainer follow-up or consolidation with #56167. The best fix should suppress exact silent-token text in Feishu `sendText`, all `sendMedia` text/fallback/comment paths, and reply-dispatcher text delivery while still sending media; export/use the helper through the Feishu SDK subpath; handle streaming-card cleanup on silent final replies; and add focused outbound and reply-dispatcher regression tests.
