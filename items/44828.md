## Summary

Keep open. Current `main` still lacks Feishu `NO_REPLY` suppression, so this is not implemented-on-main, and the submitted PR is not ready to close because review feedback identifies real uncovered paths in `sendMedia`/mixed-media delivery plus SDK-boundary and test gaps.

That means the underlying bug is still plausible on current `main` for `NO_REPLY` text reaching the Feishu send path.
