## Summary

Keep open. Current main still has the same Discord thread model-routing gap described in the prior Codex review: Discord resolves the native thread parent for routing and channel config, and core can fall back from a supplied parent session key, but normal Discord message handling only supplies that parent key when transcript inheritance is enabled. With the default `channels.discord.thread.inheritParent: false`, a thread without its own `modelByChannel` entry still cannot reach the parent channel id for model override resolution.

## Best Possible Solution

Keep this issue open and implement a narrow Discord model-routing fix: pass the resolved native thread parent channel id into model override resolution independently of `channels.discord.thread.inheritParent`, while preserving explicit thread-id `modelByChannel` precedence. Add regression coverage for a Discord thread with no thread override but a parent channel override, plus a case where the thread-specific override wins.
