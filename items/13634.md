## Summary

Keep #13634 open. Current main already has generic Project Context workspace bootstrap injection, so the AGENTS.md/SOUL.md/USER.md part is no longer Slack-specific. The remaining requested Slack account/global `channels.slack.systemPrompt` surface is still absent from Slack config types, strict schema, generated metadata/docs, and runtime prompt plumbing; Slack currently supports per-channel and `channels.slack.channels["*"].systemPrompt` for room channels only. The existing related PR #62792 is Slack file/media ingestion scope, not prompt/config parity, and the issue already has a maintainer-side Codex review comment reaching the same keep-open conclusion.

## Best Possible Solution

Keep #13634 open. Implement account-level channels.slack.systemPrompt in Slack config types, strict schema, generated channel metadata/UI hints, docs, and runtime context. Define precedence as exact channel entry, then wildcard channel entry, then account/global prompt, and apply it consistently for Slack message and slash-command handling across channels, private channels, DMs, and MPIM/group DMs. Preserve the existing generic Project Context bootstrap pipeline rather than adding Slack-specific workspace file injection.
