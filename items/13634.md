## Summary

Keep #13634 open. Current main has generic Project Context workspace bootstrap injection, and Slack supports per-channel plus wildcard `channels.slack.channels["*"].systemPrompt` for room channels. The requested account/global `channels.slack.systemPrompt` surface is still missing from Slack types, strict schema, generated metadata/docs, and runtime prompt plumbing, and DMs/MPIMs/slash-command paths do not have a global prompt fallback. The existing related PR #62792 is Slack file/media ingestion scope, not prompt/config parity.

## Best Possible Solution

Keep #13634 open. Implement account-level `channels.slack.systemPrompt` in Slack config types, strict schema, generated channel metadata/UI hints, docs, and runtime context. Define precedence as exact channel entry, then wildcard channel entry, then account/global prompt, and apply it consistently for Slack message and slash-command handling across channels, private channels, DMs, and MPIM/group DMs. Preserve the existing generic Project Context bootstrap pipeline rather than adding Slack-specific workspace file injection.
