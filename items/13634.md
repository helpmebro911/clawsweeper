## Summary

Keep #13634 open. Current main already has generic workspace bootstrap-file injection, but the requested global/top-level Slack `systemPrompt` is still missing from Slack types, schema, docs, and runtime plumbing. Slack only resolves `systemPrompt` from matched per-channel entries, including wildcard entries, and DMs/MPIMs do not get that prompt path.

## Best Possible Solution

Keep #13634 open. Implement a Slack account/top-level `channels.slack.systemPrompt` config surface, document precedence against exact channel entries and wildcard entries, and plumb it through message and slash handling for channels, MPIM/group DMs, and DMs. Do not add Slack-specific bootstrap-file injection; keep using the existing generic workspace bootstrap context pipeline.
