## Summary

Keep open. Current main still only partially overlaps #38625: the gateway has WS/control-plane update support and exposes version during WS probing, but `GET /health` and `/healthz` still return only `{ ok, status }`, no `POST /admin/update` HTTP route exists, and the typed `update.run` RPC does not accept `target_version`.

## Best Possible Solution

Keep #38625 open until maintainers either implement the requested HTTP API or explicitly choose and document a WS-only fleet-management direction. A conservative implementation would add `version` to `/health` and `/healthz` with HTTP probe tests/docs, then add authenticated/rate-limited `POST /admin/update` as a thin wrapper over the existing update runner. If version pinning is accepted, extend the typed update schema and runner target/tag contract rather than adding ad hoc package-manager behavior.
