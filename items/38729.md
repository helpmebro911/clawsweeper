## Summary

Current main still does not implement the requested Anthropic Opus 4.6 xhigh/max advertisement. Anthropic now uses resolveThinkingProfile, which exposes xhigh/adaptive/max only for Opus 4.7 and adaptive only for Claude 4.6. The PR's exact supportsXHighThinking hook is stale under the current provider-thinking API, but the underlying request remains plausible because the Anthropic transport still maps requested xhigh to max for Opus 4.6. No conservative close reason fits without maintainer product judgment or a canonical replacement item.

## Best Possible Solution

Keep the PR open, but require a rework before merge. If OpenClaw should advertise Opus 4.6 max effort, implement it in extensions/anthropic/register.runtime.ts through resolveThinkingProfile, restrict it to Opus 4.6 unless Sonnet has real distinct max support, and update extensions/anthropic/index.test.ts plus docs/tools/thinking.md. If maintainers intend xhigh/max to remain Opus 4.7-only, close this PR with that explicit policy rationale or link a canonical provider-thinking policy issue.
