## Summary

Keep this PR open. Current main still has the old CLI watchdog defaults that the PR targets: resumed CLI sessions are still capped at 180000 ms of no output by default, and the resolved watchdog timeout is passed to the process supervisor where it can terminate the run. The supplied PR diff is narrow and security-neutral, so the useful path is maintainer review/rebase with a small regression test rather than cleanup closure.

## Best Possible Solution

Keep the PR open as a plausible implementation candidate. Rebase or reapply the constants change on current main, add a focused test for fresh/resume default resolution and the existing global-timeout clamp, then review/land it or split the optional disable-cap knob into a separate issue if maintainers want that product behavior.
