## Summary

Keep open. The cache-growth problem described by the PR is still present on current main and in the latest release: the bootstrap snapshot cache, workspace file cache, cache-trace writer map, and model context cache remain uncapped Maps. The PR should not be merged as-is because its branch also includes unrelated web_search/tool-schema rewrites with credible review feedback about provider-registry regressions, but the core cache-cap fix is still actionable in this repo.

## Best Possible Solution

Keep the PR open or ask for a narrowed replacement: add size caps and oldest-entry eviction for the four identified cache Maps on current main, include focused tests for each cache, guard eviction so refreshing an existing key does not evict an unrelated entry, and leave the existing web_search runtime provider registry untouched.
