## Summary

Keep open. Current main does mitigate the Anthropic/provider replay risk by filtering `delivery-mirror` assistant rows before LLM replay, but it still performs the reported parent-session mirror write as `role: "assistant"` and has no config toggle or alternation guard. The existing closed delivery-mirror reports cover duplicate same-text/webchat rendering, not this broader cron/subagent outbound mirror behavior.

## Best Possible Solution

Keep the issue open. Add a focused regression for a cron/isolated/subagent `message` send into a parent session whose latest transcript row is already assistant-role, then either add a documented `session.deliveryMirror` policy/toggle or guard delivery-mirror appends so transcript-only mirrors cannot create unsafe assistant-to-assistant parent rows. Preserve the existing LLM replay filter and coordinate with the open delivery-mirror wake/resume work from #63779 if the final design replaces mirror-only parent signaling with a real system event.
