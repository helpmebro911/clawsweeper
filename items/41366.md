## Summary

Keep open. Current main has adjacent memory/dreaming features and Feishu/Lark mention gating, but the requested product behavior is not fully implemented: there is no governed durable natural-language rule layer with conflict/supersession semantics, and Feishu broadcast explicitly uses one active reply path rather than letting every explicitly mentioned bot reply. The linked openclaw-listen-only plugin/workaround is useful context, but the commenter also says it does not solve durable rule learning; related PR #42400 is still open and only covers listen-only suppression.

## Evidence

- **Current memory is Markdown-backed, not a governed rule layer:** The memory docs say OpenClaw remembers by writing plain Markdown files and has no hidden state. This supports persistence, but not the requested first-class durable rule log with explicit conflict handling between session rules and workspace rules.
