## Summary

Keep open. Current main has durable memory, standing-order files, mention gating, and some broadcast/multi-agent routing, but it does not implement the requested governed natural-language rule-learning layer or explicit all-mentioned-bots reply semantics. The related open PR #42400 is only a partial listen-only/no-reply suppression proposal, and flowolforg's linked openclaw-listen-only approach is a useful context-isolation workaround rather than the requested durable rule promotion/conflict model.

## Best Possible Solution

Keep this issue open and scope it into concrete design work: define a governed durable rule layer or typed memory/rule records with review, precedence, and supersession semantics; decide how promoted rules interact with AGENTS.md, SOUL.md, MEMORY.md, and live session bootstrap caches; and add channel/plugin contract support for explicit mention target sets so group routing can dispatch one reply from every mentioned bot/agent while keeping unmentioned bots silent. Review #42400 separately as a partial listen-only suppression feature, not as the canonical fix.
