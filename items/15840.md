## Summary

Current main does not implement the requested suppression option. The Signal outbound path still injects a visible `<media:${kind}>` placeholder for media-only sends, and the shared `messages` config type has no `suppressMediaPlaceholders` or granular `mediaPlaceholders` field. Related PRs in the provided GitHub context were closed unmerged, so they do not supersede the open issue.

## Evidence

- **Signal still injects media-only placeholder text:** When a media attachment resolves to a MIME kind and the outbound message text is empty, Signal sets `message` to `<media:image>` or `<media:${kind}>`; for audio this is `<media:audio>`.
  - file: [extensions/signal/src/send.ts:161](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/extensions/signal/src/send.ts#L161)

  - sha: [bd796d1c85c7](https://github.com/openclaw/openclaw/commit/bd796d1c85c7dec0dcb89ed11502e4271ded1314)
- **No shared messages config option exists:** `MessagesConfig` includes response prefix, group chat, queue, inbound debounce, ack/status reactions, tool-error suppression, and TTS settings, but no `suppressMediaPlaceholders` or `mediaPlaceholders` field.
  - file: [src/config/types.messages.ts:86](https://github.com/openclaw/openclaw/blob/bd796d1c85c7dec0dcb89ed11502e4271ded1314/src/config/types.messages.ts#L86)
