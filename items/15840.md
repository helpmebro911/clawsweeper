## Summary

Keep #15840 open. Current main still synthesizes visible `<media:audio>` / `<media:${kind}>` fallback text for media-only Signal and iMessage sends, and the requested `messages.suppressMediaPlaceholders` or granular `messages.mediaPlaceholders` config contract is still absent. The provided issue discussion also contains a recent maintainer-side Codex review comment explicitly keeping this as the canonical cross-channel request; related implementation PRs were closed unmerged.

## Best Possible Solution

Keep #15840 open as the canonical product/config request. A conservative fix should add a documented default-off `messages` option, update `MessagesConfig`, strict zod schema, generated config metadata/help/docs, and route the setting through the bundled Signal and iMessage media-only send paths while preserving current fallback behavior by default and adding focused media-only audio tests.
