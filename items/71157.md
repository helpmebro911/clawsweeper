## Summary

Keep #71157 open. Current main has the bundled WhatsApp channel plugin, but it does not expose disappearing-message expiration in WhatsApp config/types, does not resolve such a value per account, and does not pass expiration metadata through the outbound text/media/quoted/poll send paths. This is actionable in the repo’s WhatsApp plugin and config/docs surfaces rather than a ClawHub-only cleanup close.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin: add root and account-scoped typed config/schema/docs for `disappearingMessagesSeconds`, resolve account overrides through `resolveWhatsAppAccount`, pass positive values into the Baileys message generation options for text, media, quoted, poll, and message-tool sends, and keep `0`/unset behavior unchanged. Add focused tests for channel default, account override, unset/zero behavior, quoted sends, media/voice-note secondary sends, polls, and outbound payload/tool paths.
