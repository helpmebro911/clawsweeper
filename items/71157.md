## Summary

Keep #71157 open. Current main does not expose `channels.whatsapp.disappearingMessagesSeconds`, does not resolve an account-level override, and does not pass Baileys `ephemeralExpiration` through the WhatsApp outbound paths. This is not a ClawHub cleanup candidate because WhatsApp is already a production bundled channel/plugin and the request is for privacy-sensitive behavior in that supported channel. The provided related reports cover quote replies, mentions, and subagent delivery, but none supersedes disappearing-message expiration.

## Best Possible Solution

Keep the issue open and implement it in the bundled WhatsApp plugin: add typed and zod config for `channels.whatsapp.disappearingMessagesSeconds` plus account overrides, regenerate/update config metadata and docs, resolve the effective account/channel value in the WhatsApp account config path, and merge positive values into Baileys `MiscMessageGenerationOptions` for normal text, quoted replies, media/voice-note follow-up text, polls, auto-replies, and message-tool/channel sends. Tests should cover channel default, account override, unset/0 behavior, quoted text, media, and poll/tool paths.
