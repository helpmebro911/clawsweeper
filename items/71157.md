## Summary

Current main does not implement WhatsApp outbound disappearing-message expiration. The WhatsApp config schema has no `disappearingMessagesSeconds` field, the resolved account shape does not carry an expiration value, and the outbound text/media/poll send paths only pass existing options such as quote metadata, GIF playback, account id, and document filename. This is a recent, actionable WhatsApp plugin enhancement rather than a stale report or a ClawHub-only request.

## Best Possible Solution

Keep the issue open and implement it in the bundled WhatsApp plugin: add channel/account `disappearingMessagesSeconds` schema and resolved-account support, compute the effective positive value with account override semantics, pass it through `ActiveWebSendOptions` into Baileys send options for text, quoted replies, media, voice-note companion text, polls, and tool/payload sends, then update WhatsApp docs and targeted tests for configured, overridden, unset, and media/quoted paths.
