## Summary

Keep this issue open. Current main still does not enable OS-level TCP keepalive for the WhatsApp WebSocket, and there is now an open implementation PR from the same author with closing syntax. That PR should be reviewed and either landed or closed before this issue is resolved.

## Best Possible Solution

Keep this issue open while PR #72735 is reviewed. The likely fix belongs in the WhatsApp plugin socket setup or a small WhatsApp-local agent/socket helper, with focused tests proving setKeepAlive(true, 15000) or an equivalent conservative delay is applied without breaking ambient proxy handling. Close this issue only after that PR or another equivalent implementation lands on main.
