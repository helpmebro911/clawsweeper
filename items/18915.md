## Summary

Keep #18915 open. Current main at e672b61417af5c45b0431df6d9109a1f4b618ef5 still sends regular Telegram videos without width/height in both direct-send and reply-delivery paths, and there is no shipped video-dimension probe or regression test. The PR addresses a plausible reproduced Telegram bug, but the current diff should be revised before merge because it imports from core src/** across the plugin boundary and includes an unrelated global MAX_VIDEO_BYTES change.

## Best Possible Solution

Keep the PR open and request a focused revision: implement a boundary-compliant video-dimension probe through an existing or new narrow media-runtime/plugin-sdk seam, add width and height to regular Telegram sendVideo calls in extensions/telegram/src/send.ts and extensions/telegram/src/bot/delivery.replies.ts, keep GIF animations and video notes excluded, add regression tests for both send paths, and split or drop the unrelated global MAX_VIDEO_BYTES change.
