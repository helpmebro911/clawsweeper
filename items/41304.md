## Summary

Keep this issue open. Current main has useful guardrails for missing allowlisted tools and better diagnostics, but the discussion reports a broader failure mode: plugin/MCP tools can appear registered while the model gives simulated success or instructions instead of emitting a real tool call. I did not find current-main evidence that this broader registered-tool invocation problem is fixed or superseded.

## Best Possible Solution

Keep the issue open and narrow the next maintainer action: ask for a trajectory export or equivalent logs showing the final callable tool list, submitted tool schemas, selected provider/model, and actual tool-call dispatch events. If the trace shows no callable tools, route users to the shipped allowlist guard/config fix; if tools are present but no structured tool call is emitted, improve provider/model diagnostics or add a focused regression/live smoke around one plugin or MCP action tool.
