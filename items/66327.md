Keep this PR open. Current main still lacks an MS Teams outbound sendPayload path, while core only uses native interactive/channelData delivery when a channel exposes sendPayload. The PR is a focused Teams-plugin implementation candidate rather than obsolete cleanup, though it needs normal rebase/review and coordination with the open Teams messageBack inbound PR before landing.

Rebase it onto current main, preserve the Teams-plugin-owned sendPayload implementation, route card-media sends through the same outbound dependency seam as existing Teams text/media sends, and coordinate or validate the messageBack /approve click path with #64503 before landing with focused outbound tests.
