## Summary

Keep open. Current `main` already normalizes Telegram replies before delivery, so part of the PR is now partially superseded, but the core parsing bug from the review thread still exists on `main`: reply tags are removed only after MEDIA extraction, so payloads like `[[reply_to:42]] MEDIA:./photo.jpg` still will not be recognized as media. The PR discussion also shows a follow-up commit specifically addressing that remaining gap.
