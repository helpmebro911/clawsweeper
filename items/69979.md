## Summary

Keep open. Current main still forwards the raw `emoji` parameter from the Feishu `react` action directly into Feishu's `reaction_type.emoji_type`; Feishu's API expects documented enum strings such as `SMILE` or `THUMBSUP`, so the reported `emoji="👍"` 400 path is still plausible and not implemented on main. The linked PR #71970 is an active, focused proposed fix for this exact issue, but it is open and unmerged, so the issue should remain as the canonical bug until that or an equivalent fix ships.

## Best Possible Solution

Keep #69979 open and land PR #71970 or an equivalent Feishu-plugin fix that normalizes user-facing Unicode emoji input to Feishu `emoji_type` enum values before create/list/remove reaction calls, with regression coverage for `message action=react emoji="👍"` and related common emoji variants. Once merged and released, close via the fixing PR.
