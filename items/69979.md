## Summary

Keep open. Current `main` and the latest tagged release still route Feishu `message action=react` by passing the user-supplied `emoji` string straight into Feishu's `reaction_type.emoji_type` field, while the Feishu reaction helper and docs both describe that field as Feishu emoji-type codes such as `THUMBSUP`/`SMILE`. The report used raw Unicode `👍`, so the reported HTTP 400 remains plausible and is not shown fixed on `main`.

## Evidence

- **Current main passes raw `emoji` through as Feishu `emoji_type`:** `feishuPlugin` reads `ctx.params.emoji` verbatim and passes it to `addReactionFeishu({ emojiType: emoji })` with no normalization or Unicode-to-Feishu mapping layer.
  - file: [extensions/feishu/src/channel.ts:1013](https://github.com/openclaw/openclaw/blob/a4fc6c2409f9c28e55e04e22c2e243c6900d66a3/extensions/feishu/src/channel.ts#L1013)
