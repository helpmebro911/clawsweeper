## Summary

Keep open. Current main has useful partial mitigations for plan-only stalls, provider stop-reason surfacing, gateway capability probes, and exec approval visibility, but it does not appear to implement the central request: reconciling explicit operator authorization with model safety/refusal decisions for normal SSH/DevOps diagnostics. The provided related items are adjacent model/tooling reports, not a canonical duplicate or superseding tracker.

## Evidence

- **Issue scope remains core agent behavior:** The issue and follow-up comments ask for clearer separation between tool failure, authorization failure, and model-level soft refusal, plus better handling for read-only diagnostics on explicitly authorized infrastructure. This is core operator/tool-execution behavior, not just an optional plugin idea.
- **Plan-only recovery is partial, not an operator-authorization reconciliation:** Current main retries plan-only assistant turns with an act-now instruction, but the guard only targets planning/stalling patterns and skips cases with client tool calls, deterministic approval prompts, tool errors, non-plan activity, or side effects. It does not make model safety decisions aware of operator scopes.
