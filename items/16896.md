## Summary

Keep #16896 open. Current main still lacks the requested Dashboard/WebChat right-click or message-level Reply workflow: the chat view has draft/send/session/sidebar state but no selected reply target, the thread only handles scroll and code-copy clicks, message actions remain copy/expand-oriented, and `chat.send` still has no reply/quote field. Existing `replyTarget` handling is assistant directive display for `[[reply_to_*]]`, not a user-initiated quote/reply composer flow. Related #29732 is closed and cross-references back to #16896, so #16896 remains the canonical tracker.

## Best Possible Solution

Keep #16896 open as the canonical Dashboard/WebChat reply UX tracker. A complete fix should add an accessible message-level Reply action and/or context menu, store the selected quoted message in composer state, focus the input, render the selected quote clearly, and decide whether to send simple quoted text or persist reply metadata through an additive `chat.send` protocol field with controller/server/UI regression tests.
