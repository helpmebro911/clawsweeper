## Summary

Keep #16896 open. Current main still lacks a Dashboard/WebChat message-level Reply workflow: there is no right-click/context-menu or reply callback in the chat view, the composer sends plain `chat.send` payloads, and the gateway `chat.send` protocol has no reply target field. Existing reply support is only display normalization for assistant-side `[[reply_to_*]]` directives.

## Best Possible Solution

Keep #16896 open as the canonical Dashboard/WebChat reply UX request. A complete fix should add an accessible message-level Reply action/context menu in the Control UI chat surface, store selected reply target or quote state in the composer, focus the input, and send a designed quoted-message representation or additive `chat.send` reply-target field with matching protocol, rendering, docs, and UI tests.
