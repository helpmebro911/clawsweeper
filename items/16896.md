## Summary

Keep #16896 open. Current main still lacks the requested Dashboard/WebChat user reply workflow: no right-click/context-menu handler, no message-level Reply action, no composer reply-target state, and no `chat.send` reply/quote field. Existing reply-pill behavior is assistant directive display only, and related #29732 is closed/cross-referenced back to #16896, so this remains the canonical tracker.

## Best Possible Solution

Keep #16896 open as the canonical Dashboard/WebChat reply UX tracker. A complete fix should add an accessible message-level Reply action and/or context menu, store the selected message or quote in composer state, focus the input, render the selected quote clearly, and decide whether to send simple quote text or persist reply metadata through an additive `chat.send` protocol field with controller/server/UI regression tests.
