## Summary

Keep #13962 open. Current main still only partially covers the request: WhatsApp mention-gated group messages that are not processed are buffered for later context, and normal agent/channel routing can choose a model for an entire group or routed agent. It still lacks mention-state-aware model routing, background-model processing for unmentioned group chatter, and a per-group `mentionContextMessages`/context-window setting.

## Best Possible Solution

Keep the issue open until OpenClaw implements mention-state-aware routing and configurable per-group mention context sizing, or intentionally exposes a pre-gating trusted plugin/API hook that can process unmentioned group messages with a background model. Likely touch points are WhatsApp group gating, model/route selection, WhatsApp config types/schema/docs, and regression tests for skipped unmentioned messages, triggered mentions, pending-history injection, background processing, and per-group sizing.
