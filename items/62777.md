## Summary

Keep open. Current main appears to cover Part A for ordinary error runs: cron increments consecutiveErrors, resolves failureAlert, and calls the gateway delivery hook after the configured threshold. However, Part B remains present on current main: Discord explicit target parsing still returns the bare target id instead of the normalized user:/channel: target, and that parsed value is used in the cron/outbound delivery target path. The linked #62798 discussion is relevant and positive evidence, but the provided context says that PR closed unmerged and the current source still has the same target.id behavior.
