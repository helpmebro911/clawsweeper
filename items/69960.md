## Summary

Keep open. Current `main` still contains the reported symlink-skip behavior in plugin discovery, and there is no strong evidence in code or local history that the profile-specific `plugins install` path handling has been fixed on `main`. The issue timeline also already cross-references an open fix PR (#69971), which further argues against conservative auto-close.
