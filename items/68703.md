## Summary

Keep open. Current main still does not enforce requester authorization for Discord guild-admin message actions: the registered Discord action adapter has no `requiresTrustedRequesterSender` hook, the dispatcher only enforces trusted sender identity when that hook exists, and guild-admin runtime mutations still use feature gates without sender permission checks. The linked clean fix PR #68716 remains open and unmerged; the earlier PR #68705 was closed unmerged.

## Best Possible Solution

Keep #68703 open until a focused Discord fix lands. The fix should add `requiresTrustedRequesterSender` through the actual registered Discord action adapter, forward trusted `requesterSenderId` as `senderUserId` for privileged guild-admin actions, enforce the relevant Discord permission checks in `runtime.guild.ts` before bot-credentialed mutations, and add regression coverage for message-action dispatch, guild-admin forwarding, direct runtime mutation calls, and disabled-action ordering.
