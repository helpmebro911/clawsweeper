## Summary

Keep open. Current main still does not implement this PR's central auto-reply behavior: typing TTL is not budgeted from model timeout/fallback attempts, timeout failures still fall through to generic failure copy, and the proposed regression test file is absent. The PR also has an unresolved review note about undercounting configured-primary retry behavior, so the useful path is an updated/rebased fix rather than closure.

## Best Possible Solution

Keep #41957 open as a plausible focused bug fix. The best maintainer path is to ask for a current-main rebase that computes typing TTL from the same candidate semantics used by `runWithModelFallback` or a shared helper, covers the configured-primary appended attempt when no fallback override is set, keeps the timeout-specific user guidance if still desired, and adds targeted tests for both TTL bounds and timeout copy.
