## Summary

Current main still appears to have the reported regression. `/hooks/agent` creates an isolated cron-style agent job, isolated jobs still pass `forceNew: true` into session resolution, and `resolveCronSession` only reuses an existing session when `forceNew` is not set. The linked PR #70895 is an active open fix, not shipped on main, so this issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep the issue open and land PR #70895 or an equivalent fix that computes `forceNew` so webhook-sourced isolated runs with stable allowed session keys can reuse fresh sessions, while preserving fresh-session behavior for ordinary scheduled isolated cron jobs. The fix should include a regression test proving hook/webhook isolated runs pass `forceNew: false` and existing isolated cron runs still pass `forceNew: true`.
