## Summary

Keep #14251 open. Current main still lacks system-level macOS LaunchDaemon lifecycle support for gateway stop/uninstall: lifecycle commands only expose JSON output, the Darwin service adapter is still LaunchAgent-only, launchd helpers resolve the per-user LaunchAgents path and gui/<uid> domain, and the system LaunchDaemon scan is diagnostic-only under status --deep.

## Best Possible Solution

Keep the issue open until macOS gateway lifecycle management is launchd-scope-aware. A good fix should model user vs system launchd scope, detect /Library/LaunchDaemons/ai.openclaw.gateway.plist, provide clear sudo guidance or explicit --system stop/uninstall handling, ensure status reports canonical system daemons accurately, and add focused tests/docs for user LaunchAgents and system LaunchDaemons.
