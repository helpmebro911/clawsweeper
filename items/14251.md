## Summary

Keep #14251 open. Current main still treats macOS gateway lifecycle as a per-user LaunchAgent flow; system LaunchDaemons are only discoverable in deep status/doctor scans, not manageable by gateway stop/uninstall, and the prior maintainer/Codex discussion explicitly kept this issue open rather than treating #14315 as a superseding fix.

## Best Possible Solution

Keep #14251 open until macOS gateway lifecycle management becomes launchd-scope-aware: detect /Library/LaunchDaemons/ai.openclaw.gateway.plist, avoid successful stop/uninstall responses while a KeepAlive system daemon remains active, provide exact sudo guidance or explicit --system handling for stop/uninstall, and add focused tests/docs for both user LaunchAgents and system LaunchDaemons.
