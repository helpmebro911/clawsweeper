## Summary

Keep #14251 open. Current main still manages macOS gateway lifecycle as a per-user LaunchAgent in the gui/<uid> domain, while system-level LaunchDaemon handling is limited to best-effort status discovery and docs that say custom LaunchDaemons are not shipped. The existing maintainer/Codex discussion also explicitly kept this issue open and distinguished #14315 as narrower orphan LaunchAgent cleanup, not a superseding fix.

## Best Possible Solution

Keep #14251 open until macOS gateway lifecycle management is launchd-scope-aware. A good fix should model user vs system scope, detect /Library/LaunchDaemons/ai.openclaw.gateway.plist, avoid reporting stop/uninstall success while a KeepAlive system daemon remains active, provide exact sudo guidance or explicit --system handling for stop/uninstall, and add focused tests/docs for both user LaunchAgents and system LaunchDaemons.
