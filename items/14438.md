## Summary

Keep #14438 open. Current main still treats native plugin code changes as restart-only, has no supported `openclaw plugins reload` or cache-clear command, and only exposes reload metadata as config-prefix classification rather than live module/Jiti invalidation plus re-registration.

## Best Possible Solution

Keep #14438 open as core plugin loader and Gateway lifecycle developer UX work. The maintainable path is a supported Gateway RPC plus `openclaw plugins reload <id>` and/or `openclaw plugins clear-cache` that safely invalidates loader/Jiti/module state, disposes plugin-owned registrations and services, refreshes tools/hooks/providers/runtime hooks, and documents container behavior. If maintainers decide native plugin hot reload is intentionally unsupported, update the public plugin and CLI docs with an explicit supported restart/cache workflow.
