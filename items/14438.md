## Summary

Keep #14438 open. Current main still documents native plugin code changes as restart-only, has no `openclaw plugins reload` or `clear-cache` command, and only exposes cache clearing as internal loader/test machinery rather than a supported CLI or Gateway developer workflow.

## Best Possible Solution

Keep #14438 open as core plugin loader and Gateway lifecycle developer UX work. The best path is a supported Gateway RPC plus `openclaw plugins reload <id>` and/or `openclaw plugins clear-cache` that invalidates registry/jiti/module state, safely disposes plugin-owned registrations and services, refreshes tools/hooks/providers/runtime hooks, and documents container behavior. If maintainers decide native plugin code hot reload is intentionally unsupported, document that explicit product decision in the public plugin and CLI docs with a supported restart/cache workflow.
