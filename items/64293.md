## Summary

Keep open. Current `main` still treats `agents.defaults.heartbeat: {}` and per-agent `heartbeat: {}` as configured heartbeats rather than disabled ones, so the reported behavior is still present in code. There is a related patch for `#64293`, but it is not on current `main`.

## Evidence

- **Current scheduler still accepts empty heartbeat configs:** `resolveHeartbeatConfig()` returns `defaults` unchanged for the default agent and returns `{ ...defaults, ...overrides }` for per-agent config, with no empty-object guard. `resolveHeartbeatAgents()` also filters on `entry?.heartbeat`, so `{}` is treated as present.
  - file: [src/infra/heartbeat-runner.ts:187](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/infra/heartbeat-runner.ts#L187)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '187,215p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Current enablement logic treats `{}` as enabled:** `hasExplicitHeartbeatAgents()` and `isHeartbeatEnabledForAgent()` use `Boolean(entry?.heartbeat)`, and `resolveHeartbeatIntervalMs()` falls back to `DEFAULT_HEARTBEAT_EVERY` when no `every` value is set. An empty object therefore still resolves to an active heartbeat lane at the default cadence.
  - file: [src/infra/heartbeat-summary.ts:27](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/infra/heartbeat-summary.ts#L27)
  - command: `nl -ba src/infra/heartbeat-summary.ts | sed -n '1,140p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Docs still document `0m` as the disable mechanism:** User-facing docs say to disable heartbeat by setting `agents.defaults.heartbeat.every` to `"0m"`; they do not document `{}` as a disable signal.
  - file: [docs/gateway/heartbeat.md:51](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/docs/gateway/heartbeat.md#L51)

  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Related fix exists but is not on current main:** Public PR patch `#64540` is explicitly titled `fix: treat empty heartbeat config as disabled (closes #64293)` and adds the missing empty-object checks in `src/infra/heartbeat-summary.ts`. Current `main` does not contain those changes.
  - file: [src/infra/heartbeat-summary.ts](https://github.com/openclaw/openclaw/blob/2521476a938d2910643cec1e2fa362d60b966458/src/infra/heartbeat-summary.ts)
