## Summary

Keep open. Current main still treats an empty heartbeat object as enabled and falls through to the default 30 minute interval, and the provided GitHub context shows open PR #64540 explicitly targets this issue but has not merged.

## Best Possible Solution

Keep this issue open and review PR #64540 or an equivalent maintainer patch. The fix should explicitly define the empty-object heartbeat contract, apply consistently to default and per-agent heartbeat resolution, preserve the documented `every: "0m"` disable behavior, and add focused regression coverage for empty defaults and empty per-agent overrides.
