## Summary

Keep this PR open. Current main still does not implement the PR's central A2A timeout/provenance, webchat-backed main-session metadata cleanup, or outbound mirror continuity changes, but the PR branch is not merge-ready as-is because review-discussed timeout and outbound-route regressions are source-backed and the branch includes unrelated docs-template changes.

## Best Possible Solution

Keep this PR open for revision or replacement. A mergeable path should split out the unrelated docs-template commit, limit any timeout floor to true cross-agent sends with focused zero/short/long timeout tests, add the webchat-backed main-session metadata cleanup in gateway persistence with regression coverage, and preserve outbound session continuity through the existing resolver/plugin contract without overwriting the resolver-selected route key.
