## Summary

Keep #28303 open. Current main still exposes `zh-CN` and `zh-TW` as the Control UI runtime locale contract, lazy import keys, generated locale metadata, saved browser-storage values, selector option values, docs, workflow matrix, and tests. The related docs/glossary items do not implement the requested Control UI runtime rename or saved-locale migration.

## Best Possible Solution

Keep #28303 open for a focused Control UI i18n contract change: rename runtime locale IDs and generated metadata to `zh-Hans`/`zh-Hant`, update lazy imports, selector language keys, docs, workflow, and tests, and add saved-setting migration or aliases from `zh-CN`/`zh-TW` for both global and scoped browser settings. If maintainers decide the old IDs are stable API, document that decision on the issue before closing.

## Risks / Open Questions

- Renaming is compatibility-sensitive because saved values exist both in `openclaw.i18n.locale` and scoped `openclaw.control.settings.v1:*` browser settings.
- Maintainers may intentionally choose to keep region-based runtime IDs as a stable compatibility contract, but that should be an explicit product/API decision rather than conservative cleanup closure.
