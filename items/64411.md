## Summary

Keep open. Current main has a narrow active-memory fallback, but it does not implement the broader MCP/plugin fail-fast behavior requested here. Bundle MCP config only supports connection timeouts, bundle MCP tool calls still await `client.callTool(...)` without a per-call timeout or circuit breaker, and bundle MCP tools are still registered as non-optional in the normal coding/messaging profiles.

## Evidence
