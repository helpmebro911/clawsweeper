## Summary

Keep #14526 open. Current main has manual backup creation/verification, update doctor checks, plugin sync, gateway restart health/version/plugin diagnostics, auto-update timing policy, and manual rollback docs, but it still lacks the proposed opt-in safe-update workflow: automatic pre-update state backup, retention controls, channel or real-agent smoke verification, and automatic rollback/restore. Bortlesboat's linked openclaw-auto-update-kit is useful operator prior art/workaround, not a merged core implementation or superseding tracker.

## Best Possible Solution

Keep #14526 open until maintainers either implement an opt-in safe-update mode in the core `openclaw update` and `update.auto` paths or explicitly decide the workflow should live outside core. A complete core path would build on existing backup and health primitives, add pre-update backup creation, retention controls, gateway/status/channel/real-agent smoke checks, and controlled rollback/version pin plus state restore, using openclaw-auto-update-kit as useful prior art.
