## Summary

Keep #14526 open. Current main still has adjacent pieces: manual backup create/verify, update doctor checks, plugin post-update sync, gateway restart health/version/plugin diagnostics, timing-based auto-update policy, and manual rollback docs. It still does not implement the proposed opt-in safe-update workflow with automatic pre-update state backup, retention controls, channel or real-agent smoke verification, or automatic rollback/restore. Bortlesboat's linked openclaw-auto-update-kit remains useful operator prior art/workaround, not a merged core implementation or superseding tracker, and the latest provided maintainer/Codex review also kept this open.

## Best Possible Solution

Keep #14526 open until maintainers either implement an opt-in safe-update mode in the core `openclaw update` and `update.auto` paths or explicitly decide this operator workflow should remain external. A complete core path would likely build on the existing backup and health primitives, adding pre-update backup creation, retention controls, gateway/status/channel/real-agent smoke checks, and controlled rollback/version pin plus state restore, while treating openclaw-auto-update-kit as useful prior art.
