## Summary

Keep #14526 open. Current main still has manual backup creation/verification, doctor/update checks, gateway restart health diagnostics, and background auto-update timing, but it does not implement the requested opt-in safe-update workflow with automatic pre-update backup, retention policy, channel/agent smoke verification, or automatic rollback/restore. The linked openclaw-auto-update-kit is useful operator evidence/workaround, not a core implementation or superseding tracker.

## Best Possible Solution

Keep #14526 open until maintainers either implement an opt-in safe-update mode in the core update/auto-update flow or explicitly decide to leave this workflow external. A complete core solution would likely add update CLI/config flags for pre-update backup, retention limits, post-update gateway/status/channel/real-agent smoke checks, and controlled rollback/restore behavior built on the existing backup primitives.
