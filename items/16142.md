## Summary

Keep #16142 open. Current main still sends main-session cron system-event reminders through the heartbeat model path: cron enqueues the reminder as a system event, heartbeat turns it into a relay prompt, and runHeartbeatOnce calls getReplyFromConfig. Existing tests assert that behavior, docs describe cost mitigations rather than a deterministic bypass, and the linked #15403 PR was unrelated shell-environment work that closed unmerged.

## Best Possible Solution

Keep #16142 open until main-session cron systemEvent reminders have a deterministic delivery path for simple reminder text that avoids getReplyFromConfig, while preserving model-mediated handling for events that need reasoning. The likely implementation belongs around executeMainSessionCronJob, resolveHeartbeatRunPrompt, or a small delivery helper, with regression tests proving simple reminders avoid the model and docs clarifying which cost controls are mitigations versus true bypasses.
