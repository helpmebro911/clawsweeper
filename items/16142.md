## Summary

Keep #16142 open. Current main still routes main-session cron/system-event reminders through the heartbeat model path: cron enqueues the reminder as a system event, heartbeat builds a cron relay prompt, and runHeartbeatOnce calls getReplyFromConfig. The prior maintainer/Codex comment already kept it open, and the only later main commit touches WhatsApp reconnect behavior, not this path.

## Best Possible Solution

Keep #16142 open until main-session cron systemEvent reminders have a deterministic delivery path for simple reminder text that avoids getReplyFromConfig, while preserving the model path for events that need reasoning. The likely implementation belongs around executeMainSessionCronJob, resolveHeartbeatRunPrompt, or a small delivery helper, with tests proving simple reminders avoid the model and docs clarifying true bypasses versus cost mitigations.
