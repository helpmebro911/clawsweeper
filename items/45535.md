## Summary

Keep open. Current main partially covers the linked Bailian/OpenAI-compatible usage issue for prompt_tokens-only responses, but it does not implement this PR's stricter zero-before-positive alias normalization, and the CLI usage parser still lacks prompt_tokens/completion_tokens aliases.

## Best Possible Solution

Keep the PR open or ask for a refreshed patch against current main. The best fix is a small, tested normalization helper that prefers positive token aliases over earlier zero aliases in src/agents/usage.ts, adds prompt_tokens/completion_tokens support to the current CLI usage parser, and preserves the existing guard that prevents all-zero usage events from replacing prior nonzero stream usage.
