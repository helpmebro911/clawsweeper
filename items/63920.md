## Summary

Keep this issue open. Current main still routes Slack block actions and modal lifecycle events only; there is no `ctx.app.shortcut` registration, no shortcut payload summary, no shortcut system-event enqueue path, and no docs/tests for Slack global or message-action shortcuts. The current plugin-facing Slack interactive API is button/select scoped, so this is not ClawHub-ready without either bundled Slack work or a new raw shortcut/plugin API.

## Best Possible Solution

Implement this in the bundled Slack plugin, or first add a typed plugin-facing Slack shortcut/raw-interaction API. The concrete bundled path is a new Slack shortcut event module that registers `ctx.app.shortcut` for global and message-action shortcuts, acknowledges and mismatch-checks payloads, applies existing Slack sender/session authorization, enqueues stable sanitized system events with context keys, and updates Slack interaction tests plus the public Slack docs.
