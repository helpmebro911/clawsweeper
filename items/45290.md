## Summary

Keep PR #45290 open. Current main has some overlapping delivery-context and Telegram polling work, but it still does not implement several central behaviors from the PR: the `input_length` overflow matcher, leaked wrapper/tool-call artifact stripping in plain-text outbound replies, queue payload shrinking after best-effort partial delivery failure, or the Argus/lane-priority pieces. The narrow fixes that still appear valid should be separated and source-backed with focused tests: `input_length` overflow detection, plain-text leakage sanitization, and best-effort partial-failure queue payload shrinking. The Argus/lane-priority and Telegram changes need redesign against current main and resolution of the outstanding review comments before they should be considered mergeable.
