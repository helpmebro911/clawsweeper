## Summary

Keep this issue open. Current main still has no configurable per-call session scope for the bundled voice-call plugin: classic auto-response keys by caller phone number, inbound call records do not persist a per-call session key, realtime consult falls back to caller/callee phone number, and the strict config/manifest surface lacks `sessionScope`. The maintainer comment already identified the same gap, and related #59245 and #71262 do not supersede this narrower isolation request.

## Best Possible Solution

Implement this in the bundled `voice-call` plugin: add a strict `sessionScope` enum with `per-phone` as the compatibility default and `per-call` as an opt-in, expose it in `VoiceCallConfigSchema`, `openclaw.plugin.json`, UI metadata, and docs, resolve and persist `call.sessionKey` at call creation, and reuse that key in both classic `generateVoiceResponse` and realtime consult. Add focused tests proving repeated calls from the same number isolate history with `per-call` and preserve existing shared history when unset.

## Risks / Open Questions

- Changing the default away from phone-scoped sessions would break existing personal-assistant setups that rely on caller memory.
- Classic response generation and realtime consult can diverge unless the implementation resolves one session key when the call record is created and reuses it everywhere.
