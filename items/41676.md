## Summary

Keep this issue open. Current main added bounded rotation for the structured Gateway file logger, but the reported path is the macOS launchd supervisor stderr file (`gateway.err.log`), and current launchd service setup still writes stdout/stderr directly to fixed files without a size cap or rotation policy. The repeated heartbeat string is not present in current source, so the noisy producer may have moved or come from a dependency/plugin, but the defensive supervisor-log growth gap remains actionable.

## Best Possible Solution

Add a bounded retention policy for macOS launchd supervisor logs (`gateway.log` and `gateway.err.log`) or route launchd stdout/stderr through an OpenClaw-managed rotating sink. Keep the existing structured logger rotation, add targeted tests around LaunchAgent log path handling/rotation or startup truncation, document how supervisor logs are bounded, and separately rate-limit or summarize the heartbeat-timeout producer if it can be traced to a current dependency/plugin.
