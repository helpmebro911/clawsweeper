## Summary

Keep open. Current `main` has basic `memory status` diagnostics for missing extra paths, but the issue’s requested cleanup is not implemented end-to-end: builtin memory indexing still recursively ingests every Markdown file under `memorySearch.extraPaths`, there is no `excludePaths`/glob exclusion field in the runtime schema, and the current memory docs describe `extraPaths` generically without the reported path-drift / benchmark-contamination guidance.

## Evidence

- **Builtin indexing still ingests all Markdown under extraPaths:** `listMemoryFiles()` normalizes `extraPaths`, then recursively walks each directory and adds every allowed file; there is no exclusion/filter hook for Markdown files in the builtin path.
  - file: [src/memory-host-sdk/host/internal.ts:167](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/memory-host-sdk/host/internal.ts#L167)
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Schema exposes extraPaths but no excludePaths equivalent:** `MemorySearchSchema` includes `extraPaths`, QMD extra collections, multimodal, and experimental session memory, but no exclusion path/glob setting for builtin memory indexing.
  - file: [src/config/zod-schema.agent-runtime.ts:624](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/config/zod-schema.agent-runtime.ts#L624)
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Current docs only document extraPaths broadly:** The memory config reference says `extraPaths` are additional directories/files and that directories are scanned recursively for `.md` files, but it does not document benchmark/queryset contamination pitfalls or path-hygiene guidance from the issue.
  - file: [docs/reference/memory-config.md:274](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/docs/reference/memory-config.md#L274)
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)
- **Current diagnostics cover missing paths, not broader intent drift:** `memory status` reports `additional memory path missing (...)` and `additional memory path not accessible (...)`, which helps with broken paths, but there is no stronger runtime warning about paths being misaligned with the active workspace intent beyond existence/access checks.
  - file: [extensions/memory-core/src/cli.runtime.ts:537](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/extensions/memory-core/src/cli.runtime.ts#L537)
  - sha: [7d9172d5e093](https://github.com/openclaw/openclaw/commit/7d9172d5e093435d37ed3e00e476a4eaf84a252b)

- QMD has a `pattern` field on extra collections, but that is backend-specific and does not resolve the builtin `extraPaths` contamination path described in the issue.
