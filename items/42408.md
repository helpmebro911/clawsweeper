## Summary

Keep this issue open. Current main still only partially covers the report: it reports missing or inaccessible extraPaths in memory status, but builtin local memory indexing still has no excludePaths/glob exclusion surface and the public memory docs still lack the requested path-hygiene and benchmark-contamination guidance.

## Best Possible Solution

Keep open and implement this in the bundled memory-core plugin plus shared memory config/docs. A complete fix should add a documented memorySearch exclusion mechanism, thread it through config schema/types, resolveMemorySearchConfig, listMemoryFiles, watcher setup, read access, and reindex scope hashing, then add regression tests proving benchmark/queryset Markdown under extraPaths is excluded. Public docs should add path-hygiene guidance for absolute/workspace-relative extraPaths, status --deep verification, reindex-after-path-change, broad-path noise, and benchmark/evaluation contamination.
