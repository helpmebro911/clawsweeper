## Summary

Current main still appears to expose filesystem tools under restrictive profiles when a `tools.fs` block is configured, matching the reported least-privilege regression. The issue should remain open, especially with #43061 still open as an implementation candidate.

## Best Possible Solution

Keep this issue open. The best fix is to change `src/agents/pi-tools.policy.ts` so `tools.fs` and `tools.exec` configure already-granted tools but do not implicitly re-grant filesystem/runtime tools under restrictive profiles unless explicitly requested through `alsoAllow`, with deny precedence preserved. Add regression coverage for both `minimal` and `messaging` with configured `tools.fs`/`tools.exec`, and review or split #43061 as the active implementation candidate.
