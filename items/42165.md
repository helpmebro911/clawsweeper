## Summary

Keep this issue open. Current main still implicitly adds filesystem tools when `tools.fs` is configured, without checking the effective `tools.profile`; that can re-expand `read`, `write`, and `edit` into restrictive profiles before the normal policy pipeline filters tools. The docs and catalog still define `minimal` as `session_status` only and keep filesystem tools out of both `minimal` and `messaging`.

## Best Possible Solution

Keep this issue open until current main changes the implicit tool-section expansion so `tools.fs` and `tools.exec` configure already-granted tools but do not re-grant filesystem/runtime tools under restrictive profiles unless the user explicitly requests them with `alsoAllow`. The likely implementation belongs in `src/agents/pi-tools.policy.ts`, with consistency checks in `src/agents/tool-fs-policy.ts`, plus regression tests for `minimal` and `messaging` with configured `tools.fs`/`tools.exec`, preserving explicit `alsoAllow` and deny precedence.
