## Summary

Keep open. Current `main` does not include the proposed `before_tools_resolve` hook or its integration in the agent tool-assembly paths, so this PR is not implemented on `main` or the latest release. There is no strong conservative-close reason here.

## Evidence

- **Hook name is absent from the plugin hook contract:** `src/plugins/hook-types.ts` lists `before_tool_call`/`after_tool_call` and other hooks, but no `before_tools_resolve` entry in either `PluginHookName` or `PLUGIN_HOOK_NAMES`.
  - file: [src/plugins/hook-types.ts:57](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/plugins/hook-types.ts#L57)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Hook runner has no list-level tool hook:** `src/plugins/hooks.ts` exposes `runBeforeToolCall` and `runAfterToolCall`; there is no `runBeforeToolsResolve` implementation on current `main`.
  - file: [src/plugins/hooks.ts:777](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/plugins/hooks.ts#L777)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Main agent run path does not apply a pre-resolution tool-filter hook:** `runEmbeddedAttempt` builds tools with `createOpenClawCodingTools`, then normalizes schemas, adds bundled tools, and computes `allowedToolNames`; there is no `applyBeforeToolsResolveHook` call in this path.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:512](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/pi-embedded-runner/run/attempt.ts#L512)

  - file: [src/agents/pi-embedded-runner/compact.ts:505](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/pi-embedded-runner/compact.ts#L505)
