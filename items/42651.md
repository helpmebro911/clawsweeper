## Summary

Keep open. Current main has substantial memory infrastructure, but the requested MVP operator surface is still not implemented as first-class `remember`, `recall`, `review`, `edit`, and `forget` commands or equivalent skill/CLI flows. The discussion also shows useful external prior art from Engram Core, but that does not replace the unresolved bundled `memory-core` product/API decision.

## Best Possible Solution

Keep this open and narrow the v1 operator contract. A maintainer should decide whether the MVP lives in `extensions/memory-core` CLI/commands, a bundled skill surface, or a small documented alias layer: `recall` can likely wrap `memory search`, `review` can build on promotion/dreaming candidates, `remember` needs a provenance-preserving write path, and `edit`/`forget` should be coordinated with #42650. Existing daily-note, session-transcript, grounded backfill, memory-wiki, and LanceDB/plugin work should be reused rather than creating a parallel memory store.
