## Summary

Keep open. Current main has substantial memory infrastructure and some optional/plugin alternatives, but the requested MVP operator surface is still not implemented in the default memory path: there are no first-class remember/recall/review/edit/forget CLI or bundled skill workflows, memory-core only exposes search/get tools, and recall/search output still lacks a structured match-explanation signal. The Engram Core comment is useful evidence of demand and a possible external path, but it does not close the default OpenClaw memory-core contract requested here.

## Best Possible Solution

Keep this open and narrow the v1 operator contract. A maintainer should decide whether the MVP is implemented as memory-core subcommands, a bundled skill surface, plugin tools, or documented aliases. The path should map remember to a provenance-preserving write flow, recall to memory search with provenance and match explanations, and review/edit/forget to the same short-term promotion, dreaming, memory-wiki, and #42650 state model rather than creating a parallel memory store.
