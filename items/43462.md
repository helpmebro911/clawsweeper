## Summary

Keep this PR open. Current main still has case-sensitive memory-directory prefix checks in the moved memory host SDK and temporal-decay implementation, and the PR’s latest revision addresses the earlier dated-path regression. The work is still useful, but it needs to be rebased/ported to current files.

## Best Possible Solution

Port the PR’s directory-prefix behavior to current main by lowercasing only the normalized `memory/` prefix checks in `src/memory-host-sdk/host/internal.ts`, `packages/memory-host-sdk/src/host/internal.ts`, and `extensions/memory-core/src/memory/temporal-decay.ts`; use the same lowercased value for the dated-memory regex; add focused regression tests for `MEMORY/notes.md`, `Memory/daily.md`, and `MEMORY/YYYY-MM-DD.md`; and preserve canonical root `MEMORY.md` behavior rather than restoring legacy lowercase root indexing.
