## Summary

Keep this issue open. Current main still does not carry `accountId` through the direct `/hooks/agent` path: the hook payload contract and normalizer omit it, replay/dispatch scope omits it, and `dispatchAgentHook()` builds `CronJob.delivery` with only `mode`, `channel`, and `to`. The cron delivery stack itself already supports `delivery.accountId`, so the remaining bug is the hook-to-cron handoff. The same author also has open PR #43951 with closing syntax for this issue, so the issue should stay open until that PR is reviewed and merged or replaced.

## Best Possible Solution

Keep this issue open until PR #43951 is reviewed and merged or replaced by a focused current-main patch. The best fix is to add `accountId` to the `/hooks/agent` and mapped-agent hook contract, normalize and trim it, include it in replay/idempotency scope, propagate it into `CronDelivery.accountId` during hook dispatch, update the hook docs, and add regression tests proving `job.delivery.accountId` reaches isolated cron delivery.
