## Summary

#41294 should stay open. Current main still implements automatic model fallback over configured candidates, the agent model schema has no `fallbackMode`/confirmation option, and the specific interactive yes/no/pick behavior is not clearly superseded by another open item. This is core fallback runtime/config behavior rather than a clean ClawHub-only plugin close because current plugin surfaces do not appear to provide a way to intercept and confirm model fallback selection across channels.
