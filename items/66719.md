## Summary

Keep open. Current `main` still exposes ACP-only and runtime-specific `sessions_spawn` parameters in one static tool schema, `createOpenClawTools()` does not pass any schema-simplification option, and the provider-compat path only strips unsupported JSON Schema keywords rather than tool-specific properties.

## Evidence

- **Static sessions_spawn schema still includes the reported fields:** Current `main` defines one unconditional `SessionsSpawnToolSchema` with `resumeSessionId`, `thread`, `mode`, `sandbox`, `streamTo`, `lightContext`, `attachments`, and `attachAs` all present in the exported tool parameters.

  - sha: [cc0f3e0e4036](https://github.com/openclaw/openclaw/commit/cc0f3e0e40364023cc3a39227a411898ee68ce3b)
- **No schema-simplification hook is wired when tools are built:** `createOpenClawTools()` calls `createSessionsSpawnTool()` with session/runtime context only; there is no option resembling `simplifiedSchemaForGateway` or runtime-based schema pruning in the current call site.
  - file: [src/agents/openclaw-tools.ts:295](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/agents/openclaw-tools.ts#L295)
