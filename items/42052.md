## Summary

Keep open. Current `main` still treats a non-existent `--output` path without a trailing slash as an explicit file path, so the PR is addressing live behavior rather than an already-fixed issue. The follow-up PR discussion also resolved the initial review concerns about `.tar` and custom suffix handling, so the work remains useful even though the implementation now lives in `src/infra/backup-create.ts` instead of the old `src/commands/backup.ts` path.

## Evidence

- **Current main still takes missing output paths as files:** `resolveOutputPath()` catches `fs.stat()` failure and falls through to `return resolved`, with the inline comment `Treat as a file path when the target does not exist yet.` That is the exact behavior the PR is trying to change.
  - file: [src/infra/backup-create.ts:114](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/infra/backup-create.ts#L114)
  - command: `nl -ba src/infra/backup-create.ts | sed -n '86,119p'`
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Archive names are supposed to end in .tar.gz and are written gzip-compressed:** `buildBackupArchiveBasename()` always generates `<timestamp>-openclaw-backup.tar.gz`, and archive creation uses `tar.c(..., gzip: true)`. If a missing directory-like path is treated as a filename, OpenClaw will emit a gzip archive at that bare path without the normal `.tar.gz` suffix.
  - file: [src/commands/backup-shared.ts:65](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/commands/backup-shared.ts#L65)

  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **CLI/docs still present --output as either archive path or destination directory:** The CLI option text says `Archive path or destination directory`, and the docs example `openclaw backup create --output ~/Backups` demonstrates directory-style output. That makes the missing-directory case a real user-facing ambiguity, not a dead codepath.
