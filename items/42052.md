## Summary

Current main still has the reported backup output-path behavior, and this PR remains a plausible fix candidate, but it is stale against the current implementation layout and needs a rebase plus tests/docs before landing.

## Best Possible Solution

Keep this PR open for maintainer review. Rebase the useful output-path behavior onto `src/infra/backup-create.ts`, settle and document the `--output` contract in CLI help and the public backup docs, and add focused regression tests for non-existent extensionless directories, explicit archive filenames, `.tar`, custom suffixes such as `.bak`, and non-ENOENT `fs.stat` errors.

## Evidence
