## Summary

Keep this PR open. Current main still treats a missing, non-slash backup output path as an explicit file path, so the PR addresses a real remaining behavior question, but its patch targets the old backup command layout and needs a rebase, tests, and maintainer product judgment before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The useful path is to rebase the idea onto `src/infra/backup-create.ts`, add focused tests in `src/commands/backup.test.ts`, update `docs/cli/backup.md` and CLI help if maintainers accept the inferred-directory contract, then review and land or close the linked bug from the merged implementation.
