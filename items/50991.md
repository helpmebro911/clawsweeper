## Summary

Keep open. Current main has generic compaction lifecycle hook events, but the bundled `session-memory` hook still only registers and executes for `/new` and `/reset`. The requested `/compact` preservation behavior is therefore not implemented, and there is no canonical narrower open item in the provided related context that clearly supersedes this issue.

## Evidence

- **Bundled hook metadata excludes compact:** The `session-memory` HOOK metadata still advertises and registers only `command:new` and `command:reset`, with no `command:compact` or `session:compact:*` event.
  - file: [src/hooks/bundled/session-memory/HOOK.md:10](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/hooks/bundled/session-memory/HOOK.md#L10)

- **Handler ignores compact events:** The handler explicitly returns unless the event is a command whose action is `new` or `reset`, so registering compaction events alone would not currently save a memory file.
  - file: [src/hooks/bundled/session-memory/handler.ts:53](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/src/hooks/bundled/session-memory/handler.ts#L53)
