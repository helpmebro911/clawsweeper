## Summary

Keep #46356 open. Current main has partial proxy-related work, but it does not implement the PR's central webchat bidirectional mirror path, and it does not expose the PR's `channels.whatsapp.proxy` config. The supplied review discussion also contains unresolved runtime correctness concerns in the PR diff, so this is neither implemented on main nor safe to close as obsolete.

## Best Possible Solution

Keep the PR open, but ask for a refreshed, narrower branch. The best path is to split the remaining useful work into focused changes: a dedicated webchat mirror target/dedupe path that preserves current no-inherit safety, a separate Discord DM/proxy follow-up only for gaps not already solved by `proxy-request-client`, and a separate WhatsApp proxy-config proposal if maintainers still want config-level proxy support beyond ambient env proxies. The author should also resolve the supplied P1/P2 review comments before this is considered mergeable.
