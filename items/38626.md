## Summary

Keep #38626 open. Current main has useful background-task records, subagent listing, completion delivery, cancellation, and steer controls, but it still does not provide the requested durable per-subagent lifecycle timeline, persisted tool/warning/error events, artifact-pointer completion summaries, first-failure/recommended-action diagnostics, or idempotent audit-logged cancel/steer/resume supervision surface. Related #62478 is complementary background-spawn work, not a canonical replacement.

## Best Possible Solution

Keep the issue open unless a maintainer intentionally narrows it. The best implementation path is to add a durable per-run event ledger for subagent/task spawn, queued/running, tool start/end, warning/error, completion, and supervision actions; persist it in task/subagent storage; expose it through subagents list/control output, openclaw tasks show/list JSON, gateway/control UI surfaces, and docs; attach artifact pointers and terminal diagnostics; and make cancel, steer, and resume idempotent with auditable action records. Treat #62478 as complementary background-spawn work, not the canonical replacement for this observability scope.
