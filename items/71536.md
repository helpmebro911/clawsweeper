## Summary

Keep open. This is a recent non-maintainer regression report for the bundled Telegram native `/restart` command, and current `main` still has two authorization stages that can emit the same “You are not authorized to use this command.” response. I found no source, test, docs, release, or related-item evidence that the reported Telegram restart authorization failure has been fixed or superseded since the prior Codex review.

## Best Possible Solution

Keep this issue open. The maintainer path is to request or reproduce the exact Telegram config and add a focused regression test in `extensions/telegram/src` that drives native `/restart` through Telegram authorization into `handleRestartCommand` with `channels.telegram.allowFrom`, `channels.telegram.groupAllowFrom`, `commands.allowFrom.telegram`, and `commands.ownerAllowFrom`. If reproduced, fix the Telegram native preauth/owner handoff or fresh config resolution so a correctly owner-allowlisted Telegram sender can restart without weakening owner-only command gating.
