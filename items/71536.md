## Summary

Keep #71536 open. Current main still has two separate gates that can produce the exact Telegram message reported: native command sender authorization and the owner-only /restart gate. The issue is very recent, labeled regression, and the provided related reports cover adjacent restart/polling/menu problems rather than a canonical duplicate for Telegram /restart authorization.

## Best Possible Solution

Keep the issue open. Maintainers should request a redacted Telegram config and logs around the rejected /restart, then add a focused Telegram native /restart regression test covering commands.native, channel allowFrom/groupAllowFrom, commands.allowFrom, commands.ownerAllowFrom, enforceOwnerForCommands, and DM/group/topic variants. If current behavior is intended, improve the Telegram/native-command diagnostic so sender authorization and owner-only restart rejection are distinguishable.
