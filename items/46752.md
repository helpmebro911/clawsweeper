## Summary

Keep PR #46752 open. Current main still has many of the unbounded tool-schema parameters the PR is trying to harden, so the work is not implemented or obsolete. The PR is not merge-ready as-is because review comments identify several schema caps that would reject currently valid runtime/doc-supported calls; it needs a rebase and runtime-aligned bounds rather than closure.

## Best Possible Solution

Keep the PR open but ask for a rebase and a narrower, contract-backed revision: add schema bounds only where they match current runtime/provider/docs limits, preserve valid long-running subagent and node-media workflows, keep browser snapshot limits aligned with the 2000-node backend cap, preserve Discord GuildMedia support, and add focused tests that fail if schema bounds drift from runtime clamps.
