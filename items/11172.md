## Summary

Current main still does not implement the requested subagent exec identity environment variables. The exec tool has access to resolved agent/session identity, but child process env construction only merges inherited/request/sandbox env and then adds existing runtime markers such as OPENCLAW_SHELL. The related implementation PR was closed unmerged, and the open agentId schema PR does not cover env injection.

## Best Possible Solution

Keep #11172 open until OpenClaw implements or explicitly rejects a documented core exec identity contract for subagent sessions. A likely implementation belongs in the shared exec environment construction: inject OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY from the resolved agent/session context, decide whether to support legacy CLAWDBOT_AGENT_ID, define whether tool-call env may override these values, and add focused gateway/node/sandbox tests plus docs at docs.openclaw.ai/help/environment and docs.openclaw.ai/tools/exec.

- The fix spans gateway, node, and sandbox exec paths; a partial implementation could leave inconsistent identity propagation.
