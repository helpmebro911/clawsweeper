## Summary

Keep #11172 open. Current main still carries agent/session identity as session metadata, but exec child-process environments are only built from inherited/request/sandbox env plus the OPENCLAW_SHELL runtime marker. The linked implementation PR #11497 was closed unmerged, and related #53262 only concerns requiring agentId in sessions_spawn schema, not exposing identity to spawned commands.

## Best Possible Solution

Keep this issue open until OpenClaw ships or explicitly rejects a documented core exec identity contract. The likely fix belongs in shared exec environment construction: derive OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY from the resolved session context, decide whether CLAWDBOT_AGENT_ID compatibility is supported, define override/spoofing behavior, and add focused tests/docs across exec hosts and process modes.
