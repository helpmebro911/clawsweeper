## Summary

Keep #11172 open. Current main still has subagent/session identity available as execution metadata, but exec child-process environments are not automatically populated with OPENCLAW_AGENT_ID, CLAWDBOT_AGENT_ID, or OPENCLAW_SESSION_KEY. The related implementation PR #11497 was closed unmerged, and the current docs only document OPENCLAW_SHELL runtime markers.

## Best Possible Solution

Keep this issue open until OpenClaw either ships or explicitly rejects a documented core exec identity contract. The likely implementation belongs in shared exec environment construction: derive OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY from the resolved session context, decide whether CLAWDBOT_AGENT_ID compatibility is supported, define whether caller env can override identity, and add focused tests/docs for all exec hosts and process modes.
