## Summary

Current main still has the remaining live deadlock shape. Persisted transcript replay repair can synthesize missing tool results after reload, but `runReplyAgent` still drains `pendingToolTasks` with an unbounded `Promise.allSettled`, and that happens before reply-operation cleanup releases the active session run.

## Best Possible Solution

Keep this issue open and resolve it by reviewing, landing, or superseding PR #64733 with an equivalent main-branch fix: add a bounded timeout or cancellation path around the `pendingToolTasks` drain, ensure reply-operation cleanup runs, and add focused regression coverage for a never-settling tool-result delivery task.
