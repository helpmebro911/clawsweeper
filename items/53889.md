## Summary

Keep this issue open. Current main has useful transcript-repair and synthetic tool-result guard coverage, but the active reply pipeline still has an unbounded `Promise.allSettled(pendingToolTasks)` drain. A never-settling tool-result delivery task can keep the run from reaching `replyOperation.complete()`, matching the reported in-memory session lane deadlock. The related open PR #64733 is an implementation candidate, not evidence that the issue is fixed.

## Best Possible Solution

Keep this issue open and resolve it by reviewing, landing, or superseding PR #64733 with an equivalent main-branch fix: bound the `pendingToolTasks` drain in `src/auto-reply/reply/agent-runner.ts`, log or surface the timeout, ensure `replyOperation.complete()` is reached so the session lane can recover, and add a focused regression test that simulates a never-settling tool-result delivery task.
