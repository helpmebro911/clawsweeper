## Summary

Keep this issue open. Current main still has the live deadlock shape: streamed tool-result delivery promises are collected in `pendingToolTasks`, and `runReplyAgent` waits on them with an unbounded `Promise.allSettled()` before reply-operation cleanup can clear the active session run. Existing transcript repair covers replay/persisted transcript hygiene, but it does not bound the active live drain. The related PR #64733 remains open and is not a shipped fix.

## Best Possible Solution

Keep this issue open and resolve it by reviewing, landing, or superseding PR #64733 with an equivalent main-branch fix: add a bounded timeout or cancellation path around the `pendingToolTasks` drain, log or surface the timeout, ensure reply-operation cleanup runs so the session lane can recover, and add focused regression coverage for a never-settling tool-result delivery task.
