## Summary

Keep #38597 open. Current main still uses whitespace-only Discord long-line splitting and falls back to a hard limit split when CJK text has no whitespace. The active fix path is PR #71384, with prior analysis from #39399, but that work is not merged into main yet.

## Best Possible Solution

Keep #38597 open and use #71384, or a successor PR, as the active fix path. The implementation belongs in extensions/discord/src/chunk.ts: preserve existing whitespace priority, add a CJK terminal/closing punctuation fallback, add a safe CJK character-boundary fallback with a minimum-progress guard and surrogate-pair safety, and land focused Discord chunk tests for Chinese, Japanese, Korean, mixed-script, punctuation-boundary, long-token, preserveWhitespace, newline-mode, and supplementary-plane CJK cases.
