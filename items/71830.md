## Summary

Keep PR #71830 open. Current main still lacks the WhatsApp-specific XML tool-call sanitization and per-channel error visibility controls targeted by the PR, while the PR discussion and linked #71715 describe a concrete production regression and focused fix rather than obsolete work.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to land PR #71830 or an equivalent focused patch that applies WhatsApp outbound XML sanitization across sendText/sendPayload/auto-reply paths, normalizes remembered echo text, adds documented `channels.whatsapp.exposeErrorText` with account inheritance, and routes WhatsApp text-only error payloads through a suppression-capable path.
