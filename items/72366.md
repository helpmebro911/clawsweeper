## Summary

Current main implements the Bonjour/CIAO crash-loop fix reported against v2026.4.24. The discussion’s Linux, WSL, VM, and ECS confirmations match the affected path: Bonjour reached probing/announcement cancellation and the host treated it as an unhandled rejection. Main now wires scoped Bonjour process handlers into the advertiser, classifies CIAO PROBING/ANNOUNCEMENT cancellations as handled, has the global unhandled-rejection path consult those handlers before exiting, and disables or restarts the advertiser instead of crash-looping the gateway. Related ClawSweeper reviews for #72103, #72152, and #72307 reached the same implemented-on-main conclusion for this same Bonjour/CIAO crash family.

## Best Possible Solution

Close this as implemented on main and keep the current Bonjour-scoped CIAO handler, host handler registry, restart/disable behavior, and regression tests. If users still see a Bonjour crash after a release containing this mainline fix, that should be tracked as a new issue with fresh gateway logs and environment details because it would be a distinct remaining mDNS failure.

## Evidence

- **Reporter evidence:** The issue and follow-up comments all describe v2026.4.24 gateway restart loops caused by `CIAO PROBING CANCELLED` or `CIAO ANNOUNCEMENT CANCELLED`, with disabling the bundled Bonjour plugin or rolling back stopping the crash.
