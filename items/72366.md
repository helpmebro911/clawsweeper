## Summary

Current main already fixes the reported v2026.4.24 Bonjour/CIAO startup crash-loop. The Bonjour plugin now wires scoped process-level rejection and exception handlers into the advertiser, classifies CIAO PROBING/ANNOUNCEMENT cancellations as handled, and the host unhandled-rejection path returns before exiting when a registered handler claims the error. The advertiser also keeps those handlers registered across watchdog restarts and disables Bonjour after repeated stuck probing/announcing restarts instead of crash-looping the gateway. Related ClawSweeper reports #72152, #72103, and #72307 describe the same Bonjour/CIAO crash family and reached the same implemented-on-main conclusion.

## Best Possible Solution

Close as implemented on current main. Keep the shipped Bonjour hardening, and direct users still on v2026.4.24 to update to a build containing main or use the documented temporary workaround of disabling Bonjour when local multicast is unavailable.
