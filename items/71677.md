## Summary

Keep open. Current main still does not implement Feishu instant card callback updates: the registered card.action.trigger handler returns no value, handleFeishuCardAction is Promise<void> and awaits the synthetic command path, the ocf1 envelope/button helpers have no disable-state contract, and existing tests cover dedupe/reply routing rather than a returned WebSocket callback payload.

## Best Possible Solution

Implement this in the bundled Feishu plugin: extend the ocf1 interaction contract with an opt-in disable/transitional state, make card.action.trigger return an immediate Feishu callback response while dispatching the synthetic command asynchronously, preserve duplicate-token protection, and add focused tests for the returned WS callback payload plus existing quick-action and approval-card behavior.
