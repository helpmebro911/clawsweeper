## Summary

Keep open. Current main still handles Feishu card.action.trigger as a Promise<void> path with no returned callback payload, no ocf1 disable-state field, and tests only cover dedupe/reply routing rather than instant card update responses.

## Best Possible Solution

Keep the issue open. Implement this in the bundled Feishu plugin by adding an opt-in disable-state field or equivalent typed metadata to the ocf1 interaction contract, returning an immediate Feishu callback response with a transitional disabled card/toast, dispatching the synthetic command asynchronously, preserving current dedupe/retry behavior, and adding focused tests for WS/webhook callback return payloads plus existing quick-action and approval flows.
