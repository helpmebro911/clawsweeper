## Summary

Keep #70465 open. Current main has some newer MCP retirement paths, but the PR's specific gateway-routed nested `sessions_send` cleanup gap is not proven obsolete: the gateway agent handler still only forwards `cleanupBundleMcpOnRunEnd` when the request explicitly sets it to true, and the primary `sessions_send` agent call still sends a nested lane without that cleanup flag.

## Best Possible Solution

Keep the PR open for maintainer review or update it into the canonical fix. The likely path is either to default cleanup for nested gateway agent requests in `src/gateway/server-methods/agent.ts` with regression tests, or to wire a broader lifecycle cleanup that explicitly covers the primary `sessions_send` `startAgentRun` path. The updated PR should include tests for direct `sessions_send` wait/fire-and-forget behavior, no-reply/error completion, and the intended session-mode opt-out or trusted-caller policy.
