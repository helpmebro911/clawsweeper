## Summary

Keep this PR open. Current main still plans channel-account config edits as full channel restarts, while the lower channel lifecycle already supports account-scoped start/stop. The PR is a plausible focused implementation for the linked open issue #43935, but it needs a rebase/adaptation to current reload-plan fields and channel reload deferral behavior before it can land.

## Best Possible Solution

Rebase and adapt the PR onto current main, then land the account-scoped reload implementation if it preserves current reload deferral and newer plan fields. The implementation should add an account-scoped restart representation to `GatewayReloadPlan`, parse strictly `channels.<channel>.accounts.<accountId>.*` paths, keep full-channel precedence for channel-global or mixed changes, thread `accountId` through the hot-reload handler, and keep focused planner plus handler regression tests.
