## Summary

Keep open. Current main still does not implement this PR's central Mission Control runtime-ops surface: no `dashboard.summary`/`dashboard.delta` RPC/event contract, no incident handlers, no global tool activity registry, no Tool Runtime / Execution Queue / Autonomy Posture panels, and finished background processes still do not preserve `sessionKey` for session pivots. The PR also has review-identified correctness bugs, so the right path is contributor rework or maintainer judgment, not automated closure.

## Best Possible Solution

Keep the PR open for maintainer judgment or contributor rework. The best path is a narrower runtime-ops change that adds an explicit dashboard summary/event contract, a global tool activity registry, process queue visibility with retained `sessionKey` pivots, an autonomy posture view backed by existing tool/exec policy data, focused docs/tests, and fixes for the `evt.isError` classification and startup `requestContext` race while removing unrelated workspace/script/test churn.
