## Summary

Keep #22358 open. Current main still lacks the requested post-subagent-completion, pre-announcement plugin hook. Existing hooks cover subagent spawn, delivery-target routing, spawned, and ended cleanup, but `subagent_ended` is void, narrow, and deferred until after announce delivery for completion-mode runs. The provided issue discussion also explicitly keeps this request alive after adjacent delivery fixes.

## Best Possible Solution

Keep #22358 open until OpenClaw adds a public typed plugin hook at the post-subagent-completion/pre-announcement point, or explicitly folds that exact scope into a canonical lifecycle/delivery design. The contract should expose child/session/run metadata, completion output or child-history access, timing/usage where available, delivery status/error visibility, and explicit return semantics for announcement enrichment, suppression, or filtering, with docs and ordering regression tests.
