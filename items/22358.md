## Summary

Keep open. Current `main` already has subagent lifecycle hooks, but the only completion-side hook is `subagent_ended`, and for runs that expect a completion message it is deliberately deferred until announce cleanup finishes. That hook is also a void hook with a narrow event payload, so it cannot modify/suppress the completion announcement and does not expose delivery success vs failure. The requested pre-announce post-completion extension point is not implemented on `main`.

## Evidence

- **Completion hook is deferred until after announce cleanup for normal subagent completions:** `completeSubagentRun()` sets `shouldDeferEndedHook` when `entry.expectsCompletionMessage === true`, so `subagent_ended` is not emitted at raw task completion for the common announce path.
  - file: [src/agents/subagent-registry-lifecycle.ts:697](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/agents/subagent-registry-lifecycle.ts#L697)

  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **On successful announce delivery, the ended hook runs only after delivery bookkeeping:** `finalizeSubagentCleanup()` marks `completionAnnouncedAt`, sets detached-task delivery status to `delivered`, and only then calls `emitCompletionEndedHookIfNeeded()`. That is later than the feature request's requested pre-announcement hook point.
  - file: [src/agents/subagent-registry-lifecycle.ts:457](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/agents/subagent-registry-lifecycle.ts#L457)

The hook event shape has no `announceDelivered` or equivalent field, so plugins cannot distinguish success from failed delivery from the hook payload alone.

## Risks / Open Questions

- There is partial overlap with the request because plugins can already observe subagent completion via `subagent_ended` and can use `childSessionKey`/`runId` to inspect state indirectly, so a future maintainer might choose to fold this into a broader hook-shape expansion rather than add a brand-new hook name.
