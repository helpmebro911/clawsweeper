## Summary

Keep open. Current main has subagent lifecycle plugin hooks and improved announce fallback delivery, but it still does not expose the requested post-subagent-completion, pre-announcement hook that can inspect completion output, observe delivery status, and enrich or suppress the completion announcement. The latest issue discussion also contains an explicit maintainer/Codex review comment keeping this exact scope open.

## Best Possible Solution

Keep #22358 open until OpenClaw adds a public plugin hook at the post-completion/pre-announcement point, with completion output, child/session metadata, delivery result/failure visibility, and explicit announcement enrichment or suppression semantics. If maintainers prefer a broader design, fold this exact scope into a canonical lifecycle/delivery issue rather than closing it as already covered.
