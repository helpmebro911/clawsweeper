## Summary

Keep #22358 open. Current main still lacks the requested post-subagent-completion, pre-announcement plugin hook. Existing subagent hooks cover spawn, delivery-target routing, spawned, and ended cleanup, but `subagent_ended` is a void hook with no completion output, child-history access, delivery-result visibility, usage data, or announcement enrichment/suppression return path. The provided issue discussion also explicitly keeps this narrower request alive after adjacent delivery fixes.

## Best Possible Solution

Keep #22358 open until OpenClaw adds a public typed plugin hook at the post-subagent-completion/pre-announcement point, or explicitly folds that exact scope into a canonical lifecycle/delivery design. The contract should expose child/session/run metadata, completion output or child-history access, timing/usage where available, delivery path/status/error visibility, and explicit return semantics for announcement enrichment, suppression, or filtering, with docs and ordering regression tests.
