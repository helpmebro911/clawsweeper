## Summary

Current main does not implement the requested i18n for built-in Telegram bot commands or Web UI slash commands. OpenClaw has Control UI locale infrastructure and Telegram custom menu entries, but built-in command names/descriptions remain English, Telegram registers a single command list without `language_code`, and custom/plugin command seams do not cover translating reserved built-in commands.

## Evidence

- **Scope anchor:** OpenClaw’s vision says optional capability should usually be plugins, but if a useful feature cannot be built as a plugin yet, the preferred path is extending the plugin API rather than adding one-off core behavior. This request is not a clean ClawHub close because built-in command localization is not currently expressible through plugin/custom command seams.
