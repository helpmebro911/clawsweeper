## Summary

## Close Comment

Closing after Codex review because current OpenClaw already supports this structure on `main`. The default memory layer still intentionally keeps short-term notes in `memory/YYYY-MM-DD.md` and durable facts in `MEMORY.md` (`docs/concepts/memory.md:17`), but the bundled `memory-wiki` plugin provides the tree-structured knowledge layer requested here: deterministic vault directories like `entities/`, `concepts/`, `syntheses/`, `sources/`, and `reports/` are documented in `docs/plugins/memory-wiki.md:102` and created in `extensions/memory-wiki/src/vault.ts:10`. Bridge mode also imports the active memory plugin's `daily-note` and `memory-root` artifacts into stable wiki source pages (`docs/plugins/memory-wiki.md:78`, `extensions/memory-wiki/src/bridge.ts:44`, `extensions/memory-core/src/public-artifacts.ts:33`). In other words, the date-based daily files remain the raw capture layer, and the tree/indexed organization already exists as the supported higher-level memory layer, so there is no separate core cleanup to track here.
