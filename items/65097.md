## Summary

Keep this issue open. Current main still lacks a Dreaming diary language/locale setting: memory-core's narrative prompt is English-only, the narrative API and sweep call sites only carry timezone, the runtime config type/resolver omit language and locale, the strict plugin manifest schema rejects those keys, and public docs expose no language setting. Related #69354 was closed as a duplicate of this same remaining feature gap, not as an implementation.

## Best Possible Solution

Keep this issue open and implement the feature in `memory-core`: add a documented `dreaming.language` or `dreaming.locale` config contract, decide whether `auto` is supported, resolve the hint in the memory-host config path, thread it into the Dream Diary prompt/subagent run and request-scoped fallback behavior, update `extensions/memory-core/openclaw.plugin.json`, update public Dreaming and memory config docs, refresh generated config metadata if required, and add focused resolver/narrative tests.
