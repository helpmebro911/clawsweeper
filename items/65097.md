## Summary

Keep this issue open. Current main still does not provide a Dreaming diary language or locale setting: the narrative prompt remains fixed English prose, the narrative generation API and phase call sites only thread timezone/model, the resolved config contract has no language/locale field, the strict memory-core manifest would reject those keys, and the public docs list only enabled/frequency/model. Related #69354 was closed as a duplicate of this same feature gap, not as an implementation.

## Best Possible Solution

Keep this issue open and implement the feature in `memory-core`: add a documented `dreaming.language` or `dreaming.locale` config contract, decide whether `auto` is supported, resolve the hint in the memory-host config path, thread it into Dream Diary subagent/fallback narrative generation, update `extensions/memory-core/openclaw.plugin.json`, update the public Dreaming and memory config docs, refresh generated config metadata if required, and add focused resolver/narrative/phase tests.
