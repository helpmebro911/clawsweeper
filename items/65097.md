## Summary

Keep open. Current main still lacks the requested Dreaming diary language/locale control: the narrative prompt and prompt builder are English-only, the narrative runtime only accepts timezone, and the strict `memory-core` manifest/docs do not expose `dreaming.language`, `dreaming.locale`, `auto`, or a prompt-template override. The related #69354 language-instability report was closed as a duplicate of this item, and nearby dreaming model/execution config reports do not implement the language surface.

## Best Possible Solution

Keep this item open and implement the feature in `memory-core`: add a documented `dreaming.language` or `dreaming.locale` contract, decide whether `auto` is supported, thread the resolved hint into the Dream Diary system/user prompt path, update `extensions/memory-core/openclaw.plugin.json`, memory docs, and focused config/narrative tests. If prompt override support is accepted, keep it schema-validated and narrowly scoped rather than adding an unbounded hidden prompt escape hatch.
