## Summary

Keep open. Current `main` still does not expose a `runtime.usage` namespace to plugins, the shipped plugin runtime docs do not document such an API, and the prior PR review identified remaining public-SDK baseline work rather than showing the change was already implemented elsewhere.

## Evidence

- **`PluginRuntimeCore` on `main` has no `usage` namespace:** In current `main`, `src/plugins/runtime/types-core.ts` goes from `events` and `logging` directly to `state`/`tasks`; there is no `usage` property in the exported runtime core type.
