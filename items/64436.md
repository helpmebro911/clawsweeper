## Summary

Keep open. Current main still does not expose `api.runtime.usage` or an equivalent public plugin SDK path for model pricing, while the internal pricing helpers exist and are tested. This PR is narrow, active, and needs maintainer API review rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer API review. If maintainers accept the shape, the best path is to rebase onto current main, keep the narrow additive `api.runtime.usage` implementation, add SDK runtime docs and a small runtime-shape or assembly test, retain the regenerated SDK API baseline, and provide the relevant changed-gate proof before merge.
