## Summary

Keep open. Current main still lacks `api.runtime.usage` or an equivalent public plugin SDK path for resolving model pricing and estimating usage cost, while the requested helpers already exist internally. This PR remains a narrow, active plugin runtime API proposal for external plugins such as ContextClaw, not cleanup-close material.

## Best Possible Solution

Keep this PR open for maintainer API review. If maintainers accept the shape, the best path is to rebase onto current main, keep the narrow additive `api.runtime.usage` wiring, add SDK runtime docs and a small runtime-shape or assembly test, retain the regenerated plugin SDK API baseline, and provide the relevant changed-gate/API-check proof before merge.
