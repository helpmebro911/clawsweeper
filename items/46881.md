## Summary

Keep #46881 open. Current main still lacks WhatsApp inbound reaction forwarding: the monitor only subscribes to normal message upserts, `WebInboundMessage` has no reaction fields, and docs/config cover outbound and ack reactions rather than inbound reaction events. The PR is focused on the bundled WhatsApp plugin and remains useful, though it needs maintainer review and tests before merge.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to update the patch against current main, fix or consciously document the reaction-removal dedupe behavior, add targeted WhatsApp inbound reaction tests, and keep the implementation owned by the bundled WhatsApp plugin without adding unnecessary core API surface.
