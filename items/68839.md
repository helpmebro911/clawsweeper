Keep open. Current main still has the reported key-only cleanup in the auto-reply followup drain finally block, and the linked PR is a focused fix with a regression test for that exact race. No close reason applies: it is not implemented on main, not superseded, and it is a recent contributor bug-fix PR with concrete reproduction details.

The best fix remains the narrow identity guard in `src/auto-reply/reply/queue/drain.ts` plus the deterministic regression test added by the PR; if maintainers want to alter queue cleanup semantics more broadly, that should be done as a follow-up rather than closing this focused bug fix.
