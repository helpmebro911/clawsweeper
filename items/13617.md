## Summary

Keep #13617 open. Current main still does not provide the requested side-effect-free cron/skills testing framework: there is no `openclaw cron test`, `openclaw cron validate`, or `openclaw skills test`; `cron.run` queues real work; and the protocol/tool execution path lacks a global dry-run interception layer. Related #41418 is narrower and still open, so it does not supersede this umbrella.

## Best Possible Solution

Keep #13617 open until OpenClaw ships or intentionally scopes a real side-effect-free testing surface. A complete path should add `openclaw cron test`, `openclaw cron validate`, structured `--json` CI output, `openclaw skills test`, cron protocol support for dry-run/validate intent, and integration with the global tool dry-run/interception design tracked by #41418 instead of creating a separate mock path.
