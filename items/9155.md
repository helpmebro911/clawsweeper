## Summary

Keep #9155 open. Current main still does not implement `cron.defaults.delivery`: the cron config contract has no `defaults` block, validation is strict, job creation only preserves explicit delivery or defaults isolated agent-turn jobs to `{ mode: "announce" }`, and delivery planning still falls back to `channel: "last"` when no target is present. Recent session-key inference helps the related `last` routing bugs for jobs created from a resolvable chat, but it is not the requested global default delivery target.

## Best Possible Solution

Keep this issue open until OpenClaw either implements a real config-level primary cron delivery default or a maintainer explicitly chooses a different product direction. A complete implementation would add `cron.defaults.delivery` to `CronConfig`, zod/schema/generated config docs/help, cron job creation or normalization, UI/CLI/docs surfaces as needed, and regression tests showing job-level delivery overrides the global default before falling back to `last`.
