## Summary

Keep #9155 open. Current main still lacks a `cron.defaults.delivery` or equivalent primary cron delivery default. The config/schema surface only covers failure-delivery defaults, job creation still stores explicit delivery or `{ mode: "announce" }`, and delivery planning still resolves missing announce targets to `last`.

## Best Possible Solution

Keep the issue open until OpenClaw implements a real primary cron delivery default, likely `cron.defaults.delivery`, or maintainers document a deliberate alternate product direction. A complete fix should add config types/schema/generated help/docs, apply precedence as job-level delivery first, then global primary default, then `last`, and cover CLI, gateway protocol/service normalization, delivery planning, Control UI, and regression tests.
