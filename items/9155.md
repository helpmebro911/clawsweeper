## Summary

Keep #9155 open. Current main still does not implement `cron.defaults.delivery` or an equivalent primary cron delivery default; cron config only has failure-notification defaults, and omitted isolated announce delivery still resolves through `last`.

## Best Possible Solution

Keep the issue open until OpenClaw implements a real primary cron delivery default, likely `cron.defaults.delivery`, or maintainers document a deliberate alternate product direction. A complete fix should add config types/schema/help/docs, apply precedence as explicit job delivery first, then global primary default, then `last`, and cover CLI, gateway protocol, service normalization, delivery planning, Control UI, and regression tests.
