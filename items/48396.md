## Summary

Keep open. Current main still does not implement message previews in the Control UI session list, and PR #48396 is not safe to close as implemented or merge as posted: it renders `subject` instead of the existing `lastMessagePreview` field and does not wire the UI type/controller to request preview data.

## Best Possible Solution

Keep the PR open only if it is revised into an end-to-end Control UI implementation: remove the scratch `SESSION_LIST_IMPROVEMENT.md`, add `lastMessagePreview` to the UI `GatewaySessionRow` and session-event field flow, request preview data deliberately in `loadSessionsOnce()` or use the existing preview API with bounded cost, render a normalized timestamp-plus-preview line from `row.lastMessagePreview`, and add focused controller/view tests. Coordinate with #47120 and #47243 so only one corrected implementation lands.
