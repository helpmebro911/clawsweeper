## Summary

Current `main` has a generic embedded-tool schema normalizer that now flattens top-level union/`anyOf` schemas before provider handoff, so the PR root cause is partially addressed. But the Feishu source schemas in `extensions/feishu/src/doc-schema.ts` and `extensions/feishu/src/wiki-schema.ts` are still union-based, and the Feishu-specific schema/test changes from PR #42209 are not present on `main`. That is not strong enough evidence for an `implemented_on_main` close.
