## Summary

Keep open. Current main already covers cron edit and run-history UI, and the old root Vitest/jsdom rationale is partly obsolete under the current UI test config. But the PR's remaining substantive pieces are not implemented on main: the i18n fixes are still missing, and cron job chaining via onSuccessJobId/onFailureJobId is absent from types, gateway schema, persistence, UI form state, and run dispatch. There is not enough evidence for implemented-on-main, duplicate/superseded, or another close reason.

## Best Possible Solution

Keep the PR open for maintainer review, but split the path forward: land the small cron i18n fixes separately against the current UI test layout, and make an explicit product/API decision on cron-native chaining versus steering users to Task Flow/Lobster for orchestration. If cron chaining is desired, implement it with current protocol schema, persistence, run-dispatch, client artifact, docs, and regression-test coverage.
