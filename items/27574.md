## Summary

Keep #27574 open. Current main still has browser-control backend/API/CLI primitives, but no Control UI/Dashboard browser preview panel with live screenshots, URL/title, console errors, or manual interaction. The related #63926 is still open and narrower, focused on login handoff/browser management rather than the full live remote-debugging preview requested here.

## Best Possible Solution

Keep #27574 open as the broad remote browser-preview/dashboard request. The best maintainer path is either a core Control UI browser panel backed by the existing browser-control and browser.request primitives, or first adding a supported Control UI panel contribution API and then implementing the preview as a browser plugin. Coordinate with open #63926 for login handoff/browser management, but do not treat it as a complete replacement for the live remote-debugging preview request.

## Risks / Open Questions

- A live browser preview/input panel would expose browser pixels, URLs/titles, console/page errors, and possibly remote-control actions, so it needs explicit operator-admin authorization, origin protections, rate limiting, and handoff semantics.
- A plugin-based implementation likely needs a supported Control UI tab/panel contribution API first; current plugin surfaces can expose backend endpoints but not integrate cleanly into the dashboard shell.
