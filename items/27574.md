## Summary

Current main still does not implement #27574's requested live browser preview panel in the Dashboard/Control UI. OpenClaw has strong browser-control backend primitives and an admin `browser.request` gateway method, but no browser preview tab, route, renderer branch, or documented Control UI capability. The related #63926 overlaps on visible browser/login handoff, but it is narrower and remains open, so it is not a canonical replacement for this broader remote-debugging preview request.

## Best Possible Solution

Keep #27574 open as the broader Control UI browser-preview gap. The best path is either a core Browser panel backed by the existing `browser.request` and browser-control primitives, or first adding a supported Control UI panel contribution API and then implementing the preview as a browser plugin. Coordinate with open #63926 for login handoff/browser management, but do not treat it as a complete replacement for this live remote-debugging preview request.

## Risks / Open Questions

- A live browser preview/input panel would expose browser pixels, current URL/title, console/page errors, and possibly remote input, so it needs operator-admin auth, origin protections, rate limits, and clear handoff semantics.
- If maintainers prefer this outside core, OpenClaw likely needs a supported Control UI panel/tab contribution API before a browser-preview plugin can satisfy the request cleanly.
