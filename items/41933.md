## Summary

Keep open. The provided GitHub context shows open PR #41955 still explicitly targets this issue with closing syntax, so the issue should not be closed before that PR is reviewed, landed, closed, or superseded. Current main also still builds the chat session dropdown from `sessions.list` rows plus the active session key; `agents.list` is loaded and used for labels, but agents with no stored session are not added as options.

## Best Possible Solution

Keep this issue open and use #41955, or a refreshed successor, as the implementation path. The fix should merge configured `agents.list` entries into the chat session option groups using canonical main session keys, dedupe existing `sessions.list` rows, preserve current session/cron filtering behavior, and add focused UI regression coverage for agents that have no stored sessions yet.
