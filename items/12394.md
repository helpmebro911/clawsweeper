## Summary

Current main still lacks the requested channel-level `channels.whatsapp.heartbeat.target` / `targets` routing contract. WhatsApp channel heartbeat config is still visibility-only, while recipient selection is driven by agent-level `target`/`to` or the WhatsApp runtime recipient resolver's session/allowFrom fallbacks.

## Best Possible Solution

Keep #12394 open until maintainers either implement an additive channel-level WhatsApp heartbeat routing contract or explicitly choose a different product direction. A complete fix should add typed `target`/`targets` support to the channel heartbeat schema/types/docs, validate recipients against WhatsApp account scope and allowlists, apply it only to heartbeat OK routing without breaking alert delivery semantics, and add regression tests for single target, multiple targets, omitted-target backward compatibility, account-scoped config, and invalid targets.
