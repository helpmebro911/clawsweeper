## Summary

Keep #12394 open. Current main still lacks a channel-level WhatsApp heartbeat recipient contract: `channels.whatsapp.heartbeat` is visibility-only, WhatsApp recipient resolution ignores `target`/`targets`, and the existing agent-level `heartbeat.target`/`to` routes the whole heartbeat delivery rather than OK-only admin acknowledgments while preserving alert routing to relevant users.

## Best Possible Solution

Keep #12394 open until maintainers implement or explicitly reject an additive WhatsApp/channel heartbeat recipient contract. A complete fix should add typed/schema/generated-doc support for `channels.whatsapp.heartbeat.target` or `targets`, validate recipients against WhatsApp account and allowlist/pairing policy, apply those recipients only to `HEARTBEAT_OK` delivery, preserve alert/content routing, and add regression coverage for single target, multiple targets, omitted-target backward compatibility, per-account overrides, and invalid targets.
