## Summary

Current main still lacks the requested channel-level WhatsApp heartbeat recipient contract. The only channel heartbeat config remains visibility-only, while the existing per-agent heartbeat target/to path routes the whole heartbeat run and does not provide OK-only target(s) for channel-level HEARTBEAT_OK messages.

## Best Possible Solution

Keep #12394 open until maintainers implement or explicitly reject an additive WhatsApp channel heartbeat recipient contract. A complete implementation should add typed/schema/generated-doc support for channel-level OK recipients, validate targets against WhatsApp account and allowlist policy, apply those recipients to `HEARTBEAT_OK` delivery without changing alert routing, and add regression coverage for single target, co-admin targets, omitted-target backward compatibility, per-account config, and invalid targets.
