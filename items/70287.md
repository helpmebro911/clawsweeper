## Summary

Keep #70287 open. Current main at 4a3030df9efa still wires `msteams` search to `searchMessagesMSTeams`, and that implementation still builds `/messages?$search=...` with a `ConsistencyLevel` header. The existing search tests also still assert `$search`, so the PR is not obsolete or implemented on main. The related bug #70127 was closed, but the source bug remains present in this checkout.

## Best Possible Solution

Keep #70287 open and review it as the likely fix path for #70127. The implementation direction should remove `$search` from `searchMessagesMSTeams`, list recent chat/channel messages with `$top` capped at 50 plus pagination, perform body matching locally with HTML stripping, and add regression tests that assert `$search` is absent. Before merge, maintainers should verify or adjust the sender-filter behavior against Microsoft Graph’s documented query-option support, possibly moving sender filtering local as well.
