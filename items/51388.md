## Summary

Keep open. Current `main` still does not carry `tools.subagents.tools.alsoAllow` through the plugin optional-tool allowlist path, so the PR’s reported bug remains plausible and unfixed on `608c08fc54da2f86338366f8eeafd7e2a02713aa`.

## Evidence

- **Allowlist collector still ignores `alsoAllow`:** `src/agents/tool-policy.ts:67-99` defines `ToolPolicyLike` with only `allow`/`deny`, and `collectExplicitAllowlist()` iterates only `policy.allow`. An `alsoAllow`-only policy contributes nothing to the plugin allowlist.
  - file: [src/agents/tool-policy.ts:67](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/agents/tool-policy.ts#L67)

  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Subagent policy constructors drop `alsoAllow`:** Both `resolveSubagentToolPolicy()` and `resolveSubagentToolPolicyForSession()` read `configured?.alsoAllow` but return only `{ allow: mergedAllow, deny }`. In the `alsoAllow`-only case, `allow` stays `undefined`, so the explicit plugin-tool allowlist remains empty.
  - file: [src/agents/pi-tools.policy.ts:81](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/agents/pi-tools.policy.ts#L81)
