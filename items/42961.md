## Summary

Keep this PR open. Current main still applies a fixed short OpenRouter Anthropic cache_control marker and does not route the effective cacheRetention setting into the OpenRouter marker wrapper, so explicit "none" and "long" remain unimplemented for this path. The PR is a focused bug-fix candidate, but it should be refreshed against the current helper layout and the public SDK export should remain backward-compatible.

## Best Possible Solution

Keep the PR open and refresh/review it against current main. The fix should thread the resolved effective cacheRetention for verified OpenRouter Anthropic routes into the cache marker application, preserve default short caching, suppress markers for cacheRetention: "none", emit ttl: "1h" for cacheRetention: "long", keep arbitrary OpenAI-compatible proxy exclusions, and add regression tests for none/short/long/default plus non-Anthropic and custom-proxy cases. Maintain backward compatibility for the exported plugin SDK helper.
