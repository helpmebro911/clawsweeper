## Summary

Keep open. Current `main` still hard-codes the OpenRouter Anthropic cache marker to the 5-minute ephemeral form and does not thread `cacheRetention` into that wrapper, so this PR still targets a real behavior gap rather than an already-fixed issue.

## Evidence

- **Post-plugin wrapper still ignores retention:** `applyPostPluginStreamWrappers` always installs `createOpenRouterSystemCacheWrapper(ctx.agent.streamFn)` with no `cacheRetention` argument, so the OpenRouter-specific cache wrapper cannot honor per-model or per-agent retention settings.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:508](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/agents/pi-embedded-runner/extra-params.ts#L508)
  - command: `nl -ba src/agents/pi-embedded-runner/extra-params.ts | sed -n '500,512p'`
  - sha: [459d27707648](https://github.com/openclaw/openclaw/commit/459d277076488d5833a181cca7d8bf5c0f9696dc)
- **Wrapper still forces short cache markers:** On verified OpenRouter Anthropic routes, `createOpenRouterSystemCacheWrapper` always calls `applyAnthropicEphemeralCacheControlMarkers(payloadObj)`, with no branch for `cacheRetention: "none"` or `"long"`.
  - file: [src/agents/pi-embedded-runner/proxy-stream-wrappers.ts:74](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/agents/pi-embedded-runner/proxy-stream-wrappers.ts#L74)

  - file: [src/agents/anthropic-payload-policy.ts:238](https://github.com/openclaw/openclaw/blob/459d277076488d5833a181cca7d8bf5c0f9696dc/src/agents/anthropic-payload-policy.ts#L238)
