## Summary

Keep this PR open. The checked-out main still refreshes models.json only through the default-agent primary-model warmup path, and that path returns before cache generation when no explicit default primary model is configured. The PR remains a focused implementation candidate for the non-default agent startup-refresh gap described by #71192.

## Best Possible Solution

Keep this PR open for review. The best path is to land or revise a startup helper that refreshes the unique configured agent dirs independently of primary-model prewarm, with regression coverage for non-default agents and the no-default-primary case while preserving default and override agent-dir semantics.
