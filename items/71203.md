## Summary

Keep open. Current main still only refreshes `models.json` during startup through the default-agent primary-model prewarm path, and the PR’s separate configured-agent refresh behavior is not present on main.

## Best Possible Solution

Keep the PR open. The best path is to land a refined startup refresh that regenerates `models.json` for each unique configured agent dir independently of primary-model prewarm, avoids unintended synthetic-main/default-dir writes when agent-dir overrides are active, and keeps regression coverage for multi-agent and no-default-primary configs. If merged, it should close or update #71192.
