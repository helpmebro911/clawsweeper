## Summary

Keep #71203 open. Current main does not implement the PR's startup refresh for configured non-default agent `models.json` caches; startup still refreshes only the default agent dir and only inside the explicit primary-model warmup path. The PR is a recent, focused bug-fix candidate with concrete linked reproduction in #71192 and active author follow-up.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a focused startup helper that refreshes `models.json` for the default agent dir and each configured agent dir, independent of primary-model prewarm, with regression coverage for configured non-default agents and the no-default-primary shape. Maintainers should still review the review-thread concern about avoiding unintended writes for the synthetic `main` fallback when an agent-dir override is active.
