## Summary

Keep this PR open. Current main still refreshes `models.json` during startup only inside primary-model prewarm: it returns before any cache write when no explicit default primary model is configured, and otherwise writes only the active default agent dir. The PR remains a focused implementation candidate for the stale non-default agent cache behavior discussed in #71192. The security review found no supply-chain-sensitive scope in the provided diff; the remaining concerns are functional semantics around agent-dir overrides and synthetic `main` handling.

## Best Possible Solution

Keep this PR open for maintainer review or revision. The best implementation is a startup refresh helper that refreshes unique concrete configured agent dirs independently of primary-model warmup, preserves default and override agent-dir semantics, avoids synthetic `main` duplicate writes, catches per-dir refresh failures without aborting gateway startup, and adds focused regression coverage for non-default agents and the no-default-primary case.

## Risks / Open Questions

- Current main can still leave configured non-default agent `models.json` files stale after restart until a later per-agent path refreshes that specific directory.
- The PR's current helper direction still needs maintainer review for `OPENCLAW_AGENT_DIR` / `PI_CODING_AGENT_DIR` scoping and synthetic `main` fallback behavior, because `listAgentIds(cfg)` can return `main` when no agents are configured while `resolveOpenClawAgentDir()` may point at an override directory.
- Before landing, the updated branch should get targeted gateway startup test proof; this review was intentionally read-only.
