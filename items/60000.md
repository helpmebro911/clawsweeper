## Summary

Keep open. Current `main` still has the reported runtime/protocol mismatch: exec approval allowlist entries can carry `source: "allow-always"`, the Control UI round-trips that data, and the gateway AJV schema for `exec.approvals.set` still rejects it with `additionalProperties: false`. The latest release `v2026.4.23` shows the same schema, so this is not already fixed on shipped code.

## Evidence

- **Gateway schema still rejects `source`:** `ExecApprovalsAllowlistEntrySchema` allows `id`, `pattern`, `argPattern`, `lastUsedAt`, `lastUsedCommand`, and `lastResolvedPath`, but does not declare `source`; the schema sets `additionalProperties: false`, so a payload containing `source` is invalid.
