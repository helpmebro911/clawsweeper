## Summary

Current main still does not provide true progressive Feishu block delivery. Feishu still forces core block streaming off, drops non-card block payloads, and only mirrors block text into the Feishu streaming-card path when card streaming is active. The existing Feishu tests assert that suppression, so the related closed Feishu streaming reports do not supersede the remaining request.

## Best Possible Solution

Keep this open for a Feishu-owned implementation decision. A real fix should add a non-card progressive block delivery path for Feishu, wire it to the channel/global block-streaming configuration without reintroducing silent drops or duplicate final/card deliveries, and update Feishu tests plus docs/config metadata so `streaming` card preview and true block replies are clearly separate.
