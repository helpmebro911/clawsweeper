## Summary

Keep this PR open. Current main and the latest release still have the Mattermost slash-command split-loader/state shape targeted by the PR, the route can still return the reported 503 from an empty module-local state map, and the related #69038 is an open alternate implementation rather than a merged or maintainer-canonical superseder.

## Best Possible Solution

Keep this PR open for maintainer review or consolidation with #69038. The best product/code path is to land one focused Mattermost slash-command fix that makes route registration and monitor activation share the same account state across bundled-entry and runtime loader boundaries, ideally with a regression test for the dual-load case and an explicit decision on the entrypoint import-graph tradeoff.
