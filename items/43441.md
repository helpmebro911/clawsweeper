## Summary

Keep this PR open. Current main still resolves workspace templates with only the source-layout `../../docs/reference/templates` module fallback and checks `cwd` before that fallback, while the package layout ships runtime code in `dist/` alongside packaged `docs/`. The PR remains a focused implementation candidate, and its current head addresses the earlier review concern by computing fallback paths from the injected `moduleUrl`.

## Best Possible Solution

Keep this PR open as the implementation candidate. Maintainers should review the focused resolver change, prefer package-root then bundled `../docs` then source `../../docs` then `cwd` fallback, keep the packageRoot-null bundled-layout regression test, rebase if needed, and land only after the changed gate passes.
