## Summary

Keep #13570 open. Current main still lacks `contextPruning.forcePruneRatio` or an equivalent high-usage TTL bypass: config types/schemas/docs omit the key, runtime settings do not carry a bypass threshold, and the context-pruning hook returns before pruning while the TTL window is fresh. The related implementation PR #13572 is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep #13570 open until a maintainer implements or explicitly declines the feature. The likely implementation belongs in the existing context-pruning path: add `agents.defaults.contextPruning.forcePruneRatio` to config types, strict schemas, generated schema, and docs; carry it through `computeEffectiveSettings()` and runtime state; estimate context usage before the fresh-TTL early return; bypass TTL only when the configured threshold is met; preserve the current TTL gate below threshold; and add regression tests for both above-threshold bypass and below-threshold preservation.
