## Summary

Keep #13570 open. Current main at cf834e2a21eac7fd218989fb5b85e7e8dd4c3246 still does not implement `contextPruning.forcePruneRatio` or an equivalent high-usage TTL bypass. The config types/schema/docs omit the key, effective context-pruning settings do not parse a bypass threshold, and the runtime still returns during a fresh TTL window before usage-ratio pruning can run. The related PR #13572 directly addressed the request but is closed unmerged, so it does not supersede or ship the issue.

## Best Possible Solution

Keep the issue open until a maintainer implements or explicitly declines the feature. The likely implementation is to add `agents.defaults.contextPruning.forcePruneRatio` to the public config type, strict schema, generated schema, docs, effective settings, and runtime; estimate current context usage before the fresh-TTL early return; bypass TTL only when the configured ratio is met; and add regression tests for both high-usage bypass and normal below-threshold TTL behavior.
