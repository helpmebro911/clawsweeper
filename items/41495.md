## Summary

Keep open. Current main supports Telegram inline keyboards only through structured tool/presentation payloads; it does not appear to parse the reported Gemini plain-text `[[ {"text":...,"callback_data":...} ]]` output into Telegram buttons. There is also a live contract mismatch to resolve: the system prompt still tells models to use `buttons=[[{text,callback_data,style?}]]`, while the message tool schema exposes `presentation` and tests assert no top-level `buttons` property.
