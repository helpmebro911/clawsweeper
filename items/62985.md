## Summary

This issue should remain open because the reported Telegram account-level `agentId` regression is not implemented on current `main`, and an open unmerged PR (#63380) explicitly uses closing syntax for this issue. Current `main` still treats Telegram account configs as strict objects without `agentId`, while the documented supported routing path is top-level `bindings[]`; that may be the intended direction, but the open PR is the active implementation candidate and maintainer decision point.

## Best Possible Solution

Keep this issue open and review #63380 as the implementation candidate. Maintainers should either land a focused backward-compatibility fix that accepts and correctly routes or migrates `channels.telegram.accounts.<id>.agentId`, with regression coverage for the reported multi-account config, or explicitly decide that top-level `bindings[]` is the only supported path and improve Telegram upgrade docs/doctor output so users can find that migration.
