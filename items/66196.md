## Summary

Keep this PR open. Current main still sends WhatsApp inbound text and audio transcripts to the agent through raw model-facing fields, so the hardening goal remains relevant. The submitted patch is not merge-ready because it wraps only legacy envelope/body surfaces, misses `BodyForAgent`, can destabilize combined echo keys, and wraps trusted `messagePrefix`/reply-context structure inside the untrusted block.

## Best Possible Solution

Keep this PR open as an incomplete security-hardening candidate. The maintainer path is to revise or replace it so WhatsApp separates deterministic echo-key/envelope text from prompt-facing content, wraps or isolates the actual `BodyForAgent`/transcript/reply-context/group-history text that reaches the agent, keeps trusted operator `messagePrefix` outside untrusted markers, and adds focused WhatsApp regression tests before landing.

## Risks / Open Questions

- Current main still exposes WhatsApp inbound message text and successful audio transcripts through raw `BodyForAgent`/`BodyStripped` prompt text.
- Merging this PR unchanged would likely create a false sense of security because it wraps legacy envelope/body strings but not the active model-facing field.
- Merging this PR unchanged could break combined-message duplicate suppression because wrapped content includes fresh random marker IDs.
- Merging this PR unchanged would place trusted operator `messagePrefix` text and reply-context framing inside an untrusted-content block.
- The PR has no focused regression tests for direct messages, transcripts, group history, reply context, prefix handling, or duplicate echo suppression.
