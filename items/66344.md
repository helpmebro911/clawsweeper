## Summary

Keep open. Current main still has only fence-aware streaming block chunking and no Markdown table-span protection, so PR #66344 is not obsolete or implemented. The related #66568 was closed unmerged in favor of this PR, and #66614 remains the canonical open bug for the same table-splitting behavior.

## Best Possible Solution

Keep the PR open and ask for a corrected, focused implementation or maintainer-owned replacement that adds table-span protection to `src/agents/pi-embedded-block-chunker.ts`, keeps fence behavior intact, and adds regression coverage in both `src/agents/pi-embedded-block-chunker.test.ts` and the `streams-soft-chunks-paragraph-preference` integration harness. Before merge, address the review-thread edge cases for incremental partial tables at the buffer tail and no-leading-pipe table rows, then link/resolve #66614 and leave #66605 only if broader integration coverage remains separately useful.
