## Summary

Current main still matches #71646. `OpenClawChannelBridge` keeps `pendingClaudePermissions` and `pendingApprovals` as unbounded instance maps: close only clears `pendingWaiters`, Claude permission entries are removed only by a matching `yes/no <id>` reply, and approval entries are removed only by matching resolved events despite storing `expiresAtMs`. The linked fix PR #71648 is still separate from main, so the issue should remain open until that or an equivalent fix lands.

## Best Possible Solution

Keep #71646 open until #71648 or an equivalent maintainer-approved fix lands. The fix should clear both pending maps on bridge close, expire stale Claude permission and approval entries, preserve the MCP protocol/tool surface, add focused regression tests for stale entries and close cleanup, and let maintainers decide whether hard cap/FIFO burst protection belongs in the same patch or a follow-up.
