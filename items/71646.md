## Summary

Current main still matches #71646. The MCP channel bridge still stores pending Claude permission requests and pending approvals in plain instance Maps with no TTL/sweeper, no hard cap, and no close-time cleanup. The linked fix PR #71648 remains the appropriate path or needs an equivalent maintainer-approved implementation.

## Best Possible Solution

Keep #71646 open until PR #71648 or an equivalent maintainer-approved fix lands. The implementation should clear both pending maps on bridge close, expire stale Claude permission and approval entries, preserve the existing MCP protocol/tool surface, add focused regression tests, and either include a bounded cap/FIFO guard or leave that as an explicit follow-up if maintainers prefer a smaller fix.
