## Summary

Current main still matches the reported MCP channel-bridge leak: both pending maps are plain instance Maps, only clear through narrow success paths, and are not cleared on bridge close. The same-author closing PR #71648 remains open, so this issue should stay open until that PR or an equivalent maintainer-approved fix lands.

## Best Possible Solution

Keep this issue open and use #71648, or an equivalent maintainer-approved patch, as the implementation path. The fix should clear both pending maps on bridge close, avoid post-close writes, expire stale Claude permission and approval entries using existing approval expiry metadata where available, preserve MCP notification/tool protocol behavior, and add focused regression tests. A hard cap/FIFO guard can land with the same fix if maintainers want burst resistance now, or be tracked as a narrow follow-up.
