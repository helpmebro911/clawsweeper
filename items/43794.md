## Summary

Keep this issue open. Current main still has SecretRef mitigation, audit, and one-way scrubbing for supported static credentials, but it does not provide first-class encryption at rest, a native OpenClaw keychain-backed secure store, or a master-password store for config/auth-profile credentials. The threat model still lists plaintext token storage as high residual risk and still recommends token encryption at rest.

## Best Possible Solution

Keep open as core security work. The best path is to design a first-class secure credential storage layer for OpenClaw credentials: native keychain and/or encrypted-at-rest provider support, backward-compatible migration from plaintext config/auth stores, explicit handling for OAuth and rotating credentials, token-rotation guidance, and updated threat model/docs/tests once shipped.
