## Summary

Keep this PR open. Current main still lacks importance-weighted temporal decay for memory search; the active memory-core implementation only supports plain temporal decay with `enabled` and `halfLifeDays`. The PR remains a live implementation candidate that supersedes the closed earlier #18919, but it needs a rebase onto the current `extensions/memory-core` layout and cleanup of unresolved config/schema review issues before maintainer review.

## Best Possible Solution

Keep this PR open as the live implementation candidate, but require a rebase onto current main and move the feature into `extensions/memory-core/src/memory/temporal-decay.ts` plus the current hybrid/manager wiring. Thread a well-specified `importanceBoost` config through `src/agents/memory-search.ts`, runtime schemas, generated config artifacts, and docs; add focused memory-core tests for important markers, path patterns, nested defaults/overrides, and blank pattern rejection. Split or drop the already-shipped session-memory reset changes and the unrelated `periodicExtraction` schema work unless they are fully wired and tested in separate focused changes.
