## Summary

Keep #10960 open. The request is for true mid-stream or active-run soft steering, but current main still gates same-turn steer injection on `isStreaming`, rejects active non-streaming/tool-wait windows, and documents model-boundary/followup fallback rather than pause/abort/inject/resume behavior. The issue discussion is concrete, and related PRs #64758 and #65010 are useful candidate work, but the provided context has them open and their fix commits are not present in the current main checkout.

## Best Possible Solution

Keep #10960 open as the runtime/product tracker. The best fix should replace the streaming-only gates with an active agent-loop lifecycle predicate for embedded and reply-run backends, preserve safe followup fallback when same-turn steering cannot be accepted, distinguish hard `sessions.steer` redirect from soft inbound queue steering, update the queue/agent docs, and add regression coverage for streaming steer success, active-but-not-streaming/tool-wait behavior, explicit `sessions.steer` behavior, and backend failure paths. Re-review after #64758, #65010, or a successor PR lands.
