## Summary

Keep #10960 open. The requested soft-steer behavior is still not implemented on current main: steering remains gated by streaming state, active non-streaming/tool-wait windows fall back to followup, and `sessions.steer` is a hard interrupt path rather than pause/abort/inject/resume soft steering. The recent maintainer/Codex comment and related PRs #64758 and #65010 are relevant, but those PRs are still only candidate work in the provided context and the current checkout still has the old gates.

## Best Possible Solution

Keep #10960 open as the product/runtime tracker. The best fix should define active-run soft-steer lifecycle semantics across streaming, tool execution, and backend failure; replace streaming-only acceptance gates with an active agent-loop predicate where safe; preserve followup fallback when same-turn steering cannot be accepted; keep `sessions.steer` hard-interrupt behavior distinct; update queue/agent docs; and add regression coverage for streaming steer success, active non-streaming/tool-wait steering, fallback behavior, and gateway/backend failure paths. Re-review after #64758, #65010, or a successor lands.
