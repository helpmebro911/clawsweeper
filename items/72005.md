## Summary

Keep PR #72005 open. Current main does not implement the server-rendered outbound status footer, fabricated-footer stripping, or context-warning state/config proposed by the PR, and the PR is concrete but has unresolved correctness issues that should be fixed before merge rather than closed as obsolete.

## Best Possible Solution

Keep the PR open but require fixes before merge: process the same text for direct and gateway delivery, keep dry-run paths read-only, make threshold check-and-record atomic and record skipped lower thresholds, then add targeted tests around direct/gateway send paths, dry-run behavior, threshold jumps, and the config/session schema. If maintainers no longer want this as core behavior, reopen or create a narrower product decision issue rather than closing this PR as implemented.
