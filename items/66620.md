## Summary

Current `main` already contains the older Ollama simple-completion registration fix, but I did not find evidence that issue #66620's newer `lossless-claw` compaction-summary failure is fixed. The report remains plausible on current code because Ollama can still surface reasoning-only replies as `thinking` blocks while summary consumers commonly extract only `text` blocks, which matches the reported empty-normalized-output symptom.
