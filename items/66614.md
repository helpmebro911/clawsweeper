## Summary

Keep open. Current `main` still streams raw assistant chunks directly into `EmbeddedBlockChunker`, and that chunker still only treats fenced code blocks as protected spans. The two linked table-protection PRs have not landed on `main` or shipped in `v2026.4.23`, so the reported markdown-table split remains plausible and is not implemented on current OpenClaw.
