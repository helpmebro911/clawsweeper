## Summary

Keep this issue open. Current main still has only fence-aware block-stream chunking, no table-span parser or table-safe break guard, and the specific implementation candidate #66344 is still open and unmerged with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review/land #66344, or land a smaller equivalent fix. The fix should add markdown table-span detection in the shared markdown/chunker layer, gate all `EmbeddedBlockChunker` break paths on table-safe boundaries as well as fence-safe boundaries, and add focused regression tests for streaming text + table + text, partial streamed tables, fenced tables, and oversized-table fallback.
