## Summary

Current main still does not implement the requested durable early Discord commentary delivery. Commentary-phase assistant output is explicitly suppressed before partial, block, final, and message_end delivery paths, and regression tests assert that behavior. The related #55236 covered late main-result delivery and is closed, but it does not supersede this separate early-commentary loss window.

## Best Possible Solution

Keep this issue open for Discord/channel delivery work. The likely fix is to add a channel-safe materialization and flush path for approved user-visible commentary at assistant-message, tool, or restart boundaries, route it through the existing reply dispatcher/restart-deferral machinery, and add regression coverage. If commentary suppression is intentional product policy, document that explicitly and close only after that maintainer decision.
