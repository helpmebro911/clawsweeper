## Summary

Keep this issue open. Current main has adjacent Discord preview/tool-progress and reply-dispatch restart deferral machinery, but the specific requested behavior is still absent: assistant output marked as `commentary` is intentionally suppressed before block/partial delivery, so it cannot become a standalone durable Discord message before restart-sensitive work.

## Best Possible Solution

Keep this issue open for maintainer product/API work. Decide whether assistant commentary is meant to be user-visible; if yes, add a channel-safe materialization path for approved commentary at assistant-message, tool, or restart boundaries, register/flush it through the same reply dispatcher and restart-deferral model as normal replies, and cover Discord with regression tests. If suppression is intentional, document that policy and close only after explicit maintainer judgment.
