## Summary

Current main still explicitly suppresses commentary-phase assistant output before it can reach Discord-visible partial, block, final, or dispatcher delivery. The closed related report #55236 covered main-result delivery latency; this issue’s separate early-commentary commit/durability request remains unresolved.

## Best Possible Solution

Keep this issue open for Discord/channel delivery work. The likely fix is a scoped delivery contract that materializes approved user-visible status commentary before tool or restart boundaries, routes it through the existing Discord draft/dispatcher/restart-deferral machinery, and adds regression coverage for early commentary plus restart-sensitive turns. If maintainers decide commentary suppression is intentional product policy, document that decision explicitly and close only after that product call.
