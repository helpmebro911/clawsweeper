## Summary

Keep PR #72084 open. Current main still does not make the npm postinstall bundled-runtime-deps path reinstall a dependency whose sentinel package exists but has a stale or missing version, so the PR is not obsolete or implemented on main.

## Best Possible Solution

Keep the PR open and have the author or maintainer finish it by adding version-aware stale dependency detection to the package postinstall path, adding the stale-version regression test, fixing the fixture/review-comment issues, and aligning semver behavior with the existing runtime-deps repair code where appropriate.

## Evidence
