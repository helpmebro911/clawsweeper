## Summary

Keep open. Current main still lacks the stale bundled runtime dependency version check in the postinstall eager-install path, while this PR is an active implementation candidate linked to open regression #72058. A lockfile hunk from the PR now appears to be present on current main, but that is a rebase/review concern rather than a reason to close the PR.

## Best Possible Solution

Keep this PR, or a focused successor, open. The best path is to land a version-aware stale dependency check in scripts/postinstall-bundled-plugins.mjs that matches src/plugins/bundled-runtime-deps.ts, keep regression coverage for stale exact/range versions and optional native-child reinstall behavior, and rebase or split any lockfile-only drift. After that merges, #72058 can be closed as implemented if the npm update loop is covered.
