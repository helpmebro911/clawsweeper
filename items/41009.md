## Summary

Keep open. Current main still rejects Tailscale Serve/Funnel configs whose `gateway.bind` is non-loopback, and I could not find an implemented legacy migration that normalizes existing `lan`/`tailnet`/non-loopback `custom` configs to `loopback`. This PR remains a plausible implementation candidate, though it needs to be refreshed for the current migration file layout and reviewed for the `auto` bind semantics raised in review.

## Best Possible Solution

Keep this PR open and refresh it against current main. The likely path is to port the migration into `src/commands/doctor/shared/legacy-config-migrations.runtime.gateway.ts` before the non-loopback `allowedOrigins` seed, add a source-literal legacy rule for `gateway.bind=lan|tailnet|custom` with `gateway.tailscale.mode=serve|funnel`, and keep regression coverage in the current doctor/config test files. Then maintainer review can resolve the remaining `auto` bind semantics before landing.
