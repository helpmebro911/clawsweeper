## Summary

Current main does not implement the PR's ratio-based compaction budget fields. The config type, strict zod schema, runtime compaction settings, memory-flush gates, docs/help, and repository-wide search all show only the existing absolute-token fields. The PR still represents a missing core config/runtime capability, though its posted review comments identify issues that should be fixed before merge.

## Best Possible Solution

Keep the PR open for maintainer review or a revised patch. The best path is to add the four validated `*Share` fields across config types, zod schema, generated schema/help/labels, docs, and tests; resolve shares only when the active session model context window is authoritative; thread that value through Pi compaction and memory-flush consumption; and cover the preflight/no-memory-flush plus unknown-window fallback cases called out in review.
