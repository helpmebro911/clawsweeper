## Summary

Current main still has the plugin-sdk declaration boundary gap that #72108 fixes. The dts config includes all `src/plugin-sdk/**/*.ts` and excludes `*.test.ts`, but not `*.test-helpers.ts`, so `src/plugin-sdk/qa-runtime.test-helpers.ts` remains in the declaration input set. The PR is narrow, current, and still actionable.

## Best Possible Solution

Keep #72108 open and merge it, or independently apply the same one-line `tsconfig.plugin-sdk.dts.json` exclusion for `src/**/*.test-helpers.ts`. The fix belongs in the root plugin-sdk dts config because current main still includes a Vitest-only fixture in the public declaration input set.
