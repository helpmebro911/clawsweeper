## Summary

Keep #10467 open. Current main still does not expose a `lane` parameter on `sessions_spawn`, and both native and ACP spawn paths still dispatch through the fixed `subagent` lane. Configuration and docs continue to describe one global subagent concurrency cap, and tests assert that spawned work uses `subagent`.

## Best Possible Solution

Keep #10467 open until core adds an optional validated `sessions_spawn.lane` field, forwards it through both native and ACP spawn paths, defaults omitted lane to `subagent`, supports backward-compatible per-lane subagent concurrency configuration and hot reload, and updates docs plus regression tests for omitted lane, custom lanes, invalid lane names, and config fallback behavior.
