## Summary

Keep #10467 open. Current main still does not expose a `lane` parameter on `sessions_spawn`, and both native subagent and ACP spawn paths still dispatch through the fixed `subagent` lane. Gateway concurrency configuration remains a single scalar for `agents.defaults.subagents.maxConcurrent`, while docs and tests continue to describe/assert one global subagent lane. The issue discussion also includes a user +1 and a same-day maintainer-posted Codex review keeping this open.

## Best Possible Solution

Keep #10467 open until core adds an optional validated `sessions_spawn.lane` field, forwards it through both native and ACP spawn paths to the gateway agent call, supports backward-compatible per-lane subagent concurrency configuration with default fallback to `subagent`, and updates docs plus regression tests for omitted lane, custom lanes, invalid lane names, and config reload behavior.
