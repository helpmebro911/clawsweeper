## Summary

Keep open. The current main implementation still matches the reported regression: Telegram DM answer lanes force `previewTransport: "message"` before calling the lower-level draft stream, while the lower-level `auto` path would prefer draft transport for DM threads. Existing tests also assert the forced message behavior, so this is not implemented, not obsolete, and not a ClawHub/core-scope cleanup candidate. The issue comments correctly point to the upper reply/dispatch layer as the place overriding Telegram draft preference.
