## Summary

Keep open. Current main has meaningful partial fixes: normal external direct-chat failures now use generic copy, and group/channel chats are silenced for generic runner failures. The issue’s core “never expose raw LLM/API errors to users” policy is still not fully implemented because `/verbose on|full` for external direct chats intentionally forwards sanitized provider/runner details, and docs/tests preserve that behavior.

## Best Possible Solution

Keep this open for an explicit maintainer product/security decision on external-channel failure policy. A likely fix is to keep the current generic default and silent group/channel behavior, but stop forwarding provider/runner exception detail to external chat users even under `/verbose`, or restrict verbose details to operator/internal surfaces. Add regression coverage for WhatsApp/Telegram direct verbose failures and track multi-agent auto-escalation separately if it remains in scope.
