## Summary

#40078 should stay open. Current main has Discord audio preflight support, but it is a narrower spoken-mention detection path, not the requested silent transcription/history feature. The requested `preflightTranscribe` guild/channel option is not present in the strict Discord config schema, and unmentioned guild audio is still recorded into pending history from the placeholder text built before preflight transcription.

## Evidence

- **No requested config key in Discord channel/guild schema:** Discord channel overrides allow requireMention, ignoreOtherMentions, tools, skills, enabled, users, roles, systemPrompt, and thread options, then the schema is strict. Guild schema is also strict. There is no preflightTranscribe option at channel or guild level.

It transcribes the first audio attachment and returns the transcript to the caller.
