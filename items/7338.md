## Summary

Keep #7338 open. Current main still has provider-specific header/transport hooks and web_fetch User-Agent configuration, but not an opt-in OpenClaw-wide agent attestation surface for arbitrary external HTTP requests, skills, or tools. The provided discussion also makes #7338 the canonical tracker because #11137 was closed as its duplicate.

## Best Possible Solution

Keep #7338 open as the canonical product/API tracker. A complete solution needs maintainer security/product direction for an opt-in, domain-scoped attestation contract, then implementation across config/schema/docs and the relevant external HTTP paths with tests for privacy boundaries, header injection, and non-leakage to unallowed domains.
