## Summary

Keep open. Current main still lacks the PR's requested schema hint: when ACP spawning is available, `sessions_spawn.streamTo` is exposed as a bare optional enum with no description, while sibling `resumeSessionId` documents `runtime="acp"`. The executor and tests still prove the field is ACP-only by rejecting `streamTo` for `runtime="subagent"`, so the PR's model-facing documentation/test remains useful. The provided related PR #69203 is closed and unmerged, and the linked issue #69166 remains the same unresolved user problem.

## Best Possible Solution

Keep #69170 open unless a maintainer prefers a broader canonical PR. The best code path is a narrow schema change in `src/agents/tools/sessions-spawn-tool.ts` adding an explicit `streamTo` description that it requires `runtime="acp"`, plus a focused regression assertion in `src/agents/tools/sessions-spawn-tool.test.ts`. If a broader sessions_spawn fix lands first and includes the same schema hint/test, then this PR can be closed as superseded by that landed change.
