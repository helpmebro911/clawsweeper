## Summary

Keep open. Current main still contains the reported behavior: non-activating plugin registry paths validate `entry?.config` even when it is undefined, and the linked fix PR is still open/unmerged rather than shipped on main.

## Best Possible Solution

Land a focused fix in the plugin loader that defers schema validation only for non-activating metadata/snapshot loads when `entry?.config` is undefined, while preserving validation for real activation and for non-activating loads with explicit config. Keep or add regression coverage in `src/plugins/loader.test.ts` and `src/plugins/loader.cli-metadata.test.ts`, then let the merged PR close #70658.
