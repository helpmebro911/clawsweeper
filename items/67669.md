## Summary

Keep this PR open. Current main still does not implement `openclaw models custom-test`; the existing `models status --probe` surface is related but only probes selected provider/auth targets, not every configured custom provider model. The PR is focused and potentially useful, but it should not land as-is without redesign around the runtime probe path and the unresolved JSON, exit-code, coverage, and secret-redaction concerns.

## Best Possible Solution

Keep this PR open for maintainer review. If accepted, reshape it around the existing `models status --probe` runtime probing infrastructure or a shared runtime probe helper, cover all configured custom models including model-level API overrides and Responses API providers, preserve clean JSON and exit codes, redact sensitive endpoint data, and add focused tests plus docs for the new CLI behavior.
