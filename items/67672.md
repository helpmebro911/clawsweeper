## Summary

Keep open. Current main still does not await the Slack SocketModeClient disconnect before stopping the Bolt app, and the reported stale-socket restart leak remains a plausible Slack-owned cleanup bug. The PR is narrow, its earlier stopAccount concern was removed in discussion, and the best maintainer path is review plus a focused regression test rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow Slack-owned cleanup that awaits the underlying SocketModeClient.disconnect() before app.stop(), preserves the removal of the unnecessary gateway.stopAccount hook, and adds a small regression test for disconnect-before-stop/awaited cleanup so stale-socket restart cycles cannot accumulate old Slack WebSocket clients.
