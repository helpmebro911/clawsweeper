## Summary

Keep PR #71786 open. Current main already contains the non-mutating `toSorted()` part, but the PR still addresses a real, unshipped Pi TUI type-contract cleanup: current main's paged-select helper has a stale `render(): string` shim while the pinned Pi TUI component contract returns `string[]`, and the `/files` and `/diff` callers still pass object values where Pi TUI `SelectItem.value` is typed as `string`.

## Best Possible Solution

Keep the PR open for maintainer review or contributor revision. The best path is a narrow cleanup that uses the real Pi `ExtensionCommandContext`, preserves the `Component.render(): string[]` contract, maps object selections through string ids or paths for all paged-select callers including `/diff`, removes whitespace churn, and then runs the appropriate changed typecheck/test gate before merge.
