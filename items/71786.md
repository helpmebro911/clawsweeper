## Summary

Keep PR #71786 open. Current main already has the non-mutating `toSorted()` change, but the paged-select change is not safely implemented: the local helper type is wrong, while the pinned upstream Pi TUI `Component` contract returns `string[]`. The PR's join-to-string change would fight that dependency contract rather than fix the real type mismatch.

## Best Possible Solution

Keep the PR open for maintainer or contributor revision. The best fix is a narrow patch to `.pi/extensions/ui/paged-select.ts` that types `CustomUiContext.ui.custom` against the actual Pi component contract (`render(): string[]`, `done(result)` shape, ideally using imported upstream types where feasible), leaves `container.render(width)` as `string[]`, and drops the redundant sort/whitespace churn because `toSorted()` is already on main.
