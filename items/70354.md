## Summary

Keep #70354 open. Current main has the fresh-install/systemd-template mitigation (`Restart=always`), but the reported `Restart=on-failure` recovery gap is still present for existing or manual systemd units because supervised SIGUSR1 restarts still exit with code 0. The exact non-zero-exit fix is tracked in PR #70466, but that PR is open and unmerged, and a prior ClawSweeper closure of #53560 explicitly kept #70354 as the canonical remaining issue.

## Best Possible Solution

Keep #70354 open until PR #70466 lands or is replaced by a maintainer-approved fix. The best product path is to make Linux/systemd supervised SIGUSR1 restarts recover under `Restart=on-failure` without regressing launchd/schtasks, add a Linux-systemd-focused regression test, and separately decide whether update/doctor should repair older generated units to the current `Restart=always` template.
