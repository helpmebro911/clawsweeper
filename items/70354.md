## Summary

Keep #70354 open. Current main mitigates fresh/current Linux installs by rendering and documenting `Restart=always`, but the gateway run loop still exits `0` for supervised restart handoff, and service audit does not appear to flag or repair older `Restart=on-failure` systemd units. The linked PR #70466 tracks the narrower non-zero-exit fix but is still open/unmerged in the provided context.

## Best Possible Solution

Keep the issue open until #70466 or an equivalent main-branch fix lands. The best fix is to make supervised Linux/systemd restart handoff exit non-zero while preserving launchd behavior, add a run-loop regression test for the systemd-supervised path, and consider extending service audit/doctor to flag or repair existing `Restart=on-failure` gateway units.
