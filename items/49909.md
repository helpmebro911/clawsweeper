## Summary

Keep open. Current main still has the reported ClawDock behavior: `_clawdock_compose()` explicitly passes only `docker-compose.yml` and optional `docker-compose.extra.yml`, so a local `docker-compose.override.yml` is not included on `clawdock-start`. The issue comments identify the same missing conditional and ordering concern, and the current code has not implemented it.

## Evidence

- **ClawDock compose wrapper omits override file:** `_clawdock_compose()` initializes `compose_args` with `docker-compose.yml`, conditionally appends `docker-compose.extra.yml`, then invokes `docker compose` with that explicit file list. There is no check for `docker-compose.override.yml`.
  - file: [scripts/clawdock/clawdock-helpers.sh:151](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/scripts/clawdock/clawdock-helpers.sh#L151)
  - command: `nl -ba scripts/clawdock/clawdock-helpers.sh | sed -n '145,190p'`
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **Start path uses the affected wrapper:** `clawdock-start()` calls `_clawdock_compose up -d openclaw-gateway`, so the explicit compose file list is used when recreating/starting the service.
  - file: [scripts/clawdock/clawdock-helpers.sh:174](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/scripts/clawdock/clawdock-helpers.sh#L174)
  - command: `nl -ba scripts/clawdock/clawdock-helpers.sh | sed -n '145,190p'`
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **Local override file is an expected untracked user file:** The repo ignores `docker-compose.override.yml`, consistent with it being a user-local compose override file that can exist beside the tracked base compose file.
  - file: [.gitignore:4](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/.gitignore#L4)
  - command: `nl -ba .gitignore | sed -n '1,12p'`

- **Upstream Compose merge contract supports the report:** Docker's Compose merge docs state that Compose reads a base compose file plus an optional override file by default, and that multiple files can instead be specified with `-f` in order. Since ClawDock supplies an explicit `-f` list without the override file, the local override is outside the merge set.
