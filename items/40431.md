## Summary

Close PR #40431 because current `main` already fixes the reported zero-default poll misclassification, and the fix shipped in `v2026.4.22`.

## Risks / Open Questions

- `main` does not match this PR's exact `> 0` semantics for negative values; it intentionally keeps negative non-zero durations on the poll validation path. That is a semantic difference, but it still resolves the reported zero-default misrouting bug.
