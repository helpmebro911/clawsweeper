## Summary

Keep PR #72245 open. Current main still only caches provider-plugin arrays, still serializes providerRefs raw in provider hook cache keys, and still resolves hook plugins by calling runtime resolution plus scanning candidate hook providers. The related pricing hot path also remains present on main, so the PR is not obsolete or superseded by #71807.

## Best Possible Solution

Keep #72245 open for maintainer review. The best path is to review it as a narrow provider-runtime performance hardening change, validate the new cache normalization/reuse tests in a writable environment, and coordinate it with the related gateway pricing hot-path work rather than auto-closing it.
