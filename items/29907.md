## Summary

Keep #29907 open. Current main still only supports cron payloads for `systemEvent` and `agentTurn`; there is no `exec` or `lobster` cron payload, no condition/onCondition contract, and no non-LLM cron execution path. Lobster is present, but only as an optional agent tool, so using it from cron still requires an agent turn. Related #18160 tracks the direct shell-exec subset but does not cover Lobster pipelines, condition evaluation, silent no-op runs, or conditional handoff into `agentTurn`.

## Best Possible Solution

Keep #29907 open unless maintainers explicitly consolidate it into a broader canonical cron-execution issue. The best product path is an additive cron/protocol feature, or first a stable cron-executor plugin seam, covering deterministic `exec` and Lobster pipeline execution, timeout/security policy, bounded output capture, condition evaluation (`exitCode`, stdout, JSON path, always), silent no-op runs, conditional enqueue/delivery of an `agentTurn`, run-log/task-state visibility, CLI and Control UI support, public docs, and protocol/runtime tests.
