## Summary

Current `main` does not implement the requested cron `exec`/`lobster` kind or conditional handoff path. The cron contract still only supports `payload.kind` values `systemEvent` and `agentTurn`, and runtime validation enforces those session/payload pairings. Lobster exists today as an optional plugin tool and hooks can already trigger wake/agent actions from external callers, but there is no built-in cron job type that runs a deterministic exec/Lobster pipeline first and only starts an agent turn on condition.
