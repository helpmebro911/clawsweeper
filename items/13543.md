## Summary

Keep #13543 open. Current main still has session/agent-level sandbox placement plus sandbox-only tool availability filtering, not the requested per-tool placement policy that can force web_fetch/browser into sandbox while keeping exec/message/cron/gateway on host in the same session.

## Best Possible Solution

Keep this as a core sandbox/tool-routing enhancement. A good fix would add a documented, schema-backed per-tool execution placement policy, route selected tools such as web_fetch and browser through a sandbox backend while selected host tools such as exec, message, cron, and gateway remain on host in the same session, and add tests/docs that distinguish placement from tools.sandbox.tools availability filtering, tools.exec.host/elevated behavior, and browser target selection.
