## Summary

Keep #13543 open. Current main still does not implement per-tool sandbox execution routing. It supports session/agent-level sandbox modes, per-agent sandbox overrides, sandbox browser targeting only when the session already has sandbox context, and sandbox tool allow/deny filtering. Those do not satisfy selectively forcing `web_fetch` or `browser` into a sandbox while keeping `exec`, `message`, and `cron` on host in the same session.

## Best Possible Solution

Continue tracking this as a core sandbox/tool-routing enhancement. A good fix would add a documented, schema-backed per-tool execution policy in the sandbox/tool runtime path, with tests proving selected tools such as `web_fetch` or `browser` can be forced into a sandbox while other tools such as `exec`, `message`, and `cron` remain on host in the same session. The docs should preserve the distinction between sandbox placement and `tools.sandbox.tools` allow/deny filtering.
