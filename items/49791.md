## Summary

Keep open. Current main still has the observable silent-failure path for Feishu group directive-only commands when the sender is not command-authorized: Feishu normalizes and dispatches the command with CommandAuthorized=false, and the generic directive-only handler returns no reply for unauthorized senders. The reporter's group allowFrom workaround is valid, but the remaining requested UX fix is not implemented; #49821 was closed unmerged in the provided context, so it is not a canonical shipped fix.
