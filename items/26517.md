## Summary

Keep #26517 open. Current main at e672b614 still has backend support for labeled dashboard sessions and a switch-only chat session selector, but the visible WebChat new-session control still sends `/new`, which resets the current session rather than creating an isolated labeled session. The proposed agent-facing create/relabel API is also not shipped, and the related implementation PRs #52042 and #52746 are closed unmerged while #51793 was closed as duplicate/superseded in favor of this issue.

## Best Possible Solution

Keep #26517 open until maintainers either ship or explicitly narrow the WebChat product scope. The likely implementation should reuse existing `sessions.create` and `sessions.patch` backend support, add chat-surface create/switch/label management for isolated labeled sessions, and handle any agent-controlled create/relabel semantics only after maintainer review of the product and safety boundary.
