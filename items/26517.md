## Summary

Keep #26517 open. Current main still has useful backend groundwork for labeled dashboard sessions and a chat session selector, but the visible WebChat new-session control still routes to /new, the selector only switches existing sessions, and the proposed agent-facing create/relabel API is not shipped. The related implementation PRs #52042 and #52746 are closed unmerged, while #51793 was closed as duplicate/superseded in favor of this issue, making #26517 the canonical remaining tracker.

## Best Possible Solution

Keep #26517 open until maintainers either ship or explicitly narrow the WebChat product scope. The likely implementation should reuse the existing sessions.create and sessions.patch backend support, add chat-surface create/switch/label management for isolated labeled sessions, and handle any agent-controlled create/relabel semantics only after maintainer review of the product and safety boundary.
