## Summary

Keep this PR open. The checked-out main still lacks the bundled LSP process-tree cleanup and Gateway shutdown sweep the PR is adding, while the linked bug #72357 has a concrete reproduction path. The latest provided PR context is scoped to the relevant LSP runtime and Gateway close files, includes regression tests, and the second PR commit appears to address the earlier shutdown-latency review by running bundle runtime cleanup concurrently.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for #72357. The best path is to land a focused change that spawns bundled LSP stdio servers in disposable process groups where appropriate, uses `killProcessTree` for forced cleanup, tracks active bundled LSP sessions for Gateway shutdown, disposes MCP and LSP shutdown sweeps without doubling worst-case latency, and keeps regression coverage in the bundled LSP runtime and Gateway close tests.
