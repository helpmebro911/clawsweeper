## Summary

Keep this PR open. Current main still lacks bundled-LSP process-tree cleanup and Gateway shutdown sweeping for active bundled LSP runtimes, while the latest PR head is a focused implementation candidate for the concrete linked bug #72357.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land this focused lifecycle fix, or an equivalent patch, after CI passes: spawn bundled LSP stdio servers in disposable process groups, use killProcessTree for still-running servers, register a Gateway shutdown sweep for active bundled-LSP runtimes, keep MCP/LSP disposal concurrent, and retain the added runtime and server-close regression tests for #72357.
