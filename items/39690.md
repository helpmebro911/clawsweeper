## Summary

Keep open. Current main still has the direct bug reported in #39690: explicit user-configured model `input` modalities are overwritten by implicit/catalog metadata during provider model merge. The linked PR #39785 tracks the exact remaining fix but is open/unmerged in the provided GitHub context. Joel-Claw's later discussion and #67634 are useful for the related text-only primary-model fallback path, but that does not make custom vision-capable models receive native image blocks when their explicit `input: ["text", "image"]` config is merged away.

## Evidence

This matches the reported root cause for custom Ollama/OpenAI-compatible models losing user-declared image input.
