## Summary

Keep open. Current main has improved the node-backed skill install path, but the reported macOS 26/Homebrew CLT failure path still remains plausible: the macOS Skills UI uses `skills.install`, install-option selection still prefers Homebrew by default when `brew` exists, brew-only bundled skills are still common, and the installer does not classify or remediate the CLT unsupported error.

## Best Possible Solution

Keep this issue open and fix the macOS skills installer path: detect Homebrew/CLT unsupported macOS errors, return actionable remediation for installing/updating compatible Xcode/CLT, avoid selecting brew when the host brew toolchain is known unusable, fall back to uv/node/go/download when the skill metadata provides a viable alternative, and add focused tests for the error classifier and installer-selection behavior.
