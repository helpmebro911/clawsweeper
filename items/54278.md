## Summary

Keep this issue open. Current main has partial Codex OAuth and WHAM usage support, including refresh_token_reused recovery paths and Codex usage windows, but it still does not expose the requested native diagnostic contract, quota_wait versus reauth_required state, Control UI quota-wait guidance/action, or a brokered host-side reauth execution API. The earlier ClawSweeper keep-open comment remains consistent with current main at 646a268d2710bc54e478349ef5690f3c8803df4f.

## Best Possible Solution

Keep this open as a core/provider plus Control UI product issue. Add an additive diagnostic contract on the existing models.authStatus/provider-usage path with states such as ok, expiring_soon, quota_wait, and reauth_required; preserve the current WHAM-aware cooldown behavior; display Codex 5-hour and weekly renewal details in local time; and design a narrow authorized broker/adapter execution API that is only callable when diagnosis is reauth_required.
