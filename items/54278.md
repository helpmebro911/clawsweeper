## Summary

Keep this issue open. Current main has useful partial pieces for Codex OAuth reliability, including generic model auth health, Codex WHAM usage windows, automatic OAuth refresh support, WHAM-aware cooldowns, and doctor guidance for permanent refresh failures. It still does not implement the requested product shape: explicit native quota/auth diagnostic states, Control UI quota-wait vs reauth guidance/actions, or brokered host-side reauth execution.

## Best Possible Solution

Keep this open as a core/provider plus Control UI product issue. Extend the existing models.authStatus/provider-usage path with an explicit Codex diagnostic object such as ok, expiring_soon, quota_wait, and reauth_required; preserve the existing WHAM cooldown behavior; surface 5-hour and weekly renewal details in local time in Control UI; and add a narrow, authorized broker/adapter execution API only for reauth_required. Broader reason-aware guardrails can stay tracked separately in #14376, but that issue does not replace this UI/auth recovery request.
