## Summary

Keep this issue open. Current main has useful partial Codex auth and WHAM usage support, including plugin-owned OAuth refresh, usage-window parsing, status-card quota hints, and doctor guidance for permanent refresh failures. It still does not implement the requested native diagnostic state machine (`quota_wait` versus `reauth_required`), a Control UI reauth action tied to that diagnosis, or a bounded host-side reauth broker/adapter API.

## Best Possible Solution

Keep this open as a core/provider plus Control UI product issue. Add an additive diagnostic contract on the existing `models.authStatus`/provider-usage path with states such as `ok`, `expiring_soon`, `quota_wait`, and `reauth_required`; preserve the existing Codex WHAM-aware cooldown behavior; display Codex 5-hour and weekly renewal details in local time; and design a narrow authorized broker/adapter execution API that can only run when diagnosis is `reauth_required`. Include focused tests for 401/403 refresh failure, 429/quota-window behavior, Control UI rendering/actions, and the broker authorization boundary.
