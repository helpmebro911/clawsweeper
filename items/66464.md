## Summary

Close #66464 as a duplicate of the already-open #65405. Current origin/main still has the same remaining Dashboard model-dropdown gap: the Control UI asks `models.list` without any configured-provider filter, and the gateway only narrows the list from `agents.defaults.models`; otherwise it returns the full catalog. The narrower `models.mode: "replace"` form is also tracked by #48483 and PR #48535.

## Best Possible Solution

Close #66464 as a duplicate of #65405. Keep the canonical issue and related `models.mode: "replace"` issue/PR active; the eventual fix should add a configured-provider-aware `models.list` mode or UI filter, preserve the existing `agents.defaults.models` allowlist contract, and add regression coverage for `models.providers`-only and replace-mode configs.
