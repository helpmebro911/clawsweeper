## Summary

Keep this PR open. Current main still lacks Slack `allowBots: "mentions"` support, while Discord already has the tri-state behavior this PR is trying to mirror. The PR is stale because Slack moved from the old `src/slack` layout to `extensions/slack`, but it remains a focused implementation candidate rather than cleanup closure.

## Best Possible Solution

Keep this PR open and ask the contributor to port it to the current Slack plugin layout. The implementation should update `extensions/slack/src/monitor/message-handler/prepare.ts`, `extensions/slack/src/monitor/channel-config.ts`, `src/config/types.slack.ts`, `src/config/zod-schema.providers-core.ts`, Slack UI/generated metadata and docs, plus focused Slack tests for no mention, explicit mention, DM, and implicit thread participation. Maintainers should then run the Slack extension changed lane or `pnpm check:changed` before merge.
