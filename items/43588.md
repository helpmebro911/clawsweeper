## Summary

Keep this PR open. Current main still lacks Slack `allowBots: "mentions"` support in the Slack plugin runtime, public config types, strict schema, UI/help metadata, and generated channel metadata. The PR is stale against the old `src/slack` layout, but it remains a focused candidate for a real Slack parity gap rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open and ask the contributor to rebase/port it to the current Slack plugin layout: update `extensions/slack/src/monitor/message-handler/prepare.ts`, `extensions/slack/src/monitor/channel-config.ts`, `src/config/types.slack.ts`, `src/config/zod-schema.providers-core.ts`, Slack UI/help metadata, generated bundled config metadata, and focused tests for no mention, explicit mention, DM, and implicit thread participation with cache cleanup. After rebase, run the Slack extension changed lane or repository changed-check gate.
