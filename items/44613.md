## Summary

Current OpenClaw already supports the requested cross-channel session sharing on main and in released v2026.4.22: direct chats default to the agent main session, WebChat defaults to that same main session, and Telegram DM coverage asserts `agent:main:main` for private chats without a thread. The issue’s reported `agent:main:telegram:direct:<id>` key matches the opt-in isolated DM scopes, not the default behavior.

## Close Comment

Closing from a Codex review: current OpenClaw already implements this well enough. On current main, direct-message routing defaults to `session.dmScope: "main"`, so DMs collapse into the agent main session instead of staying channel-specific (`src/routing/session-key.ts:143`, `docs/concepts/session.md:24`, `docs/channels/channel-routing.md:26`). WebChat also defaults to that same main session, and the docs call out that this is how you see cross-channel context in one place (`docs/channels/channel-routing.md:132`). Telegram coverage on main asserts that a private chat without a thread uses `agent:main:main` rather than `agent:main:telegram:direct:<id>` (`extensions/telegram/src/bot-message-context.dm-threads.test.ts:82`). If you intentionally use an isolated DM scope, the supported cross-channel collapse path is `session.identityLinks` (`docs/gateway/config-agents.md:1156`). I also verified the same behavior is present in the released `v2026.4.22` tag.
