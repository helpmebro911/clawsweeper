## Summary

Keep open. Current `main` does not implement Anthropic native server-side tools: the Anthropic transport still converts OpenClaw tools into standard `{name, description, input_schema}` entries, there is no Anthropic config surface for enabling hosted tools, and the streaming/parser code has no handling for Anthropic-specific result block types like `server_tool_use` or `web_search_tool_result`.

## Evidence

- **Anthropic tool payloads are still generic function-style tools:** `convertAnthropicTools` rewrites each tool to `{ name, description, input_schema }`, and `buildAnthropicParams` uses that output for `params.tools`. That leaves no path for Anthropic-native hosted tool descriptors such as `web_search_20260209` or `code_execution_20260120`.
  - file: [src/agents/anthropic-transport-stream.ts:394](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/agents/anthropic-transport-stream.ts#L394)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Anthropic stream parsing only handles current standard block types:** In `content_block_start`, the transport recognizes `text`, `thinking`, `redacted_thinking`, and `tool_use`. There is no branch for Anthropic hosted-tool block types like `server_tool_use` or hosted search result blocks, so the issue's requested response handling is not present on `main`.
  - file: [src/agents/anthropic-transport-stream.ts:808](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/agents/anthropic-transport-stream.ts#L808)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Anthropic plugin exposes no config schema for enabling hosted tools:** The bundled Anthropic plugin's `configSchema.properties` is empty, so there is no current plugin-owned config surface for options like `tools.serverTools` or equivalent Anthropic hosted-tool toggles.
  - file: [extensions/anthropic/openclaw.plugin.json:66](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/extensions/anthropic/openclaw.plugin.json#L66)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Native hosted web search is only documented for OpenAI in the shared web docs:** `docs/tools/web.md` documents a built-in `Native OpenAI web search` path, but there is no corresponding Anthropic-native hosted web-search/code-execution section in current docs. That matches the code: native hosted search support exists for other providers, not Anthropic.
  - file: [docs/tools/web.md:117](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/docs/tools/web.md#L117)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Repo-wide search found no Anthropic hosted-tool identifiers or response types:** A repo-wide search for `server_tool_use|web_search_tool_result|allowed_callers|code_execution_20260120|web_search_20260209|web_fetch_20260209` returned no matches in `extensions/anthropic`, `src`, `docs`, or `test` on current `main`.
