## Summary

Keep this PR open. Current main still has the CLI-backend gap this PR targets: cron and subagent session identity reaches `runCliAgent`/`prepareCliRunContext`, but the CLI system prompt builder does not accept `sessionKey` or pass a derived minimal `promptMode`, so `buildAgentSystemPrompt` defaults to full mode and can include heartbeat guidance. The related issue was closed using embedded-runner evidence, but that does not cover the CLI-backend path this PR changes.

## Best Possible Solution

Keep this PR open. Rebase it onto current main, centralize or expose `resolvePromptModeForSession` from an owner-neutral helper, pass `params.sessionKey` through CLI prompt construction, pass the derived `promptMode` into `buildAgentSystemPrompt`, and add CLI-specific regression tests for no session key, regular agent session, cron run session, and subagent session before running the targeted agent/CLI checks.
