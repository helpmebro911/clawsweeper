## Summary

Keep #14629 open. Current main still has the reported sanitizer gap: final tags are stripped textually, duplicate collapse only handles exact adjacent blank-line paragraph blocks, and checked-in tests still do not cover same-line, single-newline, tag-boundary, or partial-prefix duplicate cleanup. A maintainer-side Codex review comment on the issue also explicitly kept it open for this same path.

## Best Possible Solution

Keep #14629 open. Implement a narrow sanitizer change in src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts for same-line and single-newline repeats, the reported </final> boundary duplicate shape, and conservative partial-prefix duplicates. Add focused tests in src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts that prove the reported cleanup while preserving legitimate repeated wording and normal single-newline formatting.
