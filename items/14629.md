## Summary

Current main still has the sanitizer gap reported in #14629. The implementation strips `<final>` tags as plain text and only collapses adjacent duplicate blocks split by blank lines, while tests cover paragraph-only dedupe and explicitly preserve single-newline content. The April 26 maintainer/Codex issue comment also keeps this open, and the provided related items do not supersede this sanitizer-specific request.

## Best Possible Solution

Keep #14629 open. The best path is a narrow change in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts` that conservatively handles same-line/single-newline repeats, the reported `</final>` boundary duplicate shape, and safe partial-prefix duplicates, with focused tests in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts` that preserve legitimate repeated text and ordinary single-newline formatting.
