## Summary

Keep #70055 open. Current main has removed the specific noisy `info`-level sync path described in the issue and now treats external CLI credentials as runtime-only overlays, but it still has no user-facing config, env var, or CLI flag to disable external CLI auth overlay resolution. The requested opt-out remains a real core auth/config feature request, not a ClawHub/plugin boundary issue.

## Best Possible Solution

Keep the issue open and implement a small core auth/config feature: add a documented `auth.externalCliSync` setting or env equivalent, validate it in the config schema/help/docs, and plumb the resolved policy into `external-auth`/`external-cli-sync` so Codex, Claude CLI, and MiniMax external auth overlays can be disabled globally or by provider. Add focused tests proving disabled sync does not read external CLI auth stores, does not overlay runtime profiles, and does not write `auth-profiles.json`.
