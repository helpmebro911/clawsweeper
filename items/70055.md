## Summary

Keep open. Current `main` does not expose any config/env opt-out for external CLI auth-profile bootstrap, so the requested feature is not implemented. The report's original symptom is partly outdated, though: current code overlays external CLI credentials at runtime without persisting them, and the old `info`-level sync logging described in the issue is not present in `main`.
