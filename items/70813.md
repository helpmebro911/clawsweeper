## Summary

Keep this PR open. Current main still does not implement WhatsApp edit/unsend: discovery advertises only react/poll, the WhatsApp channel supports only react execution, the runtime rejects non-react actions, and the Web listener/send API has no edit or unsend methods. The PR remains a plausible WhatsApp-plugin-owned implementation candidate because the shared message action name contract already includes edit and unsend.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land or revise a WhatsApp-plugin-owned implementation that advertises edit/unsend only when message sends are enabled, dispatches both actions through the existing gateway action surface, reuses outbound target authorization, constructs own-message mutation keys with fromMe: true and no caller-controlled participant, and keeps focused unit/seam tests plus user-facing WhatsApp docs for the new actions and gates.
