## Summary

Keep PR #70813 open. Current main still does not implement WhatsApp edit or unsend actions: WhatsApp action discovery advertises only react/poll, dispatch only supports react, and the active web listener/send API expose sendMessage, sendPoll, sendReaction, and typing only. The PR is a focused implementation with relevant tests and follow-up addressing bot feedback, so it remains useful maintainer-review work rather than a cleanup close.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a focused WhatsApp plugin change that wires `edit` and `unsend` through discovery, gateway action dispatch, Baileys send payloads, listener contracts, and targeted WhatsApp tests, while preserving bot-owned-message scoping and existing target authorization.
