## Summary

Keep #70813 open. Current main does not implement WhatsApp edit/unsend actions: the WhatsApp plugin only supports `react`, discovery only advertises `react`/`poll`, the runtime rejects non-react actions, and the Web send API has no edit/delete payload helpers. The PR is recent, contributor-authored, not protected by maintainer labels, and its head has green check-runs with targeted tests described in the PR body, so it remains a plausible implementation rather than a cleanup close.

## Best Possible Solution

Keep #70813 open and continue normal maintainer review. The best path is a plugin-owned WhatsApp implementation or this PR after review: advertise `edit`/`unsend` only when the send-message gate allows it, dispatch through the WhatsApp action runtime, use Baileys own-message edit/delete keys with `fromMe: true`, preserve existing target authorization, and keep the added unit/seam tests covering discovery, gateway dispatch, payloads, gating, and invalid input classification.
