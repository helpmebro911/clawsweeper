## Summary

Keep #11055 open. Current main has silent pre-compaction memory flush and optional user-channel compaction notices, but it still does not implement `memoryFlush.visibleWarning`, `warningPrompt`, or an agent-visible pre-compaction warning turn. The related #14790 context remains non-canonical for this request because it was closed unmerged and targeted user-facing channel notices rather than the requested agent-facing warning.

## Best Possible Solution

Keep #11055 open until maintainers either implement or explicitly reject an agent-visible pre-compaction warning. A concrete path would add validated `agents.defaults.compaction.memoryFlush.visibleWarning` and `warningPrompt` config, document it separately from `notifyUser`, inject a visible agent-facing warning before auto-compaction at the soft threshold, preserve the existing silent memory flush, and add config/runtime/docs tests for the distinction. If maintainers prefer plugin ownership, the missing core step is a supported plugin/runtime API for safe pre-threshold agent-visible injection.
