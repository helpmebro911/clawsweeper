## Summary

Keep #11055 open. Current main still has silent pre-compaction memory flush and optional user-channel compaction notices, but no `memoryFlush.visibleWarning` / `warningPrompt` config and no agent-visible pre-compaction warning turn. The existing issue discussion also already distinguishes this from #14790, which was closed unmerged and targeted user-channel notices rather than the requested agent-facing warning.

## Best Possible Solution

Keep #11055 open until maintainers either implement or explicitly reject an agent-visible pre-compaction warning. A concrete implementation would add validated `agents.defaults.compaction.memoryFlush.visibleWarning` and `warningPrompt` config, document it separately from `notifyUser`, inject a visible agent-facing warning before auto-compaction at the soft threshold, preserve the existing silent memory flush, and add config/runtime/docs tests for the distinction. This should stay in core unless a plugin API is first added that can safely inject this exact pre-threshold agent-visible warning; current compaction hooks and memory flush plan APIs do not expose that behavior directly.
