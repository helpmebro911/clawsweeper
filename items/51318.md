## Summary

Keep open. Current `main` does not implement a per-invocation `--skip-bootstrap` flag for `openclaw agents add`, and the open PR still has a credible CLI regression called out in review.

## Evidence

- **CLI flag is absent on main:** `agents add` currently exposes `--workspace`, `--model`, `--agent-dir`, `--bind`, `--non-interactive`, and `--json`; there is no `--skip-bootstrap` option or forwarding in the action payload.
  - file: [src/cli/program/register.agent.ts:175](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/cli/program/register.agent.ts#L175)
  - command: `nl -ba src/cli/program/register.agent.ts | sed -n '170,215p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Command only honors config default:** `AgentsAddOptions` has no `skipBootstrap` field, and both `ensureWorkspaceAndSessions(...)` calls use only `config.agents.defaults.skipBootstrap`. That means current main supports only the global config path, not a per-call override.
  - file: [src/commands/agents.commands.add.ts:36](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/commands/agents.commands.add.ts#L36)
  - command: `nl -ba src/commands/agents.commands.add.ts | sed -n '30,155p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Interactive path also lacks per-call override:** The wizard path still passes only `Boolean(nextConfig.agents?.defaults?.skipBootstrap)` into workspace bootstrap setup.
  - file: [src/commands/agents.commands.add.ts:368](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/commands/agents.commands.add.ts#L368)
  - command: `nl -ba src/commands/agents.commands.add.ts | sed -n '360,378p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Docs match current main, not the PR feature:** CLI docs for `agents add` list no `--skip-bootstrap` flag and state that explicit add flags force the non-interactive path.
  - file: [docs/cli/agents.md:100](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/cli/agents.md#L100)
  - command: `nl -ba docs/cli/agents.md | sed -n '100,118p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Open PR has unresolved behavioral defect:** The PR review on commit `d71aea66017dcbfc1755d7565e657c8eef7d94d5` points out that adding `skipBootstrap` to the `hasExplicitOptions` watchlist would make `openclaw agents add <name> --skip-bootstrap` silently switch to non-interactive mode and require `--workspace`, blocking the intended interactive-wizard use case.
  - file: [src/cli/program/register.agent.ts](https://github.com/openclaw/openclaw/blob/d71aea66017dcbfc1755d7565e657c8eef7d94d5/src/cli/program/register.agent.ts)
  - sha: [d71aea66017d](https://github.com/openclaw/openclaw/commit/d71aea66017dcbfc1755d7565e657c8eef7d94d5)

## Risks / Open Questions

- Closing this PR would drop a still-unimplemented user-facing feature: current main only offers the global `agents.defaults.skipBootstrap` setting.
- Merging the PR as described in review would likely regress the CLI by making `--skip-bootstrap` alone force non-interactive mode.
