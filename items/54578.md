## Summary

Keep this issue open. Current main still exposes flat `sessions`, `tasks`, `agents list`, `sessions_list`, and `/subagents list` surfaces, but no `openclaw tree`, `openclaw sessions tree`, or `openclaw agents tree` command that renders active agents and subagents as a hierarchy. The existing metadata is enough to build part of the feature, but the requested user-facing tree command is not implemented or superseded.

## Best Possible Solution

Keep this open for maintainer product judgment. A conservative implementation should add a core inspection command such as `openclaw sessions tree` or `openclaw agents tree`, build hierarchy from existing `spawnedBy`, `parentSessionKey`, `childSessions`, and task/subagent records, support text plus JSON output first, and defer `--watch` plus per-process resource accounting until the data contract is designed.
