## Summary

Keep this issue open. Current main still does not provide an `openclaw tree`, `agents tree`, or `sessions tree` command for active-agent hierarchy visualization. The adjacent surfaces are flat CLI/API views: configured agents, stored sessions, background tasks, and current-session subagent lists. Relationship metadata exists, so this is feasible core work, but the requested text tree, JSON tree, watch mode, and per-agent resource view remain unimplemented.

## Best Possible Solution

Keep this open for maintainer product judgment. A conservative implementation would add a core inspection surface such as `openclaw sessions tree` or `openclaw agents tree`, build the hierarchy from existing `spawnedBy`, `parentSessionKey`, `childSessions`, status, runtime, and task metadata, and add focused tests for text tree and JSON output. Treat `--watch` and `--resources` as follow-on extensions unless maintainers want them in the first slice.
