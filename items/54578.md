## Summary

Keep this issue open. Current main has useful relationship/status metadata for sessions and subagents, but still lacks a user-facing CLI/TUI command that renders active agents and subagents as a hierarchy with text/JSON/watch/session/resource modes.

## Best Possible Solution

Keep this open for a core inspection feature. A conservative implementation would add a sessions tree or agents tree CLI command that reuses existing session/subagent metadata (spawnedBy, parentSessionKey, childSessions, status, runtime, tokens/cost), supports text and --json first, and defers --watch plus CPU/RSS resource accounting until the data contract is explicit.

## Risks / Open Questions

- The proposed --resources option likely needs a separate data contract; current rows expose runtime, tokens, and estimated cost, but not reliable per-agent CPU/RSS ownership.
- Command naming still needs maintainer product judgment: openclaw tree, openclaw sessions tree, and openclaw agents tree are all plausible, and the implementation should avoid competing with existing status/session surfaces.
