## Summary

Keep #6615 open. Current main still lacks first-class exec-approval denylist support: no `security: "denylist"`, no `ask: "on-match"`, no denylist rule storage, and no denylist match branch in `system.run` policy. The discussion links useful prior art, including SafeClaw and CRE/claude-rule-enforcer, and there were two closed unmerged PR attempts, but neither the plugin prior art nor the closed PRs ships the requested core exec-approvals behavior.

## Best Possible Solution

Keep #6615 open until OpenClaw either implements first-class exec-approval denylist support across config/types/schema, approvals normalization/storage, gateway/node/system.run policy, CLI/Control UI, docs, and tests, or maintainers explicitly redirect the feature to ClawHub/plugin policy and track any missing authorization hooks separately.
