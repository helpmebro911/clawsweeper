## Summary

Keep #6615 open. Current main at 4a3030df9efa72c44bab567a4390ac0c4876a73f still does not implement first-class exec-approval denylist mode, denylist rule storage, `ask: "on-match"`, or a denylist match branch in gateway/node/system.run policy. The issue discussion links useful external prior art, including SafeClaw and CRE/claude-rule-enforcer, but those do not ship the requested OpenClaw core exec-approvals behavior, and an existing maintainer/Codex review comment explicitly kept the issue open.

## Best Possible Solution

Keep #6615 open until OpenClaw either implements first-class exec-approval denylist support across config/types/schema, approvals file normalization, gateway/node/system.run policy, CLI/Control UI, docs, and tests, or maintainers explicitly redirect the feature to ClawHub/plugin policy and track any missing authorization hooks separately.
