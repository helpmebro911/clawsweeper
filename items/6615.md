## Summary

Keep #6615 open. Current main still lacks the requested exec-approval denylist mode: core types and normalization accept only `deny`, `allowlist`, and `full`; prompt behavior has no `on-match`; docs and UI expose only allowlist-centric editing. The linked CRE/SafeClaw-style plugin workaround is useful prior art, but the issue asks for core exec-approval policy behavior and the linked implementation PRs were closed unmerged rather than shipped.

## Best Possible Solution

Keep the issue open until maintainers either implement denylist/on-match support in the core exec-approval policy stack across runtime, CLI/session validation, docs, and macOS/web approval UI, or explicitly decide that this should be redirected to a plugin/ClawHub policy-engine path with any missing core extension API tracked separately.
