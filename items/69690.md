## Summary

Keep open. Current main still lacks the PR’s central fix: plugin load cache keys include sorted core gateway method names, but active runtime registry state does not retain those method names, so a gateway-bindable helper/runtime load that omits `coreGatewayHandlers` can still miss the active gateway-startup registry and fall through to a fresh load.

## Best Possible Solution

Keep #69690 open. The best path is to land or review a scoped fix that records the sorted gateway method names alongside the active registry, uses them only to reconstruct a compatible gateway-bindable cache key for callers that opted into gateway binding, and keeps regression coverage for omitted handlers, differing method sets, and default-mode loads that must remain unbound.

## Evidence
