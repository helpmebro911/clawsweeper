## Summary

Keep this PR open, but require a revision. Current main still has the Vertex ADC marker and still returns no reply for empty final payloads, so the work is not obsolete. The submitted patch is not safe as-is because it would replace the existing Google thinking stream wrapper and would turn several intentional empty-payload success paths into user-facing failures.

## Best Possible Solution

Revise this PR rather than close it. The Google fix should compose a narrow google-vertex ADC marker-stripping wrapper with the existing Google thinking wrapper, preferably through the existing composeProviderStreamWrappers seam, and add regression tests for Vertex ADC, real API keys, and non-Vertex thinking behavior. The empty-payload fix should only surface a friendly retry/context error when a non-silent run truly produced no delivered output and no renderable final payload, while preserving block streaming, direct block-key dedupe, same-target messaging-tool suppression, and silentExpected turns.
