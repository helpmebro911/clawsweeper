## Summary

Keep #14206 open. Current main still does not expose a Feishu outbound minimum-interval/rate-limit setting, and the main Feishu outbound paths still send chunked replies, comment replies, message-tool text/card sends, and streaming card operations directly without a shared per-account pacing layer. The related #13615 is broader external API throttling work and does not supersede this Feishu-specific channel request.

## Best Possible Solution

Keep the issue open. Implement Feishu outbound pacing in the bundled Feishu plugin: add documented top-level and per-account minimum-interval/rate-limit config, publish it through schema, manifest, generated metadata, and docs, route reply chunks, comment replies, message-tool text/card/media sends, pairing/bot notifications, fallbacks, and streaming card send/update/close paths through one per-account pacing helper, and cover the behavior with fake-timer tests. If #13615 later provides a suitable generic limiter contract, use it as the implementation mechanism but still close #14206 only once Feishu exposes and applies the setting.
