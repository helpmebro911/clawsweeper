## Summary

Keep #14206 open. Current main still does not provide a Feishu outbound minimum-interval/rate-limit setting, and Feishu reply, comment, outbound adapter, media, and streaming-card paths still dispatch messages directly without a shared per-account pacing layer. The existing #13615 is broader external API throttling work and does not supersede this Feishu-specific channel request.

## Best Possible Solution

Keep the issue open. Implement this in the Feishu plugin surface: add documented top-level and per-account outbound minimum-interval/rate-limit config, expose it through schema, manifest, generated metadata, and docs, route reply chunks, comment replies, message-tool text/card/media sends, text-plus-media sends, fallback sends, and streaming final/card sends through one per-account pacing helper, and cover the behavior with fake-timer tests. Coordinate with #13615 only if it provides a suitable generic limiter contract; do not treat the broader issue as a replacement for Feishu-specific coverage.
