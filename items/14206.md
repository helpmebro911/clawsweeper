## Summary

Keep #14206 open. Current main has related Feishu mitigations such as streaming-card update throttling, typing-indicator backoff handling, inbound webhook rate limiting, and quota-reduction flags, but it still does not provide a Feishu outbound minimum-interval/rate-limit setting or a shared pacing hook between outbound message sends.

## Best Possible Solution

Keep the issue open and implement this in the Feishu plugin surface: add documented top-level and per-account outbound minimum-interval/rate-limit config, apply it through a shared per-account outbound pacing helper for reply chunks, comment replies, message-tool text/card sends, text-plus-media sequences, and fallback sends, then add fake-timer tests plus config schema, manifest, generated config metadata, and docs coverage. Treat #13615 as broader external API rate-limiting work, not a superseding canonical issue for this Feishu-specific channel behavior.
