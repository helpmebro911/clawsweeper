## Summary

Keep open. This is a recent broad Windows reply-latency regression report after 2026.4.12 with an Active Memory config snippet showing the feature disabled. Current main has several related mitigations for Active Memory gating, hard timeouts, plugin startup, and QMD startup scope, but I did not find a source-backed fix that clearly resolves the reported overall AGENT response slowdown on Windows.

## Best Possible Solution

Keep this issue open as a Windows reply-latency investigation. Ask the reporter to retest on the latest release/main and attach one slow-turn trace with Windows version, OpenClaw version, channel/UI, model/provider, `plugins.entries.active-memory.enabled` and `plugins.entries.active-memory.config.enabled`, `/status` output, gateway logs with hook/model timing, and whether logs show `sessions.json.lock`, Active Memory, QMD startup, or provider timeout signatures. If the trace matches a narrower open tracker such as #70857, #66792, #69182, or #72015, maintainers can then close or retitle this as duplicate with high confidence.
