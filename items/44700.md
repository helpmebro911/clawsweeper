## Summary

Keep open. Current `main` does not implement the requested installer optimization: `scripts/install.sh` still treats any existing `openclaw` binary as an upgrade, always proceeds into the npm install path, and only resolves the installed version after installation for the success message. I found no pre-install local-vs-target version check, no `install.sh --force` flag, and no docs/tests covering an `already up to date` early exit.

## Evidence

- **Existing install detection is presence-only:** `check_existing_openclaw()` only checks whether `openclaw` exists on PATH and prints an upgrade message; it does not compare installed and target versions.
  - file: [scripts/install.sh:1613](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/scripts/install.sh#L1613)

  - file: [scripts/install.sh:1991](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/scripts/install.sh#L1991)
