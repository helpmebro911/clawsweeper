## Summary

Keep this issue open. Current main has useful mitigations, but it does not yet implement the browser-specific, task-aware context budgeting requested here: default browser snapshots can still return broad page state, efficient snapshots are opt-in/config-driven, and generic tool-result guards only truncate near large single-result or total-context thresholds.

## Best Possible Solution

Keep the issue open and implement a browser-owned context-budget policy: make agent-facing snapshots default to compact/interactive bounded output when appropriate, dynamically lower browser snapshot budgets as context pressure grows, support selector/ref/detail fetch-on-demand so full page data can stay outside the main model context, and add focused tests around `browser-tool.actions`, `agent.snapshot.plan`, route behavior, and repeated browser tool loops.
