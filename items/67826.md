## Summary

Keep this PR open. Current main has superseded the setup allowFrom direction by making Telegram setup numeric-ID-only and moving legacy @username repair to doctor, but the PR's target-resolution hardening is not implemented as submitted. The remaining change is a maintainer/API decision, not a conservative close.

## Best Possible Solution

Keep the PR open for maintainer/contributor follow-up. Rebase it onto current main, drop or rewrite the obsolete setup username-resolution pieces around the numeric-only allowFrom contract, and decide whether Telegram target resolution should defensively coerce malformed internal resolver inputs while preserving or explicitly changing the public string[] resolver contract. Add focused regression coverage if that hardening is retained.
