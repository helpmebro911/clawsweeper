## Summary

Keep #71801 open. Current main still forwards Codex app-server `request_user_input` display strings directly into chat without the sanitization, per-field bounds, or prompt-length bounds described by the PR, and the existing bridge tests only cover routing/answer mapping. The PR is a recent focused bug/security-hardening change in the bundled Codex plugin, not a duplicate or already-shipped fix, though it still needs maintainer review for the unresolved bot findings around answer-object hardening and shown-question/answered-question alignment.

## Best Possible Solution

Keep the PR open and continue it, or land an equivalent maintainer patch in `extensions/codex/src/app-server/user-input-bridge.ts`. The best fix should sanitize only display text, cap question/option/prompt preview sizes, ensure the set of answerable questions matches the set shown to the user, harden answer-object creation for untrusted question ids, preserve existing request id/answer semantics, and add focused regression tests in `extensions/codex/src/app-server/user-input-bridge.test.ts`.
