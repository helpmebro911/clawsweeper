Keep PR #71801 open. Current main still formats Codex app-server request_user_input chat prompts from raw app-server supplied header/question/option strings with no display sanitizer, per-field caps, full prompt cap, or regression coverage. The PR is a focused bundled Codex plugin hardening change; the existing bot comments identify review follow-ups to resolve before merge, not a reason to close it as obsolete.
