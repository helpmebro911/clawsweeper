## Summary

Current `main` and the shipped `v2026.4.24` release already implement the requested subagent announce selection behavior: visible assistant output is preferred over raw tool/toolResult text, the `chat.history` limit is applied to the most recent visible messages, and the latest-assistant fallback strips tool messages before scanning backward.

## Best Possible Solution

Keep the current assistant-first announce extraction in core: preserve silent-token handling, prefer visible assistant text over raw tool/toolResult content, use raw tool output only as a fallback when no assistant result exists, and ask for a fresh transcript if anyone can still reproduce on `v2026.4.24` or newer.
