## Summary

Keep open. Current main has already picked up the stop-reason/context-overflow classification part, but it does not implement the PR’s central behavior: a bounded retry that suppresses prompt-hook context after compaction/truncation recovery fails. The PR is also not an obvious clean merge as-is because the discussion identifies remaining edge cases around hook scope, hook availability, and retry-loop budget.

## Best Possible Solution

Keep #44379 open unless a newer canonical PR supersedes it. The best path is a narrow maintainer-reviewed patch that adds a one-attempt overflow recovery retry for hook-injected context, gates it on remaining loop budget and actual prompt-hook context availability, preserves or explicitly documents which hook effects are skipped, and adds focused regression tests in the embedded runner overflow suite. The already-shipped stop-reason classification can be left as-is; the unrelated peer dependency and lockfile churn should be removed or handled separately.
