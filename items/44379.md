## Summary

Keep open. Current `main` only overlaps with part of this PR: raw `context_window_exceeded` messages are already treated as context-overflow text, but the PR’s main run-loop change, the one-shot suppress-hook retry after failed overflow recovery, is not present on `main`. The existing overflow loop still stops after compaction and optional tool-result truncation, so this is not `implemented_on_main`. The PR also has unresolved review concerns about retry semantics and loop-budget behavior, so it should be reworked rather than auto-closed.
