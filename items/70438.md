## Summary

Keep #70438 open. Current main still treats WebChat/Control UI attachments as image-only, the gateway parser explicitly drops sniffed non-image payloads, and the public config/docs surface exposes no `gateway.webchat.attachments` policy for PDF offload or text inline handling. Related #30759 and #60226 overlap, but they do not fully supersede this issue's specific configurable policy proposal.

## Best Possible Solution

Keep the issue open unless a maintainer explicitly consolidates it. The likely product path is to add a documented `gateway.webchat.attachments` policy, update the Control UI accept/validation path, extend `parseMessageWithAttachments` for PDF offload and capped UTF-8 text/code inline handling, and cover the behavior in gateway/UI tests plus WebChat docs. If maintainers want one tracker, copy the config/text-inline requirements into the canonical document-upload issue before closing this one as duplicate.
