## Summary

Keep open. Current main still implements the WebChat attachment path as image-only in the Control UI and gateway parser, drops non-image payloads, and exposes no `gateway.webchat.attachments`/text-inline/PDF-offload policy config. Related document-upload issues and PRs show this is an active feature gap, but none clearly supersedes the specific configurable handling requested here.

## Best Possible Solution

Keep #70438 open as the config-specific WebChat non-image attachment policy request. The best fix should extend Control UI accept/filter/preview beyond images, carry filename/MIME/type metadata through `chat.send`, add a `gateway.webchat.attachments` policy or equivalent per-tenant setting, offload PDFs as `media://inbound/<id>` refs, inline bounded UTF-8 text-like files, visibly report unsupported drops, and add focused UI/gateway/config tests. If maintainers consolidate, fold it into the canonical generic WebChat file-upload work only with those configurable handling requirements preserved.
