## Summary

Keep open. Current `main` still enforces the exact behavior the issue reports: Feishu webhook mode requires `encryptKey`, then rejects any request missing `X-Lark-*` signature headers before parsing JSON or calling `Lark.generateChallenge()`. The bundled e2e tests also lock in `401 Invalid signature` for unsigned `url_verification` requests, so there is no evidence this was implemented on `main`.

## Evidence
