## Summary

Keep this issue open. Current main still has only `systemPromptOverride`, which replaces the assembled prompt, and does not expose the requested additive `agents.defaults.systemPrompt` / `rules` or per-agent additive prompt fields. The related implementation PR is closed and unmerged, and the related duplicate issue is closed, so there is no active canonical tracker that preserves the remaining work.

## Best Possible Solution

Implement a distinct core additive prompt contribution surface: schema-backed and typed `agents.defaults.systemPrompt` and/or `agents.defaults.rules`, plus per-agent fields with documented merge semantics. Thread the resolved contribution through the shared prompt-building path, update generated schema/docs/config metadata and any Control UI or config mutation allowlists, preserve `systemPromptOverride` as full replacement behavior, and make an explicit design call for `$include` path safety.

## Risks / Open Questions

- The `$include` portion of the request needs explicit schema, path resolution, and path-safety design before implementation.
- Implementation should preserve `systemPromptOverride` as a separate full replacement escape hatch rather than overloading it with additive semantics.
- Coverage should include the shared session creation paths called out in discussion: main runs, cron-triggered runs, subagents, CLI-backed sessions, and channel-triggered sessions.
