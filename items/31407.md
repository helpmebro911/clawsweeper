## Summary

Keep open. Current `main` does not already include the requested external OpenAI compaction link, so this is not `implemented_on_main`. But the PR diff is now stale against the docs structure on `main`: `serviceTier` and server-side compaction are documented in separate sections, and the PR's insertion point would mix `responsesServerCompaction` guidance into the `service_tier` accordion, which matches the existing review concern rather than providing strong grounds to close it conservatively.
