## Summary

Keep this issue open. Current main still has the secrets RPC surface and overlapping SecretRef docs/helpers, but it does not include the requested focused secrets guidance in `src/gateway/server-methods/AGENTS.md` or a central plugin credential-persistence contract. The issue is also paired with open same-author PR #71156, which remains the active implementation candidate.

## Best Possible Solution

Keep this issue open until maintainers resolve #71156 or an equivalent focused documentation change. The best path is to add secrets-surface guidance under `src/gateway/server-methods/AGENTS.md` and extend/cross-link public plugin docs to define the credential-persistence contract for plugin-local stores: SecretRef-backed config versus plugin-local credentials, `withFileLock` plus atomic JSON write expectations, TTL/lifecycle handling, and failure behavior that must not report success before persistence or revocation guarantees hold.
