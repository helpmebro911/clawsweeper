## Summary

Keep #71116 open. Current main has substantial SecretRef documentation and reusable storage primitives, but it has not implemented the specific maintainer-facing guidance requested here: a secrets-focused `src/gateway/server-methods/AGENTS.md` section and a plugin/extension credential persistence contract. The linked PR #71156 is relevant and partially addresses the issue, but it is still open and does not clearly cover the full extension credential-governance scope.

## Best Possible Solution

Keep the issue open until maintainers either merge a scoped docs change such as #71156 and confirm it satisfies the issue, or split the remaining work. The best implementation path is to add secrets-specific invariants to `src/gateway/server-methods/AGENTS.md` and document the plugin credential persistence pattern in the plugin docs, anchored on `openclaw/plugin-sdk/file-lock` and `openclaw/plugin-sdk/json-store`; any CI token audit or optional credential helper should be separated if maintainers want that broader scope.
