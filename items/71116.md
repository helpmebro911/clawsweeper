## Summary

Keep this issue open. Current main has SecretRef runtime documentation, protocol notes, SDK file-lock and atomic JSON helpers, and tests around secrets reload/resolve behavior, but it still does not add the requested secrets hotspot guidance in `src/gateway/server-methods/AGENTS.md` or a central plugin credential-persistence contract. The same author also has paired PR #71156 open and unmerged, so the maintainer path is to review, revise, land, or close that PR rather than auto-close this issue.

## Best Possible Solution

Keep this issue open until maintainers either merge #71156 or land an equivalent focused documentation change. The best implementation is to add a secrets-surface section to `src/gateway/server-methods/AGENTS.md` and add or cross-link public plugin docs that define the credential-persistence contract: when to use SecretRef-backed config versus plugin-local storage, how to combine file locks with atomic writes, lifecycle/TTL expectations, and failure behavior that must not claim success before persistence or revocation guarantees hold.
