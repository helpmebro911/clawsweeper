## Summary

Keep #71116 open. Current main has SecretRef docs and lower-level SDK helpers (`plugin-sdk/file-lock`, `plugin-sdk/json-store`), but it does not yet add the requested secrets hotspot guidance or a central plugin credential-persistence contract. The linked PR #71156 is still open in the provided context and only covers part of the requested docs scope.

## Best Possible Solution

Keep #71116 open for maintainer triage. The best path is to let #71156 or a successor PR handle the high-privilege AGENTS.md guidance, then add a public plugin credential-persistence section that defines lock, atomic write/rename, permissions, error handling, and no-silent-success expectations for plugin-owned credentials. Maintainers should separately decide whether existing `plugin-sdk/file-lock` plus `plugin-sdk/json-store` is sufficient or whether a credential-specific SDK helper is worth adding; the CI token-scoping audit can be split into a narrower ops/security issue if maintainers want to track it.
