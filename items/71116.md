## Summary

Keep #71116 open. Current main has SecretRef runtime and credential-surface docs plus SDK helpers for file locks and atomic JSON/secret writes, but it does not yet document the credential-persistence contract requested by the issue, and the linked PR #71156 is open, unmerged, and only a partial docs follow-up.

## Best Possible Solution

Keep #71116 open and treat #71156 as a partial follow-up. The best path is to land or revise focused docs that add secrets server-method invariants, then add a plugin-facing credential persistence contract under the public plugin/SDK docs, anchored to existing helpers such as plugin-sdk/file-lock, plugin-sdk/json-store, and secret-file-runtime. If maintainers want to defer CI token scoping or a shared credential helper, split those into narrower follow-up issues rather than closing the umbrella before the docs contract exists.
