## Summary

Keep open. The report is recent, specific, and still not matched by a current-main fix or an intentional docs explanation. Current main has useful Gateway startup import and QMD/memory deferral work, but the code and tests I inspected cover RSS/import fanout and selected-memory-slot loading, not the reported 22GB Linux VIRT/VSZ anonymous-mapping symptom at Gateway startup.

## Best Possible Solution

Keep this issue open and ask for a focused current-main or latest-release reproduction: exact OpenClaw and Node versions, Gateway launch method, ps RSS/VSZ over time, /proc/<pid>/smaps or pmap -x summaries, mapping counts, enabled memory backend/plugins/providers, safe-mode or minimal-config results, and whether the unreleased Gateway startup deferral/import changes alter VmSize. If the VIRT is benign Linux/Node virtual address reservation, document that in Linux/Gateway troubleshooting; if RSS/commit/OOM pressure grows, fix the concrete Gateway startup path or dependency that owns the mappings.
