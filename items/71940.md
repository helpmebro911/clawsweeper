## Summary

Keep #71940 open. Current main still lacks the Hyprland/grim screenshot fallback, the opt-in config key, runtime schema support, docs coverage, and tests. The linked bug #64317 has concrete reproduction details and remains a real browser-plugin issue rather than a cleanup-close candidate.

## Best Possible Solution

Keep #71940 and/or its canonical bug #64317 open. The best path is a focused browser-plugin fix that adds an opt-in Hyprland/Wayland headed viewport screenshot fallback, wires the flag through BrowserConfig, strict zod schema, generated/help docs as needed, and browser Linux troubleshooting docs, with tests for route selection and virtual-output cleanup. The PR should also address cleanup/signal failure modes and remove unrelated plugin auto-enable cache work.
