## Summary

Keep PR #47562 open. Current main still only supports standard Telegram emoji reactions, while the PR targets a real missing behavior: inbound `custom_emoji` reaction events and outbound `customEmojiId` reaction sends. The PR is not already implemented or clearly superseded, but it needs to be rebased onto the current Telegram runtime files and should address the existing review comments about custom-only reaction validation and success metadata.

## Best Possible Solution

Keep the PR or request an updated patch that ports the feature to current `extensions/telegram/src/action-runtime.ts`, `extensions/telegram/src/send.ts`, `extensions/telegram/src/bot-handlers.runtime.ts`, and the shared reaction tool schema. The updated implementation should allow `customEmojiId` without an `emoji`, report custom additions accurately, preserve standard emoji behavior, and add focused action-runtime, send, and bot-handler tests for custom-only, mixed, priority, and removal cases.
