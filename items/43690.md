## Summary

Keep open. Current main still lacks Feishu card table-limit fallback: card paths send interactive cards directly, markdown table conversion remains on the plain-text path, and the provided GitHub context shows open unmerged PR #50429 with closing syntax for this issue plus overlapping PR #70651.

## Best Possible Solution

Keep this issue open until a Feishu plugin fix lands. The best implementation should detect Feishu card table-limit failures around reply and outbound card sends, retry that message via the existing sendMessageFeishu plain-text path so markdown tables are converted, preserve non-table card errors, and add focused regression tests for 230099 / 11310 / "table number over limit". Maintainers should review #50429 and #70651, choose or supersede one canonical fallback fix, and treat #42809 as a related native-table rendering enhancement rather than evidence that this bug is already fixed.
