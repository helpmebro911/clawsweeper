## Summary

Keep open. Current main still only starts the gateway memory sidecar for QMD and explicitly skips builtin memory startup work, so the reported builtin local `memory_search` cold-start prewarm is not implemented. PR #70596 directly tracks a partial fix, but it is still open, not merged, and has an unresolved review concern, so the issue should remain available as the canonical user-facing bug until a safe fix lands.

## Best Possible Solution

Keep #70460 open until a safe implementation lands. The best path is to revise or replace PR #70596 so gateway startup includes builtin explicit-local memory prewarm, uses the default cached manager rather than `purpose: "status"`, avoids poisoning the cached manager on transient provider failures, and adds gateway startup tests for explicit local, non-local skip, and failure recovery/logging. If maintainers want auto-selected local embeddings covered too, track that scope explicitly before closing this issue.
