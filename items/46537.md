## Summary

Keep open. Current main still leaves ensure-derived ACP identities as pending and startup reconciliation still scans pending identities that already have stable IDs, so this PR is not implemented on main. The PR also has a real merge-precedence concern from review: if ensure-derived identities become resolved, they can overwrite fresher persisted/status IDs when status refresh is skipped or fails, so this needs maintainer/code changes rather than conservative auto-close.

## Best Possible Solution

Keep the PR open for maintainer follow-up, but do not merge the current patch as-is. The safer path is to preserve status-derived/resolved IDs over ensure-derived IDs, then add focused tests for stable IDs from ensure, stale ensure versus fresh status, and status-refresh failure with `failOnStatusError: false`. If maintainers decide pending-until-status is the intended contract, close this PR manually as superseded by the newer ACP transcript/oneshot reconciliation work.
