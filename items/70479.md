## Summary

Keep this PR open. Current main still contains the compaction-failure session-reset path that rewrites the active session mapping, while this PR is a focused implementation candidate for the linked WebChat/session orphaning bug and is not superseded by the related overflow-budget PR.

## Best Possible Solution

Keep this PR open as the canonical implementation candidate. Maintainers should review and land this PR, or an equivalent fix, that preserves the active WebChat session mapping when overflow recovery or auto-compaction fails, surfaces explicit retry /compact /new guidance, and keeps regression coverage for embedded overflow payloads, thrown compaction failures, and unchanged session-store/sessionId state.
