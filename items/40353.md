## Summary

Keep open. Current `main` has structured JSONL logging and some request-scoped plugin context, but it does not implement the requested gateway-wide trace ID generation/propagation for log correlation.

## Evidence

- **Gateway HTTP entrypoint has no trace ID generation or request-correlation wrapper:** `handleRequest` in `src/gateway/server-http.ts:900` builds request stages directly after parsing the request. There is no trace ID creation, no AsyncLocalStorage trace wrapper, and no trace metadata attached before dispatching downstream handlers.
  - file: [src/gateway/server-http.ts:900](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/gateway/server-http.ts#L900)
  - command: `nl -ba src/gateway/server-http.ts | sed -n '900,1015p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Logger writes only provided fields plus timestamp:** `buildLogger()` serializes `{ ...logObj, time }` in `src/logging/logger.ts:176-180`. There is no hook that injects a current request trace ID into every log record.
  - file: [src/logging/logger.ts:176](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/logging/logger.ts#L176)
  - command: `nl -ba src/logging/logger.ts | sed -n '150,230p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Existing AsyncLocalStorage scope is plugin runtime context, not trace correlation:** `PluginRuntimeGatewayRequestScope` in `src/plugins/runtime/gateway-request-scope.ts:8-13` carries `context`, `client`, `isWebchatConnect`, and optional `pluginId`. It does not carry a trace/request correlation ID, and `withPluginRuntimeGatewayRequestScope()` just runs that scope unchanged.
  - file: [src/plugins/runtime/gateway-request-scope.ts:8](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/plugins/runtime/gateway-request-scope.ts#L8)
  - command: `nl -ba src/plugins/runtime/gateway-request-scope.ts | sed -n '1,120p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Subsystem logger only forwards explicit meta:** `createSubsystemLogger()` in `src/logging/subsystem.ts:329-377` forwards the caller-supplied `meta` to file/console output. There is no automatic merge of request-scope trace metadata.
  - file: [src/logging/subsystem.ts:329](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/logging/subsystem.ts#L329)

The docs separately mention OTel traces in `docs/logging.md:174-194`, but not gateway request trace IDs in log lines.
