## Summary

Keep open. Current main has adjacent Control UI diagnostics, but not the requested ephemeral pre-auth connection log/debug mode. The Logs tab tails gateway file logs only after an authenticated connection, the Debug tab shows post-connect snapshots/events/RPC, and the browser client only surfaces a final close/error string for failed connects rather than retaining lifecycle/handshake frames in an in-memory/copyable UI. The related PR #49274 explicitly targeted this issue but was closed unmerged in the provided GitHub context, so it does not supersede the remaining work.

## Evidence

- **Browser client lacks a diagnostic log hook/buffer:** GatewayBrowserClient constructs the WebSocket, listens for open/message/close/error, and on close reports only code/reason plus the pending structured connect error through onClose. It does not expose a lifecycle/handshake-frame log stream or session-only diagnostic buffer for the UI.
