## Summary

Keep open. Current `main` still derives Docker sandbox container names from the sandbox scope key slug, not from any instance-unique identity such as `HOME` or workspace path. With the documented default `scope: "agent"`, multiple co-hosted instances using the default agent id can still target the same container name. Workspace paths only affect the config hash, and the hot-container path keeps a running mismatched container instead of renaming/recreating it, so the reported cross-instance collision remains plausible on current `main` and the latest release tag available in this checkout.
