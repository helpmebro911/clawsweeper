## Summary

Keep open. Current main still has the duplicate inbound-context finalization pattern this PR targets, and there is no landed `runInboundDispatch`-style equivalent or finalization call-count guard in the current tests. The provided related items cover different iMessage, ACP, and WhatsApp inbound-context work, so they do not supersede this focused auto-reply dispatch refactor.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase if needed, preserve the public `dispatchInboundMessage()` API, introduce the private finalized-context dispatch helper, require `FinalizedMsgContext` in the wrapper-only setup helpers, and keep the focused unit assertions that buffered and hook-enabled wrapper paths call `finalizeInboundContext()` exactly once.
