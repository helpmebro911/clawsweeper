## Summary

Keep PR #71874 open. Current main still has the duplicate-finalization wrapper shape described by the PR, no equivalent `runInboundDispatch` implementation or call-count guardrail is present on main, and the PR is a focused repo-local refactor rather than something already shipped, superseded, external, or ClawHub-scoped.

## Best Possible Solution

Keep the PR open for maintainer review. The best implementation path is the PR's narrow approach: introduce a private finalized-context dispatch helper, keep `dispatchInboundMessage()` accepting raw `MsgContext`, have both wrapper paths finalize once before silent-reply and hook setup, and keep the new call-count tests in `src/auto-reply/dispatch.test.ts`.
