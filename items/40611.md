## Summary

Current main still has the reported heartbeat retry path: requests-in-flight skips are retried by the shared 1s wake retry, runner comments/tests still assert that behavior, and there is no heartbeat retryDelayMs or equivalent busy-session retry config in schema, types, or docs. The related fixing PR #40629 is closed unmerged, while #13744 and #14051 remain broader related work rather than canonical replacements for this regression.

## Best Possible Solution

Keep this issue open and fix the core heartbeat scheduling regression. Add a separate busy-session/request-in-flight heartbeat retry delay, likely defaulting around 60s, and decide whether to expose it as `agents.defaults.heartbeat.retryDelayMs` plus per-agent heartbeat config. Wire schema/types/generated config metadata/docs, update wake and runner scheduler tests so busy-session skips no longer assert the shared 1s retry, and keep #13744 for broader session-lock scope/configuration and #14051 for activity-based heartbeat semantics.
