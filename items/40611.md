## Summary

Keep this issue open. Current main still retries heartbeat `requests-in-flight` skips through the shared 1 second wake retry, and there is still no heartbeat-specific busy-session retry/backoff config in the schema, typed config surface, or heartbeat docs. The related fixing PR is closed unmerged, while the linked open issues cover broader lock-scope and activity-based heartbeat designs rather than this specific regression.

## Best Possible Solution

Keep this issue open and repair the heartbeat scheduling path. The best product direction is a refreshed version of #40629: add a heartbeat-specific requests-in-flight retry delay separate from the shared wake error retry, likely around 60s by default; decide whether it should be configurable under `agents.defaults.heartbeat` and per-agent heartbeat config; then update schema/types/generated config metadata/docs plus wake and scheduler regression tests. Leave #13744 for broader session-lock scope/configuration and #14051 for activity-based heartbeat semantics.
