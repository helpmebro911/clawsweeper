## Summary

Keep #13616 open. Current main has native local backup archive creation and verification, but it still does not implement the requested native restore/import workflow, cloud list/restore, encryption, scheduled backup surface, or component-aware cron/session restore semantics. The latest issue discussion also includes an explicit maintainer comment keeping this umbrella open while treating openclaw-workspace-sync and keepmyclaw as useful community workarounds.

## Best Possible Solution

Keep #13616 open as the native umbrella for first-party backup restore/import and disaster recovery. The concrete maintainer path is to extend the existing `openclaw backup create` and `openclaw backup verify` implementation with safe restore/import, dry-run validation, overwrite and rollback rules, component-aware cron and session semantics, and explicit product decisions for cloud storage and encryption. Community options such as openclaw-workspace-sync and keepmyclaw should remain documented workarounds unless maintainers explicitly move this scope out of core.
