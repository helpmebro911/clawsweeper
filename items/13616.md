## Summary

Keep #13616 open. Current main has a useful native local backup archive path, but the requested native restore/import, cloud list/restore, encryption, scheduled backup surface, and component-aware cron/session restore semantics are still missing. The issue discussion includes useful community workarounds such as openclaw-workspace-sync and keepmyclaw, but the latest maintainer signal explicitly kept this native umbrella open rather than treating those as replacements.

## Best Possible Solution

Keep #13616 open as the native umbrella for first-party backup restore/import and disaster recovery. The concrete maintainer path is to extend the existing `openclaw backup create` and `openclaw backup verify` implementation with safe restore/import, dry-run validation, rollback/overwrite rules, component-aware cron and session semantics, and explicit cloud/encryption product decisions. Community options such as openclaw-workspace-sync and keepmyclaw should remain documented workarounds unless maintainers explicitly move this scope out of core.
