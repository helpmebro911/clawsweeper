## Summary

Keep open: current `main` still allows the same run to re-arm the streaming watchdog after a timeout, so duplicate watchdog warnings remain possible, and there is still no user-facing `/watchdog` command or documented TUI setting for this threshold.

## Evidence

- **Watchdog timeout clears active run but does not record a warned run id:** The timeout handler resets `state.activeChatRunId`, prints the watchdog warning, and returns without storing any per-run suppression state such as a warned-run set.
  - file: [src/tui/tui-event-handlers.ts:93](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/tui/tui-event-handlers.ts#L93)
  - command: `nl -ba src/tui/tui-event-handlers.ts | sed -n '1,260p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **A later delta for the same run can become active again and re-arm the watchdog:** On every chat delta, if there is no active run, the handler sets `state.activeChatRunId = evt.runId`; for delta events on the active run it immediately calls `armStreamingWatchdog(evt.runId)`. After a watchdog timeout clears `activeChatRunId`, a later delta for that same run can therefore re-arm the timer and emit the same warning again.
  - file: [src/tui/tui-event-handlers.ts:305](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/tui/tui-event-handlers.ts#L305)
  - command: `nl -ba src/tui/tui-event-handlers.ts | sed -n '260,360p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **TUI startup does not pass any configurable watchdog value:** The TUI constructs `createEventHandlers(...)` without a `streamingWatchdogMs` argument, so the handler falls back to its internal default instead of a user-facing runtime setting.
  - file: [src/tui/tui.ts:893](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/tui/tui.ts#L893)
  - command: `nl -ba src/tui/tui.ts | sed -n '884,928p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **Current TUI slash-command catalog does not include `/watchdog`:** The built-in TUI command list includes commands such as `/model`, `/think`, `/verbose`, `/settings`, and `/exit`, but no `/watchdog` entry or argument completions for watchdog control.
  - file: [src/tui/commands.ts:66](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/tui/commands.ts#L66)
  - command: `nl -ba src/tui/commands.ts | sed -n '1,220p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **Public slash-command docs also show no `/watchdog` command:** The current documented built-in command list covers `/session`, `/think`, `/verbose`, `/fast`, `/model`, `/status`, `/usage`, and others, but not `/watchdog`, which matches the code inspection above.
  - file: [docs/tools/slash-commands.md:87](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/docs/tools/slash-commands.md#L87)
