## Summary

Keep #69978 open. Current main still has the hardcoded/default TUI streaming watchdog path, no user-facing `/watchdog` command or persisted setting, and no per-run suppression for repeated watchdog warnings. The related open issues #68596 and #69081 cover overlapping threshold/status-desync concerns, but they do not fully supersede this issue's combined duplicate-warning and TUI runtime-control request.

## Best Possible Solution

Keep the issue open and implement it as focused TUI work: add per-run warned-run suppression in `src/tui/tui-event-handlers.ts`, add `/watchdog` parsing/handling/completions in the TUI command surfaces, persist the chosen timeout or disabled state in an appropriate local TUI settings store, and update `docs/tools/slash-commands.md` plus targeted tests. Coordinate with #68596 for the broader configuration decision and #69081 for status/watchdog semantics, but do not close this issue until the duplicate-warning and runtime-control behavior is actually shipped or explicitly superseded by a canonical maintainer-tracked item.
