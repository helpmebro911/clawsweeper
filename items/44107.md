## Summary

Keep open. Current main appears to cover the large-session/chat freeze side with bounded `chat.history`, markdown rendering guards, and a ReDoS regression test, but the independent Chrome `<all_urls>` extension blank-page report is still actionable: the Control UI boot HTML has no pre-module mount sentinel or user-facing fallback, and the docs do not contain the proposed extension/incognito troubleshooting guidance.

## Best Possible Solution

Keep this issue open, or split it: treat the large-session freeze portion as likely fixed by the current bounded-history and markdown-it work, then resolve the remaining Control UI blank-page path with a small boot-time fallback in `ui/index.html` plus a public Control UI/dashboard troubleshooting note for browser extensions/incognito. If maintainers prefer narrower tracking, open a focused follow-up for the Chrome extension blank-page UX/docs and close this only after that is linked or implemented.
