## Summary

Keep open. Current main still does not render the read-tool image payload shape described in #50779. The Control UI extractor handles image blocks with source/base64 or url fields, plus image_url/input_image variants, but not direct tool-result blocks shaped as { type: "image", data, mimeType }. The live tool stream also reduces tool results to text, so a read result with "Read image file [image/png]" plus an image block still presents as text without a visible preview. Related PR #52025 tracks the same area but is open and not implemented on main, so this issue should remain open rather than be auto-closed.
