## Summary

Current `main` still implements `openclaw logs` as file-log tailing over `logs.tail`, with an automatic local file fallback on loopback pairing-required errors. The Linux/systemd journal-aware behavior and provenance output requested in #66841 are not present on `main`, and the latest release still ships the same code path, so this should stay open.
