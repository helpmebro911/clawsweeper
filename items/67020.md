## Summary

Current main still matches the reported Slack bug. `dmHistoryLimit` remains advertised in Slack config/docs, but the Slack monitor only carries `historyLimit`; 1:1 DMs are classified separately from `isRoomish`, and pending-history recording, `combinedBody`, `InboundHistory`, and history clearing are still gated on `isRoomish && ctx.historyLimit > 0`. The related PR #67054 attempted the right fix but is closed and unmerged, so it does not supersede this issue.

## Best Possible Solution

Keep this issue open and land a focused Slack monitor fix: derive `account.config.dmHistoryLimit`, carry it through `SlackMonitorContext`, compute an effective history limit for 1:1 DMs versus room/group contexts, record pending DM history, use that limit for `combinedBody`, `InboundHistory`, and history clearing, and add a regression test for a 1:1 Slack DM with `dmHistoryLimit > 0`. After Slack is fixed, decide whether the cross-channel parity concern deserves a separate issue or shared abstraction.
