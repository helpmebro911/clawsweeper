## Summary

Current main still matches the reported Slack bug: Slack accepts and documents `dmHistoryLimit`, but the Slack monitor derives, carries, records, injects, and clears only `historyLimit` behind `isRoomish`, which excludes 1:1 DMs. PR #67054 was closed unmerged, and related #64427 is broader cross-session history work rather than a clean supersession of this narrower channel-monitor wiring bug.

## Best Possible Solution

Keep this issue open and land a focused Slack monitor fix: derive `account.config.dmHistoryLimit`, carry it through `SlackMonitorContext`, compute an effective room-vs-DM history limit, record pending 1:1 DM history, use that effective limit for `combinedBody`, `InboundHistory`, and history clearing, and add a regression test for a non-threaded 1:1 Slack DM with `dmHistoryLimit > 0`. Track broader cross-session history hydration and cross-channel parity separately.
