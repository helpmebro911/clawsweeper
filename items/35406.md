## Summary

Keep #35406 open. Current main still does not guard the generic `read` tool from PDF/Office binary documents before delegating to the upstream reader. OpenClaw has MIME detection and a separate `pdf` tool, but those do not prevent `read(path="*.docx")`, `read(path="*.xlsx")`, `read(path="*.pptx")`, or `read(path="*.pdf")` from reaching the normal read path. The provided related context shows #35494 closed unmerged and #57600 still open as the draft implementation candidate.

## Best Possible Solution

Keep #35406 open and land #57600 or a maintainer-owned equivalent. The fix should add a small preflight in `src/agents/pi-tools.read.ts` before `executeReadWithAdaptivePaging`, reuse `src/media/mime.ts` for PDF and Office detection, wire both host and sandbox read paths, return a concise fallback that points PDFs to the `pdf` tool and Office files to a parser/conversion path, and add regression tests for `.pdf`, `.docx`, `.xlsx`, and `.pptx`.
