## Summary

Current main still does not implement the requested cron-scoped deny policy. Cron jobs have global/agent/provider/group/sandbox policy, delivery-mode message-tool controls, and per-job `toolsAllow`, but there is no `tools.cron.tools.deny` schema/type/docs surface and `tools.subagents.tools.deny` is still only applied to subagent/ACP envelope sessions, not `cron:` sessions.

## Best Possible Solution

Keep the issue open for maintainer product direction. A complete fix should add either `tools.cron.tools.allow/deny` or explicit cron inheritance from `tools.subagents.tools`, wire that policy into isolated cron tool construction and final embedded-runner policy, define precedence with global/agent/provider/group/sandbox policy and per-job `toolsAllow`, then update schema/types/help/docs/generated config metadata and tests covering denied `message` for `announce`, `none`, and `webhook` cron delivery modes.
