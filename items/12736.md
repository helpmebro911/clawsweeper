## Summary

Current main still does not implement `tools.cron.tools.allow/deny` and does not make cron sessions inherit `tools.subagents.tools`. Cron has a per-job `toolsAllow` allowlist and delivery-mode-specific message-tool handling, but the requested cron-scoped deny policy remains a real config/product gap. The prior issue comment by steipete already reached the same keep-open conclusion, and current main has not changed the relevant surfaces since that review.

## Best Possible Solution

Keep #12736 open for maintainer product direction. A complete fix should add a cron-scoped policy such as `tools.cron.tools.allow/deny` or explicitly define cron inheritance from `tools.subagents.tools`, specify precedence against global, agent, provider, group, sandbox, and per-job `payload.toolsAllow` policies, reconcile it with cron delivery-mode message-tool behavior, then update schema/types/help/generated config metadata, docs, CLI/config reference, and regression tests for announce, none, and webhook delivery modes.
