## Summary

Current main still does not implement a cron-scoped tool allow/deny policy and does not make cron sessions inherit `tools.subagents.tools`; the remaining work is a real product/config gap, not a closeable duplicate or already-shipped fix.

## Best Possible Solution

Keep #12736 open for maintainer product direction. A complete fix should add a cron-scoped policy such as `tools.cron.tools.allow/deny` or explicitly define cron inheritance from `tools.subagents.tools`, specify precedence against global/agent/provider/sandbox policies and per-job `payload.toolsAllow`, integrate with cron delivery-mode message-tool handling, then update schema/types/help/generated config metadata, docs, CLI/config reference, and tests across announce, none, and webhook delivery modes.
