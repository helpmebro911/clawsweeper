## Summary

Keep this issue open. Current main can display configured agent avatars and has a browser-local assistant avatar override, but it still does not implement the requested durable Agents-tab upload workflow: no `agents.avatar.upload` method/schema/scope/handler, no `workspace/avatars/` image-byte write path, and no Agents-tab file picker that updates `IDENTITY.md`.

## Best Possible Solution

Keep this issue open and implement a first-party Control UI agent avatar upload flow. The likely path is an admin-scoped RPC or upload endpoint that validates image type/size, safely writes the image under the selected workspace's `avatars/` directory, updates `IDENTITY.md` through existing identity helpers, refreshes agent identity/list/avatar display state, and adds gateway plus UI regression tests.

## Risks / Open Questions

- Implementing this later is security-sensitive because it accepts operator-supplied image bytes; it needs strict type and size validation plus safe writes that prevent path traversal, symlink escapes, and hardlink surprises.
- The browser-local Quick Settings assistant avatar override and existing `/avatar/<agentId>` display route are easy to confuse with this request, but neither updates the selected agent workspace or `IDENTITY.md`.
