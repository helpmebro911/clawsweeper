## Summary

Keep this issue open. The prior ClawSweeper comment correctly identified the gap, and current main still only displays/serves configured agent avatars or stores a browser-local Quick Settings assistant override. There is no Agents-tab upload flow, gateway RPC, binary/image upload schema, or server-side write path that saves image bytes under an agent workspace and updates IDENTITY.md.

## Best Possible Solution

Implement a first-party Control UI agent avatar upload path rather than closing this as stale. The likely maintainer path is an additive admin-scoped gateway method or authenticated endpoint with a typed protocol schema for image bytes/metadata, safe workspace-relative storage under avatars/, IDENTITY.md updates through existing identity helpers, Agents-tab UI picker wiring, avatar refresh behavior, and focused gateway/UI regression coverage.
