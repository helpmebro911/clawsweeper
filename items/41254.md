## Summary

Keep this PR open. Current main still has the exact heartbeat dedupe load/mutate/save pattern the PR targets, and the session-store API confirms that `updateSessionStore` is the lock-safe path because it re-reads inside the write lock. The PR is a focused one-file bug fix, not obsolete or superseded.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase if needed, land the `updateSessionStore` change for the `lastHeartbeatText`/`lastHeartbeatSentAt` dedupe write, and preferably add a focused regression test that proves a concurrent session-store update is preserved.
