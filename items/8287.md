## Summary

Issue #8287 should stay open. Current main still exposes nodes through declared command routing and the static `nodes` agent tool, but it does not let connected nodes advertise first-class LLM tools with schemas, lifecycle, catalog/effective inventory entries, or node provenance. The latest issue discussion also contains recent maintainer signal explicitly keeping the item open.

## Best Possible Solution

Keep #8287 open for maintainer-approved additive Gateway protocol and runtime work. A good implementation would define a validated node tool-advertisement schema, attach advertised tool lifecycle to connected `NodeRegistry` sessions, synthesize dynamic agent tools plus `tools.catalog`/`tools.effective` entries with node provenance, route execution through existing `node.invoke` authorization, and document naming, conflicts, pairing, reconnect behavior, and policy semantics. Keep `api.runtime.nodes` and the openclaw-tescmd-style plugin as interim workarounds, not the final zero-plugin node tool model.

## Risks / Open Questions

- Closing this as ClawHub would lose a real core protocol/runtime request: the current `openclaw-tescmd` plugin workaround is useful prior art, but the issue specifically asks to avoid the plugin-to-gateway loopback model.
- Dynamic node-advertised tools need explicit security and product design for pairing, declared command allowlists, node-local approvals, provenance, naming conflicts, reconnect idempotency, and `tools.allow`/`tools.deny` semantics.
