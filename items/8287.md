## Summary

Issue #8287 should stay open. Current main still has node command routing and the static `nodes` agent tool, but it does not implement connected-node tool advertisement, dynamic per-node tool lifecycle, node-sourced catalog/effective inventory entries, or node-provenance policy semantics. The issue also has recent maintainer signal explicitly keeping it open.

## Best Possible Solution

Keep #8287 open for maintainer-approved Gateway protocol and runtime work. The likely direction is an additive node tool-advertisement schema, validated storage on connected `NodeRegistry` sessions, dynamic synthesis of node-backed agent tools and catalog/effective inventory entries with node provenance, execution routed through existing node authorization, and docs covering naming, conflict, reconnect, pairing, and policy behavior.
