## Summary

Current main still does not implement node-registered agent tools. Nodes can declare caps/commands/permissions and can be invoked through the static `nodes` agent tool or low-level `node.invoke`, while dynamic agent tools still come only from core tool construction and plugin registration. The adjacent tool/hook/plugin changes do not supersede the requested gateway protocol/runtime seam.

## Best Possible Solution

Keep #8287 open until a maintainer-approved design or implementation adds an additive node tool-advertisement contract. The likely implementation path is to extend the gateway protocol schema and generated clients, store validated node tool definitions in `NodeRegistry` with connect/disconnect lifecycle, expose them through agent tool construction/catalog/effective inventory, route calls through `node.invoke`, and define naming plus `tools.allow`/policy behavior for node-owned tools. The existing plugin workaround and ClawHub direction are useful, but they do not solve the requested missing core protocol/runtime seam.
