## Summary

Keep open. Current main still hardcodes memory-core short-term promotion output to `MEMORY.md`, exposes no `promotionTarget`-style config or CLI target option, and docs still describe deep promotion as `MEMORY.md`-only behavior. The reported bootstrap-truncation concern remains plausible because `MEMORY.md` is a bootstrap file with a 12,000-character default injection cap.

## Best Possible Solution

Keep the issue open and implement this in the bundled `memory-core` plugin: add a validated promotion-target config field, thread it through CLI and scheduled dreaming promotion, preserve the default `MEMORY.md`, create missing parent files safely, update memory host events/docs, and ensure the configured target is indexed by `memory_search` regardless of whether it is under `memory/` or a root file.
