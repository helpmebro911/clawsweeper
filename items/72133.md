## Summary

Keep #72133 open. Current main already covers the channel-surface part with optional per-response `/usage` footers, but the native mobile chat request remains incomplete: iOS decodes message usage without rendering a footer, and Android does not carry usage metadata in its chat message model/parser at all.

## Best Possible Solution

Keep the issue open and scope the remaining work to native mobile: add a shared compact usage formatter for mobile chat, render it under assistant messages in SwiftUI and Android Compose, extend Android chat history parsing/model to preserve usage metadata, and include context percentage/model when the gateway session/message payload can supply them. Leave the existing `/usage` channel footer implementation in place.
