## Summary

Keep open. Current main still matches the reported Nostr restart loop: the DM subscription passes an array through `subscribeMany`, the gateway restarts `startAccount` tasks when they settle, and the Nostr gateway start path still settles immediately after startup. The related PR #42832 is an implementation candidate but remains open and was written against an older file split, while the older related reports are closed rather than usable as the canonical open tracker.

## Best Possible Solution

Keep this issue open as the canonical Nostr restart-loop/subscription tracker. Review and rebase #42832 or prepare a fresh patch against current main: pass a plain `Filter` object to `SimplePool.subscribeMany`, remove the unsafe cast, keep `startNostrGatewayAccount` pending until `ctx.abortSignal` aborts, close both the subscription and SimplePool relay connections during cleanup, and add regression coverage for serialized REQ filter shape plus gateway lifecycle so a successful Nostr start does not auto-restart.
