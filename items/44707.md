## Summary

Keep open. Current `main` still drops transcript updates when `update.message` is absent, while current code and tests still treat file-only transcript updates as valid behavior. The Control UI chat view still refreshes from `session.message` or `sessions.changed`, so there is no strong evidence that the reported post-reconnect live-update regression has been fixed on `main` or in the latest release.

## Evidence

- **Gateway still short-circuits file-only transcript updates on current main:** `createTranscriptUpdateBroadcastHandler()` returns immediately on `if (!sessionKey || update.message === undefined) return;`, which prevents both `session.message` and `sessions.changed` broadcasts for transcript updates that do not include an inline message payload.
