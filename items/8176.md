## Summary

Keep open. Current main still does not expose or apply a `promptOverride` result for `before_agent_start` or `before_prompt_build`. The related `systemPrompt` path is now wired, but that only replaces the system prompt; it does not let a plugin replace or truncate the current user prompt before `activeSession.prompt(...)`. The directly related #8178 PR was closed unmerged, and the other linked items address adjacent system-prompt or context-stripping work rather than this requested capability.

## Best Possible Solution

Keep #8176 open until OpenClaw either implements a documented prompt replacement field for the prompt-build/legacy hook path or a maintainer explicitly rejects that API. The likely implementation belongs in the public hook result type, hook result field list/merger, prompt-build helper, embedded runner prompt assembly before `prependContext`, docs, and focused regression tests for ordering.
