## Summary

Current main still does not implement a hook result that lets plugins replace the submitted user prompt. The hook contract, docs, merger, embedded runner, CLI runner, and harness path all support system prompt/context mutation, but not `promptOverride`, so #8176 remains actionable.

## Best Possible Solution

Keep #8176 open until OpenClaw either implements and documents an explicit user prompt replacement field for `before_prompt_build` and legacy `before_agent_start`, or maintainers explicitly reject that API. A complete fix should update public hook result types, prompt mutation filtering, hook merge precedence, embedded runner, CLI runner, harness/test helpers, docs, and regression tests proving replacement plus `prependContext` ordering.

## Risks / Open Questions

- The implemented `systemPrompt` side can be mistaken for this request, but it only changes system/developer prompt handling and does not let a plugin shrink or redact the current user prompt.
- A complete implementation still needs maintainer API decisions on multi-hook replacement precedence, whether later hooks see the original or effective prompt, and how `allowPromptInjection=false` filters any new replacement field.
