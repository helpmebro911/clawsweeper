## Summary

Keep #8176 open. Current main still has no hook result field or runner behavior that lets a plugin replace the submitted user prompt; it only supports model/provider overrides plus system prompt and context prepending. The related systemPrompt side has been wired, but that does not solve the requested user prompt replacement capability.

## Best Possible Solution

Keep #8176 open until OpenClaw either implements and documents an explicit user prompt replacement field for before_prompt_build and legacy before_agent_start, or maintainers explicitly reject that API. A complete fix should update public hook result types, prompt mutation filtering, merge precedence, embedded runner, CLI runner, agent harness helper, public docs, and regression tests proving replacement plus prepend ordering.
