## Summary

Current main partially addresses the broken notification path by removing the coding-agent skill's `openclaw system event` auto-notify guidance and requiring direct `openclaw message send`, but it still explicitly recommends `bash background:true` for all coding-agent runs rather than `sessions_spawn` + `sessions_yield`. Because the title and suggested fix are specifically about changing that recommendation, this is not a high-confidence auto-close as implemented.
