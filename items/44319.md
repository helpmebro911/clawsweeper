## Summary

Keep open. Current `main` still has the bug the PR describes: `resolveExtraParams` constructs the per-model config key with raw string concatenation, so provider-prefixed model IDs like `openrouter/auto` resolve to `openrouter/openrouter/auto` instead of the canonical `openrouter/auto`. The repo already has canonical key helpers and uses them in adjacent model-config lookup code, so this PR remains relevant and is not implemented on `main`.

## Evidence

- **Current main still double-prefixes provider-prefixed model IDs:** `resolveExtraParams` builds `const modelKey = `${params.provider}/${params.modelId}`` and immediately indexes `agents.defaults.models` with that raw key. For `provider: "openrouter"` and `modelId: "openrouter/auto"`, that becomes `openrouter/openrouter/auto`, which will miss the documented canonical config entry.
