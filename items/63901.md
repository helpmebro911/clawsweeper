## Summary

Keep open. Current main still does not provide the requested session-aware provider request metadata for OpenRouter/OpenAI-compatible calls: `extra_body` is static pass-through, the provider extra-param/wrap hooks do not receive `sessionKey` or `sessionId`, and OpenRouter’s stream wrapper only handles routing/thinking/cache behavior. The provided GitHub context also shows same-author open PR #64813 and related open PR #54725 as implementation candidates, so this issue should not be closed ahead of resolving those PRs.

## Best Possible Solution

Keep this issue open. The best maintainer path is to review and either land or explicitly close #64813 and #54725, converging on a narrow session-aware provider attribution seam or request-body option with tests for `sessionKey`, `sessionId`, agent id/static `extra_body` precedence, and OpenRouter/LiteLLM-compatible `user`/`trace` behavior.
