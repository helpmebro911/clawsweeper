## Summary

Keep this PR open. Current main still lacks the requested command-specific timeout cap for broad shell `find` scans over `$HOME` or `/`, and the PR remains a plausible focused fix for a real exec-budget failure mode. It needs maintainer review and parser hardening rather than stale cleanup.

## Best Possible Solution

Keep this PR open and request changes before merge. The best path is to rebase onto current main, add a shared broad-find timeout classifier used by every foreground exec route, parse all `find` start roots, resolve relative roots against command `workdir`, handle `~`, `$HOME`, and `${HOME}` forms, preserve explicit timeouts and real `-prune`/`-xdev`/`-mount` intent, avoid escape hatches from later pipeline or `-exec` arguments, and keep focused regression tests for home, root, relative workdir, multiple roots, gateway follow-up, node routing, and prompt guidance.
