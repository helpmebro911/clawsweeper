## Summary

Keep open. Current main still ignores the signal argument for node-host system.run subprocess exits, so a process killed by SIGKILL/SIGTERM resolves with exitCode undefined and error null. The open PR addresses a real remaining node-host observability gap, and the related merged Linux OOM PR explicitly treats #68680 as separate SIGKILL observability work rather than a replacement.

## Best Possible Solution

Keep the PR open for maintainer review. The best code path is to update src/node-host/invoke.ts so non-timeout signal exits produce a structured error while preserving intentional timeout semantics, with a regression test that exercises the shared helper or production exit-resolution path. Maintainers may also consider consolidating node-host execution onto the existing src/process/exec.ts signal-aware helper if that fits the node-host contract.
