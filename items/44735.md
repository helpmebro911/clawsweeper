## Summary

Keep open. Current `main` does not contain the PR's alternative-plugin guard, so this is not `implemented_on_main`. The current auto-enable path still promotes bundled `channels.<id>` configs to the built-in plugin/channel unless blocked by deny/explicit-disable or `preferOver`, and there is no coverage for the reported `channels.feishu` + enabled `openclaw-lark` case. The PR also has an unresolved allowlist correctness concern, so there is not strong evidence to close it conservatively.
