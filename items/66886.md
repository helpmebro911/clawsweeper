## Summary

Keep open. Current evidence does not support a high-confidence auto-close: the cron-specific bundled-MCP leak tied to #69145 was fixed and shipped in v2026.4.23, but current `main` still makes bundled-MCP retirement opt-in in `runEmbeddedPiAgent`, and newer open issue #71106 documents the same gateway memory/process-growth symptom through other embedded sub-agent paths. #66886 is too generic to map cleanly to only the fixed cron path, so conservative cleanup should leave it open.
