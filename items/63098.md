## Summary

Keep this issue open. Current main still routes `/vc join` through a fixed 15s Discord voice Ready wait, destroys the connection on that abort path, and returns the same `Failed to join voice channel: ... operation was aborted` message. The config schema, generated UI hints, and public Discord docs still expose DAVE and TTS knobs but no connect timeout or retry policy. Related reports #39825 and #65039 are open and corroborate the same early voice-join failure family rather than proving this is fixed or superseded.

## Best Possible Solution

Keep this issue open and fix the Discord plugin voice join path. Add a focused regression test for initial Ready abort/timeout before session activation, then choose a longer default, bounded retry/backoff, and/or a documented `channels.discord.voice.connectTimeoutMs` control. If a new knob lands, update `DiscordVoiceConfig`, strict zod validation, generated config metadata/UI hints, and Discord docs, then reconcile related open reports #39825, #63125, and #65039 after the behavior is fixed or maintainers choose a canonical tracker.
