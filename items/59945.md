## Summary

Keep open. Current `main` does not show the shared runtime-snapshot fix described in the issue discussion: the runtime snapshot setter still stores the provided config verbatim, Slack startup still consumes runtime token fields directly, and the only current runtime-snapshot write test does not cover a redacted Slack token/appToken leak. I found no shipped fix in `v2026.4.23` either.
