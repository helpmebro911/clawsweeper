## Summary

Keep open. Current main has pieces of the OpenRouter dynamic-model path, but it still does not add cached OpenRouter capability rows to the model catalog or provide an OpenRouter default-thinking hook. The remaining reasoning-default path is catalog-based, so dynamic OpenRouter reasoning models that are not in the static catalog can still resolve reasoning to off unless another config/session override supplies it.

## Best Possible Solution

Keep the PR or a successor open and implement the missing OpenRouter-owned integration on current main: expose a safe cache enumeration helper if needed, add an OpenRouter augmentModelCatalog hook that supplies cached dynamic reasoning entries through the existing catalog merge path, add or verify an OpenRouter default-thinking policy, and cover createModelSelectionState/subscribe behavior for a dynamic reasoning model not present in the static catalog.
