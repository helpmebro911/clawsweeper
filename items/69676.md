## Summary

Keep open. Current main still does not implement the PR's fix: `openclaw memory` command paths resolve to the default no-plugin-preload policy, while the GitHub Copilot memory embedding adapter is registered only by the `github-copilot` plugin. The linked issue #69180 and the Greptile review correctly identify this as a plugin-loading order bug, and the PR remains a focused useful fix rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open and rebase/adapt the focused fix onto current main: add an explicit `memory` command policy that preloads plugins before memory CLI startup, keep the command-path and startup-policy regression tests for `memory status`, then run the targeted CLI policy tests and the changed gate.
