## Summary

Keep this issue open. Current main still does not support exact Slack `messageId` reads: the Slack message-action dispatcher drops `messageId` for `read`, the runtime only forwards `threadId`, and `readSlackMessages` only returns channel history or thread replies without filtering for the requested timestamp.

## Best Possible Solution

Keep this issue open and fix it in the Slack plugin read path. Add `messageId` to Slack `read` dispatch, pass it through `action-runtime`, extend `readSlackMessages` to return/filter the exact Slack timestamp for both `messageId + threadId` and messageId-only channel reads, and add focused dispatcher/runtime/helper tests. If Slack URL parsing feeds this path, cover `pMESSAGE_TS?thread_ts=THREAD_TS` normalization too.
