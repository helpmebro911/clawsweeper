## Summary

Current main still has the Slack thread-read fix, but the narrower requested capability remains missing: Slack `message` read does not accept, forward, or filter by `messageId`, so it cannot fetch one specific thread reply or channel message by timestamp.

## Best Possible Solution

Keep this issue open and implement the missing Slack plugin read behavior: add `messageId` to the Slack read action mapping, forward it through `readMessages`, extend `readSlackMessages` to return an exact message for `messageId` alone and to filter `conversations.replies` for `messageId + threadId`, and add focused regression tests for both exact thread-reply and exact channel-message lookup. If Slack URL parsing feeds this tool path, add a parser/normalizer test for `pMESSAGE_TS?thread_ts=THREAD_TS` as part of the same fix.
