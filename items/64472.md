## Summary

Keep this issue open. Current main still rejects bootstrap-file symlinks whose canonical targets leave the workspace root, and the provided GitHub context shows a same-author open PR, #64473, using closing syntax for this issue, so it should not be closed before that PR is reviewed, merged, or rejected.

## Best Possible Solution

Keep this issue open and review #64473 as the same-author implementation candidate, alongside the broader #40210/#40230 thread. If maintainers accept `~/.openclaw` sibling directories as trusted for workspace bootstrap reads, implement the relaxation in the narrowest appropriate layer with tests for a workspace `AGENTS.md` symlink to a sibling `~/.openclaw/kb/AGENTS.md` target and tests that arbitrary outside targets remain rejected. If that trust-boundary change is rejected, close or revise the PR with a clear security rationale and consolidate the user-facing problem under the broader workspace-bootstrap symlink issue.
