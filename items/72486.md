## Summary

Keep this issue open. Current main still does not add Vietnamese to the Control UI locale union, lazy locale registry, generated locale assets, language picker labels, public docs, or i18n tests. The prior ClawSweeper comment is still consistent with the checked-out main state.

## Best Possible Solution

Add Vietnamese as a first-class Control UI locale in core: extend the Locale type, lazy registry, vi/vi-VN browser fallback, English language label, generator LOCALE_ENTRIES, generated vi locale bundle and .i18n metadata, Control UI docs, and i18n registry/translation tests. This is a core Control UI localization task, not a ClawHub/plugin cleanup item.
