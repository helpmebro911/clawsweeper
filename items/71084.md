## Summary

Keep #71084 open. Current main still explicitly skips CLI-backed providers for both preflight compaction and normal memory flush, and the test/docs still codify that skip. The related implementation PR #71081 was closed unmerged in favor of this discussion, so the remaining work is maintainer product direction rather than cleanup.

## Best Possible Solution

Keep the issue open for maintainer decision. If CLI-backed agents should participate, land a focused PR that removes the `isCli` gates in `agent-runner-memory.ts`, updates the CLI-provider test from skip to allow, adds coverage for preflight compaction with a configured CLI backend, and updates the public memory/compaction docs. If the skip is intentional, answer the design question and document the rationale clearly.
