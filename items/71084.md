## Summary

Keep #71084 open. Current main still blocks CLI-backed agents from both preflight compaction and normal memory flush, and the colocated unit test explicitly expects CLI providers to skip memory flush. The related PR #71081 was closed unmerged in favor of this discussion, so no shipped main behavior has resolved the question.

## Best Possible Solution

Keep the issue open for maintainer product judgment. If maintainers decide CLI-backed long-lived agents should share the same memory management path, the likely implementation is to remove the `isCli` gates in `src/auto-reply/reply/agent-runner-memory.ts`, update the existing CLI-provider test to assert flush/rotation, and add or adjust preflight compaction coverage for CLI sessions.
