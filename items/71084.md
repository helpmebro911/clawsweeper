## Summary

Current main still explicitly prevents CLI-backed agents from running both preflight compaction and token-threshold memory flush, and the existing unit test asserts that CLI providers skip memory flush. The related PR #71081 was closed unmerged in favor of this discussion, so this issue remains the active place for maintainer product direction.

## Best Possible Solution

Keep the issue open for maintainer direction. If maintainers accept the reported use case, the core fix should remove the CLI-only gates in `runPreflightCompactionIfNeeded` and `runMemoryFlushIfNeeded`, preserve the existing `memoryFlushWritable` sandbox/read-only guard, and replace the current CLI-skip test with coverage proving CLI-backed providers can flush and compact under the same thresholds. If maintainers intentionally want CLI sessions excluded, document that limitation in memory/compaction docs and config help.
