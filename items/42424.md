## Summary

Keep this issue open. Current main still has the core unresolved failure mode: Discord bot identity lookup can fail to undefined, self-message filters only run when that id is known, and followup/replay data does not carry a bot identity for a second self-filter. Current main does appear to have a reset queue cleanup path for explicit session rollover, but that does not fix the unresolved identity and replay/followup gaps. The related fix PR #46338 is closed unmerged in the provided GitHub context.

## Best Possible Solution

Keep this issue open and implement a Discord-owned fix: add a reliable bot identity fallback or fail-closed startup behavior, carry trusted bot identity through finalized Discord inbound context, re-check self-authored Discord turns before pending-history/followup replay, and add regression tests for failed fetchUser("@me"), allowBots self-message ingress, steer-backlog followups, and the existing /new queue cleanup behavior.

## Risks / Open Questions

- Closing now would treat the known-bot-id debounce/preflight filters as complete even though the unresolved identity path still bypasses them.

- Replay/followup filtering needs trusted identity metadata, not just display labels, or it risks either missing self messages or dropping legitimate bot relay messages.
