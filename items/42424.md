## Summary

Keep open. Current `main` still fails open when Discord bot identity lookup returns no `botUserId`, does not carry `BotUserId` into replay/followup handling, and explicitly refreshes queued followups onto rotated sessions instead of clearing them across `/new`. The linked PR #46338 was closed unmerged, and its proposed replay-path changes are not present on `main`.

## Evidence

- **Ingress self-filter still depends on resolved bot identity:** `fetchDiscordBotIdentity()` returns `{ botUserId: undefined }` on `fetchUser("@me")` failure, and both Discord self-message filters only run when `botUserId` is present (`if (params.botUserId && ...)`). That matches the reported fail-open condition rather than fixing it.
