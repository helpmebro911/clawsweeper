## Summary

Current main still has the core Discord self-message failure mode: bot identity lookup can fail to an undefined id, both live self-message filters remain conditional on that id, and replay/followup history still lacks trusted Discord author/bot identity needed for a second self-filter. One related path is improved because /new now clears session queues, but that does not solve the failed-identity ingress or replay contamination problem. The related implementation PR #46338 is closed unmerged, so it does not supersede or implement this issue.

## Best Possible Solution

Keep this issue open and implement a Discord-owned fix: make bot identity resolution reliable or fail closed, carry trusted Discord author/bot identity through pending history and queued followup metadata, re-check self-authored Discord turns before replay/followup execution, and keep the existing /new queue cleanup covered by regression tests.
