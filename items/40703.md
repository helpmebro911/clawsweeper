## Summary

Keep this PR open, but do not land it unchanged. Current main still lacks a fail-fast startup auth guard before channel startup, while the PR is stale against the current post-attach startup split and uses an auth helper that is narrower than runtime model execution.

## Best Possible Solution

Keep this PR open for revision. Rebase or rewrite the guard into src/gateway/server-startup-post-attach.ts before channel startup, preserve OPENCLAW_SKIP_CHANNELS and OPENCLAW_SKIP_PROVIDERS behavior, and base the check on the same non-mutating auth contract runtime execution uses, such as hasAvailableAuthForProvider or a safe resolveApiKeyForProvider-style probe supplied with cfg and agentDir. Add focused tests for missing auth, config-backed apiKey auth, Bedrock/aws-sdk auth, synthetic local/plugin auth, skip-channel bypasses, and non-circular remediation text.
