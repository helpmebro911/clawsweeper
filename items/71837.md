## Summary

Keep #71837 open. Current main has related memory-core cleanup and phase-signal support, but the PR's central behavior is not implemented on main: REM candidate truths are not written back as grounded short-term candidates, phase reinforcement does not satisfy deep promotion signal/diversity gates, the package session extractor remains narrower than the PR, and REM/meta filtering is still partial. The PR discussion shows concrete follow-up review comments, not an obsolete or out-of-scope cleanup item.

## Best Possible Solution

Keep #71837 open. Rebase it onto current main, preserve the already-shipped sanitizer and phase-signal behavior, wire REM candidate truths into `recordGroundedShortTermCandidates` with stable query identity and confidence-aware signal weighting, extend the package/core session stripping consistently, and validate the targeted memory-core/session extraction tests plus the changed gate before maintainer merge consideration.
