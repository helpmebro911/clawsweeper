## Summary

Keep open. Current main has tracked browser-tab cleanup for subagent, cron, session reset/delete, and delayed primary-session idle/excess sweeps, but it still does not close temporary primary-agent browser tabs/windows at ordinary task completion. The tracked-tab model also lacks an ephemeral versus keep-open/user-owned marker needed to implement the requested default safely.

## Best Possible Solution

Keep this issue open for a focused browser lifecycle change. Add task-scoped ephemeral versus preserved/user-owned metadata to tracked browser tabs, expose an explicit keep-open/debug preservation path, wire best-effort close of ephemeral primary-agent tabs into ordinary successful task or turn completion, preserve labeled/pinned/user-owned/existing-session tabs, and keep the current idle/excess sweep plus subagent/cron/session reset cleanup as fallback. Add tests for primary-agent completion cleanup, keep-open preservation, and existing-session safety.

## Risks / Open Questions

- A naive close-on-completion implementation could break workflows that intentionally reuse primary-agent tabs across turns unless explicit keep-open or reuse semantics are modeled.
- Existing-session and attach-only profiles can point at a user's running browser; cleanup must avoid closing manually opened, pinned, or explicitly preserved user tabs.
