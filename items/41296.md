## Summary

Keep open. Current main has overlapping gateway shutdown hardening for WebSocket, HTTP, and MCP shutdown, but it does not implement the PR’s central per-subsystem warning collection or `ShutdownResult` observability. The PR also needs a rebase against newer shutdown code before it could be considered for merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase it onto current main, preserve the newer WebSocket/HTTP/MCP/lifecycle shutdown hardening, decide whether `ShutdownResult` should be part of the gateway close contract, and add focused tests for the remaining per-subsystem warning behavior.
