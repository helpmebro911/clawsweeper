## Summary

Keep open. Current main has manifest-backed command-alias diagnostics, but the reported lossless/lcm path is not proven fixed, the ownerless CLI fallback can still recommend adding a command token to `plugins.allow`, and maintainer discussion explicitly left this issue open with #64732 described only as adjacent.

## Best Possible Solution

Keep this issue open until the lossless-claw command path is verified end to end. The best fix is either for `lossless-claw` to publish `lcm` and lossless runtime command ownership in the documented `commandAliases` shape, or for core to normalize slash-prefixed command tokens and harden the ownerless fallback so it never tells users to add unresolved command names to `plugins.allow`. Coordinate broader fallback cleanup with #64732 only if maintainers make that issue the canonical tracker.
