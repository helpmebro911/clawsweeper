## Summary

Keep open. Current main still does not implement PR #48375's custom Tailscale control-server feature: the gateway config schema/types, CLI options, runtime merge/validation path, and Tailscale docs only cover the existing `mode` and `resetOnExit` behavior. The linked issue #48367 was closed as superseded by this PR, making #48375 the canonical tracker rather than obsolete. The PR discussion also contains substantive late review feedback about behavior and security, so conservative cleanup should not close it.

## Best Possible Solution

Keep #48375 open as the canonical implementation PR. The best path is for the author or maintainer to rebase it onto current main, resolve the remaining review comments, ensure `gateway.tailscale.controlUrl` is either enforced against the daemon in the supported tailnet mode or reframed as documentation-only, add schema/UI hints and focused validation/runtime/CLI tests, and update the public Tailscale docs if the feature is accepted.
