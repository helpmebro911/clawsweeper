## Summary

Keep open. The report targets the bundled Feishu/Lark plugin, and current main only shows transient streaming-card tool status; it does not clearly implement the reported persistent tool-execution steps with params/results/duration.

## Best Possible Solution

Keep the issue open for maintainer/plugin-owner handling. The likely path is to retest on current OpenClaw plus current `@openclaw/feishu`, then either document that only transient tool status is supported or implement persistent Feishu card tool-step tracking in `extensions/feishu/src/reply-dispatcher.ts` using existing tool/item/after-tool event data, with focused tests covering tool name, params/result summary, duration, and final card behavior.
