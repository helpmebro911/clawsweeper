## Summary

Keep open. Current main has partial Feishu/Lark tool-progress plumbing, but the reported detailed "tool execution steps" area is not clearly implemented or shipped: Feishu streaming cards only surface a transient `Using: <tool>` status line and intentionally omit it from the final card close, while the issue asks for displayed tool name, params, result, and duration.

## Best Possible Solution

Keep the issue open for Feishu/Lark owner triage. The likely path is to reproduce on current main or the next compatible Feishu plugin/host pair, then either document that only transient tool-progress status is supported or implement a persistent card tool-step section using structured item/tool events or before/after tool hooks, with regression coverage for tool name, params summary, result/error, duration, and session-key matching.
