## Summary

Keep this issue open. Current main still has overlapping pieces, but not the requested intentional shell-facing `openclaw notify <message>` command or a generic connected-Control-UI notification broadcast. #44590 added PWA/Web Push subscription/test plumbing, and `chat.inject` can update one chat transcript, but neither is the requested admin-scoped all-UI notification surface.

## Best Possible Solution

Keep this issue open as a focused core CLI/Gateway/Control UI feature request. The likely path is an intentional `openclaw notify <message>` or `openclaw gateway notify` command backed by a clearly named admin-scoped Gateway RPC and typed broadcast event that connected Control UI clients render as an in-app notification. Reuse Web Push only as an optional fan-out for subscribed browsers, and add docs/tests for auth scope, rate limits, failure behavior, and how this differs from `chat.inject`, `system event`, and `push.web.test`.
