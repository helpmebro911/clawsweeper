## Summary

Keep open. Current main still lacks the central Gemini override behavior targeted by this PR: Gemini web search is hard-coded to the native Google endpoint, the Google plugin manifest only exposes webSearch apiKey/model, and embeddings/image/media paths do not implement GEMINI_BASE_URL, GOOGLE_GEMINI_ENDPOINT, GOOGLE_GEMINI_BASE_URL, GEMINI_API_TYPE, or OpenAI-compatible request routing. The PR is not merge-ready as-is because the provided review context shows unrelated changes and blocking review findings, but that calls for narrowing or superseding the PR rather than closing it.

## Best Possible Solution

Keep the PR open but require a narrowed replacement or substantial cleanup: limit the change to the Google plugin override behavior, remove unrelated pnpm/memory/channel/Anthropic/Volc/root-file changes, add manifest/docs for any public config surface, implement provider-specific OpenAI-compatible request/response handling, preserve native googleapis.com behavior including /openai normalization, and add focused unit tests for env/config precedence and official-endpoint exclusions.
