## Summary

Keep this PR open. Current main still does not expose `api.runtime.llm.complete()` or inject an LLM helper into context-engine runtime contexts, and the linked feature request remains unresolved. The PR is a plausible implementation candidate, but it adds a new model-inference capability for plugins/context engines and still needs maintainer API/security review before merge.

## Best Possible Solution

Keep this PR open for maintainer review or replacement. The best path is an additive `api.runtime.llm.complete()` seam with clear active-agent or explicit-agent binding, documented permission/cost semantics, SDK docs and baseline updates, and focused tests for model/auth resolution. Keep #45865 as the canonical product/API discussion if the API shape changes.

## Risks / Open Questions

- The PR exposes a new plugin/context-engine path for spending model tokens and using configured model credentials, so maintainers need an explicit permission, observability, and trust-boundary decision.
- The PR diff defaults unbound LLM completions to the config default agent when `agentId` is omitted; context-engine calls from non-default sessions could use the wrong model or auth profile unless the API binds the active session agent or requires an explicit agent.
- The proposed public SDK surface still needs docs, API baseline/export coverage, and focused tests for default/non-default agent model and auth-profile resolution.
