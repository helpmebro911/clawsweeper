## Summary

Keep #12931 open. Current main still runs a model-backed Session Startup greeting for bare `/new` and `/reset`, and the session config surface still lacks a `resetGreeting`, `resetGreetingModel`, silent reset, or static acknowledgement option. The existing Codex review comment in the issue remains accurate against current main `4a3030df9efa72c44bab567a4390ac0c4876a73f`.

## Best Possible Solution

Keep this open for a core session/reset behavior change. The best implementation would add typed and schema-validated session config for bare reset greeting behavior, support both a configured cheaper greeting model and a no-model silent/static acknowledgement path, wire it through `src/auto-reply/reply/get-reply-run.ts` and `src/gateway/server-methods/agent.ts`, update session/slash-command/config docs, and add regression tests for bare `/new`, bare `/reset`, `/new <model>`, startup-context/bootstrap interactions, and existing model override persistence.
