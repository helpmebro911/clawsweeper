## Summary

Keep #12931 open. The prior keep-open review remains accurate on current main: bare `/new` and `/reset` still become a model-facing Session Startup greeting prompt, both auto-reply and gateway agent paths still dispatch that prompt into an agent run, and the typed/session schema surface still has no `resetGreeting`, `resetGreetingModel`, silent reset, or static acknowledgement option.

## Best Possible Solution

Keep the issue open for a core session/reset behavior change. The best implementation would add typed and schema-validated session config for reset greeting behavior, support both a configured cheaper greeting model and a no-model silent/static acknowledgement path, wire it through the auto-reply and gateway agent bare reset paths, preserve existing `/new <model>` semantics deliberately, and update session/slash-command/config docs plus regression tests for bare `/new`, bare `/reset`, `/new <model>`, bootstrap/startup-context interactions, and model override persistence.
