## Summary

Keep #12931 open. Current main still routes bare `/new` and `/reset` through a generated Session Startup greeting prompt and then into the normal agent/model run. The existing model override path only handles trailing text such as `/new <model>`, and the public session type/schema/docs still have no `resetGreeting`, `resetGreetingModel`, silent reset, or gateway-ack control. The prior steipete Codex review comment reached the same result, and current `main` still matches it.

## Best Possible Solution

Keep this open for a core session/reset behavior change. A complete fix should add typed and schema-validated session config for bare reset greeting behavior, document it, and cover both requested modes: a configured cheaper model for the greeting turn and a no-model path that either sends no message or a gateway-generated acknowledgement. The likely implementation spans the bare reset prompt construction, `runPreparedReply`, the gateway `agent` bare reset handler, and reset model override handling, with tests for bare `/new`, bare `/reset`, `/new <model>`, bootstrap/startup-context behavior, and persistence of existing model overrides.
