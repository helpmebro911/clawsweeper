## Summary

Keep open. Current main still guards Feishu `MessageThreadId` behind topic-session scopes, while this PR asks for unconditional `root_id` forwarding. The PR patch is also not mergeable as written because it adds duplicate inbound-context keys and leaves the existing guarded `MessageThreadId` assignment in place.

## Best Possible Solution

Keep this PR open for contributor or maintainer revision: update the existing `ReplyToId` and `MessageThreadId` assignments instead of adding duplicate keys, add focused Feishu tests for non-topic and topic-session `root_id` behavior, and reconcile the intended metadata policy with #60848 so Feishu has one coherent threading fix.
