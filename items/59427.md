## Summary

Keep open. Current main has opt-in mitigations for context size, but the issue is a coherent, recent feature request about baseline per-message prompt load and lazy/lightweight behavior. I did not find high-confidence evidence that main implements the requested lazy-load behavior broadly enough to close it.

## Evidence

- **Issue is recent and actionable:** Issue #59427 was opened 2026-04-02, has no protected labels, no maintainer author association, and describes observed token counts plus a concrete proposal for lazy context loading and lightweight/full mode.
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Current config supports only an opt-in bootstrap reinjection mitigation:** The configuration reference documents agents.defaults.contextInjection with default "always" and an opt-in "continuation-skip" mode that skips workspace bootstrap re-injection on safe continuation turns. That helps, but it does not prove lazy loading of all context/tool/memory surfaces by reply length.
  - file: [docs/gateway/configuration-reference.md](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/gateway/configuration-reference.md)
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Schema/type surface confirms contextInjection is configurable, not default lazy mode:** The agent defaults type/schema expose contextInjection as "always" or "continuation-skip" and describe "always" as the default for workspace bootstrap files. This is narrower than the issue's proposed general lazy-load context behavior.
  - file: [src/config/types.agent-defaults.ts](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/src/config/types.agent-defaults.ts)
  - sha: [afd6b5d6fce1](https://github.com/openclaw/openclaw/commit/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386)
- **Other token controls exist but do not close the requested behavior:** Current docs also describe contextPruning for old tool results and model contextTokens caps, and code/docs mention experimental localModelLean for dropping some heavyweight tools. These are useful mitigations, not a complete implementation of lazy-loading context only when needed.
  - file: [docs/gateway/configuration-reference.md](https://github.com/openclaw/openclaw/blob/afd6b5d6fce1dac98539d8b45bd0f9db9ff25386/docs/gateway/configuration-reference.md)
