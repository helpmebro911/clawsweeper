## Summary

Keep open. The same-author PR #53315 is still the implementation candidate, and current main still treats an undiscoverable configured channel id as a fatal validation issue that can block gateway startup.

## Best Possible Solution

Keep this issue open while maintainers review PR #53315 or a replacement patch. The likely fix belongs around `src/config/validation.ts` and gateway startup handling: configured but undiscoverable plugin-owned channels should warn and be skipped while other channels still start, with regression coverage and docs updated if the validation contract changes.
