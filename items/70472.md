## Summary

Keep open. Current main still rotates the active session mapping after embedded context overflow or thrown compaction-failure handling, and PR #70479 is an open, unmerged implementation candidate with closing syntax for this issue.

## Best Possible Solution

Keep this issue open and review or land PR #70479, or an equivalent fix. The fix should preserve the active agent:main:main mapping when overflow recovery or compaction fails, avoid silent session rotation, provide explicit retry/manual session-control guidance, and keep regression coverage for embedded overflow payloads, thrown compaction failures, and the next WebChat message continuing under the same sessionId.
