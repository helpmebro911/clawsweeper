## Summary

Keep open. Current main still contains the compaction/overflow failure handling path that rotates the active session mapping, and PR #70479 is an open implementation candidate with closing syntax for this issue rather than merged behavior.

## Best Possible Solution

Keep this issue open and review/land #70479 or an equivalent fix. The fix should preserve the active agent:main:main session mapping when overflow recovery or compaction fails, return explicit retry or manual session-control guidance instead of silently rotating the session, and keep regression coverage for embedded overflow payloads, thrown compaction failures, and the next WebChat message continuing under the same sessionId.
