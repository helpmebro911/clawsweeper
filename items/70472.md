## Summary

Keep #70472 open. Current main still contains the reported compaction-checkpoint sentinel and still auto-rotates the active session after context-overflow/compaction-failure paths, so the observable WebChat session-loss bug is not implemented/fixed on main. Related PR #70479 is the right canonical fix candidate, but it remains an open/unmerged PR in the provided context.

## Best Possible Solution

Keep the issue open and handle it in core, likely by reviewing/landing the #70479 direction: stop auto-resetting the active session on compaction/overflow failure, preserve the existing `agent:main:main` mapping, surface explicit/auditable recovery guidance, and add regression tests for embedded overflow payloads and thrown compaction failures in the WebChat direct path.
