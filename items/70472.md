## Summary

Keep open. The reported bug is still present on current main: compaction/overflow failures still route through a compaction-failure reset path that generates a fresh session id and writes it back to the active session store. The linked PR #70479 is the right active fix track, but it remains open and unmerged, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep #70472 open and land/review PR #70479 or an equivalent patch that removes compaction-failure session resets, preserves the existing session mapping, surfaces explicit retry/compact/new guidance, and keeps the added regression tests for embedded overflow payloads and thrown compaction-failure errors.
