## Summary

Current `main` still does not provide a documented or implemented canonical custom-client audio ingress path for Gateway `chat.send` or `POST /v1/responses`. The issue is recent, coherent, and the code/docs support the reported gap rather than an auto-close reason.

## Evidence

- **OpenResponses docs list supported inputs but no audio input:** The public `/v1/responses` docs enumerate supported request items, then document only `input_image` and `input_file`; there is no documented `input_audio` path.
  - file: [docs/gateway/openresponses-http-api.md:67](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/docs/gateway/openresponses-http-api.md#L67)
  - command: `nl -ba docs/gateway/openresponses-http-api.md | sed -n '67,160p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **OpenResponses schema excludes input_audio:** `ContentPartSchema` only unions `input_text`, `output_text`, `input_image`, and `input_file`, so the public `/v1/responses` validator does not accept `input_audio`.
  - file: [src/gateway/open-responses.schema.ts:78](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/gateway/open-responses.schema.ts#L78)

  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **OpenResponses handler only extracts image and file parts:** The `/v1/responses` handler iterates input parts and has explicit branches for `input_image` and `input_file` only before building the prompt.
  - file: [src/gateway/openresponses-http.ts:503](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/gateway/openresponses-http.ts#L503)

- **Gateway chat attachment parser drops non-image attachments:** `parseMessageWithAttachments` warns and drops attachments when the sniffed or provided MIME is not an image, so audio attachments sent through `chat.send` do not become a supported ingress path.
  - file: [src/gateway/chat-attachments.ts:347](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/gateway/chat-attachments.ts#L347)
