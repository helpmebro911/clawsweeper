## Summary

Keep open. Current main does not implement the requested Synology Chat peer-prefix fix: outbound replies still pass the raw `to` value into `sendMessage`, and `parseNumericUserId` still calls `Number.parseInt(userId, 10)` without stripping `synology-chat:`. The related PR #50109 directly targeted this issue, but the provided GitHub context shows it was closed unmerged, and the current source matches the unfixed behavior.

## Evidence

- **Raw outbound target is still passed to sendMessage:** Synology Chat outbound `sendText` resolves the account and calls `sendMessage(incomingUrl, text, to, ...)` with the raw `to` value. There is no prefix normalization in this send path before `sendMessage`.
  - file: [extensions/synology-chat/src/channel.ts:338](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/extensions/synology-chat/src/channel.ts#L338)
  - command: `nl -ba extensions/synology-chat/src/channel.ts | sed -n '330,352p'`
  - sha: [a1090b604312](https://github.com/openclaw/openclaw/commit/a1090b604312ac7a49cf2d2ce7276b334786bb79)
- **Numeric parser still does not strip channel prefix:** `buildWebhookBody` relies on `parseNumericUserId`; the parser currently does `Number.parseInt(userId, 10)` directly for strings and returns `undefined` only for `NaN`, so `synology-chat:4` still fails to populate `payload.user_ids`.
  - file: [extensions/synology-chat/src/client.ts:272](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/extensions/synology-chat/src/client.ts#L272)

- **Inbound context still uses prefixed peer identifiers:** Synology Chat inbound context sets both `From` and `To` to `synology-chat:${msg.from}`, matching the reporter's prefixed peer-ID scenario.
  - file: [extensions/synology-chat/src/inbound-context.ts:27](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/extensions/synology-chat/src/inbound-context.ts#L27)

  - file: [extensions/synology-chat/src/channel.ts:235](https://github.com/openclaw/openclaw/blob/a1090b604312ac7a49cf2d2ce7276b334786bb79/extensions/synology-chat/src/channel.ts#L235)
