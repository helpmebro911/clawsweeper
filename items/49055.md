## Summary

Keep #49055 open. Current main does not implement the requested delivery post-condition, status propagation, or retry for recovered agent runs. The central failure remains plausible in the main agent delivery path: after a successful run/fallback, `agentCommandFromIngress` returns `deliverAgentCommandResult()` directly, and that helper still performs a single delivery call with no structured `deliveryStatus` return and no retry when delivery is skipped or returns zero results. Related PR #53961 is still open/draft in the provided context and appears to add observability/status as a prerequisite, not the full retry behavior proposed in the issue comments.

## Evidence

- **Main agent delivery helper has no delivery status or retry:** `deliverAgentCommandResult()` resolves channel/target and eventually calls `deliverOutboundPayloads()` once when `deliver` is true and a target exists, then returns only `{ payloads, meta }`. There is no structured `deliveryStatus` return, post-condition check, or retry path in this helper.
