## Summary

Keep open. The issue is recent, concrete, and reports a specific macOS WebChat interaction against v2026.4.22. Current main still hosts the macOS companion chat through the native SwiftUI WebChat view, using a SwiftUI ScrollView/LazyVStack with scrollPosition-driven auto-scroll, and I found no source, test, docs, changelog, or post-v2026.4.24 diff evidence that scrollbar-thumb dragging was fixed or covered. The related #42527 cleanup covered an older/general scrolling crash after the WebChat rewrite; this report is newer and targets the rewritten surface itself.

## Best Possible Solution

Keep the issue open for a macOS maintainer or app contributor to reproduce on current main/latest app, capture a hang sample if it still occurs, and harden or replace the SwiftUI message scroll container. A good fix would guard scrollPosition-driven auto-scroll during explicit user scrollbar dragging or move the message list to a more controllable AppKit-backed scroll view, then add focused macOS regression coverage or documented manual verification with long chat history and scrollbar-thumb drag.
