## Summary

Keep open. The report is recent and actionable, and current main still routes the macOS companion chat window through the same SwiftUI OpenClawChatView scroll stack. I found smoke coverage for opening/closing the macOS WebChat window/panel, but no implementation or test evidence that scrollbar-thumb dragging has been fixed.

## Best Possible Solution

Keep the issue open and investigate the macOS SwiftUI chat scroll path. A likely maintainer direction is to reproduce with a large chat history on macOS, identify whether the scrollPosition bottom-pinning logic fights scrollbar-thumb dragging, then either gate auto-scroll while the user is actively scrolling/dragging or replace the message list with a macOS-specific NSScrollView-backed implementation. Add a focused macOS regression test where feasible, plus manual verification notes for scrollbar-thumb dragging.
