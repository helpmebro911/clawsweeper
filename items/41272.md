## Summary

Current main still has the reported Control UI behavior: `timeoutSeconds: "0"` is rejected by validation, and even if validation changed, the payload builder currently only includes positive timeout values. The backend/protocol already supports `timeoutSeconds: 0`, so the remaining gap is specifically the UI. Related PR #41275 tracks the same fix but is open/unmerged, and PR #46003 was closed/unmerged, so this issue should stay open unless maintainers intentionally close it as PR-tracked duplicate.
