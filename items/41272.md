## Summary

Keep this issue open. Current main still rejects `timeoutSeconds: 0` in the Control UI validation path and also drops zero from the payload builder, even though the gateway cron contract and server tests already allow zero. The issue is also paired with open same-author PR #41275 using closing syntax, so the right cleanup path is to review/land or close that PR rather than close this issue first.

## Best Possible Solution

Keep this issue open and use #41275 as the implementation candidate, but require the final patch to align the full Control UI path with the existing gateway contract: allow zero in `validateCronForm`, preserve explicit `timeoutSeconds: 0` in `buildCronPayload`, update the UI validation/error tests, and adjust the user-facing timeout help/error wording as needed.
