## Summary

Keep open. Current `main` still exposes the full `sessions_spawn` schema, including `resumeSessionId`, `streamTo`, `lightContext`, `attachments`, and `attachAs`, and there is no gateway-specific stripping path in the live tool assembly. The existing provider-compat layer only normalizes schema shape and strips unsupported keywords, so there is not enough evidence that the reported strict-provider compatibility problem has been fixed elsewhere.
