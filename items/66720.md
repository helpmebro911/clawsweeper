## Summary

Keep this PR open. Current main still does not implement the requested strict-gateway schema simplification for `sessions_spawn`: ACP-only fields are hidden when ACP is unavailable, but the schema still advertises subagent-specific `lightContext`, `attachments`, and `attachAs`, and the existing provider schema compatibility code strips schema keywords rather than named tool properties. The PR remains a plausible implementation candidate for the closed companion report #66719, but it needs maintainer review and likely rework before merge.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best implementation path is a provider/deployment-aware `sessions_spawn` schema property filter wired into the actual OpenAI-compatible gateway/model compatibility path, while preserving existing ACP availability hiding and adding schema-shape tests for the default, ACP-unavailable, and strict-gateway cases.
