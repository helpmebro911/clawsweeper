## Summary

Keep this PR open. Current main only hides ACP-only sessions_spawn schema fields when ACP is unavailable; it does not implement the PR's stricter gateway/provider schema simplification for lightContext, attachments, or attachAs, and #66719 is closed as superseded by this PR.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best path is a provider/deployment-aware sessions_spawn schema filter, wired from the actual OpenAI-compatible gateway/model compatibility path that serves affected providers, while preserving current ACP availability hiding and adding tests for default, ACP-unavailable, and strict-gateway schema shapes.

## Risks / Open Questions

- Merging this PR as-is may still leave affected OpenAI-compatible gateway deployments unfixed unless a real provider/deployment compatibility path activates the stripped schema.
- The final design needs to preserve current ACP-availability hiding while adding a separate strict-gateway simplification path, otherwise it could hide useful subagent affordances too broadly.
- Schema simplification is behavior-visible to models, so it should have regression coverage for both default full schema behavior and the activated stripped schema shape.
