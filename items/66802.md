## Summary

Keep open. Current `main` still does not expose a `memoryFlush.model` setting and the memory-flush runtime still uses the session run's primary provider/model. The linked PR #66846 in the issue timeline suggests active in-flight work, so this is neither implemented on `main` nor a stale/incoherent report.
