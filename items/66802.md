## Summary

Keep this issue open. Current main still does not implement a nested memoryFlush model override: config types and schemas omit it, docs do not advertise it, the memory flush plan carries no model field, and the runtime still runs the flush through the session run's provider/model. PR #66846 is an open implementation candidate, but it has not merged.

## Best Possible Solution

Keep this issue open until PR #66846 is reviewed and merged, or an equivalent implementation lands on main. The fix should add agents.defaults.compaction.memoryFlush.model to config types, zod/generated schema, help labels, and docs; carry the override through the memory flush plan or runtime boundary; resolve provider/model/auth behavior consistently with agents.defaults.compaction.model; and add regression coverage for override and fallback behavior.
