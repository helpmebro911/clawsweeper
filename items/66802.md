## Summary

Keep open. Current main at 646a268d2710bc54e478349ef5690f3c8803df4f still does not implement `agents.defaults.compaction.memoryFlush.model`: the nested config type/schema/generated schema/docs omit the field, the memory flush plan cannot carry it, and the runtime still starts the flush turn from the session run provider/model. PR #66846 is an open, unmerged implementation candidate for this same request.

## Best Possible Solution

Keep this issue open until PR #66846 is reviewed and merged, or an equivalent implementation lands on main: add `agents.defaults.compaction.memoryFlush.model` to config types, zod/generated schema, labels/help/docs; carry it through the memory flush plan or runtime boundary; resolve provider/model/auth/fallback behavior consistently with `agents.defaults.compaction.model`; and add regression coverage for override and fallback behavior.
