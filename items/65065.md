## Summary

Keep #65065 open. Current main still has the exact CLI wiring gap the PR addresses: Discord advertises and handles channel create/edit/delete/move actions, but `openclaw message channel` only registers `info` and `list`, and current message CLI docs only document those two channel subcommands. The PR remains a focused, plausible fix rather than an obsolete or duplicate item.

## Best Possible Solution

Keep the PR open and have a maintainer rebase or apply the equivalent patch on current main: add `message channel create|edit|delete|move` in `src/cli/program/message/register.discord-admin.ts`, keep the options aligned with `extensions/discord/src/actions/runtime.shared.ts`, add focused CLI registration tests, and update the current message CLI docs page. This is core CLI wiring for an existing bundled Discord action surface, not a ClawHub/plugin-close case.
