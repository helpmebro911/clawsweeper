## Summary

Keep #12512 open. Current main still exposes skills through a model-facing available-skills prompt and instructs the model to read and follow selected SKILL.md files. Existing controls cover visibility, allowlists, install metadata, and plugin prompt hooks, but they do not implement instruction isolation, role locking, memory/tool capability gates, or behavioral monitoring for SKILL.md content.

## Best Possible Solution

Keep #12512 open as core security hardening. Define and document a SKILL.md instruction-isolation and precedence model, add enforceable per-skill capability metadata for role/memory/tool influence, enforce it through prompt assembly, tool policy, memory, and sandbox behavior, and add malicious-skill regression tests. ClawHub scanners or plugins can supplement this but should not replace the core boundary.
