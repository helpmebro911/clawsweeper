## Summary

Keep open. Current main still renders Control UI tool output URLs as escaped/plain text in expanded, collapsed, and inline tool-card paths, and native WebChat still displays tool results through plain SwiftUI Text. The directly related PR #8880 attempted the feature but was closed unmerged, so it does not supersede the issue.

## Best Possible Solution

Keep tracking #8812. The best implementation is a safe URL rendering path for Control UI tool output blocks, collapsed previews, and inline cards, with tests that assert anchors and XSS escaping. If native WebChat is in scope, add equivalent URL-aware rendering using SwiftUI-supported attributed/link handling rather than plain Text. PR #8880 can be used as prior art, but the current main code still needs an updated patch against the present tool-card architecture.
