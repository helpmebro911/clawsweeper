## Summary

Keep #8812 open. Current main still renders URL-bearing tool output as plain text in Control UI tool-card paths and native WebChat tool-result cards. Markdown linkification exists for normal chat text, but tool cards bypass that renderer, and the linked PR #8880 was closed unmerged.

## Best Possible Solution

Keep #8812 open. Implement a safe URL-aware renderer for Control UI tool output across expanded output, raw details, collapsed previews, inline short cards, and sidebar/raw views, with focused tests for anchors, escaping, unsafe schemes, punctuation, and whitespace. If native WebChat remains in scope, add equivalent SwiftUI URL rendering with AttributedString or Link handling rather than plain Text. PR #8880 is useful prior art but should not be treated as shipped because it was closed unmerged.
