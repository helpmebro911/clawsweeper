## Summary

Current main still lacks a configurable URL or hostname deny list for web_fetch. The config types, strict runtime schema, generated schema, runtime web_fetch wiring, docs, and shared SSRF policy expose existing fetch settings plus allowlist/private-network behavior, but not the requested exact/wildcard public-host deny policy.

## Best Possible Solution

Keep this issue open and implement a typed, documented web_fetch host deny list if maintainers want the behavior. A conservative implementation should add and validate tools.web.fetch.denyHosts, support case-insensitive exact and *.domain matching, enforce it before DNS and across redirects/provider fallback, return concise blocked-fetch errors, update generated schema/docs/help labels, and add focused config plus runtime tests.
